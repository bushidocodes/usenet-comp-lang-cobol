: |from platform libraries is like separating COBOL from ISAM. They
: |simply have to go together.
: ======
: I find this fascinating, because I was on the team that built the first
: COBOL compiler for the PDP-10 and therefore paid a lot of attention to
: the development of COBOL implementations in the late 60s and early 70s
: for major vendors.  I remember all too well the vast amount of code
: written in COBOL before IBM or DEC either introduced ISAM as part of
: their COBOL implementations.

: There are basically two cases
: 1) ISAM is standardized and portable.  This case corresponds to using
: standard libraries in C.  It would be perfectly possible to make
: comments about COBOL or C in such a case.
: 2) ISAM is not standardized and not portable.  This case corresponds to
: using platform-specific libraries in C.  It is impossible in this case
: to make widely appropriate comments about either COBOL or C.
