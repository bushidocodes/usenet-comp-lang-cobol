I will try to give you some help as i work with NetExpress for writing
CGI applications...
Well, did you notice the "Net" in the product name? The product is
really aimed at helping Cobol programmer get into Web progamming and is
constantly extended/improved in that direction (while keeping the "old"
features). Features it includes are:

'Form Designer': Its an integrated tool helping to create your HTML
forms. You can draw your input fields, combo boxes..., integrate
activeX, javascripts ... You associate ID and Names to those items and
the Cobol variables will be declared in copy files in the Cobol source
code.
The form will be declared something like that:
01 HTMLForm is external-form.
  02 HTMLControls.
.....

Those forms (you open with Form Designer) have a new format ".mff". The
designer also generates the ".HTML" files. All those files will be
integrated in your project.

There is also a great wizard for generating CGIs on SQL databases. It
generates the Inserts, Update, Delete with state management... The
skeleton it uses can be modified if needed.
(There are other wizards (GUI...) but as i dont use those....)

You have embedded HTML and embedded SQL:
EXEC HTML/SQL
...
END-EXEC

An integrated web server 'Solo' you can use while developing and which
enables animating CGIs.

There is a library called 'ACCCGI' which provides an extension to the
normal cobol ACCEPT which does the data decoding returned from the web
server. You will just have to DISPLAY/ACCEPT using the fields generated
by the Form Designer and the rest is transparent. I assume this would
replace your c-shell scripts.

There is a tool for publishing applications (including CGIs) to UNIX.

I also think that CGIs have to be in '.EXE' or '.DLL', no '.OBJ'.

There is an SQL assistant.

You can program object oriented (dont know if you have that yet...) and
there is class library browser.

It includes also 'Dialog System' (same remark..)

Hope this helps to give you an overwiew as this isnt all there is but
thats what i use.
If someone involved in MicroFocus (Merant?) reads this and isnt happy
with my overview i appologize:)

I will check back to see if you need some more,
Alain.
