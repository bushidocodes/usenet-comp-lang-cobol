I'm using Micro Focus Cobol workbench V3.2.20 and Visual Basic 4.0 with
Btrieve database for my application development. Currently I'm facing some
problem with the Decimal Packed field (Comp-3) in Micro Focus Cobol.

Here are the description of the problem :-

Btrieve File Layout :
KEY-FIELD PIC X(06).
DECIMAL-FIELD PIC S9(04) COMP-3.
STRING-FIELD PIC X(31).

1. I'm using Visual Basic 4.0 for my client application development running
on Windows 95 and Windows 3.1. All online data entry are done through VB4
application. Unfortunately, some of the data inserted and updated to the
database are unsigned value.

2. If the DECIMAL-FIELD value in the record contains a unsigned value,
example 1234 (hex'01234F'), when my COBOL program read the record and move
it to a field with a SIGN picture, the program will terminate with a
RTS0163 Run Time Error, Illegal character in numeric field (Error 163).

3. If I removed the SIGN picture in DECIMAL-FIELD, it work fine, but it
does not solve my problem, because I need the SIGN picture for others
arithmetic calculation.

4. I have tried putting the BADSIGNS, SIGNCOMPARE, MS"2" and a few others
directives for compiler, the problem still exist.

Any hints about how to overcome the above problem ?

The 163 error is often the result of having leading zeros set to
spaces when reading .TXT files or as a result of having a field
initialised incorrectly.

A quick and dirty way to solve the problem is to set the 'F'
runtime switch - often there is no other viable solution. This
forces the RTS to ignore such errors and pass the data.
