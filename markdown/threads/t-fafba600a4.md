```
I found this helpful:
A Quick Start Guide to Using Fujitsu 4.0 Cobol

1.	Create project folder on floppy or hard drive
   a.	Open Windows Explorer
   b.	Select file->New->folder
   c.	Give folder name such as "Project1"
   d.	Close Windows Explorer
2.	Create/edit Cobol Source Code
   a.	Start->Programs-> Fujitsu Cobol 4.0->Cobol Project Manager
   b.	File->Project Open
   c.	Navigate to your project folder
   d.	Type in "Proj1" or other useful name
   e.	Click "Yes" to create the project file
   f.	Right click Proj1.prj
  g.	Select new file
  h.	Rename "New File.exe" to "PROJ1.EXE"
  i.	Right click on PROJ1.EXE
  j.	Select New Folder->Cobol Source file
  k.	Right click "Cobol Source File"
  l.	Select "New File"
  m.	Rename "New File.cob" to "Proj1.cob"
  n.	Right click PROJ1.cob
  o.	Select "Main Program" (Icon color turns red)
  p.	Double click PROJ1.COB tp begin entering source code
3.	Compile Your Code
  a.	Left click topmost folder to highlight your project
  b.	Click "Build" icon or select Project->Build from menu bar
  c.	If you get syntax errors, double-click on red message to launch editor
and fix error
  d.	Click "Build Start" icon after returning to message window or select
Build->start
4.	Run Your Program
   a.	Click "execute" icon or select Project->execute from menu bar
```

```
Thanks for the explanation.

This brings me to another of my favorite peeves, i.e., the
complete inadequacy of operating systems and compilers to
provide a decent directory.

Trying to manage hundreds of files of all sorts is not
made easier when all you get in the directory is the
name, size and date.

The invention of the 256-character filename as a means
to improve the situation is another example of Microsoft's
remarkable ability to do things the wrong way.

A decent file-management/directory system would provide
for several items of information about each file at the
time it was created. For example, the author, name of
file, file-extension, category/project to which it belongs,
date created, date last-modified, mode, and so on.

The directory should be listable in several different
sequences with the ability to exclude certain types
from the listing, and so on.

Anyway, I wrote a program to keep track of my stuff years ago
and I still use it. The eight-character plus extension
file name is perfectly adequate if you have a file-naming
convention - and stick to it like glue.

The naming convention I have used for aeons breaks
down each of the eight characters in the name like
this:

TPPNNSSS where;

        T is the Type of file, where:
             'P' is for Program,
             'F' is for Data Files,
             'R' is for a Record Description,
             'S' is for a Subroutine,
             'D' is for Documentation,
			  and so on.

        PP is the Category or Project the file belongs to.

        NN is a number identifying the file within the Project,

        SSS is a suffix to identify the version of the file,

The DOS extension identifies the nature of the file,
e.g., EXE, COB, CBL, CPY, etc.

Between my program for supplementing the directory and batch
files to compile complete systems or sub-systems, I guess
I have the equivalent of your project file.
```

```
The OS/2 Warp operating system from IBM has a set of data
that are associated with a file called "extended attributes".
These can be used to carry additional information about
the file and can be retrieved by application programs
and OS utilities to display and/or sort files according
to the attributes. One major use in the WPS (that's the
OS/2 GUI) is to identify the program(s) that can act on
the file and what the default behavior for file open is.

You can idenify a file as "COBOL Source" and define
the default "open" to be in your COBOL editor program
(which can be defined as any executable). The file extension
has no bearing on how the file is treated (unlike Windows or DOS).

Each file can have up to a total 64 Kbytes of "extended attribute"
information. Each "extended attribute" has a name and a
value for that name. Creating extended attributes like
"Program Type" that has values of "subprogram" "main program"
etc can is verry easy and the values can be used to filter
for desktop folders or "batch" file operations. An attribute
called "Project Name" can be used to group files into
projects.

The "extended attributes" for a file are bound to a file
and are carried with it for backups, file copy operations etc.

The "extended attributes" coupled with the 256 character
path/filename available with OS/2 Warp makes file management
very easy.

This is just to let you all know that some operating
systems do provide the capabilities to provide a
decent directory and project control environment.
Mind you, the only one I know of is OS/2 Warp.
```

```
COBOL32 -iS:\A-SOURCE\SUBS\DEFAULT.CBI %1.COB
IF EXIST NOASK.TMP GOTO LIBADD

CHOICE /C:YN Add to library?
IF ERRORLEVEL 2 GOTO END

:LIBADD

\FSC\COBOL\LIB /REMOVE:%1.OBJ /VERBOSE SUBS.LIB

IF EXIST %1.OBJ DEL %1.OBJ
```

```
This works for Fujitsu COBOL 4.0, but, it should work the same...

You can select compile in the editor...

To create an exe:
1. Launch the COBOL Project Manager
2. Create a new folder for the project and source
3. Open a new project by giving it a name and selecting the folder
4. In Project Manager select/highlight the project
5. Select menu Edit... new file
6. An empty executable "New File..EXE" is created - you can change the name
7. Select it and select menu Edit... new folder... COBOL source File
8. Select the new Cobol folder - Select menu Edit... New File
9. You can rename the source file
10. Double click the COB file to edit it: type in your program
11. Once a program is typed in - Select tools ... compile in the editors
menu
12. When you are happy with the program - select the project from the
project menu
13. Select project menu ... build. The application should build ok
14. To run the executable ... double click on the executable file

note: an alternative to going to the menu is once you have highlighted a
file - you can right click and select options available to you......

Hope this is of help.
```
