First, what I'm telling you applies to mainframe IBM Cobol.  Don't
know about the PC flavors.

What you want to do is not possible with the SEARCH/VARYING because
you cannot use FROM or BY like with other verbs.  This assumes that
count is defined as an index name of your table.

However, it is very easy to set up your own serach.  You could do
something like:

PERFORM
   VARYING WS-INDEX FROM WS-MAX-SIZE BY -1
   UNTIL WS-INDEX EQUAL 0 OR (whatever your WHEN condition is)
END-PERFORM

Here's a better example using real fields and data:

In WORKING-STORAGE:

01  WS-FIELDS.
      03  WS-INDEX             PIC S9(04)  COMP  VALUE +0.
      03  WS-MAX-SIZE       PIC S9(04)  COMP   VALUE +100.
      03  WS-ITEM-PRICE   PIC S9(5)V99  COMP-3  VALUE +0.

01  WS-TABLE.
      03  WS-PRICE-TABLE  OCCURS 100 TIMES.
            05  WS-PT-ITEM    PIC X(10).
            05  WS-PT-PRICE  PIC S9(5)V99  COMP-3.

In PROCEDURE DIVISION:

PERFORM
    VARYING WS-INDEX FROM WS-MAX-SIZE BY -1
    UNTIL WS-INDEX EQUAL 0 OR
                WS-PT-ITEM (WS-INDEX) EQUAL 'CONDOMS'
END-PERFORM

IF WS-INDEX EQUAL 0
   DISPLAY NOT-FOUND 
ELSE
    MOVE WS-PT-PRICE (WS-INDEX) TO WS-ITEM-PRICE
END-IF.

Hope that helps.

Not true that the VARYING object must be the table index!  That's what I
thought, too, but instead of stating it as fact, I dug out my manual.

In

SEARCH identifier-1 [VARYING   {identifier-2|index-name-1}]
    [AT END imperative-statement-1]
    {WHEN  condition-1 {impoerative statement-2|NEXT SENTENCE}} ...
[END-SEARCH]

" identifier-2, when used, must be described as USAGE IS INDEX or as a
numeric elementary item without any positions to the right of the assumed
decimal point...

" .. If the VARYING index-name-1 phrase is specified, and if index-name-1
appears in the INDEXED BY phrase of identifier-1, that index-name is used
for this search. If this is not the case, or if the VARYING identifier-2
phrase is is specified, the first (or only) index-name given in the INDEXED
BY phrase of identifier-1 is used for the search. In addition, the following
operations will occur:
...
"b. If the VARYING identifier-2 phrase is specified, and identifier-2 is an
index data item, then the data item referenced by identifier-2 is
incremented by the same amount as, and at the same time as, the index
associated with identifier-1 is incremented. If identfier-2 is not an index
data item, the data item referenced by identifier-2 is incremented by the
value (1) at the same time as the index referenced by the index-name
associated with identifier-1 is incremented."

I have never tried this, but it sure reads as though Identifier-2 can be a
non-index dataname counter, effectively creating a numeric subscript
corresponding to the index of an indexed table .

Source: Microfocus COBOL Language Reference, Issue 14, April 1994.
In above, no items are flagged as MF-extensions, and only END-SEARCH is
hilited as "ANSI 85"

MCM
