Without knowing a great deal more about what you are doing, I can offer
that the .COB files sound like source files. The only time I know of when
a program (COBOL or otherwise) requires the source is when it is running in
a debugger, e.g. Animator. Incredible as it may sound, I've come across
more than one occasion where something has been put into production this
way.

*If* this is the source of your problem, then there are a couple of
possbile ways it has manifested itself. A) you didn't get the whole thing
off the old server, because you left behind some part which you did not
know about; or B) you didn't set up the debugger environment in such a way
as it can find it's way to the needed files.

In either of these cases, the difference between the NT client and the
Netware client is not your problem. We use both Netware and NT servers at
our site with MF COBOL programs, and they run on both. However, getting
the debugger to run on either platform against server resident source and
executable does require a little work (it took us a year to get it
reasonably straight).
