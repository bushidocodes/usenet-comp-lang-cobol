I have good news and bad news (all of this is base on your aside about using
VS COBOL II on MVS). The following assumes that you really mean that you
want to be able to handle fixed-length files of "arbitrary length/LRECL".
If what you REALLY wanted was to be able to handle records of different
length in a single variable length file, then ignore all of the following
and just code your FD with a variable format RECORD CONTAINS clause.

Good News - as long as the file is a sequential one (QSAM on MVS), then this
is possible to accomplish.

Bad New - the way to accomplish this is to use an Assembler subroutine to
Open the file (and possibly do all the I/O). But you can at least have the
FD in your COBOL program - you just need to pass it to the Assembler
routine.

Worse news - if the file is indexed or relative (any type of VSAM), then you
will need to do EVERYTHING in an Assembler routine.

FYI - if you were using a different vendor's compiler, there are some (but
certainly not ALL) that do provide native COBOL support for this type of
thing.
