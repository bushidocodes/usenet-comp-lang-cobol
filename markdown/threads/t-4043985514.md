```
The usual method of accomplishing this is to use an assembler subroutine
that invokes the MVS system service that creates a DD dynamically. The
routine will also need to be able to delete the DD. This will allow you
to process in the following manner:

(1) Dynamically allocate the DD specifying a account specific file
name.
(2) Open the file in your program as usual.
(3) Write your stuff to it.
(4) Close the file as usual.
(5) Delete the DD. (You need to delete it now so that you
can recreated the DD later with a new file name).

Repeat the above process as many times as required.

I can't give you a copy of the routine that we use. The monkey with
"Proprietary(sp)" stamp has got to it.

Regards,

Ron Voyles
```

```
The CBT tape (a collection of mostly assembler code) also on the NASP CD
contains several sets of "callable from high level langauge" DYNALLOC
routines.

Talk to sysprog at your site to see if he has the tape or NASP CD on hand.

These routines could be called from your MPP prior to OPENING a file. If
you need to change datasets you could CLOSE the file call the dynalloc
routines, then OPEN, WRITE, CLOSE etc.

These routines work fine. I cannot remember which set I picked off the
tape some years ago, nothing new involved here.

No facility in COBOL to do this, but by calling the assembler dynalloc
subs you can get what you need.
```
