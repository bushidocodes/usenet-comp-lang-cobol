In a COBOL program it is actually not that common to read just one
file.

When you see the select statement:

Select sysin assign to c:\file1.dat
    access is sequential.(etc)

You are right if you have 2 files they have to have different names.
SYSIN would be then name of a file.   On a PC you would use the file
name.  Example: ASSIGN "C:\cobol\sysin.dat".  On a large computer you
might assign a file a symbolic name such as PE11 for personel file 1.
On a mainfram if you didnt do that, everytime you used a fiel you would
have to identify where the file starts how long the record is how much
space is allocated for the file, etc.  It is a little complicated but
not too bad.  We just copy examples of other programs mostly.

Often a record has to be built from information from 2 or more files.
For instance the person's admin data may be in one file, and the
person's financial data may be in another file.  Another example would
be a batch process where a list of transactions (add, del, change,
payroll, inventory change) is in one file, and the other file is a
master file that is to be changed.  Then you have to match up one
record to the other record by some common field like a social security
number.

It is possible to do this several different ways.  You could read one
file sequentially and read maybe the second file is indexed and you
randomly search the file using the record key.  Another way is to put
the transactions in a table in the same order as the master file and
read the master file till you get a match.  Another way is to read one
file read the second file and compare one field.  Based on the
comparison you would do the transaction when they match.  Then if they
do not match you have to decide, based on a comparison of a field,
whether to read file A or file B.  This is a little harder.  It is much
easier to use a file key. However, you can read a file sequentially
faster than you can randomly access a file using a key.  A lot depends
on the size of the two files, and whether you can use virtual memory on
the computer you are using.

On large computers most files are Indexed by a key field or fields.
You just read a file, store some data in a key and use the key to find
the matching record.  Sometimes when the files are smaller it is easier
to just build a table and search the table. Other times there may be
100,000 records or more.  That is a lot to search!

It seems you just need to see what technique the instructor is deciding
to teach then implement that concept.  So you need to see what he wants
you to do.
