```
Hi,

Have you ever had to work on old COBOL programs that were both not documented
nor explicit ? And with no debugger available because hardware and system are
too old ?
Well here is a small shareware I wrote that can save you a lot of time in
understanding how the hell do those wild and unstructured programs work !
You need two things first:
1 - A way to import COBOL source codes as ascii text files on a PC running
Windows (ftp or whatever).
2 - Winword (I tested Word 97 but Word 95 should work fine. I need feedback on
this matter)

What Hypcobsh.exe does:

It reads your COBOL ascii source code and makes a .RTF file of it.
After loading the RTF file with Winword, you will see that colored hypertext
links have been added in the PROCEDURE DIVISION.
You will be able to use the Winword web tool bar to navigate into your source
code as you would do in a HTML text.
Try it. It's easy and self explicit.

you can download it here : http://multimania.com/nirrey/shareware/HYPCOBOL.ZIP
```
