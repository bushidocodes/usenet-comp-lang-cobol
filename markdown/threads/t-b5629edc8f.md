world is divided among "corporate" COBOL users and "individual" COBOL users,
that really does answer the original question/issue about the "demise" of
Micro Focus VisOC - as a "product" (not as a technology).

Thane, i do understand your confusion! these various releases were
"concurrent" (it's difficult to introduce new stuff cleanly). i can try and
clear it up as best i can, but be warned it may generate more! i'll stick
with the thread as long as you have steam! ;-)

Object COBOL for OS/2 and NT was built on "classic" technology. this is the
same core compiler technology that existed in 16-bit form on DOS, Windows
and OS/2 and in 32-bit form on UNIX, OS/2 and Win32.

VisOC was the first product release that was built on "genesis" technology
(next generation -- primarily done to address some architectural issues with
existing "classic" tech and as a means to enable new bleeding-edge
enhancements to COBOL such as multi-threading, >32-bit etc). the new
'Express line of compiler solutions from MERANT Micro Focus (Net, Mainframe
& Server) have all evolved from this technology base.

in many ways, VisOC was initially aimed at providing an MSVC++ like tool for
COBOL programmers, hence GUI was supported in a C++ like manner (either thru
Win32 and/or class libraries in combination with a dialog editor, but NOT
Dialog System). today, all of these methods are supported, plus the
traditional Dialog System (which has been enhanced to support ActiveX) and
Form Designer (an COBOL-centric HTML based painter).

hope this helps more than it confuses!
