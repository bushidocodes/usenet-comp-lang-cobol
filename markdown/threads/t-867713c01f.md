All
Since no specific tools or services have been identified for
coverntion MF Dialog System COBOL to C#, I will propose what is really
just a framework at the moment namely the zcobol translator and macro
processor.  Currently zcobol is in early development with HLASM
compatible code genertion at first target which can be run on z390 on
Windows or Linux or any HLASM compatible mainframe platform such as z9/
z10 or older vse, or even MVS Hercules systems with limitations.  At
moment the highest priority effort is getting the 459 NIST COBOL 1985
test suite running.  Progress report is here:

http://www.zcobol.org/#zcobol_NIST_Test_Results

zcobol also includes capability to generate code in other target
languages besides HLASM.  There is a demo COBOL programs that can be
compiled and executed in HLASM, C for MS Visual Express, Java for
J2SE, and MASM for native Intel:

http://www.zcobol.org/#zCOBOL_Overview

C# could be added as another zcobol target language by "simply" adding
or replacing zcobol code generation macros in new zcobol\C# directory.

All the HLASM code generation macros are in zcobol\z390.  All the non-
target language specific generic COBOL verb macros are in the zcobol
directory (these in clude IF, ELSE, PERFORM, etc which issue calls toe
GEN_ type macros such as GEN_ADD, GEN_SUB, GEN_COMP, GEN_BC to
generate target language specific code.  See zcobol\vce for MS Visual
C code generation macros for demo program.

As an example I downloaded the MF demo NEWSET.CBL and NEWSET.CPB from
here:

http://supportline.microfocus.com/examplesandutilities/nesamp.asp#DialogSystem

I ran them through the zcobol translator zc390.java/class in
z390.jar.  The only error was for a 78 level definition of a constant
which is an MF extension not defined in COBOL 1985 or the draft COBOL
1989 standard.  However I will be adding 78 level support as the goal
of zcobol is to support all COBOL dialects for multiple target
languages with HLASM as the first to be completed.  Once HLASM code
generation macros are stabilized and tested, these can all be copied
to other target language directories and modified to generate othter
target language code.

I will be happy to work with anyone interested.  All feedback and
suggestions welcome.

Don Higgins
don@higgins.net
