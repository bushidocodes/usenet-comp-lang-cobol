Anyone thinking about using PROCEDURE-POINTERs should review the information
at:
  http://publib.boulder.ibm.com/cgi-bin/bookmgr/BOOKS/igy3pg00/4.1.6

Given the fact that this is being talked about in relationship to programs
that call the same subprogram 2 or more times, you should PARTICULARLY note
the restriction that says,

"If you set your pointer item to an entry address in a dynamically called
load module and your program subsequently cancels that dynamically called
module, then your pointer item becomes undefined. Reference to it thereafter
is not reliable."
