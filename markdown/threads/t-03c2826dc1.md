```
: >What are people's thoughts on the impending demand for work for

: >My guess is this will be a) very tedious, but b) very lucrative.

The coding part of it will be easy. Finding the changes in any given
program will be easy with the proper tools, but will be tedious if
done only with an editor.

The big challenge is coordinating the changes among programs in systems,
planning subsets of changes so that testing is feasible, and planning the
whole operation on an economic feasibility basis (for example, why spend
$8,000 and the test time to bring a homegrown payroll system up to snuff
when you can buy a Year 2000 compliant payroll system from a third party
for $6,000). Testing is turning out to be the biggest part of Year 2000.
For some shops, simply finding the machine cycles to do adequate testing
is a problem.
```
