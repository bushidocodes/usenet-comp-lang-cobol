Just because your note might cause some confusion for some readers (who
haven't read the 15 or 20 times I have posted this already)

1) IBM has stated that VS COBOL II, Release 4  *WILL* be supported thru
March 2001 (not 2000 - or 1/1/00 - or any other date). This applies to the
US.  There may be a different date in some other country/counties.

2) IBM has stated that object code CREATED with the VS COBOL II compiler -
and run with the LE run-time library (either RES or linked with it if NORES)
will be supported "indefinitely".  (Assuming they don't try and use any
software/hardware interfaces that are introduced AFTER March 2001 - or that
the program uses some technique which is already documented as "illegal".)

3) Although VS COBOL II R4 is NOT on the Y2K "ready" list of IBM, there are
no KNOWN problems with running it after 1/1/00.  HOWEVER, the dates that it
produces (ACCEPT FROM DATE, WHEN-COMPILED paragraph, and CURRENT-DATE
special register, etc) all use 2-digit years - not 4-digit years.

4) if by "IBM Cobol" compiler, you mean OS/VS COBOL compiler, then you are
already in "deep do-do".  That compiler hasn't been supported for years
(already) - but once again - OBJECT code created by it (which doesn't use
"new" hardware/software interfaces) *IS SUPPORTED* - if you are using the LE
run-time library - with a couple of obscure exceptions such as odd
Communications Sections.

5) Finally, what do you mean by "support" for the OLD compilers?  OS/VS
COBOL *was* provided with source code and a logic manual.  I suppose someone
COULD try and actually provide "fixes" to the compiler if you really paid
them enough. On the other hand, VS COBOL II is - and always has been - OCO
(object code only) so no one is ever going to be able to give you a "code
fix" if something in it "breaks" after 3/31/01.  If someone tells you that
they can, I sure wouldn't "buy a bridge" (or a used car) from them.

The following views are my understanding:

IBM will support the Cobol II compiler for a while after 2000.  It will
probably work much longer than that.

If other companies were to support IBM's compiler, they must have access to
the source code.  It is not obvious that they will get it.

If you by compliant compiler mean a compiler that generates compliant
object code, then COBOL 390 is not much more compliant than Cobol II.  It
has some features that will help you write compliant programs.  Putting
your old cobol source through the Cobol 390 compiler will not in general
produce more compliant object code.

To make your program compliant you have to do some changes.  It may be
easier to do this with the Cobol 390 compiler than with the Cobol II
compiler.  But still you have to do it.
