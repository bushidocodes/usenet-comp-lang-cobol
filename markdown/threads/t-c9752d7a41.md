Is there anyway to ignore the CR/LF at the end of a sequential file.
I have to read sequential records (each record is 53 caracters long
and is on it's own line) and validate them one by one. I can trace
my program step by step with Fujitsu Cobol. I then print the first record
with errors.
Then it reads the next one. There it is. If I look at the string is has in
memory, there are two caracters more. And it seems to places them always in
the beginning of the new record read. So by the 1000th record, there are
2000 caracters more.

Help.

Thanks.

__
Pat

select clause.

SELECT FILE-IN ASSIGN TO "MYFILE.DAT"
ORGANIZATION LINE SEQUENTIAL.

This will treat your input as a line sequential (CR/LF) input file.
