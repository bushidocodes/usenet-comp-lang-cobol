```
       ENVIRONMENT DIVISION.
       SPECIAL-NAMES.
           CALL-CONVENTION 74 IS winAPI.

                :
                :

       WORKING-STORAGE SECTION.
       01  wValues.
           05  wSQLstat                PIC s9(9) comp-5.
               88 sqlSuccess            VALUES 0 1.
           05  wMy-flags               pic 9(8) comp-5.
           05  wMy-retcode             pic 9(8) comp-5.

       01  connString                  pic x(256) value
            'Driver={MySQL ODBC 5.1 Driver};Server=localhost;'
           &'Port=3306;Database=hotelDB;User=root;'
           &'Password=mypassword;Option=3;'.

       EXEC SQL INCLUDE SQLCA END-EXEC

                :
                :

       PROCEDURE DIVISION.
           PERFORM 000-Connect-toSQL


           STOP RUN.
                :
                :

       000-Connect-toSQL SECTION.
           EXEC SQL
             CONNECT TO :connString
           END-EXEC.

           MOVE SQLCODE TO wSQLstat

           IF sqlSuccess
             CONTINUE
           ELSE
             CALL winAPI 'MessageBoxA' USING
               BY VALUE 0 SIZE 4,
               BY REFERENCE ".Error : 003." & x'0D0A' &
                ".SQL-BASED data disconnection." & x'0D0A' &
                ".Call MIS group or vendor for support."
               BY REFERENCE z"My Company dotCOM"
               BY VALUE wMy-flags
               RETURNING wMy-retcode
             END-CALL
           END-IF.



Simple code to open a MySQL ODBC database. Search "ODBC connection
string" in Google (as Pete's advised).
```

Hello,

If you are interested in coding in cobol (as mentioned earlier) and have
MF NetExpress 5.1 (also as mentioned before), you have all tools to
develop you project. It is a question of methodology.
It seems that you have a running cobol application but want to connect
it to a database :
o  does the database exist ? if no :
      i.  have a look at your cobol application and see if the files can
be easily translated to a database structure
      ii. if they are close to a "normalized" (in the way of a Codd
relational definition), the job will be quite easy
      iii. if not, try to make the modification to normalize the file
structure as much as possible and have the application running under cobol.
o  design a database structure which will be a clone of your cobol file
system
o  under XP with the administration tools, create an ODBC definition to
access the database server (MySQL, MS-SQL, Access, Postgres, whatever)
o  use OpenESQL Assistant delivered with NetExpress 5.1 to create all
cobol connections and queries to the database through ODBC (many
examples come with NetExpress)
o  replace the read/write access in your application with Embedded SQL
orders and adapt the program as necessary.

This does not work 100% and depends largely on the structure of your
existing cobol code. But it can be quite a quick process to access the
data stored in a database without rewriting a complete application.
