```
I have been asked by my client to write a system which will run on both
MSDOS and Unix platforms.  The client's standard programming language is
Microfocus COBOL.  Although I am familiar with MF COBOL on Unix, I have no
experience of using this language on DOS.  Can anyone with such experience
give me some idea of how compatible the two compilers are at the source code
level?

Additionally, parts of the system will need to make TCP calls to an FTP
server.  I'm happy with writing an FTP client, and have the bones of one
already using low-level calls to Unix's TCP API, but I wonder if using
CCITCP would be a better option, as this way I may be able to avoid having
to write the TCP layer of the application twice (once for Unix using the
Unix API, once for DOS using it's API).

Reading the Toolbox guide it appears I can only connect to a server which
has previously advertised itself with the host's CCITCP daemon.  Is this
correct, or is there a workaround?  Obviously, a vanilla FTP server will not
have knowledge of CCITCP, and so will not register with the CCITCP daemon.

Many thanks in advance,

Derek Sorensen
```

```
I develop on Win-NT using 16 Bit MF Workbench V 3.4 (i.e DOS-Box
development). After finishing all tests I just copy my sources using
ftp on my Unix machine where the final compile will be done with MF
Cobol V 4.1)

Using WorkBench is much more faster and easier than using the cobol
toolbox on Unix.

If you take care upon filenames and use standard cobol (no machine
depending routines) it should work.

More or less I don't have any problems.....

HTH - Stefan Meyer
```
