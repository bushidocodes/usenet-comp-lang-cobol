We are running COBOL for MVS and VM version 1.1 at the present moment.

We will soon upgrade our operating system from existing MVS/ESA SP
version 4.3.0 to MVS/ESA SP version 5.2.2. I learnt from a document
with item number RTA000103808 (issued in MAr 97) that COBOL for MVS
and VM version 1.2 is required to run with MVS/ESA SP version 5.2.2.
However, we are running out of time to install this new version of
COBOL for MVS and VM. So, can COBOL for MVS and VM version 1.1 still
run with MVS/ESA SP version 5.2.2 ? Is there any shop running COBOL
for MVS and VM version 1.1 with MVS/ESA SP version 5.2.2
at the moment ?

As a matter of fact, when I read through IBM mauals related to MVS/ESA
SP version 5.2.2 and related program directory and PSP bucket, it was
never mentioned that COBOL for MVS and VM version 1.2 is the minimum
version required to run with MVS/ESA SP version 5.2.2. So, I
got the understanding that COBOL for MVS and VM version 1.1 should be
able to run with MVS/ESA SP version 5.2.2 until I spot that document.

We really want to keep using COBOL for MVS and VM version 1.1,
otherwise, our project of implementing MVS/ESA SP version 5.2.2 will
be much delayed. Pls help. Thxs a lot.


Regards,

Carlton
