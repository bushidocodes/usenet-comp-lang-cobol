```
R15 = 0.  PSW shows 31 bit addressing mode ON.  PSW shows you are
at 00004, which is the middle of the Restart New PSW.  This means
that you have branched, jumped, GOTO'ed to address x'00000000'
(R15).  It would appear that you have gotten a PIC 1 (ABEND
S0C1).

If you are using SYSUDUMP, you need to find R14 (8CBA83C2).  The
actual address is x'0CBA83C2'.  The save area chain is at
x'00025168', and you should run this back to see who called who. 
Also look at x'0CBA80A0' and see what program lies at that
location.  It appears that this program is the one that choked.

Assuming that you are getting RC=4 from BINDER, and that you are
not using LET and/or NCAL as an option, perhaps you need to check
with IBM for APAR/PTF info using MSGCEE0374C ABENDS0C1 for
keywords.  Also, perhaps you need to use TEST as an option for
BINDER?

This is as much as I can do for you at this point, w/o having any
further info.  I'm more of an ALC programmer, who happens to do
COBOL stuff.
```
