From my "usually reliable source",

  It really depends on what he wants to do.  The general answer
is the QUIT command, but what he writes depends on how often he
wants to test 'VAR'.

General:
    IF VAR = 'VALUE' THEN QUIT;

To test at every statement:
    AT * IF VAR = 'VALUE' THEN QUIT ELSE GO;

And to test at every statement without stopping at every statement:
  Here's the DT command I entered:
      AT *
        PERFORM
          IF WK-SEARCH-TYPE = 'D'
            QUIT ;
          ELSE
            GO ;
          END-IF ;
          GO ;
        END-PERFORM ;

This allowed the app to run without user intervention
at each breakpoint.  When the variable reached "D",
the quit was issued and user was given the end
of session prompt.
