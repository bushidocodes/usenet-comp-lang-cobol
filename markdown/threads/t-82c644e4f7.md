Using ACUCOBOL, I do the following:
I would have the following in my DATA DIVISION:

01  convert-area.
    03  convert-6bit		pic x.
    03  convert-bin		pic x comp-x
				redefines convert-6bit.
    03  result-pic99		pic 99.

and then in the PROCEDURE DIVISION, something like:

    move passed-pic-x to convert-6bit.
    move convert-bin to result-pic99.

result-pic9 is now a 2 digit display type variable.

The pic x comp-x is a strange type, which is a binary type, using 1 byte
of storage, which is exactly what you need to redefine bit data to convert
it to digits.

What compiler are you using?
