These concerns are discussed in Steve McConnell, Code Complete,
1993, Microsoft Press, Chapter 18, Layout and Style. Unfortunately,
the book does not discuss COBOL. Fortunately, the author used
several studies to guide his writing.

Four styles are discussed using the languages Ada, C, and Pascal.
These styles are presented in abstract form.

Pure-Block (recommended for Ada)

*********************************
    **********************
    ***************
********

Endline

Not shown since it has been shown as inferior.

Emulating Pure-Blocks

*********************** begin (or { )
    ****************
    ************
end (or } )

begin-end Block Boundaries

**************************
    begin (or { )
    ****************
    ************
    end (or } )

Using Pure-Block (or its emulation) for COBOL becomes something like,

section-name SECTION.
    ************************.
    ****************************************.
    *******************************.
END-SECTION.

Notice the lack of an EXIT statement! Also, note that the absence
of a paragraph-name at the beginning of the section is non-standard.
Personally, I would get rid of all periods except the period at the
end of the last statement.

[Note to Cheesle: Think about the above, I find it to be more pleasing
than having section-name-EXIT and the EXIT statement.]

Using begin-end Block Boundaries becomes someting like,

paragraph-name.
    **************************.
    **********************************.
    ***************************.

Note that I prefer a single period on the line following the last
statement.

This book also has recommendations for the layout of individual
statements and control structures. For COBOL, using END-xxx
delimiters, everywhere, would match the recommendations in the
book.

Steve McConnell offers the following opinion, "Good
programmers should be open-minded about their layout
practices and accept practices proven to be better than the
ones they're used to, even if adjusting to a new method
results in some initial discomfort."

However, I feel that, in the case of the COBOL examples, one
is not provably better than the other; hence it is a matter of
personal preference.

I think Cheesle prefers the "pure-block" style, while most of
us prefer the "begin-end block boundary" style.

I will recommend the book, "Code Complete", to those who
use other languages, particularly C. I cannot recommend the
book to those who use COBOL exclusively.

Rick Smith
