I actually wrote one of these in Access II (then someone tromped
the version and could never get it back :( )  It generated job
documentation for MVS job streams, programs called multiple times
in multiple places, etc.

The simple database I made was a list of programs with details
linked to a table of the list of errors for that program.  I
generated this in HTML.  This was followed by 'job' table linked
to a list of steps for that job.  The HTML was generated for each
job and steps with a link to the details of job in question and
each step had a jump to the program and the restart instructions
(with could also be reused).

Before this was started creating new documentation for a
essentially repeated JCL (maybe a specialized rerun) was
horrendous.

This sort of stuff is very easy.  Once you get the gist of
creating the static forms it should be easy to port this to a CGI
program (??? never done it but the theory is easy).
