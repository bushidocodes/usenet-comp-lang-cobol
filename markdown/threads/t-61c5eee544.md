It does not work with RM/COBOL, because RM/COBOL programs are compiled to
intermediate code and then interpreted by a runtime "runcobol". But Liant
has a product that used to be called DB/Plus (I think they changed the
name), that allows a transparent access to Oracle (and Informix and some
other) databases. You don't even recompile. You activate a kind of file
server, and if the runtime finds the file in the database, it "connects"
you there. If not, it looks in the normal places - the file system.
You use the database tables like normal COBOL files. Every write, update,
delete is commited, so you don't really have transactions (in the
database). Also, no SQL.
If you need SQL, you would have to write your own routines with Oracle's
ProC, and link them to the RM/COBOL runtime. Then you could call, for
example, CALL "SQL-XXX" USING "whatever".

Albert Ratzlaff
