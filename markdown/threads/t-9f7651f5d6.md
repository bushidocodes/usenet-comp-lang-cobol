With a filesystem where all files are regarded as unstructured byte 
streams, the answer would be 'no', regardless of what programming 
language is used.

The Unix utility 'wc' will, however, give you the answer to the question 
outside of Cobol, and you can possibly call the utility from inside 
Cobol. Versions of wc for Windows are freely available. The example 
below shows 63 lines, 317 `words' and 3185 characters in the file tested.

N. Shamsundar
University of Houston

----------------------------
S:\ddc>wc csort.pli
      63     317    3185 csort.pli

Create a one-record, one-field (record-count) control file.
Initialize the record-count field to zeroes.

Whenever the data file is created or maintained,
update the control file's record-count field.

To determine how many records the data file has,
open the control file and read the record.

You can add support fields to the control file
if you like, such as last date/time updated,
last update module, last update user,
last record count delta, etc.

> Unfortunately the records will be variable length and I need an exact
> figure not an approximate one.

Then either:

a) Don't use LINE SEQUENTIAL, use RECORD SEQUENTIAL and write CR+LF
terminators as the last two bytes of a space padded fixed length
record so that you can calculate. (as long as your system will write
this as fixed size blocks of course).

b) Read the file and count.

With a line sequential file the number of 'records' is the number of
record terminators in the file, usually CR+LF.  Think of the problem
of knowing how many 'A's there are in the file.
