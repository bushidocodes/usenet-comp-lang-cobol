```
As I understand it, server express is the next MF offering on unix, and it
allows for 'native files' > 2gb.  I.e it allows for the 'new' unix large
file system.

I believe this is now available on RS unix but not yet on HP -UX.  Also it
will be limited to HP-UX 11 and will not be available to earlier releases.

Does anybody know if:
. Server express is just a development system or also includes a compiler?
. If it is a development tool will MF Cobol 4.1 give the same file size
capabilities?

The reason I'm asking is that we do not develop in MF Cobol.  We develop on
other platforms and simply use MF COBOL as a compiler.  We don't normally
need anything fancy on the Unix.  So if MF supplied a simple compiler that
allowed for Unix large file systems we would be happy.

The last I heard, MF COBOL 4.1 handled large files by file striping (a
nonsense system if I ever saw one).  However the MF site says that MF Cobol
4.1 on HP-UX 10.20 is not the same as MF Cobol 4.1 on HP-UX 11.  So maybe on
HP-UX 11 it does things the right way!  Can anyone confirm this?

Ta

Rick
```
