Well, if you really mean your system has a stored a signed number field as
text in a varying field (which sounds very odd) .......

First, are you using an Oracle database, accessing via the embeded SQL
pre-processed by the Oracle pre-compiler?

If so, define the host variable in the "declare" part of working-storage
thus:

01    oracle-string           pic x(42) varying.

The pre-compiler will expand this into
01      oracle-string.
    03  oracle-string-len      pic s9(4) comp.
    03  oracle-string-arr      pic x(42).

Define a standard cobol variable:

01     cobol-string.
    03 cobol-number      pic s9(18).


The access it by
    exec sql
         select odd_string
           from  my_table
           into   oracle-string
    end-exec
    move oracle-string-arr to cobol-string
    display cobol-number

On the other hand, if you're using MF's OpenESQL, I'm not your man - sorry!!
