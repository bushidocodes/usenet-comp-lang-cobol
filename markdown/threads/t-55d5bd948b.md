```
Hello,

i think i have found a way to use the same module - subprogramm - in
CICS-Online and Batch - with file access.

This is no idea, i have created one module like the following, to make
file-access in batch and online. The data-access is different, the
data-modifikation is the same.

In Batch and Online, i need a base-date, which is stored in a
vsam-file. The submodule reads a record, incrementes a date und
rewrite the record.


Main-module - Batch:

                SET             C-INVOKE-BATCH          TO TRUE
                CALL            C-MODULE        USING           C-AREA
                END-CALL

Main-module - CICS-Online:

                SET             C-INVOKE-CICS           TO TRUE
                SET             C-DFHEIBLK      TO ADDRESS OF 
DFHEIBLK.
                CALL            C-MODULE        USING           C-AREA
                END-CALL

Submodule: (must be compiled for CICS)

        File-Secion.
                ASSIGN...
                FD
                ... and anything else you want.

        Linkage Section.
                COPY            DFHEIBLK.               (Inserted by 
translator)
                DFHCOMMAREA.
                05              C-Module                PIC X(08)      
 VALUE 'SUBPGM'.
                05              C-RELEASE               PIC X(02)      
 VALUE '01'.
                05              C-INVOKE                PIC X.
                        88      C-INVOKE-BATCH                  VALUE 
'B'.
                        88      C-INVOKE-CICS                   VALUE 
'C'.
                05              C-DFHEIBLK              USAGE POINTER.

        PROCEDURE DIVISION USING DFHEIBLK DFHCOMMAREA.

The first instruktion in Procedure-Division must be the following.
This must be done, to access the first parameter. Because the
CICS-translator inserts a modified 'PROCEDURE DIVISION'.

                SET ADDRESS OF DFHCOMMAREA TO ADDRESS OF DFHEIBLK.

                EVALUATE TRUE
                 WHEN           C-INVOKE-CICS
                        SET ADDRESS OF DFHEIBLK TO C-DFHEIBLK
        now i can use every components from CICS - like EXEC-CICS 
READ...
                 WHEN           C-INVOKE-BATCH
        and here i can use OPEN CLOSE READ and so on for BATCH...
                END-EVALUATE.

There are several other ways to do this job. This was a test and it
works fine.

Reference: CICS Application Programming Guide: Nesting - what the
application programmer must do.


- have a nice day - Andreas Lerch
```
