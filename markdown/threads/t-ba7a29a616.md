### "Sad but True" - a little known fact about past and current Standards

```
As a follow-up on the questions about how "free form reference format"
listings may end up looking (on the printed page or the screen - or
wherever) ...

Did you know that the '85 Standard (like Standards before it) *and* the
recently approved 2002 Standard is *explicit* in saying that it is totally
UP TO THE VENDOR not just what listings look like - but even whether or not
the original source code (or any variation of it) is actually produced in
"printed compiler  output"?

Whether talking about the new
    >>PAGE
         and
    >>LISTING
directives - or about the older
   "/"
in column 7,  past, present, and new Standards all talk about *IF* the
vendor produces a compiler listing, then ....

In truth and fact, as far as I know all compilers that CLAIM conformance to
the '85 Standard do have at least an OPTION to produce a source code
listing - but there are certainly some major differences in them. (Notice
that I said the compilers that I know - others may know of some that do NOT
provide this facility.) One of my favorites (and I was surprised when I
found out that not all compilers worked the way that I was used to) is what
is done with COPY/REPLACING and REPLACE statements.

Some compilers show the source code before the REPLACE/REPLACING takes place
Some compilers show the source code after the REPLACE/REPLACING takes place
Some compilers show the source code both before and after REPLACE/REPLACING
   and
Some compilers give you choice of which of these you want to see.

  ....

Now back to the question asked in the other thread,

Personally, I suspect that MOST programmers with MOST compilers that use
free form reference format will limit their line lengths to 130 or so
characters that are EASILY visible in "fixed font" online editors.  If their
source lines are so limited, then the listings should be "relatively" easy
to figure out.  OTOH, the Standard *requires* that a (2002 conforming)
compiler support up to 255 characters (that may be mixed between "single
byte" and "multi-byte" characters - each of which counts for ONLY 1 of those
255 characters).  Therefore, vendors (should there be any that actually DO
implement the full 2002 Standard) will need to come up with some
"user-friendly" way of displaying source code (such as an online source code
viewer with scroll bars).  But somehow, I think this is the LEAST of the
"inhibitors" to seeing full implementations of the 2002 Standard.
```
