OTOH it may not be the syntax that he is at a loss with, but the
concept of sharing files and locking records.

Record locking is done to communicate with _other_ processes
running at the same time and accessing the same files.  It
is done to prevent the other programs (or users) from updating
the designated record(s) for a short period of time.

If a record is read WITH LOCK then that record will be locked
until it is REWRITEn or UNLOCKed and other programs trying to
read that record will be informed (via the File status) that
this is not available for them at the moment, a later read
may find that the first process has released the lock.

Normally a WITH LOCK will release any other lock on that file
so that only one record at a time is locked in each file by
each process at max.

WITH KEPT LOCK does not release previous locks on records of
this file so that a single process can hold locks on
several records at one time.

Hope this helps.
