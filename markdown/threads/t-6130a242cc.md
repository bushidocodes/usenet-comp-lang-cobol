Bob,

The following sample code will do what you want.  You may have to play with the
Hex bytes in the "move z'\StringFileInfo\040904B0\FileVersion'" statement, as
they relate to specific code-page and language identifiers.  All of our software
version info is stamped using the same code-page and english language.  Change
the "FileVersion" to whatever version info block key you want to retrieve (ie.
ProductVersion, ProductName, etc.).  I think I got all of the WS fields used
(code excerpt) - let me know if I didn't.

Hope it helps ...

Todd Lucas
Todd.Lucas@C-Cubed.net


Sample Code:

       environment division.
       configuration section.

      ******************************************************************
      * Calling Convention notes:                                      *
      *    66  - Win32-bit C calling convention - dynamic linking.     *
      *    74  - Win32-bit C calling convention - static linking.      *
      ******************************************************************
       special-names.
           call-convention 66 is win32nol
           call-convention 74 is win32api.

       working-storage section.
       01  ws-miscellaneous.
           03  ws-product-log-path         pic x(256) value spaces.
           03  ws-buffer                   pic x(2048) value spaces.

       01  ws-subroutines.
           03  cob32api-sub                pic x(8) value 'cob32api'.

      * Win32API call fields - GetVersionInfo:
       01  ws-verinfo-ptr                  pic s9(9)  value +0   comp-5.
       01  ws-verbloc-len                  pic s9(9)  value +0   comp-5.
       01  temp-pointer                pointer    value null.
       01  load-dll-ptr                procedure-pointer.
       01  win32api-rc                     pic s9(9)  value +0   comp-5.

      *----------------------------------------------------------------*
      * This section will retrieve the product version info.   *
      *----------------------------------------------------------------*
       1010-Get-Version-Info  section.

      * Load the Microsoft Version information DLL:
           set load-dll-ptr            to entry "VERSION.DLL".

      * Assign the C3 module name to query for version information:
           move z'C3TCPCAL.DLL'            to ws-product-log-path.

      * Check for version info - will return 0 if none:
           call win32nol 'GetFileVersionInfoSizeA' using
               by reference ws-product-log-path
               by value     +0 size 4
               returning    ws-verbloc-len
           end-call.

      * If no version information exists for this file then we are done:
           if ws-verbloc-len equal +0
               go to 1010-exit
           end-if.

      * Retrieve the version information resource from the module:
           call win32nol 'GetFileVersionInfoA' using
               by reference ws-product-log-path
               by value     +0 size 4
               by value     ws-verbloc-len
               by reference ws-buffer
               returning    win32api-rc
           end-call.

      * If API call failed then we are done:
           if win32api-rc equal +0
               go to 1010-exit
           end-if.

      * Retrieve 'FileVersion' information from VersionInfo buffer:
      * The Lanuage and Codepage Identifier ("040904B0") is hardcoded
      * because we always know that it will be in English for C3 mods:
           move z'\StringFileInfo\040904B0\FileVersion'
                                           to ws-product-log-path.
           call win32nol 'VerQueryValueA' using
               by reference ws-buffer
               by reference ws-product-log-path
               by reference temp-pointer
               by reference ws-verbloc-len
               returning    win32api-rc
           end-call.

      * If API call failed then we are done:
           if win32api-rc equal +0
               go to 1010-exit
           end-if.

      * Retrieve specific version information from version info buffer:
      * The Lanuage and Codepage Identifier ("040904B0") is hardcoded
      * because we always know that it will be in English:
           move low-values                 to ws-product-log-path.
           call win32nol 'lstrcpyA' using
               by reference ws-product-log-path
               by value     temp-pointer
               returning    win32api-rc
           end-call.

      * Move the version information to the log buffer:
           move ws-product-log-path        to plf-c3-version.
           inspect plf-c3-version replacing all low-values by spaces.

       1010-exit.
           exit.
