```
Just a follow-up on a thread from last week (after several private emails
and some clarification from one of my RW experts/friends)

Although the draft of the next COBOL Standard *does* include the "ATTRIBUTE"
phrase for RW - which does mention "fonts" - the draft only allows it at the
RD (or complete report level).

My memory (once refreshed) was that this was introduced to handle such cases
as Katakana, hiragana, Kanji - and other "non-Roman" fonts.   As written it
does NOT lend itself well to specific "fonts" for specific items (e.g.
highlighting, italics, color, etc).  Certainly a specific vendor could do an
EXTENSION for this - or the Standard after this (whatever century that
happens in) might provide this.

Sorry for the mis-information that I provided before.
```
