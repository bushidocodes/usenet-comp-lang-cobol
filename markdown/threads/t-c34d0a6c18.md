It is likely that the program you are trying to run is a 'family
mode' program that can execute in both DOS and OS/2 native. The
header has additional information that is ignored by DOS and checked
by OS/2. The program can then run in OS/2 [286] protected mode
making appropriate OS calls.

Windows programs use a similar scheme. It would appear that the
program thinks it is in protected mode and has initialised
the OS/2 run-time code which then notices that it is not OS/2.

Solutions: Run it on OS/2.
Get rid of Win95 and use DOS
copy the COBLIB.DLE to COBOS2.DLL so that when it
starts the OS2 support it really gets DOS

Other operating systems running protected mode V8086 DOS tasks
also sometimes give the same problem.
