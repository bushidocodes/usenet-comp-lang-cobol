Antivirus has nothing to do with it.

These are .DLLs that need to be registered as components.

Normally they should self-register during the install process. If they are 
not doing so it may be any of the following:

1. The .DLLs are 'broken' ... probably corrupted or improper copies of the 
software.(unlikely...)
2. The program 'regsvr32' is missing from your system. (It lives in the 
windows\system32 directory on most XP systems). (unlikely...)
3. Another .DLL which is required by these .DLLs has not been found. The 
registration process checks all dependent libraries; if something is 
missing, registration fails. (pretty likely...) The hex errorcodes are COM 
return codes and generally are about "unspecified errors". This means they 
are not really helpful and can occur under many different circumstances 
(although there are "Most probable"  circumstances, a bit like Feynmann's 
sum over histories...:-)) Check out this web site: 
http://classicasp.aspfaq.com/general/why-do-i-get-non-database-related-80004005-errors.html
4. An event during the install has prevented the registration. (Maybe 
licensing or security is not correct...Fujitsu may have removed your account 
from their CASPER licencing server) (unlikely)
5. There is some other reason not covered here. (possible...)

ACTION:

1 Register them manually.  Run: regsvr32 <dll name>  ...from a command line 
or Windows Run.

2. If they fail, you will need to investigate further. Use an Object Browser 
or COM explorer to check the dependencies (You can download these tools for 
free and there are some very good ones available.) Make sure all necessary 
.dlls are installed and registered on your system.

3. Contact me privately if you need further help.

HTH,

Pete.
