In an very old application which I have to maintain (:-{) there are
sometimes deadlock situations.

first question block:
How long keeps a record locked when a file is opened with lock mode
automatic?

1. I'm sure until the next write (without keep record locked...) or the
next close statement on this file.

2. Until the next read on this file regardless if the read fails or is
successful?

3. Until the next start statement on this file regardless if the read
fails or is successful?

Or should I always use unlock before any read/start statements to be
sure the last locked record is unlocked?

Second question block:

In a deadlock situation
- we kill one program expecting the locked record will be unlocked.
- More than once the record still keeps locked and we have to kill the
second program as well.

Which is the correct way to kill (or close down) a COBOL program on
Unix so that the files are being closed by the runtime system so that
locked records will be unlocked? (Short time solution, I know...Code
review would be the best but...)

Any help would be gratefully appreciated.
