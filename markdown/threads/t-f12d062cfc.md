>From "IBM COBOL for MVS & VM Language Reference" page 291:

The EVALUATE statement provides a shorthand notation for a series of
nested IF statements. It can evaluate mulitple conditions. That is,
the IF statements can be mede up of compound conditions. The
subsequent action of the object program depends on the results of
these evaluations.

Format (From "VS COBOL II..." by James Janossy page 129):

Syntax variation Example

EVALUATE [TRUE|FALSE] EVALUATE TRUE
WHEN [relation] WHEN WS-VALUE < .75
WHEN [condition] WHEN <88-level name>
WHEN [OTHER]
END-EVALUATE.
-----------------------------------------------
EVALUATE [identifier-1] EVALUATE WS-VALUE
WHEN [literal] WHEN 'F'
WHEN [range] WHEN 'PR' THRU 'ZQ'
WHEN [identifier-2] WHEN HRS-REG-WEEK
WHEN [OTHER]
END-EVALUATE.
-----------------------------------------------
EVALUATE [expression] EVALUATE A * B
WHEN [literal] WHEN 152
WHEN [range] WHEN 25 THRU 57
WHEN [identifier-3] WHEN HRS-REG-WEEK
END-EVALUATE.

Example (Again, Janossy):

EVALUATE WS-LETTER-GRADE(WS-SUB)
WHEN 'A'
DISPLAY 'EXCELLENT WORK!'
WHEN 'B'
DISPLAY 'VERY GOOD!'
WHEN 'C'
DISPLAY 'OK, BUT YOU CAN DO BETTER.'
WHEN 'D'
DISPLAY 'MUCH ROOM FOR IMPROVEMENT'
WHEN 'F'
DISPLAY 'FAILED -- TRY AGAIN'
WHEN OTHER
DISPLAY '?? UNKNOWN GRADE ??'
END-EVALUATE.

EVALUATE really comes onto its own when one has to code in a decision
table. One can code the table structure as shown. One uses ALSO
which "seperates selection subjects within a set of selection
subjects; separates selection objects within a set of selection
objects" [IBM]. In other words, ALSO acts as an AND:

>From Janossy, page 135:

EVALUATE MACHINE-TYPE ALSO MACHINE-STATUS
WHEN 'PC' ALSO 'W'
DISPLAY 'BUY SOFTWARE FROM STORE'
WHEN 'PC' ALSO 'B'
DISPLAY 'SELL MACHINE TO FIX-IT SHOP'
WHEN 'XYZ' ALSO 'W'
DISPLAY 'BUY SOFTWARE FROM MAKER'
WHEN 'XYZ' ALSO 'B'
DISPLAY 'RETURN MACHINE TO MAKER'
WHEN OTHER
DISPLAY 'INVALID MACHINE OR STATUS CODE'
END-EVALUATE.

Placing two WHEN's consecutively with a single imiperative-statement
following serves as an OR:

EVALUATE WS-CODE
WHEN 'YYS'
WHEN 'IEI'
DISPLAY 'NEW CODES ENTERED HERE'
WHEN 'ICI'
DISPLAY 'OLD CODE STILL IN USE'
END-EVALUATE.

Once you get the hang of it, EVALUATE is a great tool.

As long as someone has brought up the EVALUATE statement, let me
egotistically (though not narcissistically) tell of how I am responsible
for a very small part of the Cobol standard. The long version of the
story can be found at the last half of
http://members.aol.com/PanicYr00/CobolProgrammer.html, so I'll just say
that the 1981 draft of Cobol-8X (as we then called it) prohibited the
use of the figurative constant ZERO (or its synonyms ZEROS and ZEROES)
as a literal in and EVALUATE statement, but at my urging this was
changed to say that ZERO/ZEROS/ZEROES was automatically the same type as
identifiers-1. I am quite sure that since 1985 someone (maybe even two
or three someones) has used ZERO for that literal, not ever realizing
that if not for my work, they would have gotten a syntax error.
