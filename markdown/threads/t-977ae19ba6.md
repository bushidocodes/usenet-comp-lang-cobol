It is my guess that this is a homework question - but I am not positive.

If this is a "real world" question, then check to see if your "compiler of
choice" supports Report Writer.  This is the type of report that is
INCREDIBLY easy to do in RW.  (It will let you do all the line/page breaks
you want for areas or sections, or whatever - and do your totaling for you).

If this is a homework assignment, then find out if you are allowed to use
report writer (and if the compiler in use for your class supports it).  If
so, that is still the way that I would go.

Either way, you might want to go to:

 http://www.spc-systems.com/

and check out the "Tutorial" section on how to use RW (in COBOL)

You will need to sort your data into something like this

01  DATA-RECORD.
      05  AREA            PIC X(05).
      05  REGION        PIC X(05)
      05  SALES          PIC S9(07)V99 USAGE IS COMP.

Read all your input, then sort the data on Area +  Region

Then read the sorted input, accumulating totals, and printing the
appropriate values on a control break

READ Sorted-Input      ( or RETURN it, if you use SORT, which is what I
would do)
  AT END
     SET End-of-Sorted-Input to true


PERFORM UNTIL End-Of-Sorted-input
  IF New Region and not start-of-sorted input
     (calculate the percentage lines and print)
  ELSE
     if new-city
       Extend the print line to the right
     else
       add the sales from this city record to the current city total, update
'grand total' accumulators
     end-if
   END -IF
   IF new-area
     print area headers
   end-if
   Read next sorted-input
      at end  set end-of-sorted-input to true
END -PERFORM


You will have to play with the control break logic, but once you get the
data sorted in control break order, that should come in a fairly
straightforward manner.

MCM
