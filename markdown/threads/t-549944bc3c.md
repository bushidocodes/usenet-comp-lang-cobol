You can try Jeff's suggestion using hex '0F'. Most Epson printers use this
code for condensed mode which should give you about 132 characters per line
on 8.5" paper.  The Fujitsu compiler may reset the printer and override your
controls, however.  I've also used the old DOS DEBUG program to set up
control codes and  set the printer using a TYPE  PRTRESET > PRN from DOS,
where PRTRESET is the name of the file I created using DEBUG.  A list of the
most frequently used printer commands might be found at www.epson.com. Good
luck.

Denny
```

On my old Commodore 128/64 you sent hex codes to the printer to change
fonts/sizes/super/subscript type as well as wide/underline/bold.

You might try printing something from a filler with hexadecimal codes in it:

       01  PRINTER-CODE  PIC X(003) VALUE SPACES.

       PROCEDURE DIVISION.

       0000-MAIN.
           MOVE '7F20' TO PRINTER-CODE
           WRITE PRINTER-RECORD FROM PRINTER-CODE
           STOP RUN
           .


This would send the character codes 127 and 32 to the printer record. The
same as :

Print #1,chr(127);chr(32);

in BASIC.

Of course you would have to have some kind of manual which tells you the
printer codes. Most of the early printer drivers put the printer in graphics
mode through Windows to do the fancy WYSIWYG stuff that the modern word
processors produce. But this would require generating a bitmap, converting
it into the priters bitmap format, then dumping to the printer. I wrote
something to do this once and it is very ugly.

Try inquiring with the manufacturer of the printer. They would be your best
bet on getting the hex codes for changing type faces and styles.
```
