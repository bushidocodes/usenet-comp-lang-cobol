Hi Christine,
  My Fujitsu 3.0 manual Requirements states Window 95/NT. I assume
your goal is self-teaching a COBOL GUI as nobody could be serious with
FSC 3.0 for a commercial product (only works on NT/95/3.1). I assumed
after file/new you right-click MainForm[form] and select open then
depress a control and left click? Or since you have the PowerCOBOL
interface up try opening up a PC 3.O project file.  However you may be
able to use the Fujitsu COBOL ANSI 85 features that are widely used in
backend systems today and port this to NETCOBOL or Fujitsu 6.0 later
with updated GUI's that have some merit. However NETCOBOL is including
MS Visual Basic NET free, which makes the next 5-year GUI lifespan of
PowerCOBOL murky at least.  If you do find an NT/95 OS note that
PowerCOBOL 4.0 versions and up syntax has changed dramatically.
 Mike
