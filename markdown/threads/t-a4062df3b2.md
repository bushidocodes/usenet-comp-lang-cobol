Judy,

Think you've done a pretty good job on your homework, but you've
confused yourself putting too much on the screen at the same time. I
tried hacking your program around, but I think you will find the
following pseudo code more useful to get what you want. This is NOT the
only way to do it, and others could give you a variation on a theme -
but it should guide you in the right direction :-

- (1) Display a Customer Screen -A/c #, Name, address etc.
- Accept Customer # (Pic x or 9)
- if Customer # = spaces/zeroes - then quit
- if not spaces/zeroes
  set CustomerFound to true
  Read CustomerFile using Customer #
    invalid key set RecordNotFound to true

NOTE : You DO NOT allow user to change an Account # once entered.
       (i.e. the file PrimeKey )

NEW RECORD : CustomerNotFound -

- Prompt user to enter fields systematically
- then display additional line "change" "OK Update" or "next"
- (2) Change - let user selectively change fields EXCEPT the Account #
- OK to update -  do validity checks :-

	if errors - highlight errors - takes you back to (2)
	if not errors WRITE record go back to (1)

- Next - ignore and go back to (1)

EXISTING RECORD : CustomerFound

-  set RecordNotChanged to true
-  display additional line "change", "delete", "OK" or "next"

- (3) if change - let user select fields to change - and as field  
selected set RecordChanged to true - Again don't let user changed
Account #
  
- if OK AND RecordChanged - do validity checks

	if errors - highlight errors - takes you back to (3)
	if not errors REWRITE record go back to (1) 

- if OK AND RecordNotChanged - go back to (1)

- if delete prompt - "OK to delete this record ?"
     - if delete 'Y' DELETE record go back to (1)
     - if delete 'N' go back to (1)

- Next - ignore and go back to (1)

As and when you get into Windows the same pseudo logic applies.

PS: In case you are not clear on the "set" business :-

 01 RecordFlag		pic 9.		These "flags" can be
    88 RecordFound      value 1.	numeric or alphabetic
    88 RecordNotFound   value 0.

 01 ChangeFlag		pic 9.
    88 RecordChanged	value 1.
    88 RecordNotChanged value 0.

......................

Hope you aren't confused <G>.

Judy:

COBOL sp2 uses ANSI standard COBOL CALL USING statements and COBOL
WORKING STORAGE variables to control the user interface.

I don't really consider it to be another platform and it does support
RM COBOL very well, but I do see a few possible problems.  I suspect
that your instructor *wants* you to use DISPLAY/ACCEPTS or screen
section for your user interface screens.

This would certainly prevent you from using sp2.

You might consider getting one or more of the introductory COBOL books
to help you with the user interface.  There are several, including:

Teach Yourself COBOL in 24 Hours by Thane Hubbell...and
Teach Yourself COBOL in 21 Days by Mo Budlong.

I believe if you access Amazon.com, you'll be able to find both of
these books.

Good luck!
