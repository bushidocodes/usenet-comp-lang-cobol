I understand that SAP can use a range of databases one of which is
SAP-DB which was originally Adabas-D and is now open sourced as MaxDB
and is under the wing of MySQL. It seems that SAP can use Oracle, DB2,
MS SQLServer, SAP-DB and possibly others.

The original Adabas of the 1970s was an inverted file system, but this
was replaced in the 1980s by a different system called Adabas-D which
had been developed at Berlin Tech and supported by Siemanns for a while
until Software AG took it over to replace their current Adabas. This
was later  over by SAP when Software AG decided it no longer wanted to
be in that business.

While the current version of Peoplesoft has COBOL in it, the COBOL in
it is not very accessible to customers of Peoplsoft. You can write
external COBOL routines to access the data as the data resides in a
RDBMs. Peoplesoft is written in C and some COBOL and uses SQL.
Peoplesoft allows end users or customizers to use a language that
Peoplesoft presents, so you would'nt be leveraging very much COBOL
code. As Peoplesoft was purchased by Oracle, Oracle has undertaken
rewritting most of the Peoplesoft code into Java. I believe that the
conversion project is slated for 2007. We'll have to wait and see if
the new edition keeps some legacy or if it is comprised of solely Java
or some mixture of languages
