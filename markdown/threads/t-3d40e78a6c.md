Fred:

Our COBOL sp2 GUI product has supported the ability to develop and
deploy Windows GUI screens for a COBOL application since Windows 3.0
in 1990.

We support all versions of Windows and our solution is compiler
independent.  That means tyou could develop a GUI for Fujitsu and run
the same screen handling code with NetExpress, or mbp Visla COBOL, or
CA-Realia COBOL, or ICOBOL, or IBM Visual Age COBOL, or Ryan McFarland
COBOL or Acucobol.

We support sophisticated screen controls such as treeview controls,
tab controls, embedded Active X controls and a host of other advanced
features.  We also allow you to run the same GUI screen definitions
and same source program as an Internet Thin Client application.  No
need to change the code.

We not only have Win32 GUI solutions, we also have web browser based
interface solutions and Windows Printing solutions as well.

Please send your questions via private e-mail.  I'll be happy to
assist you as well as provide you with an evaluation version for you
to try.




Bob Wolfe
