This is the monthly "Fujitsu is not ANSI CONFORMING for sort-files" message.
To get your program to compile with Fujitsu (and it should compile
successfully with everyone else's compiler already), change

   SELECT I ASSIGN TO "sortfile".
        to
    SELECT I ASSIGN TO sortfile.

From the JMN in the error message, one can infer that you are using
Fujitsu on a PC.
Now, the ASSIGN clause for SORT-MERGE files are 'implementor-
defined' which basically means that every compiler has its own ideas
about this, which is why you have a problem.
For the Fujitsu compiler, the rule is that the name must be a character
string not exceeding eight chars consisting of only alphabetic chars
and numeric chars with the first being alphabetic (this is bit imprecise
since alphabetic includes spaces and I don't think you can have spaces
here). To add injury to insult, the manual also states that "An ASSIGN
clause written for as sort-merge file is regarded as a comment", but
the compiler bitches nevertheless!!!.
So what's the real problem? It is that you specified quotes in the
ASSIGN clause:  "sortfile" and that sortfile was lower case, while
alphabetic chars must be upper case, so, change to SORTFILE
and the compiler is happy.

Welcome to the sad, sad world of "implementor" names!!!!.
The next Cobol standard will (sorry to say) have many more
"implementor defined" features.
