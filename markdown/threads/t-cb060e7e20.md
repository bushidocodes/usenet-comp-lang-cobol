How was the indexed file created?  This program is opening it as INPUT - so
something must have created it already.  If you tried to create it in any
way OTHER than from another COBOL program, then you probably will have
problems opening it (because the internal structure won't be correct).  Try
writing a "simple" program to read a sequential file in - and write an
indexed file out.  Then try running your program that reads it in and see if
that works.

This is how I did it from another program:

 IDENTIFICATION DIVISION.
 PROGRAM-ID. SECTIONS.
 ENVIRONMENT DIVISION.
 INPUT-OUTPUT SECTION.
 FILE-CONTROL.
     SELECT SECTIONS-INPUT-FILE ASSIGN TO 'SECTIONS.DAT'
        ORGANIZATION IS LINE SEQUENTIAL.
     SELECT SECTIONS-INDX-FILE ASSIGN TO 'INDXSEC.DAT'
        ORGANIZATION IS INDEXED
        ACCESS IS SEQUENTIAL
        RECORD KEY IS INDX-SECTION-NUM
        ALTERNATE RECORD KEY IS INDX-COURSE-NUM WITH DUPLICATES.
*
 DATA DIVISION.
 FILE SECTION.
 FD SECTIONS-INPUT-FILE     LABEL RECORDS ARE STANDARD.
 01 SECTIONS-INPUT-REC.
    05 INPUT-SECTION-NUM            PIC X(4).
    05 INPUT-COURSE-NUM             PIC X(6).
 FD SECTIONS-INDX-FILE      LABEL RECORDS ARE STANDARD.
 01 SECTIONS-INDX-REC.
    05 INDX-SECTION-NUM             PIC X(4).
    05 INDX-COURSE-NUM              PIC X(6).
    05 INDX-NUM-STUDENTS            PIC 99.
 WORKING-STORAGE SECTION.
 01 WS-WORK-AREA.
    05 MORE-RECORDS                 PIC X VALUE 'Y'.
       88 NO-MORE-RECORDS                 VALUE 'N'.
*
 PROCEDURE DIVISION.
****************************************************************
* This is the main procedure.                                  *
* It opens the files and reads the input file.                 *
****************************************************************
 100-MAIN.
     OPEN INPUT SECTIONS-INPUT-FILE
          OUTPUT SECTIONS-INDX-FILE
     PERFORM UNTIL NO-MORE-RECORDS
         READ SECTIONS-INPUT-FILE
             AT END
                 MOVE 'N' TO MORE-RECORDS
             NOT AT END
                 PERFORM 200-CREATE-RTN
         END-READ
     END-PERFORM
     PERFORM 300-END-OF-JOB-RTN
     STOP RUN.
****************************************************************
* This procedure writes to the indexed file and                *
* checks for invalid data.                                     *
****************************************************************
 200-CREATE-RTN.
     MOVE INPUT-SECTION-NUM TO INDX-SECTION-NUM
     MOVE INPUT-COURSE-NUM TO INDX-COURSE-NUM
     MOVE 00 TO INDX-NUM-STUDENTS
     WRITE SECTIONS-INDX-REC
         INVALID KEY DISPLAY 'INVALID RECORD'
     END-WRITE.
****************************************************************
* Closes files.                                                *
****************************************************************
 300-END-OF-JOB-RTN.
     CLOSE SECTIONS-INPUT-FILE
           SECTIONS-INDX-FILE.

I think you defined an alternate key when you defined (created as OUTPUT)
your file, but not when you  read (OPEN INPUT) the file in the other
program.  Some systems allow this while others do not.  Try making certain
that both your FD and your SELECT/ASSIGN statements are identical in the
program that creates the file and the one that reads it.
