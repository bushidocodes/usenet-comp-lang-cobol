You can definitely mix VB and COBOL.... and in my opinion the way you
mention is the best method.  VB can dynamically invoke the DLLs you create
in COBOL.  Under "Project Build Settings, select the Link tab, under
Category select Advanced and put CBLRTSS.LIB under "Link with these OBJs:"

If you do not statically link them you will need the runtime files to be
present (CBLRTSS.DLL at least) or the DLL will not work.  I'm not familiar
with the university edition so i'm not sure if those files come with it or
not.......
