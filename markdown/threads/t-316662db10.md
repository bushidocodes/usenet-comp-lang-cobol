```
Hi Bjorn,

I speak as an IBM MVS Cobol user when I say that end-eval is not required to
end an eval stmt. I'm not sure of other Cobol implementations. 

In addition, MVS Cobol will recognize only the first 8 characters of a pgm name
used in a CALL stmt. I noticed that at least one of your CALLs uses a 9
character module name. If your version of Cobol truncates the pgm name to 8
chars, and your system carries multiple modules that begin with those 8 chars,
this may be the cause of the error and accompanying message.

Regards, Jack.
```

```
Others have talked about adding an END-EVALUATE statement,  I don't think
that is the problem - as it is optional and this EVALUATE ends with a period
(full-stop).

However, I am a little confused by exactly what the reported problem is.
The original note says,

 "but at the second time i always get the error-message "119 Name is not
unique".

a message that says "name is not unique" sounds like a COMPILE-TIME message.
However, talking about "the second time" sounds like a run-time message.

Is this a compile-time message?  If so, can you show us the code showing
"both" the 1st and 2nd "time" you "use" this?

P.S.  What compiler and run-time environment is showing this problem? This
might help us figure out the real problem. I know that this is NOT MVS (or
any IBM mainframe) because you use ACCEPT .. AT, so whoever answered with
that "assumed" must be mistaken.

P.P.S.  If you have a compiler with intrinsic functions, are you aware that
you can change:

            evaluate fun
              when "N"
              when "n" call "neuzugang"

to

     Evaluate Function Upper-Case (fun)
         When "N"
             call "neuzugang"
 ...
```
