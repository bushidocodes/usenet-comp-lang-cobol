Here's another alternative that I use...

I installed MKS Toolkit, an earlier version, namely 5.1, on my
desktop, then copied over the binaries and the man pages to a cd-rom.
It was on the order of 16MB. (Yes, once again, like Baby Linux, that
is 16MB, not 16GB :-) )

I wrote a tiny .bat file based upon the MKS install that sets a
handfull of environment variables, like PATH, to point the binaries
and the /etc man pages, then invoked the shell (sh).

For those who prefer the command line and $ prompt this works quite
well. Because I execute the binaries off the cd-rom, I never actually
"installed" the software on the client machine. This is a useful
workaround when dealing with corporate bureaucracy.

Sadly, it breaks under Win 7, unless you invoke XP compatability mode
(which I was unable to do). However, it works great under Windows XP.
I would have used a memory stick (I did for a while) but I ran across
the corporate policy that memory sticks were verboten. Mr. Executive
VP was afraid we would download valuable organizational data. However,
the CD-ROM drives were READ-ONLY, which is all you need to invoked the
binaries. I made sure that my "working directory" was on the c: drive
where the utilities would deposit their output results, in the form of
text files. Again, which did not constitute an "Install", since they
were only text files.

I did extensive source code analysis on millions of lines-of-code
using such a tactic.

Ken
