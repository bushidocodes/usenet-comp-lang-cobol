The ICL 1900 series did not have a stack either. It used self-
modifying code for a PERFORM putting the return branch in the 'EXIT'
statement (or the implied EXIT statement) and branching to the start
point.

In other words the PERFORM with XE13, XE20, and XEKB worked like an
ALTER and a GO TO but actually did it with less instructions and
cleaned up afterwards, putting a null op in the 'EXIT' so that it
would drop through when not under the control of a PERFORM as per the
language spec. This is why it is a 'high-level' language, it explains
what is to happen and hides _how_ it happens.

Of course on stack machines the PERFORM can work differently. ALTER
can too, especially where self-modifying code can't work. This then
could be less efficient than the PERFORM.
