Minor nit, this is true only if you are using the NODYNAM compile option
(required for CICS pgms). If you're using DYNAM (recommended for batch)
COBOL will generate a dynamic CALL regardless of the format of the CALL,
i.e., the called module will not be a part of the calling program's load
module. CALLing a data name forces a dynamic call even if you're using
the NODYNAM option.
