Sorry Peter, I haven't seen your original post yet, so I couldn't CC you
in email.

Anyway, the problem is that the entry point "Test1" is included in the
DLL
in all upper-case (the command "link /dump /exports focall.dll" will
show you
this). To prevent this, ensure that you compile programs with the CASE
directive (eg, use a "$SET CASE" directive at the top of the program).

Micro Focus COBOL on Windows is case-insensitive by default (as it
provides
consistency when programs are searched for on disk). However, the
GetProcAddress call is case sensitive.

Cheers,
Kev.
