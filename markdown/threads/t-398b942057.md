We did that once before and, while the code is no longer available,
here's how we did it.

1. Run the Realia stand-alone utility that displays the ISAM file
header. Print the screen.

2. Use a hex editor to prowl around in the first record of the ISAM
file. Find the bytes that contain the values from the printout in
which you are interested (i.e., record length). Note the displacement
of the bytes of interest from the beginning of the record.

3. Define the ISAM file as [B] with a 2K record. Open as input, read
the first record. Close the file. This gets you the ISAM header
record.

Use the discovered displacements - such as for the bytes containing
the record size - in your production program.

the necessary file handle.

WIN32_FIND_DATA structure when you use FindFirstFile() or FindNextFile(). If
the file is larger than 4GB then you need to use both nFileSizeLow and
nFileSizeHigh. I ain't never called a Win32 API from Realia though...

Nice idea, but the size of the file in bytes is only a vague
approximation of the number of records in the file. 

As I understand it, the Realia file system allocates blocks that
contain both indexes and data space, and these may of course be empty.


What you REALLY need is to obtain the number of records in a given
keypath, and the percentage along the keypath of the current key.
Btrieve (and other data managres) provide this, but it takes some
hocus-pocus to obtain it from ISAM.  

About the only way you can get anything meaningful about your
CA-Realia ISAM file is to ask them how it's done.

Or, convert to Btrieve or some other similar package...

There is no API available to determine the number of records in a
Realia indexed file. this is true for both the 16 bit and 32 bit
versions. There is no method of reading a specific location in the file
to determine this value. The only way to determine the number of
records in a Realia indexed file is to sequentially read the file and
count the records.

When you use the REALIF utility, you can redirect the output into a
file and read it. Realif says a lot about indexed files without
completely reading it. It reads the header of the file, which contains
all the information.

The format is:

REALIF filename.ext > realif.out

The file realif.out will contain the info of filename.ext.
