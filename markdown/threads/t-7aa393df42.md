tur··.@pac··m.org (Lee Winson) wrote:
› 
› In IBM Mainframe COBOL (VS/COBOL or COBOL II), is it possible to obtain
› the dataset (DSN) name, volume, DCB and other information from the
› operating system from the program?  I'm not sure of any way to do that
› without using an assembler subroutine.

You have to use asm. But it's relatively trivial to do.

* KPMG R*STARS DSN retrieval
*
* Copyright (C) 1994 KPMG Peat Marwick LLP
* All rights reserved
*
* This routine retrieves the DSN for a ddname
* (It also gets stuff like JOBNAME, PROC, STEP
* -- if that's all you want, do a retrieval on STEPLIB)
*
DAFMDSN CSECT
DAFMDSN RMODE ANY
DAFMDSN AMODE 31
USING *,15
SAVE (14,12),,DAFMDSN*COPYRIGHT*KPMG*1994
BALR 12,0
BEGIN DS 0H
USING BEGIN,12
*
* Process parm list (copy dd to our work areas)
*
L 10,0(,1) R1 -> PARM LIST
USING PARM,10
SPACE 5
*
* Init return area
*
MVC PARMJOB,=CL44' '
MVC PARMPROC,=CL44' '
MVC PARMSTEP,=CL44' '
MVC PARMDSN,=CL44' '
MVC PARMMEM,=CL44' '
MVC PARMBLKS,=H'-1'
MVC PARMLRCL,=H'-1'
MVC PARMVOLS,=H'-1'
MVC PARMLDS,=H'-1'
MVC PARMORG,=CL44' '
MVC PARMRFM,=CL44' '
MVC PARMLBL,=CL44' '
MVC PARMDISP,=CL44' '
XC PARMJFCB,PARMJFCB
SPACE 5
*
* Get address of TIOT...
*
* Note that there are three ways to get this
* EXTRACT, GETDSAB and chasing the control blocks
* from PSA... All three have some restrictions and
* limiations and might fail...
*
EXTRACT EXTRDATA,FIELDS=(TIOT)
L 9,EXTRDATA
USING TIOT,9
MVC PARMTIOT,0(9) Copy TIOT (first part) to user..
SPACE 5
*
* From TIOT, grab Job, proc and step names
*
MVC PARMJOB,TIOCNJOB MOVE JOBNAME
CLI TIOCSTEP+8,C' ' Check if 2nd part is blank
BE STEPONLY so we have only a stepname
MVC PARMPROC,TIOCSTEP Otherwise we have both proc
MVC PARMSTEP,TIOCSTEP+8 step
B COMMON
SPACE 2
STEPONLY MVC PARMSTEP,TIOCSTEP Move step name
SPACE 5
*
* Point at the DD block in TIOT - We adjust base register value
* and tell the asm what it now points at...
*
COMMON LA 9,TIOELNGH POINT TO FIRST ENTRY...
USING TIOELNGH,9 UPDATE BASE REGISTER USING
XR 8,8 CLEAR R8 FOR LENGTH USE
SPACE 5
*
* Loop through each DD entry looking for a match... zero length
* means we missed it...
*
DDLOOP EQU * LOOP THROUGH EACH DD...
CLC TIOEDDNM,PARMDD RIGHT ONE?
BE FOUND
IC 8,TIOELNGH GRAB ENTRY LENGTH
LTR 8,8
BZ ENDRC8 NOT FOUND
LA 9,0(8,9) SPIN R9 FORWARD SO POINTS AT NEXT LEN
B DDLOOP
SPACE 5
*
* We've found it - so process...
*
FOUND XR 7,7 Clear R7
ICM 7,B'0111',TIOEJFCB Load in address of JFCB
LA 7,16(,7) Offset FROM EXPERIENCE
SPACE 2
USING INFMJFCB,7
SPACE 5
*
* Move data from JFCB...
*
MVC PARMDSN,JFCBDSNM DSN=
MVC PARMMEM,JFCBELNM (xxxxx)
SPACE 5
*
* Process RECFM information
*
MVC WORKBYTE,JFCRECFM Copy flag byte
NI WORKBYTE,JFCFMREC Strip off other bits
CLI WORKBYTE,JFCUND Test if U?
BNE TESTFV No - check F/V
MVC PARMRFM,=CL3'U' Set U
B ENDRECFM and we're done
SPACE 2
TESTFV CLI WORKBYTE,JFCFIX Test F
BE RECFMF
MVC PARMRFM,=CL3'V' Or it's V
B TESTB
RECFMF MVC PARMRFM,=CL3'F' Move the F
SPACE 2
TESTB TM JFCRECFM,JFCRFB Common check for Blocked
BNO TESTA
MVI PARMRFM+1,C'B'
SPACE 2
TESTA TM JFCRECFM,JFCASA Common check for ASA
BNO ENDRECFM
MVI PARMRFM+2,C'A'
ENDRECFM EQU *
SPACE 5
*
* Process DSORG
*
* We move BLKSIZE/LRECL here if NOT VSAM...
*
CLI JFCDSRG2,JFCORGAM Test if VSAM
BE ISVS
SPACE 2
* Non-VSAM, so copy BLKSIZE/LRECL
MVC PARMBLKS,JFCBLKSI BLKSIZE=
MVC PARMLRCL,JFCLRECL LRECL=
SPACE 2
CLI JFCDSORG,JFCORGPO Test if PDS
BE ISPO
CLI JFCDSORG,JFCORGPS Test if PS
BNE ENDDSORG
SPACE 2
ISPS MVC PARMORG,=CL2'PS'
B ENDDSORG
SPACE 2
ISVS MVC PARMORG,=CL2'VS'
B ENDDSORG
SPACE 2
ISPO MVC PARMORG,=CL2'PO'
SPACE 2
ENDDSORG EQU *
SPACE 5
*
* Process Label information...
* (Note that DASD is SL)
*
MVC PARMLDS,JFCBFLSQ Move label sequence number
SPACE 2
TM JFCBLTYP,JFCSL Check if SL?
BE ISSL
TM JFCBLTYP,JFCNL Or NL?
BE ISNL
TM JFCBLTYP,JFCBLP Or BLP?
BNE ENDDSLBL otherwise, leave field blank
SPACE 2
ISBLP MVC PARMLBL,=CL3'BLP'
B ENDDSLBL
SPACE 2
ISSL MVC PARMLBL,=CL3'SL'
B ENDDSLBL
SPACE 2
ISNL MVC PARMLBL,=CL3'NL'
SPACE 2
ENDDSLBL EQU *
SPACE 5
*
* Process DISP... note we only get the 1st parameter
*
SPACE 5
MVC WORKBYTE,JFCBIND2 Move flag byte to work
NI WORKBYTE,JFCDISP Mask off irrelevant bits
CLI WORKBYTE,JFCNEW Test if NEW
BE ISNEW
CLI WORKBYTE,JFCMOD Or MOD?
BE ISMOD
CLI WORKBYTE,JFCOLD Or OLD?
BNE ENDDISP None... leave it blank...
SPACE 2
MVC PARMDISP,=CL3'OLD' Set OLD
TM JFCBIND2,JFCSHARE But check for SHR
BNO ENDDISP Not then we're ok
SPACE 2
MVC PARMDISP,=CL3'SHR' Set SHR
B ENDDISP
SPACE 2
ISNEW MVC PARMDISP,=CL3'NEW' Set NEW
B ENDDISP
SPACE 2
ISMOD MVC PARMDISP,=CL3'MOD' Set MOD
SPACE 2
ENDDISP EQU *
SPACE 5
*
* Move # of volumes (1 byte) to half word...
*
XC PARMVOLS(1),PARMVOLS Zero high byte
MVC PARMVOLS+1(1),JFCBNVOL Move low...
SPACE 5
*
* Just because, copy all of the JFCB to the user's return area...
*
MVC PARMJFCB,INFMJFCB
SPACE 5
*
* Zero return code and back to caller...
*
XR 15,15
B GOBACK
SPACE 5
*
* RC=8 -- error!
*
ENDRC8 LA 15,8 SET RC=8, NOT FOUND
SPACE 5
**
GOBACK RETURN (14,12),,RC=(15)
SPACE 5
**
** Work areas
**
EXTRDATA DS F
WORKBYTE DC AL1(0)
SPACE 5
**
** Literal pool
**
LTORG
EJECT
**
** Mapping Macros
**
SPACE 5
**
** T I O T
**
TIOT DSECT
IEFTIOT1
SPACE 5
**
** J F C B
**
JFCB DSECT
IEFJFCBN LIST=YES
SPACE 5
**
** Our parameter mapping area
**
PARM DSECT
* INPUT DATA:
PARMDD DS CL8
* RETURN DATA:
PARMJOB DS CL8 8 BYTE JOBNAME
PARMPROC DS CL8 8 BYTE PROC NAME
PARMSTEP DS CL8 8 BYTE STEP NAME
PARMDSN DS CL44 44 BYTE DSNAME
PARMMEM DS CL8 8 BYTE MEMBER
PARMBLKS DS H S9(4) COMP BLKSIZE
PARMLRCL DS H S9(4) COMP LRECL
PARMVOLS DS H S9(4) COMP VOLUMES
PARMLDS DS H S9(4) COMP LABEL DS #
PARMORG DS CL2 DSORG=VS PS PO
PARMRFM DS CL3 3 BYTE RECORD FORMAT
PARMLBL DS CL3 3 BYTE LABEL (SL, NL, BLP)
PARMDISP DS CL3
PARMJFCB DS CL176
PARMTIOT DS CL256
SPACE 5
END

There are lots of other fields you can retrieve, you just need to
look at the control block layout macros and pick 'em

Enjoy!

-----Burton
