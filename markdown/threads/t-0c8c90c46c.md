```
Dennis Komeshak wrote:
>In converting to Fujitsu COBOL from MicroFocus, I use the x"f4" (pack data
>into a byte) and x"f5" (unpack data into a byte) routines. Can anyone

Below is a solution I posted to a similar problem several days ago.

Here is a general approach which will usually work, though you may need
to adapt it for your specific compiler.  Basically, you define a PIC X
field and redefine it as a binary numeric type, or vice-versa, then you
reference the binary numeric type as a numeric value, using add, subtract,
etc. to do the work.  The following Micro Focus program contains a routine
which takes a byte and splits each bit into a separate PIC 9 field, and
another routine which merges the separate fields into one byte.  Once the
bits are separated, you can easily test them individually.

       IDENTIFICATION DIVISION.
       ENVIRONMENT DIVISION.
       DATA DIVISION.
       WORKING-STORAGE SECTION.
       77  WS-CHAR             PIC  X(01).
       77  WS-VALUE            PIC  9(03).
       01  BINARY-WORK.
           03  BW-BINARY       PIC S9(04)  BINARY  VALUE 0.
           03  BW-ALPHA        REDEFINES BW-BINARY.
               05  FILLER          PIC  X(01).
               05  BW-CHAR         PIC  X(01).
           03  BW-BITS.
               05  BW-BIT128   PIC  9(01).
               05  BW-BIT64    PIC  9(01).
               05  BW-BIT32    PIC  9(01).
               05  BW-BIT16    PIC  9(01).
               05  BW-BIT8     PIC  9(01).
               05  BW-BIT4     PIC  9(01).
               05  BW-BIT2     PIC  9(01).
               05  BW-BIT1     PIC  9(01).
           03  BW-BIT          REDEFINES BW-BITS
                               OCCURS 8 TIMES
                               INDEXED BY BW-X
                                   PIC  9(01).
       PROCEDURE DIVISION.
       TEST-IT.
           MOVE "I" TO BW-CHAR.
           MOVE BW-CHAR   TO WS-CHAR.
           MOVE BW-BINARY TO WS-VALUE.
           PERFORM SPLIT-BYTE.
           DISPLAY WS-CHAR " "
                   BW-BIT128 BW-BIT64 BW-BIT32 BW-BIT16
                   BW-BIT8   BW-BIT4  BW-BIT2  BW-BIT1
                   " " WS-VALUE.

           MOVE 1 TO BW-BIT16.
           MOVE 0 TO BW-BIT1.

           PERFORM MERGE-BYTE.
           MOVE BW-CHAR   TO WS-CHAR.
           MOVE BW-BINARY TO WS-VALUE.
           DISPLAY WS-CHAR " "
                   BW-BIT128 BW-BIT64 BW-BIT32 BW-BIT16
                   BW-BIT8   BW-BIT4  BW-BIT2  BW-BIT1
                   " " WS-VALUE.
           STOP RUN.

       SPLIT-BYTE.
           PERFORM VARYING BW-X FROM 8 BY -1
                   UNTIL BW-X < 1
               DIVIDE 2 INTO BW-BINARY
                   GIVING BW-BINARY
                   REMAINDER BW-BIT(BW-X)
           END-PERFORM.

       MERGE-BYTE.
           MOVE 0 TO BW-BINARY.
           PERFORM VARYING BW-X FROM 1 BY 1
                   UNTIL BW-X > 8
               COMPUTE BW-BINARY = BW-BINARY * 2 + BW-BIT(BW-X)
           END-PERFORM.
```

```
The BITPK program from ETKPAK at http://www.etk.com/download does this:

Summary: Provides bit manipulations of up to 32 bits. Can get, set, clear,
test, and pack bits.

How to use: Instead of non-portable redefinitions to access bits packed in
numeric variables, use BITPK's get operation to convert a number into a
string of 32 characters each with values "0" or "1". You can then easily
test or otherwise manipulate these character values. The pack operation
packs the character values back into a number.

Limitations: At most 32 bits can be handled at a time.

To call BITPK, define BITPK-CONTROL as follows:

              01  BITPK-CONTROL.
                  02  BITPK-TASK-ID           PIC X(4) .
                  02  BITPK-OPERATION         PIC X(1) .
                  02  BITPK-FEEDBACK          PIC X(1) .
                  02  BITPK-NUMBER            PIC 9(10).
                  02  BITPK-BITS.
                      03  BITPK-BIT           PIC X(1)  OCCURS 32 TIMES.

Define CALL-BITPK as follows:
              CALL-BITPK.
                  CALL "BITPK"
                        USING BITPK-CONTROL
              .
BITPK supports the following operations:

G (get)
Get the bits of BITPK-NUMBER interpreted as an unsigned 32-bit number. If
the number is larger than 32 bits a (R)ANGE feedback is returned and the bit
string is all zeroes.

P (pack)
The reverse operation of get. Builds the 32-bit number having the bit string
given in BITPK-BITS.

S (set)
For each bit of the bit string given that is a one, set the corresponding
bit in BITPK-NUMBER. This is the binary OR operation.

A (and)
For each bit of the bit string given that is a one, set the corresponding
bit in BITPK-NUMBER if that bit is also a one, otherwise clear the bit. This
is the binary AND operation.

X (xor)
For each bit of the bit string given that is different from the
corresponding bit in BITPK-NUMBER, set that bit in BITPK-NUMBER, otherwise
clear the bit. This is the binary XOR operation.

C (clear)
For each bit of the bit string given that is a one, clear the corresponding
bit in BITPK-NUMBER.

1 (test for 1)
For each bit of the bit string given that is a one, test if the
corresponding bit in BITPK-NUMBER is a one. Return an (A)LL feedback if all
bits tested were ones, a (N)ONE feedback if none of the bits were ones, or a
(S)OME feedback if only some of them were.

0 (test for 0)
For each bit of the bit string given that is a one, test if the
corresponding bit in BITPK-NUMBER is a zero. Return an (A)LL feedback if all
bits tested were zeroes, a (N)ONE feedback if none of the bits were zeroes,
or a (S)OME feedback if only some of them were.
BITPK returns these feedbacks: a space means there were no errors. An "O"
means that the specified operation was invalid. An "R" means that the number
given was too large. An "A", "N", or "S" means that either All, None, or
Some bits met the condition.
```

```
I thought I would post code which uses called subprograms since this
more closely matches the "F4" and "F5". The differences are that
neither byte nor array are comp-x.

MF:  byte pic x comp-x
        array pic x comp-x occurs 8
becomes: byte pic x.
       array pic 9 occurs 8

The calls are are to "b-pack" and "b-unpack".

       program-id. b-test.
       data division.
       working-storage section.
       01 ws.
         03 ws-text.
           05 ws-char pic x.
           05 ws-position pic 9.
       01 ws-bit-value pic x value space.
       01 ws-byte pic x.
       01 ws-array.
         03 ws-bits pic 9 occurs 8.
       procedure division.
       b-test-start.
           display "Enter an action and a bit position."
           display "For example, 's1' to set bit 1"
           display "or 'c5' to clear bit 5."
           display "Enter ?4 to get the value of bit 4."
           display "Bits are numbered 1 through 8."
           display "Enter q to quit."
           move low-values to ws-byte
           perform display-bit-pattern
           perform with test after
             until ws-char = "Q"
               accept ws-text
               if ws-text ( 2 : 1 ) not numeric
                   move "0" to ws-text ( 2 : 1 )
               end-if
               move space to ws-bit-value
               move function upper-case ( ws-char ) to ws-char
               evaluate ws-char
                 when "?"
                   call "b-what" using ws-byte, ws-position,
                       ws-bit-value
                 when "S"
                   call "b-set" using ws-byte, ws-position
                 when "C"
                   call "b-clear" using ws-byte, ws-position
               end-evaluate
               perform display-bit-pattern
           end-perform
           stop run
           .

       display-bit-pattern.
           call "b-unpack" using ws-byte, ws-array
           display ws-bits(1) ws-bits(2) ws-bits(3) ws-bits(4)
               ws-bits(5) ws-bits(6) ws-bits(7) ws-bits(8)
               space quote ws-byte quote space ws-bit-value
           .
       end program b-test.

       program-id. b-what.
       data division.
       working-storage section.
       01 ws-array.
         03 ws-bits pic 9 occurs 8.
       linkage section.
       01 ls-byte pic x.
       01 ls-position pic 9.
       01 ls-bit-value pic x.
       procedure division using ls-byte, ls-position,
           ls-bit-value.
       b-what-start.
           if ls-position > 0 and < 9
               call "b-unpack" using ls-byte, ws-array
               move ws-bits ( ls-position ) to ls-bit-value
           end-if
           exit program
           .
       end program b-what.

       program-id. b-set.
       data division.
       working-storage section.
       01 ws-array.
         03 ws-bits pic 9 occurs 8.
       linkage section.
       01 ls-byte pic x.
       01 ls-position pic 9.
       procedure division using ls-byte, ls-position.
       b-set-start.
           if ls-position > 0 and < 9
               call "b-unpack" using ls-byte, ws-array
               move 1 to ws-bits ( ls-position )
               call "b-pack" using ls-byte, ws-array
           end-if
           exit program
           .
       end program b-set.

       program-id. b-clear.
       data division.
       working-storage section.
       01 ws-array.
         03 ws-bits pic 9 occurs 8.
       linkage section.
       01 ls-byte pic x.
       01 ls-position pic 9.
       procedure division using ls-byte, ls-position.
       b-clear-start.
           if ls-position > 0 and < 9
               call "b-unpack" using ls-byte, ws-array
               move 0 to ws-bits ( ls-position )
               call "b-pack" using ls-byte, ws-array
           end-if
           exit program
           .
       end program b-clear.

       program-id. b-unpack.
       data division.
       working-storage section.
       01 ws.
         03 ws-b-value pic s9(4) binary.
         03 ws-x pic s9(4) binary.
       linkage section.
       01 ls-byte pic x.
       01 ls-array.
         03 ls-array-bit pic 9 occurs 8.
       procedure division using ls-byte, ls-array.
       b-unpack-start.
           compute ws-b-value = function ord ( ls-byte ) - 1
           move all "0" to ls-array
           perform varying ws-x from 1 by 1 until ws-b-value = 0
               if ws-b-value >= 128
                   move 1 to ls-array-bit ( ws-x )
                   subtract 128 from ws-b-value
               end-if
               multiply 2 by ws-b-value
           end-perform
           exit program
           .
       end program b-unpack.

       program-id. b-pack.
       data division.
       working-storage section.
       01 ws.
         03 ws-b-value pic s9(4) binary.
         03 ws-x pic s9(4) binary.
       linkage section.
       01 ls-byte pic x.
       01 ls-array.
         03 ls-array-bit pic 9 occurs 8.
       procedure division using ls-byte, ls-array.
       b-pack-start.
           move 0 to ws-b-value
           perform varying ws-x from 1 by 1 until ws-x > 8
               multiply 2 by ws-b-value
               add ls-array-bit ( ws-x ) to ws-b-value
           end-perform
           move function char ( ws-b-value + 1 ) to ls-byte
           exit program
           .
       end program b-pack.
```

```
of is to have a table of 256  8-bit textstrings indexed by the byte
value (+1 of course).
```
