I'm not sure where the 'direct access' comes in since that is a way of reading
files.  For reading tables in working-storage, the table needs to be defined
OCCURS xx TIMES INDEXED BY index-name.  In referencing a specific item in the
table you would use 'index-name' the same way you would a subscript.  In the
procedure division, initialize your index with a SET statement before a SEARCH.
That wouldn't be necessary in a SEARCH ALL statement (binary search), but you'll
have to tell the compiler how the table is sorted (KEY).

HTH
