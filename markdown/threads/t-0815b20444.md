   If SeedNum Zero

I did post another reply related to the "solution" to my problem.

1)  It turns out that Fujitsu (at least at V4.0) takes

   01  RandomNum   Pic 9(03).
   Compute RandomNum = Function Integer ((Function Random) * 1000)

and for some (unpleasant - but legal) reason figures that you only want ONE
significant digit from RANDOM *before* multiplying it by 1000.  This meant
that the MOD that I did was  always against a number divisible by 1000 -
which gave a VERY limited selection.

2)  I still need to rework my code to get rid of the "slight" bias that
occurs when you mod a COBOL decimal field to get a random value.  For
example, if you  do

  01  BigField   Pic 9.
  Compute NumField = Function Mod (BigField 4)

if Big field can have any of the values 0 thru 9 (in an even distribution),
the result is SLIGHTHLY biased to 0, 1, 2 - and against 3 (The first 3 can
occur 3 times, while the last can only occur 2 times.)

I have figured out the code to "solve" this - but still need to implement
it.
