Note that this is definitely one time that VM (and even PC's and Unix) are
MUCH more user friendly than MVS or OS/390.

Under VM, if you don't have the CBLQDA option set to "OFF" (so you ARE
running in ANSI conforming mode), it creates a file that *IS* easy to find
and use after your program does a "stop run" or "goback".  Same for Windows
and Unix.

I have been telling IBM for *years* that they *could* have implemented the
ANSI conforming behavior in a USEFUL manner - by simply creating a
*cataloged* dataset with
  DSN = program-name.filename
