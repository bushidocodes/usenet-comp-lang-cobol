```
We have received some cobol programs for witch we should do Y2000
checks. These programs use routines of a library (also made with cobol)
and call some operating system API:s. The problem comes with debugging
(or animating as they call it at Micro Focus). The animator in Micro
Focus Object Cobol 4.0 seems to always want to have a source code for
the routine available. With the library routines we have that but for
the operating system code obviously not. Program id in the library
routines is not the same as file name witch seems to be a problem for
the animator too.

Our environment is OS/2 Warp and Micro Focus Object Cobol 4.0. We
compile our programs to .obj/.exe format. With other languages and
environment I have used to that debuggers just execute routines for
witch they don't have debugging information available. Does anyone have
an advice for me?

Reijo Koivisto
Finland
```

```
A reply from one of my "usually reliable sources",

"Animator support for debugging GNT/OBJ was introduced in the 4.0 product
(though you could accomplish the same thing in the 16-bit world several
years earlier by using Xilerator).

As to this customer's question, the reason the animator will be prompting
for the library routines written in COBOL will be because those programs
were compiled with the ANIM directive. Also, the animator does not prompt
for any source when calling operating system APIs as it knows nothing about
them, so that statement is a bit of a mystery - are they sure they are Win32
APIs that are being called ?"
```

```
Below is a Batch file I use to Animate programs with
Workbench 4.0.xx under Windows95/98/NT.

Animit.bat
@echo off
::Animit.bat
echo ganim>cobol.dir
cbllink gdfs.cbl kernel32.lib
del cobol.dir
set cobanim_2=animate
animate gdfs.exe
set cobanim_2=

The program must be Cbllink'd with the ganim compiler
directive.
The environment variable cobanim_2 must be set to the word
animate.
You can then animate the executable.
The environment variable needs to be unset when finished as
this setting will conflict with animation of Int programs.
This techniqur only works from the command line and the
graphical Animator, Anim2, can be used as an alternative.
```
