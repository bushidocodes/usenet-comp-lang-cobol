I had to look this one up (again).

There is no "line length" option in the Standard RD statement.

In the draft of the next Standard, there is a
   PAGE LIMIT IS n COLUMNS

but that isn't part of the '85 Standard.

I really, REALLY, think that it should be the RECORDS CONTAIN (in the FD)
that deals with this.  (I have checked the latest Fujitsu LRM and they *seem*
to be ANSI conforming for this.)

Can anyone using Fujitsu V5.0 check and see if this makes a difference with
the current compiler and was just a "bug" in the V3.0 product?
