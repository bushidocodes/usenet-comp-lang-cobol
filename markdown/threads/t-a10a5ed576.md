This just says that the use function 'tmpnam' (creates a temporary
name) should be relaced by 'mkstemp'.

> /opt/lib/cobol/coblib/libcobol.so: the use of `mktemp' is dangerous,

and again.

> cob.o: In function `cob':
> cob.o(.text+0xd5): undefined reference to `func'
> collect2: ld returned 1 exit status

C++ does 'name mangling'.  If 'collect2' is a C++ function that you
are trying to call from Cobol then you may find that the C++ compiler
has called it 'collect2IX4Lr' or something based on the parameters
defined for it.  This is how overloading is done in C++.

When you compile 'collect2' you need to specify that it is a C
function, not a C++ one.

> Can anybody help.........i am new to cobol

Trying using Cobol rather than using C++ in the Cobol compiler.
