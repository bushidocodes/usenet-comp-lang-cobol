Hi,
I'm on Ent Cobol 4.2 on zOS.

Running into a strange issue. Here is the relevant snippet of code

FD BARCODE-SWITCH-FILE
BLOCK CONTAINS 0 RECORDS
DATA RECORD IS BARCODE-SW-REC
RECORDING MODE IS F.
COPY BARCODE.
*****************************************************************
THIS COPY BOOK IS FOR THE BARCODE SWITCH FILE *
*****************************************************************
01 WS-BARCODE-REC.

As you can see, the intent was to have the definition of the DATA RECORD variable in the copybook but evidently there was a miscommunication between the person who wrote the program and the one who wrote the copybook. Nowhere in my program is BARCODE-SW-REC declared.

But on the host I don't get a compile error.

Down on the PC using RDz I do get an error which makes sense.

Is this a problem with the mainframe compiler?

The IBM Cobol Infocenter says
"DATA RECORDS clause

The DATA RECORDS clause is syntax checked but serves only as documentation for the names of data records associated with the file.

data-name-4
The names of record description entries associated with the file.

The data-name need not have an associated 01 level number record description with the same name.
"

In this case should it not fail the syntax check too?

Brief history of the DATA RECORDS clause(, as nearly as I can tell).

COBOL Report 1960 - The clause was required.
COBOL 1968 - The clause was documentary and not required.
COBOL 1974 - No change.
COBOL 1985 - The clause was made obsolete.
COBOL 2002 - The clause is no longer present.

"[W]as there ever a time ... had to match ..."?

Yes, COBOL 1960.

With COBOL 1968, one must distinguish between the standard and
the implementation of the standard. Because the clause was made
documentary in the standard, an implementation could have chosen
strict conformance, or not, without violating conformance. Hence,
matching was not required by the standard, but might have
been required by an implementation.

As to IBM, I don't know. I recall using the clause in the 70's
with a 1968 ANSI COBOL compiler, but not of any problems with
the clause.

In any case, shortly after I began using a Micro Focus ANSI 85
compiler, I stopped using the clause. So it has been about 25
years since my last use of the clause.


If the clause is not required and its presence may cause
problems, then why use it?
