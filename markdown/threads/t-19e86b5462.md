```
I am somewhat new to CICS programming and have a question about the use of the
'START' command. I am going to be writing a system in which students can
update some of their own data. They would first put their ID into a security
screen, transfer (XCTL) to a menu screen and then transfer to several
different update screens (XCTL). IF the student does not return to the
security screen once they are finished updating, can I issue a START command
from within the update screen that says to return to the security screen after
5 minutes of inactivity. If so, what happens to the update transaction
executing at the time? You can either mail me an answer or call me at: Howard
Schwartz (413) 545-1465. All help would be very much appreciated.
```

```
To do something like this, I'd probably START the follow-up task
immediately, and the have it issue a DELAY for 5 minutes with a REQID,
then any activity should CANCEL the REQID (and perhaps start a new
timeout task (that issues a 5 minute DELAY etc..) You could have the
REQID be a 4 character constant + the TRMID as an agreed upon
convention. Actually, in a production system, I'd probably find a
lower overhead solution that might be more work, but you did say this
was a student programming system.
```
