Hi

If you need to know, I am using Microfocus Cobol 3.2.41 and DOS 6.20

I want to be able to distinguish between an error, or a CTRL-BREAK
in an exit procedure

I have used CBL_EXIT_PROC to setup the exit proc. A simple
example of the exit proc that I want is:

IF Error-Occured = "Y"
DISPLAY "Error"
ELSE
DISPLAY "Program Aborted with CTRL-BREAK or CTRL-C".

How do i determine Error-Occured.

In Pascal there is an error address that you can check for.

Many Thanx
Keith
```

There is a way of detecting a CTRL+BREAK from COBOL and you could
use this to detect its occurance. The following program shows how
to detect its been pressed:-

working-storage section.

01 func-code pic x comp-x.
01 func-parm pic x comp-x.
01 lcount pic x(4) comp-5.
01 data-item pic x(10) value "item".

procedure division.

*
* Disable control break
*
move 4 to func-code
move 1 to func-parm
call x"b0" using func-code func-parm

move 0 to func-parm lcount
perform until func-parm = 1
accept data-item
*
* Test control break - func-parm = 0 No, = 1 Yes.
*
move 6 to func-code
call x"b0" using func-code func-parm
display "Count = " lcount
add 1 to lcount
end-perform
*
* Control break has been hit - lets tidy up
*
display "CONTROL BREAK HIT"
stop run.


It is also possible to detect at termination using errorlevel.
An errorlevel of 16 indicates termination by CTRL+BREAK and an
errorlevel of 32 indicates temination by an RTS error.
```
