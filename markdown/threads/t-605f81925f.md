### Client Server using Tuxedo, Fujitsu and SP2 on Unisys Clearpath mainframe

- **From:** jets@nbnet.nb.ca (Tony M. Mina)
- **Date:** 1998-10-10T00:00:00
- **Newsgroups:** comp.lang.cobol
- **Message-ID:** `<361edf6a.48356795@allnews.nb.sympatico.ca>`

```
We wrote a C DLL routine to do the Tuxedo calls to the Unisys
mainframe.  This C DLL was compile using Visual C++ v4.0 and is called
from Fujitsu v4.0 Cobol using SP2-32 bit as our GUI interface.  The
Fujitsu Cobol and SP2 is the client program  running on Windows 95.

We were able to schedule.multiple transactions (calling the C DLL)
back and forth on the mainframe with no problems at all.  But when we
get out of the client program we get an error on MFC40D.DLL and
sometimes KERNEL32.DLL as well as F3BIPRCT.DLL which is a Fujitsu DLL.

We don't think the problem is on SP2 because if the client program
doesn't call the C DLL everything is fine.  We did a trace on the
programs and it's dying on the "EXIT PROGRAM" of the client program.
We reported this to Fujitsu but haven't got any response yet.

I do appreciate very much if anyone can give as some clue.  The system
will be running on Windows 95 as the client and Windows NT as the
server using SP2 thin client from Flexus International.

Thanks a lot.


Tony M. Mina
```
