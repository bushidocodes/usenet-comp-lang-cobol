```
Hello all,
I'm trying to find out if my company should buy a copy of NetExpress
or not. We're using a development product called MPACT by ESI. It's
basically a GUI front-end designer for 2 and 3 tier applications using
the MF or Fujitsu compilers. It generates the vast majority of code
you will need and allows you to add your code when needed. It
provides native support for Unisys A-series computers which is a big
plus for my company.

While the product is very good, it doesn't have support for some of
the more common features one would hope, such as being able to get a
Windows printer or font dialog box. With their help, we do this API
calls. I'd like to add a File/Open/browse dialog. I'm not looking
forward to writing the API calls to do this!

My big question is: Does MF NetExpress provide for accessing Windows
common dialog boxes via CALL's like their PC_WIN_PRINT routines in the
COBOL compiler? Do they support this feature at all? I'm looking for
an easy way to do this and don't want to spend days getting the
routines written. We'd be willing to buy a copy (how much does it
cost BTW) if it provided some of these features. Could anybody let me
know if/how MF does this?

Thanks in advance.
Doug Blaze
```
