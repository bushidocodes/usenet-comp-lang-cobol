```
Concerning

02  CHAR        PIC X(1), OCCURS 8 TIMES.

There are some pretty specific rules in the ANSI Standard about
  - when it is legal to end a Picture clause with either a comma or a period
  - when (if used) the comma and period are editing symbols and when they
are separators
  - what clauses are allowed AFTER a picture clause that ends with a comma
or period

I tried (quickly) to find the exact 85 Standard rule that would "apply" to
this statement.  I think (but am not positive) that it is NON-CONFORMING
source code.

See page VI-33 which states (in part),

"If the insertion character ',' (comma) is the last symbol in the PICTURE
character-string, the PICTURE clause must be the last clause of the data
description entry and must be immediately followed by the separator period."
```

```
I agree with your interpretation. This is non-conforming code. 
Therefore any interpretation by the compiler is legal in the context of 
the standard. This is certainly an ill-advised place for an extension, 
though, as the result can be so confusing. My guess is that it simply 
resulted from a failure to add a check for the condition, rather than 
as an intentional extension. If I were using this compiler, I'd ask for 
a change.

COBOL syntax can be so messy and inconsistent ... anyone who has ever 
written even the simplest COBOL scanner (for limited program analysis) 
knows that you have to use different scanning rules for PIC clauses 
than for anything else in the language.

This is one reason that using commas for "readability" is a bad idea, 
even though they may look natural in many contexts.

Edward Reid
```
