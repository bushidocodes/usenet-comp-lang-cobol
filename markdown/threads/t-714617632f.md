```
Hi, this is the first time I've posted here.  I'm using IBM COBOL II and I
have to write variable length records to a file.  I've done this before but
with a limited number of record types, say half a dozen.  In this case the
data is much more variable.  You could say that the records could be of any
length within a minimum and a maximum of course.  Is there any way to do
this in COBOL?  It would be an easy thing to do in a language like PL1.  But

COBOL constraint?  Thanks.
```

```
If you are using the (now unsupported by IBM) VS COBOL II product *and* you
are using the NOCMPR2 compiler option, then you can use the '85 Standard
syntax:

FD  file-name
   ...
  Record Varying in Size depending on WS-ITEM ...

where WS-ITEM is a working-storage item with the length of the record to be
written (without the RDW).
```
