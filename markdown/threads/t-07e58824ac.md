The Exec SQL whenever ... declaration seems to have a serious limitation.
I does not allow a procedure to be "PERFORM" 'ed; it does however
allow GO TO. For many "sqlerror " or "not found" conditions, I would
like to return to the statement after the exec sql that caused the
condition.

Also, does there seem to be a rational explanation for the requirement
that host variables be a "01" level item (except for variable length
structures)? This effectively prohibits usage of qualified data names
for host variables (i.e. last_change_date of record-ws).

Thanks in advance for sharing your ideas.

Regarding your second point about the 01 requirement for Host Variables. We
use the Fujitsu Compiler in most of our COBOL tools.

We have also discovered this restriction in Fujitsu ODBC which has caused us
lots of problems in deciding what code to generate in our new Casegen COBOL
for Windows Professional RAD development tool. We have decided to use a
temporary table/column naming convention until the restriction is lifted,
generating the 01 level Host Variables for each column in the table.

Fujitsu has agreed to look at the problem as a matter of urgency, but I
suggest that your views would be most useful to the early resolution of this
restriction. Perhaps you could email them, their web page as you probably
know is www.adtools.com.
