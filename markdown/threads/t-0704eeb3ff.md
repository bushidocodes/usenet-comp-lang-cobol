One thing to keep in mind is that not all code is hand coded.
GOTO may be perfectly OK, even necessary in generated code.  Have
a look at the code generated by bison the parser generator for
example. 

Anyway, languages have many uses.  People often extrapolate from
their own needs and practices and sometimes seems to feel the
need to inflict their preferences on others. (not suggesting this
is the case for either of the people listed above).

Sometimes a goto can save a lot of tedious IF statements, and you
end up with a simpler program overall.  The linux kernel contains
huge numbers of gotos, and a result it runs a lot faster than it
would otherwise.

In the COBOL compiler I put in a warning about goto but only at
the highest level ("FYI Go to considered harmful") and it is
mainly just there as a joke.  

In java they have a construct EXIT LABEL, which makes it viable
to not have goto in the language.  With this construct you can
exit several layers of 'perform' in one statement.  This is very
useful for error handling when you don't want the overheads of
exceptions (create object, later garbage collect it).  

Tim Josling
