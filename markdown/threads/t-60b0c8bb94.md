Serious answer?? Well it will vary by the machine you are running on but as

high order bit used as a sign if specified. Therefore an s9(4) comp
occupies 2 bytes with 15 bits avaialable for a signed number, range =
+32767 through -32767. The comp3 you specify is not very efficent, it
should always be an odd number so the three bytes you you occupy should be
s9(5)comp3 giving a range +99999 through -99999. Once you move to 4 byte
usage the comp field starts holding far greater numbers BUT I also seem to
recal that the maths process on the IBM was most effectively carried out in
comp3 format (adding two comp3 numbers would result in a ZAP for the first
number and an AP to give the result). So generally, unless you were using
large numbers comp3 was always preffered but always with an odd number in
the brackets.
