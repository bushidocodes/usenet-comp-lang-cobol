Stefan,
Further to Gael's response, if the reason you are asking about CANCEL is
because
you want the EXIT PROC to free up resources (dynamically allocated
memory, database
connection etc) then you would probably be better off installing a
CANCEL PROC in
your program instead (see the CBL_CANCEL_PROC call - introduced in
NetExpress 2.0).
A CANCEL PROC will be called whenever a CANCEL is issued for the program
in question,
either with an explicit CANCEL statement, or during an implicit cancel
(eg, CHAIN,
STOP RUN, CBL_SUBSYSTEM removal, etc).

The manual is probably unclear because of poor COBOL language
specifications. 
STOP RUN in a program is an unconditional return to the operating
system or equivalent invoker when issued from either a main program or
a sub-routine.  EXIT PROGRAM is a return to caller if issued from a
sub-routine and the equivalent of CONTINUE if issued from a main
program.  GOBACK, an IBM extension also implemented by other vendors
and in the vaporware next standard, is a return to caller which may be
the operating system.  If the program is returning to the operating
system, it goes through clean-up which includes closing all open files
(assuming COBOL 85 or later) and freeing of allocated memory.
