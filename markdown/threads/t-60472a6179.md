I have a file which I am told is in RM Cobol relative file organisation.

It consists of fixed length 260 character records.
The first 4 bytes of each record contains hex 00 01 00 01.

Pointing me to an explanation on the web would suffice.

We do not normally document our file formats. However, this one is
pretty simple: the first two bytes contain the length of the record,
and the right-most bit in the second two bytes is '1' if the record
has not been deleted. Deleting a record sets all four bytes to 0.
