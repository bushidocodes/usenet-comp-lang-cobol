```
It has come to my attention that at least one person is confused by the
numeric USAGEs in both Standard COBOL and as non-Standard extensions.

1) Easy case - both the USAGE COMP and USAGE BINARY are defined in the
ANSI/ISO '85 (and 2002) Standards.  There is *no* limitation on what the
picture clause may contain for such USAGEs - other than the fact that they
must define a numeric (not integer) data item.  See (for example), see ('85
Standard)  page VI-46,

     "(3) An elementary data item whose declaration contains, or an
elementary data item subordinate to a group item whose declaration contains,
a USAGE clause specifying BINARY, COMPUTATIONAL, or PACKED-DECIMAL must be
declared with a PICTURE character-string that describes a numeric item,
i.e., a PICTURE character-string that contains only the symbols 'P', 'S',
'V', and '9'. (See page VI-29, The PICTURE Clause.)"

2) There is nothing to limit any vendor from "limiting" an extension USAGE
(e.g. COMP-3, COMP-4, COMP-5, COMP-X) to only integer picture clauses.
However, when it comes to COMP-5 and COMP-X (when specified with "9's" in
the PICTURE clause), the most common extension definition that I am aware of
is the same as for COMP and BINARY usages, i.e. it must be numeric - not
just integer.  See for example

IBM - LRM

http://publibz.boulder.ibm.com/cgi-bin/bookmgr_OS390/BOOKS/IGY3LR10/5.3.16.1

"The picture for a COMP-5 data item can specify a scaling factor (that is,
decimal positions or implied integer positions). In this case, the maximal
capacities listed in the table above must be scaled appropriately. For
example, a data item with description PICTURE S99V99 COMP-5 is represented
in storage as a binary half-word, and supports a range of values
from -327.68 to +327.67."

Micro Focus LRM
 http://supportline.microfocus.com/documentation/books/nx31sp1/lrpubb.htm

"8.25 The USAGE Clause
    ...

4. An elementary data item whose declaration contains, or an elementary data
item subordinate to a group item whose declaration contains, a USAGE clause
specifying COMPUTATIONAL-5, or COMPUTATIONAL-X must be declared with a
picture character-string that describes a numeric item or ..."

    ***

As I said, there MAY be some COMP-? extensions that do require an integer
picture clause, but it is certainly NOT true for the Standard "COMP" and
"BINARY" usages - nor for at least some common implementations of COMP-5 and
COMP-X.
```
