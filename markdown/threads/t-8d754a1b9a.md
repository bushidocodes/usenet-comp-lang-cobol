Grinder wrote:

>  The "buffer overflow problem" is not unique to
> windows, however.

Certainly not.  But to get a useful result from a buffer overrun it must 
overrun into a specific place and set the bytes to exactly the right value.

If all copies of, say IIS version x, are identiacal then what works on one 
copy will work on the next.  Because Apache will be recompiled for each 
distro and may use different librariy versions, different compiler options, 
even different target CPUs (386 to 686) then the bytes needed to cause a 
useful overrun will be different for every copy, or at least will be wrong 
for the majority of them.

This means that, while a buffer overrun may corrupt, say, Apache, and may 
cause it to fail, it is unlikely to cause it to execute specific malicious 
code.

Having Apache fall over is a much lesser problem than what happens with IIS 
where malicious code executes on a buffer overrun and this turns IIS into a 
weapon to attack other sites.
