    STRING
       Constant-1 DELIMITED BY SIZE
       Textfield-1 DELIMITED BY SPACE
       Constant-2 DELIMIED BY SIZE
           INTO receiving-field

reference modification, like this:

     Textfield-1 (1:Position-of-last-non-space-character)
        DELIMITED BY SIZE

   On various methods to scan for trailing spaces, search this  
newsgroup for several discussions on that. You might use keywords  
"INSPECT PERFORM VARYING REVERSE" and author name "Chuck Stevens".

might find his easiest solution to be reference modification.

PERFORM VARYING MY-INDEX FROM 79 BY -1
	UNTIL MY-INDEX < 1
                  OR MY-STRING (MY-INDEX) > SPACE
END-PERFORM
ADD 1  TO MY-INDEX
MOVE MY-STRING (80)     TO MY-STRING (MY-INDEX:).

*** Note the colon after the last MY-INDEX.  It means the TO is a string going
to the end of MY-STRING.

It's an easy task in COBOL, too, and there are several ways, but your 
question is still a tad unclear.

Nevertheless, in the most straight-forward case:

MOVE SPACE TO OUTPUT-FIELD.
STRING {constant1} DELIMITED SIZE
   {text-field} DELIMITED SPACE
   {constant2} DELIMITED SIZE
   INTO OUTPUT-FIELD.

The above presumes there are no imbedded spaces in {text-field}. If there 
are imbedded spaces in {text-field} but never two or more consecutive 
imbedded spaces, the line {text-field} DELIMITED 'bb' should be used instead 
(where 'bb' is two blanks).

There are other techniques, growing in complexity, that might be used 
depending on the anticipated nature of the data. Some involve wolverines.

You'll find that things that are trivial in one language are difficult in 
another - and vice-versa. For example, computing the hyperbolic arc tangent 
is tough in COBOL, but only one line of code in FORTRAN. Conversely, 
floating a currency symbol is piddly in COBOL but a lot of coding in many 
other languages.

Presuming you've got a COBOL compiler that conforms to the '85 standard and
includes support for the Intrinsic Function Amendment, it's not all that
complex.  The program below illustrates the technique and should, so far as
I know, work just about anywhere:

000100 IDENTIFICATION DIVISION.
000200 ENVIRONMENT DIVISION.
000300 DATA DIVISION.
000400 WORKING-STORAGE SECTION.
000500 01  INPUT-FIELD PIC X(50).
000600 01  OUTPUT-FIELD PIC X(100).
000700 01  NUM-ITEM-1 PIC S9(10).
000800 01  NUM-ITEM-2 PIC S9(10).
000900 PROCEDURE DIVISION.
001000 MAIN-PARAGRAPH.
001100     MOVE "EFG    H        I " TO INPUT-FIELD.
001200     INSPECT FUNCTION REVERSE (INPUT-FIELD)
001300         TALLYING NUM-ITEM-1 FOR LEADING SPACES.
001400     COMPUTE NUM-ITEM-1 =
001500         FUNCTION LENGTH (INPUT-FIELD) - NUM-ITEM-1.
001600     DISPLAY NUM-ITEM-1.
001700     MOVE SPACES TO OUTPUT-FIELD.
001800     STRING "ABC," DELIMITED BY SIZE
001900             INPUT-FIELD (1 : NUM-ITEM-1) DELIMITED BY SIZE
002000             ",JKL" DELIMITED BY SIZE
002100              INTO OUTPUT-FIELD.
002200     DISPLAY OUTPUT-FIELD.
002300     STOP RUN.

Note that Function TRIM has been proposed for the COBOL standard currently
in development and planned for publication in 2008.   The proposed function
accepts one or two parameters.  The first parameter is an alphanumeric or
national character data item; the second, if it is present, is a mnemonic,
"LEFT" or "RIGHT".  If the second parameter is absent, the function returns
the original string stripped of both leading and trailing spaces.  If the
second parameter is LEFT, the function returns the original string stripped
of leading spaces; if the second parameter is RIGHT, the function returns
the original string stripped of trailing spaces.  See J4/04-0198, accessible
from www.cobolportal.com/j4/ , for the latest distributed draft of the
proposal that includes this capability.
