>I received a databse from a client of me which has been written in cobol.
>The program dates from 1988 and they are still using it.

Let's get precise here. A database is not written in a language. A database may
be created by a program, or maintained by a program written in almost any
language. But there is no such thing as a "COBOL database" (or a "c database",
etc.).

Since you are referring to a COBOL program that has been in use since 1988, I
would assume that the program was initially for some mainframe. [This is just a
guess, because I'm not aware of any COBOL compilers available on workstations
back then; I could be way off here.] Do you know what kind of machine the
program was orginally coded for (or even what kind of machine it is currently
running on)?

The database itself could be stored in a number of ways; for IBM mainframes,
for example, it could be an ISAM file (pretty old), a VSAM file, a DB2
database, an IMS database, or even an in-house developed manner, or by some
third party product. For IBM mainframes, character data is stored in EBCDIX,
not ASCII. This is true for some other platforms too.

Can you tell us what kind of database format the data is currently in?

Is "example.dat" the actual name? How is your customer sending you the data?
Perhaps they don't know how to package it or transmit it.

>Please help me. In the program itself there is no export function and I need
>the data for their new databse very badly.

Please give us more information. Be calm, precise, and complete in gathering
and explaining to us the details of your platform and current database format.
