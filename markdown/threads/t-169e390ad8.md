Hello David

I use RM Cobol 6.51 for Windows since October 98. Before I was using RM
Cobol for DOS 5.32.
There are many factors to consider.
The main reason for me was to use the 32Bit OS (Win 95/98 or NT) and to be
compatible for the future.
After changing and compiling about 350 Programs within about 3 months I
would say it was worth it. I don't have limits anymore with programs that
have more than 25'000 lines of source code.
Important things to consider (but there can be many more):
Forget old self written routines in Assembler for Windows managing etc. But
you can recplace them with
the built in Windows Pop up Manager or new written DLL's . If you use
external routines for ex. in C consider, there could be some problems like
with finding the right links to the DLL's.
The System is so stable as the DOS one. I never had Cobol Program under 6.51
going down. Many other
Window Programms did.
I still use the ISAM File System, the File can be used without any changes
if you define it in the
configuration record.
Don't expect the Windows 6.51 to work as event programming. You can use
mouse events
use boxes and and and but you can recompile your original sources and they
run under Windows 6.51.
You can define many properties by default or by program and they are managed
it the Registry per machine.
or by default for all machines.
For example you can tell the 15 inch display machine to use Arial 13 and the
17 inch machine to use Arial 15.
Forget the BLINK option if used.
The next version 7 which was announced for about April this year should
bring the possibilty to use Windows
printer drivers. So start forgetting the built in printer codes and the
handling of them in your program!
If you use RM Cobol for Dos you can send me a source (not 25'000 lines!) and
I will check it
for ev. traps or problems. If you have specific questions, make them.
Greetings
John@Casagrande.ch

David,

I have been using RM Cobol 6.51 with Cobol-WOW ActiveX version for a
GUI front end. I have had some problems with WOW as it is a new
product. You have all the standard controls(tabs,bitmaps etc...) with
in it and a large selection of API calls. 

I have written several controls in VB 6 to handle JPEG's, control
network connections, access DUN connections, print images, any true
type font that is available in any colour and frames.

The rmcobol runtime is one of the most stable peices of windows
software I have ever come across even under 98.
