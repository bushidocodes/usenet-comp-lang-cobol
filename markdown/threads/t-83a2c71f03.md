### A question about variable

- **Newsgroups:** comp.lang.cobol
- **Message-ID:** `<950318124730607@star-tech.com>`

```
-> In COBOL texts I find that a variable name should be up to 30
-> characters long. Does this mean:
->
-> (1) Any variable name longer than 30 chars is simply not COBOL;
-> Although some implementation may chose to support this, no one has to
->
-> (2) Variable names are unique to the first 30 chars. This would imply
-> that:
-> AAAAABBBBBCCCCCDDDDDEEEEEFFFFF1 and
-> AAAAABBBBBCCCCCDDDDDEEEEEFFFFF2
-> cannot refer to two different variables, and therefore:
-> (a) Either this situation causes a variable name conflict
-> (variable declared twice) ... or:
->
-> (b) The two variables are treated as being identical ... (thi
-> is a dissaster).
->
-> So which is it?

I am just starting out as a COBOL programmer (landed an Associate
Programmer job yesterday and start in two weeks) so my experience is
more academic at this point. My understanding is that ANSI COBOL only
recognises 30 characters. I tried compiling a program with a 31
character variable name with Micro Focus Personal COBOL 2.0 and got the
message "COBOL word contains more than 30 characters. Word truncated.
Continue checking?". I tried compiling it using the student bare-bones
version of RM COBOL and got compile time warning and a crash when it
came to area in the proceedure division dealing with the variable.

I would say that an ANSI compliant compiler shouldn't permit more than
30 characters. But then everyone likes to add there own extensions
here and there when designing a compiler...

- Dan Hite
```
