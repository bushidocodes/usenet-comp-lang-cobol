An interesting post by Mr. Pritchett and ensuing thread by all the
regulars...

It's curious that no one has mentioned automating the clean-up process.
I've heard tell that there are these new-fangled machines out now that
can be programmed to read an input file, apply certain logical
operations to the data, and write the results to an output file. If this
is really true, then perhaps they could be programmed to transform COBOL
source using the logic Mr. Pritchett outlines. 

Trouble is, I hear these fancy new machines require very specialized
technical experts to program them. And I'm told these experts are all
very busy maintaining old and very messy "legacy" programs. 

So I guess fate decrees that we must spend the rest of our days cleaning
up COBOL source by hand. ;)

But seriously folks... I know I've played around writing such cleanup
programs, and I have a hunch many of the rest of you have too. Any
finished products out there? Or should I polish up my code and publish
it?

--John Hicks
