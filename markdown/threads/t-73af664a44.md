Hi Lars,
We need to know what compiler you are using?
Did you try to get an Informix pre-compiler?

You see, when you put an Exec Sql statement into your cobol program it is
normal that the DB vendor gives you a 'translator'. That means that your
pretty Exec Sql.....End Exec is expanded to some pure cobol statements that
actually reads the row.

Some compilers has 'built in' preprocessors. In that case, you will never
see the precompiled code. When you debug you will only se the code you put
into the program.

If you tell us what compiler you are using, you can be sure that someone has
tried this before, and he/she will help you immediately.

Let's try to solve your problem,

Regards
Ib

You are right, Sample programs are a fine way to get started.
If you want to access Informix, you need the Informix ESQL precompiler.

I've looked around for samples, and you can find some at
http://www.merant.com/ads/support/indexf.asp

Have fun while you are testing the samples, and change them to your own
flavour, but save the programs step by step. The learning process is to
create small successes for yourself. When you save the steps, it is easy to
fall back to the last reliable version.

Have fun
