This looks like a Berkley DataBase versioning issue.  BDB is fairly
fragile when in comes to compatibility between versions.

In fileio.c take a look at the lines from 60 ish to 90 ish.  You'll
see a fairly complex set of conditional includes for the variants of
db.h

From ./configure --help

  --with-cisam            (OpenCOBOL) Use CISAM for ISAM I/O
  --with-disam            (OpenCOBOL) Use DISAM for ISAM I/O
  --with-vbisam           (OpenCOBOL) Use VBISAM for ISAM I/O
  --with-index-extfh      (OpenCOBOL) Use external ISAM file handler
  --with-db1              (OpenCOBOL) use Berkeley DB 1.85
(libdb-1.85)
  --with-db               (OpenCOBOL) use Berkeley DB 3.0 or later
(libdb)(default)
  --with-lfs64            (OpenCOBOL) use large file system for file I/
O (default)

So you may want to try --with-db1  or  --with-db  depending on your
local Berkeley setup.  Or install a newer libdb.  I've never
encountered problems, but this one comes up quite often.

Cheers,
Brian
