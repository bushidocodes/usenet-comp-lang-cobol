statements through the use of an assembler subroutine that changed the

previously went to the bit bucket caused an abend (reports which
caused a DDNAME that wasn't in the JCL).  It was fun modifying the
program to use status codes on all of the OPEN and WRITE statements
for that file.  If the normal handling of output QSAM (sequential)
files in shops had been to check status codes, then CBLQDA would not
have been needed.  The 0X status code that would have resulted however
would definitely been considered a condition that had to be handled
differently and NOT the same as 00.
