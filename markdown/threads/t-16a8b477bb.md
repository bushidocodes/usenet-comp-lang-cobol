You should also look at Acucobol's Acu4gl package. It supports both Oracle and
Personal Oracle. I have not used it myself, but I've talked with several other
developers who have used it successfully. I like the concept because it enables you
to use COBOL syntax to access the DB, so additional coding is minimal. My understanding
is that the biggest problem in implementing it is in building the data dictionary, but
the compiler has the ability to build data dictionaries from COBOL FDs. (You need
to add some complier directing statements to the FDs.)
