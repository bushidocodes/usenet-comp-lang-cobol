A sequential file defaults to that behavior on most systems. If your
system does not, then try a WRITE RECORD BEFORE 1 statement.
You should note that on the PC, you will get both a CR *and* a LF.
On a Unix system, you will generally get just the LF.  The general rule
is that you will get the same format as source code for that particular
platform.  It is easily checked with your editor.

```
I was going to refrain from this one, but since all of the suggestions 
use techniques unique to various implementations, I thought I might 
give one that is standard.

You can use the SYMBOLIC CHARACTERS clause in SPECIAL-NAMES for this 
(one of the reasons it was invented).

Like

    SYMBOLIC CHARACTERS carriage-return IS 14
       ...

    MOVE carriage-return TO the-print-line (last-char + 1: 1)

If your program collating sequence is always ASCII, this will always 
work.  If not, you can make one that will work anywhere by specifying 
an alphabet-name equated to STANDARD-1 and change "IS 14" to "IS 14 IN 
alphabet-name".  Note that I used 14 even though the actual value is 
13.  That is because the number is an ordinal in the collating 
sequence associated with the alphabet.  Ordinal numbers start at 1.
```
