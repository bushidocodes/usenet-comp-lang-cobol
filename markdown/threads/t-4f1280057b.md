```
Hi,
I am trying to call a C DLL from a MicroFocus COBOL program using
calling-convention 2.  When compiled with Debug code the call works with no
problems.  When compiled without the Debug code the call works and the C DLL
performs its task but the COBOL paragraph does not return to the code that
called it.  It runs off the end of the program.  The C program is exporting the
entry points as CDECL.  Is there another calling-convention that I should be
using? I hope someone here can shed a small light on what is going on.

Thanks for any input,
Kentf@qstr.com
```

```
with no call convention (or call-convention 0) and it should work correctly.
```
