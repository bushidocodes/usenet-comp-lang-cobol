```
I haven't studied your code in detail, but the general "philosophy" for
error reports (or screens) where you have to show a "bad" input field's
value is to:

A) REDEFINE the input field as alphanumeric (PIC X) (if it isn't already)

B) Define the place in your output where you show the incorrectly input data
as alphanumeric (PIC X)

C) move the Alphanumeric version of the input field to the alphanumeric
version of the output field - and display (with messages) as alphanumeric

D) Special warning:  You may need to do SOME checking on your invalid
"input" value to make certain it doesn't include non-display characters (for
example "Line Feeds" or "attribute bytes").  In these cases, you will NOT
want to simply move these to your output field - as they could cause your
entire output line/screen to get screwed up.  (This is probably beyond class
assignments - but is worth thinking about.  When your "input" comes from a
"display only" device, this shouldn't be a problem.  However, when this may
occur, you end up having to display error messages with hex-values and more
complex ways of identifying the problem.)
```
