Ganesh, as well as being a reporting tool, FOCUS is also used to create
files, databases and GUI applications both on the mainframe and the PC.
People have been predicting its demise for years, but it is still being
used at most of the sites where I have worked. It is used mostly for
reporting and its big advantage over other reporting tools is that it can
be run against most, if not all, types of files and databases at the same
time. I've supported FOCUS applications within TSO and VM (CMS). My
impression is that it has never been marketed all that well and still
remains in relative obscurity compared to say QMF.

Focus is a 4th generation application development tool developed by
Information Builders, Inc. of New York. It has its own hierarchical
database engine but also offers read/write interfaces to many other
relational engines (DB2, SQL/DS, etc.). It can read just about any file
format around, making it a powerful reporting tool, for which it is best
known. There are versions for mainframes, minis and PC's, but most
successful on IBM mainframes and VAX machines, probably the most successful
product of its type during the 80's. I don't see much new development
going on with it, but an awful lot of legacy apps in production.
