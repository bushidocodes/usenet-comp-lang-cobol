The Fujitsu 3 'User's Guide' specifies that the SELECT ASSIGN for a
printer may be to:

             PRINTER
             "LPTn:"         (note colon, eg "LPT1:")
             "COMn:"

Note that using LPT1: may require that the printer be setup in Windows
with 'capture' for this port.

Manuals can be useful sometimes.  In fact it has a whole chapter (8)
on printing.

You don't specifiy whether it only prints the first item in the table,
or whether it prints all the items in the table but all on one line
that is hundreds of characters long (and therefore falls off the edge
of the page).

Try using a write statement that is suitable for a printer, such as:

        write print-record BEFORE ADVANCING 1 LINE

This is also covered in Chapter 8 of the manual.

Manuals are available for download from www.adtools.com.

Printers need a CR (Carriage Return - resets position to start of
current line) and LF (Line Feed - feeds paper to next line) on the end
of each print line.  This can be given by write .. BEFORE ADVANCING 1
LINE (or AFTER if you prefer) or it can be specified as being
automatically added by SELECT ... ORGANIZATION LINE SEQUENTIAL (rather
that the default of SEQUENTIAL).

Without these there is a possibility that all the print data will appear
on one line hundreds of characters long, well past the edge of the
paper.

This is covered in the manual.
