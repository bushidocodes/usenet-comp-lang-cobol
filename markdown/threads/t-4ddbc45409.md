With most PC Cobol's you do it in your code.  The usual is in
the SELECT clause, though with MF and several others, you can
also do it in the VALUE OF ID clause of the FD.  That is an older
standard, though, and normally only done for compatibility with
older systems.
Typical:

    SELECT MY-FILE ASSIGN TO "C:\myarea\myfile.dat"
            rest of select clause.

Note that if you want to vary the filename, or keep it in a setup file,
you can use a data name in working storage instead of the literal
of the example above.
IE:
    SELECT MY-FILE ASSIGN TO FILE-NAME-VARIABLE
            etc.
    WORKING-STORAGE SECTION.
    01 FILE-NAME-VARIABLE        PICTURE X(45) VALUE "c:\myarea\myfile.seq".

You can then change FILE-NAME-VARIABLE in your program
before your open, under program control.
