For what it's worth, I wound up using a sort with a compound key which
consisted of the "invoice number" and a "record type" field, then
returned the records and used a dummy record to test for when the
invoice number changed...

RETURN SORTREC
       AT END blah blah...

IF SortInvoice NOT = HoldInvoice
   MOVE SortRec TO HoldRec
   IF SortType = A AND HoldType = B
      SortInvoice is not in file B
      END-IF
   IF SortType = B AND HoldType = B
      SortInvoice is not in file B
      END-IF
ELSE
   IF SortType = B AND HoldType = A 
      matching pair found
   ELSE
      error encountered
END-IF.
