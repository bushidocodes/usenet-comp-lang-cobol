If you have a table like this:

01 TABLE-WORK-AREA.
03 TWA-X PIC 9(04) COMP.
03 TWA-LO PIC 9(04) COMP.
03 TWA-HI PIC 9(04) COMP.
03 TWA-HIGH PIC 9(04) COMP.
03 TWA-FOUND-FLAG PIC 9(01) COMP.
03 TWA-TAB-CODE OCCURS 1000 TIMES
PIC 9(04) COMP.
03 TWA-SEARCH-CODE PIC 9(04) COMP.

And assuming TWA-HIGH points to the last entry in the table
(0 if table is empty), the entries are in ascending sequence,
and TWA-SEARCH-CODE contains the code you are looking for:

MOVE 1 TO TWA-LO.
MOVE TWA-HIGH TO TWA-HI.
MOVE 0 TO TWA-FOUND-FLAG.
PERFORM
IF (TWA-LO > TWA-HI)
MOVE 2 TO TWA-FOUND-FLAG
ELSE
COMPUTE TWA-X = (TWA-LO + TWA-HI) / 2
IF (TWA-TAB-CODE(TWA-X) > TWA-SEARCH-CODE)
COMPUTE TWA-HI = TWA-X - 1
ELSE
IF (TWA-TAB-CODE(TWA-X) < TWA-SEARCH-CODE)
COMPUTE TWA-LO = TWA-X + 1
ELSE
MOVE 1 TO TWA-FOUND-FLAG
END-IF
END-IF
END-IF
UNTIL (TWA-FOUND-FLAG > 0).

If not found, TWA-FOUND-FLAG = 2.
If found, TWA-FOUND-FLAG = 1 and TWA-X points to found entry.

Requirements for this specific algorithm:
o The code / key being searched for on the table, must be unique
on the table, ie. there must be just one possible occurance that
carries the code that is being searched for.
o The code / key being searched for on the table, must be in
sequence on the table. Most easily accomplished by first initializing
the whole table to HIGH-VALUES even before the codes are loaded onto
the table from the master file.
o A predetermined ideal table size is used. This is chosen in
such a way that there is an exact middle occurance of the table with
the same amount of occurances on either side. Further, if one of
these pieces of table on either side of the middle is isolated, it by
itself must again have an exact middle occurance, until there is only
one occurance on either side of the middle.
These table sizes are found by using the simple formula
(two to the power of n) minus one
where n is any
positive integer.
The exact middle of the whole table will then be found with the
formula
two to the power of (n minus one)
and the maximum number of accesses to the table before you either find
the code you are looking for or know that it is not to be found on the
table, is given by n.
o EXAMPLE
We assume in the following example that the table shown, has already
been loaded with the details of the fifty states from a State Master
File. W-SEARCH-STATE contains the code of the state that must be
found on the table.
WORKING-STORAGE SECTION.
.
01 W-SEARCH-STATE PIC XX.
01 W-MIDDLE PIC S9(3) COMP.
01 W-JUMP PIC S9(3) COMP.
.
01 W-STATE-TABLE.
03 W-STATES OCCURS 63.
05 W-ST-CODE PIC XX.
05 W-ST-NAME PIC X(20).
05 W-ST-TOTAL PIC 9(3).
.
.
PROCEDURE DIVISION.
.
MOVE 32 TO W-MIDDLE
W-JUMP.
PERFORM DC-HALF
UNTIL W-ST-CODE (W-MIDDLE) = W-SEARCH-STATE
OR W-JUMP = 1.
IF W-ST-CODE (W-MIDDLE) = W-SEARCH-STATE
.
.
ELSE
.

.
DC-HALF SECTION.
DC-JUMP-UP-OR-DOWN.
DIVIDE 2 INTO W-JUMP.
IF W-SEARCH-STATE > W-ST-CODE (W-MIDDLE)
ADD W-JUMP TO W-MIDDLE
ELSE
SUBTRACT W-JUMP FROM W-MIDDLE.
DC-EXIT.
