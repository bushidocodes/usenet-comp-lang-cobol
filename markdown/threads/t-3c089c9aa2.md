Well, my boss finally assigned 2/5 of the staff to examining the code.
Here's an interesting one:

This is not specifically Y2K, but we found it before it bit us because
we were looking for Y2K

This was in an RPG program. It updated a master file from a trans
file that keeps MMDDY. That's right one Y. The last digit of the
year. If the Y was less than 8 we add 90 to the year, else we add 80.
Guess what happens 01/01/98? The trans get updated with a date of
1988! ARGH!
