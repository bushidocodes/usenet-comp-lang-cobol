```
We have an insurance administration system written in Micro Focus
COBOL (3.2.43) that runs under DOS in a Netware 4.1 environment.
Our users want to run multiple sessions of this application in
Windows 3.1, but the problem is that the application opens about
100 files when it starts up. We quickly run out of file handles
under DOS/Windows.

So, we started using Windows NT Workstation 3.51 SP4, with
Novell's Client for Windows NT 3.5b NTC4. Shortly after making
this switch, we began experiencing random occurrences of what the
developers have dubbed "File Bloat", where a Micro Focus VSAM
file suddenly goes from about 5 Meg to 1.2 Gig or more in the
space of a few hours. The file is corrupt and the MF REBUILD
program has a difficult time correcting the problem. We had to
back off of Windows NT last weekend...

Both Micro Focus Tech Support and the vendor of the application
are stumped. Has anyone experienced a similar problem? Any ideas?
Thanks.
```

```
It may be a good idea to try the 3.2.50 product. The filehandler
(EXTFH), was much improved in this release (in fact it is the
filehandler from the 4.0 product). Code was added, which will
retry any failed disk I/O attempts, which we found could subtly
corrupt files. These I/O failures seemed to be occuring from
slight 'blips' on the network. By retrying these failed I/O
attempts, the 'blip' are usually overcome, and processing will
occur normally.

The reason the file may be exploding, is that the free space
area may be getting corrupted (perhaps due to the NT Novell
drivers misbehaving occassionally?), causing us to write at
some stupidly large offset.
```
