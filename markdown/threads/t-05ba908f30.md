Take a look at Microfocus Net Express COBOL for .Net:

    	http://www.microfocus.com/products/netexpress

I've been playing with this for the last couple of weeks. Quite an amazing 
product, with lots of good migration features to take mainframe COBOL or 
older COBOL from VAX, Unix, PC and move it across to a modern Windows .Net 
platform. Some recoding involved - but not as much as you might expect!

Because .Net is multi-language, it lets you add new modules to the system 
in languages like C#, VB or C++ ... or you can just keep writing in COBOL 
if you want.

There is also a Fujitsu COBOL compiler for .Net but if you're using XM and 
DIALOG then Microfocus is probably a better fit.

Hope it helps,
Andrew

That will be CCI's DR-MDOS GOLD.  An enhanced version of Digital
Research's Multiuser-DOS.  I used DR-MDOS and equivalent systems.  It
is still being developed and supported by IMS Ltd - www.imsltd.com -
as Real/32.

I have been replacing my DR-MDOS based systems with Linux.  The
conection can use dial-in, or telnet or ssh over the internet, or a
VPN. Local machines will connect over the local network.  Use a telnet
client such as Token-2 or putty on Windows machines, or if no Windows
required use a telnet client box such as an old 386 or 486 with Pocket
Linux that will boot off a floppy - no hard disk, and gives several
telnet sessions.

> I currently have WB 4.0 and Dialog 2.5 and have run/animated the application
> from within WB in a DOS window as a single non-networked user.

Recompile with MF OCDS 4.1 Cobol for Linux.  No significant changes
required, you may need to change the file names. The files will copy
over with no reformatting or conversion required.

> I would like to initially migrate the application to a network environment
> with minimal change.

Once you have it on Linux you could use SP/2 for Unix to give a GUI
(actually a Text mode UI) running over telnet/ssh.

I also use LTSP for client machines locally.  These boot up over the
network and only require CPU, RAM and network card, and not much of
those either.  They give a Gnome or KDE interface and use konsole
(xterm) to run the Cobol system.  Everyone gets multiple sessions,
just as you got with DR-MDOS.
