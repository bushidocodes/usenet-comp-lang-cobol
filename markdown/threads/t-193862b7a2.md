From the README file of the PL/I Compiler (which uses the same debugger:)

---------------------------------  begin copy ----------------------------------
   C.1. Debugger
        --------

        a) Specify the /TEST and /GONUMBER options when you compile your
           application for debugging.

        b) Monitoring of expressions when the debugger is stopped
           at the main entry to the program is NOT supported.
           Unpredictable results may occur, and you may not be able
           to monitor expressions during the rest of the debugging
           session.

        c) Monitor information should NOT be saved in the debugger
           program profiles. If monitor information is saved in the
           program profiles and the program profiles are used
           during RESTART or a subsequent debugging session,
           unpredictable results may occur.

           To prevent monitor information from being saved, select:

           options->debugger settings->program profiles->select information

           and ensure that 'MONITOR' is not selected.

        d) Occasionally, you may see strange behavior in the debugger,
           such as failing to start, shutting down when stepping, etc.
           This can occur when the debugger Program Profile files
           -- which save information about previous debug sessions, such as
           window positions, breakpoints set... -- become corrupted. If the
           debugger exhibits such behavior, these Program Profile files
           should be erased.  These files have an @ sign in the
           extension (e.g., PDEBUG.@ox, or MYPROG.@op).
           On OS/2, their default location is in the OS2 subdirectory on the
           drive where OS/2 was installed (e.g., C:\OS2).

        e) If MMPM/2 is installed, the debugger may shut down when it
           tries to make a beep sound.  (The Warp operating system
           installs MMPM/2 by default.)  To avoid this problem, run the
           debugger in Synchronous PM Debugging mode or use DINSTSND.CMD
           to disable the sound portion of MMPM/2.  (You can enable
           sound again by running INSTSND.CMD.)  There is an APAR for
           Warp (PJ22061).

        f) If the Desktop Automatic Lockup feature is enabled and the
           debugger is active, the system may lock up when the timer
           expires.  Currently, the only solution is to reboot. To
           avoid this, disable the lockup feature (e.g., via the
           desktop settings notebook) before using the debugger.

        g) The debugger ignores the date attribute when evaluating expressions.

---------------------------------------- end copy --------------------------------------------------------

HTH
