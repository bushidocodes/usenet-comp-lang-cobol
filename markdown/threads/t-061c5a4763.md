What do you mean by "VS COBOL" (in the subject)?  If you mean either  IBM's

  OS/VS COBOL
or
  VS COBOL II

then the answer is that the INTRINSIC function "RANDOM" isn't valid for either. 
If you mean the "RANDOM" access method for Indexed or Relative files, then what 
don't you understand about it?

If you mean some other compiler, please tell us which compiler (exact compiler 
name and vendor) and operating system.

The first IBM mainframe compiler supporting intrinsic functions was IBM SAA 
AD/Cycle COBOL/370.  The only *currently supported* compiler for the MVS, 
OS/390, z/OS environment is Enterprise COBOL and it fully supports all '89 
Standard intrinsic functions.

NOTE:
  The CEERAN0 callable LE service - see:
http://publibz.boulder.ibm.com/cgi-bin/bookmgr_OS390/BOOKS/ceea3151/2.2.5.54

may NOT be called by either OS/VS COBOL or VS COBOL II - even if using the LE 
run-time library, but could be used from an Enterprise COBOL program.
