```
Well, the COBOL is not too relevant, really...We're amending an
existing COBOL application to turn it into a CGI application so that
we can use a browser as the front-end, it'll be an all client-server
and Internet ready (and other jargon) thingy.

That should pose no problem for running over the Internet proper,
which is one of the aims, nor for use on an Intranet, which is the
major aim. However some of our existing users are content small-scale
users and while they may have more that one machine, and even a
network(!) certainly don't have a web server as such. And this would
also apply to some potential sales. As you should realise our existing
application runs in this sort of environment but when we upgrade to
the CGI version it would be desirable to drop the old version...

So are there any really low-cost, low-resource,
low-administration-requirements web server products that could be used
on a small network or single machine for this purpose?

Now, after you've all recovered...

But seriously, we realise that we will probably have to keep the two
versions but if there are any web servers that might be suitable for
our smaller-scale users it would simplify the developments and allow
us to give a common interface to as many of our users as possible and
open up the lower end of the market to some extent.

Thanks

graham
Olympic Computers
(UK)
```

```
I am currently running Fujitsu COBOL 97 (Version 5) CGI code, and Java
(Applets and Servlets) together, using MS Personal Web Server (the one that
comes free with Win 98). It works fine and is an excellent platform for
testing and debugging Client Server applications. It should certainly
support up to 16 machines on a LAN without problem.

In fact, I was sceptical about sung COBOL as server side code when I started
it ("everybody knows" that Perl is the language for CGI scripting...<G>). I
found COBOL to be ideally suited to this task and have recommended it to
others...

Mail me privately for demo code and support.

Pete.
```
