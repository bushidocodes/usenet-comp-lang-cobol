The Microsoft COBOL version 4.5 is not a 32 bit compiler.  It compiles
16 bit executables.  If you want to compile the Microsoft COBOL source
program (without change) to a 32 bit Windows executable, the easiest
way to accomplish that would be to purchase NetExpress from Micro
Focus.

>During the last weeks I tested Fujitsu Cobol V3.0 but I could not produce a
>run-file. Same situation with Flexes-Cobol.

If when you say "Flexes-Cobol" you are referring to any of our
products, one very significant reason that you couldn't produce a run
file is because we don't sell a compiler.  Our tools are add-on tools
for the compiler.

If you are referring to the "shareware" COBOL compiler that we keep on
our web site, PLEASE do not call it the "Flexus COBOL" compiler
because it is NOT ours.

We placed it on our web site and ftp site only as a service to the
readers of the comp.lang.cobol newsgroup.  A number of people were
placing the large uuencoded files on the newsgroup containing the
cob650 shareware compiler.

Many newsgroup participants were annoyed at this, especially those
who are in countries where they must pay a per minute conect time fee.
The large uuencoded files were costing them money, so as a service
to the COBOL community, we agreed to host the shareware compiler
on our site.

Nobody seems to know what happened to the author.  He reported died
several years ago.  Nobody knows where the source code is and
therefore it would be difficult for anyone to support the product.
