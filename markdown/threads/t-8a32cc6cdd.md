I do agree with you 100%.  If Peacez simply wants to upgrade his
development system without modification to his program source or
target environment, then there really should be no need to buy or even
evaluate additional tools.

I believe that Thane assumed that Mr. Peacez also wanted to modify his
program to run in a Graphical User Interface.  We usually suggest to
customers that they try the tools which are included with the compiler
and then try various evaluation versions of 3rd party tools in
conjunction with the preferred compiler before committing to a
project.

The analysis should reveal which is the best combination of tools for
their specific operation.  Sometimes they prefer the tools which are
bundled with the compiler...sometimes they prefer others.  The best
part about this from the customer's perspective is that they can
choose which combination affords them the highest productivity,
essential features and easiest ongoing maintenance.
