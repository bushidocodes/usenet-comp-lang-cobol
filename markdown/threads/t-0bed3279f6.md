```
Hi, I have some problems calling a Cobol DLL. The problem is that
I'm importing it from a C++ program and I don't really know Cobol and
the guy who wrote the DLL doesn't know C++... I hope some Cobol AND C++
guru can help me...(The cobol DLL was made with Netexpress and the
program is compiled in Microsoft Visual C++ version 6)

The function I call in the DLL is called "SYDENTRY", and the Cobol-guy
says that in Cobol he would have called it this way:

CALL "SYDENTRY" USING XX-PARAMETER.

where XX-PARAMETER is described as:

01 XX-PARAMETER.
   05  FORNAMN   PIC  X(35).
   05  EFTNAMN   PIC  X(35).
   05  GADRNA    PIC  X(35).
   05  POSTNR    PIC  X(09).
   05  ORTNAMN   PIC  X(35).
   05  LANDKOD   PIC  X(03).
   05  KONKOD    PIC  X(01).
   05  PERSNR    PIC  X(10).
   05  STTS      PIC  X(02).
   05  MATCH     PIC  X(37).

I have no problems loading the Cobol-DLL (with LoadLibrary and
GetProcAddress), and finding the function but I there is a strange
problem when using it. When I import it I define the function in C(++)
as

const matchCodeParameterSize = 202;
typedef void (*pointerToMatchCode) (char
message[matchCodeParameterSize]);

The DLL reads inputdata from the first 8 variables and write returndata
to the last two (STTS & MATCH). I but the imput data in the big
char-array where message[0-34] is the first entry, message [35-69] is
the next one etc.

Mostly this works but there are som problems:

1) If I compile in VC++ in debugmode I get a notify that "The value of
ESP was not properly saved accross a function call. This is usually a
result of calling a function declared with one calling convention with a
function pointer declared with a different calling convention". But
mostly it works if I just press "Ignore". When compiling releasecode
theres no such warnings (and mostly the code works ok except from 2)
below)

2) Sometimes I get strange results like other variables being written
over when calling the DLL. But by just moving the place of the call the
problems disappear and everything works ok again.

I suppose the problem is something about the datadefinition and the
size. Since I usually get correct result it is not completely wrong but
obviously something is bad in the memory management...

Are there anyone out there wo can help me with this?
```

```
First, I don't know COBOL at all, but I know Fortran and they are generally
considered the same generation. From the (Swedish!) COBOL and the C++ code
it looks like what you are trying to pass as a parameter is this (expressed
in C++):

struct xx_parameter {
   char EFTNAMN[35];
   char GADRNA[35];
   char POSTNR[9];
   char ORTNAMN[35];
   char LANDKOD[3];
   char KONKOD;
   char PERSNR[10];
   char STTS[2];
   char MATCH[37];
};

You need to figure out the following:
1. Which calling convention do you need to use?
2. How must the parameter be passed, bu value or reference?

here are a few C++ prototype declarations you could try

extern "C" {
    typedef void __cdecl (*funcptr1)(xx_parameter& by_ref);
    typedef void __cdecl (*funcptr2)(xx_parameter   by_val);  // <--
equivalent to yours?
    typedef void __stdcall (*funcptr3)(xx_parameter& by_ref);
    typedef void __stdcall (*funcptr4)(xx_parameter   by_val);
}

see also my website for advice on mixed C++/Fortran programming
```
