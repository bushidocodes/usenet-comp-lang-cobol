I've just started a contract where I've gone back in time, and am using some
old MF Cobol products ! Can anyone help with this problem?

Environment - DOS box under Windows 98.

Cobol versions - MF 3.4 & 4.0

Env vars: I won't repeat these - I've been using MF s/w on various platforms
for 15 years, so I'm fairly confident I am switching these around ok.

Running: INT programs via the GUI Animator and "run.exe"

The problem I've come across is to do with the cursor direction keys (arrows
plus home, end, etc) in the separate key block and on the numeric pad.  In
all but one case, both blocks of keys work. In ALL cases, the numeric keypad
keys work.

So ......
3.4 runtime, 3.4 GUI animator - both ok
3.4 runtime, 3.4 run.exe - both ok
4.0 runtime, 4.0 GUI animator - both ok
4.0 runtime, 4.0 run.exe - separate keypad keys fails, numeric keypad
direction keys ok

I've trapped the value returned by these keys to the COBOL programs. When
working, they return x"3203" (dec 50 03), x"3204" etc. When they fail, ALL
return x"3909" (dec 57 09).

Oh, I'm using the same keyboard mapping files each time, which is backed up
by the fact that the direction keys work with the GUI animator 4.0, and the
function keys always work.

Any ideas would be welcome!
