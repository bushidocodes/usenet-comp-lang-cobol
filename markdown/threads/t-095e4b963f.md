```
1) The >>CONSTANT directive - has been changed to a >>DEFINE directive

2) It has gone back and forth, but I *believe* the current draft allows a
>>DEFINE to apply to the entire compilation group (this includes both nested
programs and a sequence of programs)

3) Some compilers can (and do) "optimize" for identified "constants" as
opposed to literals

4) Constants can be included in copy books (if appropriate)

5) The draft now distinguishes between "compilation variables" and
"constants" - HOWEVER, there is a method of getting constants from
compilation variables.  Compilation variables are used for conditional
compilation.
```
