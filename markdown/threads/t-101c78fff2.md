```
Assuming Microsoft 5 - as in your subject line when - then, yes it was
compatible with the "Micro Focus COBOL" of the same age (which is about a
decade old now).  There certainly is NO claim that current Micro Focus COBOL
is compatible with "ancient" (and long unsupported) Microsoft COBOL.
(Although there are directives and run-time switches to make it "medium
compatible".)

If you *REALLY* meant to ask if the current Micro Focus COBOL under Wintel is
compatible with the current Micro Focus COBOL under Unix, then the answer is
"basically" - but not 100% (some subroutines are operating system specific -
as are some facilities/operations) - but they are close enough be considered
"highly" compatible.
```
