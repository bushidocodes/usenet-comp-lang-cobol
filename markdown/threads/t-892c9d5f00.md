```
What is your record file description? For a record of this type you will
probably find you have a single character field, say salary-type for
example, that can only have certain values.

your record >>> data|salary-type|amount|data
your record >>> -----------Y34535-----
or                 >>> -----------H34535-----
and so on...

When you read the record you code an if statement (level 88 entries are
better if you've learned that yet).

IF SALARY-TYPE = "Y"
   MOVE REC-IN-SALARY TO WS-YEARLY-SALARY
ELSE IF SALARY-TYPE = "H"
   MOVE REC-IN-SALARY TO WS-HOURLY-SALARY
END-IF
END-IF.

If you have nothing in the record itself that tells you what type it is
your're really stuck since both fields are numeric with lengths that can
vary.
Hope this helps.
```

Since the decimal point is implied and not strongly-typed, you'll probably
need to determine the record type by testing the value against a range of
values.  Ideally, the file would contain another element on each record
occurence, denoting what type of record it is.
