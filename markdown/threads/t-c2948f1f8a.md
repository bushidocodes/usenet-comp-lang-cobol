!For COBOL FOR MVS/VM ONLY!

When I exit a paragraph pre-maturely, I usually do it like this:
     PROCESS1-PARAGRAPH.
         ...
         IF <condition1>
             SET ERROR-FLAG TO TRUE
             GO TO PROCESS1-EXIT
         END-IF
         ...         
     PROCESS1-EXIT.
         EXIT.

Does IBM COBOL FOR VM/MVS have something like this?
     PROCESS1-PARAGRAPH.
         ...
         IF <condition1>
             SET ERROR-FLAG TO TRUE
             **<exit this paragraph>**
         END-IF
         ...         

If such an instruction exists, it would eliminate the need for a "GO TO"
and would eliminate the need to code an EXIT paragraph.  Thanks!

NEXT LOGICAL COBOL SOURCE STATEMENT.  (Not verb, but complete delineated
syntactically complete STATEMEMENT.)  i.e., it goes to the next outer level

the -EXIT
statement really should be used as a range checker.

The difference between EXIT PARAGRAPH/SECTION and GO TO paragraph-name-exit
is one of avoiding "accidental" unstructured errors.   If a NEW
paragraph-name is added between the verb and the original
exit-paragraph-name, then the two will perform very differently.  Now, there
may be some question about WHICH of these is what programmers might want -
on the other hand, what the EXIT enhancements mean is that you can now
TOTALLY get rid of the PERFORM THRU structure and still get the "get out of
this paragraph/section" semantics that are often desired.
