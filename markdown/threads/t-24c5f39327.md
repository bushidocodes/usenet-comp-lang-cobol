We did the same migration many years ago, and we still use the MVS as main 
integration platform for our ERP.

We literally bought a commercial "transformer" (sources included) in order 
to migrate from MVS/VSE to AS/400 - DB2 so we've got a pack of sources 
compilable also for Unix-Linux-Win32 (we were already using a library 
architecture in order to manage the VSAM-or-DB2 issues).

Then we added a layer for handling the (? BMS ?) maps in XML and we got a 
graphical (XML-HTML) ERP ... running with a dot.net or java front end, 
Apache or IIS application server and a COBOL backend running on a platform 
choosen by the customer ... (sorry, Apple Macintosh not included).

In the last few months we've got also a couple of customer that migrate the 
OS platform from VSE to zLinux and one of them choose to migrate also the 
ERP onto zLinux with Microfocus as compiler; the only known issue is that 
...
... runtime licensing on open platforms work differently from mainframe's 
ones !!!

Best regards,

    Davide Grandi
