I am trying to link some 'C' code (Microsoft 7.0) with some COBOL code
(Realia Cobol 4.311) to allow the 'C' program access to the COBOL data
files. The Realia manual says that it can be done, however, the exact
steps are not documented. The only documentation available references
MicroSoft C 6.0 and uses the LLIBCEP.LIB library (which I think is the
PROTECTED mode version). This library is not available in MSC 7.0.
The REALIA compiler supports DOS/OS2 bound code (probably necessitating
the PROTECTED lib).

After much frustration I called Realia Tech Support. Realia said that
I needed the PROTECTED version of the MSC7.0 libs. After searching the
MicroSoft BBS I reopened the issue with Realia and was told that they
did not know for sure if there was a protected library for V7 and it
was Microsoft's problem not theirs.

I HATE THE RUNAROUND!

Has anyone else tried to tie these two monsters together? Is there
anywhere to get a copy of MicroSoft V 6.0 (even the libs)?

Please respond by E-MAIL to wkb··.@i··.net.

Thanks for your time and assistance!
