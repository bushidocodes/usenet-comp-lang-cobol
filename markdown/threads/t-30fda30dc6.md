```
A couple of (indirect) implications for COBOL (and PL/1 - because of LE
statements) users on IBM mainframes from today's announcements from IBM.

From:
  http://www.ibmlink.ibm.com/usalets&parms=H_203-13 1

"The following actions are planned to be effective with z/OS V1.6: ...

Language Environment's use of Run Time Library Support (RTLS) will be
withdrawn. This function is used primarily in run-time migration. Given the
stability and the upward compatibility being provided by the Language
Environment run-time library in recent releases of OS/390 and z/OS, this
functionality is no longer required. "

and from
    http://www.ibmlink.ibm.com/usalets&parms=H_203-128

"In order to support the C/C++ for z/VM compiler and updated runtime
libraries for C/C++, COBOL, and PL/I, the Language Environment component has
been updated to the level shipped with z/OS V1.4 and is integrated in z/VM
V4.4."

    ****

I *assume* (no statement from IBM to this effect) that the latter statement
means that the final "end" of IBM COBOL for OS/390 & z/OS can now be
announced.  It already has a "limited" life-expectancy under z/OS, but until
LE was upgraded under z/VM, IBM wasn't able to offer Enterprise COBOL for
that platform.  Now that LE is upgraded, I would expect (but have no "inside
information" to this effect) that a release of Enterprise COBOL for z/VM
should be coming sooner than later.
```
