That may be quite normal, depending on the compiler and run-time being
used.  ISAM files are usually _NOT_ just a bunch of records.

For examnple the system may create a file of a certain size ready to
write data records to it, but there are no valid records yet so the
actual content is irrelevant.  If you were to READ it the system would
report no records (AT END).

Doing this saves the system having to extend the file for every record
and thus results in better performance.

Alternately, if it is Fujitsu, eg, then there will be index data,
record size headers and all manner of other stuff required by the
system _and_ the data may be compressed.

It isn't 'junk', it is just that you don't know what it is.

> Each time I run the program it get's filled with even more junk plus
> what I'm writing to it, 

It isn't 'junk', it is things that the system requires.  You want the
file to be 'indexed', that is probably the index.

> and after a couple of runs, when I run the
> program I get an error.
> Has anybody had this problem before? 

Yes, I've had an error before.  At least two, possibly three.  If I
recall they were actually my faulty code that caused them.
