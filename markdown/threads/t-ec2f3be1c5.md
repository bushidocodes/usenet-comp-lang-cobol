====== quote from Jim Parrish's Web Site on the history of JavaScript
=====================

The year is 1995.

Netscape has just formed an alliance with Sun Microsystems. Java, a new
object-oriented language, is Sun's new baby and Netscape plans to support
not only embedding Java applets, small programs that can live on a web page,
but wishes the ability to script the applets as well. To accomplish this
goal they came up with LiveScript. This was Javascript's first incarnation.
It lasted all of about 6 months. In what some have called a pretty dumb
move, Netscape decided to capitalize on the name recognition which Sun's new
Java language was receiving. Thus LiveScript was renamed Javascript.

Of course, Javascript is neither the same as Java, nor vice-versa, however
the confusion inevitable with naming the two different languages so closely
has not really hampered their acceptance in the community at large. For one
real good discussion on what the difference is at a technical level, look no
farther than this document at Netscape - Javascript 1.5 Core Guide Details
of the Object Model.

We now come to a very important date, December 4, 1995, which is the date
Javascript was officially announced and released by the Netscape Sun
alliance.

Client-Side good, Server-Side bad
Javascript quickly gained acceptance among the web literati. Authors the
world over found they could manipulate webpages and do wonderous things to
astound users. Image flips, document verification and all sorts of magic
just waited to be exploited to amaze the customers. Javascript truly seemed
the stuff of legend.

On the other hand, a band of renegades attempted to make Javascript work on
the server-side. This did not go over so well. Other technologies, CGI, ASP,
JSP, PHP all combined to make Javascript, for all intent and purpose, fail
on the server-side. Yes, I know you can write ASP pages in Javascript ;) .
What I am talking about here is literal server-side Javascript.

Thus, while one could find Javascript used all over the client portion of a
website, one had to look quite hard to find it in use for the server and it
remains so to this day.

Yeah, it's the same, but better
Microsoft was none too happy about Netscape's success with the new language.
Let's face it, hard to run all the browsers in the world if you have to use
a Netscape controlled language to make it happen. So...they made their own
language.

Was it Javascript?

Was it like Javascript?

Nope.

It was VBScript.

WHAT???

No really, it's just like Javascript-only better. Or so we were led to
believe. However, truth be told VBScript has several BIG drawbacks. It does
not work on any browser other than Internet Explorer. It is based on Visual
Basic, which is a Microsoft language and that means it does not run on
anything but Windows (I know that is not true now, silly). Three, it comes
from Microsoft, which in some people's eyes is enough to doom it just by
default. This triad of doom secured VBScript's fate. In the present day,
VBScript is about to go away. Yes. Away. Forever. Well, hopefully. Perhaps.
Ah, right let's just move on shall we...

So, after that somewhat underwhelming debut, Microsoft came out with their
version of Javascript-JScript.

OK, let's try this again
JScript appeared on the web scene on July 16, 1996.

JScript is another version of Javascript. It is written from scratch by
Microsoft and is meant to be not only compatible with Netscape's version
(the original), but also to support the new standards put forth by the W3C.
Though it failed pretty badly at both of these things initially, it did a
darn good job considering and by now it is quite a decent language.
