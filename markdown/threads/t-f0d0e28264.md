I am having a problem with Microfocus personal COBOL for DOS 2.0.
Running Windows 95 with a new computer, etc.
I have installed it 3 times in separate directories in order to try to
deal with this problem, the last time I installed it from DOS which I
accessed from bootup without going to DOS from Windows.

The problem is in the displaying of text on the screen both in
Animator and when running a program from DOS (i.e., "pcobrun
name-of-file" from the DOS command line).

After the program displays some text, e.g., "Enter Last Name:",
and I type in the name, and then display something else, say "Enter
First Name:", the remmants of the name I typed in is often still there
on the screen at the end of the newly displayed text.

So I guess you could say that the displayed prompts/text are being
printed to the screen but are not completely wiping out the previously
displayed text, unless the new text is longer than the previously
displayed text.

This is not always the case, however. It seems to have something to do
with the ACCEPT verb. I will take another look at it today, but was
wondering if anyone has had this problem. Dejanews came up empty on
this.

Also, when running a compiled and checked .cbl program from DOS
command line, the prompts are not even seen at all, and I get the
blank line from the ACCEPT verb and so would not know that I am
supposed to enter text, e.g., "Enter First Name", etc. Also, when I
printed two records from a file to the screen, I saw only the last
record.

I called Microfocus and they told me to send email to tech support,
but the address given is no good.

TIA
