Hi Joe,

You don't tell us all the clauses in the select. It is a VSAM file, so you must
use ORGANIZATION IS SEQUENTIAL, ACCESS IS SEQUENTIAL, FILE-STATUS IS WS-etc.

If you've done this, OPEN shouldn't be a problem.

You use INPUT if the file already exists and you want to read its contents.
You use EXTEND if the file already exists and you want to add records to the end
of the file. Just remember that the must be in ascending key sequence and of a
key higher than the last record key in the existing file.
You use I-O to update any existing record "in place" by using the READ/REWRITE
sequence.
OUTPUT, of course, is used when creating a new file.

HTH, Jack.

causing the problem.

 SELECT BFEXCEPT-FILE ASSIGN       TO BFEXCEPT

    to

SELECT BFEXCEPT-FILE ASSIGN       TO AS-BFEXCEPT

ORGANIZATION IS RELATIVE is used for VSAM RRDS
ORGANIZTION IS INDEXED for VSAM KSDS
ORGANIZATION IS SEQUENTIAL for VSAM ESDS

The AS- prefix in the SELCT is required to distinguish between QSAM
(using a DCB) and VSAM (using an ACB) sequential files
