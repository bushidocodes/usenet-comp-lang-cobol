```
If you have Compuware Corporation's File-AID/MVS product (Release 8.8.1 and
above) you can generate and XML file from your application data.

If you (or someone in your shop) has access to the May 2002 issue of NaSPA's
"Technical Support" magazine, you can read my article, "Generation X Meets
Tyrannosaurus Rex: Working with XML on the Mainframe."

This article describes a new feature in each of two mainframe products: the
XMLGEN utility in Compuware Corporation's File-AID/MVS Release 8.8.1 and the
XML PARSE statement in IBM's Enterprise COBOL for z/OS and OS/390 Version 3
Release 1.  The former helps generate XML documents; the latter makes it
relatively easy to obtain data from these documents.

I hope that helps!

Larry Kahm
Heliotropic Systems, Inc.
```

```
Currently, NetExpress from www.microfocus.com can work with external XML
parsers. There's a white paper on the Micro Focus we site that discusses
XML. Some time in the not too distant future, Net Express will be able to
deal with XML directly.

www.datadirect-technologies.com have an ODBC driver which can read and write
XML directly by mapping XML documents as if they are SQL database tables -
clever stuff.

The XML ODBC driver will work with Net Express.
```
