    For heavens sake - the Mf'ers amoung us know that he is talking about
MF cobol, and the arbitrary values assigned to the various function keys
within a particular system subroutine.

    The values 53 and 54 refer to values assigned to user function keys
pageup
and pagedown.  Up and down arrow are NOT (normally) user funtion keys,
but are instead ADIS funtion keys.

    I handle this by using the program keybcf to assign user function key
values to the keystrokes normally NOT a user key.  There are 127 potential
user function keys, but only 00 thru 56 and 65 thru 98 are defined by
default.

    If you do not have the documentation, you are at a severe disadvantage.
All of these things are in the manual, if you dig.

ADIS is Micro Focus' AcceptDISplay implementation. It hold the rules for
accept and display. You are able to change the rules by configuring ADIS.
When you do that, ADIS puts your new rules in a file, named ADICTRL. You can
copy that file to all future projects and the Accept/Display behaviour will
be the way you have decided. If the file is not present, your Accept/Display
will turn back to default. Look up ADIS in your documentation.

On your keyboard you have about 100 keys. Most of them are letters and
digits, but some of them are function keys. Keys like F1-F12, PgUp, Home,
Cursor keys etc. etc. Micro Focus gives you a predefined keyboard
configuration. The configuration is a list of the keys with the a code
telling what kind of key that is. You can change that configuration with the
program called KEYBCF (KEYBoard ConFiguration). Changing the keyboard
configuration will give you a file, called COBKEYMP. You can use it in all
future applications. Look for KEYBCF in your documentation.

Regards
Ib
