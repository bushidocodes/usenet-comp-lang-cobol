```
Consider a file that has a LINAGE clause.
When a WRITE is made we have the option
to specify [NOT] AT EOP
(or END-OF-PAGE).
EOP is specified as a non-fatal error and therefore
in absence of an EOP clause one should recive
a success status. However, in absence of an EOP clause
AND with a FILE STATUS clause it would appear that
it is not possible to detect this condition.
ie. There is no FILE STATUS 0n for this and gives 00.
(With compilers that I have access to).
Yes, I know that 2002 has the exception code and can be tested
but this is about FILE STATUS.
What do your compilers do?
Oversight in the standard?

Roger
```
