A primary key should uniquely identify a single record in a file. It allows
you to directly access that record. An alternate key is just another field
in the record used as a key. In certain file systems alternate indexes are
used to directly access a base record using any alternate key. Alternate
keys allow you to reference the base file in a different sequence.

There are many real life business situations situations where it can be
useful to have multiple ways to access the same data. If you have a business
with a large number of open orders, you may work with an open order file
using an order number as a key. Every order has a customer number in it.
You may want to use customer number as an alternate key, so you can select
all the open orders for a particular customer. The customer number is also a
foreign key, because it refers to the primary key of the customer master
file.

If you have a manufacturing company that sells a lot of parts you probably
have a Part Master file. Part number would be the primary key. You probably
have parts listed in your open order file. You might want to be able to
select open orders for a specific part number. You would use part number as
an alternate key to your open order file.

Without getting into technical details, there is extra system overhead to
manage key-sequenced files with alternate keys. Record locking can become
more complicated if alternate keys are updated. There can be a performance
penalty for having multiple paths to the same data.

I hope I'm not doing your homework for you, but here goes.

A primary record key is used to identify a single record in an indexed
file structure. It used to have to be unique, but RM/Cobol now allows
for duplicate primary keys. MF and Acu don't. (I'm only have knowledge
of these three products, I don;t know what the mainframe cobols allow).
The primary record key should always be defined as the index that will
be used most often.

Alternate record keys are normally used when an indexed file may be
accessed in different sequences. Using the following example.

SELECT FGEMPL ASSIGN TO FGEMPL
RECORD KEY IS FGE-EMP-KEY
ALTERNATE RECORD KEY IS FGE-ALT-KEY WITH DUPLICATES.

FD FGEMPL.
01 EMPLOYEE-RECORD.
05 FGE-EMP-KEY.
10 FGE-COMPANY PIC 9(2).
10 FGE-EMP-NO PIC 9(8).
05 FGE-ALT-KEY.
10 FGE-SURNAME PIC X(20).

The most common way to access an individual record in this file would be
via the Primary key (fge-emp-key) which contains the Company number and
the employee number. Where recrods need to be accessed in surname
sequence the alternate record would be used.
