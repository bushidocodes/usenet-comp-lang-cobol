Gary,

Integrating Visual Basic V4 and Micro Focus COBOL V3 is not always a
straight forward task. Micro Focus COBOL V3 for DOS includes Windows 3.1
16-Bit support. Visual Basic V4 has both a 16-bit and 32-bit version. If
you trying to use VB 32-bit version with Micro Focus 16-bit version, then
you will need to "thunk" the 32-bit parameters to 16-bit. Also, you need to
ensure that your parameters between VB and COBOL are specified properly.
For more information on generic VB and COBOL parameter matching, see the
section on Parameter Passing Between Visual Basic and Fujitsu COBOL in the
white paper on integrating VB with Fujitsu COBOL at:
http://www.adtools.com/news/MicrosoftVisualBasic.htm

Provided time permits, then you might also want to take a look at Fujitsu
COBOL which provides seamless integration with VB.
