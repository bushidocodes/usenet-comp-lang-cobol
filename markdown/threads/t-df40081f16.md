Try http://www.boutell.com/faq/oldfaq/cgi.htm  to start for more
information.

The strings that come in are URL encoded a percent (%) sign followed
by a hex digit (from memory) so to decode the URL in cobol would not
be a major exercise.  Once that is finished you simply check the input
and away you go.

** Does anyone have Public Domain cobol code for this **

There are two types of input POST and another.  One puts the whole lot
on the command line, the other uses environment variables.

The output is simply an output to the sequential stdout (display if
you like).

To format the output you will probably need some basic HTML and the
header record.  This is not rocket science unless you need graphics
(not called static graphics but pie charts).

The simple way to design the output is to format it using a tool and
then look at the output and then put that output into your program
somehow (stripping what you can).

Good luck
Ken
