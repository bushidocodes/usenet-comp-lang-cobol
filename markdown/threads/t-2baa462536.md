X/Open (when it still was X/Open) most definitely DID issue its own
"Standard" document (called a "CAE - but I can't remember what this was an

Standard.  (Actually, this was the second CAE - the first was based on the
'74 Standard).  As I know that your (RW) often refer to Micro Focus for your
"real world" experience, let me point you to the following (sorry for any
formatting issues) that you can find at:


http://supportline.microfocus.com/documentation/books/sx2011sp1/sx20indx.htm

"XOPEN
Specifies that words reserved under the X/Open definition of COBOL are to be
treated as reserved words.

Syntax:
 >>-.---.-.-------XOPEN--"level"-.----------><
    .-/-+ ..----.-XOPEN----------+
           .-NO-+Parameters:
      level  Must be 3 or 4. The level of the X/Open definition COBOL to be
compatible with.

Properties:
      Default: NOXOPEN
      Phase: Syntax check
      $SET: Initial

Dependencies:
XOPEN sets RTNCODE-SIZE"4" immediately.

Comments:
The possible values of level are:

      3 Compatible with X/Open Portability Guide 1988 (XPG-3)
      4 Compatible with X/Open CAE Specification (XPG-4)

Specifying XOPEN with no parameter is the same as specifying XOPEN"4".

XPG-4 specifies options for several environments. If available, the options
are as follows:

  a.. Single-byte Internationalization Support is enabled (see the NLS
directive)


  b.. National Character Support is enabled (see the DBCS"3" directive)
See Also:
DBCS Compiler directive
NLS Compiler directive
RTNCODE-SIZE Compiler directive "



    ***



Please note well the reference to the X/Open documents.  In fact, when these
came out there were even "conformance" suite tests for them.
