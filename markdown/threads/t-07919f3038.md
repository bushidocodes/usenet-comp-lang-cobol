If you don't have control over the creation of the above record and cannot
change the layout of it, (in other words, your desired key HAS to be
non-contiguous) then you will need a version of COBOL that allows split keys
for alternate indexes. (Some do, some don't... check your documentation)

(The fact that you are seeking to change the SELECT by the addition of a new
ALTERNATE INDEX, suggests that this is not the case. Given that you do have
some control over this record layout, a simple solution is posted elsewhere
in this thread.)

Another approach would be to convert the file to a Relational table in a
RDBMS and then you can index anything you like...

Fortran, eh? I remember teaching it to myself from a "Teach yourself Fortran
programming" book back in the 1960s...

It's still being seriously used in a number of places. I remember reading
(in the 1980s) about a Hospital in one of the Southern states of the USA
which ran their computer systems entirely on Fortran. Apparently when jobs
completed, the system would prompt with "Next DO?" It amused me at the time.

Sometimes, especially when doing stuff on your own, it is good to have what
you thought could be a solution, confirmed by people knowledgeable on the
subject.

And keeping code flexible is a good practice in ANY programming language. I
try to write general solutions, which may not be quite as fast as a specific
hard-coded solution (although I write those too if performance is an
important priority), but, on numerous occasions I have been very glad that
the code was flexible enough to accommodate new requirements. You should
probably have a look at templating (creating a general "shell" of a function
then modifying it for specific use by replacing "placeholders" in it.) It is
a step towards component based solutions where you re-use components that
"plug together" like Lego blocks... very powerful and quick to build, but
also really easy to maintain.

I'm planning to discuss all of these things on the COBOL21 side of the new
PRIMA web site (http://primacomputing.co.nz) when it is officially launched
later this year.
