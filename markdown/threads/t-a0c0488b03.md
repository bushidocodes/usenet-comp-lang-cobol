You are specifically asking abut GUIs. But when you say 'Animator' are you
talking about the Micro Focus DEBUGGER - excuse the pun, but it has bugger
all to do with Screens <G>.

Assuming you are using Net Express - you have the following options :-

- Flexus SP2 - as Thane has already pointed out
- Screen IO - from Norcom
- Windows APIs
- Net Express GUI OO classes using the Net Express version of Dialog Editor

- Net Express - Dialog System - very different from Dialog Editor
- Net Express calling other languages - C++, Java, VB etc.

Certainly most of the above require a fair/big amount of effort to learn.
Can't blame you for wanting to do it the easy way - go with one of the
first two options.

I use Netexpress/Dialog System from the first version.
Dialog System is a tool that you use to create screens graphic or
character. it can be used in MS-Windows, OS/2 or Unix. The version
Mainframe Express can emulate terminals 3270. Dialog System uses an own
liguagem based on events.You choose the wanted object, clicks, it drags
and if necessary writes the dialogues for the object. example: We press
a key F3 inside of an Entry Field we would show a Dialog box.
F3
  SET-FOCUS DBOX-001

and the window would be shown.

For us to use ANIMATOR with Dialog System, we changed the option call "
DSGRUN " for call " DS ".

Maybe is what I did of any use for you: I had to transform quickly a couple
of printscreens into a couple of screensections.

To accomplish this I 'abused' the compiler directive  ' $SET
SOURCEFORMAT"FREE" '
in microfocus to have an easier to maintain screensection. This way I could
have all 80 characters of 1 line on the screen in 1 line of code.
The big advantage is that you can see how your screen will look like in your
program.

(mind: every 03 level uses a new line, but because of linewrapping in your
newsreader the code below might look differently)
