I would like to know what others have done to run MF cobol programs that
deal with files larger than the Sun-Unix 2 gig file size limit.

We have created called I/O sub programs that are executed in place of read
and write and these handle breaking up a file into parts that are each less
than two gig.

When files needs to be sorted each must be less than 500 meg or the files
will not be sorted correctly due to the sortwork file seeming to required
4-5 x the size of the input file.

Any comments would be great!

Lou d.
