Hi. I've no idea how the compiler checks for this, but assuming it uses file
modification timestamps, be sure that your source files (and COPY files) are
not datestamped into the future (or whatever your system clock thinks is the
future). If this is the case, each time you compile to .INT the .INT will be
timestamped according to the system clock, but it will still look older than
the source files (and thus the compiler still thinks the .INT has not been
built from those source files). The problem will go away once the system time
catches up with the .CBL timestamp and .INTs are therefore stamped newer than
the sources.

Just a guess, hope it helps.
