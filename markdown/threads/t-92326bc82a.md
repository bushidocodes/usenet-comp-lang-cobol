Hello Marco, this is going back a long time (you've got some pretty old
product there), and unfortunately I can't try this, but I think the error
may be caused by picking up the wrong linker. There's a LINK.EXE in the
COBOL EXEDLL directory and the MSVC BIN directory. Check your PATH and
whichever of these is being picked up (i.e. is first in the PATH), re-jig
things to pick up the other one instead. If you're not sure which one it's
finding, copy the LINK.EXE to the current directory.

Let me know if this works.
