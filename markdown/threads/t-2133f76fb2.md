The answer to your question is COBOL FormPrint from Flexus. It allows
a COBOL programmer to print documents through the Windows Print
Manager directly from a COBOL program. The only thing that the
programmer needs to incorporate into the source program is a series of
standard COBOL CALLs. The format is:

CALL QPR USING FUNCTION-PARAMETER DATA-PARAMETER.

All parameters are COBOL working storage variables. The form
definition is stored in an external file and can incorporate bitmaps,
data fields, lines and boxes, radio buttons and checkboxes. Color and
special Windows True Type fonts are also supported. Micro Focus,
Acucobol, RM, mbp Visual COBOL and CA-Realia are all supported.

We have an early release evaluation version available on our web site.
The only difference between the production version and the evaluation
version is that the eval version prints a thin "X" across the page.
We will be releasing a full function forms editor within the next few
weeks. This will also be available on the web site.

The web site is http://www.flexus.com

I don't know about the "Graphics", but you seem to indicate that you want
charts and graphs. There is an excellent package for that called Grafsman.
They have versions for DOS, Windows and UNIX. It works great, and we use
it in our Micro Focus Cobol applications. Once the graph templates are
defined, you can print or display graphs, by making a call the the Grafsman
runtime. First you make the .dat data files for the numbers for the
graphs, then you make a simple call and the chart either displays or is
printed.

If you are talking about graphic images, ie Pictures etc, you should look
at the products from Flexus. http://www.flexus.com. They have tools
windows developement, and their formprint tool will allow the printing of
said graphics.
