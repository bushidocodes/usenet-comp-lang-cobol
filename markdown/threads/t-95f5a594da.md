```
The standard COBOL 2002 syntax to switch from (the default) fixed-form
reference format to free-form reference format is
    >> SOURCE FORMAT IS FREE

The standard COBOL 2002 syntax to switch from free-form reference format to
fixed-form reference format is
    >>SOURCE FORMAT IS FIXED

In both cases the words "FORMAT" and "IS" are optional.

I can't speak to whether Micro Focus' current offerings provide the new
standard Compiler Directives mechanism currently, but given that company's
stated commitment to the new standard and to the future of COBOL, if it's
not there yet I'd be surprised if it weren't there soon. The syntax and
function of this directive have been in every draft of the standard I've
been able to locate (back to Version 1.1, at least), which means it's not
been news for at least five years.

        -Chuck Stevens
```

```
What you may be missing Robert, is that for a compiler that supports all the
features of the 2002 Standard, it is possible to create source code that
will work with *both* free and fixed form reference format.  This is
"useful" in some cases - especially in large enterprises where different
applications use the same copy members.

The following are the rules to follow:

1) Start all source code in column 8 (or later).
2) End all source code before the R-margin (whatever that is for your
compiler, e.g. 73, 80, 255, other)
3)  Use the new comment indicator "*>" as the only start of line comment
indicator
4)  Never continue a COBOL word across lines
5) Use the new format for literal continuation
6) If you have any debugging lines, use the new ">>D" format for this
7) Use the >>PAGE directive if you want source listings with page breaks

    ***

If you follow those rules, you don't need "worry" about using source-format
directives - your code will compile as either fixed or free format source
code.
```

```
Nested COPY statements are expressly disallowed in '85 standard COBOL.  This
has to be a vendor-specific extension.

In the current ISO standard for COBOL, the ability to nest COPY statements
is introduced, but only when neither the the "outer" nor the "inner" COPY
has a REPLACING phrase, and only when the text being copied does not cause
the processing of a COPY that directly or indirectly copies itself.  Given
these restrictions, the implementation must provide for a minimum of five
levels of nesting to be conformant.
```
