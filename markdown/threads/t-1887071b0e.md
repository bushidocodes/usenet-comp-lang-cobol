It is often the case that:

When reading a text file, if a CR is encountered before the record is
filled, the compiler generates code to space-fill the remainder of
defined input record (this is a COBOL standard). In other words, if
the program asks for 100 bytes and the operating system supplies only
80, the program fills the remaining 20 bytes with blanks.

The compiler, for output, takes advantage of the above rule by simply
not writing trailing blanks. This saves space on the output media. At
least that's the theory.

In your case, you'll have to over-ride this default behavior by some
appropriate mechanism unique to your compiler/operating system. It
could be in the code itself (BLOCK CONTAINS 100 CHARACTERS), another
form of the syntax (ORGANIZATION IS FIXED), or some add-on (ASSIGN TO
MYFILE.TXT[U]). Or something similar.
