```
I don't use Screen section much.  I am trying to find out some information about 
how GLOBAL works with SCREEN SECTION stuff.  I have written a program that does 
NOT compile with the version of Fujitsu that I have (but should according to the 
'02 ISO Standard).

Can those of you with compilers that support GLOBAL, nested programs, AND screen 
section, try to compile the following - and if it works, let me know what 
happens?  (If your compiler supports GLOBAL and Screen Section but doesn't 
compile this program, I would be interested in what it says).

*** source code follows:

       Identification Division.
        Program-ID. GLOBSCRN.
       Data Division.
        Working-Storage Section.
       01  anItem    Pic X(10) Value "0123456789".
       01  OtherItem    Pic X(1) Value "Y".
        Screen Section.
       01  aScreen Global.
           05  Value "Here is:"
                   Line 10
                   Column 10.
           05  Pic X(10) From anItem.
           05   Line 11 Pic X(01) to OtherItem.
       Procedure Division.
        Mainline.
           Display "before call"
           Call "Nest1"
           Display "after Call"
           Display "value of OtherItem is:" OtherItem
           Stop Run
            .
       Identification Division.
        Program-ID. Nest1.
       Procedure Division.
        Mainline.
           Display "in Nest1"
           Display aScreen
           Display "between"
           Accept aScreen
           Display "leaving Nest1"
           Exit Program
            .
       End Program Nest1.
       End Program GLOBSCRN.

*** end of source code
```
