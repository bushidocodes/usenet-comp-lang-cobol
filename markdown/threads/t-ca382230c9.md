It is not possible to do what you want, directly.

The reason it cannot be done directly is that the value of pointers
can not be known prior to execution of the program; therefore, one
must know the name of the field before one may use a pointer to
reference that field. An EVALUATE could be used to set a pointer
depending upon the name of the various field.

       EVALUATE w-fld-name
         WHEN "w-fld-2"
           SET ws-pointer TO ADDRESS OF w-fld2
etc.

Since the field name must be known to get its address, it would
be easier to replace the SET pointer with the actual move.

There is an interesting means for accomplishing this, indirectly.
Create a skeleton program, compile the program, then call it.
See the source code below.

The following source code was run on DOS using MF 3.2.50.

------- my-ws-1.cbl -----------
       01 my-ws-1.
         05 ws-01-field-1 pic x(20) value "abcdefg".
         05 ws-01-field-2 pic 9(5) value 1234.

-------- my-ws-2.cbl ----------
       01 my-ws-2.
         05 ws-02-field-1 pic s9(8) comp value 87654.
         05 ws-02-field-2 pic s9(7)v99 comp-3 value 45.35.

-------- my-defs.cbl -----------
       01 ws-table-defs.
         05 ws-td-01.
           10 td-name pic x(30) value "ws-01-field-1".
           10 td-pic pic x(20) value "x(20)".
           10 td-copy pic x(10) value "my-ws-1".
         05 ws-td-02.
           10 td-name pic x(30) value "ws-01-field-2".
           10 td-pic pic x(20) value "z(4)9".
           10 td-copy pic x(10) value "my-ws-1".
         05 ws-td-03.
           10 td-name pic x(30) value "ws-02-field-1".
           10 td-pic pic x(20) value "z(7)9-".
           10 td-copy pic x(10) value "my-ws-2".
         05 ws-td-03.
           10 td-name pic x(30) value "ws-02-field-2".
           10 td-pic pic x(20) value "z(4),zz9.99-".
           10 td-copy pic x(10) value "my-ws-2".
       01 ws-lookup-table redefines ws-table-defs.
         05 ws-td-entry occurs 4.
           10 ws-td-name pic x(30).
           10 ws-td-pic pic x(20).
           10 ws-td-copy pic x(10).

--------- my-skel.cbl -------------
       01 ws-skeleton-program.
         05 pic x(60) value "program-id. temp.".
         05 pic x(60) value "data division.".
         05 pic x(60) value "linkage section.".
         05 s-pic pic x(60) value "01 my-result pic ".
         05 s-copy pic x(60) value "copy ".
         05 pic x(60) value "procedure division using my-result".
         05 s-head pic x(60) value "    ".
         05 s-var pic x(60) value "    move ".
         05 pic x(60) value "      to my-result.".
         05 pic x(60) value "    exit program.".
       01 redefines ws-skeleton-program.
         05 ws-skeleton-line occurs 10 pic x(60).
       01 ws-skeleton-locations.
         05 s-pic-loc pic s9(4) comp value 18.
         05 s-copy-loc pic s9(4) comp value 6.
         05 s-head-loc pic s9(4) comp value 5.
         05 s-var-loc pic s9(4) comp value 10.

----------- view-ws.cbl --------------
       program-id. view-ws.
       environment division.
       input-output section.
       file-control.
           select temp-program-file assign to "temp.cbl"
               organization is line sequential.
       data division.
       file section.
       fd temp-program-file.
       01 tpf-record.
         05 pic x(7).
         05 tpf-data pic x(60).
       working-storage section.
      * Any copybooks containing data to view.
           copy my-ws-1.
           copy my-ws-2.
      * A table of data-names and display pic clauses
           copy my-defs.
      * A skeleton program to be compiled
           copy my-skel.
      * Local variables to accept the name and display results
       01 my-result pic x(60).
       01 ws-sub pic s9(4) comp.
       01 ws-tpf-sub pic s9(4) comp.
       01 ws-compile-flag pic x.
         88 ws-compile-failed value "f".
         88 ws-compile-successful value "s".
       01 ws-call-flag pic x.
         88 ws-call-failed value "f".
         88 ws-call-successful value "s".
       01 ws-char pic x.
       01 ws-result pic x comp-x.
       01 ws-function pic x comp-x.
       01 ws-parameter.
         05 ws-length pic x comp-x.
         05 ws-command pic x(80).
       procedure division.
           perform varying ws-sub from 1 by 1
             until ws-sub > 4
               perform create-program
               perform call-compiler
               if ws-compile-successful
                   perform call-temp
                   if ws-call-successful
                       display ws-td-name (ws-sub)
                       display my-result
                   else
                       display "call to temp unsucessful"
                   end-if
               else
                   display "compile failed"
               end-if
           end-perform
           accept ws-char
           stop run
           .

       create-program.
           delete file temp-program-file
           move ws-td-pic (ws-sub) to s-pic (s-pic-loc:)
           move "." to s-pic (60:1)
           move ws-td-copy (ws-sub) to s-copy (s-copy-loc:)
           move "." to s-copy (60:1)
           move ws-td-copy (ws-sub) to s-head (s-head-loc:)
           move "." to s-head (60:1)
           move ws-td-name (ws-sub) to s-var (s-var-loc:)
           open output temp-program-file
               perform varying ws-tpf-sub from 1 by 1
                 until ws-tpf-sub > 10
                   move spaces to tpf-record
                   move ws-skeleton-line (ws-tpf-sub)
                           to tpf-data
                   write tpf-record
               end-perform
           close temp-program-file
           .

       call-compiler.
           set ws-compile-successful to true
           display "cobol temp.cbl /omf(gnt); > nul:"
             upon command-line
           move 35 to ws-function
           move 0 to ws-length
           move space to ws-command
           call x"91" using ws-result ws-function ws-parameter
           if ws-result not = 0
               set ws-compile-failed to true
           end-if
           .

       call-temp.
           set ws-call-successful to true
           move spaces to my-result
           evaluate ws-td-copy (ws-sub)
             when "my-ws-1"
               call "temp" using my-result my-ws-1
                 on exception
                   set ws-call-failed to true
               end-call
             when "my-ws-2"
               call "temp" using my-result my-ws-2
                 on exception
                   set ws-call-failed to true
               end-call
             when other
               set ws-call-failed to true
           end-evaluate
           cancel "temp"
           .

Hi Mathew.

Have a look at this.

01  w-fld-name.
      03 w-fld2 pic x(01) value "B".
      03 filler pic x(19)  value spaces

  move w-fld-name to w-result.

This should move a "B" into w-result. The way as you had a default value
in w-fld-name to point it to w-fld2, sorry but this does not work, as
this becomes a litteral with no pointer to that field (e.g. text). This
should sort out your query.

If however you need to build a string with a variable name unknown, then
that is a different story. e.g.

01 sub1 pic 9(04).
01 sub2 pic 9(04).
01 first-part pic x(20).
01 second-part pic x(20).
01 result-part pic x(100).

move spaces to result-part.
move 20 to sub1.
perform until sub1 < 1
    or (first-part(sub1:1) not = spaces)
        compute sub1 = sub1 - 1
end-perform.

move 20 to sub2.
perform until sub2 < 1
    or (second-part(sub2:1) not = spaces)
        compute sub2 = sub2 - 1
end-perform.

string "move ", first-part(1:sub1), second-part(1:sub2), " to
filename.txt"
   delimited by size into result-part end-string.

Now say that first-part = "abc" and second-part = "1234". sub1 will
be calculated to 3 and sub2 calculated to 4. Therefore the command will
be as follows.

     move abc1234 to filename.txt

The first-part(1:sub1) stands for first-part starting at position 1 and
3 characters long thus being "abc". If your cobol compiler does not
handle these subscripts single fields, then I suggest that you move it
to a table, do the calculations and then back.


I hope this makes sense and will help you.

Thanks. Have a nice day.
