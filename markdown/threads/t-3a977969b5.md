Well, if it's anything like COBOL under MVS then you are mixing file types.
Variable length records contain a length of record comp field and a length
of block field at the beginning of each block. Yuk! I suggest you use
a flag field and filler and pad the different record types out to a normal
sequential file. i.e.

FD output-file
record contains 80 characters (compiler will check that lrecl = 80)
block contains 0 records (for each record below)
data record is output-rec.
01 output-rec.
05 filler pic x(80).
01 tree-record.
05 record-type-indicator-tree pic x.
88 this-is-a-tree value 'T'.
05 tree-name pic x(20).
05 tree-height pic 9(06).
05 tree-number-of-leafs pic 9(08).
05 filler pic x(46).
01 doggie-record.
05 record-type-indicator-dog pic x.
88 this-is-a-dog value 'D'.
05 dog-breed pic x(20).
05 dog-barks-per-minute pic 9(04).
05 dog-gone pic x(1).
05 doggie-tags pic x(5).
05 filler pic x(49).
They all add up to 80, though some don't use all the bytes.
Well, here the output record will be 80 bytes, here is some of the JCL:

//THEOUT DD DSN=GERBILS.STUFF,DISP=(new,catlg,delete),
// dcb=(recfm=fb,lrecl=80,blksize=8000),unit=discoduck,
// space=(cyls,(1,1),rlse)
//
your mileage will vary...

To indicate a variable length(which I seldom use, so use the following
with a way-big grain of salt)

FD kludge-meister
record contains 50 to 100 characters
data records are bob, ted, carol, alice
block contains 0 characters.
01 bob.
05 Ima-fifty-byte-rec-field pic x(50).
01 ted.
05 hunk-burning-love pic x(100).
01 carol.
05 fooled-you pic x(75).
01 alice.
05 filler pic x(80).

and jcl something like this:
//VAROUT dd dsn=dumb.stuff,disp=(new,catlg,delete),
// dcb=(recfm=vb,blksize=108)(variable blocked, length plus length fields...
etc....
