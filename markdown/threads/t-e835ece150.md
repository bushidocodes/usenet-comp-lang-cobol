Actually, you have it turned around. You can only CALL an independent
program (a nested program). That is, the object of a CALL must be the
program-id name for some other program. PERFORM can only call a section or
paragraph. If you want to PERFORM a subroutine, you could do an in-line
perform of a call:

perform until i-say-done
call 'subroutine' using parm-1,...
if xxx then set i-say-done to true
end-perform

Dear Wayne:
Usually, to call one COBOL program from another, you use the linkage
section in the subprogram, and the call statement with a list of
parameters in the calling (main) program for example suppose
prog1 calls prog2:

And prog2 adds one to a counter...

In program one
The main program:
program-id. prog1.

working-storage section.
01 ws-this-is-parm-one.
05 ws-person-name pic x(20) value spaces.
05 ws-person-phone-nbr pic x(20) value spaces.
01 ws-parm-two-count pic 9(04) value zeroes.
procedure division.

a0000-main.
* explicit program name causes link to be done...
CALL 'prog2' USING ws-this-is-parm-one,
ws-parm-two-count.
stop run.
* stop run causes end of run-unit. contrast this with exit
* program or goback verb in subprogram...

OR you can move the program name to an area for dynamic subprogram calls.

MOVE 'PROG2' TO WS-SUB-NAME.
CALL WS-SUB-NAME USING WS-THIS-IS-PARM-ONE,
WS-PARM-TWO-COUNT.



subprogram prog2:

program-id. prog2.
working-storage section.
* linkage, a characteristic of subprograms.
LINKAGE section.
01 link-this-parm-one.
05 filler pic x(20).
05 filler pic x(20).
01 line-the-count pic 9(04).
* note using statement that specifies parm list in subprogram...
procedure division USING link-this-parm-one,
link-the-count.


add 1 to link-the-count.
* exit program returns control back to calling program...
EXIT PROGRAM.


Note in the subprogram the using phrase of the procedure division
which lists the parms used in linkage section. These should match
in order the calling programs parms passed to this program.

The two programs can be compiled seperately, and the static link
can be done to combine the two object files together. In a shared
runtime, this can be a dynamic Call where the subprogram is brought
in at runtime.

Wayne,
A COBOL program that is called MUST contain all divisions. The data structure(s) passed
must be placed in the called program's Linkage Section in the order they appear in the USING
phrase of the CALL statement. The data structures in the calling program can originate in
either its Working-Storage and/or its Linkage Section.
