Volker Bandke wrote:

> Hm, did anyone check the rules?  The ISBN is a weighted (10 to 2)
> modul0 11 check digit, which means the result can be 0 to 10.  If the
> check digit would be 10, an X is printed.

OK  Let's go through it again <G>

1. The student's task was to determine the ('fixed') format but wasn't asked to validate the check digit. That got introduced because Jerry quoted the same check digit reference you did for ISBNs. So her professor specifically gave her the format
n-nnn-nnnnn-n, (either to keep it simple or he didn't know Mod 11 can give you an 'X'. Chances are, he is an academic, not a working stiff like you who teaches COBOL on the side <G>).

2. Yes Mod 11 can give you a 'result' of 0 through 10. So you have two choices, (a) ignore, that is, don't use numbers having  check digits of '10', or, (b) alternatively substitute 'X" - which was what we did back in the 60's, (except for the
merchandise Kimball tags we produced which only punch-holed numerics - zero alpha).

3. We know from the ISBN doc that the format is a bit of a moving target, but must have at least 10 digits, including the check digit (0-9 or X). Again this can probably be improved upon, but this is my first shot. And as 'legitimate' numbers I'll use
the three ISBN examples (0-571-08989-5, 90-70002-34-5 and 0-8436-1072-7) plus I'll throw in a legit number with 'X' as the check digit. Let's work on the assumption we 'prefer' the numbers to have the hyphens for readability, but they are going to
move depending upon the numeric  format - which I don't cover.

I think the approach from Dennis was just great, given the student had a fixed format and no check digit calculation was asked for.

I'm sure others can fine-tune the following, (and hopefully I haven't goofed - it's at least 20 years since I wrote a Mod 11 routine ) :-

Jimmy

*>-----------------------------------------------------------------------------
Program-id.  CheckISBN2.

*> See ISBN Document :-

*> http://www.isbn.org/standards/home/isbn/
*>                     international/html/usm4.htm
*>
*> To summarize, the ISBN must be a total of 10 digits.
*> The tenth is the check digit and where the result is '10'
*> the character 'X' is substituted
*>
*> Groupings of digits vary, separated by hyphens of which
*> there must be three. We can determine where the hyphens
*> should appear from the ISBN document. This would add
*> additional logic to get printing format accuracy.
*> It isn't done here !

*> So, including hyphens the minimum field length is 13 chars

Working-storage section.

 01 n                       pic 9(03).
 01 n2                      pic 9(03).
 01 CharCounter             pic 9(02).
 01 DigitCounter            pic 9(02).
 01 aRemainder              pic v9(02).
 01 aResult                 pic 9(04)v9(02).
 01 aSum                    pic 9(05).

 01 input-ISBN.
    05 aChar occurs 13 pic x.

 01 Temp-Number.
    05 aNumber occurs 10 pic 9(02).

 01 T1.

    *> Valid formats and check digits

    05  filler pic x(13) value '0-8436-1072-7'.
    05  filler pic x(13) value '90-70002-34-5'.
    05  filler pic x(13) value '0-330-28987-X'.
    05  filler pic x(13) value '0-571-08989-5'.

    *> Invalid Numbers or mising hypehns

    05  filler pic x(13) value '0 571 08989 5'.
    05  filler pic x(13) value '0571089895'.
    05  filler pic x(13) value '0-571-08985-9'.
    05  filler pic x(13) value '00-70002-34-5'.
    05  filler pic x(13) value '0-8346-1072-6'.
    05  filler pic x(13) value '123-4567890'.
    05  filler pic x(13) value '123-456-789-0'.
    05  filler pic x(13) value '123 456 78 90'.

 01 T2 redefines T1.
    05 TableField occurs 12 pic x(13).

Procedure Division.

Perform varying n from 1 by 1 until n > 12

   initialize Input-ISBN, CharCounter, DigitCounter
   move TableField(n) to input-ISBN

   perform varying n2 from 1 by 1 until n2 > 13  OR
           DigitCounter = 10

     Evaluate true
       when aChar(n2) is numeric
            add 1 to DigitCounter
            move aChar(n2) to aNumber(DigitCounter)

       when aChar(n2) = "-"
            add 1 to CharCounter

       when aChar(n2) = "X"
            add 1 to DigitCounter
            move 10 to aNumber(DigitCounter)
            EXIT PERFORM
     End-Evaluate

   End-perform

   Evaluate true

     when CharCounter <> 3 display
         "Oops ! ISBN " Input-ISBN " Must have three hyphens"
     when DigitCounter = 10
          perform VALIDATE-CHECK-DIGIT
     when other display
          "Oops ! ISBN " Input-ISBN " Insufficient digits"

   End-Evaluate

End-perform

STOP RUN
   .
VALIDATE-CHECK-DIGIT.

compute aSum =
  (aNumber(1) * 10) +
  (aNumber(2) * 09) +
  (aNumber(3) * 08) +
  (aNumber(4) * 07) +
  (aNumber(5) * 06) +
  (aNumber(6) * 05) +
  (aNumber(7) * 04) +
  (aNumber(8) * 03) +
  (aNumber(9) * 02) +
  (aNumber(10)    )

divide 11 into aSum giving aResult
move aResult to aRemainder

Evaluate true
 when aRemainder = 0
      display "ISBN " Input-ISBN  "  is valid"
 when other
      display "ISBN " Input-ISBN  "  is INVALID"
End-Evaluate
