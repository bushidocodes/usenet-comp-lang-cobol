enhancement?

I have a bunch of non-mainframe systems experience, but my mainframe

upgrade the applications) it would provide a "better" environment for the
future.

technical point of view).  However, getting a "well-normalized" database
takes more planning and development.

The addition of a key is not a bad solution.

However, in your heading for this thread you say "COBOL-VSAM-DB2
conversion"

I find no mention of DB2 in the text of your post.

As you are going to repopulate the existing VSAM data anyway, there
might be some benefit in taking this opportunity to move the data to
DB2.

The main benefit in doing this is that it is then accessible with
standard software via ODBC and you can run ad hoc queries and reports
from it with SQL, rather than having to write a COBOL program every
time you need data.

Of course, you would need to convert the existing software to access
the RDB instead of the VSAM data, but this is not such a big deal
(providing the software is layered so that the data access is
isolated...).

From the CICS point of view, it is all the same to CICS whether you
are accessing RDB or VSAM; in either case it is done through the CICS
command interface.

Pete.
