I did originally have just "ASSIGN TO SYS005", but I added the TFIL so I
could use the same source code for both the case where the input was
SYSIPT and where the input was a file.  There's no such thing as "//
ASSGN TFIL,SYSIPT".  With a VSAM managed SAM file having the DLBL w/o an
ASSGN is fine.  The SYS005 part in the COBOL program is ignored.  If you
ASSGN to a system CUU (SYSIPT, SYSOUT, virtual printer) then the file
name (TFIL) is ignored.

In other words:

SELECT INPUT-FILE-1 ASSIGN TO SYS005 works (should work) if SYS005 is
assigned to SYSIPT or SYSOUT (no DLBL required).
SELECT INPUT-FILE-1 ASSIGN TO TFIL works if TFIL is a DLBL pointing to a
VSAM file (no ASSGN required).
SELECT INPUT-FILE-1 ASSIGN TO SYS005-TFIL works for either of the above,
plus it also works for SAM files (both an ASSGN and a DLBL required) and
tape files (both an ASSGN and TLBL required).

Because of the latter I always like to include both the SYScuu and the
DLBL/TLBL name to make the program as flexible as possible without
requiring recompilation.

Well, the question had to do with VSE, not MVS.

Originally in the DOS world, the ASSIGN part of the SELECT
had to be coded as follows:

..... ASSIGN  SYSnnn-c-d-o-x

Where:
c=class:
	DA for direct access, 
	UT for Utility
	UR for Unit Record
d=device (which had to match class):
	2501 (card reader)
	1442x (card reader/punch were x=R/P)
	3350 (disk drive)
	2314 (disk drive)
	1403 (printer)
	etc.
	etc.
o=Organization (which had to match the device):
	S for standard sequential
	A for direct (actual track adddressing)
	D for direct (relative track addressing)
	U for direct w/ REWRITE (actual track)
	W for direct w/ REWRITE (relative)
	I for ISAM
x=FNAME (if not given, then the SYSnnn becomes the FNAME)
	FNAME is treated as DDNAME for OS type systems (VS1, MVS,
etc.).

Once updates were done for VSAM, then things changed about
ye ol' ASSIGN clause. So the answer given must comply with
what is done for VSE and an MVS based answer may or may not
work as VSE handles VSAM differently than does MVS (in fact,
up until VSAM, DOS did not have a CATALOG - although one
could argue that the standard and alternate cylinders
provided such).
