NT includes an OS/2 1.x 'personality' and MS Cobol 4.5 (but not 5.0)
includes the ability to compile OS/2 16 bit programs. Whan a 'family'
(DOS-OS/2) application is run it will test to see whether protected mode
is available and run as an OS/2 app if it can.  I suspect that you
installed MS Cobol with all options and managed to get the compiler to
run as an OS/2 program.  The default is to produce code for the same
environment as the compiler.

However, to link an OS/2 program it requires the OS/2 library which is
not provided by the compiler.  DOSWRITE etc are OS/2 API calls.

You need to specify that it is to use DOS libraries.
