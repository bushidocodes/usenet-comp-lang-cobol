I certainly wouldn't go so far as to rewrite the whole program just because
I don't like it - then again, in some cases a complete rewrite is just
what's needed - but I would certainly be hesitant to propagate bad coding
practices just to preserve the style of the existing code. That's why I
suggested doing at least the most important minor cleanups (e.g. isolating
overlapping perform/goto/fallthru ranges etc.) in the areas where I need to
make changes anyway, and then insert performs to nice new clean code if
there's enough code to be added to make it worth it. It's all a tradeoff
between spending as little time as needed to make the necessary changes vs.
leaving the program at least as maintainable as before (hopefully a little
better) vs. really cleaning it up in severe cases.
