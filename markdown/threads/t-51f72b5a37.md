Lori,

Yes, you can do the move yourself.  It's really a matter of how much you
want to re-invent.  The hardest areas will be screen handling,
references to data file locations in the Select and FD code
(FILE/LIBRARY/VOLUME vs /directory/subdirectory ...), and if your source
made extensive use of other Wang extensions to COBOL.  (MOVE WITH
CONVERSION, READ WITH HOLD, OPEN SHARED, etc.)

Converting the code is only half the battle.  If you want to preserve
the Wang "environment" (command processor, menu-driven interface,
utilities, easy administration, etc.) I wouldn't re-invent that
myself--others have already done so.

The migration companies can offer great features.  It all depends on
your application.  We chose the migration vendor approach and have been
very satisfied with the results.  However, our experience is migrating
from the VS to unix, not NT.
