A couple of answers,

1) "Relative indexing" - where you have "+/- integer" following an index or
subscript is supported in ANS'85 COBOL.

2) Using an actual arithmetic expression (e.g. (index * 2 / 4)) is NOT
supported in ANS'85 COBOL - but will be in the next ANSI COBOL and is
already supported in a few compilers as an extension.

3) I am surprised by your use of an underscore in a variable name. If you
are looking for IBM documentation, then I don't think that IBM compilers
accept this - although some other compilers do.

Subscripting can use any arithmetic expression.

Indexing can use only certain arithmetic expressions:

A literal
An index-name
An index-name plus or minus a literal


› example:
› 77 b_max pic 999.

Note that underbar is not part of the CoBOL character set, although some
compilers will accept it.


› 01 xy pic 999.

On most machines, subscripts and depending-items should have USAGE IS
COMPUTATIONAL, either S9(4) COMP or S9(9) COMP:

77 b-max pic s9(4) comp.
01 xy pic s9(4) comp.


› 01 test pic x(80).
› 01 table.
› 05 buffer pic x(80) occurs 1 to 100 times depending on b_max.

There's no need for a DEPENDING ON phrase unless TABLE is going to be the
source of a move operation. This is where you would define an index if
you were going to use one:

05 buffer pic x(80) occurs 100 indexed by buffer-ndx.


› ...
› ...
› move test to buffer (b_max - 3).

This is syntactically valid for both subscripts and indices. (Of course,
you must ensure that "b-max - 3" is POSITIVE at run time.)
