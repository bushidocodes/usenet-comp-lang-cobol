```
John Dwyer wrote in message ...
>None of my reference materials discuss in depth the procedure to pass a
>variable to a called program. Can someone suggest a web reference, or
briefly explain it.

Over-simplified:

Call-ING program:

    CALL "MYPROG" USING  Var-1, VAR-2, VAR-3.

Call-ED program

LINKAGE SECTION.
   01 VAR-1    PIC whatever.
   01 VAR-2    PIC whatever.
   01 VAR-3    PIC whatever.


PROCEDURE DIVISION USING VAR-1, VAR-2, VAR-3.


The CALL-ed program may either be a completely separate program, or can be
an embedded program; if separately compiled, make sure the CALL-ed
object/load module is available to the linker. (When embedded, that usually
takes care of itself.)

>Also, I'm having problems with multiple indexed files in my programs.  I
>can't find any references for this either.

Huh? What do you mean by "multiple indexed?"  If you mean indexed files with
multiple keys, check the ALTERNATE KEY phrase of the SELECT (these may vary
by compiler).

Note: if the alternate keys are non-unique, you can get some funky file
statii depending on the specific compiler in use.

MCM
```
