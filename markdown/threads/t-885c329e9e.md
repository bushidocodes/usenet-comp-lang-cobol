The collating sequence never impacts the order of returned records in
indexed file or external database (as is this case).  Further analysis
is required to determine a proper solution.  I would use some SQL to
eliminate the 999 records totally and rely on normal EOF processing,
then live with the ASCII collating sequence for the data.  Either that
or change the 999 records to ZZZ records.
