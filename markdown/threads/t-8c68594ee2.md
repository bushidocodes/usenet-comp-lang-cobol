RM/COBOL-85 comes with two utilities: RECOVER1 and RECOVER2 that can
repair and rebuild your indexed files.

By compression, I assume you want to compress the file in a way that
precludes access from COBOL; for instance, you want to store the file
offline. In that case, your system should already have compress and/or
pack utilities--enter "man compress" or "man pack" for help.

If you want better compression, you can get Gnu ZIP (gzip) from many archives
with ftp: the main site is prep.ai.mit.edu in the pub/gnu directory.

You can also get a more pkzip-like package called INFOZIP from a lot of
servers. Both are truly free--no strings attached. Note that these are
usually distributed as source, so you'll have to compile them yourself.
I use INFOZIP's ZIP and UNZIP under DOS and OS/2, and prefer them to
PKZIP in several ways, and the resulting archives are PKZIP compatible.

Here's a tip: the easiest way to get these is via a Web browser: the
URL is http://www.shareware.com and use ZIP or UNZIP as search keywords.
A couple of clicks and they're on their way to your machine.
