Charlie,

You have received some good responses and Jimmy is sending you code so I'll
keep this short.

The connection between COBOL and SQL is achieved by the SQL pre-processor.
This is independent of ANY Relational Database so the same COBOL SHOULD be
able to connect to MS SQL (for SQL Server or ACCESS) AND Oracle, PROVIDED
the table structures are identical. (In fact, you should be able to run
against, INGRES, ACCESS, or DB2 or UDB on a mainframe, given the same table
structures and column names.)

This miracle is achieved by the use of a separate connection to the target
RDB which occurs at run time and is system dependent. It is called Open
Database Connectivity (ODBC) and your COBOL will talk to ANY database for
which there is an ODBC driver available on your system and which has the
ODBC connection activated. Under Windows you do this through the Control
Panel and ODBC32. The exercise of setting this up is separate to the
exercise of writing your COBOL code.

As long as you avoid RDB specific constructs (like Oracle Procedures) and
simply use the RDB as a receptacle, driven from COBOL, your DB access (SQL)
will be portable.

It works and I've done it in both mainframe (accessing mainframe databases
from the PC) and PC environments. It isn't even difficult.

Pete.
