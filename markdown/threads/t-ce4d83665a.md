The RM/COBOL indexed file format is proprietary, and unless someone
reverse-engineered the file format, the document you're asking for
doesn't exist. RM does offer two products that might be of interest,
however:

1. The RM Open File Manager is a "C" library that provides access to
RM/COBOL files. There are restrictions on how it's sold, and I don't
know if your employer qualifies.

2. Our Relativity product allows you to directly access RM/COBOL files
from ODBC compliant databases.

Check out the RM/Liant web page at http://www.liant.com for more
information, or call 1 800-RMCOBOL in the US (1 512 343-1010), or
(011 44) 171 799-2434 in Europe.

I hope this helps.

The more common approach to this problem is to write a program in the COBOL to
create a sequential fixed field fixed record ASCII file and have the C program
strip the data into its own format. Many, including myself, have used this
approach successfully and no knowledge of the COBOL indexed file layout and
compression routines is needed.
