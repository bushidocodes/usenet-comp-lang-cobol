I am not aware of any such tools either, and this is not a minor task.

1) the database conversion - the DLI hierarchy would need to be unloaded to
flat files, then massaged into a format that would map to the relational
model, and converted to Ascii for loading into Oracle.

2) the program conversion - not a minor task either - major coding changes
in programming logic would be called for, as the entire data
retrieval/storage logic would need changing...

If you want to forward the client info to me, or my info to the client,
that would be fine. If they are in an area of the country I can handle,
I'd be willing to discuss further...

As an afterthought, IBM might have some tools for migrating from DLI to DB2
that might be useful, but I am not aware of them.

Bob
