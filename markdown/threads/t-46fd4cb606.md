I am running MicroFocus 4.0.32 on WinNT 4.0 and having the following
problem:

When I open the MF Command Prompt to compile and link programs, the
batch file being run (¥winnt40¥setmfenv.bat) is executing, but the SET

and the PATH is not being changed.

This quandary is further complicated by my ability to invoke a

I really don't want to fool with this every time I need to compile
something!!!

Has anyone else experienced this type of behavior from the MF Command

Does anyone have any ideas about what I might be missing here???

I don't know if it would apply, but I experienced similar problems with
CA-Realia COBOL and OS/2 about a year ago. It turned out that different
DOS sessions had different shell commands in their AUTOEXEC's, and the
DOS session that failed had insufficient environment space to hold the
environment variables. I was able to determine by testing that certain
DOS sessions had more environment space that others. Basically, I
allocated a very large environment variable, and then displayed it to
see how long it actually was.

I had to add a SHELL statement for the automatic DOS session that looked
something like:

shell=c:¥dos¥command.com c:¥dos /e:1024

The final parameter allocated 1024 bytes of environment space to that
DOS session. Again, I don't know if this is exactly the problem you're
having, and I don't know how you configure DOS sessions in NT 4.0 (I
have NT 3.51 at work). But the symptoms sound similar, and it resolved
my problem under OS/2 Warp 3.0.

I hope that helps.
