```
1) Read *both* the COBOL migration guide *and* the LE migration guide (from
IBM)

2) There are minimal (a few reserved word) differences between the 2
compilers. (In fact, you can even keep using CMPR2 - despite what you say).

3) There is a WORLD of difference between running with the VS COBOL II
run-time library and the LE run-time library - even for your existing VS
COBOL II created object code.

I strongly suggest that you plan on migrating your run-time library and
getting it quite STABLE before you introduce the new compiler into your
production environment.
```

effort to be spent on overriding default options that could've been used on
"actual work". In some places, there was so much that needed overriding

their default options would even allow programs to run above the line, thus
defeating much of the purpose of the new Cobol and LE.
