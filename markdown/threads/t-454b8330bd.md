Use a RC file for include version info for your DLL. You should add
the RC to your Net Express project and add it to your DLL.

A sample RC is:-

1 VERSIONINFO
  FILEVERSION    1,0,0,0
  PRODUCTVERSION 1,0,0,0
  FILEFLAGS      0
BEGIN
  BLOCK "StringFileInfo"
  BEGIN
    BLOCK "04090000"            /* lang-charset : US. English, ACSII
*/
    BEGIN
      VALUE "CompanyName",      "MERANT International Limited\0"
      VALUE "FileDescription",  "MERANT Net Express Demo\0"
      VALUE "FileVersion",      "1.00,1.00\0"
      VALUE "InternalName",     "ADSAMP\0"
      VALUE "LegalCopyright",   "Copyright (C) 1999\0"
      VALUE "LegalTrademarks",  "MERANT\0"
      VALUE "OriginalFileName", "ADSAMP.dll\0"
      VALUE "ProductName",      "MERANT Net Express\0"
      VALUE "ProductVersion",   "1.0.00\0"
    END
  END
END

Regards
David.

Juergen,

The problem you have is that CBLLINK only knows about .CBL, .OBJ and .LIB.
Everything else gets passed straight to the linker. However, a .RC cannot be
passed to the linker directly, it has to be compiled using the Microsoft
Win32 Resource compiler first eg

rc -r testpgm.rc

That will generate testpgm.res which you can specify on the CBLLINK line as
the linker will accept it.

I hop that solves the problem for you.
