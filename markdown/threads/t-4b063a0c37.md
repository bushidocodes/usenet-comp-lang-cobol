Use the Windows API:

GetCurrentProcessId
The GetCurrentProcessId function retrieves the process identifier of the
calling process.

DWORD GetCurrentProcessId(VOID);
Parameters
This function has no parameters.

Return Values
The return value is the process identifier of the calling process.

Library
KERNEL32.LIB

I "think" in COBOL you would..

      CALL GetCurrentProcessId RETURNING MY-PID

(I am not a Windows OO-COBOL person).


MCM
