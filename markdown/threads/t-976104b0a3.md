TRERRC = Temp Read Errors Since Last Cleaned  
TRERRI = Temp Read Errors Since Initialized    
PRERRC = Perm Read Errors Since Last Cleaned   
PRERRI = Perm Read Errors Since Initialized    
TWERRC = Temp Write Errors Since Last Cleaned     
TWERRI = Temp Write Errors Since Initialized    
PWERRC = Perm Write Errors Since Last Cleaned  
PWERRI = Perm Write Errors Since Initialized   

Yes there are bad areas on the tape as you have had 2 permanent errors since 
the last time that the tape was cleaned. This could account for the fact that 
you have fewer blocks on the second cart. Another reason could be that there 
are more than one file on the second cart, and the first 2 carts have only one 
file each on them. Do a TMS inquiry by volser to get that info.


BTW if you hit PF1 while in TMS, it will give you a description of each of the 
fields that are shown....

Are you sure that all tapes are the same length. I have never done it
for carts, but I have intentionally shortened the old round tapes. We
used to rip off 50 feet and throw it away. When creating a master tape
for duplication you need to insure that it is no longer than the tape it
will be copied to. Especially if it is a multivolume file, like the old
DASDR dumps.

Also, I believe that if MVS gets enough errors on a tape it will force
early end of volume and switch to the next tape before the current tape
is full.

Were you using compression? Was all data exually compressible? Perhaps
the data on the second tape could not be compressed well, but the other
data could.

Are you sure that all blocks are full? Theoretically you could have the
situation where all blocks on one tape are 23579 bytes each (32760 -
9181) and the next tape has blocks that are 32760. Or the data could
have been written out in short blocks intentionally. What is the
minimum, maximum, average and standard deviations for the block sizes on
all volumes?
