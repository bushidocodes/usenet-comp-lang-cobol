There are a number of constructs, such as records and groups, which
absolutely require 01 designators.
Suppose I also use 01 to designate non-subdivided working-storage
variables, such as record-counters, indexes, error-counters, etc.
If I do a Find on "01", I see all items, real 01's as well as
optional 01's--I have no choice.

On the other hand, suppose I use 77 to designate non-subdivided
working-storage variables. I can now do a Find on "01" to get
mandatory 01 items, or a Find on "77" to get only those items which
qualify as 77 items, or a Find on "01", "77" to get both types of
items in one result set.
Since this flexibility expands my power as a programmer, the only
people who could object are opinionated autocratic pedagogues.

Q.E.D.

I think there are justified uses for GO. Since the scope terminators

77 OP-INPUT PIC X.

several times. The structured way to write this is

GET-OPERATOR-INPUT.
PERFORM UNTIL OP-INPUT-VALID WITH TEST AFTER
DISPLAY "Report run -M-onthly -Q-uarterly or -A-nnual?"
ACCEPT OP-INPUT
END-IF.

but the TEST AFTER mystifies most Cobol programmers, so I prefer the
former.

I don't use GO very often, perhaps once in 5000 loc. I remove far more
GOs than I add.

I think that the reason for not using 77 levels is actually simpler than
that.
In order to make programs more self-documenting many of the items that would
normally be 77 level items are grouped together by their type. For example WS-
SWITCHES, WS-COUNTERS, WS-ACCUM-FIELDS would group those otherwise
independent items under an 01 level group. In a large commercial system this
is commonly the case, and it is quite convenient when it is implemented
consistently. I frequently use 77 level items for debugging purposes, and
remove them when debugged, with the added convenience of a compile making
SURE every last trace of that debugging code is gone.
Perhaps as a result of this, when I see a 77 level item in a production
program I immediately think something might be fishy. The sames goes for
uncommented out blank lines.
Experience is a harsh teacher, especially at 2 AM.
It's really all a matter of convention, but I think that widely accepted
practice goes this way.
