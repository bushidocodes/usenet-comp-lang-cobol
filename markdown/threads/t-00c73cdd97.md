```
First and foremost I must apologise to all whose time I wasted with my
useless example yesterday; two things need to be clarified:

1) the example was a copy sent to a friend to demonstrate to him how
remediation could introduce subtle bugs; it was *not* an attempt to
demonstrate precisely how the NJ Woe came about.

2) In my haste and sloppiness I gave an example which would not compile
because of unmatched scope-delimiters... I tried to Make Things Neat by
adhering to COBOLII/COBOL '85 standards rather than Maintain Accuracy by
giving an example of code which would willy-nilly combine the various
aspects of COBOL and COBOLII/COBOL '85 standards.

The only defense I can ofer is a variant of the Brooklyn Bridge defense
('everyone else is doing it'); I have seen examples in textbooks which
would not compile... but their sloppiness is no excuse for mine.

The corrected example is below:

--begin corrected code

MOVE MFCS-CUST-NO-01 TO WS-CUSTNO.
IF INDEPENDENT-CUSTOMER
    PERFORM A53102-ALLOCATE-LOT THRU A53102-EXIT
    PERFORM C88092-CALC-CUBES   THRU C88092-EXIT
    PERFORM E35702-FIND-SHPWGHT THRU E35702-EXIT
    INSPECT WS-CUSTNO TALLYING WS-PERFORM-SUB FOR ALL 'QZ'
    IF WS-PERFORM-SUB > 0 AND WS-DT-REC-ADDED < WS-CUTOFF
        MOVE 'Y'  TO SPECL-CUST-FLAG
        PERFORM B28095-PROCESS-SPECUST THRU B28095-EXIT
         VARYING SUB1 FROM WS-PERFORM-SUB BY 1
          UNTIL SUB1 > WS-PERFORM-SUB
    END-IF
END-IF.

MOVE MFCS-CUST-NO-01 TO WS-CUSTNO.
IF INDEPENDENT-CUSTOMER
    PERFORM A53102-ALLOCATE-LOT THRU A53102-EXIT
    PERFORM C88092-CALC-CUBES   THRU C88092-EXIT
    PERFORM E35702-FIND-SHPWGHT THRU E35702-EXIT
    INSPECT WS-CUSTNO TALLYING WS-PERFORM-SUB FOR ALL 'QZ'
    IF WS-DT-REC-ADDED-YY < 40
        MOVE 20 TO WS-DT-REC-ADDED-CC
    ELSE
        MOVE 19 TO WS-DT-REC-ADDED-CC
    END-IF
    IF WS-CUTOFF-YY < 40
        MOVE 20 TO WS-CUTOFF-CC
    ELSE
        MOVE 19 TO WS-CUTOFF-CC
    END-IF.
    IF WS-PERFORM-SUB > 0 AND WS-DT-REC-ADDED-CCYYMMDD < WS-CUTOFF-CCYYMMDD
        MOVE 'Y'  TO SPECL-CUST-FLAG
        PERFORM B28095-PROCESS-SPECUST THRU B28095-EXIT
         VARYING SUB1 FROM WS-PERFORM-SUB BY 1
          UNTIL SUB1 > WS-PERFORM-SUB.

--end corrected code.

(also note that if one so desires the two END-IFs which terminate the
unremediated code can also be removed if a period is placed at the end of
the PERFORM... I will leave it to the readers, kind as they have been in
showing me my error, to debate the 'fairness' of doing this or not as it
could be construed as a real Dialect Change between the samples)

Once again, my apologies to all; if nothing else I have once again
demonstrated that these free examples are worth exactly what one pays for
them.

DD
```
