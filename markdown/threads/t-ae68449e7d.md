It is unlikely that an exception would be raised as the programmer error 
may not be detectable by the system.  Your example is one of the reasons 
that explicit dynamic allocation of memory should be avoided.  At the very 
least the allocation should be based on 'sizeof recordnew' rather than a 
literal.  Some later changes may add to the record definition without the 
programmer updating the literal, or even knowing that it is needed.

Memory overwrites are one of the hardest bugs to find because they can 
cascade until an obvious result which may be far from where the actual 
defect is.
