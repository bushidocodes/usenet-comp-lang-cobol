If you can expect Excel to be installed on the host computer that you want
this task accomplished on, you can use Excel via OLE Automation in Acucobol
5.0 to do this. This is pretty neat, you will then be able to control Excel
from within Acucobol. If you choose to, the user will not see Excel when you
do this. Using Excel this way, you can access each particular cell of the
spreadsheet as you like.

Here is a Word example using OLE Automation from Acucobol to print a preview
of a xls file:

      * Create "Word.Application" object
             create @Application of @Word
                 handle in word-app.

      * Make Word invisible
             modify word-app @Visible = 0.

      * Open file
             modify word-app @Documents::Open("demo.doc",0,1)
                 giving word-doc.

      * Print Preview
             modify word-doc @PrintPreview().
             modify word-app @Visible = 1.

      * Destroy the objects
           destroy word-doc.
           destroy word-app.

If your host computer does not have Excel, you will need to check out if
there are any 3rd party companies that offers an xls "engine" either as a
DLL or as an ActiveX. I am certain there are, yet I don't know of any.

OLE Automation is a technology not specific to
any lanaguage at all.

A language should not have "extra" syntax just so
that it supports the latest technology.
If this were the case in 5 years time we would
end up with a language thicker than the bible!

Instead, as OLE Automation is OO based it should
only be called from OO based "standard" syntax.
For example:

If I wanted to create an instance of my Word
program then I would:

   invoke wordapp "new" returning wordServer

Then get the current document:

   invoke wordServer "getActiveDocument"
returning  theDocument

Then print it:

   invoke theDocument "PrintOut" using by value 0

This is the same as calling a file handler to get
a record:

   invoke Filehandler "new" returning filehand

   invoke filehand "getRecord" returning filehand-
Rec

   invoke filehand "print" using filehand-rec
