First thing off, I've found it is not easy to try and obtain a  straight one for
one relationship between the syntax of different languages. Currently I'm
reading up on Visual Basic to see what they have to say about components. The
book has a chapter on creating classes - believe me, although the theory is the
same, creation of VB classes has no bearing on how you set them up under COBOL -
OK so COBOL is verbose.

Your query about functions. Currently can be handled by both Fujitsu and Micro
Focus (Net Express) OO compilers, when writing methods contained in classes.

The J4 specification for COBOL 2002 (due December), has the following :-

- UDFs(User Defined Functions)
- OO - Classes containing methods

UDFs are another way of expressing Methods without getting into OO. May not be
quite what you are after but :-

*>----------
Method-id. "ImpToMetric".
*>-----------------
Local Storage Section.
*> 01 ImpToMet   CONSTANT    value 25.4.
*> I'm not using above line, just included to show you how it can be used
Linkage section.
01 ImpValue                         pic 9(02)v9(03).
01 MetValue                        pic 9(03)v9(02).
Procedure Division using ImpValue returning MetValue.
 compute MetValue rounded = ImpValue * 25.4
End Method "ImpToMetric".
*>----------------------------------------

UDFs parallel the above format.

Note that COBOL 2002 allows for the '25.4' to be stored as a CONSTANT, (cast in
stone), either in Global Working Storage or within a method's Local-Storage
Section. At the current time in Micro Focus I can use a Level 78 either globally
or in the method Local Storage :-

    78 ImpToMet                value 25.4.

Working in OO you'll appreciate that I don't have a need for UDFs, but they will
be available to non-OO developers and you can build libraries of UDFs.

You could emulate UDFs currently by calling sub-programs, particularly useful if
you grouped a series of 'like minded' functions within one non-OO program and
were using entry points. Like minded functions - that's the whole point about OO
classes, isn't it, coupled with reusability ?

Jimmy, Calgary AB
