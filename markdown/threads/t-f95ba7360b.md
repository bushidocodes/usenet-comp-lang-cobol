I just ran into the oddest thing.  IBM's compiler prohibits a
configuration section in nested programs.

I was trying to include a XML building utility program as nested to
improve performance, it contained:

 CONFIGURATION SECTION.                                     
 SPECIAL-NAMES.                                             
     CLASS NAME-CHAR-START IS ':abcdefghijklmnopqrstuvwxyz' 
                              '_ABCDEFGHIJKLMNOPQRSTUVWXYZ' 
                                                            
     CLASS NAME-CHAR       IS ':abcdefghijklmnopqrstuvwxyz-'
                              '_ABCDEFGHIJKLMNOPQRSTUVWXYZ.'
                              '0123456789'.                 

Of course, the compiles failed.

Is there another way to get the utility of character classes without
using Special-Names?  Anybody have good ideas or cute tricks?

Thanks much,

Joe Zitzelberger
```

Joe,
   The restriction on Configuration Sections *NOT* being in nested programs
is from the ANSI Standard.

Did you see the sentence at:
  http://publibz.boulder.ibm.com/cgi-bin/bookmgr_OS390/BOOKS/IGY3LR10/4.1

which says (in part),

"The entries specified in the Configuration Section of a program apply to
any program contained within that program."

Unless you want to use the SAME name with different meanings within
different nested programs, you shouldn't have any problem just putting them
in the outermost program.
```
