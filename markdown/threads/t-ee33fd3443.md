kajanz@npt.nuwc.navy.mil (BoMoF) wrote:

>I am reading COMP-2 fields but from a C++ application, so I was trying
>to find their format but could not?  I can get the binary
>representation of it but I don't know which fields are the mantissa,
>which are the exponent, and which represent the signs.  In a 64bit
>COMP-2 (dpfp) field, which bits would be which?

IEEE 754 uses 1, 11 and 52 bits, reading left to right. Subtract 1023 from e,
add 1.000 to m. That's almost surely a native double in your C++. 

Old IBM uses 1, 7 and 56 bits. Subtract 64 from m.  New IBM uses IEEE 754.
Old IBM is used for submitting files to the government under standards written
long ago e.g. FDA New Drug Applications.

To convert Old IBM to IEEE, follow instructions here:
http://en.wikipedia.org/wiki/IBM_Floating_Point_Architecture
