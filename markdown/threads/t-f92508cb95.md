I have the same problem in AcuCobol (not with the doc) and I am solving
it with a very little assembler program. The program receive, when called
from the Cobol-program, address to a four-bytes area. The assembler
program moves the four bytes to AL, AH and DX, and then an INT 14,
After the INT 14, the program moves AL and AH to the four bytes area
and the control is returned to my Cobol program.

Maybe it is a litte step towards the solution.
