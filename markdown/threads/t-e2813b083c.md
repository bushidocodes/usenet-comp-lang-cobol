The book is using a non-conforming extension. To quote IBM:

"To code a table, give the table a group name and define a subordinate
item
(the table element) that is to be repeated n times:

     01  table-name.

         05  element-name OCCURS n TIMES. "
"
The OCCURS clause cannot be specified in a data description entry that:

   * Has a level number of 01, 66, 77, or 88."

Just change 01 TABLE-STATE-RECORD OCCURS 50 TIMES.
to:
01 TABLE-STATE-RECORD.
         02       OCCURS 50 TIMES.
             05   TABLE-STATE-CODE       PIC XX.
             05   TABLE-STATE-NAME       PIC X(2).

One of the rules is "read what the error message says".
If an occurs cannot be used with an 01 level, try making
it an 02 level.  That rule, BTW, I am told is standard. Many
compilers do allow occurs on an 01, so the example
is non-conforming, but probably runs on most compilers.

I find it annoying, as I do not see any need for the 01 level rule.
However, if you go back a few weeks in this NG, you will find
a long argument over that very point.
