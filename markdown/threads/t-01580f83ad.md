```
The error message makes sense: you can never REWRITE a record unless it was
first READ and that READ was the most recent I-O operation against that
file; but it sounds like you are doing that.

So... I don't think you can REWRITE a file which is SELECTed with ACCESS IS
SEQUENTIAL. (The ORGANIZATION can be sequential). Without looking at the
SELECT or the code, I can't tell if this may be causing a problem.

So check your SELECT; and if you have to change that, you've got some other
work to do as well.
```

This isn't "Exactly" accurate.  One can rewrite and indexed file
record, using Random or Dynamic access mode, even when one has not
first read the record.
