COBOL is the name of the language, all dialects, all releases, all
standards.

COBOL II is the name given to the 1985 standard as implemented by
several compiler makers.  Also called COBOL 2 or COBOL 85 by other
compiler makers.

LE COBOL or COBOL LE is  the current version of COBOL as implemented
by IBM for operating systems which use IBM's Language Environment
product.

From the fact that the poster mentions "COBOL LE" - my guess is that he is
talking totally about IBM (probably IBM mainframe environment).  Certainly
"COBOL II" is *not* "the name given to the 1985 standard" as Stephen writes,
but is a common (mis-)name given to IBM's compiler that was FIRST a '74
Standard compiler, then an '85 Standard compiler. The *actual* (official)
"relatively" modern list of IBM compiler names (for the MVS environment, VM
is a little different VSE is more different<G>) is:

  ANS COBOL
  OS/VS COBOL
  VS COBOL II
   IBM SAA AD/Cycle COBOL/370
   IBM COBOL for MVS & VM
   IBM COBOL for OS/390 & VM
   IBM Enterprise COBOL for z/OS & OS/390

Many of these have had different "releases" and/or versions.   Some
supported multiple ANSI Standards (via compiler options, e.g. LANGLVL and
CMPR2).  The compilers that are listed AFTER "VS COBOL II" use LE (Language
Environment) as their run-time; the compilers up to VS COBOL II all provided
their own run-time component.  Only the last two are still officially
supported (as compilers) by IBM anymore.  Although the object code from all
of them is still supported in an LE environment.

According to my research of the IBM COBOL migration guides from the
Redbooks, ...

COBOL 1968 - (D)OS/VS COBOL
COBOL 1974 - (OS)VS COBOL
COBOL 1985 (i.e. COBOL2) - VS COBOL II, COBOL/VSE, COBOL/LE/VSE (per
http://www.redbooks.ibm.com/pubs/pdfs/redbooks/sg244277.pdf)

COBOL/VSE and COBOL/LE/VSE have many extensions over any other version from
IBM.  COBOL/LE/VSE v.4 is the current enterprise product.

The best way to determine what can do what is to look at what libraries are
required to run and viewing the compile maps.  The current level of COBOL
from IBM can be run recursive, re-entrant, interfaces with the web, is CORBA
compliant with SOM objects, runs in a mixed environment, (both language and
OS)...

If corporations would use tools like Visual COBOL, new programmers could
easily move to supporting COBOL environments and old programmers could
sharpen their skills.
