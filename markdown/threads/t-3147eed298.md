Not really, as "OO" is a function of the compiler, not the platform API.

The 'use of Win32 APIs' implies you are inserting lines like..

  CALL windows-function USING xxxxx, xxxx, xxxxx, xxxxx RETURNING yyyy

.. into your procedural code.  Still looks pretty procedural, huh?

While you "can" access some (much?) of the Windows API through the Component
Object Model (COM) interface used in "OO" programming, your source code
remains procedural. (And you'd best be _real_  comfy using the COBOL POINTER
and PROCEDURE-POINTER data types to even give this a shot.. because the
basic COM interface works by returning pointers to procedure-pointers).

If you want "OO programming " on your desktop,  you are best off using a
compiler designed to support "OO" development.
