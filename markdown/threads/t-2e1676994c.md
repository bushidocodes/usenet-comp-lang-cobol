In article <357··.@in··y.net>, Martin Meadows writes:

› STRING SQL-BMH(SQL-CTR)
›    DELIMITED BY SPACE
›    INTO SQL-STATEMENT
›    WITH POINTER PTR

The numeric field PTR holds the byte number within SQL-STATEMENT that will be
the starting point for receiving characters. (So PTR must be set to the correct
location before this statement is executed.) If PTR = 1, the first character of
SQL-STATEMENT will be the start of receiving the characters.

The SQL-CTR occurance of SQL-BMH will be copied, character by character, to
SQL-STATEMENT. The copying stops as soon as a space is encountered in
SQL-BMH(SQL-CTR) (the space character itself is not copied because SPACE is
specified as the delimiter), or all characters of SQL-BMH(SQL-CTR) have been
transferred if there is no space in SQL-BMH(SQL-CTR), or SQL-STATEMENT is full.

PTR is updated to point to 1 past the last character in SQL-STATEMENT that
received a character.

Any characters in SQL-STATEMENT that had not received characters will remain
unchanged.

Mark A. Young
