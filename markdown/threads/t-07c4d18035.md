I assume that you are using Fujitsu COBOL - as this same question comes up
every few months with their product (which I have already told Fujitsu is
NOT ANSI conforming because of this problem).

How to fix,

 Remove the word "RANDOM" after the words "ASSIGN TO".

"RANDOM" is not used following "ASSIGN TO" when it IS used correctly - so
the compiler thinks that is a "logical file name"  (Something that you SET
to a physical file name outside your program).

Once RANDOM is removed, you are left with "ASSIGN TO RANDOM "SORTFILE" ...
which is correct (when you follow it with a period).

Then what you need to do is delete

              ORGANIZATION IS LINE SEQUENTIAL
              FILE STATUS IS FSTATUS.

BOTTOM-LINE: the SD file is JUST a work area.  Do not try to allocate it as
a "real" file.
