```
Folks,  

I am in the process of migrating from WANG to HP-UNIX using MFCobol. 
The process is going very well to date, but I have hit a small
problem.

On the VS there was a 2024 byte Line Sequential file that we output to
tape and sent to clients.  This file had PACKED fields in it because
the longest sequential record supported on the Wang VS was 2024. 
However we had a Wang utility program to move the file to tape and do
ASCII-EBCDIC translation and we could identify packed and binary data
areas to the program so that the translation process took this
information into account and translated accordingly.

Now however our output to 3490 program permits us only 2 options -
Translate to EBCDIC or DON'T TRANSLATE.

Now I know the easy fix is unpack the file, but this affects MANY
customers that would then have to make changes themselves in both
their programs and their JCL to offload the tape.  Therefore this
seems not to be an option.

I know that we can use the FD CODE-SET clause in MF to cause a
linesequential file to be output in a different character set, but I'm
having trouble getting the pre-requisites straight.  There are issues
with the ALPHABET clause that have to be taken into account and the
documentation is well... we've all been there haven't we! :-)

If anyone can offer a little assistance in getting me going with this
minor problem it would be very much appreciated.

R Boback
Quincy Mass.

```
