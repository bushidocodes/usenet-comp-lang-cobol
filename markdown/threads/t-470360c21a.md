I can help YOU figure out what the problem is. (I am not trying to be
obscure, but without some information that you can gather yourself, this
would be medium hard to solve).

For those reading this who don't know, the IWZ message means that this is
IBM VisualAge for COBOL (or the AIX equivalent).

What your problem is pointed to by the phrase "data exception". On an IBM
mainframe, this usually ends up with an S0C7 - on the PC, you get the type
of message that you have listed. The chances are that one of your numeric
fields has non-numeric data in it - and this caused the "data exception".

The part that you are going to need to work on is what the message means
when it says, "from offset 0x290 of routine PROG5." If you compile your
program using the LIST or OFFSET compiler option, it will tell you at what
offset each verb of your source program occurs. Once you have this output,
you will be able to locate the place in your code referenced by the message.
Once you know that you should be able to look at your logic and figure out
which numeric field has gotten bad data - and what you need to do to solve
this.

Gook Luck - and I hope this helps.
