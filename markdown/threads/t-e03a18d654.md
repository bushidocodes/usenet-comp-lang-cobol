```
Please find hereafter some Questions relative to IBM Visualage for COBOL v 2.0:

1) In the documentation ('Visual Builder User's Guide'-'Working with parts in the visual
builder window'-'Creating a new part'), it is said that it is possible to create a part of
type 'Class Interface Part',but in the Visual Builder, there is only two choices Visual and
Non-visual part. Is it a bug in the documentation or the installed product is corrupted or
???

2) In the same book, it is said that it is possible to make a part inherits from another
part, but the list box of the visual builder accepts only predefined values (CFrameWindow,
CCanvas or CNotifier). Is it a bug in the documentation or the installed product is
corrupted or ???

3) How is it possible to integrate SOM classes (Source in COBOL v1.2)on the visual builder?
It seems impossible to create 'Non-visual' parts from 'real' SOM classes. For example,
, it is impossible to assign a metaclass or a specific parent class (see question 2) in the
definition of a class. It seems that the mysterious 'Class Interface Part' (see question 1)
has been designed for that, but where is it?. How can he integrate those classes and use
them on the visual builder? Writing a .vce file seems very painful because currently in the
SOM class there is only one get/set method for a structure that holds all the attributes of
the class (And not one per attribute as the Visual Builder need it). The visual builder
should help to create set/get methods for all individual attributes.

4) With VACOBOL v1.2 on OS/2, there was a 'Package' option on the project menu. There is no
such option on VA COBOL v2.0 on NT. Is there an NT product that can do the job? Or at least,
somewhere in the documentation, the list of objects that compose the 'run-time' part
of VACOBOL for NT?

5) It was said that a 'Report-Writer' helper will be included with VA COBOL. It is not the
case with version 2.0 In which version will this be included? Is it a separate product? How
people create reports with the current version of VACOBOL on NT?

Many thanks in advance for your answers.

BTW, I would like to communicate with others VA COBOL v2.0 users to exchange experience.
```
