If you make your file line sequential in the select clause
then you will get one record per line.  Make sure your record
size is big enough for the longest line.  Shorter records
will be blank filled on read.

What you have described here indicates you want LINE SEQUENTIAL organization. 
Unless specifically stated in the SELECT statement, the files
are assumed to be RECORD SEQUENTIAL.
As for the variable length issue, Your record definition does not indicate any
field that occurs 'n' times depending on 'xxxx-field'.
