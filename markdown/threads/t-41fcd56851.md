the microfocus cobol workbench to microfocus netexpress.

If i use $SET SQL(DB2) in the source and i build the

Our goal is a massmake with batchfiles like we did it with
the cobol workbench.

some compiler directives:
-------------------------
CONFIRM
EARLY-RELEASE

LITLINK(1)
CASE
ASSIGN "EXTERNAL"
OUTDD "sysout.txt"
AREACHECK

DEFAULTBYTE "0"
ARITHMETIC "VSC2"
NOQUERY
SQL(DB=Y, NOPRE, NOINIT, NOACCESS, BIND, COMMIT=1, PASS=X.Z, FORMAT=ISO)

Hi Jurgen,

If I remember there was no doucmentation with V1 of Net Express of using DB2
UDB.

This support for DB2 was introduced in version 2 of Net Express with the
documentation downloadable from the Net Express WebSync site. In version 1
the only SQL solution was to use the OpenESQL directives to access DB2 via
ODBC.

My advice here would be to get V3 (the latest version).

The format of the DB2 directives have changed from Workbench to Net Express
they now use the form DB2(...) eg

      DB2(INIT=PROT BIND COLLECTION=MYSCHEMA)

Hope this helps
David.
