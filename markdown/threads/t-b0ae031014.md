My understanding is that it is not supported (you may have one COBOL thread but
not more than one) - yet; thread safe support is planned for a later release.

That said, I haven't tried it. You can run some experiments and see what
happens. The problem, of course, is it may be one of those situations where
things go wrong with no apparent pattern, so the application starts out being
flaky and cannot get better than that until the compiler generates thread-safe
code.

Hope this helps.
