When Assembler is the high-level module, this is a problem because everytime
the COBOL II module is called the COBOL environment has to be
created and it is terminated when the last COBOL module returns.

To get around this you have 2 options:

1. Make the high-level module COBOL II - This can be just a call to your
Assembler module. That way when the Assembler module gets control, the COBOL
II environment is already established.

2. In your Assembler module, load and call IGZERRE, which will
pre-initialize the COBOL II environment. This needs to be done only once and
will have the same effect as having a high-level COBOL II module. The COBOL II
environment will not be created/terminated each time a COBOL module is called
within your Assembler code. BUT...You can only do this if your code is
compiled RENT and linked as REUS. Otherwise it will not help you much and you
may even get S0C4. The COBOL II Application manual has a small section on this
(check performance or inter-language calls.)
