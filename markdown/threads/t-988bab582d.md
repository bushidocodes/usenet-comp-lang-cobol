What are the significant differences between ISO/ANSI COBOL and a
particular vendor's COBOL?

The reason I'm asking is that I've been turned down for several COBOL
programming jobs because the clients wanted *Micro Focus* COBOL
programmers.  I've developed programs for Burroughs, Honeywell, and DEC
COBOL compilers and the COBOL code differences seemed to me to be minor.
The really big differences were in the system calls, but that's a
different bucket of worms.

Is Micro Focus COBOL so different?

I find the same problem with databases: "the qualified candidate will be
proficient in embedded Oracle".  Excuse me, but doesn't Oracle use SQL?
Sure, there may be a couple of bells and whistles added (that make
portability a pain), but I'm pretty sure "embedded Oracle" code looks
exactly like SQL.

So if you have a couple of minutes, wander over to HR and ask to look at
the IT job descriptions used for recruiting.  Consider changing the
phrase "require Micro Focus COBOL and embedded Oracle experience" with
"require COBOL and SQL experience (Micro Focus and Oracle a plus)" or
some such.

The COBOL compilers that you mention are all "mainframe" type compilers.
Micro Focus is PC/UNIX "oriented".  There are a "reasonable" number of
differences in "mind set" in programming for the mainframe as opposed to the
PC/Unix (Workstation).  That *might* be what is bothering them.

Specific "extensions" are common in all COBOL compilers.  IBM mainframes and
Micro Focus and Fujitsu all have their own "peculiarities" as far as the
language goes - and as someone mentioned the "programming
environment/interface" is QUITE different.  You might want to pick up one of
the free or low-cost workstation compilers/environments and "play with it" a
bit to see what it is like.
