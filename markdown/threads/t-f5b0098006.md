: >	By using GO TO explicitly to exit PERFORMed code, all that is
: >	required is that you PERFORM SECTIONs and not paragraphs.
: >	Then you can have a paragraph as follows:
: >
: >	EXIT-THE-SECTION.
: >	    EXIT.
: >
: >	at the end of *every* SECTION in your program.  This will allow
: >	you to copy most any routine with out having to fret about ugly
: >	GO TOs used for exiting the code.   Please note that the *real*
: >	exit point of a PERFORMed SECTION is just before the *next*
: >	SECTION definition -or- end of source.
: >
: >	This also alleviates the need for those not so pretty 
: >	PERFORM ... THRU ... statements.  And, of course, you can
: >	still PERFORM specific paragraphs within the section.
: >

: The only problem with this is is a programmer must be careful when 
: adding, deleting or moving sections of code in the program. The best 
: thing to do is avoid GO TO and PERFORM THRUs. Just use a basic PERFORM, 
: or if you must show where the end of paragraph is use an END-PERFORM. 
: That way if someone doing maintenance on a program you may have 
: written, he won't have to worry about inserting code in the wrong place 
: or deleting code that should be left alone.
