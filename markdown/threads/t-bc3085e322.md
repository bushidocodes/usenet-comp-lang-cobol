The 8-step procedure you describe does the neccessary steps, but you can do
it all with a few AMS commands in a single job step, supposing that you
have the records available in a sequential dataset (sorted on primary key).
 No JCL DD neccessary.

1. DEFINE the base cluster
2. REPRO from the sequential dataset into the base cluster
3. DEFINE the alternate indices and the paths
4. BLDINDEX from the base cluster into each path (or alternate index)

> As for using it in CICS, I'm stilla bit baffled. DO I have to 
> define each path in the FCT? Or must they be pre-allocated somehow? 

After the file has been loaded you can use it with CICS.  Each path is seen
as a separate file by CICS, so each path that you intend to use must be
defined in the FCT.  Otherwise the UPGRADE option will see to that each
index is synchronized as records are written to the base cluster.

Gunnar.

I'm sure once you get some more experience you'll understand the flow &
streamline things, e.g., develop JCL that will REPRO the data to a flat
file, DELETE & DEFINE the cluster, REPRO the flat file into the new
cluster, define the AIXes, build the AIXes, define the PATHs. You can

last step which deletes the flat file (assuming you want to free the

allocate the flat file (,CATLG) and be confident that it will run.
