Hi *,

Suppose we have the following:

SRCFLD PIC X(08) VALUE SPACES.
TARGETFLD PIC 9(04)V9(04) VALUE ZERO.

I am searching now for a short and elegant way of converting the contens
of the field SRCFLD to TARGETFLD.

Normally SRCFLD contains a number with leading or trailing blanks.

First of all we have to check if the contens is numeric (spaces should be
acceptable) or not (how best ?).

If the number is greater than 4 digits before or after the decimal point
an error message should be written.

Well, o.k. I could write an appropriate routine with multiple PERFORMs and
IFs. But I feel that there must be way of doing that more comfortable.

It could be by UNSTRINGing the TRGFLD in a PRE and a FRACT part:



....
01 NUM-ALPHA
05 PRE PIC X(08) JUST RIGHT.
05 FRACT PIC X(08).
01 NUM-NUM REDEFINES NUM-ALPHA PIC 9(08)V9(08).
...
UNSTRING SRCFLD DELIMITED BY '.'
PRE DELIMITED BY SIZE COUNT PRENUM
FRACT DELIMITED BY SIZE COUNT POSTNUM
END-UNSTRING
....
INSPECT NUM-ALPHA RELACING ALL ' ' BY '0'
MOVE NUM-NUM TO TARGETFLD.
....




Is there a better wa of doing this ? Are there any standard cobol source
sites (URL= ?) in the web ?

Thanx in advance

Michael

Simply redefine SRCFLD:
SRCFLD-EDIT PIC Z(08)
and another
TARGETFLD-INT PIC 9(8)

Then in the procedure division:
MOVE SRCFLD-EDIT TO TARGETFLD, TARGETFLD-INT
IF TARGETFLD-INT > 10000

END-IF

Should work with any ANSI 85 compliant COBOL implemented at the High
level. Some COBOL systems try to take the edit picture into account
during de-editing, so you might need to run a test or two (the standard
is silent on how the de-edit is accomplished; most vendors allow
reasonable free-form).
