pages to print but it will print out the listing file or any other selected

manufacturer regarding the handling of ASCII FF characters.

One last thought on this mess to go along with your solution of using
wordpad.

Anytime that I don't open up a line sequential file in its own application I
open it in wordpad because wordpad doesn't add any formatting unless you
save it.  If you save it as a .txt the formatting of a file will not be
retained only when you save as an .rtf file will the formatting be retained.

Files that are line sequential such as a report or a cobol listing or even
cobol source files can always be opened in wordpad and the characteristics
of the file will be retained.  This is not true if opening those same files
in word, WordPerfect or other word processing applications because these
types of applications will add formatting characters that have changed the
nature of the line sequential file.

What you want to remember is that any file that can be opened in an editor
like the dos editor on a windows system or vi on a unix system will open up
fine in wordpad.
