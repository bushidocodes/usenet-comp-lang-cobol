If you have access to a compiler (other than a Micro Focus one - which I 
have already tested) that support "ANS'85 conformance flagging" (e.g. 
Enterprise COBOL with FLAGSTD(H) ),  I am pretty certain that Fujitsu also 
has such a flagging facility.

I have another test program that I would like to know what compilers do 
with.

Can you download the program from:
  http://home.comcast.net/~wmklein/DOX/alph85.cbl

and try compiling it with ANS'85 flaggin turned on.  I am interested in what 
(if any) flagging (or error) messages the compile gets.  If you do NOT get 
any flagging messages or error messages, could you "uncomment" the last MOVE 
and display statements and see if this gets flagging or error messages.  (I 
think it will PROBABLY get error messages).  Please remember to "turn on" 
seeing informational and flagging messages - If your compiler doesn't 
default to doing this.

You can run the program, but I am not all that interested in the run-time 
results (at lest yet).

You can reply in this forum or to me off-line; whichever is easiest for you.
