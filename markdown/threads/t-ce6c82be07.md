Without the source code describing the layout of the records in the
database, you are in BIG trouble. You can guess, but it is un-
reliable, particularly if the file is an indexed sequential file or if
the database uses fields that are anything other than ordinary text.

Do a web search on this newsgoup for the postings of Michael Mattias.
These discuss the formats of cobol data files (such as they are) and
the problems involved in moving data out of a cobol data file. At
least you don't have to worry about character translations (EBCDIC to
ASCII)!

The optimum solution is to find a COBOL programmer who will write a
COBOL program which reads the existing database and extracts the data
to a line-sequential file in which all of the fields are ordinary
text. This program needs to account for signed fields, implied decimal
points and other stuff too.
