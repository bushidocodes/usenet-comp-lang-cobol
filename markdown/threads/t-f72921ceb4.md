From one of my "usually reliable sources",

"Since he apparently are working in dos and unix only, I really have no
other suggestion than that he either link in the C library functions
directly or creates a wrapper API that he links with. He would then have to
provide the HTTP functionality himself which will be quite a challenge,
unless there are thirdparty libraries available, which I am pretty sure
there are. Definitely for Unix. If he had been developing for Windows, the
answer is surprisingly easy, if he wants to rely on Internet Explorer.
Because Internet Explorer comes with a high end HTTP api, which should serve
his needs well. The API he should look for is WinInet, if he goes to the web
he will find all he needs here:

        http://msdn.microsoft.com/workshop/networking/wininet/wininet.asp

As he will notice, this does not only apply to HTTP, but also FTP and
Gopher.
Problem is of course if he must be on Unix and Dos, in that case I regret to
say that I have no exact answers other than the proposal above."
