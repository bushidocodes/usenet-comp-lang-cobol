The syntax for DISPLAY is:

DISPLAY identifier/literal ... [UPON mnemonic-name/environment-name] [WITH
NO ADVANCING]

To print a blank line using display, try the following:

DISPLAY ' '

(if your system requires double-quotes, use DISPLAY " ")

The character in the quote is just a space as a literal. Another option
would be to set up a working storage field with something like:
77 WS-SPACE PICTURE X(01) VALUE SPACE.

Then use:

DISPLAY WS-SPACE

But I think that would be redundant since I think most standard COBOLs
should allow literals on a DISPLAY line.

Hello KL,

Use DISPLAY ' ' or DISPLAY SPACE

If you are using screen description entries with a Format 2 DISPLAY
statement, consider setting up one with a line containing the BLANK
LINE clause so that it will come out that way when you display the
whole screen description entry.

If you have access to the manuals for your compiler, then there are
probably examples in the programmer's guide.  If you haven't got
manuals, then you can probably get them online from the supplier, many
are free downloads.

Although it doesn't really matter in this case, it's a good idea when
posting here to specify the compiler you are using with its version,
the hardware and operating system that you are running it on and often
the compiler options in effect.

Regards

Robert
