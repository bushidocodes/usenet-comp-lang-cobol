The most used way to access a relational is through embedded SQL statements.
This is a technology used for a long time in Cobol and you will need a
precompiler in order to access the specific RDBMS you want to use.

If you are developing Windows applications, you might want to use ODBC, so
your application can access (almost) any relational with the same code. ODBC
statements are quite tedious to write and are documented (usually) in C/C++,
so you have to do the translation to Cobol data types.

If you are using Net Express from Micro Focus, you can still use embedded
SQL statements that are automatically translated (transparently) to ODBC
calls. Also, Net Expess allows you to use the latest Microsoft technology to
access data : ActiveX Data Objects (ADO), but this needs a fairly good
knowledge of Object Cobol.

Hope this helps.

Panos
