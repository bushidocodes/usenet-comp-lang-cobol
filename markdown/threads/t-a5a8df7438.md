```
Unless there is something real specific about your compiler, user-defined
functions are not possible in ANSI COBOL 85. (I think Mr Klein said
something about UDFs being in the draft for the next standard).

Closest you can come is to write a nested program, something like...

PROGRAM-ID. Main.
....
WORKING-STORAGE SECTION.
01  MyVar       PIC S9(09) COMP.
PROCEDURE DIVISION.
     MOVE +10 TO MYVAR
     DISPLAY MyVAR
     CALL "MyFUNC" USING MyVar
     DISPLAY MYVAR.

    STOP RUN.

END PROGRAM MAIN.
PROGRAM-ID. MyFUNC.
...
LINKAGE SECTION.
01 DATA-IN     PIC S9(09) COMP.

PROCEDURE DIVISION USING DATA-IN.

     ADD +1 TO DATA-IN
    GOBACK.

END PROGRAM MyFUNC.

(BY REFERENCE assumed for CALL and USING).
```

```
Actually, since the '85 Standard, COBOL *does* fully support "nested
programs" which does allow you to have "local" and "global" variables (as
well as EXTERNAL items which available across  non-nested programs).

Having said that, if you are interested in "user-defined functions" in terms
of referencing a "procedure" and getting back a "returned value" - then it
is true that Standard COBOL doesn't inlcude that yet - although many
implementations do already provide it.  (For example - going back to the
original question - IBM does support RETURNING values on S/390 and on
Workstations - but not on OS/400.)
```

```
Hey Fabio-

COBOL doesn't really have functions like other languages, so you really
don't have to pass a value to a paragraph in COBOL like you do in C, for
example.  In essence, all the variables are 'global', to use an
object-oriented refenrence.

Example:

WORKING-STORAGE-SECTION.
01   MY-NUMBER      PIC 9(02)  VALUE 0.

PROCEDURE DIVISION.

000-MAIN.

        PERFORM 100-INIT.
        PERFORM 200-LOOP UNTIL MY-NUMBER = 10.
        PERFORM 300-END.

        STOP-RUN.

100-INIT.
        DISPLAY MY-NUMBER.

200-LOOP.

        IF MY-NUMBER < 10
            PERFORM 250-ADD
        END-IF.

250-ADD.

        ADD 1 TO MY-NUMBER.

300-END.

        DISPLAY MY-NUMBER.

So. the 250-ADD paragraph acts like a function, in that it manipulates a
variable in a way that is useful to the other paragraphs.

Hope this helps-
```
