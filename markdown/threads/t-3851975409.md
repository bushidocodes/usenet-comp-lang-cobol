- **Message-ID:** `<6n2l7i$4i2$1@news.worldonline.nl>`

```

Hi there,

I have a problem which is keeping me busy for a week or so, maybe I am
looking in the wrong places. However I noticed that questions put in this
newsgroup mostly are answered by specialists, so I hope someone has a
solution for me, because I can not believe I am the only one with this
problem.
Okay, let me explain, my company is busy implementing the Microfocus
Workbench on Windows NT 4.0 for offloading purposes. We have installed
version 4.0.32, MCO 3.0 (no party in this problem) and HCO 3.0.07. The
database we are using is DB2 5.0 (UDB), connected to MVS DB2 4.1 via DB
Connect (formerly known as DDCS2, so I am told). Now every programmer has
its own set of tables, recognized by the qualifier ("userid.table") we also
have one central set of tables ("project.table"). When a programmer checks
his program (non qualified sql) he creates a package on the mainframe in a
collection identified by his userid (last friday I think I have found out
how to redirect it to another collection (SQL(COLLECTION=xxx)), but I
haven't tested it yet). By default the package uses the userid of the
programmer as the qualifier (that's good !!). Now, when other programmers
run the program (.INT or .GNT) they have to rebind the package to use it,
but because we would like it to be possible for more then one programmer to
run the program at the same time (with there own set of tables) they have to
rebind it into their own collection. And that's the part that really hurts
because Microfocus does not give me the opportunity to select a collection
to use when running the program. It seems to me that the collection- and
package-name are stored in the .INT/.GNT files. Is there anyone who can help
me with this problem ??? The helpdesk from the distributor in Holland (VAC)
was not able to help me right a way, they are looking into it, but everybody
knows that that can take a while.......................
If you can help me or not thanks for trying !!!!

Herman
herros@bigfoot.com
```
