This is from memory, so it may possibly have minor syntax errors:

SET COST-IDX TO 1
SEARCH COST-TABLE VARYING COST-IDX
AT END
MOVE ZERO TO WS-COST-IDX
WHEN COST-VALUE (COST-IDX) EQUAL ZERO
SET WS-COST-IDX TO COST-IDX
END-SEARCH

If you are still using ANSI-74 and don't have explicit scope
terminators, then just use a period to terminate the search statement
instead of END-SEARCH.

I hope that helps.

Have you tried SET WS-COST-IDX TO COST-IDX? According to the manual, if
the sending field (COST-IDX) is an index-name, and the receiving field
(WS-COST-IDX) is an elementary integer data item, then this should work.
It will work especially well if your assignment allows you to make
WS-COST-IDX a PIC S9(3) COMP-3, or a PIC S9(4) COMP.

Not to belabor the point, but this is what manuals are for.
