I received this question before it was posted. Sorry for not getting back
to you sooner. After you have done a READ with a KEY, then if you do the
following read just as "READ NEXT" (without a key), it will use whatever key
and value you had from the last READ. (This was just very badly explained -
I hope you can understand it.)

The bottom-line is that if you follow your example code with a

READ NEXT

It will get the logically next record.

MOVE LOW-VALUES TO VAR-A.
MOVE LOW-VALUES TO VAR-B
START TTT KEY NOT < VAR-B

IF TTT-FIL-STATUS = "00"
READ TTT NEXT RECORD
IF TTT-FIL-STATUS = "00" AND
VAR-B = WHAT-U-WANT
PERFORM GOT-ONE UNTIL
VAR-B NOT = WHAT-U-WANT OR
TTT-FIL-STAT NOT = "00".


GOT-ONE.

PERFORM DO-SOMETHING.
READ TTT NEXT RECORD.

This should get you started. No gaurantees on the code.
