> Well, since so many people have added to this thread I thought I would throw in our most recent "performance" discovery.

According to an applications performance guru in my shop, youi can get
even better performance on a DYNAM procedure-pointer call by omitting
the USING parameters and using EXTERNAL data to communicate between
the programs.

You must use the DYNAM compiler option to ensure the procedure-pointer
call is dynamic.  You must then add the "IS EXTERNAL" clause to the 01
leve of all shared records (in both the calling program and the called
program, which must have identical data definitions for the shared
EXTERNAL data).  Then your "CALL procedure-pointer USING parms"
statement can be changed to just "CALL procedure-pointer".

Once you do this, the CALL statement is not only faster than a static
CALL, it's equivalent to a PERFORM of a local subroutine!
