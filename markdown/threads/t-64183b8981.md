It might be that you are trying to access ICOBOL files written with
version 1 of ICOBOL. I imagine that the Egan driver probably only works
with version 6 and upwards of ICISAM files.

I believe that Wild Hare and Transoft also offer ODBC options, while
Cybersoft (?) offer a straight forward windows-based report writer.

If your files are written with version 1 the data portion has a 512 byte
header, with each record having a six-byte header for the primary key
and a four-byte header for each alternate key.
