Accidently, coincidently ...

I work (subcontractor) for a large COBOL CICS mainframe shop, and currently am
expanding a fairly large HTNL front-end system that connects from the web-server
via a communications server with various mainframe CICSes. It uses CICS for NT
for that connection. The COBOL code on the M/F is standard IBM COBOL/CICS. It
includes DB-access (IDMS) and will have SQL (DB/2) in the future.

On the web server the programs (CGI and in the future ISAPI) are COBOL too.
Micro Focus (nowadays called Merant) NetExpress I use for that. It has a builtin
HTML designor/editor and integrated with COBOL fairly well. Some nice JavaScript
could be added if one wants (I do, animating some graphical parts of the html
pages ).

On the web server, these COBOL programs also access elsewhere located network
data bases via ODBC, i.e. standard SQL is also in the web-programs.

The app. is received (used in a inbound call center) with all kinds of
enthousiasm bu the end users. It shows them the information to answer questions
of calling (phoning) customers and intermediates for insurances. This
informationis integrated, browsable and navigatable via hyperlinks and as such
replaces a few hand-full of separate old green-on-blak mainframe screens.
Call/answer time has degraded from 10+ minutes to 2-3 minutes per phone call.
Tremendous savings AND customer satisfaction. 

Can tell you more if needed.

Frog (in a hurry, sorry for typos)
