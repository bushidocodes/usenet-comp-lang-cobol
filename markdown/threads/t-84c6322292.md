Data WILL sort differently - in EBCDIC, letters a-z sort before numbers 0-9;
in ASCII it's the other way around - unless the programs are compiled with a
different APLHABET, PROGRAM COLLATING SEQUENCE and/or sort-merge level
COLLATING SEQUENCE IS clause.

These vary tremendously by compiler and before you implement any of these
schemes you should get it written down as a firm policy or you will end up
with a terrible mess as you bring on new maintenance programmers.

Best advice: train the users to get used to seeing numbers precede letters.

MCM

I've ported a mainframe (CICS) application to a PC under UNIX, OS/2 and NT
always using MicroFocus. In order to avoid your problem, you can set the
default standard as being EBCDIC rather than ASCII. After that everything
else is transparent to you.

If you  need to have files generated by your application, used by other
'ASCII' systems, then use an EBCDIC to ASCII converter.
