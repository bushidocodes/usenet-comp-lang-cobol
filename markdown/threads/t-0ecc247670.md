COBOL for MVS and VM is the effective next release of COBOL/370.

Works just fine, but have not explored either the OO extensions or the
extensions to allow C-style calling / function value returns that were
added between COBOL/370 1.1.1 and COBOL for MVS and VM.

Some siginficant internal changes were made in the generated code to make
the product more "C" friendly. Based on these changes, I have been
letting the latest version (COBOL for MVS and VM) age a bit - using it
only for development type work - before deploying it in heavy duty
production work. Currently production work is still platformed here on
COBOL/370 1.1.1, but with LE/370 (the run-time) production at 1.5 - which
will support COBOL for MVS and VM.

Plans here are to wait for more cumulative maintenance and experience on
COBOL for MVS and VM prior to deploying it as the front line, production
platform. Since COBOL for MVS and VM went GA on 10/28/1995, I estimate
that I will deploy it as the front line compiler about 10/1996, having
first upgraded all maintenance to the most current level.

Rex Widmer
Builder of software archeology tools and other strange programs to help
survive in a legacy based world.

IBM's latest COBOL compiler is called COBOL for MVS & VM. The same compiler
has been ported to OS/2 and AIX (Windows and other UNIX in the works). My
experience is with the OS/2 version called VisualAge COBOL for OS/2 and I
love it. GUI development, OO extensions, interactive debugger, etc.

Most interesting is the Distributed File Manager which allows access to host
based VSAM, QSAM, and PDS files. No need to rehost the data just to test.
This equates directly to producivity since test data setup is minimal
(assuming you have a test environment already setup on the host).
