OK, you answered your own question. Apart from that, the only thing
"special" about them is that they are dynamically searched for and
loaded by the Micro Focus Runtime System rather than the OS.

› The files size always under
› 64K, so I think it is some kind of overlay, isn't it?

.INT and .GNT are the "root" sections of your COBOL code. If you are
using segmentation (eg XXX SECTION 51. syntax), then the .Inn and
.Gnn files are the overlays (where nn is the SECTION number).

› Does MF is a
› real compiler or just a interperter?

You've already answered this question yourself (you know that .INT is
interpreted and .GNT is native code) - it's both, depending on how you
want to compile/run your code.

› Can I decompile the INT, GNT
› files?

No. Why do you want to ?

I don't know off the top of my head, but I suspect that the licence
agreement specifically forbids attempting reverse engineer .INT or
.GNT files (not least because the Micro Focus compiler, code
generator, Animator etc are all written in COBOL).

Cheers,
Kev.

Dear Cobol Fan

The int files are intermediate internal code that is interpreted by the
cobol runtime of Microfocus.

You use .int files when your programs must be executed on systems with very
low memory.

The .int files are slower than .gnt files.

The .gnt files are also intermediate internal code but the execution speed
is a lot faster than .int files.

When you compile a program to .gnt code are your program uses segmentation
(sections with a section number lager than 49), the compiler generates
overlay segments codes G01, G02, etc.

The .gnt code is the most interesting mode because

a) you do not have to link your programs,
b) the code is very compact,
c) and the speed is extremely fast.

The copyright you see is the level number of the compiler. Your are at
level 3.2 patch 43 of the
microfocus cobol compiler.

AXCPA/000113763 is the serial number of the compiler used to compile the
programs.

The compiler at level 3.2 is a 16 bit compiler, so it the code segments and
data segments are
limited to 64 kb.

It is not possible to decompile .int of .gnt files.

Regards.
