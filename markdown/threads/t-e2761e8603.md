He has a series of direct translations that have varying lengths. There are 
several ways to do that (even in COBOL).

My preference would be (using only the data given; it's a model...):

*> define the translation variables
01     TranslationPairVariables.
          12  filler                pic x(6) value  "TG9913".
          12  filler                pic x(6) value  "TL9993".
          12  filler                pic x(6) value  "GS9910".
          12  filler                pic x(6) value  "PM9912".
          12  filler                pic x(6) value  "UG9911".
*> add any further required entries here...
01    TranslationPairTable redefines TranslationPairVariables..
         12 tptEntries occurs 5   *> adjust this number if you add more 
entries above...
              indexed by tpt-x1.
             15  sourceAlphaPair       pic xx.
             15  targetNumericPair    pic 9(4).

*>  define the single ones like "F"...
01     TranslationSingleVariables.
          12  filler                pic x(3) value  "F83".
*> add any further required entries here...
01    TranslationSingleTable redefines TranslationSingleVariables..
         12 tstEntriesoccurs 1  *> adjust this number if you add more 
entries above...
              indexed by tst-x1.
             15  sourceAlphaSingle       pic x.
             15  targetNumericSingle    pic 99.

01    WorkString.
         12 filler   pic x.
         12 theRest  pic x(14).

01     outString   pic x(30) value spaces.

01     outStringPos   pic s9(4) comp value 1.
...

         move inputString to WorkString      *> inputString is not defined 
above. It is self-explanatory...
         perform until WorkString = spaces
              if WorkString (1:1) is NOT NUMERIC
              *> search the pairs first...
                  set tpt-x1 to 1     *> initialize TranslationPairTable 
index
                  search tptEntries
                       at end
                               *> it isn't a known pair so search the single 
ones...
                                 set tst-x1 to 1
                                 search tstEntries
                                       at end
                                           *> it is a single alpha character 
that is not recognized.
                                           *> might want to notify user, log 
it to an error file, abort, whatever...
                                           *> I'm taking no action here, but 
YOU should...:-)
                                       when WorkString (1 : 1) = 
sourceAlphaSingle (tst-x1)
                                                 move sourceTargetSingle 
(tst-x1)  to outString (outStringPos : 2)
                                                 add 2 to outStringPos
                                  end-search
                       when WorkString (1 : 2) =  sourceAlphaPair (tpt-x1)
                                 move sourceTargetPair (tpt-x1)  to 
outString (outStringPos : 4)
                                 move theRest to WorkString
                                 add 4 to outStringPos
                  end-search
              else
                  move WorkString (1:1)  to outString (outStringPos : 1)
                  add 1 to outStringPos
              end-if
              *>  shift the input string one position to the left - slicing 
the carrot...
              move theRest to WorkString
         end-perform

Whilst the above is only one way to address this problem, it is kind of fun 
to write (I wrote this in 15 minutes and enjoyed it; haven't been writing 
any COBOL lately) and I think it is pretty flexible.

Notice that this caters for 1 or 2 length triggers; you could easily add 
similar tables for 3, 4, or whatever length triggers. Then it comes down to 
making sure the strings are searched from the longest to the shortest as 
shown above.

Still, assuming there is some pattern to the madness, there may be
decent approaches to the problem.  There have to be *some* rules for
the transformation.  You haven't told us what they are.

For example, let's say there's one transformation rule for a single
character and another for a pair of characters (this requires a rule
for handling sequences of three of more characters, as well).  You
could then just step through the input string left-to-right, looking
ahead no more than one character (to detect pairs), and apply the
transformation and output the results as appropriate.  Again, the
examples you've offered look pretty random.

As a general comment, unless there are substantial restrictions on the
contents of the numeric parts of the input strings (which we don't
know about), the results cannot be guaranteed to be unique (and input
of 08GS and an input of 089910 would result in the identical output)
or reversible (you couldn't get the original value back - using the
same example, if you had 089910, you wouldn't know that was originally
a 08GS or 089910).  Whether or not this matters for the application is
a different matter.

I split the field into separate numeric and alpha fields.
Converted to alpha fields using the patterns given in the
examples. Then strung them all together.

Use, of this, what you will.
-----
       identification division.
       program-id. huh.
       data division.
       working-storage section.
       01 work-flds.
         02 f1-count comp pic 9(4).
         02 f2-count comp pic 9(4).
         02 f3-count comp pic 9(4).
         02 f4-count comp pic 9(4).
         02 f1 pic x(15).
         02 f2 pic x(15).
         02 f3 pic x(15).
         02 f4 pic x(15).
         02 work-num pic x(15).
         02 work-alfa pic x(15).
       linkage section.
       01 trans-no-d pic x(15).
       01 trans-no-r pic x(15).
       procedure division using trans-no-d trans-no-r.
       begin.
           initialize work-flds trans-no-r
           move trans-no-d to work-num work-alfa
           inspect work-num converting
               "ABCDEFGHIJKLMNOPQRSTUVWXYZ" to space
           inspect work-alfa converting
               "0123456789" to space
           unstring work-num delimited by all space
               into f1 count in f1-count
                   f3 count in f3-count
      * work-alfa will have leading spaces so 'f2' is used twice
           unstring work-alfa delimited by all space
               into f2 f2 count in f2-count
                   f4 count in f4-count
           if f2-count > 0
               call "convert-alfa" using f2-count f2
               if f4-count > 0
                   call "convert-alfa" using f4-count f4
               end-if
           end-if
           string f1 (1:f1-count) f2 (1:f2-count)
               f3 (1:f3-count) f4 (1:f4-count)
               delimited by size
               into trans-no-r
           exit program
           .

       identification division.
       program-id. convert-alfa.
       data division.
       working-storage section.
       01 first-time pic x value "y".
       01 lookup-limit pic 9(4) comp value 100.
       01 lookup-table value spaces.
         02 lookup-entry
           occurs 1 to 100 depending on lookup-limit
           ascending key lookup-key
           indexed by le-index.
           03 lookup-key pic x(2).
           03 lookup-value pic x(4).
       linkage section.
       01 f-count pic 9(4) comp.
       01 f pic x(15).
       procedure division using f-count f.
       begin.
           if first-time = 'y'
               perform load-table
               move "n" to first-time
           end-if
           search all lookup-entry
           when lookup-key (le-index) = f
               move lookup-value (le-index) to f
               multiply 2 by f-count
           end-search
           exit program
           .
       load-table.
           move 6 to lookup-limit
           string
               "F 08  " "GS9910" "PM9912" "TG9913"
               "TL9993" "UG9911"
           delimited by size
           into lookup-table
           .
       end program convert-alfa.
       end program huh.
-----
       identification division.
       program-id. huh-test.
       data division.
       working-storage section.
       01 trans-no-d pic x(15) value space.
       88 test-1 value "07TG".
       88 test-2 value "07TL".
       88 test-3 value "08GS".
       88 test-4 value "0800PM".
       88 test-5 value "10F3UG".
       01 trans-no-r pic x(15) value space.
       procedure division.
       begin.
           set test-1 to true
           perform test-huh
           set test-2 to true
           perform test-huh
           set test-3 to true
           perform test-huh
           set test-4 to true
           perform test-huh
           set test-5 to true
           perform test-huh
           stop run
           .
       test-huh.
           call "huh" using trans-no-d trans-no-r
           display trans-no-d space trans-no-r
           .
       end program huh-test.
