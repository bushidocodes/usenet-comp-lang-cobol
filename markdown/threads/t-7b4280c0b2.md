From the J4 external web page (report from recent meeting)
   (http://merant.microfocus.com/Standards/)

"FINALIZER method

J4 voted unanimously to recommend to WG4 that a subdivision of the COBOL work
item be initiated immediately for a Technical Report (TR) on finalizers and
associated object-oriented language elements.

J4 will begin work on this Technical Report  via an ad hoc group. The TR
should be available in the same timeframe as the standard.

Reasons:

 * There are still major areas of controversy, including side-effects and
syntax.
 * This is a good subject for a Technical Report (TR) because not all details
have been determined. A TR allows J4 to fix the standard later and
implementations to fix problems they find during implementation.
  * Implementations will diverge if nothing is done to provide finalizers.
  * Finalizers require broader consideration of language design than there is
time for in this standard.
  * Java is evolving in the area of finalizers. If Java doesn't have it right
yet, it's premature to think that COBOL does.

This compromise is in everyone's best interests. A specification for
finalizers can be available in the same timeframe as the standard. The
standard will not be delayed. The specification can change if needed. "
