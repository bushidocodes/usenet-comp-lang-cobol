I know of no such tool that can be used with multiple languages. There
is a tool called HP Fortify that inspects code and tries to find
"holes" that could be exploited by hackers.  It is a multi-language
tool but that is the only one I know about.

Generally speaking, the idea of quality assurance for software lies in
what results are obtained when running the software.  Does it produce
what it was designed for?  Does it meet users expectations?  Is it
easy to maintain?  If any of the things you mentioned occurred, the
software would not see production until fixed or discarded as the case
may be.
