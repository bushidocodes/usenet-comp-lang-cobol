Unfortunately, there seems to be no portable way to have null-terminated
string literals in Cobol.

Chris Mason (and others) had suggested using variables similar to
the following:

› 01 I-AM-A-PORTABLE-STRING.
› 05 THE-STUFF PIC X(20) 'THE STUFF'.
› 05 FILLER PIC X(01) VALUE LOW-VALUES.

which Don Nelson pointed out was not truly portable, but does work
on all my current development platforms! :-)

I was hoping to avoid this level of abstraction, but I guess it
beats writing wrapper functions with temporary variables.

Dave

```

FYI - IBM in their new COBOL compilers has introduced Z-literals,
e.g.,

Move Z"this is null terminated" to Some-Other-field
or
Call "Bet-you-are-in-C" Using by content Z"another n-t-s"

Micro Focus has this syntax available when doing IBM compatible
development (as an early-release option with COBOL370(2)).

This is *not* portable, but check with your "vendor of choice" as
to when/if they are considering this.
```
