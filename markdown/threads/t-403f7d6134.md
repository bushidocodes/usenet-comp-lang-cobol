On a mainframe terminal there are protected and unprotected areas,
when the 'send' key is pressed the data is sent from the unprotected
fields to the computer. Thus data is sent from the *screen* to the
program.

PCs are different. Each keystroke is sent to the computer (where
ADIS initially handles it). Data is sent from the *keyboard* to
the program and only appears on the screen because ADIS sends it
back out.

In general if you want the cursor to be somewhere it must be
'unprotected' in that it must be a position that can validly
be used to echo keyboard input. If this is a console (DOS) or
QuickWin program then just allow all 2000 characters to be part
of the accept.

This is probably not what you want to do though. Why do you
want the cursor to be 'between fields' ? What could be done with
the cursor at these places ? Is it just that the users want it
because it is what used to happen, but they don't know why they
want it ? just tell them to get ovet it.
