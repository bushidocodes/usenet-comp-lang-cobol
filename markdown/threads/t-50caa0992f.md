Toby Reed wrote:
› I have been able to declare the pointer and then set it to the address
› of a Ws-variable, but how can you manipulate the ws-variable thru the
› pointer.

You can't do that using the pointer itself, as the pointers are not
typed
(the compiler doesn't know what sort of data they point to).

To get around this, indirect through pointers using LINKAGE SECTION
items
(which are also just pointers, BUT have a record definition associated
with
them).

ie:

WORKING-STORAGE SECTION.
01 TXT PIC X(11) VALUE "HELLO WORLD".
01 PTR USAGE POINTER.
LINKAGE-SECTION.
01 LK-TXT PIC X(11).
PROCEDURE DIVISION.
SET PTR TO ADDRESS OF TXT
SET ADDRESS OF LK-TXT TO PTR
DISPLAY LK-TXT.

or:
SET ADDRESS OF LK-TXT TO ADDRESS OF TXT
DISPLAY LK-TXT.

The pointer is not really needed, as you can see in the second case,
but in a real application you'd probably keep an array of pointers
or something, and set the address of LK-TXT to the correct one depending
on which you wished to manipulate.
As you can see, you can simulate local parameters to PERFORMed routines
by having the PERFORMing code set up a couple of linkage item addresses
to
point to any arbitrary WS variables and having the PERFORMed routine
only
access them via the linkage section.

Cheers,
Kev.
