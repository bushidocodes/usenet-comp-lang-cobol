Dear Cobol Gurus,

    I am very new to Cobol. I had written a Cobol program on DG machine
using MF cobol for Unix and Oracle 8.

I have a Host variable defined as PIC X(4) in Cobol program and in Oracle
table the column (a) is defined as CHAR(4). Say the table has data for a
code = 'FF  ' (Two F followed by two blanks). I had moved the same code in
Cobol code to host variable (MOVE 'FF  ' TO H-V1). The problem is - when I
use the following embedded SQL, the select statement fails and I am unable
to understand why since the Column a in table temp has 'FF  ' and host
variable is also having same value.

Select b INTO :H-V2 from temp where a = :H-V1

After some hit and trial the above statement when modified to following did
work.

Select b INTO :H-V2 from temp where RTRIM(a) = :H-V1

Is there something trivial which I am missing.

With regards,

Vikram
