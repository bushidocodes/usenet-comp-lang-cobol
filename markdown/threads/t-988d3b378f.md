to your TSO session, but not part of your SYS1.LINKLIB. See which

When you run a batch job, you normally don't have access to TSO commands, since
the TSO environment is not automatically established in the batch. You can
establish the TSO environment from your COBOL program by calling IKJTSOEV.

To do this, call IKJTSOEV passing five parameters, each of them fullword binary
fields. And you really only care about the second. So, you might include
something like this:

01 TSO-parms.
    05 word-1 pic s9(9) binary value 0.
    05 word-2 pic s9(9) binary value 0.
    05 word-3 pic s9(9) binary value 0.
    05 word-4 pic s9(9) binary value 0.
    05 word-5 pic s9(9) binary value 0.
.
.
.
         call 'ikjtsoev' using word-1,
                                           word-2,
                                           word-3,
                                           word-4,
                                           word-5

Now, assuming the value in word-2 is zero after the call, you have established
the TSO environment (if it is 8, by the way, you have established the TSO
environment but not the REXX environment).

Next, to invoke your REXX exec, do it through a call to IKJEFTSR, the TSO
service facility....

01 IKJ-parms.
    02  IKJ-flags pic x(4) value x'00010001'.
    02 IKJ-command pic x(??) 
                      value '%exec_name'
    02 IKJ-buflen pic s9(9) binary value ??.
    02 IKJ-RC      pic s9(9) binary value 0.
    02 IKJ-Reason-code pic s9(9) binary 
                      value 0.
    02 IKJ-Abend-code pic s9(9) binary
                      value 0.
.
.
.
        call 'ikjeftsr' using IKJ-flags,
                                          IKJ-command,
                                          IKJ-buflen,
                                          IKJ-RC,
                                          IKJ-Reason-code,
                                          IKJ-Abend-code

after this call, check RETURN-CODE; if this is non-zero, check the values in
the last three parameters for more detailed information. There are other
possible values for the IKJ-flags field. 

At run time, include a DD statment named SYSEXEC that points to the library
containing your exec.

All this, and more, is detailed in my course "Advanced TSO REXX Programming",
see the website in my signature.

Hope this helps get you started.
