> I can't find a file named opdynchk.obj on my system

It's in the RTS .LIB file.

> The DLL cblrtssi dosn't have an entry checkfil

There shouldn't be any such thing as CBLRTSSI.DLL - the
files should be CBLRTSS.DLL and CBLRTSSI.LIB

> So the runtime is right! it can't find checkfil.

There is a dynamically-loadable version of CHECKFIL in
%COBDIR%\UTILS.LBR
Therefore, if you make sure UTILS.LBR gets loaded at
startup time of your COBOL DLL, it should work.
eg:

cobol maindllentry.cbl INITCALL("UTILS.LBR");
cbllink -D maindllentry.int otherprogram.cbl otherprogram2.cbl
 
> If i link externl.obj i get the error
> 119 symbolic name is more then one time used.

Hmmm. I don't think this should happen (my initial thought was
that linking in EXTERNL.OBJ would fix the problem). Can you say
what your link command line was when this happened (I assume
you used CBLLINK to make the DLL) ?

> I think i have to get the file opdynchk.obj.

No, you already have this file (in the LIB).

Cheers, Kev.
