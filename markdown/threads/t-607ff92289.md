First - as I suspected you are using Screen Section - big, big challenge to
emulate a Listbox with the Screen Section.. So that you understand the
problem, lets take the 'Windows' approach using GUIs. You use a visual
design tool to set up the Listbox, then your support GUI tools gives you
code to put a table of entries (elements) into a Listbox.  Once displayed,
by moving the Mouse (a mouse event) over the scroll bar  the list
automatically moves up and down without any coding in COBOL - this is done
by Windows API support. When you want a particular record in the Listbox you
click on it and your application can pick out the record by its index
position.
(Assume your Listbox shows 25 records at a time, even though there might be
300 records/elements, (250 being 'hidden')).

Because MicroSOFT COBOL V 5.0 and Micro FOCUS COBOL V3 are virtually the
same animal the following should apply to use of the Screen Section. You
have got to emulate both the moving up and down (scrolling) and selecting a
particular record that you want. The actual Lisbox is no big deal in Screen
Section - it's that 'visually' moving the contents and selecting a record
that's your problem. You will have to confirm for me - does MS COBOL V 5

Using character mode screens it is possible to emulate some of the more
'modern' GUI controls such as grids, etc. but as other posters have
mentioned this is a tremendous programming challenge.  I rate it impossible
without some previous experience writing user controls.   Some oo
exeperience will help as well.

Most of the trick lies in hacking the screen section to display/edit data
from an externally managed data source, such as a work file (or, if you are
lucky) a dynamically allocated memory buffer.  This is highly vendor
specific and may rely on special extentions to the language to help you out,
for example mouse handling.  All things considered, though, Unless you have
built in GUI support from the vendor (AcuCorp AcuCobol), it may not be
worthwhile).

> "DeLaPena" <member25583@dbforums.com> wrote

Let us assume that you are doing a list box from an indexed file.  eg 
courses for a given semester where either the main key or an alternate key 
is giving the records in sequence.

The 'listbox' is an 10 line area of the screen.  10 records at a time will 
be shown and there may be more than 10 records.  Each'page' needs a start 
key so that any particular page of 10 records can be shown.  However we 
accept that the list needs to be shown forwards from the start (ie paged 
through) and not jumped to any point.

         01  Page-Index             PIC S9(4) BINARY.
         01  Page-Array.
             03  Page-Key           PIC X(24) OCCURS 100.
         01  Line-Index             PIC S9(4) BINARY.
         01  Line-Array.
             03  Line-Key           PIC X(24) OCCURS 10.

       

To start put the first key in Page-Key(1):

          MOVE 1      TO Page-Index
          MOVE Semester-code  TO Page-Key(Page-Index)
          .

To display a page start the file at that page's Page-Key and then read the 
next 10 records storing each key in a line table.

     Display-Page.

          MOVE Page-Key(Page-Index)  TO Course-Key
          START .. Course-Key
              INVALID
                  MOVE HIGH-VALUES TO Course-Key
          END-START
          MOVE 1   TO Line-Index
          PERFORM
              UNTIL Line-Index > 10
                 OR Course-Semester NOT = Semester-Key
              READ ...
                  AT END MOVE HIGH-VALUES TO Course-Key
              END-READ
              IF ( Course-Semester = Semester-Key )
                  MOVE Course-Key  TO Line-Key(Line-Index)
                  MOVE data items to screen line
                  ADD 1       TO Line-Index
              END-IF
          END-PERFORM
          PERFORM
              UNTIL Line-Index > 10

              MOVE HIGH-VALUES    TO Line-Key(Line-Index)
              *> this is only done if there were less than 10 in this 'page'
              blank out remaining screen-line data
              ADD 1    TO Line-Index
          END-PERFORM 
          *> save key for start of next page
          ADD 1            TO Page-Index
          MOVE Course-Key  TO Page-Key(Page-Index)
          .


If a line is selected then get the number of it and use this to fetch the 
key from the line-table. 

If 'next page' is selected (eg Page-Down key) then just perform the 
display-page above.

If 'previous page' is selected then subtract 2 from page-index, check that 
it is > zero, and perform display-page.

If 'home' is selected move 1 to page-index and perform display-page.

The last key on a page will also be the first key on the next page to give 
continuity.
