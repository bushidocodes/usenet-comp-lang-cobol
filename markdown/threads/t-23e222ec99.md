Hi Bob,

The NESTCALL directive is only needed if your program has the following
structure:

IDENTIFICATION DIVISION.
PROGRAM-ID. PROG-A.
DATA DIVISION.
....
PROCEDURE DIVISION
....

IDENTIFICATION DIVISION.
PROGRAM-ID. PROG-B.
DATA DIVISION.
....
PROCEDURE DIVISION.
....
END-PROGRAM. PROG-B.
END-PROGRAM. PROG-A.

This is a nested program because there is a program (PROG-B) inside another
(PROG-A).

The reason the compiler needs the directive is because of the way the
compiler has been written.

Use NESTCALL when you are using "nested" programs.
Do not use it otherwise.

If each source file, after processing COPY statements,
contains only one program, you are not using nested programs.

If the source file contains multiple
    END PROGRAM program-name
statements, the program may be nested.

In the source for a nested program, there will always
be at least one occurence where two end program
statements appear in sequence, such as

....
end program inner.
end program outer.

In this case, the program 'inner' is nested within
the program 'outer'.

HTH
------------------
Rick Smith
