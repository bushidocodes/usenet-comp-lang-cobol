```
I am doing demo evaluations of cobol's with integrated
GUI environments for Win95. Right now, I'm working
with ACUcobol-GT's demo disk.

Of course, for cost reasons there isn't any supplied printed
documentation. All I have to work with is the online help.
The online help is good on the compiler itself, but when it
comes to the AcuBench and ScreenPainter, its very lacking.
The online help presumes the user is already trained in
those products, which is kind of odd, since they're both
proprietary products.

I'm now playing with the ScreenPainter, but this is new
to me, and I don't understand how the generated code gets
back into AcuBench, and what to do from there.

Is there some third-party book available that gives a true
tutorial of this process. I recently did this for VisualBasic,
and of course there are endless tutorials available.

I'll soon be evaluating Fujitsu, and possibly RM-WOW.
Any tutorial pointers, and/or comments on the relative
merits of these three products would be appreciated.

don ferrario
```
