> 1)  Does any one know an other tool with equivalent power?

We have a tool called y2k-fix.
This tool applies windowing to the user-specified 
data items in the Cobol code.

It can fix both main files and copybooks.
It does not reformat the unhcnaged parts of the program,
thanks to SourcePrint capability.

You can download free evaluation package for this tool
from http://www.siber.com/sct/tools/y2k-fix.html.

This tool has several ways of selecting the 
data items that are dates: by explicit list, by picture and by name.

Our tool certainly is a way cheaper than Revolve/2000,
but it does not have a nice visual shell, it's a command line tool,
which is usually OK for big conversions that run in batches.

The big thing about this tool is that if you are willing
to program in C++, we can give you source for this tool
and you can enhance it in any way you like.

Usually end users do not want to do this,
but power users in conversion houses may need
a capability to implement additional transformations 
to their liking.

and to do transformations you need to get SmartFix/2000,
but I am not really sure.
