### sct-3-7-3: CobolTransformer Data File Readers: New Features

- **From:** Vadim Maslov <vadik@siber.com>
- **Date:** 2000-02-21T00:00:00
- **Newsgroups:** comp.lang.cobol,alt.cobol
- **Message-ID:** `<38B0D350.49A114CC@siber.com>`

```
Siber Systems is pleased to announce release 3-7-3
of our DataReader technology. Using this technology,
you or your application can read Cobol data files 
directly, without writing a Cobol program to do it.

For more info look at http://www.siber.com/sct/datafile/

The new features are:

* Readers for RM/Cobol and AcuCobol data files.

* Data-Guess program that automatically guesses
Cobol data files format (MF/FSC/RM/ACU, indexed/relative, indices)
and record layout. Data file format is guessed precisely
while the guessed record layout is approximate, it is 
based on statistical analysis of data file records.

Current state of DataReader technology:
* Can read Cobol data files produced by the following Cobol compilers:
- MicroFocus,
- Fujitsu, 
- Ryan-McFarland and 
- Acu 

* Can write data files in the following formats:
- Flat (CSV),
- dBase/IV DBF, importable to MS Excel, MS Access.
- Crystal Reports report generator.

* You can use DataAccess library in a C++ program
to read Cobol data files.

* Data-Guess will help you guess format of a Cobol
data file if source of Cobol program that can write 
or read the data file is not available.
```
