Hi there,

You could try

set environment "PRINTER" to "xxxx"

assigns PRINTER the value of xxxx in the run unit's local environment (at
least in Acu & RM cobols). To read the value later you can use:

accept xyz from environment "PRINTER"

(moves the current value of PRINTER from the environment to variable xyz).

HTH.

- Brendon Oliver.

unix script that is run with "." by the parent script when the cobol
