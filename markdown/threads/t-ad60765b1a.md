```
Does any one know how to use the MFcobol 'WINAPI' command to write a string
of data to the com-port?  I need to send a string of data to the Thermal
Printer which is connected to com1 or com2.   I am using "CreateFileA"  to
open the port, "GetCommState" to get the status and "SetCommState" to
ensure the status is set accordingly for the printer.  I have to reset the
com-port each time I print since the same port can be used for the modem
via a switch box.  I do not get any errors until I "WriteFile" and I cannot
"CloseHandle" successfully.  

Any suggestions on a better way of handling this problem, how to evaluate
my return-code with a list of meaningful errors or a solution would be
greatly appreciated.  MicroFocus support is having trouble giving me an
answer.  

Ohhh, for you old time people the command use to be 'Call X"84"  using
ws-interrup, etc,etc' for DOS and Windows3.x.

Thanks

Patti
```
