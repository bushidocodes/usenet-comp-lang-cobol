```
This probably isn't what you are looking for, but I would use a system
call to the O/S.

01  UNIX-Command-Do-Hello.
     02                          PIC X(5)  VALUE  "hello".
     02                          PIC X      VALUE  LOW-VALUES.

Procedure division.
       Call "SYSTEM" Using UNIX-Command-Do-Hello

Animator, of course, is not going to step through your C program, and
you will need to compile your C program separately, but this will get
the job done.

Now as for passing data between the programs without writing to a temp
file, if you find out, let me know, will ya?
```

```
If you create a new "runtime" with your 'C' modules in, then you will
be able  animate the COBOL and call the 'C' functions.

For example:

cc -c hello.c
ar crv libhello.a hello.o
cob -vxo myrts -L. -lhello
cob -via *.cob
COBSW=+A ./myrts mytest
```
