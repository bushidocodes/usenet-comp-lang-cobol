The field descriptions are not in the database files, they are in the
program source code, probably in a 'copy' file.

>          program ? my intention's access the database files to read and
>          write, I know how to generate a Ascii or c-Isam format by
>          rebuid, but I need to write directly in the file without change
>          the database structure file  of my DAT, IDX files...

You are more likely to corrupt the data and/or index unless you use
the exact Cobol mechanism.  For example adding a new indexed file
record may require an index block split and index rotation.  You also
need to have the same exact locking mechanism otherwise you may be
updating at the same time as a Cobol program is and the result would
be corruption.

If you have a developer licence then you should have a callable file
handler that you could use.
