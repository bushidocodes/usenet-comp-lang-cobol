Hi Alex,

Don't know too much about Net Express, but a memory violation typically 
means that you've tried to access a bit of memory that you don't 'own'. 
This is very often a result of a call by reference when you meant to 
call by value (or vice versa!), or (more likely) the size of the third 
parameter in the two programs differs. That would be the first thing I'd 
check.

Good luck!

Nigel

Hi Alex,

I think there may be limits, can't remember of the top of my head.
Check system limits in the help. But I don't think you are having
that problem here.

Check the ENTRY statement has three parameters in the USING clause
and that the definitions of the items in LINKAGE SECTION match the
definitions of the items in the CALL statement in the calling program.

If the size of the third parameter is smaller in the calling program than in
the sub program (ENTRY statement). When you try and use the third
parameter, it wil try and access more memory than has been passed.
A possible reason for the memory violation.

If you are still stuck, it will be good to see the definitions of items
param1, param2 and param3 for the CALL statement and again
what your ENTRY statement looks like and the definitions for all
the items in the USING clause.
