We're using RM COBOL v5.2 for DOS (some 5.3) which worked flawlessly
under Netware 3.12.  Since we upgraded to Netware 4.11 (1 1/2 years or
so) it seems we've been experiencing more file corruption problems.

The BIGGER problem, though, is that the RECOVER utility doesn't work
anymore, so now, instead of the users being down for the time it takes
to recover the file, they are down for the time to restore from backup
as well as losing any work down since then.  This is quite so bad if
it happens early in the morning, but when it happens at 4:15 pm - the
stuff really hits the IS department in the fan.

I know we need to upgrade to a later version of COBOL, we've been
fighting that battle for a couple of years now, but we're a small shop
and are already facing a multi-year backlog of changes, additions and
new features/systems.

Has anyone had or heard of a problem similar to this?  Any ideas are
greatly appreciated.
