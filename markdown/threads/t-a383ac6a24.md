```
There is no feature in VS COBOL II (or the newer COBOL for this-and-that)
compilers to get this information via "native" COBOL source statements.  You
can look at DejaNews for methods of calling either an Assembler *or* COBOL
program to go searching OS/390 control blocks - to find the TIOT or similar
system resources with this information.
```
