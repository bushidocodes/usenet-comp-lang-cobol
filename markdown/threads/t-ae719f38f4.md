The answer to your problem is compiler dependent.  There is no standard
COBOL syntax for testing if a key has been pressed.  For example, in
Micro Focus COBOL there are extensions for this.  In some Windows based
compilers you could call a Windows API function.  In some DOS based
compilers you could call a DOS function.  Let us know what compiler and
platform you are working with and we will try to kelp you.
```

In message <<71t7nh$ds2@news3.euro.net>> "Radoux Claude" <claude.radoux@euronet.be> writes:
> I don't know how to make a loop (like a perform) that stop when a key is
> pressed.

Good.  Looping while waiting for a keystroke uses up all the
CPU it can get hold of.  This is a very bad thing on reasonable
systems, or even on Windows, where other programs could actually
do something useful with the processor.

If you do actually find a way of doing this then please also
include an OS despatch withing the loop.  This can be done
under DOS or DOS boxes under Windows or other multitaskers
and DOS multiusers using INT 0x2F with AX=0x1680.

I just did a program that scanned a directory looking for
new files (or an abandon signal from keyboard).  It uses
almost zero CPU time while idling.

First check if there is a timeout clause on your ACCEPT
statement.  If not you need to specify the compiler and
environment.

```
