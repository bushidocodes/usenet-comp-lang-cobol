You'll notice that each file generated matches a section, like Working
Storage, etc.

You can change the code in the .cbl file, in these sections.  The reason it
says not to, is that if you change something in the Screen Painter, and
regenerate, you'll lose those chages.  Oh, one other thing, turn off Generate
Program File in the Options section of the Screen painter once you start
changing the .cbl file, so you don't overwrite your previous changes.
