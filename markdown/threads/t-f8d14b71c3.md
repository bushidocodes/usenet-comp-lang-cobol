```
I have been writing COBOL for many years but am new to writing it on UNIX.
My problem is this. I am trying to write out a fixed blocked record. My
output file is defined as record sequential and I write my record after
advancing 1 line. What a get is this.

- COBOL puts out a hex '0a' as the first record, which is a line feed.
- All subsequent records are terminated with a hex '0d' (C/R) and
'0a'(L/F).
- The very last record is only terminated with a '0d' not the '0a'.

What I want is a file with no '0a' as the first record and all records
terminated with a '0a' only. Any help would be appreciated. Please email
me directly.

Thanks!
```

```
Hi Eric,

First of all get rid of the after advancing stuff.

If I understand correctly, you have 2 choices:

1. Define the file as line sequential.
The only thing here is the trailing spaces will be truncated,
resulting in variable record lengths (a regular text file).

2. Define the file as relative (with access sequential).
This creates a fixed length record terminated with a '0a'.

Hope this helps.
```

```
It seems to be doing what you are asking it to do.

WRITE .... AFTER ADVANCING 1 LINE

is implemented by outputting a linefeed, then the record which
has a normal terminator of carriage-return. This is repeated.
ie the '0a' is being output at the _start_ of the record as
requested. Doing a WRITE .... BEFORE ADVANCING 1 may get
it to output '0a'.

As you haven't indicated which implementation of Cobol it is
impossible to give specific details of how to achieve exactly
a '0a' terminator only.
```
