It is pretty easy to skip records.  Simply READ a record and bypass 
it.  But adding that logic to an existing program may be difficult, 
depending upon how the original program was coded to process the 
results of a READ.

Skipping the last five records of a sequential file is difficult to do 
in a COBOL program, unless you know in advance how many records are on 
the file.

This sounds like the kind of thing you might need to do in order to 
create a small test file from production data, or to clean up a file 
by removing unwanted records.

Sometimes the best way to do that is to write a one-time COBOL program 
simply to select the records you want.
