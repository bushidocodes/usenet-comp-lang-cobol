You need to connect to the Access database using ODBC(Open Database
Connectivity)
Simply opening the file using traditional cobol methods will not work.
You say Mf cobol 3.0 is that netexpress?, if so then NetExpress supports
ESQL(embeded simple query language, i think! :) ), very simple SQL calls
to access your data, if you have not got this feature ie: not NetExpress
then you can use the Windows API, using call-convention 2 for 32-bit
this will work for any cobol compiler.
