Obviously an MVS question.  Please let people know in your
message.

Abend aid can be replaced with the augmented dumps provided by
language environment.  There is a runtime penalty for doing this
though but it is acheived fairly simply by changing compile
options:

TEST(PATH,SYM)

Gives a very good formatted dump that you just walk through.

Ken

The following is the reply that I received from one of my "usually reliable
sources,"

"  For debugging of ABENDS, TEST(PATH,SYM) provides no more than
TEST(NONE,SYM), but it does have an execution time penalty.  TEST(NONE,SYM)
does NOT run slower than NOTEST, it has the same execution speed, there are
no extra instructions
generated.  TEST(NONE,SYM) does cause extra information to be placed in the
object deck and load module, and the module size can be up to 50% larger.
So, there is a load module size cost, but NO EXECUTION TIME COST for the
extra
debugging ability of LE formatted symbolic dumps.

  The first suboption of TEST specifies whether debugging hooks (for Debug
Tool)
are inserted into the code. The second suboption specifies whether the
symbolic
information is placed in the object program.  It is used by BOTH LE symbolic
dumps and IBM Debug Tool.  See the COBOL Programming Guide for more
information."

I hope this helps

With path yes there is.  Once you clock a new line the line number
is recorded so that everytime the source line number is changed
the compiler introduces a move of the line number.


> Doesn't most of the extra code only get executed

I use this during test and when I spend longer than the run time
figuring out what happened.  If reexecuting the program is fast,
compile in dev a fully instrumented version, point you application
at it and let it go.  Penalty paid on on need.
