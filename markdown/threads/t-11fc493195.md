I can offer a bit of history but, unfortunately, no technical advice on
Windows ME or 2000.

RM/COBOL version 1.5e is a version of the extremely old (in 'software
years') 1974 standard based product.  Development on the 1974 product ceased
15+ years ago, when the 1985 standard based product (called RM/COBOL-85 at
the time) was introduced.  Support for the 1974 product was terminated a few
years later, and RM/COBOL-85 was then renamed to RM/COBOL.

Recompilation and file conversion are necessary when moving from the '74
product (i.e. ver 1.5e) to the '85 product.  The conversion tools are still
available.  Once on the current product, Ronald's assertion is true -- all
that is needed is the proper runtime.  The benefit of doing this, however,
is that Scott should observe about a 5x performance improvement.

BTW, version 1.5e was not even the last version of the '74 product, so Scott
is using software that is almost 20 years old!

Tom Morrison
Liant Software Corporation

That's why the version did sound familiar to me (I'm using rm now for about
17 years)
I don't think you'll need to convert anything when you keep on using 74 mode
(compile with -7)
The only thing you'll NEED to do is to read every file with comp-3 fields
move the fields to a normal field and back.
like:
       01  HULP                PIC S9(11)V9(4).

           MOVE SOS-OMZET-DP    TO HULP.
           MOVE HULP TO SOS-OMZET-DP.
           MOVE SOS-AANTAL-DP   TO HULP.
           MOVE HULP TO SOS-AANTAL-DP.
           MOVE SOS-OMZET-TM    TO HULP.

The op-code for a signed 0 field is changed. So it can happen that a field
with value 0
Doesn't respond on a zero like:
         IF    SOS-OMZET-DP > 0 DISPLAY ">0" GO TO EINDE.
         IF    SOS-OMZET-DP < 0 DISPLAY "<0" GO TO EINDE.
         IF    SOS-OMZET-DP = 0 DISPLAY "=0" GO TO EINDE.
         DISPLAY "shit".
EINDE.
It'll display 'shit'.

Another thing to look after is the value of the function key's in the new
environment.
I've never seen any conversion tools, but I've always written them my self.
Ther still was something else about the conversion but I can't remember
what.
