Hi Costas:

The movement of the cursor when the HOME key is pressed is puzzling to
say the least.  From our perspective, it appears to operate in one
manner under certain Windows applications and in another manner under
different Windows applications.

Please allow me to elaborate.

If you launch a form in one of the sample applications in Microsoft
Access, you may notice that HOME and END move the cursor to the first
and last field as you have noted.  This only happens when the entire
contents of the field are highlighted (as if the operator tabbed into
the field).  But if you click the mouse so that the contents of the
field are NOT highlighted, then the HOME and END key simply moves the
cursor to the beginning and end of the data contained within the
field.

The only problem is that if you launch other Microsoft applications
and display a dialog box with enterable fields in the dialog box, the
HOME and END key *only* moves the cursor to the beginning and end of a
field and *never* to the first and last fields on the screen....even
if the contents of the field are fully highlighted.

We're still trying to figure out exactly which one is the default
Windows behavior.

It is reasonable to assume that Microsoft would have established some
sort of standard for the cursor behavior in all the applications which
they develop in Redmond, Washington.  When you give it a try, you will
obtain mixed results.

I agree with Henry that the HOME and END key should move the cursor to
the beginning and end of the data contained in the field which has
current focus.
