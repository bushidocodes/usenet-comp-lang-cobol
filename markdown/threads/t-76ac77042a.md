```
Hi,

I am going to write a COBOL batch program that will add/modify a VSAM file. There is an
alternate index file associated with this VSAM file, defined with option UPGRADE.
Do I need to add a DD with pathname or DD with ALTERNATEINDEX name to my JCL to be sure
that Alternate Index file will reflect the changes done to the base cluster by my program?

IBM manuals says: 
"UPGRADE specifies that the cluster's alternate index is upgraded to reflect changed data
when the base cluster's records are added to, updated, or erased. "
But if I do not need an additional DD with pathname, why do I need to define a PATH?

Thanks,

Zalek
```
