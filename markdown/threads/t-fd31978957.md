GNT modules are modules created by the Micro Focus run-time system
(RTS), and can only be used by the MF RTS.  From what I understand they
are also platform portable, meaning that you could take a GNT module
created on a windows system and run it successfully under an OS/2 system
(though I have never done so).  It is an acronym for GeNeraTed code.
Again, it will not run without the MF RTS.

Three types of modules created by MF Cobol v3:

INT - Intermediate Code Module (used in animation)
GNT - Generated Code Module (faster that an INT)
EXE or DLL - Executable or Dynamic Link Library (fastest format)

Also, there is no editor that reads GNT modules that I know of.  With MF
v4, I think there is a way to animate a GNT module, but I am not for
sure on that.

Hope this helps.

TL

intermediate (say precompiled) code that is machine independent.

That's not entirely true. It depends on the number of users and the
platform, the size of the run-time, and whether or not you're using shared
libraries or DLLs for the common functions.. IIRC, gnt's are loaded into
user memory so you get an instance per user. If you have a few users then
there may be some advantage (There's certainly an advantage when compared to
statically linking all the run-time bits into each application). However,
once you have a lot of users, then you quickly exhaust system memory with
all those duplicate instances of both data and code. For more info you can
check out

http://www.microfocus.com/cgi-bin/WhitePapers2?file=1175.unix&title=Packagin
