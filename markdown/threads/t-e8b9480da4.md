As yes, extracting data from a free-form ASCII file. This is one of my least
favorite tasks.

First of all, you need to set up a series of loops to extract the record one
character at a time and assemble it into COBOL fields. From your posting, I
assume you are already doing this.

Regarding numbers of any kind, take advantage of redefines or scaling to avoid
direct interpretation of decimal points, your loop would look something like
this:


MOVE 0 TO NUM-WRK.

Start of Loop:

... code to obtain next character ...

IF CHAR NUMERIC
MULTIPLY 10 BY NUM-WRK, ADD CHAR TO NUM-WRK.

... code to determine repeating loop ...

End of Loop

... scaling code ... i.e. DIVIDE BY 10 or whatever.

If you are processing this file repeatedly, i.e. an interface file, you may
want to consider using a C program format out a data-stream into the COBOL
sequential format.

Try using the UNSTRING command to separate the fields. It looks as if you
have semi-colons which separate all the fields. Once you have the fields
separated, the numbers can be moved to a working-storage field with
the JUST RIGHT clause. This will cause the number to be justified from
right to left instead of left to right. Then you can move the JUST RIGHT
field to a numeric editied field (one with Z's so the spaces in the JUST
RIGHT field are OK.) Then you can move the numeric edited field to a
normal numeric field. COBOL will "de-edit" the number edited field and
make a number out of it for you. There may be short cuts. I'm thinking
really quickly here.

Good luck...this should be one way to get you going.
