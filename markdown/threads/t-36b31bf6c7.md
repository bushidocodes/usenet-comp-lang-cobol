```
For those interessted, the following "substantive change" item is listed in
the '85 Standard on page XVII-46

"(70) EXIT PROGRAM statement (1 IPC). The EXIT PROGRAM statement need not be
the only statement in a paragraph."

Therefore, anyone who hasn't changed compilers or programming techniques
MIGHT still have a restriction similar to this.  Of course, even their '74
Standard compiler *might* have allowed this restriction to be ignored - as a
perfectly valid extension.  Certainly, I know of no currently supported
(when run in '85 Standard mode) compiler that doesn't follow this rule.
There are, however, some '74 Standard compilers still being supported (or
'85 Standard compilers that have a '74 Standard option) such compilers
*might* still have this restriction - but again, I don't personally know of
any.

FYI - this was the "argument" against adding the GOBACK statement to the
Standard. In an '85 Standard conforming program, it was legal to code a
paragraphs such as the following:

  Leave-Program-Para.
       Display "about to this program and/or stop the run-unit"
       Exit Program
          .
       Display "Leaving the run-units main program"
       Stop Run
         .

Clearly (ignoring lower-case) this would NOT have been conforming to the '74
Standard - but was in the '85 Standard.
```
