```
2-bytes = 9(04) BINARY
4-bytes = 9(08) BINARY
8-bytes = 9(16) BINARY
```

```
AS I OFTEN SAY: "THERE IS NO SUCH THING AS 'HEX' DATA. DATA ARE DATA."

Assuming "hex byte" means an eight-bit character in which each bit
represents a power of two, that is,

hexbyte (1) =  x'01'
hexbyte (2) =  x'02'
hexbyte(15) = x'0F'
hexbyte (16) = x'10'
hexbyte(255) = x'FF'

... the below will work.

OTOH, if "hex byte" means a display character in the ranges '0' - '9' and
'A' -'F' then that's a horse of a different color and below will not work.


01  VALUE-1.
     05 FILLER     PIC X(01) value x'00'.
     05 THE-CHAR  PIC X(01)
01  VALUE-NUM REDEFINES VALUE-1    PIC S9(04) BINARY.
move byte-from-file to the-char
move value-num to anynumericfield
```

```
There may be a compiler option to allow 1-byte computational fields.
DO NOT USE THIS COBOL EQUIVALENT OF ANTHRAX. Hundreds of other
problems will be created that you will spend your career trying to
find.

You have eight bits (1 byte) containing a count. These 8-bits
(XXXXXXXX) are now addressable as one byte. For an INTEL PC you have
to get these 8 bits into two bytes that look like

XXXXXXXX 00000000

Then address these 16 bits (2 bytes) as a  number defined as COMP-5.

So, a definition like:

01  DISPLAY-DATA.
     02  WORK-CHAR    PIC X.
      02  FILLER       PIC X VALUE LOW-VALUES.
01  FILLER REDEFINES DISPLAY-DATA.
      02  MYNUMBER   PIC S9(3) COMP-5.

will work swell. Move the single byte from the record to WORK-CHAR,
add 1 to MYNUMBER, then put WORK-CHAR back in the record.

BUT. If you really want to be cute, do this:

COMPUTE MYNUMBER = FUNCTION ORD(IN-CHAR) - 1

ORD returns the number in the collating sequence of a single
character. You've got to subtract 1 from whatever ORD returns since
the 1st value in the sequence is zero. A hex '64' comes back as 101.

Unfortunately, there is no UNORD to get your result BACK to one byte.

You could, I guess, do this:

COMPUTE MYNUMBER = FUNCTION ORD(record-byte).
MOVE WORK-CHAR TO record-byte.

which has the effect of incrementing record-byte by one.
```
