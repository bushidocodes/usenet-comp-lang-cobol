```
I have written an OLE Automation server (32 bits) for PICK
and PICK-like databases. It allows you to develop a Visual Basic
application with all verbs and functions you need
to manage the database (open, read, write, call, ...).

The final Windows application you want to develop, (with Excel,
Visual Basic, Delphi ..) called the client establish a link to
the OLE server. The OLE server launch a request to the remote host.
A server program runs on the remote RDBMS and answer to
the OLE server. The final result is transmitted to the client.
The link between the host and the client is a serial or a telnet
link.

I plan to (re)write the server program for the COBOL/CISAM world.
It's about 800 lines of BASIC I should convert to COBOL. I would
like to know if this kind of tool already exists and if the project
is realistic ? I would add that the OLE server will be a
non-commercial software. Please give me your feedback.
```
