```
There are about 3 things "wrong" with this question:

1) IBM does NOT ever use the term "compliant" for any compiler (or product).
But it is true that VS COBOL II is not on their Y2K ready list.

2) ACCEPT FROM DATE on *any* ANSI/ISO conforming compiler ONLY returns a
2-digit year.  (There will be enhancements in the next Standard - which some
vendors - including IBM have included in their newest products as an
extension.  The use of the YYYYMMDD keyword - where available - should give
you a 4-digit year.)  VS COBOL II (R4.0 is supported until March 2001) will
continue to provide the correct 2-digit year in any ACCEPT FROM DATE
statement.

3) VS COBOL  II does *FULLY* support getting the correct 4-digit year
already:
  - Under CICS, use the EXEC CICS ASKTIME statement
 - Under non-CICS, use a dynamic call to the correct LE callable service
(assuming that you are already using the LE run-time library).
```
