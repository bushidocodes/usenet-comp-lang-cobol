```


Here's a problem for you...

We currently produce a data file that is then downloaded to a PC, zipped
and then transmitted to a third-party client via modem. At the moment the
files are quite small (180Kb) but within 18 months the file sizes will rise
considerable.

I want to compress the file on the mainframe (MVS5.2) during the overnight
batch work. This would reduce both the download time and the subsequent
zipping operation.

Could anyone point me towards some COBOL II source that can produce common
PC format (PKZIP, LHA, etc.) compressed files? Is there an algorithm in the
public-domain? I could use run-length-encoding or something similar, but it
needs to be uncompressed by the third-party at the other end.

Thanks,

Greg
```

compiler on the mainframe though.

We now use a tranmission tool call XCOM which compresses transmissions
on the fly and decompresses it at the other end. I understand that
there is a PC version which I have not tested but it is great to a unix
server.
