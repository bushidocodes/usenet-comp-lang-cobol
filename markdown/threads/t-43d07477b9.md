This will be tough. COBOL uses fixed-length strings, and C uses
null-terminated ones. I may be one of the few people who actually knows
how to do this :)

Normally, you will have a CALL USING for your DLL in the COBOL code.
Something like:

CALL STRINGTHING USING STRINGS

Where STRINGS is an 01 level of some sort. You can either
do it as a bunch of strings:

	01 STRINGS
		05 STRING1 PIC X(...)
		05 STRING2 PIC X(...)
		05 STRING3 PIC X(...)
		05 STRING4 PIC X(...)

Or as an OCCURS clause.

	01 STRINGS PIC X (...) OCCURS ... TIMES

Your choice. If you do an occurs, you will have to pass the number of
times on to the DLL so the C code will know how many positions the table
has! Remember COBOL has fixed length strings and fixed size tables,
there is no dynamic allocation at all: either your C code knows in
advance what to expect, or you pass it in as a parm to the call.

Anyway, your C program has an interface like:

	stringthing(char *strings) {
		...
	}

The point is, to the C program, "strings" is one HUGE chunk of memory
the size of either all the PIC X clauses added up, or the size of the
PIC X times the TIMES in the OCCURS clause. It's a pointer (all COBOL
variables are passed by pointer unless you say otherwise, and you usually
never want BY VALUE when you're dealing with C!) to a block of memory
the right size for all the strings.

You will have to copy one string at a time into the HUGE chunk of memory,
and padd it out with spaces. Be SURE to remove the null terminator,
too. You can use strcpy if you want to, but be sure to keep track of
your positions accurately!

You can also define a struct the same size as the COBOL 01 or table, and
put your strings in there (careful, get rid of that null terminator!) and
write a routine to do the space padding, then memcpy that into your
pointer's HUGE chunk of memory.

Have fun,

Scott
