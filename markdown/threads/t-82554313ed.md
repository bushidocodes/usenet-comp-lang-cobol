I believe that you have no indication that it has been especially long since a
successful compile.  Very recent compilers had backwards compatibility in order
to tolerate this kind of code. This was done to allow users to avoid code
changes.

In later compilers "Any assignment-name after the first is treated as a
comment", according to a VS COBOL II Language Reference. Although I can not say
that this applies to VSE (my experience is MVS/XA and MVS/.ESA).

You state that the code is "supposedly running" in production.  Don't assume
that the program is obsolete. Also don't be surprised if it compiles yet.  The
2314 device identifier certainly suggest a very early origin for this code. 
But I don't think you have displayed anything at all in the posted code that
supports an assumption of obsolescence of the code.  It may well be a cuttent
program.

Best Wishes,

I dug into some reference material from that era. The ASSIGN clause
appears to be ANSI 68. My IBM Full ANS COBOL manual says that
the ASSIGN clause was changed in version 3. VS COBOL was two
releases later. My best guess is version 1 which dates from about
March 1969. Version 2 was 1970; hence too late for this program.
