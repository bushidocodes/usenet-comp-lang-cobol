Some compilers have some way of determining how many variables are passed. 
The Realia compiler had a call one could make to return the number of passed 
variables. In the old IBM compiler, the high-order bit of the last parameter 
was set to indicate there are no more passed values.

are getting and, when that happens, deduce you've only received two values 
instead of three.

```
As you are using Enterprise COBOL<
  "you canNOT do this"
without modifying the CALLing program.  If you do NOT use the "OMITTED" 
parameter (with IBM Enterprise COBOL - or earlier), then the way that IBM passes 
"BY REFERENCE" parameters does *not* zero-out (or make null) the address of any 
non-passed 01-level parameters.  (Actually, it may in some cases, but results 
are UNPREDICTABLE).

This is a "critical" feature (not defect) with the way IBM does CALLs.  This 
means that addressability can be maintained for "non-passed" parameters when the 
same program is called by multiple entry- points or even from the same program. 
(For Micro Focus users, see the Sticky-Linkage directive)

The alternative of passing multiple parameters  - all as sub-items under an 
01-level, would require the subprogram to "get information" from the CALLing 
program (either length or number of sub-items).  Again, change of source code is 
needed.
```
