Wondering if anyone else has experienced the problem we are facing. 

We have
just recently switched from the 16-bit version Runtime version 6.08 to the
32-bit Runtime v 6.51 for our application. When a user tries to print from
our application from a NT Workstation using Novell's Client for NT v 4.x, the
program hangs. In discussing with RM, they aknowleded that they are getting a
different result when their runtime is going a Windows GETFILETYPE 32bit API
call than what they were expecting. They are expecting a File Type of
"Character Device", and instead with this workstation client they are getting
a file type of "Disk Device".

RM is telling me that we are the only one's
reporting this problem to them and they feel it is a Novell problem because
the return code from the API call is not correct. 

I would appreciate any
feedback you may have. If you are also experiencing these problems, please
contact Liant.

Thanks. 

Has anyone else been successful in printing from a
NT Workstation using Novell Client 4.X?  If so, I would greatly appreciate
hearing from you. I feel this is a RM issue and as such they should step up
to the plate and fix it.
