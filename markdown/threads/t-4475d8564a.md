### How spaces are handled by Pro*Cobol

A while back someone posted a question about selecting a column that
contained spaces from a Cobol program. Here's my explanation as to what is
happening.

With Oracle, trailing spaces are either removed or left depending on how
the host variable is defined, the precompiler directives used and finally
how the column is defined within the database.

The Oracle precompiler directive that controls the removal of trailing
spaces is the MODE directive. The MODE directive also controls other
behaviour, and is not necessarily the best method to be used to get the
correct results from the example query.

The next thing that the user could do is to change the definition of the
host variable. At the moment the user has it defined as

01 A PIC X(10).

The user can change it to:

01 A PIC X(10) VARYING.

This turns the host variable into a VARCHAR variable. The Oracle
precompiler will then create the following group item:

01 A.
02 A-LEN PIC S9(4).
02 A-ARR PIC X(10).

With the program the user now moves data to and from -ARR
and uses -LEN to decide how much data is in the variable.

So, the code fragment now becomes:

Move 'AAA' to A-ARR
Move 3 to A-LEN

EXEC SQL
Select col1, col2, col3
from table1
where A = :A
END-EXEC

The thing about this approach is that the column A has to be defined as
either VARCHAR(10), or more likely VARCHAR2(10). If that is the case, then
the above example would work.

The final change to the select statement to cover all cases where trailing
spaces are not required is:

EXEC SQL
Select col1, col2, col3
from table1
where NVL(RTRIM(A),' ') = RTRIM(:A)
END-EXEC

This select will strip off trailing spaces from the column and the host
variable. It will also (the NVL function) convert null columns to spaces.
If the user doesn't want to include null columns, then remove the NVL
function.

The Oracle manuals the user can read for more information are the 'SQL
Language reference' for information on the RTRIM and NVL functions, the
'Pro*Cobol supplement to the Oracle Precompilers Guide' for information on
defining VARCHAR variables and the MODE directive.

I hope this helps.
