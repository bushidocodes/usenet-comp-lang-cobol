Acucobol is no database, but a Cobol dialect. Their filesystem is named
Vision. I presume that is what you refer to.

You may access Vision files from any ODBC compliant front end utility if you
have AcuODBC, which is the ODBC driver for Acucobol Vision files.

If you don't know of this option, it is likely that you don't have it. In
that case you may want to purchase it (www.acucorp.com), you will however,
need to have access to the source of your applications to generate the
mapping tables of your vision files to be used in ODBC.

I guess that the "pointer tables" they are referring to are the XFD files,
the purpose of these files are to map the Cobol file such that when the ODBC
filter receives a record it uses the XFD file to identify the fields in the
record.

> could function.  I guess my question is, is that true and if so would

> all you helped.

It is technically possible for someone that does not have Cobol experience
to do it, but it will be highly preferable to have knowledge to Acucobol in
particular. As I first said, you would need access to the source and an
Acucobol compiler to have the files made. It *is* also possible to create
the files manually, but, it will require an experienced Acucobol programmer,
and that person would also need access to the definition of the vision file.
