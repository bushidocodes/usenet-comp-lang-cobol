I was'nt going to jump into this thread, as I sincerly suspected that the original poster was
just baiting. Ethical question? About GOTO?
But as this may be my last chance to put in my two bits on the GOTO thing (these threads have to
stop soon...), I must say that I agree with Richard.
However, I don't like and don't user PERFORM-THRU. It's only justification is to allow GOTOS to
the exit part. That means we are implementing a RETURN statement which COBOL lacks. I find that
only a small percentage of routines actually need a RETURN statement, and that does not justifty
using THRU-EXIT on *all* routines. For those cases that need it, I use a status flag to bypass
the rest of the code in the paragraph, and control jumps off the end.
What I do like, is a CONTINUE statement, which GOTOs the top of the paragraph. I use it whenever
the "loop" control does not belong in the calling routine. For example, a READ routine that
displays a "Record locked" message and tries again.

Regards
