If you wind up needing to write your own, visit:

http://www.imatix.com/html/libero/lrexam.htm#TOC2

...which offers a free program for parsing PICTURE clauses, probably
the hardest single aspect of parsing the data division.

It's probably written in C. However, it was written via a code
generator called Libero, also offered at the same site, also free.

Libero can generate COBOL as well as C, along with several other
languages. It is specialized for generating finite state machines,
which represent something of a paradigm shift if you haven't
encountered them before. For non-trivial parsing problems it's
worthwhile to learn about finite state machines anyway.

While Libero generates the framework, you still have to code the
details by hand. If you want to write your utility in COBOL, you
could get Libero to regenerate the same framework in COBOL, and
translate the details yourself.
