This makes them static calls rather than dynamic.  While you
can force a CALL "name" to be dynamic with a DLOAD option
you cannot make a CALL variable static, for obvious 
reasons.

With MF a dynamic call would seach the curent .EXE for
the name, so simply linking all the dynamically called
routines into the .EXE wouldn't make them statically
linked, just availble for the search.

I don't think that Fujitsu does any form of resolution
in this way, it is either a static linked in or a
dymanic with the .LIB linked into the main routine.
ie it is not really dynamic, just dynamic loading.
It allows modules to be replaced but not added.

With MF I can type a program name into the menu
program and have it run, Fujitsu doesn't seem to have
a mechanism to allow this.
