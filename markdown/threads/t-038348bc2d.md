```
Hello,
Question about BIGINT. IBM SQL manual says ..

Smallest BIGINT value -9 223 372 036 854 775 808
Largest BIGINT value +9 223 372 036 854 775 807

In a COBOL pgm I coded ..

IF WCA2-RESULT-INT < -9223372036854775808
OR WCA2-RESULT-INT > 9223372036854775807

AN IBM COBOL for Windows 7.6.1 compile then says ..

LineID Message code Message text
645 IGYPS0024-E A numeric literal that contained more than 18
digits was specified. The literal was truncated to 18 digits.
646 IGYPS2002-E Numeric literal "9223372036854775807" was longer
than the maximum allowed length. The literal was truncated to
"223372036854775807".

Implies that 999999999999999999 (18 of them) is the largest a COBOL
pgm can handle? Yes/no?

Please, thanks
Graham Hobbs
```
