```
Current Development Environment:
-----------------------------------------------

- NT, DB2-UDB-v7
- Multiple databases in each database instance
- Stored Procedures are developed in Cobol & deployed as DLLs
- DLLs are placed in C:\Program Files\Sqllib\Function
- Each stored procedure uses more than one program. The main program
issues CALLs to sub-modules.

Everything works like a charm for any one database.

Issue:
-----------------------------------------------

We are not able to have stored procedures for more than one database
per server. As the names of the DLLs have to be the same, we can not
copy the DLLs for the second DB/Version to the same directory where
the first version is stored. We can copy the files to another
directory in PATH. But, then, the other version can not be used.

The only way, we could think of, is to rename all the modules for each
environment. It is not the easiest as we have more than 100 modules.

Does any one of you have any other suggestions? Appreciated.

Siva
```
