Under Windows, you can use the API calls GlobalAlloc or HeapAlloc.

Under 'pure' MS-DOS, there is an int x'21' function you can use, but if you
already have a DOS program running, unless you have made arrangements in
your
program to return unused memory to the OS, your program already has all the
system memory (except you can't reach it).

Under MS-DOS, what I have seen done is to load a TSR - which does nothing
except grab a bunch of memory - BEFORE starting the using program; then the
TSR can respond to an interrupt with the address of that memory and the
calling program can use it.  OpTech Sort for MS-DOS and BTrieve for MS-DOS
work this way.
```

Which COBOL compiler are you using on the PC?  Some of them provide
"callable" memory management routines.  I know that Micro Focus (pre-MERANT)
provided *some* CEE-emulation routines, so you ;may be able to port existing
IBM mainframe code directly to the PC.  If you are a Mainframe Express (or
even still a Workbench) customer, check your documentation for which
CEE-routines are supported.

P.S.  NetExpress *also* provides its own memory management routines.  If you
happen to be using IBM VisualAge COBOL, then I think you could use EXEC CICS
GETMAIN/FREEMAIN to do this even in "otherwise non-CICS" applications.
However, I won't swear to that.
```
