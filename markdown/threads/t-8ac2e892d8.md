I am new to this particular newsgroup, so if this question has been
asked before please forgive me.

I am working on a project that requires me to migrate several hundred
batch COBOL programs and the accompanying JCL streams down to a NT 4.0
server. We are using Microfocus COBOL for the programs and have been
very pleased with the results. We are also using PROXMVS for the JCL
but have been less pleased. Our main problem is that one of the
requirements of the project is for the batch programs to be run in a
"lights out" environment (a mainframe operator runs the jobs
currently). Unfortunately PROXMVS, as far as I can tell, does not
support execution from the command line. It appears to be interactive
only, thus requiring someone to act as an operator.

When I spoke to Microfocus about this problem they told me that I
would need to rewrite the JCL streams in "the scripting language
provided by the target operating system". Which in this case would be
..BAT files. Not the most powerful tool in the world and something I
would like to avoid if possible.

So what I am looking for is either a product that will replace PROXMVS
and allow me to run the jobs unattended or some information (white
paper, book, web page, etc.) on converting the JCL to something that
can be run from the command line or some how shelled to from a driver
program.

Any help you could give would be greatly appreciated. Thanks in
advance.

Jim Sikes
jsi··.@ix.··m.com

Before you give up all hope and opt for .bat files look at the possibility
of using Rexx as a command language. Although it is remembered most for
its use on VM/CMS and as a command language for OS/2, Rexx is IBM's
official SAA language and has been ported to every major operating
system/architecture in use including a version for Windows NT. You may
even be able to find a free interpreter available on the 'Net to exeriment
with. Rexx has been promoted by IBM as a replacement for Exec, Exec II,
JCL and clists and is as powerful or better than the languages it
replaces. For documentation on the JCL -> Rexx translation I would
recommend the JCL book by G. DeWard Brown or my favorite, Practical usage
of MVS Rexx by Anthony S. Rudd ( ISBN 3-540-19952-7). You may not be able
to find it at your local Walden Books store but it should be available at
www.amazon.com.
