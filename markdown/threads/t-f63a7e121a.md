```
(related to other posts)

For IBM shops, if you are working with VSAM KSDS files, it is also worth noting 
that when "checking" file status values (vs Invalid key) that you have access to 
the 2nd file status field (not to be confused with the 2nd byte of the Standard 
file status field).  Especially when debugging IBM "9x" values, this field may 
be very useful.  ON THE OTHER HAND, those values are fairly "obscure".  (I 
usually have a long search before I can even remember where they are 
documented).

As usual, "your mileage may vary".  The one thing that I would suggest (to IBM 
mainframe shops) is that *IF* you are using File Status fields, (for VSAM files) 
that you always DO define the 2nd field as well.
```

```
When testing a file status for success only the first byte should be
checked for '0'. The second byte may contain additional information.
There are several values defined in the standard wich are not '00' and
yet indicate success.

 01  PARM-FILE-STATUS        PIC X(02)   VALUE '00'.
     88  PARM-FILE-STATUS-OK             VALUE '00' THRU '09' '10'
'97'.
```
