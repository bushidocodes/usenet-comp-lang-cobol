```
Same thing for COBOL/400 and VAX/COBOL... depends on what you're doing.

You can 'clear the screen' by using one of the clauses included in the DISPLAY
verb (forget which one it is), but, I've never actually ran across it being
used in a production program. In most cases, the 'screen is cleared' by
bringing up the next form (FMS or SDA).

In the case of Windows (FCOBOL), the 'screen' can't be 'cleared', because the
'screen' doesn't belong to any single program. The most an application can do
is create a blank window.
```

```
Simple solution (although I don't know why you would want to do it this
way):

01 BLANK-LINE PIC X(80) VA SPACES.
|
|
PERFORM CLR-SCR 24 TIMES.
|
|
CLR-SCR.
DISPLAY BLANK-LINE.
CLR-SCR-EXIT.
EXIT.
```
