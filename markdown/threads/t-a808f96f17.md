Further to Mark's answer, you could also use the Program-break feature.

When the animator first loads up, type 'e' (Environment) 'p' (Program-Break)
's' (Select), then the name of the program you want to stop in (prog-c). Now
hit Go or Zoom or whatever. The animator will stop at the first statement in
the procedure division of prog-c, when it is called. Cancel the program-break
(with 'e', 'p', 'c') and use Locate or just scroll thru the source and set your
regular statement breakpoints before continuing animation.

The drawback of this method is that if you wanted to set several breakpoints
in different programs, you'd have to know the order in which they are called
and do it in stages as you can only have one program as the target of a
program-break at any one time. This facility has been availiable in the
animator for a LONG time, so it should be supported in the DOS version you
have.

There is some support for this in the Workbench V3.2 Animator. The
support is called "Application Animation" and is described in the
COBOL System Reference Vol. 1, page 4-36. It is only available for
setting breakpoints in programs which have already been loaded.

There is another way of achieving similar results. A feature of
Animator (and Animator V2) is that breakpoints are stored for each
program until that program is rechecked. So, you could go through the
application in one animation session, set the breakpoints you need,
then the next time you animate, the breakpoints will have been reset
for you (assuming the program hasn't been rechecked).

I hope that's of some help.
