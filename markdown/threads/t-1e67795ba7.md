```
The question of converting "object code" to COBOL source code is one that
comes up occasionally.  Depending on the platform (operating system and
compiler) THERE may be ways (services) to do this.  HOWEVER, the original
note seemed to indicate that the person asking the question didn't even know
what compiler was used to create the object code.  In that case, I think
things are "pretty hopeless".

If you DO know the compiler and operating system, then "repost" with that
information and we MIGHT be able to help.

P.S.  If all you want to know is which compiler was used, you might want to
look at your object code in hex/ASCII "conversion" mode and see if any
"eye-catchers" happen to include the vendor name.  This does happen with
some (most?) object code - but is certainly no guarantee.
```

```
certainty) the rest to C, which is a few abstractions under cobol.
```
