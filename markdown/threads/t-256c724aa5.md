One would just have to ask why.  Are these BLOBs (Binary Large
OBjects) such as pictures or word processing documents ?

Typically these could be stored in separate files in a directory with
the name of the file stored in the data record.
If they are 'normal' variable length with an OCCURS DEPENDING ON for
repeated data then you should revisit the design and consider having
two files, one for the fixed information and the other with records
for each of the repeating group.
