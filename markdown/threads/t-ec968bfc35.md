### Iss-base V2.0 available

Halstenbach ACT is happy to announce the availability of
iss-base V2.0.

iss-base is especially targeted to the creation of complex,
mission critical applications with a strong focus on software
quality. Halstenbach's dedication to quality is demonstrated by the
ISO 9001 certification obtained in 1994.
As an independent developer of Eiffel technology, Halstenbach
has created a development environment covering the whole
software life cycle. Major strenghts of iss-base are
- robustness
- speed of compilation
- a sophisticated user interface management system with
full Windows'95/NT support
- an easy to use GUI-builder relying on object oriented
concepts for assembling dialogs based on reusable and
redefinable building blocks
- reliable ODBC support
- many libraries especically for writing commercial
applications, e.g., extensive date/time support,
BCD-arithmetics, regular expression matchers, ...

The free download version of iss-base V2.0 will be available
by March 2. It will include all components of iss-base:
iss-bench, iss-store, iss-uml,
iss-vision, iss-build, iss-tune,
iss-baselib, iss-webgate, iss-report,
online documentation and guided tours,
the PC-Warehouse example project.

Besides a great number of detail improvemnts and a further
increase of compilation speed the following enhancements
are included in this new release:

- The component mechanism: now it's much easier to
develop, distribute, and use libraries, frameworks,
and subsystems. The new file format for software
components dramatically reduces the number of files
needed for distribution. Now each Eiffel class implementation
can be hidden, safeguarding valuable know-how.

- Class renaming in Ace files: This represents an elegant
means of resolving name conflicts when combining
independently developed libraries as described in ETL.

- Hierarchically structured clusters in Ace files:
Eiffel classes can be grouped more clearly, according
to logical aspects, independently of the physical storage
structure.

- Precursor support: The iss-bench Eiffel compiler fully
supports the "Precursor" mechanism, just recently adopted
by NICE. This mechanism simplifies calls to the original
routine implementation from within a redefined routine.

- Improvements to the dialog management system,
including:
a new unit of measurement ("character"),
3D border display for all container controls,
the capability of specifying a minimum control size,
an event specially designed for validating input.

- The Property Sheet container: this represents a genuine
novelty. Analog to the Grid container, the Property sheet
container provides an automatic layout computation when
loading or resizing the surrounding dialog.

- The fully integrated Command Browser: this new tool
further supports iss-build's RAD capabilities. Services
provided by components are encapsulated in commands,
which can be triggered by user-interface events.
The Command Browser supports searching and browsing
in a command repository as well as the addition of new
command specifications.

- Database "positioned update/delete" support: iss-store
offers this capability for various ODBC drivers.

- Increased database caching granularity

- Context-sensitive ODBC message classification: ODBC
messages are now better integrated into the iss-base
message system.

- Central network installation

homepage at http://www.halstenbach.com or
