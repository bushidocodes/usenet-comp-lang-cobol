```
jasonc@collegeestrie.com wrote:
> 
> Would someone please explain to me the difference between MOD and REM.
> 

The difference is in handling signed items (mainly, one negative and 
one positive).  MOD is defined as (argument-1 - (argument-2 * FUNCTION 
INTEGER (argument-1 / argument-2))) and REM the same except for 
INTEGER is INTEGER-PART.  So what, you might say.  INTEGER means the 
greatest integer less than or equal to the value of the argument.  If 
you have an argument of 1.5 you get 1.  However, -1.5 gets -2.  
INTEGER-PART just chops off the fraction.  Therefore, MOD (11, -5) 
returns -4.  REM (11, -5) returns +1.  If all you are using is 
positive numbers, there is no difference.  By the way, don't start 
ranting about the COBOL Committee defining MOD in a dumb way.  That is 
the mathematical way and is used by all the other languages and 
defined that way in the ISO standard.
```
