```
(Partially, this note is in reply to the thread on ISPF options and
compiling/scanning your COBOL source code.)

I don't think it has received lots of publicity but do you know about the
IBM (mainframe and PC - I don't think it is on COBOL/400) feature that lets
you "syntax scan" EXEC CICS and EXEC SQL statements during your compile step
(without using the preprocessor or translator)? According to the latest APG,

"Use ANALYZE when you want the compiler to check the syntax of imbedded SQL
and CICS statements in addition to native COBOL statements."

For more detail, see

http://ppdbooks.pok.ibm.com/cgi-bin/bookmgr/bookmgr.cmd/BOOKS/IGYPG201/2.3.3
..2
```
