Terry Heinze wrote:

> Main programs that have PARM parameters defined DO have LINKAGE SECTIONs.

I assume that you are writing using an IBM mainfram compiler.

The '85 standard for this is:

"The LINKAGE SECTION in a program is meaningful if, and only if, the
object program is to function under the control of a CALL statement, and
the CALL statement in the calling program contains a USING phrase."

as I said in my message:
 
> > It seems to be another IBM-ism that main programs can have a LINKAGE and
> > thus may be executed as a main or be called.
 
It is possible that other systems have this as an extension, but I have
not come across any other (though there are many systems that I have not
come across at all).

Consequently the 'problem' with EXIT PROGRAM is restricted to IBM
Mainframes (AFAIK), which is why GOBACK is advocated by IBMers, and
others don't necessarily see the point.

As I said: _I_ know when to use STOP RUN or EXIT PROGRAM so it will not
'bite' me.

I have also mentioned in another thread that the 2002 Cobol standard
should be written so that it can (or even should) form the basis of the
implementors Language Reference manual with marked annotations as
necessary to explain departures from the standard and extensions (as is
done by MF).  In this way the Cobol programmers will be aware of what is
part of the Cobol standard (or of various standards) and what are
extensions that may be specific to just their particular compiler.

Page II-64, EXIT statement, paragraph 5.11.4 GR1:  "An EXIT statement serves
only to enable the user to assign a procedure-name to a given point in a
program.  Such an EXIT statement has no other effect on the compilation or
execution of the program."

ANSI X3.23-1985:
Page X-33, EXIT PROGRAM statement, paragraph 5.4.4 GR1:  "If the EXIT
PROGRAM statement is executed in a program which is not under the control of
a calling program, the EXIT PROGRAM statement causes execution of the
program to continue with the next executable statement. "
