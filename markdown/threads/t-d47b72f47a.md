You can 'sort of write a function' if you use OO - so here goes. This is 
a 'play' on what Bill Klein coded and overcomes the problem where you 
have two words with more than one space between them in the same input 
element :-

       *>--------------------------------------------------------------
       Program-id. InputText.

       Class-Control.      MakeAstring is class "makstrng".

       Working-storage section.

       01 TableA.
          10 pic x(80) value "   Humpty Dumpty      ".
          10 pic x(80) value "   sat on the wall,   ".
          10 pic x(80) value "Humpty          Dumpty".
          10 pic x(80) value "had a     great fall. ".
          10 pic x(80) value spaces.

       01 InputLength                       pic x(4) comp-5.

       01 StringInfo.
          05 StringLength                   pic x(4) comp-5.
          05 StringText.
             10 occurs 1 to 500
             depending on StringLength      pic x.

       01 StringChoice                         pic x(4) comp-5 value 0.
          88 NoSpaces                          value 0.
          88 WithSingleSpace                   value 1.

       Procedure Division.

       compute Inputlength = function length (TableA)

       *> Returned string includes NO spaces between words

       set NoSpaces to true
       invoke MakeAString "makeText"
               using StringChoice, InputLength, TableA
               returning StringInfo

       if StringLength <> zeroes
          display StringText
       End-if

       *> Returned string includes one space between each word

       set WithSingleSpace to true
       invoke MakeAString "makeText"
               using StringChoice InputLength, TableA
               returning StringInfo

       if StringLength <> zeroes
          display StringText
       End-if

       STOP RUN.
       *>--------------------------------------------------------------
       *>----------------- makstrng.cbl --------------------------------

       Class-id.       MakeAstring inherits from Base.

       Class-Control.  MakeAstring             is class "makstrng"

                       CharacterArray          is class "chararry"
                       .

       *> Note : A period (.) follows the 'final' Class-Control entry

       *>--------------------------------------------------------------
       FACTORY.
       *>--------------------------------------------------------------
       Method-id. "makeText".
       *>--------------------------------------------------------------

       *> Using input data, the class :-

       *> (1) creates a CharacterArray then
       *> (2) uses "asArrayOfSubstrings" to ignore intervening white
       *>     spaces. White space in this method, is defined as being
       *>     any character with an ascii value that is less than
       *>     or equal to 32 (space).
       *> (3) Builds the string setting the counters for
       *>     lengths of strings.


       Local-storage section.

       *> Max Values that we can check against - or change

       78 ls-StringLength                  value 80.
       78 ls-MaxLength                     value 500.

       *> other :
       01 n                                pic x(4) comp-5.
       01 ls-bytes                         pic x(4) comp-5.
       01 ls-NumberOfElements              pic x(4) comp-5.
       01 ls-TempText                      pic x(ls-StringLength).
       01 ls-TempCount                     pic x(4) comp-5.

       01 ls-SeparatedStrings              object reference.
       01 ls-String                        object reference.

       Linkage Section.
       01 In-Choice                     pic x(4) comp-5.
          88 NoSpaces                   value 0.
          88 WithSingleSpace            value 1.
       01 In-Length                     pic x(4) comp-5.
       01 In-Text.
          10 occurs 1 to ls-MaxLength
             depending on In-Length     pic x(4) comp-5.

       01 Lnk-Out.
          05 Out-Length                    pic x(4) comp-5.
          05 Out-Text.
             10 occurs 1 to ls-MaxLength
                depending on Out-Length    pic x(4) comp-5.

       Procedure Division using In-Choice, In-Length, In-Text
                          returning Lnk-Out.

       initialize lnk-Out

       if In-Length > ls-MaxLength
          display "In-Length exceeds MaxLength"
          EXIT METHOD
       end-if

       invoke CharacterArray "withLengthValue"
                using     In-length, In-Text
                returning ls-String
       invoke ls-String "asArrayOfSubstrings"
              returning ls-SeparatedStrings
       invoke ls-SeparatedStrings "size" returning ls-NumberOfElements

       initialize Lnk-Out, ls-TempCount
       Perform varying n from 1 by 1 until n > ls-NumberOfElements

           invoke ls-SeparatedStrings "at"
                  using n returning ls-String
           invoke ls-String "sizeinBytes" returning ls-bytes

           if ls-bytes > ls-StringLength
              move zeroes to Out-Length
              display "String exceeds Maximum Bytes for ls-TempText"
              EXIT METHOD
           End-if

           invoke ls-string "getValuewithSize"
                  using ls-bytes returning ls-TempText(1:ls-bytes)

           Evaluate true
             when NoSpaces
                compute Out-Length = Out-Length + ls-bytes
                String  Out-Text (1:ls-TempCount)
                        ls-TempText  (1:ls-bytes)
                        into Out-Text
                End-string

             when WithSingleSpace
                compute Out-Length = Out-Length + ls-bytes + 1
                String  Out-Text (1:ls-TempCount)
                        ls-TempText  (1:ls-bytes)
                        " "
                        into Out-Text
                End-string

             when other
                move zeroes to Out-Length
                display "Invalid String Choice"
                EXIT METHOD

           End-Evaluate

           move Out-Length to ls-TempCount

       End-Perform

       End Method "makeText".
       *>---------------------------------------------------------------
       END FACTORY.
       *>---------------------------------------------------------------
       *> OBJECT.
       *> END OBJECT.
       *>---------------------------------------------------------------
       END CLASS MakeAstring.
       *>---------------------------------------------------------------

Here is what we use : 

rtrim.cbl
      $set sourceformat"free"
1 tally-count pic 9(7).
Linkage Section.
1 lnk-len   pic s9(9) usage binary.
1 Lnk-StringX.
  2  Filler occurs 1 to 32000 Depending On Lnk-Len PIC X.

Procedure Division Using Lnk-Len Lnk-StringX.

move zero to tally-count
inspect function reverse(Lnk-StringX)
  tallying tally-count
  for leading spaces
if tally-count > 0 then
  move low-values to Lnk-StringX(lnk-len - tally-count + 1 :
tally-count)
end-if

Exit Program
stop run
.
--------

so you can write :

call "rtrim" using length of AT-Entry (1)  AT-Entry (1)
call "rtrim" using length of AT-Entry (2)  AT-Entry (2)

string 
    AT-Entry(1) 
    AT-Entry(2) 
    delimited by low-values
into
    Output-Table
end-string

Of course, you will find low-values after "ThisThat".

You can also write another code which will do the same 
trim.cbl
       working-storage section.
       01 i pic 9(4) comp.
       linkage section.
       01 ls-pStr usage pointer.
       01 ls-len   pic s9(9) usage binary.
       01 lnk-texte   pic x.
       procedure division using ls-pStr, ls-len.
       paragraphe.
           set address of lnk-texte to ls-pStr
           perform varying i from ls-len by -1
           until ((i < 1) or (lnk-texte(i:1) <> " "))
               move x'00' to lnk-texte(i:1)
           end-perform
           exit program
           .
the call is then :

       call "trim" using
           by reference address of AT-Entry (1)
           by content length of AT-Entry (1)
       end-call

They all work.

With the first version, we have been experimenting some problems with
the length of the string. We had to reduce 32000 to a lower amount due
to an error during execution.

Regards.

Alain

Robert,
   Just so you know, there is one more difference (between the requirement for 
the BASED phrase) between existing IBM and Micro Focus implementations and that 
required in the '02 Standard.

For IBM and Micro Focus, a Linkage Section item for which the SET ADDRESS OF 
statement establishes addressability need not - but MAY appear in the Procedure 
Division Using statement.  If it appears there but a SET ADDRESS statement is 
used against it, it "changes" where that item is now located.

For the '02 Standard, there is an explicit rule that an item in the Procedure 
Division Using statement may *NOT* have a BASED phrase.  Therefore, some 
existing IBM and Micro Focus applications may fail to compile with a restricted 
to '02 Standard COBOL compiler - even if BASED phrases are added.

From my personal experience, MOST Micro Focus and IBM applications using SET 
ADDRESS OF to establish addressability to a Linkage Section item do NOT include 
them in the Procedure Division header.  HOWEVER, I have also seen some that do 
(for cases where a parameter is passed to a subprogram - and then that LS item 
is "re-addressed" based on application logic)

FDIS, page 137, 8.8.4.3 Order of evaluation of conditions
"Parentheses, both explicit and implicit, denote a level of
inclusiveness within a complex condition. Two or more
conditions connected by only the logical operator 'AND'
or only the logical operator 'OR' at the same level of
inclusiveness establish a hierarchical level within a complex
condition. An entire complex condition can be considered
to be a nested structure of hierarchical levels with the entire
complex condition itself being the most inclusive hierarchical
level. Within this context, the evaluation of the conditions
within an entire complex condition begins at the left of the
entire complex condition and proceeds according to the
following rules recursively applied where necessary:

1) The constituent connected conditions within a hierarchical
    level are evaluated in order from left to right, and evaluation
    of that hierarchical level terminates as soon as a truth value
    for it is determined regardless of whether all the constituent
    connected conditions within that hierarchical level have been
    evaluated.

2) Values are established for arithmetic expressions and
    functions if and when the conditions containing them are
    evaluated. Similarly, negated conditions are evaluated if and
    when it is necessary to evaluate the complex condition that

Yes, and it 'short cicuits'.  Once T2 is < 1 it won't be used as a
subscript (or in this case reference notation) and thus won't
(potentially) cause a boundary violation - it doesn't need to make
that test.

Other languages also 'short circuit', for example C, and this can lead
to a potential problem with it where user defined functions are used
in the tests.  For example, in C:

      if ( ( value > 0 ) && ( makePayment(value) != 0 ) ) { .. };

makePayment() is a function that will only be executed if value is >
zero because the evaluate of the test stops as soon as the truth can
be determined. Any side effects from the function will not happen. 
This case is simple so it is easy to see what is happening, but it may
be wrongly assumed by the programmer that the function should have
been executed.

When user defined functions are available they will need to consider
this, or ensure they do not have side effects.

'extern "C"' indicates that a 'function type, function name, [or]
variable name' was compiled by another language and has the
language linkage of the C programming language.

"C" language linkage. [Note that in 1 and 2, the default language
linkage is C++, therefore 'extern "C++" is never required for a
'code fragment' compiled with C++.] This means that using
'extern "C"' 'to define a C++ function which will be callable with
C linkage' is an implementor extension and is inconsistent with
the intent expressed in 7.5.

libraries' does not mean that 'extern "C"', in that context, is
C++. It might be that since standard C has no means for
identifying language linkage [there is 'fortran' as a
function-specifier] it is simply more convenient for C++
implementors to 'abuse' C++.

It seems to me that the means for COBOL to call C++
within the standards for both is the CALL-CONVENTION
directive. This is where a COBOL implementor may define
the equivalent of 'extern "C++"' to permit COBOL to call
C++ directly.

...
>>CALL-CONVENTION C++
    CALL "cpptrim" USING MY-STRING
>>CALL-CONVENTION C

>>CALL-CONVENTION COBOL
