There are at least two ways that come to mind:

TECHNIQUE #1 - OPEN INPUT and check the file status values. If the
file does not exist, you'll get an appropriate file-status value
You'll ALSO get run time errors. To avoid the run time errors, you'll
need a DECLARATIVES section. In spite of what the compiler says,
DECLARATIVES do work. Here's how we do it:

  A. We use a copy book, viz:
     PROCEDURE DIVISION.
     COPY 'C:\CB\DECLAR.COB'.
     ...
     OPEN INPUT MYFILE.
     IF MYFILE-FS NOT = '00'
        OPEN OUTPUT MYFILE.

The DECLAR.COB above MUST be accessed via a COPY command. Do not use
#INCLUDE.

   B. 'DECLAR.COB' looks like:
       DECLARATIVES.
       DECLAR-INPUT SECTION.
           USE AFTER ERROR PROCEDURE ON INPUT CONTINUE.
       DECLAR-OUTPUT SECTION.
           USE AFTER ERROR PROCEDURE ON OUTPUT CONTINUE.
       DECLAR-IO SECTION.
           USE AFTER ERROR PROCEDURE ON I-O CONTINUE.
       DECLAR-EXTEND SECTION.
           USE AFTER ERROR PROCEDURE ON EXTEND CONTINUE.
       END DECLARATIVES.
       PROCEDUR SECTION.

Notice the code in the DECLARATIVES section does absolutely nothing.
We just need the presence of the section to avoid the run time errors.


TECHNIQUE #2 - Use

   CALL 'CBL_CHECK_FILE_EXIST2' USING
     filename
     filedetails
     RETURNING status-code

This method has the advantage of providing file size, create date, and
create time. "Filename" is the external name of the file (... ASSIGN
TO filename) and is delimited by low-values. DO NOT use the
corresponding CBL_CHECK_FILE_EXIST as it fails on imbedded spaces in
the path.
