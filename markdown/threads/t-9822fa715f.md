```
Hi All,

After doing quite a bit of hunting around and even contacting the
author of a book on serial communications, I'm still trying to find
out how to handle network modems/modem pools in my COBOL app - or find
out even if I have to!  The app is written in MF Cobol 4.0.32 and has
been working fine for the past 1.5 years with modems connected to the
PC it' running on.  It uses Windows API calls to talk to the COM port
and uses a GUI front end provided by MPACT from ESI.

From what I can find out, it looks like in order for an application to
talk to a network modem, there has to be some kind of a 'redirector'
resident on the PC from which the app is running which tells Windows
that this COM port is really somewhere else on the network - sort of
like mapping a printer port.  Is this correct?  The libraries that
I've found, Marshallsoft, Sax, Crescent, etc. have no mention of
network modems/modem pools in them.  You connect to a COM port and
that's it.  It also appears that the 'redirector' can vary by which
brand of network you have.  This would also imply that the end user
would have to be responsible for setting it up before my app could
access a network modem.

Any information or experience anyone has had with network modems would
be greatly appreciated.  I'm not even sure I have to do anything to my
app other than update the help file!

Doug
```
