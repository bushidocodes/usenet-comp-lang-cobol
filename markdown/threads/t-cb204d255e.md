```
Hello fionarei,

The Net Express with .NET product has a pinvoke example included in the product 
example directly that directly meets your needs but to answer your questions 
directly.

> 1. If you have a C library (not COM) that is accessed in Micro focus NetExpress 
using 
> SET W-TML-PTR TO ENTRY "xyz_c.dll".
> How do you access that library from a MF .Net

You can comment this code as it is not required for .NET, or you can place 
it under condition complication eg:
 *> NOT required for a .NET program
 $if NOILGEN set
 SET W-TML-PTR TO ENTRY "xyz_c.dll".
 $end

> 2. If you have a COBOL library that is accessed in MF Express using
> SET W-CISCO-COB-PTR TO ENTRY "abc_cobol.dll".
> How do you access that library from MF .Net ?

You just do a normal CALL statement but when you compile your managed program, 
you need to tell the compiler to resolve extra unmanaged symbols from your 
unmanaged .DLL.

In the IDE you need to add the unmanaged DLL as a reference to the project 
or from the command line use the inpinvoke"abc_cobol.dll" directive.

>  3. If you have the source code for a COBOL library, how do you convert
> that to a MF COBOL .Net library?

You create a COBOL library project, include your sources in the project, 
build it and yeh presto you have a managed assembly. 
```
