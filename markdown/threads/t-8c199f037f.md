Ken MacKenzie writes:

› Can anybody tell me what the second parameter to CEE3ABD
› (clean-up) actually achieves. Does anybody have any
› recommendations as to whether specifying 0 or 1 is best?

If you specify the second parameter as 0:
a. No LE dump is generated
b. LE condition handling is disabled (not used) for the current enclave
c. Debug tool is not invoked
d. User exits are not invoked
e. User condition handlers are not invoked

If you specify the second parameter as 1:
a. Dump / trace production depends on ABPERC and TRAP runtime options
b. Condition handlers (LE and user) get a chance to intercept the abend
c. Debug tool, if available, is invoked
d. User exits are invoked

I tend to prefer "1", but I tend to be the "neat and tidy" type (except
for my office).
