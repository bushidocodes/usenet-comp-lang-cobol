Obviously people are not reading your requirements.   A Windows only 
compiler such as MF Personal COBOL for Windows, and Fujitsu do not 
meet your needs.

You are wanting to mix apples and oranges.  If you want NT support, 
you probably desire 32 bit COBOL - which means it WON'T make DOS 
programs.

However, if you want DOS programs you need a 16 bit compiler.  The 16 
bit offerings from MicroFocus and CA-Realia fit this bill.  You can 
create 16 bit Windows executables that will work under NT with each of
these.

The one with the widest platform support is AccuCOBOL, but it is also 
the one with the largest per install cost.

If you can back off the the DOS requirement, there are more choices 
available at a more reasonable price.
