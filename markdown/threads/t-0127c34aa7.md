Okay, this crusty old mainframe programmer thought he should get
current with Cobol as it is in 1998 so I spent my hard earned shekels
on Cobol Unleashed.  I really wanted it as a reference book since all
the new stuff that I never get to use was in it.  And what did I find
but a really nice CD that had Casegen Cobol for Windows and Fujitsu
Cobol and others as well.  With new software I kind of like Tim Taylor
in a hardware store..I've got to install it and play with it.  So
after browsing through the directory on the CD I printed off the
Winword doc that was in the Casegen sub-directory.  Reading it told me
to install Fujitsu first, then Casegen.  Which I did.  No problems.
Then, continuing to read the instructions in the doc, I selected one
of the sample Casegen programs, double clicked on it, then pressed the
generate button.  When I selected RUN, I got two error messages.

1)  Registry error - cannot find compiler registry key:
Software\Fujitsu\Cobol85

2)  Compiler not found

Okay, I can read English.  Looking at my hard drive I found that
Fujitsu had installed itself in C:\Fsc and there are numerous
sub-dirctories under it.  One is called PCobol32.  In it is
application powercob.  Nowhere did I find anything called Cobol85.

Another anomaly was in executing Powercob.  When executing this
program I get an error that says:  F5BA1243 Cannot Open Registry
Information.  And then Powercob goes into a really nice loop that is
almost impossible to get out of.

Apparently, my Windows 95 registry information did not get set up
properly during the installs of the software.  Being I am crusty and
have only been working with Windows 95 from a user stand point for
about a year, I am not as versed in error correction as I am with
Windows 3.11.

Anyone know what I should do?

Thanks.

P.S.  Despite the above mentioned problems, the book is terrific.

Regards,

May we apologise to everyone who has tried using Casegen COBOL for Windows
supplied with the CD on "Cobol Unleashed" and had problems compiling their
programs.  You will have received this message:

'Registry error - cannot find compiler registry key... ".

This problem is due to an error with the Fujitsu compiler on the CD.  For
some reason the compiler does not get installed correctly and Casegen needs
this compiler to build Cobol programs generated by Casegen.

It will be necessary to install Fujitsu Version 3 or Version 4 from another
source to use Casegen.  We are currently discussing this matter with Fujitsu
to provide you with the correct software as soon as possible.

We hope to have a response in the next few days.

Again, our apologies - I know how frustating it can be trying to get
software to work, but I am sure you can appreciate that this problem is
outside our control.

If you need to try Casegen COBOL for Windows urgently, please call us on
+44 (0) 1494 451 014 during office hours (in the UK) and we will airmail you
a CD.

Simon Cordingley
