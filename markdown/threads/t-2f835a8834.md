```
It depends on the way you defined to access the file in the Environment
division.

If you use "Access is Sequential" than a read will get you the next
record, and a rewrite will rewrite the present record. If I'm not
mistaken a write can only be done if the primairy key is higher than the
highest primary record already in the file. The file will always be
accessed in sequence of the primary key!

If you use "Access is Random" you will have to fill one of the keys of
the record, an a subsequent "Read Key is key-just-filled" will give you
the record you want unless there are more records with the same key
value , or you get a "Invalid" situation. A Write and Rewrite can
allways be done in this mode, unless of course you duplicate errors etc.

With "Access is Dynamic" you get the best of both worlds. You can still
use the same method as with Random, but you can also use a "Read Next"
to read the next record in sequence of the key you specified the in last
Read. There is also a Start command you can use. Let's say you want to
read keys with a value higher than 100, then you can use the following
code:

Move 100 to key-I-need
Start file Key > Key-I-need
(invalid or not-invalid etc.)
Read file Next  

This code will give you the first record with a key > 100.

Hop this helps
```

```
[NOT ] INVALID KEY is used only on I-O for files with access random or
dynamic.  Sequential files use a relative key, indexed files use one or more
record keys.

For sequential files, you use READ and REWRITE with no intervening access of
the target file. (This may not be supported by some COBOLs on some operating
systems, I'm not sure of that).
```
