The way to find out why is you have to debug the application to see what
code may or may not be executing.  If you have an animator you can follow
the code step by step and check values and the like.  If you don't have an
animator you have to do it through a series of display and accept
statements.

After each line of code executed do a display with a message of what the
code was and an accept so it will appear on the screen.  so you know it is
the debug statement use your RETURN-KEYB field and do a statement like:

display blank-screen.
display "Add record line 1" 1525.
accept RETURN-KEYB at 2479 with no-echo.

What that will do is display the message clear the screen then display the
message in about the center of the screen then the cursor will be in the
bottom left hand corner and will not accept input data into the field but
stops the cursor so you can see the screen.
