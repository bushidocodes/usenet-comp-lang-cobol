```
Hello readers,

perhaps I am a bit 'overdozed' but when I sometimes read ancient sources
of Mainframe COBOL looking a bit 'quick and dirty' I guess others must
have had similar problems: 

No nicely arranged VALUES, every TO in a different col, no plain nesting
in WORKING STORAGE at all, no nested IF's, everything lookin a bit
unaligned. It looks like a mess and only due to this it is uncomfortable
to analyze.

Is anybody out there using a tool - just like a comb - to arrange given
source code so that it looks a bit more readable?

I spent some time on a REXX routine but I am still not satisfied with
the results.

Anybody willing to share experience?

I used tools like Cobol Workbench or Visual Cobol but these did not make
my life much easier. I want to have something available in the ISPF Edit
environment.

Okay, ducked and covered......
```

```
In 1992 I wrote a COBOL program to convert OS/VS COBOL source code to
VS COBOL II.  I was primarily concerted with fixing up OS/VS COBOL
uses of CURRENT-DATE and WHEN-COMPILED, plus converting BLL cells and
SERVICE RELOAD to COBOL II usage.  If it needed to, it inserted
working-storage items and made changes to procedure division code.  It
also had a couple of PARM overrides to change its conversion
behavior.  I always ran it as a plain batch job under MVS, but it
might be adaptable to run in ISPF.  I used it to convert about 100
programs to COBOL II, a mix of batch and CICS programs.

It also aligns PIC, REDEFINES, and VALUE clauses, and TO objects in
procedure division.  I'd say it works better on data division than
procedure division, depending on what the input COBOL source code
looks like. 

If you're interested, I could zip up a copy of the source code, plus
some notes on how to set up the JCL.  It reads source code as 80-byte
records and writes an 80-byte record file, which you can direct to a
PDS or a QSAM file.  The COBOL program was about 3800 lines, and
should compile clean under COBOL II or higher.

Let me know if you'd like a copy.
```

```
such as rewriting GO TO code into PERFORM only code.  It was
able to detect dead code, and other things.
```
