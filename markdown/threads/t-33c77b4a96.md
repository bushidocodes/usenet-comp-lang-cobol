I have Fujitsu code to do this, if you want it.  To tell the truth, though,
I am replacing it with SP2 Formprint, as it was primarily a stopgap measure,
and while it works, it is a bit clumsy.  The code works as follows:

The code is written to produce a plain ASCII text print file, as a line
sequential file complete with formfeeds.  It is then spooled using the
subroutine and a form name.  The form is the output of MS word.  That is,
create the form, and print  the form to a file. The file can be tested by
copying it to the printer, and you should get a blank form.

The spooler scans the text file for formfeeds.  Whenever it finds one, it
ejects the current page (if one is there), and reads the form file, passing
it to the printer.  When it gets to the final formfeed of the form, it
stops, resets the laser cursor to line one, column 1, resets the font to
courier, and overprints the next page of the report on top of the form
(laser printers have the ability to use cursor control, the same as a
screen). On the last page, it give a final formfeed to eject the last page.
Voila!  Report is printed on a form.

The major problem is that it is written for an HP laser IV only. By going to
FORMPRINT, you can print to any printer.
