```
This sounds suspiciously like a mismatch in the calling convention used by
VB and the convention expected by the COBOL code. Use the special-names
section to resolve it.

special-names.
    call-convention 74 is from-vb.

procedure division vrom-vb using...

That should (hopefully) fix it.
```
