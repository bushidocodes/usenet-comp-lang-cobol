Oracle has some makefiles supplied with the database, and these are
installed with PRO*COBOL.
look at $ORACLE_HOME/precomp/* structure for makefiles for cobol.
You should have a procob and a procob2 directories both with examples
of makefiles.

It is advised by Oracle to use these makefiles as base for your own,
so do look at them.

IF you do not have the above folders available then you may need to
install PRO*COBOL as this is not installed by default.

If you're working on UNIX :

In addition to what Frederico has said above, you need to have your COBOL
environment set up in order to get the option to install Pro*COBOL on UNIX.

On both UNIX and Win32 :

If having done that you rerun the Oracle installer, select a Custom install,
and then Oracle Developer, by expanding the tree you should see Pro*COBOL
available for installation. As part of the install on UNIX, it will recreate
a new run-time including the Oracle support libraries (rtsora, under
$ORACLE_HOME/bin).

Under $ORACLE_HOME/precomp/demo, you will find two subdirectories, procob --
for use with the Oracle 1.8.xx precompiler -- and procob2 -- for Oracle
8.x.xx or 9.x.xx. Those directories include sample programs, and makefiles
(for UNIX) and a batch file (for Win32), which should point you in the right
direction for the correct build command(s) to use for your application.

Hope this helps.
