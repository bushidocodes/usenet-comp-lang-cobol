You could do something independent of any special function
of any compiler.

set up a table like
01 all-hex-tab.
03 filler pic x value x"00".


and so on - then redefine it

01 all-hex-tab-1 redefines all-hex-tab.
03 filler occurs 255.
05 my-hex-value pic x.

01 my-dec-value pic 999.
01 ind pic 999.

in your procedure


move "@" to ws-x
perform find-dec-value
if w02-exist = 1
perform use-dec-value
....or what ever

find-dec-value section.
move 0 to w02-exist.
perform with test before
varying ind from 1 by 1
until ind > 255
or w02-exist = 1

if ws-x = my-hex-value(ind)
add -1 to ind
move ind to my-dec-value
move 1 to w02-exist
end-if
end-perform.

ODE is not a COBOL 9x function. The function ORD is part of the
current standard, however. By using 'COMPUTE store-val = FUNCTION ORD
("@") - 1' you get the ASCII code provided that your system's
character set is ASCII and that the program collating sequence is also
ASCII.

The hex literal X"00" is not part of any standard. It is dependent on
the compiler you are using. Therefore, the code is not independent.

01 CHAR-VAL PIC S9999 COMPUTATIONAL.
01 CV REDEFINES CHAR-VAL.
02 FILLER PIC X.
02 CHAR PIC X.

MOVE 0 TO CHAR-VAL.
MOVE Char-field TO CHAR.

CHAR-VAL now has the ASCII code for Char-field. There might be a difference
on the numeric field usage or order, but essentially the above is what you
need to do.
