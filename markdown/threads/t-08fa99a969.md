Hello,

I'm just completing a COBOL project and it seems to work fine except for one
little snag. I'm using the MicroFocus COBOL compiler/linker and I'm working
in a Win98 environment (in a dos-box). I use the following compile and link
command line: To compile: COBOL prog.cbl /NESTCALL To link: LINK prog.obj
adis.obj adisinit.obj adiskey.obj extfh.obj cobintfn.obj

Thiw works and I'm able to run my program without any problems on _my
computer_, which has COBOL installed of course. To be able to run it on a
different computer though it seems I have to link in LCOBOL.LIB but this is
were the problem lies. When I do this with this command line:

LINK prog.obj adis.obj adisinit.obj adiskey.obj extfh.obj
cobintfn.obj,,,LCOBOL.LIB,,

(all on one line of course)
Then the program crashes (Windows gives an 'performed illegal operation')
whenever I try to write/delete a record in a file.

If anybody has any ideas as to why it does this and more importantly how I
could solve this it would be much appreciated.

Thanks in advance,
