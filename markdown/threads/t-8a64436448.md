Dave, you can't insert Assembler instructions into a COBOL/370 (or later)
program.

Many years ago, some compilers let you insert Assembler by coding

ENTER ASSEMBLER
... your Assembler code here ...
END ASSEMBLER

or something like that. But I'm not aware of any modern compiler that let's you
change languages on the fly.

OTOH, if you want to change DDnames on the fly, you can do that sort of thing
using TSO commands, and you can issue TSO commands from COBOL programs(!).

But that's a different class (Advanced TSO REXX), more money.
