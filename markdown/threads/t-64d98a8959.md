```
Does anyone have some Cobol code that will do a CRC checksum on field or
data record?

I realize this is a rather vague question but what I want to do is send
1000 byte records between machines over a socket and make sure that I
receive exactly what has been sent. The machines I'm running on do not
have TCP checksum enabled by default and significant overhead is
incurred by turning it on.
```

```
computing the CRC in COBOL also incurs overhead !

The following example is not quite portable. The COMP-4 might have
to be adjusted for your compiler, but the code shows how to
add up characters of the data. For a better CRC you might want
to use a more sophisticated algorithm, but the general principle should
be clear.

000100 IDENTIFICATION DIVISION.
000200 PROGRAM-ID.    CRC.
000300
000400 AUTHOR.        LEIF SVALGAARD.
000500 DATE-WRITTEN.  99/10/14
000600     -REVISED:  99/10/14.
000700
000800 ENVIRONMENT DIVISION.
000900
001000 CONFIGURATION SECTION.
001100 SOURCE-COMPUTER. NOT-PORTABLE.
001200 OBJECT-COMPUTER. NOT-PORTABLE.
001300
001400 DATA DIVISION.
001500
001600 WORKING-STORAGE SECTION.
001700
001800 01  DATA-RECORD.
001900     02  DATA-CHAR               PIC X      OCCURS 80 TIMES.
002000
002100 01  CRC-VARS.
002200     02  CRC-SUM                 PIC S9(18) COMP.
002300     02  CHAR-PTR                PIC S9(3)  COMP.
002400     02  CHAR-VALUE              PIC S9(4)  COMP-4 VALUE +0.
002500     02  FILLER                  REDEFINES  CHAR-VALUE.
002600         03  FILLER              PIC X.
002700         03  THE-CHAR            PIC X.
002800
002900 PROCEDURE DIVISION.
003000     MOVE "SOME DATA" TO DATA-RECORD
003100     MOVE ZEROES TO CRC-SUM
003200     PERFORM FORM-CRC
003300       VARYING CHAR-PTR FROM 1 BY 1
003400         UNTIL CHAR-PTR > 80
003500
003600     DISPLAY CRC-SUM
003700     STOP RUN
003800     .
003900
004000 FORM-CRC.
004100     MOVE DATA-CHAR (CHAR-PTR) TO THE-CHAR
004200     ADD CHAR-VALUE TO CRC-SUM
004300     .
```
