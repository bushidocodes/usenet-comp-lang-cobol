```
Hi everyone,
 I'm working on a project and I'm kind of new to the real world programming.
I'm developing a program to print labels that will get the information from the
master.dat file. First the clerk enters an account number and the program
checks to see if the account number is on the master.dat if it finds it it will
move some of the fields to the labels record to print. My problem is the key. I
have not much experience with keys and I cannot read the record because I don't
know how to load the key before reading the record. Here is the file-control
for that master.dat

select armaster1 assign to "armaster.dat"
                                organization is index
                                access mode is dynamic
                                record key is ar-key
                                alternate record key is ar-name
                                with duplicates
                                alternate record key is ar-phone-4
                                 with duplicates
                                 file status is fl-stat.
could someone give me some examples of how to load the key before a read.

Thanks in advance

Robert
```

```
Simply do a move to the key field.  Using the select clause
you gave the reads would go as follows:

       move "this is key" to ar-key.
       read armaster1 invalid key
            perform key-not-in-file.
or
       move data-name-containing-key to ar-name.
       read armaster1 key equal to ar-name invalid key
            perform ...

Lots more permutations and combinations, but that
is the basic idea.  For anything but the primary key,
use the key name and a condition.
```
