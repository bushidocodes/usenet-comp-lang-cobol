BTrieve ain't Cobol.  Some Cobol implementations can use
BTrive as the file handler when using normal Cobol verbs.
In these cases the BTrieve files behave as other Cobol
files and the run time adds the appropriate START where
required (as long as appropriate compile options are used).

It may be possible to make the compiler work in a non-standard,
non-Cobol way, but that does justify claims about what
<<Cobol>> does.

> 
> You have to set the key also if it is not the primary-key.

Of course, but that just evades to issue.  It is always
valid to read through to end of file after doing an
OPEN without an explicit START when using Cobol.
