possible logic error -- referring to it by mistake.

>Then it is easier to see what is being done when the values of the

>2002 to set condition-names to false using the "WHEN SET TO FALSE"
>phrase of the VALUE clause and the format 4 SET statement.

But you cannot refer to the 88 when specifying an initial value,
whether true or false. You can initialize it dynamically with:

05   pic x.
  88  leave-birthday value 'B' false 'N'.

set leave-birthday to false.

But you cannot initialize it statically with:

05  pic x value false.
  88  leave-birthday value 'B' false 'N'.

Why not? Why does the Standard make you write 'N' twice for static
initialization, but lets you use the 88 for dynamic initialization?

To cope with this syntactic error, I recommend that the false value
always be low-value and that indicators be initialized with VALUE
LOW-VALUES on a group name or procedurally with MOVE LOW-VALUES TO
group-name. If you prefer SPACES, that doesn't give me heartburn.

Regarding the speculation about the use of English, as  close as I was 
to the project, I can say that a major part of the objective was to
remove the ability of a programmer to respond to a request to see the
code, with a copy of the code or something near that level of the other 
non-documentation driven progamming languages.

Management, and Grace pointed out that the program documentation could
be substantially iimproved if the code were readable to almost anyone.

To that objective, the trend seems to be going the wrong way. COBOL is 
beginning to look like code of any programming language driven by the 
desire to more closely relate to the other languages.

If the boss asks to see the program today, he/she may have to ask for 
help in following some of the new fangled features that apparently were 
not introduced with the same objective, but to an objective of more 
closely resembling other programming language. Or to put it another
way, people who define COBOL extensions are not inspired to follow the
original objectives. Why, because, they want to only think about those
objectives uppermost in the minds if the current participants.

Warren Simmons
