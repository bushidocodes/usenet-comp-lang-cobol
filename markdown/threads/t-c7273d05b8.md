Not sure, but without searching I'm inclined to think SCREEN is cast in 
stone as a RESERVED word. There may possibly be directives setting your 
compiler to older COBOL versions which exclude some currently RESERVED 
words, but then you might have problems where they exclude RESERVED 
words that you want !

Bear in mind that SCREEN or any other RESERVED word is no longer 
reserved if you tag something on the front or end, e.g. :-

ws-SCREEN or SCREEN-1

Your problem you've got some 250 programs to check out, but it's 
probably only a percentage of those that actually contain the word SCREEN ?

As a thought, try this using the N/E IDE :-

Main Menu Bar ----> Search---->Find in Files - then enter the word 
Screen or SCREEN

The result will bring up an output list with EVERY program in your 
current application containing the word SCREEN. Make a note of the 
program names.

Open each relevant program as your source. Back to the Search - from the 
Menu Bar---->Search-----Find Text (i.e. in the current source)

Brings up the Find/Replace box - you can now do a global replace :-

Make 'Screen' into 'A-Screen'.

It might appear half-baked, but that's the way I would go at it. And 
REBUILD ALL is going to highlight any you miss.

Your variables ending with an hyphen - I very much doubt that. Check the 
on-line or the N/E on-line Language Reference Manual. Again if that's a 
big problem, try repeating what I've written above about 'correcting' 
SCREEN.

Jimmy, Calgary AB

> SCREEN as not being a reserved word.

NOMF

See the MF Compiler directive and there should be a huge table
somewhere in the docs (language ref appendix probably) that lists all
reserved words and which directives change them (if any).

REMOVE"SCREEN"

will also remove just the one reserved word.

> Also allow it to accept variables ending with a hyphen

See Language ref, chapter: Concepts of the Cobol Language, section:
User-defined words:

"except that '-' can not appear as first or last character".

This is part of the Cobol standard.  The solution is probably to
process the source code and change these.

Net Express compiler has a compiler directive called
REMOVE that you can use to isolate a reserveWord
into a user-defined words under windows 2000 pro,
you should try it under unix and see if you can use it
with your OS environment, you can code it as follows
on the FIRST line of your source code:

$set REMOVE "reserve-word"
