I was going to try this in GnuCOBOL 3.0 pre-rel.

Ignoring all the whoswhats about Report Writer in this thread;

Red, are you sure this is valid code to start with? TERMINTATE is a typo. The WS-CD-... fields are missing along with the NO-MORE-RECORDS conditional. Extra periods in the RD, missing most of the procedure division paragraphs, and that is just at the quick glance.

If it was just a copy-n-paste issue, then maybe a repost would help. Otherwise I'd have to say this is not really a Minimally Complete Verifiable Example.

And back to testing things like this. GnuCOBOL has near complete support for Report Writer, so future proofing is now fairly secure, in that GnuCOBOL is released as source so access won't go away until the GNU project and the internet goes away.

In the meanwhile, if moving the code to a free software platform is off the table, you can at least exercise the source to see if it'll be worth trying to continue by using the Hercules emulator and Tk4-.

MVS 3.8J will let you hone the JCL, The ANS COBOL (circa 1972) included in the Tk4- bundle supports Report Writer of yore. And Report Writer of yore hasn't really changed since the definition in the 1960s. If that code is going to work in Report Writer (and it won't as posted), the old ANS COBOL in Tk4- will at least give you some output to look at, to see if it's worth pursuing on z/OS.

As is, that code needs work.

Jay Moseley wrote up a wonderful tutorial on Report Writer with Hercules using this 1970s tech. Fortunately that tech still holds on modern equipment and operating systems thanks to some (very rare in the industry) long term foresight by IBM.

http://www.jaymoseley.com/hercules/compiling/cobolrw.htm

WARNINGS and CAUTIONS and PLAY NICE sentiments...

Thinking about running Hercules for anything beyond personal use at home for training will likely get you nothing but very deep legal trouble, so... go with GnuCOBOL which is free software and in active development to run the reports, or get a proper license for the RW module for use with your COBOL under z/OS, once it seems worthy of continuing.

And by the by for anyone reading along; There is KICKS for CMS and TSO and the author, Mike Noel, has tested it under MVS 3.8J, so you can even practice all the ins and outs of COBOL with CICS (free emulator - not the real thing, but close) on a Hercules node. http://www.kicksfortso.com/

Cheers,

a subset compiler excludes one or more of the modules. IBM System/360

"115) Report writer. Previously, the report writer was a separate

facility is integrated into the specification and it shall be
implemented by a conforming implementation."
Thus no longer a subset.

Thus back to being a subset.
