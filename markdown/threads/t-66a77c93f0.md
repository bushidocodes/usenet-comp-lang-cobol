- **Message-ID:** `<3mts2e$1llc@news-s01.ny.us.ibm.net>`

```
In an earlier post I said that I was trying to get Realia COBOL 4.2 to
talk to Microsoft C 7.0. Well after my post I received a response from
a very helpful Realia product manager. I now can do most of what I
want, but there are still a few snags.

It turns out that Microsoft (in it's infinite wisdom) does not support
OS/2 DLLs in MSC7 (even though they pushed to make the OS/2 DLLs the
'STANDARD' when they were co-developing OS/2). Watcom 'C' version 10.0
does support DLLs but for some reason they don't work 100% with Realia.

Realia uses OS/2 DLLs for BOTH DOS and OS/2 applications. They seem to
be able to do this by putting a DOS STUB in front of an OS/2 EXE so
that it can load and get access to the DLLs.

Now, to the problem. I have my Realia program talking to my Watcom DLL
under OS/2, however, it will not load under DOS! SUPPOSEDLY, this
works with MSC6 DLLs. So if Watcom and MSC6 generate the SAME DLLs,
why doesn't it work with Watcom????

Has anyone got this to work in both DOS and OS/2?
```
