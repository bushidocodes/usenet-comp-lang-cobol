My first comment is that many of the things in the proposed standard were
needed last year. I thus am unhappy to hear from several sources
including a posting by Don Nelson that it will be at least 1998 before
the new standard is finalized. What can be done to speed up the process?
What can be done to have additions to the standard come out in more
manageable chunks without taking the amount of time on the part of the
standards committee that the 89 addenda took? This can include urging
our national standards bodies to modify the process or setting up of any
alternative standards body such as X-Open that might be able to move
faster because of different rules.

Could sample code be posted on some Web site so that we can see what the
standard means? I know that I am confused by the >>CONSTANT directive
and the way >>IF and >>EVALUATE directives work. IF I understand the use
of >>CONSTANT, I think that >>COMPILE-PARM would be clearer. The IF was
in capitals because I may have completely missed the point in my 3
readings of the definition.

I am unclear as to what the various character sets are. Is ISO 646
international-1 the same as 8859-1 (PC code page 850 - Latin 1 for all
Western European Languages)? What is ISO/IEC 14651 and to what PC or
mainframe code page does it relate (how many bits per character?)? Is
there a document for under 100 USD (dollars) for the various ISO 10646-1
character sets so that we can see how they relate to the standard? Would
the most recent Unicode book be helpful in this regard?

What make an exception a fatal condition? Other than the case where data
in areas other that the target of the operation (such as the result field
for a COMPUTE statement) are corrurpted, why shouldn't there be a
continue option with or without default fix up?

While not a change from the previous standard, why does the standard
require that the implementor create a file if an OPEN OUTPUT does not
find the file connector. The IBM mainframe solution was to create a
temporary file that gets deleted at the end of run unit. While this may
seem bizarre and while it took me 20 minutes to find what exactly had
been done wrong when I knew this is what had happened, I suspect that the
other choices are as bad or worse. I can see a system flooded with
unidentifable files or clogged print spools.
