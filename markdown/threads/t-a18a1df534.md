Whoops ! Just read your second message. What are you saying ?

1. Run Convert3

2. Tell it the file format you will be using from RM - needs if you like, a
dummy program containing as minimum the RM copy files for File Control and File
Section, just like you'd give the layout to a DB package. It now generates an
N/E source which you compile.

3. Going from memory, your compiled program asks for (a) name of the RM INPUT
file and (b) name of the N/E OUTPUT file. Then converts unless it hits an error.

However if RM/COBOL 85 - game's over !

Dave,

From what you've written, it doesn't seem that this would broach any M/F rules,
so I can let you have a copy. However, one proviso. Convert3 was for the 'old'
RM file formats - Version 2.? ANSI 74. It wont do it for RM/COBOL 85 - they are
a different format. (I don't know what old M/F DOS tools you have, but Convert3
along with other COBOLs, (DEC, IBM etc.), was part of the basic DOS package V
3.0 onwards).

If you are talking RM/COBOL '85 - seems the only way would be to generate Ascii
Delimiteds(CSV) from RM and feed them into N/E as line sequentials.

Let me know if you still want it.
