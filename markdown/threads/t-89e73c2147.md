```
Report on October 9-13, 1995 ANSI X3J4 COBOL Committee meeting,
Lowell, Massachusetts, USA. When a vote is shown, the numbers are
for, against, and abstain (f-a-a).

There was no primary topic at this meeting. We had a three-day
editorial ad-hoc meeting before to deal with the public review
comments. All of them were dealt with at that meeting and in a
object-oriented ad-hoc that was parallel to the regular meeting. The
biggest change was to increase the maximum size of a numeric item
from 18 to 31 digits.

The major topics/proposals we discussed were:

1. The committee is going to look into using e-mail for distribution
of documents. How this would work has yet to be decided.

2. A proposal to require headers and end markers on a line by
themselves was rejected (in the proposed standard there is no such
requirement, either in fixed-form or free-form format).

3. A proposal to define i-o for multi-octet characters (MOCS) was
discussed at length and sent back for rewrite. It adds such things
as a SELECT WHEN to figure out what record description applies on a
READ, a POSITION clause for reports and screens that states in points
where to position some text, a CONVERT clause to state that data is
to be converted on i-o, a WRITE FROM literal, and so on. Note that
this proposal works for non-MOCS as well, as in converting EBCDIC to
ASCII with embedded packed-decimal and binary fields.

4. There are many issues on COBOL and internationalization that have
to be resolved in our standard. We discussed a liaison report with
SC22/WG20 (the internationalization group) for some time. Some
features they want do not seem to be possible. Anyhow, we will
continue working on this.

5. Pointers were discussed at length. We ended up deciding that the
data pointer would be somewhat like some current implementations now
have (IBM, Tandem, MF) with some restrictions. Very little was done
with procedure pointers. A proposal to add data pointers should be
available for the next meeting. A proposal for data pointers should
be available then or at the February, 1996 meeting.

6. The OO task group met for three days. Most of the changes they
recommended were in syntax and such. No big changes in direction or
capability were made.

7. My proposal on recursive programs was discussed and sent back to
me for rewrite. It was suggested that a
category be added that would persist on all calls from the first
recursion of a program (it would be in initial state) until somehow
the iteration ended. This was rejected.

8. A proposal on adding an implementor-defined directive (>>IMP) was
passed and will be added to the base document.

9. A proposal to expand the size of a numeric item from 18 to 31
digits was passed and will be added to the base document.

10. A proposal to add EXIT PERFORM [CYCLE], EXIT PARAGRAPH, and EXIT
SECTION passed and will be added to the base document. Many
implementations already have these as extensions.

11. A proposal to make NEXT SENTENCE archaic was passed and will be
added to the base document.

The next X3J4 meeting will be in Hershey, Pennsylvania, USA, on
December 4-8 1995, with an editing ad-hoc meeting on December 3. The
main topic will be critical proposals. There will be an OO ad-hoc
meeting November 16 and 17 in the San Francisco Bay Area.

Don Nelson
```
