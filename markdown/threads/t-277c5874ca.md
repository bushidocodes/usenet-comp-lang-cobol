```
Hello, I have a problem with one of my programs.  What I am trying to do 
is sort a file using a sort file giving my ouput file.  This is simple 
enough but when I so to process the output file to a Print file, i.e 
Headings, total records etc, I have problems.  

I have Microfocus for Unix and Dos, the Dos one processed the output file 
properly and will read my output file into the print file that I have set 
up.  When I use the Unix version it just believes the output file is 
empty and therefore It will not process any of the records and so I have 
no print file. 

Here is the code I am using:

      SORT SORT-FILE
            ON ASCENDING KEY SORT-STOCK-NUM
            USING ORDER-FILE
            GIVING SORTED-FILE.
       OPEN INPUT SORTED-FILE.
       READ SORTED-FILE INTO WS-ORDER-REC
            AT END MOVE "T" TO EOF-FLAG.

The unix version of Microfocus always processes the SORTED-FILE as being 
empty.  This means my it will not read naything into the working storage 
i.e WS-ORDER-REC.

If anyone could help, I would be so grateful.

Thanx in advance.
```

```
It may be easyer to use the SORT-FILE in your case :
A sorted file does not need to be opened. You get the records by the RETURN
command.

EG :

      SORT SORT-FILE
            ON ASCENDING KEY SORT-STOCK-NUM
            USING ORDER-FILE
            OUTPUT PROCEDURE IS MOVE-RECS.


MOVE-RECS.
       RETURN SORT-FILE
            AT END MOVE "T" TO EOF-FLAG.
       PERFORM UNTIL EOF-FLAG = "T"
             MOVE SORT-FILE-REC TO  WS-ORDER-REC
             RETURN SORT-FILE
                     AT END MOVE "T" TO EOF-FLAG
       END-PERFORM.
```
