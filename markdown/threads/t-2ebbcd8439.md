We are currently looking for new Third Party products to replace our
existing G/L, A/P, Inventory, Maintenance and Purchasing applications
that are currently on the mainframe. The replacement products could
either be mainframe or client server, providing it has a means of
interfacing Big Blue with little blue.

If the application is a mainframe package, then it would need to be
written in COBOL, be Y2K tested. the Database should be either VSAM
or SQL/DS. We currently operate a couple VSE/ESA 1.3.5 guest
machines under VM/ESA 1.2, and are in the initial stages of migrating
to VSE/ESA 2.2 and VM/ESA 2.2. Sounds like fun!

Client Server products should function under a NT 4.0 environment,
with SQL server supplying the database. Other servers could be looked
at, but we prefer the SQL server environment.

Do any of you have any suggestions? We would like to look at as many
alternatives as possible. Are there any products that any of you
have encountered that you suggest that we look at?

I appoligize for posting on the VSE-l, but who better to ask then the
"experts" on the VSE Operating systems.
