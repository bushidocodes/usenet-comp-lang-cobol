Dear Eestir,

Without more details on your situation I'm not sure, but we have a
product that may solve your problem. It is called Cobol-RPC and it is an
implementation of remote program calls for COBOL. With Cobol-RPC a COBOL
call can be made to a remote computer, the program is loaded and
executed on the remote machine.

I suspect that the slowness of the report is due to the fact that the
COBOL data has to be transferred over the WAN. With Cobol-RPC the report
program can be run on the server (with NT) or on a workstation connected
to the server (with Novell). Unfortunately Novell will not support the
execution of COBOL logic on the LAN server itself, but NT will. In
either case the WAN data bottleneck is removed.

In this scenario the report program could generate the report and return
it back to the client. The report will have to be transferred over the
WAN, but the report is almost certainly substantially smaller than the
data that is being processed.

The problem in your case is that you are running the program on a
client PC and the files reside on the file server. Each file access is
done via the WAN which apparently is too slow for this.

The solutions in your case are:

1. Copy first the files to local PC, then run the programs,
accessing these files completely localy. If the files are not very
large, this is a very good option.

2. Abandon the Netware server. Get a Unix server. It can run as a
file server like the Netware did, plus you may buy the acucobol for
the Unix. This will give you the option to run the programs "locally"
on the server but the output would be on your client. We use this
method, and is the speediest in multiuser Cobol environments. Your PC
may open a terminal window to Unix host, and run the program on the
host. Only screen output is transfered via the LAN/WAN and ofcourse
the printouts.

3. Transform the application to do some kind of replication to a
local file server. If you have your programming team, and they are
skillfull enough, they can do it. I did it with Unix and acucobol,
transfering data via e-mail like EDI to two servers 500Klm apart via a
64Kbps line. This takes time though (in programming) and needs also
programming the server and I don't think that this can be done on
Netware or even windows Nt. Just UNIX....

4. Go and buy an RDBMS with replication capabilities. Buy the
product from acucobol that ties the compiler to an RDBMS (oracle for
example). Buy the replication middleware, do some programming on your
source and you are ready (or so). Too expensive, and not sure results.

The most easy and fast solution is 1. Then go on 2.


Hope this helps.
