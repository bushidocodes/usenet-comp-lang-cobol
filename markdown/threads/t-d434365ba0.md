First of All I would suggest forming a study Group.  Good Ideas are best
shared.  Second go to class and take notes.  Third read the textbook!  4th
When coding anything for the first time dont try to write the program all
at once.  Just do the simplest most fundamental thing first, like opening
and closing the file.  Often it is the operation system specific tasks
like reading a file that is the hardest.  There are always things the
instructor leaves out like what type of file access needs to be used, or
if you have to use a copy statement, or they havent made the file yet!

Here is a tip for you when you describe the file use a file-status storage
item and check the file status when you open the file and if it does not
equal '00' display what it does say.  That way if you get a dump you can
read the variable with the file status.

Also use displays so you know if the file is open at the right point.  or
if you got to the end of the process.  If you think there is an endless
loop put a counter in the middle and tell it to stop when it gets to 50.
or when you get to page 50 of the printing.

COBOL is an easy language to learn.  All programming is done one step at a
time with loops and counters.

Charles Hammond, BSIM Undergraduate Program
