Thanks for the advice Rik,

It has been brought to my attention by other means too that COM object
handling could be the solution. The anomaly was minor however. When
executing a SaveBookAs command (or invoking the method if you prefer) there
is an option to suppress the Excel dialog (name the file, are you sure and
so on), if omitted Excel should be able to open up a conversation with the
user.

In Windows (all supported versions) omission of the parameter is no problem,
the Excel dialogue comes up perfectly. In the Citrix environment however,
the dialogue seems to be started BUT does not become visible. And thus it
looks like the application freezes, which is�nt the case, it is waiting for
closing of the Excel dialogue.

I was able to fix it by adding a simple POW-FALSE after the book name
argument. Now the Excel dialogue is suppressed (not disabled however) and is
replaced by a layer FJ Cobol seems to provide (taking care of the
interaction (if needed only)).

After this minor change that particular anomaly was taken care of.

But I do guess I will encounter similar problems with other controls
(maybe).

Greetings, Danny.
