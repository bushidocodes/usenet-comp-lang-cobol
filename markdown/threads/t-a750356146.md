The answer ist very simple. The alternate-index is to define as an own
file with a key.

SELECT .... RECORD KEY....   the record key is the field(s) of the
alternate-key

It is used, when you have to insert many records, which have no unique
key (for example sales in a bank and then you want to find all sales of
an account).

record can be:

....VOLUME ... DATE ..  ACCOUNT

ACCOUNT is alternate-index and is not unique !
