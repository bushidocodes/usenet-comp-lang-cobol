```
II (primarily VSAM file structures.

Q:  The COPY command allows you to use the REPLACING option to replace
text in the Copy element; however, I have found that it does not work in
certain instances.

Example#1 - does not work:
COPY COPYBK1. (I wanted to use the original record layout as well as a
replaced version)
COPY COPYBK1 REPLACING 'PROD-REC' BY    'WS-REC1'.

Example#2 - DOES work:
COPY COPYBK2 REPLACING ==:TAG==     BY    ==REC1==.
COPY COPYBK2 REPLACING ==:TAG==     BY    ==REC2==.

Unfortunately, I need to use another department's copybook (COPYBK1)
which does not have any psuedo text-- any ideas?
```

```
Sorry about that - but with the current COBOL Standard (your IBM mainframe
compiler) it is impossible to do "partial word replacing" WITHOUT defining
your original copy member with something like
   (TAG-)
     or
   :TAG-:

It won't help you now, but the next Standard will allow you to replace a
prefix.
```

```
You can fully qualify references to the data elements:

FD  FILE-1
    DATA RECORD IS FILE-1-RECORD.
01  FILE-1-RECORD.
    COPY COPYBK.

FD  FILE-2
    DATA RECORD IS FILE-2-RECORD.
01  FILE-2-RECORD.
    COPY COPYBK.

PROCEDURE DIVISION.
    MOVE  DATA-ITEM  OF FILE-1-RECORD
                     TO  DATA ITEM OF FILE-2-RECORD.
```
