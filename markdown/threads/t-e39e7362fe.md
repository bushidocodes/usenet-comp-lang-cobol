Two comments:
The only time a spin-loop on a client machine doesn't matter is if
you're running a single-tasking operating system (i.e.: DOS). If
you're running OS/2 or NT, or maybe even Win 95, it's needlessly
using cycles that could be put to productive use. Even under DOS,
there are some add-on multi-taskers that some people use.
As far as a call to sleep for a specified period of time,
that's already listed as an enhancement request (NWB24636). If
it's important to you (and you're on maintenance), you can
call in and ask to have your company added to this enhancement
request.

Ron

You can use the UNIX sleep() function. I don't know
offhand if this is callable from within a COBOl program
just like that. If not, you just write a tiny C function
that accepts an argument from a COBOL program, calls the
sleep() function, and returns. You compile this function
and recreate the run (rts32) with this C function. It's
not really very hard.
