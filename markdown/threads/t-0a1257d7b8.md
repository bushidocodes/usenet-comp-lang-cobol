```
Hello, everybody,

            Here's my problem: Assume application A DISPLAYing a SCREEN
(80 x 25) and starting thread T which regulary  updates the time (which
is shown in the right corner) and repaints the screen.
            Then, assume application A CALLing some other procedures,
which may display there own SCREENs. (Say, error messages or help
windows) onto the main-screen
             I need a technique for thread T to determine wheter the
main-screen (the 80x25) is the active one or not, since if the thread
would DISPLAY the time while a different SCREEN is active, the effects
don't look that good.
            I know that I could declare a global EXTERNAL variable,
which is set to false when another screen is painted, but I'd prefer to
know if the 80x25 screen is active!
            I use AcuCOBOL 4.0 and Windows NT 4.0.

                Thank you in advance,


Markus Triska.
```
