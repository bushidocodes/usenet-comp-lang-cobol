Could be a compile option or a runtime option, possibly. For the second
program, is there an OUTDD compile option in effect that uses a DDname other
than SYSOUT, perhaps for which there is no DD statement? Is the first program
passed a set of LE runtime parameters? Or do either of them have a CEEUOPT
module linked in? What is the MSGFILE ddname for runtime? Do you have a SYSOUT
DD statement supplied in your run step? In fact, are these programs batch or
CICS or IMS programs? All these factors could influence the behavior you're
seeing.

Just some things to check.
