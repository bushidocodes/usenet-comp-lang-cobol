Anyone know of a COBOL style guide on the 'net?  The type that says
stuff like "thou shalt not use the ALTER verb under pain of being
vaporized by thy fellow programmers", "paragraph names should begin
with three digit numbers, with leading zeros, and be in ascending
order", and "indent nested code by two spaces".  Basically how to
write readable COBOL code ... or at least one organization's vision
(or hallucination) of readable.  Something like "Indian Hill C Style
and Coding Standards"
(http://www.chris-lott.org/resources/cstyle/indhill-annot.html), but
for COBOL.

Also, has anyone developed a style formatting program for COBOL like
C's indent program?  (http://www.gnu.org/software/indent/indent.html)

That was the "Lister" option, and it didn't carry forward into newer IBM
compilers.  There are many ways outside of the compiler for doing this.
The IBM CCCA and COBOL/SF programs, for example, will clean up code as
they convert and restructure it.  In addition, the ISPF Editor now
offers the HILITE option for helping us effectively view code in many
languages.

These capabilities are part of  VisualAge COBOL, too.

I'm not up to date on other vendor's offerings, but I'd be surprised if
most of them didn't have something along these lines.
