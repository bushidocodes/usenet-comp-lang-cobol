Hello Shea,

You didn't quite explain it all - for example, you did not indicate whether
the error was a compiler message or a runtime message - however I believe I
know the answer. The compiler is say "X is not declared" - Am I correct?

You do not have to declare X because you are not using a VARIABLE named X.
The built-in subroutine you want to use is a CALL-BY-NUMBER routine; its name
is a HEXIDECIMAL constant, X"AF". Now I will not correct your call statement
for you because learning to use the compiler to correct syntax errors is a
skill you MUST MASTER. However, think about WHY the compiler "believes" you
are using a variable named X, instead of the constant X"AF" and the answer
should come to you. If not, talk to your instructor - or reconsider your
choice of the wacky, wonderful, maddening, tediously boring world of computer
programming.
