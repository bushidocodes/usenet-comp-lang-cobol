Dear Andy:
I took several courses at Hayward State University in Computer Science,
and the language of choice then in education was Pascal. I remember
data structures class using Pascal, and some other ones two.

I took my first COBOL class under the Accounting department. I saw
a book in the Cupertino Library here in California with a title
such as Cobol for Pascal Programmers. The following is a thumbnail
sketch of Pascal versus COBOL.

These are my own opinion and many language purists may find fault,
but you might find the following useful...

1. Pascal has Global and local declared variables. COBOL tends to
have global variable in working-storage and on the input data records
coming from the input file, or variable about to written out to a file.

Thus:
WORKING-STORAGE SECTION.

* COBOL COMMENT BEGINS IN COLUMN 7 WITH ASTERISK
* HERE IS AN INTEGER:
01 MY-COUNTER PIC S9(04) COMP-5 VALUE 0.


Versus Pascal:
var myvar as integer;
(Whoops, it's been 10 years, is the above Pascal correct???)

The computation usage (shortened to COMP or COMP-5 for intel)
can be integers or floating point as in COMP-4 on ibm mainframes.
comp tends to be binary twos-complement integers. comp-5 is
intel swapped-byte integers...

COMP-3 is PACKED decimal, a left-over of IBM Mainframes.

Characters in Pascal are Pic X fields in COBOL. And strings
are also Pic x. For example:

01 mycobolchar pic x(01).


move 'A' to mycobolchar.
display mycobolchar.

In pascal I guess this might be (hedging)

var mypascalstring as string;

mypascalstring = "A";

The arrays of pascal become tables in COBOL.

For example, and array of char:

01 the-table.
05 the-char occurs 11 times pic x(01).

01 i pic 9(02) value zeroes.

Load the array with spaces:

move spaces to the-table. (treating it as a group level)

or in a loop to the array element:


perform 11 times
add 1 to i
move spaces to the-char (i)
end-perform.


The above loop is a new feature of cobol 85. old cobol would do
it by performing a seperate paragraph:


perform b000-table-process varying i from 1 by 1
until i greater 11.

b000-table-process.
move spaces to the-char (i).


Even older cobol would be done by ex-360 assembler programmers
and use the deadly 'goto' branch method of programmin'
start of program...

move zeroes to i
b000-loop-the-loop.
compute i = i + 1.
move spaces to the-char (i).
if i < 11
go to b000-loop-the-loop.
(falls thru to next statement )

One weird thing about COBOL which you get used to, is the sense
of the perform until loop structure. It is the opposite sense
to the Do while loop. You perform UNTIL something is true.
Contrast this to The Do WHILE something is true. For example:

perform until i > 11


versus

Do while i <= 11.

Writeln in Pascal in roughly equivalent to Write in COBOL, as are
READLN VS. READ.

Passing variables to subprograms is mostly pass by reference
in COBOL, unlike
C which has pass by value as the default. The subprogram can change
the passed variables values since it is actually using the address
of those vars to change them.

here is COBOL program calling another:
main program:

CALL 'MYSUB' using var-1.


hERE IS part of THE subprogram:

program-id mysub.
linkage section.
01 var-1 pic x(300).


...

exit program.
