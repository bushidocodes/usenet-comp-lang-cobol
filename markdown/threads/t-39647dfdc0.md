My editor of choice on the PC is UTRAEDIT, whether I map a net drive
or use an emulator's file tranfer to access a  mini's files.  I've
been using it for years, since version 2.0   Its up around version 8.0
now.  Do a search for it on the web and you'll easily find it.   You
can install it for 45 days trial then it stops working.  For $35 you
can buy it.   If you spend a lot of time editing text files, this
program, or one similar to it, is an absolute must have.   

If you have ever used the old Brief editor on a PC, before GUI, you
can think of Ultraedit as a Window's GUI style Brief editor, only
better, because its GUI.  Its a real programmer's text editor and it's
my most important tool.  The MDI user interface is very well designed.
The configuration is complete.  It's functionallity is rich and can be
added or removed from the tool bar to suit your needs.    The MDI
interface is also flexible.  you can use file tabs or file trees to
view loaded files.  You can load two programs and display them side by
side and make them both scroll at the same time.   I've loaded dozens
of files and done mass edits on them or mass searches with results
loaded into a new buffer.  You can load a big COBOL program and 
have it find lines containing a string which it inserts into a mini
window.  Just click on a line in the mini window and the editor jumps
to that line in the currently displayed buffer.  

I was asked to insert a line of code after a specifc line in roughly
250 JCL files.   I was expected to load, edit and save each one.  The
boss assigned groups of files to 4 different programmers so the job
would be finished by the end of the day.  Why bother.  FTPed them onto
the PC.  Loaded them all into the editor and told it not make .bak
files.  Edited the first file using Home/Find/ArrowDown/Paste with
macro recorder turned on.   Saved macro then clicked through all 249
files hitting Run macro key.  Did a Save All then FTPed them back.
Job was done in 30 minutes.  One person did it.  Boss's jaw hit the
floor when he heard.

The most usefull feature is Column mode cutting and pasting.  This
feature has saved me 100s of hours of work - I kid you not.   The
editor doubles as data file editor, so it's usefull with say a  200MB
data file that needs byte number 138 removed from every line.  No
problem, 5 minute job.

I went out their web site and found a utility that makes Windows think
that Ultraedit is Notepad.  Whenever I click on a text file, Ultraedit
loads.  If its already loaded, it inserts the file into a new MDI
window.  The editor can also be told to run seperate instances of
itself.  

Have been doing some HTML work lately.  The Editor is smart and can
display all HTML tags in a utility list window.   It will also fetch
the Browser and pass the file to it.  

It has syntax highlighting for various languages, a crude spell,  two
up page printitng for long programs,  color configuration, margin
markers, byte rulers, logical line number display,  compare, sorting
functions, unix conversion, etc.   Can't say enough about it.  

For $35 its one of the most usefull programming tools I have ever
used.
