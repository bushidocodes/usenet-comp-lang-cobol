An OCCURS clause is invalid on an 01 or 77 level.

      01  Table-State-Table.
        03  TABLE-STATE-RECORD        OCCURS 50 TIMES
                                      INDEXED BY STATE-INDEX.
         05  TABLE-STATE-CODE         PIC XX.
         05  TABLE-STATE-NAME         PIC X(20).

Some compilers allow OCCURS on an 01 (but not 77) as an extention, but
this can lead to anomolies.  For example if this was an FD what would
the 'record size' for the file be ? 22 ?

There may also be issues with 'slack bytes'.  A compiler may start
each 01 and 77 level on, say, an 8 byte boundry.  How much space would
your table take ?  1100 bytes ?  1200 bytes ?  1198 bytes ?

What if you had had (in the original) 

     01  Table-State-REDF REDEFINES Table-State-Record.

Are you redefining just one 01 level or the whole 1100/1200/1198 bytes
?

A slight, and admittedly obscure, exception:  Unisys MCP/AS COBOL(68)
allowed OCCURS on 01-level items.  What it did on its target system was
produce a multidimensional table in which the first subscript addressed an
array of descriptors, each of which described memory.  Thus, only the table
elements associated with first subscripts that were actually used in the
program had memory allocated for them, and only when an element of the table
with that first subscript was "touched".   The mechanism thus made use of
the hardware's implementation of "true" two-dimensional arrays:  an array of
descriptors for the first dimension and memory spaces for the second (and
subsequent).

This feature was troublesome both in implementation and in execution, and
had some insoluble semantic anomalies when the user neglected to code a
second dimension.  The feature was not carried forward as a feature into
MCP/AS COBOL74 (or COBOL85, for that matter), and users were urged to use
"soft" multidimensional arrays -- i.e., "normal" COBOL-style ones -- rather
than attempt to use the array-of-descriptors approach.  Unisys MCP/AS
COBOL(68) is no longer officially supported but still has a few users by
special arrangement.

    -Chuck Stevens

containing an occurs clause.  Redefining records (01 levels) should
always be valid.
