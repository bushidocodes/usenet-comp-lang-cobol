Here's a little help:

Define an SD sort-record.
Use the Sort verb. Use an ascending key sort-year and name. Use an
input and output procedure.
In the input procedure translate the st-year to a numeric
sort-year: If st-year = 'FR' move 1 to sort-year an so on.  
In the output procedure translate sort-year back to the original
values for printing.

Why not add an additional field to the sort record (in the SD) to sort on,
and in the input procedure, plug a numeric value for the appropriate
2-character ST-YEAR.

SD record layout ...

01    STUDENT-SORT-RECORD.
        05    SSR-NAME              PIC X(15).
        05    SSR-MAJOR            PIC X(15).
        05    SSR-YEAR               PIC XX.
        05    SSR-YEAR-NUM     PIC 9

INPUT PROCEDURE CODE:

STUDENT-RECORD VALUES HAVE BEEN MOVED TO STUDENT-SORT-RECORD ...

EVALUATE SSR-YEAR
    WHEN 'FR'
        MOVE 1          TO SSR-YEAR-NUM
    WHEN 'JR'
        MOVE 2          TO SSR-YEAR-NUM
    WHEN 'SO'
        MOVE 3          TO SSR-YEAR-NUM
    WHEN 'SR'
        MOVE 4          TO SSR-YEAR-NUM
    WHEN 'GR'
        MOVE 5          TO SSR-YEAR-NUM
    WHEN OTHER
        MOVE 9          TO SSR-YEAR-NUM
END-EVALUATE

RELEASE ....
