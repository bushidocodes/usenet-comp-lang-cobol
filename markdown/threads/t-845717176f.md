```
One of the reasons people move to a relational database ( RDB Migration) is
so that the data resource held in the ISAM files is available to software
other than just COBOL.

It is very advantageous to be able to access your data with standard
software rather than having to write a COBOL program every time.

Take reporting, for example...

There are some quite amazing reporting packages available (Crystal Reports,
Stimulsoft, etc.) that provide a simple WYSIWYG interface for report
development, and the days of counting COBOL printline fillers between
variables are long gone...

However, none of the reporting packages I know of can support ISAM as an
input.

Once you have migrated to RDB, this is no longer a problem as all of the
reporting packages support RDB as a data source.

Migrating can be a lengthy process (it generally isn't if you use our
tools... :-)), but there may be strategic reasons you need to delay starting
it.

There is a learning curve involved in using the Reporting packages and it is
good if you can get some practise with them as soon as possible. (maybe
while you are waiting for the Migration to happen...)

One way to do this is to convert the existing ISAM to XML (which is an
acceptable input format for Stimulsoft, at least...)

PRIMA has developed software to read Fujitsu ISAM and produce XML, very
quickly, so it can be done at report run time. The XML is "transparent" as
it is only ever used by the reporting package, but having this facilty means
you can start developing reports with a reporting package before you have
completed Migration.

Having the XML produced at runtime in the background means that it reflects
the latest data on the ISAM source. It also gives you a chance to work on
your reporting development without dependency on Migration.

(Once Migration is accomplished you simply change the data source in your
developed reports, without having to develop the report again...)

The solution we have uses both COBOL and C# components to read ISAM and
write XML. These components are callable from any language that implements
the COM interface, (including both Micro Focus and Fujitsu COBOLs.) We have
not tried it with Micro Focus ISAM but I'd be happy to make the source
available for compilation in the Micro Focus environment if anyone is
interested in getting XML out of ISAM.

If you need XML from your ISAM (for whatever reason), please contact me
privately. (there isn't necessarily a charge involved.)

Pete.
"I used to write COBOL...now I can do anything."
```
