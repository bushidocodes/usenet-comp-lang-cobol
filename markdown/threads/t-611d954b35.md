```
I did a little work on something to demo that a few years ago--this might be
enough to get you started.  You might need to change a thing or two in case
your compiler or run unit doesn't like zero subscripts.
       77  PNTR                   PIC S9(4) COMP-1.
       01  ACC-NUM                PIC 9(6)V99.
       01  ACC-AREA REDEFINES ACC-NUM.
           05  AA-THOU            PIC XXX.
           05  AA-HUNS            PIC XXX.
           05  AA-CENTS           PIC XX.
       01  SET-OF-THREE.
         05  HUNS                 PIC 9.
         05  TENS                 PIC 9.
         05  UNITS                PIC 9.
       01  TARGET-AREA            PIC X(80).
       01  FILLER REDEFINES TARGET-AREA.
         02  TA-COL OCCURS 80     PIC X.
       01  TABLES.
         05  UNIT-TAB.
           10  FILLER             PIC X(6) VALUE "ONE~".
<snip for brevity>
           10  FILLER             PIC X(6) VALUE "NINE~".
         05  FILLER REDEFINES UNIT-TAB.
           10  DIGIT OCCURS 9     PIC X(6).
         05  TENS-TAB.
           10  FILLER             PIC X(8) VALUE "UNREF".
           10  FILLER             PIC X(8) VALUE "TWENTY~".
<ibid>
           10  FILLER             PIC X(8) VALUE "NINETY~".
         05  FILLER REDEFINES TENS-TAB.
           10  A-TEN OCCURS 9     PIC X(8).
         05  FILLER               PIC X(10) VALUE "TEN~".
         05  TEEN-TAB.
           10  FILLER             PIC X(10) VALUE "ELEVEN~".
<ibid>
           10  FILLER             PIC X(10) VALUE "NINETEEN~".
         05  FILLER REDEFINES TEEN-TAB.
           10  A-TEEN OCCURS 9    PIC X(10).
-----------------------
           MOVE SPACE TO ACC-AREA.
           ACCEPT ACC-NUM LINE 3 POSITION 1 SIZE 8 CONVERT.
           IF ACC-NUM NOT NUMERIC
             GO TO EXIT-IT.
           IF ACC-NUM = 0
             GO TO EXIT-IT.
           MOVE 6 TO PNTR.
           MOVE "******" TO TARGET-AREA.
           IF ZERO = AA-THOU AND AA-HUNS
             MOVE "*****ZERO" TO TARGET-AREA
             MOVE 11 TO PNTR
             GO TO DISP-CENTS.
           IF AA-THOU > ZERO
             MOVE AA-THOU TO SET-OF-THREE
             PERFORM SPELL THRU SPELL-EXIT
             STRING " THOUSAND" DELIMITED BY SIZE
                   INTO TARGET-AREA POINTER PNTR
             IF AA-HUNS > ZERO
               MOVE "," TO TA-COL (PNTR)
               ADD 2 TO PNTR
             ELSE
               ADD 1 TO PNTR.
           IF AA-HUNS > ZERO
             MOVE AA-HUNS TO SET-OF-THREE
             PERFORM SPELL THRU SPELL-EXIT
             ADD 1 TO PNTR.
         DISP-CENTS.
           IF AA-CENTS = "00"
             MOVE "NO" TO AA-CENTS.
           STRING "& " AA-CENTS "/100*********" DELIMITED BY SIZE
                 INTO TARGET-AREA POINTER PNTR.
           DISPLAY TARGET-AREA LINE 5 POSITION 1 SIZE 79.
----------
         SPELL.
           IF HUNS > ZERO
             STRING DIGIT (HUNS) " HUNDRED " DELIMITED BY "~"
                   INTO TARGET-AREA POINTER PNTR.
           IF TENS = 1
             STRING A-TEEN (UNITS) DELIMITED BY "~"
                   INTO TARGET-AREA POINTER PNTR
             GO TO SPELL-EXIT.
           IF TENS > 1
             STRING A-TEN (TENS) DELIMITED BY "~"
                   INTO TARGET-AREA POINTER PNTR
             IF UNITS > ZERO
               MOVE "-" TO TA-COL (PNTR)
               ADD 1 TO PNTR.
           IF UNITS > ZERO
             STRING DIGIT (UNITS) DELIMITED BY "~" INTO TARGET-AREA
                   POINTER PNTR.
         SPELL-EXIT.
           EXIT.
```

```
Peter Gerard wrote:
>Is there any sites out there that would have this routine coded and
>available for download ?  I am not just looking for the $100.00 dollar
>example, but the code to turn any numeric value into a written out
>dollar value.....for printing on cheques.

Download MONEY.ZIP from my web page below.  It would convert, for example,
123.45 into:

       **ONE HUNDRED TWENTY THREE DOLLARS AND FORTY FIVE CENTS**

or 123,456,789,012,345.67 into:

 **ONE HUNDRED TWENTY THREE TRILLION FOUR HUNDRED FIFTY SIX BILLION**
  **SEVEN HUNDRED EIGHTY NINE MILLION TWELVE THOUSAND THREE HUNDRED**
             **FORTY FIVE DOLLARS AND SIXTY SEVEN CENTS**

The routine places the text lines centered into a multi-line buffer.
you can easily change the number and size of the lines to fit your
checks.  The asterisks are for check protection, of course.  Changing
the names of three digit groups (e.g. trillion, billion) for U.K.
requirements is trivial.
```
