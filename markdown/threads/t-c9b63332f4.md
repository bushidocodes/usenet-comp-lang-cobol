```
Hi.

Does anyone know the performance difference between INDEX BUILD (default)
and INDEX UPDATE, on BMC LOADPLUS, where a process loads 1 day's
data into a DB2 table of 6 months data (adding 0.6% to the volume)?

I've read the diagnostics for the LOAD phase, and it looks like the load of
the
rows constitutes less than 5%of the phase elapse time, with the rest taken
up by
the index unload/concatenation of 1 days new index entries/sort/reload.

Bearing in mind this is a 1.1 billion row table of 16 paritions, you can
start to see
the financial implications. I saw this on Thrusday, and knowing there is no
serial
processing on the table, just 2 transactions doing random access, I think I
may
be able to use INDEX UPDATE, with a weekly index rebuild and a monthly
REORG.

I also anticipate savings in the preload, if I can get LOADPLUS to stop
SORTing.
I do a SORT/split before I start the load job, creating files for each
partition load.

THX.
```
