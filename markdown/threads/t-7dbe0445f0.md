Your example leaves me a little confused.

The
        ASSIGN TO a:\dprog1.dat
   looks like you want to establish the "physical file name" within your
source program and are using PC type names. While the
        ASSIGN TO UT-S-SYSOUT
looks like you want to use a SET statement to establish the physical name
but are using (IBM) mainframe naming standards.

The 1st one is "wrong" because it isn't in quotes and ":" and "\" are not
valid in the middle of COBOL words.

The 2nd one isn't completely wrong but SYSOUT is a *very dangerous* name to
use for your logical file name.  In many (IBM-like) environments, this is
reserved for DISPLAY statements and not WRITE statements.

Finally, (and this is probably your biggest problem), your SELECT statements
use the names "INVENTORY" and  "PRINTED-REPORT" while your FDs use the names
"STUDENT-FILE" and "PRINT-FILE".  You need to change this so that the same
name is used in the SELECT statement and the FD for each file.

After using all the help in other posts, you may very well end up with something like:
SELECT ... ASSIGN TO ...SYSIN... and
SELECT ... ASSIGN TO ...SYSOUT... .
The SET statements will then still cause you errors. You should use:
set sysin=a:\dprog1.dat
set sysout=a:\dprog1.rpt
That is: no spaces around the =. Use SET VAR=VALUE and not SET VAR = VALUE.

Changing the files is then simply changing the SET statements without recompiling the
