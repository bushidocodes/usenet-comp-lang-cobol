When you say "visual cobol" - do you happen to know the vendor?  The only
vendor that "officially" has a product by that name is Mbp - but that is
"medium uncommon" (but as your ID shows you are in Europe, it is more common
there).  Micro Focus had a product called "Visual Object COBOL" (VisOC for
short).  There are a NUMBER of other compilers that have "visual programming
environments" - so it could be something totally different.

All of this relates to the fact that many (most? all?) COBOL ISAM file
systems are proprietary (some belonging to the compiler vendor - some to
other vendors) - so we would need to know which you mean to give you "useful"
information.
