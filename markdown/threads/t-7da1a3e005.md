```
Microfocus 2.2 / Unix / Oracle 9i  

When I'm trying to update a DATE column in procobol MicroFocus2.2 to
null, it stops processing. Anyone know how to make this work?

The column may or may not contain a value. If it does not have a value
I'd like it to contain a NULL.

Prior to the update, I'm either moving spaces to a pic x(9) column, or
a date into it.
I1-DATE-VALUE will contain either spaces (which is where the problem
is)
or will contain something like:  '13-JUL-02'  

I've tried updating code, to no avail, with :      
 col =   NVL(:I1-DATE-VALUE,NULL),
 col =   NVL(:I1-DATE-VALUE,''),
 col =   NVL(:I1-DATE-VALUE,''''),
 col =       :I1-DATE-VALUE,
 col =   NVL(:I1-DATE-VALUE,""),

and moving a null indicator to the column I1-DATE-VALUE and setting db
column to that.

It worked fine before in the 8.1.7 version of true64 cobol. 


-----------------------------------------------
CODE:
    05  I1-DATE-VALUE             PIC X(09)   VALUE SPACE.

     EXEC SQL
        UPDATE LE_STR_INSTRUCTOR
           SET LAST_CHANGED_DATE        =  SYSDATE,
               HUNT_CERTIFIED_DATE      =  NVL(:I1-DATE-VALUE,''),
       WHERE   INSTR_NO            =  :INPUT-INSTR-NO
    END-EXEC. 

-----------------------------------------------
ERROR WHEN RUNNING:

(Looks to be inserting a value with spaces, and it doesn't think it's
a valid date)

**************  SQLCA VALUES AT ABNORMAL TERMINATION **********
SQLERRMC..ORA-01847: day of month must be between 1 and last day of
month

------------------------------------------------
Pre-Compiled with this:

 procob iname=n475.pco format=terminal  MAXLITERAL=32 \ 
        mode=oracle  DBMS=V8 UNSAFE_NULL=YES  comp5=yes

-------------------------------------------------
Compiled with this (in the 'make' file):

COB=cob -vx -C sourceformat"'free'" -C 'makesyn "COMP-5" == "COMP"'
PCCPLSFLAGS=format=terminal dbms=v8 unsafe_null=yes maxliteral=32
```
