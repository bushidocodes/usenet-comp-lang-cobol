In order to read/write to an ISAM (Indexed Sequential Access Method)
file you need to open it as I/O and have the select/assign setup
correctly:

i.e.

Environment division.

input-output section.
file-control.
select my-isam-file
assign to "whatever"
key is my-isam-key
organization is indexed
access mode is dynamic
file status is ws-file-status.

Note that the access mode is dynamic. This lets you go both ways,
sequential and random. Sequential is the default is access mode
is not specified.

Note:
Read my-isam-file

is a random read based on exact key match.

Read my-isam-file next record

gets me the next sequential record.

And
Start my-isam-file key not less my-isam-key

gives you a new file position greater than or equal to the key.


Your error is probably because you left out the
access mode is dynamic.

Also, when you First create a new isam file, a onetime program
should open it as output, and give it records presorted in ascending
order.

After that, you can open the Existing file by another program
to open as i/o:

Open i-o my-isam-file.

is a random read
