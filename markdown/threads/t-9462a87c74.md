You know, if you're a consultant or software supplier (and it sounds like you
are from the context of your message), it seems to me it would be a good idea
to be current with what's going on in the market. There are a lot of ways to
find out and places to keep up. It's always embarrassing when you're not sure
what you're talking about and the customer is paying such a nice price.

For the MVS, OS/390 world, the most recent COBOL compilers from IBM are:

VS COBOL II - all versions and releases support both the ANSI 68 and ANSI 74
standards, using appropriate compiler settings
 Version 3 and later support the ANSI 85 standard (but not the '89 amendment to
that standard, which was mainly intrinsic functions)

COBOL/370 - version 1 release 1 of a new compiler; this and all subsequent
releases require LE (Language Environment) to run; this is the first
IBM-supplied mainframe compiler to support intrinsic functions

COBOL for MVS & VM - version 1 release 2 of the new compiler; added a ton of
new features, many of them anticipated in the next ANSI standard;

COBOL for OS/390 & VM - version 2 release 1 of the new compiler; only some
minor enhancements, from an applications programmer perspective.

There is a PTF that can update both COBOL for MVS & VM and COBOL for OS/390 &
VM to support two new formats of ACCEPT and three new intrinsic functions, all
of which enhance Y2K support in the language.

Training is available, check the web site listed below.

There is no such thing as VS COBOL II R1.5.  The last version of VS COBOL II
was V1.4.

The order of IBM MVS compilers is:

ANS COBOL (V4 was the last version in the early '80s)
OS/VS COBOL (R2.4 was the last version dropped in the mid-90's)
VS COBOL II (r1.0, R1.1, R2.0, R3.0 R3.1, R3.2, R4.0 - support for R4.0 will
end in March 2001)
AD/Cycle COBOL/370 (1st compiler with Intrinsic Functions - support already
dropped)
IBM COBOL for MVS & VM (still supported, 1st compiler with OO support)
IBM COBOL for OS/390 & VM (most current compiler)

The last 3 listed require LE (Language Environment as their run-time)
The last 2 listed have an available add-on called MLE (Millenium Language
Extensions) for additional Y2K date handling.
All products listed from VS COBOL II on have an available add-on product for
Report Writer support.

For the PC (not originally, but currently for both OS/2 and Windows), IBM
VisualAge for COBOL is available which use the identical "front-end" to the
current IBM COBOL for OS/390 & VM product  - so syntax support is identical
(except when it isn't - for example LINE SEQUENTIAL support is only
available on the PC).  The PC product does NOT support the mainframe CMPR2
compiler option so no ANS'74 support is available on the PC.

ANSI support:
   ANS V4, OS/VS COBOL (with LANGLVL(1)) support ANS'68
   ANS V4, OS/VS COBOL (with LANGLVL(2)), VS COBOL II, and IBM COBOL for
this-and-that (with CMPR2) support ANS'74
   VS COBOL II and higher all support ANS'85
   COBOL/370 and higher support ANS'85 *plus* Intrinsic Functions
   IBM COBOL for this-and-that support ANS'85 *plus* Intrinsic Functions
*plus* Corrections Amendment.

    ****

Hopefully, this answers your question (in more detail than you really
wanted).

For AIX, there is a comparable product - but I haven't seen it being
upgraded as frequently as the PC product.
