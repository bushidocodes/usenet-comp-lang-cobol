I had a problem similar to yours when using PC_WIN_PRINT_FILE_EXT.
You should be aware in the contents of the file you want to print,
especially the combination of special characters (LF, FF, CR)to
control the page break of the report.

In my cobol program, instead of inserting control characters in a
line to be printed, i used the commands "write ... after advancing
n line(s)" and "write ... after advancing page" to generate the line
and form feed in my report. I printed this file on a laser and dot
matrix printer and i got the complete report.

I hope it would be helpfull.
