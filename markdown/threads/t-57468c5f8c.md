Not necessarily. It depends on the collating sequence being used.
LOW-VALUES corresponds to the lowest character in the program collating
sequence. On most sytems, this is binary zero and LOW-VALUE is indeed
zero. One some it is not. This is especially true if you define your
own collating sequence (see PROGRAM COLLATING SEQUENCE in the
OBJECT-COMPUTER paragraph). This technique is not portable.

The only portable way (in COBOL '85) is the following:

SPECIAL-NAMES.
ALPHABET std-1 IS STANDARD-1
SYMBOLIC CHARACTERS zero-term IS 1 IN std-1
.

Probably the best way in the program is:

MOVE zero-term TO some-item (where-to-put-it: 1)
