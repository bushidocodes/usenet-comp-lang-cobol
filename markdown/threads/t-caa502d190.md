Hi, We have downloaded input files ( datasets) from mainframe to PC to work
on MF-Cobol, but the signed fields were not getting converted completely to
ASCII. So we used compiler directive SIGN"EBCDIC" after which MF-Cobol
started recognising signed fields. But the problem is it is converting
unsigned fields also. For EX say 123, it is converting as 12C ebcdic (
equivalent to +123 in ascii). Also we have tried using CHARSET"EBCDIC"
compiler directive, but it is being rejected by the compiler. Anybody
having come across this problem or can suggest a better way to download from
Mainframe to PC. Suggestions r welcome. Thanks, Kiran
