Hi. I haven't enough experience with TELON to be able to answer
questions too intelligently. However, I can tell you that TELON is a
program generator. The vendor broke down programming into small
pieces, and put each possible function on a screen of its own, so it's
very highly structured. The programmer fills in the blanks to generate
COBOL pgms, both batch and CICS. Simply skip the screens that relate
to functions you don't need in this particular program.

You can also include your own code into the pre-structured paragraphs.
It's a nice tool that doesn't take too much time to understand if you
have a bit of programming experience.

TELON is a COBOL code generator with interfaces to DB2/DL1/VSAM etc. It
was originally marketed by Pansophic which was subsequently taken over by
CA. Although they have produced a Y2K compliant version it seems they may
stop supporting the product soon.

Basically it works by defining the program structure from a screen or
report which you paint.. Next you add data accesses and finally you
create custom code (straight COBOL) which gets copied into the program at
specific sections.

It saves most time when coding a CICS transaction since it creates
standard screen mapping and paging code. The main downside is the need to
be very familiar with the generic program structure it uses to get the
best from it. I have seen programs almost entirely coded in custom code
and ignore all the TELON facilities.

Good Luck
