respectively. Both are indexing schemes, ISAM predates VSAM is mostly
obsolete on IBM mainframes. Many current database management systems

internals of the two methods if you're interested at that level.

› Can anyone tell me what's ISAM and VSAM in Cobol urgently ?!

› Thank you !

In the IBM Mainframe world, ISAM was Indexed Sequential Access Method,
and has been obsolete for many years. It was replaced by VSAM, Virtual
Storage Access Method. VSAM supports three kinds of files -- ESDS (Entry
Sequenced Data Sets), KSDS (Key Sequenced Data Sets), and RRDS (Relative
Record Data Sets).

Most mainframe programmers are at least familiar with KSDS VSAM files,
because you have to specify the KEY in the SELECT statement and in the 01
buffer under the FD (File Definition). While KSDS files can be processed
sequentially, you can also use ACCESS IS RANDOM or ACCESS IS DYNAMIC in
the SELECT statement. Random access is direct access by key, Dynamic
access is sequential access after using the START verb to point to some
KEY (or partial key) anywhere within the file.

ESDS VSAM files are just sequential files that are managed by VSAM. You
can use the FILE STATUS returncodes to analyze the success of an I/O the
same as for KSDS VSAM files. If you have access to a COBOL manual, look
it up.

RRDS VSAM files are accessed by record number. Think of the file as an
array of records. The relative record number is the subscript used to
read or write a specific record. The relative record number is NOT part
of the record, unlike a KSDS VSAM file, where the key is a field in the
record.
