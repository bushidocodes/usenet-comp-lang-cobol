Have a look at http://www.qubit.org/
- as I found, there is already a
quantum-algorithm that does searches
on unordered tables in sqrt(N) time.
See
http://www.qubit.org/intros/PhysicsWorld/PhysicsWorld.html
by David Deutsch and Artur Ekert

As Cobol has a SEARCH-statement, a
quantum-computation of this would be
a cool thing.

But:

For for one thing, there are not yet
any quantum computers, and for elaborate
table-searches, one usually uses SQL and
a database which implements many
operations on tables really fast, using
well-known classical algorithms. Also,
a compiler implementor *could* secretly
decide to augment every *OCCURS* clause
with generated algorithms so that
searches, should a programmer use them,
are carried out fast as well.

However, one should keep an eye on quantum
computation, and Cobol has the one language
element behind which a quantum algorithm
could work. So is there somewhere
already more research in the context
"Quantum Cobol" being carried out?

Herwig
