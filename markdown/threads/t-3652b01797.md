It is impossible to debug a program without the actual source.  The
'syntax' is just what you think the program should be doing and not what
it is actually doing.  ie if it worked like the 'syntax' it wouldn't
have a bug, if you wrote out what the program is actually doing you
would find the bug.

      PERFORM WW
      ...    
   WW SECTION
      ...

   YY SECTION.

(code edited)


My best guess, baseless on evidence, is that you have mixed up
(inadvertantly) performs of paragraphs and of sections, for example XX
is not actually a SECTION (ie no keyword SECTION) so that the perform of
ww includes xx as a pargraph because the WW section ends on the next
SECTION keyword.  This would give an extra perform of XX by 'dropping
into' it at the end of WW.  When 'DRU' is found the subscripts are all
with the table and no harm is done.  When 'DRU' is not found the
subscripts point beyond the table and somehow corrupt other data.
