What for? All further releases of DEC compilers such like COBOL and FORTRAN are
downwards compatible, i.e. "old" features are present in the doc (and the

> 1. I want to know if vax cobol v4.4 comply with Cobol-85 standard?

Yes.  VAX COBOL V4.4 supported the ANSI-85 standard and some
differences related to the ANSI-74 standard.  See HELP COBOL /STANDARD
for some details.  These /STANDARD differences are still supported in
Compaq COBOL V5.7 for OpenVMS VAX.  If you have the .COM files used to
build the COBOL applications with VAX COBOL V4.4, you should recompile
using the same /STANDARD option (if any) used when the applications
were built with V4.4.  The defaut in both V4.4 and V5.7 is
/STANDARD=85.

VAX COBOL V5.0 added support for the 1989 intrinsic functions addendum
to the ANSI-85 standard, but this should not impact the upgrade
project you are doing now.

> 2. If not, where can I find vax cobol v4.4 information?

The best starting point for details on changes to COBOL on VAX since
V4.4 is the release notes:

  http://www.openvms.compaq.com/commercial/cobol/VAX_COBOL_RN_V57.htm

What we've tried to do in these release note is give a comprehensive
summary of all changes which have gone into the product on VAX over
the last 8 releases since V4.4.

The Compaq COBOL V5.7 for OpenVMS VAX documentation (User Manual and
Reference Manual) is available at

  http://www.openvms.compaq.com/doc/#cobol

This documentation covers V2.7 on Alpha (OpenVMS and Tru64 UNIX) and
V5.7 on OpenVMS VAX.

Yes.  As stated in the earlier replies, the intent with Compaq COBOL
V5.7 for OpenVMS VAX is that you can recompile and relink applications
which had previously been built with an older version of VAX COBOL.

For performance reasons, you should look at the /INSTRUCTION_SET
qualifier as part of your recompilation effort.  COBOL applications
generally perform best on modern VAX systems when compiled with the
NODECIMAL option.  The DECIMAL option (still the default for
compatibility with VAX COBOL V4 and earlier) was designed for early
VAX systems like the 780 and 750 with non-emulaed decimal
instructions.

I am assuming you are upgrading to a version of OpenVMS more recent
than V5.5-2.  Compaq COBOL V5.7 requires OpenVMS VAX V6.2 as a
minimum.  Compaq COBOL V5.7 is the single version of COBOL on VAX
currently supported by HP engineering meaning there is no support
today by HP engineering for any version of VAX COBOL, DIGITAL VAX
COBOL, or Compaq COBOL on VAX/VMS V5.5-2.
