That function is exported from  ODBCCP32.DLL (I assume your "ODBCCP32.DDL"
is a typing error) as "SQLConfigDataSource" in mixed case as shown. Your
missing external is in all upper-case as "SQLCONFIGDATASOURCE".  Import and
export names ARE case-sensitive.

The error message indicates the import table of the executable is using the
wrong case; so the problem is wherever the compiler/linker is getting its
import definition (the LIB file?).

Refer to lines 1480 and 1490. There you could add whatever information
(i.e. IP address, Server Name, Database, etc.) as you would normally
have used when establishing your ODBC String. Using this "ADO.NET"
type approach you DO NOT need to set up a ODBC connection at all. This
ADO.NET way, results in you having a more dynamic connection.

Optionally, (as I demonstrate in Chapter 18 of my book) rather than
hard code the connection string information in your program, you could
make use of the .NET XML Config file. Then, you would need to add a
few lines of code similiar to the following:
