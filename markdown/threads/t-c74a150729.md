I think what we are seeing is a splitering of diciplines.  Few have
any idea what the big picture is.  I hear all the time that I should
rewrite such and such an application in JAVA.  Why I ask?  So it can
run slower?  

We use JAVA where I work.  We actually use it as stored procedures in
an Oracle database.  For what it is good at, JAVA is fine.  It's when
you are so myopic that you try to force JAVA onto a process that just
plain isn't suited for it.

I see the same thing here with Databases.  I know SQL, I work with
Oracle databases.  We were doing some queries yesterday that just
would not come together in a timely fashion.  The multiple joins to
different tables, inner and outer were just too slow.  I extracted the
data, did some minor file manipulation to eliminate some of the joins
and we very quickly had our results.  What did I use to manipulate the
data?  A JAVA program?  No -- COBOL.  COBOL excells at data
manipulation.

So I'm not saying that JAVA is bad or that in itself it will fail to
deliver.  (Or XML or any other emerging technology).  But what I do
see is people falling so head over heels in love with the "new" stuff
that they try to use it for EVERYTHING, when it's just not suited for
the task.
