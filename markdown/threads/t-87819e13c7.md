ODBC is an interface, just like ADO, JDBC and embedded SQL.

ODBC itself doesn't know much about SQL. It just passes queries (and
data) from the application to the server and retreives data from the
server.

When reading a multi-row result-set (typically generated by a SELECT
statement), ODBC has support for reading several rows for each FETCH
(i.e. array fetch). This means that data for several rows in the
result-set are copied from the network buffer to some kind of data
area in the application.

result-set are grouped together into one network package. How this is
done is not specified by ODBC (or SQL).

same network package, even though array fetch isn't specified. In this
case one FETCH must be performed for each row in the result-set.
