If you have the FUNCTION TRIM, like OpenCOBOL has :-),
then you can do -

       IDENTIFICATION DIVISION.
       PROGRAM-ID. STRINGT.
       ENVIRONMENT DIVISION.
       DATA DIVISION.
       WORKING-STORAGE  SECTION.
       77   WS-NAME         PIC X(20) value "Bob Smith".
       77   WS-ADD1          PIC X(20) value "10 The High Street".
       77   WS-BALANCE  PIC ZZZZZ9.99.
       01   FLOC3                PIC X(70).
       PROCEDURE DIVISION.
           move 123.45 to ws-balance.
           STRING QUOTE   delimited size
                  FUNCTION TRIM (WS-NAME, TRAILING) delimited size
                  QUOTE           delimited size
                  ","                     delimited size
                  QUOTE           delimited size
                  FUNCTION TRIM (WS-ADD1, TRAILING) delimited size
                  QUOTE           delimited size
                  ","                     delimited size
                  FUNCTION TRIM (WS-BALANCE, LEADING) delimited size
                  ","                     delimited size
                  into floc3.
           display floc3.
           STOP RUN.

Note the "delimited" is only there for completeness and is not required.
Test -
$ cobc -x stringt.cob
$ ./stringt

I have no trim function but I have the following routine :

rtrim.cbl
      $set sourceformat"free"
1 tally-count pic 9(7).
Linkage Section.
1 lnk-len   pic s9(9) usage binary.
1 Lnk-StringX.
  2  Filler occurs 1 to 32000 Depending On Lnk-Len PIC X.

Procedure Division Using Lnk-Len Lnk-StringX.

move zero to tally-count
inspect function reverse(Lnk-StringX)
  tallying tally-count
  for leading spaces
if tally-count > 0 then
  move low-values to Lnk-StringX(lnk-len - tally-count + 1 : tally-count)
end-if

Exit Program
stop run
.

Then, exporting a csv is easy :

1 s1 pic x(80).
1 s2 pic x(80).
1 s3 pic x(80).
1 s4 pic x(256).

procedure division.
...
s1 s2 s3 receive values
...
*> prepare strings
call "rtrim" using length of s1 s1
call "rtrim" using length of s2 s2
call "rtrim" using length of s3 s3
*> export as csv
string
  s1 delimited by low-value
  ";" delimited by size
  s2 delimited by low-value
  ";" delimited by size
  s3 delimited by low-value
  into s4
end-string

If you have a line sequential file :
write myfile from s4

I also have a ltrim function to eliminate left spaces :
ltrim.cbl
      $set sourceformat"free"
1 tally-count pic 9(7).
Linkage Section.
1 lnk-len   pic s9(9) usage binary.
1 Lnk-StringX.
  2  Filler occurs 1 to 32000 Depending On Lnk-Len PIC X.

Procedure Division Using Lnk-Len Lnk-StringX.

move zero to tally-count
inspect Lnk-StringX
  tallying tally-count
  for leading spaces
if tally-count > 0 then
  move Lnk-StringX(tally-count + 1 : lnk-len - tally-count) to Lnk-StringX
end-if

Exit Program
stop run
.


So for numbers, you can proceed like this :
1 n1 pic zzzzzz.99.
1 s1 pic x(80).

move 10.99 to n1
move n1 to s1
call "ltrim" using length of s1 s1
call "rtrim" using length of s1 s1

s1 is now ready to be used in the string instruction and you will have
"10.99" instead of "    10.99".

Remember that working with low-values can be problematic. Do not forget
to initialize all variables with spaces before using them, or use
inspect replacing low-value by space.

I use these routines daily without problem.

Hope this helps.

Oh look, embedded spaces... and just to make life easier I'll assume that 
there's an unpredictable number of embedded spaces, so that STRING WS-NAME 
DELIMITED BY ' ' (etc.) will make someone unhappy just because the name is 
'BOB(space)(space)SMITH'.  Just for amateurish starters... how about 
something like:

PERFORM VARYING WS-NAME-SUB FROM LENGTH OF WS-NAME
 BY -1 UNTIL WS-NAME(WS-NAME-SUB:1) NOT = ' '
          OR WS-NAME-SUB = 0
    CONTINUE
END-PERFORM

... so that if WS-NAME contains something simple, like:

'JIM BOB  SMITH IV             '

... then WS-NAME-SUB should tick backwards from 30 to (squint point point) 
17.  Do likewise with WS-ADD1 and see if

STRING QUOTE, WS-NAME(1:WS-NAME-SUB),
       QUOTE, ',',
       QUOTE, WS-ADD1(1:WS-ADD1-SUB),
       QUOTE, ','.
       WS-BALANCE
 DELIMITED BY SIZE
  INTO WS-CSV-REC

... will even compile, let alone give you something potentially useful.

(Truncating leading spaces on WS-BALANCE is left as an exercise to the 
reader.)

          OR WS-STRING-IDX < 1
         CONTINUE
     END-PERFORM
     SET WS-STRING-LENGTH TO WS-STRING-IDX
     EXIT SECTION.

01  WS-STRING           PIC X(60).
01  WS-STRING-LENGTH    PIC 99.
...
GET-LENGTH-OF-STRING SECTION.
     MOVE ZERO TO WS-STRING-LENGTH
     INSPECT FUNCTION REVERSE (WS-STRING)
       TALLYING WS-STRING-LENGTH FOR LEADING SPACE
     SUBTRACT 60 FROM WS-STRING-LENGTH
     EXIT SECTION.

That last one will actually result in a negative number, but since it's 
unsigned, it has the effect of flipping it on the length of the string.

I just got my interpreter running and was playing around with string and 
unstring. If you want to handle initials and more than just first and 
last names (John E. Doe III), the following does work, and it's all 
string/unstring. Depending on your compiler, replacing subscripts might 
speed it up.. At least the variables and main fit on one page!

        WORKING-STORAGE SECTION.
        01  NUMBER-OF-NAMES PIC 99 VALUE 0.
        01  NAME-PTR PIC 99 VALUE 0.
        01  NUMBER-OF-ADDRS PIC 99 VALUE 0.
        01  ADDR-PTR PIC 99 VALUE 0.
        01  CSV-CHAR-PTR PIC 99 VALUE 0.

        01  WHOLE-NAME PIC X(30) value "john  doe".
        01  STREET-ADDR PIC X(30) value "666 Cobol Expressway".
        01  ACCT-BALANCE PIC ZZZZZ9.99 VALUE 123.10.
        01  NAME-TABLE.
            02 NAMES PIC X(20) OCCURS 5.
        01  ADDR-TABLE.
            02 ADDR PIC X(28) OCCURS 10.
        01  CSV-RECORD PIC X(132).

        PROCEDURE DIVISION.
       *IN A REAL PROGRAM YOU WOULD DO SOMETHING HERE
        INIT-PARA.
            MOVE SPACES TO CSV-RECORD.
        DO-IT.
            PERFORM EXTRACT-PIECES-PARTS
            MOVE 1 TO CSV-CHAR-PTR
            STRING QUOTE DELIMITED BY SIZE
                INTO CSV-RECORD POINTER CSV-CHAR-PTR.
            PERFORM INSERT-NAMES
                VARYING NAME-PTR FROM 1 BY 1
                    UNTIL NAME-PTR > NUMBER-OF-NAMES.
            ADD -1 TO CSV-CHAR-PTR.
            STRING QUOTE "," QUOTE DELIMITED BY SIZE
                INTO CSV-RECORD POINTER CSV-CHAR-PTR
            PERFORM INSERT-ADDR
                VARYING ADDR-PTR FROM 1 BY 1
                    UNTIL ADDR-PTR > NUMBER-OF-ADDRS.
       * leave the space,nobody will notice!
            STRING QUOTE "," ACCT-BALANCE DELIMITED BY SIZE
                INTO CSV-RECORD POINTER CSV-CHAR-PTR
            DISPLAY CSV-RECORD
            STOP RUN.

        EXTRACT-PIECES-PARTS.
            UNSTRING WHOLE-NAME DELIMITED BY ALL SPACES
                INTO NAMES (1) NAMES (2) NAMES (3) NAMES (4) NAMES (5)
            TALLYING IN NUMBER-OF-NAMES.

            UNSTRING STREET-ADDR DELIMITED BY ALL SPACES
                INTO ADDR (1) ADDR (2) ADDR (3) ADDR (4) ADDR (5)
                     ADDR (6) ADDR (7) ADDR (8) ADDR (9) ADDR (10)
            TALLYING IN NUMBER-OF-ADDRS.

        INSERT-NAMES.
            STRING NAMES (NAME-PTR) DELIMITED BY SPACE
                INTO CSV-RECORD
                POINTER CSV-CHAR-PTR.
            STRING " " DELIMITED BY SIZE
                INTO CSV-RECORD
                POINTER CSV-CHAR-PTR.


        INSERT-ADDR.
            STRING ADDR (ADDR-PTR) DELIMITED BY SPACE
                INTO CSV-RECORD
                POINTER CSV-CHAR-PTR.
            STRING " " DELIMITED BY SIZE
                INTO CSV-RECORD
                POINTER CSV-CHAR-PTR.

OUTPUT:
"john doe","666 Cobol Expressway ",   123.10

Yes, I have periods scattered around, and I couldn't use end-string or
