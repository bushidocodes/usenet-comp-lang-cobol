Do you have access to a VSE (or any other IBM mainframe) system - and if so,
do you have access to a COBOL compiler on that system?

If so, it will take about less than a hundred lines to write a COBOL program
to do what you want:

1) Define the Select/Assign to read the tape file (VSE vs MVS differences
exist - but either can read this tape file)

2) Define another output file

3) Use the file-layout of your input file for the 1st file

4) Copy the file-layout of your input file - but change any numeric input to
GUARANTEE that they are USAGE DISPLAY with SIGN IS LEADING SEPARATE. Point
this output file to DISK.

5) Loop thru reading the input file, MOVE CORR (or equivalent) to the output
file - and write it.

6) End the program

7) Use any mainframe to PC file transfer program - with EBCDIC to ASCII
conversion

7)

IBM, Fujitsu, and MicroFocus all provide standard software to download files
from the mainframe to a PC. Conversion from EBCDIC to ASCII (and vice versa)
can be specified at the time of downloading. If your particular flavour
can't handle tape (and to be honest, I've never tried that although I see no
reason why it shouldn't work) you could copy the tape to disk (must be a
catalogued dataset) then do the transfer.

If there are binary fields in the tape (like numbers f'rinstance,) these
will be garbaged by the conversion (unless you specify where they are and
what size and shape they are). From your description (which offers no
description of the actual fields in the records) it looks like you are
expecting straight character data. In this case you will have no problem and
do not need any special software.
