```
*sequential* file - there are no separators.

file (when reading).

file as SEQUENTIAL (not LINE SEQUENTIAL) with an additional two bytes on the
record and move in your own separator, viz:

...
MOVE X'0D0A' TO RECORD-ADD-ON
```

```
Usually the behaviour of a "line sequential" is dependant on the 
underlying file system accessed by the COBOL program. If you want to 
use some file system convention not supported by the underlying OS you
will have to code the thing yourself. If the compiler allows you 
access to "stream I/O" routines you can open the file with those and 
split the data blocks read into records based on the 0A (line feed) 
character.

PC-DOS, OS/2 and Windows/MS-DOS use 0D 0A as record terminators for 
"line sequential", *nix OSs uses 0A as a record terminator for "line 
sequential".

For PC-DOS, OS/2 and Windows/MS-DOS you could call the underlying file
routines for "binary" files and deblock the data read. The same 
applies for *nix OSs

If the writer of the compiler provides the file system routines to 
handle either line terminator for "line sequential" they usually 
provide a way to "tell" the compiler the treatment reqired for a 
particular file.
```
