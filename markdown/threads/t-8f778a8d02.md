Hello,

I'm developing a Cobol batch program that opens a few VSAM files in
order to make some calculations. The CICS programs create and mantain
the VSAM files, some of them with alternate indexes. During a CICS
procedure I am able to access VSAM files (both via the primary or the
alternate keys) without any problems at all, but when I try to read or
write from a file which has alternate keys during a batch procedure I
get an I/O error. Probably I'm not writing correctly the jcl file
associated to the batch job (there has to be a way to tell the compiler
that the file it's trying to open has alternate keys associated!).I'm
using CA Realia for windows ver 2.0

Hereafter there is a fragment of the RDT file (the 'CICS definition')
which describes one of the files with alternate keys.

CPANAGR DFHFCT TYPE=DATASET, -

DATASET=CPANAGR, -

ACCMETH=(VSAM,KSDS,KEY), -

SERVREQ=(UPDATE,ADD,BROWSE,DELETE)
CPANAG1 DFHFCT TYPE=DATASET, -

DATASET=CPANAG1, -

ACCMETH=(VSAM,KSDS,KEY), -

SERVREQ=(UPDATE,ADD,BROWSE,DELETE)
...
CPANAGR DFHXFT TYPE=FILE,
FILE=CPANAGR,
FILENAME='C:\DAT\CPANAGR',
KEY=(8,0),
ACCMETH=KSDS,
RECLEN=600
CPANAG1 DFHXFT TYPE=ALTKEY,
DATASET=CPANAG1,
BASE=CPANAGR,
DUPVALUE=YES,
KEY=(16,08)

Does anybody have any suggestions?

Thanks in advance :-)
