Let's see where to start? I could write a book about this (Oops, I already
did - see "OldBOL to NewBOL - A COBOL Migration Tutorial" which is available
from Micro Focus publishing - Call 1-800-551-5269) Now that I have finished
the required "plug"...

1) I sure hope that you aren't really migrating to COBOL/370. That product
is 2 versions out of date and either has been or is about to be dropped from
support. What you want to move to is either
IBM COBOL for MVS & VM (only one version out of date) or
IBM COBOL for OS/390 & VM (the current strategic product)

2) You don't tell us what you mean by saying that your are using ANS'74
COBOL. Do you mean that you are using OS/VS COBOL with the LANGLVL(2)
compiler option? If so, you probably have TONS of ANS'68 code in there (and
lots of undocumented stuff as well). (From the subject of your post, this is
my guess - and you have LOTS of work ahead of you.) Or do you mean that you
are using VS COBOL II with the CMPR2 compiler option (in which case you have
a medium-easy migration ahead of you).

3) The best place to start you work is with the IBM "Migration Guide".
There is one for each of the COBOL's that you might be migrating TO (not
from). Get the one for your target product, and it will really help you
plan out your migration.

4) The IBM migration guide includes some information on the IBM products
that can be used to automate your source code conversions. If you get my
book (or some of the other non-IBM books) it will list other vendor's
products as well. (My personal preference IS the IBM product, but some of
the other products do have some benefits as well.) Make certain that you
know whether or not you have ANY "report writer" code in your programs.
This makes a big difference to what products you need. (Many shops that
THINK they don't have any, get quite surprised in the middle of their
migrations)

A while back we looked for a product of the kind you have in mind.
We ended up doing the conversion of a select group of our programs
manually, but the product that we considered was:

"COBOL Coversion Aid" from IBM (Product# 5785-AAT). According to
IBMlink:

›››››
The COBOL Conversion Aid program offering is a set of programs and
commands designed to assist the user with the conversion of
OS/VS COBOL source code into VS COBOL II source code; it is an effective
tool in expediting this tedious and error-prone conversion process.
This program offering provides: Source program and copy module
conversion ... diagnostic listing by program ... language conversion
program customization facility ... conversion management reports.
The output of the conversion is generally executable under OS/VS COBOL;
therefore, the aid can be used prior to VS COBOL II installation.
The conversion process and its customzation are driven by ISPF panels.
<<<<<<<

A pair of helpful manuals, you might want to get hold of are:
1. COBOL for MVS & VM Compiler and Run-time Migration Guide GC26-4764
2. LE Run-time Migration Guide SC28-1944
