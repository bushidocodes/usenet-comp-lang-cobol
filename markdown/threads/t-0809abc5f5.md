If you tried the PERCobol Educational version, it does not have the speed
enhancements that the Enterprise version has.  Also, the PERCobol runtimes
are about 500x faster than they were last year around this time.  Be aware
also that PERCobol programs are limited to the speed of the JVM, unless you
compile them natively.

We think of PERCobol programs as "Cobol that runs on a Java Virtual
Machine."  Since PERCobol does output Java source/bytecode, you can use it
to convert your Cobol to Java.  But it wouldn't be much different than using
other Cobol compilers to convert Cobol to Assembler or Machine Language.
Even though they do, it isn't very practical.

The great thing about PERCobol is that it provides platform independence, a
modern GUI, and access to the latest Java technology.  You can code
Servlets, HTML, TCP/IP, E-Mail, JavaBeans, and more right from Cobol.  Your
program will run as an applet or an application, right out of the box.
