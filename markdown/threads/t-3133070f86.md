```
Given a recent thread (and not knowing WHEN any implementation will be
available for the draft COBOL Standard's Test-NumVal intrinsic function) - I
was wondering if this group would come up with some "standard conforming"
(85/89/93 - not 200x) routine for testing if a "passed parameter" would or
would not be valid input to the NumVal intrinsic function.  (Once this is
created, then we could work on NumVal-C).

"assumption(s)"
A) The subprogram compiles in a program with intrinsic functions - otherwise
there is no need for this routine.
B) Performance is not as important as "maintainability" - but performance is
certainly "nice to have"
C) I have used my "own" style in my start-up code below.  Feel free to
"modify" it as you think appropriate. (If you compile your program, make
certain that you compile it with ANSI Flagging on, so your routine is
*totally* ANSI/ISO conforming.)  Because of internet/Usenet "issues" - I
have *not* provided code beginning in column 7/8.  You may do this - or
indicate that it needs to be done.

"requirements"

1) Four Linkage Section fields:
   - an integer given the length of the string to be tested
   - a PIC X(1) field with the decimal point character (i.e. "," or ".")
  - an integer returning 0 if the field is valid and an integer (listed
below) if the input is invalid
  - the actual "alphanumeric" string to be tested  (assume USAGE DISPLAY)

2)  Rules for what is "valid"
  - There must be at least one digit (0-9) and no more than 18 digits
    return 1 if this rule is violated
  -  There may be leading and/or trailing spaces.
  - There may be NO spaces between the first non-space character and the
last non-space character - except as noted in the next two rules
    return 2 if this rule is violated
  - The first non-space character may be either "+" or "-" - if so there may
be spaces before the next non-space character - which must be a digit - and
the last non-space character must be a digit or the decimal-point (not a
sign character or string)
    return 3 if this rule is violated
 - The last non-space character (or two) may be "+" "-" "CR" "DB" (c, r, d,
b are  NOT case sensitive).  This may only occur if the first non-space
character is a digit or the decimal point character. If this/these are the
last non-space character(s) then there may be spaces preceding them before
the last digit or decimal point character.
    return 4 if this rule is violated
  - There may be 0 or 1 occurrence of the "decimal point" character (this
may be the first or last non-space character)
    return 5 if this rule is violated
  - All non-space characters must be digits or + or - or in the pair "CR" or
"DB" (case insensitive)
    return 6 if this rule is violated
  - If something else is "wrong"
    return 7

   ***

To start the program out (but I haven't tried compiling this yet <G>):

Identification Division.
 Program-ID.  Is-It-Valid-for-NumVal
Data Division.
 Working-Storage Section.
01    CheckForEnd    Pic X    Value "N".
        88  End-of-Loop            Value "Y".
 Linkage Section.
* Modify the following if you want to accept more than 999 byte input
01    LengPassed         Pic 9(3).
01    DecPointChar      Pic X.
        88  DPC-Comma        Value ",".
        88  DPC-Period          Value ".".
01    Return-Value        Pic 9.
        88  RV-OK                         Value 0.
        88  RV-not-1-18-digits         Value 1.
        88  RV-BadMiddleSpaces    Value 2.
        88  RV-Bad1stChar              Value 3.
        88  RV-BadLastChar            Value 4.
        88  RV-WrongNum-DP        Value 5.
        88  RV-BadInputChar           Value 6.
        88  RV-OtherError                Value 7.
01     Passed-String.
        05    EachByte  Occurs 1 to 999 times
                                depending on LengPassed
                                indexed by ByteInd
                                    Pic X.
Procedure Division Using    LengPassed
                                          DecPointChar
                                          Return-Value
                                          Passed-String.
Mainline.
    Perform HouseKeeping
    Perform LoopRoutine
        Until (Not RV-OK
             or  End-of-Loop)
    Perform WindUp
    Exit Program
     .
HouseKeeping.
    Set RV-OK to True
    Evaluate True
        When Not DPC-Comma and Not DPC-Period
            Set RV-OtherError to True
        When LengPassed Not Numeric
            Set RV-OtherError to True
        When LengPassed Zero
            Set RV-OtherError to True
        When Other
            Continue
    End-Evaluate
     .

LoopRoutine.
    Continue
    Set End-Of-Loop to True
     .
WindUp.
    Continue
     .
```

```
Here is another way to do it (not a better way, just a different way).  It
also supports numval-c (sort of, I couldn't remember off the top of my head
what special characters it allows):


CLASS POSSIBLE-NUMVAL IS '0123456789,.'
CLASS POSSIBLE-NUMVALC IS '0123456789,.$+-%'.

WORKING-STORAGE.

01 WORK-FIELDS                 COMP.
   05 S1                                    PIC S9(9) VALUE +1.
   05 S18                                  PIC S9(9) VALUE +18.
   05 ALL-SPACES                 PIC S9(9) VALUE ZERO.
   05 LEAD-SPACES              PIC S9(9) VALUE ZERO.
   05 END-SPACES                PIC S9(9) VALUE ZERO.
   05 ALL-PERIODS               PIC S9(9) VALUE ZERO.
   05 DOUBLE-COMMAS     PIC S9(9) VALUE ZERO.
   05 IMBED-SPACES            PIC S9(9) VALUE ZERO.

LINKAGE SECTION.
01 LINKAGE-PASSED.
     05 LINK-LENGTH            PIC S9(4) COMP.
     05 LINK-INFO.
          10 LINK-CHAR           OCCURS FROM 0 TO 999 TIMES
                                                DEPENDING ON LINK-LENGTH
                                                PIC X.

PROCEDURE DIVISION.

MOVE ZERO TO RETURN-CODE.

EVALUATE TRUE
    WHEN LINK-LENGTH = ZERO
    WHEN LINK-LENGTH > S18
         MOVE 16 TO RETURN-CODE
         GO TO 9999-RETURN
END-EVALUATE.

EVALUATE TRUE
    WHEN LINK-LENGTH = S1
       AND LINK-INFO (1:LINK-LENGTH) NUMERIC
         GO TO 9999-RETURN
    WHEN LINK-INFO (1:LINK-LENGTH) = SPACES
        MOVE 12 TO RETURN-CODE
        GO TO 9999-RETURN
    WHEN LINK-INFO (1:LINK-LENGTH) POSSIBLE-NUMVAL
        CONTINUE
     WHEN LINK-INFO (1:LINK-LENGTH) POSSIBLE-NUMVALC
        MOVE 2 TO RETURN-CODE
     WHEN OTHER
        MOVE 12 TO RETURN-CODE
        GO TO 9999-RETURN
END-EVALUATE.

MOVE ZERO TO ALL-SPACES, ALL-PERIODS, DOUBLE-COMMAS
                              LEAD-SPACES, END-SPACES, IMBED-SPACES.

INSPECT LINK-INFO (1:LINK-LENGTH) TALLYING
       ALL-SPACES FOR ALL SPACES
       ALL-PERIODS FOR ALL '.'
       DOUBLE-COMMAS FOR ALL ',,'.

INSPECT LINK-INFO (1:LINK-LENGTH) TALLYING
         LEAD-SPACES FOR LEADING SPACES.

INSPECT FUNCTION REVERSE (LINK-INFO (1:LINK-LENGTH)) TALLYING
         END-SPACES FOR LEADING SPACES.

COMPUTE IMBED-SPACES =
         ALL-SPACES - (LEAD-SPACES + END-SPACES)
END-COMPUTE.

EVALUATE TRUE
   WHEN ALL-PERIODS > S1
   WHEN DOUBLE-COMMAS > ZERO
   WHEN IMBED-SPACES > ZERO
       MOVE 8 TO RETURN-CODE
       GO TO 9999-RETURN
END-EVALUATE.

9999-RETURN.
GOBACK.

Enjoy....

Dave
```
