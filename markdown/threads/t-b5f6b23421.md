```
Depends on your compiler.  For most '85 Standard compilers, you will need to
do the sorting yourself (via a loop).

However, Micro Focus (and possibly others - although I don't know of any) do
already support a "table sort" feature that is available in the 2002
Standard.

Finally, it is possible to use the "traditional file sort" (from the '85
Standard) statement with an input and output procedure to sort
working-storage items.  If you know how to use Input/Output procedures, you
just "release" all the elements of the table in your input procedure and
return them (in order) in your output procedure.
```

You don't need a different table; you can do it 'in situ'. When the number of
entries is small, as in 10, you can sort the table with a bubble sort. Pass
through comparing w-saved-gi (w-store-index) to w-saved-gi (w-store-index + 1).
If they are out of order, exchange them. When you make a pass with no exchanges,
you're done. 

When the number of entries is larger than approx. 100, bubble sort becomes
horribly inefficient compated to other algorithms. Those algorithms will be
introduced in the intermediate class. Here, it looks like the instructor is
prompting you to write a bubble sort.
```

You may also try what I call an "Insert Sort" which means you basically use
a portion of a bubble sort to insert the values into the appropriate row as
they are populated. With this you could do anything from randomizing where
you start to look to using an "Index Sort"/"Shell Sort". Which means you
simply have another field which holds the index. Rows are simply appended
and then just the indexes are resorted as you insert them.
```
