C-ISAM is an indexed file system used by Infomix database system and
also adopted by X-Open as a 'standard' implementation of indexed
files.

D-ISAM is a compatible system that was cheaper.
 
> I've been asked to extract some data from a Microfocus database.  I'm not a
> Cobol programmer and am trying to get up to speed on the file formats, etc.

> I'm doing all of this on SCO Unix as well....

It is quite possible that the MF system used C-ISAM compatible indexed
files on SCO Unix. This means that the files will be in two parts - a
data file containing fixed size records each with a cr/lf terminator
(or possibly lf only on some systems) and an index file that is
filename.idx.  The index can be ignored if you just want the read the
data records.  Read fixed length and ignore any records that do not
have cr/lf as the last two bytes.

The content of the record is determined by the application and is not
possible to know exactly without source code.  Inspection may allow
you to guess at much of it.

Bruce

C-ISAM is a file indexing library developped by Informix.
D-ISAM is a clone of C-ISAM.

MicroFocus used a modified early version of C-ISAM to handle the indexing of
their "ISAM" files.

I don't think you have to use C-ISAM.

If you look in the back of the MicroFocus Cobol system reference, you will
see an appendix full
of file layouts.  You can just ignore the File Indexes and open the data
file as an ordinary C file and read it.
