```
This should do the trick as I just chopped it out of some code I
still use today. It is a good idea to keep it as a separate program
which is called and cancelled at the start of your system as it is
operating system specific.

       IDENTIFICATION DIVISION.

         PROGRAM-ID. zzmenusz.
      *  AUTHOR.     Alister Munro, 01/03/1993.

       ENVIRONMENT DIVISION.

         CONFIGURATION SECTION.

           SPECIAL-NAMES.

         INPUT-OUTPUT SECTION.

           FILE-CONTROL.

       DATA DIVISION.

         FILE SECTION.

         WORKING-STORAGE SECTION.

      *  ************
      *  * LITERALS *
      *  ************

           78  78-kb-io                VALUE x"B0".

      *  *********************************
      *  * Used to dissable CTRL + BREAK *
      *  *********************************

           01  ws-kb-io-grp.
             03  ws-kb-io-func         PIC 9(02)   BINARY.
             03  ws-kb-io-parm         PIC 9(02)   BINARY.

         LINKAGE SECTION.

       PROCEDURE DIVISION.

         INITIALISE SECTION.

           MOVE 4 TO ws-kb-io-func
           MOVE 1 TO ws-kb-io-parm
           CALL 78-kb-io USING ws-kb-io-func ws-kb-io-parm

         EXIT PROGRAM.

       STOP RUN.
```
