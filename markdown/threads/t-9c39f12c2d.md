Hi.

It's probably best to post to the Micro Focus forum under
http://www.cobolportal.com/ to get this resolved, but here's a couple of
initial comments which should help.

The makefile as provided by Oracle to relink the COBOL run-time explicitly
passes the -t cob option, to link with the threaded COBOL run-time. This is
because the Oracle client itself is linked with the threaded C run-time. It
appears that you removed this option from the Makefile, which I believe is
incorrect.

I'm assuming that you're using a 32-bit version of Server Express, and that
your LIBPATH setting includes $ORACLE_HOME/lib32, rather than
$ORACLE_HOME/lib .

Having *only* changed -xe to read -ze -- noting that there are *two*
references wish you will need to update -- you should be able to invoke the
following to relink the shared object correctly for use with a 32-bit
product :

make RTSORA=rtsora.so LIBDIR=lib32 -f ins_precomp.mk relink
EXENAME=rtsora.so

With rtsora.so in a directory included in your LIBPATH setting, you should
then be able to compile your Pro*COBOL application with the additional COBOL
compiler directive :

-C "initcall \"rtsora.so\""

Alternatively, if you'd rather just relink the COBOL run-time, using the
original makefiles, you should be able to invoke :

make EXE=$ORACLE_HOME/bin/rtsora32 LIBDIR=lib32 -f ins_precomp.mk rtsora32

Note that if performing this latter approach, you should replace
$COBDIR/bin/rts32_t , rather than $COBDIR/bin/rts32 . I'd also strongly
recommend you back up the original files first.

Regards,
Simon.
