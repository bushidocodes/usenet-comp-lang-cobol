This is a formal Request For Discussion (RFD) for the creation of a
world-wide unmoderated Usenet newsgroup comp.lang.cobol.acucobol.
This is not a Call for Votes (CFV); you cannot vote at this time.
Procedural details are below.

Newsgroup line:
comp.sys.acucobol	All aspects of the use of Acucobol.

RATIONALE: comp.sys.acucobol

Acucobol is a company based in San Diego CA. Currently all users and
developers use email to communicate ideas and problems to Acucobol
(now known as Acucorp). These ideas and problems however are not
released to the general populous of Acucobol users. This presents a
problem for both Acucobol and the Users.

The purpose of creating this comp.sys.acucobol newsgroup is to have a
central point of contact for all users. Discussions involving
Acucobol-specific information would be greatly enhanced by this group
due to the world wide use of Acucobol.

Due to the specific nature of some of the Acucobol products, there is
a need for a seperate group to discuss these topics as opposed to just
the cobol language.

Here are some reasons in favor of a comp.sys.acucobol:

Currently, most developers cannot easily come in contact with other
developers other than by attending Acucorp's yearly developer's
conference.

Acucobol contains significant extensions to the COBOL standard.
Discussion of these extensions would be of little interest to
comp.lang.cobol readers who are not Acucobol developers.

There are a number of Acucorp products which are used in conjunction
with Acucobol that do not directly involve COBOL. Because these
products tend to be used in larger, more complex systems they also
tend to cause more discussion than the base language. Such discussion
would be only marginally on topic in comp.lang.cobol.

CHARTER: comp.sys.acucobol

This newsgroup is for discussion of all aspects of the use of
Acucobol.  Some examples of relevant topics are:

  Coding techniques
  Learning and teaching materials
  Discussions on coding
  User Support
  Technical Support
  User Discussions
  General Information

The primary purpose of this group is to pass information between the
users of Acucobol. Explicitly discouraged are the posting of
commercial advertisements, any kind of flaming and personal arguments,
as well as the posting of binaries of any kind.

END CHARTER.

PROCEDURE:

This is a request for discussion, not a call for votes.  In this phase
of the process, any potential problems with the proposed newsgroups
should be raised and resolved.  The discussion period will continue
for a minimum of 21 days (starting from when the first RFD for this
proposal is posted to news.announce.newgroups), after which a Call For
Votes (CFV) may be posted by a neutral vote taker if the discussion
warrants it.  Please do not attempt to vote until this happens.

All discussion of this proposal should be posted to news.groups.

This RFD attempts to comply fully with the Usenet newsgroup creation
guidelines outlined in "How to Create a New Usenet Newsgroup" and "How
to Format and Submit a New Group Proposal".  Please refer to these
documents (available in news.announce.newgroups) if you have any
questions about the process.

DISTRIBUTION:

This RFD has been posted to the following newsgroups:

  news.announce.newgroups,
  news.groups,
  alt.cobol,
  comp.lang.cobol

Proponent: Scott Woodward <scottw@southpacfinance.com.au>
