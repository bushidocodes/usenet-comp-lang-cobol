Hi

I need a touch of help with accepting keyboard input. I want to just
accept two digits and make sure that it is a two digit number.

So far I have:

MOVE 'N' TO HOURS-VALID
DISPLAY "Employee Name:" EMP-NAME
DISPLAY "Employee Number:" EMP-NOS
DISPLAY "Please enter the number"
DISPLAY "of hours worked by this employee"
PERFORM C-GET-HOURS UNTIL HOURS-VALID = 'Y'
END-IF.

C-GET-HOURS SECTION.
C-BEGIN.
ACCEPT WS-HOURS-WORKED.
IF WS-HOURS-WORKED NUMERIC AND WS-HOURS-WORKED
< 100 AND WS-HOURS-WORKED >0
THEN
PERFORM D-CALCULATE-PAY
PERFORM E-PRINT-PAYSLIP
MOVE 'Y' TO HOURS-VALID
ELSE DISPLAY "Invalid input please try"
DISPLAY "again two digit number"
END-IF.


This verification routine works fine if I type in characters or a
number less than 10, then it propts to re-enter. If I type in a three
digit number then it accepts it fine and in the hours-worked variable
the first two digits are in there.

Is there some sort of modifier to Accept?

Any help would be greatly appriciated.

Check to see if your compiler uses EXACT as an ACCEPT option.

If you use it on the end of the ACCEPT command it will automatically jump,
when you fill the field to it's maximum size, to the next statement or
ACCEPT depending on how your program is using ACCEPTS.

The specific way ACCEPT handles an input larger than the receiving field
is defined by the COBOL implementation. Apparently your COBOL truncates
the rightmost portion. As Bill Klein, I would also suggest ACCEPTING to
a larger field and scanning for the number. Here is a routine which will
do this for you.
