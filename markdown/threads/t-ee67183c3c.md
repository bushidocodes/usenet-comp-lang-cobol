Also - you might want to investigate the hardware...
Most off-the-shelf PCs don't use error-correcting or
parity checking memory anymore (to save a few cents.)
So, your programs (and windows) can just mysteriously
crash or produce wrong results.

In this day-and-age; running anything meaning on
a PC, it pays to specifically request ECC memory...
Believe me - I've "been there - done that."

In any event, if some machines are working with
the same workload - I doubt it's the software.
You'll need to apply good diagnostic processes to
isolate the cause, etc...
