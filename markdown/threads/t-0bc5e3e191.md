Actually, use the CURSLOC=YES parameter on your DFHMSD macro when defining
your BMS map. Upon completing a RECEIVE of the map the flag byte of the field
in which the cursor is positioned will have its second bit turned on. Although
bit testing in COBOL is not easy it is still doable. All you need to do is create
a test for each each cursor sensitive field's flag byte. This makes the actual
releative scrren byte address a non-issue and simplifies maintenance of the
program. One piece of advise, when using cursor sensitivity be aware that
3270 causes the cursor to skip to the next field when you fill the field in
question, consider how you intend to use this information. I've built a lot
of cursor sensitivity into applications that use a BMS alternative and I've
found that you need to take great care in that you really do what the terminal
intends. Anyway, hope this helps.
