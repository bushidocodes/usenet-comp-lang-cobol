With WB 4.x there were several keyboard problems, especially with
Win98. MF fixed (or better: worked around) them one by one with the
following NetExpress versions. In NetExpress 3.1 there is only one
left: exactly the one you have described.

You can read the details in Microsofts Knowledgebase, and since it is
a Windows bug, you shouldn't expect a fix by Merant. You shouldn't
expect a fix by Microsoft either, since they don't think it is that
important as it really is.

If you use your apps for internal purposes there are several tools
available that provide the keyboard mapping means you need to get your
comma working. If you have to support customers, good look to you. You
can either talk them into an other operating system or make them
install one of these mapping tools. There simply is no satisfiying
solution.
