Oops! Is this a restriction within the Fujitsu environment ? Because it is
an ANSII compliant way to read data sequential from an indexed file. This
was used in the old days to increase performance. At those days the indexed
files consisted of two seperate files (or data area's in one file)
containing data and index entries. Which means two physical reads when
accessing one record (is still something like that but modern disks are so
fast it does'nt matter anymore (usually)).

To increase performance we accessed indexed files sequentially when possible
(could be done only when the order of the records being processed did'nt
matter).

Yes you can.

Specify SELECT OPTIONAL .. and it will create the file if it doesn't
exist when OPEN I-O.  If it is OPEN INPUT and doesn't exist the file
will be treated as if it does exist but is empty (ie first READ will
follow AT END).  In both cases the file will be 'opened'.
