Having problems with handling variants in programs or documentation? 
 
You may be interested in XVCL (XML-based Variant Configuration
Language). XVCL is a meta-language, 
method and tool to facilitate software reuse, developed at the Software
Engineering Lab, National University of Singapore. 
With XVCL, you can automatically integrate variants into all kinds of
product line assets such as architecture, code components, 
documentation, even UML models. It synthesizes custom versions of those
assets from small sets of non-redundant meta-components. 
XVCL can effectively deal with any information structure of your choice,
not only product line assets, and is particularly helpful for large 
and complex information structures, affected by many inter-dependent
variants. Components exposed to multiple reuse contexts are 
certainly a good example.
 
XVCL is an open source software available at
http://fxvcl.sourceforge.net <http://fxvcl.sourceforge.net/>  This site
also contains XVCL specs, examples, demos, case studies, white papers,
etc. 
 
XVCL is a modern and versatile version of Bassett's frames, a reuse
technology that has achieved substantial productivity improvements in 
large data processing product lines written in COBOL. XVCL blends not
only with contemporary programming paradigms but can synthesize 
documents in any language, offering an effective reuse mechanism that
complements other design techniques. For example, we managed to 
eliminate redundancies in Java Buffer classes - we generate Buffer
classes in their original form from a base of non-redundant
meta-components, 
eliminating more than 60% of the code we find in the original Buffer
library. XVCL is based on a "composition with adaptation" mechanism - a
simple 
principle that appears to work surprisingly well.
