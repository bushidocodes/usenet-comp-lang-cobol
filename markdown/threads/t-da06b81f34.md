I am currently developing a Web Application using Dynamic HTML.

The Scripting Object Model provides for Objects to be included on Web Pages
and these can include standard forms and controls and foreign Objects like
ActiveX controls.

I have had no difficulty in embedding ActiveX controls and referencing their
Attributes and Methods with VBScript and JavaScript.

For simple things like static drop down list boxes, Scripts are simple and
efficient.  For slightly more complex things Java Applets work fine.

However for validations and RDB access I intend to use CGI and thought that
COBOL might be good...

My first attempts succeeded and I had no difficulty in obtaining data
submitted from Web forms, in COBOL using a CGI interface. Neither do I have
any problem in writing HTML from the COBOL program and even have it creating
a dynamic page which includes a Java Applet to display some stuff...

My current problem is that I want to access Objects on the page DIRECTLY
from the CGI COBOL code. (This is a lot different to being passed the values
of certain of their attributes through the CGI interface.)

As an example (a real one...) I have a drop down list box on the web page
and its ID is "ListBox1". This object has a Method called "AddItem" which
enables lines to be added to the list. Using VBScript or JavaScript I can
invoke this Method without any problem at all, right on the HTML page. But I
can only add static lines to the box (Scripts are severely limited in what
they can do for file access...). I actually NEED to access an ISAM file (or
RDB table) and create a set of values (possibly as a Line Sequential file)
which can be added to the list box using the "AddItem" method. These values
will change according to who is accessing the Web page and from where...

Now, it isn't very hard to write a server side CGI COBOL program that does
all the file access and either creates a simple file to be loaded into the
list box or simply invokes the "AddItem" method directly as each record is
obtained.

BUT, in order to do this, I need an Object Reference to the List Box.

In VBScript I can say:        SET ObjRef = ListBox1
and the variable "ObjRef" will contain the Object Reference to the ListBox.

(I can also invoke the required method directly:  Listbox1.AddItem ("entry 1
in the box") ) This works beautifully and is fast and efficient with no
requirement for access back to the server.

I set up "ObjRef" as a Global Variable on the web page and ensured that the
above SET statement was activated before my COBOL program.

However, when I try and get the value of "ObjRef" through the CGI interface
I receive nothing, and my COBOL consequently fails when trying to invoke a
Method from a NULL Object Reference.

I COULD create an ActiveX object that automatically loads the List Box, and
simply not use the CGI at all, but this is more work than I would like at
the moment and I really would like to find out if it is possible to pass Web
Page Object References to the CGI.

Does anybody know of an EFFECTIVE way to get a Web Based Object reference
into a COBOL CGI script?

TIA

Pete.
