Long ago I wrote an accounting & inventory system (circa 1985).  It has been
running without a problem.  I have been advising the business owner that I
wrote it for since 1995 that Year 2K was a problem and he needed to find
another product/package.  He has been unable to find anything that comes
close to what he has now.

Long story short.  I have agreed to change his system for Y2K.  I want to
convert it to a Cobol Compiler that is currently supported so that if I run
into problems I can get help.  I also used Screenio as the screen handler.
With extensive ISAM file use.

What Compiler will offer me the easiest transition?   Is Realia (CA-Cobol) a
viable solution these days?  It has been 7 years since I did any serious
programming or followed the companies developing compilers so I am really
out of touch.

Please help.  Your comments and experiences will be appreciated.

Scott

Not sure if Realia is still selling COBOL + - their version 5.0 product. 
There are significant issues in linking for a conversion.

Let me rank your potential solutions.

1.  Current Realia and Screenio - even If CA-Realia does not have a
screenio version, you can get it from Norcom.  This is the easiest solution
with the lowest learning curve.  (BTW- I have used Realia since Verison
1.0, and I still have applications in production that use Screenio).

2.  If COBOL + is not available - you may have to move to Windows, and the
32 bit version of CA-Realia - their Workbench product - which is the
currently supported version - but it does not make DOS based executables.

3.  Another solution - if you wish to move to Windows at the same time -
Convert to the COBOL compiler of your choice - I recommend Fujitsu - and
use COBOL sp2 from Flexus as a SCREENIO replacement
(http://www.flexus.com).  I have converted several of my systems that were
written using Screenio to COBOL sp2 - with relative ease.

If you have specific questions or concerns about any of these approaches -
please feel free to e-mail me!
