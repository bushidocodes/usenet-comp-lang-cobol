```
The following block of code is currently running in a program on an
Alpha OpenVMS v 7.1-1H1 with Digital Cobol v 2.5.

WHO is defined as PIC X(12) value spaces.

    IF WHO = "MS" OR "LG" OR "NC" OR "DD" OR "LY" OR "MG" OR "JR" OR
             "MEH" OR "KK" OR "RB" OR "MIS" OR "PJV" OR "RTS" OR "NJH"
        MOVE "L" TO USER-DIV
    ELSE
    IF WHO = "JN" OR "DF" OR "EW" OR "BL"
        MOVE "S" TO USER-DIV
    ELSE
    IF WHO = "SY" OR "LA"
        MOVE "A" TO USER-DIV
    ELSE
        GO TO 999-END.

When this same program is compiled on an Alpha OpenVMS 7.3 system
running Compaq Cobol v 2.7, there is no compilation or run-time error
but this same block of code does not execute correctly.  Every time it
will take the last ELSE branch which does a STOP RUN.

If I comment out the above IF block and insert the following simple IF
it works fine:

    IF WHO = "SY"
       MOVE "L" TO USER-DIV
    ELSE
       GO TO 999-END.

Also, if I change the PIC format to X(3) it works fine with the more
complex IF block.

Anyone have any ideas?  It's been awhile since I've used Cobol so I
may be missing something that is readily apparent to everyone (but
me).

Robin
```

One of us is missing something here.

Unless the last nine bytes of the field are never blank, and the third byte 
always is, I don't think that's going to help at all, and for the same reason 
that the original code doesn't work: when comparing two character fields of 
different lengths, Cobol pads the shorter one on the right with spaces to the 
length of the longer one.

Hence, the original statement is equivalent to

IF WHO = "MS          " OR "LG          " OR "NC          " ... etc.

and your suggestion is basically the same thing, with only one trailing space 
instead of ten.

Try this instead:

* Test two-letter combinations
IF (WHO(1:2) = "MS" OR "LG" OR "NC" OR ...  etc.)
* Test three-letter combinations separately
OR (WHO(1:3) = "MEH" OR "MIS" OR "PJV" OR ... etc.)
        MOVE "L" TO USER-DIV
ELSE
        IF WHO(1:2) = "JN" OR "DF" OR "EW" OR "BL"
                MOVE "S" TO USER-DIV
        ELSE
                IF WHO(1:2) = "SY" OR "LA"
                        MOVE "A" TO USER-DIV
                ELSE
                        GO TO 999-END.
