sai··.@ti··c.net (Robert Schuldenfrei) wrote:



› Without getting into a "cat fight" could you please explain how you legally use
› MicroFocus without paying a royalty?  I would love to use MicroFocus to do
› client work, but I always use C or BASIC because of the royalty issue.  Our
› firm's product is a MicroFocus application, so it is in our interest to play by
› the rules.




If you're using Workbench 3.2 (16-bit), check out the Workbench User Guide
Chapter 10. If you're using Workbench 4.0, check out the Object Cobol
User Guide - Volume 1 Chapters 11 & 25.

Both references tell how to package an application, what support modules
to include based on what your app uses and what modules require licensing.
If you don't use any of the OSX modules, you should be able to ship you
app without any additional license fee. I don't believe there is any
problem with Dialog System modules either.
