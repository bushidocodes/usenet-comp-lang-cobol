I have done things like this in IBM COBOL II and higher:

01  CUSTOMER-TABLE.
    05  CUSTOMER-ENTRY    OCCURS 5000 TIMES INDEXED BY CUST-X.
        10  CUSTOMER-NUMBER      PIC X(05)  VALUE HIGH-VALUES.
        10  CUSTOMER-NAME        PIC X(30)  VALUE SPACES.

I believe it should work with any ANSI-85 COBOL.

He did ask for the OCCURS clause to be at the elementary level.   I never have
an occurs at the elementary level, and don't know which versions of CoBOL accept
that.

All versions of CoBOL will accept the following:

01   DAY-OF-WEEK-TABLES.
      05   DAY-OF-WEEK-TABLE.
            10   THIS-DAY           PIC X(09)    OCCURS 7.
      05   DAY-OF-WEEK-TEXT    REDEFINES DAY-OF-WEEK-TABLE.
             10   FILLER              PIC X(09)    VALUE 'SUNDAY'.
             10   FILLER              PIC X(09)    VALUE 'MONDAY'.
             10   FILLER              PIC X(09)    VALUE 'TUESDAY'.
             10   FILLER              PIC X(09)    VALUE 'WEDNESDAY'.
             10   FILLER              PIC X(09)    VALUE 'THURSDAY'.
             10   FILLER              PIC X(09)    VALUE 'FRIDAY'.
             10   FILLER              PIC X(09)    VALUE 'SATURDAY'.
```

If the VALUE is the SAME for all elements at the elementary level, then the '85
Standard added support for it.  If they differ, then you need the 2002 feature
that Chuck posted.  What the '85 Standard supports is:

 01  Group-1  occurs 10 times.
     05  Group-2  occurs 5 times.
           10 Alphanum  Pic  X(10)  Value Spaces.
           10  Alpha2   Pic   X(10) Value All "A".
           10  Num1    Pic S99V99   Value Zero.
                     etc ....
```

    VALUE IS / VALUES ARE literal-1 FROM subscript-1 [TO subscript-2],
                                                 literal-2 FROM ...

I can't answer whether any given ANSI-85 COBOL implementation has provided
this feature as an extension, though.
