```
I am writing a simple little program to ask users to answer some
questions (via a screen...there are about 9 input fields). ANyway, I
need to keep track of which users have answered the questions as I don't
want someone to answer more than once per month and actually I'm only
going to allow them to answer for about the first two weeks of the
month.

I'm using DG-UX (unix system from Data General) and MF COBOL. What is
the most efficient way to find out the user name.

Currently, I call "system" using the command

who am i | cat > unique-file-name

I then open the file, read the file, save the users name, and delete the
file and continue on in my program. Although this will work, it doesn't
seem like it is the most efficient way of doing it. I had high hopes
w/ACCEPT USER NAME, unfortunately, that just gives me a user ID number,
which isn't good enough for my purposes. I will be using the system names
to send mail to the users, also.

Any suggestions?? I feel like there must be something more obvious, but
it's escaping both me and my boss. :(
```

```
The only other way I know is given by example:
identification division.
program-id. tst is initial.
environment division.

file-control.
select ss-file assign external SS
organization line sequential.
data division.
file section.
fd ss-file.
01 ss-rec pic x(80).

working-storage section.
01 xx pic x(5) value "DD_SS".
01 ab pic x(80) value "

procedure division.
display xx upon environment-name
display ab upon environment-value
open input ss-file
read ss-file into aa
close ss-file
exit program.


You have to open the file to get the command to execute.
(you can close the file, update enviroment-value then reopen it
to execute another command).
```

```
SELECT whoami ASSIGN EXTERNAL whoami
ORGANISATION LINE-SEQUENTIAL.

DATA DIVISION.
FILE SECTION.
FD whoami.
01 user-name PIC X(80).

PROCEDURE DIVISION.
DISPLAY "DD_WHOAMI" UPON ENVIRONMENT-NAME
DISPLAY "lpr" as a name and writing to a file opened OUTPUT.
```
