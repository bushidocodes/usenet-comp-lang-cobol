I believe I have seen that example. Although a good
general example, it does not really hilight
the *differences* between GN and GNP that I can
tell.

From what I gather so far, GNP does *not* set
"parentage", but is bounded by it. 
IOW, there *is* a secondary "pointer",
called "parentage" (at least informally). GN and
GU can set (move) the parentage pointer, but GNP
can *not*. It can only react to it. The parentage
pointer acts as a "ceiling" where GNP cannot
go above or around. GNP is limited to the
branches below the parentage pointer and
can not get "out" of that branch unless
the parentage is changed by a command that
can change parentage.

If this depiction is not accurate, please
let me know.

but, is it limited to one level below the
parent, or *any* level below the
parent (on the same branch)?
