James is right that an example would be helpful, but the short answer is
there are several ways to do this.

The most likely reason you're getting an RT-173 is that you're calling a C
function within a DLL but you haven't told the COBOL program which DLL
they're in.

For example, say we have a COBOL main program 'cbl1.cbl' like this:

     01 a pic x(10).
     procedure division.
     move "1234567890" to a
     display "a = " a
     call "csub1" using a
     display "a = " a
     stop run.

and this is calling a C function 'csub1', defined in the file 'csub.c':

#include <stdio.h>

void
csub1(char *a)
{
    strcpy(a, "ABCDEFGHIJ");
}

1. The simplest thing would be to link them into one exe, e.g:

    cobol cbl1.cbl obj;
    cl /c csub.c
    cbllink cbl1.obj csub.obj

- this will give us 'cbl1.exe' which will contain the COBOL and C routines.

2. If you want the C routines separately you need to give the COBOL some way
of knowing where the entry point 'csub1' is. If you just have one C routine
you can link it as a DLL of the same name, then COBOL will find it; however
it's more likely you've got a load of routines linked into one DLL. In order
that the COBOL program can find these you can load the DLL first, as follows:

    add the definition:

        01 mypointer usage procedure-pointer.

    to the working-storage section, then add the line:

        set mypointer to entry "csub.dll

    in the procedure division. This will load the DLL, and any entry points
in this DLL (e.g. 'csub1') will now be available. It's probably a good idea
to put in a test immediately after this line; if the pointer comes back bad
then it didn't find (or failed to load) the DLL.

3. Anther way of telling the COBOL program where to find the entry points is
to create an import library. For example:

    cl /c csub.obj
    cbllink /d/k csub.obj

    - this will create 'csub.dll', but it will also create 'csub.lib'.

    cobol cbl1.cbl obj;
    cbllink cbl1.obj csub.lib

    - this creates cbl1.exe, and the import library 'csub.lib' tells the exe
where to find the C entry point.

Hope this helps,


Dan.
