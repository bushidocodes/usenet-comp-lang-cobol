the S9(09) COMP field to the PIC 9(05) DISPLAY NUMERIC field.

First we need to know which COBOL you are using and on what
platform.

This is needed as in the COBOL that I use, COMP-3 is a
packed decimal field that is not used on too many platforms
other than an IBM mainframe (e.g., WANG VS systems use it,
but Intel does not).

But let me assume that you are talking about an IBM
mainframe system.

Packed decimal (COMP-3) is represented as follows:

Assume that you have an area in your record that looks like
this: nnnnns  

where n= a digit 0-9 and "s" is the sign. The sign in this
case is x'F' for positive (actually absolute), x'C' is
positive, x'D' is negative.

Now the field above could have been defined in a COBOL
program as:

 PIC 9(5) COMP-3 
 PIC 9V9999 COMP-3
 PIC S999V99 COMP-3

etc.

So, knowing what platform (architecture) you are starting
with and the COBOL you are using would make showing you
exactly what is going on much easier.

You don't specify the platform on which the files/table are held, but
presuming that COMP-3 is IBM's packed-decimal format, then the
following may help.

COMP-3 numbers are held in a compressed format such that there are two
digits per byte with the exception of the rightmost half byte, which
holds a sign instead.  In hexadecimal notation an example such as +123
in a two byte filed would be seen as follows '123F' or '123C', where
the 'F' and 'C' both represent the positive sign.  -789 would be
represented as '789D', where the 'D' represents the negative sign. 
The other hexadecimal letters A, B and E are rarely encountered as
signs, since they aren't automatically generated by any of IBM's
instructions, but would translate as either positive or negative
signs, though I can't remember which.  The easiest way to do the
conversion is to use a utility or write a program and change them all
to display format with separate signs.
