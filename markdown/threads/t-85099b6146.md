```
Mike,
I do not claim to be an incredibly intelligent COBOL person. However, I
have been one of the "standards auditors" you referred to in your post.
I have never really liked GOTO statements. The problem really is a
readability issue. If you are in a code shop full of very experienced
programmers then GOTO statements are fine. You are dealing with people who
can follow code easily. However, if you are in a shop with new
programmers, then GOTO statements can be a nightmare. A less experienced
programmer will be following the code logic and suddenly "Boom" you hit a
GOTO statement and are in a completely different part of the programming.
My experience has been that less experienced people have real trouble
following a program with a lot of GOTO statements in it.
```
