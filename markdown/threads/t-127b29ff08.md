and indicates to me you expect the data to be a zoned decimal signed
field, probably EBCDIC. Define it something like this (you could do
evaluate too):   

   05 Quantity-Purchased  Pic S9(5). 

   if Quantity-Purchased not numeric
       perform qty-is-no-good
   else
   if Quantity-Purchased = zero 
       perform qty-is-zero
   else
   if Quantity-Purchased > zero 
       perform qty-is-positive
   else
       perform qty-is-negative.

```
Just in case this is still of interest to anyone,  It is an "easy to miss"
part of COBOL that two different statements (related to zeroes) are
equivalent,

Consider

   05 NumField  Pic S999.
 * this can have any numeric picture and usage

Then you may code EITHER

   If NumField = Zero

       or

  If NumField Zero


The first is an "equality check" (relational expression) against the
figurative constant "zero" (which may be spelled zeroes or zeros as well)

The second is a "sign test" (like IF POSITIVE/NEGATIVE) and the zero may NOT
be spelled zeroes or zeros - although IBM and some other vendors allow this
as an extension).

For testing a numeric field against zero, the results are identical.
```

cause a 'negative' zero), it is poorly named.  It is really a NON-ZERO
compare

which is >, item 1or item 2.
