If you can't use COBOL SORT, use an indexed file instead.  Open it output,
close it, open it I-O and write to it instead of sort RELEASE statements.
Close the file, open it for input, and start it with a low-value for the
key.  Instead of sort RETURN statements, do READ ... NEXT statements
instead.

You would define the file as DYNAMIC.  If there is any possibility of
duplicate records, add a sequence field as the last field in the key.  When
you are shutting down the program, after closing your indexed file use a
DELETE FILE statement on it.
