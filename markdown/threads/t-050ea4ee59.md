Carlos lages wrote:

> I my  research, I think Power Doesnt Interpret the ESC (alt+27) command.
> 
> The worst thing is that I get no Erros, simply nothing happens.

The ESC sequence is probably upsetting the Windows print driver.  You are 
not supposed to send control sequences through the driver, you are supposed 
to control it with the Windows API (or have Cobol do this for you).

When you use TYPE this is a DOS program and Windows just passes whatever 
you send through without doing anything at all, thus the ESC sequences have 
no effect on Windows driver and work on the printer.

Try setting up a 'generic text only' Windows printer and send the sequences 
through that.

Do you get a document created in the print queue ?
