various cryptographic routines requires bit handling. This is
supported in the most modern COBOL compilers and the
next standard (whenever it comes).

But even in absolutely portable COBOL-74 code, bit handling
is easy.

Below is (from http://www.etk.com/download/etkpak/etkpak.htm#BITPK)
code that shows how to do that:


Summary: Provides bit manipulations of up to 32 bits. Can get, set, clear,
test, and pack bits.

How to use: Instead of non-portable redefinitions to access bits packed in
numeric variables, use BITPK's get operation to convert a number into a
string of 32 characters each with values "0" or "1". You can then easily
test or otherwise manipulate these character values. The pack operation
packs the character values back into a number.

Limitations: At most 32 bits can be handled at a time.

To call BITPK, define BITPK-CONTROL as follows:

              01  BITPK-CONTROL.
                  02  BITPK-TASK-ID           PIC X(4).
                  02  BITPK-OPERATION         PIC X(1).
                  02  BITPK-FEEDBACK          PIC X(1).
                  02  BITPK-NUMBER            PIC 9(10) COMP.
                  02  BITPK-BITS.
                      03  BITPK-BIT           PIC X(1)  OCCURS 32 TIMES.

Define CALL-BITPK as follows:
              CALL-BITPK.
                  CALL "BITPK"
                        USING BITPK-CONTROL
              .

BITPK supports the following operations:

G (get)
Get the bits of BITPK-NUMBER interpreted as an unsigned 32-bit number. If
the number is larger than 32 bits a (R)ANGE feedback is returned and the bit
string is all zeroes.

P (pack)
The reverse operation of get. Builds the 32-bit number having the bit string
given in BITPK-BITS.

S (set)
For each bit of the bit string given that is a one, set the corresponding
bit in BITPK-NUMBER. This is the binary OR operation.

A (and)
For each bit of the bit string given that is a one, set the corresponding
bit in BITPK-NUMBER if that bit is also a one, otherwise clear the bit. This
is the binary AND operation.

X (xor)
For each bit of the bit string given that is different from the
corresponding bit in BITPK-NUMBER, set that bit in BITPK-NUMBER, otherwise
clear the bit. This is the binary XOR operation.

C (clear)
For each bit of the bit string given that is a one, clear the corresponding
bit in BITPK-NUMBER.

1 (test for 1)
For each bit of the bit string given that is a one, test if the
corresponding bit in BITPK-NUMBER is a one. Return an (A)LL feedback if all
bits tested were ones, a (N)ONE feedback if none of the bits were ones, or a
(S)OME feedback if only some of them were.

0 (test for 0)
For each bit of the bit string given that is a one, test if the
corresponding bit in BITPK-NUMBER is a zero. Return an (A)LL feedback if all
bits tested were zeroes, a (N)ONE feedback if none of the bits were zeroes,
or a (S)OME feedback if only some of them were.

BITPK returns these feedbacks: a space means there were no errors. An "O"
means that the specified operation was invalid. An "R" means that the number
given was too large. An "A", "N", or "S" means that either All, None, or
Some bits met the condition.


 IDENTIFICATION DIVISION.
 PROGRAM-ID.    BITPK.

 AUTHOR.        LEIF SVALGAARD.
 DATE-WRITTEN.  96/12/06
     -REVISED:  97/01/22.

 ENVIRONMENT DIVISION.

 CONFIGURATION SECTION.
 SOURCE-COMPUTER. PORTABLE.
 OBJECT-COMPUTER. PORTABLE.

 DATA DIVISION.

 WORKING-STORAGE SECTION.

 01  BIT-HANDLING.
     02  BIT-VALUE               PIC 9(10)  COMP.
     02  BIT-TOP                 PIC 9(10)  COMP  VALUE ZEROES.
     02  THE-WEIGHT              PIC 9(10)  COMP.
     02  BIT-WEIGHTS.
         03  BIT-WEIGHT          PIC 9(10)  COMP  OCCURS 32 TIMES.

     02  BIT-COUNT               PIC S9(3)  COMP.
     02  BIT-MATCH               PIC S9(3)  COMP.
     02  BIT-NBR                 PIC S9(3)  COMP.

     02  BIT-WANTED              PIC X.
     02  BIT-STRING.
         03  THE-BIT             PIC X      OCCURS 32 TIMES.

 LINKAGE SECTION.

 01  BITPK-CONTROL.
     02  SESSION-TASK-ID         PIC X(4).
     02  BITPK-OPERATION         PIC X(1).
     02  BITPK-FEEDBACK          PIC X(1).

     02  BITPK-PATTERN           PIC 9(10)  COMP.
     02  BITPK-BITS.
         03  BITPK-BIT           PIC X      OCCURS 32 TIMES.

 PROCEDURE DIVISION
     USING BITPK-CONTROL.
 BEGIN-PROGRAM.
     IF BIT-TOP = ZEROES
         PERFORM INITIALIZE-BIT-WEIGHTS
     .
     PERFORM GET-AND-CHECK-THE-BITS
     MOVE SPACE TO BITPK-FEEDBACK
     IF BITPK-OPERATION = "G"
         MOVE BIT-STRING TO BITPK-BITS
     ELSE
     IF BITPK-OPERATION = "S" OR "O"
         PERFORM SET-THE-BITS
     ELSE
     IF BITPK-OPERATION = "C"
         PERFORM CLEAR-THE-BITS
     ELSE
     IF BITPK-OPERATION = "A"
         PERFORM AND-THE-BITS
     ELSE
     IF BITPK-OPERATION = "X"
         PERFORM XOR-THE-BITS
     ELSE
     IF BITPK-OPERATION = "1"
         PERFORM TEST-IF-ONES
     ELSE
     IF BITPK-OPERATION = "0"
         PERFORM TEST-IF-ZEROES
     ELSE
     IF BITPK-OPERATION = "P"
         PERFORM PACK-THE-BITS
     ELSE
         MOVE "OPERR" TO BITPK-FEEDBACK
     .
 RETURN-TO-CALLER.
     EXIT PROGRAM
     .

 INITIALIZE-BIT-WEIGHTS.
     MOVE 1 TO THE-WEIGHT
     PERFORM SET-BIT-WEIGHT
       VARYING BIT-NBR FROM 32 BY -1
         UNTIL BIT-NBR < 1

     MOVE THE-WEIGHT TO BIT-TOP
     .

 SET-BIT-WEIGHT.
     MOVE THE-WEIGHT TO BIT-WEIGHT (BIT-NBR)
     ADD  THE-WEIGHT TO THE-WEIGHT
     .

 GET-AND-CHECK-THE-BITS.
     MOVE BITPK-PATTERN TO BIT-VALUE
     IF BIT-VALUE < BIT-TOP
         MOVE ZEROES TO BIT-STRING
         PERFORM GET-THE-BIT
           VARYING BIT-NBR FROM 1 BY 1
             UNTIL BIT-NBR > 32

         PERFORM CHECK-THE-BIT
           VARYING BIT-NBR FROM 1 BY 1
             UNTIL BIT-NBR > 32
     ELSE
         MOVE "RANGE" TO BITPK-FEEDBACK
         MOVE  ZEROES TO BIT-STRING
     .

 GET-THE-BIT.
     IF BIT-VALUE NOT < BIT-WEIGHT (BIT-NBR)
         SUBTRACT BIT-WEIGHT (BIT-NBR) FROM BIT-VALUE
         MOVE "1" TO THE-BIT (BIT-NBR)
     .

 CHECK-THE-BIT.
     IF BITPK-BIT (BIT-NBR) = ZERO
         NEXT SENTENCE
     ELSE
     IF BITPK-BIT (BIT-NBR) = SPACE
         MOVE ZERO TO BITPK-BIT (BIT-NBR)
     ELSE
     IF BITPK-BIT (BIT-NBR) NOT = "1"
         MOVE "1"  TO BITPK-BIT (BIT-NBR)
     .

 SET-THE-BITS.
     IF BITPK-FEEDBACK = SPACE
         PERFORM SET-THE-BIT
           VARYING BIT-NBR FROM 1 BY 1
             UNTIL BIT-NBR > 32
     .

 SET-THE-BIT.
     IF BITPK-BIT (BIT-NBR) = "1"
         IF THE-BIT (BIT-NBR) = "0"
             MOVE "1" TO THE-BIT (BIT-NBR)
             ADD BIT-WEIGHT (BIT-NBR) TO BITPK-PATTERN
     .

 CLEAR-THE-BITS.
     IF BITPK-FEEDBACK = SPACE
         PERFORM CLEAR-THE-BIT
           VARYING BIT-NBR FROM 1 BY 1
             UNTIL BIT-NBR > 32
     .

 CLEAR-THE-BIT.
     IF BITPK-BIT (BIT-NBR) = "1"
         IF THE-BIT (BIT-NBR) = "1"
             MOVE "0" TO THE-BIT (BIT-NBR)
             SUBTRACT BIT-WEIGHT (BIT-NBR) FROM BITPK-PATTERN
     .

 AND-THE-BITS.
     IF BITPK-FEEDBACK = SPACE
         PERFORM AND-THE-BIT
           VARYING BIT-NBR FROM 1 BY 1
             UNTIL BIT-NBR > 32
     .

 AND-THE-BIT.
     IF THE-BIT (BIT-NBR) = "1"
         IF BITPK-BIT (BIT-NBR) NOT = "1"
             MOVE "0" TO THE-BIT (BIT-NBR)
             SUBTRACT BIT-WEIGHT (BIT-NBR) FROM BITPK-PATTERN
     .

 XOR-THE-BITS.
     IF BITPK-FEEDBACK = SPACE
         PERFORM XOR-THE-BIT
           VARYING BIT-NBR FROM 1 BY 1
             UNTIL BIT-NBR > 32
     .

 XOR-THE-BIT.
     IF BITPK-BIT (BIT-NBR) = THE-BIT (BIT-NBR)
         IF THE-BIT (BIT-NBR) = "1"
             MOVE "0" TO THE-BIT (BIT-NBR)
             SUBTRACT BIT-WEIGHT (BIT-NBR) FROM BITPK-PATTERN
         ELSE
             NEXT SENTENCE
     ELSE
         IF THE-BIT (BIT-NBR) = "0"
             MOVE "1" TO THE-BIT (BIT-NBR)
             ADD      BIT-WEIGHT (BIT-NBR)  TO  BITPK-PATTERN
         ELSE
             NEXT SENTENCE
     .

 TEST-IF-ONES.
     IF BITPK-FEEDBACK = SPACE
         MOVE "1" TO BIT-WANTED
         PERFORM TEST-THE-BITS
     .

 TEST-THE-BITS.
     MOVE ZEROES TO BIT-COUNT, BIT-MATCH
     PERFORM TEST-IF-BIT-IS-WANTED
       VARYING BIT-NBR FROM 1 BY 1
         UNTIL BIT-NBR > 32

     IF BIT-COUNT = BIT-MATCH
         MOVE "ALL"  TO BITPK-FEEDBACK
     ELSE
     IF BIT-MATCH = ZEROES
         MOVE "NONE" TO BITPK-FEEDBACK
     ELSE
         MOVE "SOME" TO BITPK-FEEDBACK
     .

 TEST-IF-BIT-IS-WANTED.
     IF BITPK-BIT (BIT-NBR) = "1"
         ADD 1 TO BIT-COUNT
         IF THE-BIT (BIT-NBR) = BIT-WANTED
             ADD 1 TO BIT-MATCH
     .

 TEST-IF-ZEROES.
     IF BITPK-FEEDBACK = SPACE
         MOVE "0" TO BIT-WANTED
         PERFORM TEST-THE-BITS
     .

 PACK-THE-BITS.
     MOVE ZEROES TO BITPK-PATTERN
     PERFORM PACK-THE-BIT
       VARYING BIT-NBR FROM 1 BY 1
         UNTIL BIT-NBR > 32
     .

 PACK-THE-BIT.
     IF BITPK-BIT (BIT-NBR) = "1"
         ADD BIT-WEIGHT (BIT-NBR) TO BITPK-PATTERN
     .

