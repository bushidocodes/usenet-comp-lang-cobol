```
The REDEFINES clause doesn't automatically perform a data conversion.  Whatever
underlying hex values are contained in SAL-SALARY, those same values will be
there if you reference SALARY-NUMERIC.  The two names refer to exactly the same
location in memory.

If you move alphanumeric values to SAL-SALARY and then reference
SALARY-NUMERIC, you don't suddenly have numeric hex values in there; it's still
the same (presumably) ASCII values, even though you re-defined the field as
numeric.  The only thing that WILL happen new if you reference SALARY-NUMERIC
after putting alpha in there is this.  If you try to perform an arithmetic
operation on SALARY-NUMERIC while alpha is in there, you will get an abend.

RoadRaat
```
