I am trying to figure out what you're wanting.  What is "a cobol
datafile"?  To me, this is a file with data in it.  It could have been
created by anything but COBOL programs can access it.  Since COBOL has
no delimiters, there's no way to determine where a field starts or ends
except by asking whoever wrote that file.

Now if you mean it reads a cobol program to determine the structure -
then I would print the COBOL program.  COBOL displays the structure
quite nicely.

Hi,

From whatever I have worked on, if u have 'file description' as copybook for
the data file and the data file, you can get a formatted output (Tabular with
headers).
This can be done by means of FILE-AID tool in mainframe.
And in PC, I know 'Data file editor' in MF Workbench can do it. But I have not
worked much on it.
Eg.
If you have 'master file' data file and you have 'master' as copybook
containing the file description (declaration), then these two are sufficient
in generating a formatted output using FILEAID.
Note: You can print data files in formatted mode (as above), Character mode
and Hex mode.
