Hi all,

the MicroFocus-runtime is linked with a bunch of c-programs;
C is main.

the situation:
The C-program cobcall's a cobol-prog.
the cobol-prog is using a c-functions.
in the c-function an error occurs and processing of the called 
cobol-prog must be terminated - *within the c-function*.
A cobcancel causes error "177 Attempt to cancel active program".

What can be done to "de-activate" the cobol-program so that it can be 
cancelled?

Thanks for reading
Wolf
```

Where will the C function return to? Its return address is inside the cobol
program it just cancelled. 

The clean way is return an error to the cobol program and have it do a goback. 

A dirty way is take a pointer to a local variable and walk the stack until you
find the cobol program's return address, memcpy it on top of your own and do a
return. If you're lucky, the cobol runtime will clean up the stack; if not,
you've created a stack leak.
```

I understand this to mean:

     +-------+
     |c-main |      +----------------+
     |call x ------>|Cobol program x |     +--------------+
     |??     |      |call y -------------> | C function y |
     +-------+      +----------------+     | cancel x ?   |
                                           +--------------+

If you had managed to cancel the cobol program then where were you
planning for the C function to go to next ?  Hint: a return() goes to
a newly dug black hole.
 
> What can be done to "de-activate" the cobol-program so that it can be 
> cancelled?

The only way is to have the C function return a value to the Cobol and
the Cobol then return to the C main so that the C main can cancel it.
