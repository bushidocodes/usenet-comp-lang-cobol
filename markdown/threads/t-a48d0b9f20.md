What you are "missing" is the fact that (basically) all 01-levels in Linkage
Section are (implicitly) just "pointers".  They have no storage allocated to
them.  Therefore, the COBOL run-time "matches" linkage section 01-levels
(via the ORDER in the Procedure Division HEADER, not the order in the
Linkage Section) with "passed" parameters.

In the case of parameters passed BY VALUE - the "address" is (depending on
the OS architecture) to "the Stack"

In the case of parameters passed BY CONTENT - the address is to a
"temporary" copy of the original item

In the case of parameters passed BY REFERENCE - the address is to the start
of the "original" storage of the passed item.

  ***

Does this help?

In COBOL, when you describe the parameters passed to a program you 
indicate what their treatment is (reference, value etc)


calling program

call "xyz" by reference doodad by value thing


called program

linkage section

01  doodad-link.
    02  filler  pic x(100).
01  thing-link  pic s9(9) comp-5.

procedure division using by reference doodad-link by value thing-link.

The "by reference" and "by value" must match in the call and the 
called using statements.

In C

xyz(&doodad, thing)

void xyz(char *doodad_link, long thing_link)

Hi,

When passing data "BY REFERENCE" the CALLer and the CALLee reference the same
physical piece of main storage for that variable. The implication of this is: if
the CALLee changes the content of the passed data field, the CALLer's subsequent
use of that data, upon return, will reflect that change.

When using "BY REFERENCE", speculation about whether the address or the data is
passed is irrelevant, since reference to the dataname in the CALLee's Linkage
Section provides access to the CALLer's data (the address is ALWAYS used to
access the data). The Linkage Section, in fact, provides the CALLee with
addressibility to the CALLer's data. But that's more than one needs to know to
use the passed data.      

HTH, Jack
