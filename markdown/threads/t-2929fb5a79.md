Are you using Object Cobol or NetExpress? I've had more problems with the

executes successfully, there is always a page fault when terminating the VB

only way I've found to avoid the page fault is to statically link all Cobol
code into one DLL.

Just an FYI: Watch out passing complex data types between VB and Cobol.
Consider this example

type VBType
someid as string * 3
devicecontext as long
end type

01 Parms.
03 someid pic x(3).
03 devicecontext pic x(4) comp-5.

The two structures are equivalent but the Cobol code will not address the
devicecontext member correctly unless a byte of filler is inserted to make
it fall on a 4 byte boundary.
