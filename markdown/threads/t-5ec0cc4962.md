A common implementation style for pseudo-conversational uses the 'NEXT
TRANID' and a COMMAREA. CICS sequesters the COMMAREA when you pass
control back to the typist. When the typist returns to your tran, the
COMMAREA is made available to your program. The COMMAREA could be all
of W-S but that is usually a lot more than you need.

In order to design a pseudo conversational system
you have to

1 Split your application to transactions - each transaction
should be a logical unit of work.
2 The transactions can pass data to each other via Temporary storage
Commarea etc.
3 The cics Initializes the working storage when the task begins.
If you need to keep some of it to a next session tou can save it in
Temp. storage.

Bjorn,

On the the mainframe, atleast, COBOL can not be assumed to  have a single
contiguous Working Storage.  To answer your question literally, we would have
to get into some routine that accessed the locations which hold a set of Base
Locators for numerous areas that hold working storage items. And that would be
very unworkable anyway.

Generally, each 01 (or 77) must be presumed to be placed in it's own unique
place! (although not really, its just that you can't assume any relationship of
proximity or sequence; especially in a virual environment).

CICS actually will copy certain work areas around dynamically too (particularly
if there are any mixtures of data above the line and below the line).

So I think you are going to need to deal in address lists, that being lists of
01 groups passed in USING clauses, Or else one by one  via CICS INTO() and
FROM() keywords.

Working storage is not a singularity. It is not contiguous. If you can break
your requirements down to more specific requests for help I am sure folks here
in the COBOL NG will be happy to interact with you.

Best Wishes

You can NOT pass more than 32K - 4bytes via
EXEC  CICS
            XCTL/RETURN [TRANSID]
          COMMAREA(some-name)
END-EXEC.

Accessing your data (WS) ina COBOL CICS pgm is just like a NON-CICS pgm.

If you need to 'retain' more than 32k - 4bytes, then you must use Transient
Data or
Temporary Storage (best).  You 'WRITEQ TS' and 'READQ TS' whenever you are
Going to XCTL, RETURN, or LINK.
