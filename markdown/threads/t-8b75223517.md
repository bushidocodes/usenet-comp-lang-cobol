Depends on how you defined the index file - if YOU defined the indexing.

If you let the COBOL compiler handle it (i.e., ORGANIZATION IS INDEXED
RECORD KEY IS whatever), then the structure of the index file is determined
by the compiler publisher and the answer is almost surely, "No."  Some
compilers create a separate physical file for the index, some include the
index info in the data file.

Regardless, in general the only way to read data from a COBOL indexed file
is to use a COBOL program created using the same compiler as was used to
create the software which created the data.
