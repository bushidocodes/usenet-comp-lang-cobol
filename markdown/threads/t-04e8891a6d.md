```
I work for a small software company that developes software using MF Cobol
(UNIX), Powerbuilder, Oracle, etc. I have been asked to R&D the possible
migration of our products to NT. I am hoping someone out there can give me a
hand or lend advice.

My primary concern is with our scripting language. Currently our cobol code
runs via Korn shell scripts. Since NT currently has no scripting language of
its own (I read somewhere that there is Visual Basic-like scripting language
planned for the near future) I need to decide on the best replacement. I have
looked at Mortice Kern's Korn shell for Windows and NT, and it is a valid
option, but we hate to add another third party to our product.

For reasons of maintenance, I would like to keep the UNIX vs NT scripts as
close in nature as possible.... identical preferred.

Consequently I am looking at Perl as an option. Can I run MF Cobol from a Perl
script? Has anyone done it, and if so, can you toss me a crumb?

Has anyone else migrated Cobol from UNIX to NT? Is there an easier option I am
missing? Those of you running MF in NT.... what are you using for scripting?

Any help or insight would be appreciated. Thanks in advance for your help.

Kevin
```
