```
1) your reference to "VS COBOL 2 Ver 2.1" has been confused and frightened.
If you really mean "VS COBOL II - Release 2" - that product has been out of
support since about 1987 - and never had a R2.1.  If, on the other hand, you
really mean IBM COBOL for OS/390 & VM R2.1 - then you are in luck and this
is pretty easy.

2) All "modern" IBM COBOLs (VS COBOL II - and the newer IBM COBOL for
this-and-that) have a relatively easy facility for "restricting" the use of
certain verbs and constructs.

Check out,
 http://www.s390.ibm.com/bookmgr-cgi/bookmgr.cmd/BOOKS/IGYIC201/1%2e5%2e3

For the web-less, this is:
 1.5.3 Creating or Modifying Additional Reserved Word Tables
in
 Title: COBOL OS/390 & VM V2R1 OS/390 Installation
 Document Number: GC26-9045-01

NOTE WELL:  One "problem" with this facility is that IBM provides little or
no assistance in "upgrading" a customized reserved word table from release
to release of their products.

NOTE2:  This facility lets you EITHER create an "error" message when one of
these verbs is used - or a "warning" message.  It also lets you create a
table that can be OPTIONALLY used by programmers at compile time (via the
WORD compiler option) - or you can set this as a non-overridable table so
programmers MUST use your choice of words.
```
