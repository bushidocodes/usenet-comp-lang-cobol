The EXIT paragraph is also used for dummy paragraphs which are performed
to meet UNTIL conditions. 

     PERFORM 1000-DUMMY 
         VARYING MY-INDEX FROM 1 BY 1
            UNTIL MY-INDEX > MY-INDEX-MAX
               OR MY-VALUE (MY-INDEX) = SPACES.
1000-DUMMY.
     EXIT.

I have mainly used older compilers, but I was thinking that an in-line
PERFORM would be cleaner here, except that I can't use the No-op command
EXIT.  What do you think?
```

This is a non-objectionable use of EXIT. You can make it better by
using a more descriptive name that makes it clear what is going on:
PERFORM LOOK-FOR-FIRST-SPACE
   VARYING MY-INDEX ...
               OR MY-VALUE (MY-INDEX) = SPACES

This nice and free comment is why actually the above construction
is better than an inline perform. This would be even more the case
if the condition were more complicated.

No, the dummy paragraph should have a name like DUMMY-PARAGRAPH.  If you
name the dummy paragraph then I have to go look at LOOK-FOR-FIRST-SPACE to
find out it does nothing, and the paragraph can't be resued for other
purposes without confusion.  Do you really want to come upon code like
this:
PERFORM LOOK-FOR-FIRST-SPACE
   VARYING MY-INDEX FROM END-POS BY -1
               UNTIL MY-VALUE (MY-INDEX) = "."

In COBOL-85 the correct form is like this:

PERFORM VARYING MY-INDEX FROM END-POS BY -1
               UNTIL MY-VALUE (MY-INDEX) = "."
    CONTINUE
END-PEFORM

It's too bad that the spec requires the CONTINUE -- the meaning is clear
without it.
```
