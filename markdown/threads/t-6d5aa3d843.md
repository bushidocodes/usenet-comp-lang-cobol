Using special names as described in Chapter 11 of the users guide (V6.1 - it
might be a different chapter in earlier releases -- look for Accept/Display) you
can get the value of different environment varibales.  The execution arguments
won't return the directory it is executed from.  The Current directory can be
obtained using a CBL_ call.

MOVE 255 TO WS-LEN.       [WS-LEN S9(9) COMP-5]

CALL 'GetCurrentDirectoryA' WITH STDCALL USING
        BY VALUE WS-LEN
         BY REFERENCE THE-PATH-NAME

The 'current' directory is not the same as the one the program was
executed from.  The 'command line' is also a red-herring.  In C the full
path of the executed program is often in argv[0] and the remaining
argv[]'s contain the command parameters.

Jerry P wrote:
> Easy:
> 
>  MOVE 'COMMANDLINE' OF POW-SELF TO COM-LINE.

That, presumably is _Power_Cobol only, given away by the 'POW-SELF'. 
This is unlikely to be of use in a 'Fujitsu Cobol' program.  I would
have expected PowerCobol to be specified it was being used.

Fujitsu version 3 (reading the manual here) does allow SPECIAL-NAMES of
ARGUMENT-NUMBER and ARGUMENT-VALUE.  These can be used in
ACCEPT/DISPLAY, though I don't know what an argument-number of zero will
return.

       WORKING-STORAGE.

       01  Run-Parameter            PIC X(80).
       01  argc                     PIC 9(4).

       PROCEDURE DIVISION.

           ACCEPT argc           FROM ARGUMENT-NUMBER
           DISPLAY ZERO          UPON ARGUMENT-NUMBER
           ACCEPT Run-Parameter  FROM ARGUMENT-VALUE

You will need to play with this code to see what, in fact, it does
return.  It may be that the first is number 1 and this is not the
executable.

There is also ENVIRONMENT-NAME and ENVIRONMENT-VALUE, I think that Win95
or so puts the command line used into COMMAND= or similar and this may
be recoverable from there.
