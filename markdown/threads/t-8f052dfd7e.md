I have written what I call COMPONNT which when given JOBSCAN output and a
PDS name containing the COBOL source code to analyze, it will report what I
call the "componnents" of the application in question. It does not expand
copyboooks, but I have written a program to do this, I just have not
integrated this into this program.

The output generated by this program can then be dirctly inported into an
ACCESS database as a comma-separated test file. I have ported all of this
over to the PC so using REGINA-REXX, FTP (included with WIN95), and
Microsoft ACCESS this information can be extracted on the PC leaving only
the JOBSCAN information to be run on the mainframe. The FTP part is used to
download the JOBSCAN info and the COBOL PDS from within ACCESS by shelling
out a DOS .BAT file.

It generates JCL dispositions for DD's, COBOL READ/WRITE/I-O info (by DD
name), DB2 table I-O info,  CICS transactions used, and sub programs called.

Is this what you are looking for?
