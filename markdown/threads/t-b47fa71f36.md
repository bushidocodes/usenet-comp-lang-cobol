Hi Folks,

Up until now, I have used the following SCREEN SECTION command structure to
enable the user to edit a variable on-screen :

LINE nn COL nn
PIC X(n) FROM Variable-1 TO Variable-1 /
Variable-2.

I have recently come across the alternative :

LINE nn COL nn
PIC X(n) USING Variable-1.

I would appreciate it if somebody could enlighten me as to the differences
between the two, apart from the fact that the former allows for the use of a
different variable from the source and thus would appear more flexible. Does
the latter option offer any advantages other than fewer key strokes when
coding?

T.A.L.I.A.

Ian S.

An input field is a screen item whose description contains a TO phrase.

An output field is a screen item whose description contains a FROM
phrase.

An update field is a screen item whose description contains a USING
phrase.

You should code the phrase that corresponds to your intent:

PIC X(n) TO variable-1
PIC X(n) FROM variable-2
PIC X(n) USING variable-3
PIC X(n) FROM variable-4 TO variable-5

It happens that FROM variable-6 TO variable-6 is equivalent to USING
variable-6. The circumlocution should be avoided, not merely because it
is longer but because you intend an update field yet do not include the
USING phrase to signal your intent to the reader.

specify.
