Hi,

   In my company we've created a mirco focus app. We compiled it 
and put an exe on a central server. All of our workstations are 
running the same exe on the server. 

   What has happened is that, on some workstations, we are 
getting problems with files being locked (error 65) even though 
those files are opened as multi user (i-o). All of the files have 
the "lock is automatic" set.

   This doesn't happen on all workstaions. What we've found is 
that, on the workstations where this DOESN'T happen, the 
microfocus development software has been installed. We believe 
that this is the problem.

   We can't install the microfocus development software on each 
workstation. We've contacted Microfocus and they are not much 
help. Anyone have any suggestions? Are there any dlls we need or 
something?

Thanks,

Hi,

As Micro Focus weren't much help, I'll try...No prizes for anybody spotting
the contradiction <g>

Check whether the machines that have the development software installed have
configuration files set up such as EXTFH.CFG, COBOPT.CFG or configuration
files pointed to by the EXTFH or COBCONFIG environment variables. If they
do, they may contain settings which affect the file locking modes in use,
which could explain the locked status you were seeing.

If that doesn't solve the problem I'm sure that if you contact them again,
the Support people in Micro Focus will do their best to help you. As I'm
sure you can appreciate, if it isn't an obvious mistake it may not have an
obvious answer.

unlocked.
