### "BY CONTENT" Storage Area Question

- **From:** "jco..." <ua-author-11185940@usenetarchives.gap>
- **Date:** 1995-07-04T20:00:01+00:00
- **Newsgroups:** comp.lang.cobol
- **Message-ID:** `<3td0ef$1fo@cdn_mail.telecom.com.au>`

```
Have been investigating the code inserted by the COBOL II compiler
when BY CONTENT is coded on a CALL statement. Noticed that when the
data structure is 65008 bytes or less, the compiler uses an area within
the program for the data area to be used in the CALL. When the data
structure is 65009 bytes or greater, subroutine IGZCSPC is called to
setup the data area, and is then called again after the CALL (to free
this area -- I assume).

Does anyone have any information regarding this? What are the parameters
used by the compiler in determining whether to use an area within the
program, or to obtain temporary storage? The number 65008 seems a strange
value to use as the threshold/limit.

JohnC
"May the Farce be with you" -- Oldie One Gone Mouldie
```
