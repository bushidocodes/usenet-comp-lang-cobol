```
I have an accounting system written in MicroFocus COBOL. Unfortunately
the consulting company that wrote it went out of business and the
source code is 'lost somewhere'. I DO have the executables and the
data files. What I'm going to try to do is use a data format translator
to convert from MF COBOL to Microsoft Access.

What's the easiest way to tell from the executable file what data file
structure a program is expecting. As an example, in one of the executables
starting around byte number 512 or so, there is what appears to be a
list of field names and other associated data. Does this 'other
associated data' include the data type and/or length?

To be honest, I know what would qualify as nothing about COBOL. I do
however know my way around PC's in general, so feel free to get down-and-
dirty with the answer if you have to. I can edit a file byte-by-byte
if needed.

If you could reply with any information or pointers thereto, it would
be greatly appreciated.

Ryan McGuire
```

```
I can't see that looking at the executable file would be much help.
You need to get all the data out in an understandable form and then
convert it to your chosen format. The easiest way might be to use
the executable to output all the data into a printfile where you can
see (or should be able to) what each field represents, and then to
write a small program that reads the printfile and writes it out in
a suitable form. Very messy and time-consuming, but perhaps the most
straightforward.
```
