```
The newer releases of Micro Focus COBOL have introduced
an error file in /usr/tmp called MFIOERR.pid.  Is there
a way to turn off the generation of these files?

We have a program that on SCO 4.0 10.06 and HP 4.1 11.02
fails but AIX 4.0 10.07 works.  If we replace the SCO
10.06 fherror.o file with the 10.02 version, the program
also works.  When it fails it fails by crashing.  It is as
if the rts is being stepped on.

Ring any bells for anyone?

					Glen Johnson
```
