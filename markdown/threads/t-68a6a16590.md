I have some performance problems using MF OOCOBOL
and Embedded SQL Toolkit
for MS SQLServer 6.0, on NT 3.51.

I use Micro Focus Object COBOL 3.3 and, since my
programs access a SQLServer
database, Micro Focus Embedded SQL Toolkit for
Microsoft SQL Server 1.0.
I use SQL Server 6.0. Somewhere I read that the
toolkit was designed for
SQLServer 4.2 and higher: I don't know if this can
suggest any suspect.

I built a COBOL program on Micro Focus COBOL for
UNIX operating system,
with an INFORMIX database. This was a read-only
program. It took about 1'30''
to run and give a report.

I transferred the same program on NT, using the
same COBOL source code, the
same database structure and data (on SQL Server, of
course), but the result
was about 7'!

I put some measure points in the COBOL source code
and retrieved the time each
critical section took: the SELECT and the OPEN
CURSOR statements were the
bottlenecks. I tried with new clustered indexes, in
order to optimize the
access to those tables, but the result didn't
change. I tried to compile
in different ways my program (SQLACCESS"" vs.
NOSQLACCESS options and so on), and
the difference was not so evident.
My suspect is that the Embedded SQL Toolkit doesn't
work so well.

Does anybody have the same COBOL/SQL configuration?
Did anybody measure his/her COBOL/SQL program
performances?
