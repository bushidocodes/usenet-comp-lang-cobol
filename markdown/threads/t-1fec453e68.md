### Conversion from DOS to Windows 95

- **From:** "c. easton beymer" <ua-author-8441803@usenetarchives.gap>
- **Date:** 1996-10-02T20:00:01+00:00
- **Newsgroups:** comp.lang.cobol
- **Message-ID:** `<01bbb163$65fd5340$72165da6@psi.rmii.com>`

```


Hi COBOL Developers!

I have a fairly large application comprised of over 300 modules and
approximately a million lines of COBOL 85 code that runs in a DOS text mode
environment. This application uses overlapping windows and was written to
be "event oriented" in that input can be random, and even a mouse can be
used. This is a commercial product that I want to move into a native
Windows 95 / NT gui environment.

I am open to other languages and development environments. Has anyone out
there had a similar experience in moving an application to Windows 95? The
COBOL tools available seem to be somewhat lacking. Realia COBOL is
expensive with no object oriented capabilities and seems to be seriously
needing an upgrade. Microfocus's products seem to be lacking strong gui
development capabilities. Ryan McFarland (Liant) and Acucobol charge
runtime royalties which is totally unacceptable. IBM's VisualAge COBOL has
recently been updated to work under Window 95 / NT but I'm not sure of the
strength of their gui development capabilites because of a disclaimer about
their product's gui strength in their advertising literature.

I am wanting to move to a strong client/server environment possibly using
ODBC. The screen design capabilites have to be first class as the product
being developed will be a "world class" product. The compiler should
produce native code. We are currently using Btrieve as our file handler,
and would like to continue using this or some other data base handler that
is extremely fast and robust. The generated code must be royalty free.

Any feedback will be greatly appreciated. Thanks in advance.

C. Easton Beymer
```
