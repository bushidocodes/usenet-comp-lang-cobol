You did not state which version of MFocus you use, but I will assume MF40NT. You cannot
put run-time switches into your source code; otherwise, they wouldn't be run-time
switches. However, there are directives that you can include in your source, and they
will modify runtime behaviour. There doesn't seem to be one that maps directly to the
'+B' switch, but I suggest you try the LOCKTYPE directive. LOCKTYPE"1" is defined as
meaning that a program cannot access locked records at all (the default is read access).
I doubt this means that the records are skipped automatically, but maybe you can trap
the error and move on to the next record. Hope this helps.

Well, actually you can set runtime switches from within the source code.
Assuming that you have the 4.0 docs, there are two call by number routines
(X"91" functions 13 and 14)documentated on page 16-26 of the Programmers
Guide to Writing Programs that will read and set the run time switches.

The reading of the runtime switches will result in a
'pic x comp-x occurs 26' parameter being returned with the current values
of each of the currently set run time switches (26 being one for each
switch - A..Z). Each bit in the relevant byte will represent the value
of the switch being set, thus the value 32 in the third (of 26)
pic x comp-x's that has been returned means that switch C5 (50 line mode)
has been set.

In your case, setting the first (or second) bit of the second byte will
result in the B (or B1) switch being set which will result in the record
pointer skipping locked records.

Remember that the function 13 will set all of the runtime switches to
whatever values you specify so make sure that you don't change
any others after reading the settings if you are not sure what they do.
A complete list of the runtime switches can be found in the Object COBOL
User Guide - Vol 1, chapter 28.
