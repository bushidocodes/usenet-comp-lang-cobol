```
>     9(3)V9(3)

The above is about all numeric types, not just packed.  Pic 9(4) has 4
numeric digits, Pic 9(3)V9(3) has 6 numeric digits.  The V is a place
holder telling the compiler where to assume a decimal.

So if your fields contained    1234  and 123456, the computer would
treat them as 1234 and 123.456;

There are some other issues in how your system does signs (even with
unsigned numbers), and where it puts slack characters.  If you want help
with these let us know your operating system.  (Or whether the file was
created by PL/I)  Actually, technical questions should always list OS,
compiler, etc., for best responses.
```

```
The following is my view of the syntax in question.

What you have is a numeric field edited as follows.

9(4)V9 = 9999.9

9(3)V9 = 999.999

the "v" is an assumed decimal point and all digits after the "v" are the number
of decimals available.
```
