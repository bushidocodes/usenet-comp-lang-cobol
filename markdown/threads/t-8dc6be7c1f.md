```
Pardon me if this topic has been discussed recently; I have not
been reading the newsgroup closely for several weeks.

I just received my Fujitsu COBOL 6.0 upgrade, and on opening it was
shocked to read the following text in the cover letter:

  "We have introduced a new license control system with this release
  so you will need to send an email note (which can be generated
  automatically) to our license server during the installation
  process to obtain your site key.  You have 30 days in which to
  register your system after your first install. ..."

Well fellow COBOL'ers, that kills Fujitsu as a compiler vendor, as
far as I'm concerned.  I have yet to use Fujitsu COBOL for a single
application, but I have been maintaining my license for several
years, as a hedge against this practice by MERANT.  If this keeps
up, I will simply start writing my applications using another
language.  Even Microsoft doesn't put you in a situation where you
can no longer install their compiler products if Microsoft goes
out of business.  And I dare say, Microsoft is less likely to do so
than MERANT or Fujitsu's software company.  And compounded by the
fact that Fujitsu has joined MERANT in charging exorbitant prices,
I see the day rapidly approaching when I sadly must abandon COBOL,
which has been my primary development language for over 3 decades.
Sigh.
```

```
Judson,

In favor of Fujitsu, the one fact I've noticed missing in this entire thread
is the fact that those of us on "Maintenance" have gotten Version 6.0
"free".  The maintenance prices have not changed, so far as I can see, i.e.
$500 for Enterprise Edition, $250 for Professional Edition, etc.  So we'll
still stick with the product, at least for a while.

However, as a person that is directly involved in a Disaster Recovery
Service, Fujitsu needs to understand that:

a.  It's not a matter of "if" hard drives will fail, it's a matter of
"when".  So one would be hard pressed to "kill" the license key on a system
with a broken hard drive.

b.  A footprint of the system that the software was installed on rarely
would help if you keep your Original Software off-site, as all prudent
businesses do, and need to reload it at your Disaster Recovery Center.

c.  Most "Disasters" (keep in mind Disasters are not necessarily fires or
tornados, but for real hardware failures) are not declared during normal
business hours.  Therefore, it would be tough to call Fujitsu during their
"9am to 5pm" normal business hours.  Since I work for a Bank, the bulk of
the processing is after 5pm.

A company that has followed a similar concept for Mainframe/PC
interconnectivity is Attachmate, which puts a Mainframe Serial number in
their software, thus rendering it inoperable in a Disaster Recovery mode
without calling them in advance.  Tough to do in a "Disaster" mode.
Therefore, the organization that I work for dropped that software entirely,
in favor of a company which at least allows to backup the software for
emergency use.  Will we do the same with Fujitsu?  Hard to tell at this
point in time.

I totally understand the Piracy issue, but isn't it about time we get out of
kindergarten, where the whole class is punished because there was one or two
"bad eggs" in the class?

Denny
```

```
cost effective to purchase maintenance contracts for software, than to buy
new versions.  I agree that it is a marketing ploy to some extent, but if
you call for support on a product and you aren't running the currently
supported version, your support often becomes "Gee, we can't help you
because that version is no longer supported".  Even though we may not always
need the latest, greatest bells and whistles, the current list price for the
Fujitsu Enterprise Edition 6.0 (excluding maintenance) is $2750.  Annual
maintenance is $500/year.  I think you'll find that 6.0 is drastically
different that 3.0 and over 4 years of development, one is still money
ahead, especially if you need technical support.  We all play the
"maintenance" game with PC operating systems.  When's the last time any one
purchased the full version of Windows 95, 98, 2000, etc?   If we are using
Windows as an operating system,  we pay the "maintenance" by buying the
upgrade versus the full version.   As it is with Windows, there is no real
requirement to upgrade from Windows 3.1 to Windows 2000, unless, of course,
you wish to use all the features of Windows 2000 and get any current
application to run properly.  Nor is there any requirement to upgrade to
Fujitsu 6.0.  In fact, since we don't need the new Web features at this
time, we may not even install the upgrade.  But it's nice to know that we
can if we choose to.

It is still disappointing to see prices spiraling upward, but trust me, they
are not even close to Mainframe Software license fees at this point in time.
Yet many companies are indeed replacing their Mainframes with Server/Client
PC based platforms.  We'll see how it all sorts out in the future, but for
now, I'd much rather import some legacy code into Fujitsu than to try to
```
