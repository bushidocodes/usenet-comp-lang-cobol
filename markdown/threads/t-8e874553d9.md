```
Hope someone can help with a problem w/ MicroFocus Cobol. This
is really puzzling.

Under NT 4.0 and Micro Focus Workbench 4.0 I wrote a
quick'n'dirty to read a text (line sequential) file and display a
particular record to the screen with a simple DISPLAY
THE-VARIABLE. The program ran fine as long as it was in the
Workbench environment - either as an .INT run by the Workbench
run-time or as a .EXE running in a command processor (COMMAND.COM
or CMD.EXE) that was spawned by the Workbench. However, if I ran
the .EXE from a command processor started directly by the OS, it
gave a run-time error 107, apparently when it hit the DISPLAY
statement. If I comment out the DISPLAY the program runs without
error, but of course is useless . . . . I tried explicitly
specifying all of the ADIS object files for the link, but that
didn't help.

Experimentation revealed that the key is to have \COBOL32\EXEDLL
in the path. An analysis of the .MAP file revealed 3 .DLL files
being referenced -- MFRTS32, MSVCRT20, and KERNEL32. The first
of these is in the COBOL32\EXEDLL directory and the other two are
in WINNT\SYSTEM32. I tried copying MFRTS32 to my 'current'
directory, but that did not help -- I *have* to have
\COBOL32\EXEDLL in the path to get this to work.

- Ed Stevens

```
