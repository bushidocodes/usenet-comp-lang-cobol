Stdin and stdout are addressable as files - you just need to use the
correct filename when you OPEN the file:

"stdin" and ":CI:" correspond to stdin.
"stdout" and ":CO:" correspond to stdout.
"stderr" and ":CE:" correspond to stderr.

If the IO is line based, make the files LINE SEQUENTIAL, as this
handles the CR/LF record delimiters for you.

Of course, straight ACCEPT & DISPLAY verbs will work also, but with
less control.

› What about closing the pipe to the COBOL-Program. Does it handle EOF
› correctly?

If you use the file based solution, yes (though I couldn't say off
the top of my head whether you get an EOF condition - it might be a
file status error indicating a broken pipe).

Kev.
