To use indexed files in am IBM mainframe environment (MVS) you have
to define it first. To do that use the IDCAMS (EXEC PGM=IDCAMS) with
a DELETE (to delete the file if it already exists) and DEFINE (to
create the file). There must be hundreds of samples in your installation:
just ask for one to someone who has already used it.
This can also be achieved using ISPF panels if you have some non-IBM
utility such as File-Aid.

Once you've defined the file in your COBOL programs there should
be something like this in your enviroment division:

SELECT any-cobol-name ASSIGN TO dd01
ORGANIZATION INDEXED
ACCESS xxxxx
RECORD KEY yyyy.

xxxxx can be SEQUENTIAL, RANDOM or DYNAMIC (both sequential and random
access) and yyyy will be the name of the field (or group) that is the
primary key of your file, as in the copybook you will include in the FD
entry for the file. In the FD entry you don't need anything special:
just "FD any-cobol-name." will do.

dd01 will be the name of the DD card (in your JCL) that points to the file
you are using (and it could be anything up to 8 characters long, with
a letter as first character).

A frequent problem: empty indexed files cannot be opened as I-O initially.
They must be opened for OUTPUT the first time they are used (unless you
have already loaded some data into it).

Hope this helps!
