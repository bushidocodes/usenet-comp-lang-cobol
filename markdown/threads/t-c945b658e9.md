Many compilers support "conditional" compilation (which was introduced in 
the 2002 Standard,not the next one).  See for example,
 Micro Focus
$IF, $EVATULATE statements

I haven't checked in detail, but I believe the Standard (2002) already 
supports:

>>IF  abe = "xyz"
>>source free
>>else
>>soruce fixed
>>endif

So, if what you are looking for is mixing conditional compilatin and other 
compiler dfirecting statements, that is already supported in "Standard" 
COBOL.

If you are looking for support for specifying options to "more typical" 
preprocessors (such as DB2, Oracle, or ReportWrioter), then you need to be 
specific about which compiler and which preprocessor you are talking about. 
As far as I know, all of the "supported" comibinations of compilers and 
preprocessors allow for specificaiton of preprocessor "directives" with 
integrated compilers.

My employer wrote a pre-processor for COBOL many years ago and it is
still in use today.  We had developed our own language to produce
reports and other things that was supposed to make it possible for
non-programmers to program.  It didn't work out that way, but the
system is useful for quick programs used in batch, online or both.  It
has its own commands which the preprocessor translates into COBOL
statements.

The preprocessor is written in Assembler and employees some C code via
a call.  Personally I don't like it and only use it when forced to by
circumstance.  Other programmers love it.

But the point is the pre-processor reads the source, translates the
"special" code into COBOL statements, creates an output file of new
source which is then read either into the CICS preprocessor or the
COBOL preprocessor.

Preprocessors can have a varying amount of integration with the
languages they support.  C's, for example, largely assumes that the
input can be sensibly tokenized by the normal C rules.  That can (for
example) make a hash out of certain PICTURE clauses, if you're feeding
it a Cobol program.  Other preprocessors are much more generic in
their text processing (m4, for example).  Although I don't think you
can really call either "the Unix preprocessor".

Of course the very wide availability of C's preprocessor has meant
that it's been (ab)used for all sorts of purposes.
