Stephen:

The ASCII characters you are speaking of include:

ASCII 176 - light "tire track"
ASCII 177 - heavier "tire track"
ASCII 178 - reverse video "tire track"

The only problem is that I've only seen these characters in the IBM
extended character set in an MS-DOS character mode environment and not
in a Windows True Type font.

Perhaps someone else has been able to locate a True Type font
containing these characters.

The other problem you may experience if you do need to utilize a
special font is the fact that you may have to distribute and install
the special font on every machine on which your application must run.
If it is not a font distributed with the standard Windows
installation, you may have to do this.

If I may offer a suggestion, I would recommend using the Wingdings
font set and use an ASCII 112 (this is a lower case "p" in the normal
text, or default font).

That gives you a very nice looking box which can be used to accomplish
the same purpose.  Wingdings is also a font which is distributed with
every Windows installation, therefore you shouldn't have to worry
about distributing and installing any special fonts.

Good luck!
