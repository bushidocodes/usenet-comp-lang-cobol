yeah, after you connect the databridge, you write an application
which does the following in ODBC:

open cobol_database
create mdb_type_database
perform until end of tables
get table from cobol_database
write table attribs to mdb_type_database
perform until end of field
get field from cobol_database
set field attribs to mdb_database
open fields
perform until end of
field
get data item
write data item
close database

then you have your database which is an exact replica
the *problem* occurs when you want to index or link
certain fields...

so you write something which converts text from
the FD of indexed key and alternate keys to
MDB index format, which is relatively simple in ODBC...
