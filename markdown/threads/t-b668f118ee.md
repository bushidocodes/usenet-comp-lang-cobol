The MSA systems are not designed for direct entry of unverified data
from simplay any source since they do form a core corporate system.
To write directly into those files would bypass, if you are
successful, formal audit requirements. This is a potentially more
serious problem than simple 'net access.

To look at the file(s), and it is likely that you will need to look at
more than one for the retrieval of meaningful information, or to
extract less information, I would be inclined to build a dataframe for
IE and use this as an access window, creating a suitable extract file
that can be interrogated.

Now, as far as updating the files is concerned, do not do it directly.
It is risky, you can easily compromise your data integrity, and it is
not the recommended way to enter data into the system. Don't forget,
you will need to set up postings to GL if you update AP. The best way
is to set up input transactions for AP. You could do this by several
routes.

Obviously a file of batch input data is one possibility, and it is by
far the safest. You could set up a DCI transaction to process this
into AP if you want a speedier result. Alternatively, you set
something up with IE to do a similar job but you will have to be
careful as to the method because of the data integrity problem.

I have mentioned data integrity a couple of times here. It can't be
stressed too much when dealing with the company's finances because if
you screw up in any way, you're going to have hell's own time in
backing it out. You will need to consult the auditors because of the
nature of the problem, they may or may not find that your update
routines are acceptable.

If they don't, then you can't implement them. If you try to implement
contrary to auditors' recommendations or approval, you will have not
only screwed up the company's files but you will also have screwed up
the company. The nett result will be no company, no job, no pay. Oh,
and instant fame.

The mainframe has all of the cgi and web access facilities that you
need, so this aspect shouldn't be a problem. I would think that an IE
program to extract a suitable file, and a CGI routine to read it will
solve the enquiry side problem, and the creation of a batch input
file, again via the cgi, will solve the input side. This way, you
should keep the accountants and auditors happy.
