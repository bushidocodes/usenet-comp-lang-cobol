If it's a straight call from C to COBOL, then pass data via the COBOL
LINKAGE SECTION. On the C side, the LINKAGE SECTION records are structures
passed to the function.

For example:


PROGRAM ID. MYCOBOL
.
.
.
LINKAGE SECTION.
01 COMM-PARM
03 FUNCTION-IN PIC X(8).
03 RESULT-OUT PIC X(8).
PROCEDURE DIVISION USING COMM-PARM.
IF FUNCTION-IN IS EQUAL TO SPACES,
MOVE 'BAD FN ' TO RESULT-OUT.
GOBACK.

--------------------------------------------------------

void call_cobol_mycobol()
{
struct comm_parm
{
char fn_in[8], result_out[8];
} Comm_Parm;
/*
** Comm_Parm.fn_in maps to FUNCTION-IN OF COMM-PARM
** Comm_Parm.result_out maps to RESULT-OUT OF COMM-PARM
*/
void MYCOBOL(struct comm_parm *); /* define COBOL pgm to C */

memset(Comm_Parm.fn_in,' ',8);

/* call cobol program and get it's result */
MYCOBOL(&Comm_Parm);
if (memcmp(Comm_Parm,result_out,"BAD FN ", 8) == 0)
printf("COBOL program MYCOBOL found a bad FUNCTION-INÂ¥n");
}

Good idea to use CALLs and LINKAGEs; however, beware of the C data
structures which are usually delimited by a single low-value (x'00').
You may need to force-code a low-value as the last logical and/or
physical byte of the structure before returning to your C program.

In addition to all the other contributors advices, I can just point out
that if you're COBOL compiler supports it, you can declare a function in
your C-program and pass it as a PROCEDURE-POINTER to your COBOL-prog.

Then in your COBOL-prog you just call the pointer.. something like CALL
my-c-callback USING whatever...

I have experienced some problems with multithreaded C++-progs.. so you
should avoid that though...
