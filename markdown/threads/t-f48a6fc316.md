Regardless of the COBOL, especially with real-time (interactive) applications, any kind of network is going to waste a lot of resouces running the ISAM handling commands through the network unless you use something like AcuServer to keep all the record locking, header grabbing etc locally on the server.

I haven't used the AcuServer product yet, but the best performance in a multiuser environment using AcuCobolGT I've come across is using Citrix NT Winframe instead of plain Windows NT. The primary advantage is that Winframe gives you a star-topology which provides minicomputer level performance with all the bells and whistles of NT. (The star-topology keeps all the ISAM handling local too.) BTW, Citrix NT is "real" MS NT; they have source license agreements with both MS (& IBM.)
