GRIFF, you want to use disp=mod. Also, IEBGENER doesn't like dissimilar
DCB characteristics unless you know the magic words... Here, I'll start
the incantation for you. Capslock on!

//GEN1 EXEC PGM=IEBGENER,REGION=2000K,TIME=10
//SYSPRINT DD SYSOUT=A
//SYSUT1 DD DISP=OLD,DSN=your.first.input.file
//SYSUT2 DD DISP=(NEW,PASS,DELETE),DSN=&&TEMP,
// UNIT=SYSDA,SPACE=(CYL,(10,10)),
// DCB=(RECFM=F,LRECL=256,BLKSIZE=256)
//SYSIN DD DUMMY,DCB=BLKSIZE=80
//*
//GEN2 EXEC PGM=IEBGENER,REGION=2000K,TIME=10
//SYSPRINT DD SYSOUT=A
//SYSUT1 DD DISP=OLD,DSN=your.second.input.file
//SYSUT2 DD DISP=(MOD,PASS,DELETE),DSN=&&TEMP,
// DCB=(RECFM=F,LRECL=256,BLKSIZE=256)
//SYSIN DD DUMMY,DCB=BLKSIZE=80
//*
//GEN3 EXEC PGM=IEBGENER,REGION=2000K,TIME=10
//SYSPRINT DD SYSOUT=A
//SYSUT2 DD DISP=OLD,DSN=your.first.input.file
//SYSUT1 DD DISP=(,PASS,DELETE),DSN=&&TEMP,
// DCB=(RECFM=F,LRECL=256,BLKSIZE=256)
//SYSIN DD DUMMY,DCB=BLKSIZE=80
//*

all your input files must be lrecl=256, if not, you'll have to use:

GENERATE MAXFLDS=5
RECORD FIELD=(256,1,,1)

to force the lrecl.

Three steps and it's done. If you need your assembler code worked on, I
do that too.

Well, if both files are straight QSAM and in the same format, I generally use
IDCAMS like this:

//STEP010 EXEC PGM=IDCAMS,REGION=600K,TIME=2
//SYSPRINT DD SYSOUT=A
//SYSOUT DD SYSOUT=A
//SYS001 DD DISP=SHR,DSN=your.HDRDATA.file
// DD DISP=SHR,DSN=your.ASMOUT.file
//SYS201 DD DISP=(,CATLG,DELETE),DSN=your.new.concatenated.file,
// UNIT=SYSDA,SPACE=(CYL,(10,10),RLSE),
// DCB=(RECFM=F,LRECL=256,BLKSIZE=256)
//SYSIN DD *
REPRO INFILE(SYS001) OUTFILE(SYS201)
//*

advance:

There are a couple of ways you can go about this, but it seems as though the two
datasets are probably of different record lengths otherwise use something like:-

//S01GEN EXEC PGM=IEBGENER
//SYSUT1 DD DSN=FIRST.DATASET,DISP=SHR
// DD DSN=SECOND.DATASET,DISP=SHR
//SYSUT2 DD DSN=NEW.DATASET,DISP=(,CATLG),
// DCB=(*.SYSUT1),UNIT=KEEP,
// SPACE=(TRK,(????,????),RLSE)
//SYSIN DD DUMMY

If the datasets are of different record lengths use IEBGENER to extend/reduce
the length oth the first dataset and then concatenate your two datasets as
above.

Another method would be to use another utility such as FILEAID, which makes the
job very simple.

This sounds to me like one of those systems that was designed with JCL
concatenation in mind. It usually isn't necessary to physically copy
HDRDATA and ASMOUT to a third dataset. The program that reads ASMOUT
just concatenates HDRDATA and ASMOUT and they _appear_ as if they were
in the same file.

If you really must produce a single physical dataset for some reason, add
a job step to initialize ASMOUT from HDRDATA before the program runs so
at least you won't have to copy ASMOUT every time.
