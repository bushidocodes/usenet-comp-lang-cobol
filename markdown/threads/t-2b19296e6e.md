I tend to agree with you. But sometimes it is convenient, I made for
instance once a function module in Cobol, working pretty much like cell in a
spreadsheet, you could even nest formulas. If I were to solve this in
another way I would have had to use indexed files, if I wanted to use static
tables, I would have had to make those huge to make sure they would keep
pace.

The linked list I ended up with was two dimensional. Held by an array
outermost.

The first dimension was each line. Each line then had a user defined set of
columns and each column could consist of formulas. Like:

Line 1
..
Line 99 (Table)

For each line there were:

Column 1 .. Column n

And each column then had a formula list:
Formula 1 .. Formula n

Where a formula could point to a new formula in itself.

I chose to limit to possible nesting to three level deep.

Obviously this was/is terrible to debug, so, I am just happy it works :-)
