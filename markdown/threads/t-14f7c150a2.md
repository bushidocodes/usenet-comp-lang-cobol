A colleague has demonstrated a very specific bug:
program calls MSDOS the first time (eg copy a file): OK;
program calls MSDOS the second time: crash, invalid opcode!

This only happens if:
the program is a Micro Focus DOS COBOL program; and
the program is running under XM, the DOS extender; and
XM is running in a DOS box under Windows 95 (original or OEM);
and the PC is a Dell PC.

Curious, isn't it?

Luckily it is easily avoidable by compiling to a Micro Focus
Windows EXE, so that XM is not needed. Or not using a Dell PC.
