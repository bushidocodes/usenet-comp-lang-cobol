```
How have you defined the receiving variable?  Is it a single digit by any
chance?

 A Pic 9(18) Comp-3 field will take 10 bytes of storage.  (9 for the digits
and 1 extra for the sign - even though it is unsigned - because there are an
even number of digits in the PIC - which is always "wasteful" for COMP-3).
If you move that to a single digit receiving item, it will truncate to zero.
```
