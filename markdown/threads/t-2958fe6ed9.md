Why not use the SORT verb?  But if you can't or won't use SORT, there
is a much faster way to do what you want, and it takes much less
storage space as well.

1. Define a table of size N = number of records you want to select.
   In your case N = 1000.  As you insert records into the table you
   will maintain the table in reverse order: table(i) >= table(i + 1).

2. Making one pass of the file.  For each record read if the amount is
   greater than the lowest amount in the stored table, insert the new
   record into the table using insertion sort.  When the table becomes
   full, the lowest entry is 'bumped off' the end each time a record
   with a greater value is inserted.

3. When your pass of the file is complete, your table contains the top
   N records, in decreasing order of sales.

Insertion sort is easy.  Starting at the end of the table, you shift
'down' toward the end of the table each entry that is lower than the
entry you are inserting.  Keep shifting entries down until you find
table(sub) >= the new entry, or sub decrements to zero.  Then insert
the new entry into table(sub + 1).

However, if you want to sort the whole file without the SORT verb,
for general purpose use I prefer Shell sort.  Note that to sort a large
number of records you may not be able to load all of them into a memory.
However, if the disk records are fixed length, you may be able to read
the amounts and record number for the amount into a table and when
complete read the records off disk from the record numbers in the
table, and write the records into a new file in sorted order.  Else,
you will need to sort the file in sections, then merge all the
sections when complete.
