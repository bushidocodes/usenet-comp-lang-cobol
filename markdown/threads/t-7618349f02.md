I don't know if anyone here can help me. If I am in the wrong place I
would appreciate being pointed in the right direction.

A little background...
My company runs a cobol based program called MCS-3 for data
entry/storage of all company information. I must access this
information from a remote site twice a day to get any new orders which
we need to fill.

When run, the program requires a user number, a company number and a
password to get to the menu system. The menu is simply a list options
each with a number. I enter the number to get to the next menu until
getting to the report generating screen where I enter the first and
last order numbers to be printed. The order numbers are deterimined by
the days date. The first 6 digits are the date and the last 2 are the
order number.

this is the batch file we run to start the database
Set COBDIR=.
Set COBSW=+M+M2+P+K2+S6
if not exist M00D0602 run -f SYSMAST
run -f MCSP0000

[Q1] Is it possible to add keystroke commands or something similar
so the program will go to a certain place. Ideally I like to have the
Report run automatically twice a day and print out any new orders.


Another option would be to extract the data directly from the data
files. My division of the company doesn't keep its information in the
MCS program. Instead I have designed a custom Database in Paradox. If
I can import the data directly into paradox that would be great.

[Q2] Is there any way to convert the cobol data files without
running the MCS program. (I know I can create ascii text files that
are used for backing up but I can only do this from inside MCS.)

In case this helps the following is the Text Header from a menu file:

Micro Focus COBOL/2 Version 2.4.38 L2.3 revision 002
AXUPA/ZZ0/31978 GNB-039049002A4 GB2.1.55 I8086
MCSP0603 92122309012260 root


I'm running this off a WindowsNT Server and accessing via WFW and also
WindowsNT workstation. I've tried sending commands to the program as
it runs in a window but have been unsuccessful so far.
