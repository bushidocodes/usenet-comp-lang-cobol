Jeff,

There is nothing wrongand nothing has changed, , but before an open of a
file, and after a close, the contents of a file buffer (the 01 in the FD) is
UNDEFINED. That means that everything is possible. Sometimes you'll have the
same values as before the close, BUT YOU NEVER CAN BE SURE. So don't touch
the buffer before open and after close, and you are safe. If you want to
maintain the last record, just use READ INTO and WRITE FROM and you always
can see your record in WORKING STORAGE.

Hope this helps

It's not that they "trash" the record contents, but that the buffer
storage that was allocated for the file at OPEN is released at CLOSE,
making it available for other uses (after all, the semantic meaning of
"CLOSE" is "I'm no longer going to use this file or its contents").  
One would hope that any pointers to the buffers would be cleared at that
point, but in any event whatever pointer value you might have at that
point can't point to the original data, because that storage is no
longer owned by the program.

recompile all COBOL programs with a "certified Y2K compliant" compiler.  In this
