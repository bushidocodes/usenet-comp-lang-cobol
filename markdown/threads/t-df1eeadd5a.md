Hope you have written a GUI Cobol Program before.
If you havent you better look for a hello world example.

Entering a class is easy.  You just collect the infomation put it in a
record and write the record.  You would have to have a student record before you
can enroll a student in a class.  In the real world you do not delete a record
for a class.  What you do instead is use a grade code for withdrawn or some
other code.  It is possible to delete a student, but only if all the other
records with their student ID are deleted first.

To change a record you have to read a record and then rewrite it.
To delete a record you have to read a record and then delete it.
Better use records with a record key/index.

On an interactive program you do not end the program when you reach the end of
the file you have to make it end when you press a F3 key or a given command like
END or a Button.

In our school we have a file record for each student with their address and
other information, then we have a file for each course with the Course code,
course number, course section (CIS 135 01).  This has just the course data like
type name hours closed/open/cancelled, and number of students enrolled and the
limit that can enroll. Then we have 2 ways to look up the class file with the
grade in it; (course, stud-ID) or (stud-ID, course). You have to do both to make
class rosters and a list of classes for each student.  We use an occurs for the
times days the course meets which isn't normal form in a database, so we could
actually put all that in a separate file.  Courses can meet up to 7 days a week,
and at different times or locations, on different days.  

Of course each course file and class file also has a key for the year and
semester.  You have to know what semester the course is counted as.  Then we
have a semester file to indicate if each student has enrolled for the semester.
You can go crazy on a program like this because the possibilities are endless.

We never delete a student record because if a student drops out then the state
requires a record of their academic status.  The only time a Student record
could be deleted using a flat file is when all the associated class records with
the student identifier are deleted too to insure data integrity.  This would
only be done if the student had given us a wrong ssn.  Even if a student does
drop all classes before they start, we still need to have a record of the
activity because we probably already created a fee record and a bill, that we
need to change also.  Then if a student doesnt pay on time, we automatically
drop them from the course also.
>
