While this doesn't answer your question, you have a logic problem in your
paragraph names. Whenever I see a program with a paragraph named "read next"
I know I'm dealing with a logic structure that attempts to have two input
paths: one that reads the first record and one that reads all others. This
is a ticket to problems down the road. I recommend rethinking the logic
so ONE paragraph reads all input records. Yes, I know many instructors
actually TEACH this technique but it is error-prone.

On your question, paragraphs should reflect decomposition, so that a reader
can identify two things: which paragraphs are at the same level within the
program and which paragraphs are subordinate to a higher paragraph. Keeping
paragraphs to single functions makes this easier. In your example (and from
my above paragraph), I don't know if START and READ-NEXT are parallel or
subordinate.

If I sound critical I don't mean to be. START inplies a time-dependent
component of the program, whereas READ-NEXT tells the reader nothing except
that it is not time-dependent. Sorry.
