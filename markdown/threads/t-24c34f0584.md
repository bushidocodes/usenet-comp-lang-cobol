```
> the XML GENERATE instruction

I had a look at Bill's reference and was not impressed by the way they
create XML.  Tying in the XML to the field names means that every minor
variation needs the program to be rewritten.  My approach to outputting
XML (or HTML or EDIFACT or printed forms or ...) is to use templates.
For XML there would be a text template giving the overall structures
required with tags indicating where the data items are to be inserted.
Repeat groups can be done using subsections or by having loop control
tags depending on how you like templating to be done.

A subprogram has to be written to do all the text processing and
substitution but once written can be used for almost any template.

:header
<personlist version="1.0" encoding="iso-273">
:person
   <person id="%id%" picture="%image%">
       <name>%name%</name>
   </person>
:trailer
   <total count="%number%" />
</personlist>

The program would have, say, a table of tag:value pairs that is passed
to the templating routine:

       move zero to tag-limit
       call template using "header" tag-limit tag-table
       read person-file next at end ...
        perform until pf-eof
           move 3 to tag-limit
           move "id"           to tag-name(1)
           move pf-code    to tag-text(1)
           move "name"     to tag-name(2)
           move pf-name   to tag-text(2)
           move "image"    to tag-name(3)
           move pf-picfile   to tag-text(3)
           add 1    to rec-count
           read person-file next at end ...
       end-perform
       move 3 to tag-limit
       move "number"           to tag-name(1)
       move rec-count          to tag-zzz9(1)
       call template using "trailer" tag-count tag-table

This way you can have alternate templates that output the same data as:
XML. HTML table, CSV file, printed form, EDIFACT, ... without changing
the program much.

The templating program can be made quite sophisticated. For example it
could have decisions:

?%image%:   <person id="%id%" picture="%image%">
!%image%:   <person id="%id%">

Where ? in col1 means 'if the tag's value is not blank' and ! means 'if
the tag value is blank'.

or

   <person id="%id%" picture="%image|default.jpg%">

where the bar starts a default value if the tag value is blank.
```
