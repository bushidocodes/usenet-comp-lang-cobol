ability to utilize the mouse in a very simple manner; to select the
data entry field only (none of the complicated routines).  This is on

without the mouse-code...

I've had ADIS mouse built into some systems for many years and never had
this problem reported, but then the users probably haven't used the
mouse much as these are keyboard based systems.

With MF 16 bit .EXEs systems I have always carefully contrived to load
the modules by having a small core module that CALLs/CANCELs each
application module and menu program in turn.  Before doing this it opens
an ISAM file to load EXTFH and does a display AT to load ADIS.  This
seems to load them low so that the applications being called/cancelled
have a contiguous area to use.

Have you got a SET COBPOOL=10 or similar ?
