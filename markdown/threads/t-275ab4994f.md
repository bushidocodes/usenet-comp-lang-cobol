```
Using
 FUNCTION Integer (whatever) + 1

will ALWAYS round up (even for negative numbers) - if what you want is for
rounding up to the next highest integer.

If, however, you use INTEGER-PART, it will not work "correctly" for both
negative and positive numbers.

OBVIOUSLY, if you are trying to round "decimal points" - from 3 to 2 (for
example) - then this technique would NOT work - unless you used something
like,

  Compute Temp-Item = Original-Item * 100
  Compute rounded-up = (Function Integer (temp-item) + 1) / 100

which doesn't seem "cleaner or easier" (IMHO) than the original adding of
.999 - but would work.
```

```
"Cliff Peterson" <cliffpeterson@spamcop.net> wrote:
> I've always used...
> FUNCTION INTEGER (whatever) + .999999999999999999
> ...or as many decimals as the compiler will take.

Actually, using more decimals than is specified in the PIC of the
target field can get you into trouble.  If intermediate results are
rounded by the hardware or software at any point before storing the
final result, any 9's to the right of the current precision could
be rounded up, causing a number to be double rounded, incrementing
it too much. :-)  This type of rounding is best performed as one
step, using the precise number of .999's to match the PIC of the
data field being rounded up.  If you need to round negative numbers,
you can use either of the following methods.  Method 1 is far more
efficient. :-)

Method 1:

    03  WS-NUMBER    PIC S999.9999.
    03  WS-INTEGER   PIC S999.

    IF (WS-NUMBER < 0)
        SUBTRACT .9999 FROM WS-NUMBER GIVING WS-INTEGER
    ELSE
        ADD .9999 TO WS-NUMBER GIVING WS-ROUNDED
    END-IF

Method 2:

    03  WS-NUMBER    PIC S999.9999.
    03  WS-UNSIGNED  PIC  999.9999.
    03  WS-INTEGER   PIC S999.

    MOVE WS-NUMBER TO WS-UNSIGNED
    COMPUTE WS-INTEGER = WS-NUMBER + .9999 * (WS-NUMBER / WS-UNSIGNED)

(Note: I know the parenthesis are not required here, but I believe
       they make the calculation clearer.)

It is surprising to me that the 1989 Intrinsic Functions module did
not include the Absolute Value and Sign functions, either of which
would have simplified this and lots of other similar code [e.g.
COMPUTE WS-INTEGER = WS-NUMBER + .9999 * FUNCTION SIGN(WS-NUMBER) ].
```
