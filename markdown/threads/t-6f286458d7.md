Have you looked at XPEDITER from Compuware?

About 5 years ago, me and another developer added the Batch Connect
Facility to XPEDITER. I don't work for Compuware any more. What it will
do is allow you to run the XPEDITER debugger under TSO/ISPF or TSO
without ISPF to debug a COBOL program running in a batch address space.
It requires a VTAM connection between the MVS batch system and the MVS
TSO system, unless of course it is a single machine.

A break point on a single statement should be pretty cheap. A breakpoint
is nothing more than a bad opcode inserted to cause a program check so
that the ESTAE routine can gain control to tell you that it happened.
Any verb without a breakpoint will run at full speed.

I have no idea if XPEDITER still supports the Batch Connect Facility.
You'll have to ask them.
