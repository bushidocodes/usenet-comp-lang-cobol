```
The Micro Focus COBOL runtime attempts to trap all of the standard UNIX
signals. The default operation is to clean up (close files, etc) and to
exit.  This includes SIGHUP (1) and SIGTERM (15)

You can specify your own signal handlers to operate in conjunction with (or
override) the RTS by using the cobpostsighandler API call. The docs for this
are in the "programmers guide to writing programs" in the "mixed language
programming" chapter.

NB: It is not always possible for the RTS to terminate and close everything
cleanly - with a SIGSEGV or SIGILL for instance, all that's really possible
is to terminate as the process is in an illegal state.  With SIGKILL (9),
the signal is totally uncatchable by any UNIX process - so, don't use it
unless your process won't respond to anything else.
```
