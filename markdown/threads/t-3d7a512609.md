```
As stated elsewhere (in this thread), this simply is not correct.

ANY ANSI '85 conforming compiler should accept (and require when in
"conforming mode") that the reference modification occur AFTER the
qualification - not after the "1st" data-name.

As I have said (and as someone else originally pointed out), the Micro Focus
compilers for MANY years have provided "IBM OS/VS COBOL" emulation mode.  As
there are some "syntax" issues with the way that this '74 (not '85) Standard
compiler works with parenthesis and  combined abbreviated conditional
expression, it does NOT allow reference modification (which was an addition
in the '85 Standard) in a conditional expression when the OSVS dialect is
selected.

It does fully support reference modification when NOOSVS is specified or
when any of the (various) '85 Standard "emulation" modes are in effect.
```
