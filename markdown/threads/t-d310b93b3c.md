Try STRING - following is very crude ( but also check STRING syntax in
your textbook/reference manuals) :-

01 field-hours		pic zzz9/9999.
01 field-minutes 	pic zzz9/9999.
01 field-seconds	pic zzz9/9999.

initialize (move spaces to) output-string *> *** good idea to do this
String	 field-hours delimited by size	  
    	" h " delimited by size
	field-seconds delimited by size
    	" min  " delimited by size
	field-seconds delimited by size
	" sec " delimited by size
	into output-string
End-string

To get the 'value' strings reduced to 'actuals' check for leading zeroes
or blanks and then you can use REFERENCE MODIFICATION to move those.
For example - field hours 4 char field = 11 then you would be
'stringing' field-hours(3:2) from third character to the end which is
the '2', or another way to express it field-hours(3: ) - the second
missing ref mod implying move 'containing up to the last'. Obviously to
use this approach you need variables to denote the positioning :-

	string field-hours (x:y) delimited by size
		etc.....

Perhaps a code fragment to show this using UNSTRING/STRING (I know it is
homework, but...)
[warning:  untested]

01  FIXED-FORMAT-LINE.
    03  pic x(16) value "You have worked ".
    03  FFL-HOURS pix Z9.
    03  pic x(4) value " h, ".
    03  FFL-MINUTES pic Z9.
    03  pic x(9) value " min and ".
    03  FFL-SECONDS pic Z9.
    03  pic x(10) value " sec".

01  CONSTRUCTED-LINE pic x(50).
01  HOLDER pic x(50).

01  F-POS  pic 99.
01  F-LEN  pic 99.
01  C-POS pic 99.
...


    move 1 to F-POS, C-POS.
    move 0 to F-LEN.
    inspect FIXED-FORMAT-LINE tallying F-LEN for characters.
    move spaces to CONSTRUCTED-LINE
    perform until F-POS >= F-LEN
        unstring FIXED-FORMAT-LINE
                delimited by ALL SPACE
                into HOLDER pointer F-POS
        string HOLDER delimited by space
                " " delimited by size
                into CONSTRUCTED-LINE
                pointer C-POS
    end-perform
