Assuming you mean GetPrinterName() returns a pointer, then the trick is
to declare a linkage item and set it's address to the value returned in
the pointer.

LINKAGE SECTION.
01  MY-VAR        PIC X(256).

    CALL APIENTRY "GetPrinterName" USING ....

   IF my-pointer <> NULL
       SET ADDRESS OF MY-VAR TO my-pointer
   END-IF

Are you CALLing a function or INVOKEing a method called "GetPrinterName"?

If you are CALLing a function, you should not be getting back an object 
reference datatype as a return value. In which case Karl Wagner's reply 
will probably help.

If you are INVOKEing a method, then it depends what type of object the 
method returns. The debugger in the IDE can give you a clue here when you 
look at the data item.

If it is an automation object, then the object will have another method 
defined in the Crystal Reports docs that gets the actual name. These 
methods should be listed under the type of object that the property 
"PrinterName" returns.

Otherwise, the object might be an instance of the class library "chararry" 
class - in which case, use the "getValue" (or other) method(s). (In this 
case, it would be almost as easy to use a pic x(n) return type.)

In each case, finalize the object returned when you've finished with it.

>Has anyone else tryed using the Crystal Reports 7.0 ocx in Netexpress ??
>It sometimes crashes my DialogSystem session , does this happen to others
>too ??

Please make sure that you have installed the latest WebSync updates. 
If the problem persists, and you think the problem lies in the Net 
Express product, please report it to Technical Support. They'll probably 
ask for some example code and (as they probably don't have Crystal 
Reports 7.0) the relevant parts of the documentation.

Steve.
