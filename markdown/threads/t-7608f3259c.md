> I'm at a loss.  Can someone help me out?

It's actually pretty easy to do.  Check out the "Open", "Read", and 
"Write" statements for your file manipulation, the "Move" command to 
move data between the input and output variables, and the "If" statement 
to check your condition.  Also, make sure you have a "Select" statement 
in the Environment Division, Input-Output Section, File-Control area, 
and make sure you have an FD (File Description) in the Data Division, 
File Section, for each of your files.

Well, it sounds like you're close, but if someone had less than 12 
hours, your Student-Name-Out will contain...

*
....5....1....5....21

...whereas when they have 12 or more, it would contain...

Gordon Sumner
....5....1....5....21

You can fix this problem 3 ways - splitting the asterisk column from the 
rest of the name (ex. instead of "Student-Name-Out  Pic X(21)", you'd 
have two variables, such as "Student-Low-Hours-Ind  Pic X(01) / 
Student-Name-Out  Pic X(20)" - this is probably the best way), reference 
modification, or using the String statement.  (Some versions of COBOL 
support concatenation as well, but mine doesn't.)

Glad it helped you...  You'll find that manuals will be your friend, and 
learning to find information in them will put you at the head of your 
class.  :)

words in small letters are field names, so you can change them into the 
names you prefer.

Want to get an A?

working-storage section.
01  name-with-asterisk.
     05  value '*' pic x.
     05  student-name pic x(30).
linkage section.
01  name-for-report   pic x(30).

if  total-credit-hours less than 12

    set address of name-for-report to address of name-with-asterisk

end-if
