Bob...

Most touch screen drivers we have encountered over the years with our
products simply allow the touch screen to operate in exactly the same
manner as a mouse.  In other words, if your current application uses a
character mode or GUI mouse, then the touch screen should operate
without automatically.

In the event that your COBOL application does not have mouse support,
then you would probably need to make sure that the COBOL application
can work with the mouse before a touch screen could be supported.
