I almost mentioned the term GOTO in my subject but decided it might be too
off-putting.  Let me give you a bit of some code.  It's bit verbose, but it
was easier to use real code than to make something up!


870-UPDATE-WIREMNT-HDR.
**** DO WE NEED TO UPDATE RECORD # 1?                        
     EVALUATE TRUE                                           
       WHEN WIRECOMM-DELETE-WIRE                             
         EVALUATE TRUE                                       
           WHEN WIREMNT-NEEDS-APPROVAL                       
             ADD -1              TO APRV-ADJ-OUT             
           WHEN WIREMNT-NOTICE-PRINTED                       
             CONTINUE                                        
           WHEN OTHER                                        
             ADD -1              TO NTCE-ADJ-OUT             
         END-EVALUATE                                        
       WHEN WS-NOW-APPROVED                                  
         ADD -1                  TO APRV-ADJ-OUT             
         ADD +1                  TO NTCE-ADJ-OUT             
         IF WIRECOMM-REF-NBR NUMERIC                         
                 AND WIRECOMM-REF-NBR >= FB-TO-FB-1ST-REFNBR 
             PERFORM 0680-ACCOUNT-LOOKUP THRU 0680-EXIT      
             IF WIRECOMM-TXFR-BRANCH = 218                   
                 MOVE +00001             TO INCOMING-BANK-NBR
             ELSE                                            
                 MOVE WIRECOMM-TXFR-BANK TO INCOMING-BANK-NBR
             END-IF                                          
             ADD +1                  TO NTCE-ADJ-IN          
         END-IF                                      
       WHEN WIRECOMM-IMMED-NOTICE                    
                 AND NOT WS-FIRST-NOTICE-PRINTED     
         ADD -1                  TO NTCE-ADJ-OUT     
       WHEN WS-FOREIGN-AMT-CHANGED                   
                 AND WIRECOMM-NEED-APPROVAL          
         ADD +1                  TO APRV-ADJ-OUT     
         ADD -1                  TO NTCE-ADJ-OUT     
       WHEN OTHER                                    
         GO TO 0870-FINISHUP                         
     END-EVALUATE.                                   
     SKIP1                                           
     IF WIRECOMM-BRANCH-NBR = 218                    
         MOVE +00001             TO OUTGOING-BANK-NBR
     ELSE                                               
         MOVE WIRECOMM-BANK-NBR  TO OUTGOING-BANK-NBR   
     END-IF.                                            
     SKIP1                                              
     PERFORM 3200-UPDATE-WIREMNT-HDR.                      
     IF ERROR-OCCURED                                   
         MOVE ERROR-MESSAGE      TO WIRB-MESSAGE-LINEO  
         GO TO 0500-SEND-MAPWIRB.                       
     SKIP1                                              
     IF WS-NOW-APPROVED                                 
             AND WIRECOMM-REF-NBR NUMERIC               
             AND WIRECOMM-REF-NBR >= FB-TO-FB-1ST-REFNBR
         PERFORM 0660-WRITE-INCOMING                    
            THRU 0660-CONTINUE                          
     END-IF.                                            
 0870-FINISHUP.                                         
     MOVE WIRECOMM-FUNC          TO WIRECOMM-MAP.       
     GO TO 0714-BROWSE.                                 

As you can see, there is an evil GOTO (in then WHEN OTHER).  I'm curious as
to how someone might restructure this to avoid the GOTO.  Of hand I can see
creating a flag called UPDATE-HDR and then doing the following:

MOVE 'Y' TO UPDATE-HDR
(prior to the evaluate)
  WHEN OTHER
     MOVE 'N' TO UPDATE-HDR

And after the evaluate put
IF UPDATE-HDR
   ...everything between END-EVALUATE and 0870-FINISHUP goes here
END-IF

I could also avoid a new flag by saying
IF ZERO = APRV-ADJ-OUT AND NTCE-ADJ-OUT AND NTCE-ADJ-IN
  CONTINUE
ELSE
   ...everything between END-EVALUATE and 0870-FINISHUP goes here
END-IF

But this seems excessively verbose...

And really, why should I do this just to avoid using a GOTO.  Can I really
justify this?  Well, no I can't, but perhaps you can.

Not to start yet another GOTO war, but I was just reviewing this code and I
couldn't help myself!  (And this doesn't even reference the second GOTO...)



--- 
Frank Swarbrick
Senior Developer/Analyst - Mainframe Applications Development
FirstBank Data Corporation

whole program for references to that label, or to prior labels that
may result in dropping through the label, or to references that may

paragraph need be examined plus any PERFORM or THRU that refers to
that prior paragraph, or the one under examination.

guarantee this is in fact the case, and that there can only be one way
the flow can exist around any label.

complexities such as 'excessive nesting of IFs' and other things they
are not used.
