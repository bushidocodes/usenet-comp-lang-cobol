Incorrect. The output of the compiler is either interpreted (.int) or
native (.gnt) dynamically loaded code. Dynamically loaded by the COBOL
runtime system. Execution of either does NOT require source to be
present (unless you are executing under the debugger, of course).
What you don't get is output as .obj files (and hence, as Bob said,
cannot create a *stand alone* executable).
