If you can get a copy of Thane Hubbell's "Teach Yourself COBOL in 24
hours" it will give you date/time examples.

Failing that you can use ACCEPT Date or Time, but preferably :-

    move function current-date to ws-DateStart *> 21 alpaha/numeric 
    move function current-date to ws-DateFinis *> 21 alpaha/numeric     

If you don't have 'move function' then use 'compute function' on a
numeric Date21N.
	
When starting/finishing you get the 21 character date/time/GMT format -
check your manuals/on-line help. Taking ws-DateStart and ws-DateFinis
you need to bring them down to the lowest denominator, subtract and then
convert back to previous units - to get your time usage. 

	1 hour   = 	xxxxx hundreths
	1 minute =	xxxxx hundreths
	1 second =	xxxxx hundreths

       01 ws-Date21.
          03 ws-yyyymmdd.
             05 ws-Year.
                10 ws-cc          pic 9(02).
                10 ws-yy          pic 9(02).
             05 ws-Month          pic 9(02). *> range 1 - 12
             05 ws-Day            pic 9(02). *> range 1 - 31
          03 ws-yyyymmddN
             redefines ws-yyyymmdd pic 9(08).
          03 ws-Time.
             04 ws-HoursMinsSecs.
             05 ws-HoursMins.
                06 ws-Hours   pic 9(02). *> range 0 - 23 past midnight
                06 ws-Minutes pic 9(02). *> range 0 - 59 past hour
             05 ws-Seconds    pic 9(02). *> range 0 - 59 past minute
             05 ws-Hundredths pic 9(02). *> range 0 - 99 past second
          03 ws-GMT.
             05 ws-PlusMinus      pic x.           *> value 0, + or -
             05 ws-GMThoursMins.
                06 ws-GMThours    pic 9(02). *> range 0 - 12/13
                06 ws-GMTminutes  pic 9(02). *> range 0  - 59

       *>  ws-hundredths = "00" if system cannot generate fractions
       *>  of seconds

       *> Greenwich Mean Time :

       *> if the system cannot provide a local time differential
       *> a value of "00000" is returned in the last five positions

       *> ws-PlusMinus refers to GMT : either '0' above or '+' or '-'

       *> ws-GMThours     range '-' 0 - 12; '+' 0 - 13
       *> ws-GMTminutes   range '-' 0 - 59; '+' 0 - 59

       *> GMT minus '-' - indicates hours/mins behind   GMT
       *> GMT plus  '+' -     ..        ..     ahead of GMT

> In a COBOL program, I need to know how long the user used the program,
when
> he quits. So I get the time when it starts and ends, but this doesn't work
> if for example the program is opened at 8 in the evening, and closed at 2

Convert both times to seconds.  If the opening and closing dates don't
match, run both of them through an integer-of-date function, multiply the
difference between them by the number of seconds in a day and add it to the
closing seconds.  Subtract the opening seconds from it and convert the
result into days, hours, minutes, and seconds.  HTH
