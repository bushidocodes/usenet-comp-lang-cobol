Is anyone out the a Realia COBOL MLI guru?

I have an interface which requires host system updates to be date/time
stamped. In order to eliminate variances in the workstation clocks I want to
have my Realia COBOL 4.29 application get the time from the NetWare 3.12
servers that host the system.

I have established that I need to do the following

1) populate the AX, CX, & DX registers as follows
MOVE 7                       TO DX.
MOVE X'14F2'                 TO AX.
MOVE 0                       TO CX.

2) Generate int 21

3) Check the AX register for error and get the results from a seven byte
server reply packet

I have ran various tests and have been able to successfully generate
interrupts using the MLI_INT function. The problem is when I generate an
interrupt that requires a reply packet to be populated, I always GPF. I have
tried various combinations of defining the server packet then using SET
ADDRESS and using DOS_ALLOC to try to allocate a specifc pointer. I have
spent a lot of time trying to get this to work but I have not had much luck.

If anyone has any code examples from Realia 4.29 that set a pointer for a
reply packet, or a way to get the time from a NetWare server without shelling
to DOS I would greatly appreciate it.
