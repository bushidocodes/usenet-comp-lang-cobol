```
Yes, as this is an alphanumeric (and not a numeric) Intrinsic Function, it
may be used as the sending operand in a MOVE statement (in any COBOL
compiler that includes ANSI/ISO conforming support for Intrinsic Functions).
Your COBOL documentation will tell you what format this will be in.  You
*may* be interested in just the first 8 bytes (and not want to use "default"
truncations) - so you may be interested in:

 Move Function Current-Date (1:8) to WA-HOLD-DATE

or if you don't want the century (after all you won't be around for the Y3K
problem <G>) you could code:

 Move Function Current-Date (3:6) to WA-HOLD-DATE
```

```
Kevin,

        Yes it can. CURRENT-DATE pulls two calls. One to CEELOCT and the
other to CEEGMTO, neither of which use any MVS Macro services.

        Alternatively, you could always issue an ABSTIME ABSTIME followed by
a FORMATTIME ABSTIME, where you can specify a plethora of different date
formats.
```
