```
I would like some advice on a COBOL problem. We are converting to an IBM
AS/400 (RISC machine) and will be using COBOL /400 ILE. This is a
variation of COBOL 85.

One of the programs which we use lists on the screen the quarter, the
course and the section of an instructor's class. Below this will be an
alphabetic list of the students. Three files are used at present-- a class
master which contains the instructor, the quarter, the section and the
course. The second file contains the quarter, the section, the course and
the name of the student. This is sequenced so that all students in a
course and section are in alpha order. Then the name is used to read
another file for student information.

As a part of the conversion we want to have a person's name only in the
student master. The student master will contain a uniqe ID number for each
student. This ID number will be in the second file with the class
information. If I read the master, then the second file, I will have the
students in order by ID or randomly. They need to be alpha.

Doing this in batch is simple on the AS/400 -- and OPENQRY file with the
required key structure for the file and I'm home free. The problem is how
can I do it interactively for people who are used to thinking that a 1
second response time is needed? I have thought about reading the ID
number and placing the names (up to 42) and their data into a temporary
file keyed by name, then read it for the display and clear it after each
class that is requested.

If this is clearer than mud--do you have any suggestions?

--Ray B.
```
