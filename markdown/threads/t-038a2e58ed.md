Compiler messages are clear. You cannot divide by letters (pic x), only by
numbers (pic 9).

There are SERIOUS structural errors in your program. You want a
three-dimensional table where one dimension is major (SCL, 3),  one is year (4),
and the third is grade triad (3). These are all accumulators to which you will
add 1. At the first and second dimension, you also want to accumulate the total
of next-level subordinates. Don't try to accumulate in DETAIL-RECORD. That's an
output format, not internal accumulators. Change the PICs in detail-record to
edited (zzz.99), and redesign the report to print three lines, one per SCL (it
appears that's what you want), with a description of the SCL. If you want a
one-line report, leave it as is. Get rid of 88s because you're not using them
anyway. The formula for percentage (do remember to say rounded) is (numerator *
100) / denominator. Do not say (numerator / denominator) * 100 because you'll
lose precision. 

In summary, learn to use tables (OCCURS) and subscripts for both accumulators
and report columns. 

Feel free to post here if you have trouble with an IMPROVED versions of your
program. We like to see the student is making an effort rather than asking us to
do the work for him. Why? Because in the real world, you don't get helpful
hints, you get fired. That can bum you out for a whole weekend.

SCL-CODE is not defined as a numeric value, but as a 1-byte text string.  
You will need to do some sort of type cast to get this into a numeric 
value, or use the "Function NumVal( SCL-CODE )" in your compute 
statements.



>    196     WRITE OUTPUT-FILE.
> "C:\My Documents\COBOL\a-7-dj.cbl"(196,18): * 330-S Not a record name

>            WRITE OUTPUT-FILE.

This is part of that freaky and not very successful effort to make Cobol 
easy to code for english speaking non-programmers.  The idea is that one 
would say "READ the-file" but "WRITE the-record".  So in the write 
statement, the file that is being written to is never mentioned.  This 
is just unusual and there really isn't a good way to learn this, you 
just have to get bitten by it a few times to remember.  Your statement 
should look like:

>            WRITE OUT-SALARY-REC.




> 
> Are my 88's wrong?

Your 88s are good, but you don't seem to be using them.  For example, 
SCL-CODE is defined with:

>            05  SCL-CODE                        PIC X.
>                88 BUSINESS                     VALUE '1'.
>                88 LIBERALARTS                  VALUE '2'.
>                88 ENGINEERING                  VALUE '3'.

but you have IF statements checking for "1" and "2" and "3".  This 
defeats the purpose of defining the 88s.  You might want to go back and 
try:

   IF BUSINESS
      stuff
   END-IF
   IF LIBERALARTS
      stuff
   END-IF
   IF ENGINEERING
      stuff
   END-IF


The same holds true for ARE-THERE-MORE-RECORDS.  You could "SET 
NO-MORE-RECORDS to true" and "PERFORM UNTIL NO-MORE-RECORDS".

The reason we write records is not to make the language english-like; we do it
to specify the record length. An FD could have three records with different
lengths. If we wrote the file, the compiler wouldn't know which record length to
use. Note that DELETE does use the file name.
