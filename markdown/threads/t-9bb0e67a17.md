```
Interesting.  I was looking through one of the IBM Language Environment
sample COBOL programs and came upon one with the following code:

  PERFORM                                                
    CALL "CEEGQDT" USING CURRENT-CONDITION               
                       Q-DATA-TOKEN FC                   
    IF SEVERITY > 0 THEN                                 
       DISPLAY "CALL to CEEGQDT failed with "            
               "Severity = " SEVERITY                    
       DISPLAY "                and message "            
               "number   = " MSGNO                       
       GOBACK                                            
    END-IF                                               
    SET ADDRESS OF Q-DATA-PTRS TO Q-DATA-TOKEN           
    SET ADDRESS OF ABEND-CODE TO Q-DATA-ABEND-CODE-PTR   
    EVALUATE ABEND-CODE                                 
       WHEN 777                                         
         DISPLAY "Severe Error!  Condition Handling "   
                 "should not get control for IMS Abends"
         SET PERCOLATE TO TRUE                          
       WHEN OTHER                                       
         CONTINUE                                       
    END-EVALUATE                                        
  END-PERFORM                                           

What's weird?  There's a PERFORM, but no UNTIL!
And it even compiles.  I wonder what their reason behind doing this was.  I
could be mistaken, but I don't imagine the program would behave any
differently with the PERFORM and END-PERFORM removed.
```

```
There are two reasons (thatI can think of) why such code might exist:

1) Simply to indicate a "logical block" of  code.
2) Because it used to be an "out-of-line PERFORM that someone moved in-line

Of course, there if you have an out-of-line PERFORM (with no UNTIL) it does 
the same thing as moving all the code "inline", too (assuming you use corret 
scope-terminators and remove all periods).

Without seeing the specific code example and knowing some of the LE 
documenation, it wouldn't surprise me if this code was done to "match" some 
C code or other language "logical block" structure.
```

```
Could have been an interesting alternative to END- statements in COBOL 1985.
 For example...

IF SOME-CONDITION
    PERFORM
        PERFORM SOMETHING
        PERFORM SOMETHING-ELSE
    END-PERFORM
ELSE
    PERFORM
        PERFORM SOMETHING-ELSE
        PERFORM SOMETHING
    END-PERFORM

Of course it'd probably also have to have a rule that IF and ELSE can be
followed by only a single "statement" (with the PERFORM block being
considered a statement), pretty much like C et al.  But not like prior
versions of COBOL, so I can see why they didn't go that way!
```
