>I do not think that anybody ever really understands how a
>computer works until they have coded in assembler.  It does
>not matter that much which assembler ... keeping in mind that
>each processor uses a different assembler. 

Definitely.  When I started, it was with Fortran, Cobol and 360BAL.
I added CDC-6000 series assembly when I worked with one of those,
then PDP-8 assembly.  I've done serious amounts of 68000 and x86
assembly since then.  

Learning one machine's assembly language will teach you lots.  Learning
a second one, especially if for a very different kind of machine, will
teach
you more.  It will help you to code better in any HLL you work with also.

Depending on where you are working 360/370 assembler, PowerPC or
x86 could be good choices.  Although x86 is arcane, not clean like 68000 or
360.

Also you get to learn the various machine architecture paradigms
at a much more detailed level than you normally see in any HLL:
(examples only, not exhaustive)
stack oriented
register oriented
memory to memory oriented
one operand / two operand /  three operand
and computational paradigms (again examples)
binary vs. decimal
sign magnitude vs ones complement vs twos complement &c
integer vs fixed point vs floating point

Enjoy!
