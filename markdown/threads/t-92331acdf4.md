Hello,

I have a rather difficult problem that hopefully someone can make a comment on.
 I am using NetExpress with Sp2 from Flexus.  I have a screen that has numerous
pushbuttons on it.  These push buttons are defined as Icon/BitMap push buttons.
 So far no problem, here is the difficult part.  Part of the BitMap is text
that the user will set up in a control file.  So I can not build these BitMaps
in advance.  The text needs to be on top of the push button so it must be part
of the bitmap.  I have create several blank template bitmaps using PaintShop
Pro.  But now the problem is merging the text in the control file with the
blank template bitmap.  The user will be changing this control file often so my
program needs to be able to read the control file and then merge the text with
the template bitmap to form the updated pushbutton bitmap.  Is anyone aware of
a third party product that i can "CALL" or "WinExec" to accomplish this merge ?
This application is for a touch screen point of sale system.  Any ideas would
be most welcomed ?
Thanks,
Bob Hennessey

Hi Bob!

I think that I have a pretty good solution for you.

COBOL sp2 supports the ability for you to embed OCX (Active X
controls) in your user interface screen.  In addition, you can control
the properties and methods for these OCX's in the same way that you
control all of the standard Windows user interface controls.

FarPoint Technologies develops and sells a set of OCX controls called
Button Objx which allows you to display and control buttons with
embedded bitmaps and text.  The text can be dynamically changed by
modifying a property prior to display of your sp2 screen.

The other cool thing about Button Objx is that you can also display
buttons which are not limited to square or rectangualr shapes...that
means you could display round, oval, triangular, etc. shapes.

You can download a trial version of the Button Objx OCX control at:

http://www.fpoint.com/products/button/button.html

You can also find a wide assortment of OCX components at:

http://www.vbxtras.com

I'll send you the COBOL sp2 Active X control add on in a private
e-mail message.



Bob Wolfe, flexus
