The CUG library contains public domain versions of lex and yacc, different
names but the same thing.

The rules aren't that bad once you get it going.

Parsing COBOL isn't that difficult, albeit cumbersome. I've done it many
times, even with COBOL. EXEC-xxxx ... END-EXEC can be troublesome as this
by definition carries a separate syntax, however COBOL "proper" is pretty
straight forward.

I've long since developed my own language parser, in function a dynamic
lex/yacc - in operation completely different, using C++. So far have not
been able to make it stumble on any free-form procedural language. However,
as I'm employed by a company, I can't release the software to the public.
