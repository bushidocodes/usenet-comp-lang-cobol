Honeywell 115 and 120 had six but characters with native character set
BCD, but no fixed word size.  Each byte in memory had the six data
bits along with a word mark bit (and I think a parity bit).  You used
the word mark to indicate the limit of various instructions.  Happily
we had the COBOL-B compiler and did very little in assembler.  The
compiler ran in 8K without mass storage.  Source on cards in, object
punched onto cards, mag tapes for intermediate storage by the
compiler.  On interesting feature of the compiler was that you only
the first few bytes of identifiers were significant(I think four).
T1A1-ABC matched T1A1-XYZ.  Seven track tape only, usually 556 BPI.
We eventually replaced the H120 computers with a Honeywell 2020

Later worked on DCD Cyber with 60 bit words.  It could use either 6
bit BCD or 8 bit ASCII.  60 Bits was great for floating point, but 8
does not go into 60 evenly.

Later still worked on Honeywell 66 which ran GCOS.  This was an
evolution of the GE mainframes.  I think the word size was 36 bits.
It had hardware emulation of the old Honeywell 2000 series computer.

The COBOL code that was written in the late '60s survived three
generations of mainframes.  Some of it was eventually moved to PC's
where a small bit continues to be used today.

---Charlie

I vehemently disagree :-)

It was never designed as a commercial machine and the 6 bit byte was a 
narrow loser in the "byte wars" . There is no particular superiority of the 
8 bit byte (apart from offering a larger character set without the need for 
control characters); it was simply that IBM established it. 60 bit words 
were excellent for floating point operations and CDC was not the only major 
company at the time that used 6 bit characters (NCR (on their 315), 
Honeywell, and ICL 1900 series ALL used 6 bit characters and were ALL faster 
than System 360. To be fair, Amdahl's 360 architecture was ground breaking 
because it used ROS to make a "soft" machine that could be easily upmodelled 
for marketing purposes, but in the lower end (like 360-30) the time it took 
to step through ROS for every execution slowed it down so much that IBM 
started moving ROS code from mylar firmware into actual wired hardware...

The  Cyber TTL logic, designed by Seymour Cray, was overtaken by MOSFET 
chips (largely in System 360) but it was more reliable (and faster) than 
early IBM 360s. The Cyber consistently creamed System 360  AND early System 
370 models in benchmarks.
