Hello, 

I was wondering if someone could point me in the right directrion on
this.

We have a program in Microfocus 2 Cobol that would read a Line
sequential file with TAB delimited input, and would then write some
data into another Line sequential file, imbedding modem charactes into
the output.  Eveything worked fine until......

The problem is that we now compile the program under Microfocus 3
cobol, and it messes up the Line sequential input and output.  On
input, it converts each TAB character to spaces.  On output, it
prefixes every non-alphaneumiric character with a binary zero.
This really messes things up.

I suspect that it has something to do with MF 3 handling international
character sets that use 2 bytes instead of one.  I tried to find how
to turn this off, but haven't had any luck.

Thanks for your time,

Paul

Look up Line Sequential Organization in the 'File Formats' appendix of the
COBOL System Reference for more information.
