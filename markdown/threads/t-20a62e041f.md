If you have access to a COBOL compiler, that would be the simplest way to
convert the files.
If not, it seems you need the definition of COMP-3, sometimes called
"packed decimal".
In this usage, every byte except the last one contains two digits.  If you
define each byte as
unsigned char, divide by 16 to get the higher digit and mod by 16 to get
the lower digit
The final byte contains the least significant digit and the sign: x'c' or
x'f' means positive while
x'd' is negative.

x'12345c' = 12345 (positive)
x'00348d' =    348 (negative)
