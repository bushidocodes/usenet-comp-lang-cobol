Most COBOL programs will manage a terminal connection over TCP/IP the same
as a local connection. For a communication port, printer service generally
works giving the device name as the filename. Sometimes sequential file
service will work this way.

If you wish to establish a pipeline via a TCP/IP connection, you should write
a routine in C or assembler callable from the COBOL program and use the socket
libraries to manage the link. You need to know nothing about the Ethernet
card as this is handled at the lowest network communication level.

If you wish to manage communication over a port, again you should write a C or
assembler program callable from the COBOL program. You may be able to use O/S
functions that require little knowledge about managing the port.

COBOL was never intended for communication management. Likewise C does poorly
for large complex record keeping. The two languages can and often are used
together to play their various roles.
