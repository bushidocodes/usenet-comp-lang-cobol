I have several multi-user systems programmed on Cobol which I run
in DOS or under Windows. The main thing is to check for record
locking when you do a read in which you could update the record.
Also watch your Read Next statements. Some Cobols will not update
the record pointer if they encounter a record locked at another
station. Some have a switch or environment variable that will
overcome this problem. On MF-Cobol, you can read next WITH IGNORE LOCK.

By the way, what Cobol compiler are you using ?
I would like to convert some of my systems to run under Linux.
