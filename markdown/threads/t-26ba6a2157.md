When you use a "format 2" RECORD clause and there are multiple length
records, then you *must* define the physical file as RECFM=V or RECFM=VB.
This is INDEED a "variable length file" - although it contains no ODO.  This
is "simply" the way the language is defined.

Once a file has been created with a format 2 record contains clause (whether
it is VSAM or QSAM or SAM on VSE), then it is *always* possible to *read*
that file using the RECORD VARYING IN SIZE format or the RECORD clause.
(Again this assumes an '85 Standard compiler - this was NOT available with
OS/VS COBOL or the first 2 releases of VS COBOL II).  This means that it is
ABSOLUTELY POSITIVELY possible to get the length of each record on a READ
statement.  You are correct that if you keep to a "format 2" RECORD clause
in the program that READs the file, that you can't get the length - but that
is like saying that if you don't use either the MULTIPLY or COMPUTE verb,
you can't do multiplication.  The ABILITY to get this information in in the
language and it WORKS.  It works no matter what type of RECORD clause was
used to create the file - or even if the file was created in a non-COBOL
application. (By the way, it works with *both* VS COBOL II, R3.0 or later
*or* with an LE-enabled COBOL - this is part of the language and has nothing
to do with the compiler or run-time used).

NOTE WELL: from the definition that you gave for format2
  "Fixed-length records are obtained when all 01 record description entry
lengths are the same. "

It simply is not true to EVER speak of a "fixed length file" when there are
records of different lengths (whether via an ODO or multiple 01-levels).
