9/41 is corrupt index file.  The file index needs to be rebuilt.
 
> the same program work well n openserver v5.0.5

Unixware and OpenServer are different product lines.  OpenServer is
derived from Xenix (originally Microsoft Xenix) which was a Unix
edition 7 (ie pre System III) licenced version.  UnixWare is AT&T Unix
via Novell to SCO.  While there has been convergence there are still
differences.

It may be that the locking mechanisms are different such that the
OpenServer run-time fails to lock records correctly for index updates
allowing two programs to mutually corrupt the file.

Or perhaps you copied the file while it was being used resulting in
corruptions.  That is when you copied the first part of the file the
last part was being updated and when it got to the last part the first
bit had been changed which was not reflected in your copy.
 
> can't open more than 286 files

Are you trying to open more than 286 ?
