CICS stands for Customer Information Control System. It's been around
for a long time on IBM mainframes -- I first encountered it in 1979, and
I believe it may have been around in the late 1960's.

IBM's two main operating systems for its mainframes are VSE and MVS, and
they are batch-only operating systems. CICS runs as a single "job" under
either of these operating systems. You can think of it as an operating
system extension. CICS owns multiple terminals and files, and programs
for CICS (whether COBOL, PL/1, assembler or whatever) are first compiled
thru a preprocessor to translate CICS API (Application Programming
Interface) calls into standard calls in the format of the language you
are using.

CICS programs run as subprograms of CICS itself. CICS provides a number
of services to support multiple terminals, including event management,
task management, timers, queues, et cetera. This allows an IBM mainframe
to support online, interactive processing, or even realtime transaction
processing. CICS programs are pseudo-reentrant, meaning that multiple
instances of a CICS online program can be executing concurrently, but
only one copy of the executable code is present in memory. Each task,
however, has its own copy of working storage areas. Each task is
uniquely identified by a four-character transaction ID and 4-character
terminal ID (Terminal ID may be optional in some cases).

A task or transaction in CICS is defined with a starting program, but
that program could link or transfer control to any other CICS program.
Any program access to terminals or files or other CICS services requires
calling CICS with a CICS API call.

I found a CICS newsgroup named bit.listserv.cics-1. You might try
looking at it. IBM also offers CICS for OS/2 and RS6000, probably for
AS400 as well. But it was originally a system 360/370 mainframe product.

It's terminal interfaces are character oriented. I've never seen
graphics on CICS, but maybe someone else has. It is very difficult to
describe CICS briefly. It's like an operating system, but it was built
to provide online interactive services on batch processing mainframes.
