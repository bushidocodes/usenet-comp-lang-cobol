Nice start, but you need to turn the "FD" file description entry to a
"SD" sort description entry in your DATA DIVISION FILE SECTION. You
also need to ASSIGN the "SD" name to the FILE-CONTROL then add two
"FD" file descriptions for the input and output files and ASSIGN their
names as well. Example:

SELECT UNSORTED FILE ASSIGN "UNSORTED"
SELECT SORTED ASSIGN "SORTED".
SELECT DATIFILE ASSIGN "DATFILE"
*
DATA DIVISION.
FILE SECTION.
FD UNSORTED.
01 FILLER PIC X(134).
*
FD SORTED.
01 FILLER PIC X(134).
*
SD DATIFILE.
01 DAT-REC.
02 DAT-TYPE PIC X.
02 DAT-DATE PIC 9(6).
02 DAT-TAKINGS PIC 9(5)V9(2).
02 DAT-REPORT1 PIC X(40).
02 DAT-REPORT2 PIC X(40).
02 DAT-RECORT3 PIC X(40).
*
PROCEDURE DIVISION.
000-START-PROGRAM-RUN.
SORT DATIFILE
ASCENDING KEY DAT-DATE
USING UNSORTED
GIVING SORTED.
STOP RUN.

The remainder of the program to fill in is left as an excersize.

Just my 1.5 cents worth, adjusted for inflation

John,

Depending on the format of your date (eg DDMMYY or MMDDYY) you will need
to amend your sort key so that the year is the major key, month is next
and Day is minor key.

eg

02 DAT-DATE.
03 DAT-MM PIC 99.
03 DAT-DD PIC 99.
03 DAT-YY PIC 99.

SORT DATIFILE
ASCENDING KEY DAT-YY DAT-MM DAT-DD
USING UNSORTED
GIVING SORTED.

This of course will fall apart in just over three years, when your code will
erroneously believe that Jan. 1, 2000 is earlier than Dec. 31, 1999.

It also would make more sense to reformat the data so that year comes
first, then month, then day -- e.g.
01 dat-date.
03 dat-yyyy pic 9999.
03 dat-mm pic 99.
03 dat-dd pic 99.
-- so that you can sort with a single key instead of three.
