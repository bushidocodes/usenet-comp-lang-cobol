I'm trying to make a dynamic (non-DLL) call from a COBOL module  to a C
module passing (BY REFERENCE) some linkage area that the C module will update
and the results will be seen in the COBOL module.  How do I do this?  The
only way I can get the COBOL module to see the C module is by putting in
main() in the C module.  The problem with this is that main() only accepts
argc and argv as arguments.  I can't "update" these and see the results in
the COBOL module.  If I try and call the C functions directly, my COBOL
program abends with "UNKNOWN MODULE".

I'm using COBOL for OS/390 and C for OS/390.  I have exhaustively gone through
IBM book manager books on ILC, C/C++ for OS/390 and COBOL for OS/390.  They
make it seem so simple.  Are they using DLL examples?  Are they static link
examples?  Is it possible to do this with a dynamic, non-DLL, call?

Yes, it's true!

COBOL support for generating and using dynamic link libraries (DLLs) was
introduced in the latest version, IBM COBOL for OS/390 & VM Version 2
Release 1 (Version 1 was called IBM COBOL for MVS & VM).

DLL support is available for applications running under OS/390, in batch,
TSO,  CICS, and IMS environments.  DLL support is not available under
VM/CMS.  The DLL mechanism is common between COBOL, C, and C++.

Refer to COBOL for OS/390 & VM Programming Guide SC26-9049-01.

Gunnar Opheim.

Colin,

Just a thought/suggestion:-

Can you get a static call to work ok?
If so what about "COB-1"    making a dynamic call to "COB-2"
(where COB-2 is simply a small stubb program (no logic in it).
And "COB-2"  has a  static call to "C-Module".

COB-1    ---..>dynamic-call--->>>    (COB-2stub / C-Module)

I don't know what your desired effect is (or problem you want to overcome), maybe
this will give you what you want?

I had an inter-language problem a few months ago and this gave me what I desired.

Good luck
