procedure division.
*****************************************************
* main body
*****************************************************
main-line section.

perform init-app
perform process-the-gui until gui-action = "done"
perform shutdown-app
stop run
.

init-app section.
*****************************************************
* open the file, test for success
*****************************************************
open i-o simidx-file
if si-stat not = zero
display "Unable to open file - status = " si-stat
stop run
end-if
*****************************************************
* initialize the constants
*****************************************************
move function current-date to complete-date
compute rate-per-mo = rate-per-yr / 12
*****************************************************
* initalize the dialog system data and control blocks
*****************************************************
initialize data-block
ds-control-block
move data-block-version-no to ds-data-block-version-no
move version-no to ds-version-no
move 256 to ds-set-name-length
move "test1.gs" to ds-set-name
move DS-NO-NAME-INFO to ds-control-param
move DS-NEW-SET to ds-control
.

process-the-gui section.
*****************************************************
* main loop - processed until gui-action is "done"
*****************************************************
move space to gui-action
call "dsrun" using ds-control-block data-block
if ds-error-code not = 0
display "Dialog System Error " ds-error-code
end-if
move zero to pgm-status
*****************************************************
* evaluate the reason (gui-action) for the RETC
*****************************************************
evaluate gui-action

when "getbal"
perform get-balance
when "delacc"
perform add-account

*****************************************************
* ideally, every RETC in the screenset should set
* gui-action to some value, but the tutorial did not
*****************************************************
when " "
move "done" to gui-action

when "done"
continue

end-evaluate
.

add-account section.
*****************************************************
move account-number to si-account-no
move account-owner to si-owner-name
move zero to si-date
move account-balance to si-balance
write simidx-rec.

get-balance section.
*****************************************************
* read the record the user requested, test for success
*****************************************************
move account-number to si-account-no
read simidx-file key is si-account-no
if si-stat not = zero
*****************************************************
* record not found. Ideally the screenset should test
* the pgm-status field and perhaps display a message box
*****************************************************
move zero to account-balance
move space to account-owner
move 1 to pgm-status
else
*****************************************************
* got the record, update it
*****************************************************
perform calc-interest
add accumulated-interest to si-balance
move curr-date to si-date
*****************************************************
* the following line commented out to support repeated
* testing. Uncomment it to actually update the file
*****************************************************
* rewrite simidx-rec
*****************************************************
* update the info in the data block
*****************************************************
move si-owner-name to account-owner
move si-balance to account-balance
end-if
.

shutdown-app section.
*****************************************************
* close the file, shutdown dialog system
*****************************************************
close simidx-file
move DS-QUIT-SET to ds-control
call "dsrun" using ds-control-block data-block
.

calc-interest section.
*****************************************************
* compute the days between the file date and today
*****************************************************
compute num-days =
function integer-of-date (curr-date) -
function integer-of-date (si-date)
*****************************************************
* convert the days to months
*****************************************************
compute num-mos = num-days / 30
*****************************************************
* calculate the compound interest
*****************************************************
compute accumulated-interest =
si-balance *
function annuity (rate-per-mo num-mos).
