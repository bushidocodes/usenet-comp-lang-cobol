```
One way to be sure that the edit routine is being executed is to put a
unique display in that paragraph.  Your previous example would never
reach F16-CHECK-COURSE because the PERFORM statement was commented
out, and the VALID-COURSE flag was unconditionally set to "Y" after
the commented-out PERFORM of F16-CHECK-COURSE.

Now when you do finally execute your class-test code, you may find
there is a problem with ALPHABETIC.  In every COBOL I have used,
spaces are valid ALPHABETIC class characters, but probably not what
you want for this program.  My guess is that you will somehow need to
exclude spaces in any position in field W-C2.

I would probably define a class in SPECIAL-NAMES containing only
alphabetic characters but excluding spaces, and then use the special
alphabetic class in the IF statement.

> 
> It's definitely reached because if you look at the output file (it's a
…[6 more quoted lines elided]…
> Please refer to test1.log.

Try something like this:

F16-CHECK-COURSE.
    DISPLAY "NOW EXECUTING PARAGRAPH F16-CHECK-COURSE"	
    MOVE "N" TO VALID-COURSE.
    IF (W-C1 NUMERIC) AND (W-C2 ALPHABETIC)
        DISPLAY "W-C1 CONTAINS (" W-C1 ") AND IT IS NUMERIC"
        DISPLAY "W-C2 CONTAINS (" W-C2 ") AND IT IS ALPHABETIC"
        MOVE "Y" TO VALID-COURSE.

This kind of test instrumentation will tell you if F16-CHECK-COURSE is
being executed, and will also tell you if blank characters are allowed
in W-C2 by the ALPHABETIC class test.
```

```
x all;f all debug~;del all nx
```
