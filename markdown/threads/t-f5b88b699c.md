An INITIALIZE only affects named items.  MergedRec contains an unnamed
item (implicitly a FILLER) which will be ignored by the INITIALIZE.  Use
MOVE SPACES TO MergedRec instead of, or before, INITIALIZE, or named the
first field.

This looks suspiciously like a homework assignment.

If it isn't, then the easiest way to merge two files is to simply
append one to the other using the MS-DOS COPY command thus :

         COPY d:\rentalrec+d:\salesrec d:\mergedrec

         The snag with this is your filenames, which in two cases are
nine bytes long, you could "cheat" by renaming them with explorer for
the purposes of using COPY, then rename them back to
their original names afterwards.

If it is a homework assignment, then what I suspect is required is to
merge the two files in key sequence, where ITEM-IN-SALES and
ITEM-IN-RENTAL are the key fields for the input files and
OUT-ITEM-IN-SALES is the key field for the output file.  If so, then
the COBOL MERGE statement would be the best way of doing it, using an
OUTPUT PROCEDURE to create the output records from the merged input
and putting spaces in the leading three byte filler field of the
output file.  Alternately, it  might be that the task is somewhat more
complex, requiring you to produce an output file that represents the
stock held after accounting for the sales made.  In this case you
would have to provide matching logic in your program code to ensure
that sales and rental records tie up even when there are missing
records for particular items in one or other of  the two input files. 
This would include the necessity to handle the situations where the
first two records don't match and/or  where the last two records don't
match.  For matching records you would have to add the fields
representing numbers of items together to produce the output result.

However, these potential solutions are speculative, we would have to
see the problem specification, for example it may be that the
video-names are the key fields.  I can't see from your program whether
the item fields are identity codes or numeric quantities.
