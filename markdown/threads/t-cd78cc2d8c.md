The draft standard says that Binary-Char must be able to
hold values in the range -127 through +127. An implementation
with 9-bit characters could allow a range of -255 through +255.
If a program depended upon the larger range of the 9-bit
implementation, the program could not be ported to an
implementation with 8-bit characters, without modification.
If the programmer did not exceed the ranges given in the
draft standard, the program would be portable.

In any case, the data may not be portable due to endian-ness.
