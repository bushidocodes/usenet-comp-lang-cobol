```
My application is MSDOS-based, and developed
using MF cobol. No problems when running under
DOS, or windows3.1.

Under Windows95 (in a DOS-box), everything is fine
except for a very long delay before printed output begins.
It can be anywhere from 30 seconds, to 15 MINUTES
before printing begins. Once printing begins, it completes
normally.

I realize that W95's print spooler is noted for being slow,
but this is ridiculous. It appears the print spooler is waiting
some time-out period to see if more data is coming into
the buffer.

Also, I have other DOS-based apps, and although their
printing is slower than under regular DOS, they do not
experience anywhere near the delay of my MF-COBOL
application.

I noticed that larger print jobs come out sooner, so I
suspected that some buffer must be waiting to be filled.
I tried sending *lots* of nulls to fill the buffer, but this had
no effect, good or bad.

I have also noticed that if I exit the application, printing
begins immediately. Apparently, exiting the app closes
the spooler buffer. I AM CLOSING THE PRINT-FILE WITHIN
THE APPLICATION, so I don't know what else I can do
from within the cobol app.

Is there anything (maybe a system call?) that can be
issued from within MFcobol to FLUSH the print spool buffer?

Maybe there is some configuration command to reduce the
wait time for the buffer to fill, before printing?

In my SELECT clause, I am opening the file "LPT1".
Is there some other device name I could open, that would
bypass the print spooler altogether?

don ferrario
```

```
Hey Don.

We had the same problem at my company, but have since resolved it.
You need to do the following:

(1) Use the +P Run-Time switch
(2) In the properties for the printer - capture LPT1:, and set the
spooler to "Start printing after the first page is spooled"

It seems that if you close the printer port after printing a job
(instead of leaving it open and asking to print more), it lessen the
amount of time it takes to print smaller jobs.

Anyhow, try that, that worked for us.
D.
```

```
I saw a similar non-COBOL problem with Win95 to a printer, via an
automatic switch, where data was lost - the printer buffer filled,
then stopped, then it printed half a page with a chunk of data
missing.

Adjusting Win95 for direct printing seemed to fix that one:
Start, Settings, Printers, [Select printer]
[Right click] Properties, Details, Port Settings
then unset "Spool MS-DOS print jobs", OK, OK.

It might not work to a printer over a network, though.
```

```
though you close
```
