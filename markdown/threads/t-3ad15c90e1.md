David White wrote:
› Is there a way to initalize a variable decalred as binary to a binary
› value using the value clause?
I'm assuming the compiler is not letting you specify VALUE x'FF' (your
own value in the quotes'. It may be because the BINARY type is the same
as COMP in VS COBOL II. You could try defining an alphanumeric field,
setting the value on it and then redefine that field as the binary one.

Ex.

05 ALPH-1 PIC X(01) VALUE X'FF'.
05 BIN-1 REDEFINES ALPH-1 PIC 9(01) BINARY.

I'm not that familiar with BINARY type, but I hope this helps.

David James

David White wrote:

› Is there a way to initalize a variable decalred as binary to a binary
› value using the value clause?

This should be simple depending on the compiler you are using. With the compiler
I use all I have to do is provide a value clause. If I want to initialize the
BINARY to some other value then I can use the HEX representation of the value
and place that in the value clause. For example:

* Puts the value 65535 into the binary
77 one-binary PIC 9(11) VALUE 65535.

* Places the HEX value BADBADBAD into the binary
77 two-binary PIC 9(11) VALUE @BADBADBAD@.

Remember that not are binaries are similiar. The binaries will differ depending
on what platform you are using. The binaries declared about are 48 bit binaries.
