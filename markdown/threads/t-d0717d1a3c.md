Most legacy code will not include Object Oriented COBOL, since it is
quite new. In addition, OO COBOL (unlike most other OO extensions) is a
strict superset of the previous version of COBOL. That means you could
learn regular COBOL and leave the Object stuff until later, when you
really need it...

COBOL is easy enough that if you need more than a few days and some
existing code to look at, something is wrong. (Assuming ordinary
knowledge of languages learned in years of work in the field.)
