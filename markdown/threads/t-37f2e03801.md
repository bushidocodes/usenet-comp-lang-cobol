You knew I had to respond <G>. Hmmmm. Softly, softly, catchee monkey. Obviously
don't go for a 'biggie'. Let's take your recurring theme, maintenance and the
necessity of testing umpteen programs as a result of a change.

I don't knw how you are setup or your breakdown between  the following :-

1 - calling 'functions' via sub-programs or stand-alone programs
2 - copyfiles, slipped into place in an exisiting program with a routine (
copies of  PERFORM(s) doing functions).

There's no immediate merit in trying to do #1 but #2 is an 'OBVIOUS'. If you do
use #2, is it perhaps also broken down into separate copyfiles - do some of
these 'separates' have an association ( in the lingo 'are they operating on the
same type of object' ?). Given say, that you had copyfiles which are the
equivalent of my routines for Imperial/Metric and Metric/Imperial conversions,
that would fit ideally into OO.

Hopefully you've picked up from me and others that you can have instances, one
source but many different 'carbon copies' functioning. My above Conversion
routines are written this way - purely because I hadn't realized that there can
be a difference between FACTORY methods and INSTANCE (Object Section methods).

In essence, you ONLY need instances when you are creating different objects to
invoke a common class. Example, I need three screens in a program (Dialogs) - I
can invoke the common support class MyDialog three times :-

invoke MyDialog "new" returning Dialog1
invoke MyDialog "new" returning Dialog2
invoke MyDialog "new" returning Dialog3

They all reference the class MyDialog, but I have three using the common methods
contained in the OBJECT SECTION of the program MyDialog.

(Now - I only truly got the significance of when you would use instances, from a
message that Bill wrote. Instead of the Bible on his bedside table, he has a
copy of the J4 Standards  <G>).

Now back to you invoking classes and my Conversion class above. As written it
does invoke "new" - but it need not have be written that way. You aren't
interested in several instances of Conversion, one will suffice for your needs.
A class can be designed to ONLY contain methods in the FACTORY - WITHOUT any
invoking "new", (which has to be a Factory Method), plus the program doesn't
have to have an active OBJECT SECTION. So........

*>----------

Class-id.                    ConvertValues
                                  inherits from IBM-Base (whatever it is)

Class-Control            ConvertValues is class "convert". *> = name of your
source
*>----------------------
FACTORY.
*>---------------------
       Method-id. "ImpToMet".
       *>-------------------------------------------------------------
       Linkage section.
       01 lnk-inches                   pic 9(03)v9(03) comp-3.
       01 lnk-mms                      pic 9(03)v9(02) comp-3.

       Procedure Division using     lnk-inches
                          returning lnk-mms.

          if  lnk-inches <> zeroes
              compute lnk-mms rounded = lnk-inches * 25.4

          else move zeroes to lnk-mms
          End-if

       End Method "ImpToMet".
       *>-------------------------------------------------------------
       Method-id. "MetToImp".
       *>-------------------------------------------------------------
       Method-id. "CtoF".
       *>-------------------------------------------------------------
       Method-id. "FtoC".
       *>-------------------------------------------------------------
       Method-id. "KpaToPsi".
       *>-------------------------------------------------------------
       Method-id. "PsiToKpa".
       *>-------------------------------------------------------------
       OBJECT.
       *>-------------------------------------------------------------
        *> Note : doesn't contain any methods
       *>-------------------------------------------------------------
       END OBJECT.
       *>-------------------------------------------------------------

       END CLASS ConvertValues.
       *>-------------------------------------------------------------

You want to use the 'Imp-to-Met" method from an exisitng program :-

*>---------------------- your caling program ---------------

Program-id.            ?????

Class-Control            ConvertValues is class "convert".

Working-storage Section.
01 ws-inches                   pic 9(03)v9(03) comp-3.
01 ws-mms                      pic 9(03)v9(02) comp-3.

Procedure Division.

move in-value to ws-inches
invoke ConvertValues "Imp-to-Met" using ws-inches returning ws-mms

move in-value2 to ws-mmms
invoke ConvertValues "Met-to-Imp" using ws-mms returning ws-inches

- See - No object reference - you are invoking a Factory method directly by the
name ConvertValues.

Food for thought - and it is a starting point. Sing out for more help as
necessary.

PS: Should you get a copy of Will Price's book "Elements of OO", for tutorial
purposes and to avoid confusing students, he tends to do it all in the FACTORY.

Jimmy, Calgary AB

   DO NOT try, use, play with, or anything else

try working with VisualAge COBOL.
