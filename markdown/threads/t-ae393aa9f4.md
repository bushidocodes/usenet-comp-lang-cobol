the DOS market.

correctly what I've read here in the past, you neeed the LINK.EXE command from
some older version of MS-DOS.

COBOL 650 is not the only COBOL compiler for MS-DOS. The following
compiler companies all have DOS versions of their compilers:

Acucobol
mbp Visual COBOL (now owned by Micro Focus)
Micro Focus (They even have a "Personal COBOL" compiler,
cheaper and it works in MS-DOS)
CA-Realia
Ryan McFarland

I may not have covered them all, but it appears that there are many more
choices than the COBOL 650 compiler.

If you decide to continue using the 650 compiler, I think you can use the
linker from MS-DOS 3.3 and earlier. You can also find more information
from the COBOL FAQ web site.
