### Alternative to ELIF

- **Date:** 1998-12-05T00:00:00
- **Newsgroups:** comp.lang.cobol
- **Message-ID:** `<36695DD8.4D90@swbell.net>`

```
I'd like to propose another way to tame the wild END-IF: bracket a
complex set of conditionals with PERFORM and END-PERFORM.  E.g.:

    PERFORM
      IF A ...
      ELSE IF B ...
      ELSE IF C ...
    END-PERFORM

This is not my idea -- someone emailed it to me.  He didn't want to 
post it himself, or let his name be used, because he didn't want to 
be pestered with requests for homework assistance.

Admittedly this syntax looks bizarre.  As a perversion of the intended 
use of PERFORM/END-PERFORM, it is likely to confuse a maintenance
programmer who is not familiar with it.  That said, let us consider
its advantages.

The example is a simple IF/ELSE chain which you might just as well
code with EVALUATE TRUE, or (if it were available) some form of ELIF.
However, the PERFORM/END-PERFORM could enclose any arbitrarily
complex set of conditionals, including stuff you couldn't code with
EVALUATE or the proposed ELIF.

The END-PERFORM terminates the scope of this complex conditional, but
it is less of a blunt instrument than a period would be.  A period
terminates all scopes, bringing you abruptly to the outermost level.
By contrast, the END-PERFORM only brings you back to the level of the
corresponding PERFORM.  Thus this construct gives a degree of 
flexibility not otherwise available except through the (possibly 
lavish) use of END-IF.

I haven't used this construct myself, and I can't decide if it's ugly
or elegant.  Let's have a long interminable argument about it.

```
