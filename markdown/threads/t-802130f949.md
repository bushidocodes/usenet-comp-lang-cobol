Sorry if this question is too silly. But since I do not know much about 
Cobol I don't know this (yet).

When writing database applications, host variables are used when 
sending/retreiving data from the database.

For regular Latin-1 character data, the typical Cobol declaration looks 
like PICTURE X(10).

But how are the host variables declared for national character data 
(Unicode), UTF-16 and UTF-32 encodings?

What I'm looking for is something like C's wchar_t.


TIA,
Jarl

Hi Jarl,

It's not a silly question. In Net Express 4.0 (the only COBOL development
tool I know of that supports Unicode) it would be defined for UTF-16 like
this:-

01 mystring	pic n(20) usage national.

Moving a PIC X(20) to PIC N(20) would change the ASCII (or EBCDIC as
applicable) to UTF-16 for you.

Short answer for IBM Enterprise COBOL,

a) compile with NSYMBOL(NATIONAL)
b) declare data items with usage NATIONAL and a PIC clause that uses N in the
same way DISPLAY uses X
c) declare literals as n'....' or nx'...'
