This problem is a perfect example for using the new 1985 standard addendum
Intrinsic Functions. Lets hope you have a current COBOL compiler and run
time to use for this assignment. Current Micro Focus products, support
this, as well as IBM's latest offerings on MVS and PC's.

One could use FUNCTION INTEGER OF DATE or FUNCTION INTEGER OF DAY to
convert the transaction date (from input) to a Jan 01, 1601 based
sequential day. (Numeric value between 1 and 3,067,671).

Use FUNCTION CURRENT-DATE to obtain a 4 digit year calendar date YYYYMMDD
plus more.

Use FUNCTION INTEGER OF DATE to convert the YYYYMMDD to the same
sequential day format.

At this point all leap year, year 2000 (and year 2100) etc provisions are
taken care of (assuming of course that the input has a 4 digit year). In
fact the program will work until either the transaction date or
current-date goes beyond December 31, 9999 (yup, 8003 from now).

A simple IF statement can now compare for the 60 day window, since all you
have are accurate sequential day information, and all leap year etc have
been handled.

Sounds like the instructor is trying to show the power of Intrinsics -
good idea.

Rex Widmer
