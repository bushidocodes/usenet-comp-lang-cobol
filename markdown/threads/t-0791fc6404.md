Mr. Robert,

If you are using Line Sequential files created by editors, try to append

FILLER PIC x(001)

at the end of your record definition in the FD. The record separator (LF
and/or CR) will be ignored and the next READs will proceed OK.

Hope this can help you and let me know.

Nilo mailto:npaim@zaz.com.br

    When reading user created line sequential files, it is always wise to
add several extra fillers
to the end of the record definition.  Most cobols will split a long input
record into 2 or more
records if the program defined record is too short.

    Also, the record separator varys from unix to "dos" (dos/win95/98/nt,
etc).  If you create
the data file in one world, and use it in another, you will have problems.
There
are many easy ways to fix this problem.  sometimes the runtime can be caused
to accept
files with the "wrong" format with a runtime switch.
