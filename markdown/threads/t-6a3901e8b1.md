You sure you understand this stuff?

You cannot have a "real program" which uses "WITH TEST AFTER" without the
words "WITH TEST AFTER" appearing in the program, so you can forget about
that one.

If you read the syntax diagrams (all flavors of COBOL), you will see that if
you do not specify "WITH TEST BEFORE" or WITH TEST AFTER", then "WITH TEST
BEFORE" is assumed.

This loop...

PERFORM [WITH TEST BEFORE] UNTIL condition
....
END-PERFORM

... may not ever execute: if condition is true when PERFORM is encountered,
the loop will never execute.

OTOH...

PERFORM WITH TEST AFTER UNTIL condition
...
END-PERFORM

... will *always* execute at least once.


MCM
