OK so it's mainframe.  Generally mainframe applications link the
output to a device or file using an external language like JCL in
MVS's case.

If they can proide a printout then they can modify the job
commands to put that data to a file and once they have the file
put in on a compatible device or FTP it to your server.  Sounds
like a lot of fun interpretting this stuff.

If they are MVS and have a report using SDSF try:

((preallocate a file named TEMP)

PRINT ODSN  TEMP * SHR
PRINT
PRINT CLOSE

(FTP temp to wherever)
