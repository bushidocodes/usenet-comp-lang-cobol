Whining aside, I can see how two PCBs would work for the case where
nesting is only one layer deep, but what about the case with more
layers? Seems to me you'd need a PCB for each layer and this means you
must guess in advance the number of layers you'll find.

Another way might be to create an array in working storage to hold
concatenated keys and use it like a FIFO stack - while GNPing if an
A002 is also an A001 push its c.k. onto the stack. Once GNP ends with
GE pop a c.k. off stack and load into SSA-A001, do GU then begin GNPing
again - this would be recursive and only require one PCB.

For example, given a BOM where each letter is an assembly and each assembly
is made up of some subassemblies (shown) and detail components (not shown).

                      | A
                      |
       _______________|________________________________
      | B       | C        | D         | E             | F      
      |         |          |           |               |
 _____|__      _|___      _|_____      |_______      __|______
| G      | H  | I   | J  | K     | L   | M     | N  | O       | P

To traverse the BOM, first load A into the FIFO stack.

Then call an outer loop (PERFORM -- UNTIL paragraph) that pops a key off
the FIFO stack, GU's it for positioning then calls inner loop PERFORM.
The inner loop PERFORM repetitively GNPs through each A002 occurance.
(now for the beauty part!) and if the A002 occurance is itself an assembly
pushs that key onto the stack.

By my thinking, after processing A, the stack contents should be:

	B - C - D - E - F

pop off B and process again, at end stack contents should be:

	G - H - C - D - E - F

pop off G and process again, at end stack contents should be:

	H - C - D - E - F

and so on and so on until stack is empty.

This would need only one PCB and one FIFO stack to work.

So of course the next question is, how do I implement such a FIFO stack
in COBOL?
