```
It is called 'EVALUATE'

the syntax is
EVALUATE conditional-expression    (conditional-expression can be level 88,
or ws-var = 'X', or
anything that can be coded 'IF ' at same location)
WHEN [TRUE | FALSE | ANY]  (any = I don't care)  [|] means pick one
    imperative statement ( DISPLAY, PERFORM, CALL, GO, ADD, etc)
WHEN [TRUE | FALSE | ANY]  (any = I don't care)  [|] means pick one
    imperative statement ( DISPLAY, PERFORM, CALL, GO, ADD, etc
....
WHEN OTHER
    imperative statement ( DISPLAY, PERFORM, CALL, GO, ADD, etc <== Optional
END-EVALUATE.

Evals can be nested
So example --==
01 WS-SWITCH      PIC X.
     88  ALL-DONE  VALUE 'Y'.
     88  NOT-DONE  VALUE 'N'.

EVALUATE  WS-SWITCH
    WHEN 'Y'
        DISPLAY 'DONE'
    WHEN 'N'
        DISPLAY 'MORE'
    WHEN OTHER
        DISPLAY 'Oh Crud!'
END-EVALUATE
 can ALSO be Coded
EVALUATE  ALL-DONE
    WHEN TRUE
        DISPLAY 'DONE'
    WHEN FALSE
        DISPLAY 'MORE!'
END-EVALUATE
or this way
EVALUATE  TRUE
    WHEN ALL-DONE
        DISPLAY 'DONE'
    WHEN NOT-DONE
        DISPLAY 'MORE'
    WHEN OTHER
        DISPLAY 'Oh Crud!'
END-EVALUATE

```
Here are a few simple examples.  One book I have lists EVALUATE the other
doesn't list it at all.  This *should* work on most compilers.

EVALUATE  field
    WHEN value1
        things to do
    WHEN value 2
        things to do
    WHEN OTHER
        things to do
END-EVALUATE.

EVALUATE TRUE
    WHEN field-a = field-b
        things to do
    WHEN field-c = field-d
        things to do
    WHEN OTHER
        things to do
END-EVALUATE.

EVALUATE field
    WHEN 1000 THRU 9999
        things to do
    WHEN 10000 THRU 99999
        things to do
    WHEN OTHER
        things to do
END-EVALUATE.

You can obviously make them more complex, however this should be sufficient
to start you off.

-Steven Kalemkiewicz
