file functions, including deletes.  It varies from one implementation to the

from the CBL_ Subroutines manual you might want to try this

CBL_DELETE_FILE2

CBL_DELETE_FILE2 deletes a file.

The difference between the CBL_DELETE_FILE subroutine and
this subroutine is that this subroutine can support a file name
containing a space.

Specification
The following explains how to specify CBL_DELETE_FILE2.

Parameter data definition
01 file-name PIC X(n).
01 status-code PIC S9(4) COMP-5.

Chapter 4. File Routines 71
Calling format

CALL "CBL_DELETE_FILE2" USING file-name
         RETURNING status-code.

Interface

file-name

Specifies a file name ending with a null character. If no path
name is specified, the current directory is assumed.

Return code
The return code is set in the status-code specified in the
RETURNING clause.

status-code
0: Successful
Other than 0: Failure

Under Windows, there are no control codes like this: everything is
virtualized. You select the fonts, colors, type size, features etc and
"paint" a page. It does not matter to the programmer/painter ( L'artiste? )
what the device is: it could be a screen, a printer, a plotter, or anything
which uses what is called a "device context."

Windows - or more properly in the case of a printer, the Driver for the
printer - handles any and all control codes necessary to render the virtual
page on the physical device. No font in the size specified? No problem:
Windows has already selected  (if you let it) the "closest matching font"
supported by the driver. No color? No problem: the printer driver will make
it grey-shaded.

While more difficult to learn than text mode printing, Windows printing is
an awful lot more flexible than "pure" COBOL or other "character file"
report generation.

For printing I use a third party shareware library called DDOC, which
consists of a DLL and an EXE file, and includes a "preview" engine. There is
no reason you could not call that library from your COBOL program. The
registered version of this library will cost you all of $22.00. You can
download the shareware version at www.basicguru.com/dickinson or some link
available from Don Dickinsons' page at "The Home of the BASIC Gurus."

The header files would need modification, but all Don uses to pass data are
null-terminated strings and 32-bit signed integers, both of which you can
create in COBOL.
