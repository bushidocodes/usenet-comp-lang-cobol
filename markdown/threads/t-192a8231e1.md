  This is correct. Unless you want to fiddle with the actual bitwise  
representation of the numeric values in memory, which would open up  
another can of worms -- don't do it.

  Actually, when the program can't generate the data at program start  
by some function, deriving the values depending on the indexes, I  
would read in the data from a file at run time. You might have a day  
where these values change, and then it is easier to change the  
parameter file instead of the program. On the other hand, having these  
parameters hard coded in the program itself, you have one file less to  
worry about.

  BTW, you don't have to invent names for the redefining data items,  
just name them all FILLER, or even leave that out, if your compiler  
adheres at least to COBOL-85.
