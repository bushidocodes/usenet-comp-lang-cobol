You'll find procedures defined in the screename.cbl file that is generated
from teh Screen Painter.  You can edit this (even though it says not to !).

Be warned, unless you tell the Screen Painter not to generate the code file
(under options, you can define which documents will be generated), then you'll
overwrite any code changes you make if you change the screen and regenerate.

It should be fairly straight forward from there.

You can avoid overwriting your code changes by going into Tools, then Options
and under the Generate Document tab, turn off Program File (the .cbl file).
Then, only the copy libs will be generated.  Mind you, it won't add any
paragraph headings etc to the .cbl file, so you'll have to keep an eye on that
(it's not usually a problem).

Hope that made it a little clearer.

there. You should be able to follow the variables in the Working Storage
(.wrk) and see the code in the Procedure Division (.prd).

The easiest way would be to compile the screen code with the listing option -
it'll give you a list of all the code from the copy files etc.
