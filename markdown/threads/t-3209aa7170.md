```
To save others from confusion:

We have a cobol/oracle program that reads a table name from standard
input and creates a copybook into standard output. When we tried to
run it from inside a script, we got:

Execution error : file 'direct'
error code: 197, pc=0, call=5, seg=0
197 Screen handling system initialisation error. 

Documentation says that this is the result of a corrupted terminfo
file. Since we had this running without error on another machine with
an identical terminfo directory, that didn't seem likely. We
eventually discovered that the COBSW variable was set to +A on the
machine that wasn't working. When we CLEARED it, our script worked
right.

Moral of the story: error 197 can also mean that you are attempting to
run the cobol animator from inside a shell script.
```
