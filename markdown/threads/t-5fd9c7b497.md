```
Arthur,
When I got a call a number of years ago to do some IMS programming, I 
just read the entire IMS Applications Programming manual.  I found that 
having done so, I was "overqualified".  The "experts" on the project 
said things like "Don't use IMS path calls (which obtain multiple data 
base segments in one call); we don't do that."

Well, of course, being an independent sort, I did what seemed to make 
sense.  Before too long, the "experts" were discredited, because they 
had never really learned IMS coding, they just did what they had seen in 
other programs.

You should be aware that IMS/DC (Data Communications) is no longer the 
current term; it is IMS/TM (Transaction Management).
```
