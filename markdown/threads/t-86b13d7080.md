It would be helpful to know what kind of 30 errors you are getting. A call
to C$RERR using either a four or eleven character parameter will return the
extended error code.  Knowing what kind of I/O statement is returning the
error would be helpful also.

What version of RM/COBOL are you using?  Version 6.51.00 is the current
version for Window 95.

If you are using Novell's Netware Client32 for Windows 95 instead of
Microsoft's client for Netware make sure that you are using the most current
version.   RM/COBOL version 6.5 for Windows 95 requires version 2.11 of
NetWare Client32 for Windows 95 (dated 8/21/96, file size 461,359 bytes).
Record locking does not work properly and file corruption may occur with
earlier versions of Client32.  Additionally there may be open errors (94,xx)
or other permanent errors (30,xx) with older versions of Client32.
Receiving error 30, MS Windows error 1 (30,12,00001 from C$RERR) may
indicate
 a broken version of Client32.

If even one client on the network is using an older version of Novell's
Netware Client32 then you may be seeing these problems.  A simple test to
ensure record locking is working properly is to run the verification program
pacetest.cob from two clients accessing the files on the server.

Hope this helps.

Robert Heady
