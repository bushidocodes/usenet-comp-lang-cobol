IBM COBOL/2 was written by Micro Focus. As far as I remember, it`s
identical to Micro Focus COBOL/2. It might have a couple of IBM tools
thrown in though as did the Microsoft version. It should be reasonably
compatible with the current product. ie. code on cobol/2 should be
compatible with Micro Focus COBOL 3.2. The reverse will not be as true
if you use any of the features added in those 7 years.

I`m not sure of the policy on being compatible with a 7 year old
product. So don`t sue me if you expect to take index files from cobol/2
and run them on 3.2.

I`m sure someone else here at MF should be able to answer this exactly.

›
› I was looking for something to practice and compile at home. I usually buy
› older software because a) It's cheap. b) It runs fast on my 386DX/25.

Should be just perfect for your purposes however, the current base
compiler will probably be quicker still although some parts of the
product now require more memory and benefit from a faster system due to
more features. I run our current OS/2 PM based Animator on a 386DX/25
with only 8Mb and whereas it's not as much fun as on my DX4/100 it's
perfectly usable.

›
› I've never seen this Cobol/2 before. Or if I have, I have forgotten. What
› have I got here and are there any major caveats to using it?

If as you say, used MF COBOL Workbench 4 years ago, you may have used
COBOL/2 without knowing as we only changed the name to just COBOL about
3 to 4 years ago.

The product has a very traditional Microsoft style command line driven
interface.

eg, to compile a program, it`s

COBOL program;
LINK program,,,LCOBOL.LIB/nop; etc.

If you have an early version, the OS/2 EXE`s are preceded by a "P" so you
have PCOBOL.EXE. At that time, we couldn`t do dual mode executables.

› BTW, I paid a whopping 50 cents for it at a library used book sale.

Bargain. that`s $49.50 cheaper than our cheapest compiler. (Personal COBOL)
