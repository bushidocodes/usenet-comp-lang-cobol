As I point out in the document at http://www.flexus.com/ebd2asc.html, to
which I referred Mr Fidel Davila Villagomez, a knowledge of the COBOL
PICTURE clause is
*absolutely required* to do any type of conversion, either EBCDIC to ASCII
or COBOL to IEEE datatypes.

But, hark! the package I supply with the DLL includes a function to parse a
COBOL  PICTURE clause and generate the necessary information to do later
conversion; a function which means you can use this software without knowing
squat about COBOL.

 From the documentation I supply:

  ====
   The COB2IEEE function is called as follows:

         CALL COB2IEEE (parm)

  Where parm is the 32-bit address of data block (a C struct, a BASIC
User-defined-type,  or a COBOL group item). (i.e., parm is passed by
reference).

The BASIC-language UDT looks like this:

TYPE ParmBlock
  Action       AS LONG
  DataInfo     AS DataInfoType
  ParseInfo    AS ParseInfoType
  ConvertInfo  AS ConvertInfoType
  STATUS       AS LONG
  ErrorMessage AS ASCIIZ * 256
  Version      AS STRING * 32
END TYPE

  ***********************************************
      LONG         = 32-bit signed integer
      ASCIIZ       = Null-terminated string
      STRING * nn  = Fixed size buffer
      xxxxType     = another data block
  ***********************************************


                               USING COB2IEEE TO PARSE

To Parse, the user supplies

  Action    = %C2I_Action_Parse
  ParseInfo = a completed ParseInfoBlock

COB2IEEE Returns:

Status  zero    : Parse was successful; a completed DataInfo block was
returned
Status  non-zero: Parse failed; a text message is in ErrorMessage

TYPE ParseInfoType
  Picture      AS ASCIIZ   * 32
  Usage        AS ASCIIZ   * 32
  SignIs       AS ASCIIZ   * 32
  Align        AS LONG
END TYPE


Picture  is the COBOL PICTURE without sign or usage. Examples:
   S9(04)V9(2)
   X(15)
   999999V9999
   A(4)


Usage is the COBOL usage clause. Examples:

   spaces/null (default value, for numeric, this will be USAGE IS DISPLAY)
   DISPLAY
   COMP
   COMPUTATIONAL-3
   COMP-1

  * Usage COMP-4, COMP-5 and COMP-X are supported only for the parse
function.
  * Usage COMP-1 and COMP-2 assume IEEE standard single and double precision
reals.
  * Edited numeric fields are supported; see notes re conversion of these
fields.

 NOT SUPPORTED:
   Scaled decimals ("P")
   USAGE POINTER
   USAGE PROCEDURE-POINTER
   OCCURS/REDEFINES
   RENAMES
   SYNC(HRONIZED)

SignIs is the COBOL sign clause. Examples:

   space/null  (default value)
   LEADING
   SEPARATE
   LEADING SEPARATE (if both LEADING and SEPARATE are used, must be
                     separated by one or more spaces).

Align is the alignment type to use for COMPUTATIONAL data.

   %C2I_Align_Byte - COMP data will be allocated exactly as many bytes as
required; no synchronization or padding bytes will be added
   %C2I_Align_Word - COMP data will be allocated either 2, 4 or 8 bytes of
storage.

   The default is WORD alignment.

   IBM mainframe COBOL uses Word alignment.
   Microfocus Personal COBOL on a PC uses byte alignment.


If COB2IEEE succesfully parsed the input, it returns a status of zero and a
filled DataInfo Structure. If status is non-zero, a parsing error occurred
and an
error message may be found in the ErrorMessage field.


TYPE DataInfoType
   DataSize     AS LONG
   DataType     AS LONG
   Digits       AS LONG
   Decimals     AS LONG
   IsSigned     AS LONG
   SignLeading  AS LONG
   SignSeparate AS LONG
   Align        AS LONG
END TYPE

DataSize      is the number of whole bytes used by the data item.
DataType      is a constant; see table
Digits        is the total number of digits (numeric fields) or characters
              (alphanumeric fields) in the data. For Alphanumeric fields,
              this will be the same as the data size.
Decimals      is the number of implied radix ten decimals in the PICTURE
IsSigned      is true (-1) if the data have a sign associated with it.
SignLeading   is true (-1) if the sign is leading (overpunched or separate)
(default=trailing)
SignSeparate  is true (-1) if the sign is separate
Align         will be the same as the alignment supplied in the parse
function,
              unless an invalid alignment was supplied, in which case
COMPUTATIONAL
              data will have WORD alginment and all other data will have a
zero
              value for alignment. No warning or message is given.

 === end of excerpt ====
