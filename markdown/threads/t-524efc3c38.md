```
QUESTION.

How do we run large Micro Focus Cobol programs, too big to fit
in Base Memory, which make use of embedded SQL for an ORACLE
database ?
We are using DOS Ver 6.22 on our client workstation. We cannot
move to Windows or OS/2, because we use some drivers which are
available in the DOS environment only.

SPECIFICATIONS.

We are using Micro Focus Cobol ver 3.0.51 Programs to interface
with an ORACLE database (Ver 7.0).
In order to communicate with Oracle we need to load the
following Drivers :
SQLSPX (52K)
SQLPME (81K)

These two drivers cannot be loaded HIGH (i.e. in extended
memory) and therefore leave us with 471K Conventional memory.

Neither the use of the XM utility nor overlays are helping to
solve this problem. The procedure division is not that large
anyway.
Even running under the Shared Run Time system is not helping.

We suspect that the main problem with memory usage is the amount
of libraries we need to link with the executable.
These include :
1) Communication Libraries for a Unisys
DP1800 Reader/Sorter (DOS version only).
2) SQLLIB for embedded SQL.
3) ORA6DOS for Oracle.
4) ADIS,ADISINIT,ADISKEY - Extended Displays.
5) MFC7INTF,C7DOSIF,C7DOSLB.
6) The standard LCOBOL & COBAPI.

If we remove the extended displays (i.e. screens) and therefore
do not link in the three ADIS libraries, most of the programs
execute without a problem. With the displays included, we find
that either the system freezes up during program execution or
we receive the message "PROGRAM TOO BIG TO FIT IN MEMORY".

We would appreciate any ideas that may help us solve our
problem. Please send your comments to :

MANY THANKS MR R.DICKSON ( RAYMOND ).
```
