It's been quite a while since I've been programming in COBOL. I have this
table :

03  WS-A-FKC-FKM                 PIC 9(4) COMP
                                   VALUE 0.
03  WS-TABEL-FKC-FKM             OCCURS 1 TO 1000 TIMES
                                 DEPENDING ON WS-A-FKC-FKM
                                 ASCENDING KEY
                                           IS WS-TABEL-DEP
                                 INDEXED BY WS-INDEX.
    05  WS-TABEL-DEP             PIC X(3).
    05  WS-TABEL-REDEN           PIC X(2).
    05  WS-TABEL-BEGIN           PIC X(8).
    05  WS-TABEL-EINDE           PIC X(8).

that is filled out with a number of elements. I need to initialize it.
WS-A-FKC-FKM needs to be 0 again, but I also want all the elements to be
removed. The results of my program should be exported to a PC-application,
so I want to avoid all LOW-VALUES.

Probably it's not very difficult, but I can't remember how it's done.

Thanks in advance,

Philippe

Judson,
  Today seems to be my day for disagreeing with you <G>  Usually, you are
completely accurate, BUT ...

Being able to INITIALIZE an 01-level above WS-A-FKC-FKM in an extension that
some - but definitely not all compilers allow - because it has a subordinate
OCCURS DEPENDING ON item.  (I don't think anyone would allow an INITIALIZE of
WS-A-FKC-FKM without any index/subscript.) The "portable" way to do this is:

 MOVE 1000 TO WS-A-FKC-FM
 Perform
   Varying WS-Index
     from 1 by 1
     Until WS-Index > 1000
        Initialize WS-TABEL-FKC-FKM (WS-Index)
  End-Perform
  Move Zero to WS-A-FKC-FM

P.S.  Before anyone asks,  "Simply" Moving 0 to WS-A-FKC-FM *may* clear out
all the elements of the table - for future use.  I know of some compilers
that work that way - but the Standard is pretty clear that what will be in
them during the next time WS-A-FKC-FM gets greater than zero is "undefined".

P.P.S.  I think the programmer needs to change
     Occurs 1 to 1000 times
           to
     Occurs 0 to 1000 times

If, OTOH, there IS always at least one element, then the "zeroes" in my
sample need to be changed to 1.
```

The COBOL 85 standard says in paragraph 6.17.3:

(4) The description of the data item referenced by identifier-1 or any
items subordinate to identifier-1 may not contain the DEPENDING phrase
of the OCCURS clause.

I had forgotten this restriction.  So the table as it stands cannot be
initialized using the COBOL 85 INITIALIZE, my method or yours. :-)

In the case of this particular table, containing only alphanumeric
fields, I would just do this:

MOVE 1000 TO WS-A-FKC-FKM.
MOVE SPACES TO WS-TABEL-FKC-FKM.
MOVE 0 TO WS-A-FKC-FKM.

But to specifically get around the restriction, you could use something
like this:

03  WS-A-FKC-FKM                 PIC 9(4) COMP
                                   VALUE 0.
03  WS-INIT-TABEL-FKC-FKM        OCCURS 1000 TIMES.
    05  WS-INIT-TABEL-DEP        PIC X(3).
    05  WS-INIT-TABEL-REDEN      PIC X(2).
    05  WS-INIT-TABEL-BEGIN      PIC X(8).
    05  WS-INIT-TABEL-EINDE      PIC X(8).
03  WS-TABEL-FKC-FKM             REDEFINES WS-INIT-TABEL-FKC-FKM
                                 OCCURS 1 TO 1000 TIMES
                                 DEPENDING ON WS-A-FKC-FKM
                                 ASCENDING KEY
                                           IS WS-TABEL-DEP
                                 INDEXED BY WS-INDEX.
    05  WS-TABEL-DEP             PIC X(3).
    05  WS-TABEL-REDEN           PIC X(2).
    05  WS-TABEL-BEGIN           PIC X(8).
    05  WS-TABEL-EINDE           PIC X(8).

INITIALIZE WS-INIT-TABEL-FKC-FKM.
MOVE 0 TO WS-A-FKC-FKM.
```
