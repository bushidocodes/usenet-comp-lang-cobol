I am in a shop which has used MF for a number of years, and can point out
one difference which is strongly pushing us toward the VA camp. It is not
a technical difference, but rather a marketing decision.

We have been using the OS/2 version of MF COBOL in order to support a third
party (Intersolv) development tool which was not available on any other
platform. We can now convert to NT or Win 95. MF wants a hefty extra
payment to convert out licenses. IBM, on the other hand, supplies their VA
tools (along with a number of other things, like terminal emulation) in a
single package which supports OS/2, Win 3.1, 95 and NT.

Now, while am perfectly aware that it is MicroFocuses perogative to charge
us this fee, it is something for a shop to consider when deciding where to
invest their money. At this point IBM (like many other vendors) seems to
want to market a license per machine, with support for whatever platform
you choose to run. IBM will also allow concurrent use licensing, which MF
will not.

We have run into similar problems with another vendor (Wall Data,
publishers of Rumba terminal emulation). When you consider the cost of
developer software, we have a serious problem with paying two licenses for
each product per machine. It's not like we could possibly run both
versions at one time (I don't know of a version of VM/SP to run on a
PC...)

Doug Partch wrote:
› Also note that VA creates true "exe" or "dll" files as compared with MF that
› require you take other steps to produce these files.

Doug,

Can you explain what you mean by "other steps" ?

Micro Focus COBOL simply requires you to compile each source file to
.OBJ (eg, cobol prog.cbl obj;) then link them all together to form a
.EXE or .DLL (using the "cbllink" utility).
In fact, you can actually give source files to "cbllink" directly, but
the two-stage approach is more useful when using "make"-type
utilities or when you need to compile different sources with different
compiler options (ie, when a project is anything more than a simple
one-source file to one-exe file arrangement).

Kev.
