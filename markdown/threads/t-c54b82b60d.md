I do this all the time in COBOL. I have full-function dynamic
allocation (SVC99) routines written specifically for use by COBOL
programs. You can dynamically specify the DSN, UNIT, DCB...anything.
After you dynamically specify the file (or rather, the DD) you then
OPEN the FD associated with the file and process it as normal. After
the CLOSE, you can the dynamically unallocate the file, or just let
the unallocate happen at EOJ.

Let me know if you would like the routines.
