I am very new to COBOL but so far in my class we have learned that to
break the program up into the following divisions, although I think
there are others out there when programs get more complicated:

Identification Division.
This one has one required entry, a Program-Id. Many programmers use
it for documentation as well.

Environment Division.
This is where you identify any equipment used, like floppy drives
and printers. You identify your input and output locations.

Data Division.
This is where you define all your data areas.

Working-Storage Section.
I don't think this one is required, but I think its used in a lot of
programs. This is where I define my variables, output formatting,
counters, etc...

Procedure Division.
This is the division where all the programming is done. You have
defined all your variables, etc. in the working-storage section and now
you have to write the program telling the compiler what to do with them.

Candi
