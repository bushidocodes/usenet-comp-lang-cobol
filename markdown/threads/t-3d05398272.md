> So how to achieve this without getting any error code in DB2.

You can't prevent DB2 returning the 02000 SQLSTATE/100 SQLCODE for a NOT 
FOUND condition, but you CAN trap it and continue.

You should replace your REDKG with an "equivalent" DB2 statement  (...WHERE 
<key> >= ID ) and use WHENEVER NOT FOUND CONTINUE to carry on processing, or 
to redirect control to a routine that deals with NOT FOUND. Remember that 
NOT FOUND may be used to indicate "end of processing" and write your code 
accordingly.

Look up the use of WHENEVER in your DB2 reference.

HTH,

Pete.
