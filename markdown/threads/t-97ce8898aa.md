aha! I think I just figured it out.  If you installed something else, 
there is a bug in the windows installer that will bugger up the path 
statement.  Re-installing windows will do the same thing.  The original 
install will set up a path that contains spaces, and must be enclosed in 
quotes for it to work.  The bug removes the spaces, so the path is 
incorrect.  As to the specific user problem, each user can be set up for 
a different environment under windows, and always could.  I suspect that 
the bottom level autoexec.bat has been buggered up, but that the user 
specific one corrects it.
