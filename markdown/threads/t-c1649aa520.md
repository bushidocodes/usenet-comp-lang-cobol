To increment, take each target byte, load it into the low-order of a
halfword, add +1 and restore it back.

03 WS-HWORD PIC S9(04) BINARY.
03 WS-HWORD-X REDEFINES WS-HWORD PIC X(02).

MOVE LOW-VALUE TO WS-HWORD-X (1:1).
MOVE WS-TARGET-BYTE TO WS-HWORD-X (2:1).

IF  WS-HWORD-X (2:1) IS LESS THAN HIGH-VALUE
    ADD +1 TO WS-HWORD
    END-IF.

MOVE WS-HWORD-X (2:1) TO WS-TARGET-BYTE.

You didn't state your platform so I'm guessing that it is IBM
mainfrmae since you mentioned VSAM.

To read a VSAM file sequentially you only need to open it as input (or
I-O) with access sequential (or dynamic) defined in the select
statement.  Then you just READ the file being sure to use the AT END
statement.  There is no need to set a key for a sequential read.

If, however you need to start reading at a particular place, then you
set a key to a value, do a Start Browse followed by a READ NEXT.
You'll need to check your file status or use INVALID KEY and AT END
clauses.  From that point, every READ NEXT will read the next
sequential record.
