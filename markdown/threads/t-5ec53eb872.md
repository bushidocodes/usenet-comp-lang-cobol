```
Hello everyone,

Recent developments in OpenCOBOL

"Discovered" Vala, a programming language in the style of c-sharp
without the gunk.  Designed for easing the use of GNOME libraries, but
keeping with a C ABI.  As it turns out, this is excellent fit for
OpenCOBOL extension writing.   http://live.gnome.org/Vala

For instance, a sample embedded WebKit based browser in 120ish lines
of Vala and some 12 lines of OpenCOBOL, mainly a simple CALL passing
the URL.  The browser window sports forward, back and refresh buttons
along with a location bar, and as it's the WebKit engine also sports
excellent support for the most sophisticated sites.

Screen shot of the browser creating a post on the opencobol.org forums
at http://add1tocobol.com/tiki-browse_image.php?imageId=14

Screen shot of the browser viewing the screen shot of the browser at
http://add1tocobol.com/tiki-browse_image.php?imageId=15

OpenCOBOL has already easily linked to many of the Vala GTK+ and glib
samples; Text entry, Text Viewer, Poppler PDF viewer, Glade based tree
views.  This was all possible with the C code that was a work in
progress before, but Vala reduces the time and code needed by one or
two orders of magnitude.  Quite exciting.

With this new tool accelerating the GTK+ linkage and opening a
plethora of other Glib features, attention turned to an IDE.  I can't
speak for everyone, but Eclipse doesn't fly fast enough on my
development box so I never got far enough to try the COBOL plug-in.
Enter GNAT Programming Studio.

GPS is working out as a very nice development environment.  Yes it Ada
centric, but it's extensible feature set is allowing the smooth
integration of OpenCOBOL, C and Vala.

I can't help myself when I get into early days of efforts, and a
screen shot of the cobc control panel (incomplete) is at
http://add1tocobol.com/tiki-browse_image.php?imageId=18

A shot of the style of the editors syntax highlighter and the
OpenCOBOL menu in action at
http://add1tocobol.com/tiki-browse_image.php?imageId=17

GPS also supports text expansion aliases and the current opencobol.xml
customization includes a handy one.  Type "program", hit Ctrl-O and it
fills in an empty OpenCOBOL template ready for fleshing out.  Screen
captures of the before and after at
http://add1tocobol.com/tiki-browse_image.php?imageId=19
and
http://add1tocobol.com/tiki-browse_image.php?imageId=20

All this and things have barely started; some 350 lines of GPS XML
customization (including syntax highlighter rules for OpenCOBOL and
Vala).  And Ada itself includes external access rules for COBOL in
it's specification, so using GNAT as a jumping off point doesn't seem
like an overly bad thing.  The OpenCOBOL FAQ already has a sample
source code for interfacing to GNAT Ada and this tool brings the two a
little closer.

As as aside; OpenCOBOL should have sample how-to links for gretl (Gnu
Regression, Econometrics and Time-series Library) and Octave numerical
analysis soon, and there are plans to include the syntax in GPS
customizing files.

Given the three tools of OpenCOBOL, Vala and GPS (along with
OpenCOBOL's easy access to all things based on the C application
binary interface) there could well be some spiffy looking GUI, network
aware, feature rich yet secure, open source, small and medium business
applications coming our way very soon.

Free and open COBOL is something to watch in my humble opinion and,
the reason for the post really, get involved in.

Cheers,
Brian
```
