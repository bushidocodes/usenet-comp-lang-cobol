More precisely, an 0C4 abend is an adressing exception in an OS family
operating system, i.e., an attempt to address a section of memory
illlegally. Exceeding a table is a good example of this, but it can
also occur if a data element in a record area is addressed after the
file is closed. These two examples account for most of the 0C4 abends
that occur.

The S0C4 abend is a memory protection exception, typically caused by a
runaway subscript, or attempting to use register 0 as a base register
(which limits you to the lowest 4K bytes of memory).

The S0C7 abend is a data exception, and usually occurs when you attempt
to perform calculation or numeric comparison to an IBM packed decimal
field, and the data is not in IBM packed decimal format.

Hope this helps.

These are system (as opposed to user) ABEND codes that are specific to IBM's MVS
operating system. Any program, not only those written in COBOL, can cause these
errors. The S0C4 is a protection exception; the program has tried to change
storage which has a protection key different from that of the program. The S0C7
is a data exception; the program has tried to perform an operation that requires
a numeric operand and has found non-numeric data in the field. A typical classic
cause of the latter is spaces in a COMP-3 field. Note that IBM's VSE operating
system simply calls these 'protection exception' and 'data exception'.
