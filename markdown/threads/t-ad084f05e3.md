```
X@yz.com (X) said:

>Anybody know how to open and write to empty VSAM file from a batch COBOL 
>program and avoid a FILE STATUS of 35 without having to resort to loading
>the file with a dummy record?

Specify the word OPTIONAL in the file's SELECT statement. When a VSAM file
in CREATE mode (i.e. just DEFINE'd) and specified as OPTIONAL is OPEN'd
for I-O, OPEN automatically does the dummy record trick and return status
05 instead of 35.

As an alternative, you can use the INITKSDS utility found in file 183 of
the CBT tape (http://cbttape.org) which performs the dummy record trick in
a separate job step.

Gilbert Saint-flour <gsf@ibm.net>
```
