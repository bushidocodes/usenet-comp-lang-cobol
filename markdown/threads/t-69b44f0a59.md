If you are writing to the Windows API then you will have enormous
problems while going up the learning curve. Things like using
hWnd when it doesn't have a valid value. Some of these 'small'
issues will seem innocuous at the time but will crash the system
later.

Get Petzold and read it twice.

If, however, you have a front-end (such as SP2) then this should
all be hidden from you.

› If anyone has any simple programs they'd like to share, I'd be very
› thankful. By simple, I mean something more than a Hello World dialog.

I have a nice Windows API MF Cobol program for loading container
ships, all graphical: views of the bays, drag and drop the containers,
bending and shearing curves, pictorial reprentation of the tanks,
runs to several megabytes of code.

In terms of the API calls, it is just like a C program so Petzold
will provide the program structure and examples if it is API level
you are doing.
