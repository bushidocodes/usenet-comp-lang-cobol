When i need to display messages on a collector on hp nonstop systems, i typically need to declare the spooler ($s.#somedest) as a file.
Everything in a tandem/hp nonstop system is a file, when you are a cobol program.

Therefore SELECT and declare a file as your destination and WRITE to it. Assign it to $0 at runtime.

Or you can use DISPLAY UPON CONSOLE in which case the GUARDIAN parameters of your process will direct the output at run-time like so:

run myprog /out $s.#somedest/

welcome to hp nonstop cobol85. it is one of the most beautifully organised cobol dialects i know. i used to teach it in Twinsoft Denmark.

//Henrik

I have not worked in your environment, however i HAVE worked in similar
ones.

I suspect you will need to insert a control code into the message to
cause the field to be inverted.

So, your message literal would be: "Help!" ... where is a
control code that your terminal can recognize.

In some systems, if you are using the COBOL screen section you define
the attributes of the display fields, so you would set that to be
inverted. But, of course, that is more forms related for a display
application.

In a straight DISPLAY, you will need to make the displayed text include
formatting for the terminal.
