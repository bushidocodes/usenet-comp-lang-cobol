The simple text output of the COBOL is easy enough to display as a web
page. And unless one wanted to add stuff like web page Title, etc. one
can do without any HTML.

One  issue you'll have to address is the question of printer-specfic
control characters that the COBOL program is generating (assuming the
program's output is presently going to the printer). In such a case,
I'd begin by first directing the output (on the host side)  to a disk
file, using JCL. Next run a utility against the output to convert the
control characters  (you may want to ensure that each text line with a
'0' in column 1 is followed by one line of spaces in the report, a
text line with '-' in col 1 is followed by two lines of spaces, etc.).


FTP the converted file to the web server and save it there simply as a
.txt file. None of the principal browsers have any difficulty in
displaying a .txt file. 

Someone on the web server side will have to write up a menu page (in
HTML) that lists report files available for viewing. The URL
associated with each menu item will then point to the appropriate .txt
file residing on the web server.

Hope this is of some help.
