Crystal Reports is made by Seagate Software, 604-669-8379. They
probably have a web page somewhere.

I use it with Visual Basic. The profesioanl version of Crystal comes
with a very powerfull DLL print engine. So if you can make DLL calls
from mf-cobol in windows, then you should be all set to use it.

In my opinion though, it would be worth your while to by a COBOL
Windows printing product that also supplies you with source. Crystal
reports makes it very easy to produce simple reports, but comlex
reports require a deeper understanding of crystal, which can be
criptic and non user freindly. I would rather invest my time in
learning how to print in windows using COBOL, and possibly even
writing your own print engine.

Have fun printing!

Paul
