```
A number of people from time to time have claimed that COBOL is obsolete
and/or dying. People have been saying this since the 1960s, yet COBOL
is going on, strong as ever.

Anyway, in the business environment, coding is only a small part of the
total application development cost. Testing is a big part, regardless
of language. And the most important part is the front-end design--the
user requirements definition and resulting system design.

I've seen many programmers with excellent machine skills but who are
lousy at preparing good user interfaces--clear reports, easy to use
screens, solid data edits, consistent design, etc. Every printed report
should have the run date, title, report number and name at a minimum,
yet many programmers dump out data with lousy titles, if any. Every
data entry point (batch or online) should vigorously edit the data.
Edits can get complicated--for example, the allowed values of fields
could vary for different transaction types.
```
