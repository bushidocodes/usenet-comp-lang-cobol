Michael,

I have had discussions with Microsoft on this very subject and the basic
problem is that as soon as you apply a lock on a network file the operating
system disables file caching which makes the file accesses a large amount
slower. Unfortunately there is nothing you can do, short of removing the
locks, to eliminate the problem.

MF changed the way, relative and line sequential files are handled in
3.2. That increased the access time extremely on DOS with smartdrive <
4.01, some NetWare requesters and some NT installations.

Try compiling your program with compiler directive CALLFH"EXTFH". If
that does not work try using coblib.dle from rev. 3.4.

Best regards

Sven
