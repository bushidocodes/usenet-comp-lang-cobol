> 3. What is the difference between "the invalid key
> condition" and "an invalid key condition"?

The invalid key condition is I-O status with a "2" in
the first position.

An invalid key condition is any conditon that causes
the I-O status to contain a "2" in the first position.

"The value of the I-O status is made available through
the use of the FILE STATUS clause in the file control
entry for the file or through the use of the
EXCEPTION-FILE or EXCEPTION-FILE-N function."

You need to look at what actions can be performed using which access. 
When I need to read this at work, I look it up in a book.

If you try to write a record that you just read you may get a condition 
causing a file access error.  i.e. Rewrite.  If you open a file and it 
fails and you do not check the status and then attempt to read that may 
cause a file access error.

There are times that you want to use the file status of the file to 
determine on a random read or a start whether there is a record.  Just 
because you look for a record and it is not there, that does not mean 
that is necessarily an error condition.  It is just a condition you need 
to be aware of in a program, so you can use some logic to make a 
decision using programming logic. If you code the different conditions 
you want to look for as conditions 88's the you can say "IF 
RECORD-FOUND" BLAH BLAH or "RECORD-NOT-FOUND" or "NO-MORE-RECORDS" do 
blah blah.
