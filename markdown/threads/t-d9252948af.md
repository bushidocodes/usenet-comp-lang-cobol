### RM/Cobol-85 r5.10 for SCO Unix, spurious 93/02 errors

- **From:** "paul" <ua-author-6589077@usenetarchives.gap>
- **Date:** 1997-01-29T19:00:01+00:00
- **Newsgroups:** comp.lang.cobol
- **Message-ID:** `<32f11328.69864@news.demon.co.uk>`

```


We've got customers running RM/Cobol-85 5.10 under SCO Unix who are
experiencing 93/02 errors (OPEN WITH LOCK attempted on file already
open, or OPEN without lock attempted on file already OPEN WITH LOCK).

The thing is that we *never* open files with lock. The code looks
clean and the fact it's only happening intermittently (frequency of
once a fortnight on busy multi-user systems) suggest to me we're
seeing a bug in the runtime. Has anyone else seen it? Does anyone know
if there's a fix in later runtime releases (we're about to start
upgrading all our sites to 6.x)?
```

In both RMCobol85 5.35 and 6.09 this error message means that the file
is locked. If you create a file, this file is locked. If your program
has finished 'anormally', the file is marked as 'owned'. Running
recovery you can repair that mark. I'm writing from my home, and
haven't the hand book, but there's a configuration file for cobol, you
should look there if you have some parameter about file locking.
