thaneH@softwaresimple.com (Thane Hubbell) wrote:
>Sorry I was unclear.  I wanted to knw when an COBOL ap was wrapped in
>an Enterprise Java Bean (Man, I feel like I'm describing a burrito!)
>where the COBOL runtime is?  Suppose the server is an IBM mainframe.
>How does the Merant COBOL runtime work when the COBOL application is
>wrapped in a EJB?

The bean makes use of MERANT's mfcobol.runtime (Java) class, which loads 
the COBOL run-time when it is required. When the bean is called, it makes 
calls to the COBOL application via Java native methods, implemented as 
part of the COBOL run-time; the run-time does the parameter translation, 
such as Unicode to pic x, to/from the COBOL application.

The COBOL application can also make calls back, either via JNI or via the 
OO Java domain (which does all of the parameter translation again, and 
means that it looks like a "normal" OO COBOL program) on Java objects, 
classes or other EJBs (if it wishes).

To the Java programmer, everything still looks like Java; to the COBOL 
programmer, everything still looks like COBOL.

Net Express doesn't run on a mainframe, but you can obviously call the 
mainframe from your PC COBOL. At some point we hope to be releasing Unix 
versions of the same support (using the same mfcobol jar) for Server 
Express.

Hope that's clear!

Steve.
