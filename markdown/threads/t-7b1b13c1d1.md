By deleting the idx file you also deleted information about the file's
key structure. You have to either rebuild the file providing the key
information on the command line or - if you don't know the structure -
do a little trick:

Copy the dat-file to a temporary directory. Restore the old idx file
from your old backup. Do a rebuild [filename] on the file. If rebuild
gives no further error messages, you can copy dat and idx-file to your
original data directory.

Hi, George/Neil & Russell -

Russell is perfectly correct to assert that, if you have a second IS file with
an absolutely identical file structure, you could use that index to re-create
your corrupt idex, but it misses the point, really; the whole idea of having a
rebuild utility is to deal with exactly your situation, where the index is
corrupted.  The MF rebuild utility includes a facility where you can include
within a text file a definition of the IS file's record and key structure,
including key start point, length & type [alpha , numeric, comp etc.]; this
definition is then used to re-create the IS file (preserving the data, of
course).  I'm sitting at home typing this and don't have access to the MF COBOL
manuals, but I believe you'll find the necessary details in one or t'other of
the "System Reference" manuals - they're reasonably detailed.  Or is it in the
"Additional Topics" manual...?

You will certainly be able to avoid going back to earlier backups.  I'm afraid I
don't have a sample text file to include here, but I've used this facility
before.  And, believe me, if I can do it, you can ;-)  If push comes to shove
and you can't get your head 'round how to create the record structure text file,
post a copy of your FD here (or, in extremis, e-mail it to me at
john.it.macaulay@cityedin.demon.co.uk).

No promises, but I'd be surprised if we couldn't give this one a quick
turnaround.
