Sure. If it's totally new development and the programmer doesn't have to
cope with existing garbage, then scan the Proc. Div. for any Alter, Go To,
Perform Thru, Exit, Section, or Next Sentence, or periods on any executable
statements except e.g. the final one and only GoBack permitted in the
entire program, at the end of the main paragraph and not within any
structure but at the top level only (periods should be by themselves in
column 12 as only the last "statement" in each paragraph); if any of these
are found, hand it back to the author to be coded right. If it passes this
step, then on to the rest of the inspection process to actually look at the
code. This initial step should eliminate a large percentage of likely bugs
before they ever get into production.
