A 'text' file has records delimited with carriage-returns or
carrige-return/line-feeds. A 'sequential' file does not have a record
delimiter. So, a 'text' file (line sequential) record of 80 bytes is really
81 or 82 bytes in length. A similar 'sequential' record is only 80 bytes
long.

So, COBOL reading a SEQUENTIAL file in the LINE-SEQUENTIAL mode reads only
ONE record irrespective of the size of the the file (the read passes data
until the computer detects a CR). Conversely, reading a LINE-SEQUENTIAL file
in the SEQUENTIAL mode generates records progressively off by 1 (or 2)
bytes.

Looks like you took a LINE SEQUENTIAL file (created with Notepad) and
created a SEQUENTIAL file (deleting the CR/LF at the end of each record).
That's OK, but not if your program is expecting a LINE SEQUENTIAL file.

Why are you running the File Utility program if you created a file in the
proper format anyway? (Hint: file extensions are irrelevant.)

To be sure whether the CR/LF (X'0D0A') bytes do or do not exist, you must
view the file with a program that displays the file in hex.
