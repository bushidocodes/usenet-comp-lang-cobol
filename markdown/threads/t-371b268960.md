```
The reason for the behavior is that the DOS (actually BIOS) screen I/O
routines (without ANSI.SYS) do not support color.  Each character on a
text display is represented in the video adapter RAM by two bytes.  The
first byte is the character, the second byte is the attribute byte.
When the screen is cleared, all the attribute bytes are set to white on
black.  When the screen is scrolled up, the attribute bytes on the last
row are not cleared, leaving the colors for the next text displayed
there.  Once you write some other color in line 25, from that point
on until you do a clear screen, every line will scroll up with the
same color attributes.  The reason you got subsequent lines in the
first color is because that happened to be the last color written to
last line on the screen.  What I usually do when I exit a program is
to blank the screen with the default white on black.
```

```
The problem is that DOS uses the attributes that are currently on the screen
when it runs in TTY mode.  If you blank the screen with the appropriate
colour before you exit, DOS will retain it.  So to get back to white on
black, do something like

       DISPLAY space
           WITH BLANK SCREEN
                FOREGROUND-COLOR IS 7
                BACKGROUND-COLOR IS 0
                AT 0000.

(I might have those reversed).
```

```
You're a persistent bugger aren't you - but you ask so nicely. Boy, did
I have to scratch the grey-matter to even figure this. Not saying this
is picture perfect, but it works :-


       Program-id. scrntst.

       Working-storage section.
       01 FlowFlag                 pic 9.
          88 ContinueProgram       value 0.
          88 CancelProgram         value 1.
       01 answer                   pic x.
          88 noLower               value "n".
          88 noUpper               value "N".

       Procedure Division.
         set ContinueProgram to true
         perform until CancelProgram
            display "This is a test Program"
                     at 0117 with
                     Blank Screen foreground-color 1
            display "Do you wish to continue ? Y/N"
                     at 1010 with foreground-color 3
            accept answer at 1050 with prompt auto

            if   Nolower or NoUpper
                 set CancelProgram to true
            End-if
       End-perform

       STOP RUN.
```
