```
I'm having problems using Novell Personal Netware and Microsoft COBOL.
We have a four PC network with a printer attached to one machine that
acts as a file & print server.

We have a COBOL application that works perfectly well on a single PC,
and works perfectly on the network, until we try to print. More often
than not, after printing has started the server freezes, the only
solution then being a reboot which causes data loss or file corruption
due to indexed files being open.

We can print all day without problem providing that no updates are done!
The freezing seems to occurs only after and update has taken place
recently, and the printer is used.

The printer is 'net captured' and the COBOL program Opens & Closes the
printer for each report.

The software used is :-
Microsoft Dos 6.22
Microsoft Cobol V3.0 (Not the current Micro Focus version)
Novell Personel Netware V1.0

I've implemented a workaround which closes & reopens files during
periods of keyboard inactivity to try and preserve data when the machine
freezes.

Has anybody has similar experiences, or know of any workarounds.

Your help would be appreciated.

Thankyou.

Tony Dick
```
