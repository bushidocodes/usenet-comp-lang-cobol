```
Without knowing your "full requirements", it is hard to say how to do this
in COBOL (as there is no "natural" / "built-in" way to do it in COBOL - as
there is in many programming languages).  However, depending upon how many
"variables" you might want to populate, you MIGHT find the following
technique useful,

01  Lots-of-Variables.
     05  Named-Var1   Pic X(10).
     05  Named-Var2   Pic X(20).
     05  Named-Var3   Pic X.
           ...
     05  Named-VarN  Pic X(99).

01  strDestination    Pic X(9).
      88  When-Var1  Value "Named-Var1".
      88  When-Var2  Value "Named-Var2".
      88  When-Var3  Value "Named-Var3".
            ...
      88  When-VarN  Value "Named-VarN".

Move  "Named-Var1" to strDestination
Evaluate True
    When When-Var1
        move value to Named-Var2
    When When-Var2
        move value to Named-Var2
    When When-Var3
        move value to Named-Var3
            ...
    When When-VarN
        move value to Named-VarN
     When Other
         Display "unknown receiving field"
  End-Evaluate
```

```
I don't think to move "nameOfADeclaredVariable" to strDestination is a good
question.
In any language, variable is referred by name. You only know the symbolic
name before compiling. At running time there is nothing for the name of a
variable, it is an address in memory. It is useless normally. Do you want to
use naming convention to simulate OO inheritance?

I think you are trying to know the type of an variable at running time.
Because there is no polymorphism, every Cobol variable has a unique type and
can be determined statically at compiling time. But for OO language (Java,
C++ and C++), it is quite different.

Thomas
```
