- **From:** "Norman Juers" <agri@gte.net>

```
Assuming that MERANT is going to charge runtime fees what would be the best
approach for software developers. It is obvious that application of runtime
fees for small software developers based on flat fees, such as, $870 for 10
runtimes is unworkable.  The model used here is clearly a corporate model.

Perhaps a constructive thing to do would be to suggest ways that MERANT
could structure the runtime fee for software developers.

The following are some ideas to start the discussion:

1.  Fee based on a percent of software sale price.
2.  One time buyout of fees.
3   Fees have a cap at a set amount.
4.  Volume discount with fees that phase out
5.  None of the above. There is no good way

A fee structure should take the following into account:

1.  Developers should be able to market low-end products.
2.  Fees should have a relationship to revenue.
3.  Accounting for fees should be simple and verifiable.

I would prefer not to have any runtime fee at all, but if MERANT is going to
charge a runtime fee I hope it is something that software developers can
live with.
```

This brings up an interesting point.  Where run-times were required in
the mainframe environment, they were generally purchased by the
purchaser of the software.  Thus if you purchased package X that
required the PL/1 run-time, you had to purchase the PL/1 run-time from
IBM.  I believe one of my previous shops had to license the Pascal
run-time because it was required by ISPF/PDF (or we didn't and couldn't
use one of the features of ISPF/PDF).  This wasn't as onerous as it
might seem because a shop normally would have that run-time for their
own development.  Also all file support was and is a part of the
operating system.  Data Base support is paid for separately.  I have not
heard of application vendors bundling compiler or data base run-times
with their packages.  The sort also is purchased/licensed by the shop
with the computer, not supplied by an application vendor.   

Another related point is what happens if I buy package A from vendor X
and package B from vendor Y, both of which are for the same set of PC's
and both of which use the Merant run-time?  What if both of them are
from Fujitsu and use the Power B sort or the multi-threaded run-time?
