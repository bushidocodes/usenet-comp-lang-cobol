Depends on the compiler and (sometimes) runtime environment.

With Fujitsu, use Declaratives: viz:
Procedure Division
Declaratives.
Declar-Input Section.
   Use after error procedure on input Continue.
Declar-Output Section.
   Use after error procedure on output Continue.
Declar-IO Section.
...
Declar-Extend Section.
...
End Declaratives.
Procedur Section.   (note deliberate misspelling)

Yes, you could do that, in every program, or you could just add into
the COBOL85.CBR, or whatever you use, run time file:

  @MessOutFile=message.txt

which puts the pop up message text into the specified file and "no
message box is displayed".

Perhaps the @NoMessage=YES will give the same effect, but lose the
message.

I find @WinCloseMsg=OFF, @EnvSetWindow=UNUSE to be useful options too.
