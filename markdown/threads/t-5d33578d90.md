```
I have often been faced with changing a sort sequence when using
IBM's dfsort and sat for quite a while trying to figure out what positions
in a file a given field is located.  Typically a sort would require
starting position, length of field and type of sort, and whether the
results are ascending or decending.

I was wondering is someone has a utility program to look at a copy book
and make a list of the fields(fixed format) and assign beginning and
ending locations in the file and field length. 

I can't help thinking there has got to be an easy way of doing this like
reading the registers or something.

I am using COBOL LE on an enterprise size legacy IBM 370 Mainframe.

We use mostly fixed length vsam and sam files often with a record size of
400 characters or larger.  It would really be helpful if there was a
simple utility program that could do this.

Any ideas can be posted or sent to me at
chammond@lc.cc.il.us

Charles Hammond, Data Processing, Lewis & Clark Community College.
```

```
FileAid does this rather nicely... but if you *really* want to do it
cheaply then code a skeleton program with an SD that duplicates the FD you
want to use and then:

PROCEDURE DIVISION.

    SORT SORTFIL ASCENDING KEY FLD1
                               FLD2
                               FLD3
     USING INFILE GIVING INFILE.
    GOBACK.

Compile this to your personal TEST.LOAD library and then browse (edit will
force a browse as well) the load module.  On the command line type

F FIELDS

... and see what you come up with.  The usual response to seeing what this
does is 'Sacred Fertiliser!'

(I *know* it is only browsing a load module... but it is almost like
programming by modifying the object-code so is it object-oriented?)
```

```
Good idea.  Taking it one step further, if you compile and run this using
DFSORT, you'll be able to see the SORT statement that COBOL generates which
will give you the sort field information you need.  Where the DFSORT messages
with the SORT statement show up depends on your set up.  Look in //SYSOUT
first.  If the SORT statement is not there and you can't find them elsewhere ,
add the following to your execution step:

//SORTDIAG DD DUMMY
//DFSPARM DD *
  MSGDDN=MYOUT
/*
//MYOUT DD SYSOUT=*

This will force the DFSORT messages to be printed in //MYOUT so you can find
the SORT statement.
```
