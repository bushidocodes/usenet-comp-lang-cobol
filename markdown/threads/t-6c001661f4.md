ETK (Easy ToolKit) solves the problem of writing 100% portable applications
using Cobol. "Write once, run anywhere" is a reality with ETK.

An application written with ETK (including ETK itself) runs on any platform
that has a viable Cobol compiler. This includes (but is not limited to):

PC (DOS, Win16/32/NT, OS/2)
Any brand of Unix
IBM AS/400
IBM 370/MVS TSO,CICS
BULL GCOS8 TSS,TP8
DEC VAX/VMS
(ETK has been ported to DG, HP, UNISYS,...)

ETK is scalable from simple stand-alone programs to major applications
involving millions of lines of code, from single-user to large-scale
deployment.

Developing an application with ETK involves the following steps:

1) Describe the high-level logic of the program as an FMS
(Finite State Machine). ETK has a special editor for this.
The FMS can be very simple (even omitted) or as complex as
the application requires.

2) Describe the screen layout if the program is interactive.
ETK has a special Screen Painter for this.
Screens can be processed in text-based, Windows GUI, and
Web browser HTML-based modes, all at the same time.

3) Describe the files and databases to be used.
ETK has tools to assist you with this.
This works with sequential/relative/indexed files, SQL,
and hierarchical databases.

4) Write the application logic (Business Rules) in a portable
subset of Cobol. ETK comes with source editors for this, but
you can use any editor or text-processing system that produces
text suitable as input to the Cobol compiler.

>From the above specifications, ETK generates native Cobol code for
the target platform. This code is compiled with the Cobol compiler
on the target platform, linked and bound the usual way. The result
is a normal, native application with maximal performance and
compatibility with existing applications, databases, and systems.

All ETK code and ETK generated applications are fully Y2K compliant,
unless you specifically include non-Y2K business rules (e.g. for
compatibility).

ETK also has an active repository. The analyst describes the application
to the repository. Then ETK generates the entire application ready to
run. Screens are generated automatically. Business rules are applied
as described in the repository. Database DDL and DML are generated
automatically. Regenerate the application for a different platform
and it is ported automatically. Or generate it on a PC and then use
the ETK porting tools to automatically port it to the target platform.
Regeneration is faster than compiling, so you normally do not maintain
the generated code; maintain the specifications and regenerate just
before compilation.

Screen handling:
Whether specified manually or automatically generated, the screen
layout is processed by a special SCRNIO module at run-time. Depending
on a configuration parameter, SCRNIO either generates native data-
streams or control codes for a variety of terminals: PC, ANSI, VTxxx,
3270, VIP7800, 5250, and more, or SCRNIO communicates with ETKWIN,
a Windows application that implements a GUI front-end, or SCRNIO
generates dynamic HTML on the fly to support a Web browser user
interface. In all cases, this is completely transparent to the
programmer. In fact, the application may be using all three modes
at the same time, depending on a user preference setting.

On the PC, ETK supports several popular Cobol compilers:
Realia Cobol
MicroFocus Cobol
Fujitsu Cobol

The PC version of ETK includes a very fast DOS-based compiler
and an evaluation copy of the 32-bit Fujitsu compiler.
