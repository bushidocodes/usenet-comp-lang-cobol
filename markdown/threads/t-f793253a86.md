    I think that the problem is that those old compilers
supplied RUN.EXE (and possibly others?) that were
what is known as "bound" exe's.  That is, they were intended
for DOS or OS/2.  These OS/2 exe's confuse Win9x and WinNt.

    To correct, either use the 16 bit RUN.EXE (install compiler
to support Windows as well as DOS (I know - it was
windows 3.1, but bear with me).  You should find a
number of DLE files.  Rename them to EXE.

    This might not be true with compilers newer than 3.1.

    Another approach is to use build to create a DOS only EXE.
This does work with Win2000.  Note that using LINK to
directly create an EXE might not work if you use XM.
Use build.

    I am told that putting the command "DOSONLY" in the
CONFIG.NT file will work as well.  I just tried "DOSONLY".
It did not help the original RUN.EXE (ver 3.4).  But RUNw
worked.  It runs only in a window, but then Win2000 does
not support DOS in full screen mode (clear screen does
not work on most or any program).

    Once you have a 16 bit dos only EXE, it should work,
so long as the COBDIR and etc are set.  Don't forget to
include +M and +P in the COBSW variable.  +P1 might
be needed if you use CALL X"E5"  to produce a beep.

Folks,

Russell is correct, the error is caused by the fact that no OS/2 COBOL
runtime can be found.

Your options are:

1) Prefix the DOS commands with XM e.g. XM COBOL ...

2) Use the Windows EXEs instead e.g. COBOLW ...

3) Upgrade to Net Express 3.1 which is fully compatible with Windows 2000.

All the best.
