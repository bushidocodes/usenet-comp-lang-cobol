*IF* you are trying to centre a dialog with a 'Splash Screen' take a
look at the N/E demos for GUIsamp, particularly the following method:
which uses a splash screen :-

*>---------------------------------------------------------------*
 method-id. "DefineInitialSize".
 local-storage section.
 01 lsX                  pic 9(9) comp-5.
 01 lsY                  pic 9(9) comp-5.
 01 lsCx                 pic 9(9) comp-5.
 01 lsCy                 pic 9(9) comp-5.

 procedure division.

     invoke wsDesktop "GetHeight" returning lsY
     invoke wsDesktop "GetWidth"  returning lsX
     divide lsX by 1.2 giving lsCx
     divide lsY by 1.2 giving lsCy
     divide lsCx by 10 giving lsX
     divide lsCy by 10 giving lsY
     invoke self "SetRectangle" using lsX lsY lsCx lsCy

 exit method.
 end method "DefineInitialSize".
*>---------------------------------------------------------------*

Personally I find the above a bit confusing and prefer to call them  x,
y, w(idth) and h(eight) instead of  lsX, lsY, lsCX, lsCY . Can't help
you with APIs, but essentially you have got to do the same as the above
with direct API calls :-

Step 1 - get the x,y,w,h for the Desktop, (or if you want to display a
child window), get the x, y, w, h for the 'Parent' Window/Dialog

Step 2 - now you "mess" around with the child window x,y,w,h to get
'centring'.

In OO with GUIs I do something like the following. Here I am not
'centring' as the left part of the main Dialog contains a pane for a
Treeview, and I'm just positioning the child window in the right-hand
part of the main Dialog :-

Dialog Creation Class :
*>-------------------------------------------------------------
Method-id. "setXYcoords".
*>-------------------------------------------------------------
Local-storage section.
01 x                                 pic x(4) comp-5.
01 y                                 pic x(4) comp-5.

move 290 to x
move 150 to y
invoke os-TDialog "zSetXY" using x, y

*> I determined the x and y values above with 'suck-it-and-see' -
*> but you could get mathematical. - getting into the world of
*> HDUs(Horizontal Dialog Units), pixels etc.....

End Method "setXYcoords".
*>-------------------------------------------------------------

TabbedDialog Class :
*>-----------------------------------------------------
Method-id. "zSetXY".
*>-----------------------------------------------------
*> sets Dialog positioning

Linkage section.
01 x                                 pic x(4) comp-5.
01 y                                 pic x(4) comp-5.

Procedure Division using x, y.
  invoke self "setXY" using x, y

*> Note your *ship* stays anchored to the above, until you change them !

End Method "zSetXY".
*>-----------------------------------------------------

If you *MUST* use direct API calls, check out the on-line help for a
sample of a Windows program "Overview - Windows API Programming".

Another resource for you, using APIs with Net Express - Dr. Young Kim's

http://www.kimsoft.com/api-cobol/api-cobol.htm
