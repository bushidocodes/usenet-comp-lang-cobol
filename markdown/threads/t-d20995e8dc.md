The poster (John) is probably correct.

Fujitsu COBOL has a "Yield" function that gives control back to Windows so 
it can deal with just such an eventuality. (It "yields" the cpu back to 
Windows and prevents COBOL from locking up the entire system.)

You should search the Micro Focus docs for a similar function. If you can't 
find one you will need to use the Windows API...

Really just a lot simpler to write in VB or C#... :-)

Of course, it might be that your output spreadsheet is gagging at 1000 
lines.  Have you checked that page breaks are being inserted? Have you 
looked at the file that is created? Is anything "cruel or unusual" happening 
around the 1000 line mark?

Paul,

You may be misunderstanding the use of "invoke".

There ISN'T a "comprehensive list of INVOKE statements" for your version of 
COBOL or any other version.

Could you find a comprehesive list of CALL statements?

Invoke is the OO equivalent of a CALL.  You CALL a callable module, you 
INVOKE an instance of an OO Class.

Excel is an Automation Server. That means it is a COM component that 
complies with COM interfacing. (Automation servers comply with the Component 
Object Model (COM)). The functions you have been "INVOKE"ing are all METHODS 
of the Excel Automation server Class.

You can think of them as "Built in functions" that are part of Excel. 
Whenever you interface with Excel (by program or by keyboard/mouse) it does 
what it does by invoking these methods internally.

Using the COM interface is like having access to Excel internals.

If you were going to look for a "comprehensive list" of the functions 
available, you would look in Excel, NOT in COBOL.

(COM methods can be invoked from MANY languages...)

Some time ago (March 26th, 2009) I suggested you use an Object Browser to 
get the list of all the Methods available (there are hundreds of them for 
Excel and it shares some of them with other Office products), and I even 
gave some detailed instructions (blow-by-blow) on how to activate and use 
the Object Browser that is available with Excel and other Office Automation 
servers (in fact, anything that can support VBA). You never acknowledged the
