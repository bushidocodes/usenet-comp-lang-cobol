Since you are using an 85 standard compiler, associate a two character
file status in WORKING-STORAGE with the file on the SELECT statement and
then test the value of the status field after the OPEN. If the OPEN was
unsuccessful the take appropriate action and re-issue the OPEN. If you
want to test for the existence of the file before OPEN, you probably will
have to call an assembler routine or system service. See your systems
people for help if that is the route you want to go.

If you use the FILE STATUS IS clause on the select on the SELECT
statement, it will return a two digit status code to you. By checking
the value in this return code, you can determine if the file was
present and opened correctly. The down side of this is that you will
be fully responsible for error handling dealing with this file. If
you don't get a good return code '00', it won't tell you unless you
check the value of the file status, and this will be the case for any
verb involving this file (OPEN, CLOSE, READ, WRITE,etc).......

Maybe I'm misunderstanding, but I use this (FLETT is the name of the file):

FILE-CONTROL.
SELECT FLETT ASSIGN TO FLETT
FILE STATUS IS W3-FLETT-STATUS.



01 W3-SWITCH.
05 W3-FLETT-STATUS PIC XX.
88 W3-FLETT-OK VALUE '00'.



A-READ.
OPEN INPUT FLETT
IF W3-FLETT-OK
CONTINUE
ELSE
MOVE 'Error on opening FLETT' TO W2-L1-TEXT
*(just some variable)
MOVE W3-FLETT-STATUS TO W2-L2-FILE-STATUS
*(just some variable)
PERFORM X8-FILE-FEIL
*(just some paragraph)
END-IF



You could have more switches, and 88-levels determining different options on
open - there are plenty of status-codes.
