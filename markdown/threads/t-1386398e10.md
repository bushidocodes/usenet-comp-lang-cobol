Hi Stan,

take a look http://www.parkway-software.com
Connectware is the right product.

They have exactly what you need
- an ODBC-File-Handler
- a Server on Unix side (to increase speed ) which will serve your
requests.
- simple user access rights handling
- not expensive (IMHO)

We're using this config in our environment (Win-NT apps build with
PowerBuilder accessing ISAM-Files on Unix) and it's working :-)

If you need to create a flat CSV-like file
(every line is a record, it contains list of values separated by a
commma or some other character), then ODBC is not needed.

You can do the conversion directly using Data2Flat converter
from Siber Systems. More info on it at
http://siber.com/sct/datafile/data2flat.html

This is ok for flat non indexed files.
But if you're using indexed files you'll have problems when writing
data from both sides. The filehandler on Unix differs to the NT/DOS
filehandler which will corrupt your idx-file.

We've had exactly those problems and it was a real pain to find the
reason.
