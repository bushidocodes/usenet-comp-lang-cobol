```
In the old days oneI did by calling a COBOL program using the fd name as a
parameter. It passed a pointer to the FCB, which contained he RBA. The program
would pick it out and pass it back with RETURNING or in a second parameter. 

Call using file name is not in the standard. I don't know whether it ever was,
but it might still work. 

You could also try: set address of fcb to address of file-name.
```

```
IBM does support (with VS COBOL II and later) compilers

   Call pgm Using File-Name

as an extension, but only if the file is QSAM, not VSAM.

See:

http://publibz.boulder.ibm.com/cgi-bin/bookmgr_OS390/BOOKS/igy3lr10/6.2.4.2

and
 http://publibz.boulder.ibm.com/cgi-bin/bookmgr_OS390/BOOKS/igy3pg10/4.2.1

In other words, a CALLed subprogram may access a file's DCB, but not an ACB.
```
