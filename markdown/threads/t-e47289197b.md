Not a solution - just a couple of points.

- Do you know which version of RM/COBOL ? If *really old* around Version 2 which
was ANSI 74 compliant, the file formats differ from RM/COBOL ANSI 85.

- Deleted records -  correct "Deletions' are marked as such. You only truly lose
them when you use RM/COBOL file REBUILD routines. Assuming same record size, for
a REWRITE the 'new' replaces the 'old'.

If you do have access to the appropriate RM compiler then you could write your
own CSVs ( in COBOL parlance, Sequential files).

Hopefully others will jump in indicating routines that could create CSVs that
you can import into Delphi.

Jimmy
