The CICS WRITE OPERATOR Command, beginning with Version/Release 4.1.0, can
write up to a single 680 byte message to the console. You'd have to break the
message up into logical sub-messages and then write the entire area. From
BATCH, you could use the CICS EXCI facility to accomplish this using DPL to a
CICS program that issues the WRITE OPERATOR. Unfortunately, SYSLOG will show
the message as coming from the CICS Region, not from a BATCH JOB, unless you
build that information directly into the message body or maybe as a header.
