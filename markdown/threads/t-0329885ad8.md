```
I'm having problems figuring out a table that I need to load into a COBOL
program.
What I mainly need to figure out is how many dimensions are there in this
table (1, 2, or 3). The table is a tax-rate table. It has several
records in it with 4 columns.
Marital Status Not lower than... Not higher than... tax
is...
S 01420 04370
0035400
M 04370 09600
0113850
To determine the tax, first you must have marital status, then taxable
income.
If there are 3 marital status, S, M, H would I have an OCCURS clause of
three, even though there are many several of each in the table?
Does this table have one, or two dimensions?
```

```
This sort of thing is usually handled in a tax table file with a separate
record for each marital type.

You can do it in two dimensions, or a split one dimension. An example
of the latter:

01 TABLE.
02 TABLE-SECTION OCCURS 2 TIMES.
03 MARITAL-STATUS PIC X.
03 TAX-TABLE OCCURS # TIMES.
04 TAX-BASE PIC 9(5).
04 TAX-END PIC 9(5).
04 TAX-AMT PIC 9(7).

References:

MARITAL-STATUS(TYPE)
TAX-BASE(TYPE, LINE)
```

```
I think you only need a one dimensional table, e.g.

01 tax-table.
03 tax-table-records occurs 10 times.
05 marital-status pic x.
05 low-salary pic 9(9).
05 high-salary pic 9(9).
05 tax-amount pic 9(9)v99.

Note that your table would have as many "occurs" as there are records, and
that you would have determine the best "picture" for your data.
```
