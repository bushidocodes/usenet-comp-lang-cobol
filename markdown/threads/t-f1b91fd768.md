I am currently working on an IDMS-COBOL program that takes about 2.5 hours to
run nightly and is about 7,000 lines of code. My job is to get the program
to run in less time. This program uses a lot of EVALUATEs instead of IF
statements, and one of my questions was does the EVALUATE statement hog more
resources and utilize more CPU time than an IF statement? Also, there are
about 2 pages of IF statements (like if wk-field = 'a' then move 'b' to
wk-fieldb. if wk-field = 'b' then move 'c' to wk-fieldb. etc...) and I was
wondering if I set up a 2dimensional table to move these values out, if that
also would decrease the amount of time the program has to run? Also, does
the END-IF statement utilize more CPU time, rather than just ending the if
statement with a period? I would appreciate any 'efficient cobol coding'
techniques a COBOL programmer has to offer. Thank you.

>From what I have seen, the evaluate is no less efficient than the if
statements. Nor do the scope terminators (end-if) slow the program
down. In fact, none of the things you list should really have much
effect (especially on something running that long). I would be
looking at the I/O. How many reads, write, opens, closes, etc is it
doing? Is there an internal sort? Is there a way to cut down the
size of the sort (gettting rid of unwanted records before the sort
instead of after)? Are you doing sequential reads when you could be
using direct reads (random access)? Are you going through the same
routines for no real reason? These are the things I would be looking
at....
