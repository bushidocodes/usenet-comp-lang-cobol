### Yes Virginia, there is a "GO TO" statement in COBOL

- **From:** "william m. klein" <ua-author-3041905@usenetarchives.gap>
- **Date:** 1998-04-29T20:00:01+00:00
- **Newsgroups:** comp.lang.cobol
- **Message-ID:** `<6i9e1s$k7j@sjx-ixn5.ix.netcom.com>`

```

A while ago, someone (and I can't remember if it was joking or serious)
indicated in a thread that there was no "GO TO" statement in COBOL - only a
"GO" statement.

This is not true.

According to page VI-89 of the ANSI'85 Standard, the name of the statement
is "GO TO". It is true, however, that the word "TO" is optional in the
syntax - but being optional doesn't impact the name of the statement. If it
did, then in the Select/Assign statement (following a certain rock and roll
celebrity), COBOL would need to have something called,

"the clause formerly known as ORGANIZATION"

because in this case the entire current name of the clause is optional.
```
