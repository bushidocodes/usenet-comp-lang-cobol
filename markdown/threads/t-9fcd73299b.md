Before I try to answer youre question, I would question the need to
take the spoolers out of the loop. I have used MF COBOL on unix for
years and always used the spoolers with no problem. The answer given below
would mean that only one user could attach to the printer at any one
time which rather defeats using UNIX in the first place however

UNIX defines devices such as printers to be a file usually in the /dev
directory they are usually called lp0, lp1 etc

If you define either a sequential or line sequential file pointing
directly at this file and open it with an exclusive lock testing
for faliure and looping withn a count and a sleep you should have no
problem writing directly to the printer.

Hope this helps
