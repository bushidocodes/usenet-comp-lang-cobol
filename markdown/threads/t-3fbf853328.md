```
This one is for the people who love COBOL data representation
details on any machine type to the assembly language level.

I'm writing a set of routines to handle some COBOL packed and
unpacked decimal data (typically pictured (NOT edited) numeric
(only picture characters  S, 9 and V) and some forms of
computational data) for a SNOBOL 4 system (the application
involves some intense character pattern matching and hence the
SNOBOL 4).  The details of the internal machine data
representation vary greatly from one machine to another, and from
one compiler (even on the same machine) to another, and I can
account for the representation in the various IBM mainframe
COBOLs.  But while I'm at it, I'd like to do a thorough job of it.
Any details that anyone can supply about the representation of
packed and or unpacked decimal data on various other machines
and COBOLs to the bit, hex. digit and byte level would be
appreciated.

In particular, does anyone know the details for Fujitsu Cobol and
IBM Visual Age for Cobol, both on x86 type PCs (Windows xx,
OS/2, .... ) ?  Details about the subject for other recent or current
machines and OS's would be appreciated too.
```
