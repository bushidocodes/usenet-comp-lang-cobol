Some of the suggestions already made may be more to the point, and
better suited for COBOL. But let me suggest some additional angles,
just in case you need them:

The C Preprocessor is, indeed, somewhat of a "universal pre-processor".
It doesn't care if it's seeing C, or Perl, or whatever. But I'm sure it
doesn't know COBOL's rules. You have to use #define macros carefully
since they may expand or line-wrap into the wrong columns for COBOL.
Features like #ifdef, #include, etc. don't disturb the format of the
lines they're processing so they should work just fine.
Remember CPP is case-sensitive.

Micro Focus and X/Open have the SOURCEFORMAT"FREE" directive which can help
the compiler to accept some of the output of CPP more easily: source code
can start in column 1 (with some restrictions) and source lines can be up
to 160 bytes long. (Sort of like playing tennis without a net!)

If PERL is available in your environment (it's very portable), you might
be able to fill in some nice preprocessing features that are lacking in
whatever other tools you're able to find -- such as forcing a stream of
CPP output tokens into the correct columns for acceptance by COBOL.

It depends on which COBOL you're using. Check your manfacturer's docs
for this kind of capability. For instance, in Micro Focus COBOL, you
can use the CONSTANT directive to achieve c-like conditionals.
In the following example, the directive is set in a $SET statement,
but it could have been an option set at compile-time.

$SET CONSTANT OS-ENV "UNIX"
* set OS-ENV to NT or UNIX, as appropriate

$IF OS-ENV = "NT"
************ Win/NT VERSION OF cmd-line-1 **********
01 cmd-line-1.
03 filler pic x(8) value
"NT stuff".
$ELSE
************ UNIX VERSION OF cmd-line-1 **********
01 cmd-line-1.
03 filler pic x(44) value
"UNIX stuff on a longer command line".
03 spcl-extra pic x(10).
$END

Hope this helps,

As far as I know, there is no such thing as macro pre-processing for
COBOL, if you're looking for something like C pre-processing. Although
there are pre-processors for CICS or SQL API calls, I've never heard of a
generic pre-processor or a third party pre-processor.

You can, of course, use COPY to embed lines of source code, which is
similar to the "include" command in C. You can also use COPY REPLACING
to alter text in the included file.

IBM's implementation of the COBOL 85 standard includes a REPLACE
statement that performs text substitution in a manner somewhat similar to
a C pre-processor. The book I'm looking at (COBOL/370 by Harvey Bookman)
gives examples of using REPLACE to convert user datanames from older
COBOL programs which are now reserved words in the COBOL 85 Standard.

An example of REPLACE might look like this:

REPLACE == DAY-OF-WEEK == BY == THE-WEEK-DAY ==.

There is also a "REPLACE OFF." command to limit character substitution to
only certain ranges of the source program.

I hope this helps!
