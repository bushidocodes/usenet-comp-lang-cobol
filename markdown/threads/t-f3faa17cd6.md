```
Fellow Micro Focus Programmers, Please bare with me.

At an office we have a Micro Focus system running under the Workbench
V3.2.46 runtime.

Last week our systems people converted the network from coax to twisted
pair. In the process all network cards were replaced and new software was
loaded on each workstation for 3Com Ethernet III.

Each workstation is configured as below:

HIGHMEM is loaded
EMM386 is loaded with NOEMS
files 55
buffers 32
everything that could be loaded high was loaded high
VLM is used

After the conversion, we are now getting a Cobol runtime error 105, a
memory allocation error.

I have tried everything that I can think of to resolve this problem. To
make matters worse, I was able to get the system to work using an OLD Cobol
runtime dated back to 3/9/1990.

I anyone has ANY idea, and I don't care if its oddball and out of left
field, please reply.

John H. Howe

Totally mystified.
```
