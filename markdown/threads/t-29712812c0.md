Anton,

It's not as hard as it seems!

In your config file (the file in which you link external Cobol filenames to
the lp spooler (DEFINE-DEVICE etc.), add a device called MAILBOX. The syntax
will be pretty close to what is used for the lp spooler. I haven't got an
example handy (I'm at home) but you could try something like
    mail $LOGNAME
or something like that.
We use it at quite a lot of client sites.

We do not allow the user to mailk reports directly to other users; only to
his or her own mailbox. That should be fine as the user can easily forward
the output to whomever he/she wants. It also ensures that the user sees (and
god willing) check what goes out.

Good luck,

Albert Gielis
