Wow, an IMS question here !!!
Haven't got the manual to hand, but here goes.

First thing you need to do is ensure that you've put a name on the PCB in the PSB,
otherwise with the exception of the IOPCB this will never work.

Assuming you're coding in Cobol (use of POINTER implies this).
Then code up the AIB in working storage in your program as per the manual.
There's a field to fill in the PCB name that you've set up in the PSB, and also a PCB
address field - which you should define as POINTER.

Code the database PCB in your linkage section - (i.e.  01 LS-PCB-MASK) as if it was being
passed from IMS in the traditional way, but DON'T include it in the PROCEDURE DIVISION
USING .... or in an ENTRY xxxx USING .....

Then it's CALL 'AIBTDLI' USING WS-INQY
                                                    WS-AIB-IOAREA
                                                     
Check the return & reason codes as per the manual and then, assuming the call worked you
use the SET verb to
SET ADDRESS OF LS-PCB-MASK TO <aib-pcb-address-dataname>

Once you've done this you can continue to use the AIB to make calls against the database,
or having SET the address, you can do all the other calls in the module using the more
traditional CBLTDLI.

Best use of using the AIB is when you're changing a common routine used by lots of
programs where you don't particularly want to have to change the linkage through all of
them to when the common routine needs to access a new database.  Use the AIB and then all
you need to do is update the PSBs - no code changes needed along the calling path.
AIB return/reason codes can also be more useful than the DL/I status codes, but are
somewhat complex.
