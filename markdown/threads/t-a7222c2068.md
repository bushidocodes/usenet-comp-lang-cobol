### building MF DLLs in 32 bit

- **From:** shrmtech@myna.com
- **Date:** 1998-09-24T00:00:00
- **Newsgroups:** comp.lang.cobol
- **Message-ID:** `<3609c3df.2270935@news.myna.com>`

```
Greetings,

I have two questions:

1) I am looking for sample source or some pointers for building a 32
bit DLL that includes other statically linked *.CBL files using MF WB
95/NTver 4.0 . We have been able to call a single source cobol routine
in a DLL by using "call-convention 74 is staticNTStdCall" for the
routine to be exported. However, we are having problems calling other
routines from within the cobol DLL that we thought were statically
linked into the DLL and are intended as external functions. Whenever,
the routines that reside in other *.CBL files are invoked, the parent
program that calls the DLL terminates without any messages. The
statically linked programs in cobol are not called with the convention
above since they are internal to the DLL. Is there anything special
declaration required ? All Microfocus examples I have seen via fax or
otherwise are several years old and do not address 32 bit development.


2) When running animator, there are programs that are in DLLs that we
would like to step over. But what happens is we see a message to the
effect that "the file or directory is not found". What we do next is
set a break point after the function call and run but we still see
this message. In the past, we would recompile the called source with
debug turned on but this seems silly to have to do so. We should be
able to "step over" and allow function to be called but not debugged.

Any information on these questions is greatly appreciated.
Thank you,

Nawien
```
