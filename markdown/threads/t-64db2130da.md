I'm using the Windows API function MAPILOGON to log into a new MAPI session.
The command I'm using is

   call WinApi MAPILogon   using by value 0 size 4
                                       by reference wl-profile-name
                                       by reference MapilogonPass
                                       by value MAPI-NEW-SESSION
                                       by value 0 size 4
                                       by reference Mapihandle
                                       returning ws-retcode
         end-call

The issue I'm encountering is that when I use the MAPI-NEW-SESSION flag, I
must also pass it a profile name.

What I am trying to do is log into a NEW MAPI session with the current
DEFAULT profile.   The Logon dialog box should also NOT display.

I noticed that there is a winapi MAPILOGONEX.  Can I use that one and
combine the MAPI-NEW-SESSION and the MAPI-USE-DEFAULT flags?

Or
Can I still use the MAPILOGON function with some combination of the flags ?

Any ideas are greatly appreciated.

I'm using Netexpress Cobol 3.0 with Flexus SP2 as the GUI front-end, on
Windows 98, to develop 32-bit Windows applications.
We have Microsoft Exchange Server 5.0 SP2 on a Network Server with Microsoft
exchange and Outlook Clients.

Thanks in advance,

Hi Juan,

Under NT the display of a Logon Dialog can be supressed in the profile
properties. There is an option in the advanced section for Logon Network
Security. Setting this to "NT Password authentication" stops the Logon
Dialog Box appearing. Not sure if 98/95 have this option.

I suspect most sites would have a standard for the profile name. You could
have this as a configurable value for your application (perhaps picked up
from a registry entry).

Hope this helps
David.
