```
Hello,

We have a little problem with a CALL statement.
We use the CA-Realia Cobol 4.2 compiler.

Our problem is the following :

In the main program, we call a subprogram called MODULE with the syntax :
CALL 'MODULE' USING ECRIRE-CLIENT, CLI.

The parameters ECRIRE-CLIENT and CLI are also defined in the linkage
section of our subprogram and have the same types as in our main
program.

Main program :
--------------

CLI is defined as follow :
DATA DIVISION.
FILE SECTION.
FD CLIENT.
01 CLI.
02 CLI-CODE PIC 9(3).
02 SIG-NOM PIC X(15).
02 SIG-ADRESSE PIC X(20).
02 SIG-FONCTION PIC X(10).
02 SIG-DATE-ENREG PIC X(10).

and ECRIRE-CLIENT :
WORKING-STORAGE SECTION.
01 ECRIRE-CLIENT PIC 99 USAGE COMP VALUE 1.


Subprogram :
------------
LINKAGE SECTION.
01 CODE-OP PIC 9(2) USAGE COMP.
01 PAR-CLIENT.
02 C-CODE PIC 9(3).
02 NOM PIC X(15).
02 ADRESSE PIC X(20).
02 FONCTION PIC X(10).
02 DATE-ENREG PIC X(10).

The compilation of the two programs is OK, but when we try to link
these two programs, we have the following errors :

Error 37 : Symbol undefined MODULE.



We hope someone will be able to help us.

Thanks.
Stephane and Arnaud.
```

```
I've used CA-Realia 4.2 and like it. I've never tried to pass a FILE
SECTION area in a CALL statement, so I'm not sure how well that works.

If you're using Realia you should have the OptLink Linkage editor.
'MODULE' needs to have MODULE as its PROGRAM-ID. When you want to
statically link a subprogram, you need to specify it to your LINKCOB.BAT
file as a command-line parameter, or else use OPTLIB to add MODULE to a
library that is passed to LINKCOB.BAT as a parameter. The command should
look something like this: LINKCOB MAINPROG MODULE. You should also
check your Realia manuals on how to link a program, and the section on
the OptLink Linkage editor.

I've also played with the Realia Educational compiler, and found that it
was more convenient to add my subprograms into my own object library
(call it MYLIB.LIB) and changed my BAT file for linking to always include
MYLIB. That way the linkage editor could always find my subprograms.

I don't know if this specifically addresses your problem, but I hope it
helps.

Good luck!
```
