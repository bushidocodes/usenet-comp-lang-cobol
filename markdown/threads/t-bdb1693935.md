Yippee! Thanks to Katie Parker (cobolnotdead@hotmail.com). The problem
clicked after Katie's response to Richard's inquiry in my previous
thread. I got Fujitsu 6.1 to work with the most popular Linux RDBMS
MySql http://www.mysql.com and this means it probably works with
PostgreSQL as well as any with ODBC. Fujitsu doesn't even realize this
according to their support pages at http://www.adtools.com. This truly
shows the importance of companies supporting user forums in addition
to service plans as expert users (like Katie) help tech support iron
out complaints of users. This means Fujitsu's COBOL for Linux should
enjoy functionality with the fastest growing OS and RDBMS upon launch
soon. My apologies for insulting FSC 6.1 drivers previously. The
problem was in my checking for a SQLSTATE = "00000" MySql returned
"s1c00" and the program went to exit verses reading a row. It is not
perfect, but at least there is communication.  I will post it here if
anyone wants to see how it worked? Forgive the awful "go to" it was
minimally modified from Fujitsu's examples till testing was passed.
   Mike

File is a 1 row(age) InnoDB table(testt) under DB(test) (row =
[name=age, type=int(2),] innodb. Data=1,3,5,7,17,27,12.
Using MySQL ODBC 3.51 System DSN with "Data Source Name:=TestDB"

  COBOL code/files
               cobol85.cbr (@ODBC_Inf=age.inf)

age.inf 
[default]
@SQL_DATASRC=testDB  (name of 
@SQL_USERID=myself
@SQL_PASSWORD=COHJLKHJEDANJIHPMLPHNK
@SQL_ACCESS_MODE=READ_ONLY
@SQL_COMMIT_MODE=MANUAL
@SQL_CONCURRENCY=READ_ONLY
@SQL_ODBC_CURSORS=USE_DRIVER
@SQL_QUERY_TIMEOUT=0

              [SQL_DEFAULT_INF]
@SQL_SERVER=default
@SQL_USERID=myself
@SQL_PASSWORD= COHJLKHJEDANJIHPMLPHNK

[CONNECTION_SCOPE]
@SQL_CONNECTION_SCOPE=THREAD

c. COBOL source code:
IDENTIFICATION DIVISION.
 PROGRAM-ID.    row1.
 ENVIRONMENT    DIVISION.
 DATA           DIVISION.
 WORKING-STORAGE SECTION.
  01 ageWK          PIC S9(2). *> COMP-5.

     EXEC SQL BEGIN DECLARE SECTION END-EXEC. 
 01  age          PIC s9(2).
 01  SQLSTATE     PIC X(5).
 01  SQLMSG       PIC X(128).
     EXEC SQL END DECLARE SECTION END-EXEC.
 PROCEDURE DIVISION.
  P-START.
* CONNECT TO THE DATABASE ON THE DEFAULT SERVER.
    
     EXEC SQL
       CONNECT TO DEFAULT 
     END-EXEC.

*need for mySql to enable transactions/ SET autocommit=0;

     EXEC SQL
       WHENEVER NOT FOUND GO TO :P-END
     END-EXEC.
* DECLARE CURSOR
     EXEC SQL
       DECLARE cur1 CURSOR FOR SELECT * from testt 
     END-EXEC.
*        IF SQLSTATE = "01000" THEN
     IF SQLSTATE = "00000" then  *>Mike toggle SQLSTATE-COMP/ =
"S1COO" THEN
       DISPLAY "SUCCESSFUL CONNECTION WITH DATABASE."
       DISPLAY " "
     ELSE
       DISPLAY "UNSUCCESSFUL CONNECTION WITH DATABASE."
       DISPLAY "       SQLSTATE : " SQLSTATE
       DISPLAY "       SQLMSG   : " SQLMSG
*> Problem statement        GO TO END-PROC
*> above statement would go to stop run without 
     END-IF.
     
* OPEN THE CURSOR. 
     EXEC SQL
       OPEN cur1
     END-EXEC.  
 P-LOOP.
* FETCH THE DATA.
     EXEC SQL
       FETCH cur1
       INTO :age
     END-EXEC.  
     move age             to ageWK.
     DISPLAY " age    =  "  ageWK.
     GO TO P-LOOP.

 P-END.
* CLOSE THE CURSOR.
     EXEC SQL
       CLOSE cur1
     END-EXEC.  
* END THE TRANSACTION.
     EXEC SQL
     ROLLBACK WORK
     END-EXEC.
* DISCONNECT FROM THE DATABASE.
     EXEC SQL
       DISCONNECT DEFAULT
     END-EXEC.

   END-PROC.
     DISPLAY "END OF SESSION".
     STOP RUN
