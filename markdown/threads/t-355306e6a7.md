I cannot believe that the rest of the responses to this mail have been

Firstly, when you say VISION PROPRIETARY FORMAT do you mean AcuCobol's
vision file system ????

If so, then you can either use the AcuODBC product available from
AcuCobol themselves ( http://www.acucobol.com ), or you can use the
U/SQL for AcuCobol product available from Transoft (
http://www.transoft.com ).

Once you have built either the .XFD file (AcuODBC) or the UDD file
(U/SQL) then your file becomes just another ODBC data source and can be
used with ACCESS or any other ODBC enabled product.

If it isn't AcuCobol Vision file system, then I suppose I'm as clueless
as the rest of them ;-)
