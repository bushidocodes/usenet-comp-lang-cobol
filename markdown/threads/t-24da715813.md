```
Kenny wrote:

 > htcobol testa.cob
 > Input file 'testa.i' not found or not readable

TC has a pre-processor 'htcobolpp', which produces the intermediate '.i'
file from the COBOL source.

The pre-processor is run as a separate process.

If the intermediate file was not created probably means that the
pre-processor process did not run.

Did you set the environment variables ?

(Example)
SET TCOB_PP_PATH=D:\TinyCOBOL
SET TCOB_OPTIONS_PATH=D:\TinyCOBOL
```
