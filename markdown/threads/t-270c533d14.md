> rcarvalho1474@my-deja.com wrote:

> > We are not acquainted with Cobol. So, we have a few questions.
> > 1) Can a MicroFocus Cobol program, compiled in a HP-UX environment, be
> > run in a Sun Solaris (Unix) environment, without further compilation?

Any MF Cobol can produce .int (intermediate) compiled programs that are
bytecode (ie not object code) that can be run by any MF run-time of the
same version.  If you compiled to .int on HP-UX then these will run on
Solaris with the appropriate run-time (application server).

The MF bytecodes were first used in around 1978.  Much the same was done
with UCSD Pascal, RM and ACCU-Cobol and, lately, Java.

The .int's are not optimised in any way.  .gnt's (generated) are usually
portable to the same processor class.  eg Compiled .gnt on SCO Unixware
will run under the Intel Linux application server (same CPU class).

> > 2) What is CobRun for ?

To run the .int and/or .gnt byte coded programs.  (just like 'javac'
will compile and 'java' will execute).

> > 3) Does MicroFocus Cobol compiler generate an executable (.exe?) or
> > rather generates an "to-be-interpreted" file ?

Yes.

You can produce .int (to be interpreted), .gnt (generated but relies on
run-time support) or stand-alone executables.  On DOS or Windows these
would be .EXE.  On DOS/Windows/OS/2 you can bury .int into the .EXE so
it acts like a stand-alone but is interpreted with the interpreter in
the .EXE.

On Unix it is rather harder to build a stand-alone executable.

> > 4) In order to be able to run a MicroFocus Cobol application (developed
> > on HP-UX) in a Sun Solaris environment, do we need any MicroFocus Cobol
> > software ? Which one ?

You need the appropripriate run-time or application server for Solaris,
or whatever they call it.  Their web-site has a list.  I understand that
Sun also has its version of MF so check the Sun site too.
