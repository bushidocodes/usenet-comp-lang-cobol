If you are talking about dynamic allocation of the memory used to store a
table, in the current COBOL the answer is no.

If you are talking about keeping track of how many elements of a table are
valid, you could always ADD 1 TO ELEMENT-COUNT when you load a new element.


>Is it possible to write an arrray to a file with each record being on a
>separate line rather than the whole array on 1 line?

Yes. Try this:
PERFORM VARYING SUB FROM +1 BY +1
     UNTIL SUB GREATER ELEMENT-COUNT
 MOVE THE-ARRAY(SUB) TO OUTPUT-FILE-RECORD
 WRITE OUTPUT-FILE-RECORD
END-PERFORM

Of course, your file will need the correct ORGANIZATION or contain "newline"
characters. This selection will be platform-dependent; what is your hardware
environment?

(Not to get too esoteric, but "new line" is kind of an amorphous term. Post
a few lines of code and show us what you want for output).

MCM
