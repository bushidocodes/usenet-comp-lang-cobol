I can see something like
    01  AccDate.
    02  AccDateYYYY PIC X(4).
    02  AccDateMM PIC XX.
    02  AccDateDD Pic XX.

with code like

    ACCEPT AccDate FROM DATE YYYYMMDD.
    MOVE CORRESPONDING AccDate TO InDate.

presuming, that is, that the COBOL implementation is compliant with ISO/IEC 
1989:2002, or includes the YYYYMMDD feature as an extension if it's 
compliant with a prior standard.

MOVE FUNCTION CURRENT-DATE TO AccDate
would also serve the function of the ACCEPT, and could be followed by the 
same MOVE CORRESPONDING, if your compiler includes support for the Intrinsic 
Function Amendment of 1989.  But that's not, strictly speaking, ACCEPT.

Using the ANS 'ACCEPT InDateMM [FROM CONSOLE]' or the X/Open ACCEPT xxx
FROM CRT AT llcc, or ...

> 01  InDate.
>  02  InDateMM  PIC XX.
…[3 more quoted lines elided]…
>  02  InDateYYYY PIC X(4).

In fact an X/Open 'ACCEPT InDate AT llcc FROM CRT' will accept the
three non-filler fields and skip over the fillers.

However a DISPLAY InDate will also skip the fillers and not show the
'/'s so you may need to redefine this as a named PIC X(10) and display
that.

The old ANSI way (applicable to COBOL 85) might use
control characters such as those defined in ANSI X3.64-1979.
For each field, position the cursor (DISPLAY control-string
WITH NO ADVANCING), then accept the field (ACCEPT
identifier).

Or, one might use an extension available from the compiler
vendor; in particular, one which is similar to the Format 3
ACCEPT in ISO/IEC 1989:2002.

Thanks all. I have picked up information from all the mails above and
implemented it below which is working.
working-storage section.
01 AcDate.
   02 AcDateYYYY pic x(4).
   02 AcDateMM pic x(2).
   02 AcDateDD pic x(2).
* Screen variable
01  InDate.
  02  InDateMM  PIC XX.
  02  filler PIC X value '/'.
  02  InDateDD PIC XX.
  02  filler PIC X value '/'.
  02  InDateYYYY PIC X(4).

procedure division.
    accept AcDate from century-date.
    move AcDateYYYY to InDateYYYY.
    move AcDateMM to InDateMM.
    move AcDateDD to InDateDD
    display InDate at line 4.
    accept InDateMM line 4 col 1 auto.
    accept InDateDD line 4 col 4 auto.
    accept InDateYYYY line 4 col 7.
    VALIDATION code added here.
