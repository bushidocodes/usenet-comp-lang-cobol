The latest version of the proposed new COBOL standard specifies a 31
digit limit, instead of 18.  If you can find a compiler which supports
that, you should be in business.

Failing that, it sounds like you need some extended precision functions.
Algorithms for multi-precision arithmetic abound, and are almost trivial
for add and subtract.  Multiply is not too much harder, but divide can
be a bit more complex to do well.  If you want to see some very high
precision (> 1000 digits) decimal floating point code in C, including
transcendentals like sqrt, Y^X, trig and logs, download BIGCALC.ZIP from
my web page below.  BIGCALC is a high precision RPN calculator program.
The math routines are pretty solid and have been licensed for use in two
commercial applications from other companies. The code is well commented,
especially the math routines.  The routines may be massive overkill for
your needs, but the comments should be helpful if you decide to code the
functions yourself.

I once had some time to kill while monitoring jobs, and wrote a passable
imitation of a CPU in DCL.  It would do arithmetic the way the CPU did,
with shift-lefts, etc.  It was quite inefficient, but wasn't limited to
what DCL could do.

If your application doesn't mind some efficiencies, you can do something
similar with COBOL.  You can treat each digit of a number separately
and  handle overflow the way you did doing arithmetic as a child.
