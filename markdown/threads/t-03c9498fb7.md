I don't know how you would do it in Fujitsu, but the following is pseudo
code as to how I would do it in NetExpress :-

	(a) Class - Business Logic
	(b) Classes - Dialog1 and Dialog2

- invoke Dialogs 1 and 2  from Business and create controls for each
- in each Dialog set each control to return an event to Business 
- Based on the event, say from Dialog1, Business then passes values to
  Dialog2 to "show"
- Same sequence if event was returned from Dialog2 to be displayed
in     Dialog1

Not necessarily the only way to do it - but letting the Business logic
control both Dialogs seems the most practical way to do it. In essence
my approach is to treat the screen as a 'dumb terminal', displays being
controlled from the Business logic.

Jimmy has started describing some Model View Controller (MVC) stuff,
it is not OO just used by them.  look at
http://st-www.cs.uiuc.edu/users/smarch/st-docs/mvc.html

Basically the theory of operation is that the input, business and
output are all separate, (this not as stupid as it sounds).  You apply
updates to the business model that calls a view that calls the update
for any registered view.

So to talk you through a basic scenario:

	enter 1 on screen a.
	1 is passed to the business model
	1 is flushed to the view for update.
	view calls all dialogues in succession to update the 1.

If you wish to add more views then you can (three dialogues no
problem). It is better than hard wiring the dual interaction today.

An example of how this might be useful numbers are entered and they
must be displayed with commas and two decimal places.  The input is
passed to the business that updates the view with the correct image
and all views are automatically updated with the correct
representation.  Beats programming it multiple times, especially when
your application heads for Europe (dots and commas are reversed in
numbers).

Also see http://www.objectmentor.com  in the publications there is a
document on taskmaster.  The early bits has a discussion on MVC.  If
you go past the first discussions and actually understand the full
document let me know :-}
