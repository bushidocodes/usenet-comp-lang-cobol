```
I've just tried this, and seen what the problem is.

The END-P directive within your cobsql.dir file should read ENDP.

What's happening is that END-P is being passed to Pro*COBOL, and as it
doesn't recognise that as a directive, it assumes that END-P is the program
name. If you specify the SQLDEBUG directive to cobsql, i.e.

-C "p(cobsql) sqldebug endp"

you will see the exact command-line being executed.

Once you've fixed the directives file, the application compiles cleanly.

SimonT.
```

Hi.

With Oracle 8 and later, you need to specify the Cobsql directive
COBSQLTYPE=ORACLE8 , rather than COBSQLTYPE=ORACLE , due to some internal
Oracle changes between Oracle 7 and Oracle 8.

With that directive set, your app should run correctly. I checked this out
with an amended version of Oracle's sample2.pco, and your the 1036 error
(with CSQLT=ORACLE), and correct execution (with CSQLT=ORACLE8).
