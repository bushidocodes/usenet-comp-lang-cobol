```
I am looking for a way to evaluate simple algebraic expressions (e.g.
A = 2 * (B - C) / (E + F)) which are defined outside my cobol program.

The program contains a small table with the actual values of the variables.
The future user wants to be able to change these computation rules
without having to change the program (the expressions must not be hard-coded).

Does anyone know of a way to solve this problem, or are there routines
that can do this ?

Thanks in advance,
```

```
Pass the data thru the JCL using--- PARM=[']##############...[']---,
quotes are needed only if you define the parm-info as pic x.
When defining this field don't forget to leave a pic x(2) before the
parm-data itself.
e.g.

01 PARM-INFO
05 PARM-DATA.
10 FILLER PIC X(2).
10 FIRST-ENTRY PIC 9(x).
10 FILLER PIC X VALUE ','.
10 SECOND-ENTRY PIC 9(x).
10 FILLER PIC X VALUE ','.

. . . and so on...where (x) is the number of bytes for that value

In your JCL say, EXEC PGM=????,PARM=[']###,####,##,##...[']

This way the user won't have to hardcode it in the program and it could be
changed from JCL which doesn't need compiling or linking.

Once you have passed the values on to working storage just load them onto
the table by using a perform varying. (build a table from a small file).
```
