Howard,
You might have been convinced by those responding to you that you shouldn't try
to move COBOL application to the Java environment. Merant has in the past
discussed wrappering COBOL logic to create Java Beans.  LegacyJ PERCobol will
compile your COBOL and create executables that will execute (without wrappering)
in the Java environment as applications, applets, servlets, EJBs and Java Beans
directly from COBOL. A PERCobol compile might be significantly easier and less
costly than rewriting an existing COBOL application in Java.

The program compiled with PERCobol can access new Java functions or CALL to
existing COBOL routines.
