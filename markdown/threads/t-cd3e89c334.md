```
Ok... here's the problem. I am going to generate COBOL source code and
a Smalltalk proxy (client) in Visual Age Pacbase. Through CICS
Tranaction Server run a Smalltalk client accessing the COBOL source on
the server with DB2 as my database. This architecture is to emulate
what the application will look like on a mainframe, thus the CICS. This
part I'm pretty confident will work. The problem occurs with
debugging. I have MF COBOL as my compiler, which I'm still not sure I
will use because I have the DB2 overhead (links, binds).

I think I can use CEDF from Transaction Server, but I'm looking to use
the Micro Focus Animator, which I beleive to be a very powerful
debugger. MF comes with its own CICS which I dont want to use.

The question is can I get MF Animator to work like Compuware's
Expeditor, i.e., place a break in a called module, let the app run from
a client initiated transaction to the break point, then show my loaded
listing, all in a CICS environment. Or am I just wasting my time?

Next... is there an Expeditor for NT? and if so... how bad does it
suck?

If anyone has any similar experience I would be interested. I'm on the
verge of attacking this problem whether it will work or not. I like to
find out the hard way, then chew out the vendors.

Opus
```
