› I need to get a "date" entered by an operator.

It probably depends on the platform.

If you are using an IBM mainframe, try something like:

05 CTL-DATE.
10 CTL-DATE-MM PIC 9(2).
10 FILLER PIC X.
10 CTL-DATE-DD PIC 9(2).
10 FILLER PIC X.
10 CTL-DATE-YYYY PIC 9(4).

DISPLAY 'PLEASE ENTER CONTROL DATE' UPON CONSOLE.
ACCEPT CTL-DATE FROM CONSOLE.

There are several ways of verifying that a valid date was entered:

1. If you are running COBOL/MVS, COBOL/VM, or COBOL/VSE, one of the Language
Environment routines can accept the date in various formats (you give it a
format string the date is expected to be in), and it will convert from
"Gregorian" ("MM/DD/YYYY") to a Lilian date or return a code indicating that
the conversion failed.

2. If you are using a compiler that includes the optional Intrinsic Library,
there is a function that converts a date to a relative day number. You might
check that out and verify that it does something reasonable (returns 0?) if an
invalid date has been entered.

3. Since dates are common things, there may already be a common date validation
routine at your shop, either as an external program you CALL with some
parameters, or as COPY book, or through some other similar facility.

4. You could roll your own date validation routine, but you have to know the
correct rules for number of days in each month, including the proper rules for
calculating leap years (in the currently adopted calendar, multiple of 4 are
leap years, multiple of 100 are not, but multiple of 400 are, thus year 2000 is
a leap year).

If you have a stock Julian date conversion library routine, or
something similar, an easy technique that should work for date
validation is as follows:
1. Convert the Gregorian date to a Julian date.
2. Convert the Julian date back to Gregorian.
3. Compare the result from Step 2 with the original date.
If they are not equal, then the original date is not a valid
date (February 30, garbage, negative, whatever).
