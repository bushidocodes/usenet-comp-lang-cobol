Hi Keith.
Well, I've got some good news, some not-so-good news, and some bad news.

The good news:
This is possible using the X/Open syntax DISPLAY ... UPON ARGUMENT-NUMBER and
ACCEPT ... FROM ARGUMENT-VALUE.

The not-so-good-news:
Due to the various methods of invoking your program not just across differing
platforms, but on the same platform (ie static linked vs dynamic loaded), the
name of the COBOL program might be ARGUMENT-NUMBER 0 or ARGUMENT-NUMBER 1.
You'd have to write a little bit of code to work out which.

The bad news:
This works fine on MF 32-bit products (ie all Unix variants, plus the 32 bit
OS/2 and Windows/NT SDKs), however there is a bug on the 16-bit product (which
I would imagine is the one you are using :( ), in that the path is stripped
off (so RUN D:\PATH\PROG gives only "PROG").
So what you need to do is raise a discrepancy report with your Technical
Support rep to get this fixed in an update.

I'm afraid I don't know enough about the 16-bit products to help any more
(I can't think of any workaround other than having a C main function which
grabs the command line parameters and checks them, but you can't have any COBOL
command-line processing then (FYI, this is also possible on the 32-bit products
using the cobcommandline() call from the C main to tell the COBOL RTS what the
command line was)).
