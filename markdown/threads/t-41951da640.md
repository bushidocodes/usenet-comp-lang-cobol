```
Well, this has been my weekend to run into wierd and
strange stuff... Has anyone here used COBOL as a
CGI program?

I had this teeny little require to read and display the
contents of an indexed COBOL file on the web. Being a basically
lazy kind of guy, I tossed together a little reader in VA/Cobol,
compiled ot to an exe file (this is under Windows NT by the way)
and let it fly...

I should have known better...

I executed as expected, read the file, and all that jazz, but
I can't seem to connect a DISPLAY to "stdout", even though the
manual says that DISPLAY to SYSOUT, SYSLIST, SYSLST, and CONSOLE
go to "stdout."

I wound up donig a real nasty fix I would like to pull out if
possible- I wrapped the COBOL program in a C program, had the
COBOL program write the indexed file's records out in a small
sequential file, then had the C program open and read the
sequential file. The users will never know there is a hack like that
in there, but *I* sure will...

I kinda figured it would work like any other language, and I can
certanly run the program on the command line and redirect the
stdout with ">" easily enough.

Anyone tangled with this before, or have any ideas?

Thanks
-Paul
```
