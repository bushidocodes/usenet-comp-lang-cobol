You indicate that you have a drive c which is fat and drive d which is NTFS
... if you are saving files to the c drive you will get an 8.3 (fat) name
when you do a directory but a long file name when you use explore as NT
keeps track of the extended attributes.  If you are saving to drive d, do
you receive an error when you open the file, etc.  Where do you see the 8.3
name in dos or from explorer?  If you think it is the compiler make a call
to the winapi CreateFileA and see if the name gets saved correctly.
