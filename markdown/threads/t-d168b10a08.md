```

When using Evaluate, with multiple conditions, when the first condition is
met does the program continue to check the other conditions in the evaluate
statement or does it exit?

Ie:

Evaluate A also B
When A = 1 Perform do-1
When B=1 Perform do-b-1
end-evaluate

When A=1 the program performs the do-1, does it then continue to check for
B=1 or does the evaluate end on the first true condition?

Thanks!
```

It quits when it sees A equal to 1 and does the perform. That is, as
soon as a WHEN is true, it executes that one and quits.
Unfortunately, the standard is not worded too well and it gives one
the idea that it evaluates all of the conditions first and then picks
the first one that is true. I don't know of any implementations that
did this and we tried to clarify it for the next standard.

Thane,

The execution of the evaluate statement ends when one of these 3 things
happens:
1. the statements associated with the selected when have been executed
2. the statements associated with the when other phrase are executed
3. no when conditions satisfied (and no other phrase).

so evaluate will end when the statements associated with the first match
are complete. a second match won't be found.
