```
hi regs, miss me?
during the time i took off, the only cobol-oriented thing i did was to
read the online manual for PowerCobol.
The mind-control implant i got with windows 98 must have kicked in
because i started fooling around with PowerCobol.
Sooooooo.....
i'm afraid i must abandon my goal to make my app mouse-free.
but on the up-side, i have 2 screens that i really like.
"Create Estimate" is the module i am currently working on.  so much for
a break, huh?  at least my writer's block broke.
Ok, in a meineke environment here's what you have to put in for a part
on an estimate:  (no, i'm not asking anyone to write code for me so
don't you dare!)
part number -- accesses the file partmaster.dat to see if it's there
lp-flag -- l = local purchase or s = stocked item
class -- exhaust, brakes, or suspension part (primarily for sales info)
warranty -- replaced, n/a, 90 days, one year, or lifetime
supplier -- maremont, walker, or some local parts house
description -- what is a 17568U?
retail -- what the customer pays for the part
jobber -- what i pay for the part
on-hand -- how many i have (never trust this number!!!!!!)
on-order -- how many are coming wednesday or friday (never trust..<g>)
minimum -- how many i keep in stock

so the create estimate form has ended up a conglomerate of static text,
textboxes, checkboxes, listboxes (learned that one today), and now for
the repetitive "misc. items" like clamps, resurface rotors/drums, bleed
system, etc, i am working on combo boxes.  (how the hell do i get items
in those?!)

wish me luck.

and send me some @#%$ beer!
```
