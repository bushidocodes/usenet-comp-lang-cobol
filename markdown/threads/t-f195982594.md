Bill,
I've been using the "Try and Buy" version of the product for a few weeks (it has
a 60 day life, so I'm about done) at work. Luckily, I was working hard on a
Year 2000 "Bridge Program Generator" program, so I had a good reason to use
COBOL heavily. (My task, BTW, was to take record descriptions of pre-Y2K &
post-Y2K records and generate a program to convert from old to new & new to old.
I wrote entirely in the "dying language" COBOL, and generated COBOL, too.)

I found the product very useable compared to working on the mainframe. I had to
make changes to the SELECT statements to add "ORAGNIZATION LINE SEQUENTIAL" for
my files, which were all mainframe QSAM. After learning how to set options for
the project, I could switch between Running and Debugging modes, using the
editor to look at my outputs (the COBOL program and JCL to run it).

I didn't see anything really exciting about this release. One thing missing
was the ability to get a "frequency listing" - how often each line of code was
executed. I went back to the host to get this from COBTEST (we're not at COBOL
for MVS&VM yet, though I've been able to try CODE/370).

I kind of liked the fact that this product looks so close to the mainframe
product - the listings have ASA carriage control chars in the first column!
(BTW, I wrote a REXX .CMD to convert these for printing, which allowed me to use
the HP LaserJet for downloaded host listings, too.)

IBM announced Ver. 1.2 on July 23, available on July 31, 1996. It appears to be
more robust. It's packaged as a trio of tools: base COBOL (VisualAge for COBOL,
Standard for OS/2), a 'redeveloper' (VisualAge for COBOL, Professional for
OS/2), and a 'tester' (VisualAge for COBOL, Test for OS/2). The base looks to
be essentially V1.1; the Professional is Standard plus redevelopment tools, and
Test is increased testing ability, including 'path coverage' (similar to the use
I make of frequency listings).

Get the announcement by sending e-mail to "ann··.@web··m.com"
with a blank subject and the message text "GET 296260" (quote marks not part of
the text).

Surely there are more than 2 or 3 folks using this product and monitoring this
new group - speak up!

Colin Campbell
