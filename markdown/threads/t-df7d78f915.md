Still trying to get a handle on setting screen
colors with RM/COBOL. Note from my previous posts
that I am using both an RM-system (UNIX) and a
MicroFocus-system (MSDOS). My particular UNIX
implementation doesn't support screen colors,
but the MF system under DOS does. I am compiling
under MF using the "RM" switch, for RM compatibility.
Source code must be compatible under both systems.

I have received email from pro··.@adi··m.uy
(Juan), who was nice enough to point me in the
direction of the CONTROL phrase. I have found
the CONTROL phrase in my RM documentation, but

DISPLAY VARIABLE-NAME, LINE 2, POSITION 10,
CONTROL "BCOLOR=BLUE, FCOLOR=WHITE".

This compiles without error under either compiler,
but does nothing at runtime. I suspect the
compiler does not check for syntax errors within
the CONTROL phrase, and the runtime ignores keywords
it doesn't support.

THAT SAID, could someone please show me the correct
syntax for setting screen colors this way, under RM.
It appears I need the correct keywords to use as an
argument to CONTROL.

thank you,

The syntax is correct. Please check your configuration file under Unix
running with RM/COBOL. First you need a TERM with color support and then
you need a TERM-ATTR Record with use-color=yes in your configuration
file. You can also set BCOLOR and FCOLOR by default in your configuration
file.

Please try with this:
Create a file with the following content: (Ex. conf.txt)
term-attr use-color=yes
term-attr bcolor=blue
term-attr fcolor=wihite

And then run your program with the following parameters:

runcobol prog -x conf.txt
