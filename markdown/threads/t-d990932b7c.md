```
Wang COMP is packed decimal, but you also have to be aware that Wang
used an "F" in the sign portion of the packed field for zero or positive
numbers.  (Negative numbers used the conventional "D".)  When we
migrated from the VS to Micro Focus, we had to convert all packed fields
so the signs were either "C" or "D".

Example:
Data = 123.99	Wang COMP = 12399F   Converted = 12399C

Fortunately, Micro Focus could *MOVE* comp fields with the "F" sign
(just don't attempt to do any math or comparisions).  In our MF
conversion utilties we moved the Wang data to a signed ZONED-DECIMAL
field, then moved it back again.  MF correctly changed the sign to "C".

Example:

01  WS-ZONED-FIELD	PIC S9(3)V99 VALUE ZERO.

01  WANG-COMP-FIELD	PIC S9(3)V99 COMP VALUE ZERO.

--conversion--

MOVE WANG-COMP-FIELD TO WS-ZONED-FIELD.
MOVE WS-ZONED-FIELD  TO WANG-COMP-FIELD.

Hope this helps.
```

COMP on the VS is packed decimal not binary.  Of course if you mean COMP as
on a pc and the fields are long integer then you are probably hitting the
big endian/lttle endian problem.   VS will store a long integer 1 as
X'00000001' whereas on a pc this is held as X'00010000'.  (I believe this is
the correct representation).  In this case you would need to swap the first
2 and last 2 bytes before using your routine.

Alternatively I think there is a COMP type in MF COBOL which defines an
integer in non pc format.   If you use this to describe your input files you
should just be able to move these fields to the numeric format you want.

Rick
