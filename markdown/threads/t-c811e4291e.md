On Tue, 25 Jun 2002 20:02:14 -0400, Ross Klatte wrote
> By the way, what's the meaning of DISPLAY WITH LOW-VALUES? 
> (Or, maybe it's DISPLAY WITH LOWER-BOUNDS??) 

WITH LOWER-BOUNDS is a Unisys (Burroughs) extension that deals with the 
method of passing parameters between COBOL and other languages. It's 
essentially a compiler directive -- it has no semantic effect but its 
presence or absence allows the generated code to be compatible with 
interlanguage CALLs.

DISPLAY means what it always does in a data item definition. In many 
Unisys shops, snippets of code have been passed around containing 
something like

  01 MY-DATA  PIC X(80) DISPLAY WITH LOWER-BOUNDS.

Because the WITH LOWER-BOUNDS is seldom understood, the entire line is 
taken as a magic incantation (which is basically true). Also, 
programmers in these same shops never write USAGE DISPLAY (that being 
the default). These factors -- the sharing of a magic incantation and 
the gratuitous presence of a seldom-used element within the incantation 
-- combine to lead to the assumption that DISPLAY WITH LOWER-BOUNDS is 
a phrase, when in fact it is two unrelated phrases.

There, that probably muddies that obscure point a bit more.

Edward Reid
