```
There was a lot of "discussion" about using IBM COBOL pre-compiled programs
that use LE's pre-initialization interface in a z/OS.e environment (and
whether it was or was not "supported") - around the time of the original
z/OS.e announcement.  I think that I have found (at least for the moment) a
pretty definitive statement that they ARE supported.

See:
 http://publibz.boulder.ibm.com/cgi-bin/bookmgr_OS390/BOOKS/ap0ovr00/A.0?

Which says (in part):

"EDC5244I The program, module or DLL is not supported in this environment.





Explanation: The program, module or DLL specified on this C/C++ function
call contains one or more languages that are not supported on this version
of the operating system. FORTRAN and COBOL, with the exception of
precompiled COBOL DB2 stored procedures, and other precompiled COBOL
applications, using the Language Environment preinitialization interface
(CEEPIPI), are not supported."

Given that EXPLICIT exception, I have to believe that all COBOL pre-compiled
programs that DO use the LE  preinitialization interface *ARE* supported
under LE.

Similarly,
 http://publibz.boulder.ibm.com/cgi-bin/bookmgr_OS390/BOOKS/ap0ovr00/1.6?

which says (in part),

 "Customers who choose z/OS.e are not licensed for the traditional
workloads. z/OS.e disables traditional workloads. z/OS.e will not execute
CICS, IMS, COBOL, or FORTRAN applications. However, precompiled COBOL stored
procedures and other precompiled COBOL applications using the Language
Environment preinitialization interface CEEPIPI are supported. You cannot
use the following compilers: COBOL, PL/I, VisualAge PL/I, and FORTRAN."

Seems clear (to me) that

A) You MAY not (cannot?) run the COBOL, PL/I, or FORTRAN compilers on your
z/OS.e system

B) You MAY run COBOL (DB2) "stored procedures" under z/OS.e

C) You MAY run "precompiled COBOL applications using the Language
Environment preinitialization interface CEEPIPI"

D) Precompiled COBOL programs that are not "preceded" by LE
preinitialization services in the "run-unit" will "die a horrible death" at
run-time

E) COBOL programs that expect IMS and CICS (or non-COBOL programs for that
matter) are "out-of-luck" under z/OS.e

  ***

FYI - for those who missed "how easy" it is to use CEEPIPI with a COBOL
program, check out (again)

 "5.3.5 An example program invocation of CEEPIPI"
at

  http://publibz.boulder.ibm.com/cgi-bin/bookmgr_OS390/BOOKS/ceea2120/5.3.5?

which walks you "hand in hand" thru what is required.
```
