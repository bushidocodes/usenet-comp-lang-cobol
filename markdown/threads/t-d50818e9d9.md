I've been a luurker for a donkey's age, and am now posing a question
for feedback about different file access techniques under PowerCOBOL
r4.

I've been a mainframe COBOL & ASM programmer for over 30 years, have
taught it at the local Polytechnic/University for 13, so have a pretty
good fix on using the batch version of Fujitsu -- I find it an
*excellent* alternative to MicroFocus etc.  However, visual OOD stuff
is new to me, and I'm building a club management system under Win98 as
a learning experience (perhaps as a revenue generating one as well).

Q1.  The PowerCOBOL tutorials use MS/Access ODBC examples for the
INVOKE examples.  They use server examples for the SQL examples.  Both
are run under ODBC.  Is it correct to assume that I could use SQL
under MS/Access?

Q2. Probably a dumb question, but does PowerCOBOL support normal FD
and SELECT and READ and WRITE etc.  Reference manuals suggest this,
but none of the user guides even mention this is possible.

Q3. What are the relative merits of the 3 methods of file access --
intrinsic INVOKEs vs: SQL statements vs: COBOL READ/WRITE etc.  I'm
primarily interested in versatility, robustness, performance etc.

advTHANKSance 
to all who have been "respondents" as opposed to "lurkers" like
myself, looking forward to some comments/opinions from those with more
experience than myself.

Rgds, Gerry
