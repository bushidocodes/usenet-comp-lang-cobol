Greetings....    the problem is the .Sort method has a specific declaration
if you are not going to be using named parameters...

in the VBA editor this is valid...   (probably similar to what you saw when
you recorded your macro)

Selection.Sort Key1:=Range("A7"), Order1:=xlAscending, Key2:=Range("B7") _
        , Order2:=xlAscending, Header:=xlGuess, OrderCustom:=1, MatchCase:=
_
        False, Orientation:=xlTopToBottom, DataOption1:=xlSortNormal

That previous call equates to

Selection.Sort Range("A7"), xlAscending, Range("B7"), , xlAscending, , ,
xlGuess, 1, False, xlTopToBottom

The parameters are this
.Sort(Key1, Order1, Key2, Type, Order2, Key3, Order3, Header, OrderCustom,
MatchCase, Orientation, SortMethod)

in your case you're not using the Type or the 3rd Range Key and 3rd order so
you just need to figure out how to pass all of the parameters but not
actually specifying a 3rd range.....   obviously in VBA it's nice and
easy.... just leave the parameter blank... ehhe.

NE3.1 over here... just drop me an email or something).....

    Member of Excel.Range

the wrong thing in the wrong place...

object. It might be worth having a look at the properties of this Object...

Hi Chet,

An optional parameter in COM is actually a Variant of type VT_ERROR
with a value of DISP_E_PARAMNOTFOUND. You can create one of these in
COBOL as follows:-

      ***** Create an Optional parameter
           move VT-ERROR to vtype
           invoke OLEVariant "newwithtype" using vtype
               returning MissedParam
           invoke MissedParam "getVariantAddress" returning vpointer
           set address of aVariant to vpointer
           move DISP-E-PARAMNOTFOUND to VARIANT-VT-SCODE of aVariant

You will need the following in Linkage
       01  aVariant             VARIANT.
and also have copybook mfole included.

You should then be able to use the sort method as follows:-

           invoke SortRange "sort"
               using SortKey1
                     MissedParam   *> Ignore. Use DefaultOrder
                     SortKey2
                     MissedParam   *> Ignore the Type parameter
                     sortorder

You can use the MissedParam as a placeholder for any optional
parameters you want to miss. This should also apply to other methods
that can take optional parameters.

Regards
David.
