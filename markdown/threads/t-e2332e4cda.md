```
I currently use Realia and produce PC MS-DOS only programs however
most of the time I operate under Win 3.1. This might seem
old-fashioned but at the moment I see no reason to change.

To develop code I use RED but I'm starting to think there must be a
better way, specifically to develop the source under Win 3.1 even
though the program will be compiled and run under DOS. I even tinkered
with the idea of setting up WordPerfect to write the code but the
number of macros necessary and the effort involved seems not worth it.

What I want is like RED except:

- mouse support (including drag and drop)
- clipboard support
- variable point non-proportional font size (say 8pt Courier--I can
get more on a screen)
- split windows (swappable via Ctrl-F6) such that I can edit in both
the WS and the Procedure division at the same time
- file (source) input capability like the retrieve function in most
word processors (RED is horribly clumsy here)
- Windows standard functions (e.g. Alt-F4 terminates with save prompt)
- Generate a direct source file for the compiler (no screwing around
reformatting it; no > 80 char records, no letting you put code past
col 72 and just truncating it)
- tried, tested, and rock-solid (no complaints about RED on this
score; just saying I don't want to end up with something that half
works...sometimes)

Oh, and I don't want to pay too much for it, I want to download it,
and I don't need it to hold my hand, become "the work environment", or
manage my source code. Keeping out of Config.Sys, Autoexec.bat,
Win.Ini, and System.Ini are pluses with any software product.

Does anyone use any product like this that they would recommend or
have any suggestions as to further searching?

Thanks in advance for your help.
```
