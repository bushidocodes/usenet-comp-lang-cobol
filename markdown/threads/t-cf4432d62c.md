I have *NOTHING* against Micro Focus for this, but you should certainly
"check out" IBM COBOL/ Set - if  you want a compiler that is "guaranteed" to
keep current with the syntax/semantics available on IBM mainframes.  It is
my *impression* that Micro Focus is (and has continued to be) pretty
compatible, but IBM uses the "same front end" for their compilers on all
(but OS/400) environments.

On the other hand, if you want compatibility with OLDER IBM compilers (e.g.
OS/VS COBOL), Micro Focus provides such - while IBM does not.

Fujitsu is somewhere between.  It provides OS/VS COBOL compatibility and VS
COBOL II compatibility, but not ALL of the current (post-VS COBOL II)  IBM
mainframe compiler features.

RM, AcuCOBOL, PerCOBOL, (as well as those listed above) etc are probably
best if you think that once you port to AIX, you will be porting to a
"variety" of other Linux or Unix environments (and are less worried about
existing IBM extensions).

NOTE WELL:
  This is intended as "personal opinion only".

No major issues, but there are subtleties to be on guard for.

- EBCDIC vs. ASCII in data files - depending on compiler, you can 
compile for either but mixing them will end up a giant headache
- Fixed record length vs. line terminated data may stipulate some minor 
changes
- Special PARM card provisions - not too tricky but you need to change 
linkage scheme and do a little addressing dance to allow COBOL programs 
bring in PARM card data that comes in as PARM=... on a JCL EXEC line
- No IDCAMS on AIX - if you have VSAM files to port over, you'll need to 
write C modules to recreate such features as load/extract
- JCL - Korn shell conversion unless you're using some integrated 
facility, use of SORT and Awk/Sed (instead of Easytrieve) and EXPORT 
(environment variable).

In one conversion I was involved with, we tried to use MicroFocus but it 
was unsuitable and did not add any efficiency to the conversion effort 
and hence was scrapped.

--Naum
