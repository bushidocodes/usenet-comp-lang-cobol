The reply from my "usually reliable sources" is as follows,

"It looks like Scott is using the "C Shell".  In his script to set the
environment variables and run his program are statements such as "set
FILEIN=/tapefs/ipclm.ebc".  His COBOL program then uses the environment
variable FILEIN to get the input file name.  But the "set" command of the
"C Shell" does not set environment variables, it sets local shell
variables.  To set environment variables he should use the "setenv"
command:
   setenv FILEIN /tapefs/ipclm.ebc
Then the COBOL runtime will find FILEIN and its value."

Let me (and/or the NG) know if this "solves" the problem.
