First, define the BAS$EDIT command codes:

REPLACE
==BAS$EDIT_STRIP_PARITY== BY ==1==
==BAS$EDIT_COLLAPSE== BY ==2==
==BAS$EDIT_STRIP_CC== BY ==4==
==BAS$EDIT_STRIP_LEADING== BY ==8==
==BAS$EDIT_COMPRESS== BY ==16==
==BAS$EDIT_UPCASE== BY ==32==
==BAS$EDIT_TRAN_BRACKET== BY ==64==
==BAS$EDIT_STRIP_TRAILING== BY ==128==
==BAS$EDIT_RESPECT_QUOTE== BY ==256==
.

Second, define your string(s). If your application needs to preserve the
original string, define two strings, but if update in place is OK, you need
only one:

01 WS-STRING-IN PIC X(20).
01 WS-STRING-OT PIC X(20).

Third, pass the string(s) BY DESCRIPTOR and the command code BY VALUE:

CALL "BAS$EDIT"
USING BY DESCRIPTOR WS-STRING-IN
BY DESCRIPTOR WS-STRING-OT
BY VALUE BAS$EDIT_COMPRESS
GIVING WS-VMS-CONDITION-VALUE.

I know of no easy way to use combinations of command codes in VAX CoBOL.
Either define ad hoc combinations at compile time:

==BAS$EDIT_COMPRESS_UPCASE== BY ==48==

or construct ad hoc combinations at run time:

05 WS-BAS$EDIT-COMMAND PIC S9(4) COMP.

ADD BAS$EDIT_COMPRESS, BAS$EDIT_UPCASE
GIVING WS-BAS$EDIT-COMMAND.

CALL "BAS$EDIT"
USING BY DESCRIPTOR WS-STRING
BY DESCRIPTOR WS-STRING
BY VALUE WS-BAS$EDIT-COMMAND
GIVING WS-VMS-CONDITION-VALUE.

You will find the BAS$ routines in SYS$COMMON:[SYSLIB]BASRTL.EXE, the BASIC
run-time library. If you want the BASIC compiler, you have to pay for it, but
BASRTL is free with VMS. BASRTL is supported but the documentation was
discontinued years ago.

An interesting aspect of VMS design is that any routine can be called from any
language. For example, you do not need VAX assembler to make VMS system calls;
you can make them directly from CoBOL.

Naturally, one should avoid undocumented calls. The STR$ library is documented
and supported, and one should feel free to use it as appropriate.
Unfortunately, the STR$ library does not include the common F$EDIT functions
COLLAPSE and COMPRESS. Some old VMS hands therefore recommend BAS$EDIT, which
does.

However, these functions are so simple that I recommend coding them in straight
CoBOL. Here is an example of a COLLAPSE in place:

01 WS-STRING.
05 WS-STRING-X PIC X OCCURS 80
INDEXED BY WS-STRING-INDEX-1
WS-STRING-INDEX-2.

SET WS-STRING-INDEX-1
WS-STRING-INDEX-2 TO 1.

PERFORM FUNCTION LENGTH(WS-STRING) TIMES
IF WS-STRING-X(WS-STRING-INDEX-1) NOT = SPACE
THEN MOVE WS-STRING-X(WS-STRING-INDEX-1)
TO WS-STRING-X(WS-STRING-INDEX-2)
SET WS-STRING-INDEX-2 UP BY 1
END-IF
SET WS-STRING-INDEX-1 UP BY 1
END-PERFORM.
