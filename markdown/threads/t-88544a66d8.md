Hi. I'm the developer at MF who has recently taken on responsibility for all
of Animator V2 (previously I wrote the editor engine and interface
configuration modules). I thought I'd respond to some of the specific points
raised:

›
› 1) Search (&replace) is very unreliable. Sometimes it doesn't find
› =anything=
› (while I know there must be at least 50 occurrences).

This has been reported by other user's and we think we have now gotten to the
bottom of this in the new 3.3 release (I don't know when that will be on
shelf) and the fixes are being readied for a new 3.2 update in the next
couple of months.

›
› 2) garbage is sometimes appended at the end of the code
›
Have to admit I've not heard of this one. Can you give me any more details on
what set of operations you were working on ?

› 3) Sometimes when deleting a line in the source it deletes a line which 
› isn't even close to the line at which the cursor is positioned on!! (this is 
› VERY annoying and dangerous! Imagine what would happen if the line was some 
› part of a calculation, which is now removed without even noticing)
› 
This has also been reported and to my knowledge fixed (I will check though
with the previous owner).

› 4) The Animator is =very= persistant. If it hangs (which it 
› does quite often when using it in combination with the DB2 precompiler) it 
› can't be killed without a reboot most of the times !
› 
› I'm not familiar with this but would like more info if anyone can supply.
 
› 5) Try 'save as' in an 'edit & execute' window. There's no way of scrolling 
› through the source after having 'saved as' without reloading it
› 
I'll check this one out. When the save-as was done, was the cursor in an
expanded copy file or in the main window ?

› When those bugs would be removed, it's quite a good IDE. I love the
› syntax-highlighting and the 'suggest' option.
In that case, you'll also love the new "Autofix" option, that corrects typos
on-the-fly (a combination of MS-Word's autotype and autotext features). This
is in 3.3 base release and the next 3.2 update.

We're currently working very hard to try to improve the stability and
performance of Animator V2. If anyone reading this has any specific niggles,
complaints or "would likes" then please let me know. You can contact me here
or via Compuserve, user-id 100065,2474.

regards, Steve.
