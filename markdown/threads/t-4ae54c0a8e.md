Hell, I *wrote* a BBS in Cobol.  Just to prove it could be done.  I designed
the entire thing using screen sections, then I wrote a time-sharing hook
(this was on a PC under DOS) and triggered once per second.  It scanned the
screen, compared it to the previous screen, and transmitted an ASCII Escape
sequence to update the remote terminal. That was written in assembler, as
were the Xmodem upload/download routines. It ran in batch mode under
Binkley, had E-mail, etc.  Ran in 512k 10 megs.

All the files were Isam.  I made a lot of Sysops Cobol fans when they found
out how easy it was to add a birthday index to a user file and personalize
birthday messages.

You could do the same thing today using nothing but Cobol.  With the kind of
processor power we have today, it should be relatively trivial.
