September 1, 1999

The following upgraded free file is available for download at
http://www.flexus.com/softwaredownload.html :

COBOL FD Analyzer
COBOL FD Analyzer This is Tal Systems' COBFD COBOL FD Analyzer v 1.50 (Aug
'99). This freeware tool runs in PC/MS-DOS and parses COBOL source or
copylibs. It reports the size and offset within each '01' level for all
datanames (group and elementary). COBFD supports any level OCCURS/REDEFINES
and most specialty compiler directives. Included are the executable,
documentation and a demo file. Interactive and command-line modes allow
operation with prompts or in MS-DOS batch files. Version 1.50 adds runtime
choice of word or byte storage for COMP (binary) items.
COBFD.ZIP (111,957; August 8, 1999)

   * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * *
* * * * * * * * * * * * * * * * * * * *
The addition of the WORD storage option for binary datatypes means that
COBFD will now report the correct offsets and record sizes for FDs with
COMP/BINARY data used on IBM mainframes. The previous release of COBFD used
BYTE storage with no option.

The only other change in the software is that it now supports the literal
"PACKED-DECIMAL" as a usage, treating that as COMPUTATIONAL-3 (BCD).

Three releases ago I said "this" was the *LAST* MS-DOS release of COBFD.
Well, let's make it four: this is the last MS-DOS release.

I have already started migrating this to Windows. When available, the new
software will feature the source code in one window with the report in
another window.  You will be able to select your COBOL source/copylib using
a "regular" Windows-style "Open File" dialog OR (shamelessly copying the
Progeni "sizer" utility)  you will be able to pop data into the source
window using the clipboard. The report window will allow you to "save as..."
so you may then print the file from Notepad or Word or Joe's Word Processor
or whatever you use.  I'm still undecided if I will let you edit the source
code in the source code window, and will almost certainly not include
printing functions in the first release.

The Windows design is not locked in stone yet, so if you want to send
suggestions, be my guest. I already have some other features I am planning
on adding, but I'm not telling you about those right now. We'll see if they
are still on The Wish List.

Thanks again to Bob Wolfe and The Flexus Corp for the web space and other
assistance.
-
Michael Mattias
Tal Systems
Racine WI USA
michael.mattias@gte.net
