```
> it brings up Micro Focus' 'editor' program.

MF 'editor' is a Cobol program and is therefore CALLable.
 
> Is there an environment or run-time parameter that I can configure to
> make this call to "edit" use 'vi' instead?

No. 'vi' is not a Cobol program and therefore cannot be CALLed.

> I know I may be dreaming, bur getting a return code would be extremely
> beneficial also.

What were you expecting the 'return code' to indicate ?  What would
count as 'success' and what as 'failure' and how would the editor know
which was which ?
 
> As I said, I know I can use something to the effect:
> 
> CALL "SYSTEM" USING Z"vi <filename>"

Then you have all that there is to know.  Upper case SYSTEM will reset
the screen environment correctly

> But this is expensive and tedious to bring back a good return value
> from.

Not only that, but it is unlikely to be anything interesting.

I would not recommend that you impose 'vi' on your users without
giving them a weeks training course.  At the very least they sould
have a choice of which editor they want to use, preferably one they
are familiar with, and most likely something very simple.

If you want to have the editor tell you that they completed the task
successfully then you will probably have to write your own in Cobol. 
They aren't hard to do with MF's ADIS.
```
