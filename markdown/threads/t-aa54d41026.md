```
mary plath wrote:
>Does anyone know whether COBOL 74 is Y2K compliant  ?
>Is there a website on this anywhere?
>I've been to www.cobol.org and they do not have the 74 standard there.

Standard COBOL 74 is not Y2K compliant, in that there is no way to get
a four digit current year.  However, most vendors who have 'current'
COBOL 74 compilers (e.g. Unisys), have provided extensions which allow
you to get the full year.  Unisys has a system call which returns the
exact structure returned by the COBOL 85 FUNCTION CURRENT-DATE:

      *
      *  ** CURRENT DATE/TIME (MUST BE 01) **
      *
       01  DW-CURR-DATE-TIME       WITH LOWER-BOUNDS.
           03  DW-CURR-CCYYMMDD        PIC  9(08).
           03  DW-CURR-HOUR            PIC  9(02).
           03  DW-CURR-MINUTE          PIC  9(02).
           03  DW-CURR-SECOND          PIC  9(02)V9(02).
           03  DW-CURR-DIFF-GMT        PIC  X(01).
      *                            '+' DIFF IS AHEAD OF GREENWICH
      *                            '-' DIFF IS BEHIND GREENWICH
      *
           03  DW-CURR-DIFF-HOURS      PIC  9(02).
           03  DW-CURR-DIFF-MINUTES    PIC  9(02).

           CHANGE ATTRIBUTE LIBACCESS OF "GENERALSUPPORT" TO BYFUNCTION.
           CALL "CURRENT_DATE OF GENERALSUPPORT"
               USING DW-CURR-DATE-TIME.

In practical terms, it is not that big a deal.  Assuming your COBOL 74
compiler will correctly return yymmdd after 1/1/2000, you can get by
for the next 100 years using this code:

           03  ccyymmdd      PIC  9(08).
           03  ccyymmdd-r    REDEFINES ccyymmdd.
               05  cc            PIC  9(02).
               05  yymmdd        PIC  9(06).
               05  yymmdd-r      REDEFINES yymmdd.
                   07  YY            PIC  9(02).
                   07  mm            PIC  9(02).
                   07  dd            PIC  9(02).

           ACCEPT yymmdd FROM DATE.
           IF yy < 99
               MOVE 20 TO cc
           ELSE
               MOVE 19 TO cc.

This will produce a correct ccyymmdd for all dates from now through
December 31, 2099.
```
