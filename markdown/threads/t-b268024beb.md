# Observation on Using // as the COBOL Inline Comment Indicator

### Observation on Using // as the COBOL Inline Comment Indicator

- **From:** "Rick Smith" <ricksmith@aiservices.com>
- **Date:** 1999-12-26T00:00:00
- **Newsgroups:** comp.lang.cobol
- **Message-ID:** `<845ar1$207h$1@news.hitter.net>`

```
A previous discussion concerning the COBOL inline comment
indicator concluded by suggesting the // might be a better
indicator that *> because // is easier to type.

It is possible that // could appear in a valid picture clause.

Given:

    77 NE-ITEM PICTURE 99//99.

and

    MOVE 1234 TO NE-ITEM

the result is 12//34. Were // an inline comment, the definition
would be a syntax error. Though the above PIC may be unusual,
adopting // could change existing progams and would require
additional rules. Of course, my early suggestion of -- would
have affected many more programs.

I am still in favor of changing the inline comment indicator; but,
as usual, there is much more to the COBOL standard than
"meets the eye." As I dig ever deeper into the draft standard,
the more I appreciate the work being done by J4 and WG4.
------------------
Rick Smith
```
