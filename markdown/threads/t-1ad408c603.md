```
Hi all,

Again I try to understand some content
in the committee draft (COBOL standard).

see 11.6.3 8)
If the PROPERTY clause (in the METHOD-ID paragraph)
is specified, the actual method name is IMPLEMENTOR-defined.

I can't understand why this decision is sensible.
Here is my attitude:

By using the PROPERTY clause for a data item,
methods for accessing the data item will be
defined IMPLICITELY. The implicit definitions
(see 13.17.38.1) take a form with a METHOD-ID
paragraph using the PROPERTY clause. Thus, the
above rule applies.

However, what is the sense of having methods (implicit
or explicit, no matter), if there is no STANDARD way for
invoking them.

Actually, I liked the property access syntax
in R. Obin's book:
MOVE Balance Of AccountHandle TO ...
MOVE ... TO Balance Of AccountHandle.

Maybe I didn't understand some rules in the draft.

Any comment?
Ralf
```
