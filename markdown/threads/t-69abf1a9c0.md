I'm totally confused why you would say Cobol is not well-suited for
interactive applications. Don't tell all the CICS/COBOL programmers and
the tens-of-thousands of applications out there they don't work well.

Many of the PC Cobol vendors implement a fairly rich set of options in
the DISPLAY/ACCEPT verbs to rather easily create interactive, text-based
applications. They can be written in native GUI coding and many
third-party tools exist for Cobol for interactive (and other) tasks.

COBOL can do virtually anything that any other language can do. For some
things it is a LOT better than for others. Here is a list of some of the
software I have written, NOT associated with my full time programming job.
I know that this software is "useful" because people pay me to use it!

1. A program for pilots to call DUATS (An FAA sponsored WX service) and
translate the
weatherease data into english. It is context sensitive and does an
admirable job. Currenly needs some modification to the new WX format, but
the FAA provides these translations free as part of the briefing now, so
there's not much call for my program anymore (But it WAS useful!)

2. A Bowling center management system. Runs on a Lan or standalone.
Handles everything except the lane scoring computers - they already have
those and don't need my software for that. However all marketing, league
income tracking, cash accounting is handled by the system.

3. A small system to translate CNC programs to work on a Pega 2000 Laser
Lathe. This saves the company upgrading to this device thousands of
dollars in re-work because the person doing the work won't mess it up!

I've several others, all written in COBOL.
