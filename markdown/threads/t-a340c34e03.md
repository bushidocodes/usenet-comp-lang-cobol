Firstly you haven't explained what you are trying to achieve (in an
overall sense) and secondly from your second posting you would appear
to be the reading the file produced as if it were text (or some
such...) and this may be a separate issue.

The "problem" with files has already been explained: in one way or
another each record has to be described (whether fixed or variable) so
that the file is not just a collection of bytes that the program wrote
(although this can be the case with a little effort). BUT however you
write the file (LINE SEQUENTIAL or whatever) you should be able to
read it back in COBOL in such a way as to have the COBOL deal with the
control information.

So: if you are trying to build sets of "words" (shall we call them)
you could write each word to a file as you have been doing as, say,
LINE SEQUENTIAL, and then re-read that file to build the words into
one large record (with appropriate spaces) initially in memory.
Alternatively you could create the same effect directly in memory by,
for example, using STRING, and assemble the set of words in that way.
In both case the result would be a data structure consisting of the
"words", having been assembled in memory. But then what do you want to
do with the, well let's call it a, "sentence"?

The system needs to be able to determine when reading a file what
constitutes a 'record'.  When the file is written each WRITE statement
outputs one record.  With LINE SEQUENTIAL the record is terminated by
CR/LF so that reading the file back the records can be separated.

With RECORD SEQUENTIAL the records may be variable length so the file
has to have length headers on each record and it also has a file header.

There is a fixed length format for SEQUENTIAL files but each WRITE will
output that fixed length so that reading the fixed length will recreate
the record as written.

You are trying to WRITE several records (each WRITE is one record) but
have them magically assembled as if they were just one record.

ie you are trying to write a BASIC (or whatever it is you already know)
program in COBOL.

The answer is to assemble the parts of the one record _before_ you write
that record.
