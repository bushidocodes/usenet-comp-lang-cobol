```
I'm using cobol 6.50 and the book Learn Cobol in 21 Days 2nd edition
published by Sams to refresh cobol skills held in my past. If anyone
else has used this combination, what did you think of Listing 6.7? All
I was able to do with it once I linked it was to get prompted to enter a
value then no matter what I entered I was told it was a nonnumeric value
from a runtime error and 00028 was written out to the terminal. I don't
know if that was the line number where the program broke or some
internal error number. By comparison, one would learn cobol far better
with the Sams book than the documentation found in the cobol650 package.

If listing 6.7 should have worked provided numeric values were entered,
I may be in the market for a commercial cobol system which runs on
msdos. A friend had Microfocus cobol and we tried it on a computer I
was using with a screen review package and Microfocus cobol doesn't
write to the screen properly. That package uses direct writes to video
and I need writing to go to bios or dos. I use screen review software
and speech synthesizers on computers since that's the cheapest
alternative blind computer users like me We weren't successful in
finding any way to force Microfocus cobol to write to the screen
correctly so I suppose I'm looking for other brands of cobol compilers.
BTW, I paid for Utah Cobol while Ellis Computing was still in business,
and found the screen writes in that package unacceptible too. That's
why I agreed to try Microfocus cobol out on a system first at all before
making a buying decission. I'll try the rest of what's in the Sams book
out on both cobol650 and utah cobol since I'm curious as to what each
compiler will do, but think it may be time to do a little more searching
soon.
```
