1.  IKJEFTSR needs to run in a TSO environment.  You can set up a
TSO environment, or you can run in an existing one.  

The easiest way to run in an existing one is:
//BATCHTMP  EXEC  PGM=IKJEFT01,DYNAMNBR=99,REGION=0M,TIME=99
//STEPLIB   DD  DSN=SYS1,SYS2.COBLIB,DISP=SHR  and other run time
stuff
//STEPLIB   DD  DISP=SHR,DSN=SYS1.CEELIB   LE/370, names vary
//SYSPROC   DD  DISP=SHR,DSN=SYS1.CLIST     CLISTS/REXX EXECs here
//SYSTSPRT  DD  SYSOUT=*
//SYSTSIN   DD  *
  CALL  'YOUR.LOAD.LIB(MYCOBPGM)' 'ANY PARAMETERS YOU NEED'
//ISPPROF  DD   ISPF profile, panels, etc.

Another advantage is you can run ISPF in batch with this setup.

2.  If you aren't invoked from a TSO terminal or via the batch
TMP, then you call call IKJTSOEV to establish a TSO environment. 
Doing this is 90% of the overhead that the TSO TMP will incur.  It
is often used for non-batch, non-TSO type programs.

3.  Long time ago, I am 99% sure that I ran a COBOL program in
batch which used IKJEFTSR to invoke an ALLOCATE command and I
didn't have to set up a TSO environment.  I could be wrong, and
IKJEFTSR has changed a lot since then.  But it might be worth a
quick shot...
