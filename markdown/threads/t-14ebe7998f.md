What you are doing is the classic: Extract, Sort, Print.

There are 3 methods available:
1. Create a COBOL program with SORT INPUT and OUTPUT Sections.
2. Create Separate Extract, and Print COBOL Programs.
3. Use the SORT utility for Extract, and Sorting. Create a COBOL Print
program.

Method 1 is the least desirable. There may be times when you have to rerun
the report. If you use this method you will need to restore a backup of the
VSAM file in order to recreate the report This backup may not be available.
The data is time dependent. You may not be able to recreate the report.

Method 2 would be my choice. At times it is necessary to know where a
particular field is used in the system. The source library is my Management
Information System. If I want to see where a particular field is used, I
will scan the source library.
It is beneficial to save the extract, preferrably after the sort, as a GDG.
If the report needs to be recreated you only have to rerun the report step
adjusting the relative generation number.

Method 3 is just a variation of step 2.

Using Alternate Indexes is slow, especially when you are accessing the
entire file. Using alternate indexes causes random access. I once changed a
program that was using alternate indexes to Extract, Sort, and Print, and
run time went from 5 hours to 15 minutes.

Programming is a lot like Chess. If you play five moves ahead, you will beat
the person who doesn't.
