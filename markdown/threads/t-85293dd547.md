I found a solution. Here it is.

.
.
.
SELECT Apples
              ASSIGN TO "apples.txt".
   DATA DIVISION.
       FILE SECTION.
       FD   F100-GROUP-APPLE
            RECORD CONTAINS 20 CHARACTERS
            DATA RECORD IS W100-INPUT-DISPLAY.

WORKING STORAGE SECTION.
01  W100-INPUT-DISPLAY
      05  W110-GROUP.
            10 W111-GROUP                    PIC   XXX.
            10 W112-GROUP-PEOPLE    PIC   99.
      05  W120-GROUP.
            10 W121-GROUP                     PIC XXX.
            10 W122-APPLES                    PIC 999.

Therefore, when i need to put in my first read record,
I can use put THIS STATEMENT  "  READ APPLES INTO W110-GROUP."
Similiar, If  i need to read my second line i can put "READ APPLES INTO
W120-GROUP"

After attending my second lecture, I just realised it can be done in this
manner. Just wanna share with people in this newsgroup with this sample
answer.

Why ?  Why do you have these lines ?  It is possible to determine how
many exist by counting the number of records with the matching key, so
this is somewhat redundant.

If you must have different record types then use a 'record type' field
(if you can).  Change this to be:
     type C=count, P=Person
      | sequence 3 digit
      | |  value 3 digit number in group/number of apples
      v V  V
   AAAC000003
   AAAP001056
   AAAP002065
   etc.

This could be an indexed file (your file cannot because there is no
uniqueness), the key would be the first 7 characters.
 
> Next 3 subsecquent line will be the group they are in follow by amount of
> apple they have.
…[12 more quoted lines elided]…
> but I still failed. Any help will be appreciated.

You probably can't tell if it is a group record or people record, though
the first of a code should be the group.

But what about it 'fails', does it fail to compile or does it fail to
open, or does the program just not work.

You may have to send more details, such as your select.  If you have
ORGANIZATION SEQUENTIAL (or no organization) then you may be getting
complete nonsense, if the file has been created with an editor then use
ORGANIZATION LINE SEQUENTIAL.
