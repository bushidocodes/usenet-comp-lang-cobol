Initial support for XML GENERATE and JSON GENERATE have made their way into GnuCOBOL 3.0 pre-rel.

https://sourceforge.net/p/open-cobol/code/HEAD/tree/trunk/

Many thanks to Edward Hart and Simon Sobisch.

In case your into those old and new fangled markup languages.

For example:

XML GENERATE out
FROM rec
WITH XML-DECLARATION
NAME OF a IS 'alpha', d IS 'ABCDEF';
TYPE OF a IS ATTRIBUTE
SUPPRESS WHEN SPACES


BBB

JSON GENERATE out
FROM rec
NAME OF a IS 'alpha', d IS 'ABCDEF'
SUPPRESS c

{"rec":{"alpha":"A","b":"BBB"}}


In other news, initial libAgar GUI support is now a GnuCOBOL contribution entry and will tracked in SVN from now on.

https://sourceforge.net/p/open-cobol/contrib/HEAD/tree/trunk/tools/cobweb/cobweb-agar/

cobweb-agar is a User Defined Function repository, some 114 functions coded so far in the 0.7 alpha release. libAgar is not just a cross-platform GUI, there is networking, system services, lots and lots of features. AG-Web support will be up next, and will provide a full framework for Web application programming with easy to wield and nicely concise COBOL functions. Samples already include an HTTP/1.0 page reader, but that is on the low tech side of things for the introductory release. /1.1 support (keepalive), HTTPS encrypted in the works.

Read all about it (and see the pretty pictures) at https://sourceforge.net/p/open-cobol/discussion/cobol/thread/c2ac66c1/

The second page has more graphics than the first. The discussion thread is from first discovery of Julien Nadeau's Agar, through to the 0.7 alpha release of the cobweb-agar function repository.

GUIs are easy in COBOL, or can be, given the right hammer. :-)

Have good,
Brian
