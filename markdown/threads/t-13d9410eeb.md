Hi,

There is no "JCL" call statement, but you can call a subroutine either
statically or dynamically:

CALL 'CALCWAGES' using RATE, HOURS.

will call the subroutine named CALCWAGES which must be link-edited into your
load module.  It's easier if you

01  SUB-NAME  PIC X(8)  VALUE 'CALCWAGES'.
CALL SUB-NAME using RATE,HOURS.

That way, the subroutine is loaded dynamically (not linked with your main
program), and can be modified without having to recompile/link the main
program.  There are no JCL requirements; the subroutine is found in your
normal JOBLIB/STEPLIB or LNKLST library.

call.  Sometimes the called program references a file or database which
the calling program doesn't reference.  The JCL has to be adequate for
both the calling and the called program.   And in test environments, it
is possible to have your called program and your called programs in
different STEPLIBs which need to be concatenated correctly.
