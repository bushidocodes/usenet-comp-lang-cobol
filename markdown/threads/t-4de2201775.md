An interesting problem has presented itself.  I have 200-odd jobs that will
run whenever some creates event X....they might (rarely, but might) create
some reject records that need to be reported on.

My problem is JES, err, with JES, the reject records write to a new GDG if
they are needed, otherwise that job step is skipped....but JES grabs +1 at
the start of job and I end up single-threading 200-odd jobs so that each can
grab GDG(+1) that it will most likely never use....

Anyway, I am sure this is a common problem, just the first time I've run into
it.  My first idea was to dynalloc the new gdg level so the JCL would not grab
it at the beginning of the job.  A quick second was to change the GDG into an
MQ...

But if anyone has tripped over this problem before and has an elegent
(and easy)solution to it I would love to hear about it....

Btw, I am not a sysprog so JES exits are out of the question...
