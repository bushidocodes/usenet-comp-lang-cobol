Is anyone doing anything similar to what I'm trying to do and if so maybe
you have some sample code that you could share.

Thanks Very Much,

Vic,

Xerox forms are not BMP they have their own format.  In order to print
the form on a desktop you must have a Xerox DJDE compatible printer or
an emulator.  My recommendation is to purchase a printer which is not
cheap.

I used Lapres for many years successfully.  There is a new product that
supersedes it release from XLprint in Australia (try the web somewhere).

The DJDE format is basically a series of lines with tags on them.  These
tags are referenced to a location by a printer definition which must be
emulated by whatever you are using.  The flow of the positioning is
'weird' it involves adding the previous font depth to the current
position so changing fonts to fit stuff into the forms causes slight
misalignments.  Even when using an emulator you should ALWAYS check the
result on the real printer before going for it (obvious really).

Xerox printers now support AFP printout.  The formats are more complex
but if you use AFP toolbox from IBM then it is not to bad.  This is
viewable using AFP Viewer and will print on far more printers.

What we do is send the "resources" (i.e. the Xerox form and any fonts required)
to the printer in as 129 byte blocked data with an "*" in col 129.  Then send
the form data as plain text line by line output with DJDE statements.
