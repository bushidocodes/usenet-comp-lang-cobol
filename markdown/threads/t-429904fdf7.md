Hello Everybody,
    I have got a sample program from Fujitsu  Corporation to access the
btrieve native requester to gain an access to the btrieve files. The call to
the btrieve requester is made as follows


     CALL "BTRCALL" WITH STDCALL USING
                          BY VALUE                BTRIEVE-OPERATION-CODE
                          BY REFERENCE    BTRIEVE-POSITION-BLOCK
                          BY REFERENCE    BTRIEVE-DATA-BUFFER
                          BY REFERENCE    BTRIEVE-DATA-BUFFER-LENGTH
                          BY REFERENCE    BTRIEVE-FILE-NAME
                          BY VALUE                BTRIEVE-KEY-NUMBER
                          BY REFERENCE    BTRIEVE-STATUS-CODE.

We tried  to compile  the program and link it with the library file
W3BTRV7.LIB. When we run the program it does not give the desired results.
The program shows Open successful for the btrieve file which is not
existing.

Can anybody give us an Idea how to access the btrieve files with the help of
the native btrieve requester "BTRCALL". We do not want to declare the files
in the environment division. The library file WBTRV32.LIB is not the right
file to be linked to access the btrieve requester.

TIA
Paul Schmitteck.
