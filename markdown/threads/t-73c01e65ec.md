### micro focus file handler (extfh) to read variable length indexed files IDXFORMAT"3" from a C program

```
My Situation:
I am a software developer/consultant helping mainframe
customers convert to UNIX systems using Micro Focus
COBOL. I have a set of utilities written in 'C' which
replace the mainframe utilities on the UNIX system.
You can see these utilities described on my web site www.uvsoftware.ca
These utilites call D-ISAM (C-ISAM compatible)
file handlers to access fixed length indexed files which
are compatible with MF Cobol fixed length indexed
files. These are IDXFORMAT"1" as described on the Micro Focus COBOL
web site in 23.3.5.1 'types of indexed files' in the File Structures manual.
This D-ISAM solution (IDXFORMAT"1") has worked
well for my previous customers.

My Problem:
A current customer requires support for VARIABLE length
indexed files which is IDXFORMAT"3" in 23.3.5.1.
This variable format is not compatible with the variable
format supported by C-ISAM. I tried to get answers from Micro Focus
COBOL but no luck since I am not a direct user of Micro Focus COBOL.

My Questions:

1. Can I call the MF Cobol file handler (ExtFH) from my
    'C' program to access IDXFORMAT"3" files ?
    Please see 10.1.3 in the Callable File Handler manual
    which mentions support for IDXFORMAT"4"

2. Are IDXFORMAT"4" & IDXFORMAT"3"
    compatible ? If EXTFH supports 4 does it also
    support 3 ?

3. Is there as sample 'C' demo program illustrating the
    calls to EXTFH ? 10.3.2 shows the call as:
    EXTFH(op-code,&fcd);

4. Exactly how do I link the EXTFH file handler into my
    C program ? I would have expected it to be in a UNIX
    library archive such as cobol/coblib/libcobol.a
    But I dont see any extfh.o components ???
    What is the name of the archive that contains the
     EXTFH file handler ???

Hoping somebody out there has some experience in this area,
Thank You, Owen Townsend
```
