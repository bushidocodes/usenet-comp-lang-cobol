Hi,

The ASLM: Semaphore failure could be due to the user not having a valid
license for the Applicatiojn Server runtime. It's all very well distributing
the Application Server but making sure that each user has a valid license is
now upon has all to ensure that we configure the environment correctly.

If the environemnet is a mapped drive environment then user should have the
variable 'ASLMFNET ' pointing to the license database, if the user is
Client/Server each with their own installation of the runtime then run the
'APPTrack.exe' utility.... see ASLM in the help, follow to Application
Server Licensing Messages and look at related topics.

Look at the utilities ASLMPCLOCATE.EXE, ASLMPCSILENT.EXE and APPTRACK.EXE.

Um....one other note...if the user is a web based user ensure that the Micro
Focus Registry entry has permissions, I've had spurious events happening
here. On some sites after n days for working 100% (Who knows could be 30
days) the web user suddenly requires access read permissions to the registry
in order to validate the license....just a thought!
