I would agree with what Frederico said earlier and have found that the
approach does usually work but we do have one instance where it
doesn't (and I will have to visit that customer shortly and sort it
out somehow!) so it obviously doesn't work in all situations.

In essence we found that while WIN95 seemed fine when printing to
"LST" or "LPT1" later WINDOWS weren't so keen on it. An alternative is
to use the UNC name as the file name so actually OPEN the file
"\\server\printer-name" where "printer-name" is the share name (but
you've got to create the printer as a local shared printer") and we
found that worked in most cases (except as above!)

Finally the best solution seems to be to be the most complicated and
that is to user the PC_PRINTER_xxx routines (although it raises other
issues and problems, as I've asked about here); these do appear to get
into the WINDOWS printer interface reasonably successfully and in the
instance above where the old style (OPEN/WRITE etc.) does not work the
PC_PRINTER_xx do write successfully to the same printer that cannot be
accessed otherwise!

No...PC_PRINTER_OPEN is set to browse a WINDOWS network and returns a
Printer Selection Window (and font selection, if desired), as in the
Help...

"A set of bits to define printer options:
bit 0  Display a printer control dialog to allow you to define the
printer characteristics (cannot be used with bit 2 or bit 3)
bit 1  Display a font dialog box to allow you to define the default
font for the document
bit 2  Print in portrait orientation (cannot be used with bit 0 or bit
3)
bit 3  Print in landscape orientation (cannot be used with bit 0 or
bit 2)
bit 4  Display a progress indicator dialog box"

so setting "flags" to 3 you get to choose the printer *and* the font.
The "document title" is just a name you give the document that would
appear in the printer spooler entry (or some such) but it is *not* a
file name.
PC_PRINTER_WRITE then is used for sending data and PC_PRINTER_CONTROL
for line and form feeds as required.


> So, do you change your programs to print everything to a file and than

and then use the handle in ..._WRITE ..._CONTROL and ..._CLOSE
