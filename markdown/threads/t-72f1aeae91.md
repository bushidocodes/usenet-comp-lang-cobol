I'm no expert on this, but what follows is my understanding of the situation
based on discussions between many of the FAQ maintainers.

The Berne Convention guarantees you copyright of any work and you would
retain that should you post it on USENET or make it available some other
way on the Internet.  However, should you wish to be able to sue for
lost income due to someone infringing your copyright, I believe that many
places require that you register a copyright.

> Also, if a person gets source code with Micro Focus COBOL and then
> changes it, do they violate copyright of Micro Focus?

I don't believe that changing the code would, but publishing the changed
code might.

> I would like to start posting some tutorials I am working on.  But I
> don't know about the legal ramifications.

Should you do so, perhaps you'd like to put them on the Micro Focus FTP
server too ?

> I know Micro Focus people read this newsgroup, and I wonder how they
> would feel if I posted a revised version of their program winhello.cbl.

Personally, I don't see why there would be a problem with this, but I'll
try to get a definitive position for you.

> In order to do this, do I have to pay royalties to the original
> copyright owner (Micro Focus)?

Because <a body> holds copyright on something doesn't mean that you
would *have* to pay royalties for using it.  The body in question may
ask for royalties or they may allow you to use the code freely as long
as you make no profit by it.  As I said, I will try to find someone who
can give you a definitive position for this.

> Since I am a man of modest means, what can I do to avoid this?
> Compromise?  Do I have to code the Message loop from scratch without
> using Winhello.cbl as a base?

There is a copyright FAQ which you might like to read for more information.
I don't know where it appears, but it's certain to be available for
anonymous FTP on rtfm.mit.edu in /pub/usenet-by-...

James.
