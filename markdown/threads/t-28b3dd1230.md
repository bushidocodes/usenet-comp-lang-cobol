While several people said use SVC 99, here is a Cobol only answer.
I tested this on PP 5688-197 IBM SAA AD/Cycle COBOL/370 1.1.1 and 
it works.  I also complied it without errors  with COBOL II compiler 
level PP 5668-958 IBM VS COBOL II Release 4.0 09/15/92, but did not 
run it but there should be no problems either.

The first thing is IBM disclaimer, then the code. While I used a nested 
program the second program could be separate and LKED statically or 
dynamically called. In a case like this I like the nested way better. 
An identifier in the USING phrase of the CALL . . . BY REFERENCE 
statement can be a file-name, in addition to a data-name. If the 
identifier is the file-name for a queued sequential file, the COBOL 
compiler passes the address of the data control block (DCB) as this 
entry of the parameter list. The identifier cannot be a VSAM 
file-name. 

File-names as CALL operands are allowed by the compiler as an extension. 
Any use of the extension generally depends on the specific internal 
implementation of the compiler. Control block field settings might 
change in future releases. Any changes made to the control block are 
the user's responsibility and not supported by IBM.
