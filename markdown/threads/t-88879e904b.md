```
PROBLEM:
     A program is required to validate batches of transactions that are 
held on a disk file.Each batch should consist of up to 100 transaction 
records followed by a batch control record.Each field of the transaction 
record needs to be validated and hash totals of all the numeric fields 
calculated.
     If the accumulated hash totals match those of the batch control 
record AND there are no invalid records AND the number of transaction 
records in the batch does not exceeds 100,then the batch is to be classed 
as valid.All valid records are to be written to a valid file.An erroneous 
batch is to be printed out on an error report that contains appropriate 
headings,with each batch starting on a new page.It is also required that 
the accumulated hash totals for that batch are printed after each invalid 
batch.A separate disk file is to be used as temporary storage of a batch 
until the validity of the batch is ascertained.

Record description is as follows:

INPUT FILE
This file is made up from two types of records,i.e.:-
     i) The actual transaction records
    ii) Batch control records

TRANSACTION RECORDS CONSISTS OF THE FOLLOWING FIELDS:-
     Record Type                             1 alphanumeric "T"
     Part No.                                8 numeric
     Customer/Supplier No.                   6 numeric
     Date supplied/Received                  6 numeric
     Quantity                                5 numeric


BATCH CONTROL RECORDS CONSISTS OF THE FOLLOWING FIELDS:-
     Record Type                             1 alphanumeric "B"
     Hash total of Customer/Supplier No's.   8 numeric
     Hash total of Dates                     8 numeric
     Hash total of Quantities                8 numeric
     Hash total of Part No's                 8 numeric



------------------  Posted via CNET Help.com  ------------------
                      http://www.help.com/
```
