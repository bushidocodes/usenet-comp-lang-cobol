```
Morning,

I as fellow developer, I will try and answer the questions.

What is "call "cob32api""?
- this is a module in the Micro Focus Runtime that makes ALL
entry-points in Win32 OS visable to .int and .gnt. The call
is not required in the application is a .exe or .dll.

Why do some Win32 APIs have a "suffix" of 'A' or 'W'.
Most APIs in Win32 environment often have to versions of the
same API, ie: one for "Ascii" and the other for "Unicode"

Do I use Unicode or the Ascii APIs?
If you application is for Windows 95 and NT, you can only use the
Ascii APIs (suffix of 'A'). If you use NT, you could use the Unicode
APIs.... infact the NetExpress product has a Win32API that will use
the Unicode APIs.

What is unicode? see: http://www.unicode.org/

Has the Microsoft APIs changed between Win16 and Win32?
Oh yes.... however Microsoft have documents on the differences
in the Win32 Developers Area.... see http://www.microsoft.com/win32dev/

What is the call-convention for Win32 APIs?
Use call-convention 74.

Oh... a handy tip..... Install the Win32 SDK, it has helps files and extra .libs, which
will be very useful for you.

PS: I have included a little "sound" demo for you.

--play32.cbl
$set defaultbyte"00" case
special-names.
call-convention 74 is WINAPI.

working-storage section.

78 SND-SYNC value h"0000".
78 SND-ASYNC value h"0001".
78 SND-NODEFAULT value h"0002".
78 SND-MEMORY value h"0004".
78 SND-LOOP value h"0008".
78 SND-NOSTOP value h"0010".

01 mfdata.
03 ptr-to-mmsystem procedure-pointer.
03 badptr procedure-pointer.

procedure division.

set badptr to entry "guess_this_is_a_pointer"
set ptr-to-mmsystem to entry "winmm"

if badptr equals ptr-to-mmsystem
display "Multimedia sound DLL not present"
stop run
end-if

call winapi "sndPlaySoundA" using
by reference "c:\win95\media\chimes.wav" & x"0",
by value SND-ASYNC size 2
end-call

if return-code not equal 0
display "Failed"
end-if
```
