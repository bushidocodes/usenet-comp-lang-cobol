The NetExpress product DOES allow you to use the Windows printing subsystem.
As you quite correctly stated, by default the product does send output for
files declared as ASSIGN TO PRINTER directly to LPT1. However, you can get
the product to send the output via the Windows print spooler without
requiring any source code changes by setting the 'printer_redirection'
tuneable (full details in the on-line help). In addition, if you want to
control the printer at a slightly lower level, there is a set of
PC_PRINTER_... calls.
