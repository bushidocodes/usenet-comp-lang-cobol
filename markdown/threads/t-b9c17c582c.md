```
There are SEPARATE migration guides for both the compiler (COBOL) and the
run-time (LE).  However, the bottom-line is that all "valid" VS COBOL II
code is supported without change with the newer COBOL for this-and-that
compilers.  (Actually, there are a few minor new reserved words - but not
many.)  Moving from the VS COBOL II run-time to the LE run-time is, however,
a bit more difficult but not nearly as bad as moving from OS/VS COBOL to VS
COBOL II.
```

Your occurs example could happen here, with say a variable length table
either filled with high values for a binary search or a range of
lengths.  If it normally reads in 500 records, but was written:
       05 MY-TABLE                          OC 10000
                indexed by my-index.
          10   MY-VALUE-1         PC X(5).
          10   MY-VALUE-2         PC X(2).

It is possible for this to compile with OCCURS 1000 (the trailing 0 is
in column 73).
