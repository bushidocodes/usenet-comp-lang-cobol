### IBM, OS/VS COBOL, CICS - Clarification (aka "back off")

- **From:** "William M. Klein" <wmklein@nospam.ix.netcom.com>
- **Date:** 2002-04-22T15:50:44-05:00

- **Message-ID:** `<aa1t39$dgb$1@nntp9.atl.mindspring.net>`

```
IBM has clarified (more accurately "backed off") its statement concerning
run-time support for OS/VS COBOL object code in the CICS release to follow
the latest CICS TS release.

Although my IBM sources indicate that this page is still a "work in
progress," (it has some pretty funny errors - IMHO)  please see:

    http://www-4.ibm.com/software/ts/cics/v2/cobol.html

Most notably, the last paragraph which says,

"So for the release after CICS TS V2.2, it will continue to provide object
compatibility, i.e. All applications will continue to work. CICS Customers
can, for example, move their existing OS/VS COBOL or COBOL II applications
to CICS TS V2.2 or later in complete safety as there are no planned changes
which will have a mandatory requirement for customers to recompile and
re-test there applications "

HOWEVER, make certain that you also read the earlier part that says,

"So if a customer running with CICS after CICS TS V2.2, has OS/VS COBOL
applications running but the run-time library is for instance COBOL II then
the problem will not be accepted by IBM support unless it is recreated using
the LE run-time libraries as this will be the only supported environment. A
similar situation exists for PL/I. So, the only migration effort is to LE in
order to remain fully supported by IBM. "
```
