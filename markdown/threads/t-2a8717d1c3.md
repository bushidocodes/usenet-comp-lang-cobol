W A N T E D

TOOLS FOR RESTRUCTURING OF COBOLSOURCE

In order to restructure FANS COBOL and VS COBOL II, I would like to be
informed about experiences with tools who are able to perform one of
the following tasks for batch or online-applications:

1. Restructuring a single program by manipulating with sections and
paragrafhs in the Procedure Division.
2. Restructuring an application by manipulating with programs
contained in that appplication.
3. Analysing which common data are used between programs in the
application (like fields in files and data parsed through the
Linkage Section). Providing a 'namelist' for the same logical
data.
4. Make it possible to give the same logical data a unique name
throughout all the programs in an application.

Looking forward to receive your response.
