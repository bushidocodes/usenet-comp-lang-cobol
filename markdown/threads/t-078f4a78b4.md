shakirvohra wrote:
> any one want to sell cobol netexpress 5.1 please?

If you are prepared to reinstall it every 30 days you could use the free 
version. (you need a virtual machine, but if you only use it occasionally, 
that's fine. You can download free virtualization software from Sun 
VirtualBox (recommended) or VMWare... both excellent products.)

Speaking as someone who has used it, I can tell you that the NetExpress 5.1 
implementation is of no interest to Micro Focus,who cannot/will not support 
it, you will need to pay a runtime license fee on anything you develop with 
it, and if you want to develop COM components it is "best avoided". 
(Although it technically supports them, the process for doing so is 
unecessarily convoluted, all COM components have to be developed through a 
Project, they require "trigger files" for registration, and there is no 
facility to batch compile such components.) For straightforward procedural 
COBOL 85 it is adequate, but Fujitsu is a better option for small developers 
because there is no run time fee.

Mocro Focus are more focused on their more recent offerings, and you cannot 
really blame them for not being enthusiastic supporters of NE 5.1.

Both Micro Focus and Fujitsu implementations of COBOL (non CLI generating 
versions... i.e. native code implementations... Say NE 5.1, Fujitsu Netcobol 
version 6.1 through version 9) are very good as far as COBOL goes, but they 
are both expensive (even now, when they have all been "replaced" by CLI 
generating versions for .Net), and, as far as I can tell, it is no longer 
possible to buy a single seat license for any of these products. (I was 
quoted $US20,000 for a 5 seat (minimum) license for Fujitsu Version 10, and 
maintenance is mandatory, not optional.)

This is just ludicrous for a product which has been superseded for years 
now; no wonder COBOL is dying.

I would suggest you try e-bay to see if someone who has moved on from COBOL 
will sell you their licensed copy of NE 5.1.

Personally, I wouldn't buy it at any price... :-)

I will sell a licensed copy of Fujitsu NetCOBOL, equivalent to NE 5.1 but 
without the runtime fees, (which is a very good native code implementation 
of COBOL, including support for COBOL '85 plus extensions, OO COBOL and 
PowerCOBOL for quickbuild GUI applications, plus mainframe flagging, but I 
only have a very limited number of licenses remaining, and maintenance is 
not included (although I will provide reasonable support to anyone who 
purchases), if you contact me privately. These are second hand transfers of 
ownership and not new products.

Pete.
