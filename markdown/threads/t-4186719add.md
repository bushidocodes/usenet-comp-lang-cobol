First off, it's not the dump you're looking at; it's the compile listing.
Finding the TGT in a dump is another issue.

The cause of your concern is the RENT option.

When a COBOL program is compiled with the RENT option, the compiler knows that
at run time a _copy_ of the TGT will be used, so it doesn't make any sense to
provide PGMLOC, which is a list of displacements of TGT fields relative to the
beginning of the program (where TGTLOC is a list of displacements of TGT fields
relative to the beginning of the TGT).

But if your program is compiled with NORENT, then the TGT compiled in the
program will be used at run time, so offsets within the program are useful for
debugging and run time support, so PGMLOC is provided at compile time. Does
this make sense?
