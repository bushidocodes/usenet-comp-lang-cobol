Then again, it may not.  Since it is complaining it cannot be found, and 
not complaining about a specific ordinal, perhaps the problem is that 
C:\Windows\System is not on the default dynamic-link search path.

This DLL seems to be central to the linkage between COBOL and Visual Basic.

I know on Windows NT or 2000, it is typical that the system searchpath 
is something like %SystemRoot%\System32.  I don't know about XP, but you 
should check if the system upgrade reused the old %SystemRoot% or not. 
You can also try copying this (and any other related DLLs) to the 
application's "bin" directory.

This DLL does not appear to be self-registering, so re-registering it 
manually will likely not work.  You may have to reinstall the app; this 
may be easier than hacking DLLs searchpaths on this machine.

Because Windows can't find it. There, that wasn't hard, was it?

If the file's not in the registry, not in the current directory, and not in
any folder mentioned in the PATH statement, Windows gives up.

So, you can:
1. Add the file to the registry,
2. Put it in the current directory, or
3. Modify the path to additionally point to the file's location.
