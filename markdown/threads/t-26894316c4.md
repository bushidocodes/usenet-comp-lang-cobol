Just read in the file in the Windows tool of your choice and before you
write it to the windows database of your choice, make sure to run strings
through the OemToChar() function.

F.ex in pascal:

    While Not Eof(Infile) Do
     Begin
      ReadLn(Infile, mystring);
     strcpy(Buffer, mystring);
     OemToChar(Buffer, Buffer);
     WinDatabase.Post(Buffer);
     End;

Since you are specifying *.DAT files, I get the impression your using
Micro Focus indexed files.  Unless your MF Configuration is setup to
compress data, you should be able to read the *.DAT file as a binary
sequential file.

As mentioned earlier you can use an ODBC to port the data to your
windows apps.

If you have a large number of files, you could do what I did and write
a simple batch program that has command line arguments, filename and
record length.  The program processes *filename* reading large blocks
(say 20480 bytes) of data into a buffer.  You move the data from the
input buffer in strings equal to the length of the command line, to an
output buffer.  After each string move, you append a x0D0A.  When the
output buffer gets full, you dump it to you output file and clean the
buffer.

Believe me when I say you want to use buffers rather than process the
files on a byte for byte basis.  The byte for byte processing is a
performance killer!  One of my On The Job Trainees did the byte for
byte business which took 10 minutes to process a 1/2 MB file where my
buffered program took 5 seconds.  Was a great learning exercise for the
OJT.

On other point you might want to make clear.  Are you converting file
formats or data type.  What I'm really trying to say is are you wanting
to convert EBCDIC to ASCII?  You can do this also in the program I
outlined above.

Siber Systems sells Data2Flat converter that does
just what you need: it takes Cobol data file and
translates it to text format. This text format
is values separated by commas (CSV).
Other separator characters can be specified.

Record layout for the Cobol data file is stored
in RDD (Record Descriptor Data) file that is
automatically produced from the Cobol program source
by Cbl2Fdd tool.

More on this at http://www.siber.com/sct/datafile/
