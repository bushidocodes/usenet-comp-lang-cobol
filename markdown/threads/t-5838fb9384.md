```
Actually, I believe the colors are brown on black, a default color
combination that could only have been selected by someone with the
ergonomic sense of a potato.  I'm sure they were inspired by the
vast number of text editors, word processors and terminal emulators
that uses that default. ;-)  No doubt, MF has failed to remedy the
gaff because so many users have come to love that illegible combo.

To correct the situation you need to do two things:

    1. Using ADISCF change the default colors to white on black.
       (Don't despair, if you're particularly clever, you should
        be able to figure out how to do it in a week or less.)

    2. In your AUTOEXEC.BAT file set the environment variable:

        set cobsw=+s5  (plus whatever other switches you may be using)

You see, the environment variable must be present so the executable
will listen to what you have already told it in ADISCTRL.  Whoever
built this nonsense into the system was determined.  What provokes
me is not that this is overly onerous, just that it is unbelievably
stupid, and should have been corrected before the first customer saw
the product.
```

```
You do it the same way it worked in 16 bit runtime.

    To set system (default) attribute:

77 SET-ATTR                     PIC X COMP-X VALUE 7.
77 THE-ATTR                     PIC X.
77 ATTR-ON-OFF              PIC X COMP-X VALUE 16.
77 ATTR-ON                        PIC X COMP-X VALUE ZERO.
77 ATTR-OFF                    PIC X COMP-X VALUE 1.

CALL X"A7" USING SET-ATTR THE-ATTR.            *> SET THE ATTR
CALL X"A7" USING ATTR-ON-OFF ATTR-ON.      *> INSURE THAT IT IS
BEING USED.

    Above assumes attribute is being defined as a hex value.
Make THE-ATTR comp-x
if you prefer decimal values.

    Above assumes int/gnt.  Probably works with EXE/DLL, but I
can not swear so.

    Above information is in NETX 3.1 help file.  Should be in
3.0.
```
