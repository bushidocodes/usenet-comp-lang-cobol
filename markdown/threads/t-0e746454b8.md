Where to start?

First, you need to take a look at:

Micro Focus COBOL Compatibility Guide

HP Part No. B2433-90062
Printed in U.S.A.
E0493

This talks about the many different issues that will have to be
considered when
porting code and data from other dialects of cobol.

Secondly, now that your in the middle of doing this, take a look at:


HP Micro Focus COBOL/UX Programmer's Guide

HP 9000 Series 700 and 800 Computers
Printed in U.S.A.
HP Part No. B2433-90020
Edition Fourth
E0796

Chapter 8 Debugging Tools and Tips


This chapter will discuss such issues as creating a stack trace.
However,
if your getting a runtime error and would like a core file to be
produced,
you can set a runtime tunable called core_dump. Here's a snippet of the
documentation:

core_dump.


Syntax: set core_dump={TRUE|FALSE}


Value: TRUE or FALSE

If TRUE, the run-time system upon processing a
fatal run-time error (whether software, operating
system, or hardware detected), that is not
intercepted by the application, prints an error
message, creates a UNIX core file, and then
terminates the process. Refer to the man page
core(5) for information on core files.

This core file can then be used by a system level
debugger (such as ADB, HP Symbolic Debugger, or
HP/DDE) to analyze the cause of the error. See
your system debugger documentation for more
details
on debugging using core files.

Default Value: FALSE. The run-time system does not create a UNIX
core file.

Hope this helps. If not, and you have purchased the product from HP and
have HP support, please give us a call where we can get more information
from you and better assist you.

Regards,

Tony Heffner
