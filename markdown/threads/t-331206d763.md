are currently running on the mainframe to a UNIX (RS6000 AIX)
environment. We are using IBM COBOL on the UNIX box.

I am getting a file status 35 error when I attempt to open a file.
Here is the select statement:

SELECT SORTINFO-FILE-IN
       ASSIGN TO SYS015
       ORGANIZATION IS SEQUENTIAL
       ACCESS IS SEQUENTIAL
       STATUS IS SORTINFO-IN-STATUS.

Here is the corresponding FD statement:

 FD  SORTINFO-FILE-IN
     DATA RECORD IS SORTINFO-REC-IN
     RECORDING MODE IS F
     RECORD CONTAINS 41 CHARACTERS
     BLOCK CONTAINS 200 RECORDS
     LABEL RECORDS STANDARD.
01  SORTINFO-REC-IN                  PIC X(41).

Here is the script that is calling the program:

#!/usr/bin/ksh

SYS015=/proddev/cobol/data/J6CDM210.050

export SYS015

/proddev/cobol/src/cdm210

where cdm210 is the Cobol program.

Any help would be greatly appreciated.

File status 35 means that "an open with i-o, input, or extend
has been tried on non-optional file that is not present"

you can explain you file as

export dd_SYS015=/proddev/cobol/data/J6etc

then you write - assign to "SYS015"

or you declare

01 SYS015       pic x(100)      value
"/proddev/cobol and so on"

then you write - assign to SYS015
