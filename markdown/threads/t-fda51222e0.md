```
Actually, this sort-of depends on the compiler and environment.  There are
operating system/compiler combinations where "header" information is placed
into the file - and that these files "only make sense" to a COBOL
application created with the same (or compatible compiler).  There are other
environments where all the ANSI/ISO defined file types (sequential, indexed,
and relative) exist in the "operating system" software and all programming
languages and compilers can access them the same way.

From the fact that this mentions ".DBF and TXT" - my GUESS is that this may
be a Windows environment.  If a COBOL program used the common PC extension
of "Line Sequential" when creating the file, the changes are that it can be
treated as a TXT file without modification (however, if it has any COMP-?
data in it, then this isn't true - but it probably shouldn't have been line
sequential in the first place).
```
