You could always do what programmers have been doing for
years:

Name your variables creatively.  If you can't do that, come as
close as you can to not naming them at all, e.g., use a single
letter with a bunch of digits.  Don't even think of indenting
your code, not even code subordinate to an IF statement,
because there will be no such code.  You'll be doing it all with
the time-honored GO TO.  In fact, you'll be doing better than
that, because security is important.  Where others might have
been content with mere spaghetti code, you will be using the
ALTER statement.

This technique has stood the test of time.  It has kept the trade
secrets in thousands of programs safe from spies (and everyone
else) no matter how long they look at the source.

Good luck.

Louis Krupp

Put the program parts that you want to keep secure in separate data files,
and get them into the programs by using COPY statements.

Although how to do it is going to depend on the platform, just about every
Cobol I have ever seen had some method somewhere that would allow control
over copied data display in the listings.

The fact that the security required source also resides in separate
files than non-secure source allows you better control at the librarying and
archiving level, as well, though that would be true regardless of language.

Under some compilers (MF for example), you can use conditional
compilation. If a condition is true, you will compile a portion of
code, if it is false, another one. I think it might be an answer to
your question. 

Under MF, the syntax is 
$if MY-CONDITION = "THIS"
  my code
$else
 another portion of cdoe
$end

You can set the condition in your source code or while calling the
compiler.
