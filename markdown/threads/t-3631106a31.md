Nope, VS COBOL II, COBOL/370 and COBOL for MVS and VM provide native
support for XA architecture only.

Seems to me there are some callable routines around to provide Windowing
services into dataspaces that can be called from code compiled with these
products.

Only product that I know of that provided direct support for dataspace
type data is a late version of VS Fortran that supported data defined in
those spaces (been a while since I researched that item) as I recall "this
is not for amateurs".
