```
If this is subprogram is going to continuously run in batch as well as CICS
you 
may want to put the guts of the code and working storage in copy members
and then code the CICS or file access in same named paragraphs (sections)
in the actual programs.  

This way when your program wants to read the invoice file it would
code :
	PERFORM READ-INVOICE-FILE
 in the copy member.  Then the CICS program would have a section like:

READ-INVOICE-FILE SECTION.

	EXEC CICS
		READ DDNAME
	END-EXEC

	(return code check)
	.

and the batch program would have a section like:

READ-INVOICE-FILE SECTION.
	
	READ FILENAME
		AT END (or otherchecks)
	END-READ
	.

This way when you have to modify the logic of the program
you only have to change it in one place (and compile and test
both).  I suspect that finding the copy member in procedure division
would encourage you to look for the other times it is used and
keep the two programs in sync.
```
