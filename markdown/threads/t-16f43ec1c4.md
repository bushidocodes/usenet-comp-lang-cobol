```
I think you are "bit confused".

LE does not (never has, never will) "support" any compiler - what it does is
support the object code PRODUCED by various compilers.

The LE in z/OS Version 1, Release 3 still supports load modules created by
all COBOL compilers from OS/VS COBOL V2.4 thru VS COBOL II (all releases)
thru SAA ADCycle COBOL/370 thru IBM COBOL for MVS & VM thru IBM COBOL for
OS/390 & VM thru IBM Enterprise COBOL for z/OS & OS/390.

Support for all of the compilers listed above up to and including IBM COBOL
for OS/390 & VM has been withdrawn or has announced withdrawal dates.
Support for all releases of OS/VS COBOL and VS COBOL II was withdrawn QUITE
a while ago.  HOWEVER, if you have (or even create new) load modules that
are link-edited with and run under a supported release of LE, they (the
object code) *IS* supported - as long as it does not use any "undocumented"
feature of the earlier compiler.

There has been an announcement that the release of CICS TS that comes AFTER
the latest one will NOT include support for OS/VS COBOL programs.  (CICS
"used to" need special support for OS/VS COBOL because it did not have a
REAL "re-entrant" feature).  Similarly, preprocessor support for currently
unsupported COBOL compilers is not available with the latest release of
CICS.
```
