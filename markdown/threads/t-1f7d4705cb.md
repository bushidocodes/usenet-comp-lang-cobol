Hi,

Server Express has the -z option on the cob command to link to shared
object. Server Express is the only Micro Focus COBOL product that directly
supports this format.

Given that you mention $COBDIR/coblib rather the $COBDIR/lib leads me to
believe you are either using OCDS or an earlier product. Shared objects (so)
are not directly supported in these products.

Are you sure you really need to use shared objects? Shared libraries can
give similar features and can be more efficient in some circumstances.

This quote from the 4.1 manual should tell you if your system supports this:

Note: The -B flag is supported only on UNIX operating systems that support
dynamic linking. If your operating system does support it (that is, the
library libcobol.so exists in $COBDIR/coblib), see your operating system
manual for more details on static and dynamic linking.

Look in your documentation about dynamic linking.
