```
If you do not specify FBA it is possible the file will be created without
them. Whatever parameters are not specified will be filled in by COBOL. Even
if you use the ADV/NOADV if you do not use AFTER/BEFORE ADVANCING the file
will not have carriage control characters in it. People sometimes write
reports without them.

Assuming I may be wrong, create a simple test program to verify this.
Compile and ruun. Only by trial and error can you be 100% sure. Create 4
programs. 2 w/ ADV specified 1 with ADVANCING clause, one without. 2 w/
NOADV clause 1 with ADVANCING clause, one without. Run all of them and check
attributes of output files. That is one way to be sure. The manual may not
tell you exactly what you are looking for.
```
