You did not specify a FILE STATUS clause on your SELECT...ASSIGN for
your SYSIN file. In the IBM world, this means that the operating system
will intercept any error and terminate your program.

If you had specified the FILE STATUS clause, then the IBM operating
system will not terminate your program when an error occurs on that
file, because you have implied to the operating system that you will
check every I/O on that file for errors. In this case you would have
received a file status of 47 if you were checking for errors. Since you
weren't checking, the operating system decided your program couldn't
continue.

A "logic error" in IBM-speak is generally an attempt to read or write to
a file that's not open, or attempting to open a file that's already
open, or attempting to close a file that's not open. Or you could be
trying to write to an INPUT file or read an OUTPUT file, or REWRITE with
an invalid record length or no prior READ.

It's generally helpful to know where the page of standard FILE STATUS
returncodes can be found in your manual or textbook. File Status "47"
means you are trying to read a file that is not open for INPUT or I/O.

I hope that helps.

Seeing that it's moaning about SYSIN, Here's some guesses:
1. There is an ACCEPT statement in the program, but no 'SYSIN DD *'
'card' in the JCL.
2. There is an OPEN INPUT SYSIN statement preceding an ACCEPT. SYSIN
doesn't need to be OPENed.
3. There is a FD for a file, and it's ASSIGNed to SYSIN, but COBOL is
getting confused because it thinks SYSIN is something special, not to be
used as the name of a 'user file'. ASSIGN the FD to some other name
(MYFILE) and change the JCL.

Offhand, I'll bet that X'0E82' is an OPEN statement.
