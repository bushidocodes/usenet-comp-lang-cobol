Actually it is the other way around.

Standard DISPLAY and ACCEPT default to being UPON CONSOLE, so your
ACCEPT is from the console.  If positioning phrases are added, such as
AT, then they default to being UPON CRT, so DISPLAY AT is on the screen.

The solution is to use all with the same variants, for screen change
your ACCEPT to be:  ACCEPT AA AT somewhere, or use FROM CRT or add
CONSOLE IS CRT in SPECIAL-NAMES.

From memory ACCEPT AT in Fujitsu requires every character to be filled
so you may need to space out to the end of AA.

If you want it all to be console then remove the AT from the DISPLAY.

Different systems interpret what the 'console' is in different ways.

> With Fujitsu the Display is actually in the "user window" (so was the

If I use Fujitsu (4) and, without special-names or compiler options, do:

      DISPLAY "Unadorned"
      DISPLAY "upon console"  UPON CONSOLE
      ACCEPT input-field

these all come out in the same window that has a title: 'Console:
program-name'.
Thus, as I said, "standard DISPLAY and ACCEPT default to being UPON
CONSOLE".

Strictly speaking the default is SYSIN and SYSOUT, but these default to
being the Console: window.

When I use AT or UPON CRT or use SCREEN SECTION then these come out in a
window that has a title 'Screen: program-name'.  If this is also called
the 'user window' then this naming is merely confusing in giving it yet
a third name (CRT and/or Screen being the obvious names). In fact I
can't find any reference to the term 'user window' in either version 3
or 4 Fujitsu Cobol manuals.

> (so was the accept).

The behaviour described in the problem was that the display and accept
were in _different_ windows.  The use of 'AT' in the DISPLAY made it use
the 'Screen:', the ACCEPT defaulted to 'Console:'.  Your claiming that
they both actually came out in the same 'user window' doesn't seem to
help really.

> To get "proper", or should I say "Accepted" behavior from Fujitsu v3
> with Display ... Accept... you need to use UPON/FROM Console, else the
> entry must be FULL in order to terminate the accept.

One can use CONSOLE or SCREEN as required but it is best not to mix
these (as the problem program did).  MF Cobol (and others) use the same
window for both, Fujitsu uses separate windows for completely unknown
reasons.

Using UPON/FROM CONSOLE means not using positioning or attribute on the
DISPLAY/ACCEPTs.
