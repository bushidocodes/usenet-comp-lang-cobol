Dear Sirs,

We are having runtime error 163 in our application of .int modules,
and we need urgent help about this.

Here is our story:

1) We have since 1992 an old 68030 Motorola with Unix
System V Relase 3 with MF runtime V1.1 revision 0.

2) Most of our programs have working-storage variable defined
like these:

01 var1 pic 9(06).
01 zar1 pic z(06).

3) We allways run our applications using cobrun -F application
and we never had problems.

4) If we don't use the -F flag, we get runtime error 163 on the
variables above.

Now we bought an Alpha 1000A with Digital Unix 4.0 and the
MF cobol runtime 3.2, and we copied all the .int modules from
the old 68030 into it.

Unfortunately, in the Alpha, we are allways having 163 run-time
errors with all the vars. PIC 9(XX) and unpredictable results
with our vars. PIC Z(XX), using the same command cobrun -F application.

We made some experiences trying to find out what was wrong.

A) We recompiled in the 68030 some programs using
cobol application.cbl SPZERO. The resulting .int went to the
Alpha computer. We tried cobrun -F in the Alpha and we got rid
of our problems both regardind the PIC 9(XX) and the PIC Z(XX)
vars.

B) We changed our vars PIC 9(XX). with PIC 9(XX) value zeros.
We compiled it on the 68030 and moved the resulted .int to
the Alpha. cobrun -F work fine, no 163 errors.

Unfortunately, in our 68030 we have something like more then 1000
.int modules and only a few ( +- 30 .CBL sources ), so recompiling
is no solution for our problem.


Is there any way to turn around this problem ??

Please, we need, repeat, need HELP.

Best Regards

Antonio Azevedo

fi··.@mai··c.pt
