```
to upper case using COBOL II.>>

Long, Long...... time ago, in the days of COBOL 68 I
occasionally had to do a conversion because the operator was
forever entering lower case (or partial lower case answers)
into the console.

It took a character by character evaluation of the subject
field. First, you move low-values to a pic 9(4) comp field.
Then you move the individual character to the last position
of a pic xx field (that redefines the pic 9(4) binary
field).  Then, check  the binary field for greater than 193
(if it is, it doesn't need converting).  If it isn't add 64
to the binary field, and move the second character of the
pic xx field back to the original location.  This assumes
you're using EBCDIC and the field doesn't contain
punctuation or spaces.  If the field can contain any
punctuation or spaces, you'll want to check for those values
and not convert them either.
```

```
If it is TRULY COBOL II - and Not "COBOL for this and that", this is 
your best solution:

Inspect Input-Field Converting "abcdefghijklmnopqrstuvwxyz" 
                                                    TO 
"ABCDEFGHIJKLMNOPQRSTUVWXYZ"

However, if you have the later cobol, use

Move Function Upper-Case (Input-Field) to Input-Field 

(Yes you CAN do this - but I prefer a different destination field).

You can LEAVE the data in mixed case in the file, and just convert to 
upper for the search.
```

```
Try INSPECT. Other COBOLs could use TRANSFORM or EXAMINE.

My COBOL II installation does not have INSPECT CONVERTING or functions,
so I have to do the following.

 INSPECT HJBDEBUG, REPLACING  ALL 'a' by 'a'
                                  'b' by 'B'  etc, etc.
 
I once worked in a shop which had a mixture of ANSI 68 and ANSI 72
COBOL.  It used EXAMINE to do the job.  Since EXAMINE was replaced by
INSPECT, I decided to use the simpler command which was available in
both ANSI versions:

TRANSFORM TEST-NAME FROM
  'abcdefghijklmnopqrstuvwxyz' TO
  'ABCDEFGHIJKLMNOPQRSTUVWXYZ'.

I thought this would be safe.  Well, now TRANSFORM is obsolete.  So
existing, working code has been changed twice to fit new ANSI standards.

Too bad, if I am between TRANSFORM and INSPECT CONVERTING, I have to
have clumsy code.
```

```
COBOL II doesn't have an UPPERCASE-TRANSLATION function. You can :

a) write a small assembler program that will do this for you, just OC the field in
question with SPACES. (possibly the fastest way)

b) INSPECT name-field CONVERTING 'abcdefghijklmnopqrstuvwxyz'
                                                       TO 'ABCDEFGHIJKLMNOPQRSTUVWXYZ'

c) INSPECT name-field REPLACING 'a'  WITH 'A'  'b' WITH 'B' ......
                                                            

d) migrate to COBOL for MVS & VM, or COBOL/390, which has FUNCTION UPPER-CASE
```
