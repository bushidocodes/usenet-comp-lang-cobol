```
I know you can direct me to several sites to check on how to handle
dates but I wonder if there isn't a case for CLC people as a group
producing 'open source' covering all date possibilities.

Purely as an exercise, and it isn't finished, I have been producing an
OO class to handle dates, e.g. 2000/01/01 Jun 20 00, Mai 20, Dimanche
23, that is trying to cover as many date permutations by language. Well
your response could well be, 'We already have installation standards to
handle that' - but as you know we do keep getting queries from
newcomers.

I'll make my OO source available, (if required) - and you are not to
laugh out too loudly at my code ! What I would suggest is that we need
three models :-

(a) Structured - no intrinsic functions ( so that includes date routines
    like Judson has produced)
(b) Structured - using intrinsics
(c) OO - using intrinsics (It doesn't make sense to use OO without 
intrinsics - if you got OO you've also got the latter).

To be of any value - as a starting point - it should address the issue
of Western European languages. Perhaps having produced a first model it
should then move on to Slavic, Kanji, you name it....

One part of the exercise would be to see how much consistency we have
between compilers to achieve clean compiles on different machines. Plus,
those 'additions' or 'language extensions' which would make it 'croak'.

If interested, may I suggest that there be three teams of co-operators -
pointless having 101 solutions - let's get groups talking together and
swapping ideas.

If there is any interest in this I'll post my OO source code.

Jimmy
```

Perhaps you should clarify whether you are looking for public domain or
open source code.  There are substantial differences between the two
concepts.

Public domain means there are no restrictions on the use of the code.
For example, anyone could take the code and sell it as a commercial
product to those who are not aware it is public domain.

Open source (I presume you mean open source as defined by the OSI
http://www.opensource.org/) is copyrighted and distributed with
restrictions.  It cannot be resold as a commercial product and any
improvements that an individual makes must also be released as open
source.

If you want people to contribute their efforts, then open source is the
way to go.

To clarify Thane,  Open source works like this:

Brainiac 1 creates a library of routines that answers some fundamental
programming need.

Joe average picks up the library, makes some small internal changes to
make it work better, these must be released back to open source.  An
then derives a product from the library.  The derived product must
reference the copyright of the library and have a method to get the
library source to the customer (any customer) this is traditionally
through a web page.  The derived product is Joes unless you in turn
choose to release it.

The library becomes stronger and more robust, more people look at it
and debug it than any other application so you have really good code. 
Bad code does not get supported by the community.

Another model of profit is to open source your application and provide
support on contract, look at the MySQL database as an example of this.
The source is open but owned by the vendor who sells support, not the
product.
