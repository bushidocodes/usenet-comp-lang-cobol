```
You *might* be thinking of EXTERNAL data.

If you put the EXTERNAL attribute at the 01-level of a COPYbook and bring
that "description" into any program (at compile-time) then the data is
AUTOMATICALLY available to every program that has it defined without passing
it in CALL statements.
```

```
Well, I know you can have GLOBAL data (q.v. in the documentation), and
there's always EXTERNAL data and there's nothing preventing you from
using...

01  GLOBAL-DATA     IS GLOBAL.
      COPY LIBNAME.

or
01  EXTERNAL-DATA  EXTERNAL.
     COPY LIBNAME.

Or for that matter, no reason not to....


LINKAGE SECTION.
01  PASSED-DATA.
      COPY LIBNAME.

Is this what you mean? (You'll need to study up on GLOBAL and EXTERNAL to
select which one you want). (For that matter, I'd have to do that, too).

IIRC (and I might not be) other programs not part of the run unit can access
EXTERNAL data and other programs which are part of the same run unit can
access both GLOBAL and EXTERNAL data.

(Style: Bias Alert) Don't code with EXTERNAL across run units, though. If
you have the option, use LINKAGE to pass data around. Lots easier to
maintain as the system expands. (GLOBAL within a run unit, yeah, I can see
that but I don't like it).
```
