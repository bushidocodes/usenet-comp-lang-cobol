I'm not certain I understand the request.

Do you want to represent the copybook as XML or the CONTENT of the
data area as XML - where you might have multiple "records" each with
the associated and appropriate XML tags?

XLST might be a way to take an XML formatted document and presesent it
as a COBOL copybook, but It can't go the other direction.

If the goal is to convert "COBOL data" (hate that term - data is data)
into XML formatted data using the copybook the generate the tags for
said data, then you might want to look at the new XML output feature
from the new version of Fujitsu COBOL.  It takes a copybook (really
record layout) and converts a data file to XML based on the layout.
