As I understand it, if you have freeform source then you don't need,
and can't have, continuations.  As freeform is an extension in '85
Cobol then any implementation can do this differently.

> (This is valid source inserted by the precompiler) 

You may need to have directives in the source to turn freeform off and
on.

> > * Accepted - sourceformat'free'

You seem to have it set for freeform.

> > -    "D  ,C.FIRST_NAME  ,C.MIDDLE_INITIAL  ,C.LAST_NAME  ,C.STREET
> > **    First character of a continued literal not a quote. Quote
> > assumed.

But continuations do not use '-' when using freeform (AFAIK).

You should be able to use the $SET SOURCEFORMAT statement within your 
code to switch between modes dynamically at compile-time as required, 
rather than reformatting your source at all, e.g. $SET SOURCEFORMAT"free" .

When in fixed format mode, the $ should be in col 7. In freeformat, this 
should start in col 1.

     .

It appears that the precompiler works on 4 byte increments (ie 28, 32, 36,
40). So, 32 bytes is as big as I can get. "02  FILLER PIC X(32) VALUE" takes
up the remaining 28 bytes for a total of 60 bytes before it wraps. The
wrapping period doesn't count in the size of the statement, so it doesn't
count in the length.

               include=/path1/path2/path3/ \
               maxliterals=32
