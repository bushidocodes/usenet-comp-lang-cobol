Jerry already mentioned that just about anything will "interface" to
Fujitsu.

You need to understand what ODBC is. IF you have a driver for ODBC and
configure it on your server or system, your Fujitsu COBOL can interface to
ANY relational Database that the ODBC driver supports.

Start with ACCESS. It is simple and a full RDB implementation. If you have
access installed you will already have the ODBC driver for it.

Some advice...

1. Start with some simple SQL embedded in your COBOL. You will need the
Fujitsu RTS configuration file set up to point at your database. (ODBCinfo).
Read the Manual.

2. Use the RDB PURELY as a receptacle for data. Drive it from your COBOL
program with embedded SQL. Do NOT USE RDB specific features like DB
Procedures on the database.

If this means nothing to you, mail me privately.

Pete.

Structured Query Language (SQL) has 7 or 8 basic "Commands" and is designed
to be easy to learn. Nevertheless, it is extremely powerful. (part of this
power comes from the fact that it is "Structured" and commands can be
"nested" inside each other, but that is more advanced usage...).

Do a Web Search for tutorials on it (Google gives at least a dozen...) I
have referred students to some of these sites and they picked it up very
quickly.

The best books are anything by Codd or Date (they developed the Relational
Model) but these are usually way beyond a beginner and a bit of an
"overkill" if you just want to manage a RDB from COBOL.

I am sending  you a paper I wrote on accessing Relational Data Bases from
COBOL. It is intended as a primer for people who have never done it before
and are not familiar with the environment. It contains sample code. This
paper has proved helpful in the past to many students.

HTH,

Pete.

If you came from mainframe, it's unknown. But I think Btrieve was the

then you may have come across it without knowing. But better check it
at www.pervasive.com.

syntax right) you just change the assignment like:

SELECT FILEA ASSIGN TO "FILEA.DAT"

to

SELECT FILEA ASSIGN TO "FILEA.DAT, BTRV"

where the keyword BTRV (if it's correct) identifies the file as
Btrieve.
