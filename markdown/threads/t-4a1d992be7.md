Any possibility you have a record which has neither Zero, nor 1, nor 2? 
That would take you to the 4th occurrence, which is undefined in the 
code fragment posted. Why do you specify "73 times"?

For such a small number of options, I'd probably code an Evaluate with 3 
cases and be done with it, e.g.:

Evaluate	CR-SATISFACTION-RATING
When		Zero
	Move  "DISS"	to	WA-RATING
When		'1'
	Move  "UND"	to	WA-RATING

You have a choice here concerning the next case, if you make it '2', I'd 
also code an Other case to catch any records with bum rating codes, if 
that's possible. Otherwise, I'd code:

When		Other
	Move  "SAT"	to	WA-RATING

End-Evaluate

I see no point in coding a Search for such a small number of arguments.

YMMV
