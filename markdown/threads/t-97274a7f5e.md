I doubt if your checking it against File Control, as such.
A KEY may be mentioned there, but that KEY is actually in
WORKING-STORAGE. That's if your ACCESS IS RANDOM/DYNAMIC.

Normally HIGH-VALUES is something you move into a field,
not normal for anyone to have this in a file.
You usually say AT END MOVE HIGH-VALUES TO a field name
of your choosing. In your case it looks like WS-STK-CONTROL
and your program is going to decide if it should go and READ
again.
e.g. IF WS_STK_CONTROL = HIGH-VALUES
PERFORM
END-OF-INPUT-ROUTINE
END-PERFORM
ELSE
PERFORM
GET-ANOTHER-RECORD
END-PERFORM
END-IF
If it's equal to HIGH-VALUES you can't read that file anymore,
it will ABORT the program, because it has already READ the
end of file mark on the disk.

Jamaaludeen

I did the VZaP course back in 1982, and I can see that not much has
changed with regard to the teaching methods.

Firstly, the File Control is used to define the links between the
LOGICAL (as used in the particular program) and the PHYSICAL (as stored
on disk, tape, printer...) files.

Each file to be used in the program needs to have a SELECT statement
that identifies where the data file is stored, and other characteristics
such as its organization, access mode, key structure ....

The FILE CONTROL area falls withint the ENVIRONMENT DIVISION of the
program because it is defines characteristics of te enviromment that the
program will be exucuted in.

>From what I remember STOREKEY is an area of working-storage that is used
to define control breaks within a batch-type program (Print, merge ...).

Good practice.
It is always important to identify the program structure before starting

In your main control section within the procedure division, it is common
practise to open all the files necessary for that particular program,
and then to perform some section until the promary file is at end. after
that, in a print program the final totals are printed, and the files
closed, and the program terminated.

This funnel that you talk about relates to the PERFORM UNTIL
END-OF-FILE. within this structure.

The line W-STK-CONTROL = HIGH-VALUES is just the end-of-file indicator.
Somewhere in the other sections there will be a read that says

READ NEXT [FILENAME]
AT END
MOVE HIGH-VALUES TO W-STK-CONTROL.

The W-STK-CONTROL is just being used as an indicator.
They could just as easily have used MOVE "Y" to END-OF-FILE-FLAG, and
then checked this condition.
