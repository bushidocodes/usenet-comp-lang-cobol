Greetings,

I am having a problem SELECTing with a CHAR column that has trailing
spaces in an Oracle database.  I'm using MicroFocus Workbench 4.0.32 and
Pro*COBOL 7.3.
Example:
column:         ClaimId CHAR(10)
column value:   'ABCD123   '
host variable:  S-CLAIMID PIC X(10).
variable value: 'ABCD123   '
Embedded SQL:   Select ClaimUID from Claim where ClaimId = :S-CLAIMID

This SELECT produces no match on the database, however

Select ClaimUID from Claim where ClaimId = 'ABCD123'
and
Select ClaimUID from Claim where Rtrim(ClaimId) = :S-CLAIMID
both produce a match on the database.

Of course we don't want to use Rtrim because it is not very efficient,
also we would like to be able to use the same code for Sybase and SQL
Server.

If anyone has the answer to this or knows who has the answer, pls email
me at kkitowski@issre.com.

Thanks in advance for your help.

Regards, Kevin

We realized this bug working with Oracle and DEC/OpenVMS with the 
Pro*COBOL pre-compiler.  The bottom line is that the variables on the 
right side of the condition are ALL rtrimed automagically.  So to get 
a match the field has to be full or you have to rtrim it.  We Rtrim 
both sides.  JUST IN CASE down the road Oracle fixes the bug.
