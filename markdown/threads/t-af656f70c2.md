```
From Fujitsu COBOL 97 you can start an EXE program by creating a windows
process.  There is an example named "CALLEXE" on www.adtools.com under
"V4 and V5 downloads".

One possible problem with this approach is that the new process is
independent of the calling program, which continues to execute and does
not wait for the CALLED program to terminate.
```
