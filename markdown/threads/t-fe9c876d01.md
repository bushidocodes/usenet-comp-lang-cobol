I'm having some problems with PowerCOBOL TAB controls.

I can "forgive" PowerCOBOL not using TabPages as objects, because the
TabPages are considered to be properties of the TAB control, but this
creates a problem with "ownership".

If you add controls to a TAB page, in the PowerCOBOL Form Designer, you
could reasonably expect that page to own them. If I ask one of those
controls to show me its parent, or its container property, I would expect
it to report either the TAB page it is on (that would be ideal and is
exactly what Windows Forms does) or the actual TAB control. (not so useful,
but still reasonable)

Instead, it seems to be reporting the actual Form...(Which is, of course,
the ultimate "parent"...)

This is stupid and pointless and violates the fact that the documentation
states that the TAB control is a "Container" just like a Group Box (Group
Boxes work perfectly and all their child controls report the Group as their
parent)...

I can't believe that PowerCOBOL is this "broken" so I think I must be doing
something "wrong" or I have a flawed understanding of it.

If anyone is dealing with PowerCOBOL TAB controls and can offer me a few
tips I'd be very grateful.

Cheers,

Pete.

"I used to write COBOL...now I can do anything."
