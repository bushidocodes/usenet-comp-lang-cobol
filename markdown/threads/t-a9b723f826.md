I need some assistance in the mechanices of calling BIN and
EXEs from Cobol in DOS using MF Cobol v3.2.50.

I have to write several small assembler programs that accept
parameters from the Cobol program. I don't know Cobol at all,
so I'm hoping someone out here can rescue me!

I don't have a problem with parameters passed "by reference".
They are all what I would call "far pointers". I also realize
that calling BIN and EXEs order the parameters differently on
the stack. (That was a bit of a suprise!)

My problem is dealing with parameters passed "by value". I
think that the "by value" parameters from the calls are
placed on the stack as 4 bytes in size. The correct value
is in the bottom 2 bytes and the top 2 bytes seem to be ???.

I also noticed that I have to return a 4 byte value. That is,
I have to place the value I want returned in reg AX and clear
reg DX in order for this to work.

Does Cobol always pass "by value" integer parameters as 4 byte
values? Is there some way to pass them as 2 bytes? What if we
pass a byte "by value"? Is it also widened to 4 bytes instead
of two? How can we achieve world peace in our times?!

Many thanks in advance for any help.


This code snippet is from the Cobol calling program.


WORKING-STORAGE SECTION.

01 WS-WORK-AREAS.
05 BUF-LENGTH PIC 9(04) COMP-5.
05 VALUE PIC 9(04) COMP-5.
05 BUFFER PIC X(23).

PROCEDURE DIVISION.

move 23 to BUF-LENGTH.
move 'This is a test message.' to BUFFER
CALL 'ComputeFunc'
USING by reference BUFFER
by value BUF-LENGTH
returning VALUE.

stop run.


//----------------------------------------------------------
// "Daniel Helgason"
//

Hi. I`m by no means an expert on DOS or our product that runs on it, so take
this with as large a pinch of salt as you can muster, but there is a "SIZE"
option to the "BY VALUE" clause :

CALL "prog" USING BY VALUE 106 SIZE 4

... will give you 106 in binary on the stack, occupying 4 bytes, in machine
order. I`ve no idea if it`ll work, but you could try playing with the specified
size. I don`t know if the system will still try to align the stack evenly or
possibly to 4-byte boundaries.

MFocus has chosen to set the size of BY VALUE and RETURNING (GIVING) fields to 4
bytes as a maximum. You should be able to lower the size of the BY VALUE field by
including a SIZE clause (number of bytes):

CALL 'ComputeFunc' USING by reference BUFFER
by value BUF-LENGTH size 2
...

I don't know what you can do about the RETURNING field.

Del.
