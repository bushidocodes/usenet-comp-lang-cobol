```
The following is the information that I received from one of my sources at Micro Focus:


"
I'm not sure that Oracle supports OCI calls from Cobol programs.

The problem the user is having is that they have not linked in the oci
libraries into the new rts. By default, the make file as produced by Oracle
does not include the OCI libraries. According to the docs, the list of OCI
libraries should be in the platform specific user guide.

› From looking at their docs & searching around the various Oracle make
files, I think the library the user needs to include is libocic.a.

The make file the user could look at is oracle.mk, this will probably be in
the $ORACLE_HOME/rdbms/lib directory.
"
```
