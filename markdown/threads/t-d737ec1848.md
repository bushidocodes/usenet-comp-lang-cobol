Hi,

Using COBOLII's pointer feature, you can access MVS control blocks to
get to the DSN.

e.g. CVT @ x'10'; CVT +0 => OLD/NEW TCB PTR;ONTP + 0 => TCB; TCB + 16
     => TIOT; TIOT + 24 => 1st DDENTRY. Find the one you want(loop).

That DDENTRY + 8 => JFCB; JFCB + 16 => DSN. GDG @ o/s 45 of DSN.

P.S.

The 1st byte of each entry contains the length of the entry unless
there are no more, then the 1st 4 bytes ore x'00'.

Open the file first, I think that makes the absolute GDG available.
If the DSN you're interested in is in a concatenation, additional
logic is necessary.

Hope this helps.

Jack

>Which offset changed at which release level?  I could be mistaken, but I

The offsets haven't changed, it's the contents of some fields that may
have changed.  For example, the 24-bit JFCB address becomes an SVA (SWA
Virtual Address) when your job runs with SWA=ABOVE.  You then have to
convert the SVA to a 31-bit adddress, if you know how to do it, of course.

The SWA=ABOVE issue has been mentioned on this list previously.  Accessing
MVS (or VSE, or any OS) control blocks using the POINTER facility is often
doable, but you'd better know what you're getting yourself into before you
start relying on it.
