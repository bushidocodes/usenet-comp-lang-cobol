```
Dear:

In all programming output, there could be output-to-sytem-storage such
as databases or data files. There could be output as reports to users.

In your case, you are talking about output as reports to users. In this
case, programmers should be the one who design the reports, that are
already formated in the programming codes. Reports should not be expected
to be formated manually by the users. That is the reason no programming
books discussing on how to use word processing to format reports generated
by a certain programming application. If you want to do that, please,
consult word processing teachers or clerks. Your instructors are wrong
telling you to manually edit a report generated by programming application.
In business, this practice may cause the security issue, for no one is
allowed to edit a report. Reports should be in their original physical
state: files or in databases in the system. A boss who attempts to edit
a report is going to commit a crime.

If you want to print your report, you may open it in any text editors
such as DOS, MS LearnC, Borland C, Visual Basic, Visual C++,
WordPerfect5.1, WordStar or MultimateAdvantageII (they are distinct few
years ago) and print the report from there. They do not have problems as
you have using Windows word processing applications.

The other way to print a report in Text format is sending it to the printer
by the following command:  C:\>copy report.txt LPT1
(LPT1 or LPT2 is the COM port the printer connect to)

Good luck.
```
