You should note that I said 'can' not 'is'.

MF Cobol can output files as .int, .gnt or .obj/.exe (depending on
version and platform).  When outputting .obj/.exe the content _can_ be
intermediate code by specifying OPT"0".

Using .int/.gnt gives portability to different machines and operating
systems.  Using .exe enforces DOS/Windows but intermediate code, while
being rather slower, is much more compact.  For interactive systems
CPU performance is usually not an issue in most of the code.

> I haven't tried this in Net Express, but I used to write some small utility

to the equivalent in other languages, but then I don't use MSC or MS
