Well, COBOL ist just an abbreviation for 'Common business oriented
language'. It's a programming language with several dialects. There are
(ANSI, ISO) standards, known as 74 and 85 (the year these standards are
published), and a draft standard called 99 which will be published ind
two or three years.

'VS COBOL II' from IBM is a standard 85 COBOL, its version 4 is still
supported until March, 2001. It contains constructs like 'END-IF',
in-line PERFORMs, SECTIONs and nested programs.
Its predecessor was called 'OS/VS COBOL' (also known as 'COBOL I'), a
standard 74 language. Not supported since years, but still works on IBMs
OS/390 platform, even under LE runtime.

'IBM COBOL for MVS & VM' is a more current version of the COBOL
language. It superseds 'IBM COBOL/370' which is not supported anymore.
The current version is 'IBM COBOL for OS/390 & VM', the current release
is '2.1.1'. All these versions need LE (Language Environment) as runtime
and have some elements of the next (draft) standard such as object
oriented possibilities and recursive calls.

With the current 'COBOL for this and that', you can interact with C/C++
modules and DLLs.

Does this answer your question?
