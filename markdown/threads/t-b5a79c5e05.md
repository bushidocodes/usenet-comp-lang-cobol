    The way we get the PID is via a C program that we link into the runtime.
When I get into the office on Monday I'll send a copy of the C program to
you via e-mail (I think its a one-line program).  Our session file is fairly
simple - We have a single key that is the user's login into our application.
While they are in the application we have the program keep this record
locked.  Then we read the file sequentially and whenever we get a "9D"
status, we know that record is locked.  We then store all kinds of
information in the record -- User's Name, date and time they logged in, PID,
current database they are accessing, etc.  Once the user logs out, the lock
is released on the record and the "old" data just sits out there.  I hope
this helps.
