```
For your information an IBM Apar is opened , by me ,  for a Cobol problem.
This apar is not accessible at this moment but it contains :

APAR= PQ61957  SER=                            IN INCORROUT
EXEC INCORRECT RESULT CMPL ARITH(EXTEND),OPT AND COMPLEX COMPUTE
WITH EXPONENTIATION EXECUTED BY PERFORM...THRU...UNTIL STATEMENT

ERROR DESCRIPTION:
A COBOL program gets incorrect results using the compiler
options ARITH(EXTEND),OPT(STD!FULL) in that a complex COMPUTE
involving exponentiation gives the answer of 000000000.0000
when the paragraph containing the COMPUTE is executed
with a PERFORM...THRU...UNTIL statement.
*
 The code involved is the following:
       WORKING-STORAGE SECTION.
       01 RESULTC         PIC 9(9)V9(4).
       01  FIELD48040-0500      PIC 9(9)V9(4) COMP-3.
      *---   SETTING TO = 48040.0500
       01  FIELD1            PIC 99V9(5) COMP-3.
      *---   SETTING TO = 01.00000
       01  DATA0                    PIC 99V9(5) COMP-3.
      *---   SETTING TO = 00.00000
       01  EXPON0                   PIC 9(2)V9(5) COMP-3.
*---   SETTING TO = 00.00000
 01 COUNT-IT                     PIC 9(3) COMP.
*---  SETTING TO = 000
 PROCEDURE DIVISION.
     MOVE 0 TO  DATA0.
     MOVE 0 TO EXPON0.
     MOVE 0 TO COUNT-IT.
     MOVE 1 TO  FIELD1.
     MOVE 48040,0500 TO  FIELD48040-0500.
     PERFORM CALCR THRU CALCR-EXIT UNTIL COUNT-IT = 3.
       GOBACK.
 CALCR.
     IF COUNT-IT = 3
            GO TO CALCR-EXIT.
     ADD 1 TO COUNT-IT.
     COMPUTE RESULTC =
                  ( FIELD48040-0500 *  FIELD1
               * ((1 +  DATA0) ** EXPON0)
               * ((1 +  DATA0) **  (1 / 2))
               * ((1 +  DATA0) ** (COUNT-IT / 360))).
            DISPLAY RESULTC.
       CALCR-EXIT.   EXIT.
****
NOTE:
The same results occur with the COBOL/OS390 2.2 compiler
and the Enterprise COBOL/zOS 3.1 compiler.
LOCAL FIX:
If any one of these changes is made the result is
correct as 000048040.00500  :
--Use ARITH(COMPAT) with either OPT or NOOPT.
--Use ARITH(EXTEND) with NOOPT.
--Perform the COMPUTE paragraph without the UNTIL clause
  or code it inline.
```
