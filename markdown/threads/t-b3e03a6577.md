```
>What would you say is "religious use of
the scope delimiters" ?

I would define religious use of anything as use
by rule. (as opposed to use when one decides
on a completely different basis ... does it make
the code read better?  ... is it REQUIRED to
make the compiler recognize the construct
correctly ...)

If every "IF" must end with an "END-IF" then use
of the scope indicators is 'religious'; however if
every "IF" statement must simply be easy to read
then they are being used correctly (in my opinion).

If
    (every "IF" must end with an
"END-IF")
then
    (use of the scope indicators is
'religious');
however
if
    (every "IF" statement must simply be
easy to read)
then
    (they are being used correctly (in
my opinion))
    end-if.

It is what you are use to. In a specific
context a given literary device may either
add to the readability or obscure it, depending
on circumstances and the skill of the writer.

>How would you characterise using a full-stop wherever it is
>possible to do so even if it is not needed for any purpose
>such as terminating a scope ?

Now that is an interesting question.
I consider

This-is-a-paragraph.
      perform first-step
      perform second-step
      perform third-step
      perform four-step.

and

This-is-a-paragraph.
      perform first-step.
      perform second-step.
      perform third-step.
      perform four-step.

To have subtly different meanings, though only to the
programmer and not to the compiler. I would assume
with the first, that the following would be true:


This-is-a-paragraph.
      perform first-step
      perform second-step
* note no periods.  For some reason the programmer wanted
*this to be kept to-gether
      perform third-step
      perform four-step.

Whereas with the second, I would have no compunction in
changing the code as follows:

This-is-a-paragraph.
      perform first-part-that-i-am-going-to-expand.
      perform second-part-that-i-am-going-to-expand.

first-part-that-i-am-going-to-expand.
      perform first-step.
      perform second-step.

second-part-that-i-am-going-to-expand.
      perform third-step.
      perform four-step.
```
