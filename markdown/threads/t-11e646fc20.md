Something we installed for a client was -

*
* GET THE CURRENT TERMINAL USING THE ASSIGN COMMAND.
*
         EXEC CICS ASSIGN FACILITY(TERMID)
*
* HAVING GOT THE TERMID, SET UCTRAN OFF.
*
         EXEC CICS SET TERMINAL(TERMID) NOUCTRAN

Warning - You'll need to use the SP option to compile this. See
http://www.s390.ibm.com:80/bookmgr-cgi/bookmgr.cmd/BOOKS/DFHLA814/2%2e87?SHELF=DFHLSH25

Combining ideas here - first you must have ASIS on your RECEIVE MAP
command, otherwise the lower case will be translated to upper case.
You need this even if you issue the SET commands in your program.

Next, if it is possible for your CICS Sys Admin to change the TCT
terminal definitions so that lower case is not automatically changed,
then that needs to be done and you don't need to put any SET commands
in your program.

However, if it is not possible (most likely case due to other apps),
then you need to issue the SET commands detailed above.  Just remember
to change it back before the program exits.  You could be causing some
undo headaches somewhere else if you leave the termianl changed or
your program abends before you are able to change it back.  Use
caution.
