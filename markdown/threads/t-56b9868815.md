Hi. You don't specify which operating system you are using. Please *always* do
this when asking questions. I'll assume you're using UNIX.

1) Compile all COBOL programs you want to animate for animation.
2) Link your C functions into the COBOL RTS.
3) Set the environment variable COBSW to include the +A switch. Export it.
4) Use the RTS with your C functions in to execute the COBOL program.

eg (bourne shell/korn shell):

$ cob -a *.CBL
$ cc -c *.c
$ cob -xo newrts *.o
$ COBSW=+A; export COBSW
$ ./newrts # if you had a main() in your C modules.
or
$ ./newrts COBOLPROG1 # if you had no main().

The animator will pop up each time you enter one of the animatable programs
(unless you alter this behaviour using 'Z'oom, 'E'nv/'P'rog-break etc).

You can also use a C debugger and Animator at the same time. To do this, ensure
you compile and link your C programs for debug (-g flag to 'cc' and 'cob') then
run the newrts with :

$ sdb ./newrts

.. or whatever your favourite debugger is. You can now set breakpoints etc in
the debugger and start running the program. Animator will still pop up when you
enter the animatable COBOL programs, and your debugger will control the C
debugging.

You did not mention the platform on which You are doing this. If it is some
kind of Unix, this may help:

a) COBOL(main) -> C(sub1) -> COBOL(sub2):
Build a new RTS including sub1. Set the environment variable for animating
( export COBSW=-A, or was it +A?). Run the program with "cobrun main". Now
all COBOL-modules for which there is no .gnt file are animated.

b) C(main) -> COBOL(sub1):
You have to call the RTS from main. I forgot the name of the special entry.
So You have to link Your main program with the COBOL-librarys.
If You set COBSW=-A (or was it +A?) COBOL-Modules are animated if there
is a .int and no .gnt.
