Hmm...the question here might be to ask, if he wants the DOS window to
maximize or minimize and not in PowerCOBOL?

The feature to minimize and maximize windows is already capable to coding
in PowerCOBOL for its GUI apps.  It's the DOS window, which tells us
another story.  From what I understand and from what I've read on the
subject, I believe you have to manually make the DOS screen full size
manually and manually minimize or maximize, whatever.

All Fujitsu COBOL products running in DOS are actually running under
Windows on their Windows  system.  Trying to run Fujitsu COBOL in DOS
without windows in memory would not work because Fujitsu COBOL has all its
compiler/runtime files meant to run using Windows. Thus the reasoning, why
not send a call to make this api function possible?  Sorry, but the
function isn't there nor will it ever be.

However I think there is a solution here on this subject. You CAN I believe
create a INF file (I think it's named, I'm probably wrong, but some of you
will know what I mean.).  It's for when running a DOS program and when a
DOS program runs, it looks for the INF?? file and if it's there, it will
look at the settings you want for that DOS window to appear. If you want
full screen, it will do that.  If you want minimized first, it will do
that.  This may be what you're looking for for those of you who want to
control the window size when it first opens.

Hope this helps.
