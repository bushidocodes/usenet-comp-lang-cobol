```
Hellough everyone:

As some know, I happily use my ancient 16-bit compiler and can say
that I
haven't learned anything new in 20 years (other than html and web
stuff).
Still, my 16-bit compiler and ancient version of SP2 still provide me
with
the tools to do the things I want to do.

I recently discovered on-line Scrabble and found myself writing a
Scrabble 'dictionary' to help my play.

The official Scrabble 'TWL98' dictionary contains 182k words and is
used
to validate play.

So I wrote a program to help me play according to the TWL98
dictionary.

The dictionary source file is TWL98.TXT containing 182,853 words
(Official
TWL98 dictionary).

The program creates the ISAM file:

        SCRABIND.DAT is  7,862,664 bytes
        SCRABIND.IDX is 18,742,272 bytes

The pertinent file description is:

       01  SCRABX-REC.
           02 SCRABX-KEY PIC X(20).
           02 FILLER REDEFINES SCRABX-KEY.
              03  SCRAB-1ST PIC X(01).
              03  SCRAB-C   OCCURS 19 TIMES PIC X(01).
           02 SCRAB-SIZE.         (alternate key)
              03  SCRAB-A  PIC 9(02).
              03  SCRAB-B  PIC X(20).

        (SCRABX-KEY and SCRAB-B are the words from the TWL98 file.)

I ran the program to create the ISAM file four different ways with
the results as follows:

Desktop Pentium D 3.7ghz 4gb RAM - 8gb memory stick. XP -  150
seconds.

Desktop Pentium D 3.7ghz 4gb RAM - 70gb hard drive. XP -  141 seconds.

Netbook Intel Atom 1.6ghz 1gb RAM 160gb hard drive. XP - 665(!)
seconds.

Laptop Celeron 1.6ghz 2gb RAM 70gb hard drive. Vista - 166 seconds.

I am curious as to how the speed of a my ancient 16-bit compiler and
ISAM
would compare to something written using modern stuff (even other
than
COBOL) such as Java, .NET, SQL or whatever.

If interested persons would write a simple program using their chosen
software to produce the equivalent of the ISAM file using TWL98.TXT
and post the resulting times, it would be good.
It would be even better if one chose to write the functional
equivalent of my COBOL program.

You can download the TWL98 text file and my COBOL program from
foodman123.com/scrab.htm.

Has the hardware become so fast that efficient software (in terms of
size and speed)
is not necessary?

I wonder, how much faster would the modern software be?

How big would the downloadable executable be in the more modern
language/access method?

How big would the equivalent of the ISAM file be?

thank you

tony dilworth
```
