The file prepared in notepad will be a text file with each record
variable in length and a CR/LF terminator.  CR=Carriage Return, LF=Line
Feed.  The deafault ORGANISATION of a file is SEQUENTIAL where the
records are fixed length.  You need to specify that this file is
ORGANISATION LINE SEQUENTIAL so that it reads each record up to its
CR/LF terminator.
 
>        FD  table-file.
>        01  table-rec.
>            05  month-num   pic x(02).
>            05  month-wrd   pic x(09).

This doesn't define exactly how the records are.  You have the file
(above) as '1  January' (with 2 spaces).  You need to define this as:

             05  Month-Num   PIC XX.
             05  FILLER      PIC X.
             05  Month-Wrd   PIC X(9).


>        open input table-file
>             output print-file

You should add a FILE STATUS in each SELECT and check this after each
OPEN and other operation. 

>        perform until eof-in
>            read table-file
>                at end
>                    set eof-in to true

>                not at end
>                    perform 1000-table-entry
>            end-read
>        end-perform

I expected '1000-table-entry' to store the record just read into the WS
area.  It doesn't.  The PERFORM will loop reading one line from the file
each time.  You will be performing 1000-table entry 12 times.

What you probably should be doing is:


	 SET X1      TO 1
>        perform until eof-in
>            read table-file
>                at end
>                    set eof-in to true
>                not at end
                     MOVE Table-Num  TO WS-Table-Num(X1)
                     MOVE Table-Wrd  TO WS-Table-Wed(X1)
                     SET X1 UP BY 1
>            end-read
>        end-perform

At this point the 12 (I assume) entries from the file are now in the
table.

>        perform 1000-table-entry
 
>        perform 3000-print
>               varying x1 from 1 by 1 until x1 > 12
…[3 more quoted lines elided]…
>        .

This bit (above) is fine, and so is 3000-print.
 
>        1000-table-entry.
> 
…[7 more quoted lines elided]…
>                   stop run

Why the STOP RUN ?  It is in fact at this point that you should be
saying 'not found'.  

>            when ws-month-num (x1) = v-month
>                 display 'found'
>                 move dl-month to ws-month-wrd (x1)
>            when ws-month-num (x1) not = v-month
>                 display 'not found'

Saying 'not found' here will result in several 'not found' messages.

>        end-search
>        .

Your current logic is that after each line read you will search an empty
WS table prucing 12 'not found' and then do a stop run.  The 2nd line
(february) is never read.
