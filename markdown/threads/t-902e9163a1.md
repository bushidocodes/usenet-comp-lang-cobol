There is a lot that is different.  One thing is the limited support 
available if you are creating PC applications - says so right in the 
manual.

Next is the fact that the indexed file structure is very VSAM like.  
It does not allow record locking (at least with the NATIVE structure) 
and does not allow split keys.

Conversion from Realia to MF would be relatively easy - MF to Realia -
virtually impossible.

Realia has no 32 bit GUI - and thier 16 bit is a joke.  You could use 
a 3rd party product with Realia to create the GUI - something like 
COBOL SP2.

The other problem (and you will have this with any COBOL except maybe 
AcuCobol  - No 32 bit OS/2 support) (their 16 bit product - reportedly
dead - did have good OS/2 support).



> We are also considering other products, such as IBM (we have a meeting with
> them thursday this week). Any comments on IBM's COBOL products?
>

All reviews so far have been stellar - but I have no direct experience
- I think you will need a different version for each platform.  I 
don't know of anyone doing end user development with it - only in 
house and mainframe connectivity stuff.
 
> Our current operating system on the workstations is OS/2, but probably soon
> will be Windows NT (or Windows 98).
> 

Other vendors to consider are AcuCobol and Fujitsu.

Fujitsu has a really good 32 bit compiler for Windows 95/98/NT that 
has a good migration path from MicroFocus.

Question - why not move to a supported MicroFocus product?  Mad about 
them pulling the rug out from under you?  

The best PC COBOL solution I have seen for end user development is 
Fujitsu COBOL with COBOL Sp2 from flexus (Fujitsu: 
http://www.adtools.com, Flexus: http://www.flexus.com) SP2 for the GUI
and thier Form Print product for printing.  It all works and it works 
well.

Good luck.

```
We've been using CA-Realia COBOL for a long time (as well as the other guys'
products), and we like it.  The 32-bit workbench, like any of the workbench
products, takes a bit of getting used to, but it works well.  Not as easy
and efficient to use as the old DOS products, but given that everyone is
chasing after Windows and workbench suites, it's a nice product.

Our experience with support is sort of ho-hum, but that seems pretty
universal unless you pay big bucks for the premium support with any of the
vendors.

It's a fact that the PC environment takes considerably more know-how than
the mainframe environment (where the Technical Services guys hide the
low-level stuff from the application programmer).  We field lots of tech
support calls where it's obvious the programmer doesn't understand linking,
argument passing, and some of the more basic stuff about how a program goes
from source code to an executable application.  FWIW, it doesn't have much
to do with using our products, but we help 'em out all the same...

Although the manual says they don't support PC application development, it
is fine for that purpose.

We're using the CA-Realia COBOL Workbench 3.0 to develop a full GUI screen
manager to follow up on our popular ScreenIO product, and things are going
extremely well.
```
