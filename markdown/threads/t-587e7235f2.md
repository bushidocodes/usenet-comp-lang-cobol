```
variable length records are defined like this:

In the FILE SECTION, say
RECORD CONTAINS integer-1 TO integer-2 CHARACTERS

In the record description, use a variable length record, with a OCCURS
DEPENDING ON clause.   like this...

01  RECORD-DESC.
    05  FIXED-PART.
       10 REC-HEADER  PIC (...)
       10 TABLE-OCCS  PIC S9(4) COMP.
    05  VARIABLE-PART.
       10  TABLE-ENTRY  PIC X(nn) OCCURS 0 TO 5 TIMES DEPENDING ON
                                  TABLE-OCCS.
```

```
Please, PLEASE don't use this technique (unless you are STUCK with a '74
Standard compiler).  With any compiler from the last decade and a half, a
MUCH better way to code this same thing would be:

In the FILE SECTION, say
RECORD Varying in Size from integer-1 TO integer-2 Depending On
Working-Storage item.

The rest of the record description is probably OK, but with the new
(improved) syntax you do not NEED to have a "counter" actually in your record
telling how large the variable part is.
```
