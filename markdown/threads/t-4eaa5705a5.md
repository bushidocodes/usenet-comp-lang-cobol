Just a reminder that in less than one week now, support for VS COBOL II
ends.  This (of course) does *not* end support for VS COBOL II object code -
when link-edited and run with a supported LE run-time/link-edit library.
However, it does mean that (at least in the US) the last IBM mainframe
pre-LE COBOL "environment" is going, Going, GONE !!!

the longer you use them for "mission critical" applications, the more danger
there is of REAL problems.
