One way would be to call an ASSEMBLER Sub-Program, which would pass back a
doubleword-binary (PIC S9(18) BINARY) value to the COBOL Program containing
the results of a STCK (Store-Clock) instruction. The caller can then
manipulate the value using different methods (such as date, time, etc) to
arrive at a random-number.

STORCLK    CSECT
                    STM          14,12,12(13)        SAVE REGISTERS
                    USING       *,15                     R15 IS
BASE-REGISTER
                    L               1,0(,1)                  PARAMETER
ADDRESSABILITY
                    STCK        0(1)                      ISSUE A
'STORE-CLOCK'
                    LM            14,12,12(13)         RESTORE REGISTERS
                    XR            15,15                    MAKE COBOL HAPPY
                    BR            14                         RETURN TO THE
CALLER
                    END

    BTW, the results of a STCK instruction turns over around Sept 2042 and
will not be in compliance. Get ready for the Y2K+42 crisis! Save it for your
grandchildren....

Cheers,

WOB,

           02  CW-RND-NUMERO           PIC 9(10).
           02  CW-RND-SGTE             PIC 9(9) COMP-4 VALUE 1.
           02  CW-RND-AUX              PIC 9(18).
      *

      ************************************************
* cp-rnd    : R.S. para obtener numero random (azar)        (PD) *
*           : Salida      :
CW-RND-NUMERO                                  *
****************************************************
*
       9999-OBTENER-NUMERO-RANDOM.
      *
           COMPUTE CW-RND-SGTE =
               (CW-RND-SGTE * 1103515245) + 12345.
           DIVIDE CW-RND-SGTE BY 65536 GIVING CW-RND-AUX.
           DIVIDE CW-RND-AUX BY 32768  GIVING CW-RND-AUX
                                       REMAINDER CW-RND-NUMERO.
