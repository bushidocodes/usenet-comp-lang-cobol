You didn't say if you were using DOS or UNIX.

I have tried a number of options under both operating systems. Under
UNIX, you can use C-ISAM as long as the programs have been linked to use
the 4.x C-ISAM libraries instead of the default ones. If you want to use
Informix to access the files, you will have a bit more work, as the keys
are defined differently, and you would have to make provision for the
Informix date and decimal field formats.

Under DOS, you could try the following:

1. Link to the MS routines supplied with COBOL. These are documented in
the manuals, and I believe that there is an example somewhere in the
demos directory.

2. Write your own. There is a structure description, again the manuals,
but this would be substantially more work than for option 1.

In the end, with everything going Windows, I tried the Parkway ODBC
driver, which seems to work OK. Micro Focus have also announced that they
will be supplying the Transoft ODBC drivers in the not too distant future
(there was a request for beta testers on this forum recently).'

You should find a file EXTFH.OBJ which is the file handler for
the ISAM files. This can be incorporated into MS C or
compatible languages to access the files. However, you need to
know how to call this. It is documented in the MF Toolset
documentation, but not in the compiler manuals, or in MS.
