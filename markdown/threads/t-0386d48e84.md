```
You need to know the format of the data. If you do not have the source code,
or at least some type of file layout you most likely will not be able to
pull it out directly.

You mentioned this file or files holds ALL of your data. If the database is
in one file changes are they used record layouts which vary in size. If this
is the case you will probably not be able to load it into EXCEL or ACCESS
directly. You could write a VB macro or EXE to extract it from within EXCEL
or ACCESS, but you still need to know the layout of the data.

Furthermore if you know the layout you are still going to need to know the
relation between the record layouts. Simply capturing the data may not do it
for you.

As far as guessing the layouts of the file you may be able to do this to a
certain extent. Most alphanumeric data is stored as straight ASCII. You may
be able to browse the file with a word processor and guess some of the
layouts and feed your results into VB/ACCESS/or EXCEL. Since you said the
files are in excess of 100Megs, you may not be able to load the entire thing
into ACCESS or EXCEL due to the record limitation in ACCESS/EXCEL and
possibly even memory limitations of the machine your system is running on.

If you have the source code to the COBOL programs you may be able to get the
layouts from the FD or WORKING-STORAGE section. If you know nothing about
COBOL this will be rather difficult. With a good COBOL manual you may be
able to figure it out.

Good Luck.
```
