There is no such thing as an 'empty' field. A field always has something in
it. Most applications define SPACES in a field as the absence of data. So:

MOVE 'TEXT' OF SCREEN-FIELD TO TEMP-FIELD.
IF TEMP-FIELD = SPACES
   do something about absence of data
   (i.e., complain to user, set default, etc.)
END-IF.

PowerCOBOL lacks the VB "Validate" method and there is no "Must-Enter"
property.

I have a "Must be entered" Text Box component that I drop on the form
instead of the standard text box, when I want a mandatory field (rare
occasions). It has all the standard text box control properties, methods,
and events, but the underlying Class has been extended to check that data
was entered... Simple OO solution. It won't allow the cursor to move away
from the field until data is entered. I'm thinking of extending it further
to produce a user dialog box if they try three times and still don't enter
data... just tell them it's a mandatory field.

I also have a "Numeric input" Text Box component that automatically
validates for numeric input and sets the number into around 8 different
formats (custom properties) for easy passing to COBOL and non-COBOL
applications. It produces a User message if the data is not numeric and
explains what is required, with examples of valid input (custom methods).

Of course, if I have a "Must Be Entered Numeric Input" field I'm a bit
stuffed at the moment...<G>

(only kidding... the numeric validation component "detaches" from the Text
Box if required and can be dropped directly on the form where it remains
invisible. You can invoke its methods and access its properties just as if
it was a GUI control. Pass it a string and invoke its "CheckNumeric" Method.
Don't need Validate...<G>)
