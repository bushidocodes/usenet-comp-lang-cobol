I may have briefly mentioned this before, but file status should only
be checked in the first byte for success and not for "00".  Valid
'success' codes already include "02", "04" and "05".  Perhaps:

     88 CTL-Status-OK  VALUE "00" THRU "09" "10" "97".

Note that status '14' also exists and this may be a result that takes
the AT END branch of a READ.
