I am an extremely biased source, but the answer is, yes, it works quite
nicely.

InstantSQL is Liant's solution for easy access to databases using Structured
Query Language (SQL) from RM/COBOL.  InstantSQL provides SQL connectivity
from RM/COBOL applications to any Open Database Connectivity (ODBC)
compliant data source, including Relativity data sources.  InstantSQL is
implemented at compile time with a set of copy files and at runtime with a
dynamic link library (DLL) or shared object (SO).

A fully functional Windows version (called the Jumpstart Kit) is available
on our web site, www.liant.com, plus plenty of documentation.  (There are
links in the 'What's New' box.)

The Linux Jumpstart Kit should be on our web site soon.  While ODBC is part
of the environment on Windows, the ODBC interface must be added to the
environment on Linux, Unix, etc.  The Linux Jumpstart Kit targets the iODBC
driver manager.  See www.iodbc.org for more details.

> Can we "export" cobol data to an sql database easily?

Yes, it is as simple as using SQL UPDATE and INSERT.  While I was testing
the Linux port, I created tables, inserted and updated data, and dropped
tables on Oracle and MySQL without any problem.
