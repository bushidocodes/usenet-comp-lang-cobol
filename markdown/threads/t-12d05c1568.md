No direct support exists in COBOL II for data spaces. Some routines exist
that allow data space access, but they are callable interfaces.

The big storage exploitation in VS COBOL II is actually for the XA model.
Single data items can be 16 MB, above the line. I have built apps using
as much as 100MB of "above the line" storage for native COBOL data
structures. This is not data space support, but just plain olde 31 bit
addressing.
