Pardon my jumping in here.
If you have to distribute a run time module or an ISAM module then
that/those modules require you or the user to pay the Vendor the
price of those modules.
I encountered this when I made the mistake of buying MS-Cobol
some years back for development of applications for clients.
Unfortunately, clients didn't want to pay an extra $200 for the
run time module on top of development costs. I don't recall what
MS wanted for the module you had to run as well if you had any
ISAM files, or the module for rebuilding corrupted ISAM files.
Needless to say, I wrote off MS-Cobol and switched to Realia Cobol
because it didn't require all these extra modules and charges.
An expensive lesson for me.
