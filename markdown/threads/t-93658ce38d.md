My solution to your problem would be:

01 AMT PIC 99.
...
ACCEPT AMT-IN FREE
ON INPUT ERROR DISPLAY "ERROR MESSAGE"
END-ACCEPT

This method will not automatically "take" the input when you have typed
the second digit. You must press return. There are other differences
in the behavior as well. In your version if only one digit was typed
and return pressed, whatever was present in the second position would
still be there after the ACCEPT. For instance if the AMT-IN field
contained "10" and you ACCEPTED a "2" and a return AMT-IN would then
contain "20". If nothing was entered, just a return it would still
contain the original value. With ACCEPT ... FREE the input is
interpretted before being moved, with a null response becoming a zero.

Hope this is of use to you.

I would suggest coding the following IF before the numeric test.
IF AMT-IN (2 : 1) = SPACE OR LOW-VALUE
MOVE AMT-IN (1 : 1) TO AMT-IN (2 : 1)
MOVE ZERO TO AMT-IN (1 : 1)
END-IF

This assumes that your ACCEPT mechanism pads with either SPACE or LOW-VALUE.
If your field become larger you will have to come up with a more complex
routine.
