tommy nilsen wrote (I snipped some):

> "Indicator variable required but not supplied"
> Does anyone know what this means, and how to avoid it ??

Likely you SELECT or FETCH a column INTO :hostvar ?!
Likely this column is not NOT NULL, so may at run-time be not filled
(nil value).
This type of columns shoud be INTO :hostvar:flagvar.
Note the *two* colons, *no* comma in between.
In the DATA DIVISION define:
    nn  HOSTVAR  PIC whatever.
...
    nn  FLAGVAR  PIC S9(4) BINARY.
        88  NOT-FILLED  VALUE -1.
These definitions may be in different group items or even be separate 01
levels.
In the PROCEDURE DIVISION, after the SELECT or FETCH, assumed that
SQLCODE = ZERO, you can test wehter the column was nil or not. IF NOT
NOT-FILLED does not read well, is it not?! I suggest you name the 88
level something like XXX-IS-EMPTY to avoid linguistical NOT NOTs. It is
good style to hqave a separate XXX-IND var for each host variable XXX.

As I recall, SQL defines -1 as "not filled" ("empry") and any other
value as "filled" ("not empty").

Good luck. I hope!, you tell? this helped.

The Frog
