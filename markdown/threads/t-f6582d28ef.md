After several interesting comments and suggestions I have re-coded the
fraction Example.  Well, recoded is too harsh - I modified it
slightly.  I made it capable of easy multiple use, and eliminated the
need to invoke the actual comutation.  Is this good or bad?  I'm on
both sides of the fence.

First, I did not put the computation in the factory object of the
class.  I could though.  If I move the whole thing into the factory I
can eliminate the invocation of the "new" method to create the object.
There is some appeal to this approach for this type of object - there
will never be a need for persistance.

This approach consists of overriding the  assumed "SET" method for the
Decimal-Num property and doing the computation to determine the
fractional equivalent from there - with no invocation of the
DecimalToFraction method from the mainline program.  In reality the
whole DecimalToFraction method could be inside the SET method -- but I
thought that was doing TOO much in one place.  But I could easily be
wrong.

Next I cleaned up the routine, based on some other suggestions I
received - removing Iterations.  The Precision property still must be
set before using the object.  I want to eliminate that also, but first
I need a simple routine to determine the number of significant digits
in the passed decimal number.  Any ideas on that front?

I changed Decimal-To-Fraction to DecimalToFraction because the dashes
were not acceptable to OS/390 in a method ID.  Not that this will work
under OS/390 - it won't.  The OS/390 implementation has not kept up
with the proposed standard --  who can fault IBM for not shooting at
this moving target?

Lastly, this may be a Fujitsu centric approach.  I am not sure that
the syntax for overriding the SET method is standard.

Do I like this approach?  Not really.  I think that what happens when
you move the decimal number into the object property is too obsure in
the mainline.  While the specific invocation of the calculation is
more code, I think it has merit.  Anyone else have an opinion?

Here's the mainline followed by the class.

 PROCEDURE DIVISION.
*
 IDENTIFICATION DIVISION.
 METHOD-ID.   SET PROPERTY decimal-num. 
 DATA DIVISION.   
 LINKAGE SECTION.
 01  decimal-in     Pic 9(9)V9(9).
 PROCEDURE DIVISION USING decimal-in.
     Move Decimal-In to Decimal-Num
     Set Odd-Iteration to True
     Move 0 to Low-Num
     Move 1 to Low-Den
               High-Num
               High-Den
     Initialize Working-Variables
     INVOKE SELF "DecimalToFraction"
     EXIT METHOD
     .
 END METHOD.
*
 IDENTIFICATION DIVISION.
 METHOD-ID.   DecimalToFraction.
 PROCEDURE DIVISION.
     compute Integer-part = function integer (decimal-num)
     subtract Integer-part from decimal-num
     perform try-fraction until break-out
     exit method
     .
 try-fraction.
     compute test-low = (low-den * decimal-num) - low-num
     compute test-high = high-num - (high-den * decimal-num)
     evaluate true
        when test-high <= precision * high-den
             set break-out to true
             move high-num to numerator
             move high-den to denominator
        when test-low <= precision * low-den
             move low-num to numerator
             move low-den to denominator
             set break-out to true
        when other
             perform setup-next-test
     end-evaluate 
     .
 setup-next-test.
     if even-iteration 
        perform even-adjustment
        set odd-iteration to true
     else
        perform odd-adjustment
        set even-iteration to true
     end-if
     .
 odd-adjustment.
     compute test-x = test-high / test-low
     compute int = function integer (test-x)
     compute num = ((int + 1) * low-num) + high-num
     compute int-num = function integer (num)
     compute den = ((int + 1) * low-den) + high-den
     compute int-den = function integer (den)       
     compute high-num = int-num - low-num
     compute high-den = den - low-den
     move num to low-num
     move den to low-den
     .
 even-adjustment.   
     compute test-x = test-low / test-high
     compute int = function integer (test-x)
     compute num = low-num + ((int + 1) * high-num)
     compute int-num = function integer (num)
     compute den = low-den + ((int + 1) * high-den)
     compute int-den = function integer (den)
     compute low-num = int-num - high-num
     compute low-den = int-den - high-den
     move num to high-num
     move den to high-den
     .
 END METHOD DecimalToFraction.
 END OBJECT.
 END CLASS Fraction.
