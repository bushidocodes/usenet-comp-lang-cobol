[Posted and e-mailed]

If you are using LEVEL II COBOL V2.6 for PCDOS or LEVEL II
COBOL V2.5 for MSDOS, it should be possible to CALL the OS
to get the date. Where you use ACCEPT FROM DATE, CALL
the following program. (This program was tested using
Professional Cobol V1.1, which is a LEVEL II implementation).

For example, CALL "OS-DATE" USING WDATE.

       IDENTIFICATION DIVISION.
       PROGRAM-ID. OS-DATE.
       DATA DIVISION.
       WORKING-STORAGE SECTION.
       01  DOS-INT  PIC X VALUE X"21".
       01  DOS-GET-DATE PIC 99 COMP VALUE 42.
       01  DOS-FLAGS PIC X VALUE X"FF".
       01  DOS-REGS.
           05  AX PIC S9(4) COMP.
           05  FILLER REDEFINES AX.
               10  AH PIC 99 COMP.
               10  AL PIC 99 COMP.
           05  BX PIC S9(4) COMP.
           05  FILLER REDEFINES BX.
               10  BH PIC 99 COMP.
               10  BL PIC 99 COMP.
           05  CX PIC S9(4) COMP.
           05  FILLER REDEFINES CX.
               10  CH PIC 99 COMP.
               10  CL PIC 99 COMP.
           05  DX PIC S9(4) COMP.
           05  FILLER REDEFINES DX.
               10  DH PIC 99 COMP.
               10  DL PIC 99 COMP.
       01 DATE-WORK PIC S9(4) COMP.
       LINKAGE SECTION.
       01  CURRENT-DATE.
           05 YY PIC 99.
           05 MM PIC 99.
           05 DD PIC 99.
       PROCEDURE DIVISION USING CURRENT-DATE.
           MOVE DOS-GET-DATE TO AH.
           CALL X"84" USING DOS-INT, DOS-FLAGS, AX, BX, CX, DX.
           MOVE DH TO MM.
           MOVE DL TO DD.
           DIVIDE CX BY 100 GIVING DATE-WORK REMAINDER YY.
           EXIT PROGRAM.
           STOP RUN.
-------------------------------
Rick Smith
e-mail: < ricksmith@aiservices.com >
