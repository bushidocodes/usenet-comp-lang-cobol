The message for this code is:

"Permanent error condition: An OPEN statement with the input, I-O, or
EXTEND phrase was attempted on a non-optional file that was not
present.".

Generally, this error occurs because the Assign To name in your Cobol
program does not have a corresponding DD statement in the JCL. You
might have also misspelled it in the JCL statement DD name.

Mike Dodas

I believe you basically you have 3 options:

(1) Write a dummy record to the file in a separate program before
opening it for I/O.

(Here's a secret: You can define the VSAM file, write a dummy record to
it, then delete the dummy record. You will then be able to open it for
I/O. The error code 35 happens when you open a file for Input or I/O
that has never been opened for output, not because it is 'empty'.

(2) Define the file in your SELECT statement as OPTIONAL.

When you OPEN it for I/O, you'll get a status code of 05 but the file
will be opened anyway ***but it may be opened for OUTPUT instead of I/O
- you should test this. Of course, if you're only adding records to the
file, who cares! ***.

(3) Attempt to OPEN if for I/O. If you get a 35, OPEN it for OUTPUT,
WRITE a dummy record, DELETE the dummy record, CLOSE it, then OPEN it
for I/O again.

This is a bit hokey but at least you won't have to write a separate
program as in (1).

>From Janossy "VS COBOL II":

OPEN with INPUT, I-O, or EXTEND specified as the SELECT/ASSIGN access
mode but the file does not exist. The DD statement for the file may
be missing from JCL. This error does not occur if the file is defined
as OPTIONAL in the SELECT/ASSIGN statement (in thst case READ actions
against the file receive "end of file" or "record not found"
indications).
