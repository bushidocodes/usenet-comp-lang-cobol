I try to use a method that works and that is comfortable to my client...  
but I'm easy that way.  The easiest example that comes to my mind
extemporaneously is (given a YYYYMMDD field) -

01  WS-MISC-STUFF.
...
    05  WS-YYYYMMDD.
        10  WS-YYYY PIC 9(4).
        10  WS-MM   PIC 99.
            88  VALID-MONTH VALUES 01 THRU 12.
        10  WS-DD   PIC 99.
            88  VALID-DAYS VALUES 01 THRU 31.
    05  WS-MO-TBL.
        10  FILLER PIC 99 VALUE 31.
        10  FILLER PIC 99 VALUE 28.
        10  FILLER PIC 99 VALUE 31.
...
    05  REDEF REDEFINES WS-MO-TBL.
        10 MAX-DAYS-IN-MO OCCURS 12 PIC 99.

...

    MOVE INPUT-DATE TO WS-YYYYMMDD.
    IF WS-YYYYMMDD NUMERIC AND VALID-MONTH AND VALID-DAYS
        PERFORM VALIDATE-DATE THRU VD-EX
    ELSE
        SET NONNUM-DATE TO TRUE
    END-IF.
...
VALIDATE-DATE.

    SET DATE-INVALID TO TRUE.
    MOVE ZEROES TO WS-JUNK02.
    DIVIDE WS-YYYY BY 4 GIVING WS-JUNK01 REMAINDER WS-JUNK02.
    IF WS-JUNK02 = 0
        MOVE ZEROES TO WS-JUNK03
        DIVIDE WS-YYYY BY 400 GIVING WS-JUNK01 REMAINDER WS-JUNK03.
    IF WS-JUNK02 = 0
        MOVE 29 TO MAX-DAYS-IN-MO(2).
    IF WS-JUNK03 = 0
        MOVE 28 TO MAX-DAYS-IN-MO(2).
    IF WS-DD > MAX-DAYS-IN-MO(WS-MM)
        PERFORM TOO-MANY-DAYS-RITUAL THRU TMDR-EX
    ELSE
        SET DATE-VALID TO TRUE.

VD-EX.
    EXIT.

The flaws in this code are obvious, of course... all caps, periods galore,
PERFORM THRU, it is limited to a particular calendar ('What happens when
we sell this code to an Israeli company?'), a paragraph-name that might
cause an adolescent to giggle ('He named it VD... haw haw haw!')...

... it also, more-or-less, works... and should be readily understood at 
3:am by a 2-year maintenance coder.

If you have the intrinsic functions INTEGER-OF-DATE and DATE-OF-INTEGER from
the '89 Intrinsic Function Amendment, and the original date THE-DATE  you
want to test is in YYYYMMDD format, it may be possible to verify the
validity of THE-DATE by testing to ensure that the value contained in
THE-DATE is the same as the result of FUNCTION DATE-OF-INTEGER (FUNCTION
INTEGER-OF-DATE (THE-DATE)).  In the MCP/AS  implementation of '85 COBOL,
INTEGER-OF-DATE returns a zero for any out-of-range argument, in which case
DATE-OF-INTEGER returns 16001231.  I would expect something analogous from
anyone whose function doesn't fall over (though the returned value from
INTEGER-OF-DATE may not be zero, it can't represent a value corresponding to
something that isn't a date to begin with, certainly not the [invalid] date
you started with).

    -Chuck Stevens

Howard (who probably knows this already) and others (who may not),

  The following solution is NOT a generic solution. (As far as I know, other
than "table of date checks" there is no generic solution before the 2002
Standard TEST-functions.) It will, however, work with either Micro Focus
(only products where LE callable services are available) or IBM environments
(I think even VA COBOL - but I am not certain about OS/400 or COBOL Set for
AIX).

There is, however,  a "question" that I would think arises with this
"solution", i.e.

   IF you are going to use LE callable services to test if a date is valid,
why not use them for everything???

Anyway, here is a program that will test (in an IBM or Micro Focus
environment) if a date is a valid date in YYYYMMDD format (including
leap-days and other oddities).  If you want to test for a YYYYDDD format, I
think you can figure out the required modification.

(see:

http://publibz.boulder.ibm.com/cgi-bin/bookmgr_OS390/BOOKS/CEEA3130/3.5.24.3
for source of this program and check your SCEESAMP library for machine
readable version.  I have removed comments in program below)

            IDENTIFICATION DIVISION.
            PROGRAM-ID. CBLDAYS.
            DATA DIVISION.
            WORKING-STORAGE SECTION.
            01  CHRDATE.
                02  Vstring-length      PIC S9(4) BINARY Value 8.
                02  Vstring-text.
                    03  Vstring-char    PIC X
                                OCCURS 1 TO 8 TIMES
                                DEPENDING ON Vstring-length
                                    of CHRDATE.
            01  PICSTR.
                02  Vstring-length      PIC S9(4) BINARY Value 8.
                02  Vstring-text.
                    03  Vstring-char    PIC X
                                OCCURS 1 TO 8 TIMES
                                DEPENDING ON Vstring-length
                                    of PICSTR.
            01  LILIAN                  PIC S9(9) BINARY.
            01  FC.
                02  Condition-Token-Value.
                COPY  CEEIGZCT.
                    03  Case-1-Condition-ID.
                        04  Severity    PIC S9(4) BINARY.
                        04  Msg-No      PIC S9(4) BINARY.
                    03  Case-2-Condition-ID
                              REDEFINES Case-1-Condition-ID.
                        04  Class-Code  PIC S9(4) BINARY.
                        04  Cause-Code  PIC S9(4) BINARY.
                    03  Case-Sev-Ctl    PIC X.
                    03  Facility-ID     PIC XXX.
                02  I-S-Info            PIC S9(9) BINARY.
            PROCEDURE DIVISION.
            PARA-CBLDAYS.
     *   fill in date to test
                MOVE "20030532"
                    TO Vstring-text of CHRDATE.
           *   fill in format of date to test
                MOVE "YYYYMMDD"
                        TO Vstring-text of PICSTR.
                CALL "CEEDAYS" USING CHRDATE, PICSTR,
                                     LILIAN, FC.
                IF  CEE000 of FC  THEN
                    DISPLAY Vstring-text of CHRDATE
                        " is A VALID date with Lilian Value: " LILIAN
                ELSE
                    DISPLAY "Input Date is not valid, Check return code for
reason"
                        Msg-No of FC UPON CONSOLE
                    STOP RUN
                END-IF
                GOBACK.
