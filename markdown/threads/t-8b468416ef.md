```
Can anyone help with this little problem ? :-

The application I work with (MVS) produces huge reports which are
subsequently fiched.

There are two problems I face with these reports. First, the fiche
production takes too long and second, when errors occur (whilst
ficheing) it is costly to restart the report.

The answer (I know) is to split the report into more manageable
sections, but how ?

I have considered a number of options, the first is to CLOSE/OPEN the
file at convienient points. Unfortunately when using FREE=CLOSE in the
JCL on a SYSOUT DD card. The system dataset created by the operating
system disappears and can't be opened again.

The second alternative is to write to seperate DD cards when
appropriate. This is very simple but could become impracticle by having
to code many more DD cards - in a program that already has quite a few.

The third method I have considered is to write the report to cassette
and use a utility to produce the report.

Is there another method lurking behind the scenes or am I to plunge
into the realms of IBM assembly coding. I've never tried to use MVS
COBOL 2 to allocate dynamic files.

TIA for any help you can give.

Benny
```
