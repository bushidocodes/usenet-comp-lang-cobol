```
Try this...

# days up to now = ((year - 1) * 365) +
((year - 1) / 4) -
((year - 1) / 100)
((year - 1) / 400) +
(# of days elapsed in current year) / 7)

Use the remainder to determine the day of week as follows:
0 = Sunday, 1 = Tuesday, ... 6 = Saturday.

I'm not sure this is exactly right, but it is a definite start.
```

```
If you have a routine or a subprogram that can calculate the number
of elapsed days between any two given dates, you can make use of that:

Find out the day of week for a past date, say 1-Jan-1990. Use the
subprogram to calculate the number of days since 1-Jan-1990 for your
date, divide the count by 7, and from the remainder figure out the
day of the week for your date.

The alogorithm that I used in an Assembler program we have is the
"Zeller's Congruence" - got it off an old (ancient?) book:
"Problems for computer solutions" by Gruenberger & Jaffray;
Wiley & Sons, 1965).
```

```
On the HP3000 system I work on there is a COBOL function available that
returns then number of days offset from Dec 31, 1600 given an eigth
digit date. Dividing this number by 7, retaining the remainder which
the only part of use in this situation. The remainder is a number
between 0 and 6 and corresponds to Sunday thru Saturday. It looks
something like this:

$CONTROL POST85
...
DATA DIVISION.
WORKING-STORAGE SECTION.
01 DATE-TEST PIC S9(8) COMP.
01 DAYS-VALUE PIC S9(9) COMP.
01 DAY-OF-WEEK PIC S9(4) COMP.
88 ITS-A-SUNDAY VALUE 0.
88 ITS-A-MONDAY VALUE 1.
...
88 ITS-A-SATURDAY VALUE 6.

...

PROCEDURE DIVISION.
...

MOVE FUNCTION INTEGER-OF-DATE TO DAYS-VALUE
DIVIDE DAYS-VALUE BY 7 REMAINDER DAY-OF-WEEK.
```
