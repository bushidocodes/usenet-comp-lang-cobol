David,

As documented in Appendix A of the User's Guide, the XX (where XX not = 01
or 02) means that file has an inconsistent structure.  The reason that you
cannot find further information these codes is that they detail exactly
which internal check failed  (for example:  I think I just read a data
block.  Does the block code = data block? No...98,XX!!).  But even if you
knew which internal check failed, it is most likely not a diagnosis of the
actual failure, since the failure is in the history of the file, the present
inconsistent state being only a symptom of the failure.

The cure does not depend upon which internal check failed, but it is
sometimes useful for our tech support analysts to know which internal
failure occurred.  Furthermore (though this has not happened in several
years) we might see a trend to a particular type of internal failure, which
might suggest a problem in our algorithms.  And, of course, if the internal
failure can be reproduced, we definitely want to know about it!
