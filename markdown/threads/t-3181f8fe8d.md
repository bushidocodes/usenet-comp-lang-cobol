```
It just occurred to me that some people will have no idea what a Mashup is.

Here is a short, but very accurate video.

http://www.youtube.com/watch?v=U9sENSA_sjI

(The presenter says that around 2.5 Mashups a day were being produced at the 
time the video was made and he predicted 10 a day by 2010. The most credible 
figures I can find, suggest that it is currently around 6 a day, but it is 
increasing...)


The fact is that not only technology, but also the way in which people use 
technology is changing rapidly.

45 years ago I used to wish that people could get access to computers 
without having to go through the barrier of analysts and programmers. If 
only everyone could just get the support and computing power they needed, 
because the interface was so simple, imagine the apps they could produce...

I spent hours writing natural language interfaces for DOS based PCs. (One of 
them was called "IGOR" and it was sold in Sydney Australia, on DOS based 
systems. IGOR helped non-technical people run their applications and 
provided help and guidance through conversation. It was very well received 
ad over 400 machines with it on were sold in a single weekend when it first 
became available. When Windows arrived the point and click interface 
rendered IGOR largely redundant although I continued to receive mail from 
people who enjoyed IGOR for a few years afterwards.)

It was this experience which first stimulated my interest in getting 
computers to write their own code and to generate from templates. (I first 
did this on a mainframe site in Spain, and it enabled people to get their 
own reports without needing to request them from Programming.)

Films like "2001 - a Space Odyssey" with the fully conversational HAL (IBM - 
1) 9000 just made me drool for such a system. (OK, I know HAL was "insane", 
driven to madness by the enormity of the Mission, but us programmers know 
that a computer could never REALLY become the effect of its own data, right? 
:-))

Now, with Mashups, it looks as if we are getting close to a truly 
non-specialist interface for machines and the fact that High School students 
are pursuing it enthusiastically and producing some very impressive results 
gives me a warm feeling.

I'm glad I lived long enough to see this.

Pete.
-- 
"I used to write COBOL...now I can do anything."
```

```
> Now, with Mashups, it looks as if we are getting close to a truly
```

```
I am not sure why you think that this is 'truly non-specialist'.
Mashups still require programming, web pages and Javascript, possibly
server based stuff too. It is done using APIs, certainly they are web-
based APIs, but the code that uses those APIs doesn't write itself.

What may finally happen with these APIs that Mashups use is that we
will get what Netscape was working towards in the early 90s and that
is a browser based system where the internet is the computer. That
frightened Microsoft, which had ignored the internet*, into ripping
off Spyglass** to get IE written and then giving it away for free and
paying OEMs to remove Netscape. Netscape was driven out of business
and so the desktop computer survived and held back the industry for
another 20 years.

Now, eventually, we are moving towards Netscape's vision with cloud
computing. The APIs which mashups use are mechanisms to get data and
process this. It is no different in concept from a program that uses,
say, a database API and a charting library API (neither of which are
part of the OS) to produce a bar chart. The only difference is that
the 'charting library' and the 'database' are probably accessed from
Javascript in a web page rather than, say, BASIC. Javascript was
developed by Netscape for exactly this purpose.

I did some work with Google maps a few years ago (2008), putting GPS
readings onto a map. Trucks were sending out GPS positions over SMS
that were being stored in a database so we should have been able to
draw a map of the route they had taken. Unfortunately the positions
were only every 10 minutes so it wasn't useful enough to implement.

Demo sample: http://Azonic.co.nz/maps/map.html



* The first edition of 'The Road Ahead' did not mention the internet
at all, it was later added. With Windows 95 there was the original MSN
which was walled off from the internet and was for W95 users only. It
failed.

** Spyglass wrote IE on the basis of a royalty payment (of $5.00 I
think ?) for every copy sold. MS claimed that it never _sold_ any
because it gave them all away, so it never paid anything to Spyglass
who then went broke. In my view if it was sold as part of Windows then
that was a sale.
```
