Arn Trembley COBOL II is just IBM's name for their ANSI-85 compliant COBOL
Aw> compiler. The COBOL II name was used just to distinguish it from
Aw> their earlier compiler, which was compatible with the 1968 and 1974
Aw> standard. For the MVS operating system the old name was OS/VS
Aw> COBOL for the old one, and VS COBOL II for the ANSI-85 compatible
Aw> compiler. It has now been superseded by COBOL for MVS and VM
Aw> (formerly called COBOL/370). COBOL for for MVS and VM is identical
Aw> to COBOL II, but includes the new intrinsic functions and uses
Aw> LE/370 (Language Environment 370), which is a common set of runtime
Aw> libraries for COBOL, PL/I, C, and assembler.

Aw> I got a nice COBOL compiler for PC DOS by buying a textbook titled
Aw> "COBOL: From Micro to Mainframe", by Robert T. Grauer and Carol
Aw> Vazquez Villar, and published by Prentice Hall in 1991, ISBN
Aw> 0-13-140179-3. The price was about $60. I used to have a 1-800
Aw> number to order it direct, but I can't find it right now. The
Aw> compiler, with examples, came on two 5-1/4" floppies, non-copy
Aw> protected. You need to supply your own linkage editor -- I used
Aw> one from Microsoft QuickBasic 4.5 and it worked fine.

Aw> The compiler has some limitations. No program can have more than
Aw> 500 lines of code, excluding comment lines. No file can be bigger
Aw> than 64K bytes. You can get around the first limitation by linking
Aw> multiple programs together, and you can get around the second
Aw> limitation by using the doscall library. Unfortunately, the only
Aw> way to get documentation on the doscall library is to get manuals
Aw> for the full-blown Realia COBOL compiler (that compiler costs about
Aw> $800). The student compiler that comes with the textbook is a
Aw> smaller version of the full Realia compiler and it supports both
Aw> ANSI-74 and ANSI-85 COBOL dialects.

Aw> You can also get MicroFocus Personal COBOL, which is pretty
Aw> inexpensive. I think you can find it for less than $100. I don't
Aw> have any personal experience with it, but I hear it's a good
Aw> product. If you look at other posts in comp.lang.cobol you will
Aw> see their address for their web page, or just search for
Aw> MicroFocus.

Thank you very much, this is very informative and
helpful.
