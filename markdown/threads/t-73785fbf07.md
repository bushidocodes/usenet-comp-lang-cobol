```
PRIMA is about to release a new software protection product which will 
guarantee Developers no piracy of their packages. (Pirate (non-validated) 
copies of a package simply won't run.)

It is called "Remote Application Validation" (RAV)

RAV doesn't use serial numbers or product activation codes, so there is no 
chance of people obtaining "cracked" copies of these from hacker web sites.

RAV is implemented by adding a single COM component to your application, and 
a web service to your web server. (If you don't have a web server (typically 
it would be running your company's web pages) we can supply one for you 
either in the cloud or at an ISP location, for around $US300 per year. Both 
of these components are tailored by PRIMA specifically for you as a 
Developer. You can protect as many packages as you like from the same web 
server and you can run as many different packages on client sites as you 
like, once you are set up. RAV has no impact on your Application 
functionality (it has no visible user interface and dos not interact wth 
Users of your package); it simply tells you what level of access is 
appropriate and what run limitations are in place (if any), and whether any 
of these rules have been exceeded. What you do after that is over to you.

It allows  different levels of access to the same software and includes run 
limitation by number of executions or a time period (or both) for DEMO 
versions of your package.

RAV recognises whether your package is being leased or is owned outright. If 
it is being leased, you can stop execution of it at any time without access 
to the client site.

It was written in response to a request from a client using TrialPro, which 
is now defunct. Anyone who has TrialPro will know what problems this 
software addresses, however it does it in a completely different way from 
TrialPro and is much more flexible and powerful.

The RAV system has currently been tried with packages written in C# and VB 
and works very well, but I'd like to try it with a completely "strange" (not 
my own) package, written, ideally, in COBOL.

If anyone here would be prepared to act as a Beta tester (and thereby get a 
free copy of the final product) please contact me. Here is the ideal profile 
I'm looking for, but if you don't quite match, but are still interested, 
please contact me anyway...

1. You have a software package which you have made a significant investment 
in.
2.  People can buy the package or lease it.
3.  You generally deploy it using a site licence (All machines at the client 
site can run it). If your package is deployed to a server at the client 
site,  RAV can optionally allow a "staggered" site licence (LOW = say, 3 
machines can use the package concurrently, MEDIUM = 10 machines can use the 
package concurrently, HIGH = 20 machines can use the package concurrently, 
ENTERPRISE = unlimited concurrent users (limited only by your LAN 
hardware/software. The numbers shown are suggestions only.).
4. RAV does not currently provide protection to an individual client machine 
(it is to a client site). This saves a great deal of effort registering 
client machines, however, if protection down to this level was absolutely 
required, it can be fairly easily extended to do so, and I'd be interested 
to talk to anyone requiring it.

RAV is targeted to small independent software developers and software 
houses, for whom piracy can be the difference between solvency and 
bankruptcy. It will be priced so people can afford to have it. I'll be 
integrating it into PRIMA's existing product range in the New Year.

Pete.
```
