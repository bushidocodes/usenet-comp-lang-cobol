Miguel and others looking for code,
here is code for the classical Towers of Hanoi problem:

Towers of Hanoi
In a monastery in Hanoi there are three diamond rods. God placed 64 gold
disks of different sizes - each with a central hole- on rod number 1 (the
魹ｽsource魹ｽ) such that the largest disk is at the bottom of the stack, the
disks then get smaller and smaller, and the last and smallest disk is on
top. The monks mission is to move all the disks onto rod number 2 (the
destination). Only one disk may be moved at a time and never may a larger
disk be placed on a smaller one. If needed, disks may temporarily be placed
on the third rod (the spare) again in such a manner that no larger disk
rests on a smaller one.
This classical problem can be solved as follows: first move all disks except
the last one from the source rod to the spare rod, then move the last and
largest disk from the source rod to its final position on the destination
rod, and finally move all disks from the spare rod to the destination rod.
Moving N-1 disks is then performed in the same manner, recursively, until
all disks have been moved.

Here is pseudo code for this:

BEGIN INTEGER M;

PROCEDURE MOVE (N, SOURCE, DEST, SPARE);
INTEGER N; STRING SOURCE, DEST, SPARE;
BEGIN
IF N > 1 THEN MOVE (N-1, SOURCE, SPARE, DEST);
WRITE (OUT, "MOVE DISK ", N,
" FROM ", SOURCE,
" TO " , DEST);
IF N > 1 THEN MOVE (N-1, SPARE, DEST, SOURCE)
END;

WRITE (OUT, "NBR OF DISKS = "); READ (IN, M);
IF M > 0 THEN MOVE (M, "SOURCE", "DEST", "SPARE")
END

explicitly managed with the variable WHAT-TO-DO and parameters (which are in
a sense local to each invocation of the procedure) are stacked along with
the program counter. Stacking the first set of parameters simulates the call
of the procedure in the main program.
