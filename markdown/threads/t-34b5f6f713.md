Hello all,

I'm having a problem with a multi-threaded app I'm writing...

1)  the structure is that a 'main' thread that's compiled as reentrant(1)
starting 6 other threads (all serial).   these 6
children thread CBL_THREAD_SUSPEND themselves
when they have no work to do and are CBL_THREAD_
RESUMED (by 1 of the other threads) when there's work
to be done.

2) the main thread (gulp) has 200Meg of w/s it's dealing with.
The stated limit is '256Meg or less', however I do pass this
data (which is defined essentially in 4 01's) around to the
other threads.

3) All threads are started by the main thread via 1 CBL_THREAD
CREATE which uses a variable to point to the proper 'ENTRY'
in the main thread (all located at the end of the main thread).

4) The code within each of the entry points is just a CALL USING
with my 4 01's (all BY REFERENCE).

The >problem< is when I compile to EXE with DLL's.   In
contrast to running (i.e., RUNM'ing) the program which works
GREAT, the EXE 'loses' place while executing...   PERFORM
THRU's (which all are performing SECTION'ed code) stops
'working' and the main thread just falls, falls falls, straight to
the end...   Which, of course is the polite way to describe
a thread (and NT as well !) dying a horrible death !

Any clues out there ?


Thanks A BUNCH !!!!

Ethan Schultz
