```
It is a good idea to let the user know that the program is actually
doing something useful. Assuming your program is doing
something, during the processing insert a counter that triggers
the message. Such as:

01  TICKER            PIC 99 VALUE 0.
01  DOT-LINE.
       02  FILLER        PIC X(10) VALUE 'Working'.
        02 DOT-LINE-DOTS       PIC X(70).
01  DOT-COUNTER    PIC 9(2) VALUE 0.



(code here)
ADD 1 TO TICKER
IF TICKER = 0
    PERFORM DOT-MAKER.
(continue with processing code)


DOT-MAKER.
    ADD 1 TO DOT-COUNTER.
    IF DOT-COUNTER > 70
        MOVE 'Working' TO DOT-LINE
        MOVE 1 TO DOT-COUNTER.
    MOVE '.' TO DOT-LINE-DOTS(DOT-COUNTER:1).
    DISPLAY DOT-LINE.

Each 100 iterations of the code gets another dot.
You can adjust TICKER to be, say, 9(3) and get a dot
every 1000 iterations.

Or, you could use:
ON 1 AND EVERY 100
    PERFORM DOT-MAKER.

Or, if you really wanted a dot every "x" time interval,
you could query the clock - but the overhead would
be ferocious.
```
