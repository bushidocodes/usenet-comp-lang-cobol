```
There seems to be a problem with using Oracle Pro*COBOL with the Micro
Focus "Free" source format. The code generated by Pro*COBOL is not
'valid' free format COBOL.  Specifically, Pro*COBOL generates a number
of long 'strings' in working storage - these strings are generated
using line continuation characters that are only valid for fixed
format source.  Therefore the Micro Focus COBOL compiler barfs.

There are a number of ways around this - essentially involving an
intermediate step (between the Pro*COBOL pre-compile and the Micro
Focus compile) that cleans up this working storage so that it will
compile.

This is fine for a makefile type build approach.  However we are
trying to move to NetExpress 4 which is much more interactive.  My
goal is to have single step compilation in NetExpress ...

My questions are:
- Does anybody else have a solution for this ?
- Is there a way to tell Pro*COBOL to generate free format compliant
code ?
- Has anybode else looked into building a Micro Focus integrated
pre-compiler to fix this (ie chain it after Pro*COBOL and have it fix
the generated statements for the COBOL compiler) ?

Thanks
Matthew James
```
