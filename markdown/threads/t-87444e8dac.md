Windows NT, 2000 etc can execute OS/2 1.x programs in text mode (I don;t
think that XP still does though).  When you start MS Cobol 4.5 or MF
Cobol 3.x it tries to run as an OS/2 program and looks for those
components but fails because the appropriate OS/2 style environment
items are not set.

Use FORCEDOS to ensure that Cobol compiler and programs try to run as
DOS and thus find the run-time correctly.

Isn't this in the FAQ ?
