```
Been meaning to comment on GEM for some time. GEM is the Fujitsu version of
SourceSafe, a source code development and management system.

We keep our source on a server. To edit/compile from there was a time-waster
inasmuch as everything had to be hauled across the network, plus FJ uses
scratch files on the same drive as the source, so God-knows how many
megabytes were moving back and forth.

So, we experimented with GEM with the working library on the local drive.
That is, modules are checked-out of the main library (on the network) and
manipulated locally. Once perfected, the module is checked back in. While
checked out, no one else can have access to the module. Whole thing works
like a charm. Compiles on our larger programs dropped from three minutes to
50 seconds. Couldn't be more pleased.

Well, yes I could. PowerCOBOL projects are treated, by GEM, as binary files.
That means that there is no source-level history and roll-back available.
You'd think FJ could do that, since GEM and PowerCOBOL are both their
products, but, hey, what's art without suffering?
```
