Probably the biggest thing to look at on an AS/400 are externally-
described files.  On the 400, everything, including both data and
screens are objects.  Files are described with a language called Data
Description Specifications (DDS) and compiled.  Metadata from the
compiled object can then be copied into all programs using the object.

You will see statements like this in the file section of AS/400 COBOL.

01 REPORTING-PERIOD-REC.
   COPY DDR-ALL-FORMATS OF RPPF.

This copies the metadata for a file named RPPF into your program
replacing all underscores in field names "_" with hyphens "-".

There are a variety of formats to the COPY DDx statements, which you
will find documented in the AS/400 Cobol manuals.

You will also find some differences in how AS/400 handles screens,
because they are objects described by metadata.  Additionally, AS/400
screens can contain "subfiles", which are constructs used to describe
repeating groups of data -- very similar to grids in visual programming
languages.  It will likely take you some time to learn how to use these.

The IBM AS/400 Cobol manuals do a good job of discussing file and
screen handling and give good examples to work from.

Good luck.

I was using a book called Programming in COBOL/400, by Cooper, Stern &
Stern. Publisher Wiley.  This book does not have much about DB2 in it
though.  You can link files together logically on the AS400 using
logical files and use files as a database.  You can also run querries
to make reports.  This book has some example programs that are pretty
good, FULL LENGTH PROGRAMS, not just bits and pieces.  It is written
with the beginner in mind.  There may be other books exclusively for
DB2 or the new universal database which works on both the AS400 and NT
and other platforms.

I had a book at work by an author by the last name of LIN who taught at
Washington University, in St Louis, MO, that had a DB2 project in it.
I think it was published by Prentice Hall.  Surely there are some
Better DB2 books out there.  Your AS400 may have RPG also.  RPG can be
useful for control break programs if you can get the hang of it.

Just look at Amazon.com  they have lots of programming books, Or just
look at the examples already in the as400.  Must be a few things you
can plagerise, I mean borrow.  Why reinvent the wheel?
