You just can't call a VB DLL directly - from ANYTHING besides VB. 
There are some tools (I don't have the reference but they have been
mentioned here before) that will convert a VB DLL such that it CAN be
called.  When you do this it's called like a C program.

If the VB code is an ActiveX control the world is more open - there
are various ways to use ActiveX controls from COBOL programs including
using Flexus Sp2 or Dialog System, or PowerCOBOL.
