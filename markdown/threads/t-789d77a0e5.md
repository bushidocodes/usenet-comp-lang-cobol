I do not believe it is a problem with LE levels or versions, but with compiler
levels or versions and how you bind your programs.

OS/390 recognizes these ways for one program to call another:
  * static call (programs bound into a single load module or program object)
  * dynamic serial (programs separately bound; use system LOAD services to
connect)
  * dynamic parallel (programs separetly bound; use system services to create
       sub-tasks or threads that operate independently; not supported in COBOL)
  * DLL (all programs bound separately; use system DLL services to connect)

If you attempt to call a program that is bound as a DLL, you must use the first
or last approach to connect the programs. What has historically been called a
dynamic call won't work; you need to call using DLL linkages.

If you compile with the DLL option, it says "make all dynamic calls in this
program use DLL linkages"; it does not say "make this program a DLL".

COBOL for MVS & VM (and earlier) do not support DLL linkages. COBOL for OS/390
& VM (both currently available releases) do support DLL linkages. The only
reasons I could see that your program worked before are: a) you did not have
your C program compiled or bound as a DLL before; or, b) you used static calls
earlier.

OS/390. Here is a link to the course description; from there, you can link to
the detailed outline.
