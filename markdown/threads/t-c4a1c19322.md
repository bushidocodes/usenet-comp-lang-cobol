```
Hi, folks.
Take a look at the following small program that reproduces an error
with computational-arithmetics using Micro Focus COBOL for AIX on 
IBM RS/6000 (even worse than the Pentium-FDIV-error, this one occurs
every 28 seconds :-)):

       IDENTIFICATION DIVISION.
       PROGRAM-ID. jtest.
       ENVIRONMENT DIVISION.
       CONFIGURATION SECTION.
       OBJECT-COMPUTER. ALTOS.
       SPECIAL-NAMES.
       INPUT-OUTPUT SECTION.

       WORKING-STORAGE SECTION.
       01 J1        PIC 9(3)V9(2) comp.
       01 J3        PIC 9(3).9(2).

       PROCEDURE DIVISION.
       move 15 to J1.
       add 100 to J1.
       move J1 to J3.
       display J3 at 0101.

       stop run.

Instead of the expected value of 115, J3 contains the value 16!!!

Using the compiler-directive "IBMCOMP" avoids the calculation-error,
but also increases the size of the comp-data-elements.

So, what can I do? Does anybody else have this or similar problems?
Even Micro Focus, Newbury, UK could not answer that question.

Please, post or mail to <cal@zls.gtn.com>.
```
