Real programmers hack the help system with a text editor in HTML.

Actually HTML is easy enough that (after a while) all that is
required is text editing and the result can be accessed on just
about anything. DOSLynx can work well, though it is rather
limiting, it is easy enough to provide alternate pages when
better facilities are required. IE or Netscape for Windows,
Mosaic for Unix.

These days I incorporate the HTML as comments in the source code
of the programs and then extract it with a simple processing
programming whenever the program has been updated. I use tags on
comments:

IDENTIFICATION DIVISION.
PROGRAM-ID. Demo.
*
* Invoice File Update
*
* Applies the current ....
* blah balh
* screen layout
* operator options
* See also
*
*Another program
*....
*
*

.........
*
* Further desription of inner workings
*

The extract looks for the *, * and * to
know which bits to write to the .HTM file and generates the
required headers and trailers filling in with what lies
between.

I find that I am far more likely to update the help when it
lies alongside the source.

I generate a menu.htm from the operator menu souce so that it
links to each of the programs and then have general descriptive
text that also links back to the menu and/or programs.

I dislike those sophisticated help systems that try to make it
'easy', they get in the way.
