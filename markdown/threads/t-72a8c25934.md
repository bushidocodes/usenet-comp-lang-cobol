Here is what the definition is of an "04"

"A READ statement was successfully executed, but the length of the record
being processed did not conform to the fixed file attributes for that file."

I know that I have heard of 04's for QSAM - and I think it was when the last
record wasn't complete for some reason. I don't think there should be any
difference between "VS COBOL II" and "COBOL/370" and "COBOL for
this-and-that" in regards to when this file status is issued (but OS/VS
COBOL was very different). If you are getting different results with the
new compilers, you might want to check and see if your programs were
compiled with the same CMPR2/NOCMPR2 option. This does impact several of
the file status codes.

It is important to remember that COBOL does treat any file status code that
starts with a zero as "successful" - even though your application may well
NOT want to do so.
