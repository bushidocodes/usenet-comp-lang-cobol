```
Have many people here used Acucobol's new thin client?

We have been testing the released version of the thin client and it
does seem to be reasonably quick as long as you have a beefed up
server to run multiple users.

During our testing we have experienced numerous "Illegal operation"
messages produced from Windows that kills the thin client on the PC.
These errors are coming up every now and then so we cannot reliably
reproduce the problem to give any evidence to Acucobol about these
errors. The trace file from Aculaunch and the runtime do not show any
information when the sessions are killed so we cannot determine if the
problem is with our application or the thin client itself. We have
been testing this running on Windows 95/98 PC's with a SCO Unix 5.05
server and also Windows PC's with a Windows NT 4 server. Both these
configurations are producing these errors.

We did not notice these errors during the beta testing cycle as their
was usually only 1 person using it, but since the released version has
come out everyone in the office (5-6 sessions at any one time) are
using it and all of us are getting these errors as many as 10 times a
day.

Once the "Illegal operation" message appears I can do an acurcl -info
and my session appears in the connectioned sessions list so it appears
as the thin client is still in communication with Aculaunch, but as
soon as I hit OK to accept the error message which kills the thin
client on the PC the process is gone from the acurcl -info display. We
initially thought they were occurring because we were running out of
memory on the server side, we have since tripled the memory on the
Unix server but these errors still persist.

I would appreciate any feedback from anyone using the thin client in a
multi user environment and any issues they have with it, especially
what do you think about the speed over a 56K modem and regular local
area networks.

Regards,

Andrew Hodges
```

```
Hi Ken:

Our Thin Client has been through over 5 years of testing and
production use in the field.  In addition, our Thin Client is 100%
COBOL compiler independent, so you don't get locked into any one COBOL
compiler.

Our thin client product offers:

- Win32 interface or Web Browser based client.
- Automated version control and automated client component
installation.
- download on demand feature which downloads client side components
only as they are needed.
- data compression feature.
- panel (GUI Screen) definitions contained in separate external files
or within the actual source code.
- launching of remote executables on client or server side.
- copy file feature which allows files to be transferred from client
to server or from server to client.
- http tunneling.

In addition, our thin client supports our FormPrint product which also
allows remote printing from the host.

Our server support includes:

- Win32
- UNIX and UNIX derivatives (including OS/390, Linux, etc.)
- VMS and Open VMS

You would need to convert your screen handling code over to the 100%
compiler independent COBOL sp2 GUI, so there would certainly be some
work involved in switching over, but we believe that the benefits of
```
