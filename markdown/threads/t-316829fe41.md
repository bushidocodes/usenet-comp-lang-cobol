Can you increase the variable in an ODO that is in linkage as you need
to. I.E. you are reading a cursor and as you determine that the
values are the ones that you want you want to put them in linkage into
an array. As you fetch more rows, you want to put them into linkage
as well. You will know the upper limit, but not how many until you
finnish the cursor. Will this work, or will I run into problems at
run time.

```
Sounds like SQL syntaxdriving the question.

Of course data that is passed winds up in linkage section in the receiving
program, but must reside somewhere real in the caller.....

Where does the data originate? No adjustable arrays (at run time) in
working storage, beyond preset maximums that I can think of. Since it is
late at night there may be some scheme to do this, but none that is
obvious now.

Passing data with ODO to a subroutine, with the receving field in Linkage
Section should work, but does not get away from the intial problem of
where to put the unknown amount of data.

With implementations such as VS COBOL II / COBOL/370 one could allocate a
very large (up to 16 MB) structure in the caller and hope that it could
hold all the data, then use ODO to tell the receiver that not all is used.
Hopefully the data structure would not require floating position data
beyond the ODO - horrible stuff that everyone's payroll system seems to
have used in the past.
```
