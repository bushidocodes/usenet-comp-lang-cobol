```
You could try some kind of Soundex algorithm. Of course you would have to
decide how to handle cases like "apt." vs "apartment". Convert the fields
to their soundex equivalent and see if they match.

You could sort the characters in the fields and then compare how many
characters in one field occur in the other. By experimenting you could
determine a percentage such that if that percentage of characters match
then you have a possible match.

Neither of these give you an exact solution, they give more of a "fuzzy
match".

```
Copy both flat files to your desktop (unless you have direct access to them via
SAMBA or such). 

Write duplicate schemas (ascii text files) describing them to the Microsoft ODBC
driver for flat files. 

Using msquery (came with Excel), ACCESS or similar, write SQL to compare them
with something like: WHERE proposal.name not like customer.name(1:5) & '%'. 
Both flat files must be in the same directory, which is what you CONNECT to.
Table names are the schema names. 

There is a learning curve. Once you know how, you can do a comparison like this
in 5-10 minutes. .
