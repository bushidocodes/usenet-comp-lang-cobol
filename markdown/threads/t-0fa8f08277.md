```
The "Standard" COBOL method is to use:

Move Function Current-Date to Date-Field.

01  Date-Field.
     03  Date-CC Pic 99.
     03  Date-YY Pic 99.
     03  Date-MM Pic 99.
     03  Date-DD Pic 99.
     03  Time-HH Pic 99.
     03  Time-MM Pic 99.
     03  Time-SS Pic 99.
     03  Time-hh Pic 99.
     03  GMT-Offset-Direction Pic X.
* GMT-Offset-Direction is + or -
     03  GMT-Offset Pic 9(4).

With MicroFocus you can also try ACCEPT The-Date from Date YYYYMMDD.

This will work if your version is recent enough.
```

```
Hi !
Try this :

WS:
       01  WS-DATUM-2000.
           03  DATUM-2000      PIC X(21).
           03  DAT-2000-R REDEFINES DATUM-2000.
           05  DAT2000         PIC 9(8).
           05  DAT2000-R REDEFINES DAT2000.
              07  D2-JJ         PIC 9(4).
              07  D2-MM         PIC 99.
              07  D2-TT         PIC 99.
           05  ZEIT2000.
             07  D2-STD         PIC 99.
             07  D2-MIN         PIC 99.
             07  D2-SEK         PIC 99.
           05  D2-HSEK          PIC 99.
           05  FILLER          PIC X(5).

Procedure :
              MOVE FUNCTION CURRENT-DATE TO DATUM-2000

The last FILLER is the difference from Greenwich-Mean-Time.

(MF-Cobol)
```
