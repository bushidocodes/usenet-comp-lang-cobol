```
jwi··.@b85··f.mil (Jon Wilson) wrote:

› Any idea how to kick off a background job in MicroFocus 3.2

Is this the sort of thing you want?

identification division.
program-id. callcmd.
working-storage section.
01 cbl-get-os-info-param.
03 param-size pic x(2) comp-x value 14.
03 p-os-type pic x comp-x.
88 os-2 value 0.
88 dos value 1, 2.
88 flexos value 4.
88 ms-windows value 5.
88 unix value 128.
88 xenix value 129.
88 os-2-32 value 130.
88 windows-nt value 131.
03 filler pic x(11).
01 dos-result pic x comp-x.
01 dos-fn-35 pic x comp-x value 35.
01 dos-param.
03 filler pic x comp-x value 0.
03 filler pic x.
linkage section.
01 sys-command-line pic x(128).
procedure division using sys-command-line.
call "CBL_GET_OS_INFO" using cbl-get-os-info-param
evaluate true
when os-2
when os-2-32
when ms-windows
when windows-nt
when dos
when flexos
display sys-command-line upon command-line
call x"91" using dos-result dos-fn-35 dos-param
move dos-result to return-code
when xenix
when unix
call "system" using sys-command-line
when other display "Unknown O/S" stop run
end-evaluate
if return-code not = zero
display "Running program in background failed"
display "Used command: " sys-command-line
end-if
.
end program callcmd.
```
