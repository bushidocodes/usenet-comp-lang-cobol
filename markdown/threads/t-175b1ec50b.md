You can control how many levels of any treeview node expand/don't expand
using standard treeview messages (or by playing with the cchild member of
the TVITEM structure) ... of course, I do not have a clue how you do it with
those "easier to use" OO properties and methods stuff.

My EDI Pal (tm) product uses a treeview, and I expand/contract the nodes all
kinds of ways... however, you should note that recursion IS required to
fully manage a 'generic' tree structure and you may have to use ENTRY with
LOCAL STORAGE to effect recursion correctly with COBOL.

(Properties?  Methods?  Real Men use the raw Windows API!).
