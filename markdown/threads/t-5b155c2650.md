Need to write a TCP/IP app in COBOL II, using the IBM TCP/IP CALL API

It will require lots of bit manipulation, not just for the TCP/IP-related
API CALL's, but for the (EDI) data stream also.

What are the bit manipulation alternatives for COBOL II ? It doesn't appear
that the COBOL II language has direct support. There's a subroutine
(EZACIC06) provided with the IBM TCP/IP product, but it looks cumbersome.

Is there a "bit manipulation subroutine" library for COBOL somewhere?

Thanks!

David Thom
NPSi

Cobol for VM & MVS can handle bit manipulation. however Cobol for VSE
doesn't yet. So if you have code that needs to be compiled on VSe, it
will not work. I am not sure why it was implemented to work on VSE but I
was told to call IBM and request it.
