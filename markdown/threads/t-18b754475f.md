The "S" means that it is a SIGNED number, so it can contain both 
positive and negative numbers.  "PIC S9(4)" means exactly four decimal 
digits of precision.  It can contain integer values from -9999 to +9999.

You didn't ask, but the harder question to answer is how the number is 
stored in memory or on a file.  That will depend on the hardware, the 
compiler vendor, and what kind a USAGE clause (if any) was present 
with the PICTURE clause.

I hope that helps!

KL wrote:
> If I see PIC  S9(4)   what does the S mean?
>

As a general rule, numeric fields should be signed even if they will only
contain positive values. This is particularily true for subscripts and
indexes.

Computations are done according to algebraic rules and the results stored in
the defined variable. Just before storing the result, the compiler notices
the absence of the "S" and introduces code to take the absolute value of the
result. To avoid this extra processing, use the "S."
