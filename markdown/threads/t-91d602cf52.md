```
If the file is indexed, then you simply move a value to the
key and read it.  For example:

    MOVE "TEES" TO THE-FILE-KEY.
    READ NAME-FILE INVALID KEY
            DISPLAY "There is no record with the key equal to TEES".

Keep in mind that alphabetic keys require an exact match,
including spaces.  The normal method is to convert key
fields to all upper or all lower case before writing the record.
```

```
You can specify ORGANIZATION IS DYNAMIC for the indexed file and:

   77  WS-NAME-PRESENT-FLAG   PIC  9(01) VALUE 0.

   MOVE "TEES" TO THE-FILE-KEY.
   MOVE 1 TO WS-NAME-PRESENT-FLAG.
   START NAME-FILE
        KEY NOT < THE-FILE-KEY
        INVALID KEY
            MOVE 0 TO WS-NAME-PRESENT-FLAG.
   IF (WS-NAME-PRESENT-FLAG = 1)
       READ NAME-FILE NEXT.

This obtains the first record in the file which has a key equal to
or greater than "TEES".  The state of WS-NAME-PRESENT-FLAG will
show if a record was found (=1) not not (=0).  You would rarely
use a direct read looking for a name, because it is too difficult
to spell names consistently correct.  Read the first one found
above, then display the next 10 or 20 names for selection, and
permit a 'view next page' option to display the next 10 or 20
names.  If COBOL 85 were not pathologically inhibited in indexed
access, you could also support a 'view previous page' option and
display the previous 10 or 20 names using READ NAME-FILE PREVIOUS.
Let's hope this clause is in the next standard.
```
