I've never done COBOL CLI (call level interface) programming, but that's the
subject you'd want to look up.  That, and whether or not you have an ODBC
driver for your particular compiler in order to allow for communication
between SQL Server and COBOL.  Oh, and maybe also see if you need a COBOL
pre-compiler for the CLI statements (not sure, but you definitely need a
pre-compiler for embedded SQL).

I already responded personally to Thane, but figured I should post something
as well... just in case anyone else is reading.  It turns out that Microsoft
ComTI doesn't support host initiated transactions.  I was previously told
(by MS) that this feature would be available with it's latest and greatest
invention HIS 2000 (Host Integration Server).   Today I was told (by MS,
again) that this functionality is not available and they do not know when it
will be.  So, although COBOL/CICS CLI (call level interface) is an option
when connecting to programs on different platforms, it is not the solution
for connecting COBOL/CICS to SQL Server.  If anyone else has any helpful
hints, our ears and minds are open!
Jess
