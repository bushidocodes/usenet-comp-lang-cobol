IBM has an HTTP server that runs on big iron
 http://www-4.ibm.com/software/webservers/httpservers/ ). The CGI interface
supports COBOL applications.

They also have an application server (WebSphere) that allows you to write
Java applications and encapsulate COBOL processes using something called
Enterprise Java Beans (EJB).

A more common model is to move the web hosting to a different box (or
cluster of boxes depending on the scale) and use something like TXSeries
(CICS) or MQSeries to co-ordinate between the boxes. Security plays a big
role in deciding to expose your mainframe to the world at large. Granted
mainframe security is as robust as it gets.

Karl Wagner

P.S. I'm doing a COBOL CGI application at the moment for an on-line ordering
system (WANG back-end) It's taken about 3 days to cobble together a working
prototype, including designing the data model. It'll take a lot longer to
turn it into a production-ready system.
