Alberto? Next time, please include the original question... took me a while to
remember what this was all about.

OK.

In the user's LOGIN.COM, the data file is copied to something called AAM.DAT,
right? Then the user runs your program from the "$" sign?

Maybe, what you want to do is write a small DCL script to do the following...

$DEL DATA:W_TESTE.DAT;* ! deletes the file already in place
$COPY DATA:AAM.DAT;0 DATA:W_TESTE.DAT;1 ! copies the new data
$DEFINE/USER INPUT_FILE DATA:W_TESTE.DAT ! creates the user mode link
$R EXE:PROG ! runs the program

This, of course, assumes that you have DATA: and EXE: all setup and pointing to
the correct directories. I'm also assuming that your program is called
PROG.EXE.

Name the DCL script RUNPROG.COM (you can ignore the comments!).

Now, in PROG.COB (the source file), code the following...

SELECT TESTE ASSIGN TO INPUT_FILE
put the necessary ORGANIZATION, ACCESS MODE, etc. stuff here

DATA DIVISION.
FILE SECTION.
FD TESTE.
put the remainder of the FD here: 01, etc.

PROCEDURE DIVISION.
OPEN INPUT TESTE.
rest of program code...

When you execute the OPEN, VMS automatically makes the connection between
INPUT_FILE in your DCL and INPUT_FILE in your COBOL program. Notice that you
never 'move' anything to INPUT_FILE inside your program, since this is a
reference pointer. And, if it makes you feel better, you can substitute W_TESTE
for INPUT_FILE in your DCL script as well as in your COBOL source. Just make
sure that it's called the exact same thing in both places, otherwise VMS will
get lost.

Now, all user AAM has to do is type @RUNPROG at the "$" sign, and everything
should work as planned.

Understand?
