you are making a *very* basic error that is common to programmers being
introduced to accounting for the first time ... probably because an
accountant tried to program. Never, never, never subtract in accounting, and
never, ever introduce a variable that acts as a pretend sign.  You will
regret it forever.

You have Debits (negative entities) and Credits (positive entities).  Those
are added together to create another debit, credit, or balanced item.  That
is it.  There are no other types of entry.  All else is documentary.

The file match construct along with the control break (see conditionals FAQ)
account easily for 80-90% of all batch programming problems. So a generalized
solution (or methodology) is extremely valuable. To provide that generalized
solution is the purpose of this FAQ and that mentiond above.
