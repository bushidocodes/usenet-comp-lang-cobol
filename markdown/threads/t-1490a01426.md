```
I am writing a client in Delphi 5.0 that accesses a server written in the
latest version of NetExress. The server is using some sort of communication
interface called CCI (Common Communication Interface?) over a TCP-IP
connection. I can access the server by using a dll written by a cobol
programmer, but that dll need 8 other dll's from the MF Cobol runtime
library...


What I would like to know is:

Is there any way to talk to a CCITCP server from another platform using a
standard socket connection, and what has to be done to accomplish this. (If
I try to open the IP:Port through winsock I get a message saying the 'server
actively refused the connection'. So I assume there would be something more
than just opening the socket and sending the data.)

Any assistance would be greatly appreciated!
Richard
```
