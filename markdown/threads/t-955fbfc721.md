I'd like to get a better understanding of how the intrinsic function
range works.  Let's assume that I have a single character field in a
record that I'd like to test against a range of values.  This range of
values is variable and could be obtained externally either from a
calling program or from a single record control file.  How could I
verify that the single character field falls within the specified range
of values using a FUNCTION RANGE construct?  Yes I know I could just
compare my field to the high and low range values to verify that it did
not fall outside the range but I'd like to know how the range function
is or can be used.

RANGE is for numeric data items. It subtracts the smallest value in the list
from the largest value in the list, returning a single value. It has nothing to
do with seeing if a character is "in the range".

When writing Sams Teach Yourself COBOL in 24 hours, I made sure to put
special emphasis on the Intrinsic functions.  These are a very
important and powerful part of the current standard.  I cover EACH
function, with examples.  While some of the functions can be used with
numeric and alphanumeric arguments, Range is only valid for numbers.
You might be better of setting up a new CLASS in the Special Names
section.  Interestingly, MIN and MAX can be used with alphanumeric
fields.  I remember writing code to search a table for the highest
value -- Sure wish I had MAX available then!
