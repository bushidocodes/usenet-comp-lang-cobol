```
We are evaluating NetExpress version 2.0 and have come across the following
problem. Since we would be converting from Microsoft COBOL we want the
Microsoft COBOL V2 file status codes returned rather than the ANS85 codes.
The NetExpress documentation (the File Handler book) confirms that this can
be done by setting two program directives viz. MS"2" and NOANS85. (BTW these
are the same directives we use in our current Microsoft COBOL 4.5 compiler).

We have done this but the ANS85 file status codes are still returned. We
have also downloaded the latest updates from the WebSync site with no
change.

Although we have reported this to our local (prospective) supplier who is
looking into it, I was hoping someone may have already solved this problem.

Regards, Robert.
```

We have a tool (written at request of a customer in 
just 2 days using CobolTransformer) that 
    adds calls to a Cobol Program that converts 
    MS IO status codes to MF IO status codes.
The calls are added immediately after each IO statement
that works on a file with FILE STATUS clause.

Example of this automatic conversion:

< 	   OPEN LOCKING IS EXCLUSIVE I-O MASTER-FILE.
< 	   OPEN LOCKING IS EXCLUSIVE I-O TRANS-FILE.
---
> 	   OPEN LOCKING IS EXCLUSIVE I-O MASTER-FILE
>            CALL "CBL_TRANSL_IO_STATUS" USING WS77-MASTER-STATUS
…[3 more quoted lines elided]…
> 	                                           .

Here SELECT for MASTER-FILE has FILE STATUS IS WS77-MASTER-STATUS clause.

So if you use our tool to convert your MS programs to MF,
you do not have to wait for the next version of NexExpress
that has the fix and/or pay for upgrade.

Only the lines that need to be converted are changed.
All other lines in a program are left intact -- so
there is no "lots of generated code" to deal with.

Conversion also happens in the copybooks (if they were 
included from Procedure Division and have IO statements). 

For more information please contact info@siber.com.

Regards,
Vadim Maslov
