As far as your question on how to view object code to see if it is doing
the correct I/O's I am not sure what you mean. Is the object code you
are referring to a compiled COBOL program or do you mean the source code
that is the original COBOL code that was entered by an editor? If it is
really machine object code I don't know how to open it up to view it.

I don't know anything about cobol/3000 or edit/3000 but I do know that
Acucobol has a COBOL-85 compiler that runs on the HP3000 but requires
MPE/iX 5.0. It runs in the POSIX shell which is very similar to Unix.
If you can upgrade to MPE/iX 5.0 then you can call 1-800-COBOL85 and
ask about getting information or a copy of Acucobol for MPE/iX on the
HP3000. The documentation is probably better than what you have.
Otherwise you probably need to call someone at HP.
