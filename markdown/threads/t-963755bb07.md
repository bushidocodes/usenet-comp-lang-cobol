Probably from reading past end of file.

>If I use the 'Start Valid/Invalid key' clause I get a Runtime error (163)
>telling me that there is numeric data in an Alphanumeric item. The item
>causing this is the record key for the indexed file.

A 163 is caused by non-numeric data in a numeric field. See below.

[...]
>000320 FD  TRANS-FILE.
>000330 01  TRANS-IR-REC.
…[7 more quoted lines elided]…
>000400     03  T-D-CUST-CODE     PIC 9(5).

For example, when TRANS-FILE is at end of file the
T-x-CUST-CODE fields will conatin HIGH-VALUES. Accessing
T-D-CUST-CODE at end of file will cause an error 163 since
HIGH-VALUES is not a valid value for numeric display.

It appears that reading past end of file may be the larger problem.
