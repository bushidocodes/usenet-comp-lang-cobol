>I have a request here for some Dibol work ... anybody used it?

DIBOL is an ancient DEC language which looks like Basic with the addition of
explicit data definition. It does NOT look like COBOL, as DEC salesmen used to
claim. 

I knew it inside-out 20 years ago, when I wrote a cross-compiler to convert
Dibol to COBOL. The translation was difficult because Dibol uses out-of-line
error handlers for things such as arithmetic overflow. COBOL does it with
in-line code -- on size error. A well written cross-compiler must simulate flow
of control through the Dibol program to know which error handler, if any, would
be enabled when an arithmetic statement is executed.

There are now several Dibol-to-COBOL cross-compilers commercially available. If
you cannot find a Dibol expert, you might consider converting. The programs are
probably not running on DEC hardware -- PDP-8 or PDP-11 -- because maintenance
has gone sky high. They're probably running on a PC under a DEC emulator.
Converting to a currently-supported language would make them easy to maintain,
run much faster and eliminate the need for an emulator. 

Robert
