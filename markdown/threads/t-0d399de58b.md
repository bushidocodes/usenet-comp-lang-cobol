You are further along, but without the actual layout of what is in the
data, you are still not far.  From this we can tell you how long each 
record is, and we can tell you what the "key" fields length and 
positions are - looks like 3 keys on that file - one primary and two 
alternate.  But you still don't know the actual content of each field 
- you can't get it without the source code.  Given the "record layout"
for each file, however, you could  use rebuild to create a level II 
indexed file - which will output the data portion in "flat" ascii, and
you can use that (once you know what is in it) to get where you need 
to go for data conversion.  But you need to pressure the vendor for 
the layouts.
