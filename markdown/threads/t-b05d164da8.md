Hi Danielle,
One very important item to keep in mind about AcuCobol is that there
are two ways to accomplish nearly every program - you can code
everything longhand in traditional COBOL methods or you can use the
code generator which is core to AcuCobol's existence.  I use both way
and am partial to the traditional method.  For what you want to do,
the easiest method is to use the code generator.  There are still a
few ways to do this but what you could do is define an event to the
button you are coding for - in the Structural View choose the screen
you are coding for then click lightly on the button you've defined, go
to the event tab of the properties window and select the type of
event, set up a paragraph name for the event and the event section
will open with the cursor underneath the paragraph title you've
defined.  Here you can put traditional code in to accomplish the
calling of your program.  Good luck.  (Sometimes the automatic code
generation can be frustrating to work with!)
