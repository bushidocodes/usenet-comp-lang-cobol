Sorry, no such specification exist for "COBOL Packed Data".
Specifications will exist for each system, or perhaps each implementation
on each system.

On IBM mainframe, COMP-3 data is an odd number of 4 bit nibbles, each
containing a decimal digit in the range of 0 to 9. This is followed by a
sign nibble, typically containing C (plus), D (minus) or F (no sign). If
the picture has an even number of digits specified, an additional nibble
containing zero will be added at the start of the field.

I.E. COMP-3 PIC S9(7) value +123456.

Would be stored as X'0123456C'
