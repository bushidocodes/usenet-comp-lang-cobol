>Yes I am looking for an ftp client that has drag and drop features.


All applications including DOS have drag and drop functionality through what
used to be called file associations in the windows explorer. These can be
created to have different actions associated with each file extension you
associate it with. If a file is simply dropped onto a DOS application the
file name is passed as if you typed the EXE name followed by the dropped
file name. In most instances in 8.3 format.

Drag and drop functionality works with nearly every application I have
tried. You probably need to figure out what information needs to be passed
to the application itself and what parameters are valid.

The MS FTP client won't help you with the server portion but will work 
fine for the client. Do an "FTP -?" at the command prompt to get a list 
of options (see the -s:<filename> switch). Just put any FTP command 
inside the redirect file.

If you're using IIS for the server, the log is pretty easy to decode and 
flexible in what it can capture. I am using the log file to send email 
transfer confirmations and update an ISAM log file. I could dig up a 
sasmple if you need.

If you want to roll your own client, look at Ftp4w (I mentioned before 
but forgot to include the URL). It is a freeware library so no 
royalties. I use the companion library TCP4U with provides a good 
sockets library. The documentation and zip file can be found at:
http://home.rol3.com/~u0389227/ftp4w302.htm

The most significant problem I ever ran into was getting the USER ID and
password from a shell extension, or file association. Assuming you set up a
generic UID and PW the following is typically what I do:

-------------------------------
rem %1 = user id %2= password %3 = mainframe file name
rem 000.000.000.000=ftp address of ftp server

del "ftpcmds.ftp"
echo %1 > ftpcmds.ftp
echo %2 >> ftpcmds.ftp
echo get '%3' "%3.txt"  >> ftpcmds.ftp
echo quit  >> ftpcmds.ftp
ftp -i -s:ftpcmds.ftp 000.000.000.000
--------------------------------

Assuming above BAT file is named "ftpget.bat" the file association would
refer to the BAT file as:
"ftpget.bat" ANONYMOUS GUEST TEST.DATA.FILE

Where
ANONYMOUS = user id
GUEST = user password
TEST.DATA.FILE = mainframe file name

If you are looking to transfer from the mainframe to a PC server, you would
have to have the command file set up then run the mainframe FTP command,
typically FTP. Try scanning your current mainframe JCL and PROCS for
examples.
