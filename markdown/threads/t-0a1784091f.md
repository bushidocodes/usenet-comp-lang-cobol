There are two ways this is generally done:

If you use SELECT sales-report-file ASSIGN TO PRINTER
you must use a SET command prior to staring the program. This
is somewhat similar to a TSO CLIST ALLOC command, or a JCL DD
statement. SET PRINTER=LPT1 or SET PRINTER=PRN.

If you use SELECT sales-report-file ASSIGN TO "LPT1"
your are assigning the device within the program directly. The
SET statement is not required. You loose the flexibility of
redirecting your output, but you also don't have to worry about
the SET statement. LPT1, LPT2, PRN, etc., are reserved device
names, so the systems knows (should) what they are. This should
for CA-Realia and Micro Focus. Other compilers probably will, too.

mike dodas
