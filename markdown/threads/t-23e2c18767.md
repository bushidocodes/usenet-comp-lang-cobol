```
It appears that the file handler in Microfocus caches the sequential
reads on a file in memory. This causes a problem if you stay in a
program for a long period of time. I've got users (as shown by the
svmon command) using 24 megs for a single file! This certainly causes
problems on a system with many users. I haven't seen an option to
limit this cache pool, do I have any options that I'm unaware of?
```
