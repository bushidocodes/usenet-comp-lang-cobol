All COMPUTATIONAL items are vendor defined and are not standard.  

It is likely that open-cobol only implements (or will implement) standard 
items such as BINARY and PACKED-DECIMAL.  Therefore there is no definition 
for any COMP- items in open-cobol.

For what any particular version of Cobol means by COMP-4 you will have to 
read the documentation of _that_ version.
