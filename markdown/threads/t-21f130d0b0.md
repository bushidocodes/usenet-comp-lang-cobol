You don't describe the file format, but your use of 'lines ... just spaces'
and 'Any words ... removed' imply a text file.

Blank lines are easy enough to reject by just comparing the input record to
SPACES.

Words are a little more complex. You don't say if the position of the
words in the line may be changed, or if the words should be compacted onto
as few lines as possible (which might be implied by the requirement to
remove blank lines), or what any delimiters might be, if other than spaces.
But I will assume the delimiters to be spaces, the spacing between words
is irrelevant, and no moving of words between lines.

Set up a working area containing as many 10-character fields as you would
ever find on a single line. I would use a subscripted table. Do an
UNSTRING ... DELIMITED BY ALL SPACES to extract all the words from the
input record into the table. Then loop through the table using INSPECT to
examine each extracted word, moving spaces to each table entry containing
bad characters. Then do a STRING ... DELIMITED BY SPACE of all the entries
in the table to insert them into the output record, separated by a space.
Other details I leave up to you.

That should be enough help for a homework assignment.
