```
No newline statement. The concept of lines really does not exist in std
Cobol.
On most systems a newline is automatically issued after each DISPLAY
statement.
So:
DISPLAY "Hello"
DISPLAY "World"
 would output two lines. If you include NO ADVANCING, no newline is output,
so:
DISPLAY "Hello " NO ADVANCING
DISPLAY "World"
would output one line.
Good luck
```

```
Jeff:

With the latest level of the COBOL Report Writer syntax, you just code a 
blank LINE clause, e.g.

    05  LINE + 1.
    05  LINE + 1.
      07  COL 25  VALUE "Next Heading".

This is not the best example perhaps because you simply have coded a 
single LINE + 2.  But it's used in cases where the blank line is optional 
(use PRESENT WHEN condition) or when you want a blank line before a 
table, e.g.

    05  LINE + 2.
    05  LINE + 1  OCCURS  2 TO 10 TIMES DEPENDING ON ... .
```

```
Depends on what you are doing. There isn't one as such. 

       display "Hello"

will display Hello and return to the next line. Use the 'no advancing'
clause to *not* get a new line. Sit down, take a deep breath, and dig
out your manual and look up the syntax for the 'DISPLAY' verb.

If you're formatting strings for dialog boxes or some other kind of
api style string then you create your strings in much the same way as
C with a null at the end.
```
