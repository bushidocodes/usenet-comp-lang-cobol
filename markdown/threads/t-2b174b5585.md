### Platform "independency" and efficiency

- **Message-ID:** `<7sq8jn$1pg@dfw-ixnews21.ix.netcom.com>`

```
(Obviously, this is a follow up on several recent and ongoing threads - but
I thought it was a topic that deserves its own thread.)

One thing that I would suggest that PAYING customers do, is to press their
vendors to improve the run-time efficiency of code created with a vendor's
compilers when "Standard conforming directives/options" are selected.

I know this best from both Micro Focus and IBM (mainframe) compilers - but I
really think that this is true of many, if not most, compilers.

The developers just don't think that there is "user demand" to spend their
time optimizing performance for programs when "Standard conformance" is
selected.  Instead, they assume that all their customers who are interested
in "efficiency" will always pick the full "optimized for this
platform/compiler" set of options - and those interested in cross-platform
independence, don't worry about performance on any specific platform.

Now, I think it is (probably) true that no vendor is ever going to be able
to get MORE efficient (or possibly even as efficient) code when "Standard
conformance" is selected (for example, just think about what the COBOL
"truncation" rules require for USAGE binary items) but I really think that
at LOT of improvements could be made in this area for all the compilers that
I know of.

Therefore, if you are interested in "cross-platform independence" but you
still want the compiler to produce as efficient code as possible, I think
that this is something that you need to communicate (probably fairly
VOCALLY) to your vendors - as I don't think they are hearing it today.

As usual, this is one of those areas that lots of people ASSUME someone else
is telling the vendors (or that they already know).  However, I really do
think that COBOL compiler vendors do listen to those customers who TELL them
what they want (especially what they are willing to pay for).
```
