```
I have a question for MicroFocus mavens.

According to the MicroFocus release notes on our HPUX 10.20 box, 
MF COBOL cannot access files larger than 2 GB.  (There are also
operating system issues surrounding the use of large files, but those 
are not my concern here.)  The recommended workaround is to replace the 
usual COBOL IO verbs with calls to C routines that are enabled for large 
files.

Fortunately this solution is reasonably doable for us, since we only
use COBOL for line sequential (i.e. text) files.  Indexed files or
variable-length record sequential files would entail a great deal 
more pain.

On the other hand we'd rather avoid this step if possible.

Soon we will upgrade to HPUX 11.x, which is much friendlier to large
files.

The question: is MicroFocus COBOL limited to file sizes of 2 GB or 
less, even under HPUX 11.x?  The release notes available to me are not
very explicit on this point.
```
