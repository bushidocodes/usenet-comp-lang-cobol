However, here at home - I do.

The 74 and 85 standards state that INVALID KEY is REQUIRED if there is no
Appropriate USE statement (Declarative).

The same is true of AT END.

Most compilers ignore this restriction.  Some don't but report errors - ala
Fujitsu.

Others (VMS COBOL) won't even compile if you follow the rules.  

A notation in the draft standard makes it clear that this restriction is
going to be removed in the next standard.
------
73) INVALID KEY phrase. The INVALID KEY phrase does not have to be
specified if there is no applicable USE
statement.
------

At any rate, a compliant compiler WILL require the declarative if no
invalid key is coded, and that is the root of the question.

BTW - to remove the annoying messages with Fujitsu COBOL - do this:

When you run the program and the runtime options window appears, in the
Environment Variables Information field type in @NoMessage=YES. Click Set
and then click Save. Follow the prompts. Then click OK to run the program.
The error message window is now disabled.
