Caveat emptor. Caveat omnis. Here there be dragons!!!

To terminate a process:

1) Call OpenProcess(PROCESS_ALL_ACCESS, TRUE, ProcId)
2) Call TerminateProcess(hProcess, 0)

see Microsoft KB Article Q90749 for details.

The harder part is determining the process Id for the application. 

Under Windows 9x you can use the ToolHelp32 functions: CreateToolHelp32Snapshot(), Process32First() and Process32Next().

Under Windows NT you use the platform services API (PSAPI): EnumProcesses(), EnumProcessModules(), GetModuleFileNameEx() and GetModuleBaseName().

see Microsoft KB Article Q175030 for more info.

There may be more recent articles that provide a more elegant solution. You might also need to use SetPrivilege() to give your application debug privileges and the power to cause havoc to your system.

Oscar provides a lot of good information.  One point worth
mentioning though, is the use of TerminateProcess.  This
is equivalent to Ctrl-Alt-Deleting the application.

Some applications, Lotus Notes for one, can realize that
they've abended and will refuse to run until a restart is
performed.

The "civil" way to close an application is to send a WM_QUIT
message to it's main window.  As it may be difficult to determine
the main window (some apps don't really have a main window,)
it's generally accepted practice to send WM_QUIT to all top
level windows.
