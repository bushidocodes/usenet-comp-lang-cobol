It's a complex question that's fairly easy to answer once I understand
exactly WHERE and on WHAT type of device you're trying to print a
bar-code on. Are you using something like a Zebra printer attached to
the parallel port? Or, are you trying to print a bar-code on a piece of
plain paper? If you're doing the former, all you have to do is
construct the ZPL-II (or another vendor's control language) to create
the control characters required to print the label and 'print' it (write
it to the printer), substituting the required variables in the string.
If you're using paper, you'll probably need to CALL a DLL to construct
the bar-code and then send it along with the document. Your printer
manufacturer should have some software and/or instructions for doing
this.

Well Jaco,

When I did it, we had a list of printers we supported and we sent the
codes to actually do the graphics. HP has a font cartridge you can buy
that has the standard ones in it. OTC makes a printer that does them
without special graphics programming, and Printronix makes the magnum
board that allows you to do it with just some control characters.

I would first explore those printers that say they can print barcodes
and then figure out the codes necessary to do it.

Failing that, you have to get into graphics programming.
