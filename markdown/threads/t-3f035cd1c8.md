I have an application that is running under Windows NT 4.0, Service Pack 3,
NetExpress 2.0 with all service packs, IBM Distibuted CICS 4.2 using IBM
DCE. The problem I am having is when I use a UNC name to access a file it
fails with a return code of 9/004 which indicates an invalid file name. The
interesting part is if I run the same code outside of the CICS transaction
it works fine. Here is a little test program that I have written to
demonstrate the problem. Any thoughts or suggestions would be appreciated.
I have tried this on a FAT drive and an NTFS drive. The UNC acctually
points to a local drive. I also added some code (not shown here) to create
the file and when I do I get a return code of 005 from WinGetLastError on
the CreateFile API. The text of that error is "ERROR_ACCESS_DENIED". It
appears when you install Distributed CICS that it creates a local account of
cics which has administrator rights. But when you look at the service
definition it logs on as a local system account.
