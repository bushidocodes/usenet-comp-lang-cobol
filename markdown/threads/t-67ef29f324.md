The following is a reply from Rex Widmer (who currently doesn't have access
to the newsgroup),

"How to retreive compiler options from a load module...

Easy way - (entering BLATANT COMMERCIAL mode) - use Edge Portfolio
Analyzer - run against loadlib, can do 1 member to whole library, supports
OS/VS COBOL, VS COBOL II, COBOL/370, COBOL for MVS and VM and COBOL for
OS/390 produces nicely formatted list of compile time options, and machine
readable version of same - suitable for generating "recompile this with same
options" JCL. (end of BLATANT COMMERCIAL)

More difficult way -

Determine that csect was created by VS COBOL II -most directly done by
viewing AMBLIST LISTIDR output, and noting that IDR reports translator
566895801.

Use IMASPZAP to dump out the VS COBOL II generated csect, about the first
x'80' bytes are interesting for this exercise.

Use the listing description in VS COBOL II Application Programing
Debugging R4 (not a licensed manual) to decode the INFO BYTES starting at
location x'2c'. While this is not an "official programming
interface" the method works across all of the VS COBOL II family, so long as
you know which options apply to which release of the product."
