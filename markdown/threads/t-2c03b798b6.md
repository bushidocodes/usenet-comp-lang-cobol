```

I am not familiar with "Supersearch" so I cannot answer the
question within that specific context.

However, I can offer some observations that will enable you to
pursue the matter.

For all the releases of OS/VS Cobol of which I am aware, a
"signature" of VSR1 is inserted in the generated code. For the
releases in our shop (from which we have only 2 remaining
homwgrown programs, altho' third-pary is another matter), the
VSR1 stamp appears at displacement 14 (hexadecimal) (i.e.
"position" 21 decimal). Therefore, we can use a utility such as
FileAid to search for OS/VS Cobol batch programs by selecting
load modules containing VSR1 @ pos 21. For CICS programs,
assuming a standard stub, search for VSR1 at position 93.

Another technique is to use a "service aid" such as AMBLIST,
although I prefer to use the PDS shareware TSO command processor
from the CBT tape, to "select" modules that contain CSECTs
starting with ILBO (unless your already converted programs are
calling the abend routine, ILBOABN0), or that contain no other
CSECTs at all; the latter will pick up assembler programs and BMS
maps as a side-effect. If you prefer to work backwards, select
all the programs which contain CSECTs starting with IGZ: these
will be the programs converted already to COBOL II or COBOL370.
```
