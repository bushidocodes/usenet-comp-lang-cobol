I am a long time (I guess your phrase was "old time") COBOL and C
programmer and have used both languages extensively over the past 16
years on Unix, MSDOS, VMS, and GCOS. Unlike most C programmers, I do
not have the typical COBOL-phobic attitude toward a language which most
of its critics have never written in. COBOL is an excellent langauge
for file/database programming and report writing - generally superior to
C. HOWEVER it is a *pathetic* language to try to use for system
programming! Your COBOL people are using COBOL because they don't know
any other tool. On the GCOS6 platform I too created all kinds of COBOL
programs which did things like you describe - and worse! Unix's number 1
strength is its ability to pipe together seemingly useless shell
programs (uniq, comm, grep, sed, ...) to create all kinds of wonderful
system utilities without any programming at all (e.g. I once created a
useable SQL facility using about 60 lines of script commands and
awk-able text database tables which handled my company's financials for
its first year - this was done on a whim, and is not recommended).

Bottom line: I agree with you, "Unix utilities should be written in C or
shell scripts". However MicrFocus COBOL is a good product and is not
"damned" - the runtime generally expects to have more to do than "run 1
line of COBOL". Try this in C: Read a binary ISAM file by its alternate
key starting the search at a record specified by a particular paritial
key value. While reading, reorder the records according to some other
sequence. Now, print all the records, being sure to insert currency
symbols, commas at thousands positions and dont forget to blank numeric
output if the value is zero. I'll take bets as to whether the C or
COBOL program is bigger! Did I mention control breaks? For our next
exercise - checkpoint/rollback/commit on database group transaction
updates ...

half-guru. Ok, in the long term, it's simpler and shorter, but if

short terms and program in Cobol... Of course, they need learning
sessions, at least they need to be sensible to the beauty of
'man' (if it's correctly installed, with the whatis database...).

utilities such as sed and awk that can seem, well, let's say a
bit complicated for a newbie, so don't push them too hard...
