date integer is to be represented for human consumption.

 They also explain this:

--------- schnipp -----------------------------------------

Interpreting 2 Digit Dates

By default, all two digit years are interpreted as 20th century years.  
21st century years are entered by specifying 4 digits. For many  
applications this is insufficient. InstantDB therefore allows the  
property milleniumBoundary to be set in the properties file. When set,  
this allows two digit dates less than milleniumBoundary to be  
interpreted as 21st century dates.

   I think that most database systems providing a date-time data type  
store this as an integer (when date only) or as real number (when date  
and time in one value, time being represented as fraction of a day).

   An exception which I know is MySQL where a date is actually stored  
as YYYY-MM-DD and where this can be an invalid date; a feature which  
can come handy when one wants only to specify  a month and date  
(recurring date each year), or an approximate date, where only year  
and month are known, but the day is left ZERO for meaning unknown.
