Todd,

There is nothing in the product that has any dependencies on the number of
CPUs in use. In fact, the operating system hides that totally from the
application level. Therefore, the most likely reason that a 173 error would
occur only on a multi-processor machine  would be as a result of the
scheduling of threads being different because they can genuinely run
concurrently on a multi-processor machine rather than being switched in anbd
out of the CPU on a single processor machine. Assuming that you are running
a multi-threaded application you will need to use synchronisation to ensure
that programs are loaded in the correct order.

If that is not the case then, as Bill mentioned, you will need to follow up
via AnswerLab.
