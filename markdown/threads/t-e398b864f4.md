```
   identification division.
   program-id. main.

   call 'function-get-time' using
```

```
A) The "entry" (not to be confused with the "ENTER") statement is not a part
of any current or recent ANSI/ISO Standard.  Therefore, it has "limited"
portability.  (Many/most IBM "compatible" compilers support it.  I don't
know about any others)

B) If the only ENTRY statement you have is "right after" the Procedure
Division header, it probably does nothing - or almost nothing "useful".
(Older - almost ancient - versions of IMS *required* an ENTRY statement, but
that hasn't been true for many releases.  You will still see it, however, in
such environments)

C) If you have MULTIPLE "ENTRY" statements in your program, this provides
some functionality.  Especially if you don't have an '85 Standard compiler
with EXTERNAL files/data and Nested Programs.  However, even there, you need
to be VERY careful if your calling program using a different number of
parameters when calling different entry points.  Addressability to
"non-passed" parameters may or may not be maintained when you call different
ENTRY points.  Compare the Micro Focus "Sticky-Linkage" and IBM
Dynamic/Static CALL restrictions).

D) In my PERSONAL opinion, if you do have an '85 Standard compiler with
EXTERNAL data/files and Nested programs, there really is NEVER a need to use
ENTRY statements in "current" (new or maintained programs).
```

```
Nested Programs are the 'new' subprograms ('85) while ENTRY is an
'old' non-standard mechanism that was, for example in MF Level II
Cobol which was '74 standard.
 
>    identification division.
>    program-id. function-get-time.
…[11 more quoted lines elided]…
>    end-program function-get-time.

The 'entry' is redundant.  Put the USING clause on the procedure
division header:
 
     identification division.
     program-id. function-get-time.
     data division.
     working-storage section.
     ...
     linkage section.
     01 get-time       comp-5 pic s9(09).
     procedure division using get-time.
     function-main.
         ...
         exit program.
     end program function-get-time.

NOTEs: 
    Your example is not 'nested'.  
    The called name is provide by the program-id.
    If the program is using dynamic linkage then the call
        will try to resolve by filename, but the cancel
        may be by program-id.
    It is 'END PROGRAM name' with a space not a hyphen.
```
