In general, the primary differences between different brands of COBOL
is in user interface and database interface, because those things
have never been standardized in COBOL.  Beyond that, COBOL is pretty
much COBOL, except that some shops still use COBOL 74, some COBOL 85,
and some use extensions that are not yet in the official standard.
But any competent programmer can learn a new environment without too
much difficulty.  It's nothing to worry about. :-)

Well, if *I* did it then YOU can do it ;-)
Here are the things you'll have to learn or adapt to:

	Display Files - AS/400 counterpart to BMS maps.
			The hardest part of learning to
			use COBOL on the AS/400. A couple
			of solid skeleton programs and
			your on your way...
	DB2 400       - The database is intrinsicly built
			within the operating system - OS/400. 
			Using it in COBOL is not difficult 
			at all (just standard stuff). 
	OS/400	      - You can be quickly overwhelmed
			by the number of available commands,
			but, you don't need to know many
			of them to just "program".
	CL Language   - AS/400's version of JCL - you'll
			love its ease of programming and
			its power.  The learning curve is
			is not too bad either.
	ILE	      - Integrated Language Environment.
			There are special considerations
			you must understand when coding
			complex interactive/batch business
			processes. The manuals are a solid
			place to helping you learn this.
 	SEU	      - Source Entry Utility - the editor 
			kinda looks like XEDIT but it's 
			*different*.  Not a big deal...

All-in-all, its a solid environment and FUN to work in.
If I'm lucky, I'll stay on this platform till I retire :)

Good Luck,
