```
Is there a word for logical infacility? When someone's language skills are weak,
not necessarily incorrect, he is called illiterate. When his math skills are
unsure, he is said to be innumerate. What do we call someone insecure with
logic? Illogical connotes incorrect rather than insecure. I usually use
intuitive, but that suggests domains other than logic. 

I thought about these people yesterday while working on a program. In
particular, people who say you should not use "not". (There's a self-cancelling
phrase.) So I 'simplified' one conditional by rewriting it to remove all use of
"not". See which version you find easier to read and understand. 

The code is going through a list of parsed words on a line, building a first-cut
template of numerics. Subsequent code will identify the most frequently
occurring patterns in order to understand where to find columns of numbers.
Thus, this code need not be 100% correct because the weight of prevelance will
overcome its shortcomings. 'No-form' means the document is suspected of being
free form i.e. it might not have any columns. 'Percent-word' means the word has
either a decimal point or percent sign. These are often bond yields buried in
the description rather than numbers in a column. At this early stage, the
program is unsure about the boundaries of a line's description. 

--- Before ---
evaluate true
 .....
 when (numeric-word or
       (punctuation-word and word-text equal to '-'
        and not previous-word-date) or
       (percent-word and word-byte (word-length) not = '%' and
        ((document-13f and not bypass-percents) or
         (document-reg and use-percents)))) and
      (title-from equal to zero or
       word-col-from less than title-from or
       word-col-from greater than (title-from + 20)) and
      (not no-form or word-col-from greater than 39) and
      format-length less than numbers-on-line and 20 and
      (not inside-parens or not number-has-decimals) and
      not looks-like-bond-stuff
     add 1 to format-length
     move low-values to numeric-column (format-length)
     move word-col-from to numeric-from (format-length)
     move word-col-to to numeric-to (format-length)
 ....

--- After ---
 perform test-for-number
 if  word-is-number
     add 1 to format-length
     move low-values to numeric-column (format-length)
     move word-col-from to numeric-from (format-length)
     move word-col-to to numeric-to (format-length)
 end-if

test-for-number.
 set word-is-number to false
 if  numeric-word
     set word-is-number to true
 else
     if  punctuation-word and word-text equal to '-'
         if  previous-word-date
             continue
         else
             set word-is-number to true
         end-if
     else
         if  percent-word
             if  word-byte (word-length) equal to '%'
                 continue
             else
             if  document-13f
                 if  bypass-percents
                     continue
                 else
                     set word-is-number to true
             else
                 if  use-percents
                     set word-is-number to true
                 end-if
             end-if
         end-if
     end-if
 end-if
 if  word-is-number
     if  (title-from equal to zero or
          word-col-from less than title-from or
          word-col-from greater than (title-from + 20)) 
         continue
     else
         set word-is-number to false
     end-if
     if  no-form
         if  word-col-from greater than 39
             continue
         else
             set word-is-number to false
         end-if
     end-if
     if  format-length less than numbers-on-line and 20
         continue
     else
         set word-is-number to false
     end-if
*>     cheated here, used Boolean algebra
     if  (inside-parens and number-has-decimals) or
         looks-like-bond-stuff
         set word-is-number to false
     end-if
 end-if.
```
