Thane Hubbell's book *is* a good one (IMO) for experienced programmers.
It does a good job explaining some of the subtleties of COBOL.

[My first programming language was not COBOL - it was FORTRAN back in
the early 60s. My primary programming language is not COBOL today.]

Variable types are intertwined with the intricacies of the MOVE verb.
There are lots of things you might do with functions in another
language that you do with MOVE, including those using REDEFINES.

In COBOL, record layout is very important. You might have seen UNIONs
or EQUIVALENCE in other languages, but the idiom is different. You
don't have FORMAT statements or arguments to PRINTF, instead you use
the PICTURE clause on a data item and MOVE data.

Control structures are different. PERFORM UNTIL is actually a WHILE NOT
() DO .. END DO - the test is before the loop. PERFORM .... THRU ....
is different.

Tables are a limited form of array. Subscripts start at 1. Unless you
have seen the idiom to initialize a table using REDEFINES, you will
scratch your head.

Of course it would be nice if someone wrote a book entitled "COBOL for
Java Programmers" instead of the existing "Java for COBOL Programmers".
