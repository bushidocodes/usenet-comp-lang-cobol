Actually this is not true.  We do not support locking for our indexed
files, other than that your ability to lock a file is wholly dependant
on the file system you are working with or database you are working
with and if that feature is supported.  There is nothing stopping you
from linking in the access library for some database and using their
calls to do locking.  What it is we are working on currently is the
SQL pre-processor so that you don't have to worry about what these
various API's look like and work, your code will always be consistent.
 We expect to release that towards the end of this summer

We have one customer that has ported hundreds of programs off of an HP
3000 to Linux using Kobol and HP Eloquence to replace the Image/SQL
database on the HP 3000 and they are having great success with it,
this is a pretty large multi-user system.

So that is a long way of saying that Kobol will scale as much as you
want to scale with the single caveat of our native indexed files,
which we'll add locking to at some point.
