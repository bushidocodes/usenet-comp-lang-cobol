```
> In Microfocus Cobol could someone explain to me in simple terms what the
> difference is between declaring a variable GLOBAL and EXTERNAL

GLOBAL in an enclosing program makes that variable visible to a nested
program with it.

      PROGRAM-ID.  Outer.
      WORKING-STORAGE SECTION.
      01  Not-Global        PIC X.
      01  Is-Global         PIC X GLOBAL.
      ...
         CALL "Inner"
      ...
         PROGRAM-ID.   Inner.      
             IF ( Not-Global .. )       <<-- error
             IF ( Is-Global .. )        <<--- OK
         END PROGRAM Inner.
      END PROGRAM Outer.

EXTERNAL data may be defined in several _separately_compiled_ (or
nested but you wouldn't) and statically or dynamically linked programs
and these all refer to the same data area.  In other words the
EXTERNAL data items are separate objects from the programs (they are
created externally to the program) which are created by the first
program containing them being loaded and are linked to by each
subsequent program containing that block by name.
```
