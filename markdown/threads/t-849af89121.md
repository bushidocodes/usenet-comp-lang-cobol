related to it have to be commented out.  REMARKS used to be a separate
entity in the IDENTIFICATION DIVISION.  It may be that REMARKS must be
commented out because it is no longer in the standard.

The only time this code has to be changed is on recompilation for
other purposes so you will be going through the processes anyway.

Testing done with this program.
=====
       IDENTIFICATION DIVISION.
       PROGRAM-ID. REMRKS.
       DATE-COMPILED. 08/03/2012.
      *REMARKS.
           COMMENTS
           MORE COMMENTS
           LAST OF COMMENTS.
       PROCEDURE DIVISION.
       BEGIN.
           STOP RUN.
=====

I found this program to be accepted on Micro Focus COBOL 3.2.50
for the IBM dialects OS/VS COBOL, VS COBOL II (all releases),
DOS/VS COBOL, and COBOL/370. The code was also accepted with
ANS85 conformance flagging on.

I suspect, that by making the "REMARKS" paragraph-name a comment,
the 'comment-lines' following it became associated with the
DATE-COMPILED paragraph.

Hope that helps.
