```
IF your output FD looks like....

FD VAR-LENGTH-FILE
  RECORD CONTAINS 10 to 100 CHARACTERS
  RECORDING MODE IS V.
01  THE-RECORD             PIC X(100).

...you can get variable-length records by doing a WRITE THE-RECORD FROM
ws-item-of-desired-length.

But I prefer to use this format:

FD VAR-LENGTH-FILE
  RECORD CONTAINS 10 to 100 CHARACTERS DEPENDING ON WS-VAR-LENGTH
  RECORDING MODE IS V.
01  THE-RECORD             PIC X(100).


MOVE FUNCTION LENGTH(ws-var-item)| numeric-literal TO WS-VAR-LENGTH
WRITE THE-RECORD [FROM ws-item]


MCM

I think there is a "minor error" in the attached.

There is a "RECORD VARYING IN SIZE DEPENDING ON ..." phrase in the '85
Standard.

There is no "RECORD CONTAINS DEPENDING ON" phrase.

Having said that, I do STRONGLY prefer the RECORD VARYING IN SIZE facility to
the older (and less well defined and less portable) "RECORD CONTAINS
integer-1 TO integer-2" syntax.
```
