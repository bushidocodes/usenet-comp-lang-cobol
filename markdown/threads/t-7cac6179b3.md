We have a lot of programs with Exit paragraphs.  It isn't the way I
would program, but they are there nevertheless.

Sometimes I will debug by putting a display in the exit paragraph.  To
compile, the EXIT statement has to be deleted.  But EXIT is a No-OP
statement, the same as CONTINUE. 

Where we have code with empty EXIT paragraphs, we get compiler warnings,
so we generally stick in EXITs.  Or I have been putting in CONTINUEs. 
The latter have the advantage in that we can stick in a DISPLAY without
commenting out the EXIT.  I guess they're both NOPs, but one gives a
compiler check and the other doesn't.

Oh, well.  Here's to the death of GO TO's in COBOL.
```

```
Just so it is clear for those who might follow up on this thread, there is a
world of difference between "exit paragraphs" (part of the current and past
Standards) and the "Exit Paragraph" statement (currently an extension in some
implementation - and will be included in the next Standard).

The former is something like:

  0123-Whatever-Exit.
       Exit.

(which is usually used as the 2nd part of a statement like
   Perform 0123-Whatever thru 0123-Whatever-Exit)

while the latter is something like

0123-Whatever.
    If Something = "A"
        Read something
            At End
               Exit Paragraph
           Not At End
              Perform Process-Rec
        End-Read
   Else
      Perform Not-An-A
   End-If
   Perform Either-A-or-Not-but-not-EOF
     .
02345-Something-Else.
      ....

       ****

Back to the original note, there are "rules" about when/how the "EXIT"
statement (in an "exit paragraph" may/must be coded).  These are QUITE
different from the rules on when/how to use a CONTINUE statement.
    Perform Even-if-no

"an EXIT statement has no ... effect on the compilation or execution."

In a well-formed COBOL program, an EXIT statement does
nothing. There is a technical explanation you might find interesting.

Given:
t100-exit.
    exit.

the compiler will see the tokens,
paragraph-name
period
separator-space
exit-symbol
period
separator-space

The combination: paragraph-name period, will be replaced
by the equivalent of a label in assembler.

The first separator-space may be replaced by code for
tracing execution or debugging, through generally this
separator-space is discarded.

The combination: exit-symbol period, is discarded.

The second separator-space is replaced by code to terminate
the paragraph.

So, quite literally, the EXIT statement does nothing. It has no
effect on compilation or execution. This is why the EXIT statement
is not required by some compilers, except for conformance to
the standard.

The confusion, with the meaning of the EXIT statement, occurs
because many do not see the separator-space following the
EXIT statement and, therefore, associate the end of paragraph
code with the EXIT statement rather than the separator-space
that follows.
------------------
Rick Smith
