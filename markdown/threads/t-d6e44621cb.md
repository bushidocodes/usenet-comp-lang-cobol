The run time library for MS-DOS is COBLIB.DLE.  This must be found on
the 'path'.

Otherwise you can link in the static run time so that it is embedded
in your .EXE with:

    LINK prog,,,LCOBOL+COBAPI;

In larger programs other bits may be needed to be linked in depending
on facilities used (EXTFH, ADIS, intrinsic functions, etc).

> for a DOS client boot diskette to be used in a Windows 2000 migration.

Windows NT (as thus maybe 2000) has multiple 'personalities' including
Windows and DOS, but also including OS/2 1.x.  MS Cobol 4.x and 5.x
are versions of MF Cobol 2.5 and 3.0.  These can run on DOS, Windows
and OS/2.  While MS Cobol 5.0 has crippled the OS/2 capability (for
completely unknown reasons ;-), it may be that when the program is run
it chooses the 'highest' environment available and then look for the
OS/2 run time library - which is not provided.

This is usually overcome in NT by using the FORCEDOS command to run it
and ensuring that the DOS run time is available.
