>1. Is ADABAS considered a 'relational' database? If not, what is it?
Software AG (SAG) claim that Adabas is relational. It certainly can be
treated as such by the programming staff. I believe it is in fact an
inverted list database. Incidentally, when I started out on
Adabas/Natural I had migrated from a DL/I database (hierarchical
database) and treated Adabas in much the same way.

>2. Is there a batch (JCL-run) utility that would create a flatfile from
>ADABAS records, similar to the one that loads/unloads DB2 tables?
There is one utility which I believe comes with Adabas and can create
flat files. This is ADAUNLD (Adabas Unload) and it creates a COMPRESSED
file which will contain a leading record detailing the field names and
formats of the file records. This can then be loaded direct to another
Adabas database (e.g. when copying from LIVE to TEST environments). The
compressed means that null fields (spaces in alphabetic and zeros in
numeric fields) are suppressed, much like a comma separated values flat
file. However, if you then run a DECOMPRESS (ADADECOM) upon the file the
file is stripped of the leading format definition record and all null
fields are padded out to full length with the appropriate fillers.
 
>3. Can ADABAS tables be accessed directly from COBOL (not using
>NATURAL)? (Similar to EXEC SQL for DB2)
Yes, but they are not tables. Originally there was an interface utility
called ADAMINT but this is no longer supported by SAG. However, you may
still have it at your shop and could still write Cobol programs using it
to access your databases. I know that with Adabas 6 installed at one of
my clients' sites the old Cobol programs (dating from something like
Adabas 2 and using Adamint) still run perfectly. However, it is only a
question of time before SAG change something in Adabas which will cause
errors. There is also a product called ADASQL which is similar to the
DB2 SQL. This can be added into Cobol programs in the same way as in
Cobol DB2 programs and will work alongside Adamint in the same program. 

>4. Can SQL be run vs. ADABAS tables?
Only in batch Cobol programs (I do not know about online Cobol
programs). Certainly I am not aware of any capacity to run SQL from TSO
(like SPUFI? on DB2).
