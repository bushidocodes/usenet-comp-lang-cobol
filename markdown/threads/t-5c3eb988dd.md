Fujitsu Cobol is definitely not vaporware.  I was attracted to its
promise of being able to use ActiveX components, and generate COM
compliant objects.  These features either don't work very well or are
not sufficiently documented to figure them out.  I am still waiting for
specific example programs that show how to use MTS.  Several ActiveX
controls I tried generated protection faults, although I got some to
work too.  I never really got the PowerCobol debugger to work consistently
and I'm not sure why.  Emails to Fujitsu took many weeks to be answered.
I like the way it  handles ODBC and SQL but the requirement to have all
host variables as 01 Levels is a killer for me.   Also a killer is the
SQL whenever only allows a "GO TO" and not a "PERFORM".
I couldn't justify its use for deployment of our complex enterprise
application because of these factors.  For small scale stuff, it probably
is Ok.
