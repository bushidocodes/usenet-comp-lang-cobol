```
Give the index structure of the BTRIEVE files, and the BTRIEVE
runtime, you can do it.  There is a chapter in the TOOLS (I think)
manual about using the btrieve file handler instead of the External
file handler.  Years ago I wrote a system to import Great Plains
accounting data into a Purchase order system for payment balancing and
I used this feature.   I wrote a simple program to read the btrieve
file and write my own indexed file.  I used the btrieve runtime as
loaded by the Great Plains software.
```

```
I have used Btrieve with MF COBOL for years in exactly that environment.
There are two methods to do what you want.  One is to define the files
as Indexed files and specify to the compiler to use Btrieve as the Indexed
file handler, which MF COBOL 3.2 can do.  The other, more complex but more
flexible, method is to define the necessary file structures in Working
Storage and call Btrieve directly.

The Indexed method is much quicker and easier, but the direct method lets
you use all the Btrieve capabilities, including those not supported by
the COBOL Indexed file model.
```
