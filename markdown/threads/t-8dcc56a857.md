Be very carefull with the interest related intrinsics. An expert from the
insurance industry detected long ago that the PRESENT VALUE function
defined by the COBOL standard is not the commonly known PRESENT VALUE
function, but (as I recall from many years ago) was actually NET PRESENT
VALUE (or something similar).

I have not heard anything about the ANNUITY function, but the big warning
is that there is no error handling behavior defined for any of the
Intrinsic Functions, so what happens if you feed it something it does not
like is strictly up to the implementor.

Syntax might look like:

01 AN-VAL PIC S9(9)V99 USAGE COMPUTATIONAL.
01 INT-RATE PIC S999V99 USAGE COMPUTATIONAL VALUE 0.10.
01 PERIODS PIC S999 USAGE COMPUTATIONAL VALUE 5.

COMPUTE AN-VAL = FUNCTION ANNUITY ( INT-RATE, PERIODS))

This was tested on IBM COBOL/370 R1.1.1 with LE/370 Run-time 1.5 under MVS
5.2.2

Results were AN-VAL = 0.26
INT-RATE = 0.10
PERIODS = 5

I beleive this is correct for principal = 1.00 (implied in function)
interest rate .1 per period
5 periods
return per period .26
Cutting from the language reference for IBM COBOL/370...

The ANNUITY function returns a numeric value that approximates the ratio
of an annuity paid at the end of each period, for a given number of
periods, at a given interest rate, to an initial value of one. The
number
of periods is specified by argument-2; the rate of interest is specified
by argument-1. For example, if argument-1 is zero and argument-2 is
four,
the value returned is the approximation of the ratio 1 / 4.

The function type is numeric.



FUNCTION ANNUITY (argument-1 argument-2)

argument-1
Must be class numeric. The value of argument-1 must be greater than
or equal to zero.

argument-2
Must be a positive integer.

When the value of argument-1 is zero, the value returned by the function
is the approximation of:

1 / ARGUMENT-2

When the value of argument-1 is not zero, the value of the function is
the
approximation of:

ARGUMENT-1 / (1 - (1 + ARGUMENT-1) ** (- ARGUMENT-2))
