```
Just thought I'd show you a snippet of some code I am investigating,
written in '74 and still running. No flames please, I'm just the
reporter :). Luckily, we're replacing the system in a few years.

The deal is that for certain programs (edits) there are records on a
semi-permanent file. Each program has a record that tells what paragraph
to use when editing an input record from the transaction file, based on
the 'card code' field of the input record.

This is used as the logic because certain card codes go thru the same
edits, so one paragraph can handle multiple card codes. The Exx-paras
are edit paras. This was a code-saving device I guess - instead of one
para for each card code. Guaranteed to cause problems for one with my
inherent typing abilities.

The search portion is looking at a table that was created from the file
which depicts the card code --> edit para for each program. T-X-CC is
the 'card code' on the transaction record to be editted. Note the
fallthru logic into B20...

SET SORTYPE-INDEX TO 1.
SEARCH CC-REC AT END GO TO B40-CC-NOT-FOUND
WHEN CARDC (SORTYPE-INDEX) EQUAL T-X-CC
SET CARD-CODE-TYPE TO SORTYPE-INDEX
MOVE CC-REC (SORTYPE-INDEX) TO W-SORTYPE.
B20-GO-TO-CARD-EDIT.
GO TO E10-06 E15-01 E15-01 E12-10 E10-06 E25-09
E110-C1 E110-C1 E110-C1 E110-C1 E110-C1 E110-C1
E110-C1 E15-01 E35-FA E40-FB E45-FC E50-FD
E55-FE E60-FF E65-FG E65-FG E70-FJ E75-FK
E60-FF E80-J6 E80-J6 E80-J6 E15-01 E15-01
E95-SB E85-ZZ E20-04 E20-04 E20-04 E25-09
E30-AA E130-30 E90-SA E100-SC E105-SD E106-FM
E107-PM E108-PN
DEPENDING ON CARD-CODE-TYPE.

Enjoy..... John
```
