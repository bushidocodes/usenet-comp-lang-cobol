- **From:** "Bob Stacklin" <robstack@columbus.rr.com>

- **Message-ID:** `<SQ0z2.955$_2.5060307@storm.twcol.com>`

```
   At my company, we are experiencing problems related to Oracle 7.3.4
compilation of COBOL programs.
  Programs compiled using the Oracle 7.3.4 libraries are completing with a
non zero return codes.  This is not occurring with programs compiled using
the Oracle 7.3.2 libraries.  I have isolated the differences to the
RETURN-CODE after the call of the "SQLBEX" module (below).  The programs
were executed using an Oracle 7.3.4 database, HP-UX 10.20, Pro*Cobol
11.30(Microfocus 4.05).

                DISPLAY 'CAT=', RETURN-CODE UPON CONSOLE
                MOVE 0 TO SQL-SQLETY

                CALL "SQLBEX" USING
                    SQLCTX
                    SQLEXD
                    SQLFPN
                DISPLAY 'CTX=', RETURN-CODE UPON CONSOLE

                IF SQLCODE IN SQLCA IS LESS THAN 0
                    THEN PERFORM 900-ABEND END-IF.
                DISPLAY 'SQLCODE=', SQLCODE IN SQLCA UPON CONSOLE.
                DISPLAY 'RET=', RETURN-CODE UPON CONSOLE.

     Executed using  Oracle 7.3.4 database
     7.3.2 Compiled Program:          7.3.4 Compiled program:
     BEFORE=+0001                      BEFORE=+0001
     AFTER=+0000                         AFTER=+5028
     SQLCODE=+000000000          SQLCODE=+000000000
     $echo $?                                 $echo $?
     0                                            196

    We have contacted Oracle , but have received no resolution.  Any
assistance would be appreciated.
Thanks,
Bob Stacklin
```
