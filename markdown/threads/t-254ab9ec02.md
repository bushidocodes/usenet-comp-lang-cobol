The first thing you do is check LINE-COUNT >=  LINES-ON-PAGE which is
true since LINE-COUNT is +99 and LINE-ON-PAGE is +55.
Then you PERFORM 200-PRINT-HEADING-LINES
which tries to WRITE PRINT-AREA AFTER ADVANCING 1 LINES.
But you haven't OPENed your files yet so you get an error trying to write to
and un-opened file. OPEN the files first.

Hello again Jim,

Since you're just beginning your pgming education and most
classes/instructors teach their students how to code, but not how to
write a pgm (a pet peeve of mine), I submit the following. It's a
simple outline of a report pgm with control breaks. This is just one
of a number of effective approaches.

You'll notice the dirth of "IF" stmts and code that obscures the
"business logic". The logic is easily adaptable to any pgm processing
any number of control breaks; just add or subtract processing blocks
(1xxx-... pgraphs) and adjust the number of "IF" stmts in the
7010-SET-BREAK-SWITCHES pgraph.

Also note that the pgraphs defining the "loop logic" appear first.
This is generally the information if 1st importance to the maintainer.
Once (s)he is familiar w/this they proceed to the processing block(s)
of interest. Notice how the logic "localizes" this processing.

Initialization, a one time process is relegated to the "back of the
book". I usually group I/O functions to the 7xxx- pgraphs; init to
8xxx- and pgm exits to 9xxx-.
  
Also note that this approach is not limited to report writing. 

Hope this gives you a little insight into the pgming beyond the
coding.

Regards, Jack.   

An input file contains detail records for products sold to a company's
customers through its network of retail stores. Print a report showing
products purchased by customers in the various stores in the network.
Print totals by product, customer and store.

The program logic is this:

To process the file process all the stores in it.    
To process a store process all the customers in it.
To process a customer process all the products in it.
To process a product process all the detail recs in it.

And remember that when you enter pgraph 1000-... the record has
already been read. You'll notice also that the init read facilitates
handling the empty input file situation, which usually (but not
always) is considered an error condition.

Some say correctly that "read into" adds overhead to the task. I say
it's acceptable if it simplifies the code and reduces the chance of an
abend or two.
 
P.S. Some of the pnames may be too long; they were used to illustrate
the logic.

**************
0000-MAINLINE.
**************
    PERFORM 7000-READ-INPUT-REC
    IF INPUT-EOF
       MOVE 'NO ACTIVITY FOR THIS PERIOD'
       PERFORM 8000-NO-ACTIVITY-RPT
       STOP RUN
    END-IF  
    PERFORM 1000-PROCESS-STORES-IN-FILE
      UNTIL END-OF-FILE 
     .
****************************
1000-PROCESS-STORES-IN-FILE.
****************************
    PERFORM 2000-STORE-HEADER
    SET RESET-STORE-SW TO TRUE
    PERFORM 1100-PROCESS-ALL-CUSTS-IN-STORE 
      UNTIL STORE-IS-FINISHED
    PERFORM 2100-STORE-FOOTER 
    .
********************************
1100-PROCESS-ALL-CUSTS-IN-STORE.
********************************
    PERFORM 3000-CUSTOMER-HEADER
    SET RESET-CUST-SW TO TRUE
    PERFORM 1200-PROCESS-ALL-PRODUCTS-FOR-CUST 
      UNTIL CUST-IS-FINISHED
    PERFORM 3100-CUSTOMER-FOOTER 
    .
***********************************
1200-PROCESS-ALL-PRODUCTS-FOR-CUST.
***********************************
    PERFORM 4000-PRODUCT-HEADER
    SET RESET-PROD-SW TO TRUE
    PERFORM 1300-PROCESS-ALL-DETAIL-FOR-PRODUCT 
      UNTIL PROD-IS-FINISHED
    PERFORM 4100-PRODUCT-FOOTER
    . 
************************************
1300-PROCESS-ALL-DETAIL-FOR-PRODUCT.
************************************
    Move/total data, etc.            
    PERFORM 7000-READ-INPUT-REC
    .
2000-STORE-HEADER.
    ---
    MOVE INPUT-STORE TO HEADER-STORE.
    ---
2100-STORE-FOOTER.
    . 
3000-CUSTOMER-HEADER.
    ---
    MOVE INPUT-CUSTOMER TO HEADER-CUSTOMER.
    ---
3100-CUSTOMER-FOOTER.
    . 
4000-PRODUCT-HEADER.
    ---
    MOVE INPUT-PRODUCT TO HEADER-PRODUCT.
    ---
4100-PRODUCT-FOOTER.
    . 
********************
7000-READ-INPUT-REC.
********************
    MOVE CURR-KEY TO PREV-KEY
    READ INPUT-FILE INTO CURR-INPUT-REC
      AT END
         MOVE ALL X'FF' TO CURR-INPUT-REC
         SET INPUT-EOF  TO TRUE
    END-READ
    PERFORM 8010-SET-BREAK-SWITCHES
    .
************************
7010-SET-BREAK-SWITCHES.
************************
    IF CURR-STORE-ID NOT      = PREV-STORE-ID
       SET STORE-IS-FINISHED TO TRUE
    END-IF
    IF CURR-CUST-ID NOT       = PREV-CUST-ID
       SET CUST-IS-FINISHED  TO TRUE
    END-IF
    IF CURR-PROD-ID NOT       = PREV-PROD-ID
       SET PROD-IS-FINISHED  TO TRUE
    END-IF
    .
