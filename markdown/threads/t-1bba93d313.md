```
In our factory we have many applications developed with the old
MicroFocus Cobol compiler v1.0.30.
A few months ago we have buyed the last version of the compiler
(Workbench 3.2) and we pay for it 6500$.
Unfortunately, the big problem we have found is not to be allowed
(or we not to be able) to use the old database (still used for many
application) with the new compiler because it shows the error
9/41 - corrupt index file.
This error is caused by the changement of the idx file format.
Neither with the OLDINDEX directive we cannot work the problem out.
Unfortunately, for compatibility's reasons with the programs actually
in use, we can't convert the Isam database from the old version to the
new one, otherwise the old applications would not be able to work.

Do you know any method or solution to solve this problem ?

Thanks in advance. Best Regards. Ugo.
E-Mail: uz··.@at··t.it
```

```
Ugo,
We had a very similar problem in our shop, and decided to solve
it the simplest way instead of a more sophisticated way.
We wrote a very simple program using the old compiler, to read
in the idx file and write it out as a sequential file (same record
format). Then we wrote an equally simple program using the new
compiler, to read the sequential file and write it out as an idx
file.
```
