This may be of no help as it's quite a while since I worked on Oracle and I
don't have a manual handy, but I think most dialects of SQL require a
"using" with EXECUTE.
DB2 has NO "using" if EXECUTE IMMEDIATE is used, but it does if EXECUTE is
used.

Long shot but try:

>exec sql
>   exec addrupd using
>     :variable
>end-exec.

EXECUTE is usually used with dynamic SQL to action a PREPARED statement.
ORACLE stretches the point to include not just a dynamic SQL statement but a
series of statements (stored procedure). It may therefore be necessary to
put the procedure name in quotes ...?

Do let us know what the solution is when you get there.

Pete.
