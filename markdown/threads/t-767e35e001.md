Hi Wolf,

If you are interested in developing Win32 GUIs, Micro Focus might give you
the answer in Net Express.

Net Express includes Dialog System, a GUI screen painter that offers almost
the same functionality like the one you describe. The user paints the GUI
interface (which includes all the GUI components and statements that handle
their behaviour) and stores the screen(s) in a separate file (called the
screenset file).

In order to work with this file, the cobol code uses a call statement
passing (and getting) all the data described in the Dialog System screenset.
So a cobol program using Dialog System includes a statement like this :
    call dialog-system using control-block, data-block

control-block, contains variables that are used to "tell" Dialog System what
to do (load a new screenset, check for errors etc) and data-block are the
data that the user enters or gets back from the application.

Bottomline, no accept/display is used and a call statement will call Dialog
System to perform any interface handling.

There is  also a character version of Dialog System, so you can design and
deploy applications in character mode (Windows and Unix).

hope this helps

regards

Panos
