```
Micro Focus has a "nifty" tool for doing this - called "WFL".  What it does,
is let you convert character data from ASCII to EBCDIC or vice versa - but
doesn't change COMP-n data (binary, packed-decimal).  Other PC or UNIX
vendors may have similar utilities.   The bottom-line is that you need
SOMETHING (utility, program, whatever) that can be "feed" your record layout
and convert character data but not binary or packed data.

The alternative is to have your program (on the original platform) write out
everything as DISPLAY.  For *all* numeric data you should use

    USAGE DISPLAY SIGN IS LEADING SEPARATE

Then if you use the CODE-SET clause of the FD, you can create "totally
portable" files.
```
