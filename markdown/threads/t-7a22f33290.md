I think that you have one of the easiest FS=39 cases to solve.  From the
information that you gave us, I think you PROBABLY have a Select/Assign
clause with something like:

 Select abc Assign XYZ
    Organization Sequential
     File status WS-field.

Change this to:

 Select abc Assign XYZ
   Organization Indexed
    Access Sequential
   Record Key FD-Key
    File Status WS-field

You can then process this indexed file sequentially - without lying to it as
to what its organization is.
```

The two remaining "most logical" causes for FS=39  are:

1) The record layout under your FD does NOT really match the record layout
(as far as length, fixed/variable, key location, etc) from the way the file
was originally created.

2) You are pointing to a "part" of the file and not the full file - however,
you relate your external file-name (in the "ASSIGN" clause) to the physical
file on disk.

  ***

Final thought.  If all you really want is to convert an indexed file to a
(record or line) sequential file, you should look in the Micro Focus product
that you have for a utility called "WFL".  This will do it for you.
```
