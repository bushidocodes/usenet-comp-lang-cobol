If you expect to use IDMS on the converted code, then I would "comment" all 
the COPY IDMS statments before running the source thru CCCA.  Then uncomment 
it before compileing with Enterpir COBOL (probably with an IDMS 
preprocessor).  If you do NOT expect to use IDMS with the output source 
code, then you will probably need to figure out what "conersion" tools are 
available for changing the IDMS source code - and do that before using CCCA.

NOTE:
   CCCA is "sensitive" to the LANGLVL compiler option which impacts the COPY 
statement.  Make cerain you know what LANGLVL value is used with the OS/VS 
COBOL compiler before using CCCA.  This won't help eiyh your COPY IDMS 
statements, but issomething to be aware of.

I know what it means.    But the only way to change it is to run it
through the IDMS pre-compiler.

IDMS COPY statements don't use standard COBOL COPY libraries, but use
the IDMS data dictionary.    

Here's an example of an IDMS command after compiling:

036500*    OBTAIN CALC IAARCAA                                     
036600*        ON DB-REC-NOT-FOUND                                 
                 MOVE 3 TO DML-SEQUENCE                            
                 CALL 'IDMS' USING SUBSCHEMA-CTRL                  
                         IDBMSCOM (32)                             
                         SR1456                                    
                         IDBMSCOM (43)                             
                 IF NOT DB-REC-NOT-FOUND PERFORM IDMS-STATUS;      
                 ELSE                                              
036700             DISPLAY 'NO IAARCAA FOUND FOR "'                
036800                           FINDER-REC(1:9) '"'               
036900         END-IF.                                             

The sequence numbers are in the source code, showing you what it
looked like.    The comments were added by the pre-compiler when it
replaced those two lines with the code that does not contain sequence
numbers.

One thing that this means is that I cannot have IDMS commands in lines
conditionally commented out with a "D" in column 7.    That's because
the pre-compiler doesn't parse this out, and the regular compiler
doesn't understand the IDMS command.

Depending on your precompiler options the following command may save
displaying the precompiler output:

000100 *NODMLIST                 
000200  IDENTIFICATION DIVISION. 
000300   

Alternatively to show it:
000100 *DMLIST                 
000200  IDENTIFICATION DIVISION. 
000300                           

Otherwise, only the precompiler errors show up.
