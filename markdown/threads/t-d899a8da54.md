OpenCOBOL with Fedora GNU/Linux generates the small library of

----errn.c-----

#include <errno.h>

int get_errno() {
    return errno;
}

int set_errno(int en) {
    int oen;
    oen = errno;
    errno = en;
    return oen;
}
--------

using

----terminal session----
[btiffin@home cobol]$ cobc -v -m errn.c
Command line:   cobc -v -m errn.c
Executing:      gcc -std=gnu99 -I/usr/local/include -pipe -Wno-unused
                -fsigned-char -fno-gcse -Wno-pointer-sign -fomit-frame-
pointer
                -fno-asynchronous-unwind-tables -U_FORTIFY_SOURCE -
shared
                -fPIC -DPIC -Wl,--export-dynamic -o "errn.so" "errn.c"
                -L/usr/local/lib -lcob -lm -lgmp -lncursesw -ldb -ldl
Return status:  0
--------

The critical bits being -fPIC, -DPIC, -shared which influence the link
phase.  PIC for Position Independent Code (and -D defined so the
assembler phase can be smart about the PIC as well). -shared to inform
the toolchain that shared object output is expected. --export-dynamic
informs the linker to include all symbols which can make using dlopen
more featureful.

The output of the above is errn.so that can be linked to by anything
that plays nice with the C application binary interface.

On *nix system not running gcc, have a look to the man pages for the
local cc, then ar and ranlib.  If you do have gcc, you might have GNU
autotools in which case you want to look into libtool.  libtool is
crafted for each supported platform to make static, shared and
convenience archives, umm, conveniently.

Or, get OpenCOBOL on the target platform and use its handy compiler
options to generate dynamic shared objects with relative ease.  In the
above, I just has to use  cobc with -v (to get verbose output of the
commands used in the toolchain) and -m for create a module please.
cobc does the right thing when asked to compile .c files, and does the
right thing on the platform for -x executables, -m modules and even -b
which combines all input files into a single module.

Under MF Cobol, I can create a .so file for the cprog.c file :
cob -Z cprog.c -o cprog.so

And then compile and link the cbl file with the .so :
cob -x cobprog.cbl cprog.so

If I still want to use ing org gnt files, I can create a new runtime
linked to the so file :
cob -xo rtsnew cprog.so

compile the cobol program :
cob cobprog.cbl
and then use :
./rtsnew cobprog
