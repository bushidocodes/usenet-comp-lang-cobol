```

I'm a Windows contractor (Visual Basic / C++ / SQL Server ) working on
an established Microfocus COBOL site. The new project will use ODBC
from a VB app to a COBOL data source, using Microfocus' "Correlate"
product, supposedly an ODBC driver which connects Microfocus COBOL
tables to a Windows client.


Any advice ? Anything at all ? I know _nothing_ about COBOL and any
advice is welcome.

Anyone used this product before ? Did it work ? How good is it ?

How good a relational interface does it provide, given that ODBC uses
SQL and the back end tables are far from relational ? Can I do JOINs,
and are there any restrictions on how sophisticated these are ?

Does the SQL dialect resemble standard ODBC SQL, and are the various
ODBC escape clauses recognised by the driver ?

Can I use a CREATE TABLE query ? UPDATE ? INSERT ? SELECT INTO ?
..or just simple SELECTs ?

What's the best way to handle date/time types ?

Can I use "stored procedures", or the local equivalent ?

Can I bind a VB Data Control to such an ODBC data source, and can I
make it editable ?


One problem we've seen so far is that the Windows '95 clients can't
talk to an NT 3.51 server. Our delivered system will be NT throughout,
but the development machines are all '95. Has anyone else encountered
this ?


Thanks,
```
