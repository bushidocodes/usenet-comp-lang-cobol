### Microfocus File Share

- **From:** "greg" <ua-author-13260@usenetarchives.gap>
- **Date:** 1998-06-09T20:00:01+00:00
- **Newsgroups:** comp.lang.cobol
- **Message-ID:** `<6ln2un$1nn$1@sparky.wolfe.net>`

```

We are currently using Microfocus 16-bit object cobol with the File Share
NLM for Novell Netware on a Netware 4.1 server.

We want to move to a Windows NT server and compile our programs using
NetExpress. We would like to use File Share with the new server, but using
it with CCITCP protocol is too slow. Using it with CCINETB protocol is also
too slow. My question is, can we run our applications, compiled in
NetExpress, and using our NT server, without running the files through File
Share? Our programs run very fast accessing files on the NT box when we
don't run the files through File Share.

When we first came up on the Novell server, we tried running 16-bit versions
of our applications without the FileShare NLM and we got corrupted indexes,
so we are uncomfortable trying to run NetExpress-compiled programs on the NT
box without FileShare. Some of our programs are pretty i-o intensive.


Any opinions? Can we run our NetExpress-compiled programs and access files
on an NT server without File Share and keep our data file integrity?
```
