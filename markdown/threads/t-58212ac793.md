```
Hey there,

  I am trying to get  an FTP control to work in a PowerCOBOL
application,
and actually got one to work in Development mode, and all is good, but
when I try to install on a client PC, the control doesnt work, and after

talking with the control author (www.mabry.com's Ftp/X), they said I
should talk to you about using Licensed Controls and how you handle
it...

Any thoughts or ideas?

Here is the relevent info from the FAQ they pointed me too...

Q2: Why do I get a nag screen saying the .LIC file cannot be found when
I
use a registered control on my customer's machine?

A2: When a control loads, it can detect whether it is in a development
mode or run-time mode. Some environments such as MS Access, MS Internet
Explorer, MS Outlook, and possibly VisualObjects, are both run-time and
development environments. In these cases, a licensed control will
require
the .LIC file to be present or the control will not load.

Since licensed controls require .LIC files when in development mode, but

most controls (including Mabry controls) do not allow distribution of
the
.LIC file, most of these environments have some technology to handle
this
scenerio. For example, Microsoft developed LPK technology for IE and
Outlook which allows the .LPK file to be created from the .LIC file and
create a run-time version of the .LIC.

You will probably need to contact the developer of the environment or
you
might try posting a question in a newsgroup for the environment asking
how
to use a licensed control in that environment.


RonO
```
