that is *larger* than the Block size that you have specified.  I don't think

found this (yet) in the dox.

ISN'T a spanned record).  If it is a spanned record, then change the BLOCK
CONTAINS to the actual blocksize.

```
In "older" (pre-85 Standard) compilers the "normal" way of defining variable
length records was using the
  Record contains N to M characters

With the '85 standard, there is now a
  Record varying in size from N to M depending on WS-item

If you have a single 01-level and use that second method, then the ANSI
Standard is *required* to treat it as a "variable length record format" -
however, the first may - but need not be treated as variable length if there
is only one 01-level or all 01-levels are the same length.
```

within the Macro, which in so many words, can't deal with a signed-field
