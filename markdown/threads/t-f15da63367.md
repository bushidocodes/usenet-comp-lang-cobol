Written 10 years ago! Not exactly random as it stands but pretty simple and you can easily
change the seed to be based on time of day. You can modify WA-NUM to request output ranging
from 0 to a specified number. Basically each time round the RA loop wWZ-NUM is the next in
the sequence - I think (haven't looked for ages I used to use this code to benchmark PC's.)

Please mention my name if you use this as it is a simplified version of something
I now use for encryption which I may want to make public at some point.

IDENTIFICATION DIVISION. DWR00010
PROGRAM-ID. DWRANDBM. DWR00020
AUTHOR. DAIVD C WHITEHEAD DWR00025
ENVIRONMENT DIVISION. DWR00030
INPUT-OUTPUT SECTION. DWR00040
FILE-CONTROL. DWR00050
SELECT RAFILE ASSIGN TO W-FNAME DWR00060
ORGANIZATION IS INDEXED DWR00070
ACCESS DYNAMIC DWR00080
RECORD KEY RA-KEY. DWR00090
DATA DIVISION. DWR00100
FILE SECTION. DWR00110
FD RAFILE DWR00120
LABEL RECORDS STANDARD. DWR00130
01 RA-REC. DWR00140
03 RA-KEY PIC 9(3). DWR00150
03 RA-CT PIC 9(6). DWR00160
03 RA-ZERO PIC 9(6). DWR00170
03 FILLER PIC X. DWR00180
WORKING-STORAGE SECTION. DWR00190
01 RA-WORK. DWR00200
* wa-num is the max rand. no. generated (ie no of records)
* wc-num is the seed.
03 WA-NUM PIC 9(6) VALUE 50. DWR00210
03 WB-NUM PIC 9(5)V9(10) VALUE 0. DWR00220
03 WC-NUM PIC 9(5)V9(10) VALUE 1.23. DWR00230
03 WD-NUM REDEFINES WC-NUM. DWR00240
05 FILLER PIC X(5). DWR00250
05 WX-NUM PIC 9(10). DWR00260
03 WY-NUM PIC 9(10). DWR00270
03 WZ-NUM PIC 9(3). DWR00280
03 WW-CT PIC 9(6) OCCURS 1000. DWR00300
* w-test is the max no of accesses for any a record
01 W-TEST PIC 999 VALUE 20. DWR00310
01 W-FNAME.
03 W-DRIVE PIC X.
03 W-NAME PIC X(12) VALUE ":RAFILE.DAT".
01 W-TIME PIC 9(8). DWR00320
PROCEDURE DIVISION. DWR00330
A-1. DWR00340
MOVE 1 TO WZ-NUM. DWR00350
A-2. DWR00360
MOVE 0 TO WW-CT (WZ-NUM). DWR00370
IF WZ-NUM < 999 DWR00380
ADD 1 TO WZ-NUM DWR00390
GO TO A-2. DWR00400
AA-START. DWR00430
DISPLAY SPACE AT 0101.
DISPLAY "ENTER DRIVE " AT 0101.
ACCEPT W-DRIVE AT 0113.
ACCEPT W-TIME FROM TIME. DWR00410
DISPLAY W-TIME AT 0201. DWR00420
OPEN I-O RAFILE. DWR00460
AA-LOOP. DWR00470
PERFORM BA-UPDATE. DWR00480
GO TO AA-LOOP. DWR00490
BA-UPDATE. DWR00500
PERFORM RA-RAND. DWR00510
MOVE WZ-NUM TO RA-KEY. DWR00520
READ RAFILE INVALID KEY DWR00530
PERFORM CA-CREATE-REC DWR00540
GO TO BA-UPDATE. DWR00550
IF RA-CT NOT = WW-CT (WZ-NUM) DWR00560
GO TO EA-ERROR. DWR00570
ADD 1 TO RA-CT. DWR00580
MOVE RA-CT TO WW-CT (WZ-NUM). DWR00590
* DISPLAY RA-KEY, " ", RA-CT UPON CONSOLE DWR00600
REWRITE RA-REC INVALID KEY DWR00610
GO TO EA-ERROR. DWR00620
IF RA-CT > W-TEST DWR00630
DISPLAY "TEST OK" UPON CONSOLE DWR00640
ACCEPT W-TIME FROM TIME DWR00650
DISPLAY W-TIME AT 0301 DWR00660
CLOSE RAFILE DWR00670
STOP RUN. DWR00680
CA-CREATE-REC. DWR00690
MOVE WZ-NUM TO RA-KEY. DWR00700
MOVE 0 TO RA-CT, RA-ZERO, WW-CT (WZ-NUM). DWR00710
WRITE RA-REC INVALID KEY DWR00720
GO TO EA-ERROR. DWR00730
EA-ERROR. DWR00740
DISPLAY "TEST ERROR", WZ-NUM, RA-CT UPON CONSOLE. DWR00750
RA-RAND. DWR00760
ADD WC-NUM TO WB-NUM. DWR00770
MULTIPLY WB-NUM BY WB-NUM GIVING WC-NUM. DWR00780
DIVIDE 1 BY WC-NUM GIVING WC-NUM. DWR00790
DIVIDE WX-NUM BY WA-NUM GIVING WY-NUM. DWR00800
MULTIPLY WA-NUM BY WY-NUM. DWR00810
SUBTRACT WY-NUM FROM WX-NUM GIVING WZ-NUM. DWR00820
RA-EXIT. DWR00840
EXIT. DWR00850

›
› Does anyone have a sample of code to perform random number generation in a

Here is something I used some time ago in a laboratory environment:

* Random Number Work Store
01 RANDOM-NOS-AREA.
03 RNDNUM PIC 9V9(09).
03 RNDNUMI REDEFINES RNDNUM PIC 9(10).
03 RND-CUR PIC S9(10) VALUE ZERO.
03 RND-I1 PIC 9(02) VALUE ZERO.
03 RND-I2 PIC 9(02) VALUE ZERO.
03 RND-I3 PIC 9(02) VALUE ZERO.
03 RND-NX PIC X(550) VALUE ALL "0".
03 FILLER REDEFINES RND-NX.
05 RND-N OCCURS 55 PIC 9(10).
03 RND-PRI PIC 9(10) VALUE ZERO.
03 RND-SEEDX.
05 RND-SEED-TIME PIC X(06).
05 RND-SEED-DATE PIC X(06).
03 FILLER REDEFINES RND-SEEDX.
05 RND-SEED PIC 9(09).
05 FILLER PIC X(03).
03 RND-WA PIC 9(04).
03 RND-WB PIC 9(02).



/ Random Number Generation - translation of RPG program in S/36
* Power Tools, which implements a Donald Knuth algorithm
Z200-RANDOM-NO SECTION.
Z200-0.
IF RND-I1 = ZERO
PERFORM Z201-RANDOM-1.
ADD 1 TO RND-I1.
IF RND-I1 > 55
PERFORM Z202-RANDOM-2
MOVE 1 TO RND-I1.
MOVE RND-N(RND-I1) TO RNDNUMI.
Z200-X.
EXIT.

* Initialize Array
Z201-RANDOM-1 SECTION.
Z201-0.
MOVE 55 TO RND-I1.
ACCEPT RND-SEED-TIME FROM TIME.
ACCEPT RND-SEED-DATE FROM DATE.
MOVE RND-SEED TO RND-N(55) RND-PRI.
MOVE 1 TO RND-CUR RND-I2.
Z201-1.
COMPUTE RND-WA = 21 * RND-I2.
DIVIDE RND-WA BY 55 GIVING RND-WB REMAINDER RND-I3.
MOVE RND-CUR TO RND-N(RND-I3).
SUBTRACT RND-CUR FROM RND-PRI GIVING RND-CUR.
IF RND-CUR < ZERO
ADD 1000000000 TO RND-CUR.
MOVE RND-N(RND-I3) TO RND-PRI.
ADD 1 TO RND-I2.
IF RND-I2 NOT > 54
GO TO Z201-1.
PERFORM Z202-RANDOM-2 3 TIMES.
Z201-X.
EXIT.

* Determine next set of 55
Z202-RANDOM-2 SECTION.
Z202-0.
MOVE 1 TO RND-I2.
Z202-1.
ADD RND-I2 31 GIVING RND-I3.
SUBTRACT RND-N(RND-I3) FROM RND-N(RND-I2) GIVING RND-CUR.
IF RND-CUR < ZERO
ADD 1000000000 TO RND-CUR.
MOVE RND-CUR TO RND-N(RND-I2).
ADD 1 TO RND-I2.
IF RND-I2 NOT > 24
GO TO Z202-1.
MOVE 25 TO RND-I2.
Z202-2.
SUBTRACT 24 FROM RND-I2 GIVING RND-I3.
SUBTRACT RND-N(RND-I3) FROM RND-N(RND-I2) GIVING RND-CUR.
IF RND-CUR < ZERO
ADD 1000000000 TO RND-CUR.
MOVE RND-CUR TO RND-N(RND-I2).
ADD 1 TO RND-I2.
IF RND-I2 NOT > 55
GO TO Z202-2.
Z202-X.
EXIT.

Kindrick
