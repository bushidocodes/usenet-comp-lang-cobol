wrote a program that generate a report using the report writer, and send the
output directly to the printer port. The length of the report lines is 200
characters, the printer is a Lexmark 2381 plus (a dot matrix printer) and
the program works fine, but if I use the run-time tunable
"printer_redirection", to redirect the output through the Win32 print
spooler using the "generic text only" driver, the program works fine only if

column 149 are LOST. I do not think it is a problem of the printer driver,
because I tryed to print a line made of 200 characters from the NOTEPAD of
Windows and it is all OK.
