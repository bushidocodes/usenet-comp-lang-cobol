>"I have a file created by a program written in C or C++,
>but I don't have access to the source code or even know
>for sure the compiler that generated the executable.
>(How) can I extract the data into something usable?"


Thats like saying I have a pile of fertilizer, can you reassemble it into
the terds which came out of the cow. Then can you further reconstitute it
into the food which the cow ate and tell me where in the world the food was
created and what type of fertilizer was used on it?

If you don't have the source code you are, forgive my French, you are
fucked.

There are several formats in COBOL which are valid across format types, for
instance any sequence of bytes can be trnsformed into a valid number. If
formats can be interchangeable you would have to do some serious context
checking to tell the format and even then you won't be 100% right.

You will never know the correct format until you look at the COBOL layouts
of each file. If you can't get them, then I suggest you take a
leave-of-absence for a couple months and hope the problem goes away by the
time you get back. Sick relatives are always good excuses for
leave-of-absences.
