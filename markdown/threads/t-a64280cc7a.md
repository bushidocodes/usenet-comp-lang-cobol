### MF Cobol/hpux record unlock problem

- **From:** "Rick Price" <rick@ramtecjr.demon.co.uk>
- **Date:** 1998-10-09T00:00:00
- **Newsgroups:** comp.lang.cobol
- **Message-ID:** `<907953548.12438.0.nnrp-07.d4e45f36@news.demon.co.uk>`

```
Does anyone use MF cobol to process index files > 1gig on hp-ux?  We have a
file this size (in a bench mark test) which is opened for shared use.

If we update records near the end of the file, they do not get unlocked.
Records earlier on the file get unlocked normally.  We don't get any error
conditions.

The manual says that for windows etc, MF Cobol can handle files of 2gig, but
only 1 gig if shared.  However for unix it just says that it can handle 2gig
files.

I don't know if this is a bug in MF, in hp-ux or in the manual.  However I
do know this is not a bug in our code.

This bench mark is being run at a user site.  I'm waiting to get this file
back to test it on our machines under different versions of the os but in
the meantime I thought I might get some feed back.

We're using mf cobol 4.0.5 on hp-ux 10.20.

tia
Rick
```
