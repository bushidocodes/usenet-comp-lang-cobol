I've actually done some small scale AP & AR work in Lotus, and found
that one problem is handling transactions when you have a changing

somewhat messy if you don't want to set up a fixed sized table with

maintaining it.

spreadsheet solution vs a typical COBOL read transaction file and
update master file program, if not for the details, at least for the
flavor.
