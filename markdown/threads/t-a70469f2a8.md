I knew about ALTER since the days of DOD COBOL 60...but the only time I
actually used it was in the mid-70s on an online banking system that ran on
Interdata minis.

And what we did was legit. For a whole bunch of reasons we were driving a lot
of dissimilar devices over a single phone line, so we'd ALTER the code to poll
each type. Really it was just setting a switch...and once Interdata gave us the
code to OS32, we took the ALTER out of the COBOL and stuck it into the PASLA
(Async line adapter) driver at the OS level instead:)

Ah...the good ole days...multitasking OS that worked on a 32K machine, source
to the OS, AND source for the COBOL compiler itself:) Yep...we rolled our own
extensions to COBOL...like a working SCREEN SECTION long before most compilers
had one.

David

In the earlier days of MVS virtual memory systems (S370-158, 1970's), we
used segmentation frequently. By using sections in the source, Csects
were generated, and you would then use the linkage editor to page align
and order the Csects in your load module for efficiency.
