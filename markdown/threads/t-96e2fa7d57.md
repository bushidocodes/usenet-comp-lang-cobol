I do remember there was a Datamation article that compared Cobol and C
in terms of 'new development' and 'maintenance'.  The survey found
that most Cobol work was 'maintenance' and most C was 'new
development.

Some clueless IT managers used this to 'prove' that if they converted
all their Cobol programs to C then they would not need to do all the
maintenance and would save money and be able to develop new systems.

Of course, in the survey, the C systems were completely different from
those in Cobol.  In DP 'maintenance' includes changes due to business
changes, while in applications in C 'maintenance' is bug fixing.

Whether you write in C or Cobol depends on what type of program is
required.  If you want to write a word processor or a graphics
manipulation program then C/C++ probably makes more sense than Cobol.
