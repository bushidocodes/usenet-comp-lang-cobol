The FACTORY data and method definitions precede the OBJECT data and
method definitions in a class program. The entire FACTORY definition
is optional for a class

Each class has one and only one factory (called the factory object).
In C++ they call this the class object I believe. X3J4 found this
term a little confusing. The factory object is used primarily during
creation of objects of a given class - hence the term FACTORY.

Factory methods are often used to initialize new members of a class or
to store or retrieve data that is global to all members of a class.

For example let us assume a class of bank accounts. You may have
methods "GetNextAvailable AccountNumber" or
"GetAverageBalanceofAccounts" ot "GetTotalBalanceAllAccounts"
These are methods associated with the class but ones you would not
want associated with any particular object of that class. Nor would
want the data fields in the class definition of each object.

Nigel,

I can't speak for OO COBOL, but in OO analysis a FACTORY is generally an
object that is used to create or modify other objects. As you know, an
object is the union of data and operations on that data. A FACTORY
object will generally provide a copy of the requested object, with or
without its data values being initialized.

Tom Blow
