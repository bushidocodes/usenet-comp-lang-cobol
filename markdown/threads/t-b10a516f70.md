```
The problem is that not all newsreaders are eight-bit-clean, not all of
those that are use the same encoding, and not all that do are currently
using a character set that includes u-umlaut.  Using characters outside
ASCII will cause problems for some readers.  Whether that's a concern
is something the author must decide.

> Traditionally, unlaut-u is alt-129. Alt-252 is superscript n. 

Under newer versions of Windows (I believe any derived from NT, ie with
Unicode in the base OS and UCS-2 APIs), Alt-0252 is different from Alt-
252.  The leading 0 tells the Windows console subsystem to produce the
character with that value from ISO Latin-1 (which mostly corresponds to
the same code points in UCS-2).  Without the leading 0, values above
127 produce the corresponding character from the "traditional" (for
Microsoft) MS- DOS "Extended ASCII" character set, I think, though it
might be those code points from the current Windows code page or
something like that.  (I never use Alt-nnn these days.)

This is useful for producing many diacriticals if you don't use them
frequently.  Of course, it's limited by what's in those first 256 bytes
of UCS-2; I know I need Z-caron and z-caron a lot more frequently than
lowercase thorn, though others may have different requirements.
```
