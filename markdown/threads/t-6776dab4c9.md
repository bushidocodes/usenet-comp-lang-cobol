It sounds like you will have a lot of rewriting to do if you wish to compile
your source in the year 2000 with a supported COBOL compiler!

The only VSE COBOL compiler supported after 1999-12-31 is the COBOL for VSE
compiler. It follows the '85 standard and has the optional intrinsic functions.
It's interpretation of statements is close to LANGLVL(2).

There is no equivalent LANGLVL(1) option.

There is no WRITE / AFTER POSITIONING clause. Such WRITE statements will have
to be rewritten with AFTER ADVANCING. If you have a new programmer doing the
conversion, make sure that person knows that there is a big difference between
AFTER POSITIONING 0 (which advances to the next page) and AFTER POSITIONING
char-field, where char-field holds '0', which is double-spacing.

TRANSFORM also does not exist.

The SELECT / ASSIGN TO clauses are interpreted differently; close enough that
it usually works unchanged, but different enough that we have been tripped up
by it.

If you do any checking of FILE-STATUS, the meanings have changed. (There is a
9x code that means successful open but implied VERIFY has been done; usually a
successful OPEN would return a FILE-STATUS of 00.)

I had expected our rewriting of our in-house programs to go very quickly. It
hasn't. I am glad we are doing the rewriting now instead of under pressure in
January 2000 when we expect a number of failures to occur. (Our work now is an
attempt to _minimize_ our Y2k-related failures, but we are realistic enough to
realize that we are unlikely to find and fix all Y2k exposures correctly.)

Mark A. Young
