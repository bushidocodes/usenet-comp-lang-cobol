```
Possible, though that seems like a fairly tame expression for a
content filter to reject.  I doubt you're sitting behind a filter
that rejects invalid poster email addresses, or you'd hardly see
anything.

I suspect it's more likely that some node in your downstream path
doesn't like Robert's messages for some other reason, possibly
because his NNTP posting host (69.115.155.63) uses dynamically-
assigned DNS records.  (It currently has working A and PTR records,
which I'd think would be enough, but maybe there's a host on the
route that wants the NNTP-Posting-Host header to have a FQDN rather
than a dotted-decimal address.  Also, I notice that the current PTR
record for that address doesn't match the name in the X-Trace
header - some hop may not like that.)

Hmm... I checked the Path for one of Robert's messages, and it
contains "167.206.3.103.MISMATCH" as one of the hops.  It's been a
while since I poked around in the NNTP specs, so I couldn't say
offhand what that "MISMATCH" means, but it does look suspect.  Maybe
a hop along your feed has a heuristic that rejects messages with
"MISMATCH" hops as possible forgeries.
```
