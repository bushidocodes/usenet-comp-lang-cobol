hi everybody!

I found a tool at: oak.oakland.edu
this is SIMTEL ftp server, in subdir: SimTel/win3/visbasic/vbdll15d.zip

This tool simulates a windows DLL with a VB EXE.
After defining this VBX control, you can call functions with parameters
too from C or Visual Basic. I tried with MS and MF COBOL and this works
well!
VBDLL is a shareware program without runtime fee. So you can forget
the very expensive GUI tools with runtime fee.
I recruited some old DOS COBOL program and I rewrite them for WIN.
I had to change only display and accept commands and compile like
a WIN batch program or QUICK WIN program for debugging.
In the VB I had to make only the GUI screens and the called functions
definitions with VBDLL. This is very simple and very fast.
You don't need to know what does the COBOL program, you can concentrate
only the GUI.
