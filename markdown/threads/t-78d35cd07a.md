It depends on your printer. You will need to make sure you have
things like emulation modes, character sets, font sizes, etc. all set
correctly. That said, Acucobol allows you to enter string values in
hexadecimal using X or H:
DISPLAY X"414243" would output "ABC", as would
DISPLAY X'414243' or
DISPLAY H"414243" or
DISPLAY H'414243'

(See section 2.1.2.2 'Nonumeric Literals' in the Acucobol Reference Manual)

So, to send an escape value, lookup the hex codes you need in your printer's
manual (most will list control codes in hex & decimal) and send them to the
printer. It'd probably be easiest to do something like
77 escape-code PIC X VALUE H"1B".
and STRING it into your output buffer. Of course, nothing prevents you from
doing something like this, either:
STRING "Barcode: " delimited by size
X"1B" delimited by size
other-codes delimited by size
into output-buffer

Now, as far as creating barcodes goes, I'm sure there are some documents a
web search would turn up but to get started, you could try converting the
BASIC program at http://www.snafu.de:80/~pbsound/sourcen/barcode.bas into
COBOL.
