Just a thought - it might be as easy to write your own scheduler, if
it is to control your own system jobs.  I wrote one in a day:  a
simple screen giving the operator a list of the 'end of day' jobs,
with the option to tick or not, and to enter some parameters.  The
COBOL program then works through in sequence, calling each job,
waiting for return, continuing on to the next job, etc.  The jobs
could be made to wait for successful completion of the previous job,

If all you're using is NT class machines (NT, 2000 and XP) then you
have better choices. The command processor (cmd.exe) is far superior
to the Win 9x version (command.com). There is a lot of stuff, like
complex for loops using files as input, if... else.. processing,
partial string manipulations, prompt for values etc.

However, if you want to build an industrial strength set of scripts
for job control consider Windows Scripting Host (part of 2000 and up
and available free for NT). This allows you to build console scripts
using Jscript, VBScript and optionally, perl (free) and rexx (I don't
know about free). This gives you amazing control and power.
