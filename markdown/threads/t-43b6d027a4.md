### VS COBOL II R4.0 support ending "soon"

- **From:** "William M. Klein" <wmklein@ix.netcom.com>
- **Date:** 2001-01-24T10:55:57-06:00
- **Newsgroups:** comp.lang.cobol
- **Message-ID:** `<94n1ei$7mj$1@slb7.atl.mindspring.net>`

```
Given the recent question (in comp.lang.cobol) about what are the
differences between VS COBOL II R4.0 and  "current" COBOL (IBM COBOL for
this-and-that), I just thought that I would send out a reminder that
"support" for VS COBOL II R4.0 ends soon (i.e. the end of March - as I
recall).

IBM *still* documents support for existing (or even new) VS COBOL II
compiled programs (object code) running with a supported level of LE (either
in the load-module if NORES - or in the Linklist/Steplib/whatever if using
RES) - but this does mean that any problems with the compiler itself will no
longer be subject to standard "support".

If your shop is still using VS COBOL II (any release - but only R4.0 is
supported now), please make certain that all your "support" people
understand this upcoming change.  Hopefully, you have a migration plan in
effect and are "almost done".  If not, I suggest that you plan one as soon
as possible.
```
