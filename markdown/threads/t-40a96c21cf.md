```
We are currently using 16-bit MF COBOL with their FileShare NLM on a Netware
4.1 server. We want to migrate to using NetExpress with an NT server. Our
bench-marking has found that NetExpress with FileShare on an NT box is
slower than the FileShare NLM on the Netware box. We tried CCITCP and
CCINETB and could not get the performance out of our applications that we
want, despite the fact that the new NT box is a much more powerful machine
than what we're running Netware on. We are considering using the
NetExpress-compiled programs on the NT box without FileShare, but we are
concerned about data integrity. Due we dare try running our applications,
some of which are very i-o intensive, without FileShare? When we tried
running our applications on the Netware box without using FileShare, we
ended up with corrupt indexes. Our applications are screamers on the NT box
when we run them without FileShare. Any thoughts on this?
```
