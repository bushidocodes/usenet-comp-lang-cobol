All languages on all platforms have what is called a run-time
environment, partly to save the poor application programer from _having_
to deal with stuff like the arrow keys. Under DEC VAX VMS, the arrow
keys are already bound to the command line editor no matter which
language you use.

If you read raw ASCII characters into a CoBOL program, your program has
to know all combinations of escape sequences from all kinds of terminals.
A better way is to use whatever screen facility is provided by the
platform. That way the operating system takes care of the terminal
dependencies.

Some platforms have screen-handling built right into the CoBOL language.
Tandem Screen CoBOL is nice, and many CoBOLs have screen features built
into the DISPLAY statement. VMS takes a different approach; instead of
enhancing each language individually, it makes the same facilities
available in all languages, ï¿½ la curses.

You can find everything you need to spice up your CoBOL programs in the
VMS Run-Time Library Screen Management (SMG$) manual.
