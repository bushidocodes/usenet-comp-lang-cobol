If your program is on an IBM OS390 / z/OS operating system and the JCL
specifies the use of batch local shared resources (SUBSYS=BLSR) and it
fails on the first read, change the JCL to not use BLSR.  This was a
disconnect between the COBOL compiler people and the access method
people because OPEN of a file for sequential read should issue an
implied start.
