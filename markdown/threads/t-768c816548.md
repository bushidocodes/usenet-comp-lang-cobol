I use OLE automation to send the document to MS WORD. It can then be print
previewed, printed, FAXed, e-mailed, or whatever the user wants, with NO
effort on my part and without re-inventing the wheel...the system has a Word
Processor, why duplicate this functionality in COBOL?

Use the same facility to do renewal letters for Membership. Wrote a couple
of MS WORD macros, set up a template for monthly Membership renewals and
another for Yearly Membership renewals. COBOL program uses SQL to select
Members who are due for renewal within User selected period and load them to
an ACCESS table which is defined as a Data source to MS WORD Mail Merge. OLE
Automation is invoked to start WORD and run the Mail Merge... User can
easily amend text of templates or even individual letters, preview, FAX,
e-mail etc. as above. All letters end up on the WORD print queue and print
in the background while the system is doing something else...

Works for me...<G> (And the users LOVE it...)

All of the above is easily "doable" from PowerCOBOL Version 5.

Pete.
