```
Hello,

I have this strange problem compiling COBOL programs with OCDS 4.1.30 on my
RedHat 6.1 Linux system (both are supposed to support shared libraries):

-------------------------------- Makefile

ALL: COBFUNC.so CALLER

COBFUNC.so: COBFUNC.cbl
 cob -Q -shared -x -o $@ $<

CALLER: CALLER.cbl
 cob -x -o $@ $< COBFUNC.so

-------------------------------- CALLER.cbl

       PROGRAM-ID. CALLER.
       DATA DIVISION.
       WORKING-STORAGE SECTION.
       01  X PIC 9(9) VALUE 123456789.
       PROCEDURE DIVISION.
      *     ACCEPT X FROM COMMAND-LINE.
           DISPLAY "CMD=" X.
           CALL "COBFUNC".
           DISPLAY "Done.".
           STOP RUN.

-------------------------------- COBFUNC.cbl

       PROGRAM-ID. COBFUNC.
       PROCEDURE DIVISION.
           DISPLAY "in COBFUNC".
           EXIT PROGRAM.


When I run 'make', both programs compile/link flawlessly, and the CALLER
program runs fine.

However, if I uncomment the 'ACCEPT ...' statement in CALLER.cbl, 'make'
issues the

following warnings:

ld: warning: type and size of dynamic symbol `_mF4701' are not defined
ld: warning: type and size of dynamic symbol `_mF3331' are not defined
ld: warning: type and size of dynamic symbol `_mF4701' are not defined
ld: warning: type and size of dynamic symbol `_mF3331' are not defined

and when I try to run the CALLER program:

Execution error : file 'CALLER'
error code: 115, pc=0, call=1, seg=0
115     Unexpected signal (Signal 4)

The problem disappears if I modify "COBFUNC" to make it call another
subprogram "COBFUNC2":

-------------------------------- COBFUNC.cbl (second version)

       PROGRAM-ID. COBFUNC.
       PROCEDURE DIVISION.
           DISPLAY "in COBFUNC".
           CALL "COBFUNC2".
           EXIT PROGRAM.

-------------------------------- COBFUNC2.cbl

       PROGRAM-ID. COBFUNC2.
       PROCEDURE DIVISION.
           DISPLAY "in COBFUNC2".
           EXIT PROGRAM.

-------------------------------- Makefile (updated)

ALL: COBFUNC2.so COBFUNC.so CALLER

COBFUNC2.so: COBFUNC2.cbl
 cob -Q -shared -x -o $@ $<

COBFUNC.so: COBFUNC.cbl
 cob -Q -shared -x -o $@ $< COBFUNC2.so

CALLER: CALLER.cbl
 cob -x -o $@ $< COBFUNC.so



Anyone understand what happens here?

TIA,

Jacques Vidal
```
