Hi there,

ftp versions I use allow a batch mode, ftp -n -s:client.ftp
(see man ftp for more info on these flags)

where client.ftp is a file listing the commands ftp needs to carry out, for
example:

open client
user guest
guestpasswd            # This line contains the password for the user on the
previous line
bin
cd somewhere
prompt
put myfiles.tar
quit

To execute the command from COBOL, depends on what COBOL system you are
using. For Microfocus COBOL you can use the X"91" routine.

For example (a fragment of a program)

  01 result pic x comp-x.
  01 func pic x comp-x value 35.
  01 cmd-param-len pic x comp-x value 1.
  01 cmd-line pic x(80).
  ...
  call-os.
      move "ftp -n -s:client.ftp" to cmd-line
      display cmd-line upon command-line
      move 0 to cmd-param-len
      call X"91" using result func cmd-param-len
      if result not = zero
          display "COMMAND FAILED " cmd-line
          stop run
      end-if
      .

or

CALL "CBL_EXEC_RUN_UNIT" USING ... (see docs for this one)

or

CALL "system" using null-terminated-command-line (UNIX only, uses standard
library system call)

If you are using another COBOL product, please tell us and someone familuar
with that product I'm sure will help.

Regards,
Steve.

Create a script to hide the specific implementation.  Here is one that
relies on a netrc to provide signon to the other computer.  the set -x
is debugging.  This way the directory on the client machine is easy to
modify.  note 'ibmlaser' is a machine name and can be an IP address. 
The netrc is documented in man pages and really handy.

Note the bin transfers binary files (no translation) ascii translates
and is expands linefeeds on unix to carriage return linefeeds in PC
land.

------
#!/bin/ksh
#  ftp Australia Combined files to the IBM Laser box

# set -x

FILE=$1
ftp ibmlaser << !!
cd /var/psf/incoming
bin
put $FILE
quit
!!
---------

The other alternative to FTP is 'rcp' if the other side is a unix
box.  possibly easier.

Ken
