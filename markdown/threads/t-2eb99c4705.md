```
I'm running a RM-Cobol (5.05) program under SCO Unix (3.2.5.05).  The 
program returns an i/o error 30,19 when opening an indexed file. 
RM-Cobol error 30 indicates a hardware problem where the following
number, 19 in this case, is to be located in the errno.h file.  errno.h
shows 19 to be ENODEV "no such device".  This same file is opened by
other RM-Cobol programs with no problems and the program that returns
the error runs successfully when using a different set of files.  This
  may be related to the deletion of several thousand records from the 
file   since the last successful execution of the program although I can't
figure out how.

I can't come up with a reason why I'm receiving this error.
```

```
Bert,

I doubt it has anything to do with record deletions.  Since you are using
the same program successfully "when using a different set of files" I would
suspect a flaw in whatever mechanism you are using to map the COBOL filename
to a physical pathname for this particular file.  Are the other programs
that have no problems using the identical mechanism.

Try cloning the code and then stripping it down so that only the problem
file access remains.  Then plug it into the *identical* environment and run
with debug.  That may lead you to a solution...

Liant Software Corporation
```
