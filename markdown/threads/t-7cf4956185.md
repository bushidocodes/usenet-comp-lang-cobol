On 4 Aug 1999 04:57:06 GMT, X@yz.com (X) wrote:

>Anyone have problems with their COBOL programs when upgrading to CICS 4.1?  
>Specifically with 4.1, is storage really not initialized to low values? 

Yes, that's the way it is, and that's the way it has been documented in the Application
Programmer's reference Manual.  In Version 1 and 2 of CICS the COBOL working storage was
initialized by the CICS GETMAIN functions, but this was never documented, and should never
have been relied upon.



> We 
>have problems with SEND MAPs when the symbolic map is loaded with junk and 
>there is no explicit code to move low values to the symbolic map before the 
>SEND.  Is this just a set up issue of the CICS environment or do we have to 
>modify our programs? 

 You can do something about it, but it is not a CICS setup thingie.  Depending on which
compilers and/or languages you use, let you MVS sysprog people check on WSCLEAR (COBOL II
runtime options) or STORAGE (LE/370) runtime options

> I'm not a CICS administrator, but I can hardly believe 
>that IBM would make such a change of questionable benefit, but  with serious 
>negative consequences.
Sorry, don't blame iBM for this one, blame the programmers that did not code according to
the rules.
