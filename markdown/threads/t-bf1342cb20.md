```
I just wanted to correct the erroneous impression that might have occurred from 
a previous post.

The draft of the next Standard does not allow a "variable group item" to be 
passed (via a CALL statement) if and only if "format 1 or 2" of the CALL 
statement is used.

It *does* allow it to be passed in a Format 3 ((program-prototype) CALL 
statement.  I don't recall the discussion on this, but especially in the case of 
inter-language CALL and "new" CALL statements, I can imagine that the intent was 
to "encourage" the use of call proto-types with inter-language calls and also to 
insure that the receiving item was "prepared" to receive a variable length group 
item.
```
