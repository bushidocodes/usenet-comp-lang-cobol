I don't know, but here are some ideas:

Ask the user if there are any related software diskettes.

The .NX file is probably an index to the data file .XD so the .NX
file can be rebuilt later. The .XD file is the important one.

Look through both files (with a suitable utility) for interesting
text strings like (c) for Copyright that might indicate the
software manufacturer.

Try to find the record length in the data file by looking for a
repeating pattern; if hex 0D 0A recurs it indicates end of record.

Try to work out the offset (if any) at the start of the files.
The first part of a file might contain field names and types,
database identity and version number etc.

Bring a summary of your findings back here and we can try again!
