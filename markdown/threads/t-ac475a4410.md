I am trying to change the value of HIGH-VALUE (& LOW-VALUE) by
specifying a program collating sequence alphabet with literals.
Example:
PROGRAM COLLATING SEQUENCE IS FUN.
SPECIAL-NAMES.
ALPHABET FUN IS "a" THROUGH "z".
...

DISPLAY "high-value is " FUNCTION ORD(high-value).
DISPLAY "low-value is " FUNCTION ORD(low-value).

I always seem to get 256 & 0, respectively.
Does anyone know how to code an example that shows HIGH-VALUE changing?

Also,
The ANSI verbage on this stuff also says that more than one
character can have the highest position in the program collating
sequence. In this case, the last character specified is associated with
HIGH-VALUE. Can someone show me a example of there being multiple
characters having the highest position?

Marc:

›› I am trying to change the value of HIGH-VALUE (& LOW-VALUE) by
specifying a program collating sequence alphabet with literals. <<

The problem with your example is that ORD(char) returns the ordinal
position of the character within the PROGRAM COLLATING SEQUENCE.
Therefore, by definition, ORD(LOW-VALUE) should return 1 (not zero as
you indicated you were getting) in any correct implementation of COBOL
ORD(HIGH-VALUE) would normally return 256 for a character-set that has
256 characters, but this could be lower if the ALSO phrase is used in
the definition of the alphabet-name used for the PROGRAM COLLATING
SEQUENCE since the ALSO phrase assigns multiple characters to the same
collating position. To code an example showing changes to LOW-VALUE and
HIGH-VALUE, you need to look at the actual character value assigned.
The easiest way to do this would be to use reference modification to
move the figurative constant to the correct byte of a binary value and
then print the binary value. That is, on an implementation that uses
big-endian storage of binary values and assigns two bytes for a PIC
9(4), use

01 BIN-LOW-VALUE PIC 9(4) BINARY.
01 BIN-HIGH-VALUE PIC 9(4) BINARY.
01 DSP-LOW-VALUE PIC 9(4) DISPLAY.
01 DSP-HIGH-VALUE PIC 9(4) DISPLAY.

MOVE ZERO TO BIN-LOW-VALUE, BIN-HIGH-VALUE.
MOVE LOW-VALUE TO BIN-LOW-VALUE(2:1).
MOVE HIGH-VALUE TO BIN-HIGH-VALUE(2:1).
MOVE BIN-LOW-VALUE TO DSP-LOW-VALUE.
MOVE BIN-HIGH-VALUE TO DSP-HIGH-VALUE.
DISPLAY "Low-Value = " DSP-LOW-VALUE.
DISPLAY "High-Value = " DSP-HIGH-VALUE.

›› Can someone show me a example of there being multiple characters
having the highest position? <<

To assign multiple characters to the highest position, one must assign positions
to all characters in the native character set; otherwise, the highest position
will be occupied by the one highest native character not given a position. If
all characters are assigned specific positions in the collating sequence, and
the ALSO phrase is used on the last two or more characters, then multiple
characters will have the same highest collating sequence. For example:

PROGRAM COLLATING SEQUENCE IS FUN.

ALPHABET FUN IS
1 THRU 253, 254 ALSO 255 ALSO 256.
