System:

It is a Dos based AR Inventory system written totally in MicroFocus

Convert ISAM file sytem to a DB system that is more compatable to

These are the main questions for now. As they do no seem to be

Any database that supports ODBC keeps your options open. There's no
mention of DB volumetrics so which brand, host, etc. cannot be
answered. Based on your suggestions and ease of implementation in
COBOL, I'd be looking at XDB (XDB Systems Inc.), DB2 (IBM) or MS SQL
Server or any other database that is supported by Micro Focus' SQL
preprocessor. Otherwise you have to write the ODBC interface in COBOL
(samples available at http://www.microfocus.com 'white papers' link).
›
› 2. According to the manuals for this version of MF COBOL they do not
› support any other type of DB file structure. IS there a later version
› that will support other DB formats.

In general, COBOL compilers do not directly support relational databases
(except IBM COBOL/400 but that's an unusual implementation). SQL
database support is an add-on product available from Micro Focus and RDB
vendors. MF COBOL is up to version 3.4 (16-bit version). There are big
differences between v3.4 and v2.4.

›
› 3. This sytem is for DOS. I would like to be able to have parts in
› Windows also. IS there a MF COBOL version that could allow me to do
› that.

MFCOBOL 3.2.50 and up. There may be some problems with 3.4 depending on
the maintenance release.
