Thane Hubbell wrote:

> Quick Questin for you Terry - why?
>
> I found Realia's sort to be one of the fastest around!  The only thing
> I found to be faster is Fujitsu PowerBsort or something from Germany
> called "Sort Solution".

The quick answer is that it has the ability to sort delimited files by
field number.  I have a program that lets users create queries from
standard Realia indexed files and output comma delimited files with
dynamic content for use in programs like Excel.  Opt-Tech's sort also has
summing capabilites for other types of files that I hope will be made
available for delimited files (if they heed my enhancement request).

As of the day before yesterday, I was successful in getting it to work.  I
did have to get a new Link.exe and a dependent DLL from Computer
Associates though.

We already have an application that uses Opt-Tech's sort with batch
processing so it didn't cost us anything to take advantage of it in this
program.

Terry
