JCL cannot handle this. Most shops have a utility program "somewhere" which
takes a PARM and writes it to a disk file which can be passed to the next
job step. JCL _does_ expand the symbolics when passed as a parm.

EXEC PGM =PARMWRIT,PARM='%TSTADDJB &IMEM &OMEM 'TYPE1ERR' &FLG &DBUG '
OUTF  DD    DSN=&&PARM,DISP=(NEW,PASS)

EXEC ERRORLG
...
>>//SYSTIN DD DSN=*.OUTF,DISP=(OLD,PASS)
