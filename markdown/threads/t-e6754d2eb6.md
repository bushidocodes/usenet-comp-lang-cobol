This is all of the top of my head, so please bear with me....

It depends on what COBOL comiler you're running. Example:

If you're running RM/COBOL-85 on a NetWare 3.12 server you would need to do
the following:

On the server, START BTRIEVE. This has some parameters, but for testing,
the defaults are fine.

On the workstation (DOS or Windows) you would need to load BUTIL with a
C=Y' parameter (to create btrieve files) from DOS and then in your
application window, load RBADAPTR.

BUTIL is the netware client btrieve handler.
RBADAPTR is the RM/COBOL interface to NetWare.

>From this point on in your session's window you will be able to create and
read btrieve files. One thing to note, though, is ALL INDEXED FILES will
be created as btrieve. If this isn't what you desire, you should create
them ahead of time and then load BUTIL C=N on your workstations.

There are also utilities available to create btrieve files from COBOL FDs.
Smithware makes a pretty good one that's windows based.
