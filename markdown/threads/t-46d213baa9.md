### MF Cobol - disappearing file

- **From:** "dave cromarty" <ua-author-17087110@usenetarchives.gap>
- **Date:** 1996-04-22T20:00:01+00:00
- **Newsgroups:** comp.lang.cobol
- **Message-ID:** `<830251234snz@imago.demon.co.uk>`

```
We have encountered a problem using MF Cobol 3.1, compiled to
intermediate code, running under an OSF/1 (Digital Unix) 3.2. The problem
is that a file is occasionally set to zero length, as though it had been
opened for output, although it hasn't actually been opened for output.
The file is the last one to be opened and is possibly infringing some
limit.

We have encountered a similar situation before in a different Unix dialect
and solved it by increasing the following Unix parameters :

Maximum number of open files (system wide)
Maximum number of open files (per user)
Maximum number of locks

Unfortunately the system managers are unable to locate these parameters
within their Digital Unix documentation. So (finally!) the questions -

Has anyone else encountered a similar problem? If so, how did you solve
it? Do you know whether Digital Unix has these parameters and if so, what
are their names?

Eternal gratitude is absolutely guaranteed for any assistance :-) Post or
email, whichever you like.

TIA

Dave

are my own unless otherwise attributed.
```
