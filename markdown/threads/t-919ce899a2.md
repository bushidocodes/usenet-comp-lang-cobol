Seeking wisdom...

We have a large COBOL application that we are currently porting to
Windows/NT 4.0.

The architecture is:

* Application programs are run under Tuxedo 6.3
* The controlling functions are written in Visual C++ 5.0
* The C++ calls COBOL application code, which handles the processing
of business rules.

The problem:

When the application program is shutdown, it crashes in the COBOL
runtime libraries. Running in the debugger reveals at stack trace of
recursive calls to routines in the DLL's SQLSVR32, MFRTS32, SQLAKW32.

If we shutdown down the process prior to it entering the SQL routines,
it shuts down fine.

What have we done wrong and/or what can we change to fix this problem?

All input appreciated.
