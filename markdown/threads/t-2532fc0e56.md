```

How do you strip leading zeroes from a field so that the first significant
digit will occupy the first position. So that a pic x(4) like 0043 becomes
43bb (b for space). PIC Z4 will strip them but leave blanks, same thing with
INSPECT REPLACING.
I need to put the stripped field into a STRING command with a delimited by
space or size option so that the output will be like '43' not '0043' or
'43 '.

I am transferring data from a DB2 table on a mainframe to an ORACLE table on a
SUN UNIX server. I will select the data from DB2, output it to dataset that
will consist of 131,000 SQL update statements, and then FTP it to the UNIX
server to update the ORACLE table. The records contain a PIC X(10) field
i.e. 0000000001, 0000013245, 0003467234 that needs to be put into a STRING
command.

One possibly is to use a bunch of IF statements testing out each position for
0 and moving only nonzero digits. Like IF field2(1:1) = 0 and field2(1:2) = 0
.. field2(9:1) =0 then move field(10:1) to out2(1:1)
IF field2(2:1) = 0 etc then string and delimit by space.

However I will end up with close to 100 statements for just one field.

Does anybody have a simpler more elegant solution?
```

```
You've already gotten some excellent advice on possible solutions.

Being a COBOL/VB person, my first reaction would be to write a multi-purpose
function to solve the problem. In COBOL terms, that would be a sub-program
with three parms: input-value, output-value, and output-length. Both the
input-value and output-value fields would be something large, like X(40).
The output-length would be S9(05) COMP-3. I'd then REDEFINE each of the
numeric items coming in from the DB2 table that you want to 'strip', as PIC
X in WORKING-STORAGE. For each of these input fields, I'd move it to the
input-value parm (which COBOL would automatically pad to the right with
spaces), move spaces and zeros to the output-value and output-length parms,
call the sub-program, then use the statement OUTPUT-VALUE (1:OUTPUT-LENGTH)
to build the appropriate SQL statement that updates the Oracle table (don't
forget the intervening spaces and terminating semi-colon!). The sub-program
would parse the input-value parm two ways. First, going left-to-right to
remove all leading zeros and/or spaces, putting the results in the
output-value parm. Then, it'll determine the value of the output-length
parm, by parsing the output-value parm from right-to-left, looking for the
first non-space byte. To make the sub-program even more useful, I'd include
a fourth parm to indicate whether leading zeros, leading spaces, both, or
neither, should be removed from the input-value parm (in some cases you may
only want to know the size of the data element without the trailing spaces).

Hope this helps! :D
```

```
None of the advice so far is incorrect, but being a die-hard COBOL coder, I
prefer to use the COBOL functions.

INSPECT and UNSTRING are two of the most powerful string functions in any
language. For your problem:

01 STRING-FIELD.
03 NUMERIC-FIELD PIC 9(9).
01 RESULT-STRING PIC X(9).

INSPECT STRING-FIELD REPLACING LEADING '0' WITH ' '.
UNSTRING STRING-FIELD DELIMITED BY ' ' INTO RESULT-STRING.

The INSPECT as you know will replace all leading 0s with a space.
The UNSTRING will take all spaces as delimiters and break the source
string into approprate result strings (this is also wonderful for name
manipulations).

If you are used to manipulating strings using MID$ and INSTR, you should
really read up on STRING and UNSTRING, they are much more powerful for
business apps than any of the BASIC string functions.
```

```
I had to do something like that recently.

The code is at work but this is at home. I am doing this from memory.

05 WS-P1 COMP PIC S9(4).
05 WS-L1 COMP PIC S9(4).

05 THE-X5-FIELD PIC X(5).

* Find start of significant (non-zero) part of the field -> WS-P1.
MOVE 1 TO WS-P1.
INSPECT THE-X5-FIELD TALLYING WS-P1 FOR LEADING '0'.
* If all zeroes, make sure we have at least one 0 to move.
IF WS-P1 > 5
MOVE 5 TO WS-P1
END-IF.
* Determine length of the significant part of the field -> WS-L1.
SUBTRACT WS-P1 FROM 6 GIVING WS-L1.

Now one can use "reference modification" to do useful things, including:

MOVE SPACES TO receiving-field.
STRING
...
THE-X5-FIELD(WS-P1:WS-L1) DELIMITED BY SIZE
...
INTO receiving-field
END-STRING.

The above should be available on any compiler written to the '85 standard and
later. (I have used code like the above in COBOL for VSE.)
```

```
To follow up on this part of the thread (related to JUST), As Don said, JUST
LEFT doesn't exist - but is the default behavior of an alphanumeric MOVE.
What this means is that you could combine the techniques talked about
earlier in the thread with a simple move to get a "left justified" field. In
other words, if you have already gotten the start position of the first
non-zero digit (via an INSPECT or IF/EVALUATE checking) and have placed this
value in the field "start-field", then the following will left justify the
number with no leading zeros or spaces

Move full-field (start-field:) to alpha-receiving field

This wouldn't really meet the original poster's question - because they
wanted to have no leading OR trailing spaces - and to use "just the non-zero
digits" in a string statement. This is why I originally suggested the
syntax

String .... full-field (start-field:) delimited by size
....

This gets you nothing more, no leading zeros or spaces, no trailing spaces.
```
