The ugly pink sweater book The Orange book is wrote:
›
› Im a hs student getting ready not to hit COBOL like a brick wall in
› college and im hoping to look at source COBOL code so i can see if
› it's like C++ (what i program in now) what is harder COBOL or C++

To answer your last question first, in my opinion C++ is harder.
Having said that, however, you may still find COBOL puzzling just
because it is so different from the language you know now. This is
one of those rare cases where what you know already may actually
get in the way of learning something new rather than help it.

COBOL is very different because...

1. COBOL gets very involved and detailed in I/O operations.
In C and C++ I/O is usually done by calling a library
function, and one could almost say that I/O isn't "really"
part of the language. With COBOL, there are I/O commands:
OPEN, READ, WRITE, REWRITE, CLOSE, etc. And there are file
definition (FD) entries which are set up first to define
more things about the file than you may ever have dreamed
existed. One becomes very familar with the details of the
file management system.

2. COBOL is very heavily into data structures. After all,
it's business and data oriented. One can, in a very
compact manner, say something such as "this field contains
a date in YYYYMMDD character format followed by a four
byte binary field." As near as I can recall:

02 FIELD.
03 DATE-FIELD.
04 YEAR-NUMBER PICTURE 9999.
04 MONTH-NUMBER PICTURE 99.
04 DAY-NUMBER PICTURE 99.
03 NUMBER-FIELD, COMPUTATIONAL, PICTURE S9(9).

3. COBOL provides a lot a capabilies for shaping and editing
printed output, much more extensive that the format strings
in C's "fprint" statement. (Again, you may see this as an
instance of COBOL being a data-oriented language.) For
example, just writing this as part of the description on an
output field:
PICTURE $$,$$$,$$9.99-
says "this field is 14 bytes long, for printing a number,
with leading zeroes suppressed, but always printing at
least one digit to the left of the decimal point even if it
is a zero, with two digits to the right of the decimal point,
with a leading dollar sign floated over to be in front of the
first digit, with commas every three digits to the left
of the decimal point, and with a minus sign put on the back
if the number is negative (but nothing put there if it's
positive.)" This is not a particularly complicated example.

4. COBOL favors and sometimes demands verbosity. (Look that word
up if you don't know it. It could show up on the SAT.) The
original intent was to make COBOL read like English, so
usually there is only one command per line. This means
that those wonderful puzzles of parentheses, pointers,
and type casting that one can get in C rarely happen.

Good luck.
-- David Martin
