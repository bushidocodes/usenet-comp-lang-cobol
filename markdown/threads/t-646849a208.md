The simplest way would be to add 'NOTIFY=&SYSUID' to the jobcard before you
submit the job. Then the user will be interrupted by the message as soon the
job will be done. Set the TSO profile of the user to INTERCOM since with
NOINTERCOM he will not get the message. If you want the user to wait until
the job is done, you might want to set the 'TSO STATUS jobname' command and
parse the answer, wait some seconds, ask the status again, wait .....

Conditional JCL is a mechanism I've used to a similar goal in the past.

IF RC=0 THEN ...
IF RC (> 0 AND RC <=4) THEN ...
IF RC > 4 THEN...

The PROC sat at the end of all my batch work and  'GENERed a command (/*M;F
SUPERSESS,IMBRCST 'message' ) to the session handler (Supersession) so you got a
smart little pop-up whatever session you were working in (this was a one LPAR
site).

You could also use the TSO MSG command but beware. When I last used it, if you got
the destination parameters wrong you ended up broadcasting to all TSO users.
Embarrassing.
