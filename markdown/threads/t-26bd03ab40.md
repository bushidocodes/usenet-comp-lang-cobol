Normally this would tell JCL to pick the best block size, but since you
specified it it must match with the JCL, thus it should be BLOCK CONTAINS 2

I believe the maximum record length for a file is 32759 as you specified,
but it is a variable length record, thus the maximum record length is 32755.

If you specify a block size in JCL it must match the BLOCK contains clause
in COBOL. So your JCL says a block contains 2 records, so your COBOL should
say BLOCK CONTAINS 2 RECORDS. Remember 4 bytes for record length, so this
does not need to be accounted for in COBOL.

I find it better to specify as few record properties as possible in COBOL.
And to always use BLKSIZE=0, in JCL if specified as all.

One thing I have found helpful is to specify as few JCL parameters as
possible. With exception of specifying LRECL & RECFM. I always specify
BLKSIZE=0 in JCL unless the files are designated otherwise. The only
required options when creating a file are DISP and SPACE.

Some of this may seem like symantics, but I have changed thousands of JCL
streams and following some standard proactices really helps. Your shop may
require you to specify certain parameters in certain ways, htis is usually
for a good reason.
