You can follow each call with a cancel, and the program space will be freed
up.  Of course, that also slows down the application as each subroutine must
be re-load each time it is called.

A better approach is to cancel the modules that are rarely used, and leave
the ones that are frequently used in memory.  That takes a bit of tinkering
to get it right, as you could run out of memory, but will result in better
execution times.

A third solution is to convert to an operating system that allows
unrestricted memory useage, and add lots of memory, though that tends to be
*lots* of work.
