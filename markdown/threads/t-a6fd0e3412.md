The traditional COBOL clause for a NO-OP is NEXT SENTENCE, if that was
your question.  Some compilers also allow CONTINUE.  Neither actually
has any effect on the PERFORM statement.  As far as I know, the only way
to exit a PERFORM loop is to incorporate a controlable condition in your
UNTIL clause.

leclaire wrote:

> The traditional COBOL clause for a NO-OP is NEXT SENTENCE, 

That is not true.  NEXT SENTENCE is _NOT_ a no-op.  It is a 'GO TO
statement after next full stop'.  However, it is only usable as a clause
in an IF statement that does not include END-IF and thus cannot be used
in any other structure.

> Some compilers also allow CONTINUE.  

All ANS85 compiler should include CONTINUE which is a no-op.

> Neither actually has any effect on the PERFORM statement.  

NEXT SENTENCE cannot be used inside an in-line PERFORM (unless the
compiler is broken) as it would require the IF to have an explicit scope
terminator which makes the clause unusable.

Instead of 
 PERFORM VARYING INDX FROM 1 BY 1 UNTIL INDX > 1000
     ...
    IF something
       EXIT PERFORM
       END-IF
 END-PEFORM.

Try it this way:
 PERFORM VARYING INDX FROM 1 BY 1
                 UNTIL INDX > 1000
                 OR    something 
     ...

 END-PEFORM.

Note that the END-IF **MUST** be coded, even tho it is NEVER
"executed"!  In a more complex statement, that EXIT PERFORM could kill
ya!  There are very good reasons for the "rule" of one entrance, one
exit in Modular Strustured Programming.  I realize that the
"something" might be very complex, but one other advantage is that you
know, going into the PERFORM, what are its limitations/conditions.

EXIT PERFORM
EXIT PERFORM CYCLE
EXIT PARAGRAPH
EXIT SECTION
are ALL non-structured syntax, and should be avoided.  I did my best
to keep them out of the new standard, but....
The EXIT PERFORM CYCLE can usually be avoided by careful construction
of the conditions inside the PERFORM.
For example:

PERFORM etc.
     statement 1
     IF CONDITION-1
          EXIT PERFORM CYCLE
     END-IF
     statement 2
     statement 3
END-PERFORM

can be replaced with 

PERFORM etc.
     statement 1
     IF CONDITION-1
          CONTINUE
     ELSE
          statement 2
          statement 3
     END-IF
END-PERFORM

or 
PERFORM etc.
     statement 1
     IF NOT CONDITION-1
         statement 2
         statement 3
     END-IF
END-PERFORM

Personally, I think it is clearer, easier to maintain, and less likely
to cause problems if the PERFORM gets complex enough to split up.

All languages are, to a greater or lesser extent, idiomatic.  Each idiom
is somewhat different from others.  The Cobol idiom for 'return to start
of loop' from a couple of decades ago was:

      PERFORM Do-it-again
          UNTIL
      ....


   Do-It-Again SECTION.
   Do-It-Start.

      ....
      IF ( rest-not-needed )
          GO TO Do-it-Again
      .
      ....

   Do-It-Exit.
      EXIT.

As you can see the 'continue' in other languages functions as a GO TO.

In modern Cobol the idiom may be:

          PERFORM UNTIL ...
              ....
              IF ( rest-needed )
                  ....
              END-IF
          END-PERFORM
