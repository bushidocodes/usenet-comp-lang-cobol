### MF COBOL file reads are slowwww....

- **From:** markd@bellsouth.net (MarkD)
- **Date:** 1998-09-09T00:00:00
- **Newsgroups:** comp.lang.cobol
- **Message-ID:** `<CZAJ1.1996$g04.1581619@news3.mia.bellsouth.net>`

```
My company is using MF COBOL under AIX and I found that reading files
is exceptionally slow.  A brain-dead program that simply opens and
reads a sequential file with 1.8 million 184-byte records takes 15
minutes using MF COBOL (open input/read next) and just 25 *seconds*
using C (fopen/fread).  I ran the MF COBOL in both .gnt and natively
compiled (-x) formats.  The performance difference is the same on
various files - both sequential and indexed.

Is this inherent in MF COBOL's file handler, or do we have a real
problem?  If it's a problem, where should we start to look?

Thanks in advance.
markd@bellsouth.net
```
