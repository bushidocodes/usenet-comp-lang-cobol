I worked on this last night, and ended up using the unload.
(thanks Daniel)



I did not get around to trying the other methods, but thanks for the
info.



The only problem with the unload is that it does not create line
delimeters.



I found a work around this when importing into SQL as every record is
the same length of characters.



I do not know much about cobol, so I copied the file I needed to a
seperate directory before I ran the unload command. (not sure if
nessesary, better safe than sorry)



I also found that UNC paths do not work very well when trying to call a
program so I created a mapping.



The path statement is because vutil32 is not loaded in memory and
resides in S: (\\server\mexx)



Below is the batch file I created, so far so good;



net use S: \\server\mexx

S:

copy /y mkcfil.dat sqlfiles\mkc.dat

path=s:

cd sqlfiles



call vutil32 -udt mkc.dat mkc.txt



Thanks again to everyone for your help



Mufasa2000
