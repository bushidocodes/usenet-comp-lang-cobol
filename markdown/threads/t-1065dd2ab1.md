default DCB, which I am sure can be over-ridden, though I don't know

At my shop, we output our EZT reports to file and then genner them to
Sarview, etc.

Basically, EZT has a default line length (which i don't think is
necessarily 132 - depends on how the shop sets it up). I always
specify a line length on the linesize parm of the Report definition. I
also specify as part of the report parms which file it goes to. You
also have to in the file defintion speciy Printer so that EZT knows to
associate your file defintion with a report.
