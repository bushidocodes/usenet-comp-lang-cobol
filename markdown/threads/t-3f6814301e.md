You cannot sensibly have packed decimal (or COMP-anything) fields in a
LINE SEQUENTIAL file.  This is because in an ASCII file a packed field
could have a value that can be interpreted as a tab or line feed
character that would disrupt the format.

> All lines are the same length, so basically it's a fixed-size
> record sequential file with an EOL field separator.

Then just specify is as sequential and include the EOL in the record
layout.

> I've tried numerous syntaxes but always get an error 007 (syntax error). Is
> it because a LS file is assumed to contain only plain text?

To sort any data: table, line sequential, or any other just use INPUT
and OUTPUT PROCEDUREs and get the data from whereevr necessary and put
it out as required.

Jacques,

I have no OCDS experience, but that shouldn't matter - MFSORT is
supposed to run everywhere, right ?

I use the following statements on Windows to sort a sequential file:


USE   T:\BCMD\TDRELDEV\BSDATA\TDTO050.DAT
RECORD,F,1455 ORG,SQ
GIVE   T:\BCMD\TDRELDEV\BSDATA\TDTO050.DAT
SORT FIELDS=(3,3,CH,A,19,9,CH,A)

Of course you'll have to change record length from 1455 to your length,
file names to your names and substitiue your FIELDS= info - but this
should work for you.

Note:  I use these statements in a TAKE file - see the MFSORT doco
on using TAKE...


Ethan Schultz
