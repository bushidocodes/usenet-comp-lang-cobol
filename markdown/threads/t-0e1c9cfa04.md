Nobody's making any money with phone apps and it is a large amount of work.

an hour... It is a labour of love rather than a living.)

I'll do it with C# if I do, but I think your COM+ solution using COBOL is
excellent.

COM is the obvious way to refactor existing legacy COBOL, in my opinion.
Wrap it as components and you can run it anywhere, including mobile devices.

But it will still require the COBOL RTS, unless it is compiled and linked to
native code, with the RTS support linked in... That's why Java has been so
successful; there is an RTS for it on most devices already...

For years now I have been persuaded of the power of Component Based design
and the Component Object Model just makes sense to me.

The PowerCOBOL Migration software I have just completed uses COM so that
COBOL can run in a C# environment (Windows Forms).

There's a video demonstration at:
http://primacomputing.co.nz/PRIMAMetro/pwrCOBMigration.aspx
