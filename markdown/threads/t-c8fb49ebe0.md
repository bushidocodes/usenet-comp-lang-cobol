Charles is the president of the company that makes it.  I'm
not affiliated in any way.  PERCobol IS an excellent
product, and combined with the new BlueJ it is even better.
I'm using it with SQL, applets, servlets, and applications,
but none of the vendor specific extensions.  The compiler is
great.

There is one caveat though.  JAVA is not quite mature.  If
you're doing it on your own, you're going to learn a lot
about JAVA if you want to take advantage of all the
features.  Setting up the environment for JAVA development
and run-time and keeping it stable takes some finagling if
you want to use all of the cool features.  JAVA as delivered
is not a complete product no matter who you get it from - at
least not yet.  The best thing about it is that it is
run-time compatible (no recompiling) across most
environments - it uses JAVA routines instead of OS calls. If
you don't use vendor specific extensions, you can distribute
your executables anywhere there is a JAVA virtual machine.
Because the JAVA virtual machines are not yet fully
developed for most environments, it is slower than
traditional compiled programs at run time.
