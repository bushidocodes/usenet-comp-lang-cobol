A less attractive solution than Paddy Coleman's - and depends obviously
on frequency with which you want to do it.

Read your Micro Focus files sequentially and create ascii delimiteds for
Access. Some while since I did it, but it works. (One minor snag, my
end-user had Access V7 - the rich oil companies he was giving the data
to had Access V6 - no likee !)

Currently I use the same concept to transfer old RM/COBOL '74 data files
to Micro Focus NetExpress - RM comp fields are converted to M/F comps,
(using an M/F sub-routine), then output ascii delimiteds to create
NetExpress ISAMs.
