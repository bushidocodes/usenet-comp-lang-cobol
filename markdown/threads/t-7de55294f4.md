Try using a literal here exactly as the VB code does...at least until
you get it working.

       Invoke CRYST_APP_OBJ "OpenReport" using by value
'C:\REPORT1.RPT'


                                              RETURNING CRYST_REP_OBJ.

>      move CRYST_REP_OBJ TO "ReportSource" OF CRViewer1.

Nope. You should avoid moving Object References about. ["ReportSource"
OF CRViewer1] MUST be an OBJECT reference. In which case, make it the
target of the above invoke. Like this...


       Invoke CRYST_APP_OBJ "OpenReport" using by value
'C:\REPORT1.RPT'


                                             RETURNING "ReportSource"
of CRViewer1


> 
>      set CRYST_REP_OBJ to null.
…[5 more quoted lines elided]…
> 

This is because the "ReportSource" of CRViewer1 is not defined as an
Object reference.
