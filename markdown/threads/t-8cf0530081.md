```
All,

I have a program that uses some of the MF Cobol byte_stream_file
subroutines and nothing else. When I compile and link the program
under MF 3.2 using lcobol+os2286, it produces an .EXE file of about
35K. When I compile and link under MF 4.0, using CBLLINK with the -T
option the resulting .EXE file is 355K; 300K bytes larger. I
believe (but I'm not sure) the difference is due to large
Working-Storage areas being included in the EXE file under 4.0 and
being dynamically acquired at run time in the MF 3.2 version since
Working-Storage is about 300K. A second example is a program that
uses ADIS and PANELS. The EXE file went from 135k to over 750k! This
is just not acceptable. I have looked everywhere and cannot find why
this is happening or how to fix it. Can anyone give me some help?

Thanks.
```
