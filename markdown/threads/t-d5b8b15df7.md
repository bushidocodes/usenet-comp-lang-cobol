If you can in all program (output of MF Cobol and input of RM Cobol)
try use the clause "SIGN IS LEADING (or TRAILING) SEPARATE CHARACTER"
so the sign is a separate byte with value "+" or "-" and all Cobol i
know accept this type of fields.

For example :

05 TOTAL-INCOME PIC S9(10).

will be

05 TOTAL-INCOME PIC S9(10) SIGN IS LEADING SEPARATE CHRACTER.

now this field is 11 bytes long and the sign is a separate character
before the digits with the value "+" or "-".
