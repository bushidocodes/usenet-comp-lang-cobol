============
RESPONSE
============

Yeah, performance will likely be adversely affected.

Assuming I understand your challenge correctly....

" Execute a query. The result set of this query is one or more rows which
provide parameters for a second query,



which returns exactly zero or one row"

If so, I would create the cursor for the first query, and PREPARE the
statement for the second query. I would not use a second cursor for the
second query.

But if .. the second query can return more than one row, you should rewrite
your SQL so the second query is a subquery of the first query. Come to think
of it, you could rewrite the SQL anyway to get at the second query's result
set using one OPEN of one cursor regardless of the number of rows returned
by either query. (And having thought about it, I would not bother with
PREPARE and EXECUTE).

If you need help doing this, please POST (do not email to me) the two SQL
statements you are using. (A little text explantion or program code would be
helpful, too). Someone here will surely be able to combine those two SQL
statements into one. (Also, I'm not really good at this myself; I just know
it can be done).

MCM
