```
First, you need to be careful when using DYNAM/NODYNAM versus CALL
"literal"/identifier to force "dynamic" calls.  If you use DYNAM, this will
force ALL calls to dynamic - which is not acceptable under CICS, IMS, or DB2
(where "system" calls need to be static).

Next, no you should not need NCAL

Finally, if you are getting an (LE) U4039 ABEND, then it sounds like you are
not running with a "particularly useful" setting of the LE TERMTHDACT
run-time option.

See:
 http://www.s390.ibm.com/bookmgr-cgi/bookmgr.cmd/BOOKS/CEEA3010/2%2e1%2e53

for U4039 and TERMTHDACT

and see

http://www.s390.ibm.com/bookmgr-cgi/bookmgr.cmd/BOOKS/CEEA6010/4%2e2

for the recommended run-time options

P.S.  The bottom-line description of U4039 is that some OTHER error has
already occurred - and wasn't "properly" handled.
```
