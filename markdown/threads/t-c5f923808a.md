```
Easy to do in 1985 Standard COBOL with the intrinsic function extension.

Use Integer of Date to generate a sequential (sometimes called Lillian)
date for the base date, and for the current date. This will allow simple
subtraction to find the elapsed days, allowing correctly for leap and
non-leap years. Multiply by 24 hours per day by 60 minutes per hour and
by 60 seconds per minute. You now have base point to current day in
seconds. Add on the appropriate seconds for today using the current-time
function and a little multiplication and you have the number you want. Of
course to go the other way, arithmetic first, then use date of integer to
reverse the process. All major nastyness is buried in the intrinsic
functions.

Let me assure you much easier to have the compiler do it for you. I have
developed emulators for all 5 date and time related COBOL intrinsics so
that older COBOL's can also benefit from the functionality without having
them in the compiler. By clever copy book manipulation we have been able
to allow interchange of intrinsic and non-intrinsic compilers without
changing the program logic.

Rex Widmer
Builder of software archeology tools and other strange programs to help
survive in a legacy based world.
```
