But keep in mind these messages don't just go to the job JESYSMSG log 
but also to the system log and probably also get displayed on MVS 
consoles.  Don't use this for writing 10,000 lines of debugging output 
(I've seen it done), or you will get Operators and others highly ticked 
off, and possibly even cause other applications to grind to a halt if 
consoles can't keep up and you cause a [hopefully temporary] exhaustion 
of console message buffers.  At our installation, a job that floods the 
console log with messages will be canceled.

Some installations also have a standard that says console messages 
should only be written by a program when they convey information useful 
to Operators for proper execution of the job.  There is already much 
clutter on the system log without adding something that doesn't really 
belong there.  If it's really information that is only relevant to the 
application and not to the Operators, an ordinary SYSOUT data set rather 
than the console would be a better choice.
