This can be done using any existing 8086 disassembler/debugger. (You
wouldn't use a decompiler as the code is NOT compiled, it is assembled.)

Try CODEVIEW which is the standard PC assembler deugging tool.

If you REALLY want to have an ASCII version of the machine language (and I
can't imagine why you would - how meaningful are several hundred thousand
ones and zeros?) then you could write a simple program to read the
command.com file byte by byte (use byte stream IO if you decide to do this
with COBOL) and simply transform each 8 bit "packet" into ASCII using a
simple table lookup of the 255 possibilities, or an intrinsic function which
converts a byte to a string, if your compiler supports this.

If you want an "intelligent reply" you must ask an "intelligent question".
