```
I am a mainframe Cobol programmer and I just took a week-long course
in Java programming to enhance my programming skills.

The course was written for C/C++ programmers and the last chapter
covered JNI.  I was wondering if it is possible to use JNI with Cobol
programs.  If JNI is not useable with Cobol, is it possible to process
files created in Cobol programs (whether sequential or indexed) in
Java and vice versa.  I know that Java doesn't have an equivalent
class for mainframe packed decimal data, so I would just use character
data.

I'd like to be able to explore it first on my own PC since I have
Fujitsu Cobol 6.1a on my own and if it works well possibly try it out
on the mainframe at work since our system does have Java installed.

Any input would be greatly appreciated.

Mike Polinske
Milwaukee, WI
```

```
Depending on the edition of the 6.1a you have - you may have access to
the Java Adapter classes.  These are the classes that Fujitsu provides
for Java communication.  I don't know of anything like that on the
mainframe directly, but I do know that if you are on a recent release
of z/OS that COBOL syntax can be used to invoke Java on the mainframe.
 I know of no method to invoke COBOL from Java on the mainframe.
```

```
Portability of Java/COBOL applications (or even coding techniques) between
Fujitsu (on the PC) and IBM mainframes is likely to be MINIMAL.

It is possible to use JNI interfaces on the mainframe *if* you are using the
most current IBM compiler, i.e. Enterprise COBOL.  With V3.2 it is even
possible to have COBOL as the "main" part of the application.

I suggest that you look at the section
  "5.2 Communicating with Java methods"

at:
 http://publibz.boulder.ibm.com/cgi-bin/bookmgr_OS390/BOOKS/IGY3PG10/5.2
```
