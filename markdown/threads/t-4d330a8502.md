But why would you want to do this ?

Presumably the COBOL source is for a business application.

Why would you want to convert from a 35 year old language to
a 25 year old one ?

Why convert from a language suitable for a business application
to one that is not ?

Why convert from a portable language to one that is only portable
if you write all the code for every target machine and have
compiler select the appropriate code to use.

If you think that C code is more maintainable (it isn't but you
may think this because you know C and don't know Cobol) then
consider that converting won't give you a C program, it will
give you back the same Cobol program with the Cobol verbs
converted (probably 1:1 or worse) into C functions calls.

DISPLAY "Hello World" AT 1020

is unliklely to be a nice printf(...) but is more likely to be:

COB_Display("Hello World",..

Similarly for other verbs: MOVE -> Cob_MOVE(field1,field2,PIC,PIC)

Even GO TO will become goto, and PERFORM may well be converted to
some code that emulates the COBOL PERFORM stack and uses goto's
to do this.

By the time that you have finished converting it, C will be a
dead language and you will be asking for a C -> Java convertor.

Meanwhile Cobol will still be humming along.
