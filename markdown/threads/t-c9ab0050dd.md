> I'd like u  to ask which is  the best way to begin to learn cobol.
> Thank u .

I have always found that the best way to learn a new language,
utility, or indeed almost anything is to start with an objective.  For
example set a requirement for a simple application - a CD Library, a
cheque book balance, or some other that you can use and thus set the
specification for.

The programming then becomes a series of problems which you must solve
by writing the code and making it work, which requires you to learn.

Also try to use a compiler that has a source code debugger.  This
allows you to step through watching the results of each statement.
