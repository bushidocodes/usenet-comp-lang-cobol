The DLOAD tells the compiler to generate code to dynamically link at run
time.  This makes it not link to your static routine.

> But then i got JMP- 0015I-U Cannot call program "FUNCTEST".
> Whats wrong ?

It is looking for 'FUNCTEST.DLL' with your routine in it.

You need to include NOALPHAL to get it to call your lowercase
'functest', or use all uppercase for the C name.

You need to have NODLOAD and link in your routine (without the main()
function) plus whatever libraries your version of the compiler tells
you.  This will probably include LibC.lib and something like
F3BICIMP.LIB.

> .
> my c program:
…[5 more quoted lines elided]…
>  }

Functest accepts a parameter that is a 32 bit value.  The Cobol below
will probably pass a pointer to a two byte area containing the ascii
value '09'.  You could change this by having:

functest(int * numbers)
..
and
   01  Anzahl        PIC S9(8) COMP-5 VALUE 5.

Or using BY VALUE on the CALL
