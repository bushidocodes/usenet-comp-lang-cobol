The reply that I received from my "usually reliable sources" was,

"The restrictions discussed in the attached note are largely correct.
i.e:
    You can't specify: LINAGE clause, BEFORE ADVANCING, AT EOP.
     You can specify WRITE ... AFTER ADVANCING though  (the note  implies
other wise).
These are documented around page 110-111 in the current LRM.

As for what the options are:
     1) use a LINE SEQUENTIAL file but handle page formatting
           via application logic using WRITE AFTER ADVANCING statements
        or
     2) write the report to a SEQUENTIAL file (with the LINAGE clause, WRITE
..... AT EOP, etc)
          and then copy the sequential file to a line sequential file.

The second choice may be better for this particular customer."
