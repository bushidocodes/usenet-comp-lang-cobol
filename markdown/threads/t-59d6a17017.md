Hi,

I am currently converting a COBOL II application to COBOL/MVS. This
application is currently running under CICS 4.1

There is a psedo-conversational task that starts up and issues a EXEC
CICS LOAD command to load a cobol program into memory and store the
address. When this task terminates, it stores that address into a TSQ,
and when restarting reads the TSQ for the address of the program that
was LOADed.

This task then calls an assembler module with the address from the
LOAD command and the assembler module links to that address and
execute the program.

The above code worked fine under COBOL II (3.2) and CICS 4.1, but
under COBOL/MVS (1.02) the first time it executes the LOADed program
it works fine, but the second time it executes the LOADed program, a
S0C4 abend occurs.

The assember module is a small (16 byte progrram) assembled using
HLASM.

I've tried to re-issue the CICS LOAD commands at every startup, but it
still fails (another module does the calls to that little assemmber
program multiple times).

I realize that this code may sound strange (IT IS). I've re-written
the assember module to COBOL with a dynamic call and that resolves the
problem, but it intruduced a performace impact (> 5%). A static call
didn't improve much (if any).

Any help or guidance would be greatly appreciated.

Just a general word of advice. If you should start using DTR
(Dynamic Transaction Routing), you'll have no guarantee that
the 2nd task will necessarily even execute in the same CICS
region. generally, attempting to rely upon addresses not changing
from one transaction to the next is becoming a bad idea.
