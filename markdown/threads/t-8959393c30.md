Hello.

I have an ASCII text string that I am sending via MQ Series to an
OS390 COBOL program.  The MQ Bridge will perform an ASCII to EBCIDIC
conversion before passing the string as an input to a COBOL program.

My problem is thus:

I need to pass "0689" to the following COBOL field:

05  IN-BUFFER-SIZE    PIC 9(4) COMP.

DATA ELEMENT   BEG END LEN  TYPE  FORMAT
IN-BUFFER-SIZE    1        2       2        C           (4)


I have two ASCII positions to store a representation of a 4 digit
number in COBOL.  What ASCII characters do I store in the two digit
field so that it will represent 0689 once it gets into the COBOL
program?

Any help would be appreciated.  Email or posted replies welcome.  
Thanks.

Hi Luke,

If I read you correctly, you want to move the string '0689' to the COMP field
IN-BUFFER-SIZE.

Here are 2 possible approaches:

MOVE X'0689' TO IN-BUFFER-SIZE(1:2)  This I haven't tried, but should work.
                                                             When a field is
reference modified, it's
                                                             treated as
alphanumeric, regardless of
                                                             its original class
(in this case, numeric).


This one I've used before and know it works:
 
05  IN-BUFFER-SIZE    PIC 9(4) COMP.
05  IN-BUFFER-SIZE-X REDEFINES   
    IN-BUFFER-SIZE    PIC X(2).

MOVE X'0689' TO IN-BUFFER-SIZE-X

Just to make sure we understand each other, if you got a hex dump of your pgm
after the move, IN-BUFFER-SIZE would contain '0689'. The decimal equivilant
would be '1673'.
