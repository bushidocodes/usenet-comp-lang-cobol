I'll assume this is a serious question rather than the flame bait it appears
to be.

Resizing is based on screen resolution, not monitor size. An application
sized for a 800x600 resolution (SVGA) will run on any montor/video adapter
combination that supports the same resolution.

If you have designed your application to require higher resolutions than the
current user configuration then you will need to make sure that all the user
monitor/video adapters are configured to the same resolution, assuming they
can support the higher resolution. Failing that, you will have to redesign
your application with the hardware limitation in mind.

If the application is character-based then it may be possible to select a
different font. Usually font selection is handled by the command shell
rather than your application. If it is graphical then you may choose to
dynamically resize the windows, fonts, etc. based on the size of the
available desktop or rearrange your interface for a smaller resolution.

What I know about AcuCOBOL can be written upon the head of a pin with a
magic marker but blaming then for your poor user interface design is hardly
fair.

It's not actually "screen size" that is the problem.  The problem is
actually "screen resolution."  The three most frequestly used screen
resolution settings are 640 x 480, 800 x 600 and 1024 x 768.
Essentially these number represent the horizontal  and vertical pixel
density supported by the monitor and video card on the specific
machine.

The problem occurs when you go from machine 1 which may be running at
800 x 600 to machine 2 which may be running at 640 x 480.

The screen size doesn't actually change...it's just that if the screen
was designed to be "full screen" on an 800 x 600 resolution monitor,
it will be too large to fit on a monitor with a 640 x 480 resolution.
Most GUI tools allow for automatic screen scrolling to accomodate the
lower pixel density.

>How can I make my applications scale to the screen size that  they are
>running on?
…[3 more quoted lines elided]…
>soon!

Don't switch compilers....COBOL sp2 provides support for automatic
resolution switching through the use of a simple environment variable.

If you design at 800 x 600 and have set the environment variable
properly, the GUI screens will automatically resize downward in a
proportional fashion for 640 x 480 resolution and automatically resize
upward in a proportional fashion for 1024 x 768 resolution monitors.

COBOL sp2 supports Acucobol very well.

Well, what you have to do is:

Make sure your window is resizeable.

Make sure you use a font that is freely resizeable

Make all controls occur using dynamic settings, which means: size, lines,
cols in variables.

First of all, I presume you do program event oriented, in that case, you
must take action on the event:

    ntf-resized

    With this event you will get line and size in respectively event-data-1
and event-data-2 (pixels)

    When this occur, you will have to resize all your controls relative to
this. This is accomplished
    by modify control and respective attributes. Remember to use a relative
factor to give the right
    proportions against the window.

Here is a very small sample of a context:

    display standard window,
        title "Scaleable window"
        lines current-lines, size current-size,
        resizable
        background-low.
    display screen-1.
    accept screen-1.
    evaluate key-status
    when event-occurred
        if event-type = ntf-resized
            divide event-data-1 by 100 giving current-lines
            divide event-data-2 by 100 giving current-size
            modify mycontrol
                lines current-lines - 9
                size current-size - 8
            end-if
    end-evaluate.

As you see, it is fully possible to resize a window and its controls in
Acucobol, so there should not be any need for replacing Acucobol.

Hope this helps.

Cheesle

The easiest way to accomplish this is to adjust the size of the font to the
resolution of the screen. This means that the window must be based on the
font as follows:
           display standard graphical window
               size   80.00,
               lines  28.00,
               screen line 1
               screen col 1
               control font fixed-font,
               cell size = label font fixed-font
               modeless
               erase,
               resizable
               background-low.

To get the right font size, first ... accept terminal-abilities from
terminal-info.
This accept will give you the number of pixels available on the screen
(horizontally and vertically).

Next, pick a font size (i.e. 5) and set fixed-font to this size with the
W$FONT routine. Then, use W$TEXTSIZE to get the size of some character (i.e.
X). Divide the horizontal text size into the number of horizontal pixels. If
it is greater than 80, set the font size up by one and set and check the
size again. Do this until you can't get 80 characters on the screen, then
set the font size down by 1.

NOTE: You need to do this before you display the main window. Also, it only
works really well if you use a fixed font (i.e. Courier New).

Hope this helps.
Ted
