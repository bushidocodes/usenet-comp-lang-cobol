Here's the situation:

I've just finished taking COBOL at my local community college, and am now
enrolled for Advanced COBOL for Spring (yay...). In the class we use a COBOL
book that includes a MicroFocus COBOL compiler for MS-DOS (the school has
the complete MF COBOL Workbench package for Windowson their PC's). Now, I
installed it no problem at the beginning of the semester, and used it a fair
amount while working on my projects (as well as using the school's computers).

However, I found a very annoying bug in the MS-DOS compiler. If I run it from
within DOS, and I attempt to do any file-related operations (open source,
save source, check&compile, etc) the compiler *insists* that the files I am
specifying do not exist, even thought they do. It goes as far as to let me
pick the file out of a file list it shows of the current directory, and then
claims the file doesn't exist!!!

Obviously this is pretty bad considering that I depend on using my home system
for doing most of my work (the school's campus is very far away and I have no
transportation beyond my bicycle). Fortunately, I discovered early on that if
I run the COBOL compiler from within Windows (just by opening up a DOS shell
box), it will function fine in regards to file operations.
However, this isn't a situation I always enjoy (having to go into Windows
first) plus it just plain bugs me. I haven't bothered to contact the company
yet as my copy is merely a student edition and I suspect I'm not entitled to
tech support.

Has anyone else run into this problem? I currently have MS-DOS 6.22 installed
on a IBM 486SX/33 with 8MB of RAM...
