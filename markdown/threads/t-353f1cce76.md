My only experience with the PC version was the old DOS, Windows 3.1, 
OS/2 version (under OS/2). The version was something like 3.5? The UNIX 
version is 4.0.05.

We used WB (Work Bench) and were using the mainframe emulation.
Under UNIX, we compile using shell script (although tbox could be used).
The animators are very similar. The PC had some flowcharting features 
that the UNIX version doesn't and is slightly more robust in general.
There is very little difference creating an *.int or *.gnt (using 
runtime environment).
The link build features (WB & tbox) are similar as well.
The standard at both shops was not to link and use the *.int.
The editors looks the same, but in the OS/2 shop (a few years back) used 
a PC version of ISPF as the editor.
The AIX shop uses an in house editor.

So IMHO, there is not much difference. At least with the versions I have 
used.

>     What are the differences between building programs with microfocus 
>cobol on the PC and microfocus cobol on unix?

Almost no difference.

You don't have to use Workbench. You can use your own editor, as I do,
and compile from a script or make file. Your editor can run on a PC ..
if you have a network connection and SAMBA or UltraEdit. Lacking
those, you can FTP source file back and forth, hopefully using a GUI
FTP. 

Source programs will compile and run on both platforms with zero
changes, unless you're doing something specific to the platform, such
as GUI. 

>     Is the link and build options just the same is that all you have to do 
>to build a runtime version on unix?

Runtimes are already built, on both platforms. Compiler options are
almost the same. You don't have to worry about linker options unless
you're doing something unusual, such as calling a third-party library.

Gnt and earlier .int are retained for backward compatibility. The
preferred way is compile the  main program to an executable, make all
called programs dynamic and compile them to .so on Unix or .dll on
Windows. 

>   Is the editor and animator the same?

I don't use the editor. Animator is the same.

>   If anyone has any notes on microfocus Cobol unix please email them to me.

The Unix manuals are at:

http://supportline.microfocus.com/supportline/documentation/books/sx40sp1/sx40indx.htm


>  If anyone knows any good website on microfocus cobol unix development 
>please let me know.

All you need is knowledge of Cobol in general and the above manuals.
Microfocus Cobol is not a different language, despite what employers
think; it is ANSI standard Cobol 85  plus some extensions that you
probably don't need, such as pointers, object oriented and
multi-threading. Demos of those have been posted in this newsgroup.
