I wrote a general note on what CGI is in the CGI authoring group.

If you want access to these files by undefined people over the net you
simply generate a static form and interpret the output that comes back
from that form in a program. To update your text files you can simply
regenerate them into HTML and then have the web server read static HTML
(no difficult CGI programming then).

If you want defined people to acess text files on your site then a
secure signon through a TELNET (terminal emulator) or even upload and
download via FTP might be a better solution.

I am sure you will get better feed back on the specifics of CGI using
various cobol compilers. What specific COBOL and Unix are you using to
let the group be more specific.

Ken

MF COBOL supports the X/OPEN DISPLAY and ACCEPT extensions for getting
and setting environment variables.

To Get an environment variable (like REQUEST_METHOD) value:
DISPLAY "REQUEST_METHOD" UPON ENVIRONMENT-NAME
ACCEPT REQUEST-METHOD FROM EVIRONMENT-VALUE

where REQUEST-METHOD is a suitable working-storage variable.

You shouldn't need to set any environment variables for CGI programs but
the technique is:
DISPLAY "REQUEST_METHOD" UPON ENVIRONMENT-NAME
DISPLAY REQUEST-METHOD UPON ENVIRONMENT-VALUE

If your version of MF COBOL doesn't support the X/OPEN extensions then
you can try using cobgetenv(). Visit
http://www.microfocus.com/cgi-bin/WhitePapers for details.

As for STDIN/STDOUT, MF file support will access these streams using
sequential file I/O by making the file name STDIN or STDOUT. Example

SELECT STDIN ASSIGN "STDIN"
ORGANIZATION LINE SEQUENTIAL
ACCESS SEQUENTIAL
FILE STATUS STDIN-FILESTATUS.

Avoid using DISPLAY statements because they write to STDOUT and
prematurely lose the STDOUT data stream, truncating your CGI response.

As for Java calling a COBOL server, it can be done but you have to get
into the socket API. There are a bunch of companies that can help
including my employer Netron. Visit
http://www.netron.com/temp/new/1997dec/motorola.htm.

Karl Wagner
