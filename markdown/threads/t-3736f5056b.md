```
Following the save area chain isn't hard.  Knowing when you are at
the beginning isn't easy.   I believe that there is a system call that
will give the address of the beginning, though.  For standard linkage
convention (the URL above also inclues FASTLINK and XPLINK),

If R13 contains a save area address then load it into another register,

   LR  R2,R13

   then

   L R2,4(,R2)

Will be the address of the callers save area. 

Register 15 in the save area points to the entry point, and 4(R15)
to the length of the name and 5(R15) is the beginning of the name.

   L   R4,16(,R2)   R15 in the save area pointed to by R2.
   SR  R3,R3
   IC  R3,4(,R4)
   BCTR  R3,0
   EX    R3,MVC 

and an MVC to execute to copy the name to buffer BUF.

MVC  MVC BUF(0),5(R4)

I hope this helps,

-- glen
```

```
Actually, this will not work in many cases.  If the program is
one by IBM and certain vendors, the R15 EPA (Entry Point Address)
may not actually be a 47F????? (Branch R15+???) to jump past the
length and literal area that you refer to.  Some times you must
use the CDE chain to figure some of this out, particularly when
the program loaded was actually a module (a group of object decks
pre-linked into one large loadable entity).

Running the R13 chain (RSA - Register Save Area) may not work, as
it may be broken by a program that violates the standard.

One may find this easier to accomplish by taking a dump using
SYSMDUMP and IPCS to read the chains to determine who called
who.  I did not look at the book that was referenced by the other
poster, however, if it is the one I'm thinking of, you will need
it.  You might also try OS Debugging for the COBOL Programmer by
Murach (assuming it is still in print) -- www.murach.com
```
