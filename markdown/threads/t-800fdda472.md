Here are some things to check:

1. Use $SET CASE so that the compiler won't uppercase the names of called
routines.

2. Use
special-names.
call-convention 74 is WINAPI.

3. Some of the windows routines have both ASCII and UNICODE versions. So
use something like this:

78 Suffix VALUE "A".
78 API-SendMessage VALUE
"SendMessage" & SUFFIX.
78 API-DdeCreateStringHandle VALUE
"DdeCreateStringHandle" & SUFFIX.
78 API-DdeInitialize VALUE
"DdeInitialize" & SUFFIX.
78 API-DdeQueryString VALUE
"DdeQueryString" & SUFFIX.

4. Be sure to use the WINAPI in the call statements:
call WINAPI API-DdeInitialize using
by reference dwDDEInst
by value ddeCallBackAddr
by value ddeCmdFlags
by value dword-zero
returning ddeReturnCode
end-call.

5. Be sure to declare the variable with the correct size for WIN32.

You have coded the calls with too many parameters. The number following
the '@' symbol
represents the number of bytes for the parameters. In general, each
parameter is 4 bytes
long.

Your call to DdeConnect() should look something like:
CALL APIENTRY "DdeConnect" using
by value idInst
by value hszService
by value hszTopic
by reference ConvContext
returning hConv
END-CALL

It has 4 parameters (16 bytes) and returns a 4 byte value (hConv).
