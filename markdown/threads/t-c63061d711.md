Ken Foskey <waratah@zip.com.au> wrote in article
<37E2447F.815DC1D9@zip.com.au>...
> Firstly by my understanding of the evaluate EVALUATE TRUE must
> (should?) be equivalent to the IF THEN ELSE chain.  This is certainly
> that way that Fujitsu has implemented it.  It is a little difficult to
> test on MVS but a suspect a quick look at the assembler will prove to
> be the same.

It is.  I've checked the output from the IBM COBOL for OS/390 & VM  2.1.1
compiler.
That is

EVALUATE A
  WHEN 'x'
    ...
  WHEN 'y'
    ...
END-EVALUATE

generates exactly the same code as

IF A = 'x'
  ...
ELSE IF A = 'y'
  ...
END-IF
END-IF

Also the same with a period as multiple closure.

But the following generates several unneccessary branches:

EVALUATE TRUE
  WHEN A = 'x'
    ...
  WHEN B = 'y'
    ...
END-EVALUATE

In all cases the comparisons stop when a match is found.

Gunnar.
