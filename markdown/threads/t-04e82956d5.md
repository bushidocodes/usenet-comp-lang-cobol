```
Right after your read statement, you move your data out of your FD
section to another location.  This instruction is illegal when the file
is currently at EOF.

Try using the "READ ... INTO ..." form of the READ instruction.

Pete
```

```
This is exactly why the "exit in the middle" without a priming read gets
people into problems.  When the program gets to end of file the record-area
for the input file is no longer defined.  The program must then "skip" to
move and write statements.  A better structure is:

>           PERFORM INITIALIZATION.
             READ PB3M-FILE-IN
                   AT END MOVE 'YES' TO EOF.
>           PERFORM CREATE
>                   UNTIL EOF = 'YES'.
```

```
solutions.  My approach (assuming you have an 85 Standard compiler) would be
to change your create paragraph to:

           READ PB3M-FILE-IN

                      MOVE 'YES' TO EOF
               Not at End
                     MOVE pb3m-rec-in TO prg2-rec-out

                     End-Write
```
