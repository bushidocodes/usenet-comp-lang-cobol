```
>(*.DAT) from cobol-made to window system...any idea/clue? :)

If you have the COBOL source and are willing to write a program,
output the data to a line sequential file using DISPLAY type fields.
You could write a C program to do the same converting a formated
bitstream.  Some companies have written utilities to manage this.
In almost all these cases, you need to know the format of the file.

By the way, outside of how some COBOLs name the index of a
file, the name of the datafile can be of any format in the various
dialects of the language.  The *.dat is often a programming
convention, not a requirement of the language.
```
