The question is a bit vague ... ?

You should be able to use an array to perform any data ordering/traversal
given the proper 'paragraph' ;-) (after all a computer's memory is
just an 'array' of bits)

To build a binary tree with an array you need a function to calculate the
index of any node, basically it's (2^(depth-1)+partial width)+1, so to go
down two and over three ... (2^(2-1)+3)=6 so array(6) is the node at
depth 2, position 3 ... this assumes that the root node is at depth 0,
position 0.

Alternately, you could build a binary tree, by creating a record with a
data part and an index part, where the index part points to the two children
under it (you could also add an extra index to point to the parent for
backward traversal). You will have to keep a variable that contains the
next available index using this approach.

The first approach takes up lots of memory, but is easy (quick) to access.
The second is more memory frugal but slower to access.
Kind'a depends on what you are trying to do.


Building a stack requires that you keep an extra variable that contains the
index of the 'top of stack'. Putting something on the stack increments
this variable, removing something decrements it.


Putting the two ideas together is rather simple ... but requires that I
know more about what you are doing (well maybe not requires, but it would
save a lot of useless typing).

I think a 'priority queue' uses the two together ... but I'm too long
out of school to remember.
