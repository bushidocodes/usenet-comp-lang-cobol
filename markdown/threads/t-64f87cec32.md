'{' as the first character of the first page, puts a four character MSG-ID
at the end of every page, and puts a '}' after the end of the document.  (a
'legacy' interface depends on these special delimiters, otherwise I wouldn't
bother...)

Both an EXIT and a CONTINUE statement are NO-OP statements.  They do
nothing.  They don't mean anything.  They should be used when you need
a statement and you don't want to say anything.
In the PROCEDURE DIVISION I use the CONTINUE statement to set off the
period at the end of a paragraph.  I don't like a period on a line by
itself.
And sometimes I use it to make a positive IF statement.  Sometimes a
positive IF statement just reads better.  So I will code the following.
IF POSITIVE-CONDITION
    CONTINUE
ELSE
    PERFORM DO-NEGATIVE-THINGS
END-IF
Usually the POSITIVE-CONDITION is something like IF A = B AND (E = D OR
F not = G).  Trying to get a negative of that has never been something
I am good at.
I also prefer CONTINUE to EXIT because there should be less confusion
with CONTINUE statement.  The name EXIT just sounds like it is going to
do something, but it is not going to do anything.
I guess you could say that if the EXIT/CONTINUE statement does anything
it satisfies the compiler's need to have a statement in certain places.
 Otherwise the compiler would generate an error.
