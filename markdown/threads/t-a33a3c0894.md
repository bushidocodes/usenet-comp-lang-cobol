```
Tommy,
ZeroMemory is not an API call but a C macro which "hides" the real API 
call RtlZeroMemory() (same parameters). There is no call convention. So, 
the following works, for example:

       working-storage section.
       01 thememory pic x(256) value "test".
       01 thesize pic x(4) comp-5 value 256.
       procedure division.
       call "cob32api"
       call "RtlZeroMemory" using by reference thememory
                                  by value thesize
       display thememory
       stop run.

(RtlZeroMemory lives in Kernel32.lib/dll)

All this does is move 0 into each byte of the memory, however - something 
which COBOL is perfectly capable of doing without API calls :-)

Steve.
```
