You say it "compiles fine" but what about the link?

References to external libraries need to be presented to the linker so that
exported and imported entry points can be resolved.

You have coded it as a static call, so it needs to be resolved at compile
time.

Maybe making the call dynamic would work? (The RTS should resolve the entry,
providing you give it a "clue"... not sure how you would dothat for GNU
COBOL.)

Is there nothing in the docs about referencing external libraries, or
including them in the link?

There is nothing wrong with the syntax of your call so the problem is
probably not with the STDCALL option (which really just decides who cleans
up the stack, caller or callee...).

Have you tried posting in the GNU COBOL support forum?

http://sourceforge.net/p/open-cobol/discussion/
