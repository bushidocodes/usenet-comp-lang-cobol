Theo,

JSP stands for Jackson Structured Programming (not Structural). It is a
technique that may be relatively unknown in the USA, but is widely known in
Germany, The Netherlands, Belgium, and (I think) England and more countries
over Europe.

About JSP (and you nay know this but others clearly don't): it is very
useful in situations where one has to make a program that has to proces
input and output data files that are sequentially accessed. Jackson
proposes that the data structure of the in/out files can be merged to one
combined I/O-structure which in turn casn be transformed to a program
without anymore structural modifications. The resulting program consists of
one or more hierarchies of processing loops and/or case/selects build from 6
basis types of subroutines.
2 types for input: READ-FILE-fff / PROCES-RECORD-rrr
2 types for output: WRITE-RECORD-rrr / BUILD-RECORD-rrr
2 types for structures: INITIALIZE-xxx / TERMINATE-yyy
So far the lessons for other readers. :))
