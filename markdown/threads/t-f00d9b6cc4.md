I am disconcerted by the suggestion that additional potentially
controversial constructs are still being considered. I really don't
want to see another full round of comments being required. At this
point we need the function that the 199X standard offers as soon as
humanly possible. I am interested in having some of the things I see
being discussed but they can wait.

It is not too early for the various standards committees to start the
work for an addendum that would go to the public for comment within a
year of approval of the 199X standard. It would be useful to have a
requirement that new function be shown as implemented in at least a
beta for before approval. This would also allow posting of samples
at a web site since there would be samples available. Indeed this
could make the review process more meaningful. It also would be far
easier for both the commenters and the standards groups if all we had
to deal with were the changes. If there were a process that stated
that a change document would come out each year (18 months or some
other reasonable interval) and changes that aren't approved in one
cycle could be reworked for a subsequent one, we might get function
in a more timely fashion. If ANSI and ISO can't handle this, I
recommend that other organizations be looked at. IEEE and the
Internet Standards Group (I don't recall the name and am too lazy to
dig up the book that has it) are two organizations that come to mind.
Others may be more suitable.

I appreciate the large amount of work the standards committees are
doing. I just want to be able to use the fruits of their labors.
