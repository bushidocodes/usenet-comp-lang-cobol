```
01  Work-Fields.
     10  SALUTE    PIC X(30) VALUE " Howdy and Hello, World".
     10  TITLE        PIC X(30) VALUE SPACE.
     10  SUB1        PIC s9(04) COMP.
      10 MaxPoint  PIC S(9(04) COMP  VALUE +0.
     10  FIVE          PIC S(04) COMP VALUE IS +5.


MOVE 1 TO SUB1
COMPUTE MaxPoint = (LENGTH OF SALUTE - 5)
PEFORM
    IF  SALUTE (Sub1:FIVE) = "Hello"
         MOVE Salute (Sub1:FIVE ) TO Title
         EXIT PERFORM
    ELSE
        ADD 1 TO SUB1
        IF  SUB1 GREATER MAXPOINT
           EXIT PERFORM
        END-IF
    END-IF
END-PERFORM.
IF Title GREATER SPACE
   DISPLAY "Found ""Hello"" IN SALUTE"
ELSE
   DISPLAY "Not Found"
END-IF.
```

```
Hello Giorgio Cordini et al.:

You wrote:

................

> Ex.:
> I know that in the variable salute there is "hello".
> Now I move "salute" to title...
> ... how to obtain "hello" from title?

.................

What you want to do isn't a macro substitution (that takes place
either before or in the earliest stages of compilation).  Rather, it
is an attempt to use the contents of one variable to designate
another variable during execution, and depending on
implementation, that goes under several names: pointer or
reference variables (the variables whose contents are used to
designate the second variable), and based or parametric variables
(variables designated) for instance, or perhaps more to your
point, associative variables (referenced using the character
contents of one variable as the character string name of another
variable as found in the source program).

Cobol doesn't have such things (at least not as of when I taught
it about 10 years ago).  The attitude of the Cobol standards
people seems to be to steer clear of anything that is a bit of a
mind-bender in English.  Other languages of course do have
such things, but if you are in a Cobol environment, you had
better use what you have at hand.

Do try another tack, even if it is a long chained IF .... ELSE IF ...
...... to handle all the possibilities, but be sure to include an
unconditional ELSE at the end of it to catch the possibility of
bad character data which references no variable in your program.
```
