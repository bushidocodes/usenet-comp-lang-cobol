```
While not strictly a COBOL posting, I believe this is
of interest to COBOL programmers, especially those
involved in maintaining production applications.

Thank you for your time. Here goes:


The PDS-to-PDS Comparison/Matching Utility Version 3
is now available.

This utility was formerly called PDSMATCH, but has been
completely rewritten and renamed.

PDS2PDS is an ISPF/REXX utility that will display
the directories of two partitioned datasets side-
by-side on the screen.

You can copy or move members from one pds to the 
other, you can edit or browse members of either
pds, you can invoke SuperCe to compare datasets
or members, and much more.

PDS2PDS offers these advantages over PDSMATCH:

o  You can now specify separate member name patterns
   and wildcard searches for each pds.
o  Error detection and reporting is improved.
o  The Help screens are now a set of ISPF Tutorial panels
o  Each Global and Line command is now a separate external
   subroutine which makes for easier modification and 
   maintenance.
o  You can easily add your own global or line commands
   to the utility.
o  The command confirmation panels are now more "ISPF"-like,
   such as the ones used in ISPF options 3.1, 3.4, etc.
o  PDS2PDS is much easier to install than PDSMATCH. One
   example of this is that the included COBOL program is now
   supplied as an executable load module. You no longer
   have to compile it yourself.

If you need to keep track of matching pds members, such
as test/production jcl, source, data, and other libraries,
then perhaps PDS2PDS would be of use. Best of all, it's free
for the taking.

You can get more details at http://wb4huc.home.texas.net/pds2pds


Thank you,

Mike Newell
Austin, TX
wb4huc@texas.net
```
