I have used the Micro Focus Products here. In one case, the IMS/DB2
helped me test a online MFS screen IMS/DC program and bring it to
production rapidly. This was using the OS/2 database manager and
OS/2.

As we went from OS/2 to Windows, I tried to find the same functionality
on the DB/2 side, and was given Microsoft SQL server as a possibility.
I tried to use it, but found that using it along with the EBCDIC option
didn't work well. However, if your company will use XDB it probably
will have the EBCDIC ability. I found Micro Focus to be very useful
in debugging bad bugs. I was discouraged when I couldn't get the
DB/2 with EBCDIC with MicroSoft SQL. But maybe that was my mistake...

Also, I used MicroFocus CICS and was able to do all my homework for
a CICS class here at U.C. Berkeley extension up in San Francisco.

Overall, I think Micro Focus is the way to go...
