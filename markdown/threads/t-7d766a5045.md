Since I posted this problem last week, I have made some progress and received
some help from the C news group. If someone from Micro Focus is reading this
group, could you comment on the error I am getting from the linker. Thank you.

This is an update of a post from March 7, 1998.

I am having some basic problems getting started with
Microsoft C++. I am trying to compile a main program
and a function. The main program is in a file called
driver.c, the function is in a file called extfh.c. I
compile the function and it compiles clean and is stored
in a file called extfh.obj. The main compiles clean, but
the linker tells me:

error L2029 '_extfh' unresolved external

I am using Microsoft Visual C++ 1.52. I have a much
newer product, but it has not been installed yet. I believe
that I must "tell" the linker WHERE to resolve its
external references, but I don't know where to place
this information. I have tried various spots in the
OPTIONS pull-down, but nothing does the trick. BTW,
I have read the excellent FAQ written by Steve Summit.
While it was helpful, it really did not tell me how to
do it.

In addition I have tried the command line link command
without much success. Again, I can not seem to get the
path to the libraries correct. This time it is the
libraries, and not the function which are unresolved. In
particular: __edata and __end can not be found by
MLIBCE.lib.

Since I posted the above message I have been able to
create a STATIC LIBRARY and place a compiled function
like dummy.c into it. This would have solved my problem
completely, however what I really want to do is place
object code that someone else (Micro Focus) has written
into my .exe file. I have written a main program called
extfhrd.c which calls a function called extfh() which is
stored in a file called extfh.obj. If I could pick up
that .obj file during the compile/link step of BUILD I would
be done. Failing that, if I could figure out some way
to place the external reference to extfh() in a .lib file,
then I could proceed just as if I had written extfh()
myself. The source code to extfh() is NOT available.

After some fine help from Dann Corbit, I am able to place
.obj files into a library file misc.lib. The trick was to
build a NEW static library. Maybe the word STATIC means
that you can not add .obj files to current library. This
works with files I write and compile in MSVC release 1.52.
It seems to fail with the Micro Focus supplied file extfh.obj.

I am getting the following error:

LIB: fatal error U1155: syntax error : illegal input

I assume that the compiler or assembler which generated
extfh.obj used linker conventions which my poor old 1.52
can not handle. I am going to try to contact Micro Focus
to see if there is some way to use 1.52, or am I going to have
to upgrade.

TIA for the help.
