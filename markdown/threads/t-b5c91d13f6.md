Report on October 14-18, 1996 ANSI X3J4 COBOL Committee meeting,
Chelmsford, Massachusetts, USA. When a vote is shown, the numbers are
for, against, and abstain (f-a-a).

Membership stands at 11 (we gained and lost one after the last
meeting). The primary topic at this meeting was action for the CD
(committee draft) review at ISO. The latest version of the base
document has been forwarded to ISO for this review. The official
start was in mid-September and the end on November 26. We decided
that we could add some additional features after the CD if comments
suggested them. There were comments for all that we have added.

The major topics/proposals we discussed were:

1. There was some discussion about the CD review. So far, only one
set of comments has been receive (as of the end of the meeting,
October 18).

2. During discussion of a paper on conformance rules it was pointed
out that the new rules for CALL state that parameters in the calling
program must match the parameters in the called program exactly (PIC,
USAGE, etc.). Before, it was for size only (the new rules
accidentally made this change). We decided that the new rule would
apply only when there is a matching entry in the repository paragraph.
Otherwise, the rules would be the same as in COBOL 85. There was more
discussion on passing pointers and a paper will be written on the
topic.

3. We had assigned a member to write a proposal on allowing ADDRESS
OF in the CALL/INVOKE USING list. We spent lots of time discussing
what is actually passed, especially when a pointer data item is
referenced (a pointer to a pointer or the contents of the pointer).
The paper will attempt to resolve the conflicts and confusion.

3. The proposal to remove AUTOMATIC and STATIC for data and add a
local storage section was discussed and sent back for rewrite.

4. A proposal to extend in-line comments to fixed-form format (they
were limited to free-form before) was passed and added to the base
document.

5. A proposal for implicit qualification of data-names (like key
names) in the OCCURS clause was discussed and sent back for rewrite
(again).

6. A proposal to not allow USAGE OBJECT REFERENCE in the file section
was passed. It would have been quite sporting when someone wrote one
that pointed to an object and read it back in a week and expected it
to be pointing to the same object. The same problem exists with data
and procedure pointers, but no changes are being made.

7. A paper on defining MERGE and SORT interaction with USE
procedures, exception handling, and other clean up of ambiguous areas
was discussed and sent back for rewrite.

8. A paper on clarifying sending and receiving items brought out
problems with the incompatible data rule. Now, you can get an
exception for incompatible data, but the rule is not clear about
exactly what that means. The proposal will be explicit by tying the
rules to the numeric class test. The proposal will be rewritten.

9. A proposal on concatenation of data items was discussed and sent
back for rewrite (in the book you can concatenate literals but not
data items).

10. My proposal on constants was discussed and modified significantly.
The final form is "CONSTANT constant-name [GLOBAL] IS literal". They
can be specified anywhere in the data division an 01 or other
equivalent item can be specified and can be used anywhere a literal
can be used. The proposal passed (7-3) and will be added to the base
document.

11. A proposal on hex literals was discussed and sent back for
rewrite. It originally had two types, numeric (H"...") and
alphanumeric (X" "). The committee removed numeric and added
boolean and national. It now has three kinds, alphanumeric (X"xxxx"),
boolean (), and national (). Alphanumeric and numeric are already
implemented by several vendors.

12. A proposal to add a date formatting function was moved to be
addressed in the next revision.

13. My proposal on READ PREVIOUS and START LESS THAN was discussed and
passed and will be added to the base document. It also provides START
FIRST and START LAST and allows them for sequential files.

14. A proposal to extend the class test (like IF NUMERIC) to
non-display items such as packed-decimal was passed and will be added
to the base document.

15. A proposal to add functions to validate the format of dates and
inputs to NUMVAL and NUMVAL-C was discussed. It was decided that an
additional parameter would be added to the date and numval functions
that would force them to check the parameters and to return a value
that would indicate whether or not the arguments are valid. Then it
was decided to add the word TEST as a parameter to other date
functions to validate the inputs.

12. A proposal on allowing underscore to be used in the same way as a
hyphen for user-defined words was passed and will be added to the base
document.

15. A proposal to allow alternate key suppression was discussed and
several changes were made. It was decided that it was not a fixed
file attribute and that READ and START would skip such records.

The next X3J4 meeting will be in Carmel, California, USA, on December
9-13, 1996. The main topic will be addressing comments on the CD
review of the base document and the above proposals that were
rewritten. There will also be an object-oriented ad hoc meeting on
the preceding Friday, Saturday, and Sunday (December 6-8) to deal with
OO comments.
