Presuming you are talking about a relational database such as DB/2, Oracle,
Sybase, etc. the Cobol part is pretty simple. A typical cobol "read"
statement against the database would look something like

EXEC SQL
SELECT EMP_NAME, EMP_NBR, EMP_DOB
FROM EMPLOYEE_TABLE
INTO :EMP-NAME, :EMP-NBR, :EMP-DOB
END-EXEC

As always this is very simplified, but points up a few things. All of your
"reads", "writes", "deletes" and "updates" are coded within a EXEC SQL
statement. The source code is fed to a pre-compiler that parses these
statements, comments them out, and replaces them in the source code with
standard Cobol CALLs to the database subsystem API. This resulting source is
then input to the standard Cobol compiler. MF integrates this precompiler
support (for some databases) into their compiler.

The other thing you need to be aware of is that setting up and maintaining a
relational database -- at least for a production system -- is a full time job
in itself.
