```
Like the other posting a reply, I may be misunderstanding your problem.  I
*think* you are saying that you have defined Working-Storage Section with
both "modifiable" and "non-modifiable" data.  BECAUSE CICS requires COBOL
programs to be "re-entrant" - each time the transaction starts COBOL itself
does a "getmain" for the entire WS and initializes all the fields with VALUE
clauses in them.  You want to avoid the getmain and initialization each time
the transaction is started - for those items that are never modified (so
don't need a separate "copy" in order to be properly re-entrant).

*IF* that is the correct statement of your problem, what I would suggest is
that you create a program or part of your logic that is executed the first
time the transaction is invoked do the following:

  - use an EXEC CICS GETMAIN to get enough storage for your NON-modifiable
stuff
    (This will end-up in your Linkage Section, not WS)
  - Put the address (and length - if variable) in your com-area (or someplace
else that each iteration of the transaction can get)
  - Do "MOVEs" to initialize the values in that storage.

Then for the 2nd thru Nth iteration of the transaction, you can just do a SET
ADDRESS OF Linkage Section "item" and it will have the same values as the
last time (and you won't go thru any new GETMAINs or initialization)
```

```
The simple answer is NO. W-S is a single entity to a COBOL program, and
is obtained by a single GETMAIN if you're re-entrant. I think the
compiler has already done (partly) what you're asking about. The
template for W-S which is part of the load module tells how to
initialize/populate W-S after it's obtained, using both the literal pool
& masks. Aside benefit to having programs RENT is that the load module
doesn't include W-S, just what's needed to initialize it after the
GETMAIN. We saved a lot of loadlib space when we switched our batch
programs from NORENT to RENT (check out something with some big tables
in W-S).

Bill Lynch

(Note: I used the terms I did above conceptually, "template", "mask",
etc. I don't know which terms are used by the compiler designers)
```
