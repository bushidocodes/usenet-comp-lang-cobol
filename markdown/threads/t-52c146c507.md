Since C, C++, and C# (I have no idea what alia might be et'd) are
different languages, there is no reason why syntax should be
consistent between them.

That said, C++ includes C (though it imposes additional restrictions,
so it actually contains a subset of C); and C#, though closer to Java
than C, is very much in the C family in its general syntax.

As for releases: there have been very few changes in the syntax of C
between the first standard (ANSI 1989, adopted wholesale by ISO 1990)
and the current one (ISO 1999 plus technical corregida). Somehow I
doubt Mr Jaffe is having problems with the introduction of the
"restrict" keyword and similar.

Of course, C is difficult to use well, because it leaves so much
responsibility to the programmer. But that is not a fault of its
syntax, nor one of consistency.

C++ has gone through quite a lot of revision over the years, and
become rather a monstrosity. If that's the source of Mr Jaffe's
frustration, he has my sympathies. C++ has far too many complicated
features with obscure pitfalls. It can be used to write very elegant
code, but only if you understand it thoroughly and exercise great
restraint.

C#, on the other hand, is relatively easy and forgiving. Its chief
drawback is that it attracts the sort of programmers who disdain
whitespace and reasonable line-lengths, and consequently produce
source that's agonizing to read. (That's mainly Microsoft's fault for
not modeling better technique in their examples and educational
materials.)
