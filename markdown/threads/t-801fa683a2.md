Not sure what you mean by "multiple functions within a C program..."

With the combination of COBOL/370 and C/370 that you cite, COBOL is not C calling / passing convention aware, so you must put the #PRAGMA statement in the C program so that the compiler will generate standard (COBOL-style) calling / passing sequences.
If you would use COBOL for MVS and VM (essentially the next release of COBOL/370), and LE/370 1.5 (needed to support COBOL for MVS and VM), then you no longer need the PRAGMA statements in C, and may have better results.

Perhaps posting a minimized example would help clarify the situation.
