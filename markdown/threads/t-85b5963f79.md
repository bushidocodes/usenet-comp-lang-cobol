Hi There,

If you are looking for high performance on Microsoft IIS Webserver then you
should look at using ISAPI DLLs. These are faster than CGI EXEs as the
operating systems does not have the overhead of creating a process to run
the EXE.

You also need to make sure that your application will work correctly in a
multi threaded environment. On the COBOL side the REENTRANT(2) directive
ensures this is OK but if you call any other DLLs then you need to make sure
they take the multitreaded environment into account. If your application
used SQL then you will also need SQL(THREAD=ISOLATE) directive for your
application.

There is however a down side in that if you have an application error in a
ISAPI DLL then it is very likely that the whole webserver will "die". CGI
EXEs should prove to more resiliant to errors and the Webserver should
continue to function.

Hope this helps.
David.
