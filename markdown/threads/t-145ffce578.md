What is my-ret-code?  If that is not returning true (non-zero) the call 
failed and GetLastError() will give you a reason code.

On problem I see with this code is,  ReadFile does not return until 
number-of-bytes-to-read bytes have been read or an error occurs.

If you read past end of file, you should get the end-of-file condition - 
Success but zero bytes read -  but you may have to read one byte at a time 
to get that to work without ReadFile waiting for more than one byte when 
only one byte is available.

Also from the SDK:

"When reading from a communications device, the behavior of ReadFile is 
governed by the current communication time-outs as set and retrieved using 
the SetCommTimeouts and GetCommTimeouts functions. Unpredictable results can 
occur if you fail to set the time-out values. For more information about 
communication time-outs, see COMMTIMEOUTS"

Rene,

The sequence is:

AT+CMFG=1       *> sets text mode
OK
AT+CMGS "phone number"
(You might need to wait on a returned ">" here, it depends on the pic 
(programmed interrupt controller))
<text of SMS message>
Ctrl/Z
ATH

As you are not getting OK returned when you set text mode, then text mode 
has not been set.

Check that the card or modem you are communicating with actually supports 
GPRS and  CMFG...

AT+CMFG?

...should return the current mode (0 = PDU, 1=SMS)

AT+CMFG=?

... should return what modes are available. (something like:   +CMFG: (0, 1) 
if SMS is implemented)

If these commands indicate the device is fine, then the problem is probably 
in the parameters you passed in your API call. Check the format of numbers 
passed by reference.

Be aware that different models of handset may implement the Hayes command 
set differently, and sometimes something that works fine with one phone will 
not work with a different brand of handset.

HTH,

Pete.
