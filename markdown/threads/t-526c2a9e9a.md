I really liked the concept of elastic COBOL and would have used it gladly. 
There are times when PRIMA clients need a COBOL compiler that doesn't cost 
an arm and a leg. It is hard to justify buying more expensive COBOL when you 
are planning to move off it...

However, it generates Java (so all the infrastructure is Java - JARS etc.) 
and it doesn't support OO COBOL, or it didn't when I last looked. (Over a 
year ago now)

The Java I can live with (it doesn't really matter what gets executed as 
long as the functionality is provided), but the OO is essential for our 
(PRIMA's)solution.

I think your point about high volume testing is a good one, Doc, but I 
suppose they would argue that you have to pay for the processing on your 
existing hardware so why not move it to the cloud and (hopefully) save on 
expensive system upgrades and maintenance?

I suspect the volumes you are talking about would be stretching the elastic 
to the point where there is a real danger of the knickers ending up around 
the ankles... :-)
