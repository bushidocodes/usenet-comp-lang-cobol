I'm a little out of practice as far as COBOL is concerned - anyway,
try the following (at least in COBOL85 this DEFINITELY works !):

Declare the IS file with ACCESS MODE IS DYNAMIC, or, if you
exclusively do sequential access throughout the whole program, with
ACCESS MODE IS SEQUENTIAL.

In the PROCEDURE DIVISION, you use the START command to move the
record pointer (my word for it) to the desired position within the
file (look up the syntax in your COBOL manual - I don't remember ;-)
and I don't have my manual within reach now). A READ NEXT command
after positioning the record pointer will retrieve the current record;
any subsequent READ NEXT will do what the command says: read the next
record.

Bernd

go to break.

repeat.

go to break.

go to repeat.

break. exit.
