I've been testing some changes to some mainframe reporting programs (that 
support a PowerBuilder application).  The report 'modules' (extract and
format) are called by 'controller' programs.  I keep getting stange errors.
The MVS dumps a 2222 error but the IDMS-STATUS dump part says 0069 (run unit
time out, not bound, or bound twice).  The only database access this
particular module does is a few table calls through a handler module in the
beginning of execution.  There is also a DNS processing error
Receive_and_Wait error on IDMS.

When I run the programs where they are constantly accessing the database,
the job doesn't cancel out.

The question arose of why the programs in testing (which run at least 12
minutes when tested separately) time out.  I'm running the jobs against the
Production database (Many think I am somewhat fearless but I've never been
scared to run against prod) but still it gets run-unit timeouts.

The only thing I can think of is the CLASS of the job.  Is this possible?
When I submit a job from ROSCOE (into JES), it goes into CLASS=C.  When the
jobs are submitted by CA-7, they go into CLASS=A.  Is it possible the
different classes have an impact on IDMS run units?

I'm kinda scared to promote these.  Have a nice day.

Jeff
