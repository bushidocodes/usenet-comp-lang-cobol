Barcodes generally do not fall into standard formatting.  If you
use barcode 128 then 20 hex stands for 20 numeric.  If spaces are
absorbed on the way out then the result will be a real space in
the barcode not the digits 20.

Most formatting tools have special code to send in digits and
output the mapping font directly to the printer.  Maybe this could
be a future extension for Bob.

If you have any real problems take a look at the font itself.  You
could remap the hex 20 value to something else and then use that
hex value instead.  I had exactly the same problem with a Xerox
DJDE conversion to IBM AFP (hex 40 for IBM though :->).

Happy hunting
Ken

Hi! I'm not currently using COBOL but barcodes are a constant problems
in medical labs.  There are several different barcodes in use, some
which include spaces in the character set while others don't.  However,
we've run across cases in which instrument manufactures advertise one of
the space supporting barcodes, but then find out the instrument doesn't
know how to read a space character.  Our software will print the space
character fine but it does us no good if the instrument can't read a
space in a barcode label.

A good introduction to several barcodes is at
http://www.idbearing.com/symbol.html.  It lists the most popular bar
codes in use, their technical specs and character sets, and show
examples of these symbologies.

Charles Hall
Brooks AFB TX
