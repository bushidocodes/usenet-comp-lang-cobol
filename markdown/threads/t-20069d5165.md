The 1991 (someone will correct me if I'm wrong) update to COBOL-1985
includes about 30 Intrinsic functions. These are used as values in the
procedure division, i.e. MOVE FUNCTION CURRENT-DATE TO TODAY.

There are no user defined functions. COBOL-1997 may change that.

Intrinsic Functions were added to the COBOL standard in a "addendum to the
1985 standard". This got it out the door, standardized, but did not force
the vendors to include it (to be 1985 conforming).

Most vendors have subsequently implemented the functions in their front
line products, but not in back level products (i.e. IBM mainframes
functions appeared in COBOL/370 and the subsequent COBOL for MVS and VM,
but were never added to OS/VS COBOL and VS COBOL II).

The MF products include the intrinsic functions.

As to where to go next - get a compiler and try building some simple apps
with COBOL. The student version of MF is probably a good choice.
