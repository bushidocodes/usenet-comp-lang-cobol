```
I have placed a copy of a PDF document concerning Report Writer on the web
at:

 http://home.netcom.com/~wmklein/dox/cobol-rw-new-standard.pdf

This document was created by the company that provides the IBM (Mainframe
and Workstation) and "other compiler" add-on Report Writer product for
ANSI/ISO '85 Standard compilers.  It includes sections on the history of RW
and information on some common (and past) "extensions" - but most
importantly it had an entire "programmer friendly" section on what is new
for Report Writer in the recently approved 2002 COBOL Standard.  For those
who like examples, this includes LOTS of examples!

FYI - for anyone who has missed it - or doesn't see it in this document, the
report writer functionality that was OPTIONAL in all previous ANSI/ISO/FIPS
Standards is now REQUIRED in the revised Standard (and it is NOT in the
"processor dependent" list).  Therefore, any compiler that claims to conform
to the new ANSI/ISO Standard *must* provide this functionality.  Such
compilers may, however, provide the functionality by use of an "add-on"
product.  (The same way that many compilers today require a "separate" SORT
product or even indexed file product).

Personal SUGGESTIONS (for those interested in native COBOL report writer
functionality):

1) Contact your compiler vendor of choice and find out when they plan on
providing all this new (and required) functionality.

2) Review the document listed above *and* the document at:
  http://www.cobolportal.com/j4/files/02-0222.doc

to see what features are being added in the 2002 Standard *and* what
features are under consideration for the next Standard. Then communicate
with J4 or your national Standards body which features you think should be
added in the NEXT revision of COBOL report writer.

3) Spread the word that COBOL report writer is "here to stay" *as long as*
standard COBOL remains.

    ***

NOTE:
  No one in the COBOL Standards committees claims that this revision (or
probably any future revision) includes a "report generator product" that is
targeted at competing with any of the existing "ad hoc report" products.
Although some features have been added - and may be added in the future - to
increase the range of types of reports that can be created and the ease in
coding any supported report type, the target (as I see it) of the COBOL
report writer functionality is for those cases where integrated "business
logic and report generation" is desired.
```
