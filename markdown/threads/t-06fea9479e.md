> Mmm' CobolScript ? Does this mean you can include COBOL (eg. CobolScript)
> inside HTML (like JavaScript) ?

No.

At least not with CobolScript.  Apparently you will be able to use
Fujitsu .NET Cobol (or whatever subset of Cobol it turns out to be) in
an ASP form but that is server side.

JavaScript is a client side script.  

CobolScript is a scripting language (ie interpreted) like Perl and
Python are, and can be used as a CGI script (as well as several other
things).  It is similar to Cobol, an average Cobol programmer should be
able to use it quickly without too many problems.  It is just that what
it does it does quite well, what it doesn't quite do can waste a lot of
time.
