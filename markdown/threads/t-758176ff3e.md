I hope you can understand (my brand of) English.
You should state what compiler/runtime generated the files you are trying to
write with VB.
As far as I know, almost every compiler manufacturer has it's own
proprietary filesystem.

There are two possibilites:

A) Your filesystem has some kind of ODBC driver, therefore you can treat
your Cobol files as if they were a (Sql) Database (some limitations apply).

B) You develop a module in Cobol to perform file i-o and call it from VB (if
possible).
