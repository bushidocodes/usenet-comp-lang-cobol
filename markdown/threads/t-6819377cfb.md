There are several possibilities, but bearing in mind MS Version 5 is MF Version
3 - the quickest, easiest, but not cheapest - is go to Net Express. Set up a
project in Net Express, include all your source files xxx.cbl. Your copyfiles
automatically get added to the project when referenced in the xxx.cbl files. Hit
the "Rebuild All" button for a compile, and in theory at least, you should see
"All compiled without errors".

Using any other alternative or 'cheaper' compiler - I can't recall, but if MS
Version 5 allows you to use MF extensions like CBL_CHECK_FILE_EXISTS or Screen
Section attributes or Mouse 'sensing', then of course these are gong to hiccup
in any other compiler.

GUI tools - Hmmmm..... One good suggestion already made, SP2 from the 'Volunteer
Fireman' at Flexus <G>. You could use Net Express GUI classes - there's a
learning curve and it 'pushes' you towards using OO which you may not yet be
ready for. If you are familiar with the likes of VB or Delphi etc., you could
use those to build Dialogs and invoke from Net Express.

Jimmy
