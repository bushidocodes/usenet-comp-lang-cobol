```
I spent a couple of hours yest. doing searches on COBOL, UNIX and debuggers
w/o a great deal of success. Our product runs on HP-UX and AIX platforms,
the former is the more impt one. The COBOL systems on both are Micro Focus.
Apart from Micro Focus Animator (we've had some technical problems) and
Fujitsu (which we are investigating), does anyone know of a suitable
debugger for the above platforms or at least the HP-UX? A standalone product
would be great as we don't really need an integrated workbench like Micro
Focus's or Fujitsu's but if we have to take the latter, we will. I accept
that I probably can't find a product that will work on MF source code so I'm
prepared for a mixed env., e.g. compile the pgm to be debugged with the
Fujitsu compiler and then run it in a with the MF executables. Fujitsu
report that a nr of their customers do this but the results cannot be
guaranteed. Also, at runtime, if the pgm to be debugged needs to read/update
MF-created C-ISAM files, this would have to be supported. Many thanks. NB. A
nr of respondents mentioned the Symbolic Debugger/ix (aka xdb) product for
HP but this only seems to be for the HP3000, not the HP9000 -- at least for
COBOL.
```

```
Acucobol has a debugger in its runtime and does not require the use of
"workbench" type system.  You compile the source with a command line option
that compiles symbolic information into the resulting p-code object file,
then run it with their runtime interpreter with a -d option to enable the
debugger.

We develop on AIX, so I know it works there and we have end users running
HP-UX, and I know our object code runs there.  Acucobol object code is
platform independent, so it can be run on any system that has the Acucobol
runtime.

I don't know if Acucobol compiles Micro Focus source without changes, but I
seem to recall that they have an option to use C-ISAM as the indexed file
format instead of their own native format.
```
