```
PRIMA offered a free tool some years back which was useful for
calculating offsets and positions in COBOL record descriptions, and was
picked up by a number of people around the world.

The original engine was written by Robert Wagner, who was a regular
visitor here.

I took Robert's code and converted it into OO COBOL as a COM component,
(so it could be used from the Desktop or the Web) then wrote a C# front
end around it so it had a graphic interface. All of this was shown on
the PRIMA web site with the stages in the source code, as an example of
converting standard COBOL into Fujitsu NetCOBOL for Windows, as a
standard OO Class.

This engine was included in several PRIMA Tools but we found in testing
it that the calculations for multi-dimensional tables were incorrect. I
looked at amending Robert's code but decided against it.

I decided to write the whole thing again in C# and did so.

The COM component itself is called COBSTRUCT, but the delivered package
(which is COBSTRUCT wrapped in a user interface) is called COBDATA.

The new C# version of the COBDATA tool was available free from the PRIMA
web site and more people downloaded it.

Unfortunately, there came a time when we needed to move to HTTPS instead
of HTTP and this broke some of the internals in the COBDATA package.

I wasn't aware of the problem and no-one reported any problems until
quite recently when my attention was drawn to it by someone who wanted
to use the tool.

The result is that I have completely reviewed and enhanced the tool and
it has been moved to .Net version 4.

The web support has been updated also and you should be able to obtain
the tool easily from:
https://primacomputing.co.nz/PRIMAMetro/newCOBData.aspx

This will explain what it does if you are unfamiliar with it, and give
you a chance to download it for free. It will bring you to the download
point below...

If you already have a copy which stopped working some time ago, please
do the following:

1. Remove all trace of the old PRIMA software from your machine. Do this
by clicking the "Publisher" header in Control Panel> Add/remove programs
and this will group everything you received from PRIMA together, so you
can then delete each item.

Now remove the \RAVComponents folder and all its sub-folders.

Your machine is now clean of PRIMA software.

2. Go to the direct download point for the new version 4 software at:
https://primacomputing.co.nz/downloads/deploy2019/newcobdata/Accounts/Demo/Installer/getit.html

3. As explained at the download point, it is a 2 phase install.
Watch the icons in your task bar because it can look as if nothing is
happening but if you click the icon you can see that the install is
proceeding.

At the end of phase 1 you will have a blank screen on your Browser tab.
(This can be deleted if you want.)

Move back to the original tab and run phase 2.

When phase 2 completes, the package will start up with its splash
screen. Click on the splash to remove it and you can suppress it in
future from the config option in the menu.

Both phases of the install should run within 10 minutes, but it depends
on your download speed of course.

You can test the tool by getting a sample COBOL record layout from the menu.

Contact me if you have any problems or feedback.

Enjoy!

Pete.
```

```
My take is that for your noble and generous effort - providing free
software that others find to be of value and use - the wrong tool was
chosen from the start.

COBOL is a COmmon Business-Oriented Language... and using it for
text-processing is like using the nail-pulling side of a claw hammer as a
screwdriver. Yes, it'll work... sometimes... and that very fact of
working can encourage further misuse ('Hey, I did it before!').

In the example I gave the tool is suited to the task... and the platforms
have been kept backwards-compatible with excruciating attention to detail
by a few large, old, wealthy and stable corporations.

That's the reason for COBOL's being known for stability; a lot of Very Big
Money is invested in making sure the accountants are kept happy (even if
one cannot tell by their facial expressions). Your own efforts were to
use a tool ill-fitted for the task and then... the endless confoundments
of platform-changes and upgrades and operating-system changes - none of it
your own doing, acknowledged - forced you to run twice as fast just to
stay in the same place until, at long last, you had to change languages
and take it from the top.

to it is admirable... and the miracle of this particular dancing bear
eventually became something that was neither ursine nor waltzing.

Meanwhile... the work-a-day ants keep trudging over the fields, nothing
spectacular, nothing of particular note... until one sees how well-suited
their particular and pedestrian pace is tailored, o'er the aeons, to the
```
