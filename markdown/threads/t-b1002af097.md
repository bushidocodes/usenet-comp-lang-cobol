You can do (almost) anything in COBOL. However, you will probably find
that your COBOL environment does not support recursion which will be a
real bummer for doing most textbook algorithms. I can't recall whether
or not it is supported in the latest standard, but it certainly isn't
something that is available in most places. You may also run into some
issues regarding storage allocation. Until recently, COBOL didn't
support in any standard way the allocation of memory (like C's malloc).

Hi Michael
You can do pretty much anything in COBOL except use pointers. There is
no pointer data type. This can make some data structures difficult to
code. The usual way to simulate this would involve the use of
subscripts.To build and use a tree structure for example, an array of
nodes could be defined and subscripts used for access. eg

01 TREE-STRUCTURE.
03 NODE OCCURS 1000 TIMES.
05 PIC X(50).
01 SUB1 PIC S9(8) COMP.
02 SUB2 PIC S9(8) COMP.

Where the values OCCURS 1000 and PICX(50) are only examples. The nodes
are accessed for example as follows.

MOVE NODE(SUB1) TO NODE(SUB2).

The downside of this technique is that the array has to be predefined
and may be larger than necessary. In some cases it can make sense to
consider the array as a heap structure from which nodes can be
allocated and returned. Paragraphs can easily be written to accomplish
these tasks. Indeed this could be taken a stage further and the heap
structure embedded in a subprogram and the complete heap management
subject to parametized sub-module calls.
To be honest, I think that COBOL is very awkward for this type of
function and the use of another language would be better. Write
complex data structure algorithms maybe in Pascal or C,C++.
Hope this helps.
Regards Bob
