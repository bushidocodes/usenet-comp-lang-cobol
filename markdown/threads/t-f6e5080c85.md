```
Reference materials for IBM products are available in soft copy format from
IBM.

IBM has an internet site with reference manuals including: COBOL, CICS, DB2,
JCL etc.

http://www.s390.ibm.com/os390/bkserv/

The other option is a set of CD roms with more than you ever wanted to know
about software you never knew existed.
I prefer the CD collections since they can be installed on a LAN or PC.
The cost of the CD set was around $100, the last time I had a client order
them.

Order information can be found at:

http://www.s390.ibm.com/products/softcopy/softord.htm

A note on the CD rom set:
The contents of the CD rom may be overwhelming. What I have done, is to copy
the publications applicable to the current releases of the software being
used at the installation to a LAN, and set up a Bookshelf with those titles.
This makes it easier to find the applicable reference manuals when they are
needed.

It amazes me that businesses will spend millions on a computer, and then
skimp on manuals.
The same people would return a $100 VCR if it came without instructions.
I have often found obsolete 10 year old manuals still in use because "the
manuals are too expensive".
```

I've been in a company where not providing manuals is common practice.

I think the main reason for this attitude is that a handful of people will
literally steal every manual in sight, and so employers eventually give up
buying them.  I saw several books vanish (and a couple of laptops, to boot).
Someone stole a $60 book of mine right off my desk shelf one day (which was
the end of me leaving my books out).  Never did get it back.

I also think, regardless of how other people may laugh at me for the
thought, that each programmer should be provided with all the vendor manuals
on each subject they have any responsibility for.  This way, there won't be
any theft, because each person will have what they need.  In only a few
cases, like tech support, is this impractical on a size/space basis.

If an employer simply kept a list of what manuals had been issued to each
programmer, and made it plain that anything not turned in at the end of a
project or employment would be docked from salary, then theft or simple loss
wouldn't be much of a problem.  Employers could even provide a few more
lockable cabinents to prevent the sort of theft that might be perpetrated
just for malicious reasons.

Ah well.  End of rant.
