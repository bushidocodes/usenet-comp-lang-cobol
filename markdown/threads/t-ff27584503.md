```
donald tees wrote in message <7uka42$bkk$1@news.igs.net>...
>Does anybody have a Win95 Cobol subroutine (I am using Fujitsu) to find
>files in a directory using wildcards?

You may use the Windows API, here is an example of how to solve it in
Acucobol:

       IDENTIFICATION DIVISION.
       PROGRAM-ID. FILELIST.
       AUTHOR.     (c) 1999, Cheesle.
       REMARKS.
      *This program must be compiled with -Dw32
      *For use with 32 bit Windows only
       DATA DIVISION.
       WORKING-STORAGE SECTION.

       77 WS-COUNT            PIC 9(03).
       77 WS-PATH             PIC X(60).

      *File attrbutes may be combined logically
       01 ATTRIBUTES.
          03 RO-FLAG          PIC 9(02) VALUE 01.
          03 HID-FLAG         PIC 9(02) VALUE 02.
          03 SYS-FLAG         PIC 9(02) VALUE 04.
          03 VID-FLAG         PIC 9(02) VALUE 08.
          03 DIR-FLAG         PIC 9(02) VALUE 16.
          03 ARC-FLAG         PIC 9(02) VALUE 32.
          03 ANY-FLAG         PIC 9(02) VALUE 63.

       01  FILE-FIND-DATA.
           03 FILE-ATTR       SIGNED-LONG.
           03 CREATION-TIME.
              05 CREAT-LOW    SIGNED-LONG.
              05 CREAT-HIGH   SIGNED-LONG.
           03 LAST-ACCESS.
              05 ACCES-LOW    SIGNED-LONG.
              05 ACCES-HIGH   SIGNED-LONG.
           03 LAST-WRITE.
              05 WRITE-LOW    SIGNED-LONG.
              05 WRITE-HIGH   SIGNED-LONG.
           03 FILESIZE.
              05 SIZE-HIGH    SIGNED-LONG.
              05 SIZE-LOW     SIGNED-LONG.
           03 WIN-RES-0       SIGNED-LONG.
           03 WIN-RES-1       SIGNED-LONG.
           03 WIN-FILE-NAME   PIC X(260).
           03 DOS-FILE-NAME   PIC X(13).

       01  FILE-HANDLE        HANDLE.
       01  FIND-STATUS        SIGNED-LONG.
           88 IS-FALSE        VALUE 0.

       01  FILE-SELECTION     PIC X(260).

       SCREEN SECTION.
       01 FILE-INFO.
          03 LINE WS-COUNT.
             05 PIC X(40) FROM WIN-FILE-NAME.
             05 " (".
             05 PIC X(13) FROM DOS-FILE-NAME.
             05 ") ".
      05 PIC Z(03)BZ(03)BZZ9 FROM SIZE-LOW.
      05 " bytes".
       01 GET-PATH.
          03 LINE 01.
             05 "Provide path including filename, *? ok:".
          03 LINE 02.
             05 PIC X(50) USING WS-PATH.

       PROCEDURE DIVISION.
       MAIN SECTION.
       MAIN-001.

    PERFORM LOAD-DLL.
    PERFORM INITIALIZE-FILE-VARS.
    PERFORM INITIALIZE-REC.
    MOVE    "C:\*.*"         TO WS-PATH.
    MOVE    2                TO WS-COUNT.
    DISPLAY GET-PATH.
    ACCEPT  GET-PATH.
           MOVE    ANY-FLAG         TO  FILE-ATTR.
           INITIALIZE               WS-COUNT.
           STRING  WS-PATH          LOW-VALUES DELIMITED BY SIZE
                   INTO             FILE-SELECTION.
           CALL    "FindFirstFileA" USING
                   FILE-SELECTION
                   FILE-FIND-DATA
                   RETURNING        FILE-HANDLE
                   ON               EXCEPTION GO TO MAIN-800.

           IF      FILE-HANDLE      < 1
                   DISPLAY          MESSAGE   BOX
                                    "No files found!"
                                    TITLE     FILE-SELECTION
                   GO               TO MAIN-100.

           MOVE    1                TO FIND-STATUS.
    PERFORM UNTIL            IS-FALSE
                   PERFORM          DISPLAY-RESULT
            PERFORM          INITIALIZE-REC
                   CALL             "FindNextFileA" USING
                                    BY VALUE FILE-HANDLE
                                    BY REFERENCE FILE-FIND-DATA
                                    RETURNING FIND-STATUS
                                    ON EXCEPTION GO TO MAIN-810
                                    END-CALL
                  END-PERFORM.
    ACCEPT OMITTED.

       MAIN-100.

           PERFORM UNLOAD-DLL.
           GO      TO               MAIN-900.

       MAIN-800.

           DISPLAY MESSAGE          BOX
                   "Couldn't call FindFirstFileA!"
                   TITLE            "Fatal error".
           GO      TO               MAIN-900.

       MAIN-810.

           DISPLAY MESSAGE          BOX
                   "Couldn't call FindNextFileA!"
                   TITLE            "Fatal error".
           GO      TO               MAIN-900.

       MAIN-900.
       MAIN-EXIT.
           GOBACK.

       DISPLAY-RESULT SECTION.
       DISPLAY-RESULT-001.

           ADD     1                TO WS-COUNT.
           DISPLAY FILE-INFO.

           IF      WS-COUNT         =  24
                   DISPLAY          "(Press ENTER)..." AT 2501
                                    WITH NO ADVANCING
                   ACCEPT           OMITTED WITH NO ADVANCING
                   DISPLAY          OMITTED AT 0101 ERASE SCREEN
                   INITIALIZE       WS-COUNT.

       DISPLAY-RESULT-900.
       DISPLAY-RESULT-EXIT.
           EXIT.

       INITIALIZE-FILE-VARS SECTION.
       INITIALIZE-FILE-VARS-001.

           INITIALIZE               FILE-SELECTION
                                    FILE-HANDLE
                                    FILE-ATTR
                                    WS-PATH.

       INITIALIZE-FILE-VARS-900.
       INITIALIZE-FILE-VARS-EXIT.
           EXIT.

       INITIALIZE-REC SECTION.
       INITIALIZE-REC-001.

           INITIALIZE               WIN-FILE-NAME
                                    DOS-FILE-NAME
                                    CREAT-LOW
                                    CREAT-HIGH
                                    ACCES-LOW
                                    ACCES-HIGH
                                    WRITE-LOW
                                    WRITE-HIGH.

       INITIALIZE-REC-900.
       INITIALIZE-REC-EXIT.
           EXIT.

       UNLOAD-DLL SECTION.
       UNLOAD-DLL-001.

           CANCEL  "KERNEL32.DLL".

       UNLOAD-DLL-900.
       UNLOAD-DLL-EXIT.
           EXIT.

       LOAD-DLL SECTION.
       LOAD-DLL-001.

           SET     ENVIRONMENT      "DLL-CONVENTION" TO 1.
           CALL    "KERNEL32.DLL"   ON EXCEPTION GO TO LOAD-DLL-800.
           GO      TO               LOAD-DLL-900.

       LOAD-DLL-800.

           DISPLAY MESSAGE          BOX
                   "Couldn't LOAD Kernel32!"
                   TITLE            "Fatal error".
    GOBACK.

       LOAD-DLL-900.
       LOAD-DLL-EXIT.
           EXIT.
```
