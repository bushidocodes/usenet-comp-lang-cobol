I use templated output for quite a number of my systems.  Once the basic 
mechanism has been worked out it becomes quite easy to use different actual 
outputs.

Basically for templated output you read in a text file that is the overall 
format for the output with tags that mark where the data items are to be 
merged in.  Additional tags mark where there is to be repitition, or where 
a table of data is to step to the next occurs.

The actual data substition may need to be done 'packed' (ie variable) or 
fixed so some additional controls in the template may be needed.

Then you can merge into a text template for simple printing, or into HTML 
or XML, or EDIFACT, or postscript, or some other.

Sometimes I do a 2 stage merge where the Cobol merges into a template to 
produce a particular format that then goes into a utility which does a 
merge to the final format.

As it is a generalised merge it is only necessary to construct a template 
to get the different outputs.

If you are looking to insert data into something like a form or a letter in
Word, you can try this.  (It is basically what someone else suggested in
another post, but we do it also.)

Create a Word document and put keywords where you want to substitute data.
For example, we have a letter that might look like this:

@name
@addr1
@addr2
@city   @state  @zip

Dear @contact:

You can do whatever formatting you want on this.  Then save it as Rich Text
Format, which is just ASCII text.

Then write a Cobol program that reads the .rtf file, substitutes values for
the keywords as they are detected, and write it out to another file.  You
can even name the output file with an extension of ".doc".  Word doesn't
care if a .doc file contains the binary .doc format or RTF - that way, the
end user doesn't have to get their panties in a bunch because it's not a
.doc file.

If you want to get into it deeper, you can find info on the MS web site
somewhere on what all of the Rich Text Format tags mean and generate RTF
files from scratch instead of a template.

Or wait until Office 11 is available, which supposedly uses XML.
