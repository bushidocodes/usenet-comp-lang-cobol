Announcing the ETK COBOL Code Generator.

ETK addresses two objectives:
1) portability; end conversions forever
2) productivity; develop applications easier

Applications made with ETK are portable to any platform with a
viable Cobol compiler (e.g. PC DOS, OS2, Windows, any Unix incl. AIX,
Vax VMS/ACMS, DEC Alpha, Data General MV, HP3000, IBM VM/CMS,
IBM MVS/TSO, IBM CICS, IBM AS/400, Bull GCOS 7, Bull GCOS8, and
more...; sorry no Mac).

By portable we mean 100% portable, not 99.9%. That is, the application
ports with NO CHANGES of any kind on your part. You specify the target
system and ETK generates all target specific code. The generated code
is native and efficient on the target platform. You can generate on the
target itself or on a more congenial platform, eg. PC or Unix.

ETK is meant for large, mission-critical, enterprise-wide applications.

You can work with ETK on two levels:

Low-level, where YOU paint the screen (with the ETK screen-painter),
write the business rule code (in Cobol), etc.

High-level, where you specify teh application in a repository or (as we
call it) a dictionary, and ETK paints the sceeen, generates the code,
generates JCL to compile/link, etc.

To port your application to another platform, offload the dictionary to
a flat txt file, file transfer the flat file, rebuild the dictionary on the
target system, regenerate and compile, and you are done!
Data can be moved as well; all databases are supported:
ISAM, VSAM, SQL, IMS, IDS2, DATACOM-DB, BTRIEVE, ODBC,
etc.

Regeneration is faster than compilation. The generated code is
very readable Cobol source and can co-exit with your current
legacy systems.

ETK runs on all platforms (is written with ETK, of course) so
you can developed where it is most convenient; PC for speed,
mainframe for security and safety.

ETK is made with the same technology that is behind the Libero
tool (http://www.imatix.com).
