```
As I said earlier, I don't know of any way (with any current hardware) to
"block" numerics at the input stage.  Therefore, would you are going to have
to do is CHECK (validate) the input that is provided - and send back an
"error message" if incorrectly formatted items are provided.  I don't (never
did) hardware, so I don't know why most systems have a way to "block" to
numerics only but don't have a way to block to alphabets only.  I assume it
has something to do with "book keeping" type data entry - but that is a
guess only.

Bottom-line:
    Do IF Alphabetic and IF Numeric and IF Space checks on each byte of your
input - and send an error message when input is incorrect.  (Or define
user-defined CLASSes via Special-Names - or if you have a system where it is
available, use the VALIDATE verb - now available as an extensions and
included in the draft of the next Standard.)
```

```
I assume you try the following

Working-Storage Section.

01 Input-Data PIC AA9AA9.

Procedure Division.

ACCEPT Input-Data FROM CONSOLE
.
.
.

And want the system to allow an input of AB1CD2

but reject, possibly block the terminal with ABC   or  X1

This would require the runtime system to monitor the terminal input and check every single
keystroke.  This is more or less unfeasible, as the CPU usually doesn't handle the
keyboard (in a mainframe environment), and only processes the complete set of data
presented only when the ENTER key is pressed.

COBOL has to maintain standards and portability across platforms, and thus the behaviour
you want cannot be provided by COBOL, as it is impossible to achieve in, let's say, a
mainframe IBM 3270 environment.
```
