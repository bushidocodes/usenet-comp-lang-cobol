opened by another user before attempting to OPEN the file ?   I have a
program which uses "COBOLASSIGN"  (Tandem Cobol) to dynamically assign

opened by another user with LOCK, my program ABENDS. Normal error
trapping code after the OPEN has no effect... the program just ABENDS
with a system message about the file being open by another user. Is

Normal functionality is to return a status code of locked when you
attempt to open a locked file.  Your problem may be in the program's
declaratives section where there may be code that is doing the abend
when the file is locked.  

It may also be in the "COBOLASSIGN".  Can you code an OPEN WITH LOCK
to check if locked before doing the COBOLASSIGN?
