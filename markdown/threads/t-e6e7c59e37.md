```
Paul wrote:

> Do I need to describe the copybook in the fujitsu compiler ( it seems
> NO_NEED )
>>
>> You can't link copybooks either.

In the Project Manager you can create 'folders' for each program (.EXE or 
.DLL).  These can be 'Source code', 'Object' or 'Libraries'.

Copy books can be added to the _source_code_ folder, but it is the COPY 
statement in the program that causes these to be included in the program.

Putting these into the folder is optional, but if included then if they are 
changed the manager will know that the program has to be recompiled to use 
the latest source version.

You seem to have added them to the 'Library' folder as they are being 
linked in.  What should go into the library are .LIB files which may be 
created if you have Cobol .DLLs that are CALLed, or are specific libraries 
(.LIB) required by your program.
```
