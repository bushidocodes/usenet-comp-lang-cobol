This is a "follow-up" from a thread a while ago about how to "specify" the
debug dataset (created with SYM,SEPARATE) at run-time.   I just received the
following from the LE documentation people (probably NOT the answer you
want, but ....)

"Thank you for bringing this comment to our attention.

The z/OS Language Environment Debugging Guide will be updated in the next
edition to add information, including that the user cannot change the DDName
or DSName specified <at compile-time> as SYSDEBUG at run time for Language
Environment dumps."

PLEASE NOTE WELL:
  That is the way things work today.  I believe that IBM would seriously
LOOK AT changing this, if an "enhancement request" came in via proper
channels (i.e. SHARE, or service REQUEST).
