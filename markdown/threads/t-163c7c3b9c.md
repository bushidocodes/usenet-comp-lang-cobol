```
If you are using ANSI standard ACCEPT statements it is not possible to right
justify an alphanumeric. If, however you use the extended ACCEPT/DISPLAY
syntax or the SCREEN SECTION there are various options available to control
how data is input and the JUSTIFIED clause is available. Your language
reference manual will have full details on how to use the extended syntax.
```

```
Have you tried that?  You will be suprised when it does not work.
Justified Right is VERY misunderstood.  It DOES NOT right justify an
item when the target of a move or an accept.

What it DOES do is facilitate the move or usage of the right most
portion of a source field into a target field.

Ie:

       IDENTIFICATION DIVISION.
       PROGRAM-ID.  TESTCX.
       DATA DIVISION.
       WORKING-STORAGE SECTION.
       01  DATA-ITEM PIC X(10) VALUE "1234567890".
       01  DATA-TARGET PIC X(5) JUSTIFIED RIGHT.
       PROCEDURE DIVISION.
       START-OF-PROGRAM.
	   MOVE DATA-ITEM TO DATA-TARGET
	   DISPLAY DATA-TARGET
	   STOP RUN
	   .

The result in Data-Target will be 67890.

If Data-Item contained "12345     " the result would be "     ".

If both fields were 10 characters long, the source and target would
always contain the same values.

Play with the program and see what I mean.
```
