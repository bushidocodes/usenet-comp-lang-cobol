```
Hi all,

I'm using MF 4.0.32 COBOL and trying to get the users mail program to
get started when I issue a 'mailto:' command to the ShellExecuteA
Windows function.  The return value is +0042 which according to the
SDK means it should be the handle of the mail program.  However,
nothing happens.  On my machine, the call works fine and brings up my
email program with the 'to:' field pre-filled as I specified.  I'm
checking the return value of the call for a value of 32 or less to
indicate an error.

Since this works by using the 'file association' method of invoking
the appropriate program, where in the registry is the default email
program key?  Is there a way to check to see if the appropriate
registry entry exists before issuing the call?  If nothing happens
even though I get a 'good' result, it will confuse the customers!

Thanks,
Doug
```
