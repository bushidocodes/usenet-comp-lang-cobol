```
I got my last question about error handling in intrinsic functions
answered quite nicely, thank you, but this time somebody's really got to
put their thinking caps on.

I have spent a lot of time staring at load modules trying to determine
where information like compile dates, object module lengths, etc. are
stored. I'd say that information sits in the beginning of the load
module, and I see what I think are some of the fields, but it would be a
lot easier if someone could recommend a manual or even post source code
for a program to map a load module. If anybody uses Easytrieve and is
interested, I can post an example of a job I run that will test load
module link dates according to control card-type selection criteria.

Now that I've got that, I'd like to be able to interrogate the various
subroutine compile/link dates within the composite load module. Again -
has anybody got any suggestions, documentation, etc? Thanks for the
help. I really enjoy what I've learned from this newsgroup.
```
