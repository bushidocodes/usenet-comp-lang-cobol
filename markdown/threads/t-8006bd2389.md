```
"MERANT EnterpriseLink Generates Components From Legacy Assets For Fast
E-Business Enablement
December 12, 2000 08:03:00 AM ET

NEW YORK--(BUSINESS WIRE)--Dec. 12, 2000-- --MERANT Egility Legacy
Transformation and Integration Solution now turns mainframe systems into
powerful new Web-enabled components easily--

MERANT (NNM:MRNT; LSE:MRN), a leading global e-business software solutions
company, today announced major enhancements to its Egility(TM) Legacy
Transformation and Integration Solution with the introduction of
EnterpriseLink(R) featuring Component Generator. EnterpriseLink is designed
to reduce the time to extend mainframe applications to the Web by more than
50 percent. This Egility solution, demonstrated today at the eBusiness
Conference & Expo in New York, integrates and extends mainframe applications
to the Web without having to undertake a major programming effort and
without making changes to existing mainframe production systems.

EnterpriseLink with Component Generator allows the capture of complete
business processes and workflows, then automatically generates reusable,
industry-standard components in the form of JavaBeans, Enterprise JavaBeans
and XML for use throughout organizations seeking to e-enable
business-critical applications. This unique technology allows
business-critical transactions within mainframe applications, written in
COBOL, to be rapidly transformed into "e-business" transactions without
taking existing systems out of production. EnterpriseLink helps customers
quickly extend the reach of these assets by enhancing them to participate in
e-business solutions.

"Application software is a corporate asset and companies should look closely
at these assets and how they can be leveraged for use across the Web, such
as creating a bridge between the legacy applications and the new Web-based
ones," said Tracy Corbo, senior analyst, Hurwitz Group. "EnterpriseLink's
Component Generator function enables developers to expose legacy
applications written in COBOL as services to Java-based applications. This
provides the double benefit of both time-to-market savings coupled with ease
of integration for companies with legacy code that can be re-used for Web
development efforts."

EnterpriseLink with Component Generator captures business workflows from
CICS mainframe applications and automatically generates re-usable e-business
components using the knowledge database contained in MERANT's Revolve(R),
the industry leading COBOL analysis tool, included with EnterpriseLink.
These can be generated as Java Beans, Enterprise Java Beans or XML
components that can be used in an open standards-based environment. In
addition, EnterpriseLink automatically provides the middleware communication
between the components and the legacy application using IBM MQSeries.
EnterpriseLink-generated components can be developed and tested in a full
end-to-end fashion using MERANT's Mainframe Express(TM), a leading
workstation-based COBOL development platform.

"This focus on providing a complete solution for legacy and web-based
systems is profoundly important for our customers seeking to e-enable their
business," said Andrew Weiss, chief technology officer, MERANT. "The
time-to-market demands, coupled with the large investment in COBOL-based
mainframe technology cannot be ignored by companies seeking to maintain
their competitive advantage. Our investment in this technology and with the
enhanced features of Component Generator is clearly in line with our
solution-based strategy that will pay dividends for customers and for
MERANT."
```

```
Said it before and I say it again:

"The future of computing is in component technology."

The logical extension of this idea is application development by simply
plugging together the appropriate components. Eventually, "clever" software
will do this, by interaction with the end user.
(I believe that will happen around 2015..."Computer programming" as we
understand it today will then be irrelevant. People will still do it, but it
won't be necessary. Unskilled (in computer technology) end users will be
able to construct their own applications by interaction with smart software.
I debated this at length in this forum two years ago and have seen nothing
in the interim to make me change my mind.)

Today they are using Java to  build the components (at least in this
example), but within 3 years the components will be constructible in ANY
language which is appropriate to the task the component is intended to carry
out (including COBOL), and the whole lot will be capable of running together
as one seamless solution. (Watch what happens when .NET becomes
established.)

Some of us are already building components in COBOL and using component
based design for application development.

Even at this early stage I have components developed in VB, COBOL, C++ and
Java, all of which peacefully co-exist and communicate with each other
without problem. (I am currently looking at Delphi as another potential
component contributor. It is a very good development tool.)

Windows Application development  (at least for me) is rapidly becoming a
simple case of selecting the appropriate components and dropping them on to
a form. Similarly, the same components can be easily dropped into HTML to
Web Enable Windows applications.

The concepts of Object Orientation are extremely applicable to component
development and it makes sense to use OO COBOL for this. However, it is not
a REQUIREMENT and good components can be built using structured COBOL. (They
just need to be "packaged" more carefully so that multiple instances can be
invoked...)

I would be interested to hear from anyone in this forum who is currently
using COBOL for component development, or would be interested in doing so.

Please mail me privately: dashwood@enternet.co.nz

Pete.
```
