The EXEs in the product contain both DOS and OS/2 code. NT defaults to the
OS/2 part. Thane Hubbel has replied in a separate note mentioning FORCEDOS
which causes NT to use the DOS part, but if you do not want to have to do
that, add the EXEDLL directory from your COBOL installation to the
Os2LibPath system environment variable.
