I once did a COBOL symbolic substitution program for JCL & PROCs.
I also have rexx macros which perform substitution in source code models, as
well as macros to convert PROCs to JCL.

I would recommend getting away from the &something in the free form input.
You might have to handle &some, and &something.

You can use a special delimiter, like ~ ? " etc, to enclose your
substitution variable. (~rundate~)
This makes parsing the input easier. You have the same delimiter to start
and end the variable.

You could also allow overriding the default delimiter.
$DELIMITER = @

SORT FIELDS=COPY
OUTFIL INCLUDE=(8,10,CH,EQ,C'@RUNDATE@'),FNAMES=FILE1
OUTFIL SAVE,FNAMES=FILE2
