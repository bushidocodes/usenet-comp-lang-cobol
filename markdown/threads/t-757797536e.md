```
Use reference modification -

03  WS-SUB PIC 9(04) BINARY.

EVALUATE COUNTER
        WHEN < 10
                MOVE 3 TO WS-SUB
        WHEN < 100
                MOVE 2 TO WS-SUB
        WHEN OTHER
                MOVE 1 TO WS-SUB
        END-EVALUATE.

DISPLAY COUNTER (WS-SUB:)

You don't need to specify the length as the compiler will calculate that for
you.
```

```
Yes, he was looking to display a left-justified value of his 3-byte
Display-Numeric field named COUNTER. Moving COUNTER to a receiving
field defined as ZZ9 would have worked as well, but that would have
been right-justified for values less than 100. His example
specifically illustrated his need for a left-justifield solution.

Using the DISPLAY verb with reference modification basically
translates to -

"I want to begin my data-display at the byte/position defined to
WS-SUB. The compiler will determine the length of my display so I
don't need to specify this length".
```
