Now I'm not a bits-and-bytes man; others will cover that more
extensively - but here's a real world example of doing it with
NetExpress :-

I am transferring RM/COBOL data files to NetExpress. Firstly the
original format as used in RM/COBOL (not their current format, I should
add):-

       *---------- CSITM2.CBL ----------- 95 MAY 24 -----------------*

       FD ITEM-NAMES.

       01 ITM-RECORD.
          03 ITM-KEY-3.
             05 ITM-SYSTEM-NUMBER              PIC X(03).
             05 ITM-KEY-2.
                10 ITM-VESSEL-NUMBER.
                   20 ITM-VESSEL-KEY           PIC X(03).
                   20 ITM-VESSEL-SUFFIX        PIC X(01).
                10 ITM-KEY-1.
                   20 ITM-ITEM-NUMBER          PIC X(06).
           03 ITM-DATA.
            05 ITM-NAME                        PIC X(40).
            05 ITM-DRAWING-NUMBER              PIC X(04).
            05 ITM-NOMINAL                     PIC 9(01)V9(04) COMP-3.
            05 ITM-CRITICAL                    PIC 9(01)V9(04) COMP-3.
            05 ITM-NUMBER-OF-POINTS            PIC 9(03) COMP-3.
            05 ITM-TEMPERATURE-TABLE.
               10  ITM-TEMPERATURE OCCURS 10   PIC S9(03) COMP-3.
            05 ITM-PROBE-TABLE.
               10  ITM-PROBE-NUMBER OCCURS 10  PIC 9(02) COMP-3.
            05 ITM-SUMMARY-FLAG                PIC X.
               88 RECALC-ITEM                  VALUE "Y", "y".
            05 ITM-PIPE-SIZE                   PIC 9(02)V9(01).
            05 ITM-CODE                        PIC X(06).
            05 ITM-QUESTION-MARK               PIC X.
            05 ITM-MATERIAL                    PIC 9(02).
            05 ITM-MAX-WORKING-PRESSURE        PIC 9(05) COMP-3.
            05 ITM-MAX-WORKING-TEMPERATURE     PIC 9(04) COMP-3.
            05 ITM-NUMBER-OF-INSPECTIONS       PIC 9(02).
            05 ITM-DATE-TABLE.
               10 ITM-INSPECTION-DATE OCCURS 10 PIC 9(04) COMP-3.
            05 ITM-DIAMETER-FLAG                PIC X(01).
            05 ITM-DIAMETER                     PIC 9(03)V9(03) COMP-3.
            05 ITM-STRESS-VALUE                 PIC 9(02)V9(01) COMP-3.
            05 ITM-CRITICAL-CALC-FLAG           PIC X(01).
            05 ITM-CSA-CRITICAL                 PIC 9(01)V9(04) COMP-3.
            05 ITM-JOINT-EFFICIENCY             PIC 9(03) COMP-1.

      *------------------------ END CSITM2/CBL -----------------------*

And using an old Merant(MicroFocus) tool from DOS days, the above is
'translated' as follows for NetExpress :-

       FD RM-File.           *> ( RM Items file )
       01 RM-record.
          05  RM-SYSTEM-NUMBER            pic x(00003).
          05  RM-VESSEL-KEY               pic x(00004).
          05  RM-ITEM-NUMBER              pic x(00006).
          05  RM-NAME                     pic x(00040).
          05  RM-DRAWING-NUMBER           pic x(00004).
          05  RM-IMPERIAL                 pic x(00003).
          05  RM-CRITICAL                 pic x(00003).
          05  RM-NUMBER-OF-POINTS         pic x(00002).
          05  RM-TEMPERATURE occurs 10    pic x(00002).
          05  RM-PROBE-NUMBER occurs 10   pic x(00002).
          05  RM-SUMMARY-FLAG             pic x(00001).
          05  RM-PIPE-SIZE                pic x(00003).
          05  RM-CODE                     pic x(000006).
          05  RM-QUESTION-MARK            pic x(00001).
          05  RM-MATERIAL                 pic x(00002).
          05  RM-MAX-WORKING-PRESSURE     pic x(00003).
          05  RM-MAX-WORKING-TEMPERATURE   pic x(00003).
          05  RM-NUMBER-OF-INSPECTIONS     pic x(00002).
          05  RM-INSPECTION-DATE occurs 10 pic x(00003).
          05  RM-DIAMETER-FLAG            pic x(00001).
          05  RM-DIAMETER                 pic x(00004).
          05  RM-STRESS-VALUE             pic x(00002).
          05  RM-CRITICAL-CALC-FLAG       pic x(00001).
          05  RM-CSA-CRITICAL             pic x(00003).
          05  RM-JOINT-EFFICIENCY         pic x(00002).

The trick is getting those input fileds described as the right length
for NetExpress. And taking the new input record, I can now move
something like RM-JOINT-EFFICINCY which was originally described as 
9(03) comp-1 to a new N/E record-field as 9(03) comp-3 (it could have
been to a 9(02) comp-3 - because this application isn't concerned with a
percentage above 85%).

Hope the above helps a bit.

Jimmy, Calgary AB
