Yes. I don't know what the exact problem is. Our executable fails in MF
Once again, stony silence from MF Support but to be fair the problem
may be with Solaris 2.5.

When we call the first MF cobol routine we we get a Segmentation
Violation in the MF signal handling code. The stack dump we get is:

_aiosigaction <- Don't know where this is coming from in OS
prevsigactionset <- Seems to be a private function in mainobj.o
mFt_os_signal_init <- Public function in libcobol, mainobj.o
etc...

The work around is swap from using the sigaction to signal system call
by modifying the $COBDIR/cobconfig file to contain the line:

set signal_interface=TRUE

The problem does not occur on all Solaris 2.5 hosts. Some run MF Cobol
run time without any problems at all. It's a mystery!

Regards,
Peter
