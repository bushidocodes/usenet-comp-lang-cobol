```
Hi,
I guess you are using some *nix compiler.
Anyway, the syntax of call is as follows: CALL "C_subprogram" USING
avar1 nvar2 avar3 giving ret-val. equivalent C call ret_val =
_subprogram(*avar1,nvar2,*avar3)
    in the above call "_subprogram" is the routine developed in C or
    ASM. avar1 is assumed alphanumeric so the pointer/address of avar1
    similaryly nvar is numeric and avar3 is alphanumeric.
While making executable you should use the libraries having bundled all
your objects/User Routines developed in C or other languages.
IMPORTANT: C, COBOL and assembly have same Parameter Passing convention
i.e. in reverse order. PASCAL has a different Parameter Passing
convention.
Hope this is useful to you.
```

```
>Anyway, the syntax of call is as follows: CALL "C_subprogram" USING
>avar1 nvar2 avar3 giving ret-val. equivalent C call ret_val =
>_subprogram(*avar1,nvar2,*avar3)

If the compiler does not use some kind of template to specify if parameters
should be passed by reference or by content (value), you must use the
explict BY REFERENCE or BY CONTENT in the COBOL CALL USING.

>_subprogram(*avar1,nvar2,*avar3)

If that's the 'c' procedure header (but it can't be, see later) (c calls use
"&" to indicate an address should be passed, the headers use "*") , the
corresponding COBOL CALL would be
  CALL "C_subprogram"  USING
     BY REFERENCE avar1
    BY  CONTENT  nvar2
    BY REFERENCE avar3
        RETURNING ret_val

Further, you'll also have to make sure that avar1, nvar2 and avar3 have the
correct PICTURE and USAGE clauses.. what to use is specified in the C
header, where each parameter's data type is explicitly defined. e.g.,:

  INT  C_subprogram  ( int * avar1, int nvar2,  float * avar3)

What is "int" or "float" is a function of both the COBOL and C compilers, so
"which" compilers are in use is important.

Bottom line is OP needs to study the manual under the CALL verb a little
more; and more than likely study up on datatypes.
```
