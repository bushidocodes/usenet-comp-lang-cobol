```
I am using VA Cobol 2.2, along with VA CICS EAD V3.1 on NT 4 SP5, and I have
encountered the following problem.

I have written 2 CICS Cobol programs. Pgm A calls Pgm B (using CICS LINK),
and passes data to Pgm B for its use.

Pgm B massages the data, and writes the information out to a DB2 database.
So far so good.

Pgm B then issues a CICS SEND command to print a report
         EXEC CICS SEND TEXT
               FROM (WS-SEND-TEXT)
               LENGTH (1000)
               PRINT
               ERASE
               WAIT
             END-EXEC.

Control is then returned to Pgm A with a CICS RETURN.

Pgm A can then immediately call Pgm B again, passing new data to it.

If this occurs, Pgm B will hang in the print of the first report.

My understanding of the WAIT in the CICS SEND, was that it will delay until
the printing is complete, but this is not the case (unless I have coded it
wrong).

I do not wish to place a delay in the program, any ideas on how to get
around the problem ?

Thanks,
    James
```
