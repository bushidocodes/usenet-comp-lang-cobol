```
To create an ISAM file with input records NOT in order, do the following:

SELECT OUT-FILE ASSIGN TO 'MYFILE.DAT[X]'
ORGANIZATION IS INDEXED ACCESS IS DYNAMIC
RECORD KEY IS OUT-KEY FILE STATUS IS OUT-STATUS..


FD OUT-FILE LABEL RECORD STANDARD.
01 OUT-REC.
02 OUT-KEY PIC X(10).
02 etc

WORKING-STORAGE SECTION.
01 OUT-STATUS PIC X(2).

PROCEDURE DIVISION.

OPEN OUTPUT OUT-FILE.
IF OUT-STATUS NOT = 'OO'
DISPLAY 'Bad open output, status=' OUT-STATUS.
CLOSE OUT-FILE.
OPEN I-O OUT-FILE.

LOOP.
IF OUT-STATUS NOT = '00'
DISPLAY 'Bad open I-O, status =' OUT-STATUS.
(more code here)
MOVE yak-yak TO OUT-REC
WRITE OUT-REC
IF OUT-STATUS NOT = '00'
DISPLAY 'Bad write, status=' OUT-STATUS.
GO TO LOOP.


Use the above technique when the output records will NOT
be processed in ascending order.

When the records are guaranteed to be written in ascending order,
you may use

SELECT OUT-FILE ASSIGN TO 'MYFILE.DAT[X]'
ORGANIZATION IS INDEXED ACCESS IS SEQUENTIAL <-- note
difference
RECORD KEY IS OUT-KEY FILE STATUS IS OUT-STATUS.

(same FD)

PROCEDURE DIVISION.

OPEN OUTPUT OUT-FILE.
LOOP.

MOVE yak-yak TO OUT-REC.
WRITE OUT-REC.
GO TO LOOP.

Check OUT-STATUS after every I/O operation. Contents other
than '00' means something bad happened.

Remember, before you can access an indexed file as I-O, the
file must already exist. Opening a file I-O does not create it. The
file must first be opened output, then closed, then opened I-O.
```
