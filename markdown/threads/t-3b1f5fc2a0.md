I think you probably mean LU6.1

LU6.1 was a predecessor to LU6.2 (originally called just "LU6", so
that may be where the "6.0" comes from).  It was implemented in IMS/DC
and CICS only, and I believe there's still support in both systems for
it, so CICS and IBM systems can use it to communicate.  You *could*
implement LU6.1 code with native VTAM (or a comparable low level SNA
API on another platform), but you do it at the raw RU level as there's
nothing like the APPCCMD interface, and it would be quite painful.

To the best of my knowledge, LU6.1 is not directly compatible or
interoperable with LU6.2, but as you've already noticed, most
functions in LU6.1 are supported in LU6.2.  In CICS, many applications
can be converted from LU6.0 to LU6.2 with modest changes, although a
few deleted features will likely cause grief if they're used (for
example, embedded FMHs).  My knowledge of IMS/DC is quite limited, so
no comments there.
