```
Micro Focus are doing very well with extending the life of COBOL.

http://www.eweek.com/index2.php?option=content&task=view&id=50422&pop=1&hide_ads=1&page=0&hide_js=1

Riding on the success of their recent PDC, there is a new spring in the step 
of many COBOL developers.

Taking COBOL into .NET and cloud computing is a move that reflects their 
close association with Microsoft, and MS are probably starting to realise 
that there is a lot of benefit for them if people can be moved off  IBM 
hardware into MS supported platforms.

I liked the referenced article because it recognises that COBOL is a lost 
paradigm, but is pragmatic enough to realise there is still scope for COBOL 
code to run in a more cost effective environment. Huge savings can be made 
by moving back end services into cloud computing and Micro Focus should be 
congratulated for facilitating this.

I see this as "buying time" for COBOL sites. I hope that time is used to 
re-invest in training for their people. As mainframers come to grips with 
the .NET and cloud environments there should be a ground swell towards more 
modern approaches.  Back end services running in the cloud on a pay as you 
go basis could be very attractive to many companies.

I'd rather see COBOL running in the cloud for batch services, than Ruby... 
:-)

Pete.
```
