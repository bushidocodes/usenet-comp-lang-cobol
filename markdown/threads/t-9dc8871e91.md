I'm trying to use the ShellExecute API call in my MF 4.0 application.
I want it to open a file selected from a pull down menu.  The file is
a Word 6.0 document (a form the customer can print out).  I'm
developing it on a NT 4.0 machine and when I run it on my machine, the
document opens in MS Word fine.  I moved the app to a Win95 machine
that doesn't have MS Office on it and it doesn't work.  I checked the
file associations in Explorer and Wordpad is the app that should open
the file.  If I double click on the file name, it opens in Wordpad as
I'd expect.  I can see the WordPad icon pop up for a few seconds, then
disappears.  I also tried it on a Win98 machine without MS Office
installed and it behaves the same way.

Here's the call as I have it coded in my program:

 Call WIN32API "ShellExecuteA"
         using by value SH-Handle
               by reference SH-Operation
               by reference SH-FileName
               by reference SH-Parameters
               by reference SH-Directory
               by value SH-ShowCmnd
         returning wreturncode.
     If wreturncode < 32
         display "ShellExecute wreturncode = "wreturncode
     end-if.

SH-Handle is the handle of the window currently in focus retrieved by
the GetActiveWindow API call.  SH-Operation is 'open' (null
terminated), SH-FileName is the file name without the path (null
terminated), SH-Parameters is null, SH-Directory is the path to the
file(null terminated) and SH-ShowCmnd is zero.  The return value is >
32 which the SDK says indicates a good status (sometimes it's very
large like +7095 and other times it's +33).  The only thing I'm not
sure of is the mention of DDE in the SDK for this call.  I've not
worked with DDE before so it's new to me.

Any suggestions?

Doug Blaze

OK, I got it to work.  The secret appears to be putting spaces in
SH-Directory and specifying the filename with the complete path.  I
ended up using SW_ShowDefault (value 10) for SH_ShowCmnd and it works
fine - once I got it work I didn't fool with it!

Example:  SH-Directory = <spaces>
          SH-FileName = c:\progra~1\myapp\test.doc (null terminated)

Doug
