```
Side Files contain debug information written to SYSDEBUG at compile
time when TEST(NONE,SYM,SEPARATE) is used. The question I asked was how
to allocate them explicitly during run time. Apparently I confused
some readers, since I got comments telling me to use DDname SYSDEBUG, 
which is the Compile time output DDname, not the run time input DDname.

I am interested in using this for getting enhanced CEEDUMP reporting
at run time for IMS MPPs and CICS production programs where SEPARATE 
is an attractive option for reducing object module bloat, etc.

The difficulty is that in an SCM enviroment the side file should be
promoted along with the source, object module, DBRMs, etc. so the
Side File DSname recorded in the object deck no longer exists.

Since there didn't seem to be any advice or reference information I 
ran some experiments and discovered that DDname 
   IGZDBGIN
is used. Attempting to preallocate a copied or renamed version of the
sidefile doesn't work. LE seems to try to dynamically allocate it and
preallocating causes dynamic allocation to fail, at which point LE
gives up and does a regular CEEDUMP and SYSPRINT.

The bypass I have for this is to use a distinctive name for the Side File,
such as coding a date/time stamp as part of the DSname, or a unique version
code. An alias can be created with IDCAMS once the side file has been 
promoted to a different SCM library. I've tried this and it seems to work
quite well, just give the copied version of the file an alias that matches
the DSName that was used for SYSDEBUG at compile time.

The only mention I could find of this DDname was a brief mention of
adding a DCT entry for IGZDBGIN to a CICS region:

http://publibz.boulder.ibm.com/cgi-bin/bookmgr_OS390/BOOKS/ceea5030/8.2?ACTION=MATCHES&REQUEST=IGZDBGIN&TYPE=FUZZY&SHELF=CEE1BK32&DT=20000711153000&searchTopic=&searchText=TEXT&searchIndex=INDEX&rank=RANK&ScrollTOP=FIRSTHIT#FIRSTHIT
```
