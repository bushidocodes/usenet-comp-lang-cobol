```
I am only interested in the DATA DIVISION, and any copybooks used therein.

compiler and/or platform the rules apply to?

If there's a web site where I can look it up myself, that's fine.

(Hey, what comes out of this will be free, so it's not like I'm in this for
money!)

ALSO (and this is a REAL inquiry about your preference):  Should I store
user settings (e.g., last five files, COMP alignment, etc)  in the system
registry or in a (editable text) *.INI file?  The software will be a single
Windows 32-bit executable file, no extra DLL's required (I don't use
'Microsoft Visual anything' to produce it, so you don't need 2Mb of run-time
support files).
```

```
Concerning free format (as defined in the draft Standard -- and as I think
that Micro Focus has it implemented - mostly)  (Fujitsu and other vendors
have different rules, I think).

1) The full draft Standard can be reached from:
  http://www.ncits.org/tc_home/j4.htm

2) Now for the rules,

A) "data" can start in any column and go to any column and be continued in
any column.  Therefore, the following is "valid". "*>" is the inline comment
indicator and ">>D" is the new debugging line indicator.

1 Group1. 2
Elem Pic
X. 02 Elem2 pic X. 01 Another-Group. 05 another-elem *> a comment at end of
line
pic
*> a comment line
       x
>>D Value "A"  *> on a debugging line
    .
05 Elem2  Pic
    X(20) Value "New Type"-
   "of continuation line".

Now, I don't think many people would program that way, but it does show many
of the possible variations.

B) It is also "legal" to switch between the two types of format in the middle
of a data item, e.g.

01 Group. 05
Elem1
>>DIR Source format  is fixed
       Pic X. 05 Elem2  *> was always legal but not used much
>>Source free
      Pic 9.

C) in the draft of the next Standard (and several vendors already do this).
The entire A-margin/B-margin rules have been removed.  Therefore, an 01-level
can start in column 13 and an 05-level can start in margin 8.
```
