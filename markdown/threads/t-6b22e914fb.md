There is a callable dynamic allocation assembler routine available on the
www.mvshelp.com bulletin board.  Got to the web-site, and perform a search on
DYNALLOC.  The assembler code is posted, as well as a COBOL copybook with which
to call it from COBOL.  I think it should still be there, but mvshelp has just
transferred to another ISP so it might not - worth checking out anyway though.

What you want can be accomplished via dynamic allocation.  The program flow
should look something like this:

Build new dataset / member name (ie. my.dataset.name1).
Call DYNALLOC program to allocate the new dataset / member.
open COBOL SELECT file for output.
write JCL to file.
Close COBOL SELECT file.
Call DYNALLOC program to unallocate the dataset / member.

repeat process with next dataset name (ie. my.dataset.name2).
...
repeat process with next dataset name (ie. my.dataset.name99).

Hope it helps!
