I have found these two subroutines from the cbttape useful:


      *****************************************************************
      *     Documentation:                                            *
      * SETPTRBY - SET pointer-name UP (or DOWN) BY value             *
      * -------------------------------------------------             *
      *     The purpose of this general routine is to set the pointer *
      * POINTER-TO, up or down (according to the value's sign), by    *
      * SOME VALUE BY-VALUE..                                         *
      *     Together with the SETPTRTO program, this little package   *
      * would allow a more flexible Pointer Manipulation than is      *
      * possible with plain vanila COBOL.  It also demonstrate that   *
      * such manipulation is possible in COBOL, without resorting to  *
      * ASSEMBLER!                                                    *
      *     This subroutine manipulate a pointer to point several     *
      * bytes forewards or backwards.  There is a need for such a     *
      * command rather than manipulating the pointer directly.  A     *
      * pointer in IBM/370 is a 32 bits entity, but since the leftmost*
      * bit is not a sign, it is not a regular arithmetic value.  In  *
      * other architectures, a pointer may or may not be an arithmetic*
      * value.  Therefore it is not recomended to add to or subtract  *
      * from it, directly.  Also, even in IBM/370, the COBOL II type  *
      * PIC 9(9) COMP is defined to contain a nine decimal digits     *
      * and not 32 bits value, and is therefore not fully compatible  *
      * with the USAGE POINTER it may REDEFINES.  The "NOTRUNC"       *
      * compiler option is not always available.                      *
      *     Usage of the routine is like that:                        *
      *    ...                                                        *
      * CALL 'SETPTRBY' USING WS-PTR WS-VALUE.                        *
      *    ...                                                        *
      * If WS-VALUE contains a positive value, the pointer is SET UP  *
      * and if it contains a negative value, the pointer is SET DOWN. *
      * Note that this value is limitted to the equivalent of 9       *
      * decimal digits, which is enough for any reasonable purpose.   *
      *     It is possible to combine the usage of this routine with  *
      * the usage of the LENGTH OF spaecial register to advance to the*
      * next array element:                                           *
      *    ...                                                        *
      * CALL 'SETPTRBY' USING WS-PTR LENGTH OF WS-ELEMENT.            *
      *    ...                                                        *
      * providing the pointer was pointing to an array element.       *
      *     This program must be compiled with NOSSRANGE compile      *
      * option (or the equivalent in your compiler) to asure no       *
      * compile or run time boundary checking.                        *
      *****************************************************************
      * CBL NOSSRANGE
        IDENTIFICATION DIVISION.
        PROGRAM-ID. SETPBY.
        ENVIRONMENT DIVISION.
        DATA DIVISION.
        WORKING-STORAGE SECTION.
-INC LCS001C
        01   J                   PIC S9(9) COMP.
        LINKAGE SECTION.
        01   POINTER-TO          POINTER.
        01   BY-VALUE            PIC S9(9) COMP.
        01   VIRTUAL-OBJECT.
             05 VIRTUAL-ELEMENT OCCURS 1 INDEXED BY I PIC X.
        PROCEDURE DIVISION USING POINTER-TO BY-VALUE.
             SET ADDRESS OF VIRTUAL-OBJECT TO POINTER-TO
             SET I TO 1
             IF BY-VALUE < ZERO
                MULTIPLY BY-VALUE BY -1 GIVING J
                SET I DOWN BY J
             ELSE
                SET I UP BY BY-VALUE
             END-IF
             SET POINTER-TO TO ADDRESS OF VIRTUAL-ELEMENT (I)
             GOBACK.



      *****************************************************************
      *     Documentation:                                            *
      * SETPTRTO - SET pointer-name TO ADDRESS OF working-storage-name*
      * --------------------------------------------------------------*
      *    The purpose of this general routine is to set the pointer  *
      * POINTER-TO to the address of the object POINTED-OBJECT. Since *
      * we do not touch the latter, its type is insignificant, and is *
      * reduced here to the minimum, PIC X.                           *
      *    This subroutine forces the address of the WORKING-STORAGE  *
      * data item to the pointer.  This subroutine should be used in  *
      * cojunction with a regular SET command in order to establish   *
      * the right addressing.                                         *
      *    Usage of the routine is like that:                         *
      *    ...                                                        *
      * CALL 'SETPTRTO' USING WS-PTR WS-AREA.                         *
      * SET ADDRESS OF LS-AREA TO WS-PTR.                             *
      *    ...                                                        *
      * and from than on, any reference to LS-AREA-A1 etc., will      *
      * actually refer to some part of WS-AREA.                       *
      *    The "Application programming Guide" (*) brings in part 4 - *
      * "Compiling Your Program", an example similar to my SETPTRTO,  *
      * under the name GETADDRESS.  This subroutine does              *
      * "SET pointer TO ADDRESS OF data-item",                        *
      * while both are in the LINKAGE SECTION.  Note that calling such*
      * a subroutine (IBM's or mine), must be BY REFERENCE (which is  *
      * the default).                                                 *
      ******************************************************************
       IDENTIFICATION DIVISION.
       PROGRAM-ID. SETPTO.
       ENVIRONMENT DIVISION.
       DATA DIVISION.
       WORKING-STORAGE SECTION.
-INC LCS001C
       LINKAGE SECTION.
       01  POINTER-TO          POINTER.
       01  POINTED-OBJECT      PIC X.
       PROCEDURE DIVISION USING POINTER-TO POINTED-OBJECT.
             SET POINTER-TO TO ADDRESS OF POINTED-OBJECT
             GOBACK.


Here is an example that use the two subroutines:


      *---------------------------------------------------------------*
      * The two callable routines SETPTO   and SETPBY  , demonstrate  *
      * how, once pointer is introduced to COBOL, no matter how limited
      * it is possible to establish addressing to one's own           *
      * WORKING-STORAGE and to directly manipulate pointers without   *
      * resorting to Assembler at all.  These routines are self       *
      * documented. This text contains excerpts of an article I have  *
      * written and explain how they might be used.                   *
      *                                                               *
      * SETPTO   was adapted, with changes from the IBM's GETADDRESS  *
      * function, as described in the publication: VS COBOL II,       *
      * APPLICATION PROGRAMMING GUIDE.  The purpose of this general   *
      * routine, which is coded in COBOL, is to set the pointer       *
      * POINTER-TO to the address of the object POINTED-OBJECT.  Since*
      * we do not touch the latter, its type is insignificant, and is *
      * reduced, internally, to the minimum, PIC X.                   *
      *                                                               *
      * The purpose of SETPBY   is to set the pointer POINTER-TO, up  *
      * or down (according to the value's sign), by some value BY-VALUE
      * The idea is to use INDEX in order to travel on the object and *
      * then to SET ADDRESS OF a subscripted item to the pointer.  This
      * idea uses a fact, simply understood by any C programmer, that *
      * indexes and pointers are really the same entity.  Note that   *
      * this program must be compiled with the NOSSRANGE compile      *
      * option (or the equivalent in your compiler) to ensure no      *
      * compile or run time boundary checking.                        *
      *                                                               *
      * One may imagine some useful applications for pointer          *
      * manipulation.  The most elegant use I've found for pointer    *
      * manipulation is to overcome another limitation of COBOL.      *
      * There is no provision in COBOL to directly manipulate variable*
      * length character strings.  The UNSTRING statement may begin its
      * process anywhere on the character string but the total length *
      * of that string must be known to the UNSTRING at compile time. *
      * But if we could cheat and tell COBOL at run time that the     *
      * string begins somewhere else, we could, in effect, manipulate *
      * variable length strings.                                      *
      *---------------------------------------------------------------*
       IDENTIFICATION DIVISION.
       PROGRAM-ID. SETPTR.
       ENVIRONMENT DIVISION.
      /
       DATA DIVISION.
       WORKING-STORAGE SECTION.
-INC LCS001C
       01  LENGTH-OF-CURRENT-STRING      PIC 9(4) COMP VALUE 19.
       01  MAX-LENGTH-OF-STRING          PIC 9(4) COMP VALUE 254.
       01  WS-STRING                     PIC X(254) VALUE
           'ABC@DEF@GHI@JKLMNOP'.
       01  FILLER.
           05 WS-VALUE                      PIC S9(9) COMP.
           05 WS-PTR                        USAGE POINTER.
           05 WS-TALLY                      PIC S9(4) COMP.
           05 WS-LEN-9                      PIC 9(4) COMP.
           05 WS-POINT                      PIC 9(4) COMP.
           05 WS-TOKENS OCCURS 8 INDEXED BY IX-TOKEN.
              10 WS-TOKEN                   PIC X(30).
              10 WS-DELM                    PIC X.
              10 WS-CNT                     PIC S9(4) COMP.
       LINKAGE SECTION.
       01  LS-STRING                     PIC X(254).
      /
       PROCEDURE DIVISION.
           COMPUTE WS-VALUE = LENGTH-OF-CURRENT-STRING -
                              MAX-LENGTH-OF-STRING.
           DISPLAY 'WS-VALUE=' WS-VALUE.
           CALL 'SETPTO' USING WS-PTR WS-STRING.
           DISPLAY 'WS-PTR=' WS-PTR.
           CALL 'SETPBY' USING WS-PTR WS-VALUE.
           DISPLAY 'WS-PTR=' WS-PTR.
           SET ADDRESS OF LS-STRING TO WS-PTR.
           COMPUTE WS-POINT = MAX-LENGTH-OF-STRING -
                              LENGTH-OF-CURRENT-STRING + 1.
           DISPLAY 'WS-POINT=' WS-POINT.
           MOVE ZERO TO WS-TALLY.
           UNSTRING  LS-STRING
              DELIMITED BY '@' OR '$' OR ALL X'00'
              INTO WS-TOKEN (1) DELIMITER WS-DELM (1) COUNT WS-CNT(1)
                   WS-TOKEN (2) DELIMITER WS-DELM (2) COUNT WS-CNT(2)
                   WS-TOKEN (3) DELIMITER WS-DELM (3) COUNT WS-CNT(3)
                   WS-TOKEN (4) DELIMITER WS-DELM (4) COUNT WS-CNT(4)
                   WS-TOKEN (5) DELIMITER WS-DELM (5) COUNT WS-CNT(5)
                   WS-TOKEN (6) DELIMITER WS-DELM (6) COUNT WS-CNT(6)
                   WS-TOKEN (7) DELIMITER WS-DELM (7) COUNT WS-CNT(7)
                   WS-TOKEN (8) DELIMITER WS-DELM (8) COUNT WS-CNT(8)
              WITH POINTER WS-POINT
              TALLYING WS-TALLY
           END-UNSTRING.
           DISPLAY 'WS-TOKEN (1)=' WS-TOKEN (1)
                   '  WS-DELM (1)=' WS-DELM (1)
                   '  WS-CNT  (1)='  WS-CNT (1).
           DISPLAY 'WS-TOKEN (2)=' WS-TOKEN (2)
                   '  WS-DELM (2)=' WS-DELM (2)
                   '  WS-CNT  (2)='  WS-CNT (2).
           DISPLAY 'WS-TOKEN (3)=' WS-TOKEN (3)
                   '  WS-DELM (3)=' WS-DELM (3)
                   '  WS-CNT  (3)='  WS-CNT (3).
           DISPLAY 'WS-TOKEN (4)=' WS-TOKEN (4)
                   '  WS-DELM (4)=' WS-DELM (4)
                   '  WS-CNT  (4)='  WS-CNT (4).
           DISPLAY 'WS-TOKEN (5)=' WS-TOKEN (5)
                   '  WS-DELM (5)=' WS-DELM (5)
                   '  WS-CNT  (5)='  WS-CNT (5).
           DISPLAY 'WS-TOKEN (6)=' WS-TOKEN (6)
                   '  WS-DELM (6)=' WS-DELM (6)
                   '  WS-CNT  (6)='  WS-CNT (6).
           DISPLAY 'WS-TOKEN (7)=' WS-TOKEN (7)
                   '  WS-DELM (7)=' WS-DELM (7)
                   '  WS-CNT  (7)='  WS-CNT (7).
           DISPLAY 'WS-TOKEN (8)=' WS-TOKEN (8)
                   '  WS-DELM (8)=' WS-DELM (8)
                   '  WS-CNT  (8)='  WS-CNT (8).
           GOBACK.

