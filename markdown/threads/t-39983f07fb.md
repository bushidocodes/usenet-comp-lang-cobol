```
Hi, Harry.

First, error 17's are almost always because Dialog System is
given a bad set of parameters for a particular command.

› The detail error codes are 3152 and 3001. These translate to
› "pf-Create-Entry-Field" and "pf-Get-Gadget-Id-From-Handle".

You were close. The 3152 error, as listed in the PAN2LINK.CPY
file shows that the problem that you ran into is related to the
PF-Create-Entry-Field. However, the 3001 error, as listed in the
PAN2ERR.CPY file, shows that the cause for the error is
(P2ERR)-Bad-Type, not pf-Get-Gadget-ID-From-Handle.

What the Bad-Type error is trying to tell you is that for the
command on which it occurred, the operation that you are trying
to perform is either illegal (ie. you are doing something that is
specific only for the Windows environment in the OS/2 PM
environment), that the type of function that you are attempting
isn't supported on that object, or one of the parameters are
incorrect. To know, specifically, which this particular case
would be, you need to use the Trace function to see the exact
Dialog command the error is occurring on. It is, usually, fairly
obvious once you where is is failing. Regards, Alan.
```
