05  The-big-field    PIC X(21).             <<, all 21 digits fit here

give up three digits at one or both ends and this preserves the decimal
alignment.

So far as I know, there is no other language commonly available that can
handle what you want.

You can use floating-point definitions in some other languages (COBOL, too),
but while floating-point allows values as large as you need (and much
larger), I don't believe you're going to get 21 digits of precision.

In other words, in floating-point:
123456789012345678901.234567890

will be indistinguishable from:
123456789012345678000.000000000
or
123456789012345678999.999999999

or some similar limitation
