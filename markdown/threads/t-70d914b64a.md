```
Frik Lombard wrote:
>When i use tha accept w-date from date the system returns a date in the
>format yymmdd. What can i do for the system to return a 8 digit date fro me
>in the format ccyymmdd.
>
>I am using a Microfocus personal cobol v 2.2 and win 95.

What you are looking for is FUNCTION CURRENT-DATE, added with the
1989 update to the COBOL-85 standard.  It returns a 21 character
alphanumeric value formatted like this:

01  HOLD-CURRENT-DATE.
    03  HCD-YEAR            PIC  9999.  GREGORIAN YEAR
    03  HCD-MONTH           PIC  99.    01-12
    03  HCD-DAY             PIC  99.    01-31
    03  HCD-HOUR            PIC  99.    00-23
    03  HCD-MINUTE          PIC  99.    00-59
    03  HCD-SECOND          PIC  99V99. 00.00-59.99
    03  HCD-GREENWICH-DIFF  PIC  X.     0,+,-
                        '0' = GREENWICH TIME (ZERO)
                        '+' = AHEAD OF GREENWICH (SEE DIFF BELOW)
                        '-' = BEHIND GREENWICH   (SEE DIFF BELOW)
    03  HCD-DIFF-HOURS      PIC  99.    00-12
    03  HCD-DIFF-MINUTES    PIC  99.    00-59

    MOVE FUNCTION CURRENT-DATE TO HOLD-CURRENT-DATE.
```

This question comes up about once a week. answers range from:
1: you call "VOODOO" using strange    to
2: you use (what you don't have) the ACCEPT ccyymmdd from DATE YYYYMMDD

It is actually quite simple:

01  DATE-NOW       PIC 9(8).
01  DATE-CCYYMMDD  REDEFINES DATE-NOW.
      02  DATE-CC           PIC 99.
      02  DATE-YYMMDD  PIC 9(6).
      02  DATE-YY-MM-DD  REDEFINES DATE-YYMMDD.
            03  DATE-YY        PIC 99.
            03  DATE-MM      PIC 99.
            03  DATE-DD       PIC 99.

....

     ACCEPT DATE-YYMMDD FROM DATE
     IF YY < 98
          MOVE 20 TO DATE-CC
     ELSE
          MOVE 19 TO DATE-CC
     .

This should work for the next 100 years or so
