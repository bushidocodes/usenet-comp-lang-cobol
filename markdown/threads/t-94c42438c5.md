```
We've got a number of MF COBOL programs that use MF extended ADIS. It 
would be nice if we could hook into the code at a low level to trap the 
actual input and output streams and redirect them. The compiler is 
NetExpress 3.1

Any suggestions?

In fact we are converting the front end into using a browser (when 
available) and while some of the programs will require significant changes 
and we are content to rewrite the interface there are quite a number where 
the user interaction is limited (enter a date or two, Yes/No etc.), and 
there are quite a few of these. We would hope to find some way of avoiding 
the major recoding of these and we thought that if we could intercept the 
data streams then we could write utilities to redirect the data as we 
like. In times gone by the data appeared to be VT100(ish) and this would 
be fairly simple to handle.

Merant support cannot offer any method themselves...

graham
```
