```
I have a Fujitsu subroutine here that may be of interest to
several people in the NG.  If someone (Bob? Leif?) will donate
a web site, I will make it public domain.

The problem:  To switch code written for pre-printed forms under
DOS (or anything else) to Windows95/NT laser, plain paper printing.
The code must have minimal changes, and the form must be completely
under control of the user.  That is, it must be changeable using a
standard word processor.

The solution: A spool subroutine that merges the output of a word
processor with standard Cobol line sequential files.  The form is
written to disk after being printed on a printer to test.  It
should print a sample blank form if copied to the printer.  The
report is written to disk, instead of being printed directly, and
then closed.  The subroutine is called with the form name, the
report name, the printer name (\\server\printer style names allowed),
the point size required for the overlay, the lines per inch and
the characters per inch.  It uses a fixed font so that columns
are retained, It will default to standard six line per inch, ten
characters per inch, 12 point, the same as a DOS report would print
on a dot matrix.

I have a beta working, as well as a test routine, test form, test
report, etc.  Anybody that wants a copy can Email me.  I need to
tune it up for speed, and want to add a "number of copies" argument
before Version 1.00, however it seems to be solid in it's current
incarnation.  It should work with any HP4 LaserJet compatible.  I
created the form with MS word (actually a secretary did).
```
