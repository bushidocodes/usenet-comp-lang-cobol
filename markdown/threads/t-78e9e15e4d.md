```
I would change your code "slightly" as follows:

 DATA DIVISION.
  01 MULTIPLIER        PIC 99  VALUE 0  OCCURS 13 TIMES.
  01 SUB                      PIC 99 .

 PROCEDURE DIVISION.
    PERFORM 200-FILL-MULTIPLIER
         VARYING SUB FROM 1 BY 1 UNTIL SUB  > 13.

 200-FILL- MULTIPLIER.
         Compute MULTIPLIER (SUB) = (FUNCTION RANDOM) * 13
            .

This will take the FUNCTION RANDOM output (which is between 0 and 1 - with
lots of decimal points) and convert it into two-digits between 0 and 13.

NOTE: Although some compiler do allow it as an extension, you may NOT use
FUNCTION RANDOM in a MOVE statement in "Standard" COBOL.

NOTE2:  The COBOL "FUNCTION RANDOM" is "pseudo-random" - which means that if
you repeat this same program over and over, you will always get the same
series of "random numbers".  (There is a long explanation of that - and if
you perform that same paragraph from multiple places in your program - you
will NOT get the same numbers each time.)  If you want a "more" random
series, then before your first use of RANDOM (without a seed-value) use one
reference to RANDOM with the some portion of the time-seconds as a seed
value.
```

```
I've run across this problem before, namely how to not have duplicates in
the selections.  The specific problem was to shuffle a deck of 52 cards
(this problem with the number 13 sounds suspeciously like it).  The trick is
to not look at the problem of generating 52 random numbers between 1 and 52
(or 0 and 51), and having to worry about whether that number has been
generated before.  After all, it could be several hundred iterations before
all 52 possible results are generated.

My solution was to build an array of the results, assign a random number to
each, then sort the random numbers.

01  TABLE-OF-RESULTS.
    05  ONE-RESULT OCCURS 52.
        10  RESULT-VALUE PIC 99.
        10  RESULT-RANDOM PIC S9V9(9).

Put the values 1-52 in RESULT-VALUE (1) - (52), put 52 random numbers into
RESULT-RANDOM (1) - (52).  Sort ONE-RESULT (1) - (52) in ascending order by
RESULT-RANDOM (1) - (52).  In the process, the values in RESULT-VALUE (1) -
(52) will be reordered in random order, and or course each will only occur
once.
```

```
This method for creating a shuffled deck avoids duplicates:
...
  05 DECK OCCURS 52    PIC 99.
  05 SHUFFLE OCCURS 52 PIC 99.
...
PERFORM VARYING X1 FROM 1 BY 1 UNTIL X1 > 52
  MOVE X1 TO DECK(X1)
END-PERFORM
PERFORM VARYING X1 FROM 52 BY -1 UNTIL X1 < 1
  R1 = //Code to pick an random integer from 1 to X1 goes here//
  SHUFFLE(X1) = DECK(R1)
  DECK(R1)=DECK(X1)
END-PERFORM
...
Whenever a card from the deck is chosen, whatever is at the current end of the
deck is moved to replace the chosen card.  HTH
```
