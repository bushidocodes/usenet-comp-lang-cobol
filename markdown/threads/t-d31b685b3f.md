Assuming that you are getting the problem on an MVS platform
(S390/OS390), and you are a current COBOL, the CEEDUMP should have the
record in question as well as all of the pertinent working storage.  The
file areas and record related to the SD are available with the CEEDUMP
as of OS390, release 2.5 and may be available via PTF on even earlier
releases of OS390.  If worst comes to worst, the simplest way to get the
file is to create a test version of the program and add a WRITE
statement for the sorted record to a special file after every RETURN
statement in the program.  You don't want the intermediate sort work
file and depending on the amount of virtual storage available, it may
not even exist (or if it does exist it may not contain any data).
