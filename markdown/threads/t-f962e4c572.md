```
I have a problem. Simply stated, how do I soft-code a two-dimensional
table? My book only explains how to load a hard-coded table. My assignment
(yes...I'm a lowly student) involves using an input-loaded table to compute
average salaries for 6 different location-performance combination employees.
The incoming file provides the necessary information which includes their
salaries, their performance code, and their location code. The location
codes include 30 (Miami), 60 (Los Angeles), and 80 (New York). Performance
codes include 01 (High Performance) and 02 (Low Performance). This
effectively creates a 3-by-2 table for the 6 location-performance
combinations (Rows 1,2, and 3 are for Miami, LA, and NY; Columns 1 and 2
designate high and low performance, respectively).

To clarify, my output will resemble the following:


LOCATION/PERFORMANCE AVERAGE SALARY REPORT

LOCATION HIGH PERFORMANCE LOW PERFORMANCE

MIAMI ZZ,ZZ9.99
ZZ,ZZ9.99
LOS ANGELES ZZ,ZZ9.99
ZZ,ZZ9.99
NEW YORK ZZ,ZZ9.99
ZZ,ZZ9.99


So...this is my table:

01 Salary-Table
05 Location occurs 3 times.
07 Performance occurs 2 times.
09 Total-Salary pic 9(6)v99.
09 Emp-Count pic 9(3).

...And my subscripts are: Location-Sub and Performance-Sub

Since the location-codes are strangely coded (they're not 1, 2, and 3) how
am I supposed to put each record's values into the table?

I'm not asking for someone to do my homework for me...I've spent hours
trying to figure out this seemingly minor task. I can handle the
computations and so forth; I just need to know how to get the incoming data
into the table.




Any help to steer me in the right direction is GREATLY appreciated,



Cody
```

```
I've been using the same book.. (If I'm correct and you are using Realia) try

FILE-CONTROL.
SELECT EMPLOYEE-FILE
ASSIGN TO UT-S-SYSIN.

SELECT PRINT-FILE
ASSIGN TO UT-S-SYSOUT.

SELECT TABLE-FILE
ASSIGN TO UT-S-TABLE.

Note that last setting... when you do this, you will have to define it in the
File Section, and before running it, when at the DOS prompt after you've set
the sysin and sysout, type set Table = emptable.dat
table as "emptable.dat"... otherwise use the appropriate filename.

Took me about 1 hour to figure out that input method.
```

```
Lets assume you have defined two subscripts in WS thusly:
77 INDEX-1 PIC S9999 VALUE +0 COMP.
77 INDEX-2 PIC S9999 VALUE +0 COMP.

Then in the Procedure Divsion you could code thusly:

IF LOCATION = 30
MOVE 1 TO INDEX-1
ELSE IF LOCATION = 60
MOVE 2 TO INDEX-1
ELSE IF LOCATION = 80
MOVE 3 TO INDEX-1
ELSE PERFORM ERROR-ROUTINE.

MOVE PERFORMANCE TO INDEX-2.
ADD SALARY TO TOTAL-SALARY (INDEX-1, INDEX-2).
ADD +1 TO EMP-COUNT (INDEX-1, INDEX-2)


You may want to check that PERFORMANCE is between 1 and 2.

These suggestions are based on IBM Mainframe COBOL, but the idea
should work with any COBOL.

Hope this helps and good luck.
```
