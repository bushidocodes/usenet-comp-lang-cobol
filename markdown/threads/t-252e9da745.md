I think your first mistake is definitions.  A table and an array are 
just two different names
for the same thing ... though maybe it could be argued that an array is 
numeric. "Array"
tends to be a mathmatics term, "table" tends to be a business term. 
 Sometimes people
also use "table" to mean an array that is pre-loaded with data. The key 
point is that
the difference is just nuance.  They are the same thing in terms of 
understanding and
writing code.

They are really quite simple ... instead of storing data under the names 
Book1, book2,
book3, etc., you declare one varriable OCCURS several times.  The number 
is the
maximum.  You then refer to them as Book (1) Book (2) Book (3), etc.

The nice thing about doing it that way, is that you can use a variable 
for the subscript.
For example, if you move 2 to the variable SUBSCRIPT-NAME,
then the data reference "Book (SUBSCRIPT-NAME)" is the same as the data
reference "Book (2)".  That allows you to use one piece of code, 
changing the
subscript each time through.

As to your second question, no.  Any data record can be made to occur 
numerous times,
so something like

    01 BOOK-TABLE.
        02 ONE-BOOK        OCCURS 50 TIMES.
            03 BOOK-TITLE            PICTURE X(50).
            03 BOOK-AUTHOR       PICTURE X(50).

contains fifty titles, and fifty authors.  All three data names, (book, 
title and author) need
to be subscripted in code, and any of the three can be used in valid 
statements.

That should at least get you started.

Donald
