For more than 10 years I have been working with two software houses who ship
their products to run on a variety of hosts (mainly Unix, Aix, NT, OS2 and
Citrix) with the AcuCorp runtimes and Vision (version 3 and version 4) file
systems.  I have found Vision fast, efficient and dependable. I have only
encountered the smallest of problems with it.  For example, while the Vision
data files are almost universally portable, on a Vax/VMS system we had to
use the VMS native ISAM  package (in place of Vision) to get reasonable
performance (at the cost of some features.)

Is there something in particular you want to know?
