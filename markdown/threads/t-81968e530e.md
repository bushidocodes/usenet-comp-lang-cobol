```
1) Use of "comp.lang.cobol" rather than "alt.cobol" OFTEN gets better/faster
replies

2) "run time errors" are NOT universal, so we would need to know (at a
minimum) which compiler (vendor and release) and operating system you are
using.

3) Do you get ANY other "run-time error" message besides the "45"?  If so,
what?

4)  Without any other information, the first GUESS that I would make (and
this may be completely wrong) is that you are getting a "file status 45" on
some I/O operation.  Look that up in your reference material and see if it
might apply to where your program is having a problem.
```
