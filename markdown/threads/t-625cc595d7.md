What is "normal", one could ask. I'll interpret "normal" as "a compiler". NE
(NetExpress) provides:
OO-COBOL, like what's coming in the next standard
Direct programming of CGI programs in COBOL
Full access to all Windows API's
Interlanguage calling/called, f.e. making DLL's callable from others like
C(++)/Pascal
OO-GUI-tree re-usable classes/objects
IDE: editor, windows resource editor (bitmap, cursor, icon a.s.o)
HTML-editor, which generates COBOL datastructures for use in CGI
> 
> 2/ We use object COBOL programs (running with rts32) instead of exe or dll
> programs. Can a COBOL cgi program be an object program ?

No of course not. A CGI-program is activated by a plain call from the
web-server-software, so the CGI has to be procedural. But you could program:
WORKING-STORAGE
01  anObj  usage object-reference.
PROCEDURE DIVISION.
Main.
    Invoke "SomeClass" "new"
          [using ...]
           returning anObj

and from there on go full-blown OO, like

    Invoke anObj "startMethod"

SomeClass and all others could be instantiated and sent messages, including
objects from the (rich) provided class tree of predefined classes.

> 
> -> at the moment we have written c-shells scripts that decode the query strings
> into single parameters then run the object COBOL program with a "cobrun progname
> parameters" string.
Not needed with NE. It gets the fields of the defined HTML-forms directly using
an COBOL accept statement. It write HTML with an "Exec HTML ... end-exec
construct, just like embedded SQL.
> 
> -> that works correctly but do you think it is possible not to use the c-shell
> scripts (for maintenance raisons we would prefer not to use them...).
Definitly. I did write a complete web-app, running under IIS without using any
scripts. Just wrote some HTML, including some JavaScript for client-side things
like hyperlink-animation, and all COBOL.

With CobolScript, the POST-method string is auto-decrypted for you with
a special command called ACCEPT DATA FROM WEBPAGE.  This command places
the values of all incoming CGI variables into like-named variables in
your CobolScript program.

CobolScript is also pretty cheap, at $49.95 per license.  Check it out
at http://www.cobolscript.com .
