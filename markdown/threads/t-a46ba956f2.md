Hi,

My background is COBOL on MVS mainframes and was wondering if you could
help me with a question about COBOL on PCs.

Basically, the question is : Is it possible to create files from within
a COBOL program on a PC ?

The scenario is that I have an input file which contains a list of
people with personal details for each person, in the following format:

NAME:ADDRESS1:ADDRESS2:ADDRESS3
NAME:ADDRESS1:ADDRESS2:ADDRESS3
NAME:ADDRESS1:ADDRESS2:ADDRESS3
NAME:ADDRESS1:ADDRESS2:ADDRESS3

What I would like to do is read each line in turn and create a file for
each record with the NAME field used as the filename. So in this example
I would end up with three files each containing
NAME:ADDRESS1:ADDRESS2:ADDRESS3.

Does it make sense ?

Many thanks,

Andrew

> Basically, the question is : Is it possible to create files from within
> a COBOL program on a PC ?

Yes! It's a piece of cake. And welcome to the wonderful world of PC
programming. (What took you so long?)

The key is in the SELECT statement: the ASSIGN can point to a field
where you store the filename you wish to use. In Realia COBOL, it goes:

    SELECT MY-OUTPUT-FILE 
        ASSIGN TO VARYING MY-OUTPUT-FILE-NAME

You can then loop for each input record:

    READ MY-INPUT-FILE 
       (Exit loop on end of file)
    MOVE NAME OF MY-INPUT-REC TO MY-OUTPUT-FILE-NAME 
       (& check for valid file name and allow for duplicates)
    OPEN OUTPUT MY-OUTPUT-FILE (& check file status for error)
    WRITE MY-OUTPUT-REC FROM MY-INPUT-REC (& check for error)
    CLOSE MY-OUTPUT-FILE (& check for error)
    (Repeat loop to read input again)

There's no need to reserve or allocate disk space outside of the
program. If the name you assign is a simple file name (without
directories), it's stored in the current directory. You are free to
override that by including a drive/directory path as part of the
filename.

Go to it!

--John Hicks
