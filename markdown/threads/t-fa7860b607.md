The Select/Assign allows you to refer to environment variables on
many platforms. For example, in MVS you can use JCL to have a
DDNAME in the program be associated with a MVS file name. Here is
some JCL with the DDNAME=INPUT1 and the tape name FUN.STU.FF(0).

In the cobol program:
Select myfile
assign to INPUT1.


//Step01 Exec pgm=myprog
//STEPLIB DD DSN=PROGLIB,DISP=SHR
//INPUT1 DD DSN=FUN.STU.FF(0),DISP=(OLD,KEEP)
//SYSOUT DD SYSOUT=*
//

And under MS-DOS with the proper compiler directive the SET DOS
command can be used to tell what the file name is...

SET INPUT1 C:\INPUT.TXT

You see, the program can have a different input file by modifying
either the JCL or the environment variable via SET in MS-DOS.

In VM/CMS you can use a command called FileDef in foreground.
