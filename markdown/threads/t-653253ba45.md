```
This is called information hiding.  If you need some common information
across many different modules you create a set of modules that will
return the information to you without rereading the information every
time.  This is fairly simple:

hidden large routine:

reads all the control data and stores it in working storage on the first
call.  All subsequent calls just return the same data.  (Maybe all
windows setup criteria from an ini file)

other modules:

a set of support modules are written that call 'hidden large routine'
and extract the small section of data that the calling program is
interested in (maybe background color)

Calling programs can then call these minor routines many times with only
a extremely small performance hit but greatly improved maintainability.

Sound like a lot of work.  Implement you monolithic call and then move
it from an ini file to a SQL server.  Then change that to a dynamic
change.

The purpose is to isolate the nitty gritty of the program from the basic
business functions.  This is difficult to do especially for 'basic'
cobol programmers who have not worked out how to call instead of using a
paragraph.

Hope this helps
Ken
```
