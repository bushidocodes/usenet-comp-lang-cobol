Intrepid Research 2 wrote:
› I need info on the following:
› 1) In the MF-Cobol screen section(in Unix), can the attributes of a part
› of an item be changed. Suppose there is an item of size 70 displayed at

I don't know of a way to do this. In our programs, we don't use a
SCREEN-SECTION at all. Instead, we use format-3 of the DISPLAY verb to
control screen field attributes and ADIS calls to handle keyboard and mouse
events. This gives the maxiumum amount of control over screen handling.

› 2) Does MF-Cobol provide the facility of designing multi-row
› screens(many records on the same screen) with a maximum of 9999 records.

I don't think so. Using the DISPLAY statement, you can depend on data
wrapping from the end of one row the the beginning of the next, but I don't
know of anything that handles lists within a screen.

› 3) How to trap function keys F1 to F24(using the Function keys F1 to F12
› and combination of Shift and Function keys), page-up, page-down,
› enter-key in MF-Cobol in Unix.

You must use ADIS functions to determine if a function key was hit. ADIS
depends on the termcap definitions for the TTY device. On many terminals it
is not possible to map function keys above F12, so Micro Focus has a
multi-key method for getting around this limitation.

We have found that the only way to handle terminals well is to use X
terminal support. Using TTY terminals always involves compromises that
either make the screens ugly or limit handling of keyboard and mouse events
(usually both).

› 5) How to change the tab-order of fields in the screen-section. If I
› have a screen that has multi-row fields(within an OCCURS sub-group)
› followed by some single-row fields, I want to set focus to the
› single-row fields first and then tab to the multi-row fields.

If you use ADIS functions to get each keystroke, you can reposition the
cursor whenever you like. If you use the normal screen handling routines,
they usually work top to bottom, left to right.

Glynn

I don't think I understand exactly what you're asking for here, but you can
create a "virtual screen" using PANELS and get ADIS (and thus SCREEN SECTIONs)
to treat it as the display area. When working in this way, ADIS automatically
treats the real screen as a "window" over this display area and makes sure that
the currently active field (with the cursor) is visible. I can't remember the
exact ADIS (call-by-number x"af") call to achieve this off the top of my head,
but it is in the docs. I seem to remember that there was a PANELS demo shipped
which contained an example of this also.
