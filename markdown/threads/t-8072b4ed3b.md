```
If you ever feel like taking a break from business and relaxing in a
nice petrabyte sized pile of subatomic analysis data, meet ROOT/CINT.
http://root.cern.ch/drupal/content/cint  Another gift to the world
from CERN.

It turns out that OpenCOBOL subprograms can be loaded into and
interactively tested using the very capable CINT C/C++ interpreter.
CINT was written to allow scientists to write scripts and interact at
a console with huge data sets.  And now that power can be leveraged to
test out subprograms.

On a Fedora 14 system, yum install root.

Then given

OCOBOL*>>SOURCE FORMAT IS FIXED
      *>
***************************************************************
      *> Author:    Brian Tiffin
      *> Date:      20101119
      *> Purpose:   Pass arguments to ROOT/CINT invoked subprograms
      *> Tectonics: cobc -fimplicit-init -C cobparams.cob
      *>
***************************************************************
       identification division.
       program-id. cobparams.

       data division.
       linkage section.
       01 a-number usage binary-long.

      *>
***************************************************************
       procedure division using by reference a-number.
       display a-number end-display
       move a-number to return-code
       goback.
       end program cobparams.

and then the commands

$ cobc -fimplicit-init -C cobparams.cob
$ root -l
root [0] gSystem->Load("/usr/local/lib/libcob.so");
root [1] .L cobparams.c+
root [2] int a = 0;
root [3] int d = 42;
root [4] a = cobparams((unsigned char*)&d);
+0000000042
root [5] printf("%d\n", a);
42
root [6]

You get interactive access to COBOL subprograms, and if the desire
takes, access to a tremendous framework for exploring the quantum
universe while pondering while payroll was off by 12 cents.

Cheers,
Brian
```

```
OCOBOL*>>SOURCE FORMAT IS FIXED
      *> *********************************************************
      *> Author:    Brian Tiffin
      *> Date:      20101119
      *> Purpose:   Pass arguments to ROOT/CINT invoked subprograms
      *> Tectonics: cobc -fimplicit-init -C cobparams.cob
      *> *********************************************************
       identification division.
       program-id. cobparams.

       data division.
       linkage section.
       01 a-number usage binary-long.

      *> *********************************************************
       procedure division using by reference a-number.
       display a-number end-display
       move a-number to return-code
       goback.
       end program cobparams.

      *> *********************************************************
      *> *********************************************************

       REPLACE ==ARRAYSIZE== BY ==450==.

       identification division.
       program-id. cobfloats.

       data division.
       working-storage section.
       01 cnt pic 999.
       01 val usage float-long.
       01 xes.
          02 an-x usage float-long occurs ARRAYSIZE times.
       01 yes.
          02 an-y usage float-long occurs ARRAYSIZE times.

       linkage section.
       01 vxes.
          02 an-x usage float-long occurs ARRAYSIZE times.
       01 vyes.
          02 an-y usage float-long occurs ARRAYSIZE times.

      *> *********************************************************
       procedure division using by reference vxes, vyes.
       perform varying cnt from 1 by 1 until cnt >= ARRAYSIZE
           compute val = cnt * function random() end-compute
           move cnt to an-x in xes(cnt)
           move val to an-y in yes(cnt)
       end-perform
       move xes to vxes.
       move yes to vyes.
       move cnt to return-code
       goback.
       end program cobfloats.

and then

$ vi cobparams.c
```
