I don't usually "blast" classes (or students) but if you (any reader) have a
class that is using "MS-COBOL", then something should be done about that
class.  That compiler is INCREDIBLY out of date (and not supported).  As
indicated, Micro Focus (MERANT) has an "academic" edition compiler that you
could be using.  You could be using the free (slightly out-of-date) Fujitsu
compiler - or any of a number of other compilers.

Please feel free to pass my email address on to ANY instructor having
students use "MS-COBOL" (and I will be MORE than happy to correspond directly
with him/her).

NOTE WELL:
   Any current "conforming compiler" (that supports the '89 Amendment to the
'85 Standard) will support a 4-digit year via the "FUNCTION CURRENT-DATE"
intrinsic function.  Many compilers ALSO include an extension (that will be
Standard with the next revision) of

    ACCEPT data-name FROM DATE YYYYMMDD

to get the 4-digit year.  HOWEVER, (as stated before) any CONFORMING compiler
will return ONLY a 2-digit year if you use the (currently standard)

   ACCEPT data-name FROM DATE

syntax.
