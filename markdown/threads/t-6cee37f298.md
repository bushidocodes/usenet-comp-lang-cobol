```
I have a COBOL routine that currently is compiled using the DEC Cobol
compiler to run on VMS. I'm trying to compile this with the Fujitsu compiler
so I can call the routine from a VC++ program running on Windows. For the
most part it seems to work ok, but as best I can tell there is a problem
with what Fujitsu thinks is 0+ (signed zero). I use this routine to convert
numbers to send to the cobol routine:

void CobolPack(char* buf, long lValue, BOOL bSigned, int iDigits, int
iPrecision)
{
 const char cSignBuf[21] = "{ABCDEFGHI}JKLMNOPQR";

 BOOL bNegative = FALSE;

 if ( bSigned && lValue < 0 )
 {
  lValue = lValue * -1;
  bNegative = TRUE;
 }

 int lBufLength = iDigits + iPrecision;

 // converts long to ascii string
 LongToString(buf, lBufLength, lValue);

 if ( bSigned )
 {
  char cLastChar = buf[lBufLength-1] - 48;
  if ( bNegative )
   cLastChar += 10;
  buf[lBufLength-1] = cSignBuf[cLastChar];
 }
}

It seems Fujitsu expects an '@' or hex 40 to represent 0+ rather than '{' or
hex 7b. Is there a way to tell the Fujitsu compiler to use '{' or hex 7b?
This is causing my calculations to be off by a small decimal amount.

Thanks,
Brian
```

```
The variable 'cSignBuf' contains EBCDIC sign characters.
DEC COBOL's use of EBCDIC sign characters with ASCII
numeric characters seems to be peculiar.

Try using ASCII sign characters by changing the string
for 'cSignBuf' to "0123456789pqrstuvwxy".

HTH
------------------
Rick Smith
```
