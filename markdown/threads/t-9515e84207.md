```
From one of my "usually reliable source" (who prefaced this by saying that
this is NOT documented),

"This query is probably not strictly a COBOL type function. That is,
accessing system labels, from COBOL. This would normally be
done by an Assembler program.

1. Pure COBOL solution. Write sub-routine to process the tape
as an unlabelled tape. Reading the VOL1/HDR1 records as if they
were DATA records. Possibly use DITTO to view the layout of the
VOL1/HDR1 records for use in the COBOL program.
Note: COBOL/LE determines the label status of a tape via the
presence in the JCL of a TLBL for the filename. If no TLBL for
the filename is found, unlabelled tape processing is performed.
If a TLBL is found, labeled processing is used.

2. If a separate program can be used, write a REXX proc to
call DITTO and store the returned information in a stem.
Write this stem to a file and later read the file from the
COBOL program.

3. Single program solution, different languages. Write an
Assembler program to read and rewind the tape and pass the
VOL1 information back to the calling COBOL program."
```
