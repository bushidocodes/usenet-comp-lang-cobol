I'm still using REALIA 3.0 occasionally. And i have also delt with this
problem. The problem started to occur with DOS 6.0 and Windows 3.0 a few
years ago. I found some correlation with the size of the program I was
compiling, so I started to remove any unnessasary TSR's in config.sys and
autoexec.bat, to make the remaining memory as large as possible. I also
found out that I had to remove EMM386 (and any load-high's) to be able to
compile again.

Now I have Windows95 and uses these boot-files:
(config.sys)
buffers=20
files=20
stacks=9,512
lastdrive=e
device=c:\windows\himem.sys
dos=high
country=046,,c:\windows\command\country.sys

(autoexec.bat)
path
c:\windows;c:\windows\command;c:\;c:\dos;c:\util;c:\bat;c:\brief;c:\cob3;
c:\windows\command\keyb.com sv,,c:\windows\command\keyboard.sys
c:\dos\fastopen.exe c:
set temp=c:\temp
prompt $p$g
mode con rate=32 delay=1

When loading Windows95 you must hit F8 and select no.5 Confirm every row.
Answer Yes to everything except the last row when Win is to be loaded.

Good luck!
