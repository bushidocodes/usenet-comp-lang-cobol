Well, the most straight forward way(IMHO) to do this is do the following
steps:

1. Write a COBOL program to read the existing ISAM file and
write out a line-sequential file with that record layout.

2. Write a COBOL load program that takes the sequential file
from step 1
to create a new version of the
isam file and open it as output initially. In other words, you
have to create a new ISAM file with the new record length...

Program one:

Select input-file
organization is indexed
access is dynamic
assign to "theold.dat"

select output-file
organization is line-sequential
assign to "gerbils.txt".


fd input-file
label records are omitted.
01 old-record-layout pic x(whatever).

fd output-file
01 new-record pic x(whatever + n).
open input input-file.

read next old-record-layout.

perform until end-of-file-input
perform move-fields-from-old
write new-record
read next old-record-layout
end-perform.


close input-file, output-file.


second program:


input file is now the sequential one,
and this time you open another NEW Isam file...

open output new-isam-file.
open input the-input-file.
read input-record at end move 'y' to completely-sick-of-it-flag.
perform until completely-sick-of-it
move fields....
write new-isam-rec
read input-record at set completely-sick-of-it to true
end-read


end-perform.
close files...

Or, you could write one cobol program to read the old ISAM file
directly and load the new ISAM file in one fell swoop...
