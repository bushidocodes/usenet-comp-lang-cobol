```
Hi,

    I use the following batch file with replaceable parameters to compile &
link COBOL source code at the DOS prompt in Windows 2000.  It may have to be
modified for Fuji Cobol V3. I call the batch file "CL.BAT" for CompileLink.
The syntax is "CL Program".

echo off
rem Compile - Link COBOL source code using Fujitsu COBOL 5

set CobPath=D:\PROGRAM FILES\Fujitsu Cobol\cobol
SET @WINCLOSEMSG=OFF
rem Compile program as MAIN, using DOS Console

"%CobPath%\COBOL32.EXE" -WC,MAIN(MAIN) %1.cob

rem Pause to allow Control/Break if errors in compile
pause

rem Link, using standard Libraries
"%CobPath%\LINK" %1.obj F3BICIMP.LIB LIBC.LIB KERNEL32.LIB
/subsystem:CONSOLE /out:"%1.exe"


End of Batch file.  Note that the lines may wrap in your e-mail program
```

```
I think that if a Visual environment is available it should be used.

Batch files are better left on DOS machines...

Before you start sharpening the stakes let me say I have written hundreds of
them and still occasionally use one, but I PREFER not to. Why write a batch
file to compile and link, when there is an environment provided that will
organize my programs into projects, automatically keep track of which
modules I updated, select the correct compiler and linker options for me
(with the option for me to override), and compile and link the whole lot
when I click on "Build"?

The only reason I can see for anyone to use independent manual compile and
link steps is because they are used to working in a mainframe (or PC/DOS)
environment where this is the the only option, UNLESS you write a Batch file
or build JCL procedures.

The more "invisible" the process of compiling and linking is the more I like
it. A series of editing sessions, followed by a click on "Build", then a
click on "Debug" or "Execute" suits me down to the ground, and I would
encourage others to do likewise...

Fujitsu V3 comes with both PowerCOBOL and COBOL 85 Project Managers (in
addition to the Wincob, Link, facilities). Using a Visual Project Manager
makes  the job of maintenance easier for both yourself and any programmers
who follow you...

BOTTOM LINE:

Batch procedures do not belong in the User Manual (or anywhere other than in
DOS based systems...), as long as there is a Visual alternative provided.
```
