The issue of allowing a 5-digit number in a PIC S9(04) COMP/BINARY field
depends on compiler options *and* which release of COBOL you are using.

If you are using the VERY latest/greatest IBM compiler, e.g. IBM COBOL for
OS/390 & VM V2R2 (not V2R1), then you should use COMP-5 and you won't have
any problems.  (This is a "new" feature in the latest compiler to GUARANTEE
that "binary" fields are NOT truncated in accordance with the PICTURE
clause). (Not this release ALSO "re-introduces" an OPTIONAL method of
detecting potential truncation at compile-time.  IBM hasn't supported this
since OS/VS COBOL - and HAPPILY (IMHO) it is NOT the default in the latest
compiler)

If you are using an earlier compiler (which most likely you ARE - as the
latest one only went GA last month), then you should check your TRUNC
compiler option.  What you set this to will depend upon which compiler you
are using.  If you are using "supported one" (i.e. NOT OS/VS COBOL or an
early release of VS COBOL II), then use TRUNC(OPT) and it should accept what
you are doing.

FINALLY, you state that the compiler is giving an "error message".  Is it
actually an ERROR message - or is it a "warning" or "informational" message?
If it is just I-/W-level, then you PROBABLY can use what you are doing
already and the application will "run as expected".
```

TRUNC     This compiler option creates code that will truncate the

          Performance advantages
                    TRUNC(OPT) does not generate extra code and generally

          Performance disadvantages

                    conforms to the PICTURE and USAGE specifications.
                    TRUNC(OPT) is recommended where possible.

If there is any chance that you will be moving a PIC S9(4) COMP field to a
PIC S(5) either display or packed decimal, I'd strongly suggest using
TRUNC(BIN).  A 5 digit number in the binary sending field WILL BE TRUNCATED
when moved to the receiving field if using the TRUNC(OPT) or TRUNC(STD)
options of the COBOL II compiler.  Read about the TRUNC compiler option in
the Programming Application Guide.  It suggests using TRUNC(BIN) when using
DB2 with COBOL.
```
