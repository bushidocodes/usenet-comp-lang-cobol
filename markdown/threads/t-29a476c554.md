Two solutions.

One, the "right" way for Windows is to have the program do a Win API
call to get the list of programs running. If its own path is on that
list, then exit with a warning to the user. Alternately exit & activate
the running instance. Advantages: it's clean & fairly simple.
Disadvantages: have to get into the Win API area.

Two, all COBOL. Use a flag file or busy indicator, the file is created
when the program starts and deleted when the program ends normally.
When initializing the program first looks for the file & if found take
appropriate action. If not found the file is created. Must be deleted
by autoexec/startup process in some way.

Advantages: All COBOL, no Win calls. OS independence. Disadvantages:
extra overhead on startup as the startup delete has to be set up on each
PC the program runs on.

This would be a good way to get started with Win API calls, they are
pretty straightforward.

JR
