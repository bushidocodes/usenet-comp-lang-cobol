Have to love it.

I love Thane's answer, where product X can do MIRACLES and
easily on top of that.

Experience will show you that there are no easy answers
to this type of situations just possibilities.

If you want to look at MF for possibilities you might want
to have a look at NetExpress.  For the GUI side of your
application Dialog System is one possibility as well
as the Web enablement tools that they provide.  If used
in conjunction with their AAI middleware you can
create an interesting, platform independant, client/server
application.

BUT you also have to look at other possibility to construct
your front-end.  Might it be Visual Basic, PowerBuilder or
whatever else you might think of, you have the pick the
set of tools that you will feel comfortable with on the
development side as well as looking ahead for the
maintenance cycles that this application will have to go
through in the future.

I am not saying that Thane's suggestion of using Flexus is
wrong, far from it.  Flexus would not be able to survive as
a company if their products/solutions were inefficient.

I am just saying that the statement is gratuitous at best
and lacks any value.

I suggest that you contact the different vendors such as
Micro Focus(Merant) or Flexus and ask to be put in contact
with customers that have gone through such a project.

Good luck on your project Jason.

You don't understand Thin Client.  The program runs on the server -
the WHOLE THING - the client is simply a "Gui" processor that takes
the Sp2 calls (screen intefface) and shows them to you on your PC.
I'm telling you that with a DIAL UP internet connection you could be
running a "Windows" application - you could not tell the difference -
and the "host" program is running on the Unix box. (written in MF
COBOL).  

It really is as magic as I said it was.  The ONLY issue is the same as
the issue would be for converting text mode applications to GUI.

If a user was already using the Motif SP2 version, they could go to
thin client in SECONDS.   

I know I sound rabid.  But it really IS that easy.  

Thin client does not mean a slow clunky HTML interface, it does not
mean ASP or JAVA programs.  It's native windows API calls on the
client - with the program running on the server (unix).
