Hi Chris.

Signal 4 is "Illegal Instruction" here (you can check this on your
system by looking in the /usr/include/signal.h or (more likely)
/usr/include/sys/signal.h file).
This means that the .gnt code is invalid either due to (a) a bug in
the code generator or a disk error causing the disk file to contain an
illegal instruction, or (b) a problem at runtime which is causing the
(dynamically loaded) program to become corrupt in memory after it has
been loaded. BTW, I'm assuming you mean CBLDC001 (a data compression
routine).

› This error is only occuring in one data file.

Hmmm. I wouldn't have thought it's an error in the data file itself,
but rather that something in the data file is triggering the above
problem (either by making CBLDC001 take a different execution path,
or causing something (either Micro Focus or your software) to corrupt
the .gnt file as loaded into memory). I have seen the second case a
couple of times (later versions of MF COBOL will set such memory to
read-only, if the OS supports it).

Out of interest, can you try running with the COBSW=-l0 switch
(that's minus-lowercaseL-zero) ? Do you get the same results ?

Cheers,
Kev.
