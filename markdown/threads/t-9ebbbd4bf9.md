Bar code printing is very dependant on the printer and the environment
which you are using, not to mention which bar coding scheme you wish to
use (many exist).

Some printers can directly generate barcode characters, in other cases you
must descend to low level modes and send bit map graphic chars. One final
method could involve the use of True Type to generate the bar code.

Hi,

As I work in the Airfreight Industry I recently had to write
a barcoded label system using MF on a Pc. I talked to Micro
Focus about how to write barcodes from a Cobol program. And
it was a case of sending out bit mapped graphics.

But, it is dependant on printer hardware. If you want to
print barcodes on a dot matrix epson compt. printer, then
I use a barcode printing program which I call from the
Cobol program.

Now we use dedicated TEC (No advertising intended) thermal
printers and these printers require you to send layout and
data strings to them, and this will auto write a barcode
and print the label for you.

What type of equipment and paper/lable are you gonna be
printing on?

As a rule avoid dot matrix, as the output is thin and
liney. Although I printed a barcode on an Epson printer
then photo copied the output and then faxed it to someone
and they culd still read it!

One other consideration is the barcode type, CODE39, 128,
etc!

Let me know this information and I will try to advise you.
