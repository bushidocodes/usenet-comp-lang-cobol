```
Hey everyone,

I encountered a very strange phenomenon recently. In a program I wrote
something like :

Move An-Element1(Counter1 + 2:Counter2 - 4) To Another-Element1
Move An-Element2(Counter3 + 5:Counter4 - 6) To Another-Element2

This code worked fine when I ran this program using a debugger (Expeditor).
When running the program without (and thus processing the production data)
it seemed like losing track of the values of this counters and by
consequence moved the wrong part of the elements.
All the counters were initialized and filled with a correct value before
they were used in the above mentioned moves.
I changed the code to :

Compute Counter5 = Counter1 + 2
Compute Counter6 = Counter2 - 4
Compute Counter7 = Counter3 + 5
Compute Counter8 = Counter4 - 6
Move An-Element1(Counter5:Counter6) To Another-Element1
Move An-Element2(Counter7:Counter8) To Another-Element2

and it worked fine.

Has anyone an idea why the first version worked with the debugger and didn't
without?
```

```
Given your reference to "Expeditor" (which I assume you mean as "Xpeditor")
my guess is that you are running on an IBM mainframe.  If this is not
correct, let me/us know where you are running and with what compiler.
Clearly this code SHOULD work the same in all environments and with all
compiler options/directives.  I have, however, a couple of GUESSES as to
what may be happening.

1) (Most likely in my opinion),  When you tested this, you compiled with
TEST,NOOPT compiler options.  When you ran the "production" version, you had
NOTEST,OPT.  Although it is fairly unusual (now a days), there have been
cases in the past when IBM's "optimizer" has failed to produce the correct
code.  If this "problem" is recreatable, (i.e. it you can show the
production and test versions yielding different results), the I think you
should report the problem to IBM.  (Assuming the "test" version is the
expected results; if not, then report it to Compuware for a problem with
Xpeditor).

2) (Less likely)  Did you "cut-and-paste" your source code below from your
actual COBOL program?  If not, be VERY careful about where you do and do not
have spaces between "+" and "-" and following integer-literals.  The
difference between arithmetic expressions and unary operators is REAL easy
to miss.

3) (Medium-possible)  Have you checked that your run-time options are
identical in your test and non-test environments?  Check WSCLEAR (VS COBOL
II) or STORAGE (LE) particularly.  Also for SSRANGE and/or CHECK options.
```
