I'm trying to replicate some programs from the IBM mainframe world which
support a variable number of arguments and are invoked from Fortran and
Cobol programs. For Example:

In Fortrans/COBOL:

CALL MYPROG(parm1, parm2,...parmn);

On the mainframe, the compilers always put a VL bit on the argument
address list entry of the last parameter so that the called program
can tell how many parameters were passed to it.

On my UNIX box, I've implemented MYPROG in C using the varargs
interfaces. From what I can tell, I need to have a extra leading
parameter which contains a count value so MYPROG will know how many
parameters to look at. Is there any way for me to allow my users to
leave their invocations of MYPROG as they were on the mainframe?

BTW, the environment is either IBM's xlf on AIX or MicroFocus COBOL on
any number of UNIX platforms.
