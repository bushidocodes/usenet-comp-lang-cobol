Both products work well, and I have sites running them.

They construct the SQL statements at runtime themselves, which means that
your COBOL application source code still has COBOL IO statements.

Advantages are

It's easy to implement as source code changes are minimal (Occurs clauses
and Multiple record type files cause problems).

As a Software vendor you can offer your product with a choice of databases.

Disadvantages

Performance - As the SQL has to be constructed at "OPEN" time this takes a
little more time than a standard open, but performance is still more than
adequate.
