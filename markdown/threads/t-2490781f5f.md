Hi, folks.

I successfully installed the new release of Micro Focus COBOL 3.2 on my
AIX-machine and it solved my arithmetic problems (100 + 15 = 16 (!), if
you remember).
"cob -V" now says "PRN:2XCLY/ZZO:7a.1a.11.06".

But now I have got a new problem that did not occur with any previous release:

I need to "OPEN EXTEND" a printerdevice several times from one run-unit to
ensure that printouts to named pipes do not get seperated into different
jobs in the printqueue, if the contents of the printout is created by
different subprograms. For example: an invoice with a barcode on the header,
created by a separate barcode-routine that is "CALL"'ed with the name of
the printerdevice as one parameter.

INVOICE-ROUTINE BARCODE-ROUTINE
--------------- ---------------
open extend PRINTER open extend PRINTER (*)
... print invoice header ... print barcode
call BARCODE-ROUTINE using ... close PRINTER
... finish invoice exit program.
close PRINTER
exit program.

Both programs define PRINTER as

select PRINTER
assign PRINTER-PATH
lock manual
status FileStatus.

The statement with (*) fails with the status-code 9/20: 'Device busy', because
the "OPEN EXTEND"-statement in INVOICE-ROUTINE used it's default behaviour to
lock the printerdevice exclusively (even with "LOCK MANUAL", which should
override that default, according to the manual).

I tried

- all kinds of "LOCK MODE"
- compiler-directives "AUTOLOCK" and "FILESHARE"
- run-time configurable "same_proc_excl_detection=FALSE"

without success.

If you can help me solving this problem, please e-mail or
post to comp.lang.cobol.

Thanx, cal.
