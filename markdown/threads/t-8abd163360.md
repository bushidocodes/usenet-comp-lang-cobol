```
Let's get back to the original question: Is is an error for a perform to
perform itself?
The answer is you will get a warning message that you may cause an
endless loop. (IBM VS COBOL II).
In the example given:

PARA-1.
PERFORM PARA-1.

You would definitely cause an endless loop.

It is strictly not illegal however to perform back to the top IF you
have provided a means of escape. For example:

05 WS-COUNTER PIC 9 VALUE ZERO.
...
PARA-1.
ADD 1 TO WS-COUNTER.
PERFORM PARA-1 UNTIL WS-COUNTER > 5.

This will end the perform after 5 loops.

In my opinion this is extremely bad code. If any of the programmers
working for me wrote like this, they'd be back in elementary COBOL
structured programming class real fast.
```
