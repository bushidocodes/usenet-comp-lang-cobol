> I've never used it, so I'm not familiar with

I've only used it for basic source control, but there are lots of
free CVS add-ons, and many other tools integrate with it.

CVS may require a somewhat different development culture than what
the OP is used to.  It doesn't lock files for changes; instead, it
automatically merges simultaneous changes and helps developers
resolve conflicts.  People used to systems that lock files for update
sometimes find this unsettling, but it works well in practice.

> It's GPL, though, so it might be worth a look...

It's also widely available, works very well for distributed develop-
ment (including over high-latency or slow network connections), has a
variety of command-line and GUI clients, is easy to script, and so
forth.  I still use RCS for some local-only files, like system
configuration stuff, but CVS is a big improvement on RCS and a *huge*
one on PVCS.
