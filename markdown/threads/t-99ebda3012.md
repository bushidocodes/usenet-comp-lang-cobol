The 1900 was quite a simple set of instructions and the architecture
catered for 'extracodes' where if it wasn't available as hardware
it was passed to Exec (the operating system) which could then do
the job using software.  This allowed the smaller machines to have
less hardware while still still being able to run programs
compiled for the larger machines.

The machines also used interrupt driven I-O which meant that the
CPU could carry on processing while peripherals used (what today
is called) DMA to access the buffers.  This gave a very efficient
multi-tasking.

I suppose in todays terms the 1900 was the Amiga and the 360/370/..
the Windows NT in that the 1900 was fast, efficient, multi-tasking
and dead.        ;-)
