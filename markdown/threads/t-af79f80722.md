There are apparently organizations that will do this work, costed per line.

I've had a few experiences moving a code base from one platform to another,
and have found marginal use for mechanical conversions.  The problem lies in
the domain of the source code -- it's just too huge for any meta-compiler to
take all of the factors into account.  As a result, many errors are
introduced, some of which can be subtle.  Ultimately, you end up analyzing
all of the code anyhow.

The most recent mass conversion I assisted on, was moving from one COBOL
screen interface unit to another.  Periodically, we would identify patterns
of translation.  In those cases, I either used a good search and replace
tool (like UltraEdit,) or wrote some small utility that could do the job.

I wish you luck in your endeavor.
