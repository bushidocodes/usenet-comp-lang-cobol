You probably want to see the CALL x"84" 'Call by Number' routine.
You should find it in the index under 'Interrupt - execute DOS'.
This will cause an INT 0x21 with the parameters you specify.

› EXAMPLE:
› -----------------
› Some of the things I want to do is a "direct console read/write"
› which is DOS function number 06 (hex) which uses one parameter.

Why would you want to do this ? At the lowest level you should
be able to use: CALL "CBL_GET_KBD_STATUS" etc, at a slightly
more refined level an ACCEPT field WITH TIME-OUT may do as
you require without dropping into assembler level.

Also doing direct console reads is usually for the purpose (but
not always) of wanting to sit in a tight CPU loop using as
much resource as possible while the user makes up his mind
which key to press. This is usually considered inappropriate
behaviour on all but plain unadorned MS-DOS.

Most programmers have discovered that there may actually be
another program that could make some use of all those wasted
cycles.

At the very least, if you do want to emulate inkey() then at
put a INT 0x2F, AX=0x1680 in the loop with it.
