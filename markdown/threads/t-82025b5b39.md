IDCAMS utility.  Once you have "FTP'ed" a "dump" of the file to the PC, you
are OUT OF LUCK.

It is possible to determine if a VSAM file has fixed-length or
variable-length records, and the maximum LRECL by running a LISTCAT
command using the IDCAMS utility.  It would also be possible to
determine the keylength for a KSDS file and the relative key position
(keys don't have to start at a zero displacement relative to the
beginning of each record).  I believe this kind of information is
stored in the cluster component of the VSAM dataset.  If you have a
KSDS VSAM file and did a binary ftp of the cluster, data, and index
components to a PC, you might be able to interpret the cluster
component to obtain LRECL, Key Length, and RKP.  

VSAM does not store any definition of fields within records other than
the key.

If I had to transfer a VSAM file to a PC, I would print the LISTCAT on
the mainframe first, then REPRO the data component to a flat file, and
only transfer the data records.

Dazza,

If your file is an exact copy of a VSAM file, the first thing to do is to
find out the CI size. A CI (Control Interval) is a fixed size container of
512, 1024, 2048, 4096 etc. bytes.
Each CI contains at the beginning some records, and at the end control
information.

The last 4 bytes in each CI contain the CIDF (Control Interval Definition
Field), a field that contains information about the freespace in the CI.
Just before the CIDF are 3-byte RDFs (Record Definition Field). They are
stored from right to left, with the last RDF contining information on the
first record in the CI and so on.
If the records in the CI are fixed length, there are just two RDFs, the
rightmost giving the length of the records in the CI, the other one the
numbre of records.
If there is just one fixed length record in the CI, there is just one RDF.
If the records are variable size, there will be one or more pairs of RDFs,
each pair containing a length in the rightmost RDF and the number of
consecutive records with this length in the leftmost RDF.

I think that with this information you can keep on playing a bit more :)
