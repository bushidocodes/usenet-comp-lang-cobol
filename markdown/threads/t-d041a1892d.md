A 39 error is file not found, so that is probably a bug in your VB.  In the
VB file, write it out as a text file, and test that the data is ok with a
standard text editor.  You can also create a test file with a text editor,
and read that into your Cobol.  That way you are testing one program at a
time.

In Cobol, a line sequential file is a standard file just like source code,
each line terminated by a CR,LF.  Make sure your record description is large
enough to hold the longest line in the file.
