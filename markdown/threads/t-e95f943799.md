```
Just to clarify a couple of previous posts on PERCobol development vs
run-time support for OS/2, I got the following note in reply to my inquiry
to the vendor,

"Bill,
PERCobol supports OS/2 as a development platform with PERCobol 2.7.  The
latest release of PERCobol  3.0 has added a new IDE built by LegacyJ rom IBM
Eclipse technology.This new IDE contains language sensitive editing (old one
did as well), incremental compiles, "local" and "global" compile parameters
for mixed COBOL source (important since PERCobol support 15 vendor dialects
of COBOL), and many other nice features.  HOWEVER, PERCobol 3.0 does not
currently support OS/2 as a development platform; the reason is that IBM
does not support OS/2 with Ecplise.  PERCobol 2.7 can still be ordered for
OS/2 but, in the future, we would recommend compiling on an other platform
for deployment to OS/2 that will remain possible for as long as the platform
is supported by IBM.

PERCobol is the COBOL compiler that compiles to Java byte code.  This is
extremely important for those who would like to deploy applications on J2EE
compliant platforms; meaning that PERCobol is the only COBOL compiler that
can generate J2EE class files that are native ONLY to the Java platform. A
wrappering technology that takes a OS native object and wrappers it to
permit it to interface with Java has only created a liability not a
solution. PERCobol  is the only solution available that can compile COBOL
and generate platform independent applications, Servlets and Enterprise
JavaBeans for deployment on Application Servers for Web Services."
```
