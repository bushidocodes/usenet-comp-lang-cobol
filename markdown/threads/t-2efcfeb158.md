   I don't believe, that a KI-language, a self-modifying, interpretative

language with a nice implementation of the lambda-construct can ever
be converted in a cobol-structur. There is nothing like
a List-constructor nor an atom in Cobol.
You can write a LISP-interpreter in cobol
(hardly, but i've done it on a sinclair spectrum with basic long ago )
but you can't convert
LISP to cobol. You can write Programms in
COBOL that does the same as your little LISP-Progs, but
not automaticly!
