Here's a quick shot at it. There are many nuances, too big to explain in
an e-mail. Try the SYBASE website for a book reference, I'd guess.


Generally you use a CURSOR to get thru a table sequentially.
DECLARE MyCursorName CURSOR FOR
SELECT TableField_1, TableField_2, ..., FROM MyTable WHERE TableField_a
= ... AND TableField_b = ... etc.

Then you need to OPEN MyCursorName, FETCH MyCursorName (until all-done),
and CLOSE MyCursorName. I'm not sure what tells you 'all-done' in your
product (perhaps a field called SQLCODE). A FETCH brings you the next
record meeting the SELECT criteria.

To go backwards, DECLARE a new cursor and append a clause to the above
DECLARE: ORDER BY TableKeyField_a DESC.

The NEXT, PREV will take two cursors - a forward-looking one and a
backward-looking one. You get the FIRST record on the first FETCH of the
forward cursor, and the LAST on the first FETCH of the backward-looker.

You can also play with the keys to go backward (subtract one from MyKey
giving MyNewKey then SELECT ... WHERE TableKey = MyNewKey), if the Key
is a number and they're incremental by 1.

There's a way to get the LAST without a CURSOR, by using MAX, using
something known as a subselect.
SELECT... WHERE ... AND TableKey = (SELECT MAX TableKey FROM....).

Simile for MIN instead of MAX, for the FIRST record.
