----------------------------------------------
INTRODUCTION

This Demo reads in the binary data of a .jpg, (*** da Vinci's 'Mona
Lisa' ***), and then writes it to a database 8KB at a time.  The .jpg is
stored in the database as an Image field. It then reads this Image field
8KB at a time into a memory buffer and then writes it to a file on disk
and then opens the file.

SOURCE FILES:

BlobDemo.cbl
SQLCA.CPY (*** M/F copyfile)
WINDOWS.CPY (*** M/F copyfile)

REQUIREMENTS:
This demo works with SQL Server.  Create an ODBC data source with the
name BlobDB. Add the stored procedures to the database.
	
OPERATION:
After you have set up the ODBC DSN and added the stored procedures to
the database, run the program.  It will create a table in the database
and then write the binary data from the .jpg to the table.  It will then
read this binary data from the table into a memory buffer and write it
to disk as a new .jpg file. It will then open this new .jpg file.

---------------------------------------------------------------------------

CBL_WRITE routines), I'll try and dig out a copy of Alan Wheeler's
original WRITE-CHUNKS.cbl, (or whatever it was called). There might be a

Hello,

You cannot transmit raw binary data as part of SOAP XML message. You
need to use one of the following methods:

1. transform JPEG into S/MIME type and then to transmit it as a
regular string (doubles image size)
2. use base64 encode to transform jpeg binary into something suitable
to be transmitted along the SOAP xml (50% more data)
3. send images as an attachment appended to your SOAP packet (this
would be easy with Cobol.net)

In either case the other end needs to know how to handle the data that
you are sending in order to recreate the jpeg. You also need a very
large PIC X field in this case, long enough to hold JPEG encoded
content.

Since you are using Netexpress you can use MSXML to base64 encode the
jpeg image, but again, it depends on web service provider chosen
method to deal with jpeg (I'm assume you are just consuming the web
service).


Here is a VB example that could be adapted to use with Netexpress $OLE
bridge.

http://www.nonhostile.com/howto-encode-decode-base64-vb6.asp

Regards, Emerson
