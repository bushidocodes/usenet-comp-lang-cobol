COBOL does not create a physical end-of-file marker.

End-of-files are the solely a function of the hardware. You cannot
obtain a byte or combination of bytes which can be decoded as an
end-of-file.

Here's the code you want:

OPEN INPUT DISK-FILE.
OPEN OUTPUT TAPE-FILE.
PERFORM WITH NO LIMIT
   READ DISK-FILE
      NOT AT END
         WRITE TAPE-REC FROM DISK-REC
      AT END
         CLOSE DISK-FILE CLOSE TAPE-FILE
         EXIT PERFORM
      END-READ
   END-PERFORM.

We once had a bottle-neck in the keypunch department. Bought an
expensive key-to-tape system. Took the tapes and ran a program to
punch cards from the tapes.

Go figure.

Hi,

One thing to keep in mind is that by default  FTP  translates its output into
line sequential ASCII files which can be a problem if you're used to receiving
your files in fixed record fixed block EBCDIC format.  If EBCDIC is to be sent
untranslated I believe that the send  mode has to be set to BINARY prior to
transmission.  Either that or you need to change your program to read ASCII
line sequential which may or may not be an option depending on the compiler you
use.

Regards,
Jim Castro
