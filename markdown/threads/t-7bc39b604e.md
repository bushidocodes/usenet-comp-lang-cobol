Mr Hubbell is yet again correct:

See the GetFileVersionInfo API call and its related functions.

Note that this only works if YOU compile the version info into the EXE (or
DLL) as string resources (see below for the resource file entry). When done,
the info WILL show up when you click in Explorer and do the "properties"
thing (it's not magic, Thane) ; or you can load the resource data from
within the EXE/DLL and use it in your "about" dialog.

I don't know if compiling resources into the executable is possible with
your compiler. But hey, it works great with my BASIC-language stuff.
