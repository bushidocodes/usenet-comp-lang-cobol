```
I tried to post a while ago, but I'm not sure it got thru. If this is a
duplicate response, just ignore it.

If I remember correctly, ANSI 74 COBOL only supported a maximum of three
dimensions in an array, at least in IBM mainframe implementations.
Therefore you would not be able to define or reference EEE. Also, your
example seems to use C-style array referencing, which would not work in
COBOL.

You have a duplicate dataname X, which I assume is just a type in your
example. And your reference to DDD would not work since the first
subscript is omitted.

So, I think the statements should look like this:

MOVE DATA TO AA (1)
MOVE DATA TO BB (1,1)
MOVE DATA TO DDD (1,1,19)

And if the compiler will support more than three dimensions, then the
final statement would look like this:

MOVE DATA TO EEE (1,2,10,1).

I hope this helps.
```
