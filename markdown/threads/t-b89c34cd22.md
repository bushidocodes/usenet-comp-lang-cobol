```
My understanding is that in a sequential file (MVS/QSAM),
there are four "hidden" control bytes at the beginning of
each record that include a reference to the length of the
given record (provided the file is defined as variable
block?).  I am wondering if it is possible to access that
length value.

This is all fairly new to me, so I could be completely
missing the boat.  Here's my dilemma.

I am reading from a variable record length sequential file
and would like to write selected records (of various
lengths) out to a second sequential file for later
processing.  When I try to just read into, and write out of,
a record of max. length, it either includes the next 'n'
bytes (from successive records) to fill the record (01 level
definition) or inserts blanks (read into working storage).
It seems to me that a feasible way of achieving the desired
results would be to read in a record of the max. length,
look at its actual length in the control bytes and then move
the first 'length' bytes to a record with length defined as
occurs depending on ... length, finally writing that record
out to the new file.

Any assistance would be greatly appreciated.

Tim
```

```
Tim,

    If you are executing COBOL2, just define an unsigned working-storage
integer and include it in your FD as 'RECORD IS VARYING IN SIZE FROM nnnn TO
nnnn DEPENDING ON working-storage integer. After the READ, the integer will
contain the length of the record just read. Remember, it must be an UNSIGNED
integer (preferably a halfword or fullword binary). Before you issue the
WRITE, populate the integer with the appropriate value.

HTH....

WOB
```
