Another possibility...if your input file is defined as line sequential,
check the 'N' run-time
switch.  The default is 'ON'.  When 'ON' and writting a file, any
character < x'1F' will have a
null (x'00') inserted prior to the character.  When reading a file, the
null will be stripped
off.  Your packed fields most likely have some nulls, so each first null
will go to the bit
bucket.
