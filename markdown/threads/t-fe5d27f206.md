The production JOB is probably calling a PROC (procedure). 
Check in the proclib for that name (people in scheduling 
could help you if you don't know how to do this, or the 
application programmers with more experience).  The actual 
PROC that was run should have the real file names - 
DSN=filename.extension.etc.etc,DCB=blksize,VOL=etc,etc,etc.

Since you are looking for OS/390 I assume that ISPF is 
available.  With the name of the PROC and the PROCLIB name 
you should be able to find it on-line.  There should also be 
a way to look at the specific job run using the number 
assigned to it when it ran or by JOB name, giving a list for 
the last several runs of that job.  These are SYSOUT 
listings that are available on-line with ISPF - again check 
with the programming department or the scheduling 
department, or call the operator - anyone of them could 
point you to how to do this.

Lastly, there is the paper output listing which should have 
the job's SYSOUT containing more information than most 
anyone needs.
