Try the following, MOVE FUNCTION WHEN-COMPILED TO curr-date.
Where curr-date is Pic x(21).

Where WHEN-COMPILED is PIC X(21) and it contains:
yyyymmddhhmmsstt+hhmm.
yyyy = year
mm   = month
dd   = day
hh   = hour
mm   = minute
ss   = seconds
tt   = hundredths of seconds
+hhmmm  = Hours & Minutes ahead/behind GMT

Rich,

With COBOL/370 you would use the CURRENT-DATE FUNCTION which will return the
following information in a 21-Byte area:

CCYYMMDD    ===> Bytes 01-08 (Current-Date as CCYYMMDD)
HHMMSSTH    ===> Bytes 09-16 (Current-Time as HHMMSSTH)
'+' or '-'  ===> Bytes 17-17 (Ahead or Behind GMT)
GMT Offset  ===> Bytes 18-21 (HHMM off GMT)

03  WS-CURRENT-DATE PIC X(21).

MOVE FUNCTION CURRENT-DATE TO WS-CURRENT-DATE.

Also, check out:

03  WS-DATE-CCYYMMDD PIC 9(08).

CALL 'IGZEDT4' USING WS-DATE-CCYYMMDD.

You may also want to check out the following Web Page address for IBM Book
Manager ===> http://ppdbooks.pok.ibm.com/cgi-bin/bookmgr/bookmgr.cmd/library

See the COBOL for OS/390 & VM Programming Guide:

--- paste from manual ---

4.7.7.5  Accept with 4 Digit Year             � Copyright IBM Corp. 1991

  The Working Draft for Proposed Revision of ISO 1989:1985 Programming
  Language COBOL defines an extension to the Format 2 ACCEPT statement to
  return 4 digits for the year.  Two optional keywords are added to return
  DATE or DAY to return dates in the format YYYYMMDD and YYYYDDD
  respectively.  For example:

      ACCEPT today FROM DATE YYYYMMDD

  would return "19970807" and

      ACCEPT today FROM DAY YYYYDDD

  would return "1997219" for August 7, 1997.

  See IBM COBOL Language Reference for complete syntax details.

--- end of paste from manual ---

Even the recent IBM COBOL for MVS & VM V1R2 does not know this form of accept,
but the FUNCTION CURRENT-DATE works fine:
