```
-> I'm about to become the proud owner of several 'legacy' computer
-> systems that report that they are written in something called DLY280.
->
-> I assume this is some type of cobol look-a-like language that was
-> invented in the 70's by some vendor or another.
->
-> Am I close ? Does anyone have a clue what this beast might be and
-> what I'm likely to encounter should I have to open it up and
-> actually have to enhance such a system.

I use Dyl280 frequently and found it very easy to learn and use. It
has its own logic cycle that can handle the standard reading and
writing on its own or you can exit the cycle and do manual reads and
writes if you prefer.

You can use COBOL copy members for file descriptions or easily write
your own. DYL280 only needs the fields you will reference in the
program described (you give the relative location of the field so you
don't need to tell it what comes before or after a field.

It is very easy to code and is great for quick and dirty applications.
Much faster to develop in than COBOL though I havn't written anything
very complex with it.
```
