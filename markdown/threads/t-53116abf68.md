```
As I understand it, you are emulating DOS under a time sliced system and
within that, you  trying to trap the system clock and redirect it. As long
as you ran the program in protected DOS mode, NT was smart enough to trap
your trap, and make it a local trap only so that you did not crash every
other program. Now, you also want to take over memory management, and trap
the NT trap that traps your original trap.

I am not surprised that NT prevents you from doing it.  In fact, I am
surprised that you got as far as you did without converting the programs to
the OS you are running on.

If you cannot convert the programs, then you are going to have to run them
under DOS.  If you convert them, then you are going to have to use a system
call to interrupt at the system clock level. Trying to write hardware
patches at the system clock level for a time-sliced OS is a pretty dangerous
game.

Basically, you are at the point that you have to pee or get off the pot.  Go
to 32 bit, or run in DOS.  The OS is getting too complex for simplistic bare
metal patches.
```
