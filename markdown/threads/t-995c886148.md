Can't help you with questions 1 and 2. But number three is completely
lacking in challenge if you take 20 minutes to read the upgrade
instructions on the CD or at http://docs.oracle.com before jumping into
it.

Be sure to make sure that shared memory and semaphore settings are correct
for 8.1.7 and your environment. Don't just trust that the SysAdmin that
did it for 8.1.6 did it correctly for 8.1.7. In Solaris set them in
/etc/system. In HP/UX using SAM.

you
are doing). So, when you change Oracle 8.1.6 for Oracle 8.1.7 you will
have to relink to produce a new Acu4GL (libraries do change from one to
another).
