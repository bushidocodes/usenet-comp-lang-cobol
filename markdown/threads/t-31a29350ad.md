```
At my company, we've just completed a system that is written in COBOL and
runs under CICS TXSeries on NT and uses embedded SQL with the DB2 UDB
pre-processor.  The SQL is all in I/O subroutines (basically one per table),
which are separately compiled and linked into .dll's.

Now we plan to re-write the I/O routines to use embedded SQL for ODBC via
the NetExpress compiler.

Has anyone done this already ?  Meaning used ODBC with TX Series.  I'm
concerned about running ODBC under TX Series and how to handle the ODBC
connections.  I'm thinking that I'll need to use named connections, connect
at the beginning of each transaction, disconnect at the end, and probably do
a set to the named connection in each call to an I/O handler.

I'd rather not have to connect at the transaction level, but am not sure how
to pool them any other way.  I could have an array of names and in-use
switches stored in temp storage.  But I'm afraid that the connections would
not persist after the transaction that first opened it ended.

Thanks in advance for any help and advice,
Ken Floyd
Shaw Systems
kfloyd@shawsystems.com
```
