MOST COBOL COMPILERS ARE SYSTEM SPECIFIC.  Perhaps the compiler has an option
for a specific font.  I deal with EBCIDIC on my compiler and we use a program
to translate the font to ascii using a conversion table.  Whenever we
download something from the mainframe.  I have no idea what happens with a PC
compiler.  I would assume on a good PC compiler if you can install the font
it can be used.  There may be an option you have to set.  JAVA should be able
to convert the font if you know what font the compiler is using.  There is
support for encoding in a specific font in JAVA using a PrintWriter Class I
do beleive.
