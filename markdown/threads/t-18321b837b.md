CreateProcess() will do it but you need to use the command processor as the
executable, for Windows 9x anyway. Query the environment variable %COMSPEC%
then use it, probably with a /c option as the executable. You can also use
ShellExecute() or ShellExecuteEx().

Karl Wagner

Hi,

This solution doesn't use any API commands but works.  String together the
batch command as you would to run the command in DOS and then call "SYSTEM"
(delimit the command with a null or it fails) -

.....

W-CMD        PIC X(255).

.....

STRING "C:\MY.BAT" DELIMITED BY SIZE
       x"00" DELIMITED BY SIZE
    INTO W-CMD.

CALL "SYSTEM" USING W-CMD.

IF RETURN-CODE NOT = ZERO
    DISPLAY "CALL OF BATCH COMMAND FAILED !"
END-IF.


Cheers

Nick Miles

I assume you are talking about the default action associated with file
extensions. If this is it. The explanation is a two step process.

First, look at an existing solution:
    Open your Windows explorer
    Choose Tools | Folder Options
    Select the "File Types" tab.
    Scroll until you find the TXT definition
    Click Advanced button (this is W2K, it might be "Change" or some on
other Wins)

    You should now see a list of actions, select the "print" action and
click the Edit button

    In the dialog you got, you will see the Action's name, and a command
line description like:

    C:\WINNT\system32\NOTEPAD.EXE /p %1

    The '/p' parameter here is what I think you are talking about, the %1 is
the name of the selected
    file.

Second, now, that you have seen how it looks like. You will have to make
your own definiton. I presume your files have an extension that does not
interfere with an existing file type. Go back to the list of file types.

    Click the New button.
    Follow the dialogs

This will be pretty extensive to document here, especially since my W2K
shows a different dialog than is used in Win9x and Win NT 4.x so I would
suggest you try yourself.

If this is what you were looking for and you head into trouble, do not
hesitate to ask.
