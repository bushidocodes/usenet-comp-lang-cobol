Files with variable length records need to have some way of knowing
where the next record starts. In this case it has added a record
header that specifies the length of that record. If you had written
several records then the bytes following the first would give the
length of the next record.

  |len|record data|len|record 2 data of some length|...

How else could it read the file as a series of records ?

With COBOL standard files the only way to avoid the |len| is to use
fixed length, the only way to avoid a terminator is to not use line
sequential.

One byte sequential does that.

Or use the CBL_ byte stream file routines:  CBL_CREATE_FILE
CBL_WRITE_FILE CBL_CLOSE_FILE.

Normally files are written so that they can be read back. There must
be some indication of where records start and end. Fixed length
sequential needs no terminators because each record can be a fixed
length slice. Line sequential have variable sized records each
terminated by a CR/LF (or CR or LF depending on various factors).
Variable record sequential may have record headers that give the size
so the system knows where the next record starts.

If you want none of these, ie variable length but no terminators and
no headers then you can do this with a record sequential file with a
one byte record length.

      SELECT Output-Bytes ASSIGN Bytes-Name.

      FD Output-Bytes.
      01 One-Byte             PIC X.

This may be rather slow.
