```
You'll need the FD entry of the data files to interpret the record
structure correctly so the original source is required. Failing that
you may be able to get the record structure of indexed files using
Micro Focus Data File Editor. This is part of Workbench 2.5.

If your intention is that you no longer access the files using the
original COBOL application, you're better off converting to another
format. Write a program to read the data in and then write to an
intermediate flat text based format (line sequential) or if MS SQL
Server is the intended target, a BCP input file. Then write another
app in the tool you intend to use to import the data.
```
