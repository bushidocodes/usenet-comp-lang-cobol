If you have ORGANIZATION IS LINE SEQUENTIAL in the SELECT statement, then all
trailing spaces are deleted; this is a standard ASCII text file. If you want
to retain the spaces, use ORGANIZATION IS RECORD SEQUENTIAL instead. NOTE:
the next program must be using a different SELECT description than the first.
Alos, note that RECORD SEQUENTIAL does NOT add the "carriage return/line
feed" characters to the end of the record - you must do this "by hand" if you
want to use a standard text editor on the file.
