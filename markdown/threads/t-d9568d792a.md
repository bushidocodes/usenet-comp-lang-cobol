› least in our eyes, we had the fail-over handled on the hardware side.

Tandem handles fault tolerance (NonStop) with a combination of
hardware and software. Disks are mirrored, channels are duplexed, and
so on. There is no single point of failure within the system. All
Tandem machines have multiple processors (up to 2000 or so if you
want). On Stratus, each process (task) uses two processors that run
in lock step and if one fails the other takes over. On Tandem, a
backup process is in one processor but it is doing nothing (the
processor can be running whatever else it wants) and another is
running the actual process. If there is a failure, the backup picks
up at a restart point. In COBOL, you code a CHECKPOINT statement to
identify the restart points. On Stratus takeover is automatic (no
programmer action required), but you have two processors locked up
instead of one.

Actually, there are two Tandem lines, the "biggies" and "smallies".
The biggies work as above. The smallies run under UNIX and work
somewhat like Stratus.
