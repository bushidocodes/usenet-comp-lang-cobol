John:


I get the very same message you indicated when I submit a compile. I
tried some time ago to figure out what caused it, but could not. I came
to the conclusion it was a "phony error message", something not being
handle right during the PROC interpretation.

However, this message should not interfere with your compile/link. This
appears to be an "informational only" message. If it were a problem,
you should get a JCL error. If you are getting through the
compile/link, you should be ok. Just ignore the damn message. Any
non-zero return codes in your run should only indicate problems with the
compile (errors) or the link.

Mike Dodas
