Folks,

  Recently we purchased MF COBOL for Linux to port our product to linux. We
  already use MF COBOL on AIX, HP-UX and Unixware.

  Here is my problem.:

  I could compile and run the COBOL code, but if I create a executable
  that is combination of C and COBOL I get following error:

  error in loading shared libraries: liboops.so: cannot open shared object
  file: No such file or directory.


  Our application is complex, it is a combination of COBOL and C programs,
  it is also exec'd from a SUID C program. At first I thought that the
  LD_LIBRARY_PATH was not being exported to the SUID program, so I set the
  environment variable in the SUID program, but I got same result. To get
  more information it exec'd strace from a SUID program. 
  
  Following is the C code for the SUID program:

   putenv("LD_LIBRARY_PATH=/lib:/usr/lib:/opt/MfCobol_V4.1_8d.H3.13.03/coblib");
   path = getenv("LD_LIBRARY_PATH");

    * argv[1] = NULL;
    * execv("./cob/run/bin/region", argv);

   execv("/usr/bin/strace", argv);
   printf("region not started, buildreg probably not done\n");
   exit(1);

  I figured out the problem, it has nothing to do with libstd++. The dynamic
  link loader (ld.so), for security reasons, will ignore LD_LIBRARY_PATH if
  if the program is USID root. 
  
  To solve the problem I had to edit /etc/ld.co.conf file and add the path for
  ${COBDIR}/coblib, then run ldconfig command to build the cache
  (ld.so.cache).

  The only problem I can see right now is that I cannot run multiple version
  of COBOL compiler at same time (i.e. test newer version along with old
  version).
