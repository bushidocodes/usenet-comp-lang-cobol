Whoa back!!!
Sure, it's easy to get to control blocks - just redefine a PIC S9(9) COMP
VALUE +16 as a POINTER to get to the CVT and then chain forward from there
to wherever you want to go. Look at the OS/390 MVS Data Areas manual to find
the links if you really want to do this.
BUT, you are almost certainly putting in code that no COBOL programmer is
going to understand, and that will have to be changed next time the SYSID
changes.
It's much more flexible to use a JCL PARM= string to say 'PROD' or 'TEST' if
that's all you really want to know.
Now, if you are doing something like retrieving the JOBNAME, or JOBID, or
executing program name, which is specific to the current instance, then
control block chaining is the way to go.  Just don't do something because
it's slick i.e. arcane.
D de J
