Hi,

I tried to port a NetExpress developed CGI program to Unix.
I did this already years ago on some SCO boxes with success. 
But now with Linux the program crashes constantly with 

Execution error : file 'ADIS'
error code: 192, pc=0, call=5, seg=0
192     Required terminal capability description 

I had the same problem with the SCO box. 
The solution then was to add the environment variable 
TERMINFO=/usr/lib/terminfo
to the shellscript, that started the Cobol program.

The Apache server sets the TERM Variable to 'dumb'. 
There is an entry for 'dumb' in /usr/lib/terminfo. 
I tried it with different settings of TERM. 
Also setting COBTERMINFO gave no result.

I tried it for several hours today - I'm slowly going nuts
Did i miss some RTFM?

Some facts:

Apache 1.3.20
SuSE 7.3
Micro Focus OCDS 4.1.30

TIA,

Pepe
