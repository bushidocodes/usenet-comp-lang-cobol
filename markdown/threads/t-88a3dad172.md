Not using text mode with N/E, I wasn't aware until Sven's reply that
performance would be poor. Which begs the question; why did you upgrade
to NetExpress if you still wanted to use Screen Section ?

You can follow Sven's suggestion and use NT if that will do it for you.
Alternatively, carry on with your application in current compiler and
then make a decision to use NetExpress with Flexus SP2 or Norcon
windowing packages geared to COBOL, Dialog System or OO/GUIs - gradually
learning the chosen screen software.

At some stage you have to get into windowing to keep abreast.
