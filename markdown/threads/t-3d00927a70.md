```
I am having difficulties reading a variable block file using COBOL II.
The file is a tape file produced off of a telephone switch system. The
file contains approx. 20 different type and lengths of records. There
may be a few (12 btyes) at the beginning of each record with the same
definition BUT the "variable" portion is not determined by a segment
counter field and is not made up or just multiple segments. The
"variable" portion is composed of various unique combinations of fields
(of course with different descriptions and lengths). I get an error
simply trying to open this file.

Specifications: IBM/MVS environment with COBOL II.
file defined as VB, blocksize = 2048, lrecl = 2044
physical records range 27 - 129 bytes (containing various
configurations of packed decimal data.

Note: I am attempting to replace an ancient Work10 program that
successfully reads this data file and, trivia of the day, Work10 actually
creates COBOL code (assumably ANSI). The Work10 program SELECTS the file
as ACCESS IS SEQUENTIAL. The FD is RECORDING MODE IS V, LABEL RECORDS IS
STANDARD, BLOCK CONTAINS 2048 CHARACTERS, DATA RECORDS ARE (and then a
list of record layout definitions ranging from 43 bytes to 85.

Any help from a COBOL guru (and you qualify if you can help n this one)
would be appreciated.
```

```
Assuming that you are using the NOCMPR2 compiler option in you VS
COBOL II (or COBOL/370 or COBOL for MVS & VM) program, then you
can easily solve this by coding

FD file-name
Record varying in size
from "smallest-lrecl" to "largest-lrecl"
depending on Working-Storage-Variable-name
...

(Make certain that you do *not* include the "llzz" field in the
lrecl values you supply in the FD.)

Then each time you read a record, the
Working-Storage-Variable-Name field (defined in your WS) will
include the correct record length of the record you just read.
And I assume you can tell which record-type you have read from
the length.
```
