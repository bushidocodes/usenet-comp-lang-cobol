```
I fully accept that MANY programmers consider your approach to be
"structured"  - HOWEVER,  you are TECHNICALLY incorrect in thinking that
there is "one entry - one exit" for each program.

The routine
  M10-GET-NEXT

actually has TWO possible exits.

If the read is successful (no AT END) then the "exit" point is the physical
end of the source code in that paragraph - and there is a "fall thru" to the
next paragraph (the EXIT paragraph).

If the AT END condition does occur, then the physical end of that paragraph
is SKIPPED and there is an explicit transfer of control to the EXIT
paragraph.

Having said that, it is true that the "logical unit of work"
   M10-LOAD-TABLE THRU M10-X

does have a "single entry and exit" - But that would ALSO be true of the
following code (that MOST people would consider NOT to be structured)

Perform Para1 thru Para5

Para1.
    Display "here"
    If A = "2"
        go to Para2
    Else
         Go to Para3
     End-If
      .
Para2.
   Display "Here 2"
   Go To Para4
     .
Para3.
   Display "Here 3"
    Go To Para5
      .
Para4.
   Display "Here 4"
      .
Para5.
   Display "everything exits from here"
      .
```

M10-GET-NEXT through the paragraph-name, M10-END
constitutes a "Mid-Loop Exit" structure. The content of this
structure is reducible to a process block with a single entry
and a single exit. I discussed the "Mid-Loop Exit" structure,
last year.

The draft COBOL standard provides PERFORM ...
EXIT PERFORM ... END-PERFORM which may be used
to eliminate the paragraph-names that form the loop and
structure boundaries. However, the draft COBOL standard
has no syntax to identify the loop as indefinite, therefore
a condition such as UNTIL 1 = 0 is needed. I also mentioned
this, last year, and suggested PERFORM WITH NO TEST
as a means for identifying an indefinite loop.

While I discussed these issues in the newsgroup, I never
submitted a comment to J4 for their consideration.

       PERFORM M10-LOAD-TABLE THRU M10-X

 M10-X.
       EXIT
       .

 M10-GET-NEXT.
       READ INPUT-FILE
       EVALUATE INPUT-FILE-STATUS
           WHEN '00' 
               SET TBL-NDX            UP BY 1
               MOVE INPUT-DATA TO TBL-OCCURRENCE (TBL-NDX)
               ADD 1                          TO TBL-SIZE
           WHEN '10'

           WHEN OTHER

 M10-END.

        .

 M10-LOAD-TABLE.
       (do some initializing stuff)
       PERFORM M10-GET-NEXT
           VARYING TBL-NDX
              FROM 1
                BY 1

 M10-X.
       EXIT
