You can use OCCURS DEPENDING and RECORD VARYING integer1 TO integer2.

Using sequential recording mode, you'd have to have some sort of a master
file update scheme (MasterIN, transaction file, MasterOUT), so a variable
length indexed sequential (ISAM) file design would probable be in order for
your purpose.

You'll have to give some thought to limiting the record length to some
finite value, such as one year's worth of error counts per error record.

This is very basic COBOL, taking in data in one format and outputting it
in another format - I am trying to think if I know of a COBOL program
which doesn't do this.

I guess your question has to do with the fact that you combine many
input files to create an output one.

The hardest part if deciding how many dates you have going across your
report.  The following code fails because the heading is wrong (You
didn't think I would do all your homework for you did you!?!?).   You
will have to decide how you are going to make your headings and handle
too many dates for a page.  You may need to do an internal sort or two
COBOL programs to determine which dates are to be handled or make some
type of business decision to do the same thing.

When I do this, I first make sure the input file is sorted by error-code
and date.

Then I loop:
    (
    READ INPUT-FILE AT END SET SW-EOF TO TRUE.
    PERFORM READ-LOOP UNTIL SW-EOF.
    IF DATE-COUNT (1) > ZERO
         PERFORM WRITE-OUTPUT
    END-IF
    ... 

READ-LOOP.
    IF ERROR-CODE = HOLD-ERROR-CODE
        CONTINUE
    ELSE
        PERFORM WRITE-OUTPUT
    END-IF    

    IF INPUT-DATE = OUTPUT-DATE (DATE-INDEX)
        CONTINUE
    ELSE 
       ADD 1        TO DATE-INDEX-MAX
       IF DATE-INDEX > DATE-INDEX-MAX
           PERFORM OVERFLOW-ROUTINE
       END-IF
       MOVE INPUT-DATE TO OUTPUT-DATE (DATE-INDEX)
    END-IF.
    ADD 1            TO DATE-COUNT (DATE-INDEX).
    READ INPUT-FILE AT END SET SW-EOF TO TRUE.

WRITE-OUTPUT.
    MOVE HOLD-ERROR  TO OUTPUT-ERROR
* May make the following a perform  
    MOVE DATE-COUNT (1)  TO OUTPUT-DATE-COUNT-1
    MOVE DATE-COUNT (2)  TO OUTPUT-DATE-COUNT-2
    MOVE DATE-COUNT (3)  TO OUTPUT-DATE-COUNT-3
    IF LINE-COUNT > MAX-LINES
        PERFORM WRITE-HEADINGS
    END-IF
    WRITE OUTPUT-LINE AFTER ADVANCING 1
    ADD 1   TO LINE-COUNT
    INITIALIZE COUNT-ARRAY.
