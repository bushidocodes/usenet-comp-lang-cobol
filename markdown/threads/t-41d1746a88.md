The control provided by Fujitsu is intended to simplify the interface to
Excel. It does this quite well. However, Excel 2002 and Excel 2003 both saw
some fairly major changes to the Excel Object Model. The worksheets are now
a collection under the Application object. There is no such Method as
OpenBook in the model so Fujitsu are "translating" this into the correct
level of Open for the collection.

The syntax and structure of Properties is also subject to change.

The only way to guarantee your interface to Excel from PowerCOBOL (or
anything else, really) is to use the Excel Object Model. It is easy
enough... Create a COM object using 'Application.Excel' and then apply the
objects, methods, and properties documented on MSDN. I did a quick search on
"Excel Object Model" and found a number of useful papers giving examples
(admittedly in VB, but pretty easy to translate).

If you don't want to do this, your other option is to ask Fujitsu for an
updated control that supports the latest Excel. (Good Luck!)
