As Bill pointed out, short answer is no.  But in OC's defence.
OpenCOBOL is open.  There is nothing hidden. Source codes are
available.  It might not always be easy, but nothing in OpenCOBOL is
"impossible" to figure out.  For some major hints on the capabilities,
visit the parser.y source in cobc/ and codegen.c in the same cobc
subdirectory of the source tar ball.

Anyone that wants help with OpenCOBOL, be sure to visit opencobol.org,
add1tocobol.com and the #add1tocobol Freenode IRC channel.  It usually
doesn't take long to get working solutions.

For DISPLAY support, this line

| x_list opt_at_line_column display_upon with_clause on_disp_exception

in the parser and then

opt_at_line_column:
  /* empty */			{ $$ = NULL; }
| _at line_number column_number { $$ = cb_build_pair ($2, $3); }
| _at column_number line_number { $$ = cb_build_pair ($3, $2); }
| _at line_number		{ $$ = cb_build_pair ($2, NULL); }
| _at column_number		{ $$ = cb_build_pair (NULL, $2); }
| AT simple_value		{ $$ = $2; }
;

line_number:
  LINE _number id_or_lit	{ $$ = $3; }
;

column_number:
  COLUMN _number id_or_lit	{ $$ = $3; }
| POSITION _number id_or_lit	{ $$ = $3; }
;

Leads to  DISPLAY variable LINE n POSITION n END-DISPLAY   being a
supported syntax.

This program worked for me here on a Debian GNU/Linux system, with a
OpenCOBOL 1.1 package date of Feb 2009.

      *>
***************************************************************
      *> Purpose:   Try positioned display
      *> Tectonics: cobc -x dispat.cob
      *>
***************************************************************
       identification division.
       program-id. dispat.

       data division.
       working-storage section.
       01 variable pic x(10) value 'abc'.
       01 x        pic 99 value 12.
       01 y        pic 99 value 12.
       01 waiter   pic xx.

      *>
***************************************************************
       procedure division.
       display variable line x position y end-display
       accept waiter at 1010 end-accept

       goback.
       end program dispat.

Oh ... why the waiter?  If not, the screen will flash in and out of
ncurses virtual screen mode too fast to even notice it happen, plus it
shows off some of the alternate syntax available for positioned screen
IO.   ;)
