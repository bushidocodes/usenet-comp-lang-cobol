I have a problem using some functions in a custom API. I have custom functions
that return "Numeric" values and those I have working fine. The problem I am
having is when I need to return a return code and a character string from the
API function.
Example:
        CALL "apiCharReturn" USING BY REFERENCE BIGCHAR,
                                   BY REFERENCE SMALLINT,
                                   BY VALUE 0 SIZE 4
                             RETURNING RETURNCODE (I want BIGCHAR too!)
        END-CALL
Now apiCharReturn does send RETURNCODE but I want RETURNCODE AND the returned
value of BIGCHAR. Is this possible? If not both, I would want the BIGCHAR
instead of the RETURNCODE (more valuable to me).

I am a VB programmer thrust in the world of MF Cobol so be gentle.
Thanks in advance for any help you can provide a novice Cobol programmer.

Please respond via email.

Peter,

As the BIGCHAR and SMALLINT parameters in your example are passed BY
REFERENCE, the addresses of those parameters are passed to the
apiCharReturn function. So, if the function modifies the contents of those
addresses what is actually happening is that the parameters in the calling
program are being modified, which is what you want.

Peter,

I have been doing alot of mixed-language programming lately using Micro
Focus Workbench 4.0 (and Netexpress 3.0) with Visual Basic 5.0.  Here is
some sample code that I got to work - I think it will do what you require.
You are correct in using the "by reference" clause.  I have found with MF
Cobol that you only need a "returning" clause when you have defined the VB
declare as "Function" instead of "Sub".  I usually just define everything as
"Sub" and use the "by reference" - different strokes for different folks...

VB 5.0 Code - saved in .BAS module:

'***************************************************************************
*
' PRNTMAN Subprocedure Definitions:
'***************************************************************************
*
Public Declare Sub PRNTMAN Lib "PRNTMAN.DLL" (ByRef DocPath As Any, ByRef
DocTitle As Any, ByRef ReturnCode As Long, ByRef ReturnMsg1 As Any)

'***************************************************************************
*
' PRNTMAN Subprocedure Parameter Allocations:
'***************************************************************************
*
Private DocPath                     As Cobol_String_80_Struct
Private DocTitle                    As Cobol_String_80_Struct
Private ReturnCode                  As Long
Private ReturnMsg1                  As Cobol_String_80_Struct

Private Type Cobol_String_80_Struct
    Cobol_String_80                 As String * 80
End Type

'***************************************************************************
*
' PRNTMAN_Print_File Subprocedure:
'***************************************************************************
*
Public Sub PRNTMAN_Print_File(ParmDocPath As String, ParmDocTitle As String)

' Ensure that received parms are in fixed-length string field:
    LSet DocPath.Cobol_String_80 = ParmDocPath
    LSet DocTitle.Cobol_String_80 = ParmDocTitle

' Call the Cobol routine to retrieve and print the specified Document:
    Call PRNTMAN(DocPath, DocTitle, ReturnCode, ReturnMsg1)

' If call to program failed, then display the results to the user:
    If Not ReturnCode = 0 Then
        msg = "Could not Print the specified Document..." + Chr(13) +
Chr(13) + ReturnMsg1.Cobol_String_80
        Style = vbOKOnly + vbExclamation + vbDefaultButton1
        Response = MsgBox(msg, Style, "Airgroup - Print File")
    End If
End Sub




VB 5.0 Code - You could use this to call the function in the .BAS module:
    Call PRNTMAN_Print_File("c:\autoexec.bat", "This is a report title")
