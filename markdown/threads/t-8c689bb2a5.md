Recently, my company has purchased a Unisys Clearpath system. The system
is a Unisys A6 mainframe and a Window NT Server. I now have the ability to
read files on the A6 from a PC network and vice versa. However, I'm having
trouble programmatically (COBOL85) writing files from the A6 to the PC side

to know how either).

If anyone has any experience working with a system like this, or in
writing ASCII files (mostly comma delimited variable length records) from

techniques.

I don't know your system, but the two things that sound like they MIGHT be
useful are to:

1) Use the CODE-SET clause of the FD for the file your are writing (and set
it to a Standard-1 character set - which should be ASCII)\

2) Look at creating your file with the RECORD VARYING IN SIZE clause and
*not* a RECORD CONTAINS clause - to make certain it is variable.

These are both "generic" ANSI'85 features so they should be good for your
system - even though I don't know it personally.
