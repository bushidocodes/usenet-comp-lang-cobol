Switch P2 is Logical Cancel. By default (16-bit shared runtime),
CANCEL releases program from memory. Specfying +P2
prevents releasing the program from memory when CANCELed.

Switch M2 is Active Program Discard. By default (16-bit shared
runtime), the RTE will discard procedure code of active programs
and reload when needed. This may cause "out of memory" if there
is no room to reload. Specifying +M2 does not discard procedure
code and may cause "memory full" if the CALLed program cannot
be loaded.

    Bottom line answer to original question - If you do not cancel, than the
memory is NEVER freed, working storage, etc anyway.  The procedure division
might be discarded (a la windows 1.0), but the rest of the meory is NEVER
released.

    The +M0  (+M) switch is supoosed to control overwriting command.com.  I
have no idea if it still preserves the memory image of command.com with the
newest versions of dos.  That switch can be used at all times unless you use
XM, in which case who cares about 30K or so of memory.

    Of course the original problem was in a 32 bit environment.  I believe
that when switch p2 is unset (-P2), that it marks the program as inactive,
but does not free memory.  If you call it again soon enough, it will run in
an initial state, but will not access the file system.  If you run out of
memory, it would do a garbage collection of sorts and discard all canceled
programs from memory.
