I have found a solution to this problem, so just posting it here for
anyone else who may come across this problem.

In order to save time, the Micro Focus file handler does only logical,
not physical, file closes until the program ends.  You can change the
behavior with the runtime tunable "multi_close_limit".  A value of "1"
will do the trick.  Documentation reference: "User's Guide", Chapter
14 "Run-time Configuration".

This completely sorted out the problem I was having. Once set, I
didn't get multiple File Handles being opened for each OPEN of the
file.

Works like a dream.

Hope this helps someone else!
