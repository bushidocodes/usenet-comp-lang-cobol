```
Hi.

I have a file with 80 character records.

I've tried to set up something to read these records
that looks like this:

File Control.
select optional input-file
assign to "input.dat"

File Section.
fd input-file
label records are standard.
01 input-record.
05 field1 pic x(20).
05 field2 pic x(30).
05 field3 pic x(30).

When I try to use this in a program to read lines from the input file
I get some really bizarre results. I assume it's because I don't
understand Cobol file i/o very well. Is it okay that my input file is
hand-made with a vi editor? I have a good Cobol text that doesn't seem
to cover this topic very well. Anybody got any good file input examples
they can share?

TIA.
Baffled.
Martin Meadows
```

```

You don't say what compiler you are using but my guess is that it is a UNIX
or PC one that supports "line sequential" formats (with CR/LF between
records).

Try adding

ORGANIZATION IS LINE SEQUENTIAL

in your SELECT/ASSIGN statement.
```
