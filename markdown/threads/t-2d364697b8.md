I take it you are experimenting, not writing a 'live' program ? Couple of points
worth noting :-

Your record layout indicates a Vendor/Customer Name and Address record,  i.e. it
is not a transaction file, but rather a Masterfile - traditionally handled as an
ISAM not Sequential or Line Sequential. Imagine if American Express used your
technique the zillions of records they would have to read before they got to
Mols, Swaans, van den Ackker or van Heusen. (You have got your file stored on
disk but think of a sequential in terms of reel(s) of magnetic tape).

Firstly use an ISAM file with a Customer Number PrimeKey, then use snaam (Last
Name or Family Name ?) as an Alternate Key WITH DUPLICATES. Pseudo code :-

        Open I-O ISAM File
        set Program NotFinis to true
        perform until ProgramFinis
            display Empty Screen Record
            accept snaam

            if   snaam <> spaces
                read ISAM File key is snaam
                       invalid key - display "not in file message"
                       not invalid key perform display CustomerRecord
                 End-read

            else set Program Finis to true
                    (OR if you have the feature - EXIT  PERFORM)
            end-if
    end-perform
    close ISAM File
    Stop Run

Note the above lets you do more than one query and the program terminates when
you enter an snaam of spaces.

PS: If you have a compiler with Screen Section - you could make it easier than
your two paragraphs 2-gevonden and 2-toon.

PPS: Curious which compiler you are using, Micro Focus Net Express didn't like
the following from your code :-

Full,

First an answer to your question - Yes do a PERFORM. Try and avoid doing the
same thing in different places, particularly EXITs or STOP RUNs - for clarity
make programs PERFORM those routines. I'm glad to see you have included
copyfiles - that makes it so much easier for you to adapt to other programs.
Also you have under 'check' Paragraph, isolated your file handling routine -
that's a good way to go.

Now I have your complete program and have compiled it - I see it works, and
does it just fine. BUT - you are only doing one enquiry. Assume you had some
three or four teams of soccer players and there are some 60 people involved -
that means 60 runs of the program to search for each individual.

Note also that if I enter a blank zoeknaam - you still search all the records
in the file.
You can overcome that one by :-

        accept zoeknaam
        if zoeknaam = spaces
           EXIT PROGRAM

To have more than one enquiry against your sequential file you would have to do
the
following :-

    accept zoeknaam

    if zoeknaam <> spaces
       perform SEARCH-ROUTINE

   else EXIT PROGRAM
   end-if.

SEARCH-ROUTINE.

    open input spelerbest
    do search
    close spelerbest

That way each "Open" gets you to the beginning of the file, which was one of
your queries. But it isn't a particularly successful solution - and in the real
world, you would be put against a wall and shot <G>.

Seeing that you mentioned this was part of a project to edit, query and print -
then the only satisfactory solution is an ISAM file. I'm not going to take up
bandwidth here on how to do that, but if you would like more information, write
to me privately. BUT first check with your COBOL teacher so that he/she doesn't
get into a snit <G>.

I think your teacher has done a good job putting the coding concepts across and
you seem to have a good handle on what you have learned so far - plus I think
your written English is EXCELLENT !

A common way of reading a file is as follows:


Read first record.
Perform read-loop until EOF.
...


Read-loop.
    Test data.
     process data.
     Read next record.
>>>>>>>>>>>>>>>>>>>>>>>>>>><<<<<<<<<<<<<<<<<<<<<<<<<<<<<

There obviously has to be some AT-END  or status code checking to set EOF. 
(if you have multiple files, use multiple status codes or EOF switches. 
Never use the same one for multiple purposes if you want to keep maintenance
programmers from messing up in the future.

Some people say having two reads isn't "structured", but changing to two
performs is not a real gain.

One advantage to this style is that there are no real switches to be set and
checked from within code.  EOF is set only immediately before it is checked.
