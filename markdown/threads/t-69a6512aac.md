You didn't understand what I was talking about. I was talking about the
persistence of the data, not the program. The way the proposal works is
quite simple and obvious and is what almost all other languages do. The
suggested data handling required keeping track of various iterations and
pages of rules on how it would work. Almost nobody would understand how
to use it.

I won't go into all of the details of my proposal. If anyone wants to
see it, I can mail them a copy (regular mail). Basically, there are two
kinds of data, automatic and static. Static stays in its last-used
state (just like when you call a program that is not initial the second
time) and automatic is initialized on every call (just like initial
programs do today). For initial programs, all data is automatic unless
you say it is static or external. For non-initial programs all data is
static unless you say it is automatic.
