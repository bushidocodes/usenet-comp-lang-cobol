Here, since the consenus seems to be this is not rocket science (and it
aint).  This is some code from a FoxPro program that you ought to be
able to figure out.  As mentioned before, this is only for data entry
validation, it doesn't get you anywhere at all as far as authorizations,
settlements, etc.  But, simply because I think you ought to be working
with your provider, processor, or whomever, I did change the weights
from their original values.  That'll be easy enough for you to correct
when you get with whomever you *should* be working with.


FUNCTION mod10check
    PARAMETERS cardno
    PRIVATE weight,sum,temp,x,ret_val
    m.cardno = ALLTRIM( m.cardno )
    IF LEN( m.cardno ) % 2 = 0 
        ** even number of digits
        m.weight = "4321432143214321432143"
    ELSE
        ** odd number of digits
        m.weight = "1243124312431243124312"
    ENDIF
    m.sum = 0
    FOR m.x = 1 TO LEN( m.cardno )
        m.temp = VAL( SUBSTR( m.cardno, m.x, 1 ) ) * VAL( SUBSTR(
m.weight, m.x, 1 ) )
        m.sum = m.sum + INT( m.temp / 10 ) + ( m.temp % 10 )
    NEXT
    IF m.sum % 10 = 0
        RETU( .T. )
    ELSE
        RETU( .F. )
    ENDIF
RETURN( .F. )
