Yes, but the methods are totally compiler/OS dependent. COBOL (currently)
does not include 'native' support for dynamic memory allocation and this is
handled though O/S services on the host computer.

> another way to validate partially full data fields. So far I have tried
> the INSPECT and ALPHABETIC classes and I have run out of ideas, as these
> don't work on partially filled data fields.

When you say, "partially filled", you mean something like a left-justified
alpha field with trailing spaces?

If so, you can use reference modification...

  MOVE zero  to SPACE-POSITION
  INSPECT The-data tallying space-position  for characters before initial
space
  IF space-position > zero
    IF the-data (1: space-position) IS NOT ALPHABETIC.....


You'll have to play around, but this should get you started. The problem
will be dealing with data with leading as well as trailing spaces.

MCM
