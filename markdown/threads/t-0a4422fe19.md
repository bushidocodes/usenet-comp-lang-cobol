```
  We have a Netexpress version2 application that needs to interface with a
VB6 application.  It shifts control and focus to the vb app successfully. 
The problem occurs when we want to transfer data back to COBOL, and reset
focus to the COBOL app.  The COBOL app regains control, but the focus is
never returned to the COBOL app.  (The focus remains on the VB window).

  The COBOL app is a console app (non-gui).  We are using a win32 api
"setfocus" within COBOL to attempt to make the COBOL app regain focus.	The
return code is zeros from the API call.  We know the COBOL code continues
execution, as the console screen is updated.

```
The problem is that the SetFocus API call requires a window handle and you
do not have a window handle when running as a console app. If you relink the
app as graphical it will display the text in a proper window and you can
then use the PC_WIN_HANDLE call to get the window handle to use for the
SetFocus.

I hope this helps.
