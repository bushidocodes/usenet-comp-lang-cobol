Your original equation read:

(log to base b+c of a) + (log to base c-b of a) =
   2 * (log to base b+c of a) * (log to base b-c of a)

Are you saying that "log to base c-b of a" should have read
"log to base b-c of a"?

If not, then something's wrong, because b-c and c-b are not both going 
to be positive, and at best, you'll have an imaginary term on one side 
of the equation.

Let's assume that should have been "log to base c-b of a" and work 
backwards from:

(log to base b+c of a) + (log to base b-c of a) =
   2 * (log to base b+c of a) * (log to base b-c of a)

I looked up log base conversion (I used to know this stuff), and log to 
base x of y is ln(y)/ln(x) where ln is the natural log (base e).  Common 
logs (base 10) would work just as well.  Converting,

ln(a)/ln(b+c) + ln(a)/ln(b-c) = 2 * ln(a)/ln(b+c) * ln(a)/ln(b-c)

Dividing by ln(a) (which assumes a isn't 1), we get:

1/ln(b+c) + 1/ln(b-c) = 2 * ln(a)/(ln(b+c)*ln(b-c))

Multiplying by ln(b+c)*ln(b-c) (assuming b and c are unequal),

ln(b-c) + ln(b+c) = 2 * ln(a)

Taking the exponent of both sides,

exp(ln(b-c) + ln(b+c)) = exp(2 * ln(a))

exp(ln(b-c)) * exp(ln(b+c)) = a**2

(b-c) * (b+c) = a**2

b**2 - c**2 = a**2

a**2 + b**2 = c**2

I was skeptical, but there it is.  I learn something every day.

Louis

  I had noticed that the form of the problem was (x+y) = (2*x*y). So 
(x+y)/(x*y) = 2 which
  gives 1/x + 1/y = 2. This led me to the log rule (see rule 1 below) that

  needed to get my equations going in the right direction.

  I posted my question to the alt.math group.  They did not give me an 
answer but they
  helped me see some flaws in the problem statement.  I think we must assume 
that b+c and
  b-c are positive because logs to negative bases would be a problem.  Also 
we need to
  assume that c-b not= 1.  For example if a=3, b=4, and c=5 then c-b=1 and 
log_(c-b) a
  would become log_1 3 and no power of 1 can equal 3.  I am not a 
mathematician and this
  stuff escaped my notice.

  Problem statement: a^2 + b^2 = c^2
                     Show log_(b+c) a + log_(c-b) a = 2 * log_(b+c) a * 
log_(c-b) a

  From a^2 + b^2 = c^2 we get a^2 = c^2 - b^2 = (c-b)(c+b) which is used 
below (by assumption)

  Also used are:  rule 1: log_b a = 1 / log_a b   for a not= 1, b not= 1

                  rule 2: log_b (x*y) = log_b x + log_b y

                  rule 3: log_b x^n = n log_b x

  Proposed solution:

  log_(b+c) a + log_(c-b) a = 1/log_a b+c + 1/log_a c-b    by rule 1

               = (log_a c-b + log_a b+c) / [(log_a b+c) * (log_a c-b)]   by 
x+y/xy = 1/x + 1/y

               = log_a (c-b)(b+c) / [(log_a b+c) * (log_a c-b)]  by rule 2

               = log_a a^2 / [(log_a b+c) * (log_a c-b)]  by assumption

               = 2 / [(log_a b+c) * (log_a c-b)]   by rule 3 and log_a a = 1

               = 2 * log_(b+c) a  * log_(c-b) a    by rule 1

Here is my summary:

log is the log to any convenient base, 10 or natural, it does not
matter

let u = log a / log (c+b) = log_(c+b) a

let v = log a / log (c-b) = log_(c-b) a

These are true by the change of base rule.

The desired equation is 2*u*v = u + v.

                     == 2 = 1/u + 1/v

Start with a^2 + b^2 = c^2
                 a^2 = c^2 - b^2
factor           a^2 = (c+b)*(c-b)
take logs   2*log a = log(c+b) + log(c-b)

divide by log a

                2 = log(c+b)    log(c-b)
                    -------- +  --------
                     log a      log a

                2 =  1/u  + 1/v

 Q.E.D.
