Richard is right with respect to EXTERNAL, but didn't explain GLOBAL.

GLOBAL means that the dataname can be used without needing to be
passed as a linkage parameter by any nested programs directly or
indirectly contained within the program that declares that dataname. 
Nested programs are those that are contained within other programs and
not those that are separately compiled and connected with the calling
program at link edit time.
