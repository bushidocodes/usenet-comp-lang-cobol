```
Does anyone have an explanation for the following anomaly ?

I have a variable length field defined in working storage :-

01 WS-TALLY PIC 99 VALUE 0.

01 WS-PARMS.
03 WS-EXCEPTION PIC X
OCCURS 0 TO 80 DEPENDING WS-TALLY INDEXED INDX1.

At program start I display the compile date & time, then after
populating WS-PARMS I display its contents to the journal thus :-

DISPLAY ' '
DISPLAY 'EXCEPTION PARMS = < ' WS-PARMS ' >'

Which produces :-

MODULE RA710001 VERSION IS 2.0, LAST COMPILED 08/20/98 18.24.33

EXCEPTION PARMS = < B,7,8,9 >

However, if there is nothing in WS-PARMS & WS-TALLY = 0 the following
appears :-

MODULE RA710001 VERSION IS 2.0, LAST COMPILED 08/21/98 10.19.58

EXCEPTION PARMS = < >N IS 2.0,

To see what would happen I amended my display message and recompiled the
program thus :-

DISPLAY ' '
DISPLAY 'EXCEPTION FILE PARAMETERS ARE : << ' WS-PARMS ' >>'

This produced the following :-

MODULE RA710001 VERSION IS 2.0, LAST COMPILED 08/21/98 10.27.35

EXCEPTION FILE PARAMETERS ARE : << >>OMPILED 0

Can anyone explain why 9 characters of the previous message appear after
my "close chevron". I should be grateful to know.
```

```
Can't say exactly what is causing this since you didn't say what
platform/Cobol you are running. However, in general, your results
will be unpredictible because the area addressed by WS-EXCEPTION was
never initialized or loaded with anything. Depending on link options
this area could contain low values (hex '00' -- see previous
threads/arguments about what low values may or may not be!) or
whatever happens to be in memory at the time. My best guess is the
length of the data is 1 byte and the buffer area used by the DISPLAY
command did not get cleared.

Maybe if you supplied more info, someone can give a better
explanation.
```
