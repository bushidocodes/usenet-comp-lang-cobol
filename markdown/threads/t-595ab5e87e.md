Lets clarify what is supported and what isn't. The language is COBOL
and it certainly is supported and probably always will be. Your
programs were written in this and will be held somewhere as source
files. They contain no references to networks, but may use record or
file locking.

These source files need to be converted to (or interpreted as) machine
code and operating system calls so they run on a particular machine and
operating system. Seems like they were converted to a machine and
operating system environment that was current in 1990. Since then you
have tried running it in a different, later environment and it doesn't
work; your new environment doesn't imitate your old one.

You simply need a new COBOL system (one that is suitable for your new
environment - ie Lantastic) to reconvert your application source files
to machine code. Alternatively, if your source code is available as
generic interpreted code, then all you need is a new COBOL interpreter.
Chances are that by using the new COBOL system, your application will
work in both the old and the new environments.

mains voltage. Problem is, you have a US mains lead when you visit

Lantastic the same way as Netware). Don't chuck out the shaver, just
get a more appropriate mains lead (or change Hotel).
