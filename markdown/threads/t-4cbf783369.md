### New Y2K Functionality with SyncSort/MVS

- **From:** "brian r. wood" <ua-author-17073253@usenetarchives.gap>
- **Date:** 1997-03-30T19:00:01+00:00
- **Newsgroups:** comp.lang.cobol
- **Message-ID:** `<3340360A.6A242A71@syncsort.com>`

```

There have been some recent inquiries made concerning how the
SyncSort/MVS date data formats (Y2C, Y2Z etc.) and CENTWIN parameter can
be used in connection with programs - notably COBOL - that have multiple
internal sorts.

When making sort related changes for the Year 2000, there are times when
it is desirable to override the existing sort control statements instead
of replacing them. This is particularly true when working with program
invoked sorts. The existing SORT, INREC or OUTREC control statements may
be overriden with by placing the new control statement in a $ORTPARM. As
long as there is only one sort in the step, this method works correctly.

When there is more than one sort in a step, the problem of providing
different control statements to the different sorts arises. This can be
overcome by coding multiple $ORTPARMs with FREE=CLOSE. The first
$ORTPARM in the JCL would contain the overrides for the first sort in
the step, the second $ORTPARM would contain the overrides for the second
sort. There would be one $ORTPARM for each sort and the overrides for
the nth sort would be in the nth $ORTPARM. Each $ORTPARM must have the
FREE=CLOSE parm coded on the DD statement.

The only restriction is that $ORTPARM be allocated in a data set that
can be FREEd when CLOSEd. This means that it must be allocated to a real
data set. It cannot be a DD * allocation. The following is an example:

//INVSORT EXEC PGM=INVOKER
//$ORTPARM DD DSN=your.cntl(ORTPARM1),DISP=SHR,FREE=CLOSE
//$ORTPARM DD DSN=your.cntl(ORTPARM2),DISP=SHR,FREE=CLOSE
//$ORTPARM DD DSN=your.cntl(ORTPARM3),DISP=SHR,FREE=CLOSE
//$ORTPARM DD DSN=your.cntl(ORTPARM4),DISP=SHR,FREE=CLOSE
//....rest of your DD statements
```
