```
A. Last question first. A CR is a character. If
the file contains only a CR it will have a length
of 1.

B. One way to discover if a CR exists is to define
the file in another SELECT statement as a
SEQUENTIAL file. In the FD, define this SEQUENTIAL
file's records as something really humoungous.
OPEN the sequential file, issue one READ, CLOSE
the sequential file. Now inspect the SEQUENTIAL
file's record for the CR character (X'0D').

C. Open the LINE SEQUENTIAL file. Blank the record
area. READ a record. If the record area is still
SPACES (and you didn't have a file error), the
record you just read consist of only a CR (or
CR/LF).
```
