Bob,

This may be oversimplifying what you are trying to do, but if you know the
memory address you could set a pointer to it, then assign the address of a
linkage item to the pointer and modify the linkage item directly.

eg
        working-storage section.
        01 ptr pointer.
        linkage section.
        01 mem pic x.
        procedure division.
            set ptr to null
            set ptr up by h"240"
            set address of mem to ptr.

Bob,

I did this at my last job - but I did it differently.  You are going 
to laugh.   What is really cool about the SURE ONE is that you can 
configure the darn ports anyway you like.  We did it two ways.

The best way was to put the PRINTER on the serial port and the cash 
drawer on the COM port.  We then used the MicroFocus Windows printing 
calls to print to the default printer which we attached to COM1 - and 
sent the code to open the cash drawer to a file assigned to: "LPT1".  

The other way was to setup the cash drawer on the serial port and send
the code to a file assigned to "COM3" or whatever.  We just didn't 
make the calls to the memory addresses anymore, and I'm not sure you 
can even do it with the Windows stuff.
