I wander if anyone came across Cobol/B compiler which has had some
publicity in UK and USA and tens of thouthands potential users when

friendly at least for those familiar w/ Borland's IDEs.

as producing commercial systems (IMHO):

It is slow, the ISAM performance is 3 -7 times slower than
MF.

It has no dynamic linking and memory management. This makes
it suitable for toy systems only (IMHO).

It has no support for C or assembler add-ins.

The screen section is only partially implemented and poorly
documented.
