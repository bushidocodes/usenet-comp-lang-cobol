Hello:
Because you are evaluating one variable, you can only say WHEN
for once value of it per WHEN
kinda like the C language switch statement.

So, code it something like this:
Evaluate scrn-cd
when zeroes
* note: this "falls thru" to the when '02'
when '02'
perform section-1
when '04'
perform section-2
when other
perform handle-exceptions
end-evaluate.

Or you can do something like this, use the TRUE option of evaluate:

EValuate TRUE
when scrn-code equal '00' or '01' or '02'
perform section-1
when scrn-code equal '04'
etc...
end-evaluate.

The second option I stumbled on while I first began using
the evaluate statement. It gives you more flexibility,
since a TRUE value can be compared to just about any conditional...

I may as well add yet another solution:

EVALUATE Scrn-Cd
WHEN '0'
WHEN '1'
WHEN '5'
PERFORM Section1
WHEN '4'
PERFORM Section2
WHEN OTHER
PERFORM Section3
END-EVALUATE

While 'OR' cannot be used in a WHEN statement, successive WHENs
are effectively ORed together.

This also illustrates that one has to be carefull to ensure
that empty WHENs are not left accidentically if code is
commented out. Either comment out the WHEN too or put in
a CONTINUE.

There are a number of ways this could be resolved.

1. define an 88 level that defines each of the codes. As in:
01 scrn-cd pic x.
88 scrn-cd-1-2 value 1 thru 2.
88 scrn-cd-3 value 3
.
.
88 scrn-cd-? value ?.

evaluate scrn-cd
when scrn-cd-1-2 do something
when scrn-cd-3 do something-else
when scrn-cd-? do another-thing
when other do the-other-thing
end-evaluate.

or

evaluate scrn-cd
when 1 thru 2 do-something
when 3 do-something-else
when ? do-another-thing
when other do-the-other-thing
end-evaluate.
