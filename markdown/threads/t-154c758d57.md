```
I have a CICS program I need to change to print to an HP 4Plus Laser Printer.
My understanding is that CICS goes through VTAM (DRS,VPS), then to the HP
printer using TCPIP.  I need to pass PCL to the HP to invoke the lower tray.
Using the LaserJet 4si User's reference manual ('Don't have HP 4 Plus
Laserjet users manual) I found a PCL code: "1B266C3448" for lower tray.  Is
this correct for the HP4 Plus printer?

Anyway I was informed that this PCL HEX is in ascii and that I need to convert
it to EBCDIC HEX which is "275093F4C8" because VPS converts EBCDIC to ascii.

I tried this recommendation but the document still printed using the upper
tray source.  Could it be that VTAM(DRS/VPS) is overriding this?  Can
VTAM(DRS/VPS) be changed to invoke the lower tray?  Currently VTAM(DRS/VPS)
has been set up to pass the correct font and page orientation (landscape)
which works.  An attempted was made to set a parm in VTAM(DRS/VPS) to select
the lower paper tray as well but still continues to retrieve from the upper
tray. If VTAM can control it, maybe I can avoid using PCL code altogether.

In my cics program I used the following CICS command.  Is this the right
command?

WS-TRAY                  PIC X(5)  VALUE  X'275093F4C8'.
WS-TRAY-LENGTH    PIC S9(04)  COMP VALUE +5

EXEC CICS SEND TEXT NOEDIT
     FROM (WS-TRAY)
      LENGTH(WS_TRAY_LENGTH
      PAGING
      RESP (RESPONSE CODE)
END EXEC

Thanks

;)Kathy

-----------== Posted via Deja News, The Discussion Network ==----------
http://www.dejanews.com/       Search, Read, Discuss, or Start Your Own
```
