```
All two-pass compilers work pretty much the same way. COBOL is no exception:

1. Scan source. Parse lines into programmer-created symbols - size and type
for datanames, library module(s) and any needed symbols for them for
language (COBOL, C++, whatever) verbs. On error, report. Retain symbol
table.

2. Scan again, converting symbols to their offsets for calls or moves. Where
necessary, insert calls to cast (convert) data to correct symbol types.

3. If no error, link generated modules with library modules, add import
tables, resolve cross-module references.

Is that what you meant? Or was there something more specific you were
interested in?
```
