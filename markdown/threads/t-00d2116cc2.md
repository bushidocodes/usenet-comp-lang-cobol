Well, I couldn't find the demo download, so I bought it.

Observations after about 20 minutes of playing:

1. It does not generate native, stand alone code. To run a program compiled
with the product, you are required to install the package on the target
machine. Further, the product doesn't generate OBJ files and does not use a
Linker, leading me to believe it generates pseudo-code requiring a runtime
system.

2. There being no documentation I can find, one wonders how to generate a
text file (CR/LF delimited). I can get a non-delimited sequential file, but
not a LINE SEQUENTIAL file. Haven't tried relative or ISAM.

3. I can't figure out how to assign a file name to a variable as in:
   SELECT MYFILE ASSIGN TO DATA-NAME

the construct:
   SELECT MYFILE ASSIGN TO 'C:\TEMP\TESTDATA.TXT'
does, however work as expected.

4. There are, of course, no windows capabilities: The product is much like
FJ V3  in that it works on/in the command line. On our XP machine, the
product works quite well.

5. The integrated editor/compiler/maker thingy works well - even has
context-sensitive colored lines. I can't figure out how to set the tabs. Oh
well.

6. So, after 20 minutes, I'd say KOBOL fits between TinyCOBOL and FJ3
(simply because FJ3 seems more robust). That opinion may change, even
dramatically, after more experimentation.

sections ".stab" and ".stabstr" in PE executables before.)

runtime DLL cygwin1.dll, from which test1.exe directly imports about 50
functions.
