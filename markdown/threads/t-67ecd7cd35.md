For those who may have missed it, from the announcement of

  "IBM Debug Tool Utilities and Advanced Functions V3.1 Helps Maximize
Availability of z/OS and OS/390 Applications"

at

 http://www.ibmlink.ibm.com/usalets&parms=H_202-193

which MUST not be confused with the announcement of:

 "IBM Debug Tool for z/OS and OS/390 V3.1"

which is at

  http://isource.ibm.com/cgi-bin/goto?it=usa_annred&on=202-194

you may well be interested in the following paragraph,

"Support for OS/VS COBOL and Other Old COBOL Programs: Source programs
written for OS/VS COBOL are supported for debugging by a "Convert and
Compile" option. This option includes source conversion during the compile
step. The compilation is done using either Enterprise COBOL for z/OS and
OS/390 or COBOL for OS/390 and VM and then the converted OS/VS COBOL source
programs can be debugged using Debug Tool. In addition, once the debugging
is completed you can choose to continue using the OS/VS COBOL compiler, or
since you have already debugged the new module and know it works, you can
use the new level of source code that was produced during the compilation
and use a newer compiler. One by one you will eventually convert all of your
OS/VS COBOL source code to the latest compilers! This feature will also
support ANY level of COBOL source program as input, including VS COBOL II,
COBOL for MVS and VM, and COBOL for OS/390 and VM programs that were
previously compiled with the CMPR2 compiler option. This source conversion
can also be done separately from the compile step if you choose. "

    ***

Now, please do NOT ask me what this "feature" does or how it relates to the
traditional "CCCA".  I am just the messenger, not the interpreter <G>
