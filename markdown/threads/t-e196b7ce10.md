Hi Dave,
Let's see if I can shed a bit of light here.

C-3 is signed packed numeric, or COMP-3. A COMP-3 field is a number,
with two digits per byte, except for the last byte which has the sign in
the lower half byte.

Therefore an S9(7)V99 COMP-3 field (say) which has a value of 123.45,
is stored in four bytes as 00 12 34 5C. If the value was negative, it
would be stored as 00 12 34 5D.

COMP fields are binary, in 2, 4 or 8 bytes. S9(4) COMP (or COMP-4) is
stored as 2 bytes, S9(5) to S9(8) is 4 bytes, S9(9) is 8 bytes. Our
aforementioned 12345 would be stored, if S9(4) COMP, as 30 39. A value
of decimal 16 would be stored as 00 10.

USAGE is a "noise" word. You can put it in but the compiler doesn't
really need it. DISPLAY is a numeric field where each digit takes up one
byte. The high-order half-byte is set to F. Therefore, to store 123.45
in a 999V99 display field. you'd need 5 bytes F1 F2 F3 F4 F5.

Notice this field is unsigned. Signed numeric display is to be
avoided at all costs!

Let me know if this helps.

Just about any COBOL book can help you figure out the copybook. To
address what you have put in here:

USAGE is an optional word that you can put in your field definition
(ie USAGE IS COMP). All it does is make the syntax a little easier to
read. If it is not there, it is assumed by the compiler and therefore
unneccessary.

01 A-FIELD PIC X(80) USAGE IS DISPLAY.

is the same as

01 A-FIELD PIC X(80) DISPLAY.

is the same as

01 A-FIELD PIC X(80).

DISPLAY means that it is display format, it is formatted so you could
see and understand it (or print it). It is the default, so is
unneccessary to note (see above).

COMP is a usage format that tells the computer that it is in Binary.
The following field:

01 A-FIELD PIC S9(4) COMP.

is a 4 digit (2 byte) binary (numeric) signed field.

The C-3 is how many compilers abbreviate COMP-3, a usage format that
tells the computer to used Packed data. I don't know if it will work
defined within the program (maybe someone else can tell us that).

I hope this will help some, but your best bet is to go out and get a
decent book. Good luck.....
