True, CA-Realia provides a library to run your DOS apps in a "DOS-like"
window. Realia also provides the libraries for native GUI, but you will
need the Windows SDK. What little native Windows programming I have
done I found to be extremely tedious and time-consuming--time I just
don't have.

I use a product called Sp2 from Flexus for my GUI screens. It is
designed for COBOL (many vendors) and uses a simple "CALL" structure
with a few parameters. The COBOL program is the control mechanism, not
the tool. Once the Sp2 product finishes a process, it returns control
to the COBOL program. Unlike many of the other tools, it does not use a
Basic-like or Scripting language--just simple "CALLS".

I found Sp2 very easy to learn and use, and I get great results. Flexus
has a web site at:

http://www.enter.net/~flexus/

Mike Dodas
