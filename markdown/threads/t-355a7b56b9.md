When you use the report writer on the UNISYS for a file that is not
assigned to PRINTER you get 4 extra bytes at the start of each record.

-> UNISYS report format for a file other than printer file.

----------------------------------
| CC | CODE | BODY |
----------------------------------

CC = 2 bytes -> the carriage control character specified as a signed
binary item with:
- negative(-) = skip page
- positive(+) = skip specified number of lines
CODE = 2 bytes -> specified by the CODE clause
BODY = the report line

Does anybody know if there is something like this in MFCOBOL?
