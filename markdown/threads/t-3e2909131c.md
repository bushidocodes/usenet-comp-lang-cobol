My personal experience with language conversion tools is that none of them
work very well. Because everyone writes code a bit differently, it's quite
difficult to construct a tool that works with a high degree of accuracy in a
completely automated fashion. You might be better off re-writing the
application from scratch.

I would also add that with all of the 3rd Party COBOL productivity tools
available maintaining and enhancing the existing application may be a better
alternative than re-writing it in a different language. Just my opinion.
