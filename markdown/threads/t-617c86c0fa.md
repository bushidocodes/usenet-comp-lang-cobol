You compile your program using the compiler. This gives you an .OBJ-
file.

Then you link it using one of two ways:

The shared run-time system. This gives you a small .EXE-file,
but it needs COBLIB.DLE to run.

The static run-time system. This gives you a bigger .EXE-file,
but it runs alone.

All is done using only Base Cobol. You do not need any of the extra's.

I assume you are using a current version of MF-Cobol (version 3.1.35+).

Further information can be found i your manual (Cobol System Reference
Volume 1), page 15-1 (for MF Cobol version 3.2) or in Cobol System
Reference page 14-1 for MF Cobol 3.1.

MF Cobol 3.1 is the same as Microsoft Cobol version 5, but the MF version
should be preferred, do you have the choice.

MF includes a Link.exe program with the Workbench (for DOS and OS/2 at
least). You have the option of performing a static link (which would include
all the run-time code) or a dynamic link (which would look for the run-time
code at execution time). If you are using any OS services, you would have to
link the appropriate OS library as well.
