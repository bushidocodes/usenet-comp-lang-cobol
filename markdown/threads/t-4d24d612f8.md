Hi all,

I've encountered the following problem using Cobol and need help from
Cobol experts. I am developing a tool that instruments a Cobol program
by inserting calls
before selected statements of the program. Those calls send the values
of variables occuring in the statements
to the message-queue middleware. The values of variables should be in a
human-readable form, so I need to
convert variables of an arbitrary type into string representation before
sending them. Some USAGES (like
DISPLAY) do not need conversion, while others (BINARY, COMP-1 ....)
should be converted.

It seems that all I need is to redirect somehow the output of the
DISPLAY statements into a memory buffer
(under which I mean a variable with PIC X(...)). So,
I wonder whether can I DISPLAY to the memory and, if yes, how this
could be done?

I use IBM Visual Age 2.2 Cobol and IBM MQ Series. If someone could
suggest any other solution to the problem,
it will be greately appreciated.

Thanks in advance.
```

A variable IS a memory buffer, so just insert a MOVE to a variable you
define with USAGE IS DISPLAY PIC X(n) for alpha, PIC  9(9)V9(4) SIGN IS
SEPARATE for any numeric. (you may need to define alternate scale and
decimals depending on the dataname, but that's no big deal). Let the
compiler handle any conversions. This also lets the compiler handle any
conversions of 'implementor-defined' numeric datatypes (e.g.,
 COMP-1,-2 -3).

If you need it, you can get the address of that variable with the ADDRESS OF
function, but if you insert a CALL you can just add this variable to the
parameter list and the compiler will handle that for you, too.

You may have some problems dealing with in-line literals because you'll have
to parse out the portion which is the literal, but as long as you are
'inserting' code, this seems pretty reasonable.
```
