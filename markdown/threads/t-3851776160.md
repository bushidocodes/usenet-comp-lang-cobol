```
I believe that there is also a "freebie" on the CBT tape - but I don't easily
have the references for that.  If no one else "jumps in" with the correct
reference, I will check further.

(you could check www.deja.com - for either comp.lang.cobol or the IBM-MAIN
list, but I am not certain what to check for.)

I tend to recommend the EPA (Edge Portfolio Analyzer) because it does a
VARIETY of tasks that are important for an LE migration.  However, I do
believe there are other products that will go thru a PDS of load modules and
find OS/VS COBOL programs - if that is all that you need.

P.S.  You are aware that if you have OS/VS COBOL R2.4 compiled programs that
they are and will continue to be supported (in object code) with LE?  If they
were compiled with NORES, then you would need to re-link-edit them with the
LE link-edit library.  However, if they are compiled RES, they are supported
"fully" with LE.

This does NOT mean that I don't recommend "upgrading" such code so that they
can be maintained and re-compiled with a newer and SUPPORTED compiler.
However, there are lots of people who still don't realize that IBM does
support OS/VS COBOL object code running under LE.
```
