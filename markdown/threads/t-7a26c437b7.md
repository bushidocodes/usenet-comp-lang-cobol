A user asked me today how he could move some data from an IBM host to a
vendor's PC (or LAN) under the following conditions:
(1) Minimal intervention - he's telling the vendor to use FTP rather
than something labor intensive like using a terminal emulator program,
(2) Data compression - the data contains names, meaning there should be
a number of spaces in most records, and
(3) Encryption - the two parties are discussing whether some or all of
the data should be encrypted for confidentiality reasons.

If anyone can shed some light on a good way to do this, or suggest
another News Group I should post to, please respond to
