I installed a 672C on Win95 and had no problems. Here are a few
things you can verify.

Find out how the printer is connected:

Press Start, Settings, Printers. In the Printers window right-click the
on the printer icon and select Properties. In the Properties Dialog,
select the Details Tab. The item, "Print to the folowing port" will
tell you the destination for printer output. It should show "LPT1: ..."

If the printer is not connected to LPT1, the program cannot print
to LPT1.

Find out if the printer port is available to DOS:

Press Start, Programs, MS-DOS Prompt. At the DOS prompt, enter
the command, "debug". At the hyphen, enter "d 40:0". the display will
have 8 lines showing 128 bytes in both hex and characters. The first
line will begin "0040:0000". Following this will be 8, 2-digit hexidecimal
numbers and a hyphen. The two pairs of hex digits after the hyphen
give the port address of LPT1 as it is available to DOS. If the value
is "00 00", the port is not available to DOS. Normal values are "BC 03"
or "78 03". At the hyphen, enter "q" to quit debug. Then "exit" to close
the window.

If the port is not available to DOS, a DOS program cannot use the
port. Though I would expect the MF run time to report error 027
"Device not available."

If the report was a single page, maybe it did print. The 672C is a
page printer. Before closing the file, force a page eject with the
following code.

    WRITE name-of your-print-record FROM " "
        BEFORE ADVANCING PAGE

This will ensure that the last command to the printer is a "form feed."
Though, if a partial page was sent to the printer, there would be a
flashing indicator to show that the page was not complete. Pressing
the page eject button should cause the partial page to print.

If these suggestions and others that may be provided are not sufficient
to resolve the problem, you might try one of the windows newsgroups.
