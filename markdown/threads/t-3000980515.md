I don't remember the exact details, but we had problems with an Acucobol
program running under NT with Samba as the server.  If I recall (like I
said, I don't remember exactly), the problems occurred when NT and Win98
were mixed on the network, accessing the same files off the Samba share.
Everything was OK if they were all Win98 or all NT.  Mixing caused problems
for some reason.

Adding these lines to the Samba config fixed it:

locking = yes
oplocks = false
blocking locks = false
