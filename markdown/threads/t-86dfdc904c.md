### When to use GO TO

- **From:** "10025..." <ua-author-17071258@usenetarchives.gap>
- **Date:** 1997-06-15T20:00:01+00:00
- **Newsgroups:** comp.lang.cobol
- **Message-ID:** `<5o4de0$ltb$2@mhafc.production.compuserve.com>`

```

Cobol is a living language. The are "dialects" of Cobol in
use in a sense of the word "dialect" different from the usual
meaning in the context, but used more as the word is used of
natural languages.

I don't code like to code programs full of paragraphs, full stops
and GO TOs. But, as an itinerant worker, I must respect the
cultures of the communities that offer me work. If the style of
programming used by people who are comfortable using GO TO is the
Ebonics of Cobol, I'd best learn at least to understand Ebonics.
Maybe I'll even code in Ebonics, to make the code more
maintainable after I have left the 'hood.

I understand reasons for not using GO TO and can recognise the
rare situations in which using GO TO is necessary. I have been
programming for a *very* long time. I use GO TO less frequently
than most people.

I used to teach Structured Programming. I introduced all my
students to a range of situations in which it would be
appropriate to use the GO TO construct. I also did a lot of
thinking about how the Cobol language required improvement to
eliminate the need to use GO TO in some of these situations.
```
