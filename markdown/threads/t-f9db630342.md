```
This is the problem at hand:

We need take a cobol74 module and convert copies of it for cobol2, mvs cobol,
cobol74 y2k, cobol2 y2k, mvs cobol y2k.....

The problem is the naming convention within the same environment.... We are
trying to keep it within 8 characters... Our extensive library has so many
differently named modules it was hard to come up with a rule.

What we finally came up with is end the program name with @ then a number 1 to
6 (representing which version it is).

Looking it up in the cobol compiler manuals this is allowed...

but then......


It fails the compile. Messages are as follows:

VS COBOL:
IKF1083I-C Illegal character. Scan resumed at next valid character.

********************************
COBOL II:
IGYDS0027-S Non-COBOL character(s) were found starting with "@" in column
26. The characters were discarded.



they both died on the program id name..... Since we werent going to play with
differently named ncals... yda... yda... yda... We thought there might be a
switch we missed in the compiler...

or if anybody has another idea for a naming standard to identify a source
module within the same environment but at a different language level and y2k
stage. Sure could use an idea....

couldnt use y2 at the end..... or numbers..... over the years we have just
buried ourselves.... Special characters was the best idea... and darn it the
compiler manual says we can do it....

please help
```
