Addressed to: Distribution list (see below)

Below is a program that loops thru all days of each year in a given range doing 1 of the
following:
1. Just displaying the dates generated excluding selected year ranges.
2. Calculating the COBOL day number using FUNCTION INTEGER-OF-DATE for all dates generated and
displaying the dates with the day number excluding selected year ranges.
3. Calculating the COBOL day number using inline code for all dates
generated and displaying the dates with the day number excluding selected year ranges.
This program is to demonstrate the relative efficiency in obtaining the COBOL day number in two
different ways and has a no calculate version so that the CPU time for generating the dates and
displaying a selected subset of them can be measured.

This program was compiled on MVS using the COBOL for MVS and VM compiler release 1.2 with
the following compile options:
OPT(FULL) - full optimization
TRUNC(OPT) - truncate based on word boundary if all operands and results are literal or binary,
otherwise truncate based on picture.
LIST - show all instructions generated
APOST - Use apostrophes (') as the QUOTE character (IBM peculiarity). If you are compiling
on another platform you may to change all of the apostrophes to quotes before
compiling.
NUMPROC(MIG) - Treat both F zoned numbers (0 - 9) and C zoned numbers (x"C0", A - I) as
numeric in signed fields.
It should compile with either no message or warning messages on most platforms if legal literal
delimiter characters (the apostrophes may have to be replaced by quotes) are used. An
executable should be generated as opposed to code run by an interpreter. If the target
platform does not have hardware decimal arithmetic instructions another algorithm for
determining leap year and number of days from the given year may be more efficient.

The JCL (work flow language or bat file equivalent) has executed successfully with site
specific information replacing certain of the generic information. It would be interesting to
see how this would be executed on other platforms and I will be interested in timing for
other platforms.

results of the calculations have been verified as being the same for displayed years by using
the IBM ISPF compare utility SUPERC.
