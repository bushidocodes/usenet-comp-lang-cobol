A few months ago, I posted in this NG that I was searching
for a VisualBasic-like development environment, but for
an old cobol programmer, like myself. I got lots of
suggestions (thank you), that included many different
products.

I'm currently evaluating demos from Fujitsu and AcuCobol.

So far as I can tell (and I'm only beginning to try these
products out...), either has the features I seek. I want to
develop products that will run under Win95/WinNT,
that will build on my old cobol syntax (rewrite the code,
but not the algorithim). I'm presently using ISAM files,
which both of these products seem to support. I may
go to an SQL type database later, and again both of
these products seem to support that also.

There is a radical difference in the cost of these two
products. Fujitsu's most complete system is $500.
Acu's is $1700.

Since I'm presently lacking any reason to spend
3 times as much for Acu's product, I'd like to hear
from users of either or both product with their
experiences. I'd love to hear from anyone who
has used both of these products extensively,
but I don't know if there is anyone who has
really done that - both products seem to be
quite new.

thank you,

don ferrario

Don, the combination I hit upon is to use Fujitsu COBOL with COBOL SP2
from Flexus. While Fujitsu has a GUI font end development tool, it
requires a particular syntax and development style that still relies on
"event driven" logic to be considered in your COBOL program, and is
divergent from the traditional "Procedural" method. COBOL Sp2 handles
the event driven portion of the system and lets your COBOL remain
procedural. This makes porting of applicaitons easier.
