the compiler, because it includes some hand holding as well, is Thane
Hubbell's book "Teach Yourself COBOL in 24 Hours".  You can also get a
slightly newer version of the compiler (though with fewer add-on's) in
"Mastering COBOL", a rather massive reference work.

maybe not a function persay, but using protypes, you can create your own
verbs...

Merant cobol

       Program-ID. "cmEnableWindow" is external.
      ******************************************************************
      * Enable a window or dialog control...                           *
      * Input = hWindow                - handle of window to be enabled*
      * Output= N/A                                                    *
      * Cut and Paste the following sample implementation...           *
      ******************************************************************
      *    cmEnableWindow WindowHandle is ??????
      *                                OR                              *
      *    cmEnableWindow ??????
      ******************************************************************
       Special-Names.
       Call-Convention 74 is WinAPI.
       Linkage Section.
       copy "W003SD.CPY".
       01 hWindow                      handle.
       Procedure Division WinAPI using by value hWindow
                 invoked as == cmEnableWindow
                               [Windowhandle is] <hWindow> ==.
       End Program "cmEnableWindow".



      $Set case Defaultcalls"74"
       Program-id. "cmDisableWindow".
       Special-Names.
       Call-Convention 74 is WinAPI.
       Working-Storage Section.
       copy "W003SD.CPY".
       01 hWindow                      handle value 0.
       01 APIReturn                    bool value 0.
       Procedure Division WinAPI using by value hWindow.

           Call WinAPI 'IsWindowEnabled'
             using by value hWindow
             returning APIReturn.

           If APIReturn equal 0
             exit program
           end-if.

           Call WinAPI "EnableWindow"
             using by value hWindow
                   by value 0
             returning APIReturn.

           Exit program.




Now you can invoke

    cmEnableWindow Hwnd.

or

  cmEnableWindow WindowHandle is Hwnd.

Thane,
  I have kept this in my "to do" list and am finally getting around to it.
What you want to look at (in Micro Focus' implementation - that has been
there for SEVERAL years) is

 - $IF "directive"
 - $IFDEF "directive"
- $ELSE and $ENDIF (no hyphen as I recall) constructs
- $CONSTANT directive

I have been told by one of my "usually reliable sources" that >>IF and
>>EVALUATE may be accepted when the ISO2000 directive is on, but they don't
think they are actually processed (yet).

Two *major* diferences between the MF (and this does date back to Micro
Focus, not just MERANT) implemenations are:

 - You can "interchangeably" use 78-levels and constants defined by $CONSTANT
 - $DISPLAY to display (during compilation) what "path" or value of constants
are in effect for that specific compilation.

In my opinion (but not that of J4) BOTH of those are quite useful
"extensions" to what will be in the Standard.
