NT servers, then there is the IBM CICS Client which has an OLE Automation

The COBOL program running within CICS will see the call from VB as an 'EXEC
CICS LINK'.

Microfocus has a wrapper product (Bridge) that make VB and COBOL talk
very easy. If Your data is on normal display format You may not this
this, it is up to You how much You will code or how much You spend on
tools.

CICS has two type av programatic client interfaces, EPI and ECI.
External Presentation Interface, External Call Interface.
EPI work on the 3270 stream level. ECI is for application talking not on
the presentation level. Both interface can be used from a CICS client
regardless of CICS vendor (IBM, Microfocus or other) or where the CICS
server is running, LAN, UNIX or MVS.
