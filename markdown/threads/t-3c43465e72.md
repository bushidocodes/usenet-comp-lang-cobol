IIF the assembler layout is very well defined, that is, actual types are 
used instead of "C" for everything.  And the duplication factors and all 
line up.  Then you can do this with the macro preprocessor.

Set up a simple assembler shell, copy in the layout, and repeatedly call 
a macro with each field in order. 

Write the macro, perhaps EMITCBFL, to use the various symbolic attribute 
checks like, for example the type is T'fldname.  I believe that all the 
information you need is available for each field, just format up a text 
line and punch it.

You may have to make some changes to this depending upon how
you see the fields being used. Some fields defined with signs
may not need to be treated as signed fields. I hope this helps you
get started.

Some working storage fields are fields referred to but which had
no definition in your post. Others are lengths of various records
that your COBOL program may not need.

Addressing fields in assembler is done using a base register and
displacement method. Each field in a DSECT has a known displacement
from the start of the DSECT. The base register associated with a
DSECT is specified via a USING directive to the assembler.

A DSECT in assembler is simply a map of storage and is associated
with a USING that specifies a register. For example USING EX7DSECT,2
would associate register two with the DSECT. When fields in the DSECT
are referenced they would then be assembled using register two as a
base register until another USING for the DSECT is encountered.
Therefore by placing a address in register 2 you can make the DSECT
reference or map storage starting at that address.

In COBOL a DSECT is equivalent to a definition in the LINKAGE-SECTION.
Suppose you had the following field defined:

       01  R2-PTR      USAGE IS POINTER.

and further suppose that it contains the starting address from which
you want the EX7DSECT to start mapping from. You can accomplish this
with:

           SET ADDRESS OF EX7DSECT TO R2-PTR.
