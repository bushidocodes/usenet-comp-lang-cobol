here, what i recevied  by roger jhonson from fujtisu

Carlos lages

NetCOBOL for Windows Version 7 - Enhancement List
General
�       The look and feel of the Documentation Viewer is now more like
Web help systems with the list of documents in the left-hand pane and
the contents of the selected document displayed in the right-hand pane.
�       The style of most of the PDF manuals has been enhanced to look
better, fit more information on a page, and use an 8.5" x 11" page
format so that user printing of pages or complete manuals will be more
convenient.
�       Common graphics have been used in different parts of the product
giving a more integrated product image. For example, the autorun screen,
Project Manager start-up banner, manual cover pages, and product boxes
all use the same artwork.
COBOL
Enhancements:
�       BSAM file system has been extended to support large files (sizes
being confirmed - probably 3.4GB for indexed, 2.0GB for other file
organizations)
�       Solaris applications can be developed remotely using Project
Manager in NetCOBOL for Windows.
�       A project property setting feature has been added for setting
project attributes.
�       Projects can be viewed in a List View in addition to the
standard Tree View. The list view can show all items in the project or
just the editable items.
�       Our performance in using ODBC has been improved (may be viewed
as a bug fix by some)
�       The compiler will check calling and called program parameter
conformance for internal programs.
�       A special register SORT-CORE-SIZE has been added so that
programs can specify the amount of memory to be used by PowerBSORT in
sort or merge operations.
The following bugs have been fixed (i.e. the following will no
longer occur):
�       JMN0102I-U message is issued if you compile your programs with
compiler option PRINT, SOURCE, and TEST.
�       Compiler abends if AS modification is specified in in-line
invocation of METHOD.
�       Compiler may get into an infinite loop if incorrect conditions
are written in the "EVALUATE WHEN" condition.
�       Message JMN3482I-S is issued if a program for AIM-NDB without
the READY statement is compiled. (AIM is a Fujitsu mainframe product
that is similar to CICS)
�       Open error occurs if the top value of the target sequential file
is 0xFFFE or 0xEFBBBF.

PowerCOBOL
The following controls have been added:
�       Edit Control (a lightweight version of the TextBox control that
also supports data binding)
�       Label Control (a lightweight version of the StaticText control)
Also there are a few additions and adjustments to properties and
parameters.
PowerFORM
No enhancements.
PowerFORM Run-time
�       Print preview ability added.
�       PNG format is available as one of the supported picture types.
Collection Class Library
No enhancements.
J adaptor Class Generator
�       JDK1.4 is supported.
DB Access Class Library
No enhancements.
Data Editor
�       Data can be imported to or exported from XML data files.
�       Supports Unicode character data.
�       Replace function supports using character data as well as
hexadecimal values.
�       Layout Definition Files (*.lay) can be used with the Layout
Definition Function.
�       Handles binary data files.
�       Can set initial values when automatically generating data.
Data Converter
�       Supports conversion between XML files and the other supported
file formats.
�       Can create DTD (Document Type Definition - used with XML) files
and matching COPY libraries from the Layout Definition file.
PowerGEM Plus
No enhancements.
PowerBSORT
�       bsortex command added. The bsortex command can perform separate
processes in the one command whereas the earlier bsort command could
only perform one function per command.
�       Record skip feature added allowing the user to specify
conditions for omitting records in the sort or merge operation.
�       The event log output feature is supported on the NT kernel
OS. When serious errors occur, error information is written to the
event log.
�       Standard input and standard output can be specified to the bsort
command as input and output files.
�       Error messages that were issued to the standard output are now
being issued to standard error.
�       PowerBSORT can access files that exceed 2 GB.
�       PowerBSORT supports the BSAM access mode. So, PowerBSORT can
access NetCOBOL files larger than 1 GB using the BSAM access mode.
