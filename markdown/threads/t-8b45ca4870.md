You might try Olivadoti Associates, Inc, at 707-763-5123, and learn if
they are still offering COBOL FDChart program. The chart program runs
on a PC. It has only a minor glitch dealing with the placement of
column headings (it sometimes doesn't print it) but otherwise it's OK
when you have to discuss a record layout to a C++ programmer who just
can't conceptualize beyond the bit-counting stage.

BTW, Olivadoti's main product is a COBOL query code generator called
COBRA, where you write a SQL subset program and the product generates
the COBOL code for you. It's limited to processing two files in
sequential fashion, the SQL code is not the full set and it cannot
handle VSAM random reads, but it can help if you want a idea what the
relationship between SQL and COBOL codes are like or need to write a
simple quick and dirty query program on one file.
