You know what I would really love?  In addition to NOHANDLE, also a
HANDLE(condition).  In other words:

EXEC CICS READ
    FILE(FILNAM)
    INTO(MY-REC)
    RIDFLD(MY-KEY)
    HANDLE(NOTFND)
END-EXEC

IF EIBRESP = DFHVALUE(NORMAL)
    SET RECORD-FOUND TO TRUE
ELSE
    SET RECORD-NOT-FOUND TO TRUE
END-IF

Why?  Because this would allow an abend for any condition other than NORMAL or
the specific conditions I've chosen to handle (NOTFND, in this case).  I realize
I could put an IGNORE CONDITION but if I don't put a HANDLE CONDITION (with no
label) after it it will continue to be ignored, which is probably a bad thing.
Plus it's ugly.

And yes, I could (and do) have some generic error checking, but I don't like it.
You can't imagine (well, maybe you can) how many programs at my shop end with
such helpful messages as "ABNORMAL TERMINATION -- CONTACT PROGRAMMING" if a CICS
file is disabled or some such thing.  I much prefer an abend with a dump using
ABEND-AID.  In programs I write I've ended up doing a divide by zero in order to
force an ASRA, but I'd much prefer the method I've shown above.

(Please no one tell me to go back to HANDLE CONDITION with labels.  Yikes!)

Andy, there's nothing magical about VSAM files, nor is special COBOL code
required.

VSAM is a file-managment system used to create files of all types in an IBM
Mainframe environment.

In your COBOL program, VSAM files are just files - you OPEN, READ, WRITE and
CLOSE just like any other file. You'll need to use VSAM (through IDCAMS) to
create files of ORGANIZATION INDEXED or ORGANIZATION RELATIVE. You can use
VSAM to create ORGANIZATION SEQUENTIAL, but a lot of shops just use the QSAM
files allocated through the DISP parameter of JCL for sequential files.
