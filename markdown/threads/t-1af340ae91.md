I have had similar experiences many times, most often I have been able to
get rid of it by createing a shortcut to the application, rightclick the
icon, choose properties and open up the Memory Tab.

From here, forget about Conventional memory

If Expanded (EMS) Memory is not already set tp None, set it to none. EMS is
an outdated memory protocol, which is rarely used anyway.

Extended (XMS) memory could be left as is in most cases, however, to be sure
set it to 0, this one is more likely that could be requested though.

MS-DOS protected-mode (DPMI) memory is usually the one that messes up for
you, by some reason, Microsoft set this one to *auto* by default, many
applications can't deal with that. Unfortunately there is no way to turn it
of, but set it to a fixed number uses to fix the problem, set it for
instance to 3096.
