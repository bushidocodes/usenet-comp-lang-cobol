I think he means use a host variable in the SQL 'CASE' function.

No reason you could not, provided...
1. The DBMS and driver support CASE
2. You are not trying to use a host variable for a table name or column name
or ORDER BY object, because those change the statement such that it must be
re-PREPAREd. Of course, if you do totally dynamic SQL from within your COBOL
program, you can do just about anything; but the table name, column name and
order by are not 'host variable items' under any circumstances.
