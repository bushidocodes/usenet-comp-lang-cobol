Why not move PowerCOBOL to a Virtual Machine running under Windows 7?

Check out ORACLE Virtual Box (which is what I use) or there is a Virtual 
Machine provided with Windows 7

I haven't tried PowerCOBOL under Windowe 7 but see no reason why it wouldn't 
work.

COBOL development for NetCOBOL and PowerCOBOL (versions 5 -10) and Micro 
Focus Net Express 5.1 all run on Virtual Machines under Win XP or Win 7. I 
have done all of these on my own systems.

The procedure is:

1. Set up a VM.  (Very easy with Virtual Box) You can do this on a Win 7 or 
Win XP Host.
2. Install windows XP or Win 7 to the VM.
3. Snapshot or clone your new VM so you can set up other VMs easily. (There 
will be a free tool to clone VMs on the COBOL21 web site within the next few 
weeks. http://primacomputing.co.nz/COBOL21)
4. Install your application software ( Fujitsu Enterprise COBOL, Micro 
Focus, or anything else.)

Run and enjoy your software.

The VM can be connected to, just like any other "real" machine on the LAN 
and so it now becomes a "COBOL server" for running compilations from any 
machine in your network. (This requires some scripting which I wrote and 
would make available if you pay for it, or you can do it yourself. It 
involves using WMI/WBEM (MS "secret" technology :-))  to start remote 
processes on networked machines. See: 
http://msdn.microsoft.com/en-us/library/ms974579.aspx)

The PRIMA Migration Toolset pioneered this approach for compiling both 
procedural and OO COBOL code  of generated Load Modules for RDBs and Data 
Access Layer objects, which can be run from "old" and "new" technology. It 
has been running successfully for around 4 years now. The Toolset can 
compile its generated code locally or using a real or Virtual remote COBOL 
server as described.  You can see a video of it doing this (for RDB Load 
modules) if you take the "5 cent tour" at:
http://primacomputing.co.nz/cobol21/demosandtutorials.aspx

(The video is just over quarter of an hour and shows legacy COBOL 
definitions being used to generate an optimised Relational Database in 3rd 
Normal Form. Code to load the legacy indexed files to this database is then 
generated, compiled and run. The end result is an optimised RDB, loaded with 
current data, in around 10 minutes. No "hand coding", fully automated, just 
a few mouse clicks.  (This is just ONE part of the PRIMA Toolset, called 
Data Conversion. The Toolset does much more than that...)

You can back up the entire VM to a USB stick and take it to any other real 
machine you want to.

VM is the way to go.

Pete.
