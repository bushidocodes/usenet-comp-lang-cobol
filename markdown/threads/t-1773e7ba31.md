Hi all,

We've run our legacy accounting software (RealWorld) for 10 years on a
DOS box, then as a DOS window on a Win95 box. Now we're trying to move
it to a Win2K box. We moved the relevant directory to the box,
attempted to run the program, and got the error message:

"Cobol run time library not installed".

This message perplexes me, because everything appears to be set up
properly:

--The batch file that starts the software includes "SET COBDIR = .",
and the COBLIB.DLE is located in the same directory as the batch file.

--We created a Windows PIF for the batch file that sets the proper
working directory.

--I even tried FORCEDOS.

So, for some reason, the library is not loading properly, although it
is most certainly present and pointed to in the proper manner.

Anyone with experience getting legacy DOS-based COBOL stuff to run on
an NT/2K box, please help!

I'm surprised that FORCEDOS did not work. However, the alternative is to add
the directory containing the COBLIB.DLL (if your app includes it) to the
Os2LibPath system environment variable. Your app will then run as a 16-bit
OS/2 app rather than a DOS app.
```

```
Try adding the Cob set statements to the %SystemRoot%\System32\autoexec.nt
file (this and config.nt are read when you launch a 16bit process), and you
may have to append your path statement so the OBJ directory comes first.

rem append search path
set path=f:\dir\obj32;f:\public;f:\public\subpub;%path%
set pathext=.com;.exe;.cmd;.bat

rem set cobol environment
set cobdir=f:\dir\obj32;p:\dp\mf40\exedll;p:\dp\mf40\lbr
set cobcpy=p:\dp\dir\src\dds
set coblib=p:\dp\dir\src\dds
set cobsw=-f0+k3+i1-p1-l6
set vcscfg=p:\dp\dir\src\pvcs
set cobanim_2=animate
set seq=f:\dir\obj32
