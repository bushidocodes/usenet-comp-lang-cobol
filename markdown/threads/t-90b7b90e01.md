John,

Here's a plug for the 'lad's book' - get yourself a copy of Thane
Hubbell "Teach Yourself COBOL in 24 Hours". His Part II covers the
various file types. I'm sure you must have it in your bookshops by now -
if not try Amazon.com , Barnes & Noble, or if you want to keep it to the
'Blitish Empire' you can always try Chapters.ca up here in Canuck-land.

As you are learning I would suggest using surnames as your PrimeKey is a
bit tricky. (It's a topic all unto itself). Start off by allocating
account numbers, say C1 = Collins, J, C2 = Cartwright, C3 = Collins,
M.J. etc.. - not alphabetical - but if you want those names in
alphabetical you do it with an "Alternate Key with duplicates" - because
you could have two or more Collins, J. (Like here in Calgary for the
Chinese surname 'Lee' there are at least five pages in the directory !)

So you create records using a Primekey - C1, C2. C3 and print them out
alphabetically using the Alternate Key - Adams, B, Adams B.G, Adams H
etc..

pseudo code :-

	Open file i-o (Random - ie. ISAM Indexed Sequential Access Mode
	Enter A/c # - read PrimeKey
		      if found display record
			 if I change this record
			    rewrite this record
		      else if notfound enter new data
			      write this new record

And you need that Alternate Key with duplicates otherwise on a write or
even rewrite (where you might have corrected the name) - you will go
belly-up without the 'with duplicates' check.

Above is very brief - but coupled with Thane's book - or any other good
COBOL text - it should put you on the right track.

The file has to be ORGANIZATION INDEXED right?

So, if you OPEN EXTEND you are implicitly adding higher keys. An
attempt to add a record smaller than the largest existing key yields
an 'invalid key' condition.

If ACCESS RANDOM you can't open the file as EXTEND (EXTEND is reserved
for SEQUENTIAL access).

What to do?

Answer: OPEN I-O.

Typical sequences for an ISAM (Indexed Sequential Access Method) are:

1. Unsorted input
(ACCESS I-O)
OPEN OUTPUT   (this creates the file)
CLOSE
OPEN I-O
WRITE records
CLOSE FILE

2. Sorted input
(ACCESS SEQUENTIAL)
OPEN OUTPUT (creates file)
WRITE records (in order)
CLOSE
This file may never again be opened OUTPUT without destroying the
original file.
