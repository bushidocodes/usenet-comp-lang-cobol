    	Ignoring any implications for "IDXFORMAT 3", rebuild in place 
assumes that the data portion is intact.  Rebuild into another file 
(note: REBUILD OLDNAME,NEWNAME) without the "/D" option assumes that both 
the data portion and index are intact.  To rebuild an indexed file with a 
corrupted data portion, you need the "/D" switch.  

    	I suppose you could rebuild in place, but the data would still be 
corrupted.  

    	Also note that when the index potion is excessively corrupted, you 
can help things along by replacing the index with a known good copy (even 
an empty one) BEFORE the rebuild in place or rebuild /D.  I have seen 
index files that were in such bad shape that rebuild could not figure out 
what the proper key strucure was.
