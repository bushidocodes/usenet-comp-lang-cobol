Daniel Demeyere wrote:
: I'm searching for a MS-COBOL routine that removes all
: spaces from a string. Does anyone know where I can find it ?
: Or can someone help me writing such a procedure ?

Someone has kindly offered to send you a proper string-handling package,
and this may be what you need. But let me suggest something that is very
nice if you have a small number of nonblank tokens:

Set up a series of temporary variables, long enough to accommodate
the nonblank strings you'll encounter. Suppose you call them
foo1, foo2, etc. Then:

move spaces to foo1, foo2, foo3, .... <<== I'm not sure you need this.

unstring source-string delimited by all spaces
into foo1, foo2, foo3, .....

The 'all spaces' construct causes the 'unstring' to treat every string of
multiple spaces as a single delimiter. So foo1 will be the first nonblank
substring, foo2 the second, etc. Each will have its nonblanks left
justified, with trailing spaces. You may find foo1 is empty if the
source-string had leading spaces, with the first nonblanks in foo2.
It won't matter either way, except you must provide enough of these
varibles to accommodate all the nonblank substrings you'll encounter.

Then do this:

move spaces to dest-area. <<<== You DEFINITELY need this.

string foo1 delimited space, foo2 delimited space,
foo3 delimited space, ...
into dest-area.

This will give you all the nonblank strings, packed together at the
beginning of dest-area, with trailing blank padding. You can use the 'with
pointer' feature in the 'string' statement to record exactly how many
nonblank characters you have.

This approach is especially nice if you only expect one or two nonblank
strings in the source-string, because you can do the whole thing in-line
with just a few lines of code. The more unpredictable the situation
gets, the more you need a proper tokenizing package.

Good luck,

One way is to:

MOVE SPACES TO Temp-Strings
UNSTRING Original-String
DELIMITED BY ALL SPACES
INTO Temp-1 Temp-2 Temp-3 ...
MOVE SPACES TO Original-String
STRING Temp-1 Temp-2 Temp-3 ....
DELIMITED BY SPACE
INTO Original-String

Where there are sufficient Temp-1, Temp-2 ... of adequate length
and these are in the group field Temp-Strings.

This is not particularly efficient, so a mechanism that shifts
the characters in-situ may be better.

MOVE 1 TO In-Index
MOVE 1 TO Out-Index
PERFORM
UNTIL In-Index > Length-Of-String

IF String(In-Index:1) = SPACE
ADD 1 TO In-Index)
ELSE
IF In-Index = Out-Index
ADD 1 TO In-Index
ADD 1 TO Out-Index
ELSE
MOVE String(In-Index:1) TO String(Out-Index:1)
ADD 1 TO In-Index
ADD 1 TO Out-Index
END-IF
END-IF
END-PERFORM
IF Out-Index < Length-Of-String
MOVE SPACES TO String(Out-Index:)
END-IF
```

Daniel,
I have not tested out this code but it should work. I don't
know MS-COBOL
so I based it on COBOLII. The (sub1:1) is known as reference
modification and
if its not supported on MS-COBOL, you can redefine "s" as a
table and just
subscript through it.

Good luck,
Mike Giaquinto


77 sub1 pic s9(4) comp.
77 subn pic s9(4) comp.
77 len pic s9(4) comp.

77 s pic x(100) value 'unknown string value with spaces
'.

move 1 to subn
* compress string
Perform varying sub1 from 1 by 1 until sub1 > length of s
if s (sub1:1) not = spaces
if sub1 not = subn
move s (sub1:1) to s (subn:1)
end-if
add 1 to subn
end-if
end-perform

* clear out end (if needed?) or use subn - 1 as the new length
compute len = (length of s + 1) - subn
If len > 0
move spaces to s (subn:len)
end-if
```
