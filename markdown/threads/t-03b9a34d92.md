public class FinancialCalculations{

   public static double calculateMonthlyPayment(double loanAmount,
   int months, double monthlyInterestRate){
      double monthlyPayment = loanAmount * monthlyInterestRate/
         (1 - 1/Math.pow(1+monthlyInterestRate, months));
      return monthlyPayment;
   }

   public static double calculateFutureValue(double monthlyPayment,
   int months, double monthlyInterestRate){
      int i = 1;
      double futureValue = 0;
      while (i <= months) {
         futureValue = (futureValue + monthlyPayment) *
         (1 + monthlyInterestRate);
         i++;
      }
      return futureValue;
   }
}

IThe current ISO standard describes the PRESENT-VALUE function as having at
least two parameters, and defines it similar to the following:

    For two parameters:
        (argument-2 / (1 + argument-1))

    For three parameters:
        (argument-2 / ((1 + argument-1) + argument-3 / (1 + argument-1) **
2)

    For n parameters, n > 3:
        FUNCTION SUM (
            (argument-2 / (1 + argument-1) ** 1),
            (argument-3 / (1 + argument-1) ** 2),
            ...
            (argument-n / (1 + argument-1) ** n))

I'd think that description ought to be sufficient to get you started.

    -Chuck Stevens

Here are a couple of ways.  You didn't specify what type of interest
was being used, so the formula differs.

The present quantity P which in n years will accumulate to the amount
A(n) [read as A times n] at the rate of interest r:

At Simple Interest:  P = A(n) divided by 1 + nr [read as n times r]

At interest compounded annually:  P = A(n) divided by (1 + r) to the n
power

At interest compounded q times a year:  P = A(n) divided by (1 + r
divided by q) to the n times q power.

Further, P is called the present value of A(n) due in n years at rate
r.  The true discount is D = An - P
