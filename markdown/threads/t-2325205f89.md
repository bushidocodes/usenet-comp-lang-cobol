I haven't "hit the limit", but I think I see why you do if you're using
60+ isam files. Each isam, or indexed, "file" is actually two physical
files, one for data and one for indexes. Add any other files MF may need
to support its operation and you reach the maximum number of files allowed
somewhere very near 60 indexed files. I think I've got that right. If
I've gone wrong somewhere, I'm sure someone will let us know :).
