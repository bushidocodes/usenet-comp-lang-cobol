I am taking this from a CICS book. EIBCPOSN returns a value = to the
position across the entire screen. The value is relative to 0, not 1.
So, the values for the first row are 0 thru 79. If you need to look for
the cursor to be in specific fields on your screen, you must compute the
position values for each field. Let's say you have a field that is in
row 15, attribute column 12, data column 13 thru 27. You must compute
the starting and ending positions of the field. The book says that the
following formula will do the job:
compute cursor-pos = (row - 1) * 80 + column-position - 1
so, for the leftmost position, (15-1) * 80 + 12 - 1 = 1131 and the
rightmost position, (15-1) * 80 + 27 - 1 = 1146. If you store these
values in working storage, you can use them to test if the cursor is in
a particular field. The book suggests:
01 cics-cursor-pos pic s9(4) comp.
88 field name value 1131 thru 1146.
Execute MOVE EIBCPOSN TO CICS-CURSOR-POS. You can then use an evaluate
statement to check for the cursor position.
Hope this helps!

As the earlier reply said, the EIBCPOSN is an absolute displacement from
the first position in the displayed screen buffer. A simple piece of
code to compute the row and column is

WORKING-STORAGE.
77 CURSOR-ROW PIC S9(04) COMP VALUE +0.
77 CURSOR-COL PIC S9(04) COMP VALUE +0.
PROCEDURE DIVISION.
DIVIDE EIBCPOSN BY 80 GIVING CURSOR-ROW
REMAINDER CURSOR-COL.
ADD +1 TO CURSOR-ROW CURSOR-COL.
