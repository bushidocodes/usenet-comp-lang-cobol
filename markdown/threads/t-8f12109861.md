It's the same as their MS-DOS versions. Select associates an internal
file name with a device such as DISK or PRINTER. The external file
name is set with the VALUE OF FILE-ID clause.

for example:

    SELECT OUT-FILE ASSIGN TO DISK
    [ORGANIZATION IS...].

    FD OUT-FILE
        LABEL RECORDS ARE STANDARD
        VALUE OF FILE-ID IS "OUT.DAT".

to compile foo.cob under CP/M:

COBOL =foo

to link, load and go under CP/M:

L80 foo/g

Under MS-DOS the commands are different and depend on the version.

        SELECT [OPTIONAL] myfile
            ASSIGN TO DISK
            ORGANIZATION etc ..

       
        FD  myfile
            VALUE OF FILE-ID IS {data-name | literal}
            .

The CP/M name must only be a 8.3 name and type optionally preceeded by
a drive letter and ':'.

I think that to create a new file that is OPEN I-O you need to specify
OPTIONAL otherwise it will report that the file doesn't exist. 
Obviously creating a file with OPEN OUTPUT will work without OPTIONAL.
