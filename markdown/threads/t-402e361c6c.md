But on to address Art's original question. The answer Art, is yes, you must
specify a fixed number of hold (receiving) areas. However, by use of the
POINTER, COUNT, TALLYING and OVERFLOW options, you can keep up with the
progress
of the UNSTRING and use it repeatedly if necessary to completely UNSTRING a
field into an unlimited number of receiving fields. POINTER specifies the
first
position in the sending field for the UNSTRING to begin. COUNT specifies a
count of the number of characters the sending field which were examined.
TALLYING specifies the number of receiving fields which were populated, and
ON
OVERFLOW tells you when there were not enough receiving fields to hold all
the
data.

Sorry, Art, I didn't catch the "I don't have any manuals available" before.
I'll try to duplicate the ANSI standard syntax diagram for UNSTRING below:

UNSTRING identifier-1
--------

[DELIMITED BY [ALL] {identifier-2 | literal-1}
--------- ---

[OR [ALL] {identifier-3 | literal-2} ] ... ]
-- ---

INTO {identifier-4 [DELIMITER IN identifier-5] [COUNT IN identifier-6] }
..
---- --------- -----

[WITH POINTER identifier-7]
-------

[TALLYING IN identifier-8]
--------

[ON OVERFLOW imperative-statement-1]
--------

[NOT ON OVERFLOW imperative-statement-2]
--- --------

[END-UNSTRING]
------------
