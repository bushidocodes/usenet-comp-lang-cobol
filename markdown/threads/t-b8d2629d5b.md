I can't think why the AS400 would care from a decimal point of view.  The
only thing I can think of is the data is unformatted on the AS400 and this
is being treated as a string or part of a string, and is used in matching..
In that case obviously X'1C' is a different character to X'1F'.

If this is true then you have to fix it on the OS390.   I can't remember for
sure but I think that when a numeric string is first moved to a packed
decimal field it has an 'F' sign and only gets the 'C' sign if arithmetic is
done on it.  If this is the case, you might be able to fix it by moving the
field to a pic 9 field and back again.

If you had the reverse problem, and had to change the field signs from 'F to
'C',  just add zero to each field.

Rick
