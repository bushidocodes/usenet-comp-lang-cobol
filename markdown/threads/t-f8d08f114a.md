### Unix COBOL Compilers: mbp vs. MicroFocus

```
For 4-5 years now, the company that I work for has been using mbp's
Visual COBOL for various Unix platforms (at least the ones that they
have ported to....). A couple of years ago, we had to make use of the
MicroFocus 3.1 compiler on a Stratus FTX machine and also a Tandem
Unix machine. We currently support both platforms.

Our current COBOL applications were originally written or based on code
written on an NCR mainframe. With the realization of the limitation of
a proprietary OS and the future of NCR, the port to Unix began.
(BTW: All of this is before my time....)

Initial Unix compilers reviewed offered strictly interpreted performance.
Also, most lacked the use of a COMP-6 data type as used in the application
(compress binary??? -- to get it with MicroFocus, we an RM-something
switch).

mbp's compiler proved to be the best solution. It offered COMP-6 use as
needed in the code as well as a "true compiled & linked" executable that
was compact in size and fast.

With the need to use MicroFocus, we lacked the COMP-6 data type as needed
until the 3.1 version. Still, we found that, with our Makefiles as
written,
our "executable" programs that we produced were *VERY* large in size. As
it turns out, this is because when creating a MicroFocus executable that
runs when you type "a.out", all the run-time library gets linked in with
the application. This results in a program compiled with mbp being 60k in
size, for example, will being greater than 1M when compiled with MF.

We were informed that the solution was to link our objects into the MF
run-time creating a new "run-time" that would make a better use of memory
(one copy of the run-time running at one compared to the other where X
number of run-times are in memory for X programs running). This would be
major undertaking on our part -- so we decided to stay as is and live with
the large MF exeutables and someday create a second set of MF-specific
Makefiles

Anyway, within the last year MicroFocus bought mbp's Visual Compiler from
EDS-Germany. With this acqusition, technology between the 2 companies
is to be merged. However, we're afraid that this is simply a way for a
larger company to "do away with" a smaller but better competitor. As far
as were concerned, mbp has the better compiler (MicroFocus has a lot of
nice utilites lacking the mbp side -- but the end result is better for
our use).

I'd like to know the following:

1) Who else out there uses mbp's COBOL Compiler as well as possibly both?

Have had similar experiences?

2) Is MicroFocus going to improve their compiler to create a better
executable program?

3) Are our fears warranted or not?

4) Any other compiler that does what mbp's does? We tried AccuCobol when
we needed to port to Stratus, but it was interpreted and ran *TOO*
slow for our needs. Has there been any improvement here?

We would hate to see the fine product from mbp be removed from the market.
Furthermore, we would hate to be stuck with the current MicroFocus
alternative
(not that we would not do the work to change our Makefiles but that the
creating new MF run-time solution does not thrill us as well).

Any replies, suggestions, & comments would be appreciated.
```
