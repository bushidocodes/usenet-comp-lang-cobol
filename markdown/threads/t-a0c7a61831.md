Look at the REVERSED option (for "single reel" sequential files) on the
OPEN statement. READing the file after it has been OPENed REVERSED will
present the records in backwards order.

This is a little used feature, and may be a performance turkey. Also it
is marked as OBSOLETE in the 1985 standard document, which means that it
is planned to remove this option from the next (199? 200? ? ???) standard.

Since "software does not rust", and "programs live forever", one might
consider using sort to resequence the file (adding a key to the records,
then sorting backwards on the key), as a better long term solution.

Liz,

I don't know how it is in MF Cobol, but I have used and still using
RM/Cobol85 V5.2 and Microsoft Cobol V4.5 on PC and Cobol85 V1.2 on a
SIEMENS mainframe. All of these support the Cobol85 feature

START ... LESS THAN ...

for indexed files. Using this, you can position the record pointer
to the previous one and then do a READ ... NEXT ... . This way you
will never have to sort records and you can browse forward and
backwards anywhere and any time you need to.

Example:

PERFORM UNTIL ...
START ... LESS THAN ...
READ ... NEXT
AT END ...
END-READ
.....
END-PERFORM

This may be somewhat slower than reading straight ahead, but it works.
Hope it helps.
