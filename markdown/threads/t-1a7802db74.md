```
I am at a customer site and we are putting together a roadmap for
Cobol upgrade. We have some very large applications that share
programs / subroutines. Rather than forcing these applications to
migrate and test together, we thought that we might be able to
determine the shared programs, update those programs first. If all the
programs that use these subroutines can still function correctly, then
we can have the applications migrate individually.
The question is, if we upgrade the shared programs / subroutines, will
the programs using these continue to work? If a change is made in an
application, using these subroutines, will that still work? Lastly, if
the subroutine is exectued dynamically, will it contine to work?
The applications will be on previous versions of COBOL and the
Subroutines will be on COBOL v4.
Thank you
```
