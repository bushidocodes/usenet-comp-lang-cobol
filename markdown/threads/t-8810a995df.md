If the external DLL is another PowerCOBOL form (which was linked as a DLL),
use this syntax:

INVOKE POW-SELF 'CallForm' USING  (entryform name) (dllname)

such as:

INVOKE POW-SELF 'CallForm' USING 'MYENTRY' 'SUBPGM.DLL'

If TEMP-1 and TEMP-2 in your examples are data parameters, consider making
them GLOBAL EXTERNALs

On another matter:

You will note I specified "CallForm" using proper casing. Beware! Using all
caps in a Fujitsu PowerCOBOL INVOKE will drop you in the ditch unexpectedly.
For example:

     INVOKE LV-BOX 'ADDSTRING' USING DATANAME

moves up to 64 bytes(!) to the Listview Box. Whereas

     INVOKE LV-BOX 'AddString' USING DATANAME

moves up to 8192 bytes (the length of DATANAME).

What's more insidious are those methods/properties that return a value. For
example:

01  NUM-LIST   PIC S9(7) COMP-5 VALUE HIGH-VALUES.

     MOVE "LISTCOUNT" OF LIST-BOX TO NUM-LIST

could be WAY different from:

     MOVE "ListCount" OF LIST-BOX TO NUM-LIST.

Here's the reason for the above oddities: All CAPS implies 'backward
compatibility mode.' In the first example, ADDSTRING only moves 64 bytes. In
the last example, the return is a 2-byte COMP-5 value (S9(4)) so the last
two bytes of NUM-LIST are unchanged from whatever garbage might be left from
some prior activity.
