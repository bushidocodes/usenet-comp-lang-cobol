The F switch switches on/off checking for valid numeric data. ie. if you
have space characters in a pic 9 field, you get an error. The best course
of action is to fix your program so that it initializes fields properly
but that isn`t always an option for some applications. 'Many weeks of
bugs' sounds like a reason to do the work though.

If you want to switch the F switch on and off from inside a program you
could do it using the call by number routine x"91" function 13. This is
documented in your COBOL System Reference manual.

›
› p.s. We use MF COBOL and we compile to EXE's if this is any help

If you're using EXE's, beware that you should really put any data-items
used by this call in the first 64K of Data Division. If you are using
LCOBOL linked EXE's, you have to call by a literal rather than a data
name. ie.

CALL x"91" rather than

CALL x91-func where x91-func is x"91".

Hope that helps.

Shaun.

You can set any of the runtime switches from within your COBOL
application by using the X"91" function 13 Call-By-Name routine
(which is documented in the Call-By-Number chapter of the COBOL
System Reference manual). That call allows you to read the 26
switch bytes and reset any byte, or bit of a byte (for those
switches like B1 or B2, which mean that when the particular bit
of the switch, 1 or 2 in this case, is on, those switches are
set).

Regards, Alan.
