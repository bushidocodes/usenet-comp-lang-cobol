I just tried this in IBM mainframe COBOL II (using COMP-3, the

So I'd hope it would work in your COBOL.
Based on that hope, I suspect that the data in amount-in is corrupt. Try

I am assuming from your message that the program compiles OK, but abends

Other things to look for - ensure the program has OPENed the file
containing amount-in; make sure the record was READ from it, make sure
it's not past the end-of-file.

record layout matches the file record layout.
Also be sure the output file is OPENed for OUTPUT before the move
statement.

when you 'can't' move it.

You don't explain why you "can't move the data into the amount field" (Does
a hand come out from the screen and smack you when you try?), so I don't
know if you're getting a syntax error or just don't see the data in the
output. If it's the latter, the explanation below is probably your answer.

The first problem I see is that on some compilers, such as MF COBOL, LINE
SEQUENTIAL is a CR/LF delimited text file. You should never store COMP or
PACKED type fields in those type files, because it can easily result in
control characters which can confuse the I/O routines.
