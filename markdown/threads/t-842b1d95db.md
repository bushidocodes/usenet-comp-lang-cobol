It is highly unlikely that you will find any ODBC type methods to access
the files without first having the original file layouts to aide in defining
the 
tables(record definitions) such that a generic flat file ODBC engine 'might'
be able to extract the data.  If there are any packed or  binary fields then
you run into even more difficulty as the data storage may not be easily
mapped to an ODBC type int/long/....
This type of question continues to come up in monthly cycles.  You might 
want to search the newsgroup from some archive location for more immediate
access to other potential solutions.
