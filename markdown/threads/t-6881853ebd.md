I'm using the API NetRemoteTOD to get the time of the server so that I can
sync my client with the server.  I'm having a bit of trouble in setting up
the bufferptr:

My code is something like this
working storage.
  01  TOD.
         05  TOD-ELAPSEDT    USAGE DWORD.
         05  TOD-MSECS       USAGE DWORD.
         05  TOD-HOURS       USAGE DWORD.
         05  TOD-MINS        USAGE DWORD.
         05  TOD-SECS        USAGE DWORD.
         05  TOD-HUNDS       USAGE DWORD.
         05  TOD-TIMEZONE    USAGE LONG.
         05  TOD-TINTERVAL   USAGE DWORD.
         05  TOD-DAY         USAGE DWORD.
         05  TOD-MONTH       USAGE DWORD.
         05  TOD-YEAR        USAGE DWORD.
         05  TOD-WEEKDAY     USAGE DWORD.

procedure division.

           MOVE "wright" TO COMPNM-SERVER.
           STRING "\\"          DELIMITED BY SIZE
                  COMPNM-SERVER DELIMITED BY SPACE
                  x'00'         DELIMITED BY SIZE
                  INTO SERVER-NAME.

           CALL WINAPI "NetRemoteTOD" USING
                BY REFERENCE SERVER-NAME
                BY REFERENCE TOD
                RETURNING NETRETCODE.


The SDK says this about the API:

NET_API_STATUS NetRemoteTOD(
  LPWSTR UncServerName,
  LPBYTE *BufferPtr
);

Parameters
UncServerName
Pointer to a Unicode string containing the name of the remote server on
which the function is to execute. A NULL pointer or string specifies the
local computer.
BufferPtr
On return a pointer to the return information structure TIME_OF_DAY_INFO is
returned in the address pointed to by BufferPtr.

This bufferptr is not the same as the other API's ...can anyone point me the
the right direction in setting this up?
When I run it it steps through the API fine bu nothing in the TOD buffer and
a 53 in the return-code...which I have no idea what that means.

Anyhelp would be appreciated.

Tom Wright
twright@larimore.net
```

```
Error code 53 is "The network path was not found." Looking at the Win32
SDK docs I think I can see a couple of problems. The SERVER-NAME
parameter needs to be Unicode rather than an ANSI string (that's why
you're getting error code 53) and the memory for the TOD structure is
allocated by the NetRemoteTOD function and returned to you in a pointer
variable. I believe that once you've finished with the structure you
would deallocate the storage using the NetApiBufferFree function. 

I've been unable to test this as the Net functions only work on NT and
I'm running Win98 but I think your program should look more like:

       working-storage section.
          01 TOD-PTR                   USAGE POINTER.
          01 NETRETCODE                PIC X(4) COMP-5.
          01 COMPNM-SERVER             PIC X(256).
          01 SERVER-NAME-LEN           PIC X(4) COMP-5.
          01 SERVER-NAME               PIC X(256).
          01 WIDE-SERVER-NAME          PIC X(512).
          01 WIDE-SERVER-NAME-LEN      PIC x(4) COMP-5.

       linkage section.
          01  TOD.
                 05  TOD-ELAPSEDT    USAGE DWORD.
                 05  TOD-MSECS       USAGE DWORD.
                 05  TOD-HOURS       USAGE DWORD.
                 05  TOD-MINS        USAGE DWORD.
                 05  TOD-SECS        USAGE DWORD.
                 05  TOD-HUNDS       USAGE DWORD.
                 05  TOD-TIMEZONE    USAGE LONG.
                 05  TOD-TINTERVAL   USAGE DWORD.
                 05  TOD-DAY         USAGE DWORD.
                 05  TOD-MONTH       USAGE DWORD.
                 05  TOD-YEAR        USAGE DWORD.
                 05  TOD-WEEKDAY     USAGE DWORD.

       procedure division.

           MOVE "wright" TO COMPNM-SERVER.
           STRING "\\"          DELIMITED BY SIZE
                  COMPNM-SERVER DELIMITED BY SPACE
                  x'00'         DELIMITED BY SIZE
                  INTO SERVER-NAME.

           MOVE 0 to SERVER-NAME-LEN
           INSPECT SERVER-NAME TALLYING SERVER-NAME-LEN FOR
                       CHARACTERS BEFORE INITIAL x"00".

           *> Get size of buffer in wide characters
           MOVE LENGTH OF WIDE-SERVER-NAME TO WIDE-SERVER-NAME-LEN
           DIVIDE 2 INTO WIDE-SERVER-NAME-LEN

           *> Convert ANSI string to Unicode
           CALL WINAPI "MultiByteToWideChar" USING
                 BY VALUE 0 SIZE 4
                 BY VALUE 0 SIZE 4
                 BY REFERENCE SERVER-NAME
                 BY VALUE SERVER-NAME-LEN
                 BY REFERENCE WIDE-SERVER-NAME
                 BY VALUE WIDE-SERVER-NAME-LEN
                 RETURNING WIDE-SERVER-NAME-LEN.

           *> Get size in bytes and double null terminate the string
           MULTIPLY 2 BY WIDE-SERVER-NAME-LEN
           MOVE X"0000" TO
                WIDE-SERVER-NAME(WIDE-SERVER-NAME-LEN + 1:2)

           CALL WINAPI "NetRemoteTOD" USING
                BY REFERENCE WIDE-SERVER-NAME
                BY REFERENCE TOD-PTR
                RETURNING NETRETCODE.

           SET ADDRESS OF TOD TO TOD-PTR

           *> DO YOUR STUFF WITH TOD STRUCTURE

           *> Deallocate the storage
           SET ADDRESS OF TOD TO NULL
           CALL WINAPI "NetApiBufferFree" USING
                    BY VALUE TOD-PTR
                RETURNING NETRETCODE.
