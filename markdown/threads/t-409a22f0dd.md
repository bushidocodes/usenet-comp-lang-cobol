Hi all,

I'm using MF-COBOL and need to store the length of an area in a
AREA-LENGTH PIC 9(4) COMP-4.
using
MOVE LENGTH OF AREA TO AREA-LENGTH.

The target field 'AREA-LENGTH' must be a 2-byte field in COBOL-byte 
order (in C would say unsigned short, net-notation).

This is no problem if the length of area is less 10000, but what do I if 
it's longer, say 32000?

Moving X'7d00' into FILED(1:2) works but that's not what I want.

I did not find a way to define a unsigned short in cobol.
Thanks for hints,
Wolf
```

I can think of three ways.

1. The compiler directive NOTRUNC, which will affect all arithmetic.

2. Move the value to a larger item, say PIC 9(8) COMP-4, then
    move the two lower bytes using reference modification.
    MOVE TEMP (3:2) TO AREA-LENGTH.

3. Use PIC X(2) COMP-X, for which the TRUNC diretive has
    no effect.
```
