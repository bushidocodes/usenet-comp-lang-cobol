```
> Is there another way?

Yes.  I suspect you want to use a loop, in which case you may choose
something like this  (adjust the picture sizes to fit your machine and
compiler):

 05  chars.
     10  char1     pic x(02).
     10  char2     pic x(02).
 05  nums          pic 9(05)   comp.  redefines chars.

You can fill up a table of these and then display char2.   If you start
using signed nums, you can have trouble.

But when I did this, I just edited a file using a hex editor.  (I have
also done so with ebcdic).  It was easier, especially since I needed to
discover which characters did funny things to my display or print. 
Remember some characters (LF) won't display correctly.  On my charts, I
had to edit those characters out.

Also, different printers will display the same characters differently.
```

```
Be ready to adjust not only the PIC, but the USAGE as well.  COMP means
different things on different compilers/platforms.  What you want here
is a binary representation.  In COBOL 85, BINARY might do the trick.
Also note that you may be able to use either CHAR1 and/or CHAR2 above,
but not necessarily both, because of the way the binary representation
and signs work on the compiler/platform.  And leave out the '.' between
COMP and REDEFINES. :-)
```
