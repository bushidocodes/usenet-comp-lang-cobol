This is very similar (identical, so far as I can see) to a COM (OLE) 
VARIANT structure.

Net Express ships with two copybooks called "mfole.cpy" and 
"vartypes.cpy". The former contains the possible VT- types and the base 
VARIANT typedefs (VARIANT and VARIANTARG) which reuse the (re)definitions 
in the latter. 

You could either use mfole.cpy/VARIANTs directly, or do something similar 
for clarity, eg:

copy "mfole.cpy".
01 PROPVARIANT typedef.
  03 PROPVARIANT-vt VARTYPE.
  03 PROPVARIANT-reserved1  pic xx comp-5.
  03 PROPVARIANT-reserved2  pic xx comp-5.
  03 PROPVARIANT-reserved3  pic xx comp-5.
  copy "vartypes.cpy" replacing ==()== by ==PROPVARIANT==.
..
(If you're not using Net Express, vartypes contains lots of redefinitions, 
such as:

         03 ()-data      PIC xxxxxxxx.
         03 ()-VT-UI1        REDEFINES ()-data PIC x COMP-5.
         03 ()-VT-I1         REDEFINES ()-data PIC s9(2) COMP-5.
         03 ()-VT-I2         REDEFINES ()-data PIC s9(4) COMP-5.
         03 ()-VT-UI2        REDEFINES ()-data PIC xx COMP-5.
One for each type}
..
01 MQMSGPROPS typedef.
  01 cProp  pic x(4) comp-5.
  01 aPropIdPtr  pointer. 
  01 aPropVarPtr  pointer. 
  01 aStatusPtr  pointer. 
..
01 myMessageProps MQMSGPROPS.
01 myPropVariants PROPVARIANT OCCURS 5.
..
set aPropVarPtr of myMessageProps to address of myPropVariants(1)

Note that you need to use pointers for C arrays, and set the value of 
those pointers to the addresses of the arrays themselves.

(Alternatively, you could follow the MSMQOLE demo shipped with Net 
Express, which demonstrates sending to MSMQ using OLE automation rather 
than MQ API and C structures. :-)

Hope that helps.

Steve.
