I am using microfocus cobol on a HP-UX platform and
I have the following problem :

I have some problems in using copy files in my cobol
programs. If I have a statement like this

copy /usr/local/cobol/cpy/mycopy.

This statement is translated to uppercase letters.

COPY /USR/LOCAL/COBOL/CPY/MYCOPY.

On unix machines file names are case sensitive so the compiler
cannot find it.
What compiler flag should you I give to use lower case
letters, so it can find the copy files.

Hi Johan,

If you place quotes round the file name the compiler will keep
the case of the copy books.

You don't say what version of the compiler you are using, but
there is a directive FOLDCOPYNAME(case) which can fold the copy
name to either lower or upper.
Example: cob demo.cbl -C "FOLDCOPYNAME(LOWER)"
Hopefully now the compiler will find your copy files.

Steve.
