I don't know what Telon stands for, but it is a cross between a code database
and a code generator. It can be used to store 'standard' sections of code
and 'custom' sections of code from which you can generate a COBOL programme
by defining it to use certain sets of the standard and custom code. This allows
you to keep common code once in the database and maintain it there.

Telon also offers support for scren design. It is used here to generate the
MFS source and screen description code for online modules to be run under
IMS.

There is a PC version of the product as well as the mainframe version. PC
version is by Computer Associates.

That's about the limit of my knowledge as I haven't really used the product
much

Matthew James
