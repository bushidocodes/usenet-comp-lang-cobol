The draft Standard *does* introduce USAGE POINTER (and PROCEDURE-POINTER). It
also introduces the ALLOCATE and FREE statements. Finally, it also introduces
"based tables/data-items".

All of this CAN be used in OO programs - or not.  Those who "hate" doing this
stuff explicitly will NOT be forced into using it.  However, those users who
interface with any API's or other facilities that expect them, will find the
support in the new Standard.

As far as those who "hate this stuff" (and/or find it error prone) and end-up
having to maintain someone ELSE's code that uses it, I have no solution for
you.
