Mario,

I don't use Dialog System so go to Micro Focus Answer Exchange and post
your question under Net Express. I'm pretty certain there are some DS
demos that cover Listbox contents.

Briefly how I do it using Dialog Editor and GUI classes - which wont be
dramatically different from Dialog System. As you indicted a separate
sub-class to create the Collection.

1. Set parameters to create Collection (ofReferences ???)
2. Initially create collection - read in records from COBOL file or DB
Table (Cursor Select)  and extract info to include in your collection
elements - do this with the "add" method.
3. Within your Dialog you have a control already designed for the Listbox
- so pass the Collection with :-

invoke YourListBoxControl "setcontents" using os-TempCollection

This is your list of EXISTING records.

4. If editing records, depends upon your design. Assume you allow user to
select EXISTING from the Listbox or key in an entirely new record. When
PB-OK, for safety's sake WRITE/REWRITE to your file or INSERT/UPDATE for
SQL.

5. Now you want to redisplay the 'changed' Listbox, (strings below are
object references) :-

(a) Add -
invoke os-TempCollection  'add' using ls-string (obj ref)

(b) Change -
       invoke os-TempCollection "replaceAll"
               using ls-string1 ls-string2 ls-index1 ls-index2

(c) Delete -
        invoke os-TempCollection "removeAtIndex"
                using lnk-index returning ls-string

Now pass the updated collection to your Dialog with :-

invoke YourListBoxControl "setcontents" using os-TempCollection

That's a 'potted' version - but go to Answer Exchange for more support.
