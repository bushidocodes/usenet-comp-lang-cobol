```
> COBOL I/O error 94, 40 REBUILD\PCLABOR\PCHEADER

That seems to indicate that the Cobol system you are using is RM Cobol
- Ryan-Mcfarland. This message is the standard format for that
implementation. The '94' file status error has a qualifier of '40',
which is how RM does indicate file-status errors.

The actual error indicated by this is that for that file the primary
key has an 'invalid or mismatched key offset'.  It seems that there is
a mismatch between how the program specifies the keys and how the file
has recorded how the keys should be specified.

This may be that the data file has been corrupted somehow so that this
information does not match.  It would be necessary to know exactly how
the program requires the file to be written in order to know if it can
be fixed.  It may be that if you move the file elsewhere (delete it in
that directory) the program will recreate an empty file.  If it does
this then it may be possible to recreate enough data (by keying it in)
to get that part working again.

The reindexing program is RECOVERY in RM systems:

RECOVERY datafile outputunrecoverablefile

But if the key information is corrupt then I don't know how to fix
this.
```
