```
I am trying to use what is described in TIM (Tandem Information
Manager on CD) as an intrinsic function. There are apparently
about 48 of these intrinsic functions. The manual says that
these can be used anywhere a data item or an identiifier is used
as the sending part of a move statement. The syntax (as best as
I can tell - ie this compiles) is as follows (using the function
CURRENT-DATE):

MOVE FUNCTION CURRENT-DATE TO WS-DATE-STRING.

However when on attempts to run this code you will get an
undefined external message and inspect (debugger) will take over
from there. No where in the manuals can I find example code as to
how these function work ini COBOL. They can be called from ANY
language on the TANDEM platform and should return a value
consistent with how they are definded in the TIM manual (COBOL85
manual under development services).

I am obviously doing something wrong here, but I'll be damned if
I can find anything in the manual to suggest exactlyhow to use
these functions. If anyone is familiar with the Tandem platform
(and thier lovely COBOL "extensions") please help me with this.

Thank you in advance for your help.
```

```
Damon,

I have successfully got the CURRENT-DATE instrinsic function to work in a
Tandem COBOL85 program. I defined the following 01 level in the
EXTENDED-STORAGE SECTION:

01 CURRENT-DATE-TIME-AREA.
05 CURR-YEAR PIC X(4).
05 CURR-MONTH PIC X(2).
05 CURR-DAY PIC X(2).
05 CURR-HOURS PIC X(2).
05 CURR-MINUTE PIC X(2).
05 CURR-SECOND PIC X(2).
05 CURR-HUNDRETHS-OF-SECOND PIC X(2).
05 GREENWICH-MEAN-TIME-IND PIC X(1).
05 GREENWICH-MEAN-TIME-HOUR PIC X(2).
05 GREENWICH-MEAN-TIME-MINUTE PIC X(2).

In the PROCEDURE DIVISION, I coded the following statement:

MOVE FUNCTION CURRENT-DATE TO CURRENT-DATE-TIME-AREA.

I compiled the COBOL85 program with ?ENV COMMON, but I see no where in the
COBOL85 manual where this should not work with the program compiled with
?ENV OLD. Also I have not tested CURRENT-DATE-TIME-AREA defined in the
WORKING-STORAGE SECTION instead of the EXTENDED-STORAGE SECTION.

Hopefully this will help with your problem.
```

```
To use the function CURRENT-DATE, you must compile your
program with the ?ENV COMMON compiler directive -- even though
the Cobol85 manual doesn't say so. This is documented in the
D42 SoftDoc for Cobol85, but I don't think earlier versions of the
SoftDoc mentioned anything.
```
