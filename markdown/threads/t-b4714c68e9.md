Hi.

You might want to consider just using the CBL_EXEC_RUN_UNIT call
instead. In NetExpress, this can be used to run system executables as
well as COBOL programs.

> The COBOL/API documentation from MF is rather poorly. Does anyone know of
> more extensive documentation? The standard API-documents usually provide
> only C-examples.

Remember that the APIs themselves don't 'belong' to MF/MERANT - it would
be unreasonable to expect the COBOL documentation to document all the
OS APIs. Unfortunately for the COBOL programmer, most OS API's are
documented in C or C++ terms only.

> But how do I declare pointers and booleans from COBOL?

01 native-pointer USAGE POINTER.
01 native-function-pointer USAGE PROCEDURE-POINTER.
01 c-boolean PIC X(4) COMP-5.

The last one is not neccessarily correct - it depends what 'boolean' is
defined as in the C code (it's a typedef, not part of the C language).
The above example assumes it's just another name for a 32-bit unsigned
integer, native byte ordering.

Cheers, Kev.
