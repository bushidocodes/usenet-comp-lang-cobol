On an IBM MVS mainframe :
PIC S9(9) comp is a 32 bit word equivalent to a long int on the PC
PIC S9(4) comp is a 16 bit halfword that corresponds to a 16 bit int on a
PC.

Now for the qualifications...
Your data types must be difined by you as INT32 and INT16, you cannot rely
on any C implementation to use any set size for an int.

The bytes of the data will be in a different order. Mainframes are
Big-endian machinces, the PC is a Little-endian creature. The order the
bytes are in are different accross different machines and you may need to
reoder them to get correct results. Look for a good library of routines
to do this unless you like writing this sort of thing yourself.
