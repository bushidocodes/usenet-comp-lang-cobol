This list is from Ann Wallace, convenor of ISO/IEC JTC1/SC22/WG4.

Attachment 1 ------------ COBOL Working Draft 1.0
Summary


Organization of the document

A major reorganization has been done by removing the module concept;
only one separate module exists -- the communications module, which
is optional and is now declared obsolete. The report writer is
incorporated as a required feature; it was previously an optional
module. There are currently no subsets, although it is possible
subsets will be added later.

New features

These are the major features incorporated in the draft; a complete
list of substantive changes is given in Annex L.

1. Object orientation -- static and dynamic objects, multiple
inheritance, non-conforming reuse, properties, parameterized classes,
polymorphism, and COBOL system classes.

2. Exception handling -- a commom method for handling all exceptions
by declarative procedures (the USE statement) and a compiler
directive for turning exception checking on and off.

3. A bit data type and bit string support

4. National character support -- datatype support for a multiple-
octet coded character set, which can be ISO/IEC 10646-1 Universal
Multiple-Octet Coded Character Set (UCS) or any other multiple-octet
coded character set.

5. Extended identifiers -- a compiler directive for specifying the
use of additional characters in user-defined words in a source
program.

6. Standard arithmetic -- an option for obtaining portable and
predictable results for specified arithmetic operations using a 19-
digit floating point intermediate data item. Native arithmetic is
the default and is upwardly-compatible.

7. Free-form source as an option; fixed-form source is the default
and is upwardly-compatible.

8. Dynamic table extension -- tables that are dynamically extended
as needed during program execution

9. Sorting of tables -- tables as input and output operands to the
SORT statement.

Anticipated additions

The following are major features considered likely to be included in
the Committee Draft, although this is not certain; the attachment of
in-progress proposals is indicated by an * preceding the item in the
list:

1. Improved COBOL and cross-language calling -- enhancements to the
CALL statement to facilitate the calling of programs written in
other languages or other dialects of COBOL. Will include the ability
to check that parameters are of the appropriate type.

2. I/O support for national character sets, including coded
character set conversion from and to ISO/IEC 10646-1 UCS-2 on input
and output, respectively.

3. Enhancements to report writer

4. File sharing and record locking capability

5. Some elements of internationalization - use of WG20's ordering
standard if done in time; use of local conventions for collating
sequence, currency sign, order of keys of indexed files.

6. *Currency sign enhancements (document X3J4/94-0490)

7. *Known Errors Annex -- a list of known errors that will not be
corrected for compatibility or other reasons (document X3J4/95-0076)

8. *A VALIDATE facility -- a data-description-driven method of
verifying data (document X3J4/95-0066)

9. *Collating sequence support for national character sets and
dynamic collating sequences in general (document X3J4/95-0084)

10. *Type declarations (document X3J4/95-0091)

11. *A screen handling facility -- local ACCEPT/DISPLAY statement
screen manipulation based on X/Open XPG4 COBOL (document X3J4/95-
0105)

[There are about 50 other changes on the list. Such as recursive
programs, constants, EXIT PERFORM, floating-point data items,
variable-length strings, conditional compilation, using any
characters for editing, hex literals, READ PRIOR and START <,
pointers, etc. -- for a complete list contact me (Don Nelson)].
