```
You don't tell us, but I'll bet that this is Micro Focus COBOL.  The 163
run-time message means that you forgot to initialize or re-initialize a
numeric field - so it has "non-numeric" data in it.

HOWEVER, as this happens on the second record - but not the 1st, my other
"guess" is that your input file is a "text" file (or as we say in Micro
Focus COBOL "line Sequential")

If you haven't (and your input is a "text" file), then add

   Organization Line Sequential

in the proper place in your Select/Assign statement - and the 163 will
probably go away.

(There are other ways to specify line sequential - and other ways for
getting rid of 163's - but this is most likely where the problem is for a
"student in a beginning COBOL class")
```
