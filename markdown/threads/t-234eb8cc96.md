```
We have a batch program which reads from a sequential file, browses the AIX
of a VSAM file, decides whether to write or rewrite and then writes or
rewrites the record on the VSAM file via the base cluster ( as opposed to the
AIX ). A problem occurs whenever one of the records on the sequential file
exists which would cause the program to re-read via the aix the same CI as
a record that has just been written ( as opposed to rewritten ). A status of
93 ( file or resource in use ) is returned from the aix read after the
write.
To rephrase the problem, the steps that cause the problem are

1) read sequential
2) browse vsam via aix
3) write record
4) read sequential
5) browse aix ( same CI as prior record ), browse returns status 93

Is there a timing problem, i.e. write is not finished before browse starts?
Is there a way to syncronize this? Any and all help would be appreciated.

The program is written in COBOL II and runs under MVS/ESA with an 8M region
size.

Northwestern University

```
