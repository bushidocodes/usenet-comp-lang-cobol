```
Here is a slightly "enhanced" version of the program that I just posted to
the NG.  This version includes comments and uses some additional functions
and extensions.  Again, for additional information on the product that uses
this as an IVP, see:
 http://www.spc-systems.com

            ***

       IDENTIFICATION DIVISION.
       PROGRAM-ID.  CXRIVP01.
      *
      *COBOL REPORT WRITER DEMONSTRATION PROGRAM.
      *THIS PROGRAM PRINTS A CALENDAR FOR THE CURRENT YEAR.
      *
       ENVIRONMENT DIVISION.
       INPUT-OUTPUT SECTION.
       FILE-CONTROL.
           SELECT PRINT-FILE ASSIGN TO CALENDAR.
      *
       DATA DIVISION.
       FILE SECTION.
       FD  PRINT-FILE
           REPORT IS CALENDAR.
      *
       WORKING-STORAGE SECTION.
       01  WORKING-LOCATIONS.
      *31DEC1899 WAS A SUNDAY. THIS AVOIDS NEEDING TO KNOW BASE DATE
           05  SOME-SUNDAY                    PIC 9(8) VALUE 18991231.
           05  NEW-YEARS-DAY                  PIC 9(8).
           05  NEW-YEARS-DAY-AN REDEFINES NEW-YEARS-DAY.
             07  YEAR-NO                      PIC 9(4).
             07  MONTH-DAY                    PIC X(4).
           05  START-DAY                      PIC 99 COMP.
           05  MONTH-NO                       PIC 99 COMP.
           05  DAY-COUNT                      PIC 99 COMP.
           05  MONTH-NAME-TABLE.
             07  FILLER                       PIC X(54)  VALUE
           " JANUARY  FEBRUARY   MARCH    APRIL      MAY     JUNE".
             07  FILLER                       PIC X(54)  VALUE
           "   JULY    AUGUST SEPTEMBER OCTOBER  NOVEMBER DECEMBER".
           05  FILLER REDEFINES MONTH-NAME-TABLE.
             07  MONTH-NAME                   PIC X(9)  OCCURS 12.
           05  MONTH-DAYS-TABLE     VALUE "312831303130313130313031".
             07  DAYS-IN-MONTH                PIC 99  OCCURS 12.
           05  DAY-TABLE.
             07  DAY-NUM                      PIC 99 COMP   OCCURS 42.
      *
       REPORT SECTION.
       RD  CALENDAR
           PAGE LIMIT 60    FIRST DE 7.
       01  TYPE PH  LINE 2.
           05  COL 50           VALUE "CALENDAR FOR".
           05  COL 63           PIC 9(4)  SOURCE YEAR-NO.
           05  COL + 3          VALUE "(LEAP YEAR)"
                                PRESENT WHEN DAYS-IN-MONTH (2) = 29.
       01  MONTH-BLOCK TYPE DE  REPEATED 4 TIMES EVERY 30 COLUMNS.
         03  LINE + 4  COL 6    PIC X(10)   MONTH-NAME (MONTH-NO).
         03  LINE + 2  COL 1    VALUE "  S  M  T  W  TH F  S ".
         03  LINE OCCURS 6      VARYING SUNDAY FROM 1 BY 7.
           05  COL 2  OCCURS 7  STEP 3  VARYING DAYNO FROM SUNDAY
               PIC Z9  BLANK WHEN ZERO SOURCE DAY-NUM (DAYNO).
       01  TYPE PF.
         03  LINE.
           05  COL 35           "PRINTED USING COBOL REPORT WRITER ON".
           05  COL + 2          PIC <X(9)B<99B9999   FUNCTION MDATE.
      *
       PROCEDURE DIVISION.
       CALENDAR-START.
           MOVE FUNCTION CURRENT-DATE TO NEW-YEARS-DAY-AN
           MOVE "0101" TO MONTH-DAY
           IF   FUNCTION MOD (YEAR-NO 4) = 0
           AND (FUNCTION MOD (YEAR-NO 100) > 0
                          OR (FUNCTION MOD (YEAR-NO 400) = 0))
                MOVE 29 TO DAYS-IN-MONTH (2).
           COMPUTE START-DAY = FUNCTION MOD
                 (((FUNCTION INTEGER-OF-DATE (NEW-YEARS-DAY)) -
                   (FUNCTION INTEGER-OF-DATE (SOME-SUNDAY))) 7)
           OPEN OUTPUT PRINT-FILE
           INITIATE CALENDAR
           PERFORM PRINT-MONTH VARYING MONTH-NO FROM 1 BY 1
                                 UNTIL MONTH-NO > 12
           TERMINATE CALENDAR
           CLOSE PRINT-FILE
           STOP RUN.
       PRINT-MONTH.
           MOVE LOW-VALUES TO DAY-TABLE
           PERFORM STORE-DAY VARYING DAY-COUNT FROM 1 BY 1
           UNTIL DAY-COUNT > DAYS-IN-MONTH (MONTH-NO)
           GENERATE MONTH-BLOCK
           COMPUTE START-DAY = FUNCTION MOD (START-DAY 7).
       STORE-DAY.
           ADD 1 TO START-DAY
           MOVE DAY-COUNT TO DAY-NUM (START-DAY).
```
