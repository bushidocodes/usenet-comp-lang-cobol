Wow, thats a lot of code for a simple thing. The REXX equivilant goes
something like this

/*Rexx*/
Arg Full_Name
Num_Words = Words(Full_Name)
Last_Word = Word(Full_Name,Num_Words)
Rest_Of_Name = SubWord(Full_Name,1,Num_Words-1)
Return Rest_Of_Name Last_Word

If the Last_Name is something like 'III' or 'JR', then repeat passing
just Rest_Of_Name, or parse into three fields.

Assembler is the *long* way of doing anything. Here is a slightly
shorter piece (than another example posted):

START 0
BALR 3,0
USING *,3
B PROCESS
* WORKING STORAGE
FIELD1 DS CL40
FIELD2 DS CL40
FLD1LEN DS CL2
FLD2LEN DS CL2
NAMEIN DS CL40
ENDNAME EQU *-1
TRTBL DC 256X'00'
BLK EQU X'40'
*
SEARCH TRT 0(0,10),0(9)
MOVE MVC 0(0,6),0(9)
PROCESS EQU *
LA 9,BLK(TRTTBL)
MVI 9,BLK
LA 9,TRTBL
LA 10,NAMEIN
LA 5,FIRST
LOOP LA 11,ENDNAME
SR 11,10
EX 11,SEARCH
AH 1,=H'1'
LA 10,0(1)
CLI 0(10),C'&'
BE LOOP
BR 5
FIRST BAL 5,LOOP
CLI 1(10),C' '
BNE NOMI
LA 10,1(10)
B MOVENAME
NOMI SH 10,=H'1'
MOVENAME LA 9,NAMEIN

LA 1,0(10)
SH 1,=H'1'
LA 6,FIELD1
SR 1,9
EX 1,MOVE
AH 1,=H'1'
STH FLD1LEN
LA 11,ENDNAME
LA 9,1(10)
SR 11,9
EX 11,MOVE
AH 11,=H'1'
STH 11,FLD2LEN

Ok, so it isn't really shorter, but it is complete. You now have your
title, first name and middle initial (if present) in field 1, and your
last name in field 2. Your last name can be more than one word with
this routine, but you must have a title. You also have the lengths of
field 1 and field 2 stored in fld1len and fld2len so you may pass them
back. Now all you have to do is decide how the name is getting to the
routine, and what you are going to do now that it is split up.
Or......

You can do your own homework and do it in COBOL (which is much

If it is guaranteed that the surname is the last single word
then it is relatively simple, just step from the far end
skipping spaces and then searching for the space before the
last word. Using FUNCTION REVERSE allows INSPECT for
'LEADING SPACES', then INSPECT using reference notation
for initial space will bound the last word. Inverrt the
boundries thus found (because the string was reversed)
to use as reference notation into the original string.

It may be just as easy to use PERFORM VARYING FROM limit BY -1
and reference notation or subscripting.

However, the real problems occur where the names are of the
form:

Dr. E H Fipps MBE

If I understand correctly, the situation is that the 'last word' always goes
in field 2 and all previous 'words' go into field 1, you could use a subscript
on the original field. Set it to the max length of the original field, then
subtract 1 repeatedly, checking ORIGINAL-FIELD-BYTE (SUB-1) for SPACE until
you find a non-space. (Of course ORIGINAL-FIELD will need to be REDEFINED as
ORIGINAL-FIELD-BYTE OCCURS n.)
Save the location (L1) of that non-space. Then keep subtracting 1 until you
find another space. Save that location (L2) also.
Then move forward with the subscript from (L2 + 1) to L1, moving each byte
into the bytes of field 2 (which requires its own subscript).

Assuming that 'Mr..' etc starts in byte 1, you can then start a subscript on
the original field from 1 up to L2 and you can use another subscript to get
the 'title and initials' into field 1.

If 'Mr..' etc. might start anywhere in the original field, then you'd need to
check for two adjacent spaces in front of the first non-space, somewhat more
complex and with several optional ways to do it.

You can also use indexes where I said subscripts. Indexes are usually a more
correct choice for processing performance.

The above won't work for names like 'Mr. J.L. van Dover, etc., where the 'last
name' is really two words.
