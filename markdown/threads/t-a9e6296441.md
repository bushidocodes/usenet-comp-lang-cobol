› select immisam
First problem:
Unless immisam.dat already exists, this should be
select optional immisam
but if immisam.dat already exists, your code will have many problems
(duplicate record keys...). The 'optional' is the ANSI/ISO means to do
an OPEN...I-O on a file that may not already exist.

›                assign          'immisam.dat'
›                organization    indexed
…[7 more quoted lines elided]…
›            end-read
Second problem: this should be
read immisam next at end continue
› perform until stveld = "10"
› [snip]
› read immisam next
And so should this:
read immisam next at end continue
›                end-read
›            end-perform
…[3 more quoted lines elided]…
›            end-read.

Of course, there are perhaps better ways to write this code (more
'bullet proof', e.g. OPEN immisam OUTPUT; write the records; CLOSE
immisam; OPEN immisam INPUT; read the records; CLOSE immisam, would not
be sensitive to a previously existing immisam), but your code is quite
easily understood. You have missed a subtle feature of COBOL--the
interaction of the various I-O verbs and USE procedures. The 'at end
continue' is necessary to specify how you wish to deal with the at end
condition; otherwise you are allowing the COBOL vendor to decide.
