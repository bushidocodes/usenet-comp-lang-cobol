Most probably. But it is possible to tell the compiler that the second (CALL 'PROGRAM')
also should be dynamic. In most situations (AFAIK them) this option (DYNAM) is NOT set, so
the difference between static and dynamic can be programmed by using CALL literal and CALL
identifier respectively. But you can not swap that reasoning if the option is set. (When
all frogs are green, it does not follow that every green thing is a frog)
