```
> Before you buy.

Once the unused allocation has been released to the system, you are stuck
with a total of 10 tracks.  Appending to your data set via the MOD parameter
will add records only as long as you don't exceed your primary allocation of
10 tracks.  Attempting to add beyond that point will result in an abend.  If
you expect to add onto an existing data set once it's been written to, I'd
strongly suggest using a secondary allocation initially or NOT using the
RLSE subparameter.  I'd recommend the former.
```

```
What happens (I think) is that your primary allocation of ten tracks has
already been made and the fact that you RLSE 3 tracks does not mean that
you can subsequently add 3 more tracks whenever you want (you have no
secondary allocation made so the system will attempt to mod onto the end
of the existing dataset within the existing allocation). If you do wish
to add the three tracks back for MOD then either remove the RLSE or code
SPACE=(TRKS,(10,10)). Remember that you have only got one primary
allocation and upto 15 secondary allocations (theoretical maximum of 160
tracks). If you use SMS then don't be surprised if fragmentation of the
disk packs and the movement of datasets by SMS results in the use of all
extents to store volumes of data less than the theoretical maximum (e.g.
SMS could result in only 16 tracks being allocated).
```
