```
Good Morning,

When we sell our system to a new customer they will often provide us with an
ASCII file containing their student information.  However, usually every school
gives us a different format so I an allways writing translation programs.  What
I would like to do is have one generic ASCII translation program that I could
use.

What I am looking for is an ASCII translation tool that our support people (non
programmers) could use that would take all these different ASCII file formats
and translate them into the one generic ASCII format that is supported by my
COBOL program.  They could use this tool to specify what the incomming ASCII
file looks like and then they would output to my ASCII format.  This tool would
have to be able to run from a command line (i.e. TRANSLATE UNIQUE.ASC TO
GENERIC.ASC).

Is anyone aware of a tool like this ?

Thanks,
Bob Hennessey
```

```
Bob:

One way I have handled this problem in the past is to use MS Excel and MS
Access.  MS Excel has an import feature that lets you manually define the
input file.  The imported file is then stored as a spreadsheet.  The spread
sheet then can be imported into MS Access - from which you can create a
query/report that meets your new specs and saved as a report that can be
read by your COBOL program.

Other possible variations - not yet tried, but in theory possible:

1.  OCDB interface to connect directly to the excel spreadsheet
2.  You may be able to import the ascii file directly into MS Access -
bypassing the first step
3.  It seems to me that Fujitsu included a file manipulation tool in their
free compiler that allowed this kind of job to be accomplished.  I don't
know if it is still available in their latest versions (free or commercial).
4.  It may be possible to use VB Script to automate anyone of these methods

These last four are just brainstorming - so try at your own risk.

Floyd Johnson
```

```
Bob,

One solution that I have used for a similar problem is to import the data into one
or more database tables.  I use MS Access because it (at the time) was the only
database product available to me.  Access has a rather straight forward "import"
wizard that simplifies this process.  Once the data is imported, there are at least
two methods to get access -

    - use ODBC (from the language of your choice) to access your data
    - create a query or report that can be read by your COBOL program

It might be possible to automate this process using VB Script - involving a minimal
amount of programming in order to define the input and output fields.

You might also look at the Fujitsu COBOL product.  Though it has been a while since
I looked at the full product, it seems to me that they use to include a file tool
that could do what you are asking.

Neither of these are, as provided, simple user processes.  But with some scripting
and assuming the files are clean (they always are, aren't they), it may be possible
to make it so.

Good luck,
Floyd
```
