You might consider using ETK.

ETK was developed by SEMA in Europe, but is now marketed by TOSC
(3900 Essex Lane, suite 250, Houston TX 77027; fax: 1(713)961-4989;
phone 1(713)961-1201 (Ted Langlie)).

What does ETK do? Allows you to develop FULLY portable COBOL
applications running on any platform (PC, Unix, MVS,CICS, Vax, AS/400,
etc). The specifications for the application is portable to any platform
and ETK then generates native code for the target you specify.
The native code is highly readable COBOL code.

The screens can be text based or (without change) GUI based. The GUI
screens allow your application to communicate with any other application
on the PC supporting DDE and OLE.

You can mix ETK generated programs with existing (legacy) programs.
