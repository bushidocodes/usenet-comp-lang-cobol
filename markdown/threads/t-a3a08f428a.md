```
One big advantage of the container ship is that labor costs are low when you
can go from truck to train to ship without unloading and loading everything a
bunch of times.  But the container isn't really necessary to do the job - it
is just efficient.

Let's say I want to download a big file to my PC.  I could use a 300 BPS
modem, I could even write an algorithm so that I could use a bunch of PCs with
300 BPS modems.  But things would be easier if I had broadband access.  If I
had to do this daily, I would look into buying a T1 line.

Most mainframes don't NEED to be mainframes.  Sure there are some "cargos"
which cannot be moved any other way - but most mainframes are doing things
which NT farms could do.  But the businesses which are running them are very
interested in cutting costs, so they make studies and go for the cheapest
solution with the required security, throughput, maintenance costs, etc..   If
NT farms were cheaper for their needs, there wouldn't be mainframes.

It is true that Wintel machines are getting more powerful.  But it is also
true that our data needs are growing even faster.
```

```
We have a small AS400 on our network at our Community College and 
it allows Students to sign on from home over the network via telnet, 
write source code from home, and be able to receive their output 
without even coming to school.  It is also able to be used as a 
web server and has networking support built in for TCP/IP or AIX.  
When it is backed up properly it usually only goes down when 
the School's network makes it unreachable.  I have seen this computer
run for several months with no down time.  We have more problems with
the printers than the AS400.  Since each user can have their own outque,
even when the printers are down they can still keep working.  It is a 
pretty dependable piece of hardware.  The only drawback is it takes 
some specialized training to set up and get running properly; Of Course 
so does a network.
```
