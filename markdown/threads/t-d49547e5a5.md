```
Hello,
I am new to Cobol and i am currently trying some freeware Cobol compilers
runing under Win98.
I found Fujitsu Cobol 3.0 and Cobol650 on the Web.
However i have problems using any of those Compilers:

1) Cobol 650:  When linking the programs, i get the message
'(FCB)-Steuerdatei fehlt', which could translate into 'FCB control file
missing'.
The message is probably a Dos/Windows message rather that a message from the
Cobol650 Compiler.  Could anyone give me a hint how to set up the Cobol650
on Win98, so that i dont get the previous message ? Maybe the Dos-Version
delivered with Win98 is incompatible with Cobol650 ?

2) Fujitsu Cobol 3.0 : I really like that program. However when it comes to
executing the programs, the 'Console-Window' reacts very strangely:
The Console Window seems not to accept the 'ENTER' key:
eg: i have a variable 'myvar' defined as PIC X(30).
then i make an input with the ACCEPT statment. (ACCEPT myvar).
When i execute the program, it wont accept the user's input until 30
characters have been typed ! I cannot type a small word of 5 characters, for
example, and hit return to finish the input. The console window simply wont
react to the 'Return' -key. (It will move the cursor to the next line, but
it wont terminate the input.)
(Imagine i have some variable like PIC X(300), i would need typing 300
characters if i want using that variable with ACCEPT !!)
Is there a way to configure Fujitsu Cobol 3.0, so that the console Window
recognizes the 'Return' -key correctly ? (so that i can terminate my input
by hitting 'Return')

Any help is much appreciated,
Claude Kaber
```
