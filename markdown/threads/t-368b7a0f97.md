X'02BB'
0 2 B B =
0*4096 + 2*256 + B*16 + B =
0*4096 + 2*256 + 11*16 + 11 =
0 + 512 + 176 + 11 = 699

X'02BF'
0 2 B F =
0*4096 + 2*256 + B*16 + F =
0*4096 + 2*256 + 11*16 + 15 =
0 + 512 + 176 + 15 = 703

Note however that you don't actually have to _do_ anything when reading a
COMP field since it is _already_ in binary.

Ian D. Kelly wrote:
If I have "O2 BB" how do I convert that to decimal.

COMP is Base 16 depiction of numbers.

right-to left, the first digit = 1's (16 to the 0 power), the 2nd digit
= 16's (16 to the 1 power), the third 256's (16 to the two power), etc.

In Base 10, A = 10, B = 11, C = 12, D = 13, E = 14, F = 15

02BB = (0 x 16 x 16 x 16) + (2 x 16 x 16) + (B x 16) + (B x 1) =
0 + 512 + 176 + 11 = 699.

› I however, can't find an explanation for the "comp" format.

COMP is the most efficient mathematical representation for that particular
processor. On IBM mainframes (e.g., MVS, VM, VSE), COMP is binary. On other
machines, COMP could very well be other representations (packed decimal, zoned
decimal, etc.). Also, it is possible that it may be different between different
vendors of a compiler on some platforms.

There is a "myth" (which used to appear in some IBM manuals) that COMP is

for this. For example, a lot of PC compilers when they are emulating IBM

environments. The only "official" requirement for COMP is that it be
numeric.

› hmm. so if you move from one type of COMP to another type
› of COMP does it translate the values appropriately?
› or does it do a direct bit for bit move?

Within a single COBOL program, MOVEing an elementary numeric field to
another elementary numeric field will always generate a type
conversion. If you move a COMP-3 variable to a COMP or COMP-5 variable
the values will be translated appropriately.

The conversion could result in high-order and/or low-order truncation,
depending on PICTURE size or format limitations.

If either the source or target field is a group item, the MOVE will
default to the rules for an alphanumeric move, at least on the platforms
I'm familiar with.
