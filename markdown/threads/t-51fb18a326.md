I have a program at work (mainframe COBOL/370) that attempts to read a
variable length VSAM KSDS. The program has an 01 level record under the FD
statement equal to the maximum record length, 4000 bytes in this case.
I've determined that the requested record does not exist in the file.
Instead of getting a file status 23 (RECORD NOT FOUND) I get a file status
04 (invalid record length for fixed-length file). I've even put the
secondary file status on the FD statement, but the function and feedback
codes come back with all zeroes. Has anybody else encountered this
particular situation? The AVGLRECL is 75, the MAXLRECL is 4000. Thanks for
any and all help on this frustrating problem,

John
```

It sounds like you've coded the FD in such a way that the compiler has
assumed the file is fixed-length. You might try something like this:

FD MY-FILE
RECORDING MODE IS V
RECORD IS VARYING IN SIZE FROM 75 TO 4000 CHARACTERS
DEPENDING ON WS-MY-FILE-LRECL.

01 MY-RECORD PIC X(4000).

Remember to define WS-MY-FILE-LRECL in WORKING-STORAGE as an unsigned
elementary numeric field. The actual length read is returned following
a successful READ, and you would set WS-MY-FILE-LRECL to the length you
want to write before any WRITE or REWRITE statement.

I hope that helps.
```

They're not JCL, they're VSAM cluster definition terms. When AVGLRECL =
MAXLRECL, you have a fixed length record dataset (cluster). When
AVGLRECL < MAXLRECL, you have a variable length record cluster. Just for
the sake of convenience, when you have AVGLRECL > MAXLRECL, you have a
syntax error (or some other flavour of error).
