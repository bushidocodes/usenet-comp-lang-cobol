The short answer on this one is, sure you can do just about anything, 
providing you are familiar with the APIs. You sure you want to 'torture' 
yourself  :-) . So a lot depends upon what support you have in Acu, to 
get at methods/properties, or what is available in the Active-x tool.

For example, Net Express has its own OO Class Datepicker - producing a 
calendar you can view backwards/forwards - but only three methods 
specific to 'Datepicker' :-

        "Getddmmyyyy",     "getDefaultAPIflags"  and   "Setddmmyyyy"

So if I wanted to do anything fancy, I would have to search up through 
the GUI hierarchy for colouring cells etc., and it is quite possible I 
might for certain features have to do API calls rather than Micro Focus 
invokes.

# 1  - you have to determine which "cells" you want coloured - and this 
for EACH month as the user clicks to go forwards or backwards. There are 
two parts to your question  :-

(a) Specific days of the week (e.g. Sundays) - probably using the COBOL 
function day-of-week. Purely off the top-of-my head, you could use a 
fixed table of 31 days or a collection/list indicating which cells are 
to be 'coloured' resulting from the 'function day-of-week'.

(b) Holidays - resulting info is stored in table/list above. Just one 
country or several ? You need to separately store a set of dates.

# 2 - can't see you having the ability to "display Xmas' in what is a 
mini-window, or 'Dialog', although we are calling it a calendar. 
Independently you would have to design "Xmas" or "Santa" as a separate 
bitmap - in all likelihood using the MS "ImageEdit.exe"  which you will 
find under \MsTools\bin. Having drawn it, then you need the 
instructions/methods/properties  to put an 'image' into a cell.

Colouring - you will have to have methods for setting colours using RGB 
values within the appropriate cells. (Do a dummy run on the ImageEdit 
above, using the 'Advanced feature' which gives you a palette with a 
spectrum of colours you can move around in until you select the colour 
setting that you like - and then store the three individual values for 
R, G and B). Bearing in mind you can zip at great speed through a 
'traditional' claendar, backwards or forwards displaying each month you 
encounter, your code will need to be slick and in very tight loops. In 
the case of  'Xmas' - that's an exception routine, not part of your 
normal loop, as are the National holidays.

These National holidays require some separate thought - probably holding 
a record for each element say in perhaps a sequential file. Try and 
store some of the info as fixed (e.g., the values of the three RGBs to 
be used), plus of course indicate those with fixed dates, July 4 and 
December 25. Others - USA Thanksgiving Day = fourth Thursday in 
November; Canada Thanksgiving Day = second Monday in October. If you 
want to include 'ecclesiastical' holidays like Easter, then you will 
have to go googling on Gregorian Calendars to determine how you 
calculate the date for this 'moveable feast'.

Go googling, checking out 'computer calendars',  'calendar components',  
MSN and the Knowledge Base for articles, plus there are sets of 'images' 
which will most definitely include 'Xmas' a Christmas tree or Santa, or 
Easter eggs - reasonably priced.

Can be done - challenging - let us know how you get on - whether or not 
you get there or fail..

Jimmy, Calgary AB
