Check Language Reference.

STOP RUN.... means as it states.. "kill myapp".
You should use GOBACK instead.
This way your "main"(Root) program and sub-programs ends normally.
In older days STOP RUN was a "undocumented" way for novice programmers to
abruptly terminate CICS...
(CICS translators scans the source for such mistaked nowadays).

So the moral is.... forget the verb STOP RUN, and use GOBACK..

Generally when STOP RUN is used it is an indication to terminate and return
to the operating system instead of progressing out via the caller where you
would still have some level off control. Note: GOBACK is also used to return
to the caller. You should only pass control back to the operating system
from the main program otherwise debugging will be difficult. I have not
seen the message itself but this problem exist even on mainframe
implementations.
