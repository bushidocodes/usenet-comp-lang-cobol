Hello,

I have a typical alternate collating sequence sorting problem.I would
appreciate your suggestions.please note that there is no scope for
another program to resolve this, but maybe some utilities should be OK.
And now to the problem statement.
My input file has 29 bytes record length.The sorting fields are at
position 1 (FLD-1) and at 11 (FLD-2) and both are of alphanumeric data
types.FLD-1 is 4 bytes long and FLD-2 is 5 bytes long.
Typical input looks like this :

COLUMN : 12345678901234567890123456789
E145......07100..............
A256......14111..............
G150......11111..............
M001......07100..............
C100......22222..............
H475......33333..............
B005......07100..............
0115......DEPTR..............
1234......CTRRT..............
0045......G0145..............
..
..
Z999.........................

Sort requirements for FLD-1 :
The normal sort sequence would sort any input from A-Z and 0-9.
But the desired sort is in alternate collating sequence.So, the numbers
should sort before alphas except those starting with M and H should sort
before A,B, etc.. i.e.,

0045....
0115....
1234....
M001....
H475....
A256....
B005....
C100....
E145....
G150....
etc.
Sort requirements for FLD-2 :
Numbers should sort before alphas as for FLD-1.But, those starting with C
and G should sort after D, i.e.,

..........07100
..........11111
..........14111
..........22222
..........33333
..........DEPTR
..........CTRRT
..........G0145
..
..
etc.
Note: There are no letters other than C,D and G in case of FLD-2 data,
and the sequence of C's and G's should not affect the desired of
FLD-1 data. The input file is generated by a previous STEP and the
following STEP needs the input to be sorted in the above said crazy
sequence.
Another restriction is, that the solution cannot contain writing
extra piece of COBOL code as it is beyond the scope of the project.
However, the PROC can contain one or two extra steps.

Good luck,

PV.
