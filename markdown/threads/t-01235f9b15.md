```
You would generally use START ... READs when you want to access a dynamic
file in a sequential manner. For instance:

If your key was:

01  KEY.
    03  KEY-1    PIC X(4).
    03  KEY-2    PIC X(4).
    03  KEY-3    PIC X(4).
    03  KEY-A.
        05  KEY-A1    PIC X.
        05  KEY-A2    PIC X(3).

Then if you wanted to read through all of the records starting from
KEY-1='0010', KEY-2='2000', KEY-3='1000', you would set those 3 keys as
shown, whilst setting the KEY-A to LOW-VALUES. You would then perform a
START function. Note that START doesn't actually READ anything for you. You
will need an initial READ directly after the START function. Once you've got
that, you can continue to read as normal, until you need to change position
on the file again, in which case you could do another START function.

```
READ  filename  will read exactly one record with the key value = to
whatever you set it.

READ filename NEXT  will read exactly one record, where the key is one key
higher than the last READ, or where a START verb has positioned the key.
START does not return a record.

So

   SELECT MyFile
     ACCESS DYNAMIC
     blah blah blah
     RECORD KEY IS My-Key


To get the record for Jones, either one of these will work:

METHOD ONE:
   MOVE "JONES" TO my-KEY
   READ MyFile

METHOD TWO
   MOVE "JONES" to My-Key
  START MyFile KEY NOT LESS My-Key   << you could use KEY EQUAL My-Key
  READ MyFile NEXT

You usually use START/READ NEXT when you want to read an indexed file
sequentially, starting someplace other than at the beginning. Let's say you
want to scan thru the "J" entries. You might use

    MOVE  MOVE LOW-VALUE TO My-KEY
    MOVE "J"  TO MY-KEY(1:1)
    START MyFile  KEY NOT LESS My-Key           << position the pointer with
the first "J"
    READ MYFILE NEXT                                       << read the first
record satisfying the key condition of the START
    PERFORM UNTIL MY-KEY(1:1) NOT EQUAL "J" << show values and loop until
you hit a key not starting with J
       DISPLAY My-Key
       READ MyFILE NEXT
    END-PERFORM
    DISPLAY 'KEY OF FIRST ITEM NOT STARTING WITH J IS'  MY-KEY


Add some error handling and give this a try. (There are more
efficient/prettier ways to code this, but this should give you the idea).

But: in your application, since you are only looking for a specific record,
you'll never use START and READ NEXT; you'll just use READ, as though the
file were ACCESS IS RANDOM. (DYNAMIC is just a combination of RANDOM and
SEQUENTIAL).
