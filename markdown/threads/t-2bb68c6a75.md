01 BAD-DATE PIC 9(8) VALUE 19991301. *> Bad Month

>> CONSTANT SIMPLE-DATE-TEST IS 0

>I don't know C; but isn't what you've described the essence of a case
>structure, i.e., where you fall thru one case after another until you
>either get a hit or you fall out the end?

In C, in a switch statement, one can have multiple case statements labeling
statements to be executed. However, unlike COBOL, when a matching case
statement value is encountered, all statements after that point are executed
(even statements for subsequent case statements) until either the end of the
switch construct is encountered or a break statement is encountered.

In COBOL, in an EVALUATE statement, one can have multiple WHEN statements
labeling a set of imperative statements, but (unlike C) when the last
imperative statement associated with a WHEN (or a series of consecutive WHENs)
is executed, control goes to the end of the EVALUATE statement, maing a "break"
statement unnecessary, but also making it impossible to drop through to the
next set of imperative statements. (Actually, I prefer COBOL's way because the
C way can easily lead to more statements being executed than one had
intended--just by forgetting to code a "break" statement. Also, COBOL is more
flexable in what is allowed in the EVALUATE expression and WHEN expressions.)

You can have multiple WHEN clauses to perform the same imperative statements,
e.g.,

 EVALUATE WSWA-PRIORITY
    WHEN 'HIGH'
    WHEN 'H'
        some code (will be executed for both HIGH and H
    WHEN 'MEDIUM'
    WHEN 'M'
        some other code (will be executed for both MEDIUM and M)
    WHEN OTHER
         code for when none of the above WHENs match WSWA-PRIORITY
 END-EVALUATE

If you want a "throw-away WHEN", the imperative statement CONTINUE can be used,
e.g.,

 EVALUATE WSWA-PRIORITY
   WHEN 'HIGH'
   WHEN 'H'
   WHEN 'MEDIUM'
   WHEN 'M'
   WHEN 'LOW'
   WHEN 'L'
      CONTINUE
   WHEN OTHER
       PERFORM PRIORITY-ERROR-ROUTINE
 END-EVALUATE
