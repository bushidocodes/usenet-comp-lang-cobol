```
My organization is still in its infancy with using MicroFocus
COBOL. We are a mainframe COBOL shop attempting to use MF COBOL
as a mainframe offloading tool.

Currently, we are looking for a way to archive PC COBOL code
that has been downloaded from the mainframe and vice versa. In
essence, we need a COBOL source version control system (something
similar to PANVALET, LIBRARIAN, or SCLM on the mainframe). The
package must be able to interact between the mainframe and PC
platforms (this also includes a Novell network).

Can anyone out there offer us any assistance in what a good choice
of software would be? We will be looking at PVCS, CA's PAN-LCM,
and Legent's ENDEAVOR. Does anyone have any experience with these
packages? I would be interested in talking to you about various
issues regarding these packages.

Thanks in advance for your help.
```
