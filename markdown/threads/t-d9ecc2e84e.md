January 2, 2004

Norcom is pleased to announce the release of the GUI ScreenIO
Client/Server Layer.  

The Client/Server Layer enables existing GUI ScreenIO applications to
run across a TCP/IP network (including the Internet) without the need
to change your application.  This makes it very easy to
Internet-enable GUI ScreenIO applications.

The application itself runs on the server and only sends screen
updates to the client computer.  The client computer runs the GUI
ScreenIO Network Client software, which handles screen displays, file
transfers, and security.  The GUI ScreenIO application behaves exactly
as though it is running locally on the client.

Client/Server computing has many advantages, including better control
of your application software.  Because the applications run on your
server, it's only necessary to update the server's copy of your
software when you make changes.  You no longer need to distribute new
versions to all of your users.

Client/Server computing is also scalable; you can add servers and
communications bandwidth as your user base grows.  The GUI ScreenIO
Server software could also be ported to any hardware platform that
supports COBOL and TCP/IP; it does not require a Windows platform.
Contact Norcom if you wish to explore this this option.
