```
We are using DFSORT Release 13 for Y2K changes. Support for both the
external and internal sorts is required. We are having a problem with
programs contains more than one internal sort. We have been unable to
make this work with out program changes.

Our site has overridden the name DFSPARM with a site specific name. 

For a single internal cobol sort, no program code changes have been
required. For programs with more than one internal sort, we have found a
work around which requires program code changes related to sort control.
With out these changes, DFSORT fails to pick up the override sort parms
from the JCL stream. 

The DFSORT write up we have describes the procedure to use for multiple
internal sorts. We have not had success with the method.

Is anyone doing DFSORT 13 Y2K multiple internal sorts for two digit year
values without programn changes? If so, any assistance would be
appreciated.
```

```
I reread your post and it seems to me that the most likely problem is that
you are using DFSORT R13, but do NOT have PTF UQ05520 installed.  UQ05520 is
needed for the FREE=CLOSE support that the described procedure for multiple
internal sorts depends on.  With the FREE=CLOSE support, DFSORT will use the
first set of DFSPARM statements for the first internal sort, the second set
of DFSPARM statements for the second internal sort, and so on. (Without the
FREE=CLOSE support, the first set of DFSPARM statements will be used for
every internal sort.)

So please make sure that you have PTF UQ05520 installed.  If that's not the
problem, please send me more details offline.
```

```
I assume you're referring to the "DFSORT writeup" in my SORT2000
document.  You can find the latest version at URL:

http://www.storage.ibm.com/software/sort/srtmycbl.htm

It describes a method using DFSPARM with FREE=CLOSE.  As far as I know,
this method does work (I've tested it).  However, there are a couple of
caveats stated in the SORT2000 document:

1) JES3 does not allow the use of duplicate DFSPARM DD statements if the
DD statements request JES3 or jointly-managed devices.  So if you have
JES3, you may not be able to use this method.
2) If you're using DFSORT R13, PTF UQ05520 is needed for the FREE=CLOSE
support that the described procedure for multiple internal sorts depends
on.  With the FREE=CLOSE support, DFSORT will use the first set of
DFSPARM statements for the first internal sort, the second set
of DFSPARM statements for the second internal sort, and so on. (Without
the FREE=CLOSE support, the first set of DFSPARM statements will be used
for every internal sort.)  So please make sure you have PTF UQ05520
installed if you're using DFSORT R13.  (DFSORT R14 has this support in
the base.)
3) If your installation is using a different name for DFSPARM, you must
use that name instead of DFSPARM wherever DFSPARM is mentioned.

If your problem isn't caused by any of the above, then please send me a
note offline (fyaeger@vnet.ibm.com) describing exactly what you're doing
and the results you're getting and I'll try to help.
```
