```
Hello everyone.

I use COBTEST which is the COBOL debugger shipped
with IBM VS COBOL II accessed via ISPF option 4.10. The COBTEST restart
option allows the programmer to change and recompile programs between tests
without exiting COBTEST. However when I run COBTEST with DB2 programs I
must close down ISPF and exit to raw TSO so I can execute the DB2 "RUN CP
PLAN(XXXXXXXX)" command and then "ISPF 4.10' which brings back the ISPF
session and the COBTEST invocation panel. I am not a DB2 expert but it
seems that this process locks the plan and reserve it solely for the
COBTEST session. If I split screen and attempt to recompile the program
the compile fails at the bind stage with a resource not available message
and a code that indicates the plan is locked by another user. I get the
same result if I split screen and use the DB2 "FREE PLAN' command. I have
to close COBTEST, exit ISPF and then enter the 'END' command to exit DSN
out to raw TSO and then enter the ISPF command to restart my ISPF session
and recompile/rebind.

Does anyone know of a COBTEST option which unlocks the plan or a method of
rebinding programs which does not involve exiting COBTEST?.
```
