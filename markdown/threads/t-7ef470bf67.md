```
Ok,  I am working up a cobol ratings site.  I want to throw a design
out for critique:

My ISP will support Linux Perl and TCL CGI scripts.  No database and
no frills site.  I have never done this so a volunteer would be
appreciated but not necessary.  I suppose I should learn this stuff.

Design constraints:

1.  It must reside in sequential files and be low tech
2.  It must be automatic to add a response file
3.  Must run on my ISP (obviously).

There is a POST page that e-mail's me with the entry from the screen.
There are a series of CGI's that read files and generate themselves.

1.  There is a file containing a list of vendors and compilers

2.  There is a file containing each compiler and the OS it supports

3.  There is a subdirectory named after each compiler

4.  Each compiler file contains compiler.html which contain compiler
information (all OS's etc).

Vendor file:

1.  Vendor name   |
2.  Compiler name |
3.  Vendor web page

Compiler file

1. Operating system |
2. Compiler name

Compiler directory contains:

1.  Index file (basic compiler details)
2.  A series of response sheets formatted as HTML tables.

(*****  scripts ****)

1.  Script 1 will read the vendor file and throw a simple page list
and point into the compiler subdirectory for INDEX CGI.

Script 2 will create a form allowing a drop down of unique operating
systems and a second drop down with Vendor.  Drop boxes have {ALL} as
default.

Script 3 lists the results of the above query.  With hyperlinks to the
various folders index files.

Script 4 will glue the index and the response files into a single page
for the return.  May have optional version / OS selection if I can
figure this out.

Post page:

.  Compiler id  (from drop down or new value)
.  Version
.  OS and version
.  Date used
.  Anonymous report (show email address?)
.  Elapsed time used
.  Support response (if any)
.  Good points
.  Bad points

*  I was considering requesting the real name of the reporter for
legal reasons, if I was challenged would I need to show the responses
came from real people or would email addresses suffice?

Anything else.  Put on your analysis hats.

Ken
```
