```
I have a programm that calls and cancels many other programms.
When using MF Workbench 3.4 or 4.0 I get several errors.
In 4.0 I get "routine table overflow". So I configure a COBOPT.CFG with
routine_table_size=1024 and it works.

But in 3.4 I get the error 157 Object file too large. I think, the
COBOPT.CFG has no function in 16 bit and I had some attempts with the
stack size (/S16000 up and down), but I still have the same error. Does
anybody has some ideas which other switches or configurations may help?
If I put the programs in a LBR, it works, but there are situations when
I don't want to have a LBR.

Frank Pohlmann
```
