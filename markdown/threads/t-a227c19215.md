```
First I'd like to thank everyone who helped with my CBL_ routines problem.
That's all solved now, many thanks.

I now have a different, very odd problem. The following paragraph is called
from an EVALUATE within a PERFORM UNTIL loop:

       400-JOINT-DEBTOR-INFO.                           *> F5
         IF DEBT-JOINT-DEBTOR = SPACES THEN
           MOVE "No Joint Debtor to view." TO COLLECT-HELP-LINE
         ELSE
           CLOSE DEBTOR-FILE  *> opened elsewhere
           <INITIALIZE PARAMETERS>
           PERFORM SET-LOCAL-FOLDER    *> Essentially just a wrapper for
CBL_CHANGE_DIR
           CALL "WDEBT03" USING   PASS-VARS
                                  SYSTEM-DETAILS
                                  PASS-NEXT-PANEL
                                  PARM-FLAG
                                  PASS-CODE
                ON EXCEPTION CONTINUE
           CANCEL "WDEBT03"
           PERFORM SET-NET-FOLDER    *> Another CBL_CHANGE_DIR wrapper
           OPEN INPUT DEBTOR-FILE
           <etc.>
         END-IF.

The problem is that when the sub-program call returns, the following code is
not executed. Execution returns to the EVALUATE in the main PERFORM loop
without completing the paragraph. This manifests as a "file not open to
close" error when the program is wound up. There are other similar calls
elsewhere in the program, called from the same structure and they all work
fine. I can't fathom what's going on. Could something in the called
subprogram cause this? Are there any punctuation traps I might be missing?
Help, please!
```

```
           CALL "WDEBT03" USING   PASS-VARS
                                  SYSTEM-DETAILS
                                  PASS-NEXT-PANEL
                                  PARM-FLAG

                ON EXCEPTION CONTINUE

to

           CALL "WDEBT03" USING   PASS-VARS
                                  SYSTEM-DETAILS
                                  PASS-NEXT-PANEL
                                  PARM-FLAG
                                  PASS-CODE
                ON EXCEPTION
                        CONTINUE
                Not On Exception
                       CANCEL "WDEBT03"
           PERFORM SET-NET-FOLDER    *> Another CBL_CHANGE_DIR wrapper
           OPEN INPUT DEBTOR-FILE
                .....

or at least I would GUESS that is what you wanted your program to do
```

```
All that stuff at the end (after the call) only happens on exception
conditions, the way I read it. "Continue" is just a no-op, it does not end
the exception statement in any way.  If you put an end-call after the
continue, I think it will do what you want.  A period will also do the
trick.
```
