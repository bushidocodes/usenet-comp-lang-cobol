One of my favorite books dealing with the ANSI-1985 COBOL standard was
written
by Jim Inglis and published by Wiley in 1989.  It was not a beginner's
book, but it
was extremely well organized and demonstrated COBOL as it ought to be
written.

Can we assume that this new book emphasizes fully-formed elementary
structures,
nested COBOL subprograms callable as protected modules via parameters,  the

many forms of the PEFORM statement (including illustrations where in-line
perform
is useful,  built-in functions,  multiple examples of the EVALUATE
statement,
reference modification, and other features that make it possible to
actually do
structured programming in COBOL.

In my view, any book that demonstrates an if statement without an end if is
leading
new programmers astray.   In fact, all of the statements in which an
end-form (READ,
SEARCH, etc.) should be encouraged with that end-form.

I suppose I need to order a copy of the book to find out for myself.
However, if someone
already has gotten a copy, it would be interesting to hear some evaluation
regarding the
conformance to structured programming.    So many COBOL books I have seen
fail to
properly demonstrate good coding style, including some of those touted as
most popular.

Lao Xiao Hai
