WITH DEBUGGING MODE does not produce output, it merely uncomments
lines of code. If those lines contain DISPLAY (or EXHIBIT or READY
TRACE), then extra output will result. Conversely, one can code
diagnostic DISPLAYs without DEBUGGING MODE.

On Unix, it is common practice to DISPLAY debug-stuff UPON SYSERR,
which keeps it separate from normal DISPLAYs.

In  shops where version control is an obstacle to recompilation,
programmers prefer to turn debugging on with an environment variable
or command line parameter.

Bill seemed to indicate earlier that the presence of an object-time switch
controlling whether the debug code was "activated" or not was an implementor
extension.  I think I'd take exception to that, because ANSI X3.23-1985
states ('74 is similar), on Page XV-2, in Section 1.3.3, "Object Time
Switch":

    "An object time switch dynamically activates the debugging code inserted
by the compiler.  This switch cannot be addressed in the program; it is
controlled outside the COBOL environment.  If the switch is on, all the
effects of the debugging language written in the source program are
permitted.  If the switch is off, all the effects described in the USE FOR
DEBUGGING statement on page XV-5 are inhibited.  Recompilation of the source
program is not required to provide or take away this facility.

    "The object time switch has no effect on the execution of the object
program if the WITH DEBUGGING MODE clause was not specified in the source
program at compile time."

DEBUG-ITEM and USE FOR DEBUGGING were marked obsolete in '85 COBOL and
dropped from the '02 standard, though WITH DEBUGGING MODE and
debugging-indicators were retained, on the grounds that implementations
typically provided interactive debug facilities that didn't require source
satements in the first place.   "Making the Debug module optional allows
existing implementations to continue offering the feature for compatibility
reasons, without forcing new implementations to provide a capability
grounded in obsolete technology."  (ANSI X3.23-1985 page XVII-86).   In all
honesty, I think I'd have argued against its obsolescence, had I been
involved in the standards process at the time.

    -Chuck Stevens
