Take a look at the STRING and UNSTRING statements and the phrase
DELIMITED BY SPACE within that statement.  E.g.

STRING "The word of the day is " DELIMITED BY SIZE VARIABLE-OUT
DELIMITED BY SPACE "." DELIMITED BY SIZE INTO OUT-BUFF.

If OUT-BUFF is large enough, it will contain "The word of the day is
WORD."

Another method I have used with AcuCOBOL is this:

77  SPACE-COUNT   PIC 99 VALUE 0.
77  VARIABLE-LENGTH PIC 99 VALUE 0.

INSPECT VARIABLE-OUT
     TALLYING SPACE-COUNT FOR TRAILING SPACES.
COMPUTE VARIABLE-LENGTH = 12 - SPACE-COUNT.
DISPLAY VARIABLE-OUT (1:VARIABLE-LENGTH).

This method will depend on your compiler's ability to use TRAILING in the
INSPECT statement, so consult your manual.  I haven't been able to find this
convention in the MicroFocus manual.

This also requires the 85 standard COBOL.

An alphanumeric move in COBOL is done from left to right.  If a longer
field is moved to a shorter field, the result is truncated to the length
of the receiving field.  If the receiving field is longer than the
sending field, the receiving field is padded with spaces.

Remember, this works for alphanumeric moves!  Both operands must be PIC
X.
