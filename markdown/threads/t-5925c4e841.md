```
C C wrote:
> 
> It was my mistake.  I had the following code which did not give me a compile 
> error:
>         ACCEPT TEST-SCREEN DISPLAY TEST-SCREEN TIMEOUT AFTER 180.

DISPLAY starts a new statement, so the TIMEOUT phrase would not apply
to the ACCEPT.

That looks like it ought to have failed to compile, since there's no
TIMEOUT phrase defined for DISPLAY. (Though there's no good reason why
not; DISPLAY can time out, of course, if the output device is not
local.) So that looks like a bug, albeit a minor one.

You don't say what MF compiler you're using, but I just tried this
with Net Express 5.1, and the compiler does indeed allow a TIMEOUT
phrase on DISPLAY. It changes the behavior of a simple DISPLAY from
ANSI to ADIS (as eg "AT LINE 1" would).
```
