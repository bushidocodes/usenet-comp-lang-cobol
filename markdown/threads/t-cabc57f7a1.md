> > Seriously, WHAT IS a COBOL Clone

> I suspect that it is repeated code patterns (even when slightly
different commands are used

There is a common coding practice of taking a copy of something
that works and making any necessary changes to create a new
program. This is sometimes known as 'cloning' (as in Dolly the
sheep).

The problem is that a maintenance change may well have to be made
to all the clones as well as the original.

This it would be desirable if (at least) all the clones could be
located and (preferably) could be merged into one.

The problems are looking through a large code base to find the
clones and in carefully comparing the clones to determine which
changes have been made. Both of these are potentially
automatable.

COBOL is not really relevant here except that the toolmaker is
addressing COBOL. Also, COBOL n (n < 2000) lacks some facilities
such as templates that could be used instead of cloning. But then
I suspect that programmers use cloning in C++ as well.

hello, and thanks to everyone that responded to the post. Sorry for not
responding sooner, but I read news at most once a day and I didn't expect
this many responses in such a short time. Martin's reply below is correct.
Clones are essentially repeated idioms that occur redundantly in the code
base. As Martin says, its quite common for developers to cut and paste some
existing code in several places, usually with some modifications. The
CloneDR finds these occurrences of the original idiom despite the
modifications, and automatically constructs a replacement abstraction
(currently a COPY lib, but we're currently working on replacing with called
sections and external procedures). The savings can be huge. We've found up
to 50% redundancy in some real Cobol code bases. And yes it happens in C++
as well though to a slightly lesser extent (more like 15%). Finding the
clones can also reveal maintenance bugs where one clone was patched but the
other occurrences were not. A more detailed explanation is available at our
web site www.semdesigns.com.

And once again, thank you all for your interest (and the DEVO references.
"Whip it good" is indeed our motto when it comes to clones)
