series of ADD, SUBTRACT, MULTIPLY, DIVIDE statements. The generated code is
more efficient. This will generally be true for other platforms.

If you just want to add 1 to a counter, it probably makes no difference
whether you use ADD or COMPUTE.

The major difference is that COMPUTE uses an intermediate data item
that is implementor-defined. ADD and SUBTRACT are always guaranteed
to give you exact answers. In every implementation I have ever heard
of if you use COMPUTE to just add and subtract you will get the same
answer as ADD and SUBTRACT statements. Where it gets tricky is when
you start using division and multiplication (and exponentiation).
Then all bets are off.

A simple statement like

COMPUTE a = (b / c) * c

can give different answers on different systems. Suppose all are PIC
999 and b has a 1 and c a 3. Some systems give you an answer of 0 and
some 1. What I suggest is that if you are going to use division use
it as the last operation in the expression. If you are going to
multiply big numbers use the MULTIPLY statement to control the
results.

There are complaints and whatnot about performance. The difference is
probably so small that you could ignore it.

As long as you are careful, COMPUTE is probably the best because it
reads better in most cases (except like ADD 1 TO b which is more
obvious than COMPUTE b = 1 + b). Just use common sense.
