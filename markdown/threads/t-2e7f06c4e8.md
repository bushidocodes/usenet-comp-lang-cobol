If you're using DOS/Windows/OS2 and Micro Focus COBOL, the following program
will tell you which drives are there. There isn't a way of working out which
are floppies, which are hard drives, cd-roms etc.

----- GETDRIVE.CBL ----------------------------------------------

special-names.
call-convention 4 is nrc
.
data division.
working-storage section.
78 pc-find-drives value "PC_FIND_DRIVES".
78 cbl-and value "CBL_AND".

01 drives5 pic x(4) comp-5.
01 drivesx pic x(4) comp-x.
01 char pic x.
01 char-x redefines char pic x comp-x.
01 special pic x(4) comp-5.
01 test-buffer5 pic 9(9) comp-5.
01 test-mask5 pic 9(9) comp-5.
01 stored-drive-char pic x occurs 26.
01 all-drive-chars redefines stored-drive-char
pic x(26).
procedure division.
main section.
call nrc pc-find-drives using drivesx end-call
move drivesx to drives5
move 1 to special
move "A" to char
perform until char-x > 91
move drives5 to test-mask5
move 1 to test-buffer5
perform test-routine
if test-buffer5 not = zero
move char to stored-drive-char (special)
add 1 to special
end-if
add 1 to char-x
divide 2 into drives5
end-perform
display all-drive-chars
stop run
.

test-routine section.
call nrc cbl-and using test-mask5
test-buffer5
by value 4
end-call
.

----------end------------------------------------------------

after that you have to parse the directory structures yourself using other RTS
calls which I don't have a program handy for to demonstrate. It's quite
involved.

Have a look at the COBOL System Reference manual and if you've got Workbench
installed, try the MFFINDER program which finds files and prints out a list.
You should be able to call it after displaying the correct command-line
parameters on the OS command-line and then interpret the output from a
redirected file. I haven't tried that but I don't see why it won't work.

Couple that with the above program and you should be able to come up with a
program that searches all drives.

On a PC you can use DOS Int 21h, functions 4Eh (Find First) and 4Fh (Find
Next) to read the entries in a directory.

The basic operation is to call Find First with a string (wildcards
permitted) that indicates what you are looking for. You can also specify
the file attributes as part of the search criteria. Find First will
retrieve the first filename that matches the search string; Find Next will
retrieve subsequent entries.

If the functions are successful, the carry flag is clear and the file
information is returned as a structure. Otherwise, the carry flag is set
and the AX register will contain 0002 (File not found), 0003 (Path not
found) or 0004 (No more files).
