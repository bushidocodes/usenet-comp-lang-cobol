Hello,

We are trying to get something done for a schoolproject... the idea is
to read some text in a variable (typically PIC X variable), and then
do some counting on that string.

If I define the variable like this:

01 MyVariable PIC X(1600)

And if I then do an accept of this MyVariable and write it back to the
screen it seems that it only accepted a patition of the text I typed,
for example if I would have typed :
"hello, world, how are you doing, this is my nice little nifty program

logic in programming else ?"

then it would only display the first 64 characters being : 

"hello, world, how are you doing, this is my nice little nifty p".

the cobol compiler is from CA-Realia.

I believe this is a limitation of ACCEPT on your compiler. IIRC the IBM
ACCEPT has a limit of 114 characters and this is probably different for
different compilers.

If your max is 64 characters then define an array or table containing 25
occurrances of PIC X(64)  (as 25*64=1600 and this is probably not a
coincidence!) then code a loop that accepts one occurance at a time until
the table is full.  Then process it.
