```
Besides what is in the FAQ (that Thane pointed to),  a few comments in
response.

A) There is no such thing as a "2000" COBOL Standard.  There *is* a draft
Standard that the current best estimates indicate will be approved at the
end of 2002.

B) A good "overview" (not quite current - but still useful) describing the
OO features can be found (in PDF format) at:
  http://www.adtools.com/info/whitepaper/oocoby2k.pdf

C) The current FULL draft Standard can be found from the web page at:
 www.ncits.org/tc_home/j4.htm
If you download that LARGE document, then look at the two appendices
  substantive changes - potential effecting
  substantive changes - not effecting
that will give you a complete list of the major changes in the draft
Standard.

D) There are a NUMBER of implementations that already support OO COBOL - as
an extension to the existing ANSI/ISO Standards.  Unfortunately, none of
them exactly match the current draft Standard - nor do any of them match the
others.

E) I notice that your email ID ends with ".de" - which as I recall is a
German address.  If so, you might want to contact the German COBOL group.  I
don't have an exact address to get to the whole group, but if you contact
Artur Reimann at:
  reimann <at> attglobal.net
I think he would be able to help you with any questions/issues that you
might have from a specific "German" perspective.
```
