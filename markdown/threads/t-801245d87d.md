The CANCEL statement does in fact cause a free() operation - However because
of the way that UNIX (including AIX) performs garbage collection the freed
memory does not actually return to the OS, instead it is available for
re-use by the same process. This memory re-use is only possible if you also
utilize the -l0 setting for COBSW.
What this means in practical terms is:1) You must set COBSW=-l0 and then the
CANCEL statement will allow memory to be re-used,  and improve memory
utilization and efficiency. If you do not use CANCEL no memory segment will
not be re-used. Memory doesn't actually get returned to the OS until the
process terminates.
