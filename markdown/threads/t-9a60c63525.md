BLANK WHEN ZERO  makes the field into edited numeric. Thus when you
move the edited numeric to a numeric it does a de-editing move, as
ANS'85 allows.

You will note that BWZ does not imply that when it is blank it should
be treated as if it has a zero value, though the de-editing will do
that, an ADD 1 TO such a field will still fail to compile. SPZERO will
allow an ADD 1 TO [numeric field of blanks] but not if it has BWZ.
