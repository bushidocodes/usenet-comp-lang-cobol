The IBM / SPC Report Writer Pre-Compiler works just fine. It can be used
as a shell to make VS COBOL II, COBOL/370 and COBOL for MVS & VM all look
like they have native report writer, or it can be used to compile out the
report writer, leaving only native COBOL.

Some structures which snuck thru OS/VS COBOL's syntax check are not
supported - all of these were out of spec with the standard, but produced
predictable results. The documentation for the pre-compiler explains
appropriate changes to fix this issue.

One minor issue that can come up is the ability to exactly compare the
output files produced by native (os/vs COBOL) and pre-compiled programs.
There is no specification for the exact sequence of records produced, only
for the visual output. Consider printing a header on line 1 of a page,
and the first detail on line 3. Several sequences of ASA standard print
records can produce the same image on the paper. Yet these will not
compare equal with traditional compare programs. A solution is available
for this minor issue (e-mail for info).
Rex Widmer
Builder of software archeology tools and other strange programs to help survive in a legacy based world.
