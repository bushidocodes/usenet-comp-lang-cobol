I have a customer that has many programs written in VAX Cobol and DiBol.
They want to move their data to Microsoft SQL Server on the NT platform and

Yes. This is possible using Liant's Relational DataBridge (tm)
product. Please see
http://www.liant.com/products/retooling/databridge/index.html for
further information.

With this solution, your COBOL and DIBOL programs continue to run on the
VAX platform, and continue to use the RMS file system. DataBridge
provides ODBC access to the data in the RMS files. SQL Server can use
ODBC to bring the data to the NT.

I am unaware of any product that would allow the existing VAX
COBOL/DIBOL programs to have access to an SQL Server database.
