Michael, Michael, Michael,

 In "ANSI standard COBOL" a COMP field can be ANYTHING the vendor wants.
Even a USAGE BINARY field can be stored in any way that the implementor
defines.  "Little-endian/big-endian" and almost everything else is allowed.
As (often) stated in this NG, there are even implementations where USAGE
COMP is stored as "packed-decimal" or "display" -  all of this is ENTIRELY
ANSI conforming.
