These questions concern use of the LINKCOUNT compile option
in the MicroFocus compiler version V3.2_7a.1a.11.09.

The operating system I am using is AIX. MicroFocus documentation
indicates that LINKCOUNT applies to all the operating systems.

For a given setting of LINKCOUNT:
I have found that when an EXTERNAL variable is used in
WORKING STORAGE, the number of allowable variables in
LINKAGE is reduced.

My guess is that when the EXTERNAL variable is used the
elementary variables in LINKAGE are applied to the limit,
and then the EXTERNAL variable is _not_ used the 01 level
variables are applied to the limit.

My questions are:
1) is this a known feature of mF COBOL?
2) is this a desired feature of mF COBOL?
3) what are the disadvantages of increasing LINKCOUNT
to overcome the decrease in allowed variables?
consider that in my production case i will have to
increase linkcount from the default 64 to 2200.
a) runtime performance.
b) compile time.
c) runtime size.

Here is my sample code. Note that linkcount is set artifically
low to make the problem appear in a small example.
---- cut ---- cut ---- cut ---- cut ---- cut ---- cut ---- cut ----
$set linkcount(5)

AUTHOR. LIFE INSURANCE DATA PROCESSING.

WORKING-STORAGE SECTION.

01 WS-AREA-1 EXTERNAL.
* 01 WS-AREA-1.
05 WS-DATA-1 PIC X(100).

LINKAGE SECTION.

01 LS1-AREA-1.
05 LS1-DATA-1 PIC X(1).
05 LS1-DATA-2 PIC X(1).
05 LS1-DATA-3 PIC X(1).
05 LS1-DATA-4 PIC X(1).
05 LS1-DATA-5 PIC X(1).
05 LS1-DATA-6 PIC X(1).
05 LS1-DATA-7 PIC X(1).
05 LS1-DATA-8 PIC X(1).

01 LS2-AREA-2.
05 LS2-DATA-1 PIC X(1).

PROCEDURE DIVISION.

MOVE '2' TO LS1-DATA-1.
---- cut ---- cut ---- cut ---- cut ---- cut ---- cut ---- cut ----

My understanding is the LINKCOUNT is only required if your program contains
file of data records in the Data Division that are marked as EXTERNAL.
Further that you use the LINKCOUNT directive to specify how many references
the link area needs to hold, which must not be less than total number of
linkage section data items, external data items and external files.

I think more simply put, the compiler cannot account for external items
which are not specifically in the linkage section, and you have to count
them.
