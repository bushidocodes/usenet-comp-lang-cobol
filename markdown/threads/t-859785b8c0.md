›       C10-CALCULATE-CELSIUS.
›           COMPUTE CELSIUS-LOW-WK = 5 / 9 * (LOW-TEMP-IN - 32).

›           COMPUTE CELSIUS-HIGH-WK = 5.0 / 9.0 * (HIGH-TEMP-IN - 32).

My only guess is that the compiler does not specifically recognize that
the value of '5 / 9' should be a floating point value of 0.55555555 and
instead calculates the integer value of zero. When zero is applied to
whatever is calculated by (LOW-TEMP-IN - 32) it will always result in
zero.

By using '5.0 / 9.0', this might tell the compiler to keep the floating
point result of 0.5555555555 to then apply to (LOW-TEMP-IN - 32).

Since you show OBJECT COMPUTER IBM-370, I assume that you are on a VSE,
VM, or MVS IBM implementation of COBOL.

An entire length appendix exists to describe the behavior of intermediate
results. In your case the 5/9 will always yeild zero due to truncation.
If you were not using literals, you could control the precision of that
critical result by the pictures of the variables, even if the values were
just 5 and 9 ... i.e.

01 five pic s999v99 value +5.
01 nine pic s999v99 value +9.

would yeild different results.

Avoid constants in COMPUTE, especially if Division is involved.
