well, before you convert the code, you have other issues:
a> is all the code in 1 language?
b> is that code in 1 language written for porting (no comp-5 fields)?
c>are you attempting to port it all, and if not, which parts are you
going to choose to port over another, and more importantly why?

if i were doing a porting project, i would port the parts which are
not needed 24/7, make sure they have UPS, mirror their drive to
another machine, ect. however, since you are talking about databases,
there are 2 questions:

a> are you porting for PC administrators who want as400 access? or
b>are you moving applications off the mainframe for local businesses
who want to maintain it themselves?
