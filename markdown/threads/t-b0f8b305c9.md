A do-nothing routine is not really a good test, as it does not replicate
what your working page set and so forth will be at execution time. It is
more like 'the absolute minimum hit' test.

We have had jobs where the change from dynamic (our default) to static
linking reduced wall clock time for a nightly run by as much as 75%. These
were jobs where the 'driver' program had about 10 called subs for record
types, each of which had a called sub for a report, and where all of the
above made, together, 100-300 calls to a date conversion routine for each
record processed.
