Hi Rob.

That's not strictly true - on Unix, at least, the system call will not run
asynchronously with the COBOL program *unless* you have an apersand (&) on the
end of the command line (what you are doing is calling the OS function system()
directly - that passes it's command line to a system shell which will interpret
the ampersand to mean run asynchronously. Lack of an ampersand gives synchronous
execution). If you are not seeing this behaviour it looks like there might
be a bug, so please can you send me a short demo program ? Thanks.

Also, FYI, Micro Focus COBOL also has a call "SYSTEM" (yes, it's case sensitive)
on Unix which is a wrapper around the OS system() routine which allows us to
make a note of the fact that another process has been executed. It should
therefore be used in preference to call "system" when, for instance, the COBOL
program calling it has any screen handling in it (we know that the screen
contents have potentially been trashed).
