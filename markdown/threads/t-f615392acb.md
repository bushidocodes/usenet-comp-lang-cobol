It depends what you mean by TSO output. 

1) If this means output from a program (i.e. via DISPLAY statements)
running under TSO, then you do exactly the same as what you would do on
the mainframe by issuing the following commands:

ALLOC F(SYSOUT)DA('USERID.MY.DATA') SHR REU
CALL MYPROG

The ALLOC command has the effect of associating the ddname SYSOUT with
the dataset 'USERID.MY.DATA'. By default under TSO, SYSOUT is associated
with the screen.

2) If this means capturing output from TSO commands, then, as you could
do on the mainframe, you could write a REXX EXEC that utilises the
OUTTRAP external function to capture command output into variables, and
then write these to a dataset via EXECIO.

3) If you just want a snapshot of the current contents of the screen,
you can right-click on the Application Output window and use the Edit
function to mark and copy the text to the clipboard. you would then open
up a new file in the editor and paste the contents of the clipboard into
it.
---- 8< --- Cut Here ------------------------------------------------

I hope this helps.
