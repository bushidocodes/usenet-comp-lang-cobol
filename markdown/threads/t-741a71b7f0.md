Don't think you can do quite what you want.  Your best option would be to use the AIB
(application interface block) option to access the database in your routine.   

The database PCB still needs to be in the PSB, but the main program doesn't need to be
aware of its presence if you use the LIST=NO option on the PCB, and give the PCB a name.

You're Cobol routine could then use the AIBTDLI interface to access the PCB by it's name,
rather than being passed it's address through the LINKAGE section.

This means that you can add a subroutine without having to change the entire calling path
to pass the PCB through.

Check out the application programming manuals or ask your DBAs for more help.
