I think you will have to expand on your problem a little to give us some
more clues. Is the compiler Net Express (University or Professional
editions) ? The only thing I can immediately offer, where you refer to '10
minutes to an hour', is that somehow your source file has got a glitch,
(perhaps some hidden character or text you can't see). Pure speculation
but that might cause the compiler to loop endlessly trying to check
something.

Based on speculation - put some *deliberate* error lines in your source,
which will cause the compiler to display errors. You can then 'watch' to
see where the compiler is perhaps 'pausing'. If you hit true frustration,
try 'chopping' the program up, say doing a compile just containing
Paragraph 1. If that comes out 'clean' now add the source for Paras 2 and
3 and do a fresh compile.  Extremely tedious - but might help you if in
fact a glitch in the source was the problem.

Coding in OO I've been totally mystified where a perfectly valid method
(paragraph/sub-program) has been rejected. Cut it out, compiles clean. Now
add it back in and it compiles clean again !

If truly stumped go to the Micro Focus Answer Exchange and ask there.
