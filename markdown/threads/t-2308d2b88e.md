```
Hi, I am posting this question on someone's behalf.

The company has engaged a software house to develop a system on AS/400 using
COBOL/400 & SQL/400. The system has come on production but performance is not up to
expection.

It was discovered that logical files are created to speed up the retrieval of the
required record; eg. logical file is created with new key creation_date (which
is not a key in the physical file). As a result, each physical file has several
logical files depending on the number of online function modules. I feel that there
will be side effects when the database grows.

Have anyone experience similar scenerios? Other viewpoints to this issue would be
appreciated before the software house is confronted. TIA.
```

```
› It has been my experience that some software vendors like to create
logical files for every little thing. The usually results in
performnce going down as each update to the physical files involved
require that the logicals also get updated. The large this tree
gets the longer it takes for the system to do it's job. I usually
find that limiting the use of local files to highly used online
functions gives much better performance.

Dan
```
