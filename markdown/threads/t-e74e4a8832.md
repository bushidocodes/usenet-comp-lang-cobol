You could always do it the same way as you do by hand.   There are several
ways of doing this.

Besides the way we learned in school, here's one you might program in:

One way would be to have a table of powers of 16.

16
256
4096
65536
1048576

Find the largest value which is smaller than your number to be converted.
Any number larger than this has to be zero:
1000 dec = 000??? hex
Divide this number into your number to be converted, saving the remainder:
1000/256= 3 r 232.  Look up the value of 3 in a table and find it equals 3
1000 dec = 0003?? hex + 232 dec
Repeat until done
232/16 = 14 r 8.   Look up the value of 14 in a table and find it equals E
1000 dec = 0003E? hex + 8 dec
Since no hex value is larger than 8, look up its value in the table and find
it equals 8
1000 dec = 0003E8 hex

This should be easy to code into a called subroutine or an object.

03 WS-PACKED    PIC 9(16)V9 PACKED-DECIMAL.
03 FILLER REDEFINES WS-PACKED.
        05  WS-FWORD PIC S9(09) BINARY.
        05  FILLER PIC X(05).
03 WS-DISPLAY    PIC 9(16)V9.
03 WS-DISPLAY-X REDEFINES WS-DISPLAY
                                    PIC X(17).

MOVE ZERO TO WS-PACKED.
MOVE +1000 TO WS-FWORD.
MOVE WS-PACKED TO WS-DISPLAY.

INSPECT WS-DISPLAY-X CONVERTING X'FAFBFCFDFEFF'
                    TO 'ABCDEF'.

* At this point, WS-DISPLAY-X is equal to '03E8000000000000'

HTH....
