Hi,

I work for Unidata (based in Denver, CO) so maybe I can shed some light
on our product. We make what we call a nested relational or multi-dimensional
database. UniData is the name of our database, UniSQL is our implementation
of ANSI SQL (not to be confused with the company called UniSQL.)

We have a number of tools as well (4GL, Report Generators, Client/
Server connectivity, etc.) including a product called COBOL Direct Connect
that allows you to migrate your COBOL application to an open systems,
relational database without forced data normalization or embedded SQL or
SQL gateways. Once you get into the database environment of course then
you have access to Client/Server tools (like ODBC access), and any of
the other tools as well.
