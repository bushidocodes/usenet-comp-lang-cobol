Since you mention CICS, it is probably a four-byte elementary item.
However, bear in mind that the size of a pointer is hardware-dependent,
so you shouldn't rely on it staying a particular size. A lot of code had
to be changed from three-byte pointers to four-byte pointers when XA came
along, and I wouldn't be surprised to see IBM introduce hardware with
eight-byte pointers. (Use the MAP compiler option to find out about
storage size.)

Whenever you use the value of a variable, you use the contents of the
hardware address that corresponds to that variable. For example, let us
suppose that the variable WIN-STATUS has the address X'FEDCBA92'. That
address contains "00" [PIC 9(2) VALUE 0] before your program starts.

The value of a variable used as a pointer is the address of another
variable. For example, let us suppose that the variable WIN-ADDR has the
address X'FEDCBA98'. If that address contains X'FEDCBA92', then we say
that WIN-ADDR "points to" WIN-STATUS.


› Another statement less important is 'COPY CFHAID'. Is this a standard
› copy book in CICS?

"DFH" is the IBM prefix for CICS. "DFHAID" is the standard CICS copy
book for the 3270 Attention ID. If "CFHAID" is not another typo, it is
probably a similar copy book for the C interface of the API.

› What is it's function?

The POINTER usage could vary in size depending on implementation. It's
intended to hold an address of a data structure that can be bound or unbound
during execution. In an IBM mainframe environment, POINTER is 4 bytes or 32
bits, and can be redefined as PIC S9(9) COMP, if you want to perform
calculations on it. Otherwise there is a limited set of operations you can
perform on a pointer data item.

In mainframe CICS you can SET ADDRESS OF linkage-section-record TO
pointer-name. That establishes addressability to a piece of storage using
the linkage section record as a layout. You can also capture the address of
a linkage section record by using SET pointer-name TO ADDRESS OF
linkage-section-record. You can also SET pointer-name TO NULL. Pointer data
types can be compared for equality, but not for greater than or less than.
You cannot MOVE a pointer to some other data item. You can move or perform
arithmetic on the redefinition of a pointer, but the results of
arithmetic might vary depending on your implementation.

›
› Another statement less important is 'COPY CFHAID'. Is this a standard copy
› book in CICS?

I think CFHAID must be specific to your CICS C API. There is a standard CICS
copybook named DFHAID, which contains mainframe codes for Attention Interrupt
values generated by a 3270 terminal, such as DFHENTER, DFHPF1, DFHPF2, et
cetera.

› What is it's function?
USAGE IS POINTER indicates a field in working storage in which an
address pointer will be stored (pointer data item). This item can be
used in a compare, a SET (pointer) statement or the using phrase of a
call statement (to pass the address of an area to the called program),
by default this include the entry statement in the called program. It
occupies 4 bytes of space.
›
› Another statement less important is 'COPY CFHAID'. Is this a standard copy
› book in CICS?
DFHAID: 'DFH' the prefix all almost all CICS copy books, 'AID'
Attention Identifier. This copybook contains values returned in EIBAID
(Execution Interface Block Attention Identifier) field. Used to
determine what key the operator pressed to return control to the
program. Enter, PF1, Clear, etc.
