First the 'V'. V in a Picture clause defines an assumed
decimal point usually on an input record. You may have
an input field that looks like:
05 input-percent pic 999V99.
The data could be for example 09450. The decimal point is
assumed to be between the 4 and 5.
If wish to print this, you could set up a print area such as:
05 print-percent pic zz9.99.
05 filler pic x value '%'.
When printing you do put the '.' because you want it to appear
on the report. The 'V' tells cobol where to position the decimal
point. A 'V' would never be used in combination with a zero-
suppression 'Z' in a picture. You specify another byte for the
actual percent sign to print. The result would be 94.50%..
This would be the usual way to put the '%' on the report.
There are a few other 'cute' ways I can think of.
05 print-percent pic ZZ9.99B.
inspect print-percent converting space to '%'.
move '%' to print-percent (7:1).
