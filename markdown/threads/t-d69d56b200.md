In COBOL numbers are entered with 'implied' decimal points. That is when
data is entered it cannot be assumed to be somewhere in the data area but
it must be exactly lined up in the correct columns. An example:

Picture
PIC 9999

input record
1234
0052 is valid data representing decimal 52
52^^ is invalid (^ to represent space)

for a picture with implied decimal places we use V as a picture string
character. The V takes no space in the data it only describes where the
integers and fractions separate. Example

Picture
PIC 99V99

input record
0520 is valid data representing decimal 5.2
^52^ is invalid.

Micro Focus allow decimal points as an input field as an extension to
their COBOL syntax - it is not widely accepted practice.
