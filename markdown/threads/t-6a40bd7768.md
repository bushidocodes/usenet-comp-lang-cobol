the version level of the message file.

Does anyone have any ideas?

37/07 is  "no access to file" (file permissions).

As for the message file it should not be the cause of the error as
long as you can access it (even if the version is incorrect).

Use "strings runcobol" and "strings RUN.MSG" and compare the versions
that will show (you may need to redirect the output to a file).

For you have said I think you are using the runtime that the
customer has on his machine (and therefore on the hard disk copy (it
is a full HD copy is it?)).

Also note that you may have more than one "runcobol" or "RUN.MSG" on
the system and the wrong "RUN.MSG" may be being used. (do a find to
to see if this is the case).
