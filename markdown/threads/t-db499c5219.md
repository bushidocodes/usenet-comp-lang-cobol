```
It sounds like you are talking about an IBM mainframe environment, but I don't 
know what you mean by "last register".

Are you asking how to read the last RECORD of a sequential file?  Then COBOL has 
the same restriction as PL/I and the OPEN INPUT REVERSED option only works for 
tape.

See:
  http://publibz.boulder.ibm.com/cgi-bin/bookmgr_OS390/BOOKS/IGY3LR20/6.2.26

This is because of the way that files are "stored" in IBM mainframe 
environments.  For "normal" QSAM files, there is no high-level language to get 
to the last record directly.  (I am not certain if Assembler supports this - but 
don't think so.)
```

```
Interesting history of this from the standpoint of the standard.

From what I gather:

ANSI X3.23-1968 allowed OPEN REVERSED only for "sequential single reel"
files.  Apparently this was a holdover from "CODASYL COBOL"; the
implementation on the B6500 allowed it only for tapes, and I suspect this
may have been explicit early on.

ANSI X3.23-1974 lifted the "single reel" distinction, requiring only that
the file be of sequential organization.  It did, however, indicate that
allowing OPEN REVERSED was dependent on the ability of the hardware to
accomplish this (ours allowed it for disk files; apparently IBM's did not).

ANSI X3.23-1985 marked OPEN REVERSED as obsolete, noting that it was a
hardware-dependent feature not frequently implemented.

ISO/IEC 1989:2002 deleted the feature (as it had been marked obsolete in the
previous standard).

    -Chuck Stevens
```
