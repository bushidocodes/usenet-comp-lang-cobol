Andy,

Look at the ACCEPT statement (Format 3):

ACCEPT identifier-1
ON EXCEPTION identifier-9
imperative-statement-1
END-ACCEPT

identifier-9 should be described as an integer numeric data item. A
numeric code which identifies the field termination key is stored in
identifier-9. These codes may be configured, but out of the box F1 is 01,
F2 is 02, ... F20 is 20, and so forth. A simple program will allow you to
explore your keyboard and display all of these codes.

01 some-text pic x(20). 88 stop-request value "STOP" false spaces.
01 esc-code pic 999.
01 esc-spot pic 999.
...
set stop-request to false.
perform until stop-request
display "Press a key: "
accept some-text line 0 position 0 cursor esc-spot
on exception esc-code
display " Exception: " line 0 position 0
esc-code
" at ", esc-spot
end-accept
end -perform.
stop run.
[warning: program untested]

imperative-statement-1 may be NEXT SENTENCE if you wish to fall through to
the next sentence upon field termination.

Information about the ACCEPT statement can be found in the RM/COBOL-85
Language Reference Manual in chapter 6, as well as in the User's Guide. If
you have no manual, you might call our London office to order one at a
nominal cost (+44.171.799.2434).

As another hint, you might look at the very flexible CONTROL phrase of the
ACCEPT statement. Many of the features of the ACCEPT and DISPLAY
statements may be changed at runtime through the CONTROL phrase, allowing
you to build table driven screens using only one ACCEPT statement and one
DISPLAY statement. Another approach to efficiently building screens is to
use RM/PANELS, if you have that available.

Good luck on your project!
