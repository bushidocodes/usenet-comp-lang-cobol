Lee Unteriener and I went over this numerous times.  There's just no
way (because of the recursion restriction) to make this work directly.
 The COM server sinks for events just are not supported - outside of
POWER COBOL.  PowerCOBOL CAN handle them as far as I am aware.

NOTE: In and out of process servers behave differently in this respect
too.

May I ask which COM server you are using and is it an in or out of
process server?

I have worked in my capacity at Flexus in helping to ensure these
things DO work with Flexus Sp2 products.  The Sp2 ActiveX support has
been recently updated to support COM, both in and out of process
servers.  We have event handling for in process servers - even those
that are non insertable.  Out of process servers ... someday.

If you happen to be an sp2 user and want more information on this
support - or want to become an sp2 user, please - visit
http://www.flexus.com.

First thanks to all the helped on this subject.

Based on what Thane said , and on an example supplied by Paulo, I
managed to solve this problem by following the following steps.

1- Create a small VB OCX that can be inserted and has events that can
be processed by POWER COBOL. Source at the very end.
This control returns an event for each event raised by the Crystal
Report being processed.

2- add this OCX to the POWER form and assign the Crystal Report object
to one of it's properties.

3- Add all the code that needs to be on a CR event to the OCX events.


Regards

Frederico Fonseca



VBCODE

VERSION 5.00
Begin VB.UserControl evWrapper 
   BackColor       =   &H80000005&
   ClientHeight    =   1176
   ClientLeft      =   0
   ClientTop       =   0
   ClientWidth     =   1776
   ScaleHeight     =   1176
   ScaleWidth      =   1776
End
Attribute VB_Name = "evWrapper"
Attribute VB_GlobalNameSpace = False
Attribute VB_Creatable = True
Attribute VB_PredeclaredId = False
Attribute VB_Exposed = True
Dim oreport As CRAXDRT.Report
Dim WithEvents oSection1 As CRAXDRT.Section
Attribute oSection1.VB_VarHelpID = -1
Dim WithEvents oSection2 As CRAXDRT.Section

Public Event CREventSectionFormat()
Private activesection As String
Public Property Let cr_report_object(cr_report As CRAXDRT.Report)
Dim index As Integer
Set oreport = cr_report
index = oreport.Sections.Count
If index > 0 Then
   Set oSection1 = oreport.Sections.Item(1)
End If
If index > 1 Then
   Set oSection2 = oreport.Sections.Item(2)
End If
'(More sections can be added)

End Property

Public Property Get cr_report_object() As CRAXDRT.Report
cr_report_object = oreport
End Property
Public Property Let cr_report_active_section(cr_report_section As
String)
cr_report_section = cr_report_section
End Property
Public Property Get cr_report_active_section() As String
cr_report_active_section = activesection
End Property
Private Sub oSection1_Format(ByVal pFormattingInfo As Object)
    activesection = oSection1.Name
    cr_report_activesection = activesection
    RaiseEvent CREventSectionFormat
End Sub
Private Sub oSection2_Format(ByVal pFormattingInfo As Object)
    activesection = oSection2.Name
    cr_report_activesection = activesection
    RaiseEvent CREventSectionFormat
End Sub
