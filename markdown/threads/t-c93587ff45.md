```
There is a run-time option in VS COBOL II called WSCLEAR. It is NOT
RECOMMENDED by this writer, as it tends to only hide other problems.

For what it's worth, from the VS COBOL II Install and Customization for
MVS book:

+--- WSCLEAR
------------------------------------------------------------+

ï¿½ -WSCLEAR=--YES---
NO

YES
clears the working storage required by a RENT program to binary
zeros
(except where value clauses are specified).

Performance Consideration: Using WSCLEAR=YES may result in a small
amount of overhead at run time.

================= end of cut from book ==================

Note that WSCLEAR does nothing for spaces, or packed data, or non-RENT
programs. Not to mention that it does nothing for data in file section
areas.

I beleive that use of "INITIALIZE identifier-1" has the potential of
doing what you want, but may require lots of initialize verbs.
```
