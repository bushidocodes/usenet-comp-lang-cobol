Hi Pandian,

What you wanna do is convert a table or a database because the needs of the
business changed.
Well I used to do these things when I needed a conversion.

1) Read the table,  reorganise the data and write it to a file.
2) Read the file and write the data to the table/database (this can be any
database on any DB/2 system).

You can also work with the DB/2 unload command and reorganise the file. This
ensures you that you don't bring the performance of DB/2 down just by
testing your stuff.
