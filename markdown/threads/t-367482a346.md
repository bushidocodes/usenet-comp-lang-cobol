you should be able to find out which calling convention to use

In MF-COBOL this is very easy using:
CALL WinAPI "WinExec" USING BY REFERENCE YourHelpFileWithParms
(<-or whatever..)
BY VALUE 0
SIZE 2

and to use CBLLINK (compile/link utility)..

Think of it as testing a program from the specification without any
knowledge of what the programmer has done. In other words write the
test data to reflect the requirement rather than the possible
implementation. And don't forget that the purpose of a test is prove
that the program does not work and that a successful test results in the
program's failure.
