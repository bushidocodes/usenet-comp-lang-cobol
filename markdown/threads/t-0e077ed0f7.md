I would definitely not recommend C. The language doesn't have the data types
needed to deal easily with the data involved. COBOL has problems dealing
with some of the data types that C handles easily but your problem is to deal
with the COBOL data types. For that, COBOL or a utility on the source system
are your best bet. If it is a normal commercial file written on a mainframe
it contains character data in EBCDIC rather than ASCII and packed decimal.
It probably doesn't contain true binary. If you must deal with this file on
the Sun, COBOL with the proper options may simplify your work because many of
the COBOL packages available in UNIX such as Microfocus provide an option to
handle IBM mainframe formatted data. You also should get the record
description(s) for the file on the tape. You also probably want to convert
all of your data to character format and the MOVE CORRESPONDING verb in
COBOL can be a simple way to do so. See your COBOL manual for details.
Basically the input and output record descriptions are the same including ALL
names except for the actual field descriptions. The statement generates
field by field MOVEs. It will be simple to convert the resulting file to
ASCII. If you can't stand COBOL pick a tool that can easily handle fixed
length fields since COBOL is a fixed length field language unlike C.

through a custom coded C app to perform your task. Grab an EBCDIC
to ASCII chart and COMP-3, COMP-1, and COMP-5 format chart and
