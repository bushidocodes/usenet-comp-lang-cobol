Hi,

I have a data file that is formatted as follows :

01 stud-rec.
05 stud-key.
10 stud-subj-code.
15 stud-school-code pic x(4).
15 stud-subj-num pic 9(4).
10 stud-subj-activity-num pic 9.
10 stud-num pic 9(7).
05 stud-allocated pic x.
05 stud-act-class-num pic 9(3).

Now, here is my problem.

The primary key is 'stud-key', and I only have the 'stud-num', of which
there can be up to 6 instances of the same number. Is it possible to
sequentially go through this file, using stud-num as a key - even though
the primary key is defined as 'stud-key'?

The reason I want to do this is so I can get the 'stud-subj-code' and
'stud-allocated' for each instance of 'stud-num'.

If so, how would I do this? I'm using MF Cobol.

Any help would be appreciated!

regards, Brad

Nope. Can't be done if stud-key is the only defined key.

Many Cobols support multiple keys on the same file (one primary key
and 'n' alternate keys); if your Cobol supports this, and you define
stud-num as an alternate key, then you can read by that key.

But not otherwise.

If you are in a situation where you can't rearrange the keys, you may
need to create an alternate index with nonunique keys. the record size
to a VSAM alternate index is coded to give the average and maximiun
lengths by the formula:

5 + (alternate index key length) + (recordsize * prime index key
length)

example:

Assuming the average number of STUD-NUM is two, the maximium allowed
is six (stated) and the primary index key length is sixteen, you set
your average and maximium alternate index recordsize as:

5 + 7 + (2 * 16) = 44 and: 5 + 7 + (6 * 16) = 108

The IDCAMS is coded as follows

DEFINE -
AIX ( NAME(LEARN.STUDMAST.NUMAIX) -
RELATE(LEARN.STUDMAST) -
VOLUMES(SYSDA) - /* or use a specific a volume number */
RECORDSIZE (44 108) -
KEYS(7 9) -
NONUNIQUEKEY -
CYLINDERS(2 1) -
SHAREOPTIONS(2 3) -
UNIQUE -
UPGRAGE -
SPEED -
IMBED -
) -
DATA ( NAME(LEARN.STUDMAST.NUMAIX.DATA) -
CONTROLINTERVALSIZE(4096) -
FREESPACE (10 10) - /* Assuming you want to update */
) -
INDEX ( NAME(LEARN.STUDMAST.NUMAIX.INDEX) )

BLDINDEX ( INDATASET(LEARN.STUDMAST) -
OUTDATASET(LEARN.STUDMAST.NUMAIX) -
WORKFILES(WORKSRT1 WORKSRT2)
DEFINE -
PATH ( NAME(LEARN.STUDMAST.NUMAIX.PATH) -
PATHENTRY(LEARN.STUDMAST.NUMAIX) )

The use of a nonunique alternate index in a COBOL program is left as
an excersise.
