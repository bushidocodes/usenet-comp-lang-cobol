If you are talking about KSDS VSAM files, there really is no such beast
as "fixed length".  All KSDS datasets are variable length under the
covers--the average record length is a user-supplied, unenforced
"estimate".  The maximum RECSIZE is the upper bound, but all record
sizes large enough to contain the key field up to that limit are always
permitted.  To copy to a flat file you must use a utility (e.g., IDCAMS
REPRO) capable of handling variable length input files and whatever
output file format is consistent with your actual records.  If you use
some utility which attempts by default to generate an output file with
characteristics similar to the input file, it may insist on creating a
RECFM=VB output file since all VSAM KSDS appear as variable length
files.  If all logical records are really the same length, you can REPRO
to an output file defined as RECFM=FB,LRECL=actualreclength .  If
logical records have varying lengths in the VSAM file, your output file
should be at least RECFM=VB,LRECL=<maxactualreclength+4>.  Note that
LRECL for the flat file is based on the actual record lengths, not
whatever length was defined as the maximum VSAM RECSZ (which could be
larger).  If you don't know actual record lengths, LRECL=maxVSAMreclen+4
will also always work for a VB output file.  If your system supports
system-determined blocksize, then it shouldn't be necessary to specify a
BLKSIZE for the output file.  If BLKSIZE is manually specified then for
an FB file use a value of n*lrecl (want n large enough for a block size
close to 1/2 track) and for a VB file use the largest block size that
will give 2 blocks/track (minimum blocksize for VB is LRECL+4).

Will you post your IDCAMS control cards and the JCL running the IDCAMS
utility, and whatever message codes IDCAMS is giving you?

 You should be able to REPRO from vsam to a recfm=fb flat file in a
single step, I do it every week.

Note, there is a ~ (tilde) character in front of the JCL identifier
slashes because Outlook Express is trying to turn the ones with text
immediately after the slashes into hyperlinks (very annoying).  Please
ignore the tildes.

Example:
~//*
~//JS010    EXEC PGM=IDCAMS
~//SYSUT1    DD DSN=your.vsam.dataset.name,
~//             DISP=(SHR,KEEP,KEEP)
~//SYSUT2    DD DSN=your.flatfile.dataset.name,
~//             DISP=(NEW,CATLG,DELETE),
 ~//             UNIT=SYSDA,
~//             SPACE=(100,(100,50),RLSE),
~//             DCB=(DSORG=PS,RECFM=FB,LRECL=100,BLKSIZE=0)
~//SYSPRINT  DD SYSOUT=*
~//SYSIN     DD *
   REPRO  INFILE(SYSUT1) -
          OUTFILE(SYSUT2)
/*
~//

The space allocation figures and the lrecl being theory here, but the
above should work.
