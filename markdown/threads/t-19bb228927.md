```
First, convert the 76 in base 10 to base 16.
76 = 16*4 + 12 = 004C

Then convert to base 2:
004C= 0000 0000 0100 1100

To get the negative, you must convert to 2's complement. this is done in 2
steps:
first, flip all the 0's to 1's and the 1's to 0's.
flip    = 0000 0000 1011 0011

Now add 1 to the previous result.
+1    = 1111 1111 1011 0100

Change back to base 16.
b16  =  F        F       B      4

Hope this helped.
```

```
E.g. to convert -1234:
65536 - 1234 = 64302
64302 / 16 = 4018, remainder 14=E
4018 / 16 = 251, remainder 2
251 / 16 = 15=F, remainder 11=B
```

```
I assume you are referring to IBM MVS COBOL here.  If so the answer is:

Negative binary numbers are stored in a form called  2's compliment.  The
process is as follows:

1. take the binary positive number

0000  0000  0100  1100      (hex 004C, dec 76)

2. change 0's to '1's. 1's to 0's.

1111  1111  1011  0011

3.  add 1 to the result

1111  1111  1011 0100  (hex FFB4, dec -76)
```
