At first sorry, that I answer so late to the responses of Karl Wagner and Gary Drummond (and to the critical observation of Jedi). At second thank all for the good answers.
Karl has given me a useful part of source code to substitute long file names. But this code runs only on WindowsNT. Sorry I forgot to mention, that the source code should (so far as possible) also run on UNIX server, and it does after a new compilation on UNIX (with long filenames).
The short-cut way of Gary is also a good idea, but the programs has to elaborate more than 20 files (for every filename a short-cut?), and some filenames must be changed during run-time.
Also I give you the answer of Micro-Focus for your information:
"In reply to your recent question regarding error 188 returned on Filenames greater than 100 characters. Having checked this is correct, in the 'Programmers Guide to Writing Programs' page 3-12 we document 'Micro Focus COBOL imposes a limit of 100 characters on the size of an external filename' ".
Now our customer has decided to use relative filenames.

Best regards
Rainer Lohkaemper, IKOSS VAN, Germany
