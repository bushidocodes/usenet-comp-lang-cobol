You've also got this problem: multiple transactions for the same employee.
What if you add an existing employee? Where is this check going to be done?
What about deleting non-existent employees?

You need to simplify this design further. Break down your requirements into
specific, single-word actions (Sort Employees, Validate Fields) and then break
those down (Sort by EmpNum, Sort by TransID). Forget tables unless you have
something like this:

05 categories.

categories to get what you need. You DON'T pull in 300k of records into a
