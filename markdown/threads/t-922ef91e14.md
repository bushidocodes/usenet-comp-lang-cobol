### Micro Focus Phantom SQL Return Codes

- **From:** "david b. vance" <ua-author-17087345@usenetarchives.gap>
- **Date:** 1996-10-17T20:00:01+00:00
- **Newsgroups:** comp.lang.cobol
- **Message-ID:** `<3267F769.661E@unipac.com>`

```

We are using the Micro Focus Embedded SQL Toolkit against SQL
Server 6.5. When run outside of Animator, the first SQL
statement in every subroutine gets a bad return code. The
return codes have been unpredictable (overflow, not found,
mismatched parameters, etc...). When we add a new first
statement and do not check the return code, the subsequent
statments perform correctly (including the statements that
previously failed).

Has anyone faced this situation? What causes the phantom
code? How can it be solved?

We run under NT and Workbench 4.0.20.
```
