```
Well, it worked for me, as expected.  ShellExecuteEx returns immediately, at
which time you go into some WaitForSingleObjectEx code, where you can wait
for a 'timeout value' or wait 'infinite.'

If you want BASIC-language code to convert to your flavor of COBOL, here is
an edited note I sent to someone via regular e-mail, about using this to
invoke a 'file copy' program (I also suggested the copyfile API):

== begin note ==

I posted the BASIC Language source code needed to monitor a process started
with ShellExecuteEx on the powerbasic bbs 6/22/01. Go to
http://www.powerbasic.com/support/forums/Ultimate.cgi

Go to the "Source Code Forum" and look for something posted by Michael
Mattias 6/22/01 under the title, "Win32: Monitor Process with
ShellExecuteEx" (or something real close to that).

The best way to retrieve that code is to hit the "edit this message" button,
right-click, select all, right click, copy, change over to notepad and
paste. After pasting, just hit the "back" button on your browser to
"escape." (Don't worry, you can't really edit the message, since you are not
the author). You should not need to register with the BBS to do this

If you are using a batch file to invoke 'copy', you'll need to open
COMMAND.EXE (win 9x) or CMD.EXE (NT/2K) as the program name. I think 'Xcopy"
is a program, so you would not need to worry about the command interpreter,
but you would need to worry about finding xcopy.exe or xcopy32.exe. The
newer Windows systems hide the old DOS utility programs and do not include
them on the path.

== end note ===
```
