If you want to do a simple rebuild of the indexes and the .IDX file
is not completely corrupted, use:-

REBUILD filename.ext

REBUILD will use the key information from the .IDX part of the
file. Most of the other parameters are for creating a new different
index, and in practice I never need to use them.

The /X parameter specifies the 'key of reference' for the rebuild. The
key of reference determines the sequence the records are written to the
new file during rebuild. The default is the record key (key 0), but you
can use the /X parameter to specify an alternate key (key 1, 2, etc.) for
this purpose.
