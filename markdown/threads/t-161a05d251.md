Don't use COPY in the Data Division nor INCLUDE in the Procedure
Division. You'd think they would act the same if a pre-processor was
not involved; they don't.

For example, a COPY statement in the Data Division often generates
spurious error messages that you don't get with INCLUDE.

You use a make facility, the same as on a mainframe.  Fujitsu have a GUI
project manager that will build all the make files automatically, if you
wish to use that instead.  I do not know of *any* Cobol that has the make
facility built into the actual compiler, but then there are lots of
compilers that I have not used.

The problem is that PowerCobol generates the code, and makes all that stuff
invisible.  PowerCobol *is* a GUI code builder, and actually calls the
standard compiler.  If you write ordinary Fujitsu Cobol, rather than using
PowerCobol, then the problem does not exist.  You set up dependancies much
the same as with any other Cobol.
