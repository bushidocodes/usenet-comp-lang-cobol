```
I have several assembler applications that LOAD compiled tables.  That is,
the table is nothing more than a set of DC statements that has been
assembled into a load module.  Is there any way to accomplish the same thing
in native COBOL?  I say "native" because I know I can call an assembler
module to load the table and pass back a pointer.  I'm wondering if there is
any construct in COBOL that will accomplish the same thing as the assembler
LOAD instruction.
```

```
I am not certain that this is exactly what you want - but (assuming that you
are using VS COBOL II or later COBOL compilers on MVS.  Your "DC" comments
make me think you are talking MVS - but I could be wrong).

COBOL "main" program defines table in Linkage Section.

Main calls subprogram passing Linkage Section table.

Subprogram defines table in Working-Storage (so it "allocates" the table -
which remains in memory as long as the subprogram is NOT cancelled).  Its
procedure division does NOTHING but
    SET ADDRESS OF LS-TABLE to ADDRESS OF WS-TABLE
and returns to main program.

The table will now be available to the main program - in its linkage section.
```
