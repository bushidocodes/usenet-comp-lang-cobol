First off, I'm going to assume plain batch, no CICS or Data
Base processing.

1) First JOB Step is the cobol compile execution.
   Takes the source via the sysin dd statement and writes
the object to a DD (specified in the COBOL parms as I
recall). It writes the listing to SYSPRINT DD.

2) IF the prior step (generally called COB or CBL) executes
with CC -= 4, then the next step, LKED runs and takes in the
object deck from step one, matches it to the libraries in
the SYSLIB DD and produces an executable program which it
writes to SYSLMOD.

3) You now take a second existing PROC and run it to pull in
this program to test it.

As for all the rest, it depends on how your LOADLIB is
built. If all the programs that make up the LOADLIB are
written in COBOL then there should be a one for one match.
If there are user written subroutines that are statically
linked, then there will be a one (LOADLIB member) to many
(Source) match.

The DD names you have do not necessarily match the PROC
keywords.

Your SRCELIB probably is used in the SYSIN DD for the COB
step. There should also be a "MEMBER" keyword that is
appended (within parenthesis) to the SRCELIB thusly:

//SYSIN  DD DISP=SHR,DSN=&SRCELIB(&MEMBER)

Your COPYLIB is probably placed into a SYSLIB concatenation,
and there should be no member names specified.

The LINKLIB information is used by the LINKEDT (Linkage
Editor or BINDER) to pull in all the routines that COBOL
needs to match to the object that it produces (output to
SYSPUNCH or some other DD that I've forgotten). 

Your MODLIB could just be the staging library that holds all
the user provided subroutines (or object decks).

Lastly, your PROCLIB will not contain any relationship
whatsoever to the load library (as to names) as all it is
supposed to contain are JCL procedures (PROCs) that are
called from your JCL.

Example:

//COMPILE1 EXEC COBCL,SRCELIB=WWY81.PROD.SOURCE,MEMBER=PGM1,
//            MODLIB=WWY81.PROD.OBJECT,
//            LOADLIB=WWY81.PROD.LOAD 

This could expand into 20-50 JCL statements, and any keyword
you didn't specify would default to some value (look in the
PROCLIB where ever it is and see).

Now, if this was a homework assignment and not some
programmer trying to figure out an "MVS" system, I hope you
are far enough along for some of this to make sense. Because
I didn't say anything about your copylib which you should be
able to figure out for yourself in any case.
