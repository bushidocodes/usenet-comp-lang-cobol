```
Richard,
I'm responding to your post on "comp.lang.cobol" concerning the
existence of a COBOL Standards Checking program. I am the author of
three of these little gems, one for OS/VS COBOL, one for VS COBOL II,
and one for "1985 ANSI COBOL", which has been applied both to VS COBOL
II and COBOL/370 (which we haven't really gotten to yet).

I have not yet figured out how to port the standards checking to places
like Micro Focus COBOL, since the programs are executed as part of our
compile process on the IBM mainframe, and IBM's 'standard' compiler
exits don't exist on other compilers. However, most of the code is
COBOL (parsing the statements, testing for compliance, issuing
diagnostic messages, etc.), so conceptually, it wouldn't be too tough.
It could even be separated from compilation, but that would require a
driver able to read COBOL and fully process COPY statements (including
the REPLACING clause, nested COPY's, etc.). I've not been asked to
write this facility, so I haven't done so.

Our thinking on COBOL coding standards has changed over time. We used
to say that a program could be no larger than some certain size, and
that procedures (paragraphs or sections) also had size limits. Now, we
count conditional verbs (IF, EVALUATE, etc.) and try to limit
complexity. Other old standby's like BLOCK 0 in the FD of every
sequential file are still there.

I work for Computer Sciences Corp., having been outsourced by Hughes
Aircraft Co. a year ago. These standards and processes are holdovers
from Hughes. My program isn't available to others without permission,
I'm sorry to say, and I haven't looked into getting permission. Feel
free to contact me if you want more info.

Colin Campbell
```

```
I work for MasterCard International and I wrote a standards checker. It
is primarily concerned with naming conventions and structured
programming, but also checks for non-referenced datanames, paragraph
names, and IF-ELSE-ENDIF alignment. It performs a static check on the
source program file and does not expand copybooks. It's about 5000 lines
of COBOL II and was ported from MVS to Realia COBOL for DOS or OS/2.

I'm not sure how helpful something like this would be to you, since my
company has some very specific dataname and paragraph name numbering
conventions. It was meant to be used for code walkthrus, so basically
all it does is print a listing like a compile with a bunch of
compliance warning messages. It attempts to measure cyclomatic
complexity, but I think there are other tools on the market which do that
better.
```
