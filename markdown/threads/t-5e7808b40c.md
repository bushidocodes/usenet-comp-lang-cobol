You assign a value toa PROC-PTR with the following statement:
- SET PROC-PTR-VAR TO ENTRY "XXX"
this will make XXX to be statically linked with your module and
PROC-PTR-VAR will contain the address of the XXX routine.
If fact the rules of the CALL statement for the static/dynamic calls apply
to this form of SET statement (call to litteral or vaibale, DYNA param...)
You can then pass PROC-PTR-VAR as an EXTERNAL variable or as a normal
parameter to others sub-routines and perform:
- CALL PROC-PTR-VAR USING ...
This will make your subroutine dynamically call the code include in the
module where the SET has been done:

This can be useful for ILC, i.e mixing PL/I, COBOL, C++, ASM... in the
same enclave and/or to keep only once at run-time the code of a
sub-routine
