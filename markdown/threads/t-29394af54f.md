The TinyCOBOL project is coming along quite nicely.

Tim Joslings COBOL compiler (GNU compliant) is DIFFERENT from
TinyCOBOL and is coming along.  

Tiny seems to be playing loose with the standards in an effort to get
something working quickly and they had a head start with source
provided for a DOS based compiler.

Tim's efforts are from scratch and are standards compliant and GNU
compliant.  When Tim gets his subset working that will allow expansion
of the compiler using this COBOL subset, I will try to get involved
and do some work on the project.
