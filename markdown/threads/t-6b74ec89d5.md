```
I think you have analyzed the problem pretty well and am happy to give you
some clues of how I would approach the problem.  I must start by saying that
this is MUCH easier in a "homework" problem (like yours) then it would be in
the "real world".  In your case, you are told that you can never have more
than 100 records in a "batch".  Keeping 100 records in storage (in your
Working-Storage Section) is fairly "reasonable" to do.  On the other hand, if
this were a "real world" problem where you could have an "undefined" number
of records per batch, another technique would probably be better.

The following "suggestion" will ONLY work if you have already gotten to the
part of your course dealing with "tables" in COBOL.  If you don't know how to
process those (yet), then you will need to use some other technique.

Pseudo-code:
Do while any more records

  Initialize table and hash accumulator

  Do while more records in batch
     read record into table
     add this records value to accumulated hash counter
  End-Do

  Read batch-record
  Compare batch total with accumulated total

  If totals are equal
     write out all records from table to "good" file
  Else
    write out all records from table to "bad" file
  end-if

End-do
```
