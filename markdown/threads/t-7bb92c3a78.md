I defined each line to be printed as a "header" line.
Here are a few example:

*................ Spacer line Heading ............................
01 SS-SPACER1.
05 PIC X(32) VALUE '================================'.
05 PIC X(32) VALUE '================================'.

05 PIC X(32) VALUE '================================'.
05 PIC X(4) VALUE '===='.
*
*................ Spacer line Heading ............................
01 SS-SPACER2.
05 PIC X(132) VALUE SPACES.
*
*................ Top of page Heading ............................
01 HL-HEADING1.
05 PIC X(30) VALUE 'COOPERATIVE DES BITS QUEBECOIS'.
05 PIC X(14) VALUE SPACES.
05 PIC X(24) VALUE 'COMPLETE EMPLOYEE SALARY'.
05 PIC X(22) VALUE ' AND DEDUCTIONS REPORT'.
05 PIC X(13) VALUE SPACES.
05 PIC X(6) VALUE 'PAGE: '.
05 HL-OUT-PAGE PIC ZZ9.
05 PIC X(6) VALUE SPACES.
05 PIC X(6) VALUE 'DATE: '.
05 HL-OUT-YEAR PIC X(2).
05 PIC X(1) VALUE '/'.
05 HL-OUT-MO PIC X(2).
05 PIC X(1) VALUE '/'.
05 HL-OUT-DAY PIC X(2).

Then to print them, I loaded them in a buffer defined as ...

*
FD REPORT-PAPER-FILE
LABEL RECORDS ARE OMITTED.
*
01 REPORT-BUFFER PIC X(132).

.............. in the data division.

The print command itself looks like this:

*
WRITE REPORT-BUFFER FROM HL-HEADING8
AFTER ADVANCING 2 LINES

Of course, the snippets all have variables that are not consistent
(the whole program is close to 300 line, I think (who keep tracks of
these things ?)

To adapt to screen, specify the buffer to be 80 characters long and
format every thing for that lenght.
