Procedure Division
     Using by Value     LS-Input-Date   LS-Input-Offset-Block
           by Reference LS-Output-Date  LS-Reject-Code
     Entry P315-Offset

Call "P315-Offset"
   Using My-Input-Date   My-Input-Offset-Block
         My-Output-Date  My-Reject-Code
on Exception
     Move 7000 to My-Reject-Code
End-Call

...  I would expect that the runtime would pass the parameters the way 
the subprogram is expecting them.  Or is it?  Would I need to change the 
call to say something like...

Call "P315-Offset"
   Using by Content   My-Input-Date   My-Input-Offset-Block
         by Reference My-Output-Date  My-Reject-Code
on Exception
     Move 7000 to My-Reject-Code
End-Call

?  If that's the case, then it begs the question "Well, what happens if 
I pass a parameter by reference that the subroutine is expecting to be 
passed by content/value?"  Just curious...  I've sort of stuck my neck 
out and said I can do this (correctly and quickly), so I'm not wanting 
to be proven wrong.  :)  Thanks...

When you pass by value, the compiler builds [a copy of] the data in the
argument list; only floating point (comp-1 and comp-1), binary (comp or binary)
or single-byte literal or data item may be passed by value.

When you pass by reference [the default], the address of the data item is
passed in the argument list, and it may point to any type of data (elementary
or group)

When you pass by content, the address of a copy of the data item is passed; it
looks to the subroutine the same as passing by reference, but if the subroutine
changes the data passed to it, that change is not reflected back in the caller.



>
>...then, in the calling program, I would say...
…[11 more quoted lines elided]…
>

A subroutine may have "by reference" (the default) or "by value" for all or any
of the parameters it expects. For items passed by value, [a copy of] the data
is passed in the parameter list, for items passed by reference, the address of
the data [or address of a copy of the data - the subroutine cannot tell the
difference] is passed. Only certain types of data may be passed by value in
COBOL. (see above).

The runtime plays no role in this whatsoever. The issues are all decided at
compile time.



> Call "P315-Offset"
>   Using by Content   My-Input-Date   My-Input-Offset-Block

>to be proven wrong.  :)  Thanks...

So, if you pass by reference and the subroutine expects it to be passed by
value, the subroutine will treat the data it finds there as data, not as the
address that it really is; a likely abend of type S0C4, S0C7, or possibly some
others, will almost certainly eventually occur.
