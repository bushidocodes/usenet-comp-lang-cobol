```
If you are a SHARE member, then you could submit this as a SHARE requirement. 
HOWEVER, I doubt that this would get much support.  It would be relatively easy 
to create a CLIST or REXX to do this.  Some of the source management systems 
(e.g. Endeavor or SCLM) may already have this functionality - I don't know.

However, do this should be a pretty unusual thing to need to do.  At least I 
can't think of many times that this would be useful.  It would, also, require 
that you keep your COPY members in a different PDS than your main source code. 
Some shops do this and some don't.

Finally, are you aware that Enterprise COBOL *does* support the compilation of 
multiple sources at one time.  You simply end the first program with an END 
PROGRAM header and then put the next code after it.  This can be done for as 
many programs as you want.
```
