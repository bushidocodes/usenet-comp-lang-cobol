Report on December 4-9, 1995 ANSI X3J4 COBOL Committee meeting,
Hershey, Pennsylvania, USA. When a vote is shown, the numbers are
for, against, and abstain (f-a-a).

There was no primary topic at this meeting, but there were several
important proposals under consideration. The meeting was one day
longer than usual because we only have one more meeting before the
feature content of the next standard is frozen.

The major topics/proposals we discussed were:

1. A proposal to define i-o for multi-octet characters (MOCS) was
discussed at length (it was rewritten after discussion at the last
meeting). It adds such things as a SELECT WHEN to figure out what
record description applies on a READ, a POSITION clause for reports
and screens that states in points where to position some text, a
CONVERT clause to state that data is to be converted on i-o, a WRITE
FROM literal, and so on. Note that this proposal works for non-MOCS
as well, as in converting EBCDIC to ASCII with embedded packed-
decimal and binary fields. Lots of major changes were made and it
was sent back for rewrite.

2. A proposal to patch up some standard arithmetic rules and add
more to the concepts on same was passed and will be added to the base
document.

3. Lots of editorial papers were passed. Nothing very important was
done in these.

4. My proposal on a RESUME statement that allows a fatal exception
declarative to cause the program to continue executing at some point
was discussed. It allowed resumption at the next statement following
where the exception occurred. This was deemed as not a good thing,
so a mechanism of identifying a recovery point was substituted. The
proposal will be changed to use RECOVER (not RESUME) in a
declarative, to allow USAGE RECOVERY, SET usage-item TO RECOVERY, ON
RECOVERY ... NOT ON RECOVERY ... END-SET. I will revise it for the
next meeting.

5. The proposal on CALL was discussed. This adds prototypes, user-
defined functions, passing parameters by value, calling other
languages easily, optional parameters, arithmetic expressions as
parameters, adds GOBACK, and more. The most controversial part of
this proposal is how to handle literals as program, function, or
method names. Case-sensitive or not. In the proposal right now it
is sometimes one way and sometimes the other. After lots of
discussion and minor changes, the proposal was sent back for rewrite.

6. The proposal on program pointers was discussed. There was
controversy over whether you could use functions as well as programs.
It was decided that you could. Because this feature fits closely
with data pointers, both the procedure and data pointer proposals
will be combined in one. The proposal will be rewritten.

7. The proposal on adding floating-point and true binary data types
was discussed. It will use usages of CHAR-[UN]SIGNED, SHORT-
[UN]SIGNED, SHORT-FLOAT, LONG-FLOAT, etc. Lots of changes were made
and the proposal will be rewritten.

8. My proposal on recursive programs was discussed. There was some
confusion about programs contained in recursive programs. It was
decided that they would be implicitly recursive. Other minor changes
were made. I will rewrite the proposal.

9. A paper on problems with merging rules for fixed and free format
reference format was discussed and a proposal will be prepared to
correct the problems pointed out.

10. A proposal on allowing multiple characters for the currency sign
was discussed and passed. The updated proposal will be reviewed at
the next meeting before being added to the base document.

11. A paper on getting rid of EXIT PARAGRAPH (recently added) was
discussed and the author was told to go through WG4 (he is from the
UK).

The next X3J4 meeting will be in La Jolla, California, USA, on
February 12-17 1995, with an editing ad-hoc meeting on February 11
and an OO ad-hoc on February 13-16. Again, there is one extra day
tacked on to the meeting. The main topic will be critical proposals
like CALL, exception recovery, pointers, etc. There will be an
additional editorial ad-hoc meeting February 7 and 8 in Palo Alto,
California.
