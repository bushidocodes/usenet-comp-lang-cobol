```
> how to declare a "static char - like in C language"

In C 'static' is used for several things.  Inside a function it makes a
declaration non-dynamic.  Outside a function, where everything is
non-dynamic, it makes it 'file local'.

> static char SCCSId = "@(#) Progname - Version ID - ModificationDate";
> and the what command returns :
> "Progname - Version ID - ModificationDate".

I don't know if 'what' is a general Unix command or something written
in house, but it seems that it simply searches through the specified
file looking for '@(#)' and then outputs up to the next null
terminator.  I have written something similar myself for the same
purpose.

The only point of being 'static' is that if modules are linked together
there is not a clash in having the same variable name.

In Cobol you would just need to have a WORKING-STORAGE variable with
the required content followed by a null terminator:

    WORKING-STORAGE SECTION.
    01  SCCSId.
         03 FILLER PIC X(nn) VALUE
              "@(#) programname ....".
         03 FILLER PIC X VALUE LOW-VALUE.
```

```
A fairly big topic to describe in full detail - depending upon your 
COBOL compiler, look at on-line LRMs (Language Reference Manuals) - 
words like PICTURE, INITIALIZE, LITERALS, CONSTANTS, REDEFINES, use of 
WORKING-STORAGE etc. Go to vendor sites and look for coded examples. 
They wont specifically cover the above topics, but you will see the 
techniques tucked away in programs describing how to use other features.

WORKING-STORAGE SECTION.

*> Note - double quotes
01 Text1	pic x(31) "This text is the size specified".

*> Note - single quotes
01 Text2        pic x(50) 'This text is less than size specified'.

*> Numerics
01 Text3	pic 9(03) value 500.
01 Text4	pic 9(03)v9(02) value 123.45.

*> CONSTANTS - Part of new COBOL 2002 Standard, but as yet not
*> implemented by any vendor. However, Micro Focus compilers have a 
current extension for CONSTANTS :-

78 Text5	value "Hello World".
78 Text6	value 1.234.
78 Text7	value 2000.

*> No initial values - so be sure to INITIALIZE before using them !!!!! 
Alternatively, give them initial values as shown

01 Text8	pic x(40) value spaces.
01 Text9        pic 9(03) value zeroes.

move 'Good morning' to Text8
move 23 to Text9

and, your own example :-

01 SCCSId  pic x(40) value "Progname - Version ID - ModificationDate".


Above is in no way exhaustive - but it's a start :-)
```

```
Hi.

Something along the lines of the following will do the trick :

        78 Progname         VALUE "Progname".
        78 VerID            VALUE "VerID".
        78 ModificationDate VALUE "ModificationDate".
        01 SCCSId pic x(60) value x"0D0A" & "@(#)"
            & Progname & " - " & VerID & " - " & ModificationDate
            & x"0D0A".

where you could pass the level 78 items -- constants -- in at compile-time.

Or, more simply, use :

        01 SCCSId pic x(60)
           value x"0D0A" & "@(#)My What String" & x"0D0A".
```
