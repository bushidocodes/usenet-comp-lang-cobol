```
In response to another thread, I received the following suggestion for how
you can (with a little work) get the current module name in a COBOL program
running under LE (IBM OS/390).

"Bill,

  It is a little stretch, but you could use a condition handler to get the
program name.  Just signal a condition (CEESGL) and in the handler
call CEE3GRN to get the routine name."
```
