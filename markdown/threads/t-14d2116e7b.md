```
Dear friends,

I've MF Cobol Workbench 4.23 (16 bit) installed on a Windows NT PC.
I also have the exact same software installed on a Windows '95 PC.

Here's a problem I'm struggling with. I hope one of you can help me
out...

I've the same .int file on both systems.
When code execution reaches a sort operation, the program on the NT
system
fails with Error 223 - Sort/Merge error whereas the one on Windows '95
breezes thru without a problem!

I looked at the probable causes for error 223 and the manual came up
with
the following:

* The environment variable TMP points to a directory that does not
actually exist.

- I verified that it does.


* There isn't enough free space in the directory pointed to by the TMP
environment
variable. An MF tech support person said that the free space in the
directory
must be atleast 5 times the size of the program being executed.

- I confirmed that there is more than enough space left on the
drive.

* There is a file 'lock' issue.

- There is no explicit file locking being used within the program.
The MF tech support
person advised me to add a "+B" to the value of the COBSW
environment variable to
get around this possibility.
I did so but the problem refuses to go away.

Whats even more bewildering is that the Windows NT PC has 64 MB of RAM,
while the
Windows '95 PC has only 16 MB!

The program is set to move into production tomorrow and I don't have a
clue how to get
this to work! Any help/suggestions will be greatly appreciated!


Thanks

SM
```
