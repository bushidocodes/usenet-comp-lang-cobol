```
In article <01bdb7f0$a3c6f100$aa0cd6d1@default>, "Pascal"
writes:

› CALL FUNCTION CURRENT-DATE USING date-identifier.

Is that the correct syntax? What we use in COBOL/VSE is:

MOVE FUNCTION CURRENT-DATE(1:8) TO date-identifier

where date-identifier is an X(8) field or equivalent to receive yyyymmdd.

MOVE FUNCTION CURRENT-DATE(1:14) TO date-time-identifier

to a 14-byte field gets yyyymmddhhmmss in local time.

The next two characters that I don't show above get tenths and hundredths of a
second, local time.

After that comes timezone information, but I don't recall the exact format.

As far as I am aware, intrinsic functions are never called with a CALL
statement, but instead must be the sending field of a MOVE statement for
intrinsic functions sending character results, or on the right side of a
COMPUTE statement for intrinsic functions that return numeric results.

Mark A. Young
```
