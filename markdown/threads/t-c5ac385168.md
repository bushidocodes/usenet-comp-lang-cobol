Seems to me you are pushing your luck on this one - you are trying to 
emulate the Windows "disable", "enable" or "readOnly".

Don't know your compiler but does it allow you to (1)ACCEPT WHOLE SCREEN 
v (2)ACCEPT THIS FIELD.  Given (1) - segment your ACCEPT to ACCEPT 
SUB-SCREEN (the majority), and programatically (2)ACCEPT THIS FIELD. 
Presumably when it comes to changing, you are prompting user for the 
Field-Number to be changed. If it is one you currently want to 'protect' 
- an error message, "Can't change this right now....".

I would have thought your choices are pretty limited - check out  the 
AcuCOBOL full screen syntax, addressing each 'word' to see if there's 
something hidden behind the general name. (I've just checked the Micro 
Focus Screen Section which is COBOL 2002 PLUS extensions - and I don't 
see anything there allowing a 'protect').

If it is part of a DISPLAY it will be displayed.  If it is part of an
ACCEPT then ...

One of the reasons why I don't ACCEPT more than one field at a time
and have avoided SCREEN SECTION is the difficulty in interacting
precisely with the user. It can be done by making the TAB key or
falling off a field into returning back to the accept, then using the
CURSOR is to set the next field for the user.  In this way you would
have to control every field change but could avoid particular fields.

You could chenge the 'using' to some work field which you initally
fill and then reset if the user changes it.

You could do it if you were using the emulated graphical controls on a
character mode screen.  For example:

01 main-screen.
    03 field-1 entry-field line 1 col 1 value ws-data-1.
    03 field-2 entry-field line 2 col 1 value ws-data-2
    03 field-3 entry-field line 3 col 1 value ws-data-3.
...
...
...
        display main-screen.
        modify field-2 enabled = false.
        accept main-screen.  | This will allow entry in only fields 1 and 3
...
...
        modify field-2 enabled = true.
        accept main-screen.  | This will allow entry in all 3 fields

Otherwise, don't accept a higher level screen section entry - accept the
lower level entries and handle moving back and forth between fields in the
code and determine which you will accept or not.
