In your RM/COBOL Language Reference Manual, check the documentation of the
ON EXCEPTION phrase of the ACCEPT Statement (Terminal I-O) in Chapter 6.
There is additional documentation in the RM/COBOL User's Guide that gives an
extensive list of "field termination codes" which identify the exact
function key that was pressed.  If you are on UNIX then you will
additionally need to use the appropriate configuration (termcap/terminfo)
for the terminal you are using; this is also documented in the RM/COBOL
User's Guide.

Best regards,
Tom Morrison

Processing F-keys as interrupts: I agree the best method is totally
platform dependent, but if you are in a looping program and want F-key
interrupts you can use a loop counter to break out to an ACCEPT BEFORE
TIME statement. I think you can use BEFORE TIME 0 and just handle the
exceptions, though I haven't done this for a while. My experience is
with RMcobol.

I didn't know students studied cobol anymore, we're supposed to be
dinosaurs. I like people to punctuate and capitalise, at least to some
extent, as it makes their posts easier to read, so I am more likely to
respond.
