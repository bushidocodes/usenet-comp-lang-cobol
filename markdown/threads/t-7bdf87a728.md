I currently have to maintain KSAM XL files that are causing me
headaches. Our OS is MPE/IX. The record descriptions are:

01 record-layout.
02 key-fields.
05 char-1-2. pic x(2).
05 record-key pic x(20).
02 data-fields.
05 all-the-rest pic x(100).

The problem occurs when we use FCOPY to create sequential data sets to
use in our MFCOBOL programs on our PC's. The char-1-2 sometimes
contains binary zeros when we expect spaces(hex 20). The copied out
record with binary zeros in char-1-2 are shifted left 1 character
like this

01 record-layout.
02 key-fields.
05 char-1-2. pic x.
05 record-key pic x(20).
02 data-fields.
05 all-the-rest pic x(100)

How can I fix this? Any help would be greatly appreciated.
