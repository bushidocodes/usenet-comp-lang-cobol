Far too much to cover here.

Here is an outline of what you need to do:

1. Install the ActiveX control and note where it is installed.
2. In PowerCOBOL go to the "Add custom controls" menu. Open it and search 
through it for the Grid component. Tick the box and close the window.
3. The Janus Grid icon should now appear on the PowerCOBOL control toolbox.
4. Drag it to your form.

You will not now have to use the CREATE-OBJECT  method of COM , as it is 
already installed on your form. Make sure it appears in the hierarchy of 
controls on your form.

You can read about its Methods, Properties, and Events, in the Help file 
that comes with it, and you can use the right mouse click in the editor to 
pop up properties and methods in code.

I have some extensive code using this grid, but it is not on the machine I 
am typing this on, so I can't refer to it at the moment.

The main problem you will have is in converting the VB parameters  in the 
examples to COBOL (especially if you are not familiar with VB). I can help 
you with this (and will do so for free) but you would need to be specific 
and post attempts you have made.

Otherwise, contact me privately and I can offer full support, explanations 
over Skpe video chat, and working code samples, but there will be a charge 
for it.

You can reference the grid just like any other object in PowerCOBOL, but if 
you want to use drag-and-drop, it won't work in COBOL using normal methods. 
(This requires object references to be established to the grid's VB6 runtime 
and it's tricky. I have solved it recently for another client and they can 
now use COBOL to do this. Obviously, having charged them for this solution, 
I can't give it to you for free, but I can make sure the charge is fair.)

Because it is a COM control you must interface to it through the PowerCOBOL 
special class *COM in the REPOSITORY paragrah of your form, and all 
parameters must be passed by reference (not by value, as they sometimes are 
in the VB examples).

Here is an outline to get you started (It isn't definitive and I can't test 
it as I write this, but it should give you the idea...)

This example assumes you have NOT bound the grid to a data source...

WORKING-STORAGE SECTION.

01  ValueOfOne                   pic s9(9) comp-5 value 1.
01  DataToBeDisplayed        pic x(13) value "Grid cell 1,1".
...
PROCEDURE DIVISION.
(Assuming PowerCOBOL has loaded the control to your project with the name 
:"Grid01"...)

*> set the current row...
        invoke
                Grid01 "SET-Row" using ValueOfOne
        end-invoke
*> set the current column...
        invoke
                Grid01 "SET-Col" using ValueOfOne
        end-invoke
*> set the data in the cell pointed at by the current Row and Column 
(1,1)...
        invoke
                Grid01 "SET-Value" using DataToBeDisplayed
        end-invoke

As you can see from the above, understandiong and manipulating properties of 
the grid object is just as important as using its methods.

Hope this helps,

Pete.
