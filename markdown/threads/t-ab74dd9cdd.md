Is the problem the instruction set or the speed of the processor?
You might want to search for MO'SLO which can be run in front of the older
program to slow the processor down. I encountered this speed situation
when switching from a 486/66 to Pentium 90.
usage of mo'slo would be moslo /50 pctools
which would effectively cut the processor to 50% performance allowing the
internal timing of the application to catch the interrupts needed for proper
program functions. (The vendor wanted me to by the latest version and then
download a patch to the latest version to get around the Processor speed
change.)

Hope that this information is of some use to you.
