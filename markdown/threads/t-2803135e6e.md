```
I have a copybook used in the DECLARATION section for handling file errors.
I use replacing to set the name for statuses and other things.  I also want
to be able to display this file name.  I currrently have coded in the
copybook CPFLEERR

MOVE WS-STATUS-:FILENAME: TO WP-FILE-STATUS
MOVE ':FILENAME:' TO WP-FILE-ERROR

and then have defined in the program

COPY CPFLEERR REPLACING ==:FILENAME:== BY SVMPAR

However the compiled listing show the following

MOVE WS-STATUS-SVMPAR TO WP-FILE-STATUS
MOVE ':FILENAME:' TO WP-FILE-ERROR

The COPY REPLACING function does not appear to replace when the field to be
replaced is enclosed by quotes.

Any ideas other than setting up variables in a similar way to the
WS-STATUS-* variables for each file?
```
