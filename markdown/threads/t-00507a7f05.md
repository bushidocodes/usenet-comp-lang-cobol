```
specific loadlib members.  I have also written REXX to run this in
foreground and display the results online.

//SYSLIB   DD DISP=SHR,DSN=<loadlibname>
//SYSPRINT DD SYSOUT=*                       
//SYSIN    DD *                              
 LISTIDR MEMBER=<programname>
```

```
My shop uses PDSMAN, a collection of utilities from LEGENT, sold now by
CA. Very pretty tool, but not free.

In your case, you could use AMBLIST to analize the loadmodules. You can
call AMBLIST from your ASSEMBLER, COBOL or PL/I program and interpret
the output. SYSIN and SYSPRINT are '&&files'. Pseudocode:

read first membername of library
while not last membername of library:
  open SYSIN
  write commands to SYSIN
  close SYSIN
  call AMBLIST
  open SYSPRINT
  read SYSPRINT until EOF
  close SYSPRINT
  read next membername of library

AMBLIST is an IBM utility, you find the description in:
OS/390 V2R8.0 MVS Diagnosis: Tools and Service Aids, Document Number
SY28-1085-06

O yes, I forgot: always call utilities with a parameter, even if you
don't need a parameter. It has to be char varying, in COBOL declare it
like:
01 parm-to-utiltiy.
   05 ll pic x(4) binary value zero.
   05 parm-content pic x(100) value spaces.
```
