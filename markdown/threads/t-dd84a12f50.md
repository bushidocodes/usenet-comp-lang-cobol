```
Hi all,

I am quite new to COBOL, but have tons of C, C++, and ASM experience.

I have, for whatever reason, a field like this:

05  ADDR  PIC S9(9) COMP-4.

A call to an assembler program is used to place in this var the address
of a field, in another program:

05  SOURCE PIC X(5)

CALL MYAMSPRG USING ADDR, SOURCE.

Basically, to keep from using the whatever the COBOL equivalent of address
getting is.

Now, I have another field,

05 TARGET PIC X(5).

I need to move the data pointed to by ADDR to TARGET.  (and remember
SOURCE is in another program, so I can't just MOVE SOURCE TO TARGET).

How do I do that in COBOL?

In C, it would be: memcpy(target,addr,5).

Thanks
Chip Scardo
```

```
Private email received which stated,

"Thanks...
IBM VS COBOL II Release 4.0"

    ***

Therefore, the answer is,

1) Change the definition of ADDR to

    05  ADDR Usage Pointer.

2) Keep your CALL statement as,
  CALL MYAMSPRG USING ADDR, SOURCE

3) Add a new entry in your LINKAGE SECTION as follows:
  01 Temp-TARGET PIC X(5).

4) After you call statement add the following two COBOL statements:
   Set Address of Temp-Target to ADDR
   Move Temp-Target to TARGET

      ****

After this statement, your 05-level "TARGET" will contain the contains of
whatever is at the storage location passed from your assembler routine.

NOTE TO C.L.C. readers,
  VS COBOL II does support USAGE POINTER, but SET ADDRESS can only be done on
a Linkage Section item.  No COBOL compiler (that I know of) allows a SET
ADDRESS of an 05-level.
```

```
the 15 or 20 most commonly used commands (ok, maybe 25).  But then, I've always
been simple-minded.
```
