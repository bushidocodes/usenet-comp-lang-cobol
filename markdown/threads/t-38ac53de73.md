Did you have Validation Rules on the field? If so you must add "Or is NULL"
to the end of the validation rule text.

"Indicator Variables"  are NOT the same thing as a normal variable in SQL.
These are used to determine whether a field has a NULL value or not and are
only accessible under certain special circumstances. It would appear that
the system believes the field in question to be NULL when you have said this
is OK...

How did you create this table? If you did it with an SQL Create, did you
specify "NOT NULL" for the column? We would expect ACCESS to then define it
as REQUIRED=YES but maybe something happened during the Create...


Try removing the field completely, then do the definition manually in
ACCESS. If there are indexes or relationships attached to the field, this
will highlight them and that could be where the problem lies.

If none of this works, mail me the database and your source code.
