Faxing reports out is relatively easy. I still have a couple of smart
fax modems that are accessed as a serial text printer from DOS or,
well anything actully, and if the first thing is 'FAX 1234567' then it
is faxed to that number, if not it is sent through to the printer that
is connected to it.

There have been FAX 'print drivers' for Windows since 3.0. Some of
these, however, tend to need manual intervention of some sort, such as
selecting where to go.

The really hard part is 'confirm they got sent' if you want the
_program_ to know.  Obviously the user can check the print queue for
what is still there.  'Confirming' that it actually resulted in black
marks on paper at the destination is impossible.  There is no way to
determine if the fax machine ran out of ink (where used) or paper or
the whole thing was thrown in the bit bucket.

> PC platform, MS compiler.

The last _MS_ Compiler released was 5.0, which was roughly equivalent
to MF 3.0 - and could do MS-DOS and Windows 3.x 16 bit programs (MS
had ripped out the OS/2 support which was in MF and MS 4.5).

The 'PC platform' may be some form of MS-DOS or compatible (I still
have some use of Multiuser-DOS/Real32) or Windows 3.0 or later.

You may need to be more specific to get a _useful_ answer.  Is it
MS-DOS programs or Windows 3.x ?

A possible solution is to use efax.com. You can format the report in any of
15-20 formats including .txt and .doc, email it to efax. They will send it to
the user's fax machine or email it in their highly compressed format, and send
you a confirmation report.

Printing on a fax machine is expensive. The user might prefer to receive it as
email so (s)he can print it on regular printer. Having a file also simplifies
archiving. Fax output tends to get lost/trashed. I've had to send important
documents two and three times.Your request for confirmations indicates you want
to eliminate lost reports, yet you've chosen the least reliable delivery
mechanism. Email will eliminate lost reports much more effectively than
confirmations. 

The user can Annotate an emailed image and send it back to you, just like a fax.
I email my time sheet to my manager in efax format. (S)he signs it
electronically with a signature image and emails it back to me. Then I send the
signed timesheet to my contracting company. The process is completely paperless
until it reaches the contracting company and timesheets never get lost. It takes
less than an hour from beginning to end. When I was faxing timesheets, they had
three opportunities to get lost .. and were lost more than half the time.

I didn't realize how long it's been since I've visited this group, but we
have an interface to Winfax Pro using the older MF 3.4 Cobol compiler using
DDE commands and setting the default printer to Winfax Pro on the Fax
server, as others have suggested.  It works pretty well, and the Winfax
software handles all the logging, busy retries and it even allows two modems
to be used at the same time, if your volume warrants it.  The only time
manual intervention is required is if Winfax goes past the number of
retries.  But an operator can just right click on the "bad" fax and select
"send now" to resend it.  It's not real sophisticated, but it's been doing
the job for 5+ years without many complaints and it's pretty cheap (I think
Winfax Pro is around $100).  If you'd like more info, shoot me an E-mail.
