Micro Focus may be the industry leader (i.e. top seller), they do not
specialize in COBOL screen handlers. Flexus is in fact the only company
which specializes in user interface tools for COBOL programmers.

In addition, many of the tools offered by the compiler companies are not
truly COBOL oriented. They require the programmer to use proprietary,
non-COBOL syntax to control screens or a derivative of BASIC for screen
handling.

The person requesting information on COBOL screen handlers said, (and I
quote) "I've found it easy to program, logical in syntax, and damn'd easy
to debug."

I would think that this person is looking for a solution to screen
handling that uses COBOL and only COBOL to control screens.

COBOL spII from Flexus allows the programmer to use native, ANSI standard
COBOL to control screens in both character mode environments and GUI
environments.
