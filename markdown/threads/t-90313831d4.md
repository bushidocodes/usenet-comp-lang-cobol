PLTPI programs execute when CICS is started. PLTSD programs execute
when CICS shuts down. I would write a program to be included in your
PLTSD list which would send an MQ message to your message program that
would contain a code to request the program terminate. modify the
message program to recognize the code and terminate. The only
drawback is that CICS shutdown will be delayed while the message
program waits for the timer to activate it so that it can retrieve the
message.

If you're asking how to cancel a CICS task that you have scheduled with
a START command, this is fairly straight forward. Specify your own REQID
when you issue the START command (REQID is an 8-byte field). When it's
time to shut down, issue a CANCEL command with the REQID() option,
specifying the REQID you used in the START command.

If you want to cancel a suspended task (one which is running but which
is suspended, as opposed to active), this is a different matter. There
is an "EXEC CICS INQUIRE" command which will give you a list of
transactions in the system and their status (you may have to issue
another command to get the details, "COLLECT", maybe). I'm not certain,
but I suspect you might be able to issue an "EXEC CICS SET" to cancel a
suspended task (you've just gotten the specifics of it thru the INQUIRE
command). Unless someone else is certain this is either true or false,
you'll have to check the doc from IBM.
