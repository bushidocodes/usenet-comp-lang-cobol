Hi Vincento.

On UNIX the environment variable names -- amongst other things -- are case
sensitive.

You need to ensure COBDIR is set to the root directory of the COBOL install,
with PATH set to include $COBDIR/bin, and SHLIB_PATH (or LD_LIBRARY_PATH for
64-bit execution) to include $COBDIR/lib .

Simon.

Hi Vince.

Thanks for sending on the mfpoll content -- I can now see the issue, which
appears to be a mismatch of environment variable settings and your working
mode (i.e. 32-bit or 64-bit).

In 32-bit mode, the OS dynamic loader will search SHLIB_PATH, yet in 64-bit
mode, the dynamic loader searches LD_LIBRARY_PATH. Although your setting of
LD_LIBRARY_PATH includes $COBDIR/lib, SHLIB_PATH does not.

Assuming you want to work in 32-bit mode, adding $COBDIR/lib to SHLIB_PATH
will resolve your problem.
