below is some portions of code using MicroFocus Netx and escape sequences
defined in HP's PCL 4&5 Printer Language. It is used to drive HP LaserJet II &
III printers.  Notice that no Windows API calls are used.  This code works
on all versions of Windows including Windows 2000.

Large portions of code have been removed.  Enough has been shown to give a 
rough
idea on how HP's PCL escape sequences can be used to control their printers
without having to resort to calling Windows printing API's. As you can see it
takes a lot of code to do anything very fancy.

All of the escape values were obtained from the HP PCL 5 Printer Language
Technical Reference Manual.

Dave

           SELECT Form-Printer ASSIGN TO Printer.
       DATA DIVISION.
       FILE SECTION.
       FD  Form-Printer
           LABEL RECORDS ARE OMITTED
           Linage IS Line-Count
           Lines AT Top 0
           Lines AT Bottom 0.
       01  Print-Line.
           03 filler                 pic x(5).
           03 form-area              pic x(80).
           03 filler                 pic x(59).

       Working-Storage SECTION.
       01 Working-data.
           03 Line-Count             Pic 99 value 0.
           03 Line-Count-66          pic 99 value 66.
           03 Spacing                pic 99 value 0.
      *****************************************************************
      *          Variables Used to control HP LaserJet III Printer
      *
      *    Symbol set .... "10U" is the HP PC-8
      *    Symbol Type ... "s1p" for proportional Scaling
      *                    "s0p" for fixed
      *    Style ......... "1s" for ITALIC
      *                    "0s" for UPRIGHT
      *    Stroke Weight . "3b" for BOLD
      *                    "0b" for MEDIUM
      *    Typeface ...... "4101T" for CG TIMES (proportional)
      *                    "4148T" for UNIVERS  (proportional)
      *                    "3T"    for COURIER
      *                    "0T"    for Line Printer

       Procedure division using Basic-Rec Work-Areas No-Copies.
      *Procedure division.
           OPEN Output Form-Printer.
           PERFORM 4000-Reset.
           PERFORM 4010-Initialize.



      * Prepare to print The Header Line
           MOVE "24.00"      TO P-Point-Size
           MOVE Italic       TO P-Style
           MOVE Proportional TO P-Symbol-Type
           MOVE Bold         TO P-Stroke-Weight
           Move P-Esc-Seq    TO Print-Line
           PERFORM 6010-Printing.
           MOVE Header       TO Print-Line
           Perform 6010-Printing.

      * Prepare to print the qualifier line
           MOVE "12.00"      TO P-Point-Size
      *    MOVE "10.00"      TO P-Point-Size
           MOVE Italic       TO P-style
           MOVE Proportional TO P-Symbol-Type
           MOVE Medium       TO P-Stroke-Weight
           Move P-Esc-Seq    TO Print-Line
           PERFORM 6010-Printing.
           MOVE Qualifier    TO Print-Line
           Write print-line after advancing 2 lines

      * Change to Fixed Courier Font for Form Printing
           MOVE Italic       TO F-style
           MOVE Bold         TO F-Stroke-Weight
           Move F-Esc-Seq    TO Print-Line
           Perform 6010-Printing
      * Write First Block
           MOVE Print-Line-1 TO Form-Area
           Write print-line after advancing 1 lines
           MOVE Print-Line-2a TO Form-Area
           Write print-line after advancing 1 lines


      * Change to Proportional scalable CG Times Font for the Footer
           MOVE "12.00"      TO P-Point-Size
      *    MOVE "10.00" to P-Point-Size
           MOVE Upright      TO P-style
           MOVE Proportional TO P-Symbol-Type
           MOVE Medium       TO P-Stroke-Weight
           Move P-Esc-Seq    TO  Print-Line
           PERFORM 6010-Printing
           MOVE Footer TO Print-Line
           Write print-line after advancing 5 lines

      *----------------------------------------------------------------------
      * Examples of shading various areas of the form
      *----------------------------------------------------------------------

      * Area-Width = 30 x Number of characters wide (69)
      * Shade the Title Areas (Date Line)
           MOVE "165"  TO X-Pos
           MOVE "164"  TO Y-Pos
           MOVE "2070" TO Area-Width
           MOVE "61"   TO Area-Height
           MOVE "08"   TO Area-Fill-ID
           MOVE Shaded-Fill TO Area-Fill-Type
           MOVE Area-Fill-Seq TO Print-Line
           PERFORM 6010-Printing

      * Shade the Title Areas (Employee Line)
           MOVE "165"  TO X-Pos

           MOVE Line-Count-66 TO Line-Count.
      *    This sets TOP Margin to 0 lines [esc &k0E] *
           MOVE X'1B266B3045' TO Print-Line.
           PERFORM 6010-Printing.
      *    This turns PERF Skip Off         [esc &l0L] *
           MOVE X'1B266C304C' TO Print-Line.

      *    Set up Number of copies
           MOVE No-Copies TO Copies

lines. To open a cash drawer, I have to send an escape sequence to

character of the print line, it doesn't work. Put the escape sequence
starting in column 3 or something and it works.

program inserts the PCL escape sequences in the print line. This

You could be entering into a bit of a sledgehammer and a nut situation here.
I've written a COBOL wrapper around the 16bit WIN API using the MicroFocus
compiler. This works well enough on WIN3.1 95 98 and NT. It includes the
features you mentioned plus a couple more like retaining the last selected
font information on disk, and correctly calculating the rendering size for
the font on an output device. Unfortunatly I'm not at liberty to supply you
with the code here.

I can say that this is not something you'd really want to do unless you're
serious about using the Windows GDI.

You'll need:
    1) a COBOL compiler that supports Windows API programming, I've only
used MF's and can't comment on this support from other compilers.
    2) The windows development kit
    3) Understanding of C is useful for reading.....
    4)A book - Programming Windows 3.1 Charles Petzold or the later 5th
edition (WIN32 API) if your using a 32 bit compiler, are widely recommended.
Either of these books includes the windows development kit, I think.

If you are already writing Windows programs via the API from COBOL then
perhaps I can give you some of the information you require. However, if
you're not yet doing that then I'd advise looking at an alternative solution
or consider learning the Win API first.

hope this helps,

Pat
