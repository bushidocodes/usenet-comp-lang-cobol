the comp.lang.cobol newsgroup).

GUIs have a lot of advantages, but they are also more work.

Oh, one more thing about Multi-Edit.

I have bought upgrades via the internet.  When I install, I keep the
MEWSETUP.EXE file.  I can run it any time to create all of the icons and
registry settings I need.  When I added a second NT partition, I ran
this and was able to run Multi-Edit without installing from scratch.  

Most Windows applications force you to install from scratch when you
want a second OS to see it.  Or when you reinstall Windows or move the
application from an old computer to a new one.  This is a terrible idea,
especially in the era of web based upgrades.

Please install a SETUP file in your applications which will correct this
terrible design deficiency!!

Line control is worse, but I like being able to type in the following
commandS:

X ALL;CHANGE P'<' P'>' 7 72 ALL  (exclude all lines, change capitalize
everything between columns 7 & 72 while unexcluding thos lines with
changes!)

With TSO, I use the exclude and flip commands a lot:

X ALL; F P'#' 8 ALL    (This finds every numeric in column 7.  I then
exclude everything from the top to PROCEDURE, and can see all paragraph
names, procided they are in the format of 0000-M-PARAGRAPH.

Some editors let me automatically put my initials and change request in
columns 73-80 where I make changes.

I haven't yet seen a PC based editor which excluded lines the way I want
(but I have seen some of the other stuff I wrote above).
