You can, syntactically, include a file name as one of the parameters of the
CALL statement. For example:

call 'subra' using work-area, infile


where 'infile' is the name in your select and on your fd. 

But there is no way to 'catch' the file. There is no 'linkage section' for
files, in other words.

Alternatively, look at the EXTERNAL attribute for files. You can share files
without passing them.
