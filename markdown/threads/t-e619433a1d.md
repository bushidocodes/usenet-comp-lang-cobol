I have developed centralized COBOL systems and a very large distributed
COBOL system. The strengths of development on the different platforms does
not relate to the language itself. The COBOLs on MVS and CA-REALIA (that
we used) are very similar in syntax, compiler and file handling. Building
the application is vastly different because you have different base
environments. MVS-CICS reentrant programming is going to look miles apart
from PC code to control Windows-based screen handling. You need to answer
big questions first in the system, such as - how much will my MF app affect
my company's CPU usage and licensed software charges? Or, how large of a
PC will my users need to buy to run this application?

In short, the technique needed will change the use of the COBOL, but the
language itself renmains much the same on the 2 platforms.
