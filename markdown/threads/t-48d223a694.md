```
I have been using COBOL for 28 years and I would like to interject
some comments in these discussions.
First, My primary rule for programming standard is
understandability, not to me, but to others.
I have converted many programs from old straight-line code to a
structured code with no GO statements. I have also modified many so-called
structured programs removing code where someone has a perform paragraph
which then only performs other paragraphs, etc...
I feel that all programming should strive for simplicity in readability
and comprehension, not to follow arbitrary rules such as tiny paragraphs.
I admit to being guilty to not using enough documtation commentary in my
programs, but I feel it is important that both data names and paragraph
namesbe descriptive.
Another practice I have seen used often which I remove form as many
programs as possible, is putting multiple statements onto a line
often breaking them up to you might have two or more
half statements on a line. I firmly believe that it is important to
to put only one statement on a line. This not only makes it
easier to read the train of logic, but also makes it easier to insert and/or
delete statements from a chain.
I have many other ideas but I appreciate the chance to
"sound off" here. Thank you!
```
