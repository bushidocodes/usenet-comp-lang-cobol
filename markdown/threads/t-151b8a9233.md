Hi,

Hopefully someone can offer some insight to this problem. The software program
i have developed runs under DOS. This program communicates to various serial
devices like a pole display, printer, and audio box. This all works fine.
When i sometimes (not all allways ) have a problem is when I load a TSR called
Bhost during my autoexec. This program (Bhost) allows me to call into this
computer with a different program called Blast. With BHost loaded as a TSR the
pole display may not function or other randam serial problems may occurs. I
believe that I have found the key to the problem. If BHost is NOT loaded and I
use the MSD function and check the IRQ's the system shows that the serial ports
are being "handled" by the BIOS. If I load Bhost and do the same MSD the
system tells me the serial ports are being "handled" by BHOST". What is really
strange about this problem is that i can take ten computers and only two or
three might have the problem and the problem is not allways the same. But when
the problem does occur it surfaces with a serial device ( slow printing, pole
display not working etc.). Could anyone offer any suggestions ?

Thanks in advance,

Bob Hennessey
