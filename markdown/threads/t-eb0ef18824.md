There are several options, including

- a freely available COBOL-to-C translator,

- a COBOL interpreter (somewhat commercial; freely available),

- AcuCOBOL, a commercial COBOL compiler,

- There's a "free" MS-DOS-based COBOL interpreter that should run in
emulation mode using DOSEMU,

- Rumour has it that someone is working on a "GNU COBOL;" no news has been
heard about this in about 6 months, so the project may be dead...

See: <http://www.hex.net/~cbbrowne/languages.html> for more details.
