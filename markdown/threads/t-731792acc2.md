```
I know this sounds like a beginner's question, but does anybody know how to
get Fujitsu to put a formfeed into a line sequential file? The following
standard Cobol does not work:

000010 IDENTIFICATION DIVISION.
000020 PROGRAM-ID. "TESTIT".
000030 AUTHOR. D. L. Tees.
000040 ENVIRONMENT DIVISION.
000050 INPUT-OUTPUT SECTION.
000060 FILE-CONTROL.
000070     SELECT PRINT-FILE ASSIGN TO "\TEMP1\X"
000080         ORGANIZATION IS LINE SEQUENTIAL.
000090 DATA DIVISION.
000100 FILE SECTION.
000110 FD PRINT-FILE
000120     LABEL RECORDS ARE STANDARD
000130     DATA RECORD IS PRINT-LINE.
000140 01 PRINT-LINE  PICTURE X(80).
000150 PROCEDURE DIVISION.
000160 MAIN-LINE.
000170     OPEN OUTPUT PRINT-FILE.
000180     MOVE "THIS IS A TEXT LINE" TO PRINT-LINE.
000190     WRITE PRINT-LINE AFTER PAGE.
000200     CLOSE PRINT-FILE.
000210     STOP RUN.

If I assign the file to "printer" then it will *print* the formfeed.
Unfortunately, the computer that it is going to run on does not have a
printer.  The file is written to disk, then printed later.  It also appears
to be impossible to use a network printer. As soon as the file is written to
anything but "printer" then all the logic changes, and and the form control
simply stops working.
```

```
Hi Tom.

PROCESSING MODE is only for PRESENTATION FILES.
This is not the case.

In this particular case if the organization is LINE SEQUENTIAL it
will strip the "^L" and "^M" from the file.
If the organization is SEQUENTIAL it will keep the"^M" but it will 
remove the "^L" if this is the first char of the file.
IF we do a
WRITE AFTER 1 and then a WRITE AFTER PAGE then we will have a "^L"
on the file.

This is one of the many thinks that makes Fujitsu COBOL hard to work
with.


Frederico Fonseca
```
