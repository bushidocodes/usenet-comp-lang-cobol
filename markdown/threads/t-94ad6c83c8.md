```
I have used the GOBACK verb in the past, to return a result from a nested
sub-program to the calling program. Now I can't do it. The COBOL
compiler is giving me "301-S ***... Unrecognized verb" for the line
containing the GOBACK statement. I can do a "goback giving ..."
in the main program, but in the nested subprogram I can't do either
a "goback giving..." nor an "exit program giving ...". It does accept
a simple "goback." statement, with no attempt to return a value.

Vic Odhner Phoenix AZ (Sample program follows)

$SET NESTCALL
$SET MF
Identification Division.
Program-ID. FOO.
Environment Division.
Data Division.
Procedure Division.
call "BAR".
goback returning 0. <<== Compiler doesn't object here.
* ====== SUBPROGRAM ===========================================
Identification Division.
Program-ID. BAR.
Environment Division.
Data Division.
Working-Storage Section.
77 PROG-STAT pic x(01) comp-x value zero.
Procedure Division.
goback giving PROG-STAT. <<== "Unrecognized verb" here.
End Program BAR.
End Program FOO.
```

```
The definition of GOBACK is:

GOBACK [{ RETURNING } ] { ADDRESS OF data-name1}]
[{ GIVING } { literal-1 } ]

So it is OK with RETURNING 0 because this is a literal, but
it needs to be:

GOBACK GIVING ADDRESS OF PROG-STAT.

Hope this helps.
```
