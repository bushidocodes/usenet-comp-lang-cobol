There must be a bug in your compiler. That is allowed in the
standard. What is not allowed in the standard but is allowed in many
implementations is that you cannot reference an integer function in a
MOVE, only in an arithmetic expression. To conform to the standard
you would have to say:

COMPUTE xxx = FUNCTION INTEGER-OF-DATE (date-item (idx))

This rule is extremely dumb in my opinion and has been removed from
the next standard.

Another possibility is that date-item is not defined as an integer.
Exactly what did the diagnostic say?
