```
We've written a Stored Procedure in COBOL which runs under DB2 7.1 in
a VSE environment.  The procedure is called by Java via JDBC (IBM's
DB2 driver over DB2Connect).  Though we've followed to the letter all
the examples we've seen, we can't get the Stored Procedure to return
values to the Java program.  Here's the setup.  Can someone tell me
what we're doing wrong?  Thanks.

DDL:
CREATE PROCEDURE PROCSAM1 (IN ITEM CHAR(10),                        
  OUT RESULT CHAR(80))                                               
  EXTERNAL NAME PROCSAM1, SERVER GROUP, PARAMETER STYLE GENERAL,     
  COMMIT ON RETURN NO,                                               
  LANGUAGE COBOL, RESULT SETS 1;

COBOL:
LINKAGE SECTION.                                   
       01 MITEM-ID        PIC X(10).                     
       01 RESULT-STRING   PIC X(80).                                  

PROCEDURE DIVISION USING MITEM-ID, RESULT-STRING.            
      * Do Some Program Logic (not shown)
        MOVE 'EVERYTHING WORKED JUST FINE.' TO RESULT-STRING.
        GOBACK.

JAVA:
  CallableStatement cstmt = con.prepareCall("CALL PROCSAM1(?, ?)");
  cstmt.setString(1, "1234567890");
  cstmt.registerOutParameter (2, Types.CHAR);
  cstmt.execute();
  System.out.println(cstmt.getString(2));
  
Unfortunately, it always prints "null".  

Bill Dennis
```
