This is a unix limit.   Later releases of unix have a 'different' file
system (I can't remember its proper name) which extends this but AFAIK
MicroFocus are not going to support this.

From MFCobol V4.1 they have something called file striping which allows a
file to be split into multiple files.  Unfortunately this is not automatic
and has to be planned for.  I haven't used it and it seems very clumsy to
me.

Be warned, later versions of MFCobol change the default locking method.
This means that shared index files start causing problems over 1GB.  This is
because the unix limit is 2GB on files opened exclusively but only 1 GB on
files opened shared.  To continue to use files > 1GB you need to set the
lock mode RTS tunable setting back to the V3 method for the runtime.

Regards
Rick
