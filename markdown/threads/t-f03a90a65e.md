```
We are using MicroFocus Object Cobol 4.1.10 on a Solaris 2.6 system.
The global "mfcobolrc" file has the LANG argument for the [EDIT-USER]
tag set to XOPEN, OSVS, ANS85, VSC2(1), VSC2(2), VSC2(3).  When we use
"tbox", we must toggle through these six languages before it is set to
blank each time we log on. Is there a way to set the LANG argument so it
will default to blank in our local ".mfcobolrc" files?

Thanks,
Rob Toftness
University of Wisconsin - River Falls
```

```
Unfortunately I don't believe that it is possible to set the default to be
blank if you want to keep the alternative options. If you only want to use
the blank option then simply delete (or comment out) all of the LANG lines.
Then you'll just be left with the blank default.

If you do want to keep the other options then my best suggestion is to add a
new LANG entry before the XOPEN one and call it something like DEFAULT (you
also have to specify at least one directive). So you'd be adding a line
something like:

LANG DEFAULT   MF

Regards.
Alan.
```
