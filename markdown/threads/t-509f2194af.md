First, I'm confused - what is a vse/MVS environment.  Usually, a environment
will me OS/390 MVS or VSE, but usually not both (in my limited experience).

second, have the programmers read and understood the migration guides for the
languages in use??
I recommend checking the LIBRARY option at the following site
http://www-4.ibm.com/software/ad/cobol/
Lots of good, free information there.

In the case of COUNT, the migration guide table of comparisons with OS/VS COBOL
compiler options says

>>COUNT/NOCOUNT  Similar function is available in the Debug Tool.        <<

In the case of FLOW, the migration guide says

>>FLOW/NOFLOW 
|Similar function is available using the Debug Tool. Compile your program with
the TEST(ALL,SYM) option and use the following Debug Tool command: 
  "AT GLOBAL LABEL PERFORM; 
         LIST LINES %LINE; GO;    END-PERFORM;"                  <<

While this may not be what your programmer want to hear, I'm not sure of a
valid reason for using these resource hogs outside of a DEBUG session - you
certainly wouldn't want them in your production environment! (my not-so-humble
opinion).
