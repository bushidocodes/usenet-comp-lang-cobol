```
Did you check the file status on the OPEN? If the OPEN failed, you'd get
that error when you tried to read it or write the file.

I see this a fair amount of the time - the programmer uses the same FILE
STATUS item for all the files (which is fine), but then has something like
this:

PERFORM OPEN-THE-FILES


OPEN-THE-FILES.

  OPEN INPUT  File1  File2
             I-O       File3 File4
            OUTPUT File5.

If you get an open failure on File2, the status which can point to the
problem is gone as soon as the OPEN is attempted on File3.
(You get this with "virgin" VSAM files on IBM mainframes real easily -
whenever you re-define the VSAM file, it cannot be opened I-O until it
contains a record or has been opened output).

Better to use something like...

PERFORM OPEN-THE-FILES
  Varying File-sub from +1 by +1 until File-Sub GREATER Num-Of-Files.



OPEN-THE FILES.
  EVALUATE File-Sub
    WHEN 1
      OPEN INPUT File1
    WHEN 2
      OPEN INPUT File2
....
END-EVALUATE

   IF WS-FILE-STATUS NOT EQUAL '00' THEN
     whatever
   .....

MCM
```
