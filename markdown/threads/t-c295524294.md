That's real savvy advice Jerry. Although I confess I still use the
DOS-based editor that came with Microsoft COBOL V5 - in preference to
the NetExpress Editor - not that there's anything wrong with the latter,
but in searching for text I find the former currently quicker.
In response to a query from Merant, I've already advised Newbury of the
'small' enhancement that would get me back into their stable.

And as you rightly say I lose 'my own' editor's features if I use a
foreign one - you can go real Technicolor in N/E highlighting reserved
words, typos that don't find a variable etc... AND I've used a very
powerful feature of N/E to 'find string xxxxx in all files' against my
old RM/COBOL source files !

So certainly for the uninitiated - stick with your compiler's editor.

I find that a powerful programmer's editor offers so many productive
features, that built-in editors pale by comparison.  I feel like I am
programming with one hand tied behind my back. :-)  Good editors do
offer syntax coloring and other features such as language templates.
I usually use my personal editor to write code or make large changes,
and use the built-in editors only for small changes during debugging.
I use Multi-Edit, which is inexpensive and very powerful, but there
are other very good editors out there.  Getting a new programmer's
editor is like getting married, and tastes preferences greatly.  And
changing editors is like getting divorced and remarried, traumatic. :-)

No, M/S took M/F COBOL V3 and rewrapped it as their V 5.0 accompanied by their own Programmer's Workbench. Although in text it has a Windows look with drop-down menus etc.

As I said, nothing wrong with the N/E editor but I do find the following

		MyApplication 	class is "myapp"

Highlight that 'myapp' Alt F to open then add the extension '.cbl'
