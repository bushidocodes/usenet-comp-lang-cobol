My first thought is that this would not work as the Windows Operating System
would be the first to complain rather profusely about the SYSTEM.DAT
information not matching the configuration of the 'new house' that the 
OS finds itself in.
If you manage to get past that part of the situation, then I would imagine
that PowerCOBOL would operate in a 'fat-hapy-dumb' mode as all the 
registry/file markers are contained on the portable drive (assuming that
this portable drive has the Boot OS and all).
If you are talking of a pluggable drive separate from the main Boot drive,
I would very much doubt the probability of success.

just my two-bits worth here.
