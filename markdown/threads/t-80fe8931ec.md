```
I would like to use our Teradata DBC/1012 SQL data base computer in our PC 
environment.  Currently all of our development work is done on the PC 
with the exception of programs which contain imbedded-SQL.  Our CODASYL 
data base is emulated, this allows us to compile and test mainframe 
programs.  This test procedure falls apart when a program with 
imbedded-SQL is encountered.  Our perferred direction would be to access 
the DBC/1012 computer directly for our testing.  We have the API provided 
by Teradata (Now AT&T GIS), but I don't wish to write a pre-compiler for 
imbedded-SQL.  How do I get a Teradata DBC/1012 SQL pre-compiler for 
Micro Focus COBOL.  If one is not available, does anyone have source code 
for a SQL pre-compiler.  This would make my life easier.
```

#### ↳ Re: Micro Focus SQL Pre-compiler

- **From:** Alan Wheeler <71333.2247@CompuServe.COM>

- **Message-ID:** `<3cpu04$9ct$2@mhadg.production.compuserve.com>`
- **References:** `<3cn6db$mbl@rigel.infinet.com>`

```
To my knowledge, there is no commercially available SQL 
precompiler for the Teradata database for use with Micro Focus 
COBOL.  There are a number of other SQL precompilers available, 
from IBM's DB2/2, to MS SQL Server, to Sybase, Oracle, Informix, 
Ingress, XDB, and maybe a few others.  If none of those will 
offer you reasonable compatability to the Teradata database, or 
embedded SQL code, then you could use the Micro Focus 
Preprocessor facility, which is part of their Workbench product, 
to write what ever precompiler that you want.

Regards, Alan.
```
