Pépé le Pew wrote:

>           IF EV-STATUS NOT = '00'
>
>          Check the file status after each OPEN, READ, WRITE,  START ... etc

When checking a file status result it is important to check successful
operation by testing the first byte only for zero.  There are success
codes that are not '00'.  For example a successful open of an optional
file will give a file status of '05' and a sucessful read or write of a
record with a duplicate alternate key will give '02'.  Other values are
possible and these may increase with later versions of Cobol.

Well, that kind of depends on what you mean by success; you might care about

No, not always, only if the file was NOT PRESENT at the time of the OPEN. 
If the OPEN was for I-O or EXTEND, the file is created.  If the OPEN was

PRESENT at the time of the OPEN, and if that occurs I would expect a File 
Status value of '00', not '05'.
