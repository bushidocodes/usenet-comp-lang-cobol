```
Just because I haven't seen it mentioned yet, ...

Today, IBM announced the "long expected" CICS TS V3.1,

IBM CICS Transaction Server for z/OS V3.1 offers improved integration, 
application transformation, and enterprise
management (118.2KB)
    http://www.ibm.com/isource/cgi-bin/goto?it=usa_annred&on=204-285

Of particular interest to those involved  with IBM (in this environment) and 
COBOL, is the (*TOTALLY* expected) paragraph:

"OS/VS COBOL:
  As indicated in previous announcements, most recently in the announcement 
of CICS TS V2.3, CICS run-time support for OS/VS COBOL (5740-CB1, 5734-CB4, 
and 5740-LM1) is not provided in CICS TS V3.1. (Refer to the Application 
programming summary section)."

If you have ANY "OS/VS COBOL" still running in your shop (and even if you do 
NOT use CICS, you want to read the "writing on the wall"), I suggest you 
make certain your management know and understand this.

NOTE WELL:
  This is NOT simply a change in "service" type support.  As OS/VS COBOL was 
incapable of producing "truly RENT" code, CICS had software support to 
enable "serially reusable OS/VS COBOL" object code or work in a CICS region. 
This support is NOW REMOVED. and OS/VS COBOL object code simply will NOT RUN 
in a CICS TS V3.1 region (regardless of run-time or anything else).
```

```
I'll take a WAG at this.  

With dynamic AORs and CICSPlex, each execution of a transaction can 
occur on a different region -- load modules need to be loaded in both, 
and be unalterable or that does not work.  If you were to allow serial 
reusability, then you would have a region affinity until task completion 
because of the state of the load module.  And given the 
psuedo-conversational nature that CICS sometimes uses, how does it 
exactly know what 'task completion' is?

With truly RENT modules, CICS must only share the state of each instance 
of a program over various region sets instead of sharing instance state 
AND load modules.

Just a WAG...
```

```
certainly be recompiled and run under the new CICS TS.  The problem is that 
there were TONS of "undocumented extensions" not to mention '68 and '74 Standard 
"stuff" that is no longer supported in the current compilers (and hasn't been in 
any compiler that IBM produced since 1984 !!!).

This is why IBM has been giving so much warning that sites HAD to upgrade to a 
"currently supported compiler".  HOWEVER, there is also no "drop" date for the 
earlier release of CICS TS (that I know of), so shops can STILL keep using CICS 
TS V2 and using OS/VS COBOL applications.  (They must use the LE run-time to be 
supported).
```
