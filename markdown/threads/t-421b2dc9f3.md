```
MOVE DOUBLE-BYTE-COMP TO NEW-VALUE.

Some compilers permit the definition of a single-byte binary (COMP)
```

```
Assuming that your compiler supports ANSI/ISO Intrinsic Functions, then use
the following code (which is independent of ASCII/EBCDIC or any other
"table" implications)

01  Input-1-byte  Pic X.
01  Output-Num  Pic 999.
   ...
Compute Output-Num = (Function Ord (Input-1-Byte)) - 1

FYI - the "minus 1" is because Function Ord goes 1 thru 256 (or whatever)
while you probably want 0 thru 255
```

```
Assuming this is an IBM Mainframe environment :

03  WS-PACKED-X PIC X(02).
03  WS-PACKED-V9 REDEFINES WS-PACKED-X
                                    PIC 9(01)V9 PACKED-DECIMAL.
03  WS-DISPLAY-X PIC X(03).
03  WS-DISPLAY-V9 REDEFINES WS-DISPLAY-X
                                    PIC 9(02)V9.
03  WS-DISPLAY REDEFINES WS-DISPLAY-X
                                     PIC 9(03).
03  WS-INS-PLAN   PIC X(01) VALUE X'A8'.
03  WS-HWORD-X PIC X(02).
03  WS-HWORD REDEFINES WS-HWORD-X
                                    PIC S9(04) BINARY.

MOVE WS-INS-PLAN TO WS-PACKED-X (1:1).
MOVE X'0F' TO WS-PACKED-X (2:1).
MOVE WS-PACKED-V9 TO WS-DISPLAY-V9.
INSPECT WS-DISPLAY-X CONVERTING X'FAFBFCFDFEFF'
                                             TO 'ABCDEF.

At this point, WS-DISPLAY-X (1:2) is equal to 'A8'.

MOVE LOW-VALUE TO WS-HWORD-X (1:1).
MOVE WS-INS-PLAN TO WS-HWORD-X (2:1).
MOVE WS-HWORD TO WS-DISPLAY.

At this point, WS-DISPLAY is equal to 168.
```

```
The compiler converts one numeric format to another automatically.  If
the source field is two bytes (or more), make sure it is defined COMP
or BINARY, and just move it to a PIC 999 DISPLAY field.  If it is one
byte, define it as alphanumeric and move it to DIELD-X in the example,
then move HALFWORD to your PIC 999 DISPLAY field.

01  HALFWORD       PIC 9(4) BINARY
    VALUE ZERO.
01  FILLER REDEFINES HALFWORD.
    05   FILLER    PIC X.
    05   DIELD-X   PIC X.

Ummmm, if your computer is Little-endian You may have to reverse the
positions of FILLER and DIELD-X.

There will be one-byte true BINARY in the next standard; if your
compiler supports it now, you can avoid the HALFWORD stuff.
```
