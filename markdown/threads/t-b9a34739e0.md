We have the "high level" (EXEC DLI) commands available, but our shop still
uses the old CALL CBLTDLI commands.  I experimented with using the EXEC DLI
commands about six months ago (as far as I could tell only one other person in
our shop even knew they existed, and even he never actually used them).  I
got both batch and CICS programs working using OS/VS COBOL, and CICS working
using COBOL II, but I never could get them working for batch programs using
COBOL II.

In any case, while it has some advantages, I found that I did not like it's
error handling.  The call stuff isn't all that bad, once you figure it out.

We are, BTW, a VSE shop...
