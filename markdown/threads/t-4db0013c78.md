There have been several Year2000 project models floating around the
industry since last year. There are similarities among them, and I have
chosen parts of each of them to create the following brief outline. I
have left out issues of project control and management funding and
oversight for the sake of brevity.

1. Catalog affected entities and design solution (40%)
1.1 Inventory source
1.2 Determine affected areas
1.3 Determine approach to modifying each system
1.4 Determine tool and consulting needs
1.5 Develop modification plan
2. Change code (10%)
2.1 Change code per modification plan
2.2 Install new systems where pertinent
2.3 Develop test plan
3. Test (50%)
3.1 Based intricacy of system interrelationships

As I stated, this is a very brief outline. Each item can (and should!)
be developed in depth. In addition, there are vendor products available
to help address the issues of each stage of the project. One should be
familiar with these.

There are also Internet resources that contain a wealth of information
pertaining to Year2000 issues. Visit my webpage (below) for links to
some of these resources.

And good luck with your project!
