```
The sequence of actions to do this is as follows :

To set up the shared memory.

1) Decide on a name for your shared memory. This is what is used in
CBL_PUT_SHMEM_PTR
2) Allocate some memory to store your data string in using CBL_ALLOC_MEM.
Remember to set the flag to say that the memory is shared.
3) Set the address of a linkage item to the pointer returned by
CBL_ALLOC_MEM
4) Store your data string in this linkage item.
5) Pass the pointer returned by CBL_ALLOC_MEM to CBL_PUT_SHMEM_PTR using the
name you decided on in 1).

To access the shared memory

1) Call CBL_GET_SHMEM_PTR using the same name you decided on previously.
2) Set the address of a linkage item to the pointer returned.
3) Access the linkage item to obtain the data string.

Note that you can only access shared memory if the second program was
started by using CBL_EXEC_RUN_UNIT. Programs that are completely separate
cannot use this mechanism.

I hope this helps.
```

```
If (and only if) the two programs are run in the same "run unit" (under the
same "CALL" structure), then you can use the EXTERNAL attribute on the
definition of the data in both programs' working-storage and everything will
be done for you. This is true for any '85 Standard compiler.
```
