EXIT PROGRAM merely returns control to the calling program. It does
not affect the contents of WORKING-STORAGE. By default, the next
entry into the same subprogram will encounter WORKING-STORAGE in its
last-used state (apart from GLOBAL or EXTERNAL items which might have
been modified in the meanwhile).

› when I try to use the "CANCEL" command after the every call to the

CANCEL will work, but it incurs needless overhead. A simpler way
to ensure that WORKING-STORAGE is re-initialized every time you CALL
a subprogram is to include an INITIAL clause:

IDENTIFICATION DIVISION.
PROGRAM-ID FOOBAR INITIAL.

(I am describing VS COBOL II; I don't know if this syntax is available
in other dialects.)

You can also let the PROCEDURE DIVISION explicitly initialize
whatever needs to be initialized, but this approach is tedious.
