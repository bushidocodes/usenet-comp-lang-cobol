Theo, here's a reply from Stephen Gennard, Windows run-time
developer at Micro Focus:

======

It looks like you have not loaded the resource DLL inside you
application.

This can be done by:

call WINAPI LoadLibrary using
by reference "MyRes.DLL" & x"0"
returning hInstResource
end-call

When you have finished with the resource DLL (or at 'STOP RUN')
do a:

call WINAPI FreeLibrary using
by value hInstResource
end-call


When you register you Window class just register the menu name
and use the hInstResource for the resource hInstance.

For example:

if hWindow = 0
move 0 to style
add CS-VREDRAW to style
add CS-HREDRAW to style
set lpfnWndProc to entry "MyWndProc"
move 0 to cbClsExtra
move 0 to cbWndExtra
move hInstResource to hInstance
call WINAPI LoadIcon using
by value 0 size Word-Len
by value h"00007f00" size Dword-Len
returning hIcon
end-call


call WINAPI LoadCursor using
by value 0 size Word-Len
by value h"00007f00" size Dword-Len
returning hCursor
end-call
move COLOR-APPWORKSPACE to hbrBackground
add 1 to hbrBackground
set lpszMenuName to address of MenuName
set lpszClassName to address of MyClassName
call WINAPI RegisterClass using WndClass
returning tmpFlag
end-call
if tmpFlag = 0
exit program returning 0
end-if
end-if

call WINAPI CreateWindowEx using
by value 0 size Dword-Len
by reference MyClassName
by reference "Dialog Demo" & x"00"
by value h"00CF0000" size Dword-Len
by value h"8000" size Word-Len
by value 0 size Word-Len
by value h"8000" size Word-Len
by value 0 size Word-Len
by value 0 size Word-Len
by value 0 size Word-Len
by value hInstResource
by value 0 size Dword-Len
returning hWindow
end-call

Hope this helps....
