Easy-peasy (when you know how !!).

To get environment variables in the procedure division, do this:

01    ws-env-val    pic x(80).
....
display "LOGNAME" upon environment-name
accept ws-env-val      from environment-value

That's all there is to it. This will get you the name of the logged-in user
on Unix, but you can use to retrieve any environment variable values you
like. You can also set values with "display .... upon environment-value",
but this can only be used to communicate with other programs within the same
process, since you cannot pass values to a parent process.

Of course, if you want to manipulate file-names, as in your example, you can
code like this:
select my-file assign to external ABCFILE.

This will be resolved by the Micro Focus rte as follows:
if the parent shell has an environment variable set with this id, it will
use that:
export ABCFILE=/adir/in/data
or the old, but still supported dd_ format:
export dd_ABCFILE=diff_name.txt
If no variable exists, the filename used will be ABCFILE in the current
directory.

Oh, finally, beware of underbars in variable NAMES when using MF Cobol.
select afile assign to external FILE_1
select bfile assign to external FILE_2
will both look for the variable FILE, as everything from an underbar onwards
is ignored (extremely irritating!).

Pete Jones
