Offhand I can think of three methods.
-------------------------------------------------------
1. 03 TA-FIELD PIC 9(06)V9(02).
03 TA-FIELDR REDEFINES TA-FIELD.
05 FILLER PIC 9(06).
05 TA-FIELD-DEC PIC 9(02).

ADD .50 TO TA-FIELD.
MOVE 00 TO TA-FIELD-DEC.
-------------------------------------------------------
2. 03 TA-FIELD PIC 9(06)V9(02).
03 TA-TEMP PIC 9(06).

COMPUTE TA-TEMP ROUNDED = TA-FIELD.
MOVE TA-TEMP TO TA-FIELD.
-------------------------------------------------------
3. 03 TA-FIELD PIC 9(06)V9(02).

COMPUTE TA-FIELD ROUNDED = TA-FIELD / 100.
MULTIPLY 100 BY TA-FIELD.
-------------------------------------------------------
Top to bottom they go from most to least efficient, and most to least trouble
to code the data description.

Try using the rounded option of the compute statement.
For example
compute ws-result rounded = ws-one + ws-two
end-compute
If this doesn't work, try doing rounding manually by adding 0.05
to the result. Actually you may have to do this manually since using
the implied decimal is telling the compiler you want to keep what is
after
the decimal. Maybe you may have to redefine the field and use if
statements
to roll you own rounding.
for example:

01 ws-result pic 9(06)v9(02).
01 ws-result-redf redefines ws-result.
05 ws-result-whole-num pic 9(06).
05 ws-decimal-part.
10 ws-decimal-part-tenths pic 9.
10 ws-decimal-part-hundreths pic 9.

compute ws-result = a + b
if ws-decimal-part greater '50'
add 1 to ws-result.
or some such. I guess this isn't so easy...
I've seen old programs that add .5 or .05 to roll their own rounding...

Good Luck...
