```
The default is by reference but you can pass parameters by value if you want
or you can use both :

call "myprog" using by value parm1
                                  by value 156
                                  by reference parm2
and so on ...

of cource the same is valid when invoking a method : invoke aCustomer
"getName" using .....
```

```
FYI,
  "BY VALUE" is a (fairly common) extension; this means that depending upon
the compiler that you use, it may or may not be valid syntax.  (It will be
included in the next Standard - along with RETURNING phrases)

HOWEVER, no one has mentioned BY CONTENT (which is Standard) and which does a
"copy" of the parameter (i.e. any changes in the subprogram do not impact the
original data item).  As far as the subprogram goes, it LOOKS like it was
passed BY REFERENCE.
```
