This could be accomplished in a very easy way or a more difficult one. I
don't know RM so I can't tell if it supports DLL's or not.

The simple method is:
Have you heard of associations? That is, MS Win has associated an executable
with various file extensions. JPG is a well known extension and should be
associated with your internet browser if installed. Otherwise it could for
instance be Microsoft Photo. You may test this by at command line type:
"This is my jpeg file.jpg" followed by enter. If there is an association,
you should a) have your computer heavily working right now or b) have the
image on your screen.

So, assuming you have an association, I bet RM has a method for execution of
SYSTEM? If so, depending on their implementation of this command you should
be able to do this:

        CALL "SYSTEM" USING "This is my jpeg file.jpg"

and get the result you want. Whether the launch becomes syncron or
asyncronous depends on the RM implementation.

Sligthly more difficult is if RM deals with DLL's, then you would typically
rely on the same approach, but using Microsoft Windows API directly, for
instance CreateProcess or the obsolete WinExec. I mention WinExec because it
has a simpler interface and is still supported.

Finally, the most difficult option, get software for displaying jpg and link
it in.

Cheesle

I am happy if I could contribute, however, you have missed something, sorry

Do you remember my note about the associations in Windows? They make it
possible to launch documents rather than executables. Thus, instead of:

        CALL "SYSTEM" USING "IEXPLORE.EXE MyFile.jpg"

you could:

        CALL "SYSTEM" USING "MyFile.jpg"

To demonstrate this behavior, just find a dos prompt of your computer, look
for a .txt file and type:

        readme.txt    ENTER

replace readme with what you found, once you have pressed ENTER you will
have the file into  windows notepad (windows default, may be altered).

The benefit of this is obvious, you do not have to care about any
application, just make sure that the user has this association set.
Individual users may then have individual choices of what to use to display
a jpg file. I for one uses Microsoft Photo Editor. I agree with your
concerns that there is to much bells and whistles with IE, hence another
choice would definitely be preferable, if not for anything else so for the
size of IE.

Let me know if you still does not understand what I mean with associations,
