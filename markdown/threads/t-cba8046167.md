```

Consider
EXEC SQL INCLUDE SQLCA END-EXEC
and
COPY SQLCA
Both get the source for the SQL Comm Area, but the first one via the SQL
precompiler, the second via the standard Cobol compiler. Allthough in some
compilers the precompile and standard compile are integrated, it still can make
a difference. The first, exec sql include is the preferred way to do, because
the the sql precompiler also knows that sqlca *is* *indeed* copied. The cobol
compile will see the resulting data definitions as well and thus not complain.

Perhaps this helped?

The COBOL Frog
```

```
Hi !

We managed to solve the problem.

1. Sybase's Embedded SQL Precompiler does not expand copybooks,
so since the program was hanging up at a certain copybook while compiling,
we manually inserted the expanded copybook and fixed the syntax errors.

2. After fixing the errors we were still getting the SQLCODE errors, we
deleted all the files that were created by the animator eg. *.csi, *.idy,
*.int, *.msg, *.trc. After this when we compiled the code the program
compiled without any errors.

Thank you for all your suggestions and help.
```
