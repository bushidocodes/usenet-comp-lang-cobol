The standard DOS-window provided by Windows 3.1 does
not allocate the full 640k memory of a standard DOS
environment.

For my development work in DOS, running under Win3.1,
I set up my own DOS environment. Use the "pif" editor
to create a new file. Use "command.com" as the program
name. Under the memory options, list "640k" as both
"required" and "desired".

You then click on the program group where you want the
icon. Then click "file" and "new". Enter "prgname.pif"
(obviously, your own prgname) for the .pif file you previously
created. You can change the program name and icon as you
desire.

When you double-click on the new icon, you will get a DOS
session with the full 640k memory allocated.

If your MF application needs more memory than that, you
can change the parameters in the .pif file.



If there is a FAQ place for this newsgroup, please post
this answer there, as the question seems to come up frequently.
