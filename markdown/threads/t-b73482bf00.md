From one of my usally reliable sources,

"The Workbench updates are still available on the FTP site at
ftp://ftp.merant.com/updates/ (which is the top level FTP directory), and at
ftp://ftp.merant.com/updates/cobol-workbench-v4.0/non-security-keyed/windows
-nt-95/
for the Non Security Keyed Windows 32 bit version of Workbench
update directory.  The Y2K problem with the Setup program is on all of the
CDs created up through the end of last year.  There is a newer CD that is
available that has corrected that Y2K problem, which has both the corrected
Setup program for the base v4.0 product and includes the v4.0.38 update on
it as well.  In the directory above are two files, fix2kset.txt &
fix2kset.zip, which include the correction for the Setup Y2K problem for the
previously delivered CDs.  The instructions for using the correction are in
the fix2kset.txt file.  Customers who wish to obtain a copy of the corrected
CD can do so by requesting a copy of that CD on the Update Request page on
our web site located at
http://support.merant.com/websupport/upgstore/mfcdupgrade.asp, by calling
the SupportNet Answerline group (1-800-443-1601, voice prompt 3) and
requesting one be shipped to them, or through asking their sales person
(1-800-547-4000, voice prompt 3) to have one shipped to them.  Obviously,
prior to sending a copy of the new CD out, we will need to check to make
sure that they had purchased the product to start with, as the Workbench
product is not license keyed (except for the security keyed versions sold
mostly outside of North America).

I'm not sure why Dave was not able to find the update on the FTP site, as it
has been there for quite some time.  We are looking at moving all of the old
product updates, however, to make it easier to find in the future.  We are
in the process of adding a new facility to the SupportNet Knowledgebase,
where we will incorporate both an Update Locator in its function, as well as
include links from the relevant Knowledgebase issues that are corrected in
product updates and have those entries have direct links to the product
updates (and patches, etc.) that correct those problems.  Once we have this
fully up and running (probably not until late October or early November),
then we will be removing the updates from the FTP site, with replacing them
with a notice to look to the Knowledgebase Update Locator from that point
forward."
