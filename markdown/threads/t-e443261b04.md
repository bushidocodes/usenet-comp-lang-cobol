The x"B0" routine is described in the manual as machine dependent and
creates conflicts with other features. I would have thought that the
best ADIS mechanism is to use the Special-Names clause "CRT STATUS IS
Key-Status" and the various function key values are put into the data
area Key-Status.

The User function keys need enabling with CALL x"AF" if they are
outside the default range of Esc, F1 to F10.

The arrow keys are normally configured as 'adis keys' which are used
during and accept (ie don't terminate it) but this can be changed
using a CALL x"AF" to make them function keys and these can be changed
back with another CALL.

There should also be an adiscf (or adiscfw for Windows) which can do
all this configuration into the ADISCTRL configuration file. However,
as you probably want to change the usage only temporarily you would
need to use the CALL x"AF".

See ADIS section in your System Reference.

       special-names.
           crt status is crt-93-etat.

           02 key-1                           pic 9.
           02 key-2                           pic 99 comp.
           02 key-3                           pic 99 comp.
       *>Parameters for ADIS.
       01 Set-Bit-Pairs        pic 99 comp-x value 1.
       01 Parameter-Block.
           02  Bit-Pair-Setting    pic 99 comp-x.
           02  filler              pic x  value "2".
           02  Bit-Pair-Number     pic 99 comp-x.
           02  Bit-Key-Number      pic 99 comp-x value 1.
....
      screen section.
       01 ecran01.
 ...
      procedure division.
...
          move 0 to wss-i2
           move 3 to Bit-Pair-Setting
           move 27 to Bit-Pair-Number
           move 2 to Bit-Key-Number
           call x"af" using set-bit-pairs Parameter-Block
           *> enable crup et crdn
           move 1 to Bit-Pair-Setting
           move 5 to Bit-Pair-Number
           move 2 to Bit-Key-Number
           call x"af" using set-bit-pairs Parameter-Block

              display ecran01
               accept ecran01
               end-accept
               evaluate true
                 when key-1 = 0

                 when key-1 = 1

                 when key-1 = 2  *> cursor up dn
                   evaluate key-2
                       when CrDn
...
                        when CrUp
...
                   end-evaluate
               end-evaluate
...
           *> disable crup et crdn
           move 2 to Bit-Pair-Setting.
           move 5 to Bit-Pair-Number.
           move 2 to Bit-Key-Number.
