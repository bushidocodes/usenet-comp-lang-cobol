```
Am I correct that the file associated with PRINT-REC is a (line) sequential
file?  If so, it is the compiler that is wrong and not you.  The "solution"
to get around this is to add an END-WRITE statement immediately after the
WRITE statement that you have already.

If the compiler were working "correctly" - then the NOT INVALID KEY phrase
would be REQUIRED to go with the READ statement (because it is only valid
for an indexed file).  However, it APPEARS from the message that the
compiler thinks that you are allowed to have a NOT INVALID KEY phrase for
your (sequential) WRITE statement.

Can any of the Fujitsu "watchers" in this group, check this out with V3,
V4.2, and V5 - to see when/if Fujitsu fixed this problem.

NOTE WELL:
  If the WRITE statement were to an indexed file, then the compiler would be
correct in saying that you had a "conditional" WRITE where an imperative one
is required/expected.
```
