Hi All,

I've got a user using Windows98 that is getting the following Fatal
Exception error when the app I wrote accesses the COM port:

"A fatal exception 0E has occurred at 0028:C00802E3 in VXD MRT
Rate(01) + 00001527.  The current application will be terminated."

This application is running fine on many other Win98 machines as well
as Win95 and NT4.0.  The code where the failure occurs is using API
calls to open the COMM port and dial into our mainframe.  The program
is written in MicroFocus 4.0.32.

I can't figure out what MRT is referring to.  I've checked the MSDN
site and find other fatal exception problem descriptions, but not this
one.  Anyone have any ideas?

Thanks,
Doug

Hi All,

Well, I had an idea and found the answer!  I searched Deja.com and
found numerous references to this same problem.  I'll have to remember
to check there in the future.  

The basic problem is that Quicken 2000 installs files called
mrtRate.sys, mrtRate.vxd, and mrtRate.DLL which provide for an
automatic background download from their web site.  It, however, can
interfere with dialup networking and printing.  There official
response is to disable this feature - that does take care of it.
There official response on their web site is at:

www.intuit.com/support/quicken/faqs/win3/5192.html

Thanks anyway!
Doug
