```
After more than 100 posts, the debate's still going strong about the following
COBOL right justification algorithms (see below). Just for grins, thought I'd
post how this would be accomplished with SAS...

*** SAS ***

OUTFIELD=RIGHT(INFIELD);

*** COBOL ***

Subject: Re: Right Justify Routine

       Identification Division.
       Program-Id.  Test3.
      * right justify
       Environment Division.
       Configuration Section.
       Source-Computer.  IBM-PC.
       Object-Computer.  IBM-PC.
       Data Division.
       Working-Storage Section.
       01  input-field     pic x(30) value spaces.
       01  output-field    pic x(30) value spaces.
       01  teststrings-def.
           05 t1           pic x(30) value "ab".
           05 t2           pic x(30) value "abcdef".
           05 t3           pic x(30) value "abcdefghijklmn".
           05 t4           pic x(30) value "abcdefghijklmno".
           05 t5           pic x(30) value "abcdefghijklmnopqrstuvw".
       01  teststrings redefines teststrings-def.
           05 ti           pic x(30) occurs 5 times.
       01  tc              pic s9(4) comp.
       01  counter         pic s9(4) comp  value 1.
       01  char pic x.
       01  i               pic s9(4) comp.
       01  j               pic s9(4) comp.
       01  k               pic s9(4) comp.
       01  l               pic s9(4) comp.
       01  p               pic s9(4) comp.
       01  time-start      pic 9(4).
       01  time-end        pic 9(4).
       01  time-loop       pic 9(4).
       01  loop            pic s9(9) comp-5 value 500000.
       01  flag      pic 9 value 0.
           88 ad           value 0.
           88 sub          value 1.

       Procedure Division.
       test-3-start.
           perform varying tc from 1 by 1 until tc > 5
            move ti(tc) to input-field
            move space to output-field
            move function current-date (13:4) to time-start
            perform test3-start-AS loop  times
            move function current-date (13:4) to time-end
            compute time-loop = time-end - time-start
            display "AS :" time-loop output-field

            move ti(tc) to input-field
            move function current-date (13:4) to time-start
            perform test3-start-AS1 loop  times
            move function current-date (13:4) to time-end
            compute time-loop = time-end - time-start
            display "AS1:" time-loop output-field

            move ti(tc) to input-field
            move function current-date (13:4) to time-start
            perform test3-start-jud-coded loop times
            move function current-date (13:4) to time-end
            compute time-loop = time-end - time-start
            display "JUD:" time-loop output-field

           end-perform

           display "press enter"
           accept char
           stop run
           .

       for-later-use.
           move function current-date (13:4) to time-start
           perform test3-start-Rick loop times
           move function current-date (13:4) to time-end
           compute time-loop = time-end - time-start
           display time-loop

           move function current-date (13:4) to time-start
           perform test3-start-Judson loop times
           move function current-date (13:4) to time-end
           compute time-loop = time-end - time-start
           display time-loop

           move function current-date (13:4) to time-start
           perform test3-start-hand-coded loop times
           move function current-date (13:4) to time-end
           compute time-loop = time-end - time-start
           display time-loop

           move function current-date (13:4) to time-start
           move "abcdefghijklm" to input-field
           perform test3-start-jud-coded loop times
           move function current-date (13:4) to time-end
           compute time-loop = time-end - time-start
           display time-loop
           display "press enter"
           accept char
           stop run
           .

       test3-start-Rick.
           move 1 to counter
           if input-field > spaces
               move function reverse (input-field) to input-field
               inspect input-field tallying counter for leading spaces
               move function reverse (input-field (counter:))
                   to input-field (counter:)
           end-if
           .
       test3-start-AS.
           if input-field not = spaces
               compute i = function length (input-field)
               perform varying counter from i by -1 until
                   input-field(counter:1) not = space
               end-perform
               compute k = function length(output-field) - counter
               move spaces to output-field (1:k)
               move input-field (1:counter)
                       to output-field (k + 1:counter)
           end-if
           .

       test3-start-AS1.
           if input-field not = spaces
             compute l = function length (input-field)
             move l to j
             move 0 to flag p
             perform until j = 1
               compute j rounded = (j / 2 )
               if ad add      j   to p
                else subtract j from p
               end-if
               if input-field(p:l - p) = space
                    move 1 to flag
                    if input-field(p - 1:) not = space
                       move 1 to j
                    end-if
               else move 0 to flag
               end-if
             end-perform
             if not ad subtract 1 from p end-if
             compute k = function length(output-field) - p
             move spaces to output-field (1:k)
             move input-field (1:p) to output-field (k + 1:p)
           end-if
           .

       test3-start-Judson.
           move 1 to counter
           if input-field not = spaces
               move function reverse(input-field) to output-field
               inspect output-field tallying counter for leading spaces
               string input-field delimited by size
                   into output-field with pointer counter
           end-if
           .

       test3-start-hand-coded.
           if input-field not = spaces
               compute i = function length (input-field)
               Move i to j
               perform until input-field (i:1) not = space
                   subtract 1 from i
               end-perform
               perform until i = 0
                   move input-field (i:1) to input-field (j:1)
                   subtract 1 from i
                   subtract 1 from j
               end-perform
               if j > 0
                   move spaces to input-field (1:j)
               end-if
           end-if
           .

       test3-start-jud-coded.
           if input-field not = spaces
               compute i = function length (input-field)
      *         Move i to j
               perform until input-field (i:1) not = space
                   subtract 1 from i
               end-perform
               compute j = function length(output-field)
               subtract i from j
               add 1 j giving k
               move input-field (1:i) to output-field (k:i)
               if j > 0
                   move spaces to output-field (1:j)
               end-if
           end-if
           .
```

Good point,

The solution I showed you was developed originally to scan the
'navigation-fields' of an application.
These navigation fields could consist of a number of choices, separated by
a dot (.).

So, someone could fill in '03.123456798.123' in a character-field about 40
positions long. A special routine would destill the 03, the 123456789 and
the 123 from that and deduce that the user wanted to go to client 03,
customer-id 123456789 and perform action 123. This routine didn't have to
take spaces inbetween texts into account. 

We used a special routine, kinda like the given example a few steps back in
the thread, to handle numeric conversion. Users wanted to be able to enter
an asterisk in a numeric field, to identify the field needed no changes (a
left-over from the old punchcard days). This meant we had to accept numeric
data in alphanumeric fields and had to scan these letter by letter to
identify the contents and convert them to proper numeric data when needed.

We went over this several times.
JUSTIFIED RIGHT when the two fields have the same
length does nothing. If the sending field is shorter,
then the sending field (with spaces and all) are right
justified to the end of the receiving field, but the text
in the field is not justified.
Simple example:
aaa pic x(3).
bbb pic x(6) justified right.
...
move "sX" to aaa      <=== where the s is a space
move aaa to bbb      <==== will give:  bbb=ssssXs

Sure, so long as you understand that one should not choose a language
based on a single function.  I do not even think such inter-language
comparisons are really useful.  You choose a language based on a whole
multitude of things, not the least of which is how well its features
fit the task.  For quick little utilities, or function prototyping, I
generally use BASIC, because it is unexcelled in such things, of the
languages I am familiar with (quite a few).  But I have only written
a few applications in BASIC, because it is not well suited for larger
projects.  I only used BASIC for those because it was for applications
on the PC before other languages became readily available.  There is
no 'best language', only languages more or less suitable for particular
kinds of tasks.  And COBOL happens to be the very best language there
is, for many types of business programming.

I love COBOL for what it is good for, I love BASIC for what it is good
for, and I love assembly for what it is good for.  I'm not sure I could
say that I 'love' C, but I appreciate C for what it is good for. :) (My
first code in publication was in C.)  I could say the same for a whole
bunch of languages I have used over the years, and my job would have
been more difficult without most of them.  All of them could have been
improved, at least for some tasks, and all of them had strengths. (Well,
maybe not RPG. :)
