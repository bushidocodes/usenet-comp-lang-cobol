There is a distinction between object deck format (RECFM FB, LRECL 80),
which may be the raw form coming from the compiler, and load module
format (RECFM=U) generated by the linkage editor. The linkage editor
can work with both forms, but they would have to be stored in PDS
libraries with different characteristics and probably managed
differently. In MVS the distinction between a main program and a
subroutine tends to be blurred, and sometimes it may make sense to allow
the same code to be executed as both a subroutine and a main program,
depending on usage context. For the same PDS member to be both executed
at some times and used as a called subroutine in a link edit step at
other times, it would have to be in load module format in a load module
PDS. This, or the desire to avoid the management of a separate object
deck library, may be the reason behind the installations standards for
saving subroutines in linked format.
