Hi:

There are special line-drawing characters for DOS.

Whether your editor will let you put them into your source
is another matter.

These characters can be typed by holding down the ALT key
and then, on the numeric keyboard, typing the three digit 
ASCII code. Try this at the DOS prompt to see them

There are double and single lines.

The codes are (not in any particular sequence):

218, 194, 191, 195, 197, 179, 180, 192, 196, 193, and 217 for single lines

and 201, 203, 187, 204, 206, 185, 186, 200, 205, 202, 188.

You should search the internet for ASCII tables which depict all 256 characters.

Good luck with them.

Tony Dilworth

This is an excellent question, and hopefully I can get some help from some

Win98, Win2000, WinNT etc., don't even have this file, this suggestion may
not even apply to you. But as I recall, if I put a

DEVICE=C:\ANSI.SYS

line in the CONFIG.SYS file, the amulet type charaters that you describe
would resolve themselves to the proper charaters, assuming you keyed them as
suggested in other answers to your question.  Of course, you would need an
actual ANSI.SYS file to begin with, so if you don't have one, you need to
get some help from some other folks who may still have DOS files on their
PCs.  But in many cases, regardless of editors, when I loaded this file, I
could see the "boxes" properly in the editor.

even code set dependent on a single type.  Using hypens and plus is
suitable for most if not all terminal types.
