```
Using ANSI Standard COBOL, you canNOT really set colors (unless you display
hex-values that change your screen). In fact, the current ANSI DISPLAY
function is VERY limited to a "line printer" type mode.

All of this will change if the current draft Standard is approved - colors,
screens (character-based), and other features will be supported.

FYI - the chances are that your current compiler may have a non-ANSI
extension which already allows displays with colors. If you would like to
know how to do this, let us know what compiler you are using under what
operating system.
```

```
As I said when the question was first asked,
What compiler on what operating system is the original poster using?

If their compiler is a Windows or Unix compiler, then there is a good chance
(but no guarantee) that it does have a SCREEN SECTION (as does the draft of
the next Standard). If their compiler is on a mainframe or other totally
proprietary mid- or large-system, then there is a good chance (but no
guarantee) that their compiler will not have a SCREEN SECTION. Without
hearing from the original poster, the rest of us are just guessing - except
for the fact they did as about ANSI COBOL - in which case (again as I
originally answered) there is no way to do this - today.
```
