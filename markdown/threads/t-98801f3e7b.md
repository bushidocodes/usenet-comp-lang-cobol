This is a main good reason to use Microfocus.
On Windows, you have a very good editor/animator/analyzer wich is
provided with all the versions (called Workbench, or NetExpress, but
near the same)
On Unix, there is also good tools: editor/anim/analyze with CSI (cobol
source information) facilitie. It is included in tbox, a part of OCDS
(object cobol dev suite).
All these tools include interactive cross reference facilities and
various fonctions very nices when your job is cobol.
This answer is not an advertisement. There is a lot of problems with
Microfocus tools... but edit/anim et so on are nices.
