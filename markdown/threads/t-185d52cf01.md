Before Richard and/or others correct me, I think that some of my examples in the 
"scope terminator" thread were non-conforming.   I still haven't researched this 
or tested it, but I *think* that one may only have an imperative statement 
nested in the conditional phrase of an outer statement (other than IF)  - even 
if there is a period at the end, i.e.

Call "AB"
   On exception
           Call "XYZ"
     .

Is OK - because the conditional statement (CALL "AB") is terminated by a period 
and the ON EXCEPTION phrase includes an imperative statement.

However, I think that

Call "AB"
   On exception
           Call "XYZ"
                 On Exception
                    Perform LMN
                          .

May not (probably is not?) valid as the first ON EXCEPTION has a conditional 
statement nested in it and not an imperative statement - even if the 2nd 
conditional CALL is terminated by the period.

Again, I haven't fully researched this or tested it, but I didn't want others to 
be "confused" by my possibly (probably?) erroneous examples.
```

Robert,

I tested this with Micro Focus 3.2.24. The second
example (the one in question) reports the syntax error
"Preceding statement is not imperative".

In the FDIS 2002, "14.4 Procedural statements and
sentences", what I think applies is:

"Wherever 'imperative-statement' appears in the general
format of a statement, 'imperative-statement' refers to one
or more imperative statements ended either by a separator
period or by any phrase associated with that general
format."

The second CALL statement is conditional. Place an
END-CALL after the last statement (to make the second
CALL statement imperative) and there is no syntax error,
even though the first CALL statement remains conditional.

Rick,
   That was certainly my memory (after I posted the original erroneous code in 
the other thread).  A period (full-stop) may terminate a conditional statement, 
but NOT change a "nested" conditional statement into an imperative statement.

Therefore, in "ye goode olde" '74 Style, you didn't code:

READ WHATEVER
    AT END
        COMPUTE A = B + 1
              ON SIZE ERROR
                       DISPLAY "WHATEVER"
                                .

Rather, one coded:

READ WHATEVER
    AT END
               PERFORM COMPUTE-PARA
                  .
...
COMPUTE-PARA.
        COMPUTE A = B + 1
              ON SIZE ERROR
                       DISPLAY "WHATEVER"
                                    .

***

Now, some specific compilers may have had extensions to allow a period to turn a 
nested conditional statement into an imperative, but that wasn't standard.
```
