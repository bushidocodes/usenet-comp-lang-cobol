Yes LE (but also the new VisualAge for COBOL for NT/OS2/AIX...) provides
this kind of stuff.

The routines for the century window problems are:
CEESCEN(In: Integer Century_Start): Set the Century Window
CEEQCEN(Out: Integer Century_Start) Query the Century Windows

This is copied from the LE docs. I can not say much about this because we
do not use this here.
It is said in the doc that the century window is set for the entire
enclave (=run-unit in 'old' COBOL)

Hope that helps...

Dan,

I'm on the road now, so I don't have my course materials with me. But the
process to convert a date with a two digit year to a date with a four digit
year is roughly this:

Format the date in any character format you already use (e.g., mm/dd/yy) and
invoke CEEDAYS (I think that's the one) passing this date and a picture item
that tells LE how to interpret your picture (e.g., MM/DD/YY) and CEEDAYS
converts the date into a Lillian date (fullword binary integer containing the
number of days that have elapsed since 15 Oct, 1582); this date is calculated
based on the current setting of the Century Window; you can use the default
setting, or you can change the setting using the CEESCEN service.

Then, if you like, you can convert the Lillian date back to a character date
using CEEDAYS (pass the Lillian date, a picture of how you would like the date
formatted, and a place to put the result).

Hope I got the service names straight.
