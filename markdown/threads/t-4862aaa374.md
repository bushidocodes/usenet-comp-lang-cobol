```
From within the control panel, go to ODBC data sources (32 bit). Highlight
the User DSN that you are using, this is the tab that shows on the screen
when you enter ODBC data source. Click on the "Tracing" tab. From there you
will see a pushbutton to start or stop tracing. My default file to do the
tracing is \SQL.LOG. I noticed during normal operations, reads and opens for
example, that this file only added a few records. But, during wildcard
searches, this file grew extensively. When I stopped tracing, everything
sped up to almost instant access, even for the wildcard search.
```
