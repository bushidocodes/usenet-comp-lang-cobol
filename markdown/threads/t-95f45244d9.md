```
As a group of highly qualified developers and program analysts with
large experience in COBOL, CICS, DB2, Web Sphere, MQSeries, Java etc.
we were vary excited to hear about your interest in getting XML output
from COBOL and other MainFrame Legacy applications.
It seems that many organizations all over the world are facing the
same challenge.
Recently we&#8217;ve developed an overall independent solution to this
problem, based on the following principals:
1. Dialog with web server via MQ Series 
2. Receive URL & XML buffers to a COBOL structure by using our COBOL
API
3. Produce output XML files from genuine COBOL structures (including
occurs, redefines, copy and value options). In addition we give more
powerful meaning to the COBOL structure by defining dynamic arrays,
optional elements and independent processes (as standalone procedures
etc.).
The API is vary easy and user friendly and supports user exit
processing. (It was build explicitly for MainFrame environment.) One
of the most important features of which is high performance. It does
not require any special installations in CICS/COBOL environment.
The output XML Document is a well-formed XML with attributes and CDATA
features. It can contain DTD or XSL assigns.
We are interested in engaging in partnership with a serious
organization that is ready to back us up in our continuous effort to
get our solution to the status of a complete product.
```
