> Deek

The PIC is always followed by at least one space. The values after
that space are always typed with no spaces between the values.

AFAIK most COBOL compilers are "case insensitive" and it does
not matter if the code is typed in UPPER lower or MixEd case.

Case only makes a difference when you are calling external
routines that are developed using a "case sensitive" language
like C or C++ or others of that ilk.

Some people code with UPPER case for variable names
but this is a convention only. Others write their code with
the COBOL keywords in upper case (ie: MOVE variablea TO gender.)
but this does not make any difference in any of the compilers
I have used.
