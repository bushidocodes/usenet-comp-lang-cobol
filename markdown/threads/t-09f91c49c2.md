Restrictions" in the on-line help. Now the 'guts' of programming is not my

returning from PERFORM loops 'randomly' to the start point - and you are talking
about 50,000 records plus, is there a possibility the stack is getting
overloaded with perform references which in theory should have been
"killed'/removed from the stack ..

Hi Bill

That's a classic. 114 on MF like 0C1 or 0C4 om IBM mean that somewhere in your 
program you smeared your memory. There is no warning to the fact, 
since checking this type of error wil reduce performance significantly. 
Therefore, you put some garbage in the address used by the read statement, and 
when applying it your program did not know what to do.

This error 114 is usually very hard to find since as I mentioned, the cause of 
the problem is not the place where the program fails.

Regards

Ilan
