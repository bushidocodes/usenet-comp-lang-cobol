energy wrote:
> Would these pascal programs differ much in the other 3 languages

In a word, yes, but not too much.  For COBOL, you'd still need the four 
divisions (Identification, Environment, Data, and Procedure), but you 
could do something similar.  I'll give you a sample on this top one, and 
then you can compare it to derive the structure of the others.

> Program repeatadd;
> 
…[10 more quoted lines elided]…
> End.

v (Column 1 = Column 8, if you're in COBOL 85)
Identification Division.

Program-ID.  RepeatAdd.

Environment Division.

Configuration Section.

Source-Computer.  <insert computer type here>.
Object-Computer.  <insert computer type here>.
Special-Names.    Printer is Printer  Console is Console.

Data Division.

01  Num    Pic 9(05) Value 0.
01  Total  Pic 9(05) Value 0.

Procedure Division.

Here-We-Go.

     Perform Test After Until Num = 0
         Display "enter number or 0 to end" upon Printer
         Accept Num from Console
         Add Num to Total
         Display "Total = " Total upon Printer
     End-Perform

     Stop Run.
----------------------------------------------------------

Some notes...
- the (05) after the Pic 9 means that the number has 5 digits - you can 
have up to 18.
- instead of Add Num to Total, that line could read "Compute Total = 
Total + Num".
- The "Test After" clause makes the Perform statement execute at least 
once.  If it's omitted, this program would do nothing, because Num has 
an initial value of 0, and the test defaults to before.

Also, on the other two, check out the Perform Varying syntax in your 
COBOL manual.  It's the closest thing to a for...next loop in COBOL.

energy wrote:

> some of the jargon is going way over my head, i

It depends on where you start from - COBOL is not as cryptic as some 
other languages, which makes it more difficult for some.  Let's take 
Joe's code above and go through it real quick.

 > Identification Division.
 > Program-ID. countdown.

The identification division does just that - it identifies the program 
to the computer.

 > Environment Division.

This sets the environment of the program.  Traditionally, you'll have a 
Source-Computer and Object-Computer paragraph, which will identify the 
type of computer you're running on.  This is also where you'll specify 
any files you need to use.

 > Data Division.
 >
 > 77 Loop Pic S9(8) Binary.

(I believe there's a missing section declaration, "Working-Storage 
Section", although some compilers may let you omit it.)  The data 
division is where all variables are declared.  There are various 
sections of the data division.
  - The file section defines the format of the files you declared above.
  - The working-storage section defines variables for your program.
  - The linkage section defines variables your program will use to 
communicate with other programs.  This is most commonly used in 
"subprograms", where you call this program from another one.
  - The common-storage section defines variables that your program and 
other subprograms can use.  The best way to do this is with copy procs, 
as you'll run into problems if the subprogram defines common-storage a 
different way than the main program.  (If the subprogram doesn't use 
common-storage at all, there's no problem with that either.)

 > Procedure Division.
 >
 > Perform varying loop from 10 by -1
 > until loop = 0

Perform is the COBOL iterative structure.  You can Perform a paragraph 
name (ex. Perform 100-Do-This-Stuff), or you can use an inline perform 
as above.  You can also specify how you want the iteration to occur. 
Perform Varying, like Joe used above, moves the value 10 to the variable 
"loop", and each time End-Perform is reached, it is incremented by -1 
(which actually decrements by 1).  The until clause is very important, 
as it tells the loop when to stop.

Perform also takes a number of times.  The perform above could be written...

Move 10 to loop
Perform 11 Times
     Display loop
     Subtract 1 from loop
End-Perform

(As you can see, Joe's method is a lot shorter - but, there are times 
you may want to perform something a set number of times.)

 > Display loop

This displays the variable "loop" on the default output device (usually 
your screen).

 > End-Perform

This terminates the scope of the Perform statement above.
 >
 > Display " "
 > Display "Lift off"
 >
 > Goback

I'll admit, this "Goback" is new to me - it's not in the COBOL manual 
for the implementation I usually use.  However, if I had to guess, I'd 
say that it terminates the program run, but doesn't actually kill the 
run, in case this program was called from some other program.  The 
sentence "Stop Run" will halt execution no matter what.

> How long does it take to learn programming cause i
> only have six weeks left at college, then hoping to go uni studying
> information systems which has more programming.

Practice, practice, practice...  :)  The more you do, the more you 
learn.  Try getting a book with a title like "Learn COBOL in 3 weeks" or 
something, and go through the tutorials in it.  It'll really help you 
understand the language, especially if you're thinking that you might 
want to make a career out of it.
