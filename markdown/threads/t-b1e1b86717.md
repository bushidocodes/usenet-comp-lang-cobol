```
We have had a few Run Time Errors in our NetExpress 3.1 software which
are probably caused by corrupted data (all in one test set). I don't
think that there will be a problem to sort that out but it brought up
a couple of oddities.

Firstly, there is the program address "failed at PC..." but how does
one get these addresses from a NetExpress compilation? There was no
problem with our previous COBOL 3.n set up, just add a few directives
- REF or similar, but that does not produce the equivalent listing in
NetExpress. I've tried a few others but nothing gave any code
addresses (DATAMAP for the data but nothing for the code). Any
suggestions?

Secondly, the error was 163 (invalid data in numeric field) and yet
NOCHECKNUM is the default, or is that only for the GNT ? This error
occurred when running an INT. As it happened I tried running with the
-F switch (as described in the Help) and an OPEN FILE failed (?!)

And thirdly, in this particular case the RTError did not come in a
dialog box but in the DOS type window (this is currently a WINDOWS
.EXE, calling INTs but using ADIS so it uses what looks like a DOS
window). I suspect that I may have built this EXE, ummm, wrongly. It
does work but there may be something odd with it. In other instances
when there have been real code bugs (e.g. 153) I have seen the proper
dialog box. Is there anything obvious here ?

All a bit confusing really...

Of course when there are no data errors the software does appear to
work correctly - so we have not seen this behavour in our testing but
it just looks like something invalid in the data has made the program
misbehave.

graham
```
