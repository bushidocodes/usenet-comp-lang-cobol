Why do you need to add line feeds?

If you use an XML viewer to view the data (Internet Explorer/XMLSpy) it
should format the document for you.  There are a number of free tools for
this.

Or write a very simple program that uses an XML parser that will display the
data exactly how you want it.  I assume that you are writing this in
UTF/Unicode.

If you aren't "viewing" the data it should matter even less and you wouldn't
ask this question.

If you hell bent, make sure you understand that the line feed is not the
same across all ascii platforms and is notorious for causing problems.

JCE

You could always add EBCDIC CR/LF chars.  The various translation 
packages I've dealt with seem to convert them nicely.

As these are always considered xml "whitespace" you can place them 
anywhere a space would be appropriate.

Depending on your preserve whitespace options in your schema, you could 
add them at the start of any tag under the ignore-leading-whitespace 
rule.

Or you can always add them between a consecutive ">" and "<" sequence.
