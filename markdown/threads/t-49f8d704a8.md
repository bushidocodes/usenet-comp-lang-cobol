We currently use CA-Realia Workbench 3.0 and Flexus SP2. CA does not
appear to have any plans to incorporate support for ActiveX/COM.
According to CA support, CA-Realia is not capable of calling ActiveX
DLLs. We would like to move our product in this direction. We like SP2
and would like to continue to use it. We make heavy use of Windows
APIs. We plan on using VB in certain instances (True DB Grid Pro 6.0).

Fujitsu advertises that it can access ActiveX DLLs, produce ActiveX
components and controls from COBOL, and that it easily interfaces with
VB. It sounds almost too good to be true.

Has anybody out there migrated from CA-Realia WB 3.0 to Fujitsu? Would
you recommend Fujitsu or Micro Focus? Can I still continue to use SP2
with Fujitsu? How do you rate Fujitsu?

Any advice and recommendations would be most welcome.

Thanks,

Since you already use Realia WB 3.0 you are generating 32 bit code, 
correct?

I suggest you contact Flexus and get their add on for ActiveX/COM.  
They presently support OCX files as part of sp2 in their 32 bit 
version.  No need to change compilers or tools.

PS - Fujitsu has a fine compiler.  Conversion from Realia to Fujitsu 
is the hardest of any of the COBOL variants - Realia did things very 
differently.  However, I have done it so I know it can be done.  I did
it to get to 32 bit without buying a $4000.00 compiler from CA.  (I 
was using 16 bit Realia before).  Anyway, the file I/O with Realia is 
the FASTEST there is, so if your application is I/O intensive you 
won't want to do it.

Second, Fujitsu DOES support OCX controls, but ONLY from PowerCOBOL - 
you won't be able to mix this with Sp2.

Lastly, Flexus (http://www.flexus.com) DOES have a version for Fujitsu
COBOL, and it does (with an add on) support OCX controls (as does the 
Realia version).

Personally, I see no need for you to switch - just get the Flexus add 
on.

You can embed Active X controls into an sp2 panel defintion with our
32 bit Windows version and control both properties as well as methods
for the OCX.

I have sent you the Active X support DLL as an attachment to a

>Fujitsu advertises that it can access ActiveX DLLs, produce ActiveX
>components and controls from COBOL, and that it easily interfaces with

We support Active X controls in sp2 regardless of which compiler you
are using, therefore you can continue to use your 32 bit CA-Realia
compiler and still be able to display and control Active X controls
from your sp2 GUI screens.

In addition, our Active X support also works with our Thin Client
add-on which means that you can also run your GUI screens across the
Internet, complete with Active X controls.

>Has anybody out there migrated from CA-Realia WB 3.0 to Fujitsu? Would
>you recommend Fujitsu or Micro Focus? Can I still continue to use SP2
>with Fujitsu? How do you rate Fujitsu?

sp2 does indeed support Fujitsu, but if the only reason that you want
to switch compilers is to gain Active X control support, you can get
it immediately with sp2.  Besides, I suspect the Active X support
capabilities with Fujitsu COBOL, only work with Fujitsu COBOL.  The
Active X control support in COBOL sp2 is completely compiler
independent.  The same screen handling source and GUI screen
definitions work the same regardless of which compiler you prefer to
use.

>Any advice and recommendations would be most welcome.

minutes.
