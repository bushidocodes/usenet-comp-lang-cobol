For regular COBOL (without JCL) and batch COBOL (with JCL), the
SELECT-ASSIGN file statements have to be changed, to show the files are
read from a floppy disk (not the mainframe).

For batch COBOL (with JCL), all JCL, before and after the COBOL program
has to be removed.

You will probably need to convert a JCL based system to one that uses a
root program to call menu programs and process programs. The other area
where such differences exist is in I/O functions. Select statements need
to list filenames (or field names containing physical file names). Screen
management may differ: check the use of ACCEPT and DISPLAY verbs. Process
code should remain the same. Also, in file management, you may need to
validate READ statements for busy record conditions. This requires the use
of Declarative proceedures if you do not already have them and the checking
of status codes after each file I/O.
