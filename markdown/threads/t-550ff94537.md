Hi Judson,

I work the same way you do - visualizing what the program does as the
data passes thru.  I'm always finding that I stop right in the middle
of coding and mentally talk the logic out - what happens if this
particular piece of data comes in, etc.

Has worked very well for more than 15 years now and I don't see it
changing either.  Shoot, most of the time I sit down with just a
 rough
idea of what needs to be done and can code it up in vry little time.
My manager always wants an estimate (gotta bill that client!) and it
usually takes me longer to work up that than it would to just program
it.
