Hi,

I've been developing a new course for a few months now, and I'm getting close
enough to start describing it to training coordinators. The two people I've
talked to about it apparently heard something different than I was describing.

My new course is an in-depth exploration of subroutines and mainlines: how to
call and be called in four different languages (Assembler, COBOL, PL/I and C),
in an LE environment. It includes (for all four languages): 

* a review of describing elementary data items, structures, and arrays
* how to handle null-terminated strings
* how to handle the PARM from the EXEC statement in batch jobs 
   (including how to use CEE3PRM, and when that doesn't return 
   what you would expect)
* syntax of call / function references
* calling to / from the same language and different languages
  (trying to avoid special cases like pragmas or calling language 
     intialization entry points)
* format of object modules (OBJ, XOBJ, GOFF)
* how the linkage editor and program binder work
* load modules and program objects
* how parameters are passed in an argument list
* passing and receiving data by reference and by value
* language-specific options for calling
* language-specific options for passing / receiving data
* omitting arguments
* handling variable argument lists
* static vs. dynamic calls: whys and hows
* module attributes and their impact: reuable, reentrant
* mainenance issues and techniques
* DLLs

When I've told people about this, they seem to hear "Oh, how to pass data
between the mainframe and a work station"! Now that sounds like a useful class,
and I will turn my attention to it next, perhaps, it is _not_ the class I am
currently developing.

My thought is that a good, snappy, descrptive title might help. So I am turning
to the usenet groups for two areas of help: 

1) can you come up with a name that would help describe and sell this course
succinctly? 

and 2) is this a course anyone would be interested in or am I wasting my time
(I've spent a lot of time, learned a lot of things that I thought I knew but
was wrong about, but it may be that this work only is helpful to me)?

Here are some titles I've been tossing about...

Secrets of InterProgram Communication
Subroutines
Advanced Subroutines
Mainlines and Subroutines


Thanks for any suggestions / feedback.
