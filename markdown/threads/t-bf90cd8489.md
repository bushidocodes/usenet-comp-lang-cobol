You can try 'SET myProcPointer to ENTRY "myDDEcallback"' where
myProcPointer is a procedure-pointer declared in working-storage. It's
an Micro Focus extension but Fujitsu may support it.

Plan 'B' would be to put the routine in a DLL, use the LoadLibraryA()
function to load the DLL (which returns an instance handle). Then use
GetProcAddressA() to convert the returned instance handle to a procedure
pointer. Ain't the Win32 API straightforward ;-).
