```
I'm a long time Realia Cobol developer. I've developed Windows
applications with Realia/Flexus SP2 in the past. Given the direction
of CA-Realia towards the workbench (it used to be a stand-alone
compiler) and mainframe, meaning less support for the windows
environment, it may be time to switch to Fujitsu. If someone
experenced with both could help me a little with the following
questions, it would be appreciated.

Please state your qualifications when answering.

Thanks!!


1) The Realia file system comes with very good native ISAM/VSAM
support. How does Fujitsu compare? File utilities?

2) Realia produces native code. Does Fujitsu have a run time
DLL that is required for distribution. If so, how big is it?

3) Does Fujitsu have Intrinsic Date functions? Can it handle
four digit year functions?

4) Realia had a terrible Windows Debugger/Animator. How is
the one for Fujitsu.

5) For backward compatibility with Windows 3.1 users, I would
probably develop using the 16 bit compiler. Are there
any disadvantages to this over 32 bit?

6) How good is the tech support for Fujitsu?

7) How do Flexus/SP2 and PowerCobol compare?

8) Would there be many problems converting a 16,000
line Realia program (using VSAM and SP2) to Fujitsu?
(I have some very large arrays in the program, which
caused some segment boundary problems in Realia)

9) Are the arithmetic functions (comp-3 and comp) completely accurate

in Fujitsu?

That's all for now.

Please email also to lon··.@tel··t.com

Thanks,

Steve Lulay
Systems Group
```
