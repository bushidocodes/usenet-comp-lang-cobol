Having ported MF Cobol programs from 95 to NT myself, one possible reason
for this error could be the number of file handles available to the program.
By default, an NT Dos box has only a small number of handles available
(20?), nowhere near enough to the 100+ you need available to run the MF RTS.
Look in \winnt\system32\config.nt for how many handles are available to you
in a Dos box.
