Yup. Since late 1980's.

Works just fine.

It will find numerous "undocumented extensions" in the OS/VS COBOL report
writer language, but tolerates most of them, and tells you what is wrong
with the rest.

Difficulties can be encountered in comparing the output, since there is no
guarantee that a given data stream will be used to produce identical
printed output.

I.e. print a header at the top of the page, then a detail on the 5th line
of the same page, using ASA control characters. Probably at least half a
dozen data streams could be used.

A local vendor built a special compare program to handle "visual
comparison" of 2 print data streams. E-mail if you would like a referral.

Builder of software archeology tools and other strange programs to help
survive in a legacy based world.
