CL**4 means Change Level 4. You cannot alter this behavior in
Panvalet. Panvalet was developed to be a source management tool, so
that previous versions could be recovered from archive tapes. The CL**4
stuff lets you know which lines were changed at each version. That's
not very helpful if cannot see the before images.

You can run a batch panvalet job (I forget whether it is PAN#1 or PAN#2)
and level the entire Panvalet member. The sytax of the SYSIN would be
something like:

++FORMAT MYPROG,COBOL
++LEVEL MYPROG,5,1

You need to FORMAT your Panvalet member before you can LEVEL it. When
you format it, the level number goes up by one. You must specify the
current level number (5 in this example) and the new level number.
