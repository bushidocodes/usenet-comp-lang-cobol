>    What has X/Open to do with COBOL, 

X/Open, now the Open Group, defined a set of standards for COBOL, C,
ISAM file structure (based on C-ISAM), and various Unix bits. The
Cobol extensions include SCREEN SECTION, and CRT ACCEPT/DISPLAY as has
been used in almost all PC compilers since the 80s.  They also defined
all the LOCK-MODE extensions for file sharing with record locking.

In other words it is the X/Open standards that made PC and Unix based
multi-user COBOL systems viable.

one of the most underused feature of COBOL, and at the same time one  
of the most powerful. I guess it has been pushed aside for the same  
reasons that shop standards precluded the use of the CORRESPONDING  
phrase in MOVE, ADD, and SUBTRACT, or the COMPUTE statement. Also,  
because it was an optional module in the standards before 2002, REPORT  
WRITER was not implemented in a number of compilers, which again  
caused quite some developers to discard it in exchange for  
portability. The combined effect of that seems to have slowed down  
development of features in the REPORT WRITER module. I'm happy to see  
that it is no longer optional in the 2002 standard.
