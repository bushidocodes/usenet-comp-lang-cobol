What I mean is that zero is used as a special value in INTEGER-OF-DAY,
INTEGER-OF-DATE, DATE-OF-INTEGER, and DAY-OF-INTEGER. Whereas an an
argument of -2 passed to ACOS or SQRT will cause the program to abend
with a range violation, invalid arguments to the INTEGER-OF-Dxxx
functions quietly return the code value zero. For symmetry, the
Dxx-OF-INTEGER functions interpret a zero argument as a code value, and
return the code value zero in turn, rather than 16000101 or 1600001.

I'm not complaining, this is a logical way to handle invalid dates, but
my point was that even the Cobol standard recognizes the use of code
values.
