The 4.0 product is thread "safe", but it is not thread "aware". I'm
not sure what specific update this appeared in, but it's certainly in
the latest!
What this means is that the runtime support libraries ensure that
only one thread may enter the COBOL environment at a time. So, if you
wanted to write an app which had two threads each executing COBOL *at
the same time*, it is not possible. However, it *is* possible to have
COBOL components running in a multi-threaded app if the architecture
of the app is tolerant of a thread blocking when attempting to enter
COBOL when COBOL is already active in another thread. By "COBOL" I
mean either COBOL programs, or the COBOL runtime system.

The Visual Object COBOL and Net Express products fully support the
writing of multi-threaded COBOL applications at a syntax level (not
CALL statements) with commands to launch new threads, manipulate
several types of synchronisation objects (mutex, semaphore, event,
monitor) and provide for a THREAD-STORAGE SECTION.

Hope this helps.

Kev.
