I have a question in regards to the execution of a COBOL program with embedded SQL. I know the process in to DB2 Precompile the source, Compile, Link and then Bind the Load Module to the DBRM created in the Precompile step. At Bind time, you create a DB2 Plan for that program. A program can be bound with multiple DB2 Plans.

We have different message regions which utilize the same load modules, and each region has it's own DB2 HLQ [or SQLID] associated with it as well as it's own Plan associated with it.

Example: For Message Region 1, DEV01.XTABLEA exists in database XDATA1DB and for Message Region 2 DEV02.XTABLEA exists in database XDATA2DB. When Load Module LOAD000 runs in Message Region 1, it utilizes plan LOAD0001, but when it runs in Message Region 2, it utilizes plan LOAD0002.

Our 2 message regions will use the same load module, but one region will access the DEV01.XTABLEA table while the other region will access the DEV02.XTABLEA table.

My question is, when you execute that load module, how does it know which Plan to use?

Hi.

I asume that You are working in an IMS environment.
You have to bind each load module against all the qualifiers You are using.
I also asume that the messages regions are within the same IMS control region.
You then need different transaction codes and PSB to get to the different tables.
The transaction code has to be routed to the rigth MPP region for by the class to get to the rigth tables within DB2.
In IMS the PSB is by default the DB2 plan name. This is the name IMS give to DB2.
If You need different names this also has to be spesified in IMS nucleus.

TimeSpinner,

Assuming IMS message regions and TRAN001 and TRAN002 are the
transactions, and Plan names. Also the owners of the tables are DEV01 and
DEV02.

All you need to do is bind plan TRAN001 with "OWNER(DEV01)" and bind plan
TRAN002 with "OWNER(DEV02)". This will allow your load module to access
DEV01.XTABLEA when running under TRAN001 and access DEV02.XTABLEA when
running under TRAN002. DB2 will use the owner defined in the plan under
which the load module is running.

Hope this helps.
