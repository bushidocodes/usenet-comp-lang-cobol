```
Hi all,

I created a small cobol app using Fujitsu 3.0.  This app calls a
function in C.
I use VC++ 6.0 to link the object code.  Everything goes well except
that:

1) There is no problem if the function has no parameter, and call many
times.
2) There is no problem if the function has parameter, but only couple
calls is made
   (2 or three calls)
3) The application crash if there are 100 of calls made in a loop.

To make things simple, the app is as simple as hello world.  The
function from C does not do anything.

Note that calling 1 time will work.  If return any value, it show it's
a correct value.

If any of you have any advice, please help.

C code:

__declspec(dllexport) int __stdcall SAYHELLO(char * something);
__declspec(dllexport) int __stdcall SAYHELLO(char * something){
         return 0;
}

Cobol code:

 IDENTIFICATION DIVISION.
 PROGRAM-ID.   HelloWorld
 AUTHOR.       asdf
 DATE-WRITTEN. Jan 23 2002
 DATE-COMPILED.

 ENVIRONMENT DIVISION.
 CONFIGURATION SECTION.
       
 INPUT-OUTPUT SECTION.
 FILE-CONTROL.

 DATA DIVISION.
 FILE SECTION.

 WORKING-STORAGE SECTION. 
 
 01  FUNCTIONCALL.
     03  MYFUNC-RETURN-CODE  PIC S999.
     03  NAME                PIC X(50).
     
 PROCEDURE DIVISION.
      MOVE LOW-VALUES TO NAME.
      PERFORM 120 TIMES
          CALL 'SAYHELLO@4' USING BY REFERENCE NAME
      END-PERFORM.
      STOP RUN.

 END PROGRAM HelloWorld.
```

```
This looks like a stack problem caused by neither the Calling nor the Called
program clearing the stack down
after the call.

You need to check the Calling Conventions in Fujitsu and see which one
matches what you are doing in the C program.

COBOL (When calling BY REFERENCE) pushes the parameter addresses onto the
stack. Whether they get popped off again depends on the calling convention
used.

I seem to recall something about the PASCAL calling convention being
required when using Fujitsu V3 but it is years ago and I'm not sure exactly
how it is implemented.

Try the manual and if you are still stuck, mail me privately and I'll look
up some old code.

Pete.
```
