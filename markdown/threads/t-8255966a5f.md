Although it may be platform dependant, in all implementations of Cobol that
I have come across indexing is much more efficient. This is because the
index is basically a direct offset into the table whereas a subcript has to
be converted from an occurance to a physical offset. Because the index is
associated directly with a table, this is why you can't use an index
declared in table "a" to access data in table "b" (especially if they are
difference sizes).
