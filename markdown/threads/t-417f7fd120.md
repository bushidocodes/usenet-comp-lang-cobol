Ah, thanks.

Interesting to see that several million started looking at Javascript.

Windows 8 development will be based on XAML and Javascript (primarily) 
although the other languages can also be used. There was a demo of a Windows 
8 app built in minutes using Javascript, CSS, and XAML at the recent MS 
Build conference. The point about this environment is that you can write the 
code once and it runs across all windows platforms, from a desktop machine, 
through Web based devices, to a mobile phone, without change. It is 
accomplished by versions of a framework (runtime) for each platform (very 
similar to, and probably derived from, .Net.)

Over the last few years it has been interesting to see the general public's 
awareness of computing shifting focus from the desktop to the Web. People 
who hear I am involved with computers automatically think I develop Web 
Sites (which I do, but that isn't primarily what I'm doing.) Social 
networking has pervaded the lives of most people (not me...:-)), from kids, 
to Grandparents wanting to keep in touch with their offspring's offspring, 
and it is all accomplished via the Web. There is a generation growing up who 
simply don't recognise a desktop application; it is all cloud based web 
stuff.

In this context it really isn't so foolish to learn Javascript. It is just 
another OO Language. I still prefer ASP.Net for Web development because it 
means I can write C# "code-behinds" which gives my web site full server side 
programming power without the use of a primitive CGI bottleneck (CGI is 
centralised and only multi-threadable with added layers (like ISAPI); C# is 
innately multithreaded and distributed to the pages that require it.) 
However, there is still a place for Client side code and Javascript has 
always been more attractive to me than VBScript for Web programming. Windows 
8 is pushng Javascript to both Client and Server, although you don't HAVE to 
use Javascript for Server side programming.

I also found out recently that kids in High School are now doing "Coding" as 
a unit of their Computer Studies.

It is easy for us to sit back and say:" Well, learning a language doesn't 
make you a programmer." No, it doesn't, but it is a step in the right 
direction and if you write enough code, eventually you will find ways to 
write it better.

Furthermore, the world doesn't have to have millions of expert computer 
programmers. New environments and software development tools are much more 
"forgiving" and much "smarter" than was once the case. Most people who are 
capable of doing a crossword puzzle or solving Sudoku are more than capable 
of tailoring their own personal software for their iPad, phone, or Notebook.

It is just possible that in my own lifetime I will see computing for the 
masses, with people taking control of their devices and without the 
mysterious veil that could only be penetrated by the priests of COBOL...:-) 
I have wanted this since 1965, so it is kind of pleasing to see things 
moving that way.

Pete.
