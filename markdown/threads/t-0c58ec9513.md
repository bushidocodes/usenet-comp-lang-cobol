link portions work fine but when I come to run the WINEXEC from the
PROGRAMMING-STAFF window it gives me an "Error Starting Program" Dialog

found."  Moving the F3BIPRCT.DLL to the folder in which the source file
is located resulted in subsequent similar errors for other F3BI*.DLLs.
The only resolution I have found is to move all the .dll files into the
folder which contains the source file.

	Is there an Option that I can set so that I can tell WINEXEC where to
look for the required .dll files?  Or, alternatively, is there another
solution to the problem?

When you do the standard install, Fujitsu modifies your
default path.  It sounds like you bypassed that step. Check
that your autoexec has "C:\FSC\COBOL97" in the path(or
wherever you installed it which could also be the problem)

You should also have the environment variable for "TMP"
set to a nice big area, and the environment variable "LIB"
set to \FCS\COBOL97 and \FSC\PCOBOL32.

Dave,

You need to check your PATH statement in the autoexec.bat file.  It should
have a path containing c:\FSC;c:\FSC\PCOBOL32;   Use c for the drive it was
installed to.  This will allow the search to look in the path and find the
DLL's that are there in the folder.  The installation should have installed
this correctly. Sometimes it doesn't work if the Path statement isn't
correctly installed originally.   For V4 compilers use COBOL97 instead of
PCOBOL32.  Hope this helps.

Regards,
Gary Roush
