Without knowing which Cobol version you use, I can only say that usualy
Cobol knows only the switches 1 thru 8.

So, it should be
SWITCH-1 ON IS SW0
SWITCH-3 OFF IS SW1.

Following from NetExpress on-line help :-

COBOL switches are numbered from 0 to 31. Each can be set to "off" or
"on" when your application is started. Their state can be read by your
application at any time when it is running.
The settings that you make on the COBOL Switches tab are defaults that
can be overridden by the user when the application is run.
The switches must be defined using a condition name in the special-names
paragraph. For example:

special-names.

switch-0 is job-origin on status is us-office
                         off status is uk-office
...
procedure division.
evaluate job-origin
when us-office...
when uk-office...
...
