Report on January 30-February 3, 1995 ANSI X3J4 COBOL Committee
meeting

As usual, most of this meeting was spent discussing proposals for
the next revision and working on editorial papers concerning the
base document. We spent over a day on editorial work and there
was a one-day editorial ad-hoc the day before the regular meeting.
Our aim is to have a fairly clean document ready for the first
public review (it will probably be sent to ISO towards the end of
March). It is not yet clear how it will be made available to
reviewers in the US. In other countries the national
representatives to ISO will provide it to interested parties. I
will post a message giving the review process when I know what it
is.

The major topics/proposals we discussed were:

1. The Object-oriented proposal was reviewed (it was passed last
meeting and some editorial revisions had to be done). Some more
changes were made and I will add it to the base document in time
for the first public review.

2. We spent some time on a proposal for multi-octet character set
collating sequences and the like. This allows dynamically
changing the program collating sequence and different collating
sequences for indexed files. There was quite a bit of controversy
over requiring support for ISO-10646 (the standard for multi-octet
characters) since some operating systems (DOS was pointed out as
an example) don't support it. It will be revised and resubmitted.

3. My proposal on a compatibility directive (causes the compiler
to diagnose features that are incompatible with the new standard)
was discussed and passed. It will be added to the base document.

4. A proposal on adding a USING phrase to SELECT ASSIGN to allow
dynamic file assignment was discussed and passed. It will be
added to the base document.

5. A proposal on ACCEPT/DISPLAY screen handling (as in X/Open and
lots of COBOL compilers) was passed after lots of revisions and
will be gone over once more before adding it to the base document.

6. A proposal on user-defined types (allows strict typing and
allows a data definition to use another data definition - somewhat
like a source COPY) was discussed and sent back for rewrite. The
main hangup was about weak vs strong typing, mainly for a group
containing a typed object. We decided to provide both.

7. A proposal on VALUE REPEATED (allows setting of tables to
various values with a VALUE clause) was discussed and was passed
and will be added to the base document.

8. A proposal on partial COPY REPLACING and REPLACE (allows
replacing partial strings) was discussed and was passed and will
be added to the base document.

5. Some minor topics: we lost three members (DEC, Wang, and
GUIDE) and are down to nine members - one of the members was the
vice-chair; the name of CBEMA (the ISO secretariat) was changed to
ITI (Information Technology Industry); and CIB-27 has been
transmitted to X3.

The next meeting is in Dana Point, California, on March 27-31
1995. The main topic will probably be discussing the first draft
review comments (if any are received in time).
