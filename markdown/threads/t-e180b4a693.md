Detecting Esc "under every condition" is going to be pretty tough. But if
you mean looking for Esc to be pressed when the program is processing, that
may not be too difficult. If you use Micro Focus COBOL on the PC, place
the following Working Storage and Procedure Division code into COPY files:

Working Storage:
*
* K E Y B O A R D I N P U T A R E A
*
01 KEYBOARD-INPUT-AREA.
03 KIA-CHAR.
05 KIA-DIGIT PIC 9(02) COMP-X.
03 KIA-KBD-STATUS PIC 9(02) COMP-X VALUE 0.
*
* ** KEY RETURN CODES **
*
78 KIC-BACKSPACE VALUE 8.
78 KIC-TAB VALUE 9.
78 KIC-ENTER VALUE 13.
78 KIC-ESCAPE VALUE 27.
78 KIC-BACKTAB VALUE 143.
78 KIC-F1 VALUE 187.
78 KIC-F2 VALUE 188.
78 KIC-F3 VALUE 189.
78 KIC-F4 VALUE 190.
78 KIC-F5 VALUE 191.
78 KIC-F6 VALUE 192.
78 KIC-F7 VALUE 193.
78 KIC-F8 VALUE 194.
78 KIC-F9 VALUE 195.
78 KIC-F10 VALUE 196.
78 KIC-HOME VALUE 199.
78 KIC-UP VALUE 200.
78 KIC-PGUP VALUE 201.
78 KIC-LEFT VALUE 203.
78 KIC-RIGHT VALUE 205.
78 KIC-END VALUE 207.
78 KIC-DOWN VALUE 208.
78 KIC-PGDN VALUE 209.
78 KIC-INS VALUE 210.
78 KIC-DEL VALUE 211.

Procedure Division:
*
* K E Y B O A R D I N P U T R O U T I N E
*
001500-KBD-INPUT.
*
CALL X"83" USING KIA-CHAR.
IF (KIA-DIGIT = 0)
CALL X"83" USING KIA-CHAR
ADD 128 TO KIA-DIGIT.
*
001500-EXIT.
EXIT.

When you need to detect an Esc (or other) character input from the
keyboard, use this code:

CALL X"D9" USING KIA-KBD-STATUS.
IF (KIA-KBD-STATUS NOT ZERO)
PERFORM 001500-KBD-INPUT
THRU 001500-EXIT
IF (KIA-DIGIT = KIC-ESCAPE)


If you want to wait for a character to be input, just
PERFORM 001500-KBD-INPUT
THRU 001500-EXIT.
and test the result in KIA-DIGIT or KIA-CHAR. Except for non ASCII keys
like the function keys, KIA-CHAR will be the ASCII character. (e.g.
pressing the 'A' key will return "A" in KIA-CHAR.)
