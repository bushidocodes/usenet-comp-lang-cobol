I wouldn't disagree with any of the comments about the PC_PRINT
routines; they are fine but, like all interfaces with a set of
parameters to set up, they take a bit of effort to get started.
However, if you want to stay with the simple WRITE then it should be
successful and we have got them to work. You may need to use NET USE
LPT1 ... to connect to the remote printer or alternatively set up the
full printer name in a SELECT ... ASSIGN statement, where you could
construct the filename (printer name) dynamically at run time. We were
told about NET USE by Merant and while it does work I wasn't
completely convinced that it behaved on all networks (WINDOWS that
is). We currently use the printer name in the SELECT in the form of
'\\computer\printer' and this does seem to work, although you have to
be careful of strange names that WINDOWS allows but the OPEN doesn't
like (in particular OPEN seems to treat a space character as a
terminator). And of course don't forget to check the status after
accesses...

graham
