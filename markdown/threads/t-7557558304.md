When passing a non-fixed-size table to a CALLed program how should it be
declared in the LINKAGE SECTION?

Lets say  you want to pass this to a CALLed program

01  STAT-DATA.
    05  STAT-DATA-ITEMS    PIC  9(3).
    05  STAT-DATA-TABLE     OCCURS 1 TO 999 TIMES
                                                    DEPENDING ON
STAT-DATA-ITEMS.
          10  DATA-VALUE        PIC S9(3)V99.
          10  VALUE-SUB-MEAN    PIC  S9(4)V99.
          10  VAL-SUB-MEAN-SQUARED   PIC  9(7)V99.

Would you declare it the same way in LINKAGE or would you use a REDEFINES?

Steve Kalemkiewicz
```

I would use the same definition in both calling and called program.  There
are a couple of questions/issues however,

1) Is STAT-DATA-ITEMS   "filled in" by the calling or called program?  If it
is the calling program, then everything should be "fine" every time.  If it
is the called program, then SOME compilers may have problems with it.

2) IBM in their "early" '85 Standard compilers did NOT handle cases where an
ODO was filled in by the called program.  Instead, they tended to "abend".
ANSI "clarified" that this wasn't Standard conforming, but I can imagine some
compilers still having problems with this.
```
