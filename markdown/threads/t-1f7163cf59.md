I am doing a lot of work with MF Cobol and AIX and Oracle. Basically it is
a lot of pain, and my first advice to anyone who has the choice is: Don't!
Applications developed using C-ISAM (indexed) files are often faster,
smaller, and use less disk space.

Given your question, you probably don't have a choice. So, here is the
script we use to compile a .cob file, with optional precompilation if the
program contains Oracle EXEC SQL statements:


#! /bin/sh
#
# cobol Compile optionally using MicroFocus precompiler
#
# Written: 92/01/01 Leif Svalgaard
# Revised: 94/08/10 Pieter Hintjens
#
if test $1; then
for i in $*; do
shift
UPCASE=`echo $i | tr "[a-z]" "[A-Z]"`
UPCASE=`basename $UPCASE .COB`

if test -f $UPCASE.cob; then
egrep -q "EXEC SQL" $UPCASE.cob
ORACLE=$? # get return code from egrep: 0 = found
if test $ORACLE -eq 0; then
echo Preprocessing $UPCASE...
# Check for common error in hand-written SQL...
egrep "END EXEC" $UPCASE.cob
mv $UPCASE.cob $UPCASE.pco
nice procob include=$ORACLE_HOME/procob/lib ireclen=132 ¥
oreclen=132 select_error=no iname=$UPCASE.pco ¥
oname=$UPCASE.cob | tail +8
fi
echo Compiling $UPCASE...
test -f $UPCASE.gnt && rm -f $UPCASE.gnt
test -f $UPCASE.idy && rm -f $UPCASE.idy
echo > $UPCASE.err `date`
nice cob -uC "directives=$ETKDIR/direct.cob list=$UPCASE.err" ¥
-N "boundopt " ¥
$UPCASE.cob
test -f $UPCASE.int && rm $UPCASE.int
test -f $UPCASE.lis && rm $UPCASE.lis
test -f $UPCASE.gnt && rm $UPCASE.err
if test $ORACLE -eq 0; then
rm $UPCASE.cob
mv $UPCASE.pco $UPCASE.cob
fi
else
echo Program $UPCASE not found...
fi
done
else
echo '.---------------------------------------------------------.';
echo '| cobol {Program}... |';
echo '|---------------------------------------------------------|';
echo '| Compile each program in turn, producing an object that |';
echo '| is suitable for dynamic loading (.gnt format). |';
echo '| If the source contains Oracle commands, the Oracle |';
echo '| pre-processor is called before compilation. |';
echo '`---------------------------------------------------------'¥';
fi

We standardise filenames as UPPERCASE.cob, and the script is set-up to
assume and handle this.

Some other points about Oracle and its Cobol interface:

- You can compile in Oracle mode or ANSI mode (add a switch mode=ansi).
There are certain differences in behaviour. ANSI mode does not allow
variable-length (VARCHAR2) text fields. ANSI mode does not allow a
FETCH to be used over COMMIT's. and so on.

- In Oracle mode, VARCHAR2 fields are truncated into NULLs if they
contain spaces. This appears to be a basic flaw (and quite fatal)
in the Cobol interface. We get around it by using ANSI mode.

Hope this helps.

Pieter A. Hintjens
