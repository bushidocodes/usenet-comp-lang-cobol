I don't know if you text book (or other class material) covers this or not -
but you might want to look for the subject "priming read".  (This is one of
those things that CAN cause "religious debates" among programmers - but the
"change" that you made will not work as you desire in all cases - but might
in some others.)

Another topic that you might want to look for in your class material is the
"WITH TEST BEFORE" and "WITH TEST AFTER" options of the PERFORM statement.
Normally, this is NOT taught right at the beginning of a COBOL class - but
if you have had some "programming logic" courses or have programmed in other
"structured" languages, this may help you understand the issue.

P.S.  Please ignore if these terms don't mean anything to you - but the
default COBOL "PERFORM UNTIL" is equivalent to the "normal" (theoretical
programming) DO WHILE loop and is NOT the same as a "DO UNTIL" loop.  This
often causes a LOT of confusion to new COBOL programmers - who are familiar
with other languages or approaches.

If you're just learning COBOL why not learn to use the balanced
constructs, e.g.:

Process-Records.
        Read Student-File
           At End
                Move 'NO' to Data-Remains-Switch
           not At End
                process the record
        End-Read.

This way you don't need a priming read. Your logic works even if you
have a null input file. The way you've coded it, you'll process the last
record twice, once when you read it & again when you you reach the "at
end" condition. Also, you have an infinite read loop. I'll let you
figure out why.

LiamD

-----------------
when you have a series of statements based on an if statement the if
ends after the first period and no "END-IF" is required.

Usually "END-IF" is only used when nesting if statement one after
another.

Usually EOJ, the part of the program that closes the files and ends the
program is in a separate paragraph.  That way it can be used in a
perform statement if you try to open a file that does not exist.  Thus
if an error code or bad file access code is found you can say if file-
status not = zero perform end-of-job.

HERE IS HOW A SIMPLE PROGRAM THAT READS ONE FILE IS NORMALLY WRITTEN IN
STRUCTURED PROGRAMMING (as opposed to top-down programming):

001-MAINLINE-LOGIC.
    PERFORM 101-INITIALIZE.
    PERFORM 103-PROCESS
       UNTIL END-OF-FILE = 'YES'.
    PERFORM 102-END-OF-JOB.
    STOP RUN.

Usually read is separate paragraph and one read is in the initialize
paragraph and then another read at the 'END' of the process.  This pre-
reads the first record.  You do not really want to write a header file
or anything if you can not read the first the record. That would be a
waste of time and paper.

A top-down approach isnt incorrect, but the reason for paragraph
notation in COBOL is to make the program have reusable code.  This was
a pre-cursor to object oriented programming (OOP).  So COBOL has some
aspects of OOP already built in.  I don't think the term OOP was
conceived until after COBOL came into use.  Just don't tell your C++ or
JAVA instructor that COBOL is an OOP language unless you want to
infuriate them!

I think COBOL is the easiest programming language to learn.  If you
want to know if a program works just copy it and rewrite it and run it.

  Try it the isntructor's way first and then try something else and see
what works better!!!  You may actually see that the instructor's way is
better.  COBOL isnt just a programming language but a unique way of
thinking.  When you have to do a hard program, do it in steps.  Just
get part of it to work and then work on the rest.  If you do it all at
once you will not know where to start, when you get a whole bunch of
errors after you compile the program.  Have fun!

Well I totally agree below is a shell program that I use to start like 95%

       PROGRAM-ID.    PGMSHELL.

      *                    P G M S H E L L                            *
      *---------------------------------------------------------------*
      *     DEPARTMENT NAME HERE-- SAMPLE SHELL PROGRAM               *
      *---------------------------------------------------------------*
      * MODIFICATIONS/ENHANCEMENTS:                                   *
      *                        COMPILE                                *
      * REVISION  PROGRAMMER     DATE   MODIFICATIONS                 *

      *---------------------------------------------------------------*
      * LAST COMPILE:   YYYY, MM, DD      TIME: HH:MM AM              *
      *---------------------------------------------------------------*
       ENVIRONMENT DIVISION.
       INPUT-OUTPUT SECTION.
       FILE-CONTROL.

           SELECT IN-FILE1
             ASSIGN BDWAIFLE
             ORGANIZATION IS SEQUENTIAL
             ACCESS IS SEQUENTIAL
             STATUS I-FILE1-FS.

           SELECT O-FILE1
             ASSIGN BDWAOFLE
             ORGANIZATION IS SEQUENTIAL
             ACCESS IS SEQUENTIAL.

           SELECT I-CARD-FILE
             ASSIGN BDWACARD
             ORGANIZATION IS SEQUENTIAL
             ACCESS IS SEQUENTIAL
             STATUS I-CARD-FS.

       DATA DIVISION.
       FILE SECTION.

      /
       FD  IN-FILE1
           BLOCK CONTAINS 0
           RECORDING MODE IS F.
      *01  I-FILE1-REC            PIC X(9999).
       01  I-FILE1-REC.
       COPY INCPYBK.

      /
       FD  O-FILE1
           BLOCK CONTAINS 0
           RECORDING MODE IS F.
      *01  O-FILE1-REC            PIC X(9999).
       01  O-FILE1-REC.
       COPY OUTCPYBK.

       FD  I-CARD-FILE
           BLOCK CONTAINS 0
           RECORDING MODE IS F.
       01  I-CARD-REC             PIC X(80).

      /
       WORKING-STORAGE SECTION.

           05 O-FILE1-CTR         PIC S9(11) VALUE +0.

           05 D-O-CTR             PIC ZZ,ZZZ,ZZZ,ZZ9.

       01  STATUS-AREA.
           05 I-FILE1-FS          PIC X(02).
              88 I-FILE1-AT-EOF             VALUE '10'.
           05 I-CARD-FS           PIC X(02).
              88 I-CARD-AT-EOF              VALUE '10'.

      *01  RETURN-CODE GLOBAL     PIC S9(4) BINARY VALUE ZERO.

       01  DATE-AREAS.
           05 WS-CARD-REC.
              10                  PIC X(10)  VALUE SPACES.
              10 WS-HEADING-1     PIC X(60)  VALUE SPACES.
              10                  PIC X(10)  VALUE SPACES.

       01  DATE-AREA-2.
           05 WS-SYS-DATE.

                 15               PIC X(9) VALUE ' NOVEMBER'.
                 15               PIC X(9) VALUE ' DECEMBER'.
              10 MONTH-TABLE REDEFINES MONTHS-OF-THE-YEAR.
                 15 MONTH-NAME    PIC X(9) OCCURS 12.
           05 PRINT-DATE.
              10                  PIC X(02)   VALUE SPACES.
              10 O-TYPE-DATE      PIC X(16).
              10                  PIC X(02)   VALUE SPACES.
              10 O-MM             PIC X(09).
              10                  PIC X       VALUE SPACE.
              10 O-DD             PIC Z9.
              10                  PIC X(02)   VALUE ', '.
              10 O-YYYY           PIC 9(04).
           05 PRINT-TIME.
              10                  PIC X(02)   VALUE SPACES.
              10                  PIC X(16)   VALUE '    SYSTEM TIME:'.
              10                  PIC X(12)   VALUE SPACES.

              10                  PIC X       VALUE ':'.

              10                  PIC X       VALUE ':'.

       E000-INIT.
           ACCEPT WS-SYS-DATE FROM DATE

                DISPLAY ' '
                MOVE 16 TO RETURN-CODE

           END-READ

           EXIT.

       X000-EOJ.
