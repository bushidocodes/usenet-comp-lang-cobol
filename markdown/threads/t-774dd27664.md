```
Does anyone have suggestions for this situlation?

I have several very large PL/I include files for a large IMS database. I
would like
to use them with COBOL programs that will be accessing the same IMS
database. The include files can not be altered except with text inside of
PL/I
comments, far to many PL/I programs currently use these. An example of a
structure
looks like this.

/* ITEM SEGMENT STRUCTURE */
3 GT001,
5 ITEM_NUM CHAR(20), /* ITEM KEY */
5 ITEM_DESC FIXED DEC(9),
.
.
.
5 ITEM_TYPE CHAR(3); /*001=X3
002=Z2

009=A2 010=W9 */

Most structures are 100+ lines. I don't want to have to make a copy of the

file and then have to maintian a COBOL file and a PL/I file.

It may be possible to convert the structure when it is being copied into
the
COBOL program but I am not sure how to get rid of the text within the /*
*/

I think that I could make a copy of the entire structure, changing the
syntax to
make it legal for COBOL and place it after the PL/I structure, in
comments. The PL/I complier would consider the second structure one large
comment. If I could
have the COBOL program start reading the file at the 2nd structure I could
just
replace / and * with spaces. I don't know if the copy command can start
reading
a file at specifc point or not.

I would appreciate any comments, suggestion or new ideas on this problem.

Thanks,
gt
```
