```
I am running "Micro Focus COBOL for UNIX         V4.1 revision 030" on a
Linux box

I am trying to compile a program using the htmp preprocessor.  If I put a
display of an external-form inside a if/end-if statement i get an error.

For example

-bash-2.05b$ cat example.cbl
       data division.
       working-storage section.
       77 77-if-field pic 9.
       01  output-form is external-form identified by "outpage1.htm".
           03  name-field2 pic x(30) identified by name.
       procedure division.
       a10.
           if 77-if-field = 1 then
              display output-form
           end-if
           stop run
           .
-bash-2.05b$ cob -x example.cbl -C "preprocess(htmlpp)" ACCCGI.o
Micro Focus Accept/Display CGI and Embedded HTML Preprocessor
Version 1.58 Copyright (C) 1984-1999 MERANT International Ltd.
Micro Focus CP
V990720 140438 Copyright (C) 1993-1999 Micro Focus Ltd.
    10     end-if
* 564-S**********
**
**    A scope-delimiter did not have a matching verb and was discarded.
cob: error(s) in compilation: example.cbl
-bash-2.05b$


It seems that the preprocessor is expanding the display statement and has
inserted a full stop somewhere.

Anybody heard of this before?

Is it a bug?

Is there a fix?


regards
```
