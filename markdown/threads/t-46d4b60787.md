Okay, let's try.  You have a subprogram S which is called by someone, either under CICS or
in BATCH.

The program S needs to be translated by the CICS translator, which inserts 2 arguments at
the PROCEDURE DIVISION USING phrase. You must take care of this in the CALLING program,
which has to provide two dummy arguments

Also, You cannot use the EXEC CICS commands outside of CICS.

Therefore, you need a little program that checks if you are running under CICS.  An
Assembler program that uses the DFHAFCD TYPE=LOCATE,CB=AFCB macro would return that
information.  Then you would have to write the program in a two thronged approach, one
path for the case when you are in CICS (using the EXEC CICS commands), and the second path
when running in batch (not using the EXEC CICS commands).

BTW, if you look at the expansion od the DFHAFCD MACRO, you can do the same thing under
CICS, without having to ressort to Assembler.  It is just a simple matter of following a
control block chain

Ken,

When calling a CICS Program from Batch (a program that has CICS
commands in it), addressability must be satisfied for the first two
parameters that the sub-program is expecting (DFHEIBLK and DFHCOMMAREA)
in your CALL statement, such as:

03 DUMMY-ARG PIC X(01).
03 THIRD-PARAMETER PIC X(whatever).

CALL SUBPGM USING DUMMY-ARG, DUMMY-ARG, THIRD-PARAMETER.

The CICS Translator inserts (during the TRANSLATION Stage):

PROCEDURE DIVISION USING DFHEIBLK DFHCOMMAREA.

The sub-program source must specify:

PROCEDURE DIVISION USING THIRD-PARAMETER.

During the TRANSLATION Stage, the Translator will then insert DFHEIBLK
and DFHCOMMAREA just prior to THIRD-PARAMETER, resulting in the correct
parameter sequence.

I'm curious of how it was done without these requirements ? Did the sub-
program call another sub-program to determine the execution environment
or was there intelligence placed in the THIRD-PARAMETER ?

However, if the sub-program does NOT contain CICS Commands, then this
is perfectly legitimate co-usage and none of what I've said is
applicable.
