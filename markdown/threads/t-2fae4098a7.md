On Wed, 26 Aug 1998 06:45:36 -0400 "Jim Gallagher" <jimgall@sprintmail.com>
wrote:

:>Hello,
:>I am new to the IBM mainframe environment (not new to COBOL)  - I am writing
:>batch programs using COBOL II and DB2 in an OS390 environment.  One program
:>must create a file for each of multiple institutions. The files would be
:>named BK.PROD.001INST, BK.PROD.002INST etc. Institution number is a 3
:>position field.  Is there a way in IBM COBOL (other than allocating and
:>declaring FDs for 999 files in my program) to handle this ? i.e. create
:>BK.PROD.001INST, when institution # changes close it and reopen it as
:>BK.PROD.002INST.

:>Any pointers would be most appreciated.  BTW this is not homework !!

Jim, 

I am just back from a US trip and it appears that no one has suggested the
following solution.

Assuming that you will always be producing all the files, the following JCL
will do the job.

//MYDD   DD  DSN=BK.PROD.001INST,......,FREE=CLOSE
//MYDD   DD  DSN=BK.PROD.002INST,......,FREE=CLOSE
//MYDD   DD  DSN=BK.PROD.003INST,......,FREE=CLOSE
//MYDD   DD  DSN=BK.PROD.004INST,......,FREE=CLOSE
//MYDD   DD  DSN=BK.PROD.005INST,......,FREE=CLOSE

Each time you close the file assigned to MYDD the current allocation will be
released and when the file is reopened the next MYDD will be used. 

Of course, with this approach, if you for example wish to skip 003INST you
would still have to open and close it.

IBM has spent considerable effort over the years in standardizing
calling conventions and run-time environments to allow linking together
in a single load module code produced by Assembler and a multitude of
compilers.  It wouldn't make sense (unless the standard language
definition required it) to imbed/replicate assembler-like capabilites
into other compilers, when this can be done with greater clarity by a
separate assembly, using the linkage editor to merge.

The original problem posed (allocating a large number of datasets with a
"data-related" dataset name with a minimal number of FD's) would
probably be done at our shop using a CALL to a DYNAM subroutine
(implemented in assembler to supply functionality not available in
COBOL).  This subroutine, originally obtained from the MVS MODS tape
(available through SHARE, NaSPA, and possibly other sources), provides
an interface for dynamically allocating and deallocating a data set to a
DDNAME (in other words, performs at program execution time using runtime
data the equivalent effect of including a DD statement in the JCL), but
allows re-use of the same DDNAME and all the allocation parameters can
be variables under program control.  The general sequence of actions is
DYNAM ALLOC CALL, OPEN, process, CLOSE, DYNAM DEALLOC call; and this
sequence may be repeated many times for a single DD NAME (FD name in
program).
