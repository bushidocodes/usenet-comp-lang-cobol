Michael,

The DLL you have created will have been linked with the shared runtime
on COBOL 4.0. This means that COBOL runtime support files need to
be available for the DLL to run.

I suspect the error 53 is being generated because these cannot be found
i.e. your DLL is found OK but when this attempts to load the runtime DLLs
they cannot be found.

You need to ensure that the runtime is either in the current directory or on
the PATH as well.

Hope this helps.
