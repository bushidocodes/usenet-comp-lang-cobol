```
No idea how to do this using Dialog System, but if that system passes
windows messages or supplies an 'event' of some kind, the control's parent
will get an LVN_COLUMNCLICKED code in a WM_NOTIFY message whenever the user
clicks the box. At that point you can interrogate the checked status and do
something.

If nothing else, hope this tells you where to look in the Dialog System
manual/help file.
```

```
Marcos,

as you know, you may have more than one listview items (lines) checkboxed.
If I have well understood, this is not the best way to select a line to be
displayed in an entryfield since when you have marked many lines (without
selecting the line) you will not know to which specific one the entryfield
data corresponts.
Anyway,
to achieve what you requested, follow the next steps :

1.  In the Listview program (that created by NetExpress) in the
    Register-Callbacks section add :

    MOVE '"' & ProgramID & z'CheckClicked"' TO MessageName
    INVOKE EntryCallback "new"  USING MessageName
                                RETURNING aCallback
    MOVE p2ce-ValueChanged TO i
    INVOKE aControl  "setEvent" USING i aCallback
    INVOKE aCallback "finalize" RETURNING aCallback

2.  Add a section that defines an Event-Type value for the case a checkbox
    is selected :

       Item-CheckBoxChanged section.
       Entry ProgramID & "CheckClicked" USING anEvent.
           PERFORM GetDataBlockPointer
           INVOKE anEvent "getSelectedItem" RETURNING aListViewItem
           IF  aListViewItem NOT = NULL
               INVOKE aListViewItem "class" RETURNING aClass
               IF aClass = ListViewItem
                  INVOKE aListViewItem "getCheckBoxValue"
                                             RETURNING itemCBState
               END-IF
           END-IF
           MOVE 34594  TO Event-Type
           CALL ProcessItem USING aListViewItem
           EXIT PROGRAM
           .
 3.  In Dialog Definition of corresponding screenset you add in the
     user-event dialog :
     xif= 34594 $event-data lview-item-checked

4.   lview-item-checked procedure should contain a RETC to program
     to locate listview item index.

5.   So, from field ietmCBState (see step 2) you get checkbox value
     (0=unselected / 1=selected) and from lview-item index (see step 5)
     you get lview-item data (that is all columns values).
```
