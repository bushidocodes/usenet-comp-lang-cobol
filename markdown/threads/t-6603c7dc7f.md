of the COBOL scripting language. It must be a better tool. So, any feedback

To call the "scripting" language in Net Express COBOL is bogus and
misleading in my opinion. What they are talking about is Dialog System -
at least for Windows development.

The syntax is not procedural, not like COBOL at all.

For example:

A simple move.

COBOL:

Move A to B.

Dialog:

MOVE A B

A simple conditional perform:

COBOL

IF A = B Perform The-Routine

Dialog

XIF= A B THE-ROUTINE

A conditional GO TO

COBOL

IF A = B GO TO THE-ROUTINE

Dialog

IF= A B THE-ROUTINE

There is no simple way to do something like this in Dialog:

IF A = B MOVE B TO C

You have to do something like this

XIF= A B MOVE-ROUTINE

MOVE-ROUTINE
MOVE B C

So if anyone claims that Netexpress /Dialog system is COBOL scripting, they
are misleading you.

If you want to stick with STRAIGHT COBOL I suggest using the PC COBOL of
your choice and COBOL SP2 from Flexus international: http://www.flexus.com

It uses a simple Ansi standard call interface to the GUI and ALL procedure
takes place in your COBOL code.
