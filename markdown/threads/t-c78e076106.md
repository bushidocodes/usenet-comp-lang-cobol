```
I am not sure if this problem is a Fujitsu COBOL problem, a Visual Basic
problem, or a Windows NT problem,  it is however a PROBLEM.

I have installed on my machine Visual Basic Enterprise 5.0 as well as
Fujitsu COBOL 4.0 enterprise version.  I purchased Fujitsu COBOL because it
advertised the ability to work well with Visual Basic.

I am writing an application that uses Visual Basic for all main parts of the
application, which calls some DLL's (with linkage sections) that I have
created using Fujitsu COBOL Project Manager.  This application is to be used
on computers running Windows NT.  The application works perfectly on my
Windows NT machine.  I am trying to distribute this application to others.
I have used the Visual Basic Application Setup Wizard to create distribution
diskettes.  I have made sure that the setup includes all of the DLL files
and I am loading them into whatever directory the application is being
loaded into.  I can load up from the setup diskettes on my production
computer and everything works fine, however when I load the setup diskettes
onto a clean computer (without Visual Basic or Fujitsu COBOL) on it the
application does not run properly.  All aspects of the Visual Basic part of
the application run perfectly, but when the application tries to run any of
the COBOL created DLL's I get a Visual Basic runtime error saying that file
not found for whichever Fujitsu COBOL DLL is being called.

My question is do these DLL's have to be entered into the registry for the
application to know where to find them?  If this is the case is there a way
in the Fujitsu COBOL compiling, linking process that I can make the DLL self
registering?  If not any good ideas how to get the DLL's registered via
distributed setup diskettes? Any and all information/help would be
appreciated.
```

```
You can do it in the COBOL.CBR setup file, like so:


[WEIGH IN]
@CBR_SCR_KEYDEFFILE=\ADSN.PROGRAMS\COBOL.KBD
@NoMessage=YES
@EnvSetWindow=UNUSE
@ScrnFont=(Terminal,20)
@ScrnSize=(54,7)
@WinCloseMsg=OFF

[WEIGH IN.ENTRY]
SioBaud=WSC32.DLL
SioDTR=WSC32.DLL
SioGetc=WSC32.DLL
SioParms=WSC32.DLL
SioReset=WSC32.DLL
SioRTS=WSC32.DLL
SioDone=WSC32.DLL

The program name "WEIGH IN" is contained in quotes in the program-id.
```
