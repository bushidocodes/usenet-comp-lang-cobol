I am currently using VB for a gui shell and Realia 5 for the workhorse
underneath. The problem is this: I would like to be able to just have
the Realia process a file completely before it returns control to VB.
However, VB users these days expect to see a progress bar or some
other visual indicator of something happening. Has anybody written a
routine (involving the windows API I'm sure) to do that? If I make one
that is controlled from VB then I basically have to call Cobol one
time for each record. The overhead results in a 15-fold time increase
to read through the file. Whiel we're at it, does anyone have a better
idea?
