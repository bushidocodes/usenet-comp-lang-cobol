Hi YM,

The rule for initial entry into a PERFORM loop with TEST BEFORE
(default) is:

Set the identifier (SUBW) to the current FROM value [“+1“]
If Condition-1 [“SUBW > MAX-WIRES, i.e. +1 > 0”] is true, exit PERFORM
Else, execute specified set of stmts

You stated that MAX-WIRES is pic 9999 value 0 and SUBW is set to +1 by
the PERFORM stmt. With these values upon entry to the PERFORM the
PERFORM set of stmts is  NOT executed.

If “COMPUTE SUBW = MAX-WIRES + 1” is the only place in your pgm that
modifies MAX-WIRES, you’ll NEVER execute those PERFORM stmts because
SUBW [“+1”] is ALWAYS  > MAX-WIRES [“0“].

Regards, Jack.

My previous suggestion sounds like clutching at straws and it is, but
it's based on this premise:

Something in the code is causing it not to execute as expected or is
causing the compiler to misinterpret the code. So, eliminating pieces
of the code should eventually excise the offending segment (or
eliminate the code entirely :-).)

Since you can't eliminate required segments, you're limited to
optional segments. The parens are the only optional segments of the
code, sooo.
