Greetings,

1) I have written a program in Net Express (3.0) for our intranet.
   It responds quickly when accessed from a browser on the same machine
   that Solo is running on.  But when accessed from a browser on another
   machine it takes at least 30 seconds to respond.  I assume that Solo
   has to wait for a 'time slice'.  Any suggestions?

2) We also have an existing program which uses MQSeries (written in
MicroFocus)
   which we are having difficulty in converting to Net Express.  When we run
it
   it cannot seem to find the module MQCONN.  We tried specifying the name of
   the .LIB file in the link.  We tried SET xxx TO ENTRY "MQMCB1" to get the 
   .DLL loaded at runtime.  But all to no avail.  Any ideas?

Thanks (in advance)  Alan H

We have managed to fix our problem with calling (MQ Series).

the procedure division to CALL "__MQCONN" it now works.
