I think you'll find that you're using DOS methods in a Windows environment
that don't work any more.

The windows runtime will be executed as a new process and reset back to the
initial 'new process' settings.

Have a look in the Define Device part of the manual (User Guide
Configuration File).  This will basically map your traditional 'dumping' of
data to a printer port into the correct windows Print Manager/Spooler/Print
Driver etc.  You can also look at the Synonyms (Windows Config in UG) to map
printer names to different devices for different programs or select a
different Config file for each Icon.....

Another bit of advice is to make sure you've upgraded the Runtime to the
latest V7.00.03 with patches.  These can be downloaded from www.liant.com.
There are some enhancements in printing over earlier V7 releases.

For more details/specific help email to support@liant.com

The Windows printing support in V7 is very good.  You should be able to
support 'legacy' character reporting (Esc sequences etc) in a Windows
environment without any code changes (so long as you haven't done silly
things like specifically for Epson printers (nobody would be that sillly
would they......)).  You can also do sexy new things that people outside
this NG would certify you for if you ever talked about them - so don't tell
anyone!
