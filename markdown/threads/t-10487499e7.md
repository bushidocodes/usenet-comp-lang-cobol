```
I was in a similar situation last year, but I simply wanted to be able to
process MVS file names through FTP. I was unable to find one, as you. So I
wrote my own. For lack of an actual programming language or real FTP client,
I used Microsoft Access VB and the FTP client which comes with Win9x.

What I did was use VB to create a BAT file to call FTP.EXE and route its FTP
directory listing to a text file. I then imported the text file into a MS
Access table. I then used the CLICK event to indicate which file I wanted to
download. Then simply use another BAT file to call FTP.EXE to download the
file. In your case you would then change the output filename based on the
last node of the file name. Further you could incorporate a file association
manually, or use EXPLORER and the windows built in associations established
by your applications to open the files with the appropriate application.
There is a way to use VB to access the registry to find which application is
associated to each extension and the application file path. But unless you
would have to talk to someone with more VB experience to figure that out.

The MS Access database I wrote is rather simple and not too big. I could
e-mail ot to you if you are interested.
```
