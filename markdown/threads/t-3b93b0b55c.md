Another possible issue is how you have determined that the delete did NOT
work.  There are some file systems that leave a "physical record" in a
file - while logically deleting them.  Therefore, if you use COBOL to read
the file, the record will appear to be gone - but if you look at it thru
some hex editor or other system, the record may appear to still be there.

'delete' records (which only turns off a flag relative to the record position)
and leaves teh record intact.  To 'purge' a record from an ISAM/Indexed file is
to do a rebuild after marking records for deletion, similar to 'packing' a
dBase file.
