It is quite simple.  The default is to check before so:

   MOVE "Y" TO END-FLAG.
   PERFORM READ-FILE UNTIL END FLAG IS EQUAL TO "Y".
*****does not execute READ-FILE at all*****

   MOVE "Y" TO END-FLAG.
   PERFORM READ-FILE  WITH TEST AFTER
          UNTIL END FLAG IS EQUAL TO "Y".
***** executes READ-FILE once(provided END-FLAG is not reset)*****

The same thing is true of varying. So if the condition is true on entry, the
perform would not happen. IE:-

PERFORM XXX VARYING SUB1 FROM 1 BY 1
    UNTIL SUB1 IS EQUAL TO 1.

Would never execute XXX, as the condition would be true on the first
instance.

PERFORM XXX VARYING SUB1 FROM 1 BY 1
    UNTIL SUB1 IS EQUAL TO 2.
executes 1 time.

PERFORM XXX VARYING SUB1 FROM 1 BY 1
    UNTIL SUB1 IS GREATER THAN 2.
executes 2 times(and is the common usage).

Default tests after is equivalent to  DO/UNTIL

     PERFORM VARYING SUB FROM 1 BY 1
             UNTIL ((SUB > SUB-LIMIT) OR
                        (EARLY-EXIT))
             some data manipulation
             some test for a condition
                   set EARLY-EXIT TO TRUE
                   end-if
    END-PERFORM

     PERFORM VARYING SUB FROM 1 BY 1
             UNTIL ((SUB > SUB-LIMIT) OR
                        (EARLY-EXIT))
             WITH TEST BEFORE
             some data manipulation
             some test for a condition
                   set EARLY-EXIT TO TRUE
                   end-if
    END-PERFORM

Pay no attention to the man behind the curtain. WITH TEST BEFORE
is the default. WITH TEST AFTER, if desired, must be coded.
The difference is whether the data manipulation code within
the loop gets executed at the terminal condition, viz:

PERFORM UNTIL (condition)
     [test for exiting condition is done here]
     data manipulation
     END-PERFORM.

PERFORM WITH TEST AFTER UNTIL (condition)
     data manipulation
     [test for exiting condition done here]
     END-PERFORM.

Specifically,

PERFORM VARYING COUNT FROM 1
     BY 1 UNTIL COUNTER = 3
     ADD COUNT TO SUM
     END-PERFORM
(Result: SUM = 3)

differs from

PERFORM WITH TEST AFTER VARYING COUNT FROM 1
     BY 1 UNTIL COUNTER = 3
     ADD COUNT TO SUM
     END-PERFORM
(Result: SUM = 6)
