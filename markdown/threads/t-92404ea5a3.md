The COBOL standard describes how an indexed file should be accessed by
COBOL using the record KEY and the READ, WRITE, REWRITE, and START
verbs.  It does not specify the underlying physical access method to be
implemented by a particular compiler or operating system.  It is
therefore possible that Liant's implementation is proprietary, and the
spec may not be available for free.

I code COBOL in an IBM mainframe environment.  I prefer all uppercase
characters simply because the search function on my text editor works
better on all uppercase.  Modern COBOL compilers allow upper/lower/mixed
case source code, but COBOL is not case-sensitive when interpreting
reserved words and user-defined names.

As far as paragraph names and data identifiers go, there are lots of
different opinions on what constitutes good COBOL coding conventions.  I
work in a shop that prefers 3-digit numbers as dataname prefixes, with
the same prefix for all datanames within an 01 record.  Paragraph names
are all prefixed with a 4-digit number, and we use a numbering scheme
that reflects the hierarchical structure.  I am comfortable with these
rules, but Richard Plinston, a regular in comp.lang.cobol, will tell you
that paragraph numbers are a complete waste of time.  Since COBOL
datanames and paragraph names are limited to 30 characters, using
numeric prefixes leaves fewer characters for meaningful text.

One rule we have is that a paragraph name should begin with an action
verb, have a direct object, and describe the function of that
paragraph.  So I work with programs that have paragraph names like
1100-DISPLAY-START-MESSAGES, 1200-OPEN-THE-FILES,
8000-GET-NEXT-ACCOUNT-RECORD.  Obviously, if you don't care for
paragraph numbers you can omit them, but I like the verb/direct object
rule.  If the paragraphs are named in a meaningful way, the COBOL
program will be easier for a maintenance programmer to understand.

The RM/COBOL runtime is customizable.  Look for a sub.c file that comes
with your COBOL distribution (it's probably in /usr/rmcobol).  You can add
your own routines to that and rebuild the runcobol distribution -- it
should be possible to add a sleep() function call with the period
in the USING (I'm a C programmer who has to do this for our COBOL people,
so I know that it's possible and relatively easy)..

The files needed are standard with the RM/COBOL UNIX installation...if
you don't have them, call up Liant Software (www.liant.com is the
website).
