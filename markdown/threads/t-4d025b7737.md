If you use the correct setting for the "MF" directive (I think that COBOL
2.4 was about "MF(8)" - but I can't remember for sure), then there should be
NO differences for existing source code. In other words, you can compile
and execute your existing programs with no modifications.

If you then want to TAKE ADVANTAGE of the new features of NetExpress, you
can do so on a program by program basis - or with any new applications that
you develop.

P.S. The first paragraph is sort-of a theoretical statement. There are
actually some semantic differences - but those should be documented in the
MF "compatibility" documentation. Where you are making such a HUGE jump
(skipping about 5 releases between 2.4 and NetExpress) you may need to ask
MF for a few old documents for the old upgrades. However, they should be
able to provide them all for you (if you ask them).

Moshe,

Bill is correct. Generally speaking your source code from 2.4 should
re-compile and run under NetExpress. The areas you are most likely
to have issue with are where you have used undocumented features
of 2.4 (which may be no longer supproted in NetExpress) or anything
which is environment specific e.g. if you have used DOS interrupts
or used DOS assembler routines. These will need to be replaced
with a Win32 equivalent.
