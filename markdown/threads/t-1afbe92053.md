Though the ordinal number of  '9' is greater than 'A' in EBCDIC, this is
not true in ASCII. And even in EBCDIC there are more characters that
come between 'A' and '9' than just the letters and numbers (specifically,
hex 'CA' through 'CF', 'DA' through 'DF' and 'EA' through 'EF'). Also,
what about lower case? Some names begin with lower case letters, and
all lower case letters collate outside your range, in both EBCDIC and
ASCII.  I.E. your solution is neither complete nor general.

You could do it like this:

    SPECIAL-NAMES.
        CLASS ALPHA-NUMERIC IS
            'a' THRU 'i'
            'j' THRU 'r'
            's' THRU 'z'
            'A' THRU 'I'
            'J' THRU 'R'
            'S' THRU 'Z'
            '0' THRU '9'.

The above is complete, and would work in both EBCDIC and ASCII
(though ASCII would not require that the alphabet be divided in this
way). But it would still not be general for every possible collating
sequence. That would require that you list every single character,
like this:

    SPECIAL-NAMES.
        CLASS ALPHA-NUMERIC IS
            'a' 'b' 'c' ... 'y' 'z'
            'A' 'B' 'C' ... 'Y' 'Z'
            '0' '1' '2' ... '8' '9'.

The above is complete and general but, depending on how the compiler
implemented it, could potentially be VERY inefficient.

A simple complete and general solution is:

    IF (Name(1:1) IS NOT ALPHABETIC)
         AND
        (Name(1:1) IS NOT NUMERIC)
          <error>

It may not have the 'elegance' of the special names class solution, but
it has the advantage of being immediately obvious. :-)

write name handling code very carefully with lots of thought, and test
it on lots of real data. Sadly, as you point out, this is not always done.

More precisely, 'A' THRU '9' would include characters with collating
sequence between 'A' and '9' inclusive in the character set being used by
the program when it is run. There are no characters between 'A' and '9'
in the ASCII set, because 'A' (hex 41) is greater than '9' (hex 39). In
EBCDIC 'A' is less than '9', but also included are characters between
'I' and 'J', 'R' and 'S' and 'Z' and '0'.
