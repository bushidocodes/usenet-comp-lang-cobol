       DISPLAY WS-1

       .

```
The '85 Standard (with or without any amendments) does NOT allow for nested COPY 
statements - AT ALL.  Although a "medium-common" extension to that Standard, it 
was not made standard until the '02 Standard.  In that Standard (and with SOME 
implementors' extensions to the '85 Standard) there is the following documented 
restriction.

Page 35 (of the '02 Standard)

 "11) If the REPLACING phrase is specified, the library text shall not contain a 
COPY statement."

(in other words,
   "thou shalt not mix nested COPY statements with the REPLACING phrase.)

For any implementations that DO allow this "mixing" then you need to find out 
how they do it (and I doubt it will be portable).

NOTE:
  I won't swear to this, but my MEMORY is that with the '02 Standard, all COPY 
statments are resolved BEFORE all REPLACE statements (not replacing phrase). 
Therefore, a REPLACE statement will impact nested copy statments.  If Chuck or 
Thane are "watching" this, they may be able to confirm or correct what I have 
just said.
```
