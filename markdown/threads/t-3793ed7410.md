```
Good Morning,

I hope someone can comment on this problem because i have no idea what to do to
get this to work.  Here is my problem:

I am using NetExpress as my development system in Win9x.  I have written a
point of sale system that works on an IBM SureOne.  This terminal comes with a
thermal serial printer (Star).  My boss wants me to print a bitmap which is the
logo of a potential customer on the printer.  There are no print drivers for
this printer.  I use control codes to set the bold print, center text etc.  I
sent IBM a note asking how to accomplish the printing of the bitmap.  They told
me that the  SureOne only supports graphics printing of graphic images that are
defined
as part of the application program.  In the technical documentation there is an
example of a control code that allows you to execute normal density dot
graphics.   They then give an example of the control codes and hex values that
are sent to the printer in order to get a "train" to print on the thermal
printer.  My boss gave me a business card of our potential customer and told me
he wants that to print on the thermal printer.  I have four days to get this to
work.  Could anyone give any suggestions ?

Thanks,
Bob Hennessey
```

```
Well fellow, you ain't got much time. Best suggestion I have depend on one
issue, is the POS application of yours a native Windows one, being capable
of loading windows printer drivers?

If so, and there really is no windows printer driver available, your best
shot is to replace the printer. There is a number of alternatives out there
which do have windows printer drivers check http://www.starmicronics.com/
and/or http://www.epson.com/

You will then have to alter you application to use windows printer driver,
and my guess, given the need for errortesting and my knowledge to POS
systems, is that 4 days is way too short.
```
