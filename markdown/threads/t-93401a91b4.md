```
Hello,

I tried using the following evaluate statement:

01 WSWA-PRIORITY  PIC X(6).

EVALUATE WSWA-PRIORITY
  WHEN 'HIGH'
    ...
  WHEN 'MEDIUM'
    ...
  WHEN 'LOW'
    ...
END-EVALUATE

When I compiled it I got the following error on each "WHEN" statement:
  "An invalid abbreviated relation condition was found."

All the documentation I looked through were indicating that this should
work fine.  The following evaluate does work fine:
 
EVALUATE TRUE
  WHEN WSWA-PRIORITY = 'HIGH'
   ...
  WHEN WSWA-PRIORITY = 'MEDIUM'
   ...
  WHEN WSWA-PRIORITY = 'LOW'
   ...
END-EVALUATE

Does anybody have an explanation why the first evaluate statement
doesn't work.  In case you want the following info I'm working with the
"IBM VS COBOL II Release 3.1" compiler.

Thanks a lot,
  Claus
```

I figured out the cause of the problem.  All the time I was using
evaluate statements of the following kind:
01  WSWA-VARIABLE              PIC XXX VALUE 'YES'. 
01  WSWA-INDICATOR             PIC X   VALUE SPACES.
    88  WSWA-INDICATOR-OK              VALUE 'Y'.
    88  WSWA-INDICATOR-NOT-OK          VALUE 'N'.

EVALUATE TRUE
  WHEN WSWA-INDICATOR-0K
   AND WSWA-VARIABLE = 'YES'
    ...
  WHEN WSWA-INDICATOR-NOT-OK
   OR  WSWA-VARIABE = 'NO'
    ...
END-EVALUATE

In this case AND and OR are fine.  However, in my other situation the OR
does not work. (I was so unfortunate and tried to simplify things by
leaving the or out.)

EVALUATE WSWA-PRIORITY
  WHEN 'HIGH'
   OR  'H'
     ...
  WHEN 'MEDIUM'
   OR  'M'
     ...
  WHEN 'LOW'
   OR  'L'
    ...
END-EVALUATE

It would be nice to do things like that.

Thanks for all your help,

Claus,
you're right, it would be nice to write these kind of things.
Unfortunately, in COBOL 85, it is not allowed, it's just not valid
syntax. Your expectation is that every combination of selection subject
and selection object is valid as long as it results in a valid
condition.
The draft new standard does allow this. It introduces another kind of
section object called "partial expression", which is used to supplement
the corresponding selection subject to form a conditional expression.
This is just one example of many in the draft standard where the
language does what you intuitively think it does.
Artur
