```
Ultimately it really doesn't matter.  The DECLARE statement generates no
code within the module.  It is there only to let DB2 know what you are
intending.  The DECLARE can be coded in Working Storage or Procedure
Division depending on standards, rules of thumb, whim.........

I generally decide by the scope of the SQL for which the DECLARE is based.
If this is a subroutine which only processes the SQL that requires the
DECLARE then the best place is probably Working Storage.  If the program
does a lot of other things and one of those things is a paragraph which
contains the SQL in question then it would make sense to have the DECLARE in
or near that paragraph.

I like to keep my declares as close to the SQL using it as possible for
clarity's sake.  The bottom line is that the positioning of the DECLAREs
should make their use clear to the next programmer that comes along!!
```
