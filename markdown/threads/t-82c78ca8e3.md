```
We are converting a bunch of COBOL-74 programs to COBOL for MVS (COBOL
II). Many of these programs read a QSAM file "randomly" using a
relative record number using the NOMINAL key clause. Of course, the
ideal (and possibly only) technique is to move the data to VSAM ESDS
files using relative record number (RRN) keys. Is it possible in the
latest versions of COBOL to still read QSAM fixed-length records by
relative record number without converting the files to VSAM ESDS?
```

```
From what I can garner from SG28-6483-2 "IBM OS/VS COBOL Compiler and
Library Programmer's Guide", NOMINAL KEY is used for "Relative Files",
in addition to ISAM files.

Relative files are/were accessed sequentially (BSAM) and randomly
(BDAM) using NOMINAL KEY.

A more recent, but not current, manual that I have mentions "relative"
VSAM files being accessed randomly in a similar way using "RELATIVE
KEY". Makes me wonder if you might be able to access those files you
have using VSAM "RELATIVE KEY". It's worth a shot.
```
