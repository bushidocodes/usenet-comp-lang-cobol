The result can only be the same, if the storage layout is the same;
that is, if you did not change 'char-buffer', as indicated.

The library routine 'call' is writing 80 characters beginning at the
address of 'char-buffer'. After adjustment for the 'pic x(2) comp-x'
for 'string-len', the storage layout, you provided, has one horizontal
character (x"c4") at 'char-buffer', seven spaces (or low-values),
one low-value (from 'string-len'), one "P" (value 80 from 'string-len'),
six spaces (or low-values), and 64 undefined character positions
of spaces (or low-values). This layout is what you saw.

Changing 'char-buffer' to 'pic x(80) value all x"c4"' gives a storage
layout of 80 consecutive character positions, all with the value x"c4"
(a horizontal line). This layout is what you said you want to see.

This type of problem occurs when meaningful names are not
used. The combination 'character-buffer' and 'string-length'
may be confusing; but 'character-buffer' and 'buffer-length'
would be more clear. Also, the use of 'string' in the sense of
the C programming language may be confusing to those
not familiar with C. The stated purpose of the library routine
is: "Writes a string of characters to the screen." This might
be clearer if stated as, "Writes a buffer ..."

Look at the routine CBL_WRITE_SCR_N_CHAR which writes
a single character to successive positions on the screen. This
routine does what you wanted, in the way you expected. Even
here, the use of "N" in the name of the routine was, likely, derived
from the use of 'n' within some string routines in C. Also, the
routine is described as writing "to a string of positions" which
is less clear than "successive positions".

In COBOL 'string' is a verb, using it as a noun may be confusing
to some.
