I use SPF/PC as the editor, a Microfocus as the compiler. Comes closest
to what I wanted. Much more power in some of MF's offerings, but this
combination seemed to have the shortest learning curve. If you are OS/2
based, then you also have a choice of the IBM Visual Age COBOL, which is a
very good match for COBOL for MVS and VM / COBOL/370 / VS COBOL II. MF
has excellent match for OS/VS COBOL as well as the later stuff.

All of this stuff works, the extent of success varies with the design and
infrastructure requirements.

Of course you could always upgrade your PC to an IBM Server 500 P/390
system and run MVS with the native compilers, databases, etc. Then who
cares about emulation. The best of both worlds - Pentium and PC
technology for moderate cost, P390 to run MVS and have complete
transparency.

Rex Widmer
Builder of software archeology tools and other strange programs to help
survive in a legacy based world.

a) probably not; b) yes, yes etc.

The snag is that Workbench is pretty pricey. MF also do 'educational'
Personal Cobol products that are pretty cheap, and presumably great
for learning the OO extensions, etc. This may be OK for prototyping
code to upload to MVS, but I think only Workbench gives you the
cross-platform support you'll need for serious testing (its debugging
tool Animator is wonderful - you'll wonder how you survived etc etc).

BTW, MicroFocus experience (DOS WB devt targetting VSE mainframe at
first) helped me jump from mainframe to Unix/PC work, so it could be a
horizon-broadening experience.

PS I don't work for MicroFocus!

I just finished my first COBOL course (I'm a newbie) and purchased one
of Micro Focus' educational products, Personal COBOL for Windows. All
the code I wrote on the personal edition carried over to the MF Workbench
at school without a problem. I don't know if this will give the cross platform
support needed, but for $47 it maybe worth a try. Also the editor in
the windows edition is fantastic. The editor allows you to cut and paste,
scroll though the code and perform all those other Windows tricks. It also
color codes your code (verbs are green, variables brown, etc.) which makes
finding errors much easier. The Animator is included and has all the
functions of the Workbench version (It looks and acts just like the editor).
I hope this helps.
