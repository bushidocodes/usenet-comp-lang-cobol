Paul,

Rick has already pointed out where you are having a problem.

As regards your last comment, I doubt it very much. I originally suggested that you delete
compilation files and rebuild, to see if that would highlight your problem - it gave you the 173
'Program Not Found'. Having to delete compilation files and rebuild the project is not a panacea
for all ills- in fact it would be a lousy compiler if you had to keep doing this all the time.
(There is another compilation file - xxxxx.msg, listing compilation error messages. Delete that
also along with the INT and IDY files. Pure speculation on my part, but I suspect that VERY, VERY
occasionally, the compiler gets out of sync because of  the errormessage file).

It's aeons since I used VISOC but I can say with a fair degree of confidence that my OO code in
Net Express V 3.1 would work with VISOC. Sure, VISOC had bugs which people found, but I never
came across them, bearing in mind the OO syntax was an entirely new baby. What you are doing with
VISOC for Screen handling is an extension of what was well proven in the DOS (Text) V 3.0
compiler, long before VISOC was introduced.

As you know from the IDE to compile you can choose - (1) CTRL F7 - recompile current source, (2)
F7 - Build all programs changed and  (3) Rebuild ALL. Until you are totally comfortable with the
product, choose # 2. When satisfied the project is complete, then you can opt for #3 to make sure
all is kosher.

One thing I have found - something doesn't work and I say to myself, "Bloody compiler !". I solve
it, then say to myself, "You dumb bugger - it's your fault !". <G>

Please confirm, are you using the editor in the VISOC IDE or are you jumping out to your
'favourite' Editor to create/change your source ?
