### Another example of a good use of Evaluate.

```
Here's the situation. In a POS system, there can be 2 tax tables. Items
can be taxable for both rates, taxable for neither, taxable for one or
taxable for another. This is a windows based POS and after reading a price
lookup file, the state of 4 radio buttons is determined based on the
taxability of the item related to the 2 tables. Here's an exellent place
for an evaluate.

In the old days it would have been either 4 separate IF statements with
AND's, or it would have been an IF ELSE ELSE ELSE.

Here ya go.

EVALUATE CD-TAXABLE ALSO CD-TAXABLE-2
WHEN "Y" ALSO "Y"
MOVE 1 TO TAXABLE-RB-STATE
MOVE 0 TO EXEMPT-1-RB-STATE
MOVE 0 TO EXEMPT-2-RB-STATE
MOVE 0 TO EXEMPT-RB-STATE
WHEN "N" ALSO "N"
MOVE 0 TO TAXABLE-RB-STATE
MOVE 0 TO EXEMPT-1-RB-STATE
MOVE 0 TO EXEMPT-2-RB-STATE
MOVE 1 TO EXEMPT-RB-STATE
WHEN "Y" ALSO "N"
MOVE 0 TO TAXABLE-RB-STATE
MOVE 0 TO EXEMPT-1-RB-STATE
MOVE 1 TO EXEMPT-2-RB-STATE
MOVE 0 TO EXEMPT-RB-STATE
WHEN "N" ALSO "Y"
MOVE 0 TO TAXABLE-RB-STATE
MOVE 1 TO EXEMPT-1-RB-STATE
MOVE 0 TO EXEMPT-2-RB-STATE
MOVE 0 TO EXEMPT-RB-STATE
END-EVALUATE
```
