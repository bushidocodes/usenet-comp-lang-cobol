Practical Extraction and Reporting Language.

It started as a mix of awk, sed and bash. The main problem with it is
that everything can be done in several different ways. This means that
someone who can write PERL in one style may not be able to understand
programs written in a different style. The standard stype is 'write-
only'.

It also does not have OO, just a kludge that sort-of makes it like OO.

says 'show the UI' and does an autoconnect of the events in the UI to
functions in the program.

This runs on Linux, Windows and even my pocket Nokia N800 with no
source code changes.
