Are you saying that you need to move backwards and forwards among people - or
among transactions for a single person?  If it is the latter, then I still
say you should be creating the table (we don't call them "arrays" in COBOL)
to hold all the transactions for a single person.  This will do exactly what
you want.

If you really do have a need to move backwards and forwards from person to
person, then COBOL has a solution for this too - it is called "indexed files"
(accessed randomly or dynamically).  All you would need to do this (in an
OS/390 environment) is to do an IDCAMS REPRO from your sequential file to an
indexed file - and you will be able to process "across" people with no
difficulty (and probably won't need a table at all).
