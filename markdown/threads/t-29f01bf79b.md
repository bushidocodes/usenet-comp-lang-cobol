```
We have a product which provides a web-browser interface for
certain mainframes. It requires the customer have a "middle-tier"
Microsoft NT server with their IIS web server. I have seen
many shops using some sort of mid-range UNIX platform (like Sun) for
their web server however and thought it might be useful for
us to come up with a port of this product which supports this
setup instead of NT. I was wondering what others from the mainframe
end here might think. Thanks in advance.
```

```
Seeing as there's a server built in to OS390, and Domino will also soon
be available, an ISPF browser, 3270 or Workstation Agent (for Graphics)
would make more sense. Most Enterprise type shops use OS/2 to work with
their mainframe - buying into an untried platform like NT would be
suicide. Give it a couple of years when IBM has NT support completed for
most products, (like VisualAge COBOL etc.) and this will change. Now would
be too early for shops that are still thinking about using the Y2K problem
as an excuse for eliminating their large DOS base and finally converting
from Cobol VS to something more recent (Like COBOL 85, yuk, yuk) to even
consider something that isn't integrated like OS/2 or AS/400.
```
