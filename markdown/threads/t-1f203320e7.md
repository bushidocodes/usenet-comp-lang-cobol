Henry Ford produced an original concept when he said that you could have any
colour you wanted so long as it was black.  Some companies today are using
the same concept for their data - it can be in any format so long as it is
Oracle!

There are admin benefits to RDBMS - adding columns/indexes etc without all
programs needing to know about the structure.  I agree with Ken when he says
that SQL  reduces the amount of code - this make programing quicker and
easier to maintain (so long as you know SQL!).   I'd clarify this though - I
believe that SQL is only good for simple queries once requirements get more
complicated I believe SQL becomes complete spaghetti (if thats spelt
correctly) and uncomprehendable.  You can therefore not maintain it whereas
the COBOL rules are easier to maintain and less likely to introduce bugs.

Similarly I don't think SQL is a patch on COBOL batch/update operations (eg
update an invoice, invoice items, customer, stock, sales, VAT etc).

There's good reasons for RDBMS, but there's also the argument for keeping
COBOL file systems and viewing them as a RDBMS through a COBOL specific
driver.

Gerald
