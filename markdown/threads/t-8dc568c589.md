Yes.

I must admit I keep my ISPF in very simple CLISTs and use a real
language to do the rest. In order to do any testing during the
progress of the code you would need to exec a command (even if it is
only debugging code). I have never done it but you might consider
throwing a SELECT panel that, does not wait for a response, calls the
Rex program automatically. Do not know how it is done just read about
it many moons ago.

If you want to test before and after you might just want to call you
cobol under a rexx and dump after the cobol code has finished.

I also rarely do any work with cobol directly accessing variables. I
tend to build ISPF tables and then pass this to file tailor. If you are
performing a repetative task then consider recording it in a table and
then calling a panel to display the table data after you have finished.

Hope this helps
Ken
