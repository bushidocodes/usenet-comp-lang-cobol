Help!
I have a client with several PC computer systems on a Novell network
using a network printer. The client is using an application called
"Winning." On the faster systems, the print action fails when print
requests are made. When the application is closed the print operation
starts. On slower systems DX2-66 or less the same print request
operations work as expected. ie the printer starts each print job
after each request while the application is active. DX4-100 and up
fail. I slowed a DX4-100 down by taking it out of turbo mode and it
then worked. The netware version is 3.12
Thank to any that have an idea.

You will have to look at your "CAPTURE" parameters.
I have had comparable problems and set
CAPTURE with NO TIMEOUTS (NO AUTOMATIC ENDCAP)
I then ENDCAP when I want to which releases the print so that multiple
jobs do not shuffle the print together.
I used Realia and executed CAPTURE and ENDCAP's internally.
works fine....
