```
Assumptions (mine)
 - you are working with IBM z/OS
 - Possibly using Enterprise COBOL (but also possibly an earlier compiler)
 - You have a COBOL "main" program that is activated (in batch) by JCL
 - currently the program does NOT expect *ANY* parameters to be passed in the 
JCL "PARM="

If this is true (and you now want to add a "parm" option in JCL)

Some answers:

1) NO, these will NOT show up as "spaces" to programs where the PARM is not 
passed
2) When you use "PARM=" in JCL to pass to a COBOL main, you can ONLY pass one 
parameter.  If you use commas in the PARM= string, it will still come in as one 
parameter to the COBOL main program (the "length" word at the beginning tells 
you the length of the entire PARM string

If you are actually talking about a COBOL *sub* program (one called by another 
COBOL or Assembler or whatever main program) then the way the COBOL subprogram 
handles "passed parameters" is VERY different from what happens with JCL PARM= 
strings.

 * * * *

Clarify what you are actually currently doing and what you want to change and we 
may be able to help more.
```
