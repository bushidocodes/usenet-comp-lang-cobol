The old bang ('!') addresses... hey, now I can bring this back to COBOL!

It wasn't all that long ago... '86, '87... and they wanted to use the same
module to read the indexed master file for an individual account or a
full-file report. One of the Corner Office Idiots proclaimed that we
couldn't change the OPEN status of the dataset - 'It might screw up
someone else's process, don't do it!' - ...

... pause for wry smiles and derisive laughter ...

... so one of the code-jockies fired up a smoke and stared at the
ceiling...

... and grabbed a fresh cuppa coffee, fired up another smoke and stared
some more...

... and said 'OK... we need one byte from the FILLER-FFU at the bottom of
the link parameters to code

01 LINKPARMS.
05 LS-QUACK PIC X(005).
05 LS-ANDWOOF PIC 9(009).
...
05 LS-FILLER-FFU PIC X(100).
05 REDEF REDEFINES LS-FILLER-FFU.
10 LS-PROCESS-TYPE PIC X(001).
10 FILLER PIC X(099).

... AND THEN... excuse me, and then (the coding of the WORKING-STORAGE
SECTION and 88s are left as an exercise for the reader)...

MOVE LINKPARMS TO WS-LINKPARMS.
IF PROCESS-INDIVIDUAL
MOVE WK-LS-ACCTNO TO MF073612-CUSTMAST-ACCTNO
PERFORM 68815-PROCESS-CUST THRU 68815-EX
GO TO 99999-EOJ
ELSE
MOVE LOW-VALUES TO MF073612-CUSTMAST-ACCTNO
PERFORM 46612-START-CUSTMAST THRU 46612-EX
PERFORM 69900-PROCESS-FULL-CUSTMAST THRU 69900-EX
CALL 'PRTCMSTR' USING WS-FULL-CUSTMAST-PARMS
GO TO 99999-EOJ.

... or something like that... my memory is, admittedly, porous.

(... and the battle-cries of other dinosaurs are heard across the land)

DD
