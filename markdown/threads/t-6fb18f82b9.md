In COBOL, there are data format for variables that have numeric values
that can be used in Mathematic Calculations. They are called COMPUTABLE.
They are COMP-1, COMP-2, COMP-3, and COMP-4.

Programmers use COMP-3 most, for it is COBOL packed-decimal-format.
I do not think the other computer languages can read COBOL COMP-3 format.
The number 123 is 3 bytes but it can be in compressed format as 2 bytes.
The number 12345 is 3 bytes in compressed format instead of 5, for example.
To determin the size of a number in compressed format, just add 1 to the
size of the number and then devide by 2. COBOL programmers use this
format to save the room to store variables.

In the past, I have done experiments with VB binary files and VC++
binary files. They are not compatible. I have never done experiments with
VB reading COBOL COMP-3 format, but I have done experiments with VB
reading COBOL ASCII format and they are compatible.

If you want to use VB with COBOL files, use COBOL to convert COMP-3 (packed
decimal format) into its display format (ASCII) and they will work
together. When you use VB to read COBOL output files, you should use
Open statement for random, and the fields all are String, including numeric
fields, and do not forget to add the last 2 bytes for the NewLine and
Carridge Return for each line.
