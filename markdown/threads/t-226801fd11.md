Hi Thane,

WB3 uses the c-tree file system, developed by Faircom Corporation.  This
is (finally!) a true record-locking file system and does not suffer from
the file-locking that the Realia file systerm v2.1 had, nor the dreaded
Index File Error.  Most implementations of c-tree provide for ODBC
access; unfortunately, Realia's implementation omits two key elements
necessary for ODBC access to the file system from outside apps.  Based
on conversations I had with Faircom (about 2 years ago), those two
elements were called IFIL and DODA.

Besides that disappointment, our experience with C-Tree has so far been
positive.

I have not tried it, but the COBOL 6.0 that is part of WB3 supposedly
allows for a ( [Y]? ) file descriptor that will allow reading of the old
file system.  I'm not sure about writing.   The REALCOPY that comes with
WB3 will read the old format and write a c-tree file, which means that
if you are porting an app from COBOL 4.x or 5.x, and not changing FDs,
you don't have to write any special conversion programs to get your data
into c-tree.

Hope this is informative.

Regards,
Jim Elden
