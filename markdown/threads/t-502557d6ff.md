GENERALLY - the column names in the database tables have underscores
(and MAY not have dashes for all I know).  Until the next COBOL
standard the UNDERSCORE is not a valid COBOL character so cannot be
used where COBOL variables are declared.

For my own use in my embedded SQL coding - I use dashes in my variable
names between my DECLARE and END-DECLARE statements  - and I make
those dashes correspond to where the database uses underscores.
