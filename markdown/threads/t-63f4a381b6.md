Lars,

COBOL has no intrinsic capability to start a subtask.

You CAN run multi-threaded COBOL tasks if you have a runtime that supports
it (Fujitsu does...).

Obviously, it is an OS function to allow and control timeslicing between
subtasks or threads.

You don't say what environment you are running in, but as it is a "hobby"
program, I am assuming it is NOT running on a mainframe...<G>? (IBM
Mainframes DO support subtasking, but you need to initiate them from CICS or
from Assembler, not from COBOL)

If you are developing under Windows for instance, you CAN start different
COBOL threads as subtasks. You would need to use the Win32API or a
proprietary development tool targeted at that environment (I use Fujitsu
PowerCOBOL).

I know very little about LINUX or UNIX style OSes, but I would think they
have the same facilities.

The lack of such a facility in COBOL has never been a big deal. COBOL is
designed to run across platforms and environments and the approach has been
that COBOL will call the facilities in the Environment if it needs
environment-specific functions (like multi-threading). Part of the problem
has also been that COBOL has never had the facility to save and restore
registers so that true re-entrancy of shared code can be achieved. (Some
very ingenious schemes for "simulating" this in COBOL have been around for
many years and are implemented in CICS and IMS/DC, but the code is said to
be "serially reusable" rather than truly re-entrant.)

If you use Object Oriented COBOL (and I strongly recommend you do...<G>)
then you can instantiate as many instances of a process (or sub-process) as
you like and they can be processed concurrently if the OS and/or Run Time
System supports it.

Hope this helps,

Pete.

options.  The saving of registers has little to do with it since
mainframe COBOL always followed standard linkage conventions for that
platform.  It's just that IBM COBOL before the ANSI 85 COBOLs treated
the Working Storage and internal work areas as a part of the load module
rather than acquiring the areas separately upon entry.
