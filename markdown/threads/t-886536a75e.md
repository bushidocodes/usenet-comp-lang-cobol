```
I'm having to perform transformations on variable length cobol records
contained in a sequential file.  I am not a cobol programmer.  I was
given the copy books for the records, which made perfect sense.  I could
determine the byte offset for the occurs variable to determine the
variable length portions of the records.

When I started parsing the input file I was surprised to find that each
record is prefixed and suffixed with the length of the record (4 byte
little-endian integer).  This made it much easier to read each record.
But, I'd like to know if this a standard way for creating a sequential
cobol file, or if it implementor defined.  The compiler that was used is
Fujitsu for Win95 version 3.  Can I expect the same say for MF Cobol and
Visual Age Cobol?  If so, can I expect the length prefix/suffix to be
big-endian on big-endian platforms such as AIX and Sun?

Thanks....
--matt
```
