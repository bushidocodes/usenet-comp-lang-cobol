Les,

If you want to view the offending data, you would issue a FIND P'.' command.
This will move your cursor to the first instance of an offending character.
If you read the message, it says that the data contains an invalid
(non-display) character. Ignore the invalid part and regard only the
'non-display' part.

To view the offending character in meaningful characters, type HEX ON at the
top of the editor. This will make the screen require twice as much space
and each line will be represented by two rows of characters. The offending
character should still be on the top line and the cursor should still be
able to find the particular instance (FIND P'.'). My advice is to ignore
this error (unless you aren't getting a good compile).

When you use packed decimals (e.g. PIC S9(09), or something similar) you are
essentially representing two numbers in the space normally used to represent
one. This requires the use of extra bits that generate non-printable
characters in 'normal' display mode.

Good luck with your work -- let me know if this is any help to you!

Cheers,
Jeremy
