>From what I can gather MF Personal COBOL for Win 3.1 should run in Win 95 ok
but I may be wrong as the source code is text anyway.

One way to print in MF cobol is to use the following environment division
statements.

ENVIRONMENT DIVISION.
INPUT-OUTPUT SECTION.
FILE CONTROL.
SELECT PRINT-FILE ASSIGN TO PRINTER.


DATA DIVISION.
FILE SECTION.
FD PRINT-FILE
01 PRINT-RECORD PIC X(150).
and using standard WRITE PRINT-RECORD FROM .................. statements
will send output to the printer

As simple as that the MF run time system sorts out the actual printing.


Nick Crewdson

I wrote the following simple test program:

********************************************************************

IDENTIFICATION DIVISION.
PROGRAM-ID. PRINTER-TEST.
AUTHOR. FLOYD H JOHNSON.

ENVIRONMENT DIVISION.
CONFIGURATION SECTION.

SOURCE-COMPUTER. IBM-PC-COMPATIBLE.
OBJECT-COMPUTER. IBM-PC-COMPATIBLE.

INPUT-OUTPUT SECTION.
FILE-CONTROL.
SELECT PRINT-FILE ASSIGN TO PRINTER.

DATA DIVISION.
FILE SECTION.
FD PRINT-FILE.
01 PRINT-RECORD PIC X(80).

WORKING-STORAGE SECTION.
01 PRINTER-RECORD PIC X(80).

PROCEDURE DIVISION.

1000-MAIN-CONTROL.
OPEN OUTPUT PRINT-FILE
MOVE SPACES TO PRINTER-RECORD
MOVE "THIS IS A TEST" TO PRINTER-RECORD
WRITE PRINT-RECORD FROM PRINTER-RECORD
CLOSE PRINT-FILE
STOP RUN.

******************************************************************

It compiles without errors with or without the OPEN/CLOSE
statements.
