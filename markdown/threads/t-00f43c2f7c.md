I use AIX COBOL a lot and am very satisfied with it. A quick check shows it is still orderable online, and I have not heard of any
withdrawals on it. Remember, most of the people you talk to at IBM Direct are less than knowledgeable about anything more  esoteric
than Windows 98. (*sigh*)

It does not support the SCREEN section, so if you are using that, you would have to port over to something else. Talk to Bob Wolfe
over a Flexus for some good ideas, or you can simply write the screens in C using Curses and CALL them that way, or you can do the
best thing, and use CICS. <grin> (Well, that idea of the 'best thing' may be a tad biased... :)

I do not know what is involved with using it for Oracle, but I am sure that Oracle does, so I would give them a call.

Speedwise, compile wise it is pretty efficient and comparable with Microfocus except when dealing with Operating System files
directly. (i.e. A Sequential File is not just a LF delimitated file under UNIX, however a LINE SEQUENTIAL file is.)

-Paul

    We are using Acucobol(5.1.0.4) on AIX(4.3.3) with Oracle(8.1.7)
and have had very good luck. Acucorp's Acu4GL database interface
drivers allow you to access the database tables without changing the
COBOL read/write/rewrite/start/delete/etc statements to SQL. This
means that once the index files have been converted to database tables
there is no need to alter or rewrite any program code to perform IO
operations. The only problem we have encountered is doing some types
of read operations on alternate keys with duplicates. The performance
can be awful if the table is large. I have been told by Acucorp
support that this problem has been addressed in version 5.2 of the
runtime.
    One issue to be aware of though is cost. Oracle is rather
expensive and so are Acucobol's runtime and Acu4GL licenses.
