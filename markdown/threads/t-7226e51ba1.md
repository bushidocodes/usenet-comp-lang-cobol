The scenario is that, given an action flag, I want to scan a database and
get all the "Members" or "Friends" of a particular organisation, and string
all their email addresses into a text file that the user can cut and paste
into the Blind Copy (BCC) part of an outgoing email. In this way it is
possible to "mass mail" all the "Friends" or the "Members" of the society
with a single message. (Actually, it isn't, because most mail servers have a
limit on the number of recipients, so the function below actually breaks the
file into records containing 25 emails; this does not overload the mail
server, but it means the user has to cut and paste into as many mails as
there are rows...

Note that because the length of email addresses can vary, the 25  varying
addresses in each record will cause the record length to vary also.

Of course it would be fairly easy to actually pop up the outgoing mail form
with the BCC filled in, using CDOSYS but, after discussion with the users
they decided they'd rather just have the list somewhere handy so they can do
this whenever they have time. The answer seemed to be to put it on the
desktop and that's when everything turned to custard for me...

Anyway, it is resolved now, works like a dream, and everybody is happy.
Here's the COBOL...

 identification division.
 program-id.     'BCCLIST'.
 installation.
 author.        Peter E. C. Dashwood.
 date-written.   Oct 2004.
*
*remarks.
*
*      Function to Create Blind Copy lists of email addresses for
*             bulk mailing Friends and Members in MEDEA.
*
*
*
*    INPUT Parameters:
*
*
*    1. An Action Flag.
*
*       Indicates whether to Create for Members only, or for all "Friends"
*       of the society.
*
*
*    OUTPUT from the module:
*
*    1. A line sequential file with records containing 25 recipients
*       email addresses separated by semi-colons, suitable to load into a
mailto:
*
*    Remember to comment out Linkage section and swap
*    Procedure Division headers when debugging as a standalone program...
*
*    *******************************************************
 environment division.
 configuration section.
 source-computer. IBM-PC.
 object-computer. IBM-PC.
 special-names.
 input-output section.
 file-control.
     select BCCList-FILE assign OUTFILE-NAME
