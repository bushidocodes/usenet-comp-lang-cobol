```
Cesar

I'm assuming you're asking this question as a COBOL programmer who wants
to know a little more about COBOL programming under CICS.

In that light, you might also appreciate the CICS Application
Programming Primer at -
 http://www.s390.ibm.com:80/bookmgr-cgi/bookmgr.cmd/BOOKS/DFHZP104/CCONTENTS?SHELF=DFHHSH21

As a CICS Systems Programmer (albeit with COBOL experience) I found it a
very useful way of learning what my applications colleagues were up to.
The important thing about this book, compared with the Application
Programmers Guide (APG) or Application Programmers Reference (APR) is
that it explains how to write a CICS application from the ground up.
Assuming you complete the exercises in the book, by the time you reach
the end, you've written a passable CICS application and its modular
approach means you've got a library of reusable procedures. From my
point of view this is important because it means you're not going to try
and run batch type code under CICS.
```
