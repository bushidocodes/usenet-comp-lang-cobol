Number crunching.

Things like weather prediction or finite element analysis which require
streaming enormous amounts of data through the cpu. Fortran has always
been in front with optimisation and vectorisation and so produces the
fastest code.

Much of the difference is how you model reality. Fortran models are
usually rectangular arrays with a few properties at each node. The
emphasis is how to stream the arrays through the cpu. APL is an
extreme version of the array model.

C and Pascal tend to use networks with a bundle of properties at each
node. The emphasis is how to build and maintain the network most
efficiently. Many languages are similar to C and Pascal.

Cobol models tend be indefinitely long lists with a bundle of
properties at each node. The emphasis is on how to stream these files
through the machine, and on how to quickly do associative-like searches
in the files.

Also the different languages interact with the programmers different.
Some people find Fortran too ugly to use. Or Cobol too wordy. Or C too
dangerous. Or Pascal too straiting. This more a question of personal
preference.
