I am assuming in the statement of the problem that
WS-ALPHA is a floating decimal field with a possible leading
sign. If it were a fixed numeric-edited field with a
picture of ZZ.99999999- then the code would be Move WS-ALPHA
to WS-COMP-w. The code below is complex because COBOL is a
fixed field, fixed point language which does not handle
floating data well. This is in part due to history.
FORTRAN and subsequent languages in the
scientific/engineering community such as C which have
delimited strings and floating point handle the
above problem easily, at least for the programmer. COBOL
and other fixed position languages which come out of the
business data processing environment don't have easy native
language to handle floating decimal point position fields.
Business data was originally on punched cards in fixed
positions. This was necessary for processing on unit record
equipment. When computers came in, fixed position data was
easier than floating position data to handle in Assembler
and the languages that followed.
I have assumed that there could be an optional leading
plus or minus sign. Since the ultimate receiving field is a
floating point number, I did not have to write code to
handle the problem of either the integer or fraction
(decimal) portion of the number exceeding the size of the
field. While the source code I suggest is multi-line
compared to the probable one or two lines of code in C or
FORTRAN, if C or FORTRAN were to generate the code in line
there might not be as much of a difference in the resulting
machine code with C or FORTRAN still being more efficient
than COBOL.

The Working-storage items and code that I would use are
as follows. I am assuming the 85 standard.

01 Work-area.
05 Ws-pointer Pic S9999 comp sync.
05 Ws-integer-count Pic S9999 comp sync.
05 Ws-rcv-pointer Pic S9999 comp sync.
05 Ws-sign Pic X.
05 Ws-delimiter Pic X.
05 ws-integer-temp-x Pic X(12).
05 Ws-integer-x Pic X(12).
05 Ws-integer redefines ws-integer-x Pic 9(12).
05 Ws-fraction-x Pic X(11).
05 Ws-fraction redefines ws-fraction-x Pic V9(11).
Procedure Division.
.
.
.
Move 1 to ws-pointer.
Move "+" to ws-sign.
Move space to ws-delimiter ws-fraction-x ws-integer-temp-x
Move zero to ws-integer-x.
If ws-alpha (1:1) = "-" or "+"
move ws-alpha (1 : 1) to ws-sign
move 2 to ws-pointer.
end-if.
Unstring ws-alpha into ws-integer-temp-x delimited by "." or
space
delimiter in ws-delimiter
count in ws-integer-count
pointer ws-pointer
If ws-delimiter = "."
unstring ws-alpha into ws-fraction-x
delimited by space.
end-if.
Inspect ws-integer-temp-x replacing all space by zero.
Inspect ws-fraction-x replacing all space by zero.
If ws-integer-temp-x not numeric or ws-fraction-x not
numeric
perform 2000-error-routine
end-if.
If ws-integer-count not zero
Compute ws-receive-pointer = 13 - ws-integer-count
move ws-integer-temp-x (1 : ws-integer-count)
to ws-integer-x (ws-rcv-ptr : ws-integer-count)
end-if.
Compute ws-comp-2 = ws-integer + ws-fraction.
If ws-sign = "-"
compute ws-comp-2 = ws-comp-2 * -1
end-if.
