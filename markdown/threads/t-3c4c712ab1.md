Iam trying to pass the VB 5.0 definition below.  If I define it this way most
of the parms come back good except the var5 has garbage. But what is
interesting is if I take out the definition for var5 and just say that var4
occurs 200 times it all comes back fine.  I think it is an alignment problem
but I am not sure. Any thoughts?


This is NetExpess 2.0 with all patches running on NT 4.0 Service Pack 3.

COMPILER DIRECTIVES

$set Ans85 DefaultByte"00" Mf

CALL CONVENTION DEFINITION

call-convention 66 is NtStdCall.

--------------------------------
Visual Basic 5.0 Type Definition
--------------------------------
Global DLLReturnCode As Integer

Type cblParms
    var1(1 To 100) As Long
    var2(1 To 100) As Long
    var3(1 To 100) As Long
    var4(1 To 100) As String * 8
    var5(1 To 100) As String * 8
End Type

Declare Sub TESTCA02 Lib "VBTEST01.DLL" (DLLReturnCode As Integer,
integerString As cblParms)

--------------------------------
NetExpress Definition
--------------------------------
01 aReturnCode          Integer.

01 cblVars.
   10 cblVar1         Long occurs 100 times.
   10 cblVar2         Long occurs 100 times.
   10 cblVar3         Long occurs 100 times.
   10 cblVar4         pic x(8) occurs 100 times.
   10 cblVar5         pic x(8) occurs 100 times.

Entry 'TESTCA02' NtStdCall using aReturnCode, cblVars.

I cannot see any alignment problem. If VB5.0 inserts the
equivalent of slack bytes within a Type then the size of a
variable of that type would be greater than the sum of the
individual items. In this case, the size should be 2800.

I pasted the Type into a module and added the following
text to a Form:

Private Sub Form_Click()
    Dim x As cblParms
    If (MsgBox("Size of cblParms is " & Len(x), _
        vbOKOnly)) Then
    End If
End Sub

Clicking on the Form gives 2800, as expected.
