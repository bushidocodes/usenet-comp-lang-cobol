>You cannot get COBOL source code from executables. That would be like
>getting 'cow' from hamburger.

If it was compiled with DEBUG information in the executable, you can recover
data and paragraph names. 

If file records are described in copybooks or other programs, you can use those
data names in the reconstructed source. 

If neither of the above, reasonable names can be assigned manually by someone
familiar with the data and application. This is a tedious reiterative process --
after some names are resolved others fall into place. You need an editor that
can display instant cross-references. That's how reverse-engineering is done. 

Pseudo-code, as produced by Acucobol, is no more nor less difficult than machine
language assuming you have a definition of the pseudo language.
