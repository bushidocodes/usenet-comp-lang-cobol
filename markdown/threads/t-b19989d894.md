```
Does anyone know of a way to easily change the date and time that
is returned in the EIBDATE and EIBTIME for CICS programs?
Basically I'd like to do a CICS transaction on the fly that will
allow me to forward the date by a certain number of days and a
certain number of minutes.  This is, of course, for Year 2000
testing.  We do have a separate Y2K LPAR that can be IPL'd to
a particular date, but as far as I can tell it's a time consuming
pain to do that, plus only the systems programmers know how to do
it.
```

```
Frank, if you go to my web page at:
    http://home.att.net/~arnold.trembley/y2kdl.htm
There's a link to an IBM web page where you can download IBM SupportPak
CH15, the Time Machine for CICS.  It is licensed software, but there is
NO CHARGE for it.  You will need PKUNZIP or INFOZIP to decompress it,
but the entire package will fit on 3.5 inch floppy decompressed. 
There's also a manual in Adobe Acrobat format, and one in another
format.  You will need to TSO/ISPF to deblock the runtime library after
transfering it from the PC to the Mainframe.  You may need a systems
programmer to install it for you.

I heartily recommend it.  We used it extensively for unit-testing our
CICS applications starting in 1997.  CH15 works on CICS 3.3, 4.1, or
Transaction Server for OS/390.  It gives you a transaction (CTMC) that
allows you to set the date and time up to 31/12/2099 and a PF key to
restore the system date and time.

Good luck!
```
