Hi Masood,

I have not used this language from a very long time. Here is the answer.

First in your select clause change your Random to DYNAMIC it will allow 
you to access your indexed file randomly as well as you can also read it 
as sequential. This is good practice.

Now if you want to write it to this file your key there is Five-Digit. 

Move Input-Five Digit to Five-Digit.
Read File-1 Invalid Key
      Perform ADD-Record-To-File 
        Not Invalid Key 
            Perform Update-Existing-Record. (Invalid key means that record 
                                             does not exists in file)

       In your ADD-Record-To-File.
       Move your input fields to file fields and write the record.

       Write Record-1 Invalid Key
             (here you must code error handling paragraph).


       Update-Existing-Record.
 
       Update your file fields. 

       Rewrite Record-1 Invalid Key
              (here you must perform the error routine).

You get the following advatage of the DYNAMIC access. For example you want 
to read the from starting from 10000 record until EOF. (End of file).

Move 10000 to Five-Digit.
Start File-1 Key is Not Less Than Five-Digit.
Read File-1 Next Record 
      At End 
          Perform End-Of-Program.

The most basic use is to inspect data elements within the file.
Indexed files have two types of records: 1) Data (the stuff in which
you are interested) and 2) Control/positioning information. The latter
makes sense only to the file access system but is interspersed with
the data. A trivial metaphor is the printed page in a book; it
consists of words and page numbers. Usually you are not interested in
page numbers.

Presuming you are interested in the data and not the file control
information, the file utility program filters out the irrelevant
control and positioning material - which would make no sense to anyone
anyway - and presents only the data. The file utility also presents
the data in an orderly fashion (by key, for example). In the file
itself, the data are scattered hither-and-yon in locations known only
to the file control system.
