I suppose 'obsolete' is a poor choice of words. Any talk of coding
techniques is likely to start another holy war but at the risk of
inciting a riot here goes...

1) The program (and I use the term loosely) does not take advantage of
any ANSI-85 'END-' conditions. If it did, most of the missing period
problems would go away, and the logic problem with doing a single read
and multiple writes would stick out. 

2) A few level 88 variables would go a long way to avoiding that mess
with the EOF/END-OF-FILE "Y"/"YES" problem.

3) Testing an AT END condition instead of a file status code means that
the program will not handle any of the other errors that could occur.
What happens if the input file is missing? Of course, this **ahem**
program doesn't have any exception processing so the point is moot.

4) Putting sequence numbers in the labels made a lot of sense when the
only way to analyze the code was by reading the listing. Editors do a
pretty good job of providing a 'FIND' function. There are tools to do
structure analysis and give you pretty pictures about how the paragraphs
which do a better job than paragraph naming conventions.
