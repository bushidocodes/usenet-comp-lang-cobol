There are two issues in your comment: "similar results" and "identical
compiler" results.

Any conforming compiler (and Fujitsu, IBM, MERANT, RM, Acucobol, all are)
provides a method of "flagging" when you use extensions to the Standard - at
compile time.  If you turn this on, you SHOULD be guaranteed that your code
will compile identically with different compilers.  (Well, there are some
"implementor defined" areas - such as Select/Assign statement - but these are
usually minimal.)

As far as "identical" run-time results go, this is less easy to check for and
guarantee.  HOWEVER, if you are going to become a "real world" programmer, it
is probably useful to learn about such differences early on.  Learning to
read a vendor's documentation - and how to adjust your program's behavior to
the  desired results is a useful skill and one that I would encourage you to
learn - while you are still learning how to program in COBOL.
