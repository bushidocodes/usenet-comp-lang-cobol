On the MVS side, you could use the following TCP/IP entry points.  They are
documented in the "IBM TCP/IP for MVS - Application Programming Interface
Reference, V3R2" manual.  I don't think they are documented in any of the OS/390
manuals, though they are still supported as our products use them (currently
running OS/390 V2R7).

CALL  EZACIC04,(CIFIELDS, CILENGTH),VL  *Convert MVS EBCDIC to PC ASCII
CALL  EZACIC05,(CIENTRY, CILENCVT),VL   *Convert PC ASCII to MVS EBCDIC

Hope it helps ...

Are you using the EZA calls for your socket interface?  If so then
this call does the translation:

     if client-is-ascii                            
        Call 'EZACIC05' using read-buffer-total     
            read-request-read                       
      end-if             

read-request-read is a counter of length.  This converts EBCDIC to 
ASCII.  

I used the EZA calls for all the socket stuff.
