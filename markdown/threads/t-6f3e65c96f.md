being a simple person I sort my files before I process them
then in the set up

perform do-set-up-stuff thru set-up-stuff-end
perform read-file-a     thru read-file-a-end
perform read-file-b     thru read-file-b-end
perform compare-files   thru compare-files-end
                       until eof-a and eof-b
perform end-job-stuff   thru end-job-stuff-end
goback.


compare-files.
*** note this assumes file-a has only one record to match to file-b ***

if match-key-a = match-key-b
    perform keys-equal   thru keys-equal-end
    perform read-file-a  thru read-file-a-end
    perform read-file-b  thru read-file-b-end
else
     if match-key-a > match-key-b
	perform unmatched-b  thru unmatatched-b-end
        perform read-file-b  thru read-file-b-end
     else
	perform unmatched-a  thru unmatatched-a-end
        perform read-file-a  thru read-file-a-end	
     end-if
end-if.

truth be told, I usually code the read in the equal, high or low logic
paragraphs (which would have the appropriate processing, matching,
formating or writing steps, any required counters and so forth).

for three file matching (matching 3 files) is more complex and if you
have multiple correct matches then I would add add a flag test and if
the low-record was a match in the previous test, let it go, but if it
was never matched, then error code and report.

good luck -- Susan
