If I had access to the system I'd just write some code to do this and look

I basically understand COMP fields. A value is translated to binary and
stored using 4 or 8 bytes when saved to media. But how *exactly* does it

Finally, what if there is an implied decimal point? Is the number stored as
if there were no decimal point and then it's up to COBOL to place the
decimal? Meaning if the field were COMP 9(4)V99 with a value of 1.60 would
it be on disk as "00 00 00 A0"?

Thanks so much for any guidance.
