For those who may be interested:

My research uncovered the following:

- 2 gig limit related to 32 bit address limitation
- Most file systems under Unix will support only 2 gig isam files.
Databases can grow beyond this by using multiple "extents" (So I am told).
- I am left with three options which are currently being explored:
1) Split the isam file
2) Convert the isame file to a database (Can grow beyond 2 gig)
3) Possibly use some nfs technology with a file system that
can handle in excess of 2 gig data segements.

I am still in the process of detailing each of the above possibilities.
If I have omitted an option, feel free to comment.

Thanks for all of your help!!!

Matt
