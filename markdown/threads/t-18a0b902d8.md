I want to make a READ PREVIOUS with ESQL in MF Cobol 4.0 and Oracle 7.3.

For READ NEXT I DECLARE a CURSOR SORT ASC and then a FETCH. That works
fine. For the READ PREVIOUS I DECLARE a CURSOR SORT DESC and then a
FETCH. That works but not fine. If there are many rows in the table, it
is very slowly. When animating the FETCH is the slow statement.
Any ideas how to do a READ PREVIOUS by another way?
In Informix there is a FETCH NEXT and FETCH previous (I read so), but I
didn't find something like that in Oracle.

Hi Frank,
my suggestion would be to define a index on the column you are using in
the select part of the declare of the cursor. You can define the index as
DESC to get it in the order you require.

Also, you may want to do an array fetch rather than a single row fetch.
That would reduce the number of times you need to access the database &
should speed things up.

Hope this helps.
