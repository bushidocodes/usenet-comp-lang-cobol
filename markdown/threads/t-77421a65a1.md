```
MS Cobol 3.0 appears to be a version of MF Cobol/2 version 2.1.  In
order to use the product beyond simple play and tests you should have
the proper documentation.  Language reference and User Guide for that
particular compiler would be minimum requirement.

MF Cobol 2.1 (and thus MS 3) did not have the CBL_ routines that would
read and write characters and attributes from the screen that would make
it relatively easy.

You could use SCREEN SECTION to define your screens with all atrributes
and fields and then redisplay the whole screen after the pop-up box.  In
general I would reccomend reserving a 'message area' in each screen,
such as line 24, where all errors and non-standard requests are made.
```
