Here's another practical example, just from last week. I had a
five-minute panic attack. My user 'phoned me. 'Could I please change the
old RM/COBOL system, written 20 years ago, to read data from the CD ?'
He had an engineer who wanted to view the data on his laptop. Now who in
his right mind would want to run a system from a 5.25 on drives a or b
back in those days? (Oh, yes, and while I was at it - make it read from
D or E). How many days to Y2K ?.

Some thirty programs to change, back to uppercase, areas A and B, no
End-ifs etc... Help !!! Pleasantly surprised myself - the system was
designed with a fixed set of parameters passed from the Main Menu to all
programs - regardless of whether or not all parameters were required -
not necessarily terribly efficient, but it worked. Fortunately I had a
copy file :-

    .................. copy ..........
    Linkage Section.
    01 param-1 etc...
    Procedure Division using param-1 etc....

    ......................................

Some ten minutes of thought, five lines of code in the Main Menu,
re-compile and done in half an hour. Separate start-up batch-files for
drives a, c, d, e with run-time switches set to denote drive and the
system runs from all four. I used drive a: to test it on my machine.
Pheww! - thank the Lord.

Jimmy, Calgary AB
