### Should compilers be in public domain?

- **From:** "kim..." <ua-author-3680929@usenetarchives.gap>
- **Date:** 1995-01-24T08:09:01+00:00
- **Newsgroups:** comp.lang.cobol
- **Message-ID:** `<3g2u5g$as6@ns>`

```
It seems that all basic information processing tools such as compilers should be in
public domain or should come with an operating system (MS Wnodows, OS/400,,,). Along
the same line of arguments, compilers should come with reusable (subroutines,
intrinsic functions, or whatever you call them) codes. One good feature of Fortran
is that it normally comes with an extensive subroutine library. Why COBOL users have
to reinvent the wheel?

COBOL compilers should be in public domain (remember COBOL was originally paid for
developed by the Navy - with tax payers money) and supported by the users on the
internet. After all, many internet tools are free.
```
