Such a thing is usually used for debugging and/or error routines.

For debugging, one common purpose of this is so that the most recently
entered paragraph will have its name in the referenced data item's
location in a memory dump.

If this is used for error routines, it might be for something of this
nature:
DISPLAY "XYZ Error in paragraph " PARAGRAPH-NAME.

To gain more insight on your particular program, you may want to
search the source (or its data name cross reference) to find how the
name PARAGRAPH-NAME is used in the program, other than as the
receiving field of the MOVEs you mentioned. (You may also have to
refer to the DATA DIVISION for the item to which PARAGRAPH-NAME is
subordinate. Also look for REDEFINES involving that data name.) If
you can find no use of PARAGRAPH-NAME other than the MOVEs and its
definition, then its use is probably for memory dump analysis.
Otherwise, other uses may become evident from the program.
