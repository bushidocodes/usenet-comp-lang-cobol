A book called "COBOL from Micro to Mainframe" comes with CA-Realia
Classroom COBOL on diskette. It is a DOS command-line compiler with
an interactive source level debugger. It isn't an integrated
development environment. You compile, then you link, then you
execute either with or without the debugger. The debugger is a very
good one with breakpoints, watch variables, highlighting of the current
statement, all the standard visual debugger stuff.

I paid about 50 USD here in the states. ISBN 0-13-310764-7.

Limitations of the implementation: source programs limited to 500
statements; execution time files cannot exceed 64Kb; all programs,
regardless of size, must compile in the available memory. However,
it appears multiple programs can be linked together.

The full ANSI COBOL-85 appears to be implemented, but I haven't actually
checked out Report Writer or Communications Module.

There is no line sequential file type. Line sequential is not standard.
Also, the ASSIGN TO clause is compatible with the IBM mainframe
compilers, which means you can't use DOS filenames in your program. You
have to set an environment variable to be equal to your filename. (This
is how they imitate DD statements with DOS.) So you have to set your
environment size pretty large.

All told, I think it's a pretty good deal.
