Hi Christophe.

As Gael has already mentioned - unless the command_line_linkage tuneable
is set to FALSE (by default it is set to TRUE), your main COBOL program
will receive exactly one parameter via the linkage section. IIRC, the
format is:

01 command-line.
  03 cmd-line-length pic 9(8) comp.
  03 cmd-line-text pic x(1000).

You then access it something like:

procedure division using command-line.
display cmd-line-text(1:cmd-line-length)

Therefore, if you just run an arbitrary sub-program via cobrun, the
first parameter (in your case, lkqui) does not have a NULL address and
will contain "garbage" values.

Look up the "cobconfig" configuration file in your manual for a
description of the tuneable.

Cheers, Kev.
