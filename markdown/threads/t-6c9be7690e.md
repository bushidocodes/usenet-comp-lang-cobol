Hi All,

I have an application that is in production written in MF Cobol
4.0.32.  I have a setup screen that has a field on it that has the
directory that the user specified during installation that contains
the files used by the program (stored in a INI file).  To change the
directory at this time, the user has to type in the complete path to
the files. I'd like to add a 'browse' button that will allow them to
select the directory like they did during the installation (I used
InstallShield 5.5 Professional).  The app was written in MPACT CSU 2.3
(Flexus is the GUI painter).  I've looked for API's to do this but the
closest I can come to the FileOpen common dialog box.  I've got a test
program that will invoke the dialog box but it won't let me just
select a directory - it, as the name implies, wants you to select a
file :-)   Any ideas?

Thanks in advance,
Doug

It ain't pretty, but it's easy.

You should note that the File Open (you should be able to use one that
MPACT provides) - (Flexus has one, but I'm not sure of the integration
in MPACT) -- continuing .. File Open CHANGES the current directory.
It also USES the current directory.  So... Use the CBL_ call to CHANGE
the current directory then issue the call to the File Open.  

I do this, and I also make sure to store and return to the drive and
diretory I started in when I am done.

Doug,
There is a Win32 API function called SHBrowseForFolder that allows the 
user to select a folder and returns it to the calling program.  See the 
Window help for a complete description of the function and the 
parameters that it requires.  If I remember correctly, you specify a 
root folder to start from (default is the desktop folder) and you are 
returned a pointer to an IdList from which you get the path from (using 
SHGetPathFromIDList).  

Hope this helps.
