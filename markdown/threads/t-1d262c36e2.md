```
No -
  There is no currently supported IBM mainframe COBOL compiler that has the
"table sort" feature defined in the draft of the 200x ANSI/ISO COBOL
Standard.

If this is an "important" feature for you, I suggest that you either use
SHARE to submit a requirement for IBM to implement this "sooner than
later" - or use the normal support channels to submit a REQUEST to do this.

Otherwise, you can wait until (*IF*) IBM implements the next COBOL Standard.

P.S.  If you use both an Input and an Output Procedure, you COULD do this
with a current COBOL compiler.  You would need to "loop" thru the input
table and move each table "element" (row) to an SD, then release it.  Then
in your Output procedure you would "build" your sorted table with each
returned item.
```

No, there isn't a table sort in any MVS COBOL, however, your last
statement is a misunderstanding. There's nothing in an output procedure
that requires a file. I have many programs that sort and then my output
procedure does something other than write the sorted results to a file.

Given that, there's no reason you could not write an input procedure
that selected an item from your WS table and released it to the sort,
and then an output procedure that simply returned each sorted row and
inserted it back into your WS table.

Why not write a little code that inserts the items into the table in the
proper sequence?  All it takes is a loop that moves the last item to last item
+ 1, then the next to last item to last item, etc., until the new item is
higher than the table item being tested.  If the input table is in sequence,
no moves ever take place, except adding the new item to the end of the table.

This can even be used when the table is "dynamic"; that is, not loaded
entirely at one time.

If items may be deleted from the table, another simple algorithm can be used
to delete item n, then move n+1 to n, item n+2 to n+1, etc., until the table
is completed.

In my experience, this works best with OCCURS ... DEPENDING.

If I were at work, I'd post the code, but you'll enjoy the challenge. <G>
