The great power of the EVALUATE verb lies in the ability to translate
logic tables directly to code, e.g. given three conditions a, b, and c, and
a table describing the actions to be taken depending on different
combinations of the conditions

a b c Action

T T F xyz
T F T or F pdq
F T T abort

This is coded as
EVALUATE a ALSO b ALSO c
WHEN TRUE ALSO TRUE ALSO FALSE PERFORM xyz
WHEN TRUE ALSO FALSE ALSO ANY PERFORM pdq
WHEN FALSE ALSO TRUE ALSO TRUE PERFORM abort
END-EVALUATE

EVALUATE TRUE works a lot like the BASIC ELSEIF. You can put a lot of
unrelated tests under it and it take the first true case.

EVALUATE TRUE
WHEN A = 1 PERFORM ITS-A-ONE
WHEN B NOT = 5 OR DD ALPHABETIC
MOVE "N" TO SW-5
ADD 2 TO COUNTR
WHEN NO-INFO GO TO SOMEPLACE-ELSE
WHEN OTHER DISPLAY "WHAT TO DO, WHAT TO DO"
END-EVALUATE

This is the same as:
IF A = 1
PERFORM ITS-A-ONE
ELSE
IF B NOT = 5 OR DD ALPHABETIC
MOVE "N" TO SW-5
ADD 2 TO COUNTR
ELSE
IF NO-INFO
GO TO SOMEPLACE-ELSE
ELSE
DISPLAY "WHAT TO DO, WHAT TO DO"
END-IF
END-IF
END-IF

Hope this gets you started.
