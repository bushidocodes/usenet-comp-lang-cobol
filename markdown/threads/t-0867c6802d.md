Not tested may be valid. Consider that I have an install program that I
have used for years. The install program checks the existing installation.
Since the install program uses a 2 digit year, it will not replace files
that had been installed in 1999 or earlier. The install program is not
Y2K compliant. When I remediate the install program it applies from
that version forward. Since I shipped a non-compliant install program
earlier, there is no need for me to test whether it will fail. I have
already confirmed that it will.

Because the install program is not Y2K compliant, I then state that the
entire product is not Y2K compliant because one portion of it fails.
I can take this position even if I do not actually test other features
of the prior versions.

In fact, I may need to take this position to protect my business.
Furthermore, supplying any information as to which parts of the
product might fail is unadvisable because you may not have caught
everything.
