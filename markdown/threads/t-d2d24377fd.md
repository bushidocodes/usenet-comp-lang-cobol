```
According to the LE manual, in order for routine CEEGMT to return the
current GMT Date-Time, the System-Time MUST be set to GMT (A/K/A Standard
Epoch). However, not all shops follow this rule.

Because of this inconsistency, I chose to use CEEGMTO, which will pass back
the GMT Hours/Minutes Offset to Local Time. If you generate the ASSEMBLER
expansion for a FUNCTION CURRENT-DATE, you'll see a BALR to CEEGMTO, which
returns this offset in bytes 18-21, with byte-18 equal to a '-' or a '+',
followed by 'HHMM'.

I can successfully call CEEGMTO, get back a binary-zero (normal)
feedback-list, but the hours and minutes returned are binary-zeros. BTW, I
still get the same results after issuing a CURRENT-DATE, which I would
expect.

Does anyone know if the rule for CEEGMT regarding system time in GMT also
applies to CEEGMTO?

I know that when attempting to use the TIME Macro requesting GMT, a special
PDS member SYS1.PARMLIB(CLOCKxx), must be set to the proper GMT Offset to
Local. I wonder if CEEGMTO uses this same animal?

Any help would be appreciated.

TIA....

Cheers,

WOB,
Atlanta
```

```
Ok,  If the intrinsic function CURRENT-DATE is showing the correct offset
from GMT and the CEEGMTO callable service is NOT, then my suggestion is to
use the intrinsic function to get the information.

I can think of one environment (that may be yours) that would explain the
need to use the callable service.  IBM documents that you can dynamically
call the LE date/time services from VS COBOL II compiled programs - when
running under LE.  Is this your environment?  If so, (and I assume you do
have the COBOL for this-and-that compilers - or otherwise you wouldn't know
that the intrinsic function is giving the correct answer), I would suggest
writing your own MINIMAL subprogram that just gets the offset via the
intrinsic function, compile this with the new compiler, and call it from
your VS COBOL II program.

P.S.  I have sent a note to my "usually reliable" LE source to try and find
out why/when CEEGMTO gets a different answer than FUNCTION CURRENT-DATE -
and I will pass it on, once I get it.
```

```
offset field in his CURRENT-DATE function results.  It was

not seconds, by the way.  We have had complains about this,
for some reason!"
```
