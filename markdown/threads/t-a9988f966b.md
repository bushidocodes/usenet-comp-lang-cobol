```

Does anybody know of a neat way of finding file extensions when using 
Explorer. Column 1, Explorer lists the filename but not the file 
extension, followed by Size and a file type description. Background :-

I've known for quite a number of years that M/F Net Express has the 
ability to generate templates of OO classes and methods. Having created 
a class template, from the IDE you can click on OO Method and it adds a 
blank template method to your class. Like anybody else I have my own 
preference for layout, and getting at those templates I'm reasonably 
confident I can achieve what I want by changing the text of the M/F 
Templates - BUT FIRSTLY saving the M/F set of templates to a backup 
folder, in case anything goes wrong. In addition to what I've indicated 
here there is one template which generates some 7 different versions of 
'True' OLE classes (Client or Server, generate Registry entries etc....)

I tried to find them knowing they are written using MS Word - couldn't; 
obviously they must be in an M/F directory which in my case starts at :-

\Program Files\Merant\Net Express\Base.

obvious search key seemed to be - *.doc, starting at Merant if you like. 
Only one comes back - a reference to an OLE Demo in their samples 
programs. I eventually tracked them down :-

\Merant\Net Express\Base\Bin - 14 of them

I tried using the Help within MS Word to see if it spelled out 
extensions, some 12 suggestions but not one relating exactly to what the 
extension is for MS Word Wizards - this latter type only became apparent 
when I found the 14 above.

I haven't found anything in Windows that directly lets me get at 
extensions. Fortunately I have one of those old programs from PC 
Magazine where you laboriously keyed in the hex values and a checksum 
tells you if you had made an error. For DOS it's called DR.com 
(Directory Listing) - put it into the above directory and it displayed 
those MS Word Wizards as 'abcdefg.wiz'. Any way in Windows to get that ?

Now I know where they are I can dabble to making changes to the M/F source.

Jimmy, Calgary AB
```

At the command prompt, type "ATTRIB /?" to get help.

Displays or changes file attributes.

ATTRIB [+R | -R] [+A | -A ] [+S | -S] [+H | -H] [drive:][path][filename]
        [/S [/D]]

   +   Sets an attribute.
   -   Clears an attribute.
   R   Read-only file attribute.
   A   Archive file attribute.
   S   System file attribute.
   H   Hidden file attribute.
   [drive:][path][filename]
       Specifies a file or files for attrib to process.
   /S  Processes matching files in the current folder
       and all subfolders.
   /D  Processes folders as well.

I would probably open a CMD.EXE window, navigate to the "my pictures" 
folder and type something like this:

ATTRIB -R *.*

That would select all files ("*.*") in the current directory, and turn 
the READ-ONLY attribute OFF.

But if you can see the file in Windows Explorer, right-click on it and 
then click on "properties", then if necessary select the "General" tab. 
You should be able to clear the read-only and hidden attributes by 
unchecking them.  You can select the advanced tab to uncheck the Archive 
attribute.

You might try setting the READ-ONLY attribute ON, and then OFF, by 
checking it Read-only box, clicking on the Apply button, then unchecking 
the Read-only box and clicking on the Apply button again.
