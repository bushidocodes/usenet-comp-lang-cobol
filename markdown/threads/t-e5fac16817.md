```
Actually, it is starting two new processes.  It starts COMMAND.COM, and
COMMAND.COM starts the EXE.  Each process inherits the environment of the
one that started it.  When you change the environment in your EXE, you are
changing the copy.  When it stops, the new environment area (the one you
changed), is returned to free space.
```

```
Progger wrote:
> 
> ....so if I execute an exe in a dos bat file it is starting a new process?

Yes.  Whenever a new process starts in DOS a _copy_ of the environment
is made for that process.  When the process completes that environment
is discarded.  If you type SET at the DOS command prompt it will list
the environment.  If you then run a batch program with SET in it the
list will be the same values but is a _copy_ of the environment.  Your
first program gats a new copy, can change it but it is then discarded. 
If you started a child process then that would receive a copy of what
you had changed.

It is likely that environment items are not what you want to do.  For a
start a copy is made only to the size (rounded to 16 bytes) of the
existing items so adding new items, or making existing items larger,
will probably fail.

I usually add into AUTOEXEC.BAT SET ZZZZ=ZZZZZZZ...ZZZZZZZ to fill up
space then put SET ZZZZ= in the batch file to free some up.

What you may want to do is to have a Cobol controlling program that
CALLs your programs passing an interface area:


    prog0:

             WORKING-STORAGE SECTION.
                 COPY "interfce.ws".
             PROCEDURE DIVISION.
             Control-Program.
                 MOVE SPACES    TO Interface
                 CALL "Prog1"  USING Interface
                 CALL "Prog2"  USING Interface
                 STOP RUN.

   prog1:
             LINKAGE SECTION.
                COPY "interfce.ws".
             PROCEDURE DIVISION USING Interface.
    
                 blah blah
                 MOVE "I'm happy."   TO Interface-Message
                 EXIT PROGRAM.

   prog2:    as above.

Then you can get a bit sophisticated and have the control program call
the name that is in an area in the Interface:

   prog0:         MOVE "prog1"    TO Program-Name
           
                  PERFORM
                      UNTIL Program-Name = "FINISH"

                      CALL Program-Name
                          USING Interface
                  END-PERFORM

   prog1:         MOVE "prog2"    TO Program-Name
                  
   prog2:         MOVE "FINISH"   TO Program-Name
```
