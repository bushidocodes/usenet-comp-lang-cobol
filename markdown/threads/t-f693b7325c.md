Hi,

Recently we migrate from MF-COBOL V3.2 on AIX 3.2 to MF-COBOL V4.0 on
AIX 4.1
and I have noticed in few cases with a very strange behaviour of
MF-COBOL
which I cannot explain. Maybe someone can suggest any explanation.

1) The following program when invoked return with an error:

Execution error : file 'myprog'
error code: 105, pc=0, call=1, seg=0
105 Memory allocation error

I noticed that when the group AA is not holding amount of 48 bytes in
memory
(for example AA-1 is PIC X(47), or if I'll add an item AA-2 PIC X),
then the program executes as expected.
and this in only when the group is EXTERNAL.

IDENTIFICATION DIVISION.
PROGRAM-ID. MYPROG.
*---------------------------------------------------------------------*
DATA DIVISION.
WORKING-STORAGE SECTION.
01 AA IS EXTERNAL.
03 AA-1 PIC X(48).
* ^^
* If I change here to 47 the it is OK !
*---------------------------------------------------------------------*
PROCEDURE DIVISION.
A-MAIN SECTION.
A00.
display 111111.
A-EXIT. EXIT PROGRAM.
A-STOP.
STOP RUN.

On AIX 3.2 this program run as expected.

2) Maybe the most severe problem, I cannot compile a program with -g
for debugging, the link editor complains:

cob -gx myfile.cob

ld: 0711-593 SEVERE ERROR: Symbol C_BSTAT (entry 66) in object myprog.o:
The symbol refers to a csect with symbol number 0, which was not
found. The new symbol cannot be associated with a csect and
is being ignored.
ld: 0711-593 SEVERE ERROR: Symbol C_BSTAT (entry 70) in object myprog.o:
The symbol refers to a csect with symbol number 0, which was not
found. The new symbol cannot be associated with a csect and
is being ignored.

again, this is NOT happening on AIX 3.2

3) When trying to compile a file with a name longer then 27 characters
the compiler return an error:

cob -x a2345678901234567890123456789.cob

Syntax

and off course, this is NOT happening on AIX 3.2
