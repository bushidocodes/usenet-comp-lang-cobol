       CBL  NOTERMINAL
       CBL  NOSEQUENCE
       IDENTIFICATION DIVISION.
       PROGRAM-ID.  CALENDAR.
      *this program prints a calendar for a given year
      *or the current year (range 1951 to 2050)
      *
      *to run type CALENDAR and on prompt press return for this year,
      *or a 2-digit (year) value
      *
       ENVIRONMENT DIVISION.
       INPUT-OUTPUT SECTION.
       FILE-CONTROL.
           SELECT FS-CALENDAR
                  ASSIGN "CALENDAR.RPT"
                  ORGANIZATION LINE SEQUENTIAL
      *this MODE makes the output easier to read on-screen
                  MODE IS NOCC.
      *
       DATA DIVISION.
       FILE SECTION.
       FD  FS-CALENDAR
           REPORT IS RS-CALENDAR.
       WORKING-STORAGE SECTION.
       01  WS-GENERAL.
           05  WS-TODAY.
             07  WS-YEAR-XX.
               09  WS-YEAR         PIC 99.
             07  FILLER            PIC X(4).
           05  WS-START-DAY        PIC S9(4) COMP.
           05  WS-MONTH            PIC S9(4) COMP.
           05  WS-DAY-COUNT        PIC S9(4) COMP.
           05  WS-LEAP             PIC S9(4) COMP.
           05  WS-TEMP             PIC S9(9) COMP SYNC.
           05  WS-MONTH-NAME-TAB.
             07  FILLER            PIC X(54) VALUE
           ' January  February   March    April      May     June '.
             07  FILLER            PIC X(54) VALUE
           '   July    August September October  November December'.
           05  FILLER REDEFINES WS-MONTH-NAME-TAB.
             07  WS-MONTH-NAME     PIC X(9) OCCURS 12.
           05  WS-MONTH-DAYS-TAB   VALUE '312831303130313130313031'.
             07  WS-DAYS-IN-MONTH  PIC 99 OCCURS 12.
           05  WS-DAY-TABLE.
             07  WS-DAY-NUM        PIC 99 COMP  OCCURS 42.
       REPORT SECTION.
       RD  RS-CALENDAR   PAGE LIMIT 42   FIRST DE 7.
      *this is the page heading:
       01  TYPE PH  LINE 2.
           05  COL 50   'Calendar for'.
           05  COL + 2  '19'    WHEN WS-YEAR > 50
                        '20'    WHEN OTHER.
           05  COL 65   PIC 99  WS-YEAR.
      *this prints any single month;
      *the REPEATED clause puts them side-by-side "4-up"
       01  RS-MONTH-BLOCK  TYPE DE     REPEATED 4 TIMES EVERY 30 COLS.
         03  LINE + 4   COL 6   PIC X(10)  WS-MONTH-NAME (WS-MONTH).
         03  LINE + 2           '  Su M  T  W  Th F  S '.
      *the OCCURS clause defines an array of lines or columns:
         03  LINE   OCCURS 6    VARYING RS-SUNDAY FROM 1 BY 7.
           05  COL 2    PIC Z9  OCCURS 7 STEP 3
      *SOURCE and VARYING place different contents in each item
                    VARYING RS-DAYNO FROM RS-SUNDAY BY 1
                    SOURCE WS-DAY-NUM (RS-DAYNO)  BLANK WHEN ZERO.
      *this is the page footing
       01  TYPE PF  LINE.
           05  COL 35  'Produced using COBOL Report Writer on'.
      *there are a number of functions - note the variable PICTURE here:
           05  COL + 2  PIC <X(9)>              FUNC DAY.
           05  COL + 2  PIC <X(9)>B<99>B9(4)    FUNC MDATE.
      *
      *%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
       PROCEDURE DIVISION.
      *%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
       CALENDAR-BEGIN.
           DISPLAY 'Input required year or return for this year: '
               WITH NO ADVANCING
           MOVE SPACES TO WS-TODAY
           ACCEPT WS-YEAR-XX
      *if blank or bad, assume today's year
           IF   WS-YEAR NOT NUMERIC
                ACCEPT WS-TODAY FROM DATE.
      *calculation uses a well-known congruence:
           DIVIDE 4 INTO WS-YEAR GIVING WS-TEMP REMAINDER WS-LEAP
           COMPUTE WS-TEMP = (1461 * WS-YEAR) / 4
           IF   WS-LEAP = 0
                MOVE 29 TO WS-DAYS-IN-MONTH (2)
           ELSE ADD 1 TO WS-TEMP.
      *necessary since we arbitrarily assume 00-50 is century 2000:
           IF   WS-YEAR < 51
                ADD 6 TO WS-TEMP.
           DIVIDE 7 INTO WS-TEMP GIVING WS-TEMP REMAINDER WS-START-DAY
      *now print the calendar!
           OPEN EXTEND FS-CALENDAR
           INITIATE RS-CALENDAR
           PERFORM VARYING WS-MONTH FROM 1 BY 1 UNTIL WS-MONTH > 12
                INITIALIZE WS-DAY-TABLE
                PERFORM VARYING WS-DAY-COUNT FROM 1 BY 1
                  UNTIL WS-DAY-COUNT > WS-DAYS-IN-MONTH (WS-MONTH)
                     ADD 1 TO WS-START-DAY
                     MOVE WS-DAY-COUNT TO WS-DAY-NUM (WS-START-DAY)
                END-PERFORM
      *the GENERATE statement produces one entire month's data:
                GENERATE RS-MONTH-BLOCK
                DIVIDE 7 INTO WS-START-DAY
                  GIVING WS-TEMP REMAINDER WS-START-DAY
           END-PERFORM
           TERMINATE RS-CALENDAR
           CLOSE FS-CALENDAR
           DISPLAY "Calendar printed! - please see CALENDAR.PRT"
           STOP RUN.
      *37 data entries;  27 procedural "verbs"
      *END%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
