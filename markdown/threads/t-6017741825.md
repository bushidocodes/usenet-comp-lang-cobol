Others have responded with NUMVAL solutions, but that won't work for this
case.  This is the REVERSE of a NUMVAL situation.  The code "attempts" to
move a NUMERIC (non-integer) value to an alphanumeric item.

FIRST - why would you ever want to do this?  I can't think of any "typical"
coding cases for this, but if you want to "do it" - what you probably want
is a simple redefine, i.e.

 05 GLAETTUNGSWERT      PIC S9V999999.
 05 Alpha-GLAETTUNGSWERT      redefines GLAETTUNGSWERT
                                                 Pic   X(8).
 05 S-GLAETTUNGSWERT    PIC X(20).
 ....
 MOVE 0,000499 TO S-GLAETTUNGSWERT
 MOVE Alpha-GLAETTUNGSWERT TO S-GLAETTUNGSWERT

HOWEVER, this will treat the embedded sign as part of one of the
"characters".  MAYBE, what you want is a "display" version of the numeric
field.  In which case what you want is:


 05 GLAETTUNGSWERT      PIC S9V999999.
 05 Display-GLAETTUNGSWERT      PIC +9,999999.
 05 S-GLAETTUNGSWERT    PIC X(20).
 ....
 MOVE 0,000499 TO S-GLAETTUNGSWERT
 MOVE GLAETTUNGSWERT TO Display-GLAETTUNGSWERT

Tell us what you WANT this "move" to do, and we probably can help with
better suggestions.
```

In your sample, you had a numeric field with

     Pic  S9V99999

That doesn't have *either* "," or "." as the decimal point.  When you have
   Decimal-Point is Comma
in the COBOL special-names paragraph (as I assume you do) and you want to
change "," to "." in a NUMERIC-EDITED field, all you need to do is to use
INSPECT REPLACING, e.g.

   05  Num-Ed1   Pic +9,99999.
   05  Alpha redefines Num-Ed1   Pic  x(8)
        ...
  Inspect   Alpha replacing "," by "."
        or even easier (as long as this is "fixed format input") would be
  Move "." to Alpha (3:1)


HOWEVER, once again, your original post was for a numeric, not a
numeric-edited field and that has *NO* "," or "." in it.
```
