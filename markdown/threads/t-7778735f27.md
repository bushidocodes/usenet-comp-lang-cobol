You raise an interesting point here. I mean "asking the wrong
question."

I guess the typical "How do you dynamically allocate memory in
Cobol?" would probably be a wrong question. A better question
might be, "If I would normally be dynamically allocating memory to
achieve this or that goal, what do I do in Cobol _instead?"_

Disk files do have a size you don't know in advance when you code
the program (that's the whole point of dynamical memory
allocation), and these days disk access is typically cached even
when you use DOS, so a temporary disk file actually resides in
memory most of the time. What looks like a (temporary) disk file
at first glance might basically be dynamically allocated memory, in
a sense. (It's a shame I didn't think of it when I needed it.)

So it's probably not that Cobol absolutely can't do this or that.
The question is, how long does it take you to figure out what you
_can_ do instead of what you would normally be doing in C or Pascal
(or whatever language you are familiar with)?
