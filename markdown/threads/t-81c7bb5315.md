  In GENERAL (in an ANSI/ISO conforming program) you are correct.  HOWEVER,
as this was explicitly a NetExpress question, you might be interested in the
"well-documented" extension expressed in syntax rule 2 of the NE V4 LRM
which states (in its entirety),

"When the level-number of data-name-1 is other than 01, it must specify the
same number of character positions that the data item referenced by
data-name-2 contains except that the data-name-1 area can be either smaller
than the data-name-2 area

*>>> the next part is the relevant MF extension

or be larger than the data-name-2 area. In this case, extra storage is
reserved to provide sufficient storage for the number of character positions
in the largest of the redefining or redefined items.

*>>> end of documented extension

It is important to observe that the REDEFINES clause specifies the
redefinition of a storage area, not of the data items occupying the area."

Odd by true fact ...

 The ENTIRE procedure division is Optional in the '85 ANSI/ISO Standard.  It
is entirely "valid" to create such a program - and there are even some ODD
(and I have never seen them in the "real world") cases where it accomplishes
something.

For a SUB-program (not main program) there is an "implicit" Exit Program at
the end of "execution".  Therefore, if (for example) you have a main program
CALL a sub-program with an EXTERNAL item in its Working-Storage Section -
but NO procedure division, this will ALLOCATE the external item (if not
already done so in the run-unit).

I see NOTHING wrong with a WARNING message for a program structured this
way - but it certainly is NOT an "error"
