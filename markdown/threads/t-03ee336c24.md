Unfortunately, you can't get the DSName in COBOL. You have to pass the FD
(not the record) to an Assembler routine that can then make sense of the
DCB. (Note: There is *no* way to get the DSName for VSAM files - with the
current COBOL products - only QSAM)

Not completely true - if you're willing to use Assembler (and who
isn't?), you would call a subroutine passing the DDNAME as a parameter.
The subroutine uses DYNALLOC (SVC 99) to retrieve the DSN associated
with the DDNAME. I do this in my new (and improved) date routines, so I
know if I'm running in a test or prod environment (in my case the DDNAME
is not passed, as it's a required DD card in every batch step (our
control file)). The logic is the same, and I anticipate the results,
too. I could send some code if you want to pursue this.

The access method doesn't make any difference, nor does the subroutine
need addressability to your ACB.

You are correct that access method does not really matter - that assembler
can find the DSName from an ACB as well as a DCB. The problem is that the
new COBOLs (VS COBOL II and COBOL for this-and-that) do not allow a clean
compiler if you try to pass a file-name (FD) when the file is defined as
ORGANIZATION IS INDEXED (but does allow it if ORGANIZATION IS SEQUENTIAL).
This is why I say this technique works for QSAM but not VSAM.

Hope this helps:
data division.
working-storage section.
01 tcb-address-pointer value x'0000021c'.
02 tcb-addr-pointer pointer.
01 tiot-seg-point.
02 tiot-seg-pointer pointer.
02 tiot-seg-pnt redefines tiot-seg-pointer
pic s9(9) comp.
01 jfcb-point value low-values.
02 jfcb-pointer pointer.
02 jfcb-point-red redefines jfcb-pointer.
03 pic x.
03 jfcb-low-3 pic x(3).


linkage section.
01 tcb-pointer pointer.
01 tcb.
02 pic x(12).
02 tiot-pointer pointer.
01 tiot-start pic x(24).
01 tiot-seg.
02 tio-len pic x.
02 pic x(3).
02 ddname pic x(8).
02 jfcb-addr pic x(3).
01 jfcb.
02 pic x(16).
02 dsname pic x(44).

Procedure division.
set address of tcb-pointer to tcb-addr-pointer.
set address of tcb to tcb-pointer.
set address of tiot-start to tiot-pointer.
set tiot-seg-pointer to tiot-pointer.
add 24 to tiot-seg-pnt.
set address of tiot-seg to tiot-seg-pointer

perform until tio-len = low-values
move jfcb-addr to jfcb-low-3
set address of jfcb to jfcb-pointer
display ddname space dsname
add 20 to tiot-seg-pnt
set address of tiot-seg to tiot-seg-pointer
end-perform

goback.
