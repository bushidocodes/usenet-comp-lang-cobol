You should understand that "DAT" has no intrinsic meaning unique to COBOL;
the interior format of the file could be anything (including a dbf file).
There is no requirement that ties a file generated by a COBOL program to any
external filename.

A (suitably written) COBOL program can generate text files, undelimited text
files, tab-delimited text files, relative files, dbf files, PDF  files,
Excel or Access files, HTML files, or any other format, and then give the
result any name desired by the programmer. Conventionally, "DAT" means
"data" without regard for the internal format.

So, then, your quest is not so much with converting a "COBOL" file as it is
with converting the format you've got to something else. To do that, you
obviously have to define your present file - the definition of your present
file does not include the word "COBOL."
