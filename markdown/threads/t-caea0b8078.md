Hello.
   We are running Merant (formerly Micro Focus) Server Express 1.0,
fixpack level 4, on AIX, hardware an IBM 12-way S80.  We are using all
IDX8-format files, and have been experiencing periodic index corruption.
Most types of corruption have been recoverable using rebuild, but
several weeks ago we experienced corruption where the file header said
the file was 10 gigabytes when the file was actually 8 gigabytes, and
rebuild could not fix this type of corruption.  Merant has been notified
and is investigating, but I wanted to find out if others are using this
version of COBOL and have experienced periodic file corruption, and if
anyone has theories as to the causes.  So far we have been unable to
recreate the corruption in our testing.
    I'd also be interested in other issues people have found with Server
Express IDX8 files.

Thanks.
Bonnie
