```
COBOL sub-program parameters are, by default, *reference* parameters (you
may have come across the 'BY REFERENCE' and 'BY VALUE' clauses of the 'CALL'
verb) which means that the *address* of the linkage item is passed to the
sub-program. This allows two-way communication:

* The caller may place data values into the linkage area

* The callee may, also, place values into the linkage area. On
   return to the caller, the linkage area may be read and any
   value altered by the callee available

Thus, you might want to make a couple of small changes to your code:

* Alter your linkage section:

 LINKAGE-SECTION.
 01 LS-PAY-DATA.
    05 LS-ANNUAL-SALARY PIC 9(6)V99.
    05 WS-REG-PAY  PIC 9(7)V99.

* Alter your 'call' statement in the 'caller':

  CALL "SUBPROG" USING LS-PAY-DATA .....
  ....

* Alter your sub-program:

  110-CALCULATE-REG-PAY.
   DIVIDE WS-ANNUAL-SALARY BY BI-MONTH
   GIVING WS-REG-PAY

   MOVE WS-REG-PAY TO LS-REG-PAY

   EXIT PROGRAM.

   or, you could place the value straight into the linkage area:

   ...
   GIVING LS-REG-PAY
   ...

To answer your other question, yes, you can also use multiple parameters, if
you wish, or just use a single grouped item (each of its elements would be a
particular parameter value).

A couple of other things:

* You could use a parameter to return a 'status' code indicating
  successful execution or not

* You might want to check for 'division by zero' and an 'on exception'
   check in your sub-program

I hope this helps.
```

```
No, you do NOT need to put the parameter of the main program in Linkage
Section.  Normally it is either in the Working-Storage or File Section.

Remember that "linkage section" has no STORAGE by itself.  Therefore, if
your main program calls a subprogram and passes one or more parameters, then
they should be in WS (or file section).

If, on the other hand, your main program passes a parameter to a subprogram
and that subprogram passes the parameter to ANOTHER subprogram, then the
field would be in WS of the main program - but Linkage section of the two
subprograms.
```

```
05  NEW-VALUE   PIC ...

CALL 'MYPROG' USING RAW-VALUE   NEW-VALUE


In the subprogram (you must have an LINKAGE SECTION 01 level (in the
subprogram) for each data name mentioned in the subprogram's  USING
statement(s)):

LINKAGE SECTION.
01  RAW-VALUE   PIC ...
01  NEW-VALUE  PIC ...

PROCEDURE DIVISION USING    RAW-VALUE    NEW-VALUE

(calculation stuff yielding NEW-VALUE)
```
