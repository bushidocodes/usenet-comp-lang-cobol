>Here's the problem. I am working on a report that sorts all records by
>location, id number, then name, for example.  However, now if the location
is
>15, they want these records to be sorted by a new field(class code).


Although I belive this could be better done using an external sort in JCL, I
would do the following if my life depended on it:

Filter off all records into 3 files, with location code <15 =15 >15.

Sort files <15 and >15 as normal, keeping files separate.

Sort =15 file as needed.

Then either concatenate them in order internally or use the output in JCL as
a concatenated DD statement:

//SORTIN   DD DSN=SOTIN.FILE,DISP=SHR
//SORTLT15 DD DSN=&&LT15,DISP=(NEW,PASS,KEEP)
//SORTEQ15 DD DSN=&&EQ15,DISP=(NEW,PASS,KEEP)
//SORTGT15 DD DSN=&&GT15,DISP=(NEW,PASS,KEEP)
......THEN WHEN YOU NEED TO INPUT INTO NEXT STEP....
//INFILE   DD DSN=&&LT15,DISP=(SHR,PASS,KEEP)
//         DD DSN=&&EQ15,DISP=(SHR,PASS,KEEP)
//         DD DSN=&&GT15,DISP=(SHR,PASS,KEEP)

or after the 3 files have been created, simply read them in one at a time
writing them all out to the same file.


Excuse me if my clarity is insulting, but I just wanted to make sure you
understood.
