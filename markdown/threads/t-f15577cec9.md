### Screen Scraping Works, BUT...

- **From:** Greg Ferro <gferro@sprynet.com>
- **Date:** 2001-12-01T18:50:15+00:00
- **Newsgroups:** comp.lang.cobol
- **Message-ID:** `<3C092773.E6F113ED@sprynet.com>`

```
There was a screen scraping debacle at a major credit card company
several years ago.

A user department was tired of using 3270 emulation on their PC's, so
they simply hired some consultants to perform screen scraping and
produce a GUI interface to the legacy system.

Unfortunately, the user department didn't inform the Information Systems
department that they were now screen scraping.

When the IS department made cosmetic changes to the mainframe screen
formats, screen scraping stopped working and so did the user
department!  When the users complained to Quality Assurance, QA checked
the screens using 3270 terminal emulation.  Finding no problems, QA
closed out the trouble tickets as "problem not reproducible".

If you are going to use screen scraping, make sure the mainframe people
know what you are doing!
```
