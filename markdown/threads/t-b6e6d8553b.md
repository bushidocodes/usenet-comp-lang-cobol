For those that have not yet understood the problem!!.

PKZIP for MVS has a command line option  -DATA_TYPE that has a default
value of "detect".
This means that if this option is not set in the command line, then it
will search each individual file that is being compressed, and if any
"binary" character is found then it will treat the whole file as
binary, thus not perform any ebcdic to ascii conversion. The "tab"
character is among the ones that make it "binary" type.

So in order to make this file work the PKZIP command line needs to
include the "-DATA_TYPE=text" option so it converts the file to ascii
without trying to determine the file type automatically.
