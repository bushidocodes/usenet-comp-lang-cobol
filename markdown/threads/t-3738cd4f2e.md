```
Hello:

I have a variable defined as

	01	FILE-NAME			PIC X(60).

I'd like to be able to extract from this variable the actual name of the
file, that is, without the path. So I'm looking for the LAST occurence of /
(the directory separator) and from there I'd like to copy the content up to
the first trailing blank. I can think of many ways of doing this, but they
involve a loop running through 60  characters, managing a tiny state
machine. I'm hoping there might be a way to use UNSTRING to do this quickly
and simply. Any ideas?

Thanks,

Mayer
```

```
You could do something like the following.  Just increase the table
depending upon the # of subdirectorys you think there will be.  I used
reference modification to boost performance in the file-part check for
spaces - no need to check all 60 characters of each occurance.  Hope it
helps...

       WORKING-STORAGE SECTION.
       01  file-ptr                pic s9(3) comp.
       01  file-name               pic x(60) value 'c:\a\b\c\d.dat'.
       01  file-parts-table.
           03  file-part           pic x(60) occurs 9 times.

       PROCEDURE DIVISION.
           move spaces                 to file-parts-table.
           move 'c:\a\b\c\d\e.txt'     to file-name.
           unstring
               file-name delimited by all '\'
               into file-part(1)
                    file-part(2)
                    file-part(3)
                    file-part(4)
                    file-part(5)
                    file-part(6)
                    file-part(7)
                    file-part(8)
                    file-part(9)
           end-unstring.

           perform
           varying file-ptr from +1 by +1
           until file-ptr greater than +10
           or    file-part(file-ptr)(1:5) equal spaces
               continue
           end-perform.

           subtract +1                 from file-ptr.

           display 'last file part is: ' file-part(file-ptr).
```
