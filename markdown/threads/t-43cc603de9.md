```
Hi:

I was wondering (again) and wonder how many people writing
new COBOL applications for Windows use the Windows 'style',
i.e., event-driven, menu bars, listboxes, mouse-oriented,
radio buttons, etc.?

Personally, I find the whole Windows, mouse-oriented thing very
annoying and klutzy for business applications. The supreme
example of klutziness is the use of one of those listbox things
for the entry of the State. Nineteen clicks to replace two
keystrokes is the ultimate dim thing to do.

So, I never use any of those Windows features except pushbuttons
and those rarely. My programs are not event-driven and they
lead the user by the nose making them enter what I want them
to enter when I want them to enter it.  Unfortunately, non-computer
users like my customers perceive my stuff to be DOS-like and that
makes it hard to sell. In actuality, the software is a marvel
of simplicity (hey, if you don't blow your own trumpet, ain't
nobody gonna blow it for you). People expect things to be like
everything else even if everything else doesn't make sense.

So, the question is: how many of you develop things the Windows
way versus some other way?

Thanks

Tony Dilworth
```

```
Whether a system was designed for DOS or Windows, should be
irrelevant. A system should be designed to be supremely
efficient in terms of usage (number of keystrokes/clicks required
to perform a given function). It should also be completely
self-explanatory and require no external documentation.

The only difference between a DOS version and a Windows version
of such software would be the appearance of the screen. Using
Windows it is easy to make 'prettier' screens than DOS. DOS
screens are universally perceived to be dull and boring.

The very best-designed system would be operationally identical
without regard to the OS.
```
