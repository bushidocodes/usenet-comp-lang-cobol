should be concatenated in the DFHRPL for the CICS region before
SCEERUN.  Note that neither library should be in the link-edit
concatenation for linkage editing and that only SCEELKED should be
there.  Note that I am using the DD names known to systems maintenance
and those names will be prefixed by SYS1. or some other high level
indexes.

> In former days, on IBM, most 77-levels were byte aligned, while 01 was word

IIRC it was the other way around (the saving storage part). Both 01's &
77's are double word boundary aligned, but the fields in an 01 are
contiguous (save any BINARY SYNC fields which might force alignment). If
you defined one 77 as PIC S9(9) COMP and another as PIC X (maybe a
switch or flag), you'd use 16 bytes of memory for only 5 bytes of data,
i.e., 4 bytes wasted in the first one and 7 wasted in the 2nd one.

Grouping miscellaneous flags and data under an 01 does not necessarily
'save memory'.  While it uses less working-storage it may use
more code if the machine requires additional instructions to 
extract non-aligned bytes and/or mask off the rest of the word.

A few decades ago I recall helping a client to get his program to
fit into his 12Kword memory by changing all his character flags
to words ( specifically PIC 9 to PIC S1(23) COMP SYNC - it was a
24bit word).
