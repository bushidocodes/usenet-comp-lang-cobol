jmoore wrote:
> I need to make fields with punctuation removed and combined. ie:
> FM-25 to FM25. What is the best way?

The BEST way is to use a regular expression which CAN remove your 
unspecified "punctuation". (in effect doing the "shift left" for you.)

You didn't state what "punctuation" you want removed. The example shows a 
hyphen. What about other symbols?

However, you would need to call a RegEx engine (because it isn't 
intrinsically available in COBOL). If you are running in a Windows 
environment you could use the Microsoft one. You have a choice of using the 
.Net engine or an older desktop one.

If you are using a version of OO COBOL (Fujitsu or Micro Focus) you can 
invoke it directly on an object instance, otherwise you need to call  a .DLL 
or execute a WSH script that uses the .Net  RegEx engine.

That's the BEST way (because it is more flexible than writing it in COBOL 
and the engine is designed for doing exactly what you are asking.)

However, if you don't really want the BEST way, just a way that is OK, and 
you don't want any of that new fangled 21st century stuff,  and using ONLY 
procedural COBOL, then here's some advice:

1. DON'T use INSPECT ...REPLACING. You will end up double handling the 
string to remove spaces.

2. Use TWO strings and a control loop...    The OUTSTRING must not be 
defined as shorter than the INSTRING. Here's some (almost) code...
      MOVE SPACES TO OUTSTRING
      SET K = 1
       PERFORM
            VARYING J
                  FROM 1
                       BY  1
                 UNTIL J > [length of INSTRING. Can be hard coded from the 
picture length for efficiency, but using a LENGTH function would be more 
flexible if the size of the input is ever likely to change...]
                       IF NOT (INSTRING (J:1) = "-" [OR anything else you 
want removed])
                            MOVE INSTRING (J:1) TO OUTSTRING (K:1)
                            SET K UP BY 1
                       END-IF
       END-PERFORM
       MOVE OUTSTRING TO INSTRING   [so references to the string are still 
good. OUTSTRING is really just a work area.]

Obviously J and K are defined as usage is index. If you use reference 
modification, as shown above, there is no need to redefine either string 
with OCCURS on it.

HTH,

Pete.

It shouldn't be that inefficient on System Z (IBM 370 and successors), 
because each class test should be generated as a single 6-byte TRT 
instruction (Translate and Test), although each TRT instruction does 
require a data table up to 256 bytes long.

If the field DUMMY-X is defined as PIC X, the NUMERIC class test should 
only allow the digits 0-9, and should disallow any sign characters such 
as "+" or "-".

It's also possible to define your own class in the CONFIGURATION 
SECTION, SPECIAL NAMES paragraph and create your own class test.

http://publib.boulder.ibm.com/infocenter/iadthelp/v7r0/index.jsp?topic=/com.ibm.etools.iseries.langref.doc/evfeb4ls17.htm

ENVIRONMENT DIVISION.
CONFIGURATION SECTION.
SPECIAL-NAMES.
     CLASS NAMECHAR IS
           'A' THRU 'I'
           'J' THRU 'R'
           'S' THRU 'Z'
           '0' THRU '9'.

IF DUMMY-X IS NAMECHAR
     MOVE INSTRING (I:1) TO OUTSTRING (J:1)
     ADD 1 TO J
END-IF

On 19 ago, 09:21, jmoore <jmoore...@gmail.com> wrote:
> I need to make fields with punctuation removed and combined. ie:
> FM-25 to FM25. What is the best way?

Hi,

I'm not sure if this would be the best way, but certainly it works for
me. The following are two method that I've created for my Cobol
precompiler class. You may find it useful (I do!).

  usage:

   to replace a substring with nothing
   invoke yourClassInstance "ReplaceText" using textTobeReplaced
"whateverYouWant"  singlespace returning texttobereplaced

   to replace a string with something else
   invoke yourClassInstance "ReplaceText" using "Cobol is dead"
"dead" "very alive!"  returning finalText


   it could be improved by using dynamic memory allocation etc but a
1024 chars limit is all that I need.




 method-id. ReplaceText.
 data division.
 working-storage section.
 01  sourceString         pic x(1024).
 01  tobeReplaced         pic x(1024).
 01  toReplaceWith        pic x(1024).
 01  newString            pic x(1024).
 01  temp                 pic x(1024).
 01  sourceIndexOf        pic 9(04).
 01  newCurrChar          pic 9(04).
 01  nextSubString        pic 9(04).

 linkage section.
 01  inString             pic x any length.
 01  inSubString1         pic x any length.
 01  inSubString2         pic x any length.
 01  outString            pic x any length.

 procedure division using inString inSubString1 inSubString2 returning
outString.

     move   spaces        to  newString
     move   inString      to  sourceString
     move   inSubString1  to  tobeReplaced
     move   inSubString2  to  toReplaceWith
     move   1             to  newCurrChar

     perform until function stored-char-length(sourceString) = 0
             *> looking for substring
             move  self::"IndexOf"(sourceString, tobeReplaced)   to
sourceIndexOf
             *> if does not exist any more, get out
             if    sourceIndexOf = zero
                   move   sourceString                           to
newString(newCurrChar:)
                   exit perform
             end-if

             *> check if substring exists after first char to move
whatever exist before to the new string
             if    sourceIndexOf > 1
                   move  sourceString(1:sourceIndexOf - 1)    to
newString(newCurrChar:)
                   compute newCurrChar = function stored-char-
length(newString) + 1
             end-if

             move   toReplaceWith                               to
newString(newCurrChar:)

             if     toReplaceWith > spaces
                    add   function stored-char-
length(toReplaceWith)    to  newCurrChar
             end-if

             move  sourceString(sourceIndexOf + function stored-char-
length(tobeReplaced):)   to  temp
             move  temp                                         to
sourceString
     end-perform

     move    newString            to  outString

 end method ReplaceText.

 method-id. IndexOf.
 data division.
 working-storage section.
 01  indice      pic 9(5).
 01  temp        pic x(1024).
 01  temp2       pic x(1024).

 linkage section.
 01  inString        pic x any length.
 01  inSubString     pic x any length.
 01  outIndexOf      pic 9(5).

 procedure division using inString inSubString returning outIndexOf.
     if    inString = spaces or low-values
           move  zero                    to  outIndexOf
           exit  method
     end-if

     move  1                             to  outIndexOf
     move  inString                      to  temp
     move  function upper-case(temp)     to  temp
     move  inSubstring                   to  temp2
     move  function upper-case(temp2)    to  temp2

     perform until outIndexOf > function stored-char-length(temp)
             if    temp(outIndexOf:function stored-char-length(temp2))
= temp2
                   exit method
             end-if
             add   1         to  outIndexOf
     end-perform

     move    zero            to  outIndexOf

 end method IndexOf.
