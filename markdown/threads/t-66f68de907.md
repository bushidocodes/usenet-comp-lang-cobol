"Mr CBL" <MrCBL@cbl.com> wrote

> Another way toi eliminate this is to set the NOBOUNDS directive

First of all I don't think that the problem is a 'subscript WC-IND'
problem:
   
    a) the program runs correctly when otherwise compiled
    b) the program runs correctly when a DISPLAY is added
    c) a BOUND check failure is a completely different message

But if is _was_ a bound check error then 'eliminating' the error
message by using a NOBOUNDS is probably the stupidest thing to do.

Using a NOBOUNDS when there is a known subscript bug means that the
program is allowed to overwrite memory that it should not be.  This
will cause problems later, possibly just giving the wrong results,
maybe going completely berserk.

I think that you should revise your 'quality control' mechanisms so
that you can effectively arrive at bug-free programs rather than just
trying to hide the problems.
