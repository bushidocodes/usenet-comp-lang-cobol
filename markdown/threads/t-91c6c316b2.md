> What's SDK?

Microsoft's Software Development Kit.

> Where can I seek?

You can download the MS SDK at
http://www.microsoft.com/msdownload/platformsdk/sdkupdate/. The description
of GetSystemInformation can be found on
http://msdn.microsoft.com/library/default.asp?url=/library/en-us/fileio/volr
ef_6wfi.asp. You can also download the windows api written for Visual Basic
at http://www.allapi.net/agnet/apiguide.shtml.

What you must do when you gathered the information is to rewrite the used
Visual Basic/C++ structures into COBOL. You must make a call like the
following:

call "GetSystemInformation" with STDCALL using [args].

The [args] are described on the ms website and in the SDK. You need to make
typedefs within cobol or make sure that you pass the arguments in the right
form. For instance, the COBOL equivalent for DWORD is PIC S9(9) comp-5.
Though this is based on my experience with NetCOBOL 6.1a. Your compiler
might need an other defenition, see manual.

On thing to remember, is that the output of an API call must be directed to
pointers in your based-storage area and are passed through to Windows using
'by reference' instead of 'by value' in the initial call.

On the dowload part of the www.netcobol.com website you can find an API call
for messageboxes and starting an exe, check it for an example.

Hope you can use this information. Remember that using the Windows API isn't
that easy, but when you get the hang of it, it can be tammed.

Rik Hendriks.

SDK = Software Development Kit / Microsoft.

Something like this:

CALL 'GetVolumeInformationA' WITH STDCALL USING
   BY REFERENCE lpRootPathName  (String)
   BY REFERENCE lpVolumeNameBuffer (String)
   BY VALUE   nVolumeNameSize (4byte integer)
   BY VALUE  lpVolumeSerialNumber  (4byte integer)
   BY VALUE  lpMaximumComponentLength (4byte integer)
   BY VALUE  lpFileSystemFlags   (4byte integer)
   BY REFERENCE  lpFileSystemNameBuffer  (string)
   BY VALUE  nFileSystemNameSize  (4byte integer)

and something like weasels.
