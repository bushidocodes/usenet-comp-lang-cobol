Stephen:

COBOL sp2 supports Realia Workbench 3.1 and supports embedded OCX
(Active X) controls (com objects).  You can send and receive e-mail
through one of these.  Although I don't know of anyone using one of
these with Groupwise, I don't see why it wouldn't work.

I also believe that Thane Hubbell has developed a solution for
sending/receiving e-mail messages with an add-on tool to sp2.

In addition, Neil Williamson has developed an add-on tool for sp2
which allows intergration of Microsoft Office applications in your sp2
application.  This will allow you to send/receive e-mail through
Outlook Express.

These tools also work with our Thin Client software, so you could even
run your e-mail application on a server in one location and allow
clients in remote locations to run the application and send e-mail.

If we can be of assistance, please send a private e-mail message to
rtwolfe@flexus.com.
