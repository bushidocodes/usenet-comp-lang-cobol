Not only *can* programs share dlls, they *must*. If you have two
identically named dlls in different directories, if one is already
loaded and a program tries to call the other, the program will be linked
to the dll already loaded! Even if the dll is in a directory the
calling program will never search, a loaded dll with a matching name is
always used.
