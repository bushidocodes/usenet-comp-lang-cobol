IF in an IF and have statements around it like this:

IF x = 0 THEN
DISPLAY x
IF y = 0 THEN

Mike,

END-IF is the best solution so far if your compiler accepts it.

Since you do not use END-IF at your example so I assume you are using
a COBOL 74 compiler. To solve your problem, using a perform statement
to replace the IF Y = 0 and DISPLAY Y statements is one of the choice.

IF X = 0
DISPLAY X
PERFORM XXXX-DISPLAY-Y
DISPLAY Z.

XXXX-DISPLAY-Y.
IF Y = 0
DISPLAY Y.

Another possible solution is to reconstruct the nested IF logic like

IF X = 0 AND Y NOT = 0
DISPLAY X
DISPLAY Z
ELSE
IF X = 0 AND Y = 0
DISPLAY X
DISPLAY Y
DISPLAY Z.

A bit stupid isn't it. But it works. Personally I hate to code like
this cause it makes the code harder to maintain later on. I get a
headache each time I try to maintain a program with a sequence of
IF ... ELSE IF... ELSE IF... and there is a lot of AND and OR
operators within the condition expression.

The last method I can think of recently is a Q&D (Quick and Dirty)
way. Rewrite your code like this also work.

IF X = 0
DISPLAY X
IF Y = 0
DISPLAY Y
IF X = 0
DISPLAY Z.

I prefer the first method as it clearly show the logic of the
program and is a lot more easy to maintain.
