:>I have a problem making a tbdisplay in a rexx zdspdvtb after calling a
:>clist zd087z from the same rexx.
:>zd087 is a cobol program.
:>After returning in the base rexx I make a tbtop tbskip and tbdisplay.

:>00000017)
:>Can you give me a suggestion how I can solve that kind of problem. 

That is a typical abend for an overlay or freed storage.

By using ADDRESS TSO to invoke the clist and DB2 you are causing the same REXX
function pool to be used. When these programs end the storage used is gone.

Try using 
         ADDRESS ISPEXEC 
         "SELECT CMD(EXEC 'DBP3.DD.TXCODE2.ISPCLIB(ZD087Z)')"
to invoke the CLIST and see if the problem goes away.
