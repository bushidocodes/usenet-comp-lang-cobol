```
I'm using NetExpress 3.0 to create an Internet application.  My problem is
that the entry fields that I create on the HTML form act strange.  For
example, I set the attributes on a field to have a max of 8 characters and
the pic statement is 9(8).  When the form is loaded into the web browser and
you tab to this field, the default is to not type over the character in the
field.  This poses a problem because the field begins with the following
value 00000000.  When the user tabs to this field they cannot type any
additional characters since the max attribute for the field is set to 8 and
there are already eight characters in the field.  If the user presses the
insert key on their keyboard, it types over the values present for that
field.  However, they have to press the insert key on each and every
additional field to get it to type over.

First question: How do you set the default property of the field to act like
the insert button has already been pressed,  in other words, type over what
is initially there?

Second question: How do you prevent your Internet application from
initializing these HTML numeric fields with zeros?  I would rather have
these fields initialized with nothing in them.
```

Don't set an initial value for the field. You can add validation
to the field to make sure only numeric data can be entered by
right-clicking the field and then select validation from the
content menu. You will have to manually move the digits from the
input field to your pic 9(x) field.

For the field properties, make sure the attributes tab is
selected. Blank out any entry for the Value keyword (bottom of
the list).

What I believe you are saying, is that in the HTML form you have
specified 9(8)? If so, this is incorrect, you must specifify x
(8). Then use the validation routine for the field to make sure
that only whole numbers are entered in the field.

>
>That would work.  However, I will still be left with the
problem of having
>an alpha-numeric field that has 8 characters in it, at times,

Once the field is identified as alphanumeric, it will be
initialized as spaces. The user can then type what ever they
wish. Remember to set the validation for the field to number. Be
aware, the data will be left justified as oppose to left
justified, which is what you would expect for a numeric field. I
use the following the take care of the alphanumeric to numeric:
(others may have a better suggestion)

       01  misc-data.
           05  sub                     pic 9(04) comp-5.
           05  i-sub                   pic 9(04) comp-5.
           05  login-id                pic 9(07).
.
.
       linkage section.
       01  ls-loginid                  pic x(07).
.
.
               move length login-id to sub
               perform varying i-sub from length ls-loginid by -
1 until
                       i-sub = zeros
                   if ls-loginid (i-sub:1) numeric
                       move ls-loginid (i-sub:1) to login-id
(sub:1)
                       subtract 1      from sub
                   end-if
               end-perform
