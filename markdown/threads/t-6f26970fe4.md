Can anyone explain the difference between the NetExpress "CBL_ALLOC_MEM" and
"CBL_ALLOC_SHMEM" calls?  And also, if using the "CBL_ALLOC_MEM", what
"flags" should I be using to force the shared memory to be released when the
run-unit is terminated?

I am using the "CBL_ALLOC_MEM" call for a program that uses IBM Personal
Communications CPI-C support.  I set the "flags" parameter to a value of "1"
(see code excerpt below), meaning allocate a shared memory segment.  I do
not issue a free memory call to release the memory, but rely on the
termination of the run-unit to free it (I know, bad programming practice!).
The memory is allocated successfully the first time through, but subsequent
run unit requests fail - I'm not for sure what the return code is ...  I
guess my real question is what do bits 2 and 3 of the flags parameter
control on the CBL_ALLOC_MEM call?

Any help would be appreciated ...

Thanks - Todd

CPI-C Manual Excerpt:
When a program issues a call that supplies a data buffer, Communications
Manager determines whether the data buffer can be shared (giveable memory)
across processes.  If it can, Communications Manager uses the buffer
directly as the source or destination of data. If the buffer cannot be
shared, Communications Manager must copy the data between memory it
allocates and the program's buffer. Therefore, by using memory that can be
shared for the buffer, the program can avoid the extra copying of the data
and the memory allocation that Communications Manager would otherwise do on
each applicable call.

It is recommended for future migration that buffers be defined with the
tiled memory allocation option.
