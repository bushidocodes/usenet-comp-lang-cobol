  LOCAL FIX:

  PROBLEM SUMMARY:
  ****************************************************************
  * USERS AFFECTED: Users who compiles source codes that         *
  *                 are created beyond year 2000.                *
  ****************************************************************
  * PROBLEM DESCRIPTION: The compiler can not extract the        *
  *                      timestamp of a file that is created     *
  *                      after year 2000.                        *
  *                                                              *
  *                      This causes problem in compiling codes  *
  *                      involving file's timestamp              *
  *                                                              *
  *                      For example:                            *
  *                      #pragma comment (timestamp)             *
  ****************************************************************
  * RECOMMENDATION:                                              *
  ****************************************************************
  The routine that retrieves VM file's timestamp must be
  able to recognize a timestamp that is beyond year 2000.

  The cause of this problem is due to an effort in making the
  CMS system Y2K compliant.  In order to allow the system to
  recognize Y2K date, a century bit was turned on when the year
  is beyond 2000.  The compiler needs to make a corresponding
  change to recognize this century bit.


  PROBLEM CONCLUSION:
  The compiler routine that retrieve the VM file's timestamp
  has been modified to recognize the century bit."
