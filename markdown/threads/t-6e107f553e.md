```
I recall a posting about this a while back. I had a similar problem
today where SCO open server V gave a too many locks message (9/213 or
similar) The problem was actually caused by the /etc/hosts on each
system not having EXACTLY the right name for each machine (including
DOMAIN name) and the file /etc/default/tcp not having the correct
domain name. Once these were all set up the whole thing worked a
treat.
I hope that helps someone else - as I sweated blood !

Nick.
```
