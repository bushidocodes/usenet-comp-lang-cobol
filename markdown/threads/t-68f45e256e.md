Microsoft used to have a cobol compiler that came out back in the 8086 days.  A source 
program was first compiled, and then linked into a ".EXE".  There is a slightly later 
version which compiles into a ".INT', which then requires the presence of COBRUN.EXE for 
it to execute.  I do not think that a decompiler exists for either of them.
