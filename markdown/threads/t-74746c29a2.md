I think it's the other way around - the syntax errors are already
there, but they seem to be confusing the compiler enough to cause the
spurious .lst file lines.

The error messages from the compiler seem to be sufficient to get you
looking at the right place in your COBOL text book/manual :

› FILE (Test.cbl)
› --------------------------------------------
…[4 more quoted lines elided]…
›        PROGRAM-ID.  E1.CBL
Try:

PROGRAM-ID. E1.

or

PROGRAM-ID. "E1.CBL".

› CONFIGURATION SECTION.
› SOURCE COMPUTER. Pentium.
› OBJECT COMPUTER. Pentium.

Those should be SOURCE-COMPUTER and OBJECT-COMPUTER.

Cheers,
Kev.
