```
I got a copy of Fuj. Cobol 3.0 to learn Cobol and right now I'm trying to do
a lesson with alphanumeric variables.  But I'm having a bit of a problem.

When I compile and try to run a program that uses an alpha variable, it
doesn't get past the ACCEPT part until I enter enough characters to equal
the variable size; numeric variables don't seem to pose a problem.

For example, in a program that accepts input and then redisplays it, if the
variable size is set to 10, such as in:

01 THE-NAME PIC X(10).

if I enter 'John', I'm still left with the prompt awaiting more input until
there are 10 characters entered.  So if I enter 'John' two more times, it
displays:

JohnJohnJo


It seems like it won't accept anything less than the specified variable
size.  Is this some "feature" of the compiler that can be turned off, or
something that must be coded around?  Like I said, numerics (eg, PIC 9(2). )
don't seem to have a problem.

Thanks for any help.

- John Wolanski

Here is the code:

@OPTIONS MAIN
IDENTIFICATION DIVISION.
PROGRAM-ID.  HELLO02.
ENVIRONMENT DIVISION.

DATA DIVISION.
WORKING-STORAGE SECTION.
01 THE-NAME PIC X(10).

PROCEDURE DIVISION.
PROGRAM-BEGIN.
DISPLAY "Enter someone's name.".
ACCEPT THE-NAME.
DISPLAY "Hello " THE-NAME.
PROGRAM-DONE.

STOP RUN.
```

One of the questions that we seem to get once every few months.

With Fujitsu (but not all COBOLs), you will get the desired behavior if you
change:

 ACCEPT THE-NAME.

     to

 ACCEPT THE-NAME from Console.

Yes.  Keep in mind there is both a screen and a console.  The console is
intended as the console of a mainframe ... messages like "please mount
tape".  The screen is for programmed I/O.

I rather like having two separate devices.  It is handy for debugging, and I
use the console only for that.  However the console routines are as
primitive as hell.
