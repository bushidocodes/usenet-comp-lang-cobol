Can you set an environement value from C?

If so, you can the value from the environment like this:

DISPLAY "MSG" UPON ENVIROMENT-NAME.
ACCEPT WS-MSG FROM ENVIRONMENT-VALUE.

and lo and behold: the contents of the MSG environment value will be in
WS-MSG.

The EXTERNAL clause is for sharing data between different COBOL programs in
the same run-unit.

```
You need to give the external clause the name of the symbol and ensure the
types match up.... for example:

/a

Before ./a
Before COB Hello.....
COB : Hello.....
After COB There.....
After     ./a

and the 'C' is:

#include <stdio.h>
#include <stdlib.h>
#include <string.h>

char *MSG;
void COB();
int main(int argc, char *argv[])
{
    MSG=(char*)malloc(11);
    memset(MSG,0,11); /* reserver a NULL */
    printf("Before %s\n",argv[0]);
    strcpy(MSG,"Hello.....");
    printf("Before COB %s\n",MSG);
    COB();
    printf("After COB %s\n",MSG);
    printf("After     %s\n",argv[0]);
}

and the COBOL is:

       program-id. "COB".
       working-storage section.
       01 MSG is external by "MSG" usage pointer.

       linkage section.
       01 lnk-msg pic x(10).
       Procedure Division.
         set address of lnk-msg to MSG
         display "COB : " lnk-msg
         move "There....." to lnk-msg
         exit program.
```
