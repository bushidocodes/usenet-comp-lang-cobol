When I enter our Micro Focus environment and before running any applications
there (I telnet to a SunOS Unix server), the numeric key pad on my IBM PS/2
terminal recognizes all of the numbers 0-9 and the 'ENTER' key, but when I
attempt to enter numbers as input for the applications they terminate as if I
had pressed the ESCape key. When I return to the native Unix environment,
the numbers from the keypad are no longer recognized, i.e., I get those
strange characters of non-recognition, e.g., ^[Oq and ^[Or - the last
character appears to be the alphabet in sequential order.

My question is: Is/are there an environment variable(s) I can set in our
users' profile that will ALWAYS enable the recognition of the numeric keypad
while running Micro Focus COBOL on an Unix server?
