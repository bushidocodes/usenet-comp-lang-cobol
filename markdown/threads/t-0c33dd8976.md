In VS COBOL II:
Pointer data items can contain virtual storage addresses. You define them explicitly
with the USAGE IS POINTER clause in the Data Division or implicitly as ADDRESS OF
special registers.

Pointer data items can be:
Â° Passed between programs (CALL ... BY REFERENCE statement)
Â° Moved to other pointers (SET statement)
Â° Compared to other pointers for equality (relation condition)
Â° Initialized to contain an invalid address (VALUE IS NULL).


You can use pointer data items when you want to accomplish limited base addressing,
particularly when you want to pass and receive addresses of a variably located record
area.

You can also use pointers to handle a chained list.


POINTER Phrase:

A data item defined with the USAGE IS POINTER clause is a pointer data item.
A pointer data item is a 4-byte elementary item that can be used to accomplish limited
base addressing. Pointer data items can be compared for equality or moved to other
pointer items.

A pointer data item can only be used:
Â° In a SET statement (Format 5 only)
Â° In a relation condition
Â° In the USING phrase of a CALL statement, an ENTRY statement, or the Procedure
Division header.

The USAGE IS POINTER clause can be written at any level except level 88.
If a group item is described with the USAGE IS POINTER clause, the elementary items
within the group are pointer data items; the group itself is not a pointer data item and
cannot be used in the syntax where a pointer data item is allowed. The USAGE clause of
an elementary item cannot contradict the USAGE clause of a group to which the item
belongs.


Pointer data items can be part of a group that is referred to in a MOVE statement or an
input/output statement. However, if a pointer data item is part of a group, there is no
conversion of values when the statement is executed.

A pointer data item can be the subject or object of a REDEFINES clause.

SYNCHRONIZED can be used with USAGE IS POINTER to obtain efficient use of the pointer
data item.

A VALUE clause for a pointer data item can contain only NULL or NULLS.

A pointer data item cannot be a conditional variable.

A pointer data item does not belong to any class or category.

The JUSTIFIED, PICTURE, and BLANK WHEN ZERO clauses cannot be used to describe group or
elementary items defined with the USAGE IS POINTER clause.

Pointer data items are ignored in CORRESPONDING operations.

A pointer data item can be written to a data set, but, upon subsequent reading of the
record containing the pointer, the address contained may no longer represent a valid
pointer.

Note: USAGE IS POINTER is implicitly specified for the ADDRESS OF special register.


Incrementing Addresses Received from Another Program:


The data passed from a calling program might contain header information that you want to
ignore (for example, in data received from a CICS application that is not migrated to
the command level).

Because pointer data items are not numeric, you cannot directly perform arithmetic on
them. However, you can use the SET verb to increment the passed address in order to
bypass header information.

You could set up the LINKAGE SECTION like this:


LINKAGE SECTION.
01 RECORD-A.
05 HEADER PIC X(12).
05 REAL-SALARY-REC PIC X(30).
.
.
.
01 SALARY-REC.
05 PTR-NEXT-REC POINTER.
05 NAME PIC X(20).
05 DEPT PIC 9(4).
05 SALARY PIC 9(6).


Within the Procedure Division, "base" the address of SALARY-REC on the

address of REAL-SALARY-REC:

SET ADDRESS OF SALARY-REC TO ADDRESS OF REAL-SALARY-REC

SALARY-REC is now based on the address of RECORD-A + 12.

Here's my example.

test1.cbl:

data division.
working-storage section.
01 dummy pic x(12) value "hello world" & x'00'.
01 dummy-pointer pointer.
01 dummy-pointer-num redefines dummy-pointer
pic 9(9) comp.

procedure division.
main.
set dummy-pointer to address of dummy.
perform 12 times
call "c_call" using by value dummy-pointer
add 1 to dummy-pointer-num
end-perform.

test2.c

c_call(pointer)
char *pointer;
{
printf ("%s\n", pointer);
}

compile with:

cob -vx -C CONVERTPTR test1.cbl test2.c

Run by typing "test1". Should get this output:


hello world
ello world
llo world
lo world
o world
world
world
orld
rld
ld
d


Hope that helps. Oh the compiler directive CONVERTPTR is required if
you wish to modify that numeric value of the pointer itself.

Been a while since I've done it and I'm working from memory so apologies
for any syntax errors, but here's a quick go...

Will use this data div example for discussion below

WORKING-STORAGE SECTION.
01 MY-RECORD PIC X(500).
01 MY-POINTER USAGE IS POINTER.

LINKAGE SECTION.
01 PARM-1 PIC X(30).
01 LINK-RECORD PIC X(500).

PROCEDURE DIVISION USING PARM-1.


The pointer is allocated by declaring usage as pointer in w-s section.
Can't be sure of syntax so you'll have to double-check this yourself
(sorry).

Can then set this to point (i.e. refer) to a data item in w-s or linkage,
e.g.

SET MY-POINTER TO ADDRESS OF MY-RECORD.


You can do things like allow a linkage record to refer to data that a
pointer is pointing at.

Linkage addressability is usually set by using statement on procedure
division (i.e. set to refer to passed parameters). The setting of PARM-1 is
an example of this.

However, you can also set the linkage area's address like this

SET ADDRESS OF LINK-RECORD TO MY-POINTER.


Can be useful if your COBOL program is passed the _address_ of a record in
storage. You can then refer to and manipulate the record by setting a
linkage record to refer to the storage area pointed at by this address,
e.g.


LINKAGE SECTION.
**** Pointer list contains a list of addresses.
01 POINTER-LIST.
05 RECORD-PTR-1 USAGE IS POINTER.
05 RECORD-PTR-2 USAGE IS POINTER.
05 RECORD-PTR-3 USAGE IS POINTER.
01 LINK-RECORD PIC X(500).

PROCEDURE DIVISION USING POINTER-LIST.

SET ADDRESS OF LINK-RECORD TO RECORD-PTR-1.


With a bit of help from an assembler GETMAIN routine, you could get clever
and build up a linked-list using pointers.

Use GetMain to allocate storage, set a linkage record to the address
returned by GetMain. Define a pointer inside this linkage record to be of
usage pointer. You can now getmain a 2nd record and set the pointer in the
1st record to the address of the second record; thus linking to nodes
(records) together.

Some notes (again from memory)...
- I think pointer is an extension (IBM in my case) to COBOL.
- You use SET to assign an address to a pointer.
- You can set a pointer to NULL to indicate that it does not refer to
anything.
- You can only point at level 01 or level 77 data-items.

Hope this gives some clues,
Steve.
