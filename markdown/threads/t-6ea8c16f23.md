```
Actually I'm porting a cobol host application with embedded SQL
to Fujitsu Cobol Version 3.0 on Windows 2000. The DBMS is IBM's
DB2 in Version 7.2.

It seems that there is a awful limitation in the SQL SELECT
statement. The number of selected columns in the database table
is restricted to a magic value (lower than 59).

      EXEC SQL
         SELECT
            column1,
            column2,
            ...
            column59
         INTO :host-var-structure
         FROM table1
         WHERE column1 = :SELECT-REC-ID
      END-EXEC

When I reduce the number of selected table columns to 30 or 40
columns my program runs properly, otherwise it will be stopped
by the OS in order to a memory violation. I have changed the
subset of selected columns to be sure there is no other problem.

Any help will be appreciated. 

Please send a copy of Your response to my eMail address:
klaus.duttle@icongmbh.de

thanx

Klaus Duttle
```
