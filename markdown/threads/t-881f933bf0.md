I am having a DMSII problem on a V Series system.  $SAVECP is set in the
compile, which is supposed to cause all record and path pointer operations
to behave exactly the same as on A Series.

I'm looping through a data set adding one to the year in all date fields
(Y2K testing).  There is a set with a date in the key and NO DUPLICATES.
I'm trying to do this as follows:

    SET KEY-SET TO ENDING.
LOOP-HERE.
    DELETE PRIOR KEY-SET.
    IF DMSTATUS(NOTFOUND) GO TO LOOP-EXIT.
    RECREATE DATA-SET.
    fiddle with key
    STORE DATA-SET.
    GO TO LOOP-HERE.

(I also tried this with the DELETE split into MODIFY PRIOR KEY-SET and
DELETE DATA-SET.  According to the documentation, these should function
identically, and they do.)

This loops forever on the last record.

In V Series COBOL ANSI-74 Programming Reference Manual Volume 2, on page
1-10 there is a chart labeled Current Path Pointer States.  The footnote
says the following:  "The state of the current path pointer is not
affected by a FREE operation, a CREATE operation, a RECREATE operation, a
STORE operation, or any unsuccessful DMSII operation".

DELETE sets the current path to deleted state pointing where the deleted
record was.  A subsequent DELETE PRIOR (or MODIFY PRIOR) should get the
record before that.  The intervening RECREATE and STORE shouldn't change
this.  Yet I keep getting the last record over and over.

I find the following on page 1-62:  "Current path pointers are not by the
STORE operation unless the current path is pointing at the record and the
key is changed."  This contradicts page 1-10.  Is the current path
pointing at the record?  I wouldn't expect this to apply when I've done a
DELETE and RECREATE; it should still be pointing where the deleted record
was.  Does RECREATE resurrect this association?  Even so, following the
STORE, if the current path now points to the record just STOREd, shouldn't
DELETE (or MODIFY) PRIOR find the record before that?

How do I get the record before the one I just stored?  Does anyone know if
A Series and V Series (with $SAVECP set) behave differently on this point?
