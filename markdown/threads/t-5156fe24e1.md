You don't say so, but my guess is that you are talking about IBM MVS COBOL
products.  (Sorry Mark, if he really is VSE, you can modify this for your
side of the world)

COBOL I (and it is *NEVER* called that) is probably the OS/VS COBOL compiler
(supporting the ANS'68 and ANS'74 Standards - as well as many extensions).
It is "long obsolete" (but still in all too common use)

COBOL II (and it is called that, although its correct name is VS COBOL II)
from Release 3 on supported the ANS'85 Standard (and with very minor
exceptions ANS'74).  It has been removed from marketing by IBM already and
will lose support very soon after Y2K time (for which it has minimal
support).

VS COBOL II was replaced by the now dead COBOL/370 which has been replaced
by IBM COBOL for MVS & VM which was replaced by IBM COBOL for OS/390 & VM
(with or with out the Millennium Language Extensions).  All of these require
LE (Language Environment) to be available as their run-time.

The farther along this chain you go, the more "modern" features you get and
the more of the old (particularly the undocumented ones) extensions you
lose.

I hope this VERY brief note answers your question.  As I wrote a book on it,
needless to say, there is MUCH more that could be said.

What is COBOL I?

COBOL had had extensions and changes over the years.  ANSI creates
standandards which are accepted by the various compiler companies - or
not, and the compiler companies often have extensions.

I have used ANSI 68 & ANSI 72 which have minor differences.  Then I
worked on a VAX which had a COBOL compiler with useful extensions.  When
I later worked on an IBM again, I found out that COBOL II was IBM's name
for the version of COBOL which I had been working on the VAX (but
interviewers still wanted to know if you had COBOL II, rather than
whatever ANSI version that was).  But COBOL II still wasn't Y2K
compliant and has been replaced by a COBOL which is (and had better
called libraries).

Here's an incomplete list of differences.
CURRENT-DATE is gone
EXAMINE is still gone
EXHIBIT is gone  (was only IBM, but I miss it)
OTHERWISE is gone
REMARKS is gone
THEN is gone
TRANSFORM is gone
WRITE AFTER POSITIONING is gone
REPORT WRITER is gone
UNSTRING DELIMITED BY ALL is gone
READY TRACE is gone  (I miss this)


Commands have scope terminators, most of which are pretty much unneeded,
but the following can be very useful:
   END-IF
   END-EVALUATIE
   END-SEARCH
   END-PERFORM
ALPHABETIC compares include lower-case
We now have in-line PERFORMS
CONTINUE is a nop within if clauses which does not go to NEXT SENTENCE.
PERFORM has an option for "WITH TEST AFTER" instead of the default
testing before
All? test options now understand NOT
EVALUATE is a new command which is very useful and widely used 
INITIALIZE initializes data to zeros or spaces (it can initialize to
values too).  It is often mis-used.
We have pointer variables.
We can find LENGTH OF FIELD-A
the SET command is greatly expanded.  New COBOL II programmers will use
it primarily to SET an 88 level to TRUE.
You can have nested COPY statements.

Usage has new reserved words, PACKED-DECIMAL, BINARY, and POINTER
COBOL II will let you move a field with a PIC 9999.99 to one PIC 9999V99
You can have nested programs
You can have double byte character strings (just when everybody else is
going to unicode!)
Source can be in mixed case.
