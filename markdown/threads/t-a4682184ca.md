Linc is a Unisys product.
It generates datasets for a DMSII database, along with
the appropriate maintenance screens, very rapidly.
I do not see how it would be a useful tool for Y2K.
If you are already a Linc expert, you can write new systems
quickly; but it would not be useful for modifying existing
COBOL systems.
You can probaby find some experts at comp.sys.unisys.

One company that uses it (unfortunately) is Spartan Mills.
Some others include Westinghouse, Home Shopping
Network, Riverwood International, De Kalb County Schools,
etc.
