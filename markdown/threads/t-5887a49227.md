Another type of cobol available to you is acucobol.  It offers many
advantages.  Outside of a mainframe OS, and an Apple OS it runs on every
other platform.  Like java, it's portable - compile once and deploy
anywhere.  The makers of it have also built in a lot of extensions, such
as the ability to build graphical interfaces, make calls to databases,
run a cobol program on web server as a cgi program or run in a web
browser as a plugin, etc.  It's also very reliable.

Several and on many operating systems such as Unixware, Open Server, Linux, 
MS-DOS, System Manager, and something called Windows too.

> I know that
> 'power' cobol by fujitsu will run on the pc, unix, hpux, and solaris.  

No.  Fujitsu Cobol is available for Windows, Linux (RedHat), Solaris and 
HPUX.  'PowerCobol' is only for Windows.

> Are there versions uniquely designed for the pc?  

What exactly are you referring to when you say 'pc'.  The first 'pc' was 
the Apple ][, since then it has been used for many different machines but 
mainly the 'IBM PC' (or clone).  However these can run a variety of 
operating systems including Unix, Linux and Windows.

If you mean 'Windows' then say so.

'PowerCobol' is uniquely for Windows, though it probably shares compiler 
code with the other versions.  Some other compilers have been MS-DOS only.

> what about the IBM mainframes? what about other types of mainframes?  

I can't think of a mainframe that wouldn't run Cobol, but there may be some.

see www.microfocus.com

Micro Focus has various COBOL compiler products that run on Windows, Linux
(RedHat & SuSE), Unix (Solaris, HP-UX, AIX, Tru64, Dynix, SCO[Caldera]
OpenServer, OpenUnix and UnixWare and others).

The Windows product is NetExpress which is as you describe in another post
as "Best of Both Worlds" allowing development of character, gui, html, com,
mts, java-wrapped etc etc applications and programs in cobol. It allows
interoperability with .NET and a full integrate .NET product is under
development.

For the major Unix platforms there is Server Express which allows
development of cobol applications allowing interoperability with java.
Server Express also allows the deployment of HTML applications developed
using Net Express.

For the Minor Unix platforms and Linux there is have Object COBOL Developer
suite. This is as Server Express but does not support multi threading and
Java interoperability.

From late February there will be a Server Express for the zSeries
implementation of Linux.

For developing mainframe applications on the PC there is Mainframe Express.
This allows you to use the power of your PC to edit test and debug your
mainframe COBOL application before deploying it to the mainframe. This can
save you lots in terms of connection costs and CPU usage costs.

Micro Focus also have product set called Revolve which is a very powerful
application code analysis tool.
