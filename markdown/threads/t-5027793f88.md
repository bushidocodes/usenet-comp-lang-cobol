Parrish:
Yes, I do. But then I'm a Tech Support Rep for Acucobol. If you're
using the Windows runtime or or NT runtime you can write a .DLl which
uses the "C" calling convention and call it directly from Acucobol
with no relinking of the runtime. If the .DLL has multiple entry
point one of your first lines of code should be CALL NAME.DLL. This
will like the DLL into us and allow for the calling of any entry
point. Note that, for now, you can only use the "C" calling
convention. A later release (I think in less than 6 months) will
allow for either the "C" or Pascal calling conventions. The
difference I guess is the order that the arguements are analyzed. "C"
goes one direction and Pascal the other.

At any rate if you write the .DLL routines the same way you would
write a "C" routine to link into the runtime (see sub.c and direct.c)
then it should work.

If you have any additional questions please feel free to contact us
at 1-800-399-7220.

Pete Withey
Acucobol Tech Support
