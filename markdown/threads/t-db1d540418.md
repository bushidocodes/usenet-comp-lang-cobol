```
Hello all,

I have run into a minor problem in configuring Micro Focus Cobol and
DB2/2 on a Windows NT workstation.

Here is the situation:
----------------------

I am attempting to connect a Windows NT client running Micro Focus
WorkBench, HCO, CICS, DB2/2 v2 Single User to a OS/2 Server running
DB2/2 Server. The Data Base Manager Configuration file for both the
client and server are set up for AUTHENTICATION SERVER.

I would like to have DB2/2 for Windows NT pass the Username and Password
of the user that is logged on to the server for the authentication
rather the explicitly supplying it with every application. DB2/2 v2 for
OS/2 by default passes the user name and password when you specify
authentication server.

Current Work Around in Micro Focus Cobol:
-----------------------------------------

When I compile you must set the Compiler Directive SQLPASS equal to the
user name and password that you wish to compile with.

When I bring up a CICS region I must set the variable DBPASSWORD equal
to the user password in the DFHTX.CFG. It appears that this is passed
via HCO to the database server for initialization when the region comes
up.

I personally do not feel comfortable implementing with a user's password
stored in both of these files.

If any one could assist me I would greatly appreciate it.

Thanks,

Doug Schmucker
```
