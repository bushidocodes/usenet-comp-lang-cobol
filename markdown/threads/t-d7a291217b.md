#### ↳ gdb seems unable to display variables

- **From:** "Torsten Reichert" <Thorsten.Reichert@dresdner-bank.com>
- **Date:** 2002-10-15T17:06:27+02:00
- **Newsgroups:** alt.cobol,comp.lang.cobol
- **Message-ID:** `<aoh9ss$gng2@news-1.bank.dresdner.net>`
- **References:** `<aoh5d8$gn62@news-1.bank.dresdner.net>`

```
After starting a program, attaching gdb and stepping through, it does not
display variables:
gdb) p KONSTANTEN
$2 = {PGM-NAME = <optimized out or zero length>, CVS-TEXT = <incomplete
type>}

That they are not optimized away, is suggested by the following observations
1) xldb displays all variables
2) cob2 does not even have optimization options

I suppose the problem comes from the dash signs taht are valid in COBOL
names, that gdb interprets as minus signs.
Another possibillity would be that gdb does not know the storage layout of
IBM's compiler.

Has anyone found the solution already ?

Thank you
```
