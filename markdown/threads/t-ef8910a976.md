Bob,

Firstly, I would ask if the "terminals" are of a similar specification?  If
so, then
in general terms you would expect to see this sort of result.  However, bear
in mind that many aspects can affect performance.

Windows 95/98/ME are a hybrid operating system which contains a great
deal of 16 bit code (for backwards compatibility with DOS etc.).  This means
that these operating systems run 16 bit code well.  They also have the
ability
to execute 32 bit code but the compromises made for backwards
compatibility mean that this is not the optimum solution.

Windows NT/2000 is the opposite.  NT's support for 16 bit software is
quite basic.  However, NT is 32 bit through-and-through (again,
generally speaking - lets not get in to the "old" OS debates) and this
yields better performance when running 32 bit code.

As Microsoft themselves say, if you want to run primarily 16 bit programs
and games then use 95/98/ME.  On the other hand, if you intend to run 32 bit
code in the main then NT should be your choice.

I hope this helps.

Paddy Coleman
Manager, E-Business Support, EMEA
MERANT International.
