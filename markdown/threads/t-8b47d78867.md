Carlos,

You are correct.

It does only show 4 decimals.

But there are very sound reasons for this which you have overlooked. As I
remarked earlier, you are missing something here.

POW-NUMERIC can only be used in two cases:

1. With a ListView (where you want elements of the list to be numeric. In
this case it's use must be subscripted to indicate the level of the list you
are referring to...).
2. With the Fujitsu EXCEL control where you are exchanging information with
EXCEL via OLE.

In both of the above cases this Property (POW-NUMERIC) has an OLE type of
VT_CY.

That means it is intended to be a fixed point currency field.

Fixed point currency fields are ALWAYS pic s9(14)v9(4) in their COBOL
equivalence. So if you use POW-NUMERIC, the system correctly assumes you are
using fixed point currency and shows 4 decimal places.

In other words, POW-NUMERIC has an implicit definition of s9(14)v9(4). You
cannot change it.

The solution is NOT to use POW-NUMERIC.

Bill Klein posted a redefinition solution which is the one I would also have
used.

Consider this...

When you do internal calculations that must be sent to a report you normally
edit them with an edit string, before printing, right?

Exactly the same consideration should apply to your screen output. Treat the
data as a string and you will have no problems.
If you read the article on components I wrote in www.aboutlegacycoding.com
(Components101) this month, it will tell you where to download a component
that does exactly what you need, with input and output strings on screens. I
have put this component in the public domain and it is free. (I have already
had feedback from several people who are using it and they like it.) You can
use it in your Windows Applications, in your Client and server Web
applications and in inter-program communication to verify that strings are
numeric and all the details about how many numbers on each side of the
point, etc. The component has properties that provide the number in about 6
different useful formats, including the standard currency format.
