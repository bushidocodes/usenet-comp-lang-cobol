- **From:** "JerryMouse" <nospam@bisusa.com>

```
1. In all our programs that print, we have:

SELECT PRINT-FILE ASSIGN TO PRTFILE

2. In the COBOL85.CBR file we have:

PRTFILE=DEFAULT.PRT

3. In the DEFAULT.PRT file we have:

PRTDIALG    N
FORMKIND    C
SETPRTDIALOG    3

The goal of the above is to popup the Windows Printer Dialog box to get the
user's choice of a printer.

The above works swell on a development machine. On a non-development machine
(i.e., the application as distributed to a customer), the requested dialog
box does not appear and all printout is directed to the Windows default
printer. We, of course, want the printer selection box available on the
deployed application.

The COBOL85.CBR and DEFAULT.PRT files are installed in the same folder, on
the customer's machine, as the executables.

If we can't get this to fly, we'll have to drop back to invoking the Windows
Common Dialog box manually. The problem with the common-dialog box technique
is that it takes the user's choice for a single job and sets that choice as
the system's default printer thereby confusing subsequent print jobs from
other applications.

We've queried Fujitsu on this issue, via their problem-reporting website,
but what with different time zones 'twixt here and Japan, and assorted other
confusions, it's possible someone here can provide the answer faster.

Thanks ever so much.
```
