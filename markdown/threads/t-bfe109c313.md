The above errors are reasonable... A cursor by definition canot be used for
a singleton select.

Try this:

Declare C1 as the cursor for a query to be used in
positioned updates of the table NAMEADDR. Allow the completed
updates to be committed from time to time without closing the
cursor.

     EXEC SQL
          DECLARE C1 CURSOR WITH HOLD
                FOR
                     (SELECT * FROM NAMEADDR
                     FOR UPDATE OF NAME, ADRESS, OTHERSTUFF)
     END EXEC

OPEN then FETCH the cursor into the host variables.

Modify the HVs then issue your UPDATE. (Assuming it is a row you WANT to
update...)

You can also issue a COMMIT  say, every 500 records, because the WITH HOLD
maintains the cursor position after a dequeue boundary or commit.

This code is NOT specific to the environment you mentioned, but I believe it
SHOULD work.

No harm trying...<G>

Mike

I know I'm biased, but if "It won't work in SQL 7's Query Analyzer
either" why should the Realia preprocessor accept it? If your target
is SQL Server and this is new code, write for the platform. SQL Server
doc says that "with update" is redundant as it is the default.
Secondly, their syntax of the DEFINE CURSOR statement suggests that
this may be the correct syntax. "for update is a modifier of the
declare sursor, not a modifier of the select statement.

exec sql declare c1 cursor with hold for
       (select * from personal) for update of firstname
  end-exec.

Here's the syntax from MS.

DECLARE cursor_name [INSENSITIVE] [SCROLL] CURSOR FOR {select_stmt |
prepared_stmt_name} [FOR { READ ONLY | UPDATE [ OF column_list ] } ]

Also, i'm not suggesting that this syntax would work with the Realia
pre=processor. But will it work with the SQL 7 Query Analyzer? That's
a first step.

connect
declare cursor
open cursor
begin loop
    fetch into local variables
    update using 'where current of' option
end loop
close cursor
connect reset

with results for another hstmt'. After some research, this seems to make
sense, since the cursor is waiting to process every record found in the
