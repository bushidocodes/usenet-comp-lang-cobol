It looks like RM-COBOL is reading the input file as binary sequential (the
default) rather than line sequential. The program will then read a
record's worth of characters (101 in this case) including the the DOS CRLF
line ending. Line sequential organization tells the program to expect a
variable record and to ignore the line ending. Try adding the line
ORGANIZATION LINE SEQUENTIAL
to both input and output file SELECT clauses.
