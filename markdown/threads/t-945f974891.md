```
Hi Group,

I have an application that creates a flat EDI file that must be zipped
before transmission. To make it easier for the user, I want to zip it on the
fly from within the Cobol application. I'm using the MicroShaft 5.0
compiler. A compiler change is not an option. Operating system is XP
Professional

This is the code:

      01  zip-param.
           05  filler                  pic x(17)  value "\anes\data\edi\TH".
           05  zp-date             pic 9(06).
           05  filler                  pic x(05)  value ".zip ".
           05  zp-file-nam       pic x(27).

    accept zp-date              from date.
    move tft-fil-nam             to zp-fil-nam.
    call "PKZIP" using zip-param.

The results are a screen full of garbage, the application terminates to the
MSDOS prompt, and no zip file is created.

I've been programming in Cobol for 25 years, but have never had to call a
utility such as this. What am I doing wrong, or not doing. I want to do a
dynamic call so I don't have to go through the linking BS.

Thanks in advance for any help or advice.

John
```

```
You can't 'CALL' a utility like this, you can only CALL Cobol programs
and library routines.

You must execute a command to run PKZIP as a separate program.

MS Cobol 5.0 is a version of MicroFocus version 3.0 (with the OS/2
option removed).  Assuming you are using it to compile DOS programs
rather than Windows 3.x ones you could use the CALL x'91' function 35
to execute a command.

      01  C-Result        PIC 99 COMP.
      01  C-Function      PIC 99 COMP
      01  C-Parameter.
          03  C-PSize     PIC 99 COMP.
          03  C-PText     PIC X(25).

      DISPLAY "COMMAND /C PKZIP zipfile list of files"  UPON
COMMAND-LINE
      MOVE 35     TO C-Function
      MOVE ZERO   TO C-PSize
      CALL x"91" 
          USING C-Result
                C-Function
                C-Parameter

Other ways are possible too.
```

```
You van use zlib at  http://www.gzip.org/zlib/ 
It is a free c library for creating zip files.
It comes with the source code and the makefiles. There is one for msc.
If you use MS-Cobol, you can link cobol to c libraries. See the
section in the doc. You also have an article br Microsoft :
http://msdn.microsoft.com/archive/default.asp?url=/archive/en-us/dnarvc/html/msdn_cblcapp.asp

- an example of a cobol program creating a zipfile.
- documentation on how to call c from cobol.
```
