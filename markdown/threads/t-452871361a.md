```
All of these products operate in IBM's MVS/ESA and VM/ESA environments.

COBOL/370 (now back level), replaced by and renamed to COBOL FOR MVS AND
VM is a follow-on to COBOL II.

The last version of VS COBOL II is release 4. VS COBOL II does not
contain the intrinsic functions additions to the COBOL language. It does
fully conform to the 1985 standard for COBOL. This product is bundled
with it's own run time, which supports VS COBOL II R4 and earlier product
- at least back to OS/VS COBOL, and older (but not officially for the even
older stuff).

COBOL/370 R1 / R1.1 is a follow on product. It is a full 1985 conforming
product (just like VS COBOL II R3 or later) It had the intrinsic
functions added from a language point of view. From a packaging point of
view, the compiler, debugger, and run-time were split to separate
products. This product requires the LE/370 multi-language run-time
package. LE/370 generally supports older stuff as well, but with some
asteriks and BTWs which keeps me employed in helping folks migrate to that
new technology.

COBOL for MVS and VM is COBOL/370 renamed. It is a full 1985 conforming
product. R2 - available 10/95 - (the first version under the new name)
includes Object Oriented extension. It also operates under LE/370 (and
requires a very recent release of that package).

Bottom line, VS COBOL II R4, COBOL/370 R1 / 1.1, and COBOL for MVS and VM
are not the same. The compilers are of the same family, each succeeding
one implemented more stuff, they came from the same folks, the run time
requirements are not the same. The latest run time - LE/370 R5 will
support object code from all of the compilers.

Hope this eliminates the confusion.
```

```
They are not the same compiler. COBOL II release 4 has been functionally
stabilized by IBM: it will not be enhanced. To get the use of instrinsic
functions (which were added as an amendment to the '85 standard in '89)
you must move to COBOL/370. But this means you must also get LE/370
(Language Environment), which is a run time library that supports multiple
languages (COBOL, C, PL/I, Assembler).

The most recent release of LE is 1.5; it has been renamed to "Language
Environment for MVS & VM". The most recent COBOL compiler from IBM,
COBOL/370, has been renamed as of release 1.2 to "COBOL for MVS & VM" (get
the pattern?).

LE 1.5 added run time support for Fortran. Release 1.2 of COBOL added
Object Oriented COBOL constructs, as well as recursive programs,
local-storage, and several other nifty features.

IBM seems to be doing a terrible job of telling this story. LE has a lot
of nice facilities, including tools to help solve the Year 2000 problem,
dynamic storage allocation / deallocation from all languages (including
COBOL!), and so on. I find few customers know about these products, and
therefore few licenses have been sold.

In March, when IBM releases OS/390 (a package including MVS and a couple
of dozen products in a single offering), LE will be automatically
included, I understand.


Neat stuff available, but it's hard to keep current on everything these
days.

Hope this helps.
```
