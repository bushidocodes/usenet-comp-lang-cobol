Be sure that you have the main program and all subroutines in the
calling chain compiled with the same options. Specifically,
DYNAM and RES. Mixing DYNAM/NODYAM and RES/NORES load modules in
the same run unit will cause these kind of abends. To be sure,
recompile EVERY program in the calling chain and try again.

Good Luck.

One additional approach that has a good probablility of finding the
problem.

Recompile all modules of the program (might fix any res/nores issues as
well), with SSRANGE specified at compile time. Execute with SSRANGE
enabled in the runtime.

As the other poster stated, make sure run time components are compatable
with the compiled code - run-time must be at a level equal to or later
than compikled code.

One other thing to watch for - forced AMODE / RMODE at linkage edit time.
Two kinds of folks force these options - those who are very clever and
completely understand what they are doing, and those who have no clue as
to what they are doing.
