As I do it in N/E using OO :-

WORKING STORAGE SECTION.
01 ws-Index                        pic x(4) comp-5.
01 os-List                         object reference.
01 os-OldString                    object reference.

*>---------------------------------------------------------
Method-id. "CheckObjects".          *> Callback   PRIVATE
*>---------------------------------------------------------

*> Using method "KeysAndDataDo", the index and element are
*> automatically passed to this method

01 ls-Bool                      pic 99 comp-5.
   88 IsTrue                    value 1.
   88 isFalse                   value 0.

Linkage section.
01 lnk-Index                    pic x(4) comp-5.
01 lnk-Element                  object reference.

Procedure Division using lnk-Index, lnk-Element.

   invoke os-OldString "equal"
          using lnk-Element returning ls-Bool

   if IsTrue
      move lnk-Index to ws-Index
      invoke os-List "quitIteration"
   End-if

End Method "CheckObjects".
*>---------------------------------------------------------

Note I'm using Pic 9(02) comp-5 because that's what is shown as the
returning parameter for this M/F support method. Check the on-line help
for "Bool" and "Boolean".

Never tried it, but if you were CALLING,  presumably you could :-

01 ReturnCode			pic x(4) comp-5.
   88 IsTrue                    value 1.
   88 isFalse                   value 0.

Call MyProgram using xxxx, returning ReturnCode
*> MyProgram sets the Return code to zero or one.

if isTrue
	..........

Depends on the version of NetExpress but this is what's in the current
COBOL standard:

05  BoolVar Pic 1.

You can use Usage Bit if you want- resulting in 7 slack bits in this
particular case, but for just plain Boolean I don't think it will
matter to you.
