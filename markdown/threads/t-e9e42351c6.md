Hmmmm Gloria;
   Something is going on here, lots of our customers run a peer to peer
network system and performance is rarely the big issue. We don't have anyone
using AcuServer, as that is only when you want to split the data across
several machines, (ie: Accounting on the IBM Netfinity, Manufacturing on
the Compaq Server etc.)  Having said all that I would encourage you to
move to a true client server system.  Put the Vision Files up on the
NT/2k box and install Terminal Server if you want to be a Micro$oft
customer. In the peer to peer environment someone will lock the record
for the rest of the users if they are sitting on it, (and go to lunch for
instance.) The real problem comes when the users are impatient and excute
the dreaded three finger salute (cntrl/alt/delete) while running the
application.  Now the Vision file system is likely to have a transaction
inflight and what do you think happens to the file integrity?
   My own first choice is to put the files on a Linux server (the old NT
box perhaps,)  install AcuThin and keep the Windows95 clients thin and
clean.  Another solution is to get a true networking system from Novell
and let that manage the Vision file access.  Third choice is to use
Windows NT/2k with Terminal server, Citrix, Tarantella etc.etc. 
   Your English is a lot better than my Italian :-}  
                                    Ciao
                                    Doug Robinson
