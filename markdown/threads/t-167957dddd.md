sequence. If you are using a really weird collating sequence, you may be

I'm making a guess and some assumptions here.

You mentioned Oracle.  This makes me think you have NULL values in the
table that you are trying to handle.  These are NOT zeros, or low
values.  This is a BIG SECRET, but I'll let you in on it because it
took me a while to grasp it.

Let's say you have a table and the element is called "Element".

If you do:

Select Element from Table where Element < 1.

It won't find it.

Say you try:

Select Element from table where element not = 1

That doens't return it either even though it's not equal to 1!

Null's are SPECIAL.  

They have NO value not LOW value.

How about:

Select Element from table where element is null;

that returns the record.


So, let's say you want to "assume" a value for the element when it's
not there, or null

Select nvl(element,0) from table where nvl(element,0) = 0;

will return all the "null" or "zero" elements in the table.

Does that help clear up the mystery?
