Dave?,

The short and sweet answer is that Object Oriented programming is a
style, and not a language. In my opinion the object oriented style of
programming helps tremendously in writing good, RE-USABLE code. Most
good programmers, even if not using an OO language, probably already
use OO techniques as they lend themselves to good programming
practices.

C++ was developed to make it easy to write OO programs, as it has
built into it the mechanics necessary to write in this style. I have
never tried any OO Cobol, but I suspect it was written to ease the
transition of a Cobol programmer to the reallm of OO languages.

If you dont need the Cobol side of it, I would just go with straight
C++, or even Java, depending on what platform you need to write for.

Hope this helps,

Paul
