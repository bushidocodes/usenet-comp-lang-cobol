HeyBub wrote:
> I have to modify a program written in PowerCOBOL to accept Cyrillic

Having accepted them, presumably you must then process them? This sounds 
interesting...:-)

>
> The selected font is Tahoma which admits of Cyrillic letters.
>
> Any up-front tips before I start digging?

Yes.

1. Change the font property on Text Box controls which will be accepting 
Cyrillic, to be Tahoma.

You said "accept Cyrillic" but if you need to display Cyrillic as well...

2.  For displayed Message boxes ensure that there are no hard coded literal 
displays. (Always display a field in memory that has been pre-processed to 
contain Cyrillic)

It will be difficult to manipulate Cyrillic characters internally unless you 
switch to a Cyrillic code page. If you have mixed input where it is 
sometimes Cyrillic and sometimes something else, you may need to detect what 
was entered and switch code pages accordingly. This sounds challenging to 
me.

Let us know how you solve it.

I take it there is no available function that can "translate" between 
Cyrillic and normal text?

Pete.

Turns out it was the program that was the problem, not the font.

I suppose one can be TOO careful...

After accepting input data the program checked each character for being in 
the range " " to "~", thereby preventing the user (or the user's cat) from 
inputting invalid characters.

viz.:

    IF INPUT-STRING(I:1) < " " OR > "~"
        MOVE SPACE TO INPUT-STRING(I:1)

Actually, the test is still there unless the installation specifies they 
will be using a non-western font. If so, the user will just have to be 
careful and consign the felines to the hall.

The purpose of this drill is to allow a new customer some degree of local 
compatibility. The new customer is in Azerbaijan (!).

Thanks for thinking about the issue.
