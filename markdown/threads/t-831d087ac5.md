Jimmy,
Yes, "self" is the object currently being invoked. The equivalent of 
"this" in C++ or Java. "super" refers to the same object, but its use has 
a different meaning.

If you invoke a method on "self" it will invoke a method within the same 
object, either implemented by the class of the object or the 'closest' 
superclass of that class, following standard inheritence rules.

If you invoke a method on "super" it will invoke a method within the same 
object, but only in the 'closest' superclass of the current class 
(which may be a superclass of the class of the object) and not in the 
current class or its subclasses.

"super" is usually used with a method to invoke the superclass' overridden 
method of the same name. C++ and Java use the :: operator and "super" 
respectively.

"self" and "super" obviously have no meaning when not executing a method.
