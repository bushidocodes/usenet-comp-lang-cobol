I think our new mf2fsc converter will be able to help.
We have a Fujitsu Preprocessor mode in it.
When you run converter in this mode, 
it takes MF input and produces FSC output with full inlining.
Also the output file has #FILE and #LINE directives,
so both Fujitsu compiler and Fujitsu debugger 
will work in terms of lines of original MF file.

In fact, it creates an illusion that Fujitsu compiler
parses and executes MF Cobol program "directly".

Fairly soon you will be able to say something
like "cobol32 /mf file.cbl" and you will have your MF
application running under FSC, whille you can continue
maintaining it as an MF applcation.
