```
I am using MicroFocus COBOL on UNIX.

mf using an environment variable, COBPATH, to set the directory
path to the programs to be executed.  I have thousands of programs
in many directories so the COBPATH method does not lend itself
to my situation. i.e. my COBPATH will be huge and I will have
a correspondingly large search time to find programs.

I _do_ have a technique to determine the exact location of the
program based on the name of the program.

Does anyone know of a technique in mf COBOL that will allow me
to directly point to the program to be executed rather than
use COBPATH.

here is the original code:
-------------------------------------------
              CALL "program".
-------------------------------------------

here are the methods I have considered:

1) -------------------------------------------
              CALL "dir/subdir/program".

    I would rather not use that method as the programmer will
    have to hard code the dir and subdir.  Plus this is not
    easy when the program name is a variable.
-------------------------------------------

2) -------------------------------------------
              CALL MYCALL "program".

    Using my own program to resolve the name and in turn call the
    correct program will not work when variables are passed to program.
-------------------------------------------

3) -------------------------------------------
   COBPATH=dir

              CALL "subdir/program".

    Has anyone gotten this to work?  I only get program not found.
    plus this will not work when the program name is a variable.
-------------------------------------------

4) -------------------------------------------
          01 NewProgName PIC X(80).


              CALL MYPROGCONVERTED NewProgName "program".
              CALL NewProgName.

      Actually, i have not tried this yet.  please comment.
-------------------------------------------

allow me to add one more twist:
anything i do to change the program has to be via a preprocessor.
the original program runs on MVS; for ease of maintenance I
do not want to change the program by hand.
```

```
Method nbr 4 is the way to go.
You resolve the program name in a central utility
program according to whatever rules you want to use.
You might want to have a dummy call of the program inside the central
utility to be sure hte linker links in the program:

move "prog1" to pgm-to-call
call "myconvpgm" using pgm-to-call
call pgm-to-call

------------

myconvpgm:
if pgm-to-call = "prog1"
    move "c:\dir1\dir2\prog1" to pgm-to-call
else
if pgm-to-call = "prog7"
    move "d:\mydir1\prog7" to pgm-to-call
else
...

move pgm-to-call to dummy
if dummy = "1"
    call "c:\dir1\dir2\prog1"
else
if dummy = "2"
    call "d:\mydir1\prog7"
else
....
```
