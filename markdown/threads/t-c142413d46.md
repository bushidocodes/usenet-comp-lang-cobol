Yes, you can "write a program" to create an INDEX FILE.  HOWEVER, many
operating systems and/or COBOL systems have "utilities" to do this.  For
example, on IBM mainframes, you use IDCAMS.  Both MERANT (Micro Focus) and
Fujitsu have options to do this on the PC.

What is your OS and COBOL system?

Depends on the hardware and specific COBOL compiler.

For example, in IBM COBOL/390 you can create an indexed file which you can
read from a COBOL program using the IDCAMS utility to define an indexed VSAM
file (called a KSDS) and then use the REPRO function of IDCAMS to copy a
sequential file into the keyed file with the utility handling the "key
building."

I don't know if there is a similar "utility" approach for other compilers on
other platforms.

   ACCESS IS DYNAMIC...

(read sequential file - write ISAM file)
CLOSE ISAM-FILE.

(misc processing)
CLOSE ISAM-FILE.
DELETE FILE ISAM-FILE.
STOP RUN.
