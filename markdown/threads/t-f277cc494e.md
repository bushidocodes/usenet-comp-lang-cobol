You have me a little confused.  The term "extended file status code" in Micro 
Focus is what they use for the case where the FIRST byte of a two-byte status 
field is "9" and the 2nd byte is a COMP-X field.

How have you defined your status field?  If defined as Pic 99 or Pic XX (usage 
display) what is in the first byte and what is in the 2nd byte.  If you have 
defined them some other way, exactly how are they defined.

In GENERAL,  if you get a file status value of "2x" (two-bytes usage display), 
then the operation is UNSUCCESSFUL and I would not expect any actual I/O to 
occur.
