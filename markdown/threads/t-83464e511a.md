To give you something, I suppose you've Micro Focus.
Two possibilities: 1) Single-entry-point DLL and 2) multiple entry-point
DLL.

1)
DLL:
Identification division.
Program-id. Single.
Environment division. ...
Data division. ...
Procedure division [using ...].
    ...
    Exit program.
end program single.

Calling the DLL:

...
    CALL "SINGLE" [using ...]

2)
DLL:
Identification division.
Program-id. Multiple.
Environment division. ...
Data division. ...
Working-storage section.
... global data, available from within all DLL-entries
Procedure division [using ...].
Main-para.
    Display "this code is reachable by call as in single"
    Exit program.

    ENTRY "Multi1" [using ...]
    ...
    Exit program.

    ENTRY "Multi2" [using ...]
    ...
    Exit program.

    ENTRY "Multi3 etcetera" [using ...]
    ...
    Exit program.
end program Multiple.

Calling the DLL:

Procedure division.
    ...
    Set pp to entry "Multiple" *> just once needed,
            this makes the DLL addressable, I think that a "LoadLibrary"
win-API call is done behind the scenes.
    ...
    CALL "MULTIn" [using ...] *> as many as you like

I think the question is relating to the command
required to *create* a DLL from a COBOL source file,
rather than a CICS LINK statement or similar.

> > What is the command?
> > Or short example how to do that?

Again, assuming a reasonably up-to-date Micro Focus system,
the simplest form is:

cbllink -D source.cbl

There are additional options for being more specific
about which RTS the DLL should use etc.
