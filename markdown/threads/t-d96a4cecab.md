### Cobol Data Readers ver 3-8-1: Announce

- **From:** Vadim Maslov <vadim-cbl@siber.com>

- **Newsgroups:** comp.lang.cobol,alt.cobol
- **Message-ID:** `<39996F01.FED21CF2@siber.com>`

```
Siber Systems Inc is pleased to announce 
release 3-8-1 of our Cobol Data Reader technology. 
Using this technology, you or your C++/C application 
can read Cobol data files directly, without writing 
a Cobol program to do it or using expensive ODBC drivers.

For more info look at http://www.siber.com/sct/datafile/

In this release:
* Added readers for Acu Cobol Vision 3 and Vision 4 file systems.
* Added reader for compressed Micro Focus Cobol data files.
  Fixed a number of bugs in Micro Focus data file reader.
* Improve Crystal Reports DLL conflict resolution,
  that prevented Data2Cr from working with certain CR versions.
* Considerably improved intellect of Data-Guess program.


Overview of Data Reader technology and its tools:

* DataAccess library is at the core.

  It reads index and relative Cobol data files from 
  these vendors: Micro Focus, Fujitsu, Acu and Ryan-McFarland.
  We can implement data reader for any Cobol data file format.
  Also DataAccess library reads any sequential Cobol data file.

  Then DataAccess converts data to standard formats used in
  C/C++/Java applications. It also provides extensive browsing 
  capabilities both at the record fields and at the file 
  records level.

  DataAcess library takes file description from File Descriptor 
  (FDD) and Record Descriptor (RDD) text files. These files 
  can be generated by Cbl2Fdd program, Data-Guess service or 
  you can build them manually.


* Cbl2Fdd utility generates FDD and RDD files for all data files
  found in a given Cobol program.


* Data-Guess service allows you to build FDD and RDD files 
  for a data file even if you do NOT have the source Cobol 
  program that can read or write this data file.

  Just imagine: you have an important 3rd party business application 
  written in Cobol and you need to export some data generated by 
  this application into a new application that you are writing.
  Or you simply need to rewrite this application in a modern
  language. Or it is your own application but you lost its source.

  How can you extract the data from cryptic Cobol data files?
  Until recently the answer was simply: you cannot.

  But now you can use Data-Guess service by Siber Systems.
  We would build FDD and RDD files for your Cobol data file
  even if you do not have the Cobol program that wrote these files.
  And we will find a best way to integrate the legacy Cobol data
  with you new applications.


* Data Converters: 
  Data2Flat converts Cobol data files to flat CSV text format.
  Data2Dbf converts Cobol data files to dBase-IV DBF format.


* Crystal Reports Data Reader:
  This is a small DLL that allows you to use Cobol data files
  directly in Crystal Reports (the most popular report generator).


Best Regards,
Vadim Maslov
```
