```
We want to have a Visual Age cobol program call a Micro Focus cobol
DLL.  When the Visual Age cobol program is compiled and linked, it
wants to call the entry point in the Micro Focus program with an
underscore on the front (eg. CALL MFPROG in the Visual Age program
looks for "_MFPROG" in the Micro Focus DLL).  So we're trying to
compile and link the Micro Focus program so the DLL has an underscore
in the entry point.  We thought we could modify the EXPORTS statement
in the DEF file (used into the Micro Focus link step) to add the
underscore; but we can't figure out how to feed this new DEF file into
just the link step - when we try to only link, it compiles and links
and recreates a DEF file with our change wiped out.  In the Micro
Focus source code we've tried different combinations of
CALL-CONVENTIONS (74 MOSTLY).  Does anyone have ideas or a sample that
would help?  Thanks.
```

```
are also issues with the MAPNAME directive, but it has been too long for me
to be certain what setting would be best for that.)
```

```
you are using the NetExpress development environment go into Project ->
Build Settings -> Link -> Advanced and add the .DEF file to the additional
CBLLINK directives and it should pick up your own .DEF file okay.
```
