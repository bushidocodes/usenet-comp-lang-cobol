not quite but almost

COMP-3 is piggy backed binary coded decimal plus a 4 bit sign. the
sign is either F C or D. F=unsigned (but treated as positive) C = +
and D = -

now for the piggy-back all numbers have an odd number of digits
with the addition of sign yielding an even number of half bytes.

so +123 is 2 bytes long byte 1 is 12 (0001/0020) and byte 2 is
3+ (3C or 0011/1100) c=binary 1100 d=1101 and f=ffff

if a picture has an even number of digits pic 9(4) comp-3 then the
number will be packed with a leading 0 (0000).

ok. 1234+ (0000/0001 0010/0011 0100/1100)
0 1 2 3 4 +
123- (0001/0010 0011/1101)
1 2 3 -

now just to confuse things a few manufacturers did provide for comp-3
with LEADING SIGN. so the nybble is moved to the front.

In reply to question on COBOL Packed fields:

COBOL has several types of Data Definitions.

PIC X(##)............Character.........Printable
PIC 9(##)............Zoned Decimal.....Printable
PIC S9(##) COMP-3.....Packed (signed)...Viewable in hex
PIC S9(##) COMP.......Binary (signed)...Viewable in hex

Signs '+' => FaCe '-' => BaD


PIC ?(##).....## is the number of digits

Calc # of bytes:

PIC X & PIC 9 (Char/Zoned Dec)
# of bytes = # of digits

PIC 9 COMP-3 (Packed)
# of bytes = (# of digits + 1) / 2
Note: Packed fields should always be coded with an odd number of digits.
This avoids problems with slack bits.


Samples:
PIC X(03) Value '123' .......F1 F2 F3 (3 bytes)
PIC X(03) Value 'ABC' .......C1 C2 C3

PIC 9(03) Value 123 .........F1 F2 F3 (3 bytes)
PIC S9(03) Value +123.........F1 F2 C3 (signed) prints as 12C

PIC S9(03) COMP-3 Value +123..12 3C (2 bytes)
PIC 9(03) COMP-3 Value 123...12 3F

PIC S9(04) COMP Value +123....00 7B (2 bytes - Halfword)
PIC S9(09) COMP Value +123....00 00 00 7B (4 bytes - Fullword)
Note: a good practice is to use 4 and 9 digits for binary. This insures
the maximum usage and prevents errors when exceeding the halfword and
fullword boundary limits.


To convert Packed Fields to a readible format on a report or in a file,
just MOVE w-packed-filed TO w-print-field

.05 w-packed-field PIC S9(03) COMP-3...
.05 w-print-field PIC 9(03)..........
or
.05 w-print-field PIC S9(03)..........

Note: If on a report, don't use signed. +123 becomes 12C.
