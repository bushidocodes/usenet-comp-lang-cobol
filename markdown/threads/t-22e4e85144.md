COBOL isn't known for it's treatment of pointers.  There is no way to
dereference a pointer and fields with a USAGE POINTER are rarely used
outside of calling a sub program.  FWIW, parameters are passed by reference
when calling sub programs unless explicitly specified otherwise.

The (PCHAR) &iValToSet passes the address of the variable iValToSet. Using
only &iValToSet would accomplish the same thing, but the called function
expects the adress to point to a character. So, the (PCHAR) casts the
integer address to a character address.

As fo converting to COBOL, all you ought to need is a COBOL S9(9) COMP
variable [or whatever has the same length as a C integer on your system;
i.e. whatever would have a size equal to the word-size of your machine).

When your COBOL program calls the sub-routine and passes the COBOL variable,
the COBOL program will pass the address (by default).

Actually, the bigger problem would be passing items that the C-program
expects to be passed "by value".

What I have done in some cases is to write a layer of C functions as a
"wrapper" over the C-functions that I want to call.

- Darius Cooper
