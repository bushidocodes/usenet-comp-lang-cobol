```
> How do you handle Client access to a Server database?

You will need to ask a clearer question.  What exactly do you mean by
'client access to server database' in the context of CGI programs ?

In this case the 'client' is an HTML web browser and the 'server' is a
web server that activates the Cobol program that gets the data using
ACCEPTs and sends back the results using DISPLAYs.  The Cobol program
access whatever data files are necessary for the reply.

> Is that done all in COBOL?
> Or does the COBOL code make calls on non-COBOL routines?

That depends on the environment, on Windows I use Xitami server as it
does real CGI and it is just ACCEPT and DISPLAY.  On Linux under Apache
the code is the same, under OS/2 with Xitami I do a CALL "DosRead" for
posted data.  For some Windows Web servers I would probably need to use
WinCGI in which the data is passed in files, so it is only necessary to
open and read a text file.  This is sorted out in one of the Cobol
modules.

For 'Long running' or persistant CGI I would need to call the
appropriate library (exactly as any C, C++, Pascal or other program
would).

For thin client where the Windows panels are exported to the client so
that it runs as if it were a local application, but the code and data
access are on the server, then I can use SP2 thin client with no changes
to the normal SP2 program.
```
