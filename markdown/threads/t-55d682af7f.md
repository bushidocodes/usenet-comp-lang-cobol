The advantage of using putenv and getenv is that they are the DOCUMENTED (in
the COBOL) way of doing dynamic allocation. If you do a search in the
current IBM COBOL documentation for "dynamic file allocation" - this is what
you will find.

Down-level is anything *below* IBM COBOL for OS/390 & VM V2R2 (where this
capability was introduced).
