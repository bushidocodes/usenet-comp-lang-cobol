A while back I had posted about strange happenings using FUNCTION NUMVAL. I was
asked for some code and didn't get to it until last Friday (the usual panics at
month-end and then other stuff came up).

While merrily cutting and pasting from the Xpediter session I finally saw what
was actually happening instead of what I thought was there.

The caculation was taking place after the WRITE statement (this was a CICS
program for those that don't remember).  The WRITE torqued over the first 8
bytes of the record which of course included one of the fields I was using in
the calculation.  Moved the calc to before the WRITE and all is well.

Program went production today :).

Thanks for making me take another good look!

Eileen
