The problem has been resolved & I want to post the resolution.

Now, I can use the MFCobol Precompiler by using a DIRectives file.
The directives file contains the following.
   use"wbmf.dir"
   sql"ibm"
   sqlcommit"2"
   nosqlinit
   sqldb"db"
   nosqlbind
   sqlaccess""
   sqlpass"uid.pwd"
   sqldb2
   omf(obj)

I can compile the program by using
    COBOL x.cbl use"x.dir"

I do not need to wrap all the host variables in between
   BEGIN DECLARE SECTION &
   END DECLARE SECTION.

Then, I can use CBLLINK to create a DLL that is copied to
SQLLIB\FUNCTION
