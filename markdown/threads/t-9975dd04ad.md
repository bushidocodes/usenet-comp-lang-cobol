Eddie,

If the DB2 you speak of is for OS/390 then, DB2 has only two types of 
KEYS.  Primary and Foreign.  It allows an index on any column 
combination.  But, if it is defined with a Primary key, it must be 
Unique.  The purpose is to prevent duplicates of the Prime key (SQLCODE -
803).  You can check in the DB2 catalog to determine if your table has 
such a prime key.  SELECT * FROM SYSIBM.SYSTABLES WHERE NAME = 'table-
name';  Then check, the KEYCOLUMNS column.  If it is greater than zero, 
you have that many columns in that table's primary KEY.  Then names of the 
column, that make up this Primary key are found in SYSIBM.SYSCOLUMNS under 
the KEYSEQ column; which identifies the order of the columns in the 
PRimary Key, then see the column called NAME for the name of the column.
