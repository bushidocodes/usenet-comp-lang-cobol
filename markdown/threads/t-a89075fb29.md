I'd try and stay current on both.  But, if you have to choose, a lot
depends on what kind of programming you wish to do.

Both languages are very good and radically different.  Which means,
their particular strengths should be used for completely different
systems.

If you like business applications, go with COBOL.  If you prefer
utilities, architure and support systems featuring, go with C++.

>language?

(not counting copies).  That might give you an idea how much business
application software demand there is.  What do you think?
