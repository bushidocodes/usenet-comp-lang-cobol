Bob, to pass a pointer to a structure you define the structure as normal eg

01 NetResource.
    03 dwScope         DWORD.
    ...

then pass the item BY REFERENCE when making the API call. Note that BY
REFERENCE is the default anyway.

>It refers to a pointer to a structure.  Also the
>dwFlags refers to a set of bit flags.  Is this just a working storage item
>defined as DWORD ?

Yes, dwFlags is defined as DWORD. You then simply set the appropriate bit by
moving the value to the data item.

Bob,

Here is a quick a dirty sample application using NetExpress and
the win32api "WNetAddConnection".  My version of NetExpress did
not contain a LIB file containing WNetAddConnection, but my copy
of VC 5.0 did so it was used instead.

To compile:
        cbllink netcon.cbl wsock32.lib mpr.lib

I hope this helps.

Dave Maxwell


*----------------------------------------------------------------
      $set ans85 case
       identification division.
           program-id. netcon.

       environment division.

       configuration section.

       special-names.
           call-convention 74 is win32api.

      * -------------------------------------------------------------- *
       working-storage section.
       78  ERROR-ACCESS-DENIED             value 5.
       78  ERROR-BAD-DEV-TYPE              value 66.
       78  ERROR-BAD-NET-NAME              value 67.
       78  ERROR-ALREADY-ASSIGNED          value 85.
       78  ERROR-INVALID-PASSWORD          value 86.
       78  ERROR-BAD-DEVICE                value 1200.
       78  ERROR-DEVICE-ALREADY-REMEMBERED value 1202.
       78  ERROR-NO-NET-OR-BAD-PATH        value 1203.
       78  ERROR-CANNOT-OPEN-PROFILE       value 1205.
       78  ERROR-BAD-PROFILE               value 1206.
       78  ERROR-EXTENDED-ERROR            value 1208.
       78  ERROR-NO-NETWORK                value 1222.
      *
       01  WS-Return-Code                  pic x(4) comp-5.
       01  ErrMsg                          pic x(30).
      *
       01  Connection-Stuff.
           03  lpRemoteName     pic x(20) value
                                 "\\bigcat\windows 95" & x"00".
           03  lpPassword       pic x(01) value x"00".
           03  lpLocalName      pic x(03) value "r:" & x"00".

      * -------------------------------------------------------------- *
       procedure division.
       MyMain section.
      *
      * -- Clear screen
      *
           display "COBOL NetAddConnection Application " at 0101
                 with blank screen
                 background-color 0 foreground-color 7
      *
      * -- test the WNetAddConnection win32api
      *
           call win32api "WNetAddConnectionA" using
                   by reference lpRemoteName
                   by reference lpPassword
                   by reference lpLocalName
                   returning WS-Return-Code
           end-call

           if WS-Return-Code <> zero
              perform Get-API-Error
              display "WNetAddConnection Error: " at 0201
              display ErrMsg
              stop run
            else
              display "WNetAddConnection was Successful " at 0201
              stop run
            end-if
            .

      * -------------------------------------------------------------- *
       Get-API-Error SECTION.
           call win32api "WSAGetLastError"
                   returning WS-Return-Code
           end-call
           perform Get-Error-Msg
           .

      * -------------------------------------------------------------- *
       Get-Error-Msg SECTION.
           evaluate TRUE
             when WS-Return-Code = ERROR-ACCESS-DENIED
                  move "Access Denied"                   to ErrMsg
             when WS-Return-Code = ERROR-BAD-DEV-TYPE
                  move "Bad Device Type"                 to ErrMsg
             when WS-Return-Code = ERROR-BAD-NET-NAME
                  move "Bad Net Name"                    to ErrMsg
             when WS-Return-Code = ERROR-ALREADY-ASSIGNED
                  move "Already Assigned"                to ErrMsg
             when WS-Return-Code = ERROR-INVALID-PASSWORD
                  move "Invalid Password"                to ErrMsg
             when WS-Return-Code = ERROR-BAD-DEVICE
                  move "Bad Device"                      to ErrMsg
             when WS-Return-Code = ERROR-DEVICE-ALREADY-REMEMBERED
                  move "Device already Remembered"       to ErrMsg
             when WS-Return-Code = ERROR-NO-NET-OR-BAD-PATH
                  move "No Net or Bad Path"              to ErrMsg
             when WS-Return-Code = ERROR-CANNOT-OPEN-PROFILE
                  move "Cannot Open Profile"             to ErrMsg
             when WS-Return-Code = ERROR-BAD-PROFILE
                  move "Bad Profile"                     to ErrMsg
             when WS-Return-Code = ERROR-EXTENDED-ERROR
                  move "Extended Error"                  to ErrMsg
             when WS-Return-Code = ERROR-NO-NETWORK
                  move "No Network"                      to ErrMsg
           end-evaluate
           .

I am not a net express user, but I notice that no one else has been
able to help so maybe a couple of suggestions ...

A 173 error is a dynamic load error, I think you need to force
those to static links.  I am not sure how to do that with a given
name, the norm is that an underscore as first character forces
it.  But try it with a litlink option and see if you get a linker error.

The other thing that I have found with Cobol and API's is that
Cobol is not case sensitive, whereas the API names are. With
Fujitsu you also need a special compiler directive to force the
names to retain the case, otherwise it converts them to capitals
and you cannot find the API.
