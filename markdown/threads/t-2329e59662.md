I just read an article that was interesting for two reasons. The first
is the source, the June 1996 issue of "Object" magazine (pg. 46-50). It
seems that OOCOBOL is beginning to be taken seriously. The gist of the
article "When opposites attract" is that a highly productive programming
environment can be created by using Visual Basic for the Graphical User
Interface and capturing the business model using Micro Focus Visual
Object COBOL. The two languages can be seamlessly integrated using DLLs.

The second point of interest was an undocumented statement in the
article that as of December 1995, 65% of !!NEW!! code development
projects were using COBOL (primarily procedural I would assume).
