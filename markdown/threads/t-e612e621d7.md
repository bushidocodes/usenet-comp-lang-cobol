COBOL is best for file I/O, printing reports, and possibly in accounting and
data manipulation.  It is really best suited for large scale, high speed
computer systems.  It is also used for updating and accessing Database
storage.  I saw a class recently at a university for databse design with
Oracle and COBOL being used together.

The best thing about COBOL is it is less complex, easier to use, and simpler
to learn.  Since it is used by so many businesses all over the world, it is
kept to a standard.  If the programmer avoids the Compiler specific add-ons
it can be quite portable and cross-platform capable. This language tends to
be more common in the Mainframe world.  It was originally developed for large
businesses using mainframe computers.

COBOL was designed to read like English for ease of understanding, and
to allow continuing maintenance of installed programs.  Imagine
reading a C program aloud to someone over the phone, compared to
reading a COBOL program aloud.

While COBOL is best suited to business accounting problems I have
heard of creative uses.  I remember reading about 25 years ago an
article by a hobbyist who needed a cross-assembler to build object
code for a microprocessor (it may have been a Motorola 6800 or a
Mostek 6502).  He wrote the cross-assembler in COBOL on a mainframe,
and it generated binary code to run on the target microprocessor. 
That would have been an interesting COBOL program!

The story of Realia COBOL compiler is similar.

The compiler was written in COBOL in the early 1980s. It was initially
compiled on a small IBM mainframe machine and emitted Intel compatible code
that was downloaded to a PC. (8088 in those days). It then was used to boot
strap itself by recompiling itself on the PC.

all further development was on the PC.

We still have the compiler written essentially all in COBOL and do
regression tests on the compiler by compiling the compiler and comparing the
results.

Later the Realia CICS emulator was also implemented in COBOL.

The COBOL run-time however was written in assembler and more recently C.
