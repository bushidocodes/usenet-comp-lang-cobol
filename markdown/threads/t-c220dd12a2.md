Mr. Bdoctor:

It appears that you are comparing development tools alphabetically.
Is the next one Fujitsu COBOL and Foxpro?

My personal opinion is that you are comparing apples and oranges...or
perhaps a better way to put it is that you are comparing a small
private airplane (Access) with a Boeing 747 (COBOL). The private
plane is really nice when you only have to carry one or two people
short distances, but when you need to move a lot of people halfway
around the world, you better use the 747.

Microsoft Access is a proprietary database management system designed
to run under Microsoft Windows environments...ONLY. When using it to
develop applications which must process massive amounts of data, it
crumbles miserably....but the screens look very pretty!

Microsoft is the ONLY company to offer Access and makes ALL decisions
on how the product should work. This means that if the programmer
doesn't agree with changes Microsoft makes in a future version,
chances are, the entire program will have to be re-written from
scratch.

Historically, Microsoft has dramatically changed the way their
products work from one version to the next, which means that some
modification may be necessary in order to keep your Access application
up and running.

Access has been available for a few years now and is good for quickly
writing small applications that a larger business would not rely on
for storing critical company data.

------------------------------------------------------------------------

Acucobol is a specific brand of compiler for a programming language
called COBOL (COmmon Business Oriented Language). It runs in a ton of
operating systems. When using Acucobol (or any of the COBOL compilers
for that matter) to develop applications which must process massive
amounts of data, it performs magnificently...and there are many tools
available to make the screens look pretty too!

Acucobol is only ONE vendor of MANY who produce COBOL compilers.
Decisions on the COBOL language standards are made by an independent
standards committee. Many people from many different organizations
meet and decide on how the language is supposed to work. This means
that if the programmer doesn't agree with changes one COBOL compiler
vendor makes in a future version, chances are, the programmer can use
a compiler from a different vendor and none of the program will have
to be re-written providing that the programmer adheres to the ANSI
standard.

Historically, the COBOL language has remained quite stabile so that
from one version to the next, modifications are almost NEVER necessary
in order to keep your COBOL application up and running.

COBOL is a proven programming language which has been available for a
few decades now and is good for writing any size application that any
size business can rely on for storing critical company data.

Hope this helps.
