```
These are all the methods I am aware of to achieve a GUI using COBOL -
some are compiler specific.

AcuCOBOL GT
Fujitsu PowerCOBOL
MicroFocus Dialog System
IBM VisualAge COBOL
Objective COBOL
COBOL WOW
Mpact CSD
ETK
Casegen
COBOL SP2
Windows API calls
Visual Basic or other Language front end calling COBOL

There may be more - if I missed any I'm sorry, but post about it so I
can add it to my list.

My personal favorite is COBOL Sp2.  Why?

Compiler independent, simple call interface.  FAST execution time,
royalty free application distribution, Thin client cabable with
minimal application modifications.

Of the two you are asking about, CASEGEN is closer to the Windows API
than Fujitsu Power COBOL, so it should be faster, however screen
layout and design is easier with Power COBOL.
```
