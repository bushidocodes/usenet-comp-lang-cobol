The SEARCH  [ALL] verb can only be used on an array contained in memory -
either in WORKING-STORAGE, LINKAGE or FILE SECTION (after the successful
retrieval of a record). In addition,  the array (or 'table') needs to be
have an index defined to it - OCCURS n TIMES  INDEXED ndx.

If you have a sequential file, you will need to read the file sequentially
until you've found what you are looking for. If it is a keyed file, and you
know the key of the information, a random read will do the trick.
Seq:
    READ anyfile
        END
            ...
        NOT END
            IF i-found-it
                ....
            END-IF
    END-READ

Random:
     READ anyfile
        INVALID
*        not found               ...
        NOT INVALID
*         i-found-it
       END-READ

Regards,
John

The SEARCH statement is for searching tables, not files.  There are two
basic ways to search a file for some value.

The most efficient way is when you are looking for a simple equality or
range, and the file is an indexed (or database) file which has a key by
the desired field.  Then it is simply a matter of reading the file by the
key using the desired value.  If you are looking for a range of values,
then read the first record where the key is not < lowest value, then loop
reading the next records until you encounter the end of file or a record
whose key value is > highest desired value.

Much less efficient, but works on any file for any kind of search criteria,
you can read the entire file and examine each record, selecting the records
that match your criteria.

A huge number of methods exist for finding records meeting certain criteria,
but they will generally fit into one of the above types, or are designed for
some specialized purpose.  There isn't enough room here to list them all
(even if I or anyone else knew all of them :), not to mention describing
them. :-)
