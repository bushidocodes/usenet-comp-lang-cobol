```
It depends on what type of Cobol file you are working with.
If it is an indexed file, then you would simply write a small
Cobol program to write it out as a flat ASCII file.

If it is already a flat ASCII file, there is no reason that
you cannot read it with Delphi as it sits. The same is
true of all the other file types.  You need to know what
is in it, of course, and how the data is laid out. Cobol
uses standard coding for data (ASCII on a PC) unless
the programmer overrode that deliberately for C-type
word or binary numbers.

Cobol can also access various data-bases. If that is
the case, then you would have to use that data-bases
access methods, the same as Cobol does. SQL would
probably be the easiest method under those circumstances.
```
