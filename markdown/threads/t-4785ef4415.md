See the replies to this same question in the Micro Focus
CompuServe forum.

Bottom-Line: the Current-Date special register is only available
when using the OS/VS COBOL or DOS/VS COBOL emulation mode - this
is not available with Micro Focus Personal COBOL - but is
available with all other Micro Focus COBOL products.

In ANSI conforming COBOL you can do a

Move Function Current-Date to receiving-field

Or you can reference modify that to get a part (for example the
YYMMDD part only).
```

That's because it doesn't exist AFAIK. CURRENT-DATE does but it's an OSVS
extension. You should be using

ACCEPT xxx FROM TIME
ACCEPT xxx FROM DATE etc.

Both of which are ANSI 85 syntax and should be available to you under
Personal COBOL. You could also use the CURRENT-DATE Intrinsic function. Also
ANSI 85 syntax. Where possible use ANSI 85, it's more portable and future safe
than a single vendors extensions. It's a real standard after all rather than a
defacto standard.

I'm not sure if the docs are the problem as I don't have a copy of the
Personal COBOL docs handy.

CURRENT-DATE was part of IBM OS/VS COBOL but was dropped in VS COBOL II.
You might try the COBOL II (ANSI-85) way, which should work like this:

01 MY-DATE.
05 MY-YEAR PIC 9(02).
05 MY-DAY PIC 9(02).
05 MY-MONTH PIC 9(02).

ACCEPT MY-DATE FROM DATE.

This works in Realia COBOL 4.2, the educational version of Realia COBOL,
IBM COBOL II, and should work in IBM COBOL/370.
```
