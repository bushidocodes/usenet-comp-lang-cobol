You can write multi-user programs for DOS, Windows, OS/2, or any other
OS which can accommodate a network and record/file locking.  However,
you will probably be somewhat memory constrained using DOS.  I have
some fairly large multi-user systems (>100k lines of code) which run
under DOS, but you have to be careful how you structure your system to
do it.  Right now Microsoft Windows is by far the most popular OS, and
love it or hate it, Windows is the platform of choice for most users.
If your multi-user system needs to access an ODBC database you are more
likely to find the tools on Windows or OS/2.
