The multi-threading support in CA-Realia COBOL does not allow for
"reentrant" programs which implies common storage. Instead, the DATA
DIVISION of a CA-Realia COBOL program is associated with a CA-Realia COBOL
run-unit entity, and for each thread in a process that executes a CA-Realia
COBOL program, a run-unit is established. Therefore, CA-Realia COBOL
programs are "instanced" per-thread allowing the same CA-Realia COBOL
program to be called on multiple threads. To accomplish shared storage
between threads, the storage area to be shared must be passed as parameter
to CA-Realia COBOL programs executed on multiple threads. Of course, it is
then the application programmer's responsibility to synchronize access to
this shared storage.

Further, in regard to the threading implementation, the 85 standard
indicates that a STOP RUN returns control to the operating system.
CA-Realia COBOL Version 6.0 implements this by raising a exception that in
the case of CA-Realia COBOL main program is handled by the CA-Realia COBOL
runtime start-up code and control is returned to the operating system,
quietly. When CA-Realia COBOL is not the main program of the process or is
executed on a secondary thread, action must be taken by the programmer to
handle STOP RUN (as well as run-time errors) for the CA-Realia COBOL
run-unit, otherwise, as the standard indicates and the implementation
requires, an unhandled exception will occur resulting in process
termination with control being given back to the operating system.
CA-Realia COBOL programs that are the first program of a secondary thread
do not qualify as main programs, again, an unhandled exception will occur
resulting in termination of the process.

CA-Realia COBOL run-unit exceptions can be handled in two ways: 1) by
wrapping execution of CA-Realia COBOL run-units (i.e. at the first
CA-Realia COBOL program of the call stack) in a C try block, or 2) by
registering a user-exit function with the CA-Realia COBOL run-unit using
the RealiaCOBOLRegUserExit function. See the CA-Realia II Workbench
Release 3.0 COBOL User and Reference guides for more information.

Please note that by implementing STOP RUN and run-time errors as
exceptions, CA-Realia COBOL gives notice to other programming languages on
the call stack that the thread or process (or at least thread up to the
first CA-Realia COBOL program on the call stack) is being terminated. This
provides those languages (most notably C++) with an opportunity to clean-up
before termination.
