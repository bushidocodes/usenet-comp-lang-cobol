The best way to do this is NOT by starting IE from the command line , OR by
using the API ShellExecute command.

IE is an OLE component. You can invoke it directly (along with any of its
methods and properties) by treating it like a callable subroutine (COM
component).

I don't use NetExpress, but I understand they have support for OLE just as
Fujitsu do. Read the manual on using components (especially OLE or COM) and
you can make IE (or almost any other MS product like WORD, EXCEL, ACCESS, or
PowerPoint) turn cartwheels...

Basically, you need to create an instance of IE , using the OLE
"CREATE-OBJECT" method. You will have an object reference returned. From
then on anything that uses that object reference will use that instance of
IE.
