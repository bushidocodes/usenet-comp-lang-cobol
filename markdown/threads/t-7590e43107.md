Our Year/2000 tool MVS COBOL Transform Plug-Ins for Year 2000
compliance finds code that needs to be changed, and transforms this
code to handle dates in and beyond the Year 2000 correctly.

Our Year/2000 tool is accurate, automated, extensible and scalable.
Language gateways load ASCII source code into the Code Based Mgmnt
System (CBMS). Gateways are available for a variety of languages
including ANSI COBOL68, COBOL74, COBOL85, IBM OS/VS COBOL, VS COBOL
II, ANSI C, VAX Fortran, Fortran77, Ada and Natural.

It is designed to support the four most popular strategies for
achieving Year 2000 compliance: field widening, windowing (fixed or
sliding window), bridging and data encoding.

The COBOL Language Gateway can be used to capture code as abstract
syntax trees in the CBMS. In the CBMS your code is positioned for
further transformation such as porting to modern platforms,
converting to relational DBMS and client-server architectures, and
extraction of business rules for conversion to SAP, Baan and other
modern MIS environments.
