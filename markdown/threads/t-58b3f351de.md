IBM's VisualAge COBOL includes OO extension based on the ANSI 97 draft
standard. It has most of the features you would expect in an OO
language including inheritance, polymorphism, and encapsulation.
Encapsulation isn't completely "enforced", it is up to the programmer
you use these devices properly. Since VA COBOL is SOM enabled, it can
be used with any other SOM enabled languages. This means I can define
class in C++ and override its methods in COBOL.



At present, DSOM is not supported in VA COBOL, so all objects must exi
in the same process. These objects extensions have been delivered on
several platforms: MVS, OS2, AIX, WINDOWS 95/NT (in beta now).
