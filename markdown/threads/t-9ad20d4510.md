LABELS:
      The labels statement is used when you want to read the data set
      labels as data or data set descriptors.  Only one LABELS
      statement will used, the last, if more than one is included.
      The format of the LABELS statement is:

       {label} LABELS {CONV=(PZ | XE)}
                      {,DATA=(YES | NO | ALL ONLY)}

      CONV= defines the type conversion to be done, 'PZ' converts
      packed decimal to decimal, and XE converts alphanumeric to

Good call.  My boss sent DD the JCL we use to do this - using IEBGENR.
Dunno if I should post it or not, but here goes:

Courtesy of my boss Dwight Miller, PHD - who because of our companies
narrow minded view of the value of news groups - can READ but not
post... One doctor to another said:

Use IEBGENER!  IEHLIST does PDS/PDSE and (shudder) CVOL, not tape.

//BLPLOOK JOB (OS390),VERIFY,MSGCLASS=H,TIME=1,CLASS=9 
//NANCY PROC  TAPE=TAPE,SPLV=     
//ID EXEC PGM=IEBGENER                   
//SYSPRINT DD SYSOUT=*                     
//SYSIN DD DUMMY                                 
//SYSUT2 DD SYSOUT=*                          
//SYSUT1 DD UNIT=&TAPE,                   
// VOL=SER=&SPLV,LABEL=(,BLP),DSN=WHATS.THE.VOLSER,     
//  DCB=(BLKSIZE=8000,RECFM=U)      
//   PEND
// EXEC NANCY,SPLV=MV0570                                     
//* TO READ ROUND TAPES, ADD ",TAPE=ROUND" AFTER THE VOLSER (SPLV)
//*
FOR EXAMPLE:     EXEC NANCY,SPLV=DS7036,TAPE=ROUND  

Change JOB card as appropriate, ditto for tape esoteric names,
symbolics,
etc as dictated by local practice.  The key to it is the BLP in the
LABEL
parameter.  Only 'gotcha' is job execution class must allow
specification
of BLP (JES2 startup parameter).
