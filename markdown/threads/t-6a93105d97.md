DD is correct that using the NOSEQ compiler option should solve your problem.
See:

   http://publibz.boulder.ibm.com/cgi-bin/bookmgr_OS390/BOOKS/igy3pg10/2.4.43

However, it is POSSIBLE to get this message whenever the sequence area (column
1-6 of your source code) is not in ascending order.  If you use ISPF (or Roscoe
or almost any other online editing system) *with* its "automatically number for
COBOL" feature turned on, then you (probably) will never see this EXCEPT for
COPY statements (including those created by generators, precompilers, etc).
However, you may want to use whatever your editor has for a feature to "look at"
column 1-6 to see exactly what IS there.

FYI,
  If you compile with the FLAG(I,I) compiler option, you SHOULD see the message
wherever the problem actually occurs in your source code (I think - some
messages only appear at the beginning or end, but I don't think is one of them).
