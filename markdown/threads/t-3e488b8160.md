Hi

Can anyone please help me.

First, some background. I am using windows for workgroups 3.11, micro
focus cobol 3.2.43 (DOS). The printer is connected to over the
network.

When printing I use

SELECT PRN-FILE ASSIGN TO "LPT1:"
ORIGANIZATION IS LINE SEQUENTIAL.

FD PRN-FILE.
01 PRN-REC PIC X(80).

OPEN PRN-FILE OUTPUT.
WRITE PRN-REC.
CLOSE PRN-FILE.

Problem is that it now takes 60 seconds before the printer starts
printing. Surely the printer should start as soon as the prn-file is
closed.

It is not a windows for workgroups problem, as other dos apps print
immediately.

Any ideas anyone?

Regards
Keith

Keith:

Is the network Novell. The other programs you are using may be giving
the End Capture (endcap) command or something like it. If your program
is not using that command than the system may be waiting for the automatic
time out feature before printing your output from the queue.

Micro Focus, at one time, had a sample program that had some basic queue
managment commands. Sorry I don't have it now.

There also is a X(hex) function call that could be used to issue the "endcap"
followed by the "capture" commands through a dos call.

Scott
