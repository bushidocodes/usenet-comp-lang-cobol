Hi. This is because for 16-bit products, you need to ensure the correct DLL is
in memory before calling the routines. The DLL in question is MFPRNT16.DLL.

I believe you can do this using SET TO ENTRY or
using an .INI file - best to check your docs for this (I'm a 32bit/Unix/NT
person, so this question is 16 bits short of my understanding :) ).
