```
I fought and fought something last night, and I learned something
interesting I want to share with you. CA-Realia has an interesting
"Feature". When compiled for Windows, OPEN INPUT statments searches the
PATH for files. I have a system that will allow selection of a file, and
if that file is in a different directory than the current directory, it
will check to see if a file of the same name exists in the current dir. If
so, it prompts the user for overwriting the file.

The method I use to see if a file exists is to assign it to "file.ext"
dynamically. Then I do an open input. If I get a status 35 I know it does
not exist. I was doing this and and getting a status "00" when the file
was NOT in the current directory! I checked and found a file of the same
name in a different directory that was in the PATH. Removing the dir from
the path yielded the expected "35" status!

I tried the same thing with MF COBOL and it does not search the path.

Very interesting!
```
