Here is something I wrote that does what you need. Enjoy.

Gilbert Saint-flour


Identification Division.
Program-ID. Cob2Job.
Author. Gilbert Saint-flour .
Data Division.
Working-Storage Section.
01 job-name Pic x(8).
01 step-name Pic x(8).
01 program-name Pic x(8).
01 job-number Pic x(8).
Linkage Section.
01 cb1. 05 ptr1 Pointer Occurs 256.
01 cb2. 05 ptr2 Pointer Occurs 256.
Procedure Division.
tcb SET Address OF cb1 TO ptr1(136)
tiot SET Address OF cb2 TO ptr1(4)
MOVE cb2(1:8) TO job-name
MOVE cb2(9:8) TO step-name
jscb SET Address OF cb2 TO ptr1(46)
MOVE cb2(361:8) TO program-name
ssib SET Address OF cb2 TO ptr2(80)
MOVE cb2(13:8) TO job-number
DISPLAY job-name ' '
step-name ' '
job-number ' '
program-name
GOBACK.

The TCB pointer in the PSA points to the TCB of the task that is currently
executing. Each time the MVS dispatcher gives control to a task (read: a
program), then it updates the TCB pointer to point to that task's TCB. A
program fetching the TCB address from the PSA will always get the address
of its own TCB. This is an important element of the MVS architecture.

There is only one PSA per processor (one for each processor in a
multi-processor system); the PSA is 4096 bytes long and always start at
address zero. Some fields in the PSA never change (the pointer to the
CVT, for example), others change constantly. All of the programs that
execute on a given processor (one at a time) see the same PSA, however
some of what they see is the same, some is not.

hope this helps
