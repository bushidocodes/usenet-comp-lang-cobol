H�kon:

I'm sure that it is very feasible.  You need to address the impact of
possible changes to your source program.

I believe that Micro Focus does not support the OS/400 operating
system, but you should first check with them to see if they have a
compiler which does indeed support that environment.  That would be
the "path of least resistance" so to speak.  In other words, using the
same compiler on the new environment should result in minimal (if any)
source code changes.

If Micro Focus does not have a compiler which runs on an AS/400
machine, then you may wish to contact IBM and Ryan McFarland (Liant).

I'm quite sure that both IBM and Ryan McFarland have COBOL compilers
supporting this platform.

Here are the web sites:

Micro Focus COBOL

http://www.mfltd.co.uk

IBM COBOL

http://www.software.ibm.com/ad/cobol/cobol.htm

Ryan McFarland COBOL

http://www.liant.com/products/language/rm/rmcobol.html

You may have quite a task ahead of you in converting your source code
to run with a different compiler.  It depends on such issues as:

- use of proprietary or platform-specific compiler extensions used in
your existing source program

- file handling differences between the various compilers

- block mode screen handling on an AS/400 (5250 terminal emulation
issues) versus character mode screen handling on the PC

...and probably many more things that I haven't even thought about.
I'm not trying to discourage you, just alert you to the possibility
that there will most likely need to be source code changes when you
make the move.

Good luck!


Bob Wolfe, flexus
