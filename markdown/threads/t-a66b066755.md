MS-COBOL 3.0 upwards equates to various versions of MF COBOL. I
think v4.5 equates to 2.x and v5.0 to v3.x.

If the files are Indexed, then the data/index format is
proprietory, but see if there any import facilities for MF Cobol.

Failing that, write a program to extract the data into LINE
SEQUENTIAL format using commas as field delimeters. output numeric
fields as DISPLAY, not COMP-3 and you have created a .CSV file which
will then read into most systems.

This does assume that you have the file layouts and compiler, etc.
