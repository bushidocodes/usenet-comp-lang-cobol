I'll supply the answer to my own question since (with Seagate Softwares
excellent support) I have resolved the problem and got the program to
work.

The logon structure is a fixed structure size, not as I had assumed a
variable size.  The size of the structures is defined in the
GLOBAL32.BAS file that is included with the API.  So the answer was
right there all the time.  Here is how the structure actually looks
when coded correctly:


  01 WS-LOGON-STRUCT.
     05  WS-LOGON-LEN  PIC S9(04) COMP-5 VALUE 514.
     05  WS-SERVER     PIC X(128).
     05  WS-DB         PIC X(128).
     05  WS-USER       PIC X(128).
     05  WS-PASS       PIC X(128).

the GLOBAL32.BAS has the correct information of all the API structures.
