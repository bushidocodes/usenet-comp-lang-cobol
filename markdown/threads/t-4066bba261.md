=> Hello,
=> I am new to the IBM mainframe environment (not new to COBOL) -I am writing

=> must create a file for each of multiple institutions. The files would be
=> named BK.PROD.001INST, BK.PROD.002INST etc. Institution number is a 3
=> position field.  Is there a way in IBM COBOL (other than allocating and
=> declaring FDs for 999 files in my program) to handle this ? i.e. create
=> BK.PROD.001INST, when institution # changes close it and reopen it as
=> BK.PROD.002INST.
=> 
=> Any pointers would be most appreciated.  BTW this is not homework !!

Since I work in a VAX environment, this may be less than helpful, but VMS
allows us to keep multiple versions of a file within a directory. If we
open and close a file ten times, we end up with 
                  filename;10
                  filename;9
                  filename;8
                  filename;7
                  filename;6
                  filename;5
                  filename;4
                  filename;3
                  filename;2
                  filename;1
Then the DCL command procedure that invoked the COBOL program can run a
loop that renames the files appropriately:
      rename filename;1   BK.PROD.001INST
         .
         .
         .
      rename filename;10   BK.PROD.010INST

The DCL command proc stops renaming when it detects that filename;nnn does
not exist.

If the actual institution number can fall arbitrarily within the range
000-999, then you can write the actual sequence to a simple text file
and have the command procedure rename the filename;x according to the
values the command proc reads from the text file.

Before invoking the COBOL program, the command proc makes sure that there
are no files with the name filename;x either by deleting them or by
renaming the old versions to filename.backup.timestamp, or some such
thing. Ditto for the auxialry text file if you need to go that route.
