This appears to be Micro Focus COBOL - if it's not then what I will
say obviously does not apply!

The "%cob0.o" file in the temporary directory is a file that "cob"
creates which contains stub routines to handle various options relating
to mapping symbol names, dynamic loading of symbols (eg as .gnt) etc.

I think you must have an option on the "cob" command line, something
like "-d SYMBOL_NAME" or "-m SYMBOL_NAME=OTHER_SYMBOL_NAME". It's
difficult to tell without the exact command line you are using, but
that's my guess.

What you could also do is grab the %cob0.o file and dissasemble it
(assuming it's created directly - if not, there will be a %cob0.s file
you can grab instead). That may provide more clues (you have to be
quick, though - the file gets removed when "cob" has finished).

Cheers,
Kev.
