I totally agree with you. The 'solution' is not a solution because
1) you cannot compare old and new formats,
2) code still needs to change to accomodate the two possibilities,
3) forcing yourself into this proprietary (and patented) format commits a
company to license the software forever
4) quick maintenance of a program depends on the programmer being able to
read the contents of the fields in a file or dump quickly. This is why
one does not see binary fields in file often and
5) it is yet another 'silver bullet' hype approach that these companies
try to get the brass to commit to before the people in the trenches have
a chance to say NO
