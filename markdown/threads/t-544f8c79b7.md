```
We have a curious (ie frustrating and immensely tedious) problem. We have
a system which uses Micro Focus Cobol to access Btrieve data files on an
SFTIII system (Novell 3.11). We have no problem at all until we start
doing heavy housekeeping transactions when the system fails. The error
seems to be associated with XM (we have traced the problem as far as
XFH2BTR) and returns an error code of 199.
The PCs we are using run DOS 5.0 (though we have tried with 6.0 and
6.22), have 486 DX66 processors and 8Mb of memory.

* Has anyone else seen this.
* Can anyone suggest a fix.
* Does anyone know what the error is.

Ian Sanders
```

```
corrupting memory used by XM ?
```
