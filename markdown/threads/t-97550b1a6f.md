I use Mantis on MVS about 13 years ago so my knowledge is more
than a little dated.

Supra is not relational it is a network database, for example:

customer has transaction list:
customer contains pointer to first transaction
each transaction contains pointer to next transaction.

In relational the transactions would contain a customer number.

The access is very fast but if it is corrupted a restore is the
only choice. (It only happened once in five years though).

When I left they were just starting to support SQL which makes the
database pretend to be relational.  If you are using this it
really is no great worry so long as you know SQL.

MANTIS:  This is a pascal like language that works very easily on
sets of data (for example 20 rows of transactions to present on
the screen). It really requires you to use a lot of subroutines
(not paragraphs more of a call with parameters).  If you have had
some non cobol experience then it should be a breeze.

ODBC was not invented when I was playing with Mantis but it did
have DB2 interfaces which (assuming it worked) would be really
easy to work with using Mantis.

Good luck
Ken

I last used Supra and Mantis about 4 years ago.  There are 2 versions
of Supra.  The 1.x version is a network database with a relational
layer bolted on top but the database itself does not use SQL.  The 2.x
version is a completely different engine that is relational and
supports SQL.  Both databases are somewhat dated and lack all of the
features and functionality one would find in a RDBMS like Oracle.

MANTIS is a character based (non GUI unless you buy some add on
products) 4GL.  I enjoyed using it especially against DB2 on MVS; it
allowed the use of embedded SQL from which you could generate DB2 plans.

I doubt whether if you will find many online references to either
product due to their waning popularity.
