I am using Server Express 4.0 SP1 on HP-UX 11.11.

I have what I hope will be a quick, simple question.

Does anyone know of a way to tell the compiler, using the COBCPY
environment variable, to search sub-directories also?

I have a large number of source files for my application (over 13,000)
contained in my /app/source directory. In order to manage them more
effectively at the UNIX level, I have divided them into multiple
subdirectories containing smaller numbers of files.

One drawback is that I need to specify all of these subdirectories in
the COBCPY variable, for example:

COBCPY=/app/source/d1:/app/source/d2: ... :/app/source/d99

As you can imagine, this is quite cumbersome.

Is there a way for me to tell the compiler to search subdirectories
when processing the COBCPY variable? For example:

COBCPY=/app/source/%D

Where the %D tells the compiler to look at all directories below
/app/source?

I know I may be reaching, but this would be most helpful.

Thanks in advance.

Chris
