```
The example code

unstring xyz delimited ", " or " "
  into last-name first-name middle-initial

will NOT work correctly if your input may have spaces within the last name
field, e.g.
    Mac Cumber   (rather than MacCumber)

If the original question is for a "simple" class type exercise, it should
suffice nicely.  However, for "real world" cases, you need to know EXACTLY
what "validation" is done before the UNSTRING statement.  Can your input
ever have such things as:

   Mac Cumber, John Joseph K
   Smith III, Thomas
    Jones, Jr, Ian Ivanovich

etc ....

"complex name parsing" is a quite a CHALLENGE with "real world" proper
names.
```

```
That's true

How about

move space to fld1 fld2 fld3 fld4 fld5 fld6 fld7
move space to del1 del2 del3 del4 del5 del6
unstring xyz delimited by "," or all space
into fld1 fld2 fld3 fld4 fld5 fld6 fld7
delimiter in del1 del2 del3 del4 del5 del 6

move space to last-name first-name middle-initial

move 1 to lst-ptr

if del1 = space
  string fld1 delimited " "
  " " delimited size into last-name pointer last-prt
else
  move fld1 to last-name
  move fld2 to first-name
  move fld3 to middle-initial
else-if del2 = space
  string fld2  delimited " "
  " " delimited size into last-name pointer last-ptr
else
  string fld2 " " delimited size into last-name pointer last-ptr
  move fld3 to first-name
  move fld4 to middle-initial
else-if del3 = space
  string fld3  delimited " "
  " " delimited size into last-name pointer last-ptr
else
  string fld3  delimited " "
  " " delimited size into last-name pointer last-ptr
  move fld4 to first-name
  move fld5 to middle-initial
else-if del4 = space
  string fld5  delimited " "
  " " delimited size into last-name pointer last-ptr
else
  string fld5  delimited " "
   " " delimited size into last-name pointer last-ptr
  move fld6 to first-name
  move fld7 to middle-initial
else-if del5 = space
  string fld6  delimited " "
  " " delimited size into last-name pointer last-ptr
else
  string fld6 " " delimited size into last-name pointer last-ptr
  move fld7 to first-name
else-if del6 = space
  string fld7 delimited " " into last-name pointer last-ptr
end-if

If there is no comma "," in the name string then the last-name field 
contains the orignal xyz string with only a single space separating 
the words.

This does assume that the string will be a multi part last name 
followed by a comma "," followed by a single part first name followed 
by an initial. To add the code to handle a multi part first name 
followed by an initial would require some additional tests on the 
contents of the fields following the delimiter containing the comma.

Something like

if fld7 not = space
  string 
    fld2 delimited " "
    " " delimited size
    fld3 delimited " "
    " " delimited size
    fld4 delimited " "
    " " delimited size
    fld5 delimited " "
    " " delimited size
    fld6 delimited " "
    into first-name
  move fld7 to middle-initial
end-if

etc etc

The resulting code would be kind of long but I think it would do the 
job If the fields used to contain the unstring results were in an 
array you could probably do it with a nested perform loop.
```

```
In the real world, you either have a well defined, already parsed out name, in
which case this is trivial - or you have a much more complicated situation with
multiple middle names, last names in multiple parts, suffixes, prefixes, and
such that need to be parsed.   Companies such as Cahner's (which produces free
magazines for people who qualify), have extensive data cleansing and parsing
algorithms that have been developed in clean rooms for trade secret purposes.
```
