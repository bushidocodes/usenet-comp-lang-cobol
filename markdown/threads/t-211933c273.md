I don't know under UNIX, but under DOS it's like this :

CALL X"B7" USING function,parameter,buffer.


Where :

function is a PIC 99 COMP-X which contains :
	0	to read characters from the screen
	1	to write characters to the screen
	2	to read attributes from the screen
	3	to write attributes to the screen
	4	to clear characters from the screen
	5	to clear attributes from the screen

parameter is a group item consisting of three data items :

	- a PIC 9(4) COMP-X field showing the length of the data to be read or
written

	- a PIC 9(4) COMP-X field giving the start position on the screen. Top
Left is 1, next line start at 81 etc.

	- a PIC 9(4) COMP-X field giving the start position in the buffer. The
starting buffer position is 1.

buffer is the COBOL data area. It's a PIX X(n) field and may be as small or
large as you need. An entire screen of text or attributes is 2000 bytes (80
* 25)

Maybe this also works under UNIX, you'll have to try it out.

Good luck,

Peter Stoops
