I've been following his along for a bit now, and it's been interesting,
but what about device control? How, for example, do I set a baud
rate up from a Cobol Program? Currently, I would call an assembler
subroutine, but that is getting more and more awkward as the OS's
advance. What is really needed is a special class of "devices" that
are really device control engines. Just write a standard ASCII string
into a line sequential device controller.

It is simply getting too hard to maintain fluency in more than a few
languages. I have been thinking of having that sort of thing jobbed
out. Take all the serial devices, have a serial server with a 30-40
meg memory disk sitting as a network device. Then write pure
Cobol accessing the memory disk, and directing the device handlers.
They could be written in any language, but it would be tidy to write
them in Cobol as well.
