```
When you are coding a CICS BMS screen edit paragraph, frequently you need to
bail out of the paragraph after the first error. The easy way to do this
would be to just 'GO TO' an exit, but horrors, that would violate the
structured programming paradigm. Here's a sneaky way to do the same thing.
If you follow the rule that the only period coded in a paragraph is the
mandated period at the end, a 'NEXT SENTENCE' statement exits the paragraph.
For example:

IF WS-FLD1 = 'A' OR 'B'
  CONTINUE
ELSE
  MOVE 'Y' TO WS-ERR-SW
  NEXT SENTENCE
END-IF

EDIT OTHER FIELDS (no periods except at end of paragraph)


The COBOL compiler won't accept this because 'NEXT SENTENCE' must be the
only conditional sentence so you have to code a little stupidly (hence the
message title)


 IF WS-FLD1 = 'A' OR 'B'
  CONTINUE
ELSE
  MOVE 'Y' TO WS-ERR-SW
  IF TALLY = TALLY NEXT SENTENCE END-IF
END-IF

EDIT OTHER FIELDS
```

```
Yes this is cute. One could 'improve' the looks of it
by having an 88 condition that was always true with
the name GOTO, then the code would read:

IF WS-FLD1 = 'A' OR 'B'
     CONTINUE
ELSE
     MOVE 'Y' TO WS-ERR-SW
     IF GOTO NEXT SENTENCE END-IF
END-IF

EDIT OTHER FIELDS

This would make it clear that we are simulating a GO TO.
My reason for not allowing real GO TOs is a technical one,
not a stylistic one. I don't think a GO TO is that ugly (but the
xxx-EXIT is!).
Just like Judson, I also work with a code generator;
my generator works like a finite state machine, with
tables controlling the execution of program modules.
Each module is PERFORMED when it should be
according to the tables, but it is mandatory that
EVERY module (read paragraph) returns to the
control loop of the finite state machine. If even a
single module did a GO TO, control would be lost.
There are various solutions to this, one is to end
every module with a GO TO to the control loop,
but I prefer that the modules don't even *know* about
the control loop. Another would be to use sections,
but that seems a bit heavy and is error prone during
maintenance.

So, people have all kinds of good reasons for doing
what they do; only rarely is it stubbornness, ignorance,
old-fashionedness, etc.
```
