```
Greetings.

I have enjoyed reading this NG and hope someone can help me with a problem I'm
up against.

In PowerCOBOL 4.2 I am using the GetFileName method to let the user browse
our software's data directories and select a ledger file to open or save. (I
am working porting an accounting system to a GUI) Everythings works fine: I
set the file mask to '*.GLA' and if there are any GLA files in the data
directory the user is presented with a nice file browse window in which one
can be selected. The problem occurs if there are no GLA files in this
directory. In this case GetFileName open a browse window on the user's "My
Documents" folder. Our desire is to keep the user as close to the recommended
data location as possible and not encourage wide distribution of data files
across the hd or network.
```

```
I don't know PowerCobol, and I don't know which GUI you are porting to,
however, I have a suggestion if your cobol handles basic MS Window datatypes
and that is your target platform.

You may then use the Kernel32.dll and its FindFirstFileA and FindNextFileA
functions. In this case you would have to develop your own dialog, or you
could check for the existence before launching the host dialog.

Code is for the Acucobol dialect, approximately 100 lines (unfortunately it
is sectioned :-) ) Here are the basics;

       PROCEDURE DIVISION.
       MAIN SECTION.
       MAIN-001.

    PERFORM LOAD-DLL.
    PERFORM INITIALIZE-FILE-VARS.
    PERFORM INITIALIZE-REC.
    MOVE    "C:\*.*"         TO WS-PATH.
    MOVE    2                TO WS-COUNT.
    DISPLAY GET-PATH.
    ACCEPT  GET-PATH.
           MOVE    ANY-FLAG         TO  FILE-ATTR.
           INITIALIZE               WS-COUNT.
           STRING  WS-PATH          LOW-VALUES DELIMITED BY SIZE
                   INTO             FILE-SELECTION.
           CALL    "FindFirstFileA" USING
                   FILE-SELECTION
                   FILE-FIND-DATA
                   RETURNING        FILE-HANDLE
                   ON               EXCEPTION GO TO MAIN-800.

           IF      FILE-HANDLE      < 1
                   DISPLAY          MESSAGE   BOX
                                    "No files found!"
                                    TITLE     FILE-SELECTION
                   GO               TO MAIN-100.

           MOVE    1                TO FIND-STATUS.
    PERFORM UNTIL            IS-FALSE
                   PERFORM          DISPLAY-RESULT
            PERFORM          INITIALIZE-REC
                   CALL             "FindNextFileA" USING
                                    BY VALUE FILE-HANDLE
                                    BY REFERENCE FILE-FIND-DATA
                                    RETURNING FIND-STATUS
                                    ON EXCEPTION GO TO MAIN-810
                                    END-CALL
                  END-PERFORM.
    ACCEPT OMITTED.

       MAIN-100.

           PERFORM UNLOAD-DLL.
           GO      TO               MAIN-900.

       MAIN-800.

           DISPLAY MESSAGE          BOX
                   "Couldn't call FindFirstFileA!"
                   TITLE            "Fatal error".
           GO      TO               MAIN-900.

       MAIN-810.

           DISPLAY MESSAGE          BOX
                   "Couldn't call FindNextFileA!"
                   TITLE            "Fatal error".
           GO      TO               MAIN-900.

       MAIN-900.
       MAIN-EXIT.
           GOBACK.

       DISPLAY-RESULT SECTION.
       DISPLAY-RESULT-001.

           ADD     1                TO WS-COUNT.
           DISPLAY FILE-INFO.

           IF      WS-COUNT         =  24
                   DISPLAY          "(Press ENTER)..." AT 2501
                                    WITH NO ADVANCING
                   ACCEPT           OMITTED WITH NO ADVANCING
                   DISPLAY          OMITTED AT 0101 ERASE SCREEN
                   INITIALIZE       WS-COUNT.

       DISPLAY-RESULT-900.
       DISPLAY-RESULT-EXIT.
           EXIT.

       INITIALIZE-FILE-VARS SECTION.
       INITIALIZE-FILE-VARS-001.

           INITIALIZE               FILE-SELECTION
                                    FILE-HANDLE
                                    FILE-ATTR
                                    WS-PATH.

       INITIALIZE-FILE-VARS-900.
       INITIALIZE-FILE-VARS-EXIT.
           EXIT.

       INITIALIZE-REC SECTION.
       INITIALIZE-REC-001.

           INITIALIZE               WIN-FILE-NAME
                                    DOS-FILE-NAME
                                    CREAT-LOW
                                    CREAT-HIGH
                                    ACCES-LOW
                                    ACCES-HIGH
                                    WRITE-LOW
                                    WRITE-HIGH.

       INITIALIZE-REC-900.
       INITIALIZE-REC-EXIT.
           EXIT.

       UNLOAD-DLL SECTION.
       UNLOAD-DLL-001.

           CANCEL  "KERNEL32.DLL".

       UNLOAD-DLL-900.
       UNLOAD-DLL-EXIT.
           EXIT.

       LOAD-DLL SECTION.
       LOAD-DLL-001.

           SET     ENVIRONMENT      "DLL-CONVENTION" TO 1.
           CALL    "KERNEL32.DLL"   ON EXCEPTION GO TO LOAD-DLL-800.
           GO      TO               LOAD-DLL-900.

       LOAD-DLL-800.

           DISPLAY MESSAGE          BOX
                   "Couldn't LOAD Kernel32!"
                   TITLE            "Fatal error".
    GOBACK.

       LOAD-DLL-900.
       LOAD-DLL-EXIT.
           EXIT.
```
