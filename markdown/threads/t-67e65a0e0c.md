Don't know if anyone's interested but when I was learning MF GUI Dialog
after doing character for years I wrote an export file health checker
because of the many gaps(features?) in GUI Dialog. I know it was a bit of a
pain in character dialog having to define a procedure before you actually
called it but at least this ment you had less chance of putting the
wrong procedure number in when you called it etc...
Anyway, I kept chipping away at it since then and Im pretty chuffed with the
result so if anyone wants the source they're more than welcome to it.
You export the screenset and then run the program against the export
file

It checks:
Procedures called that don't exist (spelling mistakes probably)
Procedures that exist but aren't called (maybe perfectly legitimate)
Duplicate Procedure Names
Conflicting duplicate procedure names
(i.e same name in object/window/global dialog, even expands the dialog
so you can compare the procedures!)

In fact Ive just had a thought, I can also show redundant dialog
(i.e after a branch-to-procedure) - i'll get on that tommorrow

If someone does want it, just remember that I wrote it.....

to pay!:-)
