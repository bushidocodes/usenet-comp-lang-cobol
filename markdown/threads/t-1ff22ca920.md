Yes it worked great for the season or two that I used it.  I was going
to expand the program to track for multiple users but just never had
the time.

Basically it used the week number (of the season) as the key.  I would
input the entire NFL schedule at the beginning of the season.  Then
each week I could bring up that week's schedule, enter my predictions
and it would update the files.  Then I'd come back on Tuesday and
enter the scores of all the games.  It would summarize wins and losses
and how many points were accumulated that week and season to day based
on my predictions.  The pool was a weighted pool.  You selected a
winner and gave it a weight from 14 to 1 (there were only 28 teams in
the NFL then).  Generally all the high weights were put on the games
you felt were a lock, i.e., a week team playing a powerhouse.

I spent a lot of time making the input easy and screens "pretty".  I
did have visions of making it commercially available but work got in
the way.  It was a great way to learn Basic and Commodore Assembly and
the use of system calls to do things.  That saved a lot of basic code.
The program even supported two floppies (one to load the program in
and another to read and write the data from and to).

Another plus was I had a basic "compiler" so I could compile the
program into object code and run that.  It was pretty fast considering
everything that was going on in it.
