I assume the values are SMALL!!!
Standard prime-test methods do NOT find the factors!
Trial-division methods are *extremely inefficient*, for example a
40-digit number would take a million years to factorize this way!
Do you mean display its prime factors?
Try this code but *note*: I've omitted the Working-Storage (my extra
locations are probably PIC S9(4) COMP). I've used an advanced Report
Writer syntax - if you don't have it you need to do this part from first
principles or use a series of DISPLAY ... [WITH NO ADVANCING]  The
clean-up / debugging / testing is *100% up to you*!

REPLACE ==&max-factors== BY ==12==.
WORKING-STORAGE SECTION.
01  W-WORKING-LOCATIONS.
    05  W-FACTOR-TABLE.
      07  W-FACTOR-ENTRY   PIC S9(4) COMP  OCCURS &max-factors.
    05  W-EOF   PIC S9(4) COMP VALUE 0.
      88  END-OF-FILE    VALUE IS 1.
....  to be completed
REPORT SECTION.
RD  FACTOR-REPORT   PAGE LIMIT 60.
01  FACTOR-REPORT-LINE  TYPE DE.
   03  LINE  + 1.
      05  COL 1      VALUE "GIVEN VALUE".
      05  COL + 2   PIC ZZZ9   SOURCE W-VAL.
      05  COL + 2   VALUE "IS".
      05  COL + 2   VALUE "PRIME" WHEN W-FACTOR-CNT = 0
                              VALUE "COMPOSITE" WHEN OTHER.
      05  PRESENT WHEN W-FACTOR-CNT > 0.
          07  COL + 2     VALUE "FACTORS ARE".
          07  COL + 2     PIC ZZZ9   OCCURS 1 TO &max-factors DEPENDING
ON W-FACTOR-CNT
                     VARYING R-CNT  FROM 1 BY 1    SOURCE W-FACTOR-ENTRY
(R-CNT).
PROCEDURE DIVISION.
MAIN-CONTROL.
    OPEN INPUT INPUT-FILE OUTPUT REPORT-FILE
    INITIATE FACTOR-REPORT
    PERFORM READ-INPUT-RECORD
    PERFORM CALC-PRINT-READ UNTIL END-OF-FILE
    TERMINATE FACTOR-REPORT
    CLOSE INPUT-FILE REPORT-FILE
    STOP RUN.
CALC-PRINT-READ.
    PERFORM VARYING W-VAL-CNT FROM 1 BY 1 UNTIL W-VAL-CNT > 6
         MOVE IN-VAL (W-MOD) TO W-VAL
         COMPUTE W-MAX = FUNCTION INTEGER-PART (FUNCTION SQRT (W-VAL))
         MOVE 0 TO W-FACTOR-CNT
         INITIALIZE W-FACTOR-TABLE
         PERFORM VARYING W-FACTOR FROM 2 BY 1 UNTIL W-FACTOR > W-MAX
               MOVE 1 TO W-DIVIDES-IT
               PERFORM UNTIL W-DIVIDES-IT = 0
                    DIVIDE W-FACTOR INTO W-VAL GIVING W-TEMP REMAINDER
W-REM
                    IF   W-REM = 0
                           MOVE W-TEMP TO W-VAL
                           ADD 1 TO W-FACTOR-COUNT
                           IF  W-FACTOR-CNT > &max-factors
                                 DISPLAY "too many factors! ("
W-FACTOR-CNT ") - recompile!"
                           ELSE
                                MOVE W-FACTOR TO W-FACTOR-ENTRY
(W-FACTOR-CNT)
                           END-IF
                    ELSE
                           MOVE 0 TO W-DIVIDES-IT
                    END-IF
               END-PERFORM
         END-PERFORM
         GENERATE FACTOR-REPORT-LINE
    END-PERFORM
    PERFORM READ-INPUT-RECORD.
READ-INPUT-RECORD.
    READ INPUT-FILE AT END MOVE 1 TO W-EOF.

John.

* DETERMINE SIGNS TO BE USED SO THAT LIMITS MAY BE

	       IF C > ZERO
		  IF B < ZERO

	       IF C > ZERO
		  IF B > ZERO
		     MOVE 1 TO D.
	       IF C < ZERO
		     MOVE 0 TO D.
*     * SET THE LIMITS FOR THE FIRST SET OF LISTS.
*     * F IS HIGH LIMIT, E IS LOW LIMIT
	       IF D < 0 MOVE 0 TO F.
	       IF D < 0 COMPUTE E = C * -1.
	       IF D > 0 MOVE 0 TO E.
	       IF D > 0 COMPUTE F = C.
	       IF D = 0 COMPUTE E = C
			COMPUTE F = C * -1.
*     * H IS THE LIST ELEMENT WE ARE WORKING WITH.
	       MOVE 0 TO H.
	       PERFORM VARYING G FROM E BY 1 UNTIL G > F
		  COMPUTE I = C / G
*     * SEE IF ANY REMAINDER
		  IF I-REM = ZEROS

		     MOVE I TO L2 (H)
		  END-IF
	       END-PERFORM.
* determine the other limit
* assume always positive
	       MOVE 1 TO D.

	       IF D < 0 COMPUTE E = A * -1.
	       IF D > 0 MOVE 0 TO E.

*     * H IS THE LIST ELEMENT WE ARE WORKING WITH.

*     * SEE IF ANY REMAINDER
		  IF I-REM = ZEROS

	       END-PERFORM.
* the 2 sets of lists are created.

* now we need to loop through the multiply till we get a hit!
      PERFORM VARYING CNTR FROM 1 BY 1 UNTIL CNTR > 1000

 GOT-HIT.
