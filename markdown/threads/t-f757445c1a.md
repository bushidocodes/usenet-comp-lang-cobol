I think he/you mean BIOS.  CMOS is a semiconductor process.

> Most modern OSes, simply use it for the BOOT phase, thereafter, they
> provide their own considerably faster drivers.

Absolutely.  Windows NT, Solaris, *BSD, and Linux don't use a single 
scrap of BIOS code AT ALL post-boot (with the possible exception of 
Power Management in Linux's case).

In fact, the BIOS is only used to get the first stage loader into the 
system and for that loader to read enough sectors to load a kernel of 
some kind that uses its own routines from that point forward.

The time spent during that phase is less than a second.
