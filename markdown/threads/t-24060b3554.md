First off - check the adtools site to see if there are any Fujitsu
examples for using Screen Section. I use Micro Focus, but I *think* the
F/J and M/F approaches are fairly similar. Firstly some pseudo code to
achieve what I *think* you want to do.

Assume you have a screen :-

01 Scrn-CustomerRecord.
    05 Scrn-CustomerAccountNo line x col x pic ...
    05 Scrn-CustomerName line x col x pic
    05 Scrn-City....
    05 Scrn-AccountBalance.....

I'm vague on the syntax as it's a long time since I used Screen Section.

Pseudo :-

    Set RecordNotFound to true     (Level 88s)
    Set Record NotChanged to true (Level 88s)
    Display empty CustomerScreen
    accept Scrn-CustomerNo

    if  CustomerNo is blank - end of routine (user doesn't want to enter
info)
    if not blank Read CustomerFile
       if RecordFound
        set RecordFound to true
        display fields
          now go to 'Change' routine
       else set RecordChanged to true
            start accepting individual fields, incrementing through your
field numbers -
                accept scrn-name, accept scrn-city, accept scrn-balance.
                having accepted all input fields for new record now go to
'Change'routine

    Accept Scrn-CustomerName - if blank you have an error
    use the bottom Line (24) to display your error message with BLINK,
        HIGHLIGHT, REVERSE (???)
    now go back to Accept Scrn-CustomerName

    if that one is OK now accept scrn-Nextfield2 etc

    when you have finished new record input you go to 'Change' routine

You may or may not have CBL_MOUSE routines - if you do it makes life
easier.
If you haven't MOUSE routines use Line 24 to display

'Enter field Number to change :-   ..  Press <Enter> to finish"

To get Field Numbers part of your initial display has to contain 'labelss'
before each field, such as :-

    .... Account Number -------
    (1) Name ------------------
    (2) City--------------------
    (3) etc----------------------.
    (4) Account Balance -----------

How do you position - Note I have NO FIELD NUMBER for Account Number - we
don't want the user changing this, (they can enter an AccountNumber BUT
they can't change it !), so when you accept a FieldNumber, something like
following :-

    AcceptFieldNumber
    evaluate true
        when FieldNumber = zero - end of routine
        when FieldNumber = 1  perform ACCEPT-NAME (accept scrn-name)
        when FieldNumber = 2  perform ACCEPT-CITY
        when FieldNumber = 3 etc....
    End-evaluate

Each time you change a field set RecordChanged to true. User keys in say
'4' and you then go to routine to Accept Scrn-AccountBalance. When user
enters field number zero - they are at the end. Now display another
message line with

                " (O)K or (C)ancel"

    if OK ( 88 OK value "O', 'o")

        if RecordChanged

            if RecordFound
               perform REWRITE-record

                else perform WRITE-new-record
            End-if
        End-if

   else  display Empty Screen
End-if
