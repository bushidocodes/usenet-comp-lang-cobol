According to my MF 3.2 Error Messages manual, run time error 199 says
"Operating System error code lies outside expected range (Fatal) -- A system
call has returned an unexpected error number which is not documented. --
Contact Technical Support who will help you to discover the specific cause of
this error."

It sounds to me like the file i/o call is getting an error "25" returned from
the OS, and it doesn't know what that error is.

In MF Cobol - any time you get a file error "9/nnn" the "nnn" is a run-time
error and can be found in the manual as such.  In this case, I would believe
what the manual says and call MF Tech Support.
