"10" as a file status is guaranteed in Cobol to have a specific
meaning.  It should be unnecessary to look it up.

'FILE-IS-AT-END' _may_ have a value of "10" as a 78 or a PIC XX, or it
may be an 88 level (and thus have a different usage) or it _may_ be a
PIC X field that is designed to be set as a flag when AT END is taken.

If you never doubt what it means then you are not being defensive about
your code and there will be tears.
