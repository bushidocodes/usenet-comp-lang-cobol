### small utility illustrating line sequential processing

- **From:** bberman@netbox.com (B Berman)
- **Date:** 1999-10-09T00:00:00

```
Sometimes when I see a thread of interest developing in a newsgroup,
(this one or others) I will wait until it builds up a volume of
messages, and then save the entire thread to disk as a text file, for
reading later.  Unfortunately, many, many header lines from the
messages are saved as well, and interfere with reading, so I wrote a
small routine to strip them out.  I had originally written it in VB4,
and now in Fujitsu4.  I was amazed at how much faster it runs in
Cobol.


 identification division.
 program-id.  Filestrp.
 author.  b.berman.
 installation. p&w.
 date-written. August 1999.
 remarks.
* this program reads a line sequential txt file, strips out
*     unwanted lines, and outputs a new line sequential txt file.
*     It was specifically written to reformat a file consisting
*     of a saved newsgroup thread, and the unwanted lines are
*     all the various headers that are not necessary, and only
*     interfere with the readability of the file.
 environment division.
 configuration section.
 source-computer. ibm-pc with debugging mode.
 object-computer. ibm-pc.
 input-output section.
 file-control.
     SELECT TextFile-In ASSIGN TO Trimmed-Path-In-W
             ORGANIZATION IS line SEQUENTIAL
             FILE STATUS  IS FileStatus-In-W.
         
     SELECT TextFile-Out ASSIGN TO Trimmed-Path-Out-W
             ORGANIZATION IS line SEQUENTIAL
             FILE STATUS  IS FileStatus-Out-W.

 
 data division.
 file section.
*  the manual leads one to believe that text character strings
*      (lines) are limited to 80 characters, but this is not
*      true, and I have encountered lines of 570 characters
*      before the CRLF.

 FD  TextFile-In
     record is varying in size from 1 to 1000 characters
             depending on Record-Size-In-W
     data record is Record-In.
		
 01  Record-In.
     05  filler                  pic x
                                 occurs 1 to 1000 times
                                 depending on Record-Size-In-W.		
 FD  TextFile-Out
     record is varying in size from 1 to 1000 characters
     depending on Record-Size-Out-W
     data record is Record-Out.
		
 01  Record-Out.
     05  filler                  pic x	
                                 occurs 1 to 1000 times
                                 depending on Record-Size-Out-W.


 working-storage section.
 01  File-Work-Areas-W.
     05  PathName-In-W           pic x(255).
     05  FileStatus-In-W         pic 99.
     05  DriveName-In-W          pic x.
     05  Folder-In-W             pic x(255).
     05  FileName-In-W           pic x(255).
     05  PathName-Out-W          pic x(255).
     05  FileStatus-Out-W        pic 99.
     05  DriveName-Out-W         pic x.
     05  Folder-Out-W            pic x(255).
     05  FileName-Out-W          pic x(255).
     05  Input-EOF-Flag-W        pic x.
         88  Input-EOF                   value 'Y'.
     05  Input-Validation-Flag-W pic x.
         88  Input-Line-Valid            value 'Y'.
         88  Input-Line-Substitute       value 'S'.
         88  Input-Line-Skip             value 'N'.
     05  Record-Size-In-W        pic 9(4)    comp    value 0.
     05  Record-Size-Out-W       pic 9(4)    comp    value 0.
  		
 01  File-Name-Area-W.
     05  Trimmed-Path-In-W.
         09  Filler              pic x
                                 occurs 1 to 255 times
                                 depending on Path-Cntr-W.
     05  Trimmed-Path-Out-W.
         09  filler              pic x
                                 occurs 1 to 255 times
                                 depending on Path-Cntr-W.
  
 01  Program-Counters-W.
     05  Input-Line-Cntr-W       pic s9(7)   comp-3  value +0.
     05  Output-Line-Cntr-W      pic s9(7)   comp-3  value +0.
     05  Path-Cntr-W             pic s9999   comp    value +0. 

 linkage section.
 screen section.

 procedure division.

 Begin-it.
     initialize  File-Work-Areas-W.
     initialize  Program-Counters-W.
     move  'N'  to  Input-EOF-Flag-W.
     perform  Get-File-Names.
     open 	input 	TextFile-In.
     open	 output  TextFile-Out.
		
     perform  Read-Textfile-In.
     perform  Process-Input-File  until  Input-EOF.
		
     perform Terminate-Routine.
     goback.	

		
 Process-Input-File.
*  this is the main routine which examines the input line for
*  any of the keywords which identify a header line.  If a
*  header line is detected, the line is skipped.  If the
*  beginning of a new posting is encountered, a separator block
*  of 3 lines is output.  Because some lines consist only
*  of a CRLF, where Fujitsu reads a record with 0 length, it
*  cannot handle writing a 0 length record, so 1 blank charater
*  is added to each record to avoid abending on the write.

     perform  Check-Input-Line.
     evaluate  True
         when  Input-Line-Valid
             move  Record-Size-In-W  to  Record-Size-Out-W
                 add  1  to  Record-Size-Out-W
             move  Record-In  to  Record-Out
             perform  Write-TextFile-Out
         when  Input-Line-Substitute
             move  5  to  Record-Size-Out-W
             move  spaces  to  Record-Out
             perform  Write-TextFile-Out
             move  40  to  Record-Size-Out-W
             move  '---------------------------------------'
                                 to  Record-Out
             perform  Write-TextFile-Out
             move  5  to  Record-Size-Out-W
             move  spaces  to  Record-Out
             perform  Write-TextFile-Out
         when  other
             continue
     end-evaluate.
     perform  Read-Textfile-In.

		
 Check-Input-Line.
*  these character strings identify header records.  The x'1a'
*  is a character that I occasionally encounter.  It is an
*  unprintable character, with nothing else on the line, so the
*  line is skipped.

     if  Record-In (1:1)  =  x'1A'
             or
         Record-In (2:1)  =  x'1A'
             or
         Record-In (1:3)  =  'To:'
             or
         (Record-In (1:5)  =  'Xref:'  or
                              'Path:')
             or
         Record-In (1:6)  =  'Lines:'
             or
         (Record-In (1:7)  =  'Sender:'  or
                              'X-Auth:')
             or
         (Record-In (1:8)  =  'X-Trace:'  or
                              'X-Cache:'  or
                              'X-Admin:')
             or
         (Record-In (1:9)  =  'Reply-To:'  or
                              'X-Mailer:'  or
                              'X-Notice:')
             or
         (Record-In (1:10)  =  'X-Mimeole:'  or
                               'X-MimeOLE:'  or
                               'X-MIMEOLE:')
             or
         (Record-In (1:11)  =  'Newsgroups:'  or
                               'Message-Id:'  or
                               'Message-ID:'  or
                               'References:'  or
                               'X-ELN-Date:'  or
                               'X-Original-'  or
                               'X-Priority:')
             or
         (Record-In (1:12)  =  'Followup-To:'  or
                               'X-Auth-User:'  or
                               'X-Post-Time:'  or
                               'In-Reply-To:')
             or
         (Record-In (1:13)  =  'Mime-Version:'  or
                               'X-Newsreader:'  or
                               'Organization:'  or
                               'MIME-Version:'  or
                               'Content-Type:'  or
                               'X-No-Archive:'  or
                               'X-Abuse-Info:'  or
                               'Distribution:'  or
                               'X-Http-Proxy:')
             or
         (Record-In (1:14)  =  'X-Problems-To:'  or
                               'X-MyDeja-Info:'  or
                               'X-NETCOM-Date:')
             or
         (Record-In (1:16)  =  'X-Complaints-To:'  or
                               'Cache-Post-path:')
             or
         (Record-In (1:18)  =  'NNTP-Posting-Host:'  or
                               'Nntp-Posting-Host:'  or
                               'X-Posted-Path-Was:'  or
                               'NNTP-Posting-Date:'  or
                               'Nntp-Posting-Date:'  or
                               'X-MSMail-Priority:'  or
                               'X-Accept-Language:'  or
                               'X-Http-User-Agent:')
             or
         (Record-In (1:20)  =  'X-NNTP-Posting-Host:'  or
                               'X-Nntp-Posting-Host:')
             or
         Record-In (1:23)  =  'http://www.dejanews.com' 
             or
         Record-In (1:24)  =  'X-Article-Creation-Date:'
             or
         Record-In (1:26)  =  'Content-Transfer-Encoding:'

             or
         Record-In (1:28)  =  '-----== Posted via Deja News'

             move 'N' to Input-Validation-Flag-W
     else  if  Record-In (1:6)  =  'From -'
             move  'S'  to  Input-Validation-Flag-W
     else
             move 'Y' to Input-Validation-Flag-W
     end-if.                     
          
		
 Read-Textfile-In.
     read  TextFile-In
         at end
             display  'Input EOF File Status = '  FileStatus-In-W 
                 move  'Y'  to  Input-EOF-Flag-W
         not  at end
             add  +1  to  Input-Line-Cntr-W
     end-read.

		
 Write-Textfile-Out.
     write  Record-Out.
     if  FileStatus-Out-W  =  '00'
         add  +1  to  Output-Line-Cntr-W
     else
         display  'Output Write File Status = '  FileStatus-Out-W

         display  '*****   PROGRAM ERROR   *****'
         goback
     end-if.

		
 Terminate-Routine.
     move  5  to  Record-Size-Out-W.
     move  spaces  to  Record-Out.
     perform  Write-TextFile-Out.
     move  40  to  Record-Size-Out-W.
     move  '---------------------------------------'
                                 to  Record-Out.
     perform  Write-TextFile-Out.

     close  TextFile-In	TextFile-Out.
     display  ' '.
     display  ' '.
     display  'Input Lines Read  = '  Input-Line-Cntr-W. 
     display  'Output Lines Written = '  Output-Line-Cntr-W. 
     display  '*****   Successful Program End   *****'.

		
	Get-File-Names.
*  this routine accepts the input path and file name from the
*  console.  It assumes drive c:, and a file extension of .txt.
*  The output file name is the same as the input, with the
*  addition of the letter 's' appended to the filename.  Note
*  that Fujitsu seems to require 8.3 path and file names, and
*  abends with long path/filenames.  I'm not sure if it was
*  necessary to right trim the resulting path/file name,
*  but I did it anyways.

     display  'key in the input folder path (no embedded spaces),'
                                 ' and press enter'.
     accept  Folder-In-W  from  console.
     display  'key in the input file name (no embedded spaces),'
                                 ' and press enter'.
     accept  FileName-In-W  from  console.
		
     string  'c:\'  delimited by size	
             Folder-In-W  delimited by space
             '\'  delimited by size 
             FileName-In-W  delimited by space
             '.txt'  delimited by size
             into  PathName-In-W.
     move  +0  to  Path-Cntr-W.
     inspect  PathName-In-W  tallying Path-Cntr-W
             for  characters  before  initial space.
     move  PathName-In-W (1:Path-Cntr-W)  to  Trimmed-Path-In-W.
		
     string  'c:\'  delimited by size 	
             Folder-In-W  delimited by space 	
             '\'  delimited by size 
             FileName-In-W  delimited by space
             's.txt'  delimited by size
             into  PathName-Out-W.
     move  +0  to  Path-Cntr-W.
     inspect  PathName-Out-W  tallying  Path-Cntr-W
             for  characters  before  initial space.
     move  PathName-Out-W (1:Path-Cntr-W)  to  Trimmed-Path-Out-W.
```
