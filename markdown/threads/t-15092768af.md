```
This is for zoned numeric on an IBM mainframe (signed numeric fields, rightmost
"digit", display format).

Unsigned: characters 0123456789 (hex F0 through F9).

Signed positive: {ABCDEFGHI (hex C0 thorugh C9) Note: for reading zoned
numeric, the "usigned" characters will produce the same numeric results as the
"signed" characters. If doing numeric compares, it will work.

Signed negative: }JKLMNOPQR (hex D0 through D9)

These are in the EBCDIC collating sequence. The { and } might not translate
correctly during upload/download when ASCII<->EBCDIC character translation
takes place.
```

```
Overpunching (12 ISTR) simply produced a character with a hex value of
between D0 and D9 for the negative sign. No overpunching was needed
for positive numbers because that's what they are by default, and why
F0-F9 are assumed positive in decimal arithmetic. It was usual for
only the last digit to be overpunched, since that is all that is
needed, and it still kept the number relatively legible on the card.
```
