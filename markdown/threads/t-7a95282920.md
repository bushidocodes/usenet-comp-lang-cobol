› are packed decimal and COMP-3 the same animal?
Yes. COMP-3 is stored by converting each digit of the original
number into a 4-bit (i.e. half byte) binary representation. Then,
a half-byte is added on for the sign.

I think some packed decimal formats also store the position
of the decimal place. Thios is not required for COMP-3, because
the inmterpretation of the number (i.e. the decision about where
the decimal point is) is code in the PIC clause.

Matt,

You need to be able to tell one COMP from another. COMP (or COMPUTATIONAL) is
a part of the COBOL standard. Things such as COMP-2 and COMP-3 are not. They
are implmented by various compiler vendors to use the characteristics of a
particular machine. COMP-2 will be one thing one one machine and something
else on another. This may be where your confusion is coming from. The book
COBOL For Dummies was written strictly to the ANSI standard, so it never talks
about COMP-2 or COMP-3. Find yourself a VS COBOL manual to get the information
about these special language extensions.

Also, don't let other folks opinions slow you down. We were all beginners
once. As long as you don't claim to be anything other than a beginner, you
will never harm anyone--least of all the industry. Good luck.

Arthur Griffith
