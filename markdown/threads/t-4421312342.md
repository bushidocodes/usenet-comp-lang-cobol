24 hours.  While this is not your exact error, I think it has the same

Non US readers may encounter the following error message when trying
to compile the examples in Hours 3 and 4 relating to the use of $ by
COBOL to indicate currency:

JMN1142I-S INVALID CHARACTER IS SPECIFIED IN PICTURE CHARACTER-STRING.

PICTURE 9 ASSUMED.

The simplest solution to this problem is to go into the Windows
regional settings from control panel, and set the currency symbol to
$. Another solution is to change the compiler directives from:

@OPTIONS MAIN to 

@OPTIONS MAIN,CURRENCY($)

=====================
The problem comes in when Windows has an "S" specified for the
"currency" sign.
