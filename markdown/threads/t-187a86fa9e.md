```
> I am looking for ideas regarding how to convert our COBOL program screens

If these are text mode ACCEPT/DISPLAY and you want to move to Cobol
CGI then there will be some radical changes required.

CGI in Cobol is quite easy to do, even without using proprietry
libraries, but the interaction is stateless.  Each GET or POST from a
form starts a new iteration of the program and may be from different
users.  It is usually necessary to store some state reference in a
HIDDEN field and/or in a cookie so that the appropriate state can be
recovered and the transaction continued.

You should probably use templates for the pages and merge the data
into these rather than directly outputting html, it makes it so much
easier to change the layout without changing the program(s).
```
