```
Hi all reading this,

i am loocking for a COBOL-File-Editor. What does that mean?

There is a UNIX-file (i.e. a bucket of bytes) and there is a COBOL-Copy
containing a record layout for this file. In most cases it is a fixed
record length but a file with variable record length written with
MF-COBOL 3.2 should be readable too.

The editor will be called with the name of the file to be edited an the name
of the file containing the record layout and has to display the file record
per record in its COBOL layout and the operator may or may not edit a field
in a record (at least for fixed record length).

This should not be too complicated, but i dont want to invent the wheel
again (as we say in germany :-) ).

Anyone who knows where to get such a tool (free software in source COBOL or C,
or pay software), please post a followup or email to jo··.@kph··n.de .

Joerg Ahrens
```
