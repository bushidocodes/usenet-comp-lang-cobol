Actually it is a program generator with a programming language of its
own but very similar to cobol. The generator can generate runtime
(cobol or C++) code for a number of operating systems such as
VSE/CICS, MVS/CICS, VM/CMS, OS/400, AIX, OS/2, Windows and some other
ones that i don't remember.

I contains a GUI builder based on VisualAge Smalltalk parts.

The current product name actually is VisualAge Generator (unless it
has been changed again very recently). Check the following link for
further information:

http://www.software.ibm.com/ad/visgen/

well depends on what version you are talking about... most CSP shops use
V3.3. This version does not yet generate the code into COBOL nor C.  no GUI
interface either. Just plain CICS like mainframe textbased screens. sure you
can do pop-ups... but you have to code for it!!!! and define maps....
floating points...etc...

CSP, on my opinion, is just one of the attempts to simplify COBOL/CICS
programming, NOT!!  as one of my co-workers say, it is just one way of
complicating COBOL.

CSP has about 12 verbs to use in programming. so it is very very easy to
learn.

V4 i think is the version that would do COBOL generation. The visual Gen /
visual Age
products is the new offering of IBM that will do internet commerce.

a good COBOL programmer can do CSP in no time.  but be ready to re-think how
you
do your "structured programming"...
