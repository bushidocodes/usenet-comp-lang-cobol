I had a look at this as I might have a need for something along these lines
shortly.

Here are my findings <G>:

As I am not interested in running Perl on my system at the moment I
downloaded the executable only for Windows as a self-extractor.

It extracted and installed correctly under XP Pro. No problem.

I then downloaded the documentation as a WinZIP and that also was no
problem.

The HTML documentation looks pretty comprehensive and it had a link saying
"Using txt2pdf from COBOL" (or somesuch...) which I took. It took me to the
aboutlegacycoding site but returned a 404., so no joy there...

I decided to just test it so I did.

I tried it with a word document and it failed... well, of course, it is
called TXT2pdf, after all...<G>

I converted the word document to RTF and tried it on that... no go.

Converted it to plain text... it worked.

Good job.

Then I got to thinking...

The main reason I want to produce something as PDF is because it has images
or diagrams in it (maybe a Company logo) and I'd like it to be viewable on
all platforms.

If I can only convert TEXT why would I bother converting it at all? TEXT IS
viewable on all platforms.

If I really want it to look like a WORD document, containing images, (and I
do...) then I can convert it to HTML and that is also viewable on all
platforms (well, all the platforms I'm interested in...).

I guess I'm just missing something here, but unless this tool can convert a
document in RTF or DOC format (which may contain some images), what useful
purpose does it serve?

Pete.
