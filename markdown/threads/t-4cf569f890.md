```
If you want both a group-item and all of its subordinate elementary items to
be available as "numeric" fields, then your first method is THE way to go.
It is easy to get a group item as alphanumeric and all the subordinate items
as numeric (which is what you actually always get  - by default) and is what
happens with the 2nd set of code that you have shown.  You can also -
sort-of - get a pseudo-group item as numeric and subordinate items as
alphanumeric by using reference modification of a USAGE DISPLAY numeric
elementary item, but in that case the subordinate items don't get a name -
which is OK for an occasional use, but not great for maintenance.
```
