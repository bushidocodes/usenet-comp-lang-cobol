I am considering taking a contract to upgrade IBM mainframe applications 
from Cobol-74 and Cobol II to Cobol-390. 

The company is already using LE, but since all database IO is done by 
Assembler IO modules, there is some confusion/uncertainty about link

straightforward, recompiling & testing.  The upgrade from Cobol-74 will

Trace), and replacing those constructs with newer variations (perhaps 
declaratives).

Can anyone think of any other considerations, or point me to 
readings/resources that might be helpful ?

Bob

Huh?

One does not need to linkedit a routine AMODE(ANY) to exploit XA storage. 

In fact, all linking AMODE(ANY) does is tell the contents supervisor that the
routine should be invoked in the AMODE of the caller (and does force the
loader to load the program in 24 bit storage).

Also, old assembler routines cannot be safely assumed to be able to run in 31
bit mode. It is quite possible that the top byte of registers are used for
data and not cleared.
