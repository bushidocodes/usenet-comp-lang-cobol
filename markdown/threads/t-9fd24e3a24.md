Oh Goody, a chance to write some COBOL.... :-)

 Working-Storage Section.
 ...
 01  name pic x(40).
 01  reversed-name pic x(40).
 01  word-array.
     12 word pic x(30) occurs 3
             indexed by wd-x.
 01  ptr  pic s9(4) comp-5  value 1.
 01  J  usage index.
...

 Procedure Division.
 main section.
 p000.
     move spaces to word-array
     set wd-x to 1
     perform
         varying wd-x
            from 1
              by 1
           until wd-x > 3
                perform until word (wd-x) not = spaces
                              OR
                             name (ptr : function LENGTH(name)
                                             - ptr) = spaces
                     unstring name
                          delimited by space
                             into word (wd-x)
                          with pointer ptr
                     end-unstring
                end-perform
     end-perform
     move spaces to reversed-name
     perform
        varying wd-x
           from 3
             by -1
          until wd-x NOT > zero
             if word (wd-x) NOT = spaces
                string
                   word (wd-x)
                      delimited by space
                   "$"
                      delimited by size
                         into reversed-name
                end-string
                set J to wd-x
                perform
                   varying wd-x
                      from 1
                        by 1
                     until wd-x = J

                       if word (wd-x) NOT = spaces
                          string
                             reversed-name
                                delimited by space
                             word (wd-x)
                                delimited by space
                             "$"
                                delimited by size
                                   into reversed-name
                          end-string
                        end-if
                 end-perform
                 set wd-x to 0
                 *>set wd-x down by 1
             end-if
     end-perform
     inspect reversed-name replacing ALL "$" by space
     .
 p999.
     stop run.

There are some advantages in the above that are not so easy if you do it 
byte-by-byte...

1. There can be multiple spaces between names. It will always resolve it to 
a single space. Often, with names, people mis-type and get two or more 
spaces between the words.

2. It correctly deals with a middle name or initial whether it is present or 
not.

Although it looks pretty dreadful here, if you paste this code into your 
COBOL IDE it is actually quite elegant. As soon as you get it out of the 
proportional font it is in here, it immediately looks much better. 
Everything nested is paired and indented and all scope delimiters are 
properly matched.

But probably most importantly, it works correctly. (Tested with Fujitsu 
NetCOBOL Version 9)

(Although I enjoyed writing it, I still prefer C# :-))

This problem really comes down to detecting the last word in the name string 
and creating a new string with that word at the start, and the remaining 
words concatenated into the new string.

There are many ways to do that which are better than the above but they are 
not necessarily available in COBOL. The "best" way is probably with a 
Regular Expression, for instance.

Since regular expressions are not available in standard COBOL,
their use cannot be "best". <g>

Nonetheless, there are features in standard COBOL that simplify
the task.

=====
      $set ans85 flag"ans85" flagas"s"
       identification division.
       program-id. str-fmt.
       data division.
       working-storage section.
       1 in-name pic x(40) value spaces.
       88 test-1 value "John Q Smith".
       88 test-2 value "Sally Jones".
       1 out-name pic x(40).
       1 work-fields.
        2 fld-cnt comp pic 9(4).
        2 fld-1 pic x(20).
        2 fld-2 pic x(20).
        2 fld-3 pic x(20).
        2 fld-4 pic x(20).
       procedure division.
       begin.
           set test-1 to true
           display in-name
           perform reformat
           display out-name
           set test-2 to true
           display in-name
           perform reformat
           display out-name
           stop run
           .
       reformat.
           initialize out-name work-fields
               replacing numeric by zero
                   alphanumeric by spaces
           unstring in-name
               delimited by all spaces
               into fld-1 fld-2 fld-3 fld-4
               tallying fld-cnt
           evaluate fld-cnt
           when 3
               string fld-3 delimited spaces
                   space delimited size
                   fld-1 delimited spaces
                   space delimited size
                   fld-2 delimited spaces
               into out-name
           when 2
               string fld-2 delimited spaces
                   space delimited size
                   fld-1 delimited spaces
               into out-name
           end-evaluate
           .
       end program str-fmt.
===== (output)
John Q Smith
Smith John Q
Sally Jones
Jones Sally
=====

Notice that 'reformat' requires only five instructions with
no programmer supplied loops, versus fifteen instructions
and four loops.

I used four fields in the 'unstring' statement and coded the
'evaluate' so that, if the number of fields is other than
two or three, 'out-name' will contain spaces (see 'initialize').
That makes it easy to check for errors in the input data.

We like our names to be Last-name and suffixes, comma, honorifics, given 
names.
Jones, Tom
Smith M.D., Bob
Smith, Dr. Bob
White Jr., Bill

This is OUR standard, but it makes it easy to reformulate.

To begin with, does the supplied name contain a comma? If so, presume the 
name conforms to the above standard and pass it unchanged.

If no comma, UNSTRING the name into ten or so fields. Look at the last 
non-blank field. If it is a known suffix (Jr, II, III, IV, V, MD, M.D., RN, 
R.N. etc, concatenate it with the next to last field.

Now put the last field as the first item in the output field. Blank the last
