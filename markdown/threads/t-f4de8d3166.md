```
Has anyone managed to get table handling down in cobol 6.50? It appears
indexed by isn't supported in that syntax. As is normal, documentation
isn't present either in source code or documentation from what I've
found. If the following helps any, copy files do work. COPY
FILENAME.CB1. gets that done. inspect also works in cobol
6.50. To change all instances of lowercase "a" in a variable to
uppercase "A": INSPECT VAR REPLACING ALL "a" BY "A". gets that done.
All of this appears to be from a very old standard of cobol. Materials
I'm using are evidently a couple versions newer. To force all lowercase
in a string to uppercase try 26 inspect statements in a copy file with
extension .cb1 and pull that into procedure division where needed. One
final clue, cobol 6.50 likes its occurs clause on a lower level number
line than 01. From what I can determine search isn't supported unless
there's alternate syntax for indexed by either.

To the vendors on the newsgroup, I don't do windows. It's either msdos
with the cobol or no cobol.
```
