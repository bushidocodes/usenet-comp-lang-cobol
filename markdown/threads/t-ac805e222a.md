Around 20 years ago Report Writer was unpopular with some of our contract 
programmers because they were paid by number of lines of code. When print 
program specs were doled out, the managers always worked on the 
assumption that COBOL was strictly a third generation language, so there 
was a fairly generous expectation for print programs. Now a programmer 
using Report Writer was in a fix. He loses one line per "variable" in the 
printout because RW does it in one line with a SOURCE (cutting out the 
MOVE). Then he loses almost half the fields used for print-line 
definitions because RW doesn't need FILLERs. So the ones that were 
enterprising enough to teach themselves the RW syntax and used it had 
their programs retrospecively re-assessed. This affected not only the RW 
users but also the ones that didn't use it, because the effect was 
averaged over the whole project.

My most successful project had a timescale that couldn't have been met 
with just "vanilla" COBOL, and for some good reason the whole thing had 
to be written in COBOL. Furthermore the project was top-heavy with 
printouts. So a taskgroup was formed to learn the RW syntax (not an easy 
matter as there were no texts other than the Language Reference - ugh!) 
and teach it to the others. To cap all this there was a commandment "Thou 
Shalt Use Report Writer" for every program that wrote anything to the 
printer. This forced everyone to get to know it. The opinion most people 
took away was that it was soooo much easier than it looks in the Language 
Reference. I'd recommend Steve Eckol's book for those who don't know it. 
There are some good courses now (just do a websearch on "COBOL Report 
Writer").
