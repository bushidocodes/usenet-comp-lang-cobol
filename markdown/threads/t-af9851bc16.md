I'd recommend scratching the system call, and doing a "fork" and "exec".
The code could look something like this:

#include
#include

main {
int pid;

printf("Here we are in a C routine\n");
if ( (pid = fork()) == 0 ) { /* See if we're the child */
execlp("PROGRAM1", "PROGRAM1", (char *) 0);
perror("execlp - PROGRAM1");
return(1);
}
else if (pid < 0) {
perror("fork failed");
return(1);
}
wait((int *) 0);
printf("PROGRAM1 has finished running now\n");
exit(0);
}

------
In a nutshell, the "fork" function "clones" your C-program. After the
fork, you'll have 2 copies of it running at the same time. Both
programs will be identical, having the same memory, environment, open
files, and will be executing at the same place in the code. The only
difference between them is the value returned by the fork function. Zero
is returned for the clone. A positive value for the original. A negative
value indicates an error.

The "execlp" will terminate the currently executing program, and replace
it with whatever you tell it to execute.

The "wait" will suspend the original program until the clone finishes.

One problem you may run into: It's possible that anything you type will
be accepted as input for both programs. You may need to clear the
standard input buffer after the "wait" in the original program.

Hope this helps.

Ken Row
