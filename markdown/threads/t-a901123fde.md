- **Message-ID:** `<4vjb19$lok@globe.indirect.com>`

```

Situation - a COBOL program is running and dynamically calls a COBOL subroutine.
The first time the subroutine is called, the module is loaded into memory and used
during subsequent calls.

Question: what happens to the main program when the subroutine is recompiled
and re-linked while the main program is still running? Will it abend? Continue to
use the (now old) module in its memory? Detect the new module and reload it the
next time it is called?
I ran a test on this this morning and it did not abend, but I don't know if it picked up
the new module or not.

The OS is MVS, all programs are COBOL II.

TIA,
Doug McKibbin
```
