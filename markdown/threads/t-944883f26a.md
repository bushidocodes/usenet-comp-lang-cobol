```
pty0 wrote:

> I'm looking for the API function or another method that allows me to
> identify who's using an EXE file in a network. This is, I have an EXE
> file, for example MENU.EXE, that is shared by other users in a network
> (Windows 2000), and I want to know if this EXE is active (being used by
> someone) and if possible to know who's using it.

Have the program write a record into a shared file on the server and delete 
that record when the program exits.  Use the machine name and process-id 
(or window handle) as record key as there may be more than one copy running 
(especially on TSE).

Of course spurious records may become lodged in the file if client machines 
crash or reboot or ...  Have the server startup delete the file or use some 
other clean-up.
```
