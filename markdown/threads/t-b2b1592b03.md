Report on August 5-9, 1996 ANSI X3J4 COBOL Committee meeting,
Minneapolis, Minnesota, USA. When a vote is shown, the numbers are
for, against, and abstain (f-a-a).

Membership stands at 11, with one new organization (Fujitsu) being
added after the meeting. The primary topic at this meeting was
action for the CD (committee draft) review at ISO. The latest
version of the base document has been forwarded to ISO for this
review. The official start is expected in mid-September and the end
in mid-December.

The major topics/proposals we discussed were:

1. There was a great deal of discussion about the CD review. The
main points were: how are we going to respond to comments, will we
need another review; what can we add (the document is supposed to be
more or less frozen as far as features are concerned); and so on.
All of this was ironed out. We decided that we can add a few
additional new features that are not major changes.

2. We added a proposal to allow INITIALIZE to default to
initializing FILLER items. You add a clause in the OPTIONS paragraph
to do this. Even though this was not exactly editorial, WG4 said
they wanted it.

3. WG4 had said they wanted the method of specifying automatic and
static storage allocation to be done with a local storage section (as
in some current implementations) as opposed to using AUTOMATIC or
STATIC on a record or FD. We were not able to get this done for the
CD, so AUTOMATIC and STATIC are in this version. A proposal is being
developed to take them out and add local storage section. It will
not be possible to make a file connector automatic or static, but the
committee (except for me) felt that compatibility with current
implementations is more important.

4. A proposal for implicit qualification of data-names in the OCCURS
clause was discussed and sent back for rewrite.

5. It was decided that we would require items with a usage of bit to
be aligned on word boundaries when passed by reference. A proposal
will be written for this.

6. It was decided that the syntax for repeating table items with
VALUEs was too complicated. I will revise the proposal that brought
this up.

7. My proposal on problems with non-numeric figurative constants
moved to numeric items was discussed, passed, and will be sent to WG4
for approval. WG4 originally wanted this to be archaic, but we feel
that it should not be allowed (it is the source of lots of bugs).

8. My proposal on constants was discussed. We decided that they
could be on 01-levels only, they can have GLOBAL, they can be
anywhere in the data division, and they can be duplicated as long as
the definitions are the same. It was sent back to me for rewrite.

9. My proposal on allowing numeric functions to be used anywhere a
numeric sender can be used was approved and will be added.

10. My proposal on READ PRIOR and START LESS THAN was discussed and I
will rewrite it. READ PRIOR was changed to READ PREVIOUS.

11. The proposal on allowing a class test for numeric on a non-
display item (such as packed-decimal) was discussed and will be
rewritten.

12. A proposal on allowing underscore to be used in the same way as a
hyphen for user-defined words was discussed and assigned to me for
rewrite.

13. My proposal on allowing literals to be inserted in editing and
any literal for sign-sensitive editing and floating editing was
discussed and sent back to me for rewrite.

14. A proposal to allow a nonnumeric VALUE to be specified without a
PICTURE passed and will be added. The PICTURE is assumed.

15. A proposal to allow alternate key suppression was discussed and
reassigned to me for rewrite.
16. We went through all of the items that were being considered for
the standard. Most were changed to be worked on for the next
standard after this one.

17. Several interpretations on the current standard were discussed
and a couple were passed.

18. The rest of the time was spent on correction and clarification
proposals. No major changes were made.

The next X3J4 meeting will be in Chelmsford, Massachusetts, USA, on
October 14-18, 1996. The main topic will be addressing comments on
the CD review of the base document and the above proposals that were
rewritten.
