Offhand, it looks like you don't have enough DOS memory to run your
program. The easiest fix would be to upgrade to v6 of the runtime,
which will execute your code in DPMI memory. You could also try
disabling the runtime buffer pool to make more memory available:

RUN-FILES-ATTR BUFFER-POOL-SIZE=1

in the runtime configuration file will disable buffering, and free
up a significant chunk of memory. However, file I/O performance
will be impacted. If that works, try increasing the value 16K at
a time.

If you don't use SORT or MERGE, you can also reclaim the memory
used by that subsystem by specifying T=0 on the command line.

I hope this helps.
