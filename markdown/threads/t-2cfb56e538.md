A while ago, there was a prolonged thread about "expandable" tables. I
thought I would put out (in the newsgroup) a description of a substantive
change coming in the next COBOL Standard (always subject to changes to the
draft). If this is a feature that you are interested in sooner rather than
later, you might want to contact your COBOL compiler vendor and determine if
it could be added (as an extension) before the next Standard gets approved.

"Dynamic tables. Tables may grow beyond their initial storage allocation.
They may be declared with or without a maximum number of occurrences. If
the table is specified without a maximum number of occurrences, the size of
the table is limited only by the resources available at run time. The
system automatically expands the size allocated for a table when the table
is referenced after the logical size of the table, specified by the value of
the data item specified in the DEPENDING ON phrase of the OCCURS clause,
changes. The maximum number of occurrences allowed for the table may be
increased and decreased with a new format of the SET statement. The storage
allocation/de-allocation scheme is implementor-defined.ï¿½
