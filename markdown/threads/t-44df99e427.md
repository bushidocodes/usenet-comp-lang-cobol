Correct information follows (word-smithing is probably needed):

When any COBOL program calls any LE callable service, the COBOL Return-Code
special register is set to zero (regardless of whether or not the callable
service was successfully executed). Similarly, when a COBOL program calls any
other non-COBOL subroutine (whether user-written or vendor supplied), the
return-code special register reflects whatever is in R15 of the called
subprogram. See the warnings at:

http://publibz.boulder.ibm.com/cgi-bin/bookmgr_OS390/BOOKS/igy3pg20/3.1.1.7

and

http://publibz.boulder.ibm.com/cgi-bin/bookmgr_OS390/BOOKS/igy3pg20/3.2.1.4

Therefore, if your LE Enclave has a COBOL program as its main program and an LE
condition handler may (or may not) have been activated within the run-unit, it
is important that immediately before your STOP RUN or GOBACK statement (in the
main COBOL program) that you make an explicit call to the CEE3GRC callable
service and move the resulting Enclave Return-Code to the COBOL RETURN-CODE
special register. If your application may have explicitly set the RETURN-CODE
special register, then you should also use the CEE3SRC callable service to set
the value in the LE Enclave Return-Register; otherwise this value may be
overwritten by a final call to CEE3GRC before the GOBACK or Stop Run statement
and/or the return-code value will not be percolated to non-COBOL LE conforming
main programs.
