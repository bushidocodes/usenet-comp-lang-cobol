- **From:** "Sal Cambareri" <sal@lasermax.com>

```
I have recently started using the Casegen/Fujitsu COBOL package, that I
licensed last summer.  While porting an application that I had previously
written for Microfocus COBOL and Acucobol, I have encountered a problem. I
hope someone can point me in the right direction toward a solution.

The application allows the user to write (binary) information directly to
any printer port (or to a disk file).  It is crucial to the application that
the operator be able to select the file name from within the application.
In Microfocus and Acucobol I did this very easily with the select statement:

SELECT  PRN ASSIGN TO PRINTER PRN-FILE-ID
                           ORGANIZATION IS LINE SEQUENTIAL
                           FILE STATUS IS FS.

To satisy the Fujitsu compiler I changed it to:

SELECT OPTIONAL PRN ASSIGN TO PRN-FILE-ID
                                     ORGANIZATION IS LINE SEQUENTIAL
                                     FILE STATUS IS FS.

However, under Fujitsu, when PRN-FILE-ID is set to an actual device name
such as LPT1, I get on "OPEN EXTEND PRN" the message:

jmp03101-I  OPEN ERROR. FILE = LPT1. 'UNSUPPORT'. (address stuff . . . )

If I then try "OPEN OUTPUT PRN" I get a file status (FS) = "37".  (For
status 37, the Fujitsu documentation only tells me "The specified function
is not supported.")

The program works fine if PRN-FILE-ID points to a disk file name.  Any help
would be greatly appreciated.

Thanks,
Sal
Reply to sal@lasermax.com
```
