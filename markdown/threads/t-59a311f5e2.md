This is not a recommendation, but more of a suggestion of something to
consider.

You could look at Acucobol.  One thing that I like about it (as a software
vendor) is that it compiles to psuedo-code (p-code, byte-code,
intermediate-code, or whatever you want to call it) instead of native binary
code, although the latest release can optionally do native code on some
systems.

This means that you could purchase the development license only for Win32,
where you want to develop and test.  Since the object code is portable, you
would only need runtime licenses for the Sun development and production
systems.  There would be no recompile necessary to run on the Sun systems.

Yes, native code runs faster, but that's not always a factor - in our case,
our software is not that compute-intensive.  Portable object code is much
more important for us.  Internally, we have Win32, Linux, and AIX systems,
but our end users also have Sun, HP-UX, SCO, etc.  We don't have any of
those systems, but we don't have to compile on end user systems or go to
porting centers - the object code is all the same.

This is not a knock against Fujitsu (in fact, we are looking at their
Cobol.NET).
