```
Thanks for the heads-up.  I too have postponed installing the 6.1
upgrade for fear of unforseen hassles, at least until I'm done with some
current projects (as if that ever happens!).  Anyway, I saved your msg
for reference in case anything like this happens to me.

By the way, you may want to check into a couple things.  See if the
files you were "missing" are actually somewhere else on your system. 
Fujitsu stores a lot of excess baggage in far-off folders.  Their
install also can do severe damage to your autoexec.bat file (exceeding
the DOS length limit, unless you're using the shell command or running
NT).  
Anyway, I did a couple things which I *know* will complicate the
install.  1) I have copied several of the frequently used libraries
(f3bicblr, kernel32) into my Projects folder, which by the way is NOT in
with the obscure Fuji folders, and 2) modified my DOS path to include
Fuji's LIB folder so when I test apps outside of the IDE they can find
all the DLL's.

Just thought you might want to check and see if those "can't find"
messages merely meant the necessary files just weren't where the install
expected them to be.
```
