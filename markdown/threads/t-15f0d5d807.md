IBM VS COBOL II is either a 1985 conforming compiler with vendor
extensions, or a 1974 conforming compiler with even more extensions.
These compilers operate on MVS (XA or ESA) or VM (maybe VSE, but I do not
recall).

For releases 3.0, 3.1, 3.2 and 4.0 the mode of the compiler is set by
compiler options - CMPR2 - if set causes the 1974 interpretations to be
used.

The circa 1989 Intrinsic Function addedum is not supported on any release
of VS COBOL II. To get the intrinsic functions, one has to go to later
compilers, either COBOL for MVS and VM (current), or COBOL/370 (was
current offering till October 1995).

All of the above compilers can flag non-standard constructs using the
FLAGMIG compiler option (but you will flag a lot, i.e. quotes, COMP-3,
etc).
