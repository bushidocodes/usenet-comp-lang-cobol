Peter, 

I use these calls extensively and haven't run across this problem.
When you say the _FREE didn't 'execute', I assume you mean that
you did the call, but the memory was still allocated ?

Let me ask, how are you sure the memory is 'allocated' ?   Are 
you looking at TaskManager or something ?   I assume since you
mentioned looking for tools to view allocated that this is the 
closest you might have come to actually knowing for sure if the 
memory is allocated or not.

If you're simply accessing the memory that you _FREE'ed and not 
getting a 114 this doesn't necessarily mean the memory is still
allocated this is expecially true in multi-threaded programs.

The biggest issue I've run across with using DYN memory is making
sure the POINTER fields are reset to NULL when the _FREE is done,
also make sure you check RETURN-CODE after ALL you DYN calls.

Hope this helps.
