Alright, let's see how well my memory works... been a while.

First, one needs a SELECT INTL-RDR ASSIGN TO INTLRDR.

Next, of course,
FD INTL-RDR
quack quack quack
woof woof woof.
01 INTLRDR-REC PIC X(80).

Then, in WORKING-STORAGE one needs lines like

05 JBCD01.
10 F PIC X(20) VALUE '//JOBNAM JOB JOBNO,'.
10 F PIC X VALUE QUOTE.
10 F PIC X(14) VALUE 'SOME TEXT HERE'.
10 F PIC X VALUE QUOTE.
05 JBCD02.
10 F PIC X(49) VALUE '//
CLASS=?,NOTIFY=&UID,MSGCLASS=?,MSGLEVEL=(?,?)'.
(note: this can be broken up as JBCD01 to allow construction of
symbolics during runtime).

Finally, of course,

OPEN OUTPUT INTL-RDR.
...
WRITE INTL-RDRREC FROM JBCD01.
WRITE INTL-RDRREC FROM JBCD02.

... and that should about do it... of course, one must have the DD of

//INTLRDR DD SYSOUT=(,,INTRDR)

... or something like that, it *has* been a few years.

DD
