COMP-3 is signed Binary Coded Decimal. This mean each digit occupies 4 bits.
The last four bits contain the sign designation, 0xC for negative, 0xD for
positive if memory serves me right. COMP-6 is the same, only without the
sign.

Binary, COMP-1 is a signed 2 byte integer with the sign occupying the first
bit of the first byte (exception being compilers in reversed byte machines
that hold the value in native machine format).

Floating point binary, COMP-4 if I remember correctly, is like COMP-1 with
a value portion and exponent portion, both binary.

SYNC I'm not familiar with since it isn't really used much outside of the
mainframes, but I believe has to do with aligning fields on word boundaries
in memory. Its only affect would be field size or placement, not format.

Dear GT:
There are various computational types in COBOL. COMP or COMP-2 is usually
the same as an integer in C. It is a binary number. COMP-3 is base 10
compacted a little. Each digit in COMP-3 is stored in half a byte (a nibble).

For example, the number 1234 in a PIC 9999 comp-3 would be stored as follows:

0000 0001 0010 0011 0100 1111
1 2 3 4 weird sign in last area...

There are assembler commands on system 360 to deal with packed decimal
numbers. (Such as ZAP, zero the register and add packed)
Believe it or not, there is also floating point in IBM mainframe cobol.

comp-1.

Sync is short for synchronized, as in on a double-word boundary.

I have called assembler routines that required the parm on a double
word boundary in the address space. The system 360 word size of the CPU
was 32 bits, and the double word was 64 bits. Thus a double word
boundary would be numbered every four bytes beginning at zero...
