No, strictly you just need a resource compiler (and a linker, which
is in the Cobol product), which can be gained from most of the MS
32bit SDKs (not just the Windows 95 SDK).

› We have found that if you are willing to distribute your .ico and .bmp

Yes, you can use .ICO, .BMP, .CUR and .ANI (in later Cobol OSX product
updates) directly in your .ICN file, without any resource number attached.

The files should be on the path/system path/EXE path/COBDIR.
eg (in the .ICN):
[win-bitmaps]
apple : apple.bmp ; this is a standard Windows bitmap
[win-icons]
orange : orange.ico ; this is a standard Windows icon

ICO is Windows-format only (on Windows)

Firstly, to build the DLLs you need to have the 32-bit Windows LINK.EXE
and RC.EXE programs available.

If you had the following in an .RC file, called SAMPLE.RC

100 ICON mainicon.ico
200 BITMAP splash.bmp

then to build this into a DLL called sample.dll all you should
have to do is the following

rc sample.rc
link -out:sample.dll -dll -noentry sample.res

If I remember correctly the Microfocus docs refer to an extra file
at the link stage called dsnull.obj, or something like that. This
should not be required.
