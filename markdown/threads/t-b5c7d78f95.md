>A good thought.  What would it be any different that I/O interrupts from

>Warren Simmons

The difference is synchronous versus asynchronous.

Exceptions in COBOL are synchronized to the statements that
cause those exceptions. After processing the exception in the
declarative, control resumes at the statement following the
statement that caused the exception to be recognized.

With event-driven or interrupt-driven processing, the event or
interrupt is asynchronous to the execution of the rest of the
program. After processing the event or interrupt, control is
returned to the the operating system or runtime.
