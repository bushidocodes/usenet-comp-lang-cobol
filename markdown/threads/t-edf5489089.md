```
Hi everybody,

I've been programming in COBOL for some time, but last week I ran into a
problem I cannot seem to solve.

Since last week our team is able to use an Alpha with Open VMS (7.2), DBMS
(7.0) and COBOL (2.6). If this doesn't impress you, let me tell you: it's
quite an improvement since we have been working on one of the most worn-out
VAX-machines ever with Open VMS 6.something. However, my problem....

Because we now have the Alpha available we started compiling our sources on
the Alpha. Besides the usual alignment problems and such, we didn't find
anything strange... Until!! we started using the COBOL to bind our
DBMS-database, freshly transported from the VAX to the Alpha. We already ran
a
DBO/CONVERT and a DBO/CONVERT/CDD. So both CDD and database were converted.

But lo and behold!! our COBOL could not bind the database, although we
didn't
change anything in the code (under Data Division we have a DB
DEFAULT_SUBSCHEMA etc.... line, which always worked on the VAX).

Has anybody else ran into this problem when migrating from VAX to Alpha? And
if so, how can this problem be solved?

Any help would be greatly appreciated.

Cheers,

Bart Lupker.
gondolin@freeler.nl
```
