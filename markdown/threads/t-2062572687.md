Jimmy,

I missed the first part of this (been incommunicado for the last few
weeks), so I hope this is what you're looking for:

GUI ScreenIO provides "attributes" for each control you define on your
panel.  These attributes can be used to set the color of a control,
protect(disable) a control, make it invisible, make a control active
when the panel is displayed, and so on.  GUI ScreenIO's attributes
have their origin waaaayyy back in our old CICS days; but are more
extensive than CICS supports.

GUI ScreenIO also allows you to define multiple controls in the same
screen location.  You might do this, for example, if you want to have
a field that appears to behave differently in different situations.
You just define fields with the behaviors you want, and at runtime set
the unwanted ones to invisible (which is implicitly
protected/disabled).  

A control which is invisible is, effectively, not there at all.

It's easy to use field attributes in GUI ScreenIO because the
generated copybook names your attributes using your field name plus a
suffix for the attribute's function.  

The possible values for field attributes are documented in comments in
the panel copybook so you don't even have to look 'em up in a manual.
(Nearly all of the values you'd use to control your GUI ScreenIO
application are defined in the panel copybook via 88-level items or,
in a couple of cases, comments).

We also use attributes to inform you which field was active when
control returned to your program, whether the data in fields was
modified by the user, and a bunch of other useful stuff.
