Judging from the screen shots http://winmerge.org/about/screenshots/, winmerge looks a lot
like Beyond Compare. Key features:

.. Manually (or automatically?) MERGE differences. 

.. Compare entire DIRECTORY trees, not just two files. 
    So you can answer the question 'What's the difference between 3.0 and 3.1?', 
    referring to a large system, not one program.

Compare tools look especially polished when linked to GUI version control, such as WinCVS.
You start with a graphic treeview of a program's versions, select two of them, perhaps in
different branches, hit one key and SHAZAM, see the differences on one screen, in context
and color coded. 

Typically, WinCVS and the compare tool run on your Windows desktop, the repository is on a
remote Unix server, the programs' target machine can be anything, even a mainframe.
