Douglas, please find some coding example that may be helpful:

1) Example of code to call an OS/2 system command from VA COBOL:

IDENTIFICATION DIVISION.
PROGRAM-ID. EXAMPLE.
ENVIRONMENT DIVISION.
DATA DIVISION.
WORKING-STORAGE SECTION.
77 sysCmd PIC X(30) VALUE 'COPY C:¥FILE1.TXT C:¥FILE2.TXT'
PROCEDURE DIVISION.
CALL 'system' USING SYSCMD.
STOP RUN.

2) Example of code to call an OS/2 API function from VA COBOL:

IDENTIFICATION DIVISION.
PROGRAM-ID. EXAMPLE.
PROCEDURE DIVISION.
CALL "DOSBEEP" USING BY VALUE 2000 1000.
STOP RUN.

3) To get/set the environment variables (Copied from the IBM COBOL
FORUM, Author: Jim R Piggott):

WORKING-STORAGE SECTION.
01 w-nameZ ... (name of variable, followed by zero byte)
01 w-env-pointer usage pointer.
LINKAGE SECTION.
...
01 l-env-string pic x(200). (any length will do)
PROCEDURE DIVISION.
...
CALL "DosScanEnv" USING w-nameZ w-env-ptr
IF RETURN-CODE not = 0
GO TO (not found).
SET ADDRESS OF l-env-string TO w-env-ptr
*l-env-string is now the value followed by a zero byte

4) Example of code to call a REXX procedure from VA COBOL (Copied from
the IBM COBOL FORUM, Author:Jim Grochala.

01 WS-Work-Fields.
05 WS-Result-Area PIC X(255) Value Spaces.
05 WS-Argument-Area PIC X(255) Value Spaces.
05 WS-Parm1 PIC X(50) Value Spaces.
05 WS-Parm2 PIC X(8) Value Spaces.
01 WS-RexxStart-Parameters.
05 WS-Rexx-Argument-Count PIC S9(9) Value +1 COMP-5.
05 WS-Rexx-Argument-List.
10 WS-Arg-Length PIC 9(9) COMP-5.
10 WS-Arg-Pointer POINTER.
05 WS-Rexx-Program-Name PIC X(255) Value Low-Values.
05 WS-Rexx-Env-Name PIC X(20) Value Low-Values.
05 WS-Rexx-Return-Code PIC S9(9) Value 0 COMP-5.
05 WS-Rexx-Result.
10 WS-Result-Length PIC 9(9) COMP-5.
10 WS-Result-Pointer POINTER.
05 WS-Rexx-Interpreter-RC PIC S9(9) COMP-5.

Set WS-Arg-Pointer To Address of WS-Argument-Area.
Move Length of WS-Argument-Area
To WS-Arg-Length.
String WS-Parm1 Delimited by Space
' ' Delimited By Size
WS-Parm2 Delimited by Space
Into WS-Argument-Area
End-String.
String 'MYCMD' Delimited by Size
X'00' Delimited by Size
Into WS-Rexx-Program-Name
End-String.
String 'CGISRV' Delimited by Size
X'00' Delimited by Size
Into WS-Rexx-Env-Name
End-String.
Set WS-Result-Pointer To
Address of WS-Result-Area.
Move LENGTH OF WS-RESULT-AREA To WS-Result-Length.

Call 'RexxStart' Using
By Value WS-Rexx-Argument-Count
By Reference WS-Rexx-Argument-List
By Reference WS-Rexx-Program-Name
By Value 0
By Reference WS-Rexx-Env-Name
By Value 0
By Value 0
By Reference WS-Rexx-Return-Code
By Reference WS-Rexx-Result
Returning WS-Rexx-Interpreter-RC.

I agrre that IBM should give a little more explanation on how to
convert the C api calls to Cobol calls, but a few rules will help


c data type cobol

signed INTEGER S9(9) COMP-5
unsigned INTEGER 9(9) COMP-5
small Integer s9(5) comp-5
unsigned small INTEGER 9(5) comp-5

string data is always passed via a pointer. In C, the end of a string
is denoted by a NULL x'00', so before you can pass a string you have
to place x'00' as the last character. You can use the string statement
to do this. For example you have NAME x(20) that you want to pass
to a api call, you can code STRING NAME DELIMITED BY SPACE
x'00' DELIMITED BY SIZE
INTO WS-TEMP-STRING
the receiving string must be at least 1 byte larger than the original
to allow for the x'00'

pointer is that same thing as a address

if the call requires a pointer to a pointer, you would code it as
03 ws-pointer pic pointer
then in the procedure division you would need to
SET WS-POINTER TO ADDRESS OF ...........


in the actual call, if C asks for a value then code USING BY VALUE
if the call asks for pointer, code BY REFERENCE
to get the api return code code RETURNING

example:
DosBeep requires 2 unsigned integer values

03 ws-frequency pic 9(9) comp-5
03 ws-duration pic 9(9) comp-5
03 ws-return-code pic 9(9) comp-5

move 300 to ws-frequency
move 1000 to ws-duration (in miliseconds
1000 miliseconds = 1 min)

call 'DosBeep' using by value ws-frequency
by value ws-duration
returning ws-return-code

if you dont want to change the values, you could also code

call 'DosBeep' using 300 1000 returning ws-return-code

the returning is optional, but it does give you a error message
if the call was not sucessful
