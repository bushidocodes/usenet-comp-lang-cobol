Did you mean to change the DDNAME? This is likely to cause problems. Use
the same DDNAME you'd use for SYSOUT=* and code file parameters,
instead. One suggestion regarding a previous example which contained
"BLKSIZE=13000", code "BLKSIZE=0", assuming you're running MVS/ESA, or
higher. The access method will determine an optimum BLKSIZE for the
device (usually half-track or third-track, depends on track size).

BTW, coding ",DCB=(,,,,)" has become a throwback to the Jurassic age of
MVS JCL. The former keyword subparameters of the DCB= keyword can now be
coded as parameters on their own, e.g.:

//SYSPRINT DD DSN=some.file.name,
// DISP=(,CATLG),
// SPACE=(CYL,(12,12),RLSE),
// DEVICE=something,
// LRECL=133,
// DSORG=PS,

and so on. The model (DCSB?) you use when creating generations of a GDG
is positional, so it's still DCB=. Someone may say (quite rightly) that
coding CYL in the SPACE= parameter is also Jurassic (agreed), but I
don't remember how to code for MB's instead without looking it up .

Bill Lynch
