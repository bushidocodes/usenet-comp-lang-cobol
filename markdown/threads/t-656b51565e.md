The DATEOFFSET attribute causes any current date call (including FUNCTION
CURRENT-DATE and ACCEPT FROM DAY/DATE) run on that stack to get a date
that many days in the future.  The attribute is inherited and may be
modified or interrogated anytime.  The range of the attribute is 0 to
8191.  Too bad it won't go negative; it would be useful for reposts.

This attribute was added recently.  It is not in the base release of the
44.1 or 44.2 MCP, but it's in an early patch release.

For users of Unisys A Series COBOL74, the extensions toACCEPT FROM DATE and
ACCEPT FROM DAY described in the draft standard are available in SSR 45.1.
A parallel feature for the Unisys extension TODAYS-DATE is being introduced
in SSR 46.1.  And the current Interim Correction versions for all supported
releases of A Series COBOL74 contain these features.

For users of Unisys A Series COBOL85, all three of these features are being
introduced in SSR 46.1 and are also available in the Interim Correction
versions for all supported releases of A Series COBOL85.

The syntax for these three features looks like:
    ACCEPT <numeric-item-1> FROM DATE YYYYMMDD.
    ACCEPT <numeric-item-2> FROM DAY YYYYDDD.
    ACCEPT <numeric-item-3> FROM TODAYS-DATE MMDDYYYY.
The intent of each of these, as well as the capacity requirements of the
three destinations, should be obvious from the syntax.

    -Chuck Stevens
