I'm interested to know whether it's possible to do the following:

In the SCREEN SECTION you define a data input screen of some sort,
such as

  05 LINE 1 COLUMN 1 VALUE "Enter Name: "

DISPLAY Data-Input-Screen
ACCEPT Data-Input-Screen

ACCEPT  to instead move the data from working storage?

Yes, you can accept from command-line, from line xx column yy or you can
accept from a screen, defined in screen section.

If you look in language reference, you will see, that they describe 3
formats, and I can recommend that you use accept format 3 - That's the one
you show in your example. But you should refine it a bit. Try this (most
compilers will allow you to do it)

DISPLAY Data-Input-Screen
ACCEPT Scr-Name

Why? Because it allows you to do field-by-field validations and it is
probably what you want.
Adding some colors to the screen is another thing you can play with. All the
time, all your screen definitions are at one place, and very easy to
maintain.

Regards
Ib

Yes,
That's lesson 2 :-)
You can accept from fields in screen section, but you can display them as
well, and you don't need to move them. In screen section you say USING
ws-something or FROM ws-a-read-only-field

You can get extreemly high performance if you only display what's needed. so
from your sample, you can easily write:

Display Scr-Name
Accept Scr-Name

Or, whenever your WS-name changes (E.g. after reading from a file) you can
say

Display Scr-Name

Another gimmic is to define a Screen with date and clock (without Blank
Screen). You can display it from time to time. An idea could be to make a
loop with and 'Accept <some field> with timeout 1'. Every second the accept
times out and you can say 'Display time-scr' and return to the the original
accept. Then you have a running clock on the screen.

Hope it helps.
Ib
