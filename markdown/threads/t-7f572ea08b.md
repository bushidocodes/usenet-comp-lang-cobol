idxformat is similar to idxformat "0" (On pc's), but
remove limitations on number of records that may
contain the same thing in a key that allows
duplicates.  The standard "0" allows no more than
65,535 records with the same thing in a key field.

Also improves speed under those conditions.



    0   System specific default (for this COBOL system, same as 3).
    1   C-ISAM format.
    2   Micro Focus Level II format.
    3   The format used by this COBOL system.
    4   An optimized form of the format used by this system, for fast
        duplicate key handling.
    5   Btrieve format (with ANSI conformance emulation).
    6   Btrieve format (without ANSI conformance emulation).


There may be more in NetX.

Another possibility is to use an ODBC driver for the Micro Focus COBOL files
and use the Perl ODBC library.  This is quite easy on Windows; on Unix it
would require downloading/installing iODBC.  While I haven't tested this
solution with Perl, I have used PHP with success, so it should work with
Perl as well.
