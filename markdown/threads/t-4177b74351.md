```
This question is asked so often, it ought to be in the FAQ. 

There is no standard format for a "Cobol file". Each compiler supplies
a proprietary file system (a few use semi-proprietary such as VSAM or
Btrieve). Generally, the file must be read by a Cobol program or
utility.

If records do not have binary nor packed decimal fields, use a utility
that came with the Cobol compiler to copy the file to flat sequential.

If records do have binary or packed decimal fields, or the above is
not feasible, you need a Cobol program that converts them to display,
and probably converts the record format to comma-delimited or
tab-delimited (csv or tsv). This can be accomplished in several ways:

..Use the cob2csv program I posted here. It must be compiled once with
the Cobol compiler on the source machine.
..Write a custom program in Cobol.
..Convince the people at the source to give you a flat (LINE
SEQUENTIAL) file. 
..Convince the people at the source to use a real database that you
can read.
..Write, in Cobol, a callable function that returns the next record in
a format you can use.
```
