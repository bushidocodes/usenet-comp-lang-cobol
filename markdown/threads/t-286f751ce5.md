```
Fixed Block Ansi - the output contains the Ansi characters for printer control.

FBM = Fixed Block Machine where the printer control characters are machine
dependent.

In MVS we put the output to a file defined FBA and a record length of 133
instead of 132 (the FD in the program I believe is 132 though), and then use
the utility IEBGENER to print out the report when necessary.
```

```
I passed on your question to one of our VSE peopl and this is what he
said.  I hope it helps.

The "RECFM=FBA" is a parameter of the DCB option in MVS JCL and
indicates that a print file will be fix blocked and contain ASA type
printer carriage control characters in the first position of the
record written by the Cobol program.  There is NO equivalent parameter
in VSE JCL for specifying printer carriage control characters.  All
this information comes from the Cobol program in VSE.

The compiler option ADV/NOADV indicates whether or not the compiler
needs to adjust the size of the print record by one byte to contain
the carriage control character.

The "ADV" compiler option is only meaningful if the
WRITE.....ADVANCING format of the write statement is used when
printing lines.  If ADV is specified in this case, the compiler will
add one byte to the record length of the print file to accommodate the
carriage control character.

If the record length, and print line layout, already contains a byte
for the carriage control, then the NOADV compiler option should be
used.

01  PRINT-LINE.
     12  FILLER            PIC X(01).
     12  PRINT-REC      PIC X(132).

The Cobol for VSE Programming Guide contains a good explanation of the
use of this compiler option.
```
