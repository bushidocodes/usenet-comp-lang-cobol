The following is the note that I received from one of my "usually reliable
sources",

PN79703, <is the relevant APAR but,> it is also in the LE Programming
Reference, from Release 6 on. LE Release 6 was shipped with OS/390 R2, so
every OS/390 customer should have a new enough copy...

 Here is the text of the APAR:

Make the following changes to the manuals:

COBOL for MVS & VM Programming Guide  SC26-4767-01
  In the Release 2 book on page 451, change the NOTE: to read:

  "Note: There is no way to get 4-digit year dates using
   OS/VS COBOL programs even if they are running under Language
   Environment.  You can use DYNAMIC calls from VS COBOL II
   programs to Language Environment Date/Time callable services.
   You cannot use STATIC calls from VS COBOL II programs to
   any Language Environment callable services."

   The list of LE services that can be dynamically called
   from VS COBOL II (any release) programs is:

         CEECBLDY      CEEISEC
         CEEDATE       CEELOCT
         CEEDATM       CEEQCEN
         CEEDAYS       CEESCEN
         CEEDYWK       CEESECI
         CEEGMT        CEESECS
         CEEGMTO       CEE3CTY

COBOL for MVS & VM Migration Guide    SC26-4764
  In Appendix A, "Commonly Asked Questions", change the answer
  to the first question under the heading "Language Environment
  Services".


  > Can VS COBOL II programs CALL Language Environment callable
    services?

   "You can use DYNAMIC calls from VS COBOL II programs to
    Language Environment Date/Time callable services.
    You cannot use DYNAMIC calls from VS COBOL II programs to
    other Language Environment callable services, nor can you
    use STATIC calls to any Language Environment callable
    services from VS COBOL II programs."

LE/370 Programming Reference SC26-3312-00 (Release 3),
LE for MVS & VM Programming Reference SC26-3312-01 (Release 4)
                                  and SC26-3312-02 (Release 5)
  Add the following note in 3 places after the bulleted list
  of languages which can call LE callable services:
    Release 3, at the beginning of chapter 2, page 29,
    Release 4, at the beginning of chapter 3, page 42, and
    Release 5, at the beginning of chapter 3, page 49

 "NOTE: You can use DYNAMIC calls from VS COBOL II programs to
        Language Environment Date/Time callable services.
        You cannot use DYNAMIC calls from VS COBOL II programs to
        other Language Environment callable services.
        You cannot use STATIC calls from VS COBOL II programs
        to any Language Environment callable services.

        The list of LE services that can be dynamically called
        from VS COBOL II (any release) programs is:

             CEECBLDY      CEEISEC
             CEEDATE       CEELOCT
             CEEDATM       CEEQCEN
             CEEDAYS       CEESCEN
             CEEDYWK       CEESECI
             CEEGMT        CEESECS
             CEEGMTO       CEE3CTY "
"
