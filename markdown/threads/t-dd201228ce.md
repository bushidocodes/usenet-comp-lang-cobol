First check you table definition and make *sure* you are allowin multiple
values in there. use something like this to test it:

select hhName, hhID from Household order by hhID;

You may have hhID in there as a unique index or something.

I just quick wrote a little table and here are the results:


› SQL> select * from hhTEST order by hhID;
› 
…[11 more quoted lines elided]…
› 8 rows selected.
 
 
› When I select for what you asked for I get:
 
 
› SQL> select hhName, hhID from hhTEST where hhID IN (1,2,3,4,5);
› 
…[9 more quoted lines elided]…
› 6 rows selected.


I just noticed you duplicated the values in the IN clause,
which is an error and should have been rejected by your SQL
parser. Try it as above and if you don't have a unique index
in there, you should be flying buddy.

-Paul
