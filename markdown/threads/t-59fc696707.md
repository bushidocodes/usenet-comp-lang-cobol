If you can't have your Translator Options changed to support 'SP', then try
issuing a CICS 'LINK' to the CEMT-Programmable Interface Entry Point
'DFHEMTP', with the following:

03  WS-COMMAREA PIC X(21) VALUE 'SET TDQ(XXXX) OPE ENA'.

EXEC CICS LINK
          PROGRAM ('DFHEMTP')
          COMMAREA(WS-COMMAREA)
          END-EXEC.

You won't need the 'SP' option. However, you may get a NOTAUTH response when
you attampt the LINK. You won't need to specify the LENGTH as COBOL2 (or
greater) will build it for you. The one drawback is that 'DFHEMTP' does not
return a response-code, indicating success or failure. The other programmable-
interface entry point is 'DFHEMTA', but I'll save that for another time.

HTH....
