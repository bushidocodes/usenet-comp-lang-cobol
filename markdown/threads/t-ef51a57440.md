```
To anyone familiar with the RWCS:

I have recently written a number of Cobol report programs (using Microfocus
Cobol under Unix) using the facilities of the report writer control system
(RWCS) and have a question: Is it possible to cause the report output to use
form feed characters (as if a "write rec after advancing page" type
statement were used) rather than printing blank lines to fill the "page"
definition in the RD.

Such a capability would make reports more portable between printers with
slightly different page lengths without having to alter the definitions.

Any experience/advice gratefully received.

Regards,

Paul Shanley
```

Define a detail report item with the clause 'NEXT GROUP NEXT PAGE' and either
explicitly GENERATE the item or use control fields to automatically generate
the group. That will force the next detail group to be produced on the next
physical page.

```
If your only concern is whether you fill out a page (from the last
"information" on one page to the top of the next page) with blank lines or
whether you do a "page feed" control character, there is a MF
compiler-option that controls this.  I don't have my system reference with
me, but I think the name has "form feed" in it.

If your concern is doing blank lines versus control characters for skipping
1 or 2 lines (like IBM mainframe DCB=FBM or FBA) then I don't think that MF
(or the PC world in general, i.e. in a printer independent way) provides you
a way to do that.
```

The Micro Focus directive that you want to look at is:

  RWHARDPAGE

It is true that the default is NORWHARDPAGE - but RWHARDPAGE is available
with all current Micro Focus compilers - and will issue a "control
character" to send out the desired control character.
