As Frederico Fonseca has already told you, Relational DataBridge makes
this quite easy. Your COBOL data, accessed directly from live data on
the SCO Unix server, appears as relational tables at the PC NT client,
using standard ODBC, which is usable by VB5 through the RDO engine (and
the ADO, as well as DAO in ODBCDirect mode).

The benefit to using ODBC, rather than writing data access DLLs (as
suggested by David Sands), is that any ODBC-compliant tool (of which
there are many) can have access to the data. And, of course, using the
RDO engine in VB5 is a lot easier than calling DLLs.
