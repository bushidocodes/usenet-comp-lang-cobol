I just completed a project request that is similar to this. the process was
to format messages that can be sent through email, notifying various
recipients of late orders. Process use, Cobol II,DB2 in OS390. i needed to
have multiple files to set up my messages because different messages can go
to different recipients (more like your institutions). what i did was
format the files and separate each group by job control cards. and then
submit the jobcontrol cards via iebgener on the next job step to submit it
(checking the return code prior to submitting generated jobs, return code =
2, then no jobs were created). the number of jobs spawned will be equal to
the number of files/email messages generated. in each job generated there
is an iebgener to create the files. BUT, i was limited to 80 bytes length
which is enough for an email message/report formatting. then I save each
job created in a GDG for restart/rerun purposes.

hope this helps.

ps. if you find a way to increase the limit on the length, give me a buzz.

Aedehl
