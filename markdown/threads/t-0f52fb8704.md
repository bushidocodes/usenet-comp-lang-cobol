If the CALLed program is not CANCELed it should remain as it last
exited and so it can keep the file open. I do this frequently,
load a file handler which opens the file on the first call and
then just accesses it when called to do so.

To be tidy there should be a 'close file' call but a CANCEL
should cause the file to be closed correctly when you have
finished with the routine.

If the subroutine is actually a called COBOL program, you need to keep the
program in memory by not CANCELLING it on return and keep the files open in
the called program until its use is complete. To effect this, you may want
to pass a function flag to the called program one of which is to open files
another to close files. If the calling routine is not a COBOL program, you
may not be able to accomplish this.

Whether or not you are using called programs, you should keep track of which
files are open using flag fields and only OPEN closed files. Remember to
CLOSE the files when you are through with them.

Example:

IF fileopenflag NOT = 1, OPEN filename ALLOWING ALL, MOVE 1 TO fileopenflag.


On final termination:

IF fileopenflag = 1, MOVE 0 TO fileopenflag, CLOSE filename.


The above sequences are significant in that they avoid repeating a
potential for error by changing the flag status in a manner that would
not cause looping with an error routine in DECLARATIVES.
