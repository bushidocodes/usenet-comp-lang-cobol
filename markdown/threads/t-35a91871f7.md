```
I tried to be a bit dynamic so it would replace multiple
occurrences (I chose 2 to test), and strings of varying lengths.
I tested this using MicroFocus Workbench 4.0.38 and the ANS85
compiler option. This worked pretty well except for one thing.
The reference modification (1:string-length) in the 'inspect
replacing' did not work as expected. Even though animator
showed that string-length resolved as 3 bytes it replaced starting
at the first 'abc' the following 20 bytes with high-values (ie the
picture length of 'inspect-string'). In other words, it appears to
have located (1:3) accurately but replaced 20. When I hard coded
'inspect-string (1:3)' the replacement occurred only on the
expected 3 bytes. I think this is a compiler bug but I'd like
anyone's opinion. Otherwise, it worked successfully.




       identification division.
       program-id. test1.
       environment division.
       data division.
       working-storage section.
       01  in-area  pic x(41) value 'This isabcmy testabcarea'.
       01  inspect-string pic x(20).
       01  string-length  pic 99.
       01  tc             pic 99.
       01  unstring-area.
           05 us-1 pic x(41).
           05 us-2 pic x(41).
           05 us-3 pic x(41).

       procedure division.
      *--> get inspect string from whatever source
      *--> using literal for test program.
           move low-values to inspect-string
           string 'abc' delimited by size into inspect-string
           move zero to string-length
           inspect inspect-string tallying string-length
             for characters before initial low-value
           inspect in-area
             replacing all inspect-string (1:string-length)
             by high-values
           move spaces to unstring-area.
           unstring in-area delimited by all high-values
             into us-1 us-2 us-3
           move zeros to tc
           inspect function reverse (us-1)
             tallying tc for leading spaces
           move low-values to us-1 (length of us-1 - tc + 1:)
           move zeros to tc
           inspect function reverse (us-2)
             tallying tc for leading spaces
           move low-values to us-2 (length of us-2 - tc + 1:)
           move zeros to tc
           inspect function reverse (us-3)
             tallying tc for leading spaces
           move low-values to us-3 (length of us-3 - tc + 1:)
           move spaces to in-area
           string us-1  delimited by low-values
                  space delimited by size
                  us-2  delimited by low-values
                  space delimited by size
                  us-3  delimited by low-values
             into in-area

           goback.
```

```
The behavior (behaviour) he describes is intended.  An excerpt from the 
2002 standard, section 14.8.39.3, paragraph 3a, states...

"Characters from literal-1 or from the content of the data item 
referenced by identifier-1 are transferred to the data item referenced 
by identifier-3 in accordance with the MOVE statement rules for 
alphanumeric-to-alphanumeric moves or, when identifier-3 is of class 
national, national-to-national moves, except that no space filling is 
provided. If an identifier-1 references a zero-length item, that sending
operand is ignored."

Note the "no space filling is provided" phrase - that's where it 
differs.  :)
```

```
You can do this automatically using the COUNT clause of UNSTRING to
give the length of the field that was output.

>            move zeros to tc
>            inspect function reverse (us-2)
…[14 more quoted lines elided]…
>            goback.

You could also do this in a loop using 'pointer's:

         MOVE SPACES TO Out-Area
         MOVE 1      TO In-Point
         MOVE 1      TO Out-Point
         PERFORM 
             UNTIL In-Point > <length of in-area>

             MOVE SPACES     TO US-1
             UNSTRING In-Area
                 DELIMITED BY <some terminator>
                 INTO US-1 COUNT IN TC
                 WITH POINTER In-Point
             MOVE LOW-VALUE  TO US-1(TC+1:1)
             
             STRING US-1  DELIMITED BY LOW-VALUE
                 INTO Out-Area
                 WITH POINTER Out-Point          END-PERFORM 
This would cater for any number of copies of <some terminator>.

However it seems that you want to remove blocks of the form
'<something>' in which case you may want to use the loop and have two
UNSTRINGs in it, one looking fof '<' and outputting to the out-string,
another delimited by '>' which discards the result but increments the
POINTER.  This way you won't care about what is between the '<' and
'>'.
```

```
It seemed interesting to me also.
Here is another sample, different method.
I address removing multiple spaces, after the offending characters, or
strings are removed.


       01  STRC-COMPRESS-STRING-WORK.

           05  STRC-STRING-IN          PIC X(100).
           05  STRC-STRING-OUT         PIC X(100).

           05  STRC-IN-LEN             PIC 9(4)  BINARY  value 41.
           05  STRC-IN-POS             PIC 9(4)  BINARY.
           05  STRC-OUT-POS            PIC 9(4)  BINARY.

           MOVE  SPACES  TO  STRC-STRING-OUT (1:STRC-IN-LEN)

           MOVE  1  TO  STRC-IN-POS
                        STRC-OUT-POS
           PERFORM  UNTIL  STRC-IN-POS  >  STRC-IN-LEN
             PERFORM  VARYING STRC-IN-POS  FROM  STRC-IN-POS  BY  1
                        UNTIL STRC-IN-POS  >  STRC-IN-LEN
                           OR STRC-STRING-IN (STRC-IN-POS:1) = SPACE
                MOVE  STRC-STRING-IN  (STRC-IN-POS:1)
                  TO  STRC-STRING-OUT (STRC-OUT-POS:1)
                ADD  1  TO  STRC-OUT-POS
             END-PERFORM
             IF  STRC-IN-POS <  STRC-IN-LEN
                 MOVE  SPACE
                   TO  STRC-STRING-OUT (STRC-OUT-POS:1)
                  ADD  1  TO  STRC-OUT-POS
             END-IF
             PERFORM  VARYING STRC-IN-POS  FROM  STRC-IN-POS  BY  1
                        UNTIL STRC-IN-POS  >  STRC-IN-LEN
                           OR STRC-STRING-IN (STRC-IN-POS:1) NOT = SPACE
               CONTINUE
             END-PERFORM
           END-PERFORM
```
