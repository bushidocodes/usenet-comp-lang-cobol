If I remember correctly, Xpeditor needs the copy in order to 
  allow a line by line step through of the probram, allowing 
breakpoints to be set, and data to be seen and/or changed 
for testing purposes.  After you feel the program is safe to 
go into production, then use COPY SUPRESS to supress the 
copy.  Most sites have two sets of compile JCL, one for 
testing which produces a copy for debugging either on 
"paper" or using a utility like Xpeditor.  The other set of 
JCL would be for the production compile and is often set up 
by the systems programmers based upon site requirements.
