RF,
The vast majority of Java programs run multi-threaded.  In addition, because
of the way Java uses exception handling, any program that wants to
interoperate with Java needs to tolerate asynchronous signals.
If you are using an IBM compiler you neet to compile your COBOL
into a DLL on the USS side of OS/390 (or z/OS)
use THREAD, RENT and DLL options.  You'll find more info on
compiling and linking for USS in the Programming Guide.  One is
available online at
http://www-4.ibm.com/software/ad/cobol/library/booksghi/igypg205.pdf

Hope this helps.

Gary
