There is no reason for users to be involved. If output files from a regression
test compare exactly equal AND test cases were sufficiently numererous to
exercise all paths, there is no change in functionality. 

Designing a test plan for 'negative-finding' tests (i.e. no difference) is
different from a test plan for 'positive-finding" tests. The number of samples
must be much larger. I generally use a full production run. The reason is
negative-finding tests are measured by beta, the probability of a false
negative, whereas positive-finding tests are measured by alpha, the probability
of a false positive. Beta is much more heavily influenced by sample size than
alpha. 

Most 'professionals' took Statistics 101 in college. Soon after they graduated,
all that went out the windows and they reverted to intuitional standards. As a
high-school dropout, it took me longer to learn this stuff, but once I groked
it, it sunk in deeper.
