Hi,

I am getting problems in compiling Microfocus COBOL/CICS programs in unix. The
CICS translator is introducing a special name called "litlink" in between the
CALL statements.

For Example,

EXEC CICS ASKTIME ABSTIME(WS-TIME) END-EXEC will be transalted as

CALL litlink 'CICSAPI' USING CICS-ARGS. And also a SPECIAL NAMES para is
introducted right below the FILE-CONTROL as

FILE-CONTROL.
SPECIAL NAMES.
call-convention 8 is litlink.

This will result in compilation errors because litlink is not a Cobol reserved
word. You may be knowing that the above cics statement should be translated to
CALL 'CICSAPI' USING CICS-ARGS and there should not be any SPECIAL-NAMES para.

Please let me know if anybody has the solution for this problem. Thanks in
advance.

Thanks,
Laxmi.
