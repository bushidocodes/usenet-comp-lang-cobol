```
We have a Sun sc2000 and are trying to bring up Micro Focus Cobol V3.2.
Our Sparc C compiler is release 3.0.1, and the Cobol install is failing
because it expects SC2.0.1.

When the product is installed it creates a "liblist" file that specifies
the path of the C compiler. This liblist file is created from a template
called liblist.base. Since the initial installation of COBOL we have
upgraded the C version from SC2.0.1 to 3.0.1 and removed the old version.

The liblist.base hard-codes the SC2.0.1 path for the 2.0.1 compiler.
We deleted this file, but when we re-ran the install program, it still
referenced the SC2.0.1 path.

If we go into the file to change the path specification manually, it still
does not work becaus the file names that were referenced for C SC2.0.1 do
not exist in the SC3.0.1 version.

We'd be grateful for any advice.

Thanks in advance,

Vic Odhner Phoenix AZ
```
