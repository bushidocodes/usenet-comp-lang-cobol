```
That code is broken.  Right now a birth year of 99 would be
in the 1800's, not 1900's.  It is also not a sliding window,
because the ranges are fixed.

Birth date is one of the dates in which windowing is often
not appropriate, because many people live to be over 100.
If it is for students at a grade school, or patients at a
pediatrician's office, or other such restricted birth ranges,
it would be okay.  Windowing is a technique used to resolve a
two digit year into a four digit year.  For conversion use, a
static window is usually acceptable, because the conversion
will be done within a limited time.  But to be effective for
indefinite use, you must use a sliding window, based on the
current date at the time.  Sliding windows using the current
date can be very useful, and a completely valid method for
conversion and permitting users to continue to enter two digit
years indefinitely, so long as the dates they are entering can
always be assumed to be within a single century defined by the
current date at the time of entry.

The following code will resolve a six digit 'input' date into
a century that is 'windowed' from the current year.  This
'window' is defined by low and high range.  The low and high
range can be any offset from the current year, but since the
range must be exactly 100 years, the difference between high
range and low range (RANGE-HIGH - RANGE-LOW) must = 99.  The
example below uses a common range, current year -79 to current
year + 20.  You could use low and high of 0 and 99, -99 and 0,
-120 and -21, 10 and 109, or any other range, so long as you
always choose a low and high where high - low = 99.

     03  RANGE-HIGH           PIC S9(02) VALUE +20.
     03  RANGE-LOW            PIC S9(02) VALUE -79.

     03  TODAYS-CCYYMMDD      PIC  9(08).
         05  TODAYS-CCYY      PIC  9(04).
         05  TODAYS-CCYY-R    REDEFINES DW-RESOLVE-CCYY.
             07  TODAYS-CC        PIC  9(02).
             07  TODAYS-YY        PIC  9(02).
         05  TODAYS-MM        PIC  9(02).
         05  TODAYS-DD        PIC  9(02).

     03  WINDOW-CCYYMMDD      PIC  9(08).
         05  WINDOW-CCYY      PIC  9(04).
         05  WINDOW-CCYY-R    REDEFINES DW-RESOLVE-CCYY.
             07  WINDOW-CC        PIC  9(02).
             07  WINDOW-YY        PIC  9(02).
         05  WINDOW-MM        PIC  9(02).
         05  WINDOW-DD        PIC  9(02).

     MOVE TODAYS-CC TO WINDOW-CC.
     MOVE INPUT-YY  TO WINDOW-YY.
     MOVE INPUT-MM  TO WINDOW-MM.
     MOVE INPUT-DD  TO WINDOW-DD.
     IF (WINDOW-CCYY > TODAYS-CCYY + RANGE-HIGH)
         SUBTRACT 1 FROM WINDOW-CC
     ELSE
         IF (WINDOW-CCYY < TODAYS-CCYY + RANGE-LOW)
             ADD 1 TO WINDOW-CC.

Note that if you want to window to the century immediately
prior to the exact current date, such as for birth date,
you will need to compare the whole date rather than the
year, but you do not need to test for a lower range.

     MOVE TODAYS-CC TO WINDOW-CC.
     MOVE INPUT-YY  TO WINDOW-YY.
     MOVE INPUT-MM  TO WINDOW-MM.
     MOVE INPUT-DD  TO WINDOW-DD.
     IF (WINDOW-CCYYMMDD > TODAYS-CCYYMMDD)
         SUBTRACT 1 FROM WINDOW-CC.

Analogous rules apply if you want to window to the century
immediately subsequent to the exact current date.

     MOVE TODAYS-CC TO WINDOW-CC.
     MOVE INPUT-YY  TO WINDOW-YY.
     MOVE INPUT-MM  TO WINDOW-MM.
     MOVE INPUT-DD  TO WINDOW-DD.
     IF (WINDOW-CCYYMMDD < TODAYS-CCYYMMDD)
         ADD 1 FROM WINDOW-CC.
```

```
Don't know what the example was, but it was not a "sliding window".
That is fairly straightforward ... to understand it consider a "fixed
window" first.  You can handle 2 digit dates quite simply by assuming
that every date with a year less than 50 is in the 21st century, while
every date with a year greater than 50 is in the 20th century.  That
gives you a "window" of dates --- from January 1, 1951 to December
31, 2050 that the code works for.

To make it a sliding window, you calculate the window based on the
current date, so the software will work in a window starting
(for example) twenty-five years ago, and ending 75 years from "now".

As the date changes, the dates that the software will handle changes,
thus the "sliding" nomenclature.  Obviously, the methodology will not
work with some applications, but for a system that managed (for example)
mortgages, given that no mortgage lasts over 35 years, it can save
having to modify all the data files for a four digit date.
```
