```
Mapping to external devices requires the system to understand what you
are trying to do. In some cases the printer should be addressed as
'ASSIGN TO PRINTER' and the default will be used, in others there will
be an extra set of environment names that map to devices, or to
processes.

For example with MF one can have some environment variables:

dd_lpt1=>lpr -p name [options]
dd_lpt2=>lpr -p another ..
...

and then an ASSIGN TO "lpt1" will redirect the stream to the lpr
program specifying the device 'name'.

This works dynamically too so that 'ASSIGN TO Print-Name' is a variable
such that 'MOVE "lpt2" TO Print-Name' means that the open will get the
printer 'another'.

The program can also change the environment variables using
DISPLAY/ACCEPT so almost anything is possible.

Other cobol systems do it entirely differently. Fujitsu on Linux has
several wonderous mechanisms for accessing printers, none of which are
appropriate to my needs, but may be perfect for others, so I had to
invent my own mechanisms which wasn't hard to do.
```
