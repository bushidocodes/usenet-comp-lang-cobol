|
|--- reference modification [position,length]
| (1:1) means the first character only
| (2:) means start at the second character of the
string. When no length is given it means
'from the start position to the end of the
string.

Two inspect statements are used to guarantee that source strings
like 'JOHN', 'john', 'John', 'jOhN' all end up as 'John'.

----------------------------------------------------------------------

As an aside, this technique can be used to validate that strings only
contain a given set of characters.

E.g. A surname might be restricted to having the first character
in "AEIOUBCDFGHJKLMNPQRSTVWXYZ" but other characters may also
contain "'-()". The best way of checking this that I have found
is:-

01 NAME_FLD pic x(30).
01 WORK_FLD pic x(30).
01 VALID_1ST_CHAR pic x(26) value "AEIOUBCDFGHJKLMNPQRSTVWXYZ".
01 VALID_OTHER_CHARS pic x(30) value "AEIOUBCDFGHJKLMNPQRSTVWXYZ'-()".
...
Move NAME_FLD to WORK_FLD.
Inspect WORK_FLD (1:1) converting VALID_1ST_CHAR to spaces.
Inspect WORK_FLD (2:) converting VALID_OTHER_CHARS to spaces.
If WORK_FLD not = spaces
then
.... validation has failed; NAME_FLD contains other than
the required set of characters....
end-if
