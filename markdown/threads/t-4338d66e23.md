```
Easy answer, there  is not now, nor has there ever been any such  product as
   "Cobol LE"

There are a number of IBM COBOL compilers that require the LE run-time
library (and don't have their own run-time library).  This is in addition to
the currently unsupported OS/VS COBOL, DOS/VS COBOL, and VS COBOL II
products that work with LE, but didn't require LE.  (They *do* require LE to
have IBM "run-time" support today).

The names of the products (on MVS, OS/390, and z/OS) that require LE are:

  IBM SAA AD/Cycle COBOL/370
  IBM COBOL for MVS & VM
  IBM COBOL for OS/390 & VM
  IBM Enterprise COBOL

(All but the last one of these also work with LE under VM.  VSE has another
set of LE-enabled compilers)

Having said this, the best place to learn about the *current* IBM z/OS COBOL
compiler, is in the Migration Guide at:


http://publibz.boulder.ibm.com/cgi-bin/bookmgr_OS390/BOOKS/IGY3MG10/CCONTENT
S

If you don't know about LE (which is NOT COBOL specific), you should also
read the latest migration guide for it at:


http://publibz.boulder.ibm.com/cgi-bin/bookmgr_OS390/BOOKS/CEEA6130/CCONTENT
S

If you have specific question (after looking at these manuals) let us know
and I am certain that we can provide more information.
```
