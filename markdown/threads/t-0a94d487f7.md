```
I had the same problem and got an answer from MF that they do not directly
support MS Help. I have recently switched to WB4.0 and 32-bit DS and the
situation is the same. To solve this problem, I wrote a program that just
calls Windows Help 4.0 and compiled it to a DLL. It must be compiled with
the CASE compile option. The source is below. No comments please about
my use of mainly UPPER-CASE throughout the program, it works for me and I
have nobody to answer to since I work for myself. I am also one of those
dinosaurs who remember 026/029 card punches, 056/059 verifiers, 082/083
card sorters, 188 collators and whateverelse mentioned in another thread.
I used all of them when working with my first system, an IBM 305 RAMAC,
before I became officially involved in programming. Back then operaptors
also needed to know a little about programming as well as wiring plug
panels. Those were the good(?) old days. : >)

If any of the data names are not self-explanatory, please e-mail me and I
will post you an explanation of the data names that you don't understand.

This DLL works fine when called from the Cobol program, but I have a small
problem when calling from DS.

Here is the source:

$SET MFOO OOCTRL(+N)

******************************************************************
******************************************************************
** THIS PROGRAM IS USED TO CALL WINHELP 4.0.
******************************************************************
******************************************************************

PROGRAM-ID. CALLWHLP.

*AUTHOR. G FOUNTAIN.

* INTERNET - guy··.@ini··o.ie.
*REVIEWED BY. REVIEWER'S NAME.
DATE-WRITTEN. * DATE CODING STARTED.
*DATE-REVISED. * DATE OF REVISION.
* * INSERT LATEST DATE AT TOP.
DATE-COMPILED.
*PROGRAM-STATUS. M.
* U = UNMODIFIED SKELETON M = MODIFIED SKELETON
* C = CODED T = TESTED
* P = PRODUCTION R = UNDER REVISION

ENVIRONMENT DIVISION.
CONFIGURATION SECTION.
SOURCE-COMPUTER. PC-COMPATIBLE.
OBJECT-COMPUTER. PC-COMPATIBLE.
SPECIAL-NAMES.
CALL-CONVENTION 74 IS WINAPI.

WORKING-STORAGE SECTION.

77 COMP-CTXT-NMBR PIC 9(4) COMP-5.
77 COMP-WIND-HNDL PIC 9(8) COMP-5.
77 HELP-FILE-PNTR POINTER.
77 WIND-HELP-FILE PIC X(128) VALUE
"C:¥WINDOWS¥HELP¥WINHLP32.HLP" & x"00".

78 HELP-CONTENTS VALUE h"0003".
78 HELP-CONTEXT VALUE h"0001".
78 HELP-CONTEXTPOPUP VALUE h"0008".
78 HELP-FINDER VALUE h"000b".
78 HELP-HELPONHELP VALUE h"0004".
78 HELP-QUIT VALUE h"0002".
78 ONE VALUE 1.
78 SUFFIX VALUE "A".
78 WINHELP VALUE "WinHelp" & SUFFIX.

01 APPL-HELP.
03 APPL-HELP-FILE PIC X(127).
03 FILLER PIC X VALUE X"00".

LINKAGE SECTION.
01 WIND-HELP-PARM.
03 TYPE-HELP PIC X(10).
03 WIND-HNDL PIC 9(8).
03 HELP-FILE PIC X(127).
03 CTXT-NMBR PIC 9(4).

PROCEDURE DIVISION
USING WIND-HELP-PARM.

MOVE CTXT-NMBR TO COMP-CTXT-NMBR.
MOVE WIND-HNDL TO COMP-WIND-HNDL.
MOVE HELP-FILE TO APPL-HELP-FILE.
SET HELP-FILE-PNTR TO ADDRESS OF APPL-HELP.

EVALUATE TYPE-HELP

WHEN "CONTENTS"
CALL WINAPI WINHELP
USING BY VALUE COMP-WIND-HNDL
BY VALUE HELP-FILE-PNTR
BY VALUE HELP-CONTENTS
BY VALUE COMP-CTXT-NMBR

WHEN "INDEX"
CALL WINAPI WINHELP
USING BY VALUE COMP-WIND-HNDL
BY VALUE HELP-FILE-PNTR
BY VALUE HELP-FINDER
BY VALUE COMP-CTXT-NMBR

WHEN "CONTEXT"
CALL WINAPI WINHELP
USING BY VALUE COMP-WIND-HNDL
BY VALUE HELP-FILE-PNTR
BY VALUE HELP-CONTEXT
BY VALUE COMP-CTXT-NMBR

WHEN "POPUP"
CALL WINAPI WINHELP
USING BY VALUE COMP-WIND-HNDL
BY VALUE HELP-FILE-PNTR
BY VALUE HELP-CONTEXTPOPUP
BY VALUE COMP-CTXT-NMBR

WHEN "USINGHELP"
CALL WINAPI WINHELP
USING BY VALUE COMP-WIND-HNDL
BY REFERENCE WIND-HELP-FILE
BY VALUE HELP-HELPONHELP
BY VALUE COMP-CTXT-NMBR

WHEN "QUIT"
CALL WINAPI WINHELP
USING BY VALUE COMP-WIND-HNDL
BY VALUE HELP-FILE-PNTR
BY VALUE HELP-QUIT
BY VALUE COMP-CTXT-NMBR

END-EVALUATE.

EXIT PROGRAM.
```
