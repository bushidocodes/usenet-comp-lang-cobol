One thing I like about C is you can do the following

    while (readfile(filename) ! = EOF) {
        processrec(record);
    }

in more of a cobol style you have to do something like

    result = readfile(filename);
    while (result != EOF) {
        processrec(record);
        result = readfile(filename);
    }

In both cases readfile(filename) is performed one more time than
processrec(record), with the final call of readfile() returning an EOF
condition.  Does the new COBOL standard allow for anything like this?  I guess
it probably does for user-defined functions, but not for performs.  Alas.  I
like COBOL, but sometimes it's frustrating.
