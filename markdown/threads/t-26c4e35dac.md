I finally found a copy of this book at Computer Literacy Bookshops in San Jose, CA.
The
bookshop has a nice reading area where one can review a book before purchasing it. So
I spent about an hour looking at the Third Edition of  "Teach Yourself COBOL in 21
Days."
If this is the most recent edition, it is still a pretty poor book on COBOL.   The
code
examples are woefully out-of-date and represent some of the worst practices of COBOL
programming.  Those examples might have been OK twenty years ago, before the arrival
of ANSI standard 95, but they are likely to mislead new COBOL programmers into
poor coding style and bad habits today.    Of course, if you are still using COBOL-68
or COBOL-74, as some companies still are, it might be all right.

To its credit, the book does mention, all too briefly, some of the more current COBOL
capabilities.   This book might not be so bad if someone took the time to update the
long code examples to reflect more contemporary COBOL practice.

On the same shelf, at the same bookstore, was another book from the same publisher,
"Teach Yourself COBOL in 24 Hours."    That is much better book. The code examples
represent much of current COBOL,  the discussions of  scope (not called that in the
book,
but that's not a problem) and control statements is up-to-date.   I wonder why the
editors
at SAMS could not have demanded the same quality of the 21 Days book they have
demonstrated in the 24 Hours book.

"one hour lessons".  I did quite a lot of work on Lesson Plan and
tried mightily to keep the tone conversational -- which by the way is
