Using COBOL for VSE, is it possible for a called subroutine to know if the
system date has been overridden using the // DATE card in the JCL?

We have a subroutine that is called by every production program.  It's
purpose is to record how often every program is run.  If the system date is
overridden, it then collects incorrect data and all of the program history
stats become unreliable.

Never used anything like that EXCEPT for the VIASOFT VIADATE that allows you
to override ANY/ALL DATE/TIME functions in Batch/Online Programs
(COBOL/PLI/ASM)
by using:
//VIADATE  DD DSN=XXXX.XXXX.XXXX,EXPDATE=20001231

VIADATE is also installed in the OS and can INTERCEPT INTERUPT 11
(DATE/TIME)
If your //DATE OVERRIDES the SYSTEM DATE, then ask yourself, how can an app
tell that you overrode anything?  All it knows is what it gets!

If you are using LE, you can call CEEGMTO--Get Offset from Greenwich
Mean Time to Local Time - It normally gives the time difference
between your time zone and GMT. It returns an offset of zero hours if
a // DATE statement is in effect.

If I recall correctly, the date won't change at midnight if  // DATE
is being used even  without LE- if this is true, you could schedule
your prog to be running at midnite and have it check to see that date
changes during the run. 

 See the LE/VSE Programming Guide for more info.
