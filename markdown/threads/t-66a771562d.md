Try setting the 's' (LOWER CASE) switch in $COBSW to a large number.
This switch indicates to the internal sort routines how much memory
it should allocate to do the sort. The more memory you give it, the
less disk I/O it needs to do.

eg: COBSW=-s1048576
to set it to 1 meg. You'll probably need to play around with that
to find the optimal value for your particular environment.

Also, check on your usage of the 'S' (UPPER CASE) switch in $COBSW.
This switch indicates whether a "stable" sort or a quicksort can be
used. The "stable" sort keeps duplicate keys in order, the quicksort
doesn't. If you don't need the stable sort (COBSW=+S), use the
quicksort (COBSW=-S (the default)).

Kev.
