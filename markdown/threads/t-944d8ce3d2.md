### Returning a variable to a batch file (DOS)

```
I am using NetExpress 3.1 with all flavors of windows and DOS on a Novell
Network. We have a batch file which includes running a menu program (Jobctl)
which was created in the DOS version on Merant MF.  As part of this program,
it creates a unique directory to run the selected job for each user.  The
ending path would look like F:\users\%user%\jobctl.999 (Where 999 is
replaced by a number from 1 to 999).  The current directory is then changed
to this new path and a temporary batch file created to run the selected job.
The program exits.  The origional batch file then runs the temporary batch
file.

With NetExpress, when the JOBCTL program exits, although the new path and
batch file have been created (and the current directory changed) the current
directory reverts back to the current directory of the batch file.  All my
research says that the environment that NetExpress runs in disappears (and
doesn't affect any other environments) when the program terminates; with
possibly the exception of the returning option (as in "exit program
returning xxxx") .   I am looking to return the extension to the jobctl
directory to the batch file.

Where does the returning variable reside in memory and is there any way that
DOS can access it?  Might there be another method to pass this 3 digit
variable back (in addition to creating yet another file to be read?
```

You can probably use the RETURN-CODE register to do this. The batch file
would have to test ERRORLEVEL which would make for a 1000 lines of 'IF'
statements. You might be better off having the main batch file provide the
name of the temporary file to your COBOL program rather than the other way
around. Then all the batch file needs to do is test for it's existence after
the COBOL program completes.
