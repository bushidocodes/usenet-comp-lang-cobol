Actually (as stated elsewhere) the character is the chiropractor that is
represented by the EBCDIC or ASCII (which ever is in effect) bit-pattern that
combines the numeric value (in one nibble) and the sign (in the other
nibble).

P.S.  This question gets asked fairly often in this newsgroup.  If you want
to "manually" convert these characters, you can look (in www.deja.com) for
threads that give the values of each number/sign combination.  If, instead,
you are interested in "tools" to do this conversion for you, you can find
those mentioned in deja.com as well.

John,

The last digit contains a combination of the sign and the last numeric
digit. What we COBOL programmers refer to as "right overpunch".
This is a space saving holdover from the days of punched cards.

How much control do you have of the COBOL program that creates
this file? I think the easiest solution is to change the output 
picture to one of these:

S9(7)V99 SIGN IS TRAILING.

displays positive or negative sign (+/-) in it's own byte to the right
of the last numeric digit.

+++++++9.99

displays floating positive or negative sign (+/-) before first numeric
digit and includes an explicit decimal point ".".

-------9.99	

displays floating minus sign only when value is negative.

If you can not have the generating program changed one of these
ways then you will probably have to look at the last character and
replace the signed digit with the appropriate unsigned digit and
identify the difference between negative and positive numbers and
multiply by -1 for the negative numbers. This should not be too
tough since there are usually only 20 possibilities 0-9 positive & 
negative unless you have to provide for the possibility of unsigned
numbers then there are 30 possibilities. The actual values depend
on wether the source of the data is ASCII or EBCDIC.

We usually use a separate sign when converting from EBCDIC to
ASCII anyway.

I don't know C so I can't advise if there is some more elegant
conversion function.

Mike

  You are getting EBCDIC characters. The last character is the
digit+sign.
   In your example, the last 'D' is a '4', with positive sign.

Here are some tables to use to convert with:
Positive digits (EBCDIC)
   Char    Hex      Digit
     }      C0        0
     A      C1        1
     B      C2        2
     C      C3        3
     D      C4        4
     E      C5        5
     F      C6        6
     G      C7        7
     H      C8        8
     I      C9        9
Note: for positives, an unsigned could have the hex 'C' replaced with
       hex 'F' as the first nibble of the hex value.

Nehagive digits (EBCDIC)
   Char    Hex      Digit
    }       D0        0
    J       D1        1
    K       D2        2
    L       D3        3
    M       D4        4
    N       D5        5
    O       D6        6
    P       D7        7
    Q       D8        8
    R       D9        9
