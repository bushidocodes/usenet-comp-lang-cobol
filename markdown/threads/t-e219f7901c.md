MFCOBOL buffers the index information while always writing the data
portion. If a pc is rebooted, turned off or otherwise powered down with
an indexed file open, BANG the index is dead.

Look in you documentation for the compiler directive WRITETHRU, or at
least I think that is the correct spelling. This directive forces a write
of the index everytime that you write a record. Performance is
compromised but what the hell, the index is safe (will at least safer).
