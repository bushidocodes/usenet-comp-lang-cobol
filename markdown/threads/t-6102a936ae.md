Here's some code for you to re-map ENTER to TAB. If the last field in the
screen section is properly defined (Auto I think) then it should work as it
did before. There are 2 calls. First to re-define enter to tab and then
to undo the re-definition at the exit of the program. Undoing the
definition is important as the runtime retains the key redefinition even
after programs are canceled.
Working Storage:

01 ADIS-KEY-MAPPING.
03 ADIS-MAPPING-BYTE PIC 9(2) COMP-X.
03 ADIS-KEY-NUMBER PIC 9(2) COMP-X.
01 SET-MAP-BYTE PIC 9(2) COMP-X VALUE 3.
To redefine the key:

*
* X"AF" ADIS CALL - CHANGE MAPPING OF ENTER KEY FROM TERMINATE
* ACCEPT TO MOVE TO NEXT FIELD.
MOVE 2 TO ADIS-KEY-NUMBER.
MOVE 11 TO ADIS-MAPPING-BYTE.
CALL X"AF" USING SET-MAP-BYTE ADIS-KEY-MAPPING.

to UNDO the define:


*
* X"AF" ADIS CALL - CHANGE MAPPING OF ENTER KEY FROM NEXT FIELD
* TO TERMINATE ACCEPT.
MOVE 2 TO ADIS-KEY-NUMBER.
MOVE 0 TO ADIS-MAPPING-BYTE.
CALL X"AF" USING SET-MAP-BYTE ADIS-KEY-MAPPING.
