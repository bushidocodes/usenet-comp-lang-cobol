Various possibilities here. I suspect that when you do a CEMT I against 
the file(s) affected you will see that the file is "Clo Une" meaning 
Closed but Unenabled, thus preventing automatic open at first reference. 
This may be due to file definition (use CEDC to discover if this is so, or 
ask whomever is responsible for the definitions to check) or the file has 
been specifically closed (more likely) by someone to take it offline for 
some reason. If the latter is so, the problem is that the same person who 
is taking the file offline should bring it back online once it is sensible 
to do so.

If all of the FCT's are defined as "Firstref", then instead of the
operator opening all files, just have operations do a "CEMT SET FIL
(...) ENA" and simply enable them.

You can view the FCT's via CEDC. You may need permission/authorization
to use this transaction.

Programmatically, you could issue a LINK to "DFHEMTP" and enable them.
The commarea would be the same command as the operator enters manually.

One problem you could have opening all the files is that your MVS/TIOT
may need an adjustment if you have greater than approximately 1600
files defined.

A more modern way is to use the SET command from the System Programming
Interface (SPI):

EXEC  CICS  SET
	FILE(xxxxxxx)
	OPEN
END-EXEC

You'll need the "SP" option for the translator. Rather than hard coding
the file names you could browse the FCT to get the file names & other
attributes. I can't remember the exact format of the start browse
command for the FCT, but it's in the SPI guide.

If the files are ENABLED, set for FirstRef and they don't open, there's
a problem. Check SDSF for your CICS region and find the error messages
for your file. It might not be defined, might be defined incorrectly,
might be open in batch, etc.

My shop uses AFCS to do this. If you don't run a 2nd job to ENABLE (at a
minimum) after the CICS region is back up, your files will be closed &
disabled. Been down this road more than once. AFCS brings CICS files up
in the same state they were in when you shutdown. Of course, if you're
not bouncing the region but just running your batch stream, you have no
choice but to ENABLE whem when you're done (or code your software to
deal with the CLOSED & DISABLED status).
