    If the "P" (P0) switch is set, the built in function to check printer
status does not work.  If you do NOT use the +P0 switch (same as +P), than
network printers often do not work.

    Also note that if the +P0 switch is set, you MUST use the +L0 switch.
If not, you get the DOS abort, retry message.  Abort aborts by god, leaving
files open, etc.  The safe message is blinking and worded differently.

    The best bet is to use -P0 (default) when running DOS no network, else
trust the OS, etc to make a printer available.

    The connection with EXTFH is very odd indeed.

    If the users are running the program in a dos prompt under win95 etc,
you could try XM.  It works almost the same as without, with some (very)
small exceptions (ie you must use MODE to set com port specs, not ASM/ INT
calls).
