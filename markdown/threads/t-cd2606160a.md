```
We have received the new V3R1 of IBM Debug Tool, which will offer a kind
of low level Assembler Language debugging support (no symbolics, but
otherwise, normal facilities, I think).  It should be installed in a few
days (the Sys Prog told me he had to order and install a prerequiisite
first).

I was wondering whether anyone has used this version to debug Assembler
code?  If so, what do you think of it?  I've used the Assembler Debugger
that comes with the HLASM Toolkit, and I don't much like it - maybe it
is just "too VM/CMS" to suit me.

I'm working on a COBOL replacement of a 35 year old Assembler program
that does manufacturing assembly item "explosions".  I plan to try to
put the Assembler program and the COBOL program both under Debug Tool,
so that I can test them side by side.  If the plan doesn't work out, I
can always use Debug Tool for the COBOL and the HLASM Toolkit's Debugger
for the Assembler program.
```

```
Roland,
Thank you for the response.  One reason that I posted to the COBOL group as well
as the ASM370 group is that Debug Tool can be used on COBOL programs that were
not compiled with the TEST option, through use of what is called the
"disassembly view".  In other words, I think the same feature that lets you
debug true Assembly Language code is what allows you to debug COBOL compiled
without TEST.

Did you get the Advanced Functions optional part of the product?  Some of our
programmers are fans of Xpediter, because it will show the variables used in
each statement as it is executed.  Advanced Functions gives Debug Tool that
capability.
```
