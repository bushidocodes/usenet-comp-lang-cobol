```
Hi:

Printing problems with MF COBOL on serial ports. My application software
prints on LPT and COM ports. The serial printers are dot-matrix,
roll-paper receipt printers.

Such printers require a MODE:COMx,9600,n,8,1,P command to be in
the autoexec.  If the MODE command is not present and you send
data to COM1, the data will not print. Instead, all you get is
question marks and nothing else. After putting the MODE command
in, the question marks go away and the data will print - usually!

Sometimes, like now, the MODE command doesn't seem to work. The
MODE command is in the AUTOEXEC, but I still get question marks.
Whether one copies a file from the DOS prompt to the printer,
or you print via the COBOL program, all you get is ????????????
marks.

In such cases, one can go into control panel and print a test
page on the appropriate printer and then the question marks
go away. This is a pain in the neck for the customer to have
to go through this ritual every day, to say the least.

The plot thickens - I have a customer who, after putting the
MODE command in and then executing my application gets
RT error 004 (Invalid File Name) when my application attempts
to open a file. It seems that the application doesn't acquire
the file name at all (since the application displays an error
message including the file status and the file name and the
file name is spaces).  He removes the MODE command and he
doesn't get the RT004 error, but he gets the question marks
because the MODE command isn't there?!?!?!?!?

How can a MODE command produce an RT004 error?

Has anyone had similar problems and how did you fix it?

Does anyone know how to TOTALLY BYPASS Windows and print
directly to the COM ports?

Has anyone used the FLEXUS formprint thing to print line-by-line
to a serial port (I couldn't get it to work.)

Does anyone know of any solution to this very annoying problem?

Thanks

Tony Dilworth



Sent via Deja.com http://www.deja.com/
Before you buy.
```

```
>Does anyone know how to TOTALLY BYPASS Windows and print

Get the MarshalSoft serial I/O DLL, and program the com port yourself.  I
have not done that with MF, but I have done it with Fujitsu, and it is not
very difficult.  The mode command is only setting up the UART.  If that is
the only problem, and I suspect that it is, then you should be able to
initialise the port correctly with your own code, then open it output using
normal Cobol.
```
