Sunder,

The easiest solution would be to do a simple READ-MOVE-WRITE routine
where you read in the ISAM file, move the fields into a SEQUENTIAL
file (suitable for DOS) with all numerics being defined as DISPLAY,
and then WRITE the SEQUENTIAL file.

A few caveats:

1) If your input file has a lot of REDEFINES this technique will
require some additional work to determine how to restructure the data,
especially if these redefines affect COMPUTATIONAL numeric fields.

2) You will have to check on how the compiler/platform combination
will create the output file. Each will have various idiosyncracies
about how it terminates each record and file. You will have to make
sure that the record terminators are compatible with the DOS side of
things. Just write a few test records once you resolve caveat 1 to
test how to set this up properly.

Boris
