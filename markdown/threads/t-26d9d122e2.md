This is a structural (architectural) question. You can do it either way.
'Simple' systems .. defined as containing a few dozen or hundred programs ..
should do it on a single level. 'Large' systems should do it level by level. 

If your system was 'large', you'd probably have an architect on staff and not be
asking on CLC. Therefore, I assume your system is 'simple' or medium-sized. So
just do it the simple way, on one level.

I dislike the Fujitsu 'project manager' and 'staff'.  They are fine
for simple test programs, but get really tedious for anything useful. 
It is also a pain that only certain file 'types' are recognised.  In
taking an existing system to Fujitsu (purchased version), I
preprocessed the source code to change COPY statements so that Fujitsu
looked in several subdirectories \fd \sa \ws \pd and copied the
copybooks here with .cob 'type'.  Previously I had the files as
name.FD, name.SA, name.WS etc.

I also built a makefile that does the depndencies based on actual COPY
statements.  Most programs are built as DLLs and are called with
dynamic calls (DLOAD) so no linking of .LIBs.
