NOT OPTIONAL is an extension (a rather dumb one in my opinion). For
sequential files, the OPTIONAL phrase has existed in a standard since
COBOL 68. In COBOL 85 it was extended to apply to relative and
indexed files. Before COBOL 85, it was undefined as to what happened
when a file opened for I-O did not exist. Some implementations
bombed the job, some created the file. In COBOL 85 it was changed to
be an error unless you said OPTIONAL. In that case, an empty file is
created.
