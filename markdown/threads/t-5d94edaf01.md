```
DLL is a standard format in Windows platform and it is indenpendent of
COBOL.

Once you got the DLL, you can use JNI (Java native interface) to call DLL.
But JNI is actually is an interface between Java and C/C++. Therefore you
have to find your function prototype in term of C/C++.

Find the type converson and make up a prototype file for your dll
or it is already generated by your compiler. look for something like *.h.
Make sure your DLL can be called by C/C++.

Suggestion: test a simple one and then do your real work.


Thomas Li
```
