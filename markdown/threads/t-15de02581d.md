One way would is to write two paragraphs; one for converting to low and
one for converting to upper. Each could contain an EVALUATE statement
coded like:

TO-UPPER:
EVALUATE PASSED-CHAR
WHEN 'a'
MOVE 'A' to PASSED-CHAR
WHEN 'b'
MOVE 'B' to PASSED-CHAR
.
.
.
etc

If you're using COBOL on mainframe (EBCDIC) then look at your handy
reference card. "A" is hex C1. I believe "a" uses a lower hex number
also ending in 1 (i.e. x1). This means you could define a halfword
COMP field...redefine it with a field, PASSED-CHAR for one byte, at the
bottom. Then you could simply subtract or add the proper amount to the
COMP field to achieve the desired HEX EBCDIC code...moving PASSED-CHAR
at the end.

Hope this gives you some food for thought.

If you are in an IBM Mainframe environment, the newest version of COBOL
provides intrinsics function to do this. The newest COBOL for my OS is
IBM COBOL for VSE/ESA.

01 Item-1 Pic x(30) Value 'This is a test'.
01 Item-2 Pic x(30)


To convert string to all upper case
===========================================

Move Function Upper-case(Item-1) to Item-2.

Item-2 would be 'THIS IS A TEST'.


To convert string to all lower case
===========================================
Move Function Lower-case(Item-1) to Item-2.

Item-2 would be 'this is a test'.

In addition to the ACCEPT version offered by someone else,
a more general-purpose version would be to :

MOVE FUNCTION UPPER-CASE(input-string) TO upper-string

where input-string and upper-string are some PIC X(n) variables
you've declared elsewhere.
This is standard, but not all compilers yet implement the
intrinsic function module.

No silly questions here!

01 ALPHA-CONVERT.
03 UPPER-CASE-LETTERS PIC X(26) VALUE
"ABCDEFGHIJKLMNOPQRSTUVWXYZ".

03 LOWER-CASE-LETTERS PIC X(26) VALUE
"abcdefghijklmnopqrstuvwxyz".

01 YOUR-FIELD-NAME PIC X(80).


Inspect YOUR-FIELD-NAME Converting LOWER-CASE-LETTERS To
UPPER-CASE-LETTERS.

You can also referse the Converting options to convert upper-case
to lower-case.

The Converting option of INSPECT is an ANSI '85 feature,
so your compiler must be at the '85 level. Most PC compilers
are, if this is the platform your question pertains to.

Hope this helps,

Mike Dodas
