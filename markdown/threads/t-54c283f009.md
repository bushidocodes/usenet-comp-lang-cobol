What OS390 restriction are you talking about?

What is it you want in this table anyway?

A VALUE on an elementary item can only be the
length of the picture, ie. two bytes in your case. BUT
since it has an OCCURS a 2 byte value is replicated across the
table: 02 var1 occurs 10 times pic xx value '01' results in
the table containing '01010101010101010101'. That's what
happens on micro-focus anyway. value '010203...' is a
compile error.

HOWEVER....
Assuming that the VALUE clause is what you want to have in
your table you do realize that in the way you posted this question
Var1 (1) contains '01'
Var1 (2) contains ' 0'
Var1 (3) contains '2 '
Var1 (4) contains ' 0'
Var1 (5) contains '3 '  and so on....

Assuming you want each occurance to be an ascending sequential
number because that's the way you posted the question...
AND assuming again you do not want embedded spaces between the values
because that would be dumb ...
you can only have up to value '99' because Pic X(02) only
allows two characters. What do you want after that?

So if you want a table with 1035 occurances containing sequentially
ascending numbers do this:

05  sub pic 9(4).
05  var1 occurs 1035 times pic 9(4).

perform varying sub from 1 by 1 until sub > 1035
  move sub to var1 (sub)
end-perform.


If you want to hard code values such as days of the week then
do something like this:

05  week-days pic x(21) value 'SunMonTueWedThuFriSat'.
05  filler redefines week-days.
    10 week-day-table occurs 7 times pic x(3).


If neither of these is what you want...go back and figure out what you
do want and come back and tell us.

Hello Binoy,

You should reread your manual a bit more carefully.  Being able to
value the elements of a table independently is a new feature of the
2002 standard and if your compiler supports this particular feature,
then the rules for it are somewhat different.  Also, according to the
both the 1985 and 2002 standards, you can not define an 01 level data
item with an OCCURS clause.

It would be helpful when posting queries to specify the compiler as
well as the operating system being used.

To do this job using the rules for the 2002 standard you would need
code along the following lines:

01  VAR1-TABLE.
    05  VAR1 OCCURS 1035 PIC X(2)
    VALUE '01' FROM (1)
          '02' FROM (2)
          '03' FROM (3)
          etc

As is obvious, this is rather cumbersome for this particular task, I
would recommend that you do the initialisation in the procedure
division using a PERFORM statement as follows:

PERFORM VARYING SUB FROM 1 BY 1 UNTIL SUB > 1035
    MOVE SUB TO VAR1 (SUB)
END-PERFORM.

where SUB is a subscript defined in the data division as follows:

01  SUB PIC S9(4) BINARY.

and VAR1 is defined as:
 
    05  VAR1 OCCURS 1035 PIC 99.

though using PIC X(2) would probably still work as the compiler would
probably do the conversion automatically, but you should check your
manual to ensure that it does.

You should also check your manual to ensure that a USAGE of BINARY for
SUB is suitable to represent a PIC S9(4) number, it probably is for a
compiler that would run on an OS390 operating system.

Another way of doing what you want for rather smaller tables of no
more than 160  characters would be as follows:

01  VAR1-TABLE VALUE '01020304050607080910'.
    05  VAR1 OCCURS 10 PIC X(2).

good luck

Robert

In the compiler I use, you can't have a literal longer than 160 
characters.  (You also can't have a Value clause on an Occurs clause.) 
If these are always ascending, it's easy to fill the table initially 
using an inline perform in the Procedure Division.

(I'm not sure what the limits are, but this does 1296 occurrences...)

=-=-=-=-=

01  WS-Table.
     03  WS-Entry        Occurs 1296 Times  Pic X(02).

01  WS-Chars   Value "0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZ".
     03  WS-Char         Occurs 36 Times  Pic X(01).

01  X                   Pic 9(02).
01  Y                   Pic 9(02).
01  WS-Tbl-Idx          Pic 9(04).

...

Perform Varying X from 1 by 1
   Until X > 36

     Perform Varying Y from 1 by 1
       Until Y > 36

          Compute WS-Tbl-Idx = ((X - 1) * 36) + Y

          String WS-Char (X)  WS-Char (Y)
            Delimited by Size into WS-Entry (WS-Tbl-Idx)

      End-Perform

End-Perform

=-=-=-=-=

Entry 1 in the table is now "00", entry 37 is "10", entry 361 is "A0", 
and entry 1296 is "ZZ".
