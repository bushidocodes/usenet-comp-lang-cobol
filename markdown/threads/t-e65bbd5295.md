I don't have COBOL source, either.  ^ is the "bitwise exclusive or"
operator.  It's used to compare two operands, bit by bit.  If the bit being
compared is a 1 in either one operand or the other, but not both, then the
corresponding bit in the result is set to 1, otherwise it's set to 0.  The
function you show expects to be passed three parameters: a 16 bit unsigned
integer (presumably appropriately initialized), a pointer to a character
string, and a 16 bit unsigned integer (presumably set to the length of the
string).  The value of the first parameter is shifted right eight bits, so
the low-order eight bits are dropped as the high-order eight bits are moved
right and replaced by zeros.  This value and the first character of the
string are then compared with the "bitwise exclusive or".  The result is
then used as an index to obtain a 16 bit unsigned integer value from the
huge table.  The original value of the first parameter is then shifted left
eight bits, so the high-order eight bits are dropped as the low-order bits
are moved left and replaced by zeros.  This value and the value obtained
from the table are then compared with a "bitwise exclusive or", and the
value of the first parameter is replaced with the result.  Next the pointer
to the buffer is incremented and the process is repeated, using the next
character and the updated value of the first parameter.  This is repeated
for each character in the buffer.  Upon completion, the function returns a
sixteen bit unsigned integer equal to the value of the final updated value
of the first parameter, which is the 16-bit CRC checksum hash of the
character string.

I think that's what it will do...

Here is a little simplier explanation/flow of CRC16 that might help
you code it in Cobol:

Remember a CRC16 has a 17bit divisor. Bit 16 (As you know we start at
bit 0) is 1. When not we have again a smaller divisor.

     CRC : wordvalue

     { This routine has to be executed 8 times }
     IF CRC bit 15 = 1
     then
     shift left 1, divide by divisor (16 bits)
     else
     shift left 1, {do not divide cause we can't}

     {After this put in the next byte}
     CRC = CRC + inputbyte

     {end of this routine}


Since you have a problem reading C, maybe this routine in Pascal will
be easier:

      inpbyte = input byte for CRC
     oldCRC = old crc value
     divisor = the least 16 bits of the divisor string

     Function CRC16 ( inpbyte : byte, oldCRC : word, divisor : word )
:
     word ;

     var
     tel : word;
     temp : word;  {A simple variable to use to store the CRC)

     begin
     temp := oldCRC;
     for tel := 1 to 8 do
     begin

       If (temp and $8000)= $8000
       then
         begin
           temp := temp shl 1;
           temp := temp xor divisor;
         end
       else
         begin
           temp := temp shl 1;
         end;

       { Now we have to put in the next byte }

       temp := temp xor inpbyte;

       CRC16 := temp;

     end;

     {End of routine}

This routine is easily to expand to CRC32. In that case you have to
expand your divisor and temp and CRC function to LONG value's.

Some additional information about CRC's :

CRC16 divisor =  $1021 ( + bit 16 = $3021 )
The CRC16 feed value (when you first call the CRC routine) is $0000
CRC32 divisor =  $77073096 ( + bit 32 = $17707306 )
The CRC32 feed value (when you first call the CRC routine) is
$FFFFFFFF

You will need to look at the Cobol LE callable routines for bit
shifting.
