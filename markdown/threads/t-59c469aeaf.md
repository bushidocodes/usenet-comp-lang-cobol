COBOLers:

I sell a good general purpose accounting system that is suitable for
running medium sized businesses. Since it is strong for the "Back
Office" we sometimes want to "marry" it to a more specialized system.

I am considering quoting our system in a situation where we want to
get data from the HELMS Student Loan System which is written in Realia.

The SouthWare system which I sell is written in Acucobol. It has an
importing module for importing ASCII comma delimited or DIF format.
We have source code for SouthWare, but not for the HELMS System. I am
told by potential client that the HELMS system will not produce the
ASCII comma delimited or DIF files to go to SouthWare. The files we
want to connect are primarily A/R Customer master files and transaction
files. Most of the "flow" will come from transactions in HELMS to be
passed into SouthWare for passing to G/L, etc.

Any ideas?
