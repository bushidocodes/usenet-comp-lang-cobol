Others have talked about how you should work with the information on a File
Status of 46. Let me add one other bit of information. If you compile your
program with either the LIST or OFFSET compiler options, you will be able to
find out what verb in your program is at offset X'0C6A. This is a useful
technique for any type of error message that you get that tells you that you
have a problem in your program at a specific offset.

In MVS at the ISPF command line type

qw IGZ0020S


It gives you the following info:

Message Format: IGZ0020S A user logic error occurred. Neither file
status nor a declarative was specified for
file filename in program progname at relative
location xxxx. The status code was stcde.

Description: An I/O logic error (i.e., WRITE against a file opened for
INPUT, REWRITE without a previous READ, etc.) was detected
in the indicated program during an I/O operation against
the indicated file and no EXCEPTION/ERROR DECLARATIVE or
FILE STATUS clause was specified. The job step is
cancelled.

User Action: Check the program logic to determine the circumstances
which led to the illogical I/O operation.


Also an error 46 is defined as
A sequential READ statement was attempted on a file open in the I-O
mode and no valid next record had been established because:
1 - The previous READ statement was unsuccessful, but did not cause
an at end condition
2 - The preceding READ statement caused an at end condition

which means you are trying to read beyond the end of the file.

hopes it helps

Dan
