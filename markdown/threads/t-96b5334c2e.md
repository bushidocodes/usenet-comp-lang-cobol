The COPY x REPLACING LEADING ... BY ... is needed but we need a better
means of qualification.  As I have posted previously I want to be able
to code MOVE file-name.record-name.group-name.field-name TO x instead of
having to code MOVE field-name OF group-name OF record-name OF
file-name.  I believe that it is easier to read, is more in keeping with
the way the English language works (we say the boy's dog rather than the
dog of the boy in most cases) and follows the form of many other
languages.  I know that many people in the shops where I have worked
have found the use of qualification made the program harder for them to
read.  I dislike the awkward syntax but put up with it because I don't
have to code the same record description twice thus eliminating the
problem of differences.  Unfortunately because qualification was viewed
as awkward we are stuck with needing a construct that allows a copy with
a change of prefix.
