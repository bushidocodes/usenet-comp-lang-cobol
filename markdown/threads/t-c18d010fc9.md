A shop I just joined is migrating from mainframe to client server and is
using VB and MF COBOL against a SQL Server database. We have been told
that because we are using ODBC we can't use embedded SQL in our COBOL
programs due to the overhead involved in using MF and ODBC so instead we
are using COBOL objects for all database IO.

I am new to both MF and ODBC and wanted to ask if objects are indeed the
only way to go with MF and SQL Server. It would seem to me that there
should be a way to configure the compiler to handle all the bindings and
such so that we could use embedded SQL.

Any responses would be appreciated. Thanks!
