```
170 is system program not found. It is likely that it is
searching down the COBDIR=path for this and is not finding
it. Check that the SET COBDIR= that should be in your
autoexec is actually working correctly. Sometimes a
SET command fails because the OS runs out of environment
space and it may not be obvious that this is happening.

›
› 2-
› We have a few *.dat files we need to import to a paradox table. Is there
› any tool available to look at the data and transfer it.

You will probably need a specific extract program to create
the data in a suitable format, for example as a comma
separated value file. For my systems I have written a
generalised reporting program which can output data
as reports, CSV files, or even as HTML tables, but
it requires an interface module that will understand
the data file contents.
```
