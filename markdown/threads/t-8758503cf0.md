Michael, I assume you're using IBM COBOL for MVS V1R2? The Debug Tool
supports interactive debugging of BMPs and batch programs only in
conjunction with a workstation debug front end. The workstation front
end is available as part of the old CODE/370 product, and as part of
the new VisualAge COBOL product. In lieu of this, I believe you can
run BMPs and batch programs in TSO under BTS (but I'm no BTS expert).
You can also specify debug commands in your JCL.

Barbara Price
IBM COBOL Development

›  A fellow programmer wrote a program to match the counts generated by
› the debug tool with the sysprint member. I provide the JCL with a 'prim'
› file that contains the debugger commands (set frequency, run, ..etc).  I
› tried having just one command: "set screen on".
The set screen command is not valid when running in batch. It's
available only when running in the foreground, to tell the Debug Tool
whether you want to display the Debug Tool full screen interface (set
screen on) or you want to enter debug commands in line mode (set
screen off). For batch, neither of these is an option. You can only
provide the primary command file, as you seem to have done.

› The job abended.
Although the set screen command is invalid in batch, it shouldn't
cause the job to abend! What kind of abend was it? You might want to
contact IBM service about this.

› I have also tried to issue to BTS commands to make my
› prog. appear to be a transaction. It didn't work. I would really like to
› get interactive debugging for the batch programs.
(Again, disclaiming much knowledge about BTS...) It looks like you
should be able to run both BMPs and IMS batch under BTS via a TSO
CLIST. In the documentation I see a BTS input parm of KW(BMP) in the
section on using a TSO command list to run BTS. Perhaps if you posted
a question about this, with BTS in the subject line, someone on this
news group would be able to help.

›
› BTW, I use REflections to telnet to our IBM. We do not have other
› workstations.
