Serge,

Part of the learning process is to put your source through a compiler to check for
syntax errors firstly, before your own logic checks. The program is well laid out,
but you will learn far more if you put it through a compiler first.

Some tips -

(a) you are missing important "." (periods/full stops), and have a semi-colon (:)
where you should have a period.

(b) CODE is a RESERVED word - use Un-CODE or le-CODE if your prefer

(c) You have specified 01 un-CODE with Level 88's and then you test :-

    if un-Code = "xxxxx"

try the following :-

01 Un-Code                pic x.
  88  ARRIVEE           value "A", "a".
  88  SUPPRESSE      value "S", "s".  *> Suppression is probably a reserved
                                                                    word
  88 IDENTA              value "I", "i".
  88 RECHERCHE     value "R", "r".
  88 QUITER              value "Q", "q".

Evaluate true
    when ARRIVEE perform.......
    when SUPPRESSE perform....
    when IDENTA perform.....
    when RECHERCHE  perform.....
    when QUITER  CLOSE AUTEUR
                                           TITRE
                              STOP RUN
    when other
                   DISPLAY "CODE INCONNU"
                   GO TO MENU
End-evaluate

- see - you don't reaslly need BonCode.

(d) You can't tell the program to CLOSE a PARAGRAPH  - you perform PARAGRAPHs or
GO TO PARAGRAPHs :-.

IF CODE = "A" CLOSE ARROUV.
*> I can't see a reference to ARROUV

(e) it is GO TO  and not GOTO

I could go on and on, but you need to put it through a compiler first.
