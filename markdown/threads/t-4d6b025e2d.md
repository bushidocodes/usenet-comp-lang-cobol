```
I am currently working on a sequential file maintenance program
and have encountered a problem with reading past the end of at least one
of my input files. I have two 88 conditions (no-more-trans and
no-more-masters) which are set to high-values when the end of each
respective input file is reached. I suspect that the problem may be with
my perform until statement in the main module of my program, whic reads:

perform until no-more-masters and no-more-trans
perform 02100-process-records
read-rental-car-master-file
into master-record-work-area
at end set no-more-masters to true
end-read
read rental-car-transaction-file
into trans-record-work-area
at end set no-more-trans to true
end-read

Is a perform until condition-name-1 and condition-name-2 a valid
statement? Any suggestions will be greatly appreciated.
```

```
SET 88-level TO TRUE

is a new feature in COBOL '85. It sets the value of the underlying
value to the first value:

01 FRODO-SWITCH PIC X.
88 FRODO-IS-BILBO VALUE "A" "C" "D".

SET FRODO-IS-BILBO TO TRUE

is equivalent to

MOVE "A" TO FRODO-SWITCH
```

```
reach eof1, but not eof2, you do the loop again-- which may be what
you want-- but you do BOTH READs again. You don't want that. You
want to READ only IF not eof of that particular file-- because you may
be at the end of one & not the other.
```
