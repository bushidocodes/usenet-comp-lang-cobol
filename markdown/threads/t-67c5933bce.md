Those are MS-DOS batch file commands to configure a serial port. Unless your 
laser printer has a serial interface -- most don't -- it isn't going to work.

The strings "1 9600 8 0 1 8" and "1 1200 7 1 1 8" are configuring the serial 
port transmission parameters. Without documentation, it's not possible to say 
with certainty which parameters are which, but the second parameter 
(9600/1200) is certainly the baud rate. The third parameter is probably the 
number of data bits, and the fourth one is probably the number of stop bits. 
I'm guessing that the first parameter indicates parity. No idea at all what 
the 5th and 6th parameters represent.

I imagine that "1 9600 8 0 1 8" means 9600 baud, even parity, 8 data bits, 0 
stop bits, and "1 1200 7 1 1 8" means 1200 baud, even parity, 7 data bits, 1 
stop bit. But don't take that to the bank.

Worse than any of the above - they're commands that are likely to be in a 
DOS batch file - but for the most part they're not DOS Batch commands, 
they're commands for proprietory executables.

The strings with the leading colons are labels - there should be a "GOTO 
labelname" elsewhere in the full batch.
SET establishes an environment variable
XCOMMS would be a custom executable, doing heaven-knows-what
PORTCON%PVER% would evaluate to PORTCON0 and PORTCON1 for the two frags 
posted. These are likely to be batch files to which the parameters listed 
(and likely correctly analysed by others) are delivered - but may also be an 
executable.

Probably better to post on alt.msdos.batch.nt, but with a far more 
substantial codefrag and be able to supply such info as what XCOMMS and 
PORTCONn are - they would be found as .EXE, .COM, .CMD or .BAT files 
somewhere in the path....
