1. If you're using the newest version of Fujitsu's compiler on a Win
machine, you can use:

COMPUTE NUMCHAR = FUNCTION STORED-CHARACTER-LENG(MYLINE)

2. A common way is:

INSPECT MYLINE TALLYING LEN FOR CHARACTERS BEFORE 'bb'

where 'bb' is a double space.

3. Another way:

PERFORM VARYING LEN FROM MAXLEN BY -1 UNTIL LEN < 1
    OR MYLINE(LEN:1) NOT = SPACE
   END-PERFORM.

4. Another way:
UNSTRING MYLINE INTO JUNK DELIMITED 'bb' COUNT IN LEN.

Efficiency in this case is virtually irrelevant; you can use any of these
methods at a rate of millions of fields per second. But, if you REALLY want
the MOST efficient, write a quickie program will all the above methods and
time the results through, say, one million iterations each.

If you do the above experiment, please post the results here!

At our shop, we have a callable subroutine that left-justifies a string,
squeezes out multiple blanks, and (optionally) returns the character count.
What with the linkage and all, not as efficient as it could be, but we only
have to write the routine once.

    This was my solution in populating a VARCHAR DB2 column:

               10  WS-NBR-OF-TRAILING-BLANKS   PIC S999  COMP-3.

           10  WS-PLCM-T-POLICY-COMMENT.
               49  WS-PLCM-T-COMMENT-LEN   PIC S9(4)       USAGE COMP.
               49  WS-PLCM-T-COMMENT-TEXT  PIC X(462).

           INITIALIZE                      WS-NBR-OF-TRAILING-BLANKS.
           INSPECT FUNCTION REVERSE (WS-PLCM-T-COMMENT-TEXT)
               TALLYING WS-NBR-OF-TRAILING-BLANKS FOR LEADING SPACE.
           COMPUTE DPLCM-T-COMMENT-LEN
               =   LENGTH OF WS-PLCM-T-COMMENT-TEXT
               -   WS-NBR-OF-TRAILING-BLANKS.
           IF DPLCM-T-COMMENT-LEN = ZERO
               MOVE +0001                  TO DPLCM-T-COMMENT-LEN
           END-IF.
           MOVE WS-PLCM-T-COMMENT-TEXT (1:DPLCM-T-COMMENT-LEN)
             TO DPLCM-T-COMMENT-TEXT.
