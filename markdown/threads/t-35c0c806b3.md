> Donald

Firstly, greatly surprised you were even using N/E, then was thrown by your use
of the word "make", assuming you were going from command line.

However I read on and you are creating a project with a controlling App file -
correct - so you are doing it from the IDE with a Build or Build All ? Now
sometimes, and very rarely, things just seem to get out of sync - can't think
when I had the last one. They are a pain in the butt and you have to get really
innovative to find them.

Now I do a lot of to-ing and fro-ing between my MSCOBOL V.5 DOS Text Editor and
the N/E IDE. I would say 99.9% of the time the N/E IDE recognizes I've updated a
source outside of N/E.

By no means a complete list, but some suggestions :-

- Are there any errormessage files (xxxxx.msg from memory), kicking around in
the Debug directory - delete 'em and try a Rebuild. (As appropriate delete any
int or gnt files - but you will find you will have to exit N/E to do this).

- Delete the copyfile from the Project IDE - do a Rebuild All, (or Rebuild just
this program, if only one is affected) and you are obviously going to get errors
because the copyfile is missing. Add the copyfile back into the project and
Rebuild All again. Most times this gives you the complete solution.

To ensure it's happening as you want - on adding or deleting, I make the project
change and get out of the IDE so that I know the alterations will get included
in the App file before it closes. Then reopen the IDE, in theory, with those
alterations included.

I know above is a bit weird - but it's you trying to beat the logic of the IDE
<G>

If above doesn't work and you are using University Edition, there is a separate
query area on Answer Exchange covering UE - where you could post your question.

Jimmy

I forget the actual sequencing, because it's only rarely I've had to do it. Now
my memory may not be correct on this one, but assume you remove the alphabetical
source from the right pane - I think because it is in the left-hand tree you are
going to get some sort of error message. The left-hand tree is what the
application is being built from..

However, if you add or 'put back' a source in the right-hand pane it should
AUTOMATICALLY get added to the left-hand tree. If you add back into the right
pane for something which is *still* in the left-hand tree - not quite sure what
happens.

I'm assuming for your problem you are going with the IDE defaults - I just don't
mess around with them. Just in case, from the IDE Menu Bar check out Animate
settings and the Pull-down Project Menu, the various sub-menu items,  and see if
you are set to defaults. Another thought, are you switching to and fro between
Debug and Release modes - or are you currently solely in Debug mode for testing
with the Animator ? Conceivably, although it shouldn't, but to-ing and fro-ing
*might* just create a problem.

Somebody did raise a similar query to yours recently on Answer Exchanges (for V
3.1), and M/F admitted there were some problems when using OO (which is what I
guess you are doing) - but what the problem was/is - I'm not sure. Other than
very infrequent "obscure" problems which I mentioned having, I don't have
problems with OO-ing - certainly not a whole application going belly up !.
