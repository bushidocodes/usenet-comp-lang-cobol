We are using NetExpress 3 on NT4 and have discovered a need for a file
editor. Although NetExpress has one which seems to be rather capable, it has
one glaring flaw - it tries to load sequential files entirely into memory
(relative and indexed it doesn't). We are trying to get access to large
sequential files (100+MB) where this just doesn't work (or at least isn't
practical).

Does anyone have a suggestion on a good file editor? It doesn't even need to
be COBOL specific (probably better that way) but having some COBOL-specific
features such as mapping to a record description would be a bonus.

Although it may appear so, the Data File Editor does not load the
entire file into memory. What it is doing is  building up pointers to
each record so that you can move around (backwards and forwards the
file once it is loaded ) and insert/delete records at arbitrary
positions in the file.

For fixed length records this isn't necessary, but for variable
length, the effort is well worth it.
