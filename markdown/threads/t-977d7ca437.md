comp-3 stores in binary coded decimal with the last nybble being a sign.

eg pic 9(3) comp-3 value 123 stores 1 in the first nybble, 2 in the
second nybble, 3 in the third nybble and the last nybble will be F (in
hex) if unsigned, C if positive or D if negative. In the case of just a
9(3) rather than S9(3) you'll only get F.

9(3) comp-3 is two bytes if that helps. I don't remember Pascal having a
BCD datatype so you'll have to do some conversion on a per nybble basis.

I am going to show my IBM mainframe ignorance here, but I think
I can help you a bit.

If I remember correctly, COMP-3 refers to a compressed numeric
field. (If that is incorrect, you can stop reading now, cause
the rest won't make any sense).

You can not download a compressed field to a PC. Why; because
when you translate from EBCIDEC (or however it is spelled) to
ASCII, the translator tries to translate the whole bytes that
have compressed digits in them, not the individual digits
themselves.

Try copying these fields to a dataset and storing them there
in an uncompressed format; then download the and have a look
at it.


Hope this helps.

David.
