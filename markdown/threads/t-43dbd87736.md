On Thu, 27 Jun 2002 10:40:41 -0400, Russell Styles wrote
> I always thought that rule was rather dumb.  What would
> be the harm in allowing an end-perform, even if redundant?

You have to either require it or prohibit it -- no optional -- because 
otherwise END-PERFORM becomes ambiguous. For example, in

  PERFORM UNTIL WHATEVER
    PERFORM 1100-PARAGRAPH
  END-PERFORM

which PERFORM does the END-PERFORM match?

But you can't require it for an out-of-line PERFORM, because that would 
be incompatible with existing programs (COBOL74 programs).

So it can't be optional and can't be required, thus it must be 
prohibited.

A lot of this mess is because PERFORM is already overloaded -- it's 
both a procedure call (albeit a poor one) and a looping mechanism. I 
see frequent confusion in programmers due to this overloading.

Edward Reid
