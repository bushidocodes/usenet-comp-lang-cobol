Abend AICA is a CICS Runaway (Looping) Task. The timeout-interval for
runaway tasks is defined to the SIT (System Initialization Table). A
common value is 5000ms/10000ms (milliseconds).

As far as how to prevent AICA's ? You will have to debug the program
using a 3rd party debugger (or, if you're handy with CEDF, that could
work also). You'll then have to locate the loop and correct the problem.

Making the determination as to what the problem could be must be
performed by the programmer. There are no hard and fast reasons. Just
grunt time to figure it out.

AICA's are VERY common in CICS Applications, so don't despair. you'll
find it.
