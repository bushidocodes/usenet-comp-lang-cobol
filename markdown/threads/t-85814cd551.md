There is no single definition for COMP-5. It is not defined in any
standard, and thus its interpretation is dependent on the platform and on
the design choices of the authors of the particular COBOL implementation
for that platform.

For example, in the COBOL (68) implementation on Unisys A Series systems,
COMP[UTATIONAL]-5 is used to specify a double-precision operand (96 bits)
in A Series internal floating point format.

True in "essence" - not quite true in fact. COMP-5 *is* defined in one
Standard. The X/Open Standard *does* define it - and defines it in a way
that is compatible with how most PC and Unix vendors have implemented it.
I.e. (as stated in other posts) in the machine hardware "native" binary
format. (For IBM mainframers - of particular interest - you *CAN* have one
byte binary fields with this usage - as defined by X/Open).

Just to be clear, all COMP-n (and for that matter COMP itself) USAGEs *are*
implementor defined (according to the ANSI and ISO Standards).
