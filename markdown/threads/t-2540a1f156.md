Hi again.
I don't recognise these symbols as being part of the COBOL system,
so I would say they are either Oracle or O/S functions (probably
Oracle).

› My linker end ok, but when I start the executable, it abend with runtime
› error 173 looking for __inf_prob_notify.

OK. This is because the "cob" command will automatically create
stub routines for the symbols which can't be resolved at link time.
Each stub routine, when called, simply calls the Micro Focus Runtime
System and passes it the name. The Runtime System will then attempt
to dynamically load the program. This functionality is provided for
when the static-linked part of your application (for instance, a
C function) wants to call a .int or .gnt program dynamically.

However, in your case, it is an error that the routines cannot be
found at link time, as they are not .int or .gnt COBOL programs
('cob' doesn't know this, so it assumes the symbols will be found at
runtime and creates the stubs). You need to ensure that the .a/.o
files containing those symbols are on the 'cob' command line.

You could use the 'nm' command to locate those symbols.

Cheers,
Kev.
