Glenn,

It wasn't clear to me (and maybe still isn't) that you want to just
suppress this, and not display the dialog.  I had assumed you wanted
to recover and have the code continue to run.

This is what I set with v6.1 and v7 - not sure it will work with v5
but I think it will.  All three are needed in the COBOL85.CBR.

@MessOutFile=fuj.lst
@WinCloseMsg=OFF
@CBR_JUSTINTIME_DEBUG=NO

In addition you should compile the programs using the NOTEST option
and OPTIMIZE.
