It is possibly a problem with termination.  The routine may be using
characters until it finds a space or low-values.  When Local-Drive is
a single character embedded in a structure the next byte is whatever
Net-Drive contains and the routine takes the value being passed as
"CF".  When it is an 01 level this takes up 4 or 8 bytes and the value
is "C     " which the routine correctly identifies as a single drive
letter.  Similar with the path.

When a parameter is passed in Cobol only the start address is given to
the routine, it must decide how long it is from the definition or from
context.
