```
There have been a number of threads recently about the sign-nibble in IBM
mainframe files (either USAGE DISPLAY or USAGE PACKED-DECIMAL/COMP-3).

It is true that such sign nibbles will ALWAYS include a C, D, or F - if the
file.  HOWEVER, there are some "obscure" cases where sign-nibbles may be A,
B, or E.  Current (VS COBOL II and above) IBM mainframe COBOLs have methods
for handling such data (as input) - but will NEVER produce them as output.

For information on IBM COBOL processing of these, see


http://www.s390.ibm.com/bookmgr-cgi/bookmgr.cmd/BOOKS/IGYPG204/1%2e3%2e5%2e2

(You would need to check out the "Principals of Operations" manual - to see
how/when these values get created.)
```

```
IBM COBOL doesn't really require any special code to process any sign
nibbles in range A-F, as all of these values have been accepted as legal
packed-decimal sign values since day one of S/360/370/390 architecture,
with "B" and "D" interpreted as negative, all others interpreted as
positive.  However, the hardware packed-decimal arithmetic operations
will always generate results with sign nibbles of "C" or "D".  A sign
nibble of "F" can also be generated by the commonly used techniques of
converting character representations to packed decimal. A COBOL (or
Assembler) programmer would have to go to extra effort to generate any
other sign nibble values, and doing so would generally be regarded as an
"unnatural" act.
```
