It is definitely possible to have alternate indexes over an ESDS base
file.  We do that all the time in CICS, but we use EXEC CICS calls to
browse and read the records by alternate indexes.  The alternate index
is built normally, but instead of a primary key to reference the KSDS
base record it uses the RBA to reference the ESDS base record.

In CICS, writing the ESDS base record causes all alternate indexes to
be updated.

I'm not sure how to code standard batch COBOL verbs to access this. 
My guess is you would treat the alternate index as if it were a KSDS
file, and reading the alternate key would also obtain the ESDS record.

I have maintained IDCAMS JCL that built alternate indexes and paths
over ESDS files.

You said exactly what I was going to say. There is a popular misconception
that  VSAM ESDSs cannot be indexed. In fact, it is often (not always) more
efficient to use them than it is to use KSDSs and that is one reason why DB2
defaults to ESDS (or it did the last time I used it on a mainframe...10
years ago now).

KEY  clause in the definition of the ESDS which (if memory serves) is
defined as a standard sequential dataset (I believe you can't use the RBA
from COBOL, but I could be very wrong on this). Not sure, but it should
work.

Okay, I was confused.  I was ready to make these arguments that Arnold
and Pete were the ones that were confused because it made no logical
sense that an ESDS file, which is defined as NON-INDEXED, could have
an alternate index.  Obviously this restriction has either never been
there and I just never saw it or it is a recent change.  I was
convinced this could be done after reading the following in the latest
IBM Access Method Services manual:

RELATE(entryname[/password]) 
names the alternate index's base cluster. The base cluster is an
entry-sequenced cluster or a key-sequenced cluster to which the
alternate index is to be related. You cannot relate an alternate index
to a reusable cluster or a relative record cluster. 

RELATE, for those who don't know, is a required paramater on the
DEFINE ALTERNATEINDEX command.
