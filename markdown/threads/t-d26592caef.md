COBOL is COBOL.  The whole idea behind third-generation computer languages 
is that they use the same source-level statements and verbs and functions 
regardless of hardware, operating system or compiler publisher.

Sure, there are differences in how you edit source code and compile/link 
your programs, and different hardware/operating systems offer different 
exploitable capabilties; but IMNSHO if you can create applications using 
"brand X" COBOL but not using "brand Y" COBOL you are stretching the truth 
when you claim to be a COBOL programmer.
