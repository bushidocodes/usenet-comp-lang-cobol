If it's not too late, the best way to convert data from one platform to
another is often to (using a utility if possible or writing a program if
necessary) convert all binary (including BCD) fields to display numeric
with sign leading separate on the From platform, transferring the resulting
character-data-only file to the To platform (with EBCDIC to ASCII
conversion as necessary) via a simple utility, and then converting the data
back to its original Cobol format on the To platform. That way the
differences in platform and compiler implementation become irrelevant. Or
it can easily be used to feed an SQL loader for tables.
