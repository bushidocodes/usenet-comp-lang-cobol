```
As pointed out, you can do linked lists without using pointers.

In a more general vein, you should not pick an implementation method (e.g.,
"pointers") until you have decided what to implement (e.g., "a linked list")
AND have reviewed several choices of implementations (e.g., "pointers,"
"subscripted table"  "other random access")  for your real needs.

Unless, of course, this is homework, in which case you should learn on your
own time.
```

```
'Pointers' are usually raw machine addresses, as used in machine code
and assembly programming.  Cobol is a high level language and avoids
such low-level grubbiness.

However, there is no difficulty in doing linked lists, or trees, or
other structures, just use an array and subscripts or indexes.

      01  LL-Root                   PIC S9(4) VALUE 1.
      01  LL-Limit                  PIC S9(4) VALUE ZERO.
      01  Linked-List-Table.
          03  LL-Item               OCCURS 5000.
           05 LL-Next               PIC S9(4).
           05 LL-Previous           PIC S9(4).
           05 LL-Data ....

The LL-Next and LL-Previous do the linking as if they were 'pointers'
but using these won't corrupt memory or cause a core dump and compiling
with BOUND (or similar) will debug their usage for you.

Of course this is usually a good reason to share data and so most
applications would store data in a file, usually ISAM, rather then keep
it in a private memory structure.
```
