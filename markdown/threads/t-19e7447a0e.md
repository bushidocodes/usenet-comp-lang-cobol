After we upgraded to v2.10, we started receiving abends in one of our
applications. We get abends in different programs, different ddnames,
and different jobs, but the first occurrence always seems to start
with a message like the following:

There was an unsuccessful OPEN or CLOSE of file SYS040 in program
RI531S at relative location X'092C'. Neither FILE STATUS nor an ERROR
declarative were specified. The status code was 90. From compile unit
RI531S at entry point RP531S at compile unit offset +0000092C at
address 38C1CFDC.

In our procedures, we have the following dd statement in different
steps

//sys005  dd ddname=sysin

When we want to override this statement, our jcl looks as follows:

//step.sysin  dd dsn=whatever

If we change the override to be

//step.sys005  dd dsn=whatever

then the abend goes away.

We may get other abends, but we always get the one above.  And the
abend is consistent.  We will continue to get it until we change the
override.   What confuses me is that this abend never occurs for any
other application even though they also use the same way to define and
override sysin.  There must be something different about this
application, but I can't figure out what.  We are using IBM COBOL for
OS/390 & VM  2.2.0.

Does anyone have any ideas as to why we are getting this and why it is
only this one application?  Note that the abend message is always for
a ddname that follows sys005 in the proc.

Any suggestions would be deeply appreciated as this is driving me
nuts!

Alexa
```

CBLQDA(ON) is particularly dangerous in a multi-step job.  If you make a
typo in your JCL for the DDName in one step, it will create a file which
does NOT get cataloged - and which canNOT be found in the next step.  This
is why your proc override gave me the clue that it was probably what your
problem was.

Although CBLQDA(ON) was the default in earlier releases of LE, even IBM has
changed to "OFF" as the default in the latest releases.  This violates some
rules of the ANSI Standard, but makes life MUCH easier for most OS/390
shops.
```
