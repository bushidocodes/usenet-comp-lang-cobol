```
I work for a state agency and have worked on a Windows version of an
existing DOS program created by an older version of Micro Focus COBOL.
The 2.4 version of MF COBOL would not install on my Win 98 machine
flashing some error during the installation process and would not
install at all.  So we get a few copies of Net Express at $3500 and
proceed to make my program.  After finishing the debugging, I make my
static build executable, copy the necessary files and try running it on
another computer.  It doesn't work, it wants dsgrun.  Then it wants a
few other modules as well.  After linking those in on a static build, it
wants pan2nt.dll.  After emailing Merant and getting a call back, we
find out an outrageous detail.  ANYONE who wants to run any programs
created by Net Express has to shell out $80-$90 for the runtime licenses
or do without.  This is a load of CRAP!  What do you think would happen
to Microsoft if THEY required you pay a sizable sum of money to run any
program someone created with Visual Basic or Visual C++?  The government
would be on their butts a LOT LOT worse than they are now.  And the
salesmen who sold us the copies of Net Express said NOTHING about it,
and the documentation says NOTHING about it.  What in the world is the
benefit for such a setup like it?  Its only for Merant trying to squeeze
a few more bucks out of people like they aren't making enough from
copies of the software or selling tech support for $600 bucks a year.

So my question is this...is there any way to make a Windows COBOL
program with Net Express and not have to install the run-time stuff on
other computers for it to run?  And certainly to avoid all of our
providers from having to pay this cost for a Windows program that they
have for free in a DOS program.
```

since my days using CA-Realia COBOL.  I bought a C compiler from Borland for <

When considering COBOL compilers - here's the state of affairs as related to
Runtimes - as I know them.

Note:  This is US Centric - some vendors might charge runtimes in Europe and
other contries, but not in the US.

Fujitsu - No runtime fee for the single threaded runtime.  Enterprise Edition
users are exempt from the Multi-Threaded runtime fee.  There are runtime fees
for optional things like PowerBSort (even in the US).

Merant - Runtime fees are confusing.  There's a special allowance for small
developers upgradeing from NetExpress 3.0 to 3.1 - but no allowance that I am
aware of for new 3.1 customers.  For Server Express licensing - running on a
server - it's EXPENSIVE (Around $10,000 per server).

RM - There is a per user runtime fee.
AcuCorp - There is a per user runtime fee.

PerCOBOL - No runtime Fee
CA-Realia COBOL - No Runtime Fee

TinyCOBOL (Free/Open Source/Linux) No runtime Fee

ICOBOL - There is a runtime fee

Objective COBOL - (From Brazil - Unknown)

IBM Visual Age COBOL - I've heard both, but I think it's No Runtime Fee (but you
have to RENAME the modules so the user can't easily tell they are from IBM)

COBOL. They all have licence charges but they all have different
schemes - now there's a surprise! However there is no harm to try a

be successful if you have a significant number, in effect if you can

developers as a heavy weight language for use by big business and thus
can justify a hefty charge for the compiler etc. Whether the Tiny
COBOl or GNU compilers will be useful as a 'personal' development

lower volume end of the COBOL development market will have to move. At
