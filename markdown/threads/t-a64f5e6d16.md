I don't want to start a language war but...

Most of the new 'business' applications I'm doing these days are in Delphi. 
Whereas I think it's a very good technology for GUI work and interfacing with 
SQL, I find it a royal pain in the ass to maintain other peoples code and the 
performance of typical applications is knowhere near that of COBOL (Micro 
Focus) in my experience. The overheads of file handling via SQL are largely to 
blame. In COBOL I use my own custom non-sql database handler. Raw file 
handling is just so much quicker in COBOL. 

Delphi's threading sucks too. MF COBOL's knocks seven bells out of it.

Where it does score well is in it's framework paradigm. I can re-use a lot of 
code very easily. I have an asset retrevial system with some 1500+ assets to 
call on, most of which are Delphi. I think I put one COBOL asset in there!  OO 
COBOL is not as good and few people understand or use it although I must say 
it's a much fuller OO language than Delphi. Why Borland stuck with Pascal 
instead of Modula (2/3) or Oberon is somewhat puzzling.

Delphi is also superb at prototyping GUI interfaces and doesn't have arcane 
languages behind the app like MF's Dialog system. Mix OO COBOL with Delphi's 
GUI creation ability and I'd be more than happy.


Shaun
