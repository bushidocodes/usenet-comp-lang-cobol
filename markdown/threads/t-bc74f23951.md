using the EXTERNAL keyword on data item.  This is the way in which
PowerCOBOL shares data between sheets.  For example:

01  MyApplicationState    IS EXTERNAL
                                            PIC 9(4) COMP-5 .

Donald,

A GLOBAL declaration provides access to the data item from within ANSI
COBOL85 Nested Program modules.  An ANSI COBOL85 EXTERNAL declaration is a
shared data item between two or more programs in the same run unit.

In Fujitsu PowerCOBOL if you want to share a data item between separate GUI
events, then you want to designate it as GLOBAL and if you want to share a
data item between separate sheets, then you want to designate it as
EXTERNAL.  You can combine GLOBAL and EXTERNAL so that a data item is shared
between events and sheets.

If you want to share data between two separate COBOL run units then you need
to use Fujitsu Common Communication Interface which is fully demonstrated in
SAMPLE09.
