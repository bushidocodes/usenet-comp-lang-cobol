```

I Standard COBOL, wherever you can code a single imperative statement
(PERFORM xyz), you can also code a series of COBOL *imperative* statements
(e.g., ADD ... ADD ...). The only problem would be if you wanted one or more
of these to be conditional statements (e.g. Add ... ON SIZE ERROR xxxx).
This is NOT allowed. However, you can change a conditional statement back
into an imperative one by putting in a scope terminator (e.g. End-ADD).
```

you can do any number of things within a evaluate when clause. your
first example, provided syntactically correct, should work fine.

just as a point of reference, you might want to consider the following
instead (going back to the old thread of readability/functionality/
whatever it was we were arguing about):

evaluate true
when out-of-state
do whatever-out
when in-state
do whatever-in
when other
do something-else
end-evaluate.

where out-of-state and in-state are 88 levels of the field you are
'testing', and other is a construct available when using evaluate. this
just makes it more obvious what you are actually basing your actions
upon, rather than '1' which could, at some time in the not so distant
future, mean something entirely different, and therefore change the
whole processing of this statement if someone doesn't catch the
reference within it to the field, or catch the significance of the
reference.
