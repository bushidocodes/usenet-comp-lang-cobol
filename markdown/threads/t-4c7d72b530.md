However, whatever you string will go if you say delimited by size.  So, 
given

  01  My-Record.
      12  My-Group-Item-1.
          16  My-Elementary-Item-1        Pic X(30).
          16  My-Elementary-Item-2        Pic 9(07).
      12  My-Elementary-Item-3            Pic X(45).

  01  My-String-Target                    Pic X(100).

, and My-Elementary-Item-1 contains "Now is the Time", 
My-Elementary-Item-2 contains "0000037", and My-Elementary-Item-3 
contains "for All Good Men", if you were to say

  Move Spaces to My-String-Target
  String My-Record  "** THE END **"  Delimited by Size
    into My-String-Target

, you would have "Now is the Time" starting in column 1, "0000037" 
starting in column 31, "for All Good Men" starting in column 38, and "** 
THE END **" starting in column 83.  Now, if you were to say

  Move Spaces to My-String-Target
  String My-Record, "** THE END **" Delimited by Space
    into My-String-Target

, you would have "Now**" in the first 5 columns of My-String-Target.

Now, if you were wanting to combine all the non-space characters in 
My-Record, that's pretty easy too.  Given these two additional variables...

  01  My-In-Idx                           Pic 9(03).
  01  My-Out-Idx                          Pic 9(03).

, you can now say

  Move Spaces to My-String-Target
  Move 1      to My-Out-Idx

  Perform Varying My-In-Idx from 1 by 1
    Until My-In-Idx > 100

      If My-Record (My-In-Idx:1) Not = Space
          Move My-Record        (My-In-Idx:1)
            to My-String-Target (My-Out-Idx:1)
          Add 1 to My-Out-Idx
      End-If

  End-Perform

, in which case, given the original values above, you would have 
"NowistheTime0000037forAllGoodMen" in My-String-Target.

Spaces do not have anything (necessarily) to do with a STRING statement.

The syntax is:

STRING
   data-field1 DELIMITED BY (delimiter-1)
   data-field2 DELIMITED BY (delimiter-2)
   ...
   INTO receiving-field.

(delimiter-n) can be SPACE (or 'b') or any elementary item, such as:
   "***" or "bbbbbb" (b=blank) or MY-TAB (X'09')

We routinely assign a file name thusly:

MOVE SPACE TO FILE-ID.
STRING DIRECTORY-NAME DELIMITED 'bb' (two spaces)
   FILE-NAME DELIMITED 'bb'
   LOW-VALUES DELIMITED SIZE INTO FILE-ID.

For bonus points, what would happen in the above example if DIRECTORY-NAME
did NOT have two consecutive spaces?

P.S.
    Yes, there's an extra step to squeeze out multiple spaces in both
DIRECTORY-NAME and FILE-NAME.

Here's another variation on STRINGing, using 'Double spaces' and "Single
spaces", plus handling the situation where some of the fileds you wish to string
together are blank  :-

 01 VESSEL-record.
    05 VESSEL-PrimeKey        pic x(04).
     05 VESSEL-Prefix             pic x(20).  *> "Colt"
     05 VESSEL-NameKey       pic x(04).  *> "WIP "
     05 VESSEL-Suffix              pic x(12).  *> (north)
     05 VESSEL-etc....

 01 ws-Name30              pic x(30).

Note that Ves-Prefix and Ves-Suffix are an EITHER/OR situation, they can be
blank or contain data in the Vessel Record; also their sizes are sufficient to
allow for TwoSpaces at the end of each. The major description is VERY
repetititive so Ves-NameKey contains a menomonic code - do a File/DB Table
Lookup and get the description for Ves-NameKey which is put into ws-Name30.

Now String :-

*>----------------------------------------------------------
*> CONDITIONS :
*> Prefix = spaces                            y   y   n   n
*> Suffix = spaces                            y   n   y   n
*>---------------------------------------------------
*> ACTIONS :
*> move   name                                x   -   -   -
*> String name with Suffix                 -   x   -   -
*> String name with Prefix                 -   -   x   -
*> String name with Prefix & Suffix   -   -   -   -

Linkage section.
01 lnk-fullname                  pic x(50).
Procedure Division returning lnk-fullname.

  move spaces to lnk-fullname
  evaluate true also true
    when VESSEL-Prefix =  spaces also VESSEL-Suffix = spaces
         move ws-name30 to lnk-fullname

    when VESSEL-Prefix =  spaces also VESSEL-Suffix <> spaces
         string ws-name30       delimited by "  "
                " "             delimited by size
                VESSEL-Suffix   delimited by "  "
                into lnk-fullname
         End-string

    when VESSEL-Prefix <> spaces also VESSEL-Suffix = spaces
         string VESSEL-Prefix   delimited by "  "
                " "             delimited by size
                ws-name30       delimited by "  "
                into lnk-fullname

                " "             delimited by size
                ws-name30       delimited by "  "
                " "             delimited by size
