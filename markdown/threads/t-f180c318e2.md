The programs created in the Unix machine do not need to be recompiled,
so you only need to copy the objects to the NT machine.

The same applies to the files.


Although the above is true in general, there may be a few things that
will prevent you from doing this wi though code changes or/and file 
conversions.

1- Program changes may be required in the following situations
1.1 - if you use escape sequences in your program to change screen
      attributes, or to display graphic lines.
1.2 - if you have foreign characters (like some used in french), and
      these are in a different codeset than the one used in the NT
      machine (most probable).
1.3 - if you do specific system calls to accomplish certain tasks.
1.4 - (HELL to whoever does it). If inside a run unit you issue a
      "call system" to start a new run unit


2- File conversions.
2.2 - if you have foreign characters (like some used in french) in
      the files, and these are in a different codeset than the one
      used in the NT machine (most probable).
   The work involved in this last case will be more difficult if the
   characters are used as a primary key.


There are other issues like file naming, and printers assignment,
but these can normally be bypassed by environment variables.


So the best thing for you to do, is to get a "borrowed" runtime and
try it.
If it doesnï¿½t run try and figure out exactly where does it stop, and
why.
