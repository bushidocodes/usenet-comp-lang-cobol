>> >     - Question 2 -  Including Java and Visual Basic, how many computer
>> > languages have tried to bury and replace COBOL in the last forty (40)

Unfortunately, the idea of burying an entrenched language on a legacy
system is folly.  I'd be interested in someone finding factual proof
that Java was invented to "bury COBOL."  A language only deserves to
be buried when it is intrinsically flawed.  Otherwise, it will
naturally hold on, and COBOL has done just that.  I don't like COBOL,
but it does get its job done.

>> >     - Question 3 -  Unlike Java and Visual Basic, which programming
>> > language truly runs on more computing platforms than any other?
>> >     - Answer: COBOL

I'm not a COBOL guru, but last I checked, There were multiple
different flavors of COBOL, making this an apples-to-oranges
comparison.  COBOL is kinda like C...there's the veneer of it being
the same everywhere, but there's also the underbelly that everyone
seems to have their different idea of what's a good implementation of
the existing standards.  Given this, a COBOL program based on one
"flavor" must have work done to it to port to another "flavor."
That's the way things go when you compile to native code.  Trying to
say COBOL runs on more platforms than Java is a rediculous comparison
anyway- it compiles to native code!  COBOL is a prettier way of
dealing with native code, much like C is.  By comparison, Java is a
prettier way of dealing with Java bytecodes.  The former members of
this comparison (C and COBOL) are going to be able to target more
systems because all that is necessary is to make a new native code
compiler to hit these systems.  However, C and COBOL aren't "running"
on that target system.  They just make binaries to do that.  This also
means there's no guarantee of portability...hence the folly in
claiming that COBOL runs on any platform.

By comparison, it is semantically true that Java runs on certain
platforms.  Java is a bytecode execution engine.  It is an actual
program written for various platforms.  Java bytecodes have a high
level of universiality because of this.

>> >     - Question 4 -  Which is the only programming language

>> >     - Answer: COBOL

I have heard about Visual COBOL and OO-COBOL.  Is there a COBOL

>> >     - Question 5 -  Which business oriented programming language has

Yep.  That's because they know that businesses would like to bring

COBOL had a mountain of legacy code that could benefit from the
wonderful new .NET CLR libraries, while Java had its own library set
to stand on.

Then there's the political issues between Microsoft and Sun that
almost make the arguments implied in Question 5 a bit dishonest.
