Try using a generic parsing program like ParseRat 
(http://www.guysoftware.com/parserat.htm).

If you have no idea what the original format is, you will have to click several of 
the option buttons before you see data which you recognise. (The file could be EBCDIC or 
ASCII, the numbers could be packed, binary, zoned etc. If binary they could be 
big-endian or little-endian depending on the original machine).
There might be a bit of trial and error involved.  It will be easier if you have a small 
sample file and a report or other printout of what the data in the file SHOULD be.
