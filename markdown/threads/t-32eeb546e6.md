```
There were several answers to this same question within the last few weeks -
or you can check DejaNews and the question gets answered every few months.
The bottom-line (IMHO) is that if you have a fully "current" COBOL compiler,
use the INTEGER-OF-DATE intrinsic function and divide by 7.  The remainder
gives you the day of the week (the same value as the ACCEPT FROM DAY-OF-WEEK
statement - except you get a 0 instead of 7).

If you don't have access to the COBOL intrinsic functions, then multiple
routines have been posted - or provided via ETK.
```

```
  05 DAY-OF-THE-WEEK      PIC 9.
  05 CALENDAR-DATE        PIC 9(8). (YYYYMMDD FORMAT).

  COMPUTE DAY-OF-THE-WEEK =
    FUNCTION MOD (FUNCTION INTEGER-OF-DATE (CALENDAR-DATE), 7) + 1.

  Result is 1=Sunday 2=Monday and so on.
```

```
An Fear Ar An nGealach wrote:
>How can you calculate the Day of the week, given the date.

The following is good for any date in the Gregorian calendar, through
9999/12/31.

       01  DATE-WORK-AREA.
           03  DW-WORK-CCYYMMDD            PIC  9(08).
           03  DW-WORK-CCYYMMDD-R          REDEFINES DW-WORK-CCYYMMDD.
               05  DW-WORK-CCYY                PIC  9(04).
               05  DW-WORK-MM                  PIC  9(02).
               05  DW-WORK-DD                  PIC  9(02).
           03  DW-WEEKDAY                  PIC  9(01).
           03  DW-TEMP-CCYY                PIC  9(04).
           03  DW-TEMP-MM                  PIC  9(04).
           03  DW-TEMP1                    PIC  9(08).
           03  DW-TEMP2                    PIC  9(08).

      *            C A L C U L A T E   D A Y   O F   W E E K
      *
      *    WHEN GIVEN ANY DATE THIS ROUTINE RETURNS A NUMBER FROM
      *    1 TO 7 INDICATING THE DAY OF WEEK ON WHICH THE DATE FALLS.
      *
      *    USAGE:  MOVE <DATE> TO DW-WORK-CCYYMMDD
      *            PERFORM CALC-DAY-OF-WEEK
      *
      *    RESULT: DW-WEEKDAY = DAY OF WEEK
      *
      *    THIS ROUTINE USES ZELLER'S CONGRUENCE, EXPRESSED BY THE
      *    FOLLOWING FORMULA:
      *
      *        <WEEKDAY> = (INT(13 * MONTH + 3) / 5)
      *                   + DAY
      *                   + YEAR
      *                   + INT(YEAR / 4)
      *                   - INT(YEAR / 100)
      *                   + INT(YEAR / 400)
      *                   + 1) MOD 7) + 1
      *
      *    WHERE: WEEKDAY = 1 TO 7: 1=SUN, 2=MON, ..., 7=SAT
      *           DAY     = DAY OF THE MONTH
      *           MONTH   = MONTH OF THE YEAR (JAN AND FEB COUNT AS
      *                      MONTHS 13 AND 14 OF THE PREVIOUS YEAR)
      *           YEAR    = FOUR DIGIT YEAR (YEAR - 1 IF MONTH IS
      *                      JAN OR FEB)
      *           INT(...) MEANS TAKE THE INTEGER PART (NO ROUNDING)
      *           X MOD Y  MEANS THE REMAINDER AFTER DIVIDING Y
      *                      INTO X
      *
       CALC-DAY-OF-WEEK.
           MOVE DW-WORK-CCYY TO DW-TEMP-CCYY.
           MOVE DW-WORK-MM   TO DW-TEMP-MM.
           IF (DW-WORK-MM < 03)
               ADD 12 TO DW-TEMP-MM
               SUBTRACT 1 FROM DW-TEMP-CCYY
           END-IF.
           COMPUTE DW-TEMP1 = (13 * DW-TEMP-MM + 3) / 5.
           ADD DW-WORK-DD DW-TEMP-CCYY TO DW-TEMP1.
           DIVIDE DW-TEMP-CCYY BY 4 GIVING DW-TEMP2.
           ADD DW-TEMP2 TO DW-TEMP1.
           DIVIDE DW-TEMP-CCYY BY 100 GIVING DW-TEMP2.
           SUBTRACT DW-TEMP2 FROM DW-TEMP1.
           DIVIDE DW-TEMP-CCYY BY 400 GIVING DW-TEMP2.
           ADD DW-TEMP2 TO DW-TEMP1.
           ADD 1 TO DW-TEMP1.
           DIVIDE DW-TEMP1 BY 7 GIVING DW-TEMP2 REMAINDER DW-WEEKDAY.
           ADD 1 TO DW-WEEKDAY.
```

```
You've had a couple of answers showing the Cobol way of doing this. If you
still want to use lib$day_of_week then you need to change the definition of
your variables.

According to VMS help (help RTL LIB LIB$DAY_OF_WEEK), the first parameter
should be an unsigned quadword, and the second an unsigned longword. In VMS
Cobol terms this translates as:

01      THE-DATE        PIC 9(18)   COMP.
01      DAY-NUMBER      PIC 9(9)    COMP.

Try this and see what happens.
```
