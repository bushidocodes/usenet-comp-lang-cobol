4.2 works fine on NT 3.51. I havent tried it on NT 4.0 yet (just
installed it), but I dont anticipate any problems.

When invoking the debugger use the syntax "FORCEDOS REALDBUG
whatever_other_syntax_you_need". This makes the debugger run as a DOS
program instead of as a 16 bit OS/2 program. 16 bit OS/2 exe's still
work.

The debugger DOES NOT work on Win95, however CA just recently annouced
FREE upgrades from Realia COBOL to a new Win95-tolerant release.

Likewise, if you have COBOL and CICS, you can get the new
Win95-tolerant workbench for free. Maintenance does up a little, but I
think it a good deal despite that.

Im not related to CA in any way.
