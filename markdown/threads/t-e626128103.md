Subject: Setting up Micro Focus COBOL on a laptop w/o CD-ROM

This may be slightly off-topic, but then again maybe not. I am
providing this information in case anyone else is confronted with this
somewhat unusual problem:

The MF Cobol/Windows was sent to me on CDROM. It does work with
Windows 95 and given the contemporary times, Win95 PC's are likely to
have a CDROM so you can install it on that PC, but you have a problem
installing it on a Notebook or other PC that runs Windows 3.1 but has
no CDROM.

Assuming you have LapLink/Win31, this is what you can do:

You can't run LapLink for Windows with a Windows 95 machine.
So, make a boot disk using Windows 95 DOS (SYS: A:)
(If you use a DOS 6.22 boot disk, you can't see the C: drive)
Copy the Lap Link floppy onto this new read-write floppy
Create an autoexec and config.sys with the CDROM Drivers
Boot with the new floppy, you will now see the Win95 HDD and E:
Run Lap Link on both machines, using special disk for Win95 m/c
Now the machines will connect, and you can see the CDROM on W95 m/c
Copy all the executables over into c:\COBOL on the target (appr 8 MB)
Install using Windows Setup
If your COBOL on the target machine still does not work:
Copy the missing (4) files from the Win95 PCOBOL directory on W95 m/c
This should do it, now the MF COBOL should work on your Note Book
