I'm not used to work with MSAccess, but I think that there is a problem
regarding authentication. Are you setting any meaningful values for "UserID"
and "Password" (these are properties of your Cmdb1 control) ? I think that
the ODBC driver for Access has a way of turning off this authentication.
Check in the DataSource creation panel in the Control Panel.
Any how, I find very useful to turn on the tracing feature of the ODBC
driver and inspect the generated log. Normaly, things seem confusing there,
but the messages are all there too.
