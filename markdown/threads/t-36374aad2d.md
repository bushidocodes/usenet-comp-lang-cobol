Yes, Java is a 'fashionable' language, as was Algol, PL/I, Pascal, Ada,
dBase, C, etc. And as is VB, Perl, C++.  The 'Computer Science' types
will move on to the next language when that comes along: C#, Python, or
something new next year.

Businesses cannot change just because 'Computer Science' fashions do,
they have systems that last for decades and cannot afford to discard
them or rewrite them because some students want to play with newer toys.

Cobol has been used for 40 years and can be moved from old system to
current ones, and will move to the next range of computers.  Other
languages change from year to year, which is OK for toys that will be
thrown away, but is useless for businesses.

Java is the language that was seen as the most likely to remain stable
for some time.  At the time it came out C++ was being revised and
changed with its library going through a huge growth, a new VB was
breaking all old programs (which will happen again with next release),
and others were fragmenting into different versions.

Personally I think that Java has passed its peak and the CS types will
move to some new toy such as C# or a derivitive and then this will
fragment.  Meanwhile Cobol will continue while Python will grow as the
internet and scripting language.

As for 'Web-based' I use Cobol for business applications on the web and
Python for scripting.
