```
I was trying to use CBL_ALLOC_MEM with 3.2.24 and got these
results with the program shown below:

In DOS animator, the CALL to CBL_ALLOC_MEM takes a very
long time to execute ( >30 seconds). When it finally completes, the
status is 157, Not enough program memory: object file too large
to load.

In the Graphical environment, the program failed with 157 and
Animator also failed with 157.

When linked with "cob -x:l allocmem", it worked fine.

I upgraded to 3.2.50 by downloading and applying the update
from the Micro Focus ftp web site.

In DOS animator, there was no difference.

In the Graphical environment, the program failed, as before, but
the Animator no longer failed.

When linked with cob, as before, it now gives error 116, Unable
to allocate memory.

Any suggestions?
-------------------------------
Rick Smith
e-mail: < ricksmith@aiservices.com >

---------------
       program-id. allocmem.
       data division.
       working-storage section.
       01 alloc-pointer pointer.
       01 alloc-mem-size pic x(4) comp-5.
       01 alloc-flags pic x(4) comp-5 value 2.
       01 alloc-status pic x(2) comp-5.
       01 char pic x.

       linkage section.
       01 allocated-memory pic x(32).

       procedure division.
           move function length ( allocated-memory )
               to alloc-mem-size
           perform alloc-memory
           set address of allocated-memory to alloc-pointer
           move all "*" to allocated-memory
           display allocated-memory
           perform free-memory
           accept char
           stop run
           .

       alloc-memory.
           call "CBL_ALLOC_MEM" using alloc-pointer
               alloc-mem-size alloc-flags
             returning alloc-status
           if alloc-status not = 0
               display alloc-status
               accept char
               stop run
           end-if
           .

       free-memory.
           call "CBL_FREE_MEM" using alloc-pointer
           .
------------------
```
