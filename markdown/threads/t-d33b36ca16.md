What style you choose to use is a matter of opinion, I'll give you that.
I still think the use of periods are old-style.  In the old days you didn't
have any choice.  Now the programs may have a better structure using
if/then/else/end-if and evaluate/when/end-evaluate.  The period enforces
multiple closure of all open ifs and make some program constructs very
difficult.  As an example, how would you implement the following construct
using period and no end-if (I know how to do it but it's not nice)?

if condition-1
   statement-1
   if condition-2
      statement-2
   end-if
   statement-3
end-if

Gunnar.
