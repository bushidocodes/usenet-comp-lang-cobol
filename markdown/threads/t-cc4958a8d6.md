Try this. Nested program with REPLACE. For multiple sorts I set it up
as a COPY and use COPY REPLACING ==:NAME:== BY ==progrname==.
      * :NAME:   = NAME OF PROGRAM
      * :LENGTH: = LENGTH OF EACH ENTRY
      * :KO:     = KEY OFFSET RELATIVE TO 1
      * :KL:     = KEY LENGTH
      * :COMP:   = 4 (FEWER THAN 32768 ITEMS), 8 (UP TO 2 GIG ENTRIES)
      * :A-OR-D: = < FOR ASCENDING, > FOR DESCENDING SEQUENCE
           REPLACE ==:NAME:==      BY ==RPSORT==
                   ==:LENGTH:==    BY ==47==
                   ==:KO:==        BY ==1==
                   ==:KL:==        BY ==25==
                   ==:COMP:==      BY ==4==
                   ==:A-OR-D:==    BY ==<==.
       IDENTIFICATION DIVISION.
       PROGRAM-ID. :NAME:.
      * COPYRIGHT 1979 BY JOHN YINGLING.
       ENVIRONMENT DIVISION.
       DATA DIVISION.
       WORKING-STORAGE SECTION.
      *---------------------------------------------------------------*
      * TREESORT3 WORKS
       01  WORKS SYNC.
           05 I                                  PIC S9(:COMP:) COMP.
           05 LS                                 PIC S9(:COMP:) COMP.
           05 MS                                 PIC S9(:COMP:) COMP.
           05 AS                                 PIC S9(:COMP:) COMP.
           05 FS                                 PIC S9(:COMP:) COMP.
           05                                    PIC X.
              88 0-W-INLOOP-DONE                 VALUE LOW-VALUES.
              88 W-INLOOP-DONE                   VALUE 'Y'.
       01  HOLD                                  PIC X(:LENGTH:).
      *---------------------------------------------------------------
       LINKAGE SECTION.
      * # OF ENTRIES IN THE TABLE
       01  MAX-SUB                       PIC S9(:COMP:) COMP.
       01  SORT-T.
      *         OCCURS IN LINKAGE NOT NECESSARY WHEN NOSSRANGE
      *         IT JUST WASTES TIME AND STORAGE FOR A BLL EVERY 4096.
           03 S OCCURS 1 TIMES
                INDEXED BY A F L M       PIC X(:LENGTH:).
       PROCEDURE DIVISION USING MAX-SUB SORT-T.
       MAIN SECTION.
           PERFORM SORT-IT
           GOBACK
           CONTINUE.
       SORT-IT SECTION.
      * FROM SORTING AND SORT SYSTEMS - APPENDIX B A-65
      * WITH A CORRECTION TO INITIALIZE L
           COMPUTE MS    = MAX-SUB
           COMPUTE LS    = MS
           SET M        TO MS
           SET L        TO M
           COMPUTE I   = MS / 2
           PERFORM VARYING I FROM I BY -1
                     UNTIL I < 2
               COMPUTE AS = I
               SET  A     TO AS
               MOVE S (A) TO HOLD
               PERFORM UPTREE
               IF I NOT = AS
                   MOVE HOLD TO S (A)
               END-IF
           END-PERFORM
           MOVE 1 TO I
           SET LS       TO M
           PERFORM VARYING L FROM M BY -1
                     UNTIL L < 2
               COMPUTE AS    = 1
               SET  A       TO 1
               MOVE S (1)   TO HOLD
               PERFORM UPTREE
               IF 1 NOT = AS
                   MOVE HOLD TO S (A)
               END-IF
               MOVE S (1)   TO HOLD
               MOVE S (L)   TO S (1)
               MOVE HOLD    TO S (L)
               SUBTRACT 1 FROM LS
           END-PERFORM
           CONTINUE.
      ****************************************************************
       UPTREE SECTION.
           SET 0-W-INLOOP-DONE TO TRUE
           PERFORM WITH TEST AFTER
                 UNTIL W-INLOOP-DONE
              COMPUTE FS = 2 * AS
              SET F TO FS
              IF FS < LS AND
                 S (F) (:KO:::KL:) :A-OR-D: S (F + 1) (:KO:::KL:)
                  SET F UP BY 1
                  ADD 1 TO FS
              END-IF
              IF FS <= LS AND
                 HOLD (:KO:::KL:) :A-OR-D: S (F) (:KO:::KL:)
                  MOVE S (F)          TO S (A)
                  COMPUTE AS           = FS
                  SET  A              TO F
              ELSE
                  SET W-INLOOP-DONE TO TRUE
              END-IF
          END-PERFORM
          CONTINUE.
       END PROGRAM :NAME:.
          REPLACE OFF.

Huh?  Depending on your O/S and/or compiler, SORT *is* a callable service.

But...

I think Microfocus COBOL supports a COBOL SORT of a table (array) in
addition to standard COBOL sorts of files defined with SD's (i.e.,
RELEASE..RETURN or USING...GIVING).

In the mainframe environment, it's quite common to sort files outside the
COBOL program before the COBOL program ever sees the data, using DFSORT or
SYNCSORT. You can then sequentially load a table in sorted order (presumably
to facilitate binary searches).

Depending on your OS, there are expensive third party sort libraries
 SyncSort, Opt-Tech Sort) and free sorting tools written by talented and
generous developers (e.g., moi and the soon-to-be-mentioned Mr Maclendon).

You can write your own table sort using standard COBOL verbs. I think Judson
Maclendon has some examples on his web page (check one of his posts in this
forum for the URL).

Last but not least, if that's not enough to stimulate your grey matter, surf
on over to http://www.mvshelp.com and browse through the messages in the
"sort" and "programming languages" sections of the help board. That's all
mainframe, but you might get some ideas which port to other OS's.

Oh, yeah: what the hell is wrong with the COBOL sort anyway?

MCM
