COBOL is not well suited for this kind of thing.  But you can usually get
the job done, although the solutions tend to be system/compiler specific.
Micro Focus COBOL provides bit fiddling functions for this sort of thing.
Here is a general approach which will usually work, though you will need
to adapt it for your specific compiler.  Basically, you define a PIC X
field and redefine it as a binary numeric type, or vice-versa, then you
reference the binary numeric type as a numeric value, using add, subtract,
etc. to do the work.  The following Micro Focus program contains a routine
which takes a byte and splits each bit into a separate PIC 9 field, and
another routine which merges the separate fields into one byte.  Once the
bits are separated, you can easily test them individually.

       IDENTIFICATION DIVISION.
       ENVIRONMENT DIVISION.
       DATA DIVISION.
       WORKING-STORAGE SECTION.
       77  WS-CHAR             PIC  X(01).
       77  WS-VALUE            PIC  9(03).
       01  BINARY-WORK.
           03  BW-BINARY       PIC S9(04)  BINARY  VALUE 0.
           03  BW-ALPHA        REDEFINES BW-BINARY.
               05  FILLER          PIC  X(01).
               05  BW-CHAR         PIC  X(01).
           03  BW-BITS.
               05  BW-BIT128   PIC  9(01).
               05  BW-BIT64    PIC  9(01).
               05  BW-BIT32    PIC  9(01).
               05  BW-BIT16    PIC  9(01).
               05  BW-BIT8     PIC  9(01).
               05  BW-BIT4     PIC  9(01).
               05  BW-BIT2     PIC  9(01).
               05  BW-BIT1     PIC  9(01).
           03  BW-BIT          REDEFINES BW-BITS
                               OCCURS 8 TIMES
                               INDEXED BY BW-X
                                   PIC  9(01).
       PROCEDURE DIVISION.
       TEST-IT.
           MOVE "I" TO BW-CHAR.
           MOVE BW-CHAR   TO WS-CHAR.
           MOVE BW-BINARY TO WS-VALUE.
           PERFORM SPLIT-BYTE.
           DISPLAY WS-CHAR " "
                   BW-BIT128 BW-BIT64 BW-BIT32 BW-BIT16
                   BW-BIT8   BW-BIT4  BW-BIT2  BW-BIT1
                   " " WS-VALUE.

           MOVE 1 TO BW-BIT16.
           MOVE 0 TO BW-BIT1.

           PERFORM MERGE-BYTE.
           MOVE BW-CHAR   TO WS-CHAR.
           MOVE BW-BINARY TO WS-VALUE.
           DISPLAY WS-CHAR " "
                   BW-BIT128 BW-BIT64 BW-BIT32 BW-BIT16
                   BW-BIT8   BW-BIT4  BW-BIT2  BW-BIT1
                   " " WS-VALUE.
           STOP RUN.

       SPLIT-BYTE.
           PERFORM VARYING BW-X FROM 8 BY -1
                   UNTIL BW-X < 1
               DIVIDE 2 INTO BW-BINARY
                   GIVING BW-BINARY
                   REMAINDER BW-BIT(BW-X)
           END-PERFORM.

       MERGE-BYTE.
           MOVE 0 TO BW-BINARY.
           PERFORM VARYING BW-X FROM 1 BY 1
                   UNTIL BW-X > 8
               COMPUTE BW-BINARY = BW-BINARY * 2 + BW-BIT(BW-X)
           END-PERFORM.
```

There are only 256 different values that can be held in a byte.
A table of all the values could be used then it is only
necessary to use the input values and the original value
to select the appropriate result.  This way the change
can be done without changing any bit values at all.

It is also possible to do it aritmetically, but adding or
subtracting must only be done if the bit is not already
in the desired setting.

```
Jason,

    Conceptualize exactly what an 'NI' (AND-IMMEDIATE) accomplishes. Example
in IBM ASSEMBLER ===> NI    BYTE, B'01000000' with 'BYTE' equal to X'FF'
before the instruction. Subsequently, you'll wind up with a X'40' (A SPACE
IN EBCDIC).

    Now, in COBOL, set up a table of S9(04) BINARY fields, with the
following values:

Occurs 08 ===> 128
Occurs 07 ===> 064
Occurs 06 ===> 032
Occurs 05 ===> 016
Occurs 04 ===> 008
Occurs 03 ===> 004
Occurs 02 ===> 002
Occurs 01 ===> 001

03 WS-TBL OCCURS 8 TIMES PIC S9(04) BINARY.

Let's say you want to turn-off the 8th-Bit (leftmost) in a field called
WS-BYTE, which has a value of X'C0'. Move WS-BYTE to the low-order of a
redefined halfword, ensure the halfword's high-order byte is X'00' and
SUBTRACT the value found in occurrence 08 in the table, which is decimal 128
(X'80'). The result will be X'40', which is B'01000000'. You've just issued
an 'AND-IMMEDIATE' but in COBOL.

03    WS-HALFWORD    PIC S9(04)    BINARY
03    WS-HALFWORD-X    REDEFINES WS-HALFWORD PIC X(02).
03    WS-BYTE PIC X(01) VALUE X'C0'.

MOVE LOW-VALUE TO WS-HALFWORD-X (1:1).
MOVE WS-BYTE TO WS-HALFWORD-X (2:1).
SUBTRACT WS-TBL (8) FROM WS-HALFWORD.

WS-HALFWORD-X now equals X'0040', which is decimal 64. Now, MOVE
WS-HALFWORD-X (2:1) to WS-BYTE and you've got your answer.

You can now use the table components to turn-on and turn-off bits in a given
byte.

HTH....

WOB,
Atlanta
