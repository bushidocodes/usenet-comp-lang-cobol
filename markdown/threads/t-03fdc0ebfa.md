Well, Fujistu offers an older (version 3 COBOL compiler that runs just fine on Windows,
and the price cannot be beat - it is free. Check out www.adtools.com...)

But beyond that, ask yourself if you are primarily interested in applications under Linux or in
doing "systemprog" like things under Linux.

C - Can do *anything* under Linux, is well supported, easy to learn, and equates to assembler
       on the mainframe.

C++ - can do most applications, but has a much harder to learn paradigm and syntax than standard
           C, and I don't recommend it to people just starting in the Unix/C world. Stay with standard C.

Java - Is great for applications, is perfectly portable between platforms, and with a little effort,
          can be used for systems level stuff.

I'm a little bit confused what you mean by a Java program that does useful things - a google search will bring up thousands of
examples like you ask about.  There are three linked below, but when I said thousands, I really meant more like 10's of thousands.
;)

All in all, I would recommend, in particular since you re talking Linux, you learn C and develop your application in C, unless you
want a GUI, in which case, use Java.

Well, as an out-of-work COBOL programmer with six years experience who is
having problems getting work, I'm currently in the process of learning both
languages to make myself more attractive to employers.

Unless you've done Object-Orientated COBOL, the main hurdle to overcome is
that they are both OOPLs and requires a slightly different mindset.

Java is the easier language to learn and you can achieve results quicker
with it. There also seems to be a few books and texts along the lines of
"Java for the COBOL Programmer", as if thinking in Java was akin to thinking
in COBOL. A chap by the name of Bruce Eckel has put up on the web to down
for free a few of his books http://www.mindview.net/Books/. Titles include
Thinking in Java and Thinking in C++. There are more free books out there on
Java and C++, try a Google search for them.

Take a trip to your local library and see if they have any progamming books
on Java/C++. I borrowed Teach Yourself Java in 24 Hours and am currently up
to Hour 17. It won't make you an expert in Java but does give you a taster
of the language.
