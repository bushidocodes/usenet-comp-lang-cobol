```
There have been a couple of posts pertaining to the handling of Variable Length
Records, one of them unclear as to platform, the other for IBM MVS.

I responded to one of them, saying that the RECORD 0 clause could be used to get
around problems with checking the system might do on OPENing of VB data sets.
There were several later posts, still unhappy about the fact that VS COBOL II's
behavior when not trying to emulate OS/VS COBOL's behavior was giving them
problems. Both compilers have had this feature (RECORD 0) in them all along.
Has anyone tried it yet? It should be really quick to modify your program and
run it....

(Be sure you define the FD's records in such a way as to make the compiler
understand they are variable length, or add the proper clauses/phrases to the
FD.)
Colin Campbell
```
