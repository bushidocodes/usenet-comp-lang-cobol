Thank you for your replies (and I really do appreciate it), but what you
told me was actually the easy part. The BTRV file assignment in the
select statement is probably the only piece of BT documentation that
exists in the Fuji manuals.  But there are a few other things I've
discovered along the way ...

It is not at all necessary to reference any sort of key to read these
files sequentially.  Simply omit any ISAM-like refences in the select
statement and do READ NEXT's all the way thru to the end.

I have been able to do STARTS once I figured out how to identify the
primary key.  The first byte of the key (and all alphameric fields for
that matter) is the field length in binary form.  Upon opening the file,
I read the first record and store the field lengths for later WRITEs. 
It's a workaround and there's probably a better way to do it but at
least it works.

Speaking of field lengths, the CURRENCY data type is stated in the
manual as being equivelent to PIC s9(14)v9(4) COMP-5.  But using that
picture really screwed things up.  Turns out what seems to work is
S9(14)v999 COMP-3 with a left-over filler byte (note comp-3 not
comp-5).  Go figure.

These files have a number of alternate keys, but if I specify any of
them in the Select statement then Fujitsu gives me an open error.  I
guess I'll have to keep playing with them and something will click, but
at least for now I've discovered enough that I can hobble together a few
quick queries.

Again ... if anyone out there (like someone who has done this before)
could possibly refer me to some documentation or give a few tips it
would save me a lot of time.  Thanks.
