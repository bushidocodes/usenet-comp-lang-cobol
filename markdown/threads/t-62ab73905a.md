The error is not a problem with the program source code. Part record
read error is because a file that is record sequential must be a
multiple of record-length in size. If you have a file defined with
an actual record size of, say, 100 bytes and yet the file is not a
multiple of 100 bytes the the last record will not be a complete
record giving this error.

It may be that you have defined the file incorrectly, or it may be
that the file is actually LINE SEQUENTIAL with variable records
terminated by CR-LF, or it may be the file is corrupted.
