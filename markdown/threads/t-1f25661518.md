Florian Teichert <floteichNO@SPAMhrzpub.tu-darmstadt.de> wrote:
>I could although use DS is there an easy method, too??

It would essentially be the same method, except that you would have to 
subclass (ie create another callback procedure for, see below) one of your 
Dialog System windows. Dialog System would also take care of the Windows 
message loop.

>What do you mean by "callback-based"? Do I have to look at the message queue
>at specific places in my program to check if something is in for me?
>I'd rather want an event to be triggered when a message is posted to me.

Callback-based means that you have to export an entry point in your 
program that the system calls you back on to inform you that something has 
happened. The whole of the Windows GUI is a callback-based system, where 
you create a window with a registered callback procedure, and Windows 
sends you messages on that procedure.

If you want to pick up a specific message sent to the window, you evaluate 
the messages sent to the window in the procedure. You can then call 
whatever message/"event"-handler you want. In the example I pointed you 
to, the procedure "MyWndProc" is the callback, and it deals with some 
standard Windows messages such as WM_COMMAND.

So, if you can get Delphi to post a message to your window, you can check 
for that value in the callback procedure that handles messages for that 
window! I don't know Delphi, but it is almost certainly doing something 
similar.

Steve.
