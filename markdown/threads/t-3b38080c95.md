```
1. Check your library search by using the set command. Look for the
lib variable.
2. Check your compiler directives. You maybe telling the compiler
to compile as a coblib shared runtime object, while you are linking
to the static lcobol libary. Your compile step produces and object
module module.obj that has a reference to coblib.lib, and you need
to change your compiler directives using a text file cobol.dir or
other method.

I think the compile directive you need is this:
OMF"OBJ" (you may be producing a .gnt file...)

Otherwise, you can stop the linker from searching further by
using the linker direct /NOE (no extended search) this may
stop the duplicate names between coblib and lcobol, since
it may stop it from looking at coblib...

like so...
link compta+mdisplay+maccept+...,,nul,lcobol+cobapi/NOE;
```
