New state---
MF Cobol RTS32 running in POSIX environment with programs build with
'callable' I-O modules, 'callable' businessfunctions and 'callable'
frequently used general functions. Data Management on a separate database
server (Oracle) via middleware communication(TCP) between RTS and Oracle
server.

Problem description.

Middleware communicates via TCP the FileControlData as well as buffer
content for each proces-id from the client towards the database server and
vice versa.

114     Attempt to access item beyond bounds of memory (Signal 11)
Occurs in the following situation:

Main program doesn't do anything with a needed table. The OPEN of this table
is done by a called I-O program (io_tablename.int).
Traces show a succesfull operation.
A business function is called from the main program that calls in turn the
forementioned I-O program with specific functions like START / READ NEXT.

As soon as a buffer is retransmitted (READ or READ NEXT) from this, as a
third level, called I-O module to RTS the above Error occurs.
When done the same I-O action directly from the main program, as a second
level, everything functions as planned.

Summary:
1. All programs, main and the called ones, have the same proces-id
2. The I-O modules work with tables(Files) NOT declared in the main program,
but only declared in the called I-O programs.
3. When calling a I-O module with buffer data exchange from main no problem
arise. When calling from a called business function RTS complains with the
forementioned error message and signal preceded by the program name of the
main program that isn't active at that moment!
4. Called programs aren't CANCELled.
5. RTS seems to interpret FCD information from a, in a second level, called
program to connect to the main program and obvious these address information
is in a total different part of memory and so beyond bounds of MAIN program
memory.
6. Working in a non Client/Server structure didn't result in errors.

Is there anyone that have some thoughts, ideas or theories that might
explain the different results when doing ALMOST? the same within the one
environment?

Kind regards,

Peter Samson
