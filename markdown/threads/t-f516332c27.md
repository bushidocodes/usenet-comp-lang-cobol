That would depend (a lot) on your particular brand compiler (MF, AccuCobol,
etc) and your operating system (MS-DOS, Windows, Unix).

I think "Microsoft Cobol" was only offered in an MS-DOS version, and only a
very long time ago.  I am not aware of any software designed specifically to
read/write Access(r) databases from MS-DOS, so that would leave you only the
option of calling a Windows program from your DOS program.

Or, I guess,  figure out the structure of the *.dbm files and  try to read
those as a stream-type input.

That is, if there is no canned software which can read/write Access Db's
from MS-DOS, you are basically SOL.

MS Cobol 4.5 (1991) is equivalent to MF Cobol 2.5 and will create DOS,
OS/2 1.x and Windows 3.1x programs.  MS Cobol 5.0 (1992) is == MF
Cobol 3.0 and will do DOS and Windows 3.1 programs, the ability to do
OS/2 1.x was crippled by MS.

5.0 (later in MF 3.4), ie drawing graphics in several windows and
drag&drop etc.
