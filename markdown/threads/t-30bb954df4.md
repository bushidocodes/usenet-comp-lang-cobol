```
>  On our Solaris machine, if we
> "open output" a file that already exists, cobol restarts the file as
new
> and doesn't complain.

I think that's the way an operating system/COBOL is supposed to work.

"If you attempt to open for output a file that already exists, all of
its
existing records are deleted; otherwise a new file is created."

If your issue is whether to open as I-O (for a file that already
exists) or output (for a new file), try opening I-O first and check
the status bytes: if the open was successful, the file is there;
if the open failed, the file is not present and you will have to
open as output.
```
