There are libraries which allow the direct creation of PDFs using
CALLs from COBOL. I don't recommend this as it locks the format into
the program.

I use a postscript template and merge the data into it then either
print this or use ps2pdf to convert to PDF. This allows different
templates to be selected by the program and for these to change
without amending the program.

Postscript templates can be created by a number of drawing programs, I
use tgif

 http://bourbon.usc.edu:8001/tgif/index.html

The template is drawn with tags to indicate where the data items are
to go. For example:

   <!%ADDRESS%--------------->
   <!>
   <!>
   <!>

A single data item just has the <!%name%--> tag. The - are to fill out
the length. A multiline item has <!> tags for the following lines.

Some drawing programs are completely unsuitable because they break up
the text fields making it impossible to find the tagged fields.

I wrote a merge program many years ago that takes a text file with the
names and data and fills the template so all that is necessary is for
the Cobol to write the text data and CALL "SYSTEM" to merge, ps2pdf
and email (or print).
