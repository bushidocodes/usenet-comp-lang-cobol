I'm writing some generic COPY members to handle LOCK situations, and found
that the START statement in ILE COBOL has a NO LOCK clause. This will cause
the START not to lock a record.

In OPM COBOL the documentation doesn't mention the NO lOCK clause, but the
compiler seems to handle it the same as the ILE compiler. I've looked at
several versions of the documentation on IBM's AS/400 online library but all
versions I checked (v3r7 - v4r3) point to the same  COBOL/400 reference.

Can anyone tell me if there's a more up to date  COBOL/400 reference available
somewhere, or if the NO LOCK clause is an undocumented feature?
