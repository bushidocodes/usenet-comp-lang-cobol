This is another of the comments that I am posting in order to stimulate
discussion on the new standard and also to get feedback before making my
submission.

I applaud the addition of object oriented capabilities to COBOL. Since I
have yet to understand object oriented programming I can not comment on
the specifics. However I am assuming that the interface definition in
the language is such that COBOL can invoke objects written in other
languages that conform to recognized standards and that objects written
in COBOL can be invoked from code written in other languages. If this is
not the case then the effort has been counter-productive. The overhead
involved in object orientation has to be balanced by being able to have
access to services written in other languages and all of the things that
the proponents of OO talk about.

I don't believe that any OO language has an aim of utilising
objects written in other languages. ("Other languages ? What
other langues?"). If fact it is difficult to reuse objects
in a different version of the same language. C++ achieves
its 'type safety' by doing name mangling. Unfortunately
most mangling schemes seem incompatible so one cannot even
use calls to a .DLL, let alone static linking.

However, I don't see that the only benefit of OO is to be able
to access objects in other languages. I also don't believe
that that there is necessarily an overhead (presumably at runtime)
to using OO as against doing the same thing by non-OO methods.

While certain OO implementations have a small overhead compared
to the non-OO this is being eliminated by better optimisers.

One of the primary design goals of OO COBOL was to allow use of
objects written in other languages. Hopefully, we achieved that. It
does require that the implementor provide it because COBOL can't
enforce it. Also, work on CORBA and other object request brokers is
trying to achieve this. Unfortunately, MS is trying to shoot down
CORBA for their own scheme and while CORBA takes COBOL into
consideration the MS one does not.

Other langauges, mainly C++, could care less about objects written in
languages other than their own. As far as they are concerned, there
is only one language and that is their own (this is the view of the
C++ types mainly - Ada is not like that and has considered COBOL).

back about the Emperor's new clothes. A bit of a putdown of OO. I
