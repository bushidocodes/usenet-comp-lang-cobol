Which IBM compiler are you using?  I just built a small test program
in COBOL for MVS & VM, version 1.2.2, with no periods at end of
statements.  I put an "EXIT." statement at the end of every paragraph,
and the compiler didn't even generate a single I-level diagnostic. 
The program executes correctly too.

Every IBM mainframe COBOL compiler I have ever used would allow EXIT
as the last statement of a multi-statement paragraph.  That includes
DOS/VS FCOBOL, OS/VS COBOL, VS COBOL II, and I expect it will work for
COBOL for OS/390 & VM when we install it this fall.

The first time I ever saw this curious use of "EXIT." was with DOS/VS
FCOBOL around 1981.  When we first started writing structured programs
in that shop we frequently had a "READ ... AT END..." sentence at the
end of a paragraph.  The compiler would generate a really ugly W-LEVEL
message that read something like "EXIT FROM PERFORMED PROCEDURE
ASSUMED BEFORE PROCEDURE-NAME".  We eventually figured out that
putting any imperative statement after the READ would eliminate the
diagnostic, such as MOVE SPACE TO DUMMY-FIELD.  One wise guy just
figured out he could put EXIT at the end of any paragraph, no matter
how many statements it had, and it would eliminate the diagnostic with
no bad side-effects.

I think IBM documents that they allow EXIT as the LAST statement of a
paragraph (as an extension) in the COBOL for MVS & VM manual.
