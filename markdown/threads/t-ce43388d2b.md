```
All
On the subject of COBOL vs Java capabilities:

1.  I started writing mainframe assembler 1970-1980.
2. I bought a TRS-80 Model II in 1980 and developed PC/370 in Z80
assembler and then ported it to IBM PC with Intel 8086 in 1984.
3.  In 1988 I developed mainframe assembler in Intel MASM Assembler
and licensed it to Pansophic for TELON and Micro Focus.
4.  In 1995 I joined Micro Focus and rewrite mainframe assembler in
COBOL for Mainframe Express.
5.  In 2004 I left Micro Focus and rewrote mainframe assembler and
emulator in open source Java named z390 (www.z390.org) that runs on
Windows and Linus.
7  In 2008 I started developing mainframe COBOL compiler called zcobol
(www.zcobol.org) written in open source z390 structured conditional
macro assembler.  I'm currently working on getting the NIST COBOL 1985
test suite running using zcobol.

The z390 and zcobol HLASM assembler and COBOL tools are build on J2SE
Java and can run on any java platform.  The generated HLASM compatible
souce code for both z390 and zcobol can also be run on native z9/z10
mainframe using the latest machine instructions including HFP, BFP,
and DFP.

After the initial 6 months painfully leanring all the java class and
methods used for 90% of the code, I have become a java fan in addition
to my life long assembler bias.  But the fact that there is still tons
of COBOL running the world's busness transactions  has lead me back to
COBOL learning every COBOL verb and phrase of every dialect.

Java is not an operating system.  It is a very power portable language
that can be used to do everything that COBOL and assembler can do
(except for the priviledged OS stuff which is better left to the OS
anyway).

Don Higgins
don@higgins.net
```

```
with COBOL. Reasons : (1) I'm familiar and very comfortable with it (2)

was say 60-ish, I rebelled at the thought of having to switch from the 
familiar to the cryptic language du jour. Surprise, surprise We are now 
being told it's all about C# - so start over again boys.

Now am I incapable of learning 'cryptic'; No, I could do it with effort, 
I've done it before in my youth in the RAF, with less effort than it 
would be now. You want cryptic - take a gander at the following :-

http://en.wikipedia.org/wiki/Pitman_Shorthand

Now figure out how you write the phrase 'Happy Birthday". It's a 
combination of the table of consonants, and as you will see lower down 
the vowel-sounds, a,e,i,o,u. Where you put the vowel 'dot' against the 
consonant 'squiggle',  top, bottom, left, right - that's what gives you 
the 'sound' result.

I hated shorthand - dropped out of it at first opportunity ! (And
```
