If you want to create a shortcut then the simplest approach would be to use
the Shell object from Net Express.

Code required is below. It will add a Shortcut to the Micro Focus site your
StartMenu. This works on 3.1 and 4.0 but I have not ran it on 3.0.

Code is supplied as-is.

Regards
David.

Code >>>>>>>>>>>>>>>>>>>>>

      *> This is required for OLE applications
      $set ooctrl(+P)
       program-id. wshshortcut.
       special-names.
           call-convention 66 is winapi.  *> Non Litlinked apis

       class-control.
       *> OLE automation classes
           wshshell is class "$OLE$WScript.Shell"
           .

       working-storage section.
       01  theShell            object reference.
       01  thedesktop          object reference.
      *01  thestartmenu        object reference.
       01  specialfolders      object reference.
       01  theshortcut         object reference.
       01  thelinkpath         pic x(300).
       01  anint               pic 9(9) comp-5.

       01  thestartmenu        pic x(300).
       01  procptr             usage is procedure-pointer.

       copy "wintypes.cpy".

       78 CSIDL-DESKTOP                  value h"0000".
       78 CSIDL-INTERNET                 value h"0001".
       78 CSIDL-PROGRAMS                 value h"0002".
       78 CSIDL-CONTROLS                 value h"0003".
       78 CSIDL-PRINTERS                 value h"0004".
       78 CSIDL-PERSONAL                 value h"0005".
       78 CSIDL-FAVORITES                value h"0006".
       78 CSIDL-STARTUP                  value h"0007".
       78 CSIDL-RECENT                   value h"0008".
       78 CSIDL-SENDTO                   value h"0009".
       78 CSIDL-BITBUCKET                value h"000a".
       78 CSIDL-STARTMENU                value h"000b".
       78 CSIDL-DESKTOPDIRECTORY         value h"0010".
       78 CSIDL-DRIVES                   value h"0011".
       78 CSIDL-NETWORK                  value h"0012".
       78 CSIDL-NETHOOD                  value h"0013".
       78 CSIDL-FONTS                    value h"0014".
       78 CSIDL-TEMPLATES                value h"0015".
       78 CSIDL-COMMON-STARTMENU         value h"0016".
       78 CSIDL-COMMON-PROGRAMS          value h"0017".
       78 CSIDL-COMMON-STARTUP           value h"0018".
       78 CSIDL-COMMON-DESKTOPDIRECTORY  value h"0019".
       78 CSIDL-APPDATA                  value h"001a".
       78 CSIDL-PRINTHOOD                value h"001b".
       78 CSIDL-COMMON-FAVORITES         value h"001f".
       78 CSIDL-INTERNET-CACHE           value h"0020".
       78 CSIDL-COOKIES                  value h"0021".
       78 CSIDL-HISTORY                  value h"0022".

       01 retval                          BOOL.

       procedure division.
       start-it section.

           invoke wshshell "new" returning theShell

           set procptr to entry "shell32.dll"
      *
      *    Use CSIDL-COMMON-STARTMENU to get the ALL Users startmenu
      *
           call winapi "SHGetSpecialFolderPathA"
               using by value     0
                     by reference thestartmenu
                     by value     CSIDL-COMMON-STARTMENU
                     by value     0
               returning retval
           string thestartmenu                 delimited by x"00"
                  z"\Micro Focus Web Site.lnk" delimited by size
               into thelinkpath
           end-string

           invoke theshell "CreateShortCut" using thelinkpath
               returning theshortcut

           invoke theshortcut "setDescription" using
                                       z"Micro Focus Web Site"
           invoke theshortcut "setTargetPath" using
                        z"http://www.microfocus.com"

           invoke theshortcut "save"

           stop run.

To use IShellLink directly using Net Express you need to have a
understanding of how COM works at the interface level.

Basically COM presents an Interface as a VTABLE which is an array of
procedure pointers in COBOL which correspond to the Methods in the
Interface.

You have to be very careful that you get the ordering of the functions
correct or your likely to get a crash.

The other key point is that when you get back a Interface it is actually a
pointer to the pointer to the VTABLE (2 levels of indirection). This should
explain what:-

           set address of vtabptr to IPersistFile
           set address of lnkvtab to vtabptr

is doing.

The code itself just adds a shortcut to the startmenu.

As you can see the code for WSH Objects is far simpler as that is exposed as
an automation object (hence you can use Invoke syntax on it).

Code is provided as-is.

If anyone wants a working Net Express project for this then just send me an
email.

Regards
David.

Code>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>
      *> This is required for OLE applications
      $set ooctrl(+P)
       copy "windows".
       program-id. hshortcut.
       special-names.
           call-convention 74 is winapi.

       class-control.
       *> OLE automation classes
           dummy  is class "$OLE$dummy"
           olesup is class "olesup"
           .

       working-storage section.
       01  theShell            object reference.
       01  thedesktop          object reference.
      *01  thestartmenu        object reference.
       01  specialfolders      object reference.
       01  theshortcut         object reference.
       01  thelinkpath         pic x(300).
       01  unipath             pic X(500).
       01  anint               pic 9(9) comp-5.

       01  thestartmenu        pic x(300).
       01  procptr             usage is procedure-pointer.

      *copy "wintypes.cpy".
       copy "mfole".
       78 CSIDL-DESKTOP                  value h"0000".
       78 CSIDL-INTERNET                 value h"0001".
       78 CSIDL-PROGRAMS                 value h"0002".
       78 CSIDL-CONTROLS                 value h"0003".
       78 CSIDL-PRINTERS                 value h"0004".
       78 CSIDL-PERSONAL                 value h"0005".
       78 CSIDL-FAVORITES                value h"0006".
       78 CSIDL-STARTUP                  value h"0007".
       78 CSIDL-RECENT                   value h"0008".
       78 CSIDL-SENDTO                   value h"0009".
       78 CSIDL-BITBUCKET                value h"000a".
       78 CSIDL-STARTMENU                value h"000b".
       78 CSIDL-DESKTOPDIRECTORY         value h"0010".
       78 CSIDL-DRIVES                   value h"0011".
       78 CSIDL-NETWORK                  value h"0012".
       78 CSIDL-NETHOOD                  value h"0013".
       78 CSIDL-FONTS                    value h"0014".
       78 CSIDL-TEMPLATES                value h"0015".
       78 CSIDL-COMMON-STARTMENU         value h"0016".
       78 CSIDL-COMMON-PROGRAMS          value h"0017".
       78 CSIDL-COMMON-STARTUP           value h"0018".
       78 CSIDL-COMMON-DESKTOPDIRECTORY  value h"0019".
       78 CSIDL-APPDATA                  value h"001a".
       78 CSIDL-PRINTHOOD                value h"001b".
       78 CSIDL-COMMON-FAVORITES         value h"001f".
       78 CSIDL-INTERNET-CACHE           value h"0020".
       78 CSIDL-COOKIES                  value h"0021".
       78 CSIDL-HISTORY                  value h"0022".

       01 retval                          BOOL.

       78  IID-IConnectionPointContainer
                      VALUE x"84B296B1B4BA1A10B69C00AA00341D07".
       78  CLSID-ShellLink
                           VALUE x"0114020000000000C000000000000046".
       78  IID-IShellLink
                           VALUE x"EE14020000000000C000000000000046".
       78  IID-IPersistFile
                           VALUE x"0b01000000000000C000000000000046".
       01  IShellLink         pointer.
       01  IPersistFile       pointer.

       78  CLSCTX-INPROC-SERVER    value h"00000001". *> 0x1

       01  hres        DWORD.
       01  ws-success  PIC 9(9) COMP-5.

       linkage section.
       01  vtabptr                     pointer.
       01  lnkvtab.
           03  fptr        procedure-pointer occurs 100.
       01 lnk-IShellLink redefines lnkvtab.
           03 IUnknown-QueryInterface        usage procedure-pointer.
           03 IUnknown-Addref                usage procedure-pointer.
           03 IUnknown-Release               usage procedure-pointer.
           03 IShellLink-GetPath             usage procedure-pointer.
           03 IShellLink-GetIDList           usage procedure-pointer.
           03 IShellLink-SetIDList           usage procedure-pointer.
           03 IShellLink-GetDescription      usage procedure-pointer.
           03 IShellLink-SetDescription      usage procedure-pointer.
           03 IShellLink-GetWorkingDirectory usage procedure-pointer.
           03 IShellLink-SetWorkingDirectory usage procedure-pointer.
           03 IShellLink-GetArguments        usage procedure-pointer.
           03 IShellLink-SetArguments        usage procedure-pointer.
           03 IShellLink-GetHotkey           usage procedure-pointer.
           03 IShellLink-SetHotkey           usage procedure-pointer.
           03 IShellLink-GetShowCmd          usage procedure-pointer.
           03 IShellLink-SetShowCmd          usage procedure-pointer.
           03 IShellLink-GetIconLocation     usage procedure-pointer.
           03 IShellLink-SetIconLocation     usage procedure-pointer.
           03 IShellLink-SetRelativePath     usage procedure-pointer.
           03 IShellLink-Resolve             usage procedure-pointer.
           03 IShellLink-SetPath             usage procedure-pointer.
       01 lnk-IPersistFile redefines lnkvtab.
           03 IUnknown-QueryInterfaceX       usage procedure-pointer.
           03 IUnknown-AddrefX               usage procedure-pointer.
           03 IUnknown-ReleaseX              usage procedure-pointer.
           03 IPersist-GetClassID            usage procedure-pointer.
           03 IPersistFile-IsDirty           usage procedure-pointer.
           03 IPersistFile-Load              usage procedure-pointer.
           03 IPersistFile-Save              usage procedure-pointer.
           03 IPersistFile-SaveCompleted     usage procedure-pointer.
           03 IPersistFile-GetCurFile        usage procedure-pointer.

       procedure division.
       start-it section.

      *    invoke wshshell "new" returning theShell
           invoke olesup "singleThread" returning ws-success

      *    Use CSIDL-COMMON-STARTMENU to get the ALL Users startmenu
      *
           call winapi "SHGetSpecialFolderPathA"
               using by value     0
                     by reference thestartmenu
                     by value     CSIDL-COMMON-STARTMENU
                     by value     0
               returning retval
           string thestartmenu                 delimited by x"00"
                  z"\Micro Focus Web Site.lnk" delimited by size
               into thelinkpath
           end-string

      ***** Initialize COM Libraries

           call winapi "CoInitialize" using by value 0 size 4

      ***** Create an Instance of IShellLink
           call winapi "CoCreateInstance"
                       using by reference  CLSID-ShellLink
                             by value      0 size 4
                             by value      CLSCTX-INPROC-SERVER
                             by reference  IID-IShellLink
                             by reference  IShellLink
               returning hres
           end-call
           perform check-hres

      ***** Setup the IShellLink VTABLE

           set address of vtabptr to IShellLink
           set address of lnkvtab to vtabptr

           call winapi IShellLink-SetPath
                   using by value IShellLink
                         by reference z"http://www.microfocus.com"
               returning hres
           end-call
           perform check-hres
           call winapi IShellLink-SetDescription
                   using by value IShellLink
                         by reference z"Micro Focus Web Site"
               returning hres
           end-call
           perform check-hres

      ***** Get IPersist file

           call winapi IUnknown-QueryInterface
               using by value IShellLink
                     by reference IID-IPersistFile
                     by reference IPersistFile
               returning hres
           perform check-hres

      ***** Convert Path to Unicode
           call winapi "MultiByteToWideChar"
               using by value CP-ACP
                     by value 0  size 4
                     by reference thelinkpath
                     by value -1 size 4
                     by reference unipath
                     by value length of unipath size 4
               returning hres
           end-call

           set address of vtabptr to IPersistFile
           set address of lnkvtab to vtabptr
           call winapi IPersistFile-Save
               using by value IPersistFile
                     by reference unipath
                     by value 1 size 4
               returning hres
           end-call
           perform check-hres


      ***** Cleanup used Interfaces

      ***** Release IPersistFile
           call winapi IUnknown-Release
               using by value IPersistFile
               returning hres
           end-call

      ***** Release IShellLink
           set address of vtabptr to IShellLink
           set address of lnkvtab to vtabptr
           call winapi IUnknown-Release
               using by value IShellLink
               returning hres
           end-call

           call winapi "CoUninitialize"

           stop run.



       check-hres section.

           if hres not = 0
               display "Error : Die Gracefully"
               stop run
           end-if
           .
