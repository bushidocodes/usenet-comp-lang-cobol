and the machine on which the data was created.

It is probably floating-point -- equivalent to double in C; but
may be IEEE, IBM, or other and, if IEEE floating-point, may
be either big-endian or little-endian.

COMP-1 and COMP-2 data are *often* stored as IEEE single and
double-precision floats respectively, but there is no standard method and
the storage is "implementor defined."

As a good example of this, in IBM COBOL for MVS/VM (mainframe) , there is a
system-wide option (I think set through Language Environment) which allows
you to select either FLOAT(IEEE) or FLOAT(NATIVE) as the method of storage
on the system. (You can override this setting on a program by program basis,
but that might upset one or more of your colleagues).
