```
If anyone is writing applications that have IP addresses in the data stream
(maybe processing server logs or drilling into email, for example...), you
might like to have something that shows you the location of a given IP
address?

No GPS is involved; it works off the locaion of the machine that used that
IP address.

The component (at the moment...) takes a passed IP address and provides the
city, region, country, ZIP code, and Latitude and Longitude.

It is designed for interactive viewing of reports or Forms that contain IP
addresses, but it would be very easy to suppress the current pop-up box so
you could get a batch stream out of it.

The component can also interface to Google Maps, so you can get the above
data and also have a Google Map centred on the Latitude and Longitude, and
zoomable/scrollable in the usual way.

It looks really cool in operation, so I am prepared to provide it to people
who are interested, so they can use it with COBOL.

(I wrote it (in C#, as a COM component) as a "spin-off" from some work
connected with the new PRIMA migration of PowerCOBOL to .NET.

http://primacomputing.co.nz/PRIMAMetro/pwrCOBMigration.aspx

Hablas Espanol?: http://www.adamed.es )

Requisites for use from COBOL:

1. A version of COBOL that supports the Component Object Model (COM).
Fujitsu, RM, and Micro Focus COBOL are all COM compliant and can interface
to the component directly...Mainframe is able to access through a Java
"wrapper". (Sorry, GNU COBOL guys... you'll need a C#, C. C++, VB.Net or
Java interface...)

2. The application running it needs internet connection for it to work. (It
won't stop any other functionality in your application, but if there is no
Internet connection available, it cannot get to the web services it needs,
so it returns nothing. (It does not abort the process.)

Mail me or respond here if this might be of use to you, or you have any
questions.

Pete.
"I used to write COBOL...now I can do anything."
```

GeoIP is a local database that can be installed (with yum) and updated (with geoipupdate). geoiplookup is a command. There is a C API to the database that can be used by GNU Cobol.

I use Python these days and an example of that is:
-------------------------
#!/bin/env python

import pygeoip

geolocator = pygeoip.GeoIP('/usr/share/GeoIP/GeoIPCity.dat')
location_data = geolocator.record_by_addr('64.233.161.99')
print location_data

--------------------------

Running this produces:

$ ./testgeoip.py
{'city': u'Mountain View', 'region_name': u'CA', 'area_code': 650, 'time_zone': 'America/Los_Angeles', 'dma_code': 807, 'metro_code': 'San Francisco, CA', 'country_code3': 'USA', 'latitude': 37.419199999999989, 'postal_code': u'94043', 'longitude': -122.0574, 'country_code': 'US', 'country_name': 'United States', 'continent': 'NA'}

location_data is a dictionary that can be accessed as eg: location_data['longitude']

If a map is required then there are several web sites that can be used, such as:

http://www.geoipview.com/?q=72.22.33.1&x=0&y=0

these could be started from COBOL using something like:

CALL "SYSTEM" USING "firefox http://www.geoipview.com/?q=72.22.33.1&x=0&y=0" & x"00"

Accuracy is not particularly good. My IP shows in one as K'Road (where my ISP is) and in another as the middle of Tairua harbour.

using IPAddress
end-invoke
*> Load the WS with the component's properties, which are set

returning Region

end-invoke

returning ZIPCode (02142)
end-invoke
invoke objIPLoc "GET-Latitude"
returning Latitude (42.3626)
end-invoke
invoke objIPLoc "GET-Longitude"
returning Longitude (-71.0843)
end-invoke
...

Obviously, once you have the values you can display them or use them any way
you want. In particular, it is easy to pass the Latitude and Longitude to a
web service like Google Maps API, by shelling to an instance of your Browser
(as Richard showed) or using the "FeedMap" method of the component, which
will open a Browser instance showing the Google Map with all the usual
controls for zoom and scroll, centred on the passed geo-location:

invoke objIPLoc "FeedMap"
using Latitude
Longitude
end-invoke

Although the above is COBOL-friendly, it is much less verbose in a different
language.

IPLocator objIPLoc = new IPlocator(); //requires

objIPLoc.IPLocation("23.66.166.151"); // can be hard
