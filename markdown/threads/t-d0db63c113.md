How very interesting... learn something new every day! The
situation,
if we recall, went something like this:

› i have the following: (and this is REALLY what i would like to do)
› 
…[15 more quoted lines elided]…
› 
Someone suggested that subscripted 88s do not work (not my experience
but
hey, my experience is limited) so an alternative structure was
suggested,
viz.,

01 TEST-AREA.
05 TST-TOP-EAC PIC X.
88 TST-I-A VALUE '1'.
88 TST-I-B VALUE '2'.
88 TST-I-C VALUE '3'.
88 TST-I-D VALUE '4'.
88 TST-I-E VALUE '5'.
88 TST-II VALUE '6'.
05 TST-EAC-SEQUENCE PIC X.
88 TST-FIRST VALUE '1'.

... and a PERFORM VARYING to check this. Another suggestion was to SET
ADDRESS; I objected to this based on the Two-Year Programmer rule. The
rebuttal, of course, was one based on speed; I counter-rebutted that
since the MOVE was PIC X to PIC
X the generated instruction would be MVI, not MVC (speed difference
approx 1:3). Mr Lynch then stated that the MOVE in such a structure
would be an MVC; I professed to having insufficient data and said I
would research it.

In short, Mr Lynch is correct... and even More Learning resulted. After
changing the last 88 from 'FIRST' to 'EAC-FIRST' I generated the
following, exquisitely dull
code (SUB1) is defined as pic 9(4) COMP):

PROCEDURE DIVISION.
*
THE-PERFORM.
PERFORM MOVE-IT THRU MI-EX
VARYING SUB1 FROM 1 BY 1
UNTIL SUB1 > 20.
GOBACK.
MOVE-IT.
MOVE TOP-EAC-LINE-NO (SUB1) TO TST-TOP-EAC.
IF TST-I-A
DISPLAY ' TST-I-A'.
IF TST-FIRST
DISPLAY 'TST-FIRST'.
MI-EX.
EXIT.

Using IKFCBL00 with LIB,FDUMP,XREF,VBREF,RES,PMAP (PMAP/LIST gives the
Temple of Truth referred to in the subject line) I got:

*MOVE-IT
0003C6 PN=03 EQU *
MOVE 0003C6 41 40 6 018 LA 4,018(0,6)
0003CA 48 30 6 008 LH 3,008(0,6)
0003CE 5C 20 C 034 M 2,034(0,12)
0003D2 1A 43 AR 4,3
0003D4 5B 40 C 034 S 4,034(0,12)
0003D8 50 40 D 210 ST 4,210(0,13)
0003DC 58 E0 D 210 L 14,210(0,13)
0003E0 D2 00 6 040 E 000 MVC 040(1,6),000(14)
IF 0003E6 95 F1 6 040 CLI 040(6),X'F1'
0003EA 58 B0 C 04C L 11,04C(0,12)
0003EE 47 70 B 0AE BC 7,0AE(0,11)
DISPLAY 0003F2 58 F0 C 00C L 15,00C(0,12)

*clearly* showing an MVC, not the MVI I predicted. (Note that the IF is
still a CLI).

Using IGYCRCTL with LIB,FDUMP,XREF,VBREF,RES,LIST I found:

*MOVE-IT
MOVE
1A4 4820 9008 LH 2,8(0,9)
1A8 4C20 A00C MH 2,12(0,10)
1AC 1A29 AR 2,9
1AE D200 9040 2016 MVC 64(1,9),22(2)
IF
1B4 95F1 9040 CLI 64(9),X'F1'
1B8 4770 B082 BC 7,130(0,11)
DISPLAY
1BC 0520 BALR 2,0

... again, showing Mr Lynch's MVC.


Then, just for yucks, I did it *this* way:

THE-PERFORM.
PERFORM MOVE-IT THRU MI-EX
VARYING SUB1 FROM 1 BY 1
UNTIL SUB1 > 20.
GOBACK.
MOVE-IT.
IF PART-I-A (SUB1)
DISPLAY ' PART-I-A'.
IF EAC-FIRST (SUB1)
DISPLAY ' EAC-FIRST'.
MI-EX.
EXIT.

... with subscripted 88s. IKFCBL00 gave:

*MOVE-IT
0003C6 PN=03 EQU
IF 0003C6 41 40 6 018 LA
0003CA 48 30 6 008 LH
0003CE 5C 20 C 034 M
0003D2 1A 43 AR
0003D4 5B 40 C 034 S
0003D8 50 40 D 210 ST
0003DC 58 E0 D 210 L
0003E0 95 F1 E 000 CLI
0003E4 58 B0 C 04C L
0003E8 47 70 B 0A8 BC
DISPLAY 0003EC 58 F0 C 00C L

Notice the work needed to calculate the displacement and compare the
first MOVE's
generated statement with this second IF's... look familiar? (LA, LH, M,
AR, S, ST, L...
use your yellow-card to translate these if necessary) *Everything*
except
the actual MVC is there.

IGYCRCTL generated

*MOVE-IT
IF
1A4 4820 9008 LH 2,8(0,9)
1A8 4C20 A00C MH 2,12(0,10)
1AC 1A29 AR 2,9
1AE 95F1 2016 CLI 22(2),X'F1'
1B2 4770 B07C BC 7,124(0,11)
DISPLAY

... *much* more compact, a Load Halfword, Move Halfword, Add Register,
Compare Logical Immediate and a Branch Conditional (sorry, GO TO-less
lovers!) and there
she be.

What does this all *mean*, outside of proving me wrong and Mr Lynch
right? Well,
meaning is the result of interpretation, and interpretation is in the
mind of the
beholder... so, behold.

DD
