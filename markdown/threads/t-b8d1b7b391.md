What Carlos means, I think, is that when calling a PowerCOBOL program from
another PowerCOBOL program, you must use the following statement:

invoke pow-self "CallForm" using "Formname" ["Dllname"] [CloseChildID]

pow-self: object reference to the active screen/form
Callform: method of the object, can also be OpenForm or ShowForm
Formname: the name of the form that resides in the other file [required]
DLLname: the name of the dll file that contains the other form(s) [optional]
CloseChildID: an integer which can be checked on in the CloseChild event of
the calling program to handle the closing of called forms. [optional]

You *cannot* supply linkage reference when opening a form from powercobol:
you should use the global external clause to share the data area between
PowerCOBOL forms.

When calling programs using non-GUI a.k.a. COBOL97, you can use linkage and
it should work like any other compiler.

Rik Hendriks

There is a difference between dynamic and static linking.

RM is defaulting to dynamic loading of modules.  This means that at
compile/link time it neither knows nor cares whether the called
program exists or not.  At run time when the CALL is executed it
searches in the specified places for a file of the appropriate name
and loads it into memory for executing.

Fujitsu defaults to 'Static' or 'Dynamic Link structure'. When linked
there must be a file of the form 'progname.lib' which is included in
the resulting execuatable.  This can be the complete program or will
normally be a stub which will link to a .DLL.  The .LIB is produced by
the compile/link of that module.

This means that all the called routine must be produced first from the
lowest level so that the links of the calling programs will work, the
main program being linked last.

You can make Fujitsu use dynamic loading (called 'dynamic program
structure') and then not have to bother linking in the .LIB files. 
Use the DLOAD compiler directive or simply CALL using variables
instead of literals:

        MOVE "PROGNAME"      TO This-Name
        CALL This-Name USING Linkage-Items
        CANCEL This-Name
