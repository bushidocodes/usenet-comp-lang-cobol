Here's the deal. I've used reference modification for some time now.
But I've hit some differences between how Micro Focus COBOL and
CA-Realia COBOL handle it. I much prefer the CA-Realia method, but I
need to know which is correct, for future portability.

In Micro Focus you cannot do direct compares on reference modified
data items:

ie: IF DATA-FIELD (3:1) = "*"
..... is invalid, and disallowed by the compiler, while it
works just fine in CA-Realia.

In addition you cannot perform any EVALUATE or other operations on
reference modified items in Micro Focus COBOL.

Now... there IS something that you CAN do in MF that you can't do with
CA-Realia:

INSPECT DATA-FIELD TALLYING CNTR FOR ALL TEST-FIELD (1:3).

MF allows this, CA does not.

Which of these 2 is correct/accepted by other compiler vendors?
