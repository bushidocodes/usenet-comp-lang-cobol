```
Does anyone know of a way to get the DB2 pre-compiler to correctly
interpret the COPY REPLACING command so that fields in this copybook
could be used as host variables in a DB2 SQL statement?

COPY ALPHA REPLACING ==:TAG:== BY ==WS-==.
=============================================
01  ALPHA-GROUP.
    05  :TAG:TIMESTAMP-FIELD       PIC X(26).
    05  :TAG:TEXT-FIELD            PIC X(40).
=============================================

EXEC SQL
    SELECT TIMESTAMP_FIELD, TEXT_FIELD
      INTO :WS-TIMESTAMP-FIELD, :WS-TEXT-FIELD
      FROM TABLE
END-EXEC

The ALPHA copybook doesn't get pulled in with the DB2 pre-compile, but
this would be fine if the copybook fields were not used as host
variables.

Any ideas?

Thanks!!!
```

```
My *guess* is that this does NOT work - UNLESS - you are using the most
recent IBM COBOL compiler, i.e. IBM Enterprise COBOL for z/OS & OS/390.

In this version (or with IBM COBOL for OS/390 & VM V2R2 - with the
appropriate PTFs) you can use an "integrated" DB2 precompiler that should
handle this correctly.
```
