Well, Here is my two cents worth.

In my opinion COBOL is far from dead. It will be around for a long, long,
time. Much longer than "Mainframes".

We have about 500 programmers in our shop, 450 work in COBOL, some work in
RPG and others in more exotic things like smalltalk, Visual Basic, or C. We
hire mostly university grads, computer science or business majors. We
(re-)teach them COBOL before letting them loose on our applications.

In my opinion there are more important things to teach students than COBOL
(in no particular order).
1) Think clearly,
2) Present ideas clearly,
3) Adapt to new situations rapidly (ie. new technology),
4) Keep a cool head when under pressure,
5) Perform quick and rapid problem determination.

COBOL may be a useful tool for teaching these things, maybe not. Whatever
you choose, it should be a language that most of your students don't
already know (C, Basic, etc.)

Regarding the 'balance-line' algorithm. In the olden days it was very
useful for the updating of sequential master files from transaction files.
But in the olden days, that was all there was. The balance-line algorithm
is still quite popular, It is the MERGE of SORT/MERGE operations and a
implementation the the relational JOIN operator. It is a very important
tool in some database update applications where the old balance-line
algorithm can improve efficiency considerably.

By the way, we run COBOL applications on many different platforms. These
include: MVS/ESA, AIX, OS/2, AS/400, Windows, and Windows NT. One thing I
have found. It is easier to write platform independent code in COBOL than
it is in C. And easier to write platform dependent but portable code in C
than COBOL.
