>I recently acquired a job working with cobol on a as/400 system, I

Which model and which version of the system? I have been using a S/38 and
will be converting to an AS/400 model 400 (the RISC version) in the next
few months. The COBOL is fairly standard except that you use display files
to display information and to accept input. This is one of the tricky
things. The other is the whole area of logical views with omit and select
logic and keys. If you have specific questions I might be able to help
you. You should have a full set of manuals around the shop. They are
fairly good. If you have a later version see if there are copies of the
manuals online-- InfoSeeker is the reader and keeps track of the books.
You might also see if someone has ordered a "Red Book" on Cobol. They are
supposed to be excellent.

--Ray B.
