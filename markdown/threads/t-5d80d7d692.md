"LISTCAT ... ALL"  does not give RECFM, LRECL, or BLKSIZE for VSAM data
sets because these attributes do not apply to VSAM datasets.  Records in
VSAM datasets are organized into Control Intervals (CI's) and Control
Areas (CA's).  The rules governing record placement into the external
file are much more complex than for sequential files and the user is
more insulated from the internal structure of the data set than with
sequential files.  All records in a [KSDS or ESDS] VSAM file may be any
length up to the max record length (as long as KSDS records are at least
long enough to contain the key field).  There is also an "average"
record length defined for a VSAM file, but other than usage when
allocating space by records, it is basically just "best guess"
documentation and is neither calculated nor enforced (i.e., making
average and max recsize the same does not "make" the VSAM records fixed
length, it only indicates that perhaps the definer expects all the
variable length records to have the same actual length, but any
enforcement is done by the updating programs, not as a result of the
VSAM file definition).
