Olli,

There is an easy way and a hard way to get where you want to go:

1. If you have the DG REORG utility you can dump the DG ISAM to a fixed length
sequential file and use that to build your MF ISAM.

or

2. You can write a program to access the .xd portion of the DG ISAM as a fixed
length sequential. In your program's FD for this file add a leading 6 bytes to account
for DG's record pointers. This is trickier because deleted records may be present
in the .xd file. (I'm not sure.) If so the record pointers are probably some special
value for the deleted records.
