Hey,
Nothing is wrong.

You have learned that a proportional font is not the way to show printed
output.

If you use Word as 'viewer, then change the font in Word. Select everything
and change the font to courier, Terminal or Lineprinter. They are all
bitmapped - all letters including space are exactly same size.

You might like to change the filename to 'Printer.Txt' - the .DOC might
confuse Word, and check that you define the file as line sequential. Some
compilers will default to line sequential when you assign to printer, but
you changed that.

After this you should be able to see your output in Word.

I like Notepad as a viewer. You can turn off word-wrap and choose 'fixed
sys' as font, but keep the extension .TXT on a txt-file.

Hope this helps.

Eckhard:

I just created a FormPrint form in landscape mode with a data field
formatted to X(205), using a 7 point regular weight Courier New font.

I used a Courier New font because in most cases where a single string
of data of that length needs to be printed, it is comprised of many
fields concatenated to form one string.  Printing with a single data
field and a proportionally spaced font would not allow columns of
logical data items to line up properly in a vertical manner.

I set the page up as US Letter size which is 8.5" x 11" (269 mm width
by 207 mm height).  This is close to an A4 format (297 mm width by 210
mm height in landscape mode).

A4 landscape format should actually allow you to print an even longer
text string because the paper is wider than US Letter format.

I initialized it with 205 characters and it printed fine in Windows 98
as well as Windows NT on an HP Laser Jet 4 plus.  Please let me know
if you would like to obtain a copy of the print and I'll be happy to
send it to you.

Please bear in mind that regardless of which tool you use for your
printed output, the specific printing device(s) which will be used is
a very important consideration.  All printing devices which support
the Windows Print Manager are subject to a maximum printable area as
defined by the manufacturer's printer driver.

Ink jet printers almost always have a smaller printable area (larger
minimum margins), therefore it is impossible to fit as many characters
on a printed page rendered by a typical ink jet than one printed on a
typical laser printer.

If you control the type of printers which will be used, a 205
character line is possible, providing that the printable area is large
enough and the font is small enough.  If you don't control the
printing devices which will be used, it will be much more difficult to
fit 205 characters on certain printing devices.  It is possible to use
a smaller point size, but smaller fonts may be too small to be easily
read.  I also printed with a 5 point font and it appears that you can
print an additional 47 characters (252 total) on the page.  It is
readable, but quite small for my aging eyeballs!

This same output could also be printed from a COBOL program running on
a remote server across a TCP/IP network (internet/intranet) using the
Thin Client in conjunction with FormPrint.  This can be accomplished
in a stand-alone Thin Client program or directly from a web browser
with the Thin Client browser plug-in.  It provides much more flexible
output than an HTML based printed page.

I hope that this was helpful.  Send private e-mail if we can be of
assistance in your project.
