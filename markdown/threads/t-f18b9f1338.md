How about redefining the input record? IE:

FD master-file
RECORD CONTAINS ......
LABEL RECORDS ARE......

01 MASTER-RECORD.
05 WO-LOAN-NUMBER PIC X(5).
05 WO-BALANCE PIC 9(8).
ETC.

01 MASTER-REC-REDEF.
05 WW-LOAN-NUMBER PIC X(5).
05 WW-BALANCE PIC 9(6)V99.
ETC.

Use the first for your indexing, and the redefines for your
calculations. The redefines clause allows you to redefine part of
storage, in this case, the input record. Both wo-balance and
ww-balance will point to the same storage location, but they will
interpret the info found there differently (ie. one with the internal
decimal, the other without). Because it is within the file section,
you do not have to specify the redefines clause, as it will be
assumed. I suggest that you look in your book to fully understand
REDEFINES.
