Hello,

I'm a Belgian student and I'm having a problem with SQL in cobol. At
school we use a simple MS Access database. An example of a table
"Student" in the database could be:

Student
-------
id       Automatic Numeration
name     Text
adress   Text
age      Numeric

The problem is that I can only read fields with the type "Text".
I use this code:
*******************************************************
$SET SQL(dbman=ODBC, pass=admin, init, db=student)
...
Working-storage section.
EXEC SQL INCLUDE SQLCA END-EXEC
EXEC SQL BEGIN DECLARE SECTION END-EXEC.
01  sstudentid  pic x(3).
01  sname       pic x(15).
01  sadress     pic x(70).
01  sage        pic x(3).
EXEC SQL END DECLARE SECTION END-EXEC.
...
Procedure division.
...
student-lijst.
  EXEC SQL
    DECLARE stud-curs CURSOR FOR
    select id, name, adress, age from student
  END-EXEC

  EXEC SQL OPEN stud-curs END-EXEC
  if sqlcode not = 0
    display sqlcode
    display sqlerrmc
  else
    perform until sqlcode = 100
      EXEC SQL
        FETCH stud-curs
        INTO :sstudentid, :name, :adress, :age
      END-EXEC

      display sstudentid " " sname " " sadress " " sage
    end-perform
  end-if
  EXEC SQL CLOSE stud-curs END-EXEC
*********************************************************

The error I get is this:
------------------------
114 Attempt to access item beyond bounds of memory (Signal 11)
No source available for _SQLODBC

If I only read the fields "name" and "adres", then there is no
problem.

Is there anyone who tell me if, and how it is possible to read fields
with a type other than "Text"

Kind Regards,

Dieter Moeyersons

Dieter,

 From your "$set SQL" directive - it looks to me like you are using Net 
Express - University Edition ?

1. If you go to microfocus.com and check out "Academic and University 
Edition" there is a forum where you can post questions relating to your 
version of Net Express.

2. Again if it is Net Express UE, there's even self-help you can engage 
in. Firstly, outside of COBOL, set up an xxxx.mdb in MS Access with the 
Table you want to include BOTH Text and Numeric fields - just 
experiment. Having set up your MS Access now go to the Net Express IDE 
and from Main Menu Bar-----> Tools---->OpenESQL.

This will bring up a list of DBs registered with the ODBC - it looks a 
little complicated - but is quite comprehensive when you get the hang of 
it. (Of course those DBs appearing have firstly been registered with the 
ODBC Driver through the Windows Control Panel).

Subject to User Access rights, you can now click on the particular mdb 
and the Treeview will expand to show each of your tables. You can now 
select a particular Table and you will be prompted for the type of SQL 
query. It will list the text for SQL Statements which you can now 
COPY/PASTE into a COBOL source program. When using statements with 
cursors, it automatically generates the next Cursor Number.

Select "Auxiliary Code"--->Host Variable Declaratives and it will 
generate a copyfile in three sections containing field definitions 
including NULLs.

Subject to the type of query, your can RUN the query from within the 
ESQL Assistant without actually writing any COBOL source program.

A little patience required, but I think you will find this approach 
absolutely bulletproof to generate your SQL entries.

Bon chance,

Jimmy, Calgary AB
