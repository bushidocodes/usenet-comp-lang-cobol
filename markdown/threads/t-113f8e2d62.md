Easy - but not very friendly answer.

You haven't told us what compiler (and/or operating system) you
are asking about. The object code (and hence the performance)
for nested IF's versus EVAULATE are entirely compiler-dependent.
What may run better on one machine/one compiler will be less
efficient on another. Furthermore, even if you use the same
compiler, it may have "enhanced" performance at one maintenance
level over another.

My personal recommendation is that EVALUATE is "easier" and
cleaner to maintain - but I am certain that will depend on your
specific application and programming style.

My educated guess would be that "IF" would generate
more efficient code than "EVALUATE" for most compilers.
An IF statement generally gives you more control of the
order of the comparisons and, therefore, is somewhat less
dependent on the "intelligence" of the comiler.

Different versions of different compilers for different platforms
will probably give you different results. Perhaps your particular
compiler will optimize code better with an EVALUATE statement.

The only way to get the true answer is to do it both ways and
count the number of machine instructions generated. Take into
consideration the number of instructions actually executed for
each condition.

I use EVALUATE, and I use IFs. One thing I have noticed with an EVALUATE
is that it is much more difficult to do something stupid with it. An IF
can be nested in many different ways (you all know what I mean, so don't
try to deny it), whereas the EVALUATE is equivalent to a disciplined
IF...ELSE IF...ELSE IF...ELSE. I try to keep my IFs to that format, or
non-nested, in which case either is equally readable. However, I keep
running across those god-awful 12 page 15 level nested IFs
(IF...IF...IF...ELSE IF...IF...ELSE ELSE...etc.) which are nearly
impossible to really understand, especially when they have a few EXEC
CICSs in them, and might branch out on an exception condition.

Therefore, IMNSHO smart coders can probably use either IF or EVALUATE and
write good code, but the EVALUATE is harder to screw up for a sloppy
coder. If that is all it does, then it is worth it.
