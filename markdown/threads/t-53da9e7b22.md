Anyway, I'll try to get you started.

Think of an array or table as a list. Like a list of countries. Each line of
the list is called an entry. Each entry is numbered: the first entry is number
one, the second is number two, and so on. In COBOL you must specify how big the
list can be, perhaps like this:

01 Country-list.
05 Country occurs 30 times pic x(25).

To reference an item in the list, you must tell COBOL which number it is in the
list. You use an occurrence number. For example:

if country(5) = 'France' then ...


this asks if the 5th country in the list is 'France'. Instead of hard coding
the occurrence number you can use a variable, so that you can change which
country you are asking about. For example:

if country(c-num) = 'France' then ...

the variable, c-num in the example, must be a numeric data item (preferrably
integer). We call 'c-num' a subscript. You can ask about each country in your
list if you set up a loop that runs the 'if' statement for all entries in your
list; that is, if you let c-num run from 1 to 30.

There are a number of ways of doing this, one of which is:

move 1 to c-num
perform until c-num > 30
if country(c-num) = 'France' then ...
.
.
.
else
.
.
.
end-if
add 1 to c-num
end-perform


There's lots more, but hope that gets you started.

Good explanation. I would suggest a different way of controlling the
loop, e.g.,:

PERFORM VARYING C-NUM FROM +1 BY +1
UNTIL C-NUM > +30
IF COUNTRY(C-NUM) = 'FRANCE'
...
ELSE
....
END-IF
END-PERFORM

I was taught (and believe it's a good idea) to use the VARYING phrase
instead of incrementing the subscript myself. Over time the compiler
will make fewer errors and the construct is guaranteed to work from one
implementation to the next.

To add to what Steve has given you:

A golden rule is to DRAW THE TABLE on PAPER, whenever you decide on
how it should look and its definition and coding.
Making a drawing like this helps tremendously in understanding the
table, consentrating on your program's logic as you follow the
different processes with the table and it helps your brain to link
your thought processes to something you can see in front of you and
develop as you work with it.

Another advantage of a drawing is that you do not have to have it
absolutely right the first time. Draw it as you think it should look
- then follow all the steps that could happen to that table:
1. initialize it
2. load it
3. accumulate into it
4. use it
