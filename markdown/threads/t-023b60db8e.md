Here's a follow-up report:
I renamed the source to coblib.c, and that seems to have the same effect as
putting 'extern "C"' in the cpp source - it eliminated the name-mangling in
the exported LIB symbol - but then I got the following rather nasty-looking
error when Realia tried to link coblib.lib with the COBOL program:

D:\CICSCBK\Programs\TestCall\CobLib.lib : fatal error LNK1106: invalid file
or disk full: cannot seek to 0x3b656c75


Since the DLL and LIB files looked OK (DUMPBIN had no problem reading them),
and of course the disk was not full, I tried replacing Realia's older
(version 4) LINK.EXE and LIB.EXE with the current version 6 programs. And
now it works! Thanks again for the help
