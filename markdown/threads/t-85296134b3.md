Hi,
I believe, that both units are listeners until something in either end of
that fine RS232 cable provokes the unit to go to send mode.

If you start the transmission with an STX, you can look for that on the
Com-port. The sender should end the transmission with an ETX. In that way
the sender can say to receiver: 'I will wait for your ACK'.

When ACK is received, both units go listening again.

Now you are telling me, that you do not receive the ACK. Did the receiving
unit actually turn to send mode and was the ACK sent?

You can display the dialog on screen in TTY-manner if you pack up received
data in - let's say - 70 bytes blocks. Displaying the dialog could show you
what goes on.

I beleive that one of the units forgets to change from Send to Receive or
reverse.

Hope, that it helps

Ib
