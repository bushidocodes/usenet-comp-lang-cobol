Why do you prefer the function over the accept?  Admittedly on an IBM
environment, that command won't get century, but in an IBM environment,
we can now use the following:

 DISPLAY '  PASSED-SHOW=' PASSED-SHOW '.'
 ACCEPT WS-YYYYMMDD FROM DATE YYYYMMDD.
 DISPLAY 'TODAY = ' WS-YYYYMMDD.
 DISPLAY 'FUNCTION CURRENT-DATE = ' FUNCTION CURRENT-DATE.
 MOVE FUNCTION CURRENT-DATE(1:8) TO WS-YYYYMMDD.
 DISPLAY 'TODAY = ' WS-YYYYMMDD.

On a VAX, they always figured centuries were part of the date, so they
included in the standard accept forever.  I figure DEC was correct and
IBM wrong in this.
