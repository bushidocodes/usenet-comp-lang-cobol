Please help me write a procedure division or give me any advice for the
following problem
A program iS required to validate batches of transactions that are held on a
disk file Each batch should consist of up to 100 transaction records
followed by a batch control record Each field of the transaction record
needs to be
validated and hash totals of all the numeric fields calculated

If the accumulated hash totals match those of the batch control record and
there are no invalid records and the number of transaction records in the
batch does not exceed 100, then the batch is to be classed as valid All
valid records are to be written to a valid file An erroneous batch is to be
printed out on an error report that contains appropriate headings, with each
batch starting on a new page It is also required that the accumulated hash
totals for that batch are printed after each invalid batch A separate disk
file is to be used as temporary storage of a batch until the validity of the
batch is ascertained
  Thank you

PS. I am new to news groups so I hope I don't upset anyone with my ignorance
```

Colin Birch wrote in message <782o7i$ugs$1@news6.svr.pol.co.uk>...
>Please help me write a procedure division
>This is my second attempt at this problem.       PROCEDURE DIVISION.

       MAIN-CONTROL.
          PERFORM INITIALISE
          PERFORM TRANS-PROCESS UNTIL NO-MORE-RECORDS
          PERFORM TEMP-VALIDATE
          PERFORM CLOSE-DOWN
          STOP RUN.

       INITIALISE.
          OPEN INPUT TRAN-FILE
               OUTPUT TEMP-FILE VALID-FILE PRINT-FILE
          PERFORM HEADINGS.

       TRANS-PROCESS.
          READ TRAN-FILE AT END
             MOVE 'YES' TO EOF
             NOT AT END
             EVALUATE TYPE-T
                WHEN 'T' PERFORM TRANS-PROCESS-2
                WHEN 'B' PERFORM HASH-PROCESS
             END-EVALUATE
          END-READ.
       TRANS-PROCESS-2.
          IF REC-PART NOT NUMERIC
             OR REC-SUP NOT NUMERIC
             OR REC-DATE NOT NUMERIC
             OR REC-QUANT NOT NUMERIC
                MOVE 'Y' TO W-ERROR-FLAG
          ELSE
             ADD 1 TO REC-COUNT
             ADD REC-PART TO W-HASH-PART
             ADD REC-SUP TO W-HASH-SUP
             ADD REC-DATE TO W-HASH-DATE
             ADD REC-QUANT TO W-HASH-QUANT
             WRITE TEMP-REC FROM TRANS-REC
          END-IF.

       HASH-PROCESS.
          IF HASH-PART NOT NUMERIC
            OR HASH-SUP NOT NUMERIC
            OR HASH-DATE NOT NUMERIC
            OR HASH-QUANT NOT NUMERIC
               MOVE 'Y' TO W-ERROR-FLAG
          ELSE
               WRITE TEMP-REC FROM TRANS-REC
          END-IF.

       TEMP-VALIDATE.
          CLOSE TEMP-FILE
          OPEN INPUT TEMP-FILE
             READ TEMP-FILE AT END
             MOVE 'YES' TO EOF
             MOVE 'N' TO W-ERROR-FLAG
             NOT AT END
                IF W-HASH-PART = HASH-PART
                   AND W-HASH-SUP = HASH-SUP
                   AND W-HASH-DATE = HASH-DATE
                   AND W-HASH-QUANT = HASH-QUANT
                   AND REC-COUNT <= 100
                   WRITE VALID-REC FROM TEMP-REC
                   CLOSE TEMP-FILE
                   OPEN OUTPUT TEMP-FILE
                      MOVE ZERO TO HASH-PART
                      MOVE ZERO TO HASH-SUP
                      MOVE ZERO TO HASH-DATE
                      MOVE ZERO TO HASH-QUANT
                ELSE
                   MOVE 'Y' TO W-ERROR-FLAG
                END-IF.

       PRINT-DETAIL.
          IF W-ERROR-FLAG = 'Y'
             OR REC-COUNT > 100
             IF W-DETAIL-LINE NOT = SPACES
                MOVE REC-PART TO INVAL-PART
                MOVE REC-SUP TO INVAL-SUP
                MOVE REC-DATE TO INVAL-DATE
                MOVE REC-QUANT TO INVAL-QUANT
                ADD HASH-PART TO HASH-PART-TOT
                ADD HASH-SUP TO HASH-SUP-TOT
                ADD HASH-DATE TO HASH-DATE-TOT
                ADD HASH-QUANT TO HASH-QUANT-TOT
                WRITE PRINT-REC FROM W-DETAIL-LINE
                WRITE PRINT-REC FROM W-TOT-LINE
             END-IF
          END-IF.

       HEADINGS.
          WRITE PRINT-REC FROM W-HEADING AFTER PAGE
          WRITE PRINT-REC FROM W-HEADINGS-2 AFTER 1.

       CLOSE-DOWN.
          IF REC-COUNT > THAN ZERO
             OR W-ERROR-FLAG = 'Y'
             PERFORM PRINT-DETAIL
          ELSE
             CLOSE TRAN-FILE TEMP-FILE VALID-FILE PRINT-FILE
          END-IF.
I can't work out why HASH-FIELDS holds SPACES in the ADD W-HASH-PART TO
HASH-PART etc.when I put 'VALUE ZERO' in the DATA DIVISION. Any advice on
the logic of the program would be greatly appreciated.

THANKS.
```
