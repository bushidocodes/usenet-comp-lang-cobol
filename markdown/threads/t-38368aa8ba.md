I have a simple benchmark I use to test different development tools; it
involves calculating primes via an inefficient algorithm. Recently,
puzzled by the slow performance of a CASE-generated COBOL program on
HP-UX, I ported the benchmark code to COBOL, compiled it with Micro
Focus, and ran it. The resulting executable ran 50 times slower than a
C version, and 3 times slower than a Perl version. Is Micro Focus
really an interpreter under the covers? Am I missing a crucial
optimization switch (my compilation command line is "cob -X -O
")? Unfortunately I didn't bring the code home with me and I
can't post to Usenet from work, but the code's been looked at and
optimized by several local COBOL folks, and it's a very simple program
anyway. If anybody's interested I can post it tomorrow.

Microfocus can compile to INT code which is interpreted. But if
compiled to GNT or an EXE then it is native machine code. Both the GNT
and INT code need runtime libraries but the EXE is stand alone. I have
found in the past that INT is slow for caclulation modules.

It's worth checking the format that you are compiling to.

Dave.
