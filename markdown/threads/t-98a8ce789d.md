The book is not lying, Fujitsu uses whatever you assign the sort work
to as a comment - except when you specify a literal (in quotes, or tic
marks) in which case it recognizes it as an error. Remove the tic
marks (apostrophes for the terminally accurate) and don't surround the
name with quotation marks and life will be wonderful again.
```

Gee,  I think it has been two weeks rather than the usual weekly period
between users asking this question.

Fujitsu COBOL is *NOT* ANSI conforming and does not support ASSIGN TO
"literal" for a file associated with an SD.  To get your program to compile
(believe the message) and change your source code to:

SELECT SORT-WORK-FILE
       ASSIGN TO ANYTHING.

Don't worry about "assigning" that ANYTHING to a real file or disk - it gets
ignored anyway.  (And if you are paying for maintenance - send a note to
Fujitsu - demanding that they either make their product ANSI/ISO conforming
or give you your maintenance payment back!)
```
