Our organization is not allowed to endorse any products. So this is just
a reply to say it can be done. There are surely a number of products.

We chose SYBASE OpenServerConnect to do it.

Basically we modified a native CICS COBOL program that comes with that
product. The modifications are to (CALL or XCTL or LINK) to your own
code to do your processing. Mine calls subroutines to do DB2 stuff.
On the server side, a SYBASE Stored Procedure kicks off the CICS program
thru a gateway. You can pass parameters to some extent (length
considerations).
Ours is more complicated than the vanilla - we use Java inside a product
called NetDynamics to do screen and Server database stuff, then call the
SYBASE Stored Procedure which gets the COBOL underway. And then return
over the reverse trail.

We're in the pre-alpha stage in the ND/Java, but have the SYBASE and
COBOL all working fine. And we have ND/Java calling SYBASE from screens
on NT 4.0.

The only difference in this and what you asked is that ND runs its java
on a server of its own. So we do kick the whole shebang off from a
client, but the actual CICS invocation comes from a ND server.
