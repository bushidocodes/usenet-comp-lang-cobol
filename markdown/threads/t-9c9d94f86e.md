```
Descr: FTP and MicroFocus negatively signed characters in Display Format

A dataset containing negatively signed characters in Display Format use ASCII
characters p(hex 70)-y(hex 79). When this dataset is FTP'd to MVS, they remain
p-y characters, which are not the correct EBCDIC characters for negatively
signed characters in display format. They should be '{' (hex D0) through
'R' (hex D9).

Moving the sign to a separate character via the SIGN IS TRAILING SEPARATE
CHARACTER cobol clause works but creates a problem with CSF, which doesn't
support it. Changing ftp translate tables will not work either.

Any ideas?
```
