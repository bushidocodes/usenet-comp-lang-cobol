```
hihi all,

We're on our fourth programming assignment in a first COBOL course for an IS
degree. The professor admited that this would be the hardest program of the
semester. I've got everything stupendously beautiful (even got the
control-breaks working right) except one little detail: the detail line.

What the prof is wanting is for us to process budget records sorted by two
codes, cost (major) and budget (minor). For the first detail line in each of
the cost groups (four in total), we are to print both the cost code and the
budget code. In the subsequent detail lines, we aren't supposed to print the
first two column contents until there is a new budget code, at which point
we are to print just the budget code information. When the cost code
changes, we're to print both the cost and budget codes again in the first
two columns.

When we expressed our confusion on Monday about doing this, she didn't give
too much in the way of help, except to say we don't need two detail lines
(maybe we need three? *chuckle*). Our book isn't any help either, as it
approaches the sorting of material differently than she is in this program.
We are supposed to have two write routines, so I figure maybe I'm supposed
to do something there, but can't see it right now.

If anyone has any clues on how to implement this, I'd be so greatly
appreciative. Please email me, if you can, or just respond to the group.
Thanks :)

Rainya S. Wallner
RainyaS@aol.com
```
