```
Its been a long time since I've used C-ISAM files, but essentially what
I want to do is "extract" a subset of data from a LARGE indexed file.

For example, indexed file A has over 1,000,000 records in it - but the
subset of data I am looking for is only 15,000 of those records. I want
to create a new indexed file, A1, with all of the same indexes but
containing only the pertinent data from file A.

Does anyone know of any new tricks whereby I could possibly use a
single set of SELECT/FD statements/defs in order to accomplish this? I
know I can use COPY ... REPLACING to create two distinct named files,
but I'm wondering if a "cleaner" extraction method exists.

I hope this makes sense.

Thanks in advance,

Chris
```

```
> Does anyone know of any new tricks whereby I could possibly use a
> single set of SELECT/FD statements/defs in order to accomplish this?
I
> know I can use COPY ... REPLACING to create two distinct named files,
> but I'm wondering if a "cleaner" extraction method exists.

You could write a 'file handler' subroutine that used the same names
(except the disk file name) and just did a COPY without needing
REPLACING at all.

The main program would open and read the main file and then, when
required, call the subroutine which would write the passed record to
the extract file.
```
