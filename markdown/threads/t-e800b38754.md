### sct-3-6-1: CobolTransformer Data File Readers: new features

- **From:** Vadim Maslov <vadik@siber.com>
- **Date:** 1999-10-25T00:00:00
- **Newsgroups:** comp.lang.cobol,alt.cobol
- **Message-ID:** `<3814D53A.B3133D6F@siber.com>`

```
Siber Systems is pleased to announce release 3-6-1
of our DataReader technology. Using this technoliogy,
you or your application can read Cobol data files 
directly, without writing a Cobol program to do it.

For more info look at http://www.siber.com/sct/datafile/

The new features are:

* Date and Time recognition in record items.

For every data item you can now specify special date-time 
picture string that will tell DataAccess library that 
this data item should be interpreted as date-time and 
it will specify the format of this date-time data item.

Once data item is interpreted as date-time, you will be
able to reformat it to your liking using data output
facilities such as Crystal Reports generator.

Example: if you have numeric-edited data item that
stores date time as "98/12/31 23:59:59", it is 
automatically turned into numeric data item 981231235959.
You use date-time picture string YYMMDDHHNNSS
to cause this field to be interpreted as date-time.


* Data2Dbf converter.

This converter converts Cobol data files to dBaseIV DBF format.
While dBaseIV is not used much these days, this format
can be used as an intermediary format for importing Cobol data
into spreadsheets such as Excel and other databases such as Oracle.
Data2Dbf read FSC, MF and other Cobol data files.


* EBCDIC <-> ASCII translation.

Cobol Data File reader will be able to perform
EBCDIC <-> ASCII translation, so that mainframe
Cobol data files can be read on PC.


* DataAccess library API and manual released.

We finally formalized the API for DataAccess library.
Now you can license the DataAccess library from us
when you need to read data from Cobol files directly
into your C++ or C application. All read data is 
converted to C/C++-friendly data formats.


We also recap some of the important news from 
the previous DataReader technology releases:

* data2cr: Crystal Reports Reader for Cobol Data Files.
After you install the data2cr software (CR Data Source), 
your Seagate Software Crystal Reports report generator
will be able to read Fujitsu, Micro Focus and other
Cobol data files directly.

Just imagine: you will be generating elegant and informative 
reports from your Cobol data files directly, using 
powerful graphical user interface of Crystal Reports, 
instead of tediously programming report generation
directly in Cobol language.


* data2flat: Convert your Micro Focus, Fujitsu and other 
Cobol data files to comma-separated-values (CVS) text file.
This makes examining contents of you data files easy 
and it also helps a lot when importing Cobol data files
into other spreadsheets such as MS Excel or into databases
such as Oracle.

Best regards,
Vadim Maslov
```
