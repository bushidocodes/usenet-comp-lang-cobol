Looks reasonably well laid out so any comments are minor:

1.  In the working storage section put explanations of field contents
with the relevant dataitem. Otherwise, 15 years down the line the
comments will have been lost or neglected and therefore be out of date.

2.  Put each FD on a new page.

3.  Use sections and not paragraphs. Prevents the risk of dropping
through. (Don't spam me I've already seen the lengthy debates :-).

4.  I don't like the MAIN section. I usually structure the MAIN section
with performs of INITIALIZE, MAIN and CLOSE sections and stick the read
ahead in the Initialize section. You are doing a read ahead (good) but I
don't like the multiplicity of checks that you do for errors after
completing the main process. 

If you read Michael Jackson's book on structured programming and follow
the advice to the extent of deducing the annotated program structure
diagram you should then be able to code direct from the diagram without
producing spaghetti code. Junk anything he says about pseudo code.
