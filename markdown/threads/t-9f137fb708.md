```
Hi all,

Thank you for contributing to my previous thread on the program and file
migration issue from RM/Cobol85 (tm) files to the Fujitsu (tm) file system.

So far I got the following tip :

COMP-n is not supported by Fujitsu (they're not making a secret out of that)

Next I noticed JUSTIFIED RIGHT is not implemented according to the
RM/Cobol85 rules

If there are any more pitfalls please let me know.

So for the time being I intend to proceed with the following solution (for
data migration) :

I will generate (not write, if you are interested in the tool (written in C,
including source code) just reply) a conversion program for each file
containing COMP-n fields. It will read the source file, unpack any COMP-n
fields and writes a new file with the unpacked fields. All within a
RM/Cobol85 environment.

The resulting files, according to your input so far, should be compatible
with the Fujitsu file system.

I tested Fujitsu with RM/Cobol85 indexed files (without any COMP-n fields)
and all seems to work perfectly.

Any feedback is welcome

Danny Maijer
```

```
If you are using a RM/COBOL release 6 or greater, be shure that the
file version of the newly created files is 0; otherwise, Fujitsu will
be able to read records but not write or rewrite them. There are at
least 2 ways of acomplishing this, but I think the most sensible is
using "rmdefinx" to set the correct file version before any record is
written to it.

Regards,

Paulo Vieira, Emporsoft

PS.: I think that the only COMP-n that isn't commom between the 2
compilers is COMP-6 (packed unsigned) which has no equivalent in
Fujitsu. COMP-3 should work OK although I think it is preferable to
keep everything unpacked since the file system compression does the
job.
```

```
If the files really are RM/85 as you say, and if you are converting to
Fujitsu then you may not need to do anything but to run Fujitsu
recovery program on the files, as they use the same filesystem.

Whether or not you can do this will depend on the type of information
you hold on the file.

If you have a FD that contains any COMP-6 (and eventually some of the
other COMP-xxx) then you will not be able to use the data directly,
and will need to do a program in RM to convert to a plain file, and 
then another one to load the info into the new file (with the correct
COMP-XX fields as supported by Fujitsu). Fujitsu File Utility program
may also be used to load the file from the plain file to a Indexed
file (never used it though).

If you need help with this conversion my company has loads of
experience in doing this type of conversions and we would gladly help
you doing it if you posses the File descriptions.
```
