======

here's a simple example i have for an older MERANT compiler (Visual Object
COBOL). the example should work just fine with the current product (Net
Express).

a nice/slick alternative to using external-forms to output a page is to
simply embed the HTML into your program. in this case the 01 output-page
item is not needed and instead of the 'display output-page', you have:

exec html
    copy "output.htm"
end exec

well, something like that anyway. it's been a while. you can of course just
enter HTML directly; you don't have to use copybooks. anyway, pretty simple,
COBOL/SQL-like implementation.

==>start of cobolcgi source
identification division.

working-storage section.
01 input-page is external-form.
   03  name-field pic x(30) identified by "Name".
   03  phone-no   pic x(30) identified by "Phone".
   03  email      pic x(30) identified by "Email".

01 output-page is external-form identified by "output".
   03  name-field pic x(30) identified by "Name".
   03  phone-no   pic x(30) identified by "Phone".
   03  email      pic x(30) identified by "Email".

procedure division.

    accept input-page
    move corresponding input-page to output-page
    display output-page
    .
==>end of cobolcgi source

==>start of input.htm
<HTML>
<HEAD>
<TITLE>Sample Form for VISOC</TITLE>
</HEAD>
<BODY>

<P>Visual Object COBOL Input Form

<FORM METHOD="POST" ACTION="/cgi-bin/cobolcgi.exe"></P>
<PRE>
Name     <INPUT TYPE="TEXT" NAME="Name" VALUE="" MAXLENGTH="30" >
Email id <INPUT TYPE="TEXT "NAME="Email" VALUE="" MAXLENGTH="30" >
Phone    <INPUT NAME="Phone" VALUE="" MAXLENGTH="12">
<INPUT TYPE="SUBMIT" VALUE="Send Form"><INPUT TYPE="RESET">
</PRE>
</FORM>

</BODY>
</HTML>
==>end of input.htm

==>start of output.htm
<HTML>

<P>*** SAMPLE OUTPUT FORM ***</P>
<P>Dear :name-field</P>
<P>blah blah blah</P>

</HTML>
==>end of output.htm

I just brougth a job to acceptance testing stage: A full fledged web
(intranet) app written in COBOL + HTML and some JavaScript. All I needed
to know came from the Merant/Micro Focus docs that come with their
NetExpress product and from experience. The basic needs are:

* HTML and JavaScript knowledge.
  Having some years of script/GML experience, this is learnable in two
weeks.
* COBOL
  ... what need I say
* Accept groupItem from HTMLCOBOLdefinition
  Generated code and datastructures, using NetExpress' HTML editor.
* Exec HTML / End-Exec
* Understanding of pseudo-conversational programming like under CICS.

Within 2 months I had set up a working environment, including library
management (Visual Sourcesafe) for multi-user development, some
skeletons for HTML-pre- and post-page processing-programs and a state
maintaining mechanism.
Within two more months an app with 5 logical screens was written, using
frames, navigation via hyperlinks etc. etc. This app with separate I/O
modules.
This all accesses data via an application server coupling CICS on NT
with CICS in mainframe. The data comes from various databases (IDMS,
DB/2) on the main frame and from some SQL/ODBC (Access and SQL/server).
The 5th month was to cook it together, install the NetExpress RTS on the
web server and copy the app (CGI-exes, some statis HTML and GIFs, some
JavaScript and a bunch of DLL's) to it.

Lots of sample code with NetExpress too. Get their CD, its free to use
for 30 days, thereafter the IDE won't work, but the demo's stay on
harddisk.

NetExpress turned out buggy (for web-app development), but giving great
tools. Version 3 recommended!

Proud Frog
