COBSQL is used to preprocess the output of DBMS vendors precompilers.
So you would need to install Pro*COBOL from Oracle before using
COBSQL. COBSQL does not contain the Oracle precompilers but it will
invoke them for you depending on the directives set. If you use the
following directive CSQLT=ORA COBSQL will try to invoke the 1.8
precompiler if you use CSQLT=ORA8 COBSQL will try to invoke the 8.4
precompiler from Oracle. When you get your NetExpress product go to
the Online documentation, Database Access, Chapter 11 COBSQL. There
are Oracle sections there to answer you questions in more detail.

I recommend setting up Oracle first, make sure you can precompile
their sample programs using just ProCOBOL. Also make sure you can
connect to an Oracle database if you are planning on animating you
programs, using SQLPlus or another tool. In NexExpress you are just
going to add the program to a project, select COBSQL as your
preprocessor and add any COBSQL directives and Oracle directives you
need. The Online doc gives good examples of sample sets of directives
needed when using COBSQL + Pro*COBOL.
