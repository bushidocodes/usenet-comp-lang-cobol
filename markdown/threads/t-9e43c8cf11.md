We distribute RPV with our old DOS programs.

You modify your programs to write their output to a disk file (with ".RPV
extension) instead of sending the output to the printer.

RPV (Report Program Viewer) is a (free) Windows program that interrogates a
disk folder looking for files with an .RPV extension. Finding one, it grabs
the file and sends it to the Windows printer spooler.

What this accomplishes is to insulate your DOS program from the oddities of
print spooling and allows Windows to handle such things as weird printers or
ports (USB).

Just a quick look shows that the RPV people have sexed-up the standard RPV
by allowing you, as you create this print file, to insert formatting
commands (font, size, graphics, etc.), but the principle seems to be the
same: you generate a file and RPV sends it to Windows.

I haven't gotten down in the weeds of "Steel," but I hope my understanding
of the concept helps.
