Sean,

Word of advice - consider the following :-

01 DataFlag.
    88 DataRemainsFlag        value "NO", "No", "no".
    88 NoDataRemains         value "YES", "Yes", "yes".

01 NewDataFlag.                    pic 9.
    88 ThereIsData                    value 1.
    88 ThereIsNoData                value 0.

You've already found the problems with 'DataFlag', whereas I consider the second
more succint and - DON'T MOVE values into the flag.

    set ThereIsData to true
    perform until ThereIsNoData
        read next record
            at end set ThereIsNoData to true
            not at end .... continue, still looping through the perform
                ( BUT.... it might also be if record-type <> what I want
                                    set ThereIsNoData to true etc.....)
        end-read
    End-perform

The checking is a variation on a theme but setting to ThereisNoData gets you out
of the routine without second-guessing is it "NO" or "no".

Whichever approach, use meaningful condition-names even if they do appear
verbose - IveGotALovelyBunchOfCoconuts, IhaventGotALovelyBunchofCoconuts -
subject of course to size of condition-names <G>.

Jimmy, Calgary AB
