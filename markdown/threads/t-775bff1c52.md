```
> data... it gave the following error...

> "Segmentation fault...." what must be the reason for this error?

There's a bug in your program, or in something that it calls. An
unhandled illegal memory access is always a bug (unless you're testing
what happens when a memory access violation isn't handled). For that
matter, a handled illegal memory access is often a bug.

There is a considerable collection of literature on how to diagnose
segmentation violations on Unix (which I presume you're using, since
most other popular OSes use different terminology for illegal memory
access). I suggest consulting it.
```

```
If you are seeing "funny" (unexpected) characters ONLY as the last character if 
a field, then you may want to tell us how you are "looking at" the data.  Often, 
a single byte of data will contain both a numeric digit AND a nibble of data 
indicating the sign of the entire field.  When looked at by SOME external 
(non-programming) methods, such bytes may be displayed as "funny characters".

Such fields WILL pass an "IF NUMERIC" test - and should be processed correctly.

NOTE WELL:
  I don't know of any time that "bad data" causes a "segmentation fault" - with 
the possible (but unlikely) case of where the "bad data" is in the object of an 
OCCURS DEPENDING ON clause *or* when it is used in an ALTER ... DEPENDING ON 
clause.  The former usually causes different symptoms and the latter is so 
unusual that I can't even guess what symptom it would cause.
```
