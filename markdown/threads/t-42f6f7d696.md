Hi,

We just converted MF Cobol from Mainframe to Unix environment. We found that
we have to recordize(add new line character) or unrecordize (remove new line
character) files before or after Cobol programs. For example, one Cobol
program need to read a sorted file. I had to recordize that file and use Unix
sort command to sort, then unrecordize the file and feed to Cobol program to
process.

Is there a way to avoid this recordize and unrecordize processing?

Thanks, Andy

There is a directive SEQUENTIAL that specifies the default file type for
sequential access files.

I think it may be SEQUENTIAL"LINE" or SEQUENTIAL"RECORD" that you need to
use.
