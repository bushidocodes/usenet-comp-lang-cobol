It appears there are two 'bugs'.

Consider,

9 move -n to r
* 14-S**********
** Invalid operand

In this case '-n' is an expression that is invalid in that context. But,

FUNCTION MAX (argument-1 ...)

doesn't specifically permit expressions. The compiler should have flagged
the '-n' in each of the three cases as 'Invalid operand'.

Interestingly, Animator(r) 'hangs' when trying to process intermediate
code with unary minus, line by line; but not with 'Zoom'.

Animator 'hangs' with this code,

compute r = -n
compute r = function max (n r)
39+

But not with this code,

compute r = -1 * n
compute r = function max (n r)
39+

'-n' is an expression and MAX cannot accept an expression. The reason
appears to be that the values accepted must belong to a 'class';
integer, numeric, alphanumeric, etc. Expressions do not have a 'class',
but identifiers do.
