```
There have been several recent posts (questions, answers, and more
questions) about how to handle "internationalization" issues in COBOL (for
example, upper-/lower-case conversion of European accented letters,
alphabetization of upper and lower-case letters, etc).  The draft of the
next Standard provides some "nifty" solutions to these problems which I
think that many of this NG's readers might be interested in.  Therefore, I
have put out on the web an excerpt from the "Concepts" section of draft 1.4
of the next Standard.

  NOTE WELL:  Draft 1.5 is expected "momentarily" and there are changes -
both in details and facilities - in this draft for internationalization (and
other features).  Therefore, this concept section should only be read for
"general over-view" NOT as details that you should expect to see "as is".

Please look at the section at:
   http://pw2.netcom.com/~wmklein/INTER.HTM
```
