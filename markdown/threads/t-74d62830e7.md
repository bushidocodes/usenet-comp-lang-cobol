True that Will does cover the topic, but only in a general tutorial sense,
aligned with the main topic, Web Programming. He doesn't cover features like
'Normalization' etc. And Will's text is backed up by the Net Express Open SQL
Assistant module which  allows you to select type of Query, generates the bare
bones, then allows you to include columns from your DB tables, query conditions
and then generates the complete  syntax between EXEC SQL ....... and.......
END-EXEC, plus picking up from your DB Table, it automatically generates a
copyfile in three parts (1) Table References INT, VARCHAR etc., (2) COBOL pic
x's and pic 9's, plus (3) NULL indicators.

All of this can be copied into your current program source. Add CONNECT etc to
the OpenSQL  'construction boxes' and you can do a test run against your DB and
view the results, without any compiling of COBOL source. It does an excellent
job, but is very specific to Net Express. (The only very MINOR criticism is
that what it generates is 'wordy; it uses Aliases and prefixes column IDs with
the Table Name - but a feature necessary when linking multi-tables with
non-unique column IDs. For a 'general' package, the verboseness is acceptable).

To Dennis Edward who was after 'tutorials' do a search on SQL using google.com
or Web Ferret. Two immediate sources :-

- http://www.fluffycat.com/sql/ - you'll see the author is nuts about movies
<G>

- http://www.highcroft.com/highcroft/sql_intro.pdf

this latter one is a good lead in for what you are after and has a whole bunch
of links to other SQL sites.
