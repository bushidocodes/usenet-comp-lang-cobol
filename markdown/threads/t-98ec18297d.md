At work we (not me) use:-

net express Cobol on windows
server express Cobol on Unix
cobol400 or ILE Cobol on AS400

above languages.

wondering if
there was a universal solution to communicate between VB and the above Cobol
versions?

If not what is the best approach for each individual platform and are there
any limitations?

We would like to be able to call COBOL programs and pass parameters from VB
and visa versa.

Our AS400 is running V4R5 but would like a solution for older and newer
operating systems.

I am currently using an SQL database to pass info between ASP/VB and net
express.
Someone has a java interface for passing info to and from server express
and I am accessing db2/400 from VB using IBMDA400 client access driver.

Any snippets of code or links would be most helpful.

thanks,

Henry:

I know that it's not VB, but COBOL sp2 supports the ability to build a
full GUI interface (complete with embedded OCX Controls).  When
combined with our Thin Client product, the COBOL program runs on a
variety of different server types while displaying the screens scross
a TCP/IP network.

The nice thing about it is that it allows you to maintain a single set
of source code and still support multiple server environments.
