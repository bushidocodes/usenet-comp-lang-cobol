```
We have an application written in MF COBOL which works fine locally, but
which is giving us severe problems running over ISDN.

The problem occurs if data is being entered into a text field (using a
normal Accept) and the line drops - usually due to the user answering the
phone.

The line is not picked up again, and the result is system hangs, error 43's,
file corruptions and deeply dissatisfied customers.

We are using 3Com office connect 531 routers on an NT4 box.

Has anyone any ideas???

Gordon Wallace mailto:grwallace@sol.co.uk
```

```
1) Change your file handling so that is has transaction control (SELECT ...
WITH ROLLBACK, COMMIT, ROLLBACK)

2 Eliminate physical file locks and implement logical file locks by adding a
timestamp to every record and testing against the timestamp when updating

3) Close files before a DISPLAY and reopen them after the ACCEPT

4) Look into the 'ON EXCEPTION' clause for ACCEPT statements. It may allow
you to catch a lost connection or timeout condition and allow you to cleanly
terminate the application.

4) Implement a client/server architecture

6) Use a different file system like BTRIEVE which may offer better recovery
options. Micro Focus provides a file handler for BTRIEVE
```
