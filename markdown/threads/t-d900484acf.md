I tried reinstalling the whole COBOL package, but it didn't solve anything.
Is it a compatibility problem? Or maybe are there issues I'm not aware of
with dynamic linking - the dynlink document seemed somewhat outdated?

Hi Jacques,

I would not advise you use OCDS and Java together because Java requires that
the language being called be thread-safe.   OCDS is *not* thread-safe and
does not support shared object either.

However if uses ServerExpress v2.0.10, you will find this includes Java
support for you, it is also thread-safe and includes shared object support.

We provide a Java package that allows you to call COBOL or Object COBOL
from Java, which means you do not have to use JNI....
