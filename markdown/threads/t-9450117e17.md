I think you are on the right track in looking at Acucobol to meet your needs. Over the past 6 years I've been working with a couple of COBOL software houses who have been able to make their code hardware (and OS) independent by using Acucobol. It has been possible for two reasons:

1. The object is p-code, not native. This means that the "COBOL" code is identical on all platforms. (Neither software house has reported any performance issues related to the speed of the interpreters.)

2. Their Vision ISAM package is available on all platforms except VAX/VMS which uses the native ISAM.

These two things make it all but trivial to mix and match platforms with no COBOL changes. i.e. To move from one platform to another, all you need to is to install the interpreter, copy the program files, and copy the data files.
