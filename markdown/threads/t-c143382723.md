In the same vein, those who face conversion from one-character binary to 
hexadecimal or other such games might consider a different approach.  
Construct a table (at execution time or compile time) of the answers, and 
use (incoming value + 1) as a subscript.  For examples, the table for hex 
conversion would look like:
     01  HEX-VALUES.
       03  PIC X(32) VALUE "000102030405060708090A0B0C0D0E0F".
       ...
       03  PIC X(32) VALUE "F0F1F2F3F4F5F6F7F8F9FAFBFCFDFEFF".
     01  REDEFINES HEX-VALUES.
       03  HEX-VALUE PIC XX OCCURS 256 TIMES.

I have used this method quite successfully in several instances, 
including translating 7-bit no parity Ascii to 8-bit Ascii, with 
Even/Odd/No parity specified at execution time.  In another case it was
much easier to break bytes to nybbles for packed no-sign decimal (two 
digits per byte, no sign nybble).
