To answer the original question:   NATURAL is the reporting tool for ADABAS
(no "e"!) which is similar to Easytrieve, Culprit, and Focus [blech!! and double
blech!!].  Before the advent of SQL, each database vendor derived their own
proprietary adhoc reporting/query tools to save application maintainer the cost
of coding in COBOL.  These were interpreted languages, but most use a vector-
function call approach now and have improved efficiencies on the higher-speed
enterprise servers.  ADABAS is the database engine from Software AG [Germany?];
it is one of the few "inverted-list" engines running.

I've found focus on application experience not tech experience,
and skill of problem-solving [regardless of tools invoked] to be
transferrable across varied tool platforms.  Tools come and
go, but opportunities to resolve crises will always exist.

"Have mind, will travel"
ej

I'll tell you what it is, but you probably don't want to know.  This is known as
a 4th Generation Language.  It was developed by two guys in Germany and the
company is headquarters in Germany.  The north american hqs is in Denver and is
called Software AG of North America.  You can probably find there web site.  I
used it while working for the State of Washington.  You can develop programs
much faster than COBOL, that's the good news, now the bad news, its not too hot
and fully functional as COBOL is.  It is a real resource hog and isn't
documented very well.  You can't have more than two dimensional arrays.  Batch
programs take forever to run.  The language is called Natural2 and the database
is call ADABAS.  It's an inverted database like FoxPro.  If something abends in
this language, good luck.  Some people love it, but it had too many quirks for
me.  Sometimes the production modules will just not execute and they don't know
why, neither does Software AG.  We had a ABABAS problem where I was working at
Dept of Transportation and they never did get it fixed.  How's that for vendor
support.  You can jump in, but make sure you have a life raft, inner tube, etc.
Good Luck.

Barry Seattle

Natural is an English language programming language similar in many
respects to Cobol. If you know Cobol then, with the aid of a manual, you
will be able to read and understand the Natural source code. Natural was
developed as a 4gl tool to extract data from the Adabas database (an
inverted list database) developed by Software AG of Darmstadt in
Germany. IMHO it's strength is in developing online applications using
the inbuilt map painter. It is true, as pointed out elsewhere, that it
is not as efficient as Cobol but then you should use the right tool for
the right job. It is generally easier to debug than Cobol (I have used
both languages in equal measure) but you can get stuck for no obvious
reason and sometimes have to recompile core modules or go to SAG for a
fix. Cobol can access the Adabas database by either using ADAMINT
(precompiled access modules) or ADA/SQL. Natural can access VSAM and
DL/I and DB2 databases. There are pc versions of Natural (eg Natural
Lightstorm) which attempt to enforce a project environment upon the
user. Adabas/Natural also exists on IBM, ICL mainframes, and Unix boxes.
The editor is ideosyncratic but for some extra money you can use an ISPF
lookalike.
