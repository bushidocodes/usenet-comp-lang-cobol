```
Since the rather lengthy discussion on palindromes, I have been thinking
about a way to "squeeze out" all the non-character "bytes" from a text
field.  As I indicated in a separate note, I really do not like "looping
thru individual bytes" (as this always seems to me to be a "failure" on the
part of native COBOL - and is often hard to maintain).  I know that such
loops are almost ALWAYS more efficient at run-time, but I was wondering ...

 Can anyone come up with a "clear" way to get rid of all non-alphabet-
characters from an input field when creating an output (temp) field?
Assume that the input is 80 bytes long and may include spaces (any number at
the beginning, end, or middle) with any extraneous punctuation characters at
any point - but no "unprintable" characters.

It seems to me that there should be a way to "unstring" all the input and
then string the resulting "tokens" back together.  However, I haven't come
up with any thing that really seems like a worthwhile algorithm.

Anyone want to approach this totally hypothetical semi-challenge?  (All
Standard verbs and constructs are allowed - just not a method that does a
character by character loop)
```

```
#include <stdio.h>
main()
{
  char istr[50] = " Now is the time. ", ostr[50], *iptr, *optr;
```

```
Actually, a C program would not need to loop char by char. Use

This program gives the same results as my 'no-punc2', which
was posted elsewhere in this thread.

NOTE: I did not actually link the object into a COBOL program!

----------------
/* NoPunct.c - C version of no punctuation */

#define NOPUNCT_TESTING

#include <stdlib.h>
#include <stdio.h>
#include <string.h>

/*---------------------------------------------------------
   This routine is callable from COBOL. It assumes that
   neither the input or output string should be terminated
   by a nul character. Both istr and ostr must point to
   80 character records.
---------------------------------------------------------*/
void NoPunct ( char *istr, char *ostr )
{
 char *iptr;
 char *tokens = "~!@#$%^&*()_+`1234567890-={}|[]\\:"";'<>?,./";

 /* move spaces to output string */
 memset ( ostr, ' ', 80 );

 /* get memory for a copy of the input record */
 iptr = (char *) malloc (81);
 if ( iptr != NULL )
 {
  char *next_token;
  char *optr = ostr;

  /* create a nul terminated copy of input record */
  strncpy ( iptr, istr, 80 );
  iptr[80] = '\0';

  /* get the address of the first token */
  next_token = strtok ( iptr, tokens );
  while ( next_token != NULL )
  {
   /* copy the token into the output string */
   strncpy ( optr, next_token, strlen(next_token) );
   optr += strlen(next_token);

   /* get the address of the next token */
   next_token = strtok ( NULL, tokens );
  }

  /* free the memory */
  free ( iptr );
 }
 else
 {
  /* memory allocation failed */
  /* output record will contain spaces */
 }
}

#ifdef NOPUNCT_TESTING

int main ( void )
{
 char instring[81], outstring[81];
 char * string1 = "A man, a plan, a canal -- Panama.";

 memset ( instring, ' ', 80 );
 instring[80] = '\0';
 strncpy ( instring, string1, strlen(string1));
 NoPunct ( instring, outstring );
 outstring[80] = '\0';
 printf ( "%s\n", instring );
 printf ( "%s\n", outstring );
 return 0;
}

#endif
----------------
```

```
I have supplied two subprograms. The first, "no-punc1," extracts the
"words" and places them in the output string  with the words separated
by a single space. The second, "no-punc2," is probably closer to what
you intended. It finds the non-alpha, non-space characters replacing

       identification division.
       program-id. klein.
       data division.
       working-storage section.
       01 string1 pic x(80) value
             "A man, a plan, a canal -- Panama.".
       01 string2 pic x(80) value
             "  Able was I, ere I saw Elba.".
       01 string3 pic x(80) value
             "%^*(^(*a}{:>b|}{^%c@#$%^d>:{+e>;'-7258f*&()".

       01 string-out2 pic x(80).

           display "start"

      * ASCII character set assumed
       01 non-alpha-char pic x(42) value
             "~!@#$%^&*()_+`1234567890-={}|[]\:"";'<>?,./".

      * non-alpha-space must have the same size as non-alpha-char

           move function upper-case (input-string)

             converting non-alpha-char to non-alpha-space

           move 1 to in-ptr

      * prevent leading space in output
               if token not = space

      * output the token followed by a single space

                     space delimited by size
                     into output-string with pointer out-ptr

               end-if
           end-perform

               string token delimited by low-values
                 into output-string with pointer out-ptr

           end-perform
           exit program.
       end program no-punc2.
```

```
To me it seems this the place for character by character code.  An 'INDEX BY
SUB1' phrase
can be added to the BUFF1 line if you prefer but SUB2 should be left as an
ordinary subscript.
Note SUB2 will contain a count of the number of characters when this perform is
over.

01  BUFFER-1.
     05 BUFF1 OCCURS 80  PIC X.
01 BUFFER-2.
     05 BUFF2 OCCURS 80 PIC X.
------------------------------------
MOVE ZERO TO SUB2.
PERFORM VARYING SUB1 FROM 1 BY 1 UNTIL SUB1 > 80
  IF BUFF1(SUB1) ALPHABETIC AND BUFF1(SUB1) NOT = SPACE
    ADD 1 TO SUB2
    MOVE BUFF1(SUB1) TO BUFF2(SUB2)
  END-IF
END-PERFORM.
```
