```
On June 27th, the following question was asked,

"Dear all,
According to IBM APAR II06138, we should not use STRING, UNSTRING, INSPECT
because they will eat up our 24bit Getvis area. Any recommendation of
alternatives for these 3 words???? "

There were then responses showing that this didn't SEEM to apply to the
current COBOLs, but the original questioner replied,

"Thanks for your detail explanation. But IBM guy told me the issue applies to
COBOL/VSE even through it state in the COBOL manual that it only applies to
DOS/COBOL. Is that true??"

I have now received a reply from my "usually reliable sources" that states,

"I believe, from entries in the VSE-DIGEST newsgroup, that Patrick Lee was
referring to APAR II06138. That APAR certainly reported the problem with
STRING, etc. BUT it was raised in 1992, some 3 years before the advent of
COBOL/VSE. The APAR does say in the text that the COBOL involved is DOS/VS
COBOL. The verbs referred to are not restricted in COBOL/VSE applications for
CICS, and if there is a problem with those verbs it should be reported to
IBM."

> > > >

Sorry for the delay, but I hope that getting this "final" answer is useful to
anyone running across this thread or hearing the out-of-date information
elsewhere.
```
