```
I recently downloaded the free Fujitsu v3 from http://www.adtools.com
and used it to compile a 4,500 line IBM COBOL II mainframe program into
a dll.  This program, an actuarial calculation module with no I/O
except through the LINKAGE SECTION, proved a very easy port to Fujitsu
and is working great when called from Visual Basic 6.

In thinking about turning this functionality live in production,
questions have been raised concerning how this fits into our client
operating system plans.  We are being told that the government
agency I work for will require all PC's to be migrated to Windows 2000
as early as (gulp) March 2000.  So:  Has anyone running a Windows 2000
beta tried running Fujitsu v3 dll's and, if so, what have experiences
been?

An obvious answer to this email is to buy a commercial Fijitsu
product.  Well, maybe we will, especially if we find good uses for
Fujitsu COBOL beyond this one actuarial program.  However, it would be
nice to have a reasonable expectation that this will continue to be
optional.

Thanks for all wisdom and opinions.

Steve Eisenberg    Wynnewood, Pennsylvania  U.S.A.

Before you buy.
```
