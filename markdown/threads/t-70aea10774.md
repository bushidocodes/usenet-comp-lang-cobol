Although, support was changed in later versions (those currently supported)
of Micro Focus COBOL, I think that version ONLY supports recursive
programs - for programs with a Local-Storage Section defined (it does NOT
need any data items in it).  Obviously, if you WANT "instance-specific"
data, then you must use Local-Storage (not Working-Storage) section.

You could put an ENTRY point in a and call that from b.

You could split a into x and y where x is a small control module that
calls y, then x call b which calls y.

You could do a 'reverse call' where a calls b repeatedly with b setting
a flag that is passed in the call.  If the flag is 'X' then a stops
calling b, if the flag is 'Y' then a performs some code that is what you
need to do when 'b calls a'.  So instead of 'b calling a' it sets the
flag to 'Y' and does a EXIT PROGRAM. and then carries on when the next
call is made to it.
