Hi group,

I'll answer the questions here and set things straight.  The book is
actually $49 and comes with the "watered-down" version of Professional
COBOL. It may be sold for less depending on who is selling it. It is not
crippled.  If the person chooses the Enterprise version, SOME versions that
come with Enterprise will be timed, but the Professional portion or the
regular version part will not time out.  The Mastering COBOL CD offering
the V4 compiler with COBOL97 and PowerCOBOL does NOT contain everything,
especially for PowerCOBOL.   For those wanting to know the differences
between Standard, Professional, and Enterprise, it is explained in
http://www.adtools.com in the link "What COBOL".

Now to show what has been removed from the regular version comparing the
Mastering COBOL CD to the purchased one from Fujitsu, read below:

Fujitsu COBOL97 Compiler V4L10 Level with the following restrictions:

* Optimization
* ODBC (SQL)
* Runtime Installer
* CBL_ routines (Import Library is removed)
* Micro Focus to Fujitsu conversion utilities (Source and file converter
are
removed)
* CGI routine (COPY Book and DLL are removed)
* Call interface for Fujitsu File system (LIB and DLL are removed)
* 2G bytes file system (DLL is removed from CD-ROM)
* Simplified inter-application communication (Server EXE is removed)
* Accessing Btrieve file function (Btrieve files are removed)

PowerCOBOL GUI Builder at V4L10 level with the following controls (dlls)
removed:

* drivelist and folderlist and filelist control
* Timer and drivelist and folderlist and filelist control
* shape control
* image, animation and mci control
* excelconnection and db access control
* DDE and excelconnection and db access control

Hope this helps clear up the question regarding pricing. Purchasing the
compiler directly from Fujitsu will give you all the tools you want from
the compiler.  Also because the features in 4.2 are strongly improved,
programmers will find want to upgrade to 4.2 after finding satisfaction to
begin with basically from using the watered down version.

Regards,
Gary Roush
Fujitsu COBOL Support
