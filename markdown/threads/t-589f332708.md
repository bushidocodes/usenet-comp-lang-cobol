One (over-kill / expensive / hard-to-justify) approach I have used with
great success is to design a special-purpose computer, then write a Cobol
program to simulate it.

A program 'written' in the new machine's language might appear like the
expressions in a spread-sheet, and (if loaded from a configuration file)
is far easier to maintain / test / etc.

Sometimes the thinking and design process involved helps suggest a better
(more traditional) solution.

I got the idea from one of the first Fortran compiler for IBM/360 DOS
machines. They designed a computer which was optimal for compiling
Fortran, then wrote an emulator. The emulated machine featured stacks of
tokens with the usual push/pop stuff, and also a 'roll' instruction to
process multiple input stacks. This was one of the most bug-free
compilers I have ever used, and at the time, a rarity.

Usually you can locate the repetitive coding, isolate the
variables and set up tables with suitable values for them.
It helps if you're using COBOL 85 with the EVALUATE verb.
A tip: always test for OTHER values, it will surprise you.

Example (completely untested):
IF NAME = "ABC CORP"
COMPUTE PENSION = 17 * SALARY / 100
ELSE
IF NAME = "XYZ CORP"
COMPUTE UNION-FEE = 1 * SALARY / 100
ENDIF
[etc etc]
ENDIF

could become (completely untested):
01 DATA-TABLE-A.
05 FILLER PIC X(20)
VALUE "ABC CORP A017000".
05 FILLER PIC X(20)
VALUE "XYZ CORP B000001".
[another 8 entries]
01 FILLER REDEFINES DATA-TABLE-A.
05 ENTRY-A OCCURS 10 TIMES INDEXED BY IX-A.
10 COMPANY-A PIC X(8).
10 FILLER PIC X(1).
10 ACTION-A PIC X(1).
10 PENSION-PERCENT PIC 9(3).
10 UNION-FEE-PERCENT PIC 9(3).
10 FILLER PIC X(4).

[SEARCH table to match NAME to COMPANY-A (IX-A)]
[IF successful..]
EVALUATE ACTION-A (IX-A)
WHEN "A"
COMPUTE PENSION = PENSION-PERCENT (IX-A) * SALARY / 100
WHEN "B"
COMPUTE UNION-FEE = UNION-FEE-PERCENT (IX-A) * SALARY / 100
WHEN OTHER
PERFORM ZZ-DEBUG
END-EVALUATE.

A commercial version would have more error-checking, eg NUMERIC,
comments attached to the table(s) and some optimisation.
