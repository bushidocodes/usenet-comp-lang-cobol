These two sound to me more like terminal definition problems than
ones with vim or the syntax definition file - possibly an error in
the definition of rmso ("remove standout modes") in the terminfo
entry, or TERM is set to the wrong terminal type.

I personally hate syntax coloring, though, so I've never tried out
the vim COBOL syntax support.

However, though vim is my editor of choice on Unix and Windows, I
find it cumbersome for editing COBOL (unless free-format, which is
generally the case with the code I write, but not with code written
by others but maintained by me), because of the lack of support for
arbitrary tab stops.  So while I continue to generally use vim for
COBOL, I admit it's a hassle when I have to do anything substantial
to fixed-form source.

> Does anyone know of an editor for UNIX that is COBOL specific, or
> perhaps a better COBOL definition file for vim?

Not offhand, though I thought I might take a look at cobol.vim to see
if I could fix the problem Kris Dames raised.[1]

I'm sure emacs could be customized up the wazoo for COBOL.  Whether
that would be a solution depends on your attitude toward emacs (mine:
"ugh"), whether someone else has done the work already, and if not
whether you're inclined to do it.
