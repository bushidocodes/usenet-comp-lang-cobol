```
We recently purchased some Micro Focus COBOL products for our Unix server,
namely, ToolBox, OSX and the compiler. The ToolBox provides our development
environment which includes other features such as the Advanced Animator, the
Screens facility, etc.

As I said, the MF software resides on our Unix server so I 'telnet' into it
from my IBM PS/2 to work on programs while the users will to execute programs
- we all have IBM PC's.

My problem/question is the MF documentation books provide alternate key usage
because the software is in fact emulating from a Unix environment, for
example, F2 is configured as /2, but although there are examples and
documentation in the books to utilize the function keys in application
programs, no matter what I do, the keys are not enabled.

I have even used the Adis and Keycf (sp) facilities, which change the
configurations for the keyboard, to enable the function keys but nothing seems
to work; even compiling and running the example programs which use the
function keys as examples do not enable the keys. (I was sure to return the
configurations to their original settings!)

Also, we do a lot of accounting numerical data entry meaning I need to find a
way to enable the numeric pad - these keys are also dead.

Any suggestions?
```

```
Since you are telnet'ting you are probably emulating a vt100 terminal (you
should confirm your terminal type).
Make sure the terminal your PC telnet software is emulating matches the
setting of the unix $TERM env. var.
If you still can't get your F-keys to work, ask your system's administrator
to check the standard terminfo AND mf terminfo files to see if they match
the codes being output by your terminal emulator.

You may not be able to get your numeric keypad working, it depends on your
keyboard, pc telnet program and terminfo entries. (I'm sure it could be
done using a 101 keyboard and ms-kermit, and very likely could be done
(perhaps more easily) with many professional emulators, of course YMMV).)
```

```
Also, if you're getting "corrupted" characters (ie, the trailing part of the
fkey sequence gets put in to ACCEPT's etc) then try setting the COBKEYTIMEOUT
environment variable. This controls the rate at which the RTS will timeout
after getting an escape (x"1b") - the rest of the sequence may come slowly,
and the RTS will think you just pressed the escape key.
```
