That depends on the cobol compiler you use.

In Acucobol this is just an ACCEPT FROM ENVIRONMENT .
-------
In MF Cobol this is a little bit difficult. At First you have to declare
a cobol variable to set the and a variable to hold the value
to get or set in to the environment variable. This is done by defining
the following in SPECIAL-NAMES.
ENVIRONMENT-VALUE IS ENV-VAL
ENVIRONMENT-NAME IS ENV-NAME

To get an environment variable:
DISPLAY UPON ENV-NAME.
ACCEPT FROM ENV-VAL
To set a new value:
DISPLAY UPON ENV-VAL.

If you use .gnt modules don't forget to CALL "MFEXTMAP.DLE" initially.

which in turn depends on which OS you are running on and/or which COBOL
implementation you are using.

If your compiler supports the X/Open COBOL spec then you can use:
DISPLAY env-name UPON ENVIRONMENT-NAME
ACCEPT env-value FROM ENVIRONMENT-VALUE

If you are using an X/Open (XPG-4) compliant compiler, then look
at the ACCEPT from ENVIRONMENT-NAME and ACCEPT from
ENVIRONMENT-VARIABLE syntax. If you are using a COBOL compiler
that does NOT support X/Open syntax, you will need to use
whatever (non-portable) solution that they provide - possibly
even a CALL to that *other* language that starts with a "C" (but
stops theres too).
