The capability requested by Peter is largely controlled by the server 
application, not the client.

Word registers itself as "Multiple use" server when it starts running. 
This means that each client, when asking for the server via the standard 
OLE API(s) to create server objects (CoCreateInstance(Ex)), gets its own 
version of the server started for it (at least for local servers).

(Had Word registered itself as a "Single use" server, each client would 
connect to the same running version.)

Another API (CoGetInstanceFromFile) does allow you to connect to 
some existing servers when specifying a file (although in Word's case, it 
doesn't actually load the file!). The current Micro Focus Net Express 3.0 
product doesn't (yet) support this use directly via its run-time OLE 
support.

Steve.
