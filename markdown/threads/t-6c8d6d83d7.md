You must have mighty old reference books, since this is part of the ISO/ANSI
'85 standard.
 The notation you are referring to is called "reference modification" in the
COBOL world ("substringing" in most other programming languages). The first
number represents the starting location within the named field; the second
number represents the number of bytes to participate. The colon must always be
specified to distinguish reference modification from subscripts or indexes.

>MOVE FIELD2(2:1) TO FIELD1(3:2)

This is called "Reference modification" and is part of the '85 standard.

Reference modification implies that no data conversion takes place, but
otherwise acts similar to an elementary alphanumeric move.

FIELD2(2:1) designates the part of FIELD2 that starts in the second byte
position for length of 1.

FIELD1(3:2) designates the third byte of FIELD1 for length of 2, i.e., the
third and fourth bytes.

The above statement moves 'B' and a space over the '37'. (The space comes from
the destination of the MOVE being longer than the source, so COBOL
automatically treats this as if the sending field was extended by spaces.) The
resulting FIELD1 value is '12B 8'.

One thing not mentioned above is that one can leave off the length and it would
imply the rest of the field, e.g., MOVE FIELD2(2:1) TO FIELD1(3:) would imply
the third and subsequent bytes of FIELD1 will be modifed, e.g., as if MOVE
FIELD2(2:1) TO FIELD1(3:3) were coded, or MOVE FIELD2(2:1) TO FIELD1(3:1) MOVE
SPACES TO FIELD1(4:2).

In my AcuCobol-85 manual it's referred to as 'reference modification' .
I quote from my manual (sorry Acu :)  Reference Modification is a method of
dynamically redefining a data item at execution to refer to just part of it

General format

data-name ( leftmost-position : [length] )

Syntax rules:

1) both 'leftmost-position' and 'length' are arithmetic expressions. 2)
unless otherwise specified reference modification is allowed anywhere a data
item of the class alphanumeric is permitted. 3) 'Data-name' may be qualified
or subscripted.  Reference modification is done after both qualification and
subscripting.

This feature is a real boon - means some of the most murky pieces of cobol
can be written in a clear concise fashion - no more redefines, redefines,
redefines

for your example data  FIELD1  FIELD2  BEFORE 12378  ABC  AFTER 12B 8  ABC
note the space 'cos target field longer than source and Cobol space fills
then.

No, the syntax is fieldname(start:length) or fieldname(start:).

Like group moves, moves using reference modification are treated like
alphanumeric elementary moves but without data conversion. That means that if
the destination is longer than the source, the bytes in the destination after
the bytes received from the source will be filled with spaces.
