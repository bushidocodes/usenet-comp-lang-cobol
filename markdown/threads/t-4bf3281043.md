```
The COBOL II and COBOL for MVS and VM programmer's guides both claim
that use of the AWO (apply write only) option will optimize the size of
output blocks for variable length non-VSAM files.  Since this was an
underneath the covers way of applying the APPLY WRITE-ONLY clause (only
WRITE x FROM y used never WRITE x) to a variable length record file,
this would make a difference.  A WRITE x statement used code to build
the record in the output file buffer so a block had to be written any
time the maximum size record could not be added to the block.

The reason for the question is that I have the feeling that IBM COBOL no
longer builds the output record in the output file buffer but in effect
alway uses PUT MOVE rather than PUT LOCATE.  Could someone on this
newsgroup confirm or disprove my theory.  This is in response to a
question from a fellow worker.

Clark Morris, CFM Technical Programming Services morrisc@nbnet.nb.ca
```

I can't specifically address your PUT MOVE vs PUT LOCATE issue - but I do
know that the "newer" COBOLs are *supposed* to work whether the DCB is above
or below the line.  Hence it wouldn't surprise me if all records are "built"
somewhere other than in the buffer.

My guess is that this is *NOT* a "documented" interface - so my "usually
reliable sources" won't be able to give me (you) an official answer.  But I
can ask "just in case".
