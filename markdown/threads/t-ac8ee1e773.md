```
Just as a follow-up on another thread

1) Having NOT INVALID KEY coded *before* an INVALID KEY was illegal in the '85 
Standard.  The '02 Standard lets them go in either order

2) START REVERSED was non-conforming in the '85 Standard.  In fact, IBM *still* 
doesn't support "reversed" READ or START.

3) All the "NO" phrases were added in the '85 Standard.  Therefore, if working 
with "old" COBOL code (originally developed for the '74 - or even '68 
Standards), you won't see these constructs (or CONTINUE).
```
