Ken,

Ignoring that runtime fee problem <G>. This sort of touches on that other topic
Components. Call it what you like. Appointment List, Booking List and the
Calendar feature (class TimePicker in N/E)..

Whether you are talking, beauty/hairdressing  salons, dentists, medics,
chiropractors, veterinarians, lawyers, accounting firms...... the list is
almost endless - they all have a requirement for a 'booking application'.
Agreed there are differences, but essentially the requirement is the same - the
ability to flip backwards and forwards through an appointment register and book
slices of time to potential 'customers'.

I call my own family GP a computer freak - I've told him so <G>. The only pen I
ever see in his hand is a light pen !

Problem is - not sure it will jive with an API approach - I think it would have
to be OO-based sending and receiving messages, allowing COBOL to interface with
other languages. Perhaps you should take a look at the N/E Java classes, and
the White Papers on Java use. There are some very sophisticated components out
there already, which also link to DBs - but can't see any harm in one being
available in COBOL.

PS: Picky, picky. It's not exactly shareware if you want to price it - but I
understand what you are saying

Jimmy, Calgary AB

As this is comp.lang.cobol I'll throw out one of my wishlist items
that I never seem to have enough time to write, but would happily pay
<$50.00 if it was available.

A Cobol source pretty-printer and/or html-izer that is not dialect
specific.  Possibly with a plug-in extension architecture for user
written munging.

Of the two features I would consider an html-izer to be more valuable.
 I often like to post example code as part of system documentation,
but it can be painful and time consuming to convert a program into
good html (variables as hotlinks to defs, procedure names as hotlinks,
etc).  More often than not I end up wrapping each line in a set of
&ltcode&gt&lt/code&gt tags saying "I'll get back to it later".

I often wish I had a pretty-printer that I could use as a preprocessor
for version compares to make it easier to identify what is really
different and what just got reformatted/uppercased/name-changed/etc.

Regards,

Joe Zitzelberger


BTW -- I know about CobolTransformer and CobClean.  But CobClean
charges $350, FAICT, to add flower boxen and remove sequence numbers. 
CobolTransformer would easily do what I want for the price of a small
vehicle.
