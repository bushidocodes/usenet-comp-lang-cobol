I guess it is more a question of change of environment, when I did the move
some years ago, we did not alter much. we had not used a configuration file
before, we added one and here are the entries we introduced:

screen          size=25,80 window=999,999
white-fill      1
color-model     5
3d-lines        1
background-intensity 1
translate-to-ansi    1
quit-mode       11

The most important one if you are non-english is the translate-to-ansi
variable, which lets you keep your data in ascii, while the screen / printer
io goes in ansi. Screen, white-fill, color-model, 3d-lines,
background-intensity, quit-mode are windows specific stuff.

And that's basically it, there may be a few quirks that I don't remember,
but those are probably not important.

Markus Triska wrote in message <379CA72B.46AD5C42@netway.at>...
>By now, I've found out that also the ALT + Function-key - combinations are
>disabled for programs.

That is the general setup, you may however alter this programmatically,
like:

           SET     ENVIRONMENT      "KEYSTROKE" TO
                                    "Exception=1  ^[".

Which terminates an accept if ESCAPE key was pressed. To obtain the same
behaviour with for instance Alt F1, you would:

           SET     ENVIRONMENT      "KEYSTROKE" TO
                                    "Exception=101  U1".

Which terminates an accept when ALT F1 is pressed and sets escape key to
101.

>When will version 4.3 be released?

I have no idea, they told about it at a conference I was attending.
