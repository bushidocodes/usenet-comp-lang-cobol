Hi Frank,
You don't say which product you have (Workbench 3.4, 4.0 or NetExpress 2.0),
but Panels2 documentation was shipped as on-line files with Workbench in
PANELSV2.HLP and PANELSV2.HNF (note the V in those).

Pf-Display-Bitmap doesn't take a .BMP file directly, but a resource name.
This resource name may be associated with a .BMP or a bitmap compiled into a
resource DLL or the EXE itself. It is passed as the third parameter (after
P2-Parameter-Block and P2P-Parameter-Block).

To associate the resource name with the bitmap, you should either use a side
file (see Pf-Initialize and Pf-Process-Side-File) or you can request Panels2
to dynamically add the resource by using Pf-Add-Resource.

See the docs for the syntax to these calls.

Hope this helps,
Steve.
(MF Development)
