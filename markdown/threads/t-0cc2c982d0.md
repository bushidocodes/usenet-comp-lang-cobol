obvious.

Not long ago I found a statement like this in a program I debugged:

IF A NOT = 1 OR 2 OR 3

and a little investigation revealed that the programmer obviously had
meant:

IF ( A NOT = 1 ) AND ( A NOT = 2 ) AND ( A NOT = 3 )

which could have been written in the form that amuses (some of) us:

IF A NOT = 1 AND 2 AND 3

The funniest thing is that the program had been in production for several
years and nobody had noticed anything unusual.

The most elegant solution, as one poster demonstrated, is to use an 88
level symbol having all the relevant values.  But guess what the compiler
produces from that!  In plain English:

IF C-BR-CODE = 1 GOTO FALSE.
IF C-BR-CODE = 2 GOTO FALSE.
IF C-BR-CODE = 3 GOTO FALSE.
...
true clause.
FALSE.

which is correct, of course, and spelled out in clear, like some posters
preferred.
