```
On an IBM RS/6000 under AIX 3.5 with MF-COBOL 3.2, we get
a strange effect when the system is heavily loaded: COBOL
programs start failing to open files correctly. It looks
like there is a resource problem. Strangely, this only
affects COBOL programs: other UNIX commands like cat work
perfectly. The problem is sporadic, so that after a few
attempts, the program succesfully opens the file.

Has anyone experienced such problems, and if so, did you
find a solution?
```

```
The resource problem could be to do with file/record locks. By default
MF COBOL gets an exclusive lock on a file (unless opened for input) and
if there are lost of programs running the system might hit the locked
file limit. I'm not sure how record locks are handled. I don't know
AIX particularly, but I remember some Unix systems have a per user or
per process limit and a total system limit. But then I would expect the
file status error description to mention this like status error 9/213 or
219.
```
