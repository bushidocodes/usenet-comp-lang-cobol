- **From:** erik.melen@seb.se (Erik Melen)

- **Message-ID:** `<3a757899.24939861@news.sebank.se>`

```

We at SEB IT Partner has found a fairly simple way to create DFSORT
symbol
definitions from COBOL copies.  The COBOL copy is wrapped in a dummy
program, compiled, and processed by a simple REXX program that eats
the compiler listing and creates a SYMNAMES file.  You can then use
the COBOL names (except with underscores instead of hyphens) in
DFSORT and ICETOOL control statements.

As most of the records in the world have a description in the
form of a COBOL COPY , this significantly increases the usability of
DFSORT symbols.

You can find the details, including our freeware REXX program at:

http://taz.vv.sebank.se/cgi-bin/pts3/pow/kampanjer/it_partner/

If you're not familiar with DFSORT symbols, you can read more about
them
at:

http://www.storage.ibm.com/software/sort/srtmasym.htm

Erik Melen
SEB IT Partner
Development Environments
```
