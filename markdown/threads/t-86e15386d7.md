COBOL will always DISPLAY the full Working-Storage length of a dataname.

If you wish to truncate the name, you have to find the end of the meaningful
text yourself and only disply that portion of the dataname.

Since this looks like homework, all you get is a hint: Look up the INSPECT
verb with the TALLYING option; then look up "reference modification."

Alternately, make up the reply in another W-S item using STRING and use

Or, you can do it manually like this (not guaranteed to compile
precisely as written, but it's a neat trick):

PERFORM WITH TEST AFTER
  VARYING CHAR-COUNTER FROM LENGTH OF NAME-FIELD BY -1
  UNTIL NAME-FIELD (CHAR-COUNTER:1) NOT = SPACE AND LOW-VALUE
      OR CHAR-COUNTER < 2
CONTINUE
END-PERFORM

Now, CHAR-COUNTER is the number of characters in the field before the
trailing space(s).  Even if the text string contains  embedded spaces.


You need to deal with the special case of an empty field...

So, then you just:

DISPLAY NAME-FIELD (1:CHAR-COUNTER)

which will display only the number of characters you want to display.

You will probably want to limit test (e.g., verify that it works with
strings of length zero, 1, and field-length) because lots of code
looks good but breaks when you force it to minimum/maximum values.

There are many ways to manipulate strings in COBOL; you can use
INSPECT, STRING, UNSTRING verbs; all of them have their place, and
unfortunately, too many COBOL guys are unfamiliar with them.  

'Course you could say that about a bunch of verbs, I guess.
