I know that at various release/maintenance levels of IBM's mainframe
compiler, it has NOT worked.  This was the origin of the original ANSI
interpretation request that cleared up the difference between "results are
undefined" and "the returned value is undefined".  (See substantive change
potentially effecting documented in the ANSI/ISO Amendment 2 - Corrections
Amendment)

Please, PLEASE, don't suggest that people rely on any value that HAPPENS to
be returned for this situation - unless YOUR compiler does document what
value will be returned.   I know that Micro Focus and IBM (both Workstation
and Mainframe) do NOT define what value will be returned - and if it happens
to be zero today, there is certainly no guarantee that it will be tomorrow.

NOTE PARTICULARLY - that the results are STILL undefined in the draft of the
next Standard - although an exception could be used for testing for this
situation.
