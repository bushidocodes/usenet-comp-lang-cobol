```
jnhemley@earthlink.net wrote:
> Thanks for getting back to me but what is the problem ? Is the Borland
> object incompatible with Winsock ?

Borland C/C++ is incompatible with MicroFocus Cobol 3.x. See MicroFocus
3.2 User Guide where it states that Borland can only be mixed with
Borland when you use C to call a Cobol DLL.

You can't call a Borland DLL because some versions of Borland C/C++
DLLs scribble back over some memory locations in the caller which is OK
when the caller is also Borland C/C++.
```
