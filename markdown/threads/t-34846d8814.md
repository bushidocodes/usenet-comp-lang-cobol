```
I thought I'd add some comments to the interesting linked-list -vs-
table debate currently being held in a neighboring thread.  This is a
small, simple example of a vector, or resizeable array, that starts
quite small, but can grow quite large.

Comments and flames are welcome.

Regards,

Joe Zitzelberger



 Identification Division.
 Program-ID. Cobvectr.
*---------------------------------------------------------------*
*
* Simple vector example program done in Cobol.
*
* This should help fan the flames of the linked-list -vs- table
* discussion going on under the 'Best Practices' thread.
*
* This requires a user to actually have some method of getting
* and freeing storage.  I have used only one of many, please
* route all comments that start with "We allocate storage by..."
* to /dev/null.
*
* This also requires a smarter LENGTH OF operator.
*
*---------------------------------------------------------------*
 Environment Division.
 Data Division.
 Working-Storage Section.

 01 Cobvectr-storage.
   05 Getmain-Pgm                     Pic X(8)  value 'CEEGTST'.
   05 Getmain-Address                 Pointer   value null.
   05 Getmain-Length           Binary Pic S9(8) value 0.

   05 Freemain-Pgm                    Pic X(8)  value 'CEEFRST'.
   05 Freemain-Address                Pointer   value null.

   05 Heap-Zero                Binary Pic S9(8) value 0.

   05 I                        Binary Pic S9(8) value 0.

 Linkage Section.

* Max size map for memory move.
 01 Wk-Buff                           Pic X(799928).

* Map of vector.
 01 Simple-Vector.
   05 SVA-Prefix.
     10 SVA-Max-Size           Binary Pic S9(8).
     10 SVA-Increment          Binary Pic S9(8).
     10 SVA-Size               Binary Pic S9(8).
   05 SVA-Items.
     10 SVA-Item               occurs 0 to 9999
                               depending on SVA-Size
                               Pic X(80).

 Procedure Division.

     Perform Init-Vector

/----Load up lines until the user enters "DONE"
     Perform with test after
       until 'DONE' = Function Upper-Case(SVA-Item (SVA-Size))

       If SVA-Size = SVA-Max-Size
           Perform Grow-Vector
       End-If
       Add 1 to SVA-Size
       Accept SVA-Item (SVA-Size) from SYSIN

     End-Perform


/----Display size and dump lines
     Display 'Final Vector Size = ' SVA-Size
     Perform varying I from 1 by 1
       until I > SVA-Size

       Display I ',' SVA-Item (I)

     End-Perform

     Goback.


/----Set up to initially hold 5 items and grow by 10 at a time.
 Init-Vector.

     Compute
         Getmain-Length = Length of SVA-Prefix +
                         (Length of SVA-Item (1) * 5)
     End-Compute
     Perform Do-Getmain
     Set Address of Simple-Vector to Getmain-Address

     Move 0 to SVA-Size
     Move 5 to SVA-Max-Size
     Move 10 to SVA-Increment

     Exit.


/----Grow vector by increment size
 Grow-Vector.

*----Get area with room for another 10 items
     Compute
         Getmain-Length = Length of SVA-Prefix +
                (Length of SVA-Item (1)
                      * ( SVA-Size + SVA-Increment ))
     End-Compute
     Compute
         SVA-Max-Size = SVA-Max-Size + SVA-Increment
     End-Compute

*----Move old array to new, larger, area
     Perform Do-Getmain
     Set address of Wk-Buff to Getmain-Address
     Move Simple-Vector to Wk-Buff (1:Length of Simple-Vector)

*----Free old area
     Set Freemain-Address to Address of Simple-Vector
     Perform Do-Freemain
     Set address of Simple-Vector to address of Wk-Buff
     Exit.


/----Requires length (input) returns address (output)
 Do-Getmain.

     Set Getmain-Address to null
     Call Getmain-Pgm using
         Heap-Zero
         Getmain-Length
         Getmain-Address
         Omitted
     End-Call
     Exit.


/----Assumes address only required.
 Do-Freemain.
     Call Freemain-Pgm using
         Freemain-Address
         Omitted
     End-CAll
     Set Freemain-Address to null
     Exit.

 End Program Cobvectr.
```
