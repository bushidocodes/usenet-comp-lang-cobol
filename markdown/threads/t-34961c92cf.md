Cobol uses temporary storage areas for intermediate results that are
to be stored or used in later calculations.  Thus your statement


    multiply var-02 of calc by var-01 of calc
                               var-02 of calc
                               var-03 of calc

is compiled into language like:

    MOVE var-02 of calc to temp
    MULTIPLY temp by var-01 of calc
    MULTIPLY temp by var-02 of calc
    MULTIPLY temp by var-03 of calc

I think this is done because the way the statement is coded, you are
really saying "Take the contents of var-02 an mulitply it into var-01,
var-02, var-03".

The results you were originally expecting (IE 4 4 8) would be
equivilent to 3 Seperate multiply statements:
   MULTIPLY var-02 by var-01
   MULTIPLY var-02 by var-02
   MULTIPLY var-02 by var-03
This would do the 4 4 8 result.  

Hope this makes some sense,

Paul

It is useful to learn the order of actions with OR, AND, COMPUTE, etc,
and then never, ever have code dependent upon them.  Redundant
parenthesis work wonders at clarifying and speeding up understanding of
your code and intentions.

e.g.  COMPUTE X = ((5 * 3) - 2)
gives the same result as
      COMPUTE X = 5 * 3 - 2
except that your intention is obvious, and the maintenance programmer
who decides to multiply the whole thing by Y doesn't do the following:
     COMPUTE X = 5 * 3 - 2 * Y

This is trivial, but a bunch of long data names over several lines can
make the mistake easier to happen.

How do people here like to type the following code?:

IF   (
      ( A = B)
OR    ( C = D)

     )

I like to be able to comment out a line if possible, which is not as
easy if you do:

IF  (( A = B) OR

Combine this with some knowledge of how computers work.  On a simple
level, computers store data in general storage AND computers do math
on data in registers.

Applying these rules, yields the following:
	Values are stored in registers from general storage
            Calculations are done
            Values are returned to general storage

Your example has 4 variables in general storage to deal with, so that,
   The value (2) in storage location assigned to variable B would be
moved to some register (call it R1)  
   The value (2) in storage location assigned to variable A would be
moved to some register (call it R2)
   The value (2) in storage location assigned to variable B would be
moved to some register (call it R3)
   The value (2) in storage location assigned to variable C would be
moved to some register (call it R4)

Then the math is done:
   R2 is multipled  by R1 (on some computers this is really a series
of ADDs)
   R3 is multipled by R1
   R4 is multipled by R1

Then the results are returned to general storage:
   R2 is moved to variable A
   R3 is moved to variable B
   R4 is moved to variable C

Exactly as written.

This is a not a compiler issue, this is the way it was defined to be.
