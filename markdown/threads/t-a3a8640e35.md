```
Subject: Cobol Direction / Help

We have an accounting package originally written in RM Cobol that we run
through MFocus 32bit.  We had some printing & misc problems when running
their 16bit version so we decided to get them fixed with the 32 bit version
release.  Since installing that version in about 50 customer sites (there's
no turning back), we've had nothing but problems.  The biggest being speed
on a multi-user network and keyboard problems.

The keyboard problems we've had are as follows

1.  If your mouse is moving when you  run exe with mouse and all programs
wait for you to hit a key on the keyboard.  The same program runs fine
through dos or if you do not move your mouse while executing.

2.  Caps Lock - if caps lock is on number keys work like the shift is down .
This seems to be only a Windows 98 problem.

Bottom line is that we've about had it with Microfocus.  We've installed the
latest update and that fixed nothing.  They are also telling us that the
4.38 release is the last.

Can anyone recommend a direction we should go in?  Should we shift gears
completely and go with Fujitsu, IBM, or Accucobol?  Are any of those system
compatible with RM?

We're in a big bind and any help would be greatly appreciated.

Scott
Ucs, Inc.
```

```
Imagine for instance, that Microsoft Windows NT Server is being installed
from Microsoft with fileserver capabilites off! What is that for a server?
What they do is to optimize the software as an application server/
workstation to gain optimum performance for C/S server apps. Traditional
filesystems suffer significantly of this, particularly when it comes to
record lock.

If you are mixing Microsoft and Novell its even worse, due to configuration
issues and bugs.

If this is the cause, Microfocus is not the one to blame, other than that
they should have helped you by providing how-to's. I don't know Microfocus
products i particular though.

Generally speaking, I would guess what your target environment is, is
something which will rule the recommendation.

I have been using Acucobol for several years and are well satisfied with
them. I will particularly honor their support which is quick and efficient.
I came from Microsoft, so I have no experience with it, but Acucobol is said
to be RM compatible, you may look up their web (www.acucobol.com) to get
details about that. Acucorp are starting their annual conferences very soon
now, you may want to do as we did back then when we decided, we participated
a conference to see what it was, meet people who used it and gain
impressions. We went for it, and were never sorry about it.

Someone else here told that Fujitsu had a problem being ported between
different versions of Windows, regarding IBM Visual Cobol, I have no idea.

I guess it will not be an easy choice, there is a number of good products
out there, I whish you good luck with your decision. I am certain it will be
wise, whichever you choose.
```
