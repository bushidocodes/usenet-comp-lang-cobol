```
Good Morning,

I am trying to write data to certain memory locations with NetExpress Version
2.0. The purpose of this is to open a cash drawer which is controlled by a
application-specific intergrated circuit (ASIC).  This is on a IBM SureOne
Point of Sale Terminal running Win95.  

If I use the following "debug" commands the drawer opens:

Debug
o 240 28 (Enter)
o 241 06 (Enter)
o 240 29 (Enter)
o 241 01 (Enter)
q (Enter)

The following is my COBOL program:

WORKING-STORAGE SECTION.

   01  ASIC-INDEX-PORT-POINTER             POINTER.
   01  ASIC-DATA-PORT-POINTER              POINTER.

LINKAGE SECTION.

   01  ASIC-INDEX-PORT-MEMORY              PIC X(01).
   01  ASIC-DATA-PORT-MEMORY               PIC X(01).

PROCEDURE DIVISION.

       SET ASIC-INDEX-PORT-POINTER TO NULL.
       SET ASIC-DATA-PORT-POINTER TO NULL.

       SET ASIC-INDEX-PORT-POINTER UP BY h"240".
       SET ASIC-DATA-PORT-POINTER UP BY h"241".

       SET ADDRESS OF ASIC-INDEX-PORT-MEMORY TO
                                    ASIC-INDEX-PORT-POINTER.
         
       SET ADDRESS OF ASIC-DATA-PORT-MEMORY TO
                                    ASIC-DATA-PORT-POINTER.

       MOVE h"28" TO ASIC-INDEX-PORT-MEMORY.
       MOVE h"06" TO ASIC-DATA-PORT-MEMORY.

       MOVE h"29" TO ASIC-INDEX-PORT-MEMORY.
       MOVE h"01" TO ASIC-DATA-PORT-MEMORY.

       EXIT PROGRAM.
       STOP RUN.

i compile my program to an EXE and try to run it on the terminal.  The cash
drawer does not open and I get the following error:
     Execution error : file"
     error code: 114, pc=0, call=1, seg=0
     114 error message text not found (signal 11)

Would anyone have any comments on what the problem might be ?

Thanks,
Bob Hennessey
```
