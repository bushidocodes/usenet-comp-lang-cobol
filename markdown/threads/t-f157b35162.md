Thanks to all who are trying to help on this project! I'll try to provide
more information.  The Version 5 install actually sets the "path" and "lib"
options in the autoexec.bat file for you. I already removed all references
to c:\fsc\...whatever. The Powercobol build process works fine as long as
you don't add any addition link options.
In the Powercobol build process, you right click on the Module name, do a
properties, click on the link tab, then enter any additional modules that
you might need for external references.  As long as this field is blank, it
works OK. When I change the lib reference from the version 4 project to the
version 5 lib path, I get the error after the .rc file is created. I
remember with the regular Fujitsu Cobol, you had to set the program to be
"Main", but I didn't need to do that with Powercobol, at least in version 4.
If I get time, I'm going to create a new project with external lib
references to see if I get the same results. I'm also looking for any .ini
files that might be involved.

Hi all!  Problem solved.  I just knew it had to be something simple that I
wasn't considering.  Actually, all of you were correct in a way, but
Frederico gave me the real clue.  I went to the Microsoft support site
regarding spaces in a path statement and they are legal as long as they are
in quotes.  The Fujitsu install, puts quotes on the path statements in the
autoexec.bat file, but if you do a PATH command from a DOS prompt, you don't
see them.

So I put quotes on the link module, i.e. "C:\Program Files\Fujitsu
COBOL\COBOL\F3BICBLR.LIB" and it linked fine.  I suppose I should have known
better, but I didn't.  I thought I would pass this along, in case anyone
else runs into this.  Now I need to E-mail Fujitsu with a humble pie
attachment.

Thanks again, everyone.

Denny
