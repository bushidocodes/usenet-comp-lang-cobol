I'm using MF 4.0.32 Cobol. I want to use their printing routines for
Windows since it makes it easier to print within Windows than getting
into the API calls directly. I'm getting an RunTimeSystem 114 error -
"Attempted to access item beyond bounds of memory (signal 11)" on the
PC_WIN_OPEN_PRINTER call. The program is as follows:

procedure division WinApi32 using by value instance
by value PreviousInstance
by reference CommandLine
by value CommandShow.
main.
perform Connect thru
ConnectExit
stop run.

Connect.
Display "At PC_WIN_SET_PDEFAULT".
call "PC_WIN_SET_PDEFAULT"
using by reference "Lexmark Optra S 1650" & x"00"
returning status-code
end-call.
If status-code = 0
display "SET_PDEFAULT Successful"
else
display "status-code after PDEFAULT = "status-code
Stop "OK to continue".
Display "at PC_WIN_OPEN_PRINTER".
call "PC_WIN_OPEN_PRINTER"
using by reference op-printer-handle
returning status-code
end-call.
If status-code = 0
display "OPEN_PRINTER Successful"
else
Display "status-code after PC_WIN_OPEN_PRINTER = "status-code
Stop "OK TO CONTINUE".
Display "At PC_WIN_SET_PFONT"
call "PC_WIN_SET_PFONT"
using by reference op-printer-handle
by reference "Courier New" & x"00"
by value 8 size 2
by value 0 size 2
returning status-code
end-call.
display "after PC_WIN_SET_PFRONT status-code = "status-code
Display "At PC_WIN_PRINT_FILE_EXT"
call "PC_WIN_PRINT_FILE_EXT"
using by reference "c:\dougset.txt" &x"00"
by reference "My Config.sys" &x"00"
by value 3 size 2
returning status-code
end-call.
call "PC_WIN_CLOSE_PRINTER"
using by reference op-printer-handle
returning status-code.
display "status-code after PC_WIN_CLOSE_PRINTER = "status-code.
ConnectExit.
Exit.

I've got the displays in place for debugging. I've also run it with
Animator V2. The PC_WIN_SET_PDEFAULT works and returns a status-value
of zero (success). The PC_WIN_OPEN_PRINTER fails with or without the
PDEFAULT routine. The odd thing is that the printer icon does show up
in the system tray and I can select it and tell it to show active
printers which it does. And it's for the printer I specified in the
calls. I'm running on Windows NT 4.0 with 64-meg of memory and even
with nothing else running but this program it fails - doesn't seem to
be a memory limitation. I also tried it on a Win95 workstation with
16-megs and it behaves exactly the same. Any ideas?

Thanks in advance,
Doug Blaze

Hello all,
I finally figured it out! I had the printer handle variable in the
Linkage section instead of the Working-Storage section. Once I moved
it all works well. One bit of warning: If you specify an invalid
printer name in the PC_WIN_SET_PDEFAULT call, it will return a good
status-value (mispell the printer name as known by Windows). However,
the PC_WIN_OPEN_PRINTER done after the PDEFAULT call will return RTS
114!

Thanks again,
Doug
