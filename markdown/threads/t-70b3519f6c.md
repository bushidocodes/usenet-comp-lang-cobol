### Internal representation of some Cobol data ?

- **From:** "John H. Lindsay" <eil@aracnet.net>
- **Date:** 1999-06-21T00:00:00
- **Newsgroups:** comp.lang.cobol
- **Message-ID:** `<376EA228.CA10FBA5@aracnet.net>`

```
I'm looking for detailed information to the byte and bit level
of COBOL pictured numeric data (only S, 9 and V picture
characters) and packed decimal data on various hardware types
and under various compilers, as might be needed by an assembly
language programmer having to handle such data in files.  Where
it does exist, packed decimal data is ordinarily available as one
of the COMPUTATIONAL or COMP-n USAGEs.  The internal
format of edited data or of other forms of COMP- data such as
machine floating point or integer (fixed point) binary is not of
immediate interest to me.  The meaning of the various forms of
COMPUTATIONAL or COMP-n data varies greatly from
machine to machine and from compiler to compiler even on the
same machine.

I'm already familiar with the above for the I.B.M. mainframe
computers and the COBOLs that run on them.

Any information or pointers that anyone can give would be
greatly appreciated.  I'm trying to do a run-time library for a
SNOBOL4 (Spitbol) system to enable it to handle such data; I
can guess at a spectrum of possibilities, but I'd like to do a
thorough job here.  SNOBOL4 is a very powerful string
processor, character pattern matching and data structures
language.
```
