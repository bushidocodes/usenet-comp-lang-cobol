Jim: In order to answer this question intelligently, more information
is required. Do you have the source for the programs which are creating
the XX.dat files? If so, look at the SELECT statement for the file(s)
and tell me what the ORGANIZATION IS clause says (possibilities are
ORGANIZATION IS SEQUENTIAL, RELATIVE, INDEXED or LINE SEQUENTIAL). In
addition, you need to look at the record definition for the records
being written to see if there is any BINARY, COMP, or COMP-n data in the
record which would appear as garbage in the output file.

If you don't have the record layouts for the output files, I'm afraid
you're probably out of luck though.

Dennis

A little more information is needed here. When you COBOL in DOS is
that PC DOS or mainframe DOS? If it is mainframe DOS, then you can
use COBOL via the select statement to create as ASCII file from an
EBCIDIC file. If it is PC DOS, then unless the program is doing some
compression or creating binary data, the file should already be in
ASCII format.
