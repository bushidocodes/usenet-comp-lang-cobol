80 seems awfully long...  
In any case, you declare a variable DATE-FIELD of 80 chars.  
Then you redefine it as DATE-ARRAY, an array of 80 1-byte fields. 
Move the input date to DATE-FIELD
Then you can 
(A) set the index to 80 
     see if the 80th byte of DATE-ARRAY is a space.
     if it is, stop
     if it isn't, decrement the index by 1
     etc.
(B) inspect the DATE-FIELD looking for the first occurrence of
two spaces.  
If the input date is coming from a human, this may not
always work.

First, find out the business rules for what you want.   For instance:

     Whether or not your parsing will be case sensitive  FUNCTION UPPER-CASE
will help if not.
      Whether or not you want to allow multiple spaces, commas, etc.

Second, create your test plan.

Third, start your design and coding:

If you don't have functions, do a table translate on each character to get
everything upper case for your test.  If you need case sensitive matching,
you will have more work to do.

Then you can separate your words, either with an unstring, or looping through
the characters.

Then you might want to check out the first word against a table of months.

Then check to see if the third word is numeric and within your valid range.

The most complicated check will be the second word.  Split it between numeric
and non numeric parts.   Look at the last digit of the numeric part, and see
if the alphabetic part matches 0 & th; 1 & st; 2 & nd etc.    Then look at
the number and see if it is within the range allowed for the month.  Do you
know how to check for leap year?

      *    Left Justify
      *
      *        This subroutine left justifies a string by deleting
      *    leading spaces, underlines and binary zeros.
      *        Any underlines, binary zeros or blanks imbedded in
      *    the string are also deleted.

       environment division.
       configuration section.


       input-output section.
       file-control.

      /
       data division.
       working-storage section.



      *local-storage section.

       01  input-subscript                   pic 9(4) comp.
       01  output-subscript                  pic 9(4) comp.

       linkage section.

       01  input-data.
          05  input-datax                    pic x(01)
                                             occurs 9999 times.
       01  input-data-length                 pic 9(8) comp.

       procedure division  using  input-data,
                                 input-data-length.

          if  input-data-length  not >  zero
              go to   0010-exit.

          move  +1  to  input-subscript.
          move  +1  to  output-subscript.

       0010-loop.
          if  input-subscript  >  input-data-length
