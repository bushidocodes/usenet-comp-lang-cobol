Your example has me a  little confused.  Assuming what you really have is

  Identification Division.
  Program-ID.  Pgm1.
  Data Division.
  Working-Storage Section.
 *  Not File Section.
 01  AAA Global Pic  XXX.
 Procedure Division.
   Whatever.
        Call "pgm2"
        Stop Run.
 Identification Division.
 Program-ID.  Pgm2.
 Data Division.
 Working-Storage Section.
  01 BBB.
     05  AAA Pic XXX.
  Procedure Division.
   Still-Whatever.
       Display "I want to reference the OUTSIDE program AAA"
            AAA
       Exit Program.
  End Program Pgm2.
  End Program Pgm1.

Then, the bad news is that it can't be done.  Assuming that a data-name is
"unique" in the nested program and is an 01 level (outside the File
Section - where you can qualify by the file-name) in the containing program,
then the nested program can NEVER reference the containing program's name.

Some "easy" work-arounds would be to change the 01 structure in the outside
program to

 01  AAA.
       05   I-can-be-referenced Pic XXX.

In which the name is NOT found in the nested program.  Another, is to change
the name in the nested program.

My assumption is that there are other reasons why you want the two names to
be identical - in which case, I don't see how to solve this.

Can't be done.  AAA is ambiguous, and the first one cannot be made unique
via qualification.  This can even happen with a field that can be
qualified.

 FD THE-FILE.
 01  THE-RECORD.
      05  AAA Pic  X.
      05  BBB.
           07 AAA Pic X.

Even if you type AAA OF THE-RECORD OF THE-FILE that's not unique, so there
is no way to access the first AAA.
