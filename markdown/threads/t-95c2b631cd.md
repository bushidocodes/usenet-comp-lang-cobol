Yes, you can suppress it. Try the following code:

accept ws-variable with no-echo.

OR

accept ws-variable with secure.

Both should not display the characters typed into the accept
statement. Instead spaces will be displayed and the cursor will
display and move as each character is input.

This is for Microsoft/Microfocus cobol version 3.XX and is
described in the manual under "Screen Section". However, I've
successfully used it on fields which NOT screen section.

It depends on what COBOL compiler you're using, but if you're using
a non-ANSI ACCEPT (ie, ACCEPT AT or some such extension), then there
is usually an option you can supply to the ACCEPT to achieve this. In
Micro Focus COBOL, it would be something like:

ACCEPT x AT 1010 WITH NO-ECHO.

"NO-ECHO" and "SECURE" are synonymous.
