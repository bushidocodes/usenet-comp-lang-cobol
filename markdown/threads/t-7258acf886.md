Larry,

The MULTIPLY statement refers to TABLE-ENTRY but that is a group item.
Another problem might be that you use quotes in the VALUE clauses for
numeric items. Even if your compiler allows that, others don't, so
it's a bad habit.
I propose the following definition for your table:
01 TABLE-DEFINITION.
05 PIC 9V99 VALUE 1.14.
05 PIC 9V99 VALUE 1.77.
05 PIC 9V99 VALUE 1.92.

01 TABLE-REDEFINITION REDEFINES TABLE-DEFINITION.
05 TABLE-ENTRY OCCURS 3 TIMES.

'Table-Entry' is a group field, a struct in C. It is defined to
contain 3 sub-fields, and there are 3 occurances of the group
giving a total of 9 sun-fields. This is not what you intended.

I would do it as:

01 Rate-Table.
03 PIC 9V99 VALUE 1.14.
03 PIC 9V99 VALUE 1,77.
03 PIC 9V99 VALUE 1.92.
01 Rate-Table-Rdf REDEFINES Rate-Table.
03 Rate PIC 9V99 OCCURS 3.

MULTIPLY Salary-Base BY Rate(Code) GIVING New-Salary.

Note first that "1.14" is a string and not a numeric value.
This table create a struct of 3 elements with values. Then
this is then 'union'ed to a struct containing an array of rates.

It is sooo easy. ;-)

Larry,

In the multiply statement you refer to TABLE-ENTRY but that is a group
item with three elements. Which element should Cobol use for the
computation? Another problem I see is that you use quotes for the
values of numeric items. Even if your compiler lets you it is a bad
habit, because not all compilers allow that.

01 TABLE-DEFINITION.
05 PIC 9V99 VALUE 1.14.
05 PIC 9V99 VALUE 1.77.
05 PIC 9V99 VALUE 1.92.
01 TABLE-REDEFINITION REDEFINES TABLE-DEFINITION.
