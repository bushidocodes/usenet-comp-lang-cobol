I believe that Mr. Moore is literally referring to the Tiny COBOL
(a.k.a. GNU COBOL) project.  It is an "open source" project in which
anyone who wants to work on the compiler can obtain the source code
for the compiler (just like Linux).  If it is to be distributed in the
same manner as Linux, it will be a free  user-supported COBOL
compiler.

The Tiny COBOL compiler is currently only available for the Linux
environment.

The Tiny COBOL project (1) is not part of, and never was part of the GNU
COBOL project (2), also know as the COBOL2C project (3).

FYI, the GNU COBOL project is inactive, with any future development is
unlikely. 

However there is project called COBOL for GCC (4). It's objective is to
develop an ANSI/ISO COBOL 85 (20xx ?) compliant COBOL compiler, which
will be integrated with the GNU Compiler Collection (GCC).
