```
> John

As long as he's considering the inspect verb, he might as well do what
programmers would do and use the STRING verb.  But since his teacher
told him to use tables, I would go ahead and use tables.

Move each name from one table of characters to a combined name which is
also redefined as a table of characters.

Since first and last names can have spaces in them, start any searching
from the end.

    move first-name to combined-name.
    perform find-last-space varying combined-index from 60 by - 1 
       until combined-name-char (combined-index) > space.
    add 2 to combined-index
    perform move-middle-name varying middle-index from 1 by 1
       until middle-index > 20.   (or whatever, I don't remember your
sizes)
    perform find-last-space varying combined-index from 60 by - 1 
       until combined-name-char (combined-index) > space.
    add 2 to combined-index
    perform move-last-name varying middle-index from 1 by 1
       until middle-index > 20. 

find-last-space.
    exit.
move-middle-name.
    move middle-name-char (middle-index)  to combined-name-char
(combined-index)
    add 1                                 to combined-index.
move-last-name.
    move last-name-char (middle-index)    to combined-name-char
(combined-index)
    add 1                                 to combined-index.
    

A lot more work than using the STRING command, but it illustrates table
manipulation
```

```
Several others have talked about the STRING approach - which is probably the
way this particular problem would be handled in the "real world". (On the
other hand, in the real world, people would be worried about names that had
imbedded spaces).  ANYWAY - if you teacher WANTS you to use tables, my guess
is that they want something like the following.  (Sorry other readers, I
can't think of any way to show this - without "doing the homework")

  NOTE: I have not compiled and testing this.  Sorry if I have made some
obvious (or not obvious) mistake - but I think the logic is valid.
  NOTE2:  If your teacher REALLY wants you to use 2 tables - rather than
one, then you probably need to "build" your output name in a table rather
than reference modification.

01 EMP-NAME.
   05 LAST-NAME       PIC X(20).
   05 FIRST-NAME      PIC X(10).
   05 MIDINIT-NAME    PIC X(01).

01 EMP-OUTPUT.
    05  EMP-FULL-NAME    PIC X(31).

01 Only-One-Table-Needed.
      05   Subscript    Pic  S9(04).
      05   Letter-Table occurs 20 times.
             10   Each-Letter   Pic X.

   ....

*  Get/Read each Record
 Perform Each-Input-Name
*  Finish program
    .
Each-Input-Name.
   Move FIRST-NAME to Letter-Table
  Perform
        Varying Subscript from 1 by 1
         Until Subscript > 20
              or Each-Letter (subscript) = Space
                Continue
  End-Perform
  Move First-name (1: Subscript) to Emp-Output    *>  This uses reference
modification
  If First-Name (Subscript:1) = Space
      Move  MIDINIT-NAME  to Emp-Output (Subscript + 1 : )
      Move LAST-NAME to Emp-Output (Subscript + 3 :)
   Else   *>  for a 20 character first name
       Move MIDINIT-NAME  to Emp-Output (Subscript + 2: )
       Move LAST-NAME to Emp-Output (Subscript + 4 :)
   End-If
    .
```
