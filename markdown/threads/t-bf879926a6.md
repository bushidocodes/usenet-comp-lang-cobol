It has been so long since the "final resolution" on this issue, that I had
to look this up.  I think there is still some "confusion" related to
Working-Storage and Local-Storage for object (in the final draft).

1) You *may* define Working-Storage in an "instance definition" (this is the
data division "at the top" of the object instance definition - AFTER the
factory definition and before any methods).  Such data is "by definition"
GLOBAL and available to ALL methods of that class.  Such data is
"individual" for each instance (each gets its own copy). Such data
"persists" for the life of the object instance.

2) Local-Storage may exist inside a method definition.  Such data does NOT
persist for the life of the object instance.  However, if you "recurse" into
the same method, it is "pushed" and "popped" with the recursion).  (The same
is true of Local-Storage in PROGRAM definitions that use recursion - even if
you never "touch" OO.)

    ***

The only thing "missing" is a type of data that is local to a specific
method and that persists for the life of the object instance.
