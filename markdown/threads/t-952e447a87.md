I'd be just a bit careful with seemingly "dead" - i.e., never executed -
code.  I have on occasion come across code which has been deliberately
isolated because it was anticipated that it might be needed some other
time.  Now if the person that did it was conscientious s/he put an
explanation to that effect - which of course usually doesn't happen.  In
other cases, I have found that it was decided to leave the code in a
program because (documentation practices being so abominable although of
course it wasn't stated that way) there was no other way of being sure
of retaining it otherwise: the theory being that if all else is lost
we'll still have he source code.  (Although followers of this ng will be
