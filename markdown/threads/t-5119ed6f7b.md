This is highly compiler- and platform-dependent. The following works on
IBM CoBOL Set 1.1 for AIX 4.2.

LINKAGE SECTION.
01 LS-ARGC PIC S9(9) COMP.
01 LS-ARGP USAGE IS POINTER.
01 LS-ENVP USAGE IS POINTER.

* These are essentially DSECTs so their lengths are meaningless
01 LS-ARGV PIC X(1024).
01 LS-ENVV PIC X(1024).

PROCEDURE DIVISION USING
BY VALUE LS-ARGC
BY REFERENCE LS-ARGP
BY REFERENCE LS-ENVP.

0000-MAIN.
SET ADDRESS OF LS-ARGV TO LS-ARGP.
SET ADDRESS OF LS-ENVV TO LS-ENVP.

* SET ADDRESS is a nonstandard verb available in most UNIX CoBOLs. If
* your compiler doesn't have it, you can dereference pointers with a
* downcall.

* Here's an example of scanning LS-ARGV
MOVE 1 TO WS-NDX.

PERFORM
WITH TEST BEFORE
VARYING WS-SUB FROM ZERO BY 1 UNTIL WS-SUB = LS-ARGC

MOVE ZERO TO WS-LEN
INSPECT
LS-ARGV(WS-NDX:)
TALLYING WS-LEN
FOR CHARACTERS BEFORE INITIAL X'00'
DISPLAY 'ARGV[' WS-SUB '] "' LS-ARGV(WS-NDX:WS-LEN) '"'
ADD WS-LEN, 1 TO WS-NDX

END-PERFORM.

(You can scan through LS-ENVV in a similar way, except that the
termination is by double null instead of by count, but I strongly
recommend using getenv() instead).

Sample output:

$ demo a bc def ghij klmno
ARGV[000000000] "demo"
ARGV[000000001] "a"
ARGV[000000002] "bc"
ARGV[000000003] "def"
ARGV[000000004] "ghij"
ARGV[000000005] "klmno"
$
