A couple of years ago, we solved this kind of interprocess communication
making a DLL that the cobol application loaded. Once the DLL were loaded, we
used memory files to communicate between the cobol application and the
Delphi application that we had. It was seamless and efficient. The Delphi
application ruled the Cobol application by sending commands and data and
receiving calculated data in return. This was of course all handled locally
on one computer. However, I guess extending it to do RPC or DCOM should be
easy.

BTW: We used Acucobol 3.2.2 and Borland Delphi 3
