I've been reading about the OO Extensions for MVS COBOL. Before we dive in
to try it, we have some questions:

1. How difficult is it to set up the environment to support it?

>> LE and SOM are easy to install, but moving all of your applications
>> to LE can be easy or hard depending on your particular applications.

2. Where can I find a reference to set up the environment?

>> It depends if you mean SOM or LE or OS/390 or ?

3. I see that it uses SOM, but I seem to recall seeing somewhere that IBM
is not going to be supporting SOM in the near future. Is this true?

>> SOM is no longer strategic, but IBM COBOL will continue to run even
>> after SOM is no longer supported.  We may include SOM or a SOM
>> replacement with COBOL or change underlying support, but it will
>> not impact customer code that was written for SOM originally.

4. Can C++ classes written for MVS extend COBOL classes, and Vice-Versus?

>> Yes, as long as the C++ classes are written using SOM.  C++ supports
>> both SOM and non-SOM OO.

5. Is anyone out there using it?

>> Yes.

6. What are your experiences with it?

>> Any volunteers for this one?

7. Are there recommended standards for class names, method names, etc?

>> No, I don't think so.
