```
Folks,

I have a question about VS COBOL II copy and replacing. For example, if
I have the following copybook

** COPYBOOK TESTING
01 XX-TESTING.
05 XX-FIELD1 PIC X(01).
05 XX-FIELD2 PIC X(02).

Now, I want to use the keyword "copy" to include the copybook into my
program and replace all XX with AB, so I type my line in the
program as below

COPY TESTING REPLACING == XX == BY == AB ==.

As I will expected when I compile the program the copybook would be
included and looked like:

01 AB-TESTING.
05 AB-FIELD1 PIC X(01).
05 AB-FIELD2 PIC X(02).

However, it NEVER, XX still there. Is there anyway I can replace all XX
by AB by using VS COBOL II function?

Thanks.

Rgds,
Rodenia
```

```
Rodenia,

What you are trying to do is called "partial word replacing". You need to
alter your
copybook slightly to get it to work:

COPYTEST.CPY

01 (XX)-TESTING.
05 (XX)-FIELD1 PIC X(01).
05 (XX)-FIELD2 PIC X(02).

The COPY statement should read:

COPY "COPYTEST" REPLACING ==(XX)== BY ==AB==.

This will replace the (XX) with AB i.e. (XX)-FIELD1 becomes AB-FIELD1. The
parenthesis are delimiters and I believe others can be used. Brackets seem
to
be the most common however.

Hope this helps.
```

```
Copy Replacing *only* replaces text words and not parts of words. Change
your COPYLIB to

01 :XX-:TESTING.
05 :XX-:FIELD1 PIC X(01).
05 :XX-:FIELD2 PIC X(02).

and change your COPY statement to

COPY TESTING REPLACING ==:XX-:== BY ==:AB-:==.

and it will do what you want. HOWEVER, be aware that if some other program
has a COPY TESTING without any replacing phrase, this will now fail. There
is no way (until the next Standard) to get partial word replacing on text
that will compile cleanly both with and without a REPLACING phrase.
```

```
I don't have manuals handy here at home but from memory, I think the
copybook you are pulling in has to look something like this:

01 %XX%-TESTING.
05 %XX%-FIELD1 PIC X(01).
05 %XX%-FIELD2 PIC X(02).

Then the substitution will get done. The reason is Cobol only
substitutes complete words but the percent signs fool the compiler
into thinking XX is a word.
```
