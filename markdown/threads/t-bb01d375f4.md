```
For the '85 Standard (and any "conforming" implementation)

   Pic  999ppp
        is valid and is equivalent to
   Pic  999pppV

Similarly,
   Pic  ppp999
        is valid and is equivalent to
   Pic Vppp999

This "thread" has pointed out an UNINTENDED (minor) bug in the FCD (draft of
the next Standard) but everyone "agrees" that it is intended that these
"variations" will continue to be supported as long as COBOL supports P's in
PICTURE clauses.

The '85 Standard (along with the IBM documentation) EXPLICITLY states that
the "V" is redundant when it is at the very left or very right of a string
of P's.
```
