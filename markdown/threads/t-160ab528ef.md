Here's a subprogram that I use.  It converts a given character to its
2-character hexadecimal representation.  It's all standard COBOL, so
should be quite portable.

      IDENTIFICATION DIVISION.
      PROGRAM-ID. CHAR-TO-HEX.
      DATA DIVISION.
      WORKING-STORAGE SECTION.
      01  HEX-TABLE    PIC X(16)    VALUE "0123456789ABCDEF".
      01  CHARNUM      PIC 999      COMP.
      01  HEX1         PIC 99       COMP.
      01  HEX2         PIC 99       COMP.
      LINKAGE SECTION.
      01  CHAR         PIC X.
      01  HEX          PIC XX.
      PROCEDURE DIVISION USING CHAR, HEX.
      BEGIN.
          COMPUTE CHARNUM = FUNCTION ORD (CHAR) - 1
          DIVIDE CHARNUM BY 16 GIVING HEX1, REMAINDER HEX2
          MOVE HEX-TABLE (HEX1 + 1 : 1) TO HEX (1:1)
          MOVE HEX-TABLE (HEX2 + 1 : 1) TO HEX (2:1)
          EXIT PROGRAM.
      END PROGRAM CHAR-TO-HEX.

Walter

Hi. I just knocked this up in 5 minutes, so take it at face value (ie, test it
and if it appears to do what you want, feel free to use it at your own risk).

----- 8< ----- Cut Here -------------------------------------------------------
       DATA DIVISION.
       WORKING-STORAGE SECTION.
       01 Hex-Table VALUE "0123456789ABCDEF".
          03 Hex         PIC X OCCURS 16.
       01 Binary-Offset  PIC 9999 COMP.
       01 Text-Offset    PIC 9999 COMP.
       01 High-Nybble    PIC 99 COMP.
       01 Low-Nybble     PIC 99 COMP.
       LINKAGE SECTION.
       01 Binary-Length  PIC 9999 COMP.
       01 Binary-Buffer  PIC 99 COMP OCCURS 100.
       01 Text-Buffer    PIC X OCCURS 200.

       PROCEDURE DIVISION USING BY REFERENCE    Binary-Length
                                BY REFERENCE    Binary-Buffer
                                BY REFERENCE    Text-Buffer.
       Main SECTION.
       Main-Para.
           IF Binary-Length > 100
               EXIT PROGRAM RETURNING 1
           END-IF

           MOVE 1 TO Text-Offset
           PERFORM VARYING Binary-Offset FROM 1 BY 1
                   UNTIL Binary-Offset > Binary-Length
               DIVIDE 16 INTO Binary-Buffer(Binary-Offset)
                         GIVING    High-Nybble
                         REMAINDER Low-Nybble
               MOVE Hex(High-Nybble + 1) TO Text-Buffer(Text-Offset)
               MOVE Hex(Low-Nybble + 1) TO Text-Buffer(Text-Offset + 1)
               ADD 2 TO Text-Offset
           END-PERFORM
           EXIT PROGRAM RETURNING 0
           STOP RUN RETURNING 1.
----- 8< ----- Cut Here -------------------------------------------------------

And here's the quick test program :

----- 8< ----- Cut Here -------------------------------------------------------
       DATA DIVISION.
       WORKING-STORAGE SECTION.
       01 Text-Buffer    PIC X(8).
       01 Binary-Size    PIC 9999 COMP.
       PROCEDURE DIVISION.
       Main SECTION.
       Main-Para.
           MOVE 4 TO Binary-Size
           CALL "hexcvt" USING BY REFERENCE Binary-Size
                               BY REFERENCE x"0a0e0f82"
                               BY REFERENCE Text-Buffer

           EXHIBIT NAMED Text-Buffer
           STOP RUN.
----- 8< ----- Cut Here -------------------------------------------------------

You can make all sorts of improvements to this like passing the size BY VALUE
(allowing you to use literals and stuff), but I've no idea (1) what compiler
system you're using or (2) exactly what is/isn't in what standard (not being
a COBOL programmer myself, and being too busy to look it up right now).

However, I'm fairly sure that the conversion program is ANSI-85 compilant with
the exception of the RETURNING keywords and the possible exception of
lower-case usage and the expressions in the subscripts (but all of those are
very easily fixed). The test program definately contains MF extensions.

Cheers,
Kev.
