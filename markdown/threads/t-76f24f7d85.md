```
In article , Don Nelson wrote:
› 
› [...] However, why not use the intrinsic functions?  That is, I assume you are
› using an up-to-date COBOL 85 compiler (probably the wrong assumption since
› lots of you seem to be stuck with COBOL 74 or with COBOL 85 without functions
› (which were added to the standard in 1989)). [...]

When I get a CoBOL job, I have a small program that I type in to find out which
CoBOL features I can use there (and to learn the editor). I use FUNCTION
LENGTH() in this program to find out if they have the intrinsics.

About two years ago I got a contract at a ($500M) company which shall remain
nameless. My test program with FUNCTION LENGTH() compiled and executed, so I
used FUNCTION DATE-OF-INTEGER() and FUNCTION DAY-OF-INTEGER() in the
deliverable. I am always happy to use the intrinsics instead of having to type
in my old date routines again.

Well, that sucker compiled, but it wouldn't link! These guys had the compiler,
but they didn't have the intrinsics library. Some intrinsics, like FUNCTION
LENGTH(), are usually implemented inline, whereas other intrinsics, like
FUNCTION SIN(), are usually implemented externally, and on this machine FUNCTION
DAY-OF-INTEGER() was implemented externally.

I now use FUNCTION SIN() instead of FUNCTION LENGTH() to find out if intrinsics
are available.
```
