```
I want to print a list of names alphabetically, 3 columns in landscape.
(8.5 x 11). The easiest way is to print :-

A.........	B...........	C.........
D........	E..........	F.........
G.........	H...........

But I would prefer to print :-

A..........	D.............	G..........
B..........	E.............	H..........
C.......... 	F.............

and of course, page-change after x maximum lines.

I can come up with all sorts of convoluted solutions - like reading the
file once to get a record-count, then re-reading and giving each record
a table position.

As I'm using OO I could create a SortedCollection (alpha sequence), then
invoke :- 

	01 ls-size		pic x(4) comp-5.

	invoke os-SortedCollection "size" returning ls-size

then perhaps use a callback to calculate each record's position in a
table having determined from ls-size the (a, b) positioning.

There's got to be a simple way. Any neat suggestions ?

Jimmy, Calgary AB
```

```
Build a table of size maximum lines * 3 to hold all the info you want to
print.  After the table is filled with sorted information, divide by number
of entries in the table by 3 (last page may not be filled) to determine the
max number of lines on that page.  In a perform of your subscript from 1 to
max lines, move your table entry (subscript) to your left print area, table
entry ( maxlines + subscript) to the middle column, and table entry(maxlines
* 2 + subscript) to the right column.

HTH
```

```
Jimmy,

Are you running on a platform that supports the enhanced COBOL Report Writer
option?  If so, that should do what you want nicely.  Just GENERATE the
left-hand side and use SET COLUMN and SET LINE to do the middle and
right-hand big columns.  RW has the option to buffer the whole page till
you're ready to let
it print.

Another way is to read ahead one big column of lines and use the REPEATED 3
TIMES clause on your report group description, e.g.

(in Report Section)
  01  BIG-COLUMN  REPEATED 3 TIMES EVERY 40 COLS.
    03  LINE + 1   OCCURS 60 TIMES  DEPENDING ON W-COUNT  VARYING R-LNO.
      03  COL 1    PIC X(20)   SOURCE W-TABLE (R-LNO).

...
(in Proc Div)
     (read up to 60 records with count in W-COUNT)
     GENERATE BIG-COLUMN

I used an OCCURS DEPENDING ON to handles the unused lines at the end of the
last big column.  Unused big columns on the last page are automatically left
blank by the REPEATED clause.  I've missed out the Page Heading etc. from
the above scheme.

These features are described on www.spc-systems.com/refce/cobrwr.htm.
```
