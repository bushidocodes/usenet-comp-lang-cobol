The DMS Reengineering Toolkit is generalized compiler technology used
to automatate analysis, modification and translation of large-scale software
sources.
DMS provides parsing, tree-generation, tree-walking, attribute-evaluation,
source-language-level transformations, prettyprinting, and a lot more.
DMS has off the shelf parsers/prettyprinters for COBOL, C, C++, Java, XML,
SQL and many other legacy languages.

See http://www.semdesigns.com/Products/DMS/DMSToolkit.html.

It could be used to effect a reliable translation.  Usually, such
translations
are custom, because customers want to translate applications using
execution environment assumptions, so the problem isn't just translating
the source code; one also wants to translates the "assumptions" (i.e,
replace system calls for one kind of system with possibly different
calls for the new target system.

This also only makes sense if the translation scale is some tens of
thousands
of lines or more; otherwise, a hand translation is about as cost effective.
