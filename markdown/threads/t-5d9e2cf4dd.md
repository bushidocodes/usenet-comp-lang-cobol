Here's a thought:

Create a clean Virtual Machine image with Windows XP or whatever your 
favourite OS is.

(You download the Virtual Box software from Oracle, then create a new 
machine using your Windows install disk. Save the clean machine image as a 
snapshot; it is a mouse click.)

Download the full Edition of Net Express, which has a free trial for 30 
days, to the Virtual Machine.

Put your COBOL development  Project sources on a different machine, or 
organise them into a single directory on the VM so you can back it up.

Poddle away with NetExpress for 30 days.

After 30 days, backup your project sources to a different machine or 
external device.

Restore the clean machine from the snapshot.

Install NetExpress.

Restore your project sources.

Good to go for another 30 days...

Now, I'm not advocating this for people who are seriously developing in 
COBOL; you should bite the bullet and buy the compiler.  But for people who 
just want to dabble, or for hobby development, it is not a bad solution and 
you aren't limited to 2000 lines etc.

I have need of MF COBOL (NetExpress) VERY occasionally when I need to check 
out how something works in that environment. I do not develop solutions in 
that environment. (The fact that I would have to license the runtime is a 
showstopper for me and also the support for COM is not good enough to meet 
my requirements. (You cannot batch compile COM projects in NetExpress and 
every component you build HAS to have an NE Project so the methods can be 
registered.) The whole business of developing COM in NE, although viable, is 
ugly and cumbersome compared to Fujitsu where it is easy and simple. Having 
said that, the implementation of COBOL as a language is excellent in NE just 
as it is in NetCOBOL, but NetCOBOL has no runtime fee.)

My advice to people who want to play with COBOL would be as follows:

1. Use Open COBOL. (If it supported OO, I'd be using it exclusively...It's 
an excellent product.)

2. Learn C#. (It's free and you're a "programmer", right?) There are some 
really superb free videos to get you started and that was how I learned it. 
There may be little commercial advantage in learning C# (or any "desktop" 
programming language apart from scripting) as the world moves on to mashups 
and pre-packaged functionality, but you can do more with it and you learn a 
lot of other stuff along the way which means that your programming becomes 
richer and you can take on more challenging tasks. You can also get stuff 
working quicker because the tools are better.

I guess if you want to learn something with commercial advantage, CSS3, 
HTML5, Javascript, VBscript, and general Web Technology is the way to go.
