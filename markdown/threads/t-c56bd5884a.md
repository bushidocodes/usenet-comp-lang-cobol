```
We are unable to use an MF COBOL program running on one UNIX machine
to access an indexed data file on another UNIX machine even though the
directory containing the data file is mounted and made shareable
through NFS. We can access and execute the program through NFS, but
when the program tries to access the data file (which resides in the
same shared mount point), we get a status 043, File Information
Missing for Indexed File. When I look at the data directory
afterwards, I find that the .dat file is gone; it's been deleted
somehow. I restore the .dat and .idx files, run the program again,
abort again, and again the .dat file is gone. We would sure like to
hear some suggestions about where to find the solution to this
problem.

The code is being developed on a SUN SS2 under SOLARIS 2.3 without
NFS. The program runs fine on this platform. The problem of the
missing ,dat file occurs on SUN SS20s that are running SOLARIS 2.3
with NFS.


The vfstab of the local machine contains:

vulcan:/home10/manper /home10/manper nfs - yes rw,bg,soft

The dfstab of the data server contains:

share -F nfs -d "MANPER Directory" /home/manper


The program works fine if the data files are copied to another
directory on the local machine, but we want to support multi-user data
access, so replicating data is not the answer for us.

Does anyone have any idea about what the cause of our problem is and
how we can correct it?


Lee Shafer
lsh··.@b85··f.mil
```

```
In my experience using any indexed file across NFS (or RFS or
whatever) is not good. Assuming you get past problems like
lock daemon hangups, support for long filenames etc, the
performance is likely to be poor as each COBOL i/o verb causes
numerous system calls over the network. Go for MF Fileshare
instead if it is available.
```
