Most of the time I get this error, is because I have a window running the
program and am trying to compile it on another. The OS locks the executable
whenever it is loaded, thefore preventing the compiler from deleting it. The
solution, of course, is to terminate the runing unit before compiling.
Sometimes it isn't enough; for instance, if runing the program caused a
serious error on KERNEL32.DLL, the runtime system is unable to unload the
program from memory and the only solution is to reboot.
