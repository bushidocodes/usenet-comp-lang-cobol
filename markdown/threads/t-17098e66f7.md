Besides which, even the fastest Pentium chip will operate no more
quickly than the fastest disk drive will feed it data, if you're
talking about a truly data-intensive operation (such as graphics).

I used Fujitsu COBOL 3.0 / 4.0 to process identical 75MB data files in
a read-flat-file/write-indexed-file benchmark on a 75MHz Pentium and a
350MHz Pentium.  Less than a 10% gain on the faster machine.  And,
since we're talking about sub-2 minute operation, it hardly seems
worthwhile to migrate upwards.  (And since this is a COBOL group, the
point seems germaine.)

So, wrt this group and the results I have gotten, I think we can
safely conclude the 400+MHz Pentia (? plural ;-) are really not
exceedingly useful in a traditional data processing application.
