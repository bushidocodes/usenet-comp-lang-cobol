```
> What does a "copy-overlay" do in ISPF?

The source line is marked with a "C" in the ISPF prefix area.  The
target line is marked with an "O" in the prefix area.  A range of
target lines is marked with "OO" in the beginning and ending line.

For every matching character position in the source and target
line(s), the source character will be copied to the matching position
in the target line if the character is blank in the target line:

For example, BEFORE executing:

C..... 05  MY-DATA        PIC X(04) VALUE SPACES.
            
OO.... 05  FILLER    
...... 05  FILLER 
OO.... 05  NAME

and AFTER the copy-overlay is executed:

...... 05  MY-DATA        PIC X(04) VALUE SPACES.
            
...... 05  FILLER         PIC X(04) VALUE SPACES.    
...... 05  FILLER         PIC X(04) VALUE SPACES. 
...... 05  NAMETA         PIC X(04) VALUE SPACES.

It can be a very useful command in the right situation.

I hope that helps!
```

```
It could be done without sed & awk - as I suggested before, check out Vim at
www.vim.org - it has a built-in macro language that could probably do
something like this pretty easily - and it's still "vi" (but improved!).

If you did write a sed or awk (or perl) script to do it, you can execute it
through vi without leaving vi.

Taking your example:

C  foo-
OO     bar1
       bar2
OO     bar3

If you had a shell script in your path named "coverlay" that executed a sed
command (or awk or whatever) that read stdin and wrote stdout, from within
vi, move the cursor to the first line (C foo-) and type:

4!!coverlay

where 4 = number of lines to send to stdin of coverlay.  vi will then
replace those 4 lines with the stdout of coverlay.
```
