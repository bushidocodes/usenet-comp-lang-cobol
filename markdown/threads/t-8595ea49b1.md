```
There are entry level jobs in COBOL, but you will have to look at the
bigger companies.  I got into mainframe technology in the Y2K good ole
days, when they would sit a warm body in a chair and call you a
programmer.  We went to a cobol "boot camp" and 3 months later I was a
consultant doing exciting stuff like changing the mm/dd/yy to
mm/dd/ccyy.  It was so glamourous!
If you really want to work Old Skool, check industries like Healthcare,
Insurance, Banking, etc.  Call their HR departments or email and ask if
they have a trainee program (we have one here at our company if you
like to work for peanuts) but once you get some actual work experience
you can move on from there.  You could talk to consulting companies if
you can stand it - but usually they aren't going to talk to you unless
you already have a job, and don't want another, from my experience.
Cobol programming is not at all "hot" or "glamourous" but it is stable
and in my humble opinion is not going away any time soon.  I've worked
at companies where they tried to get off the mainframe, spent millions
of $ and found out it doesn't work when they are overbudget and way
past their deadline.  For large amounts of data processing, it is hard
to beat the processing power of a mainframe (maybe a unix system but
probably not Windows)
You also want to make sure you have some other skills besides mainframe
apps.  To be able to use existing legacy code and either convert it to
languages that can be used on a windows or unix server is a plus.  Then
you can market yourself as a "legacy migration consultant" and make big
bucks promising to get off the mainframe, which probably won't work
anyway.  In Healthcare knowing stuff like Hipaa transactions, edi, aix,
scripting helps too.
People can use cobol in combination with client server technology.
They just don't want to use it, or like it, because Cobol isn't cool.
But, if you have experience in it, and are pretty young, then you may
find that it is useful after all the older dudes start to retire, which
is starting now, and the new college grads don't know it because
everyone thinks it will be obsolete.  Guess what - in corporate
america, especially in the backwaters outside of the west coast like in
the south, they are VERY slow to adopting new technologies because it
takes money, skills, and some understanding on management's part (which
is a rarity).
```
