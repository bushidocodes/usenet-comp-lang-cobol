I think that VAX COBOL has two different source modes. Perhaps
you are in the "free form" source input mode, wherein the
compiler thinks that the asterisk is a arithmetic operator.
This "free form" (don't know what the VAX COBOL name for it might be)
would have different commenting conventions.

The format you are using is American National Standard CoBOL Reference Format,
or ANSI format for short. It's probably a good idea to learn this format at
least once, since it is the only universal format.

VAX CoBOL defaults to a vendor-defined extension that DEC calls terminal format.
It has several advantages over ANSI format, but it is not, alas, universal, and
it is unwise to get too used to it.

If you say COBOL/ANSI instead of just COBOL, the compiler will expect your
source in ANSI format. The errors on the comments will go away, but you will
probably get dozens of other errors on typos that are acceptable in terminal
format but not in ANSI format. Persevere. As I said, it's probably a good idea
to learn this format at least once.

You will often find the VAX/VMS online help to be more useful than this
newsgroup. Start with HELP COBOL. Here's what you would get for HELP
COBOL/ANSI:


/ANSI_FORMAT

/ANSI_FORMAT
/NOANSI_FORMAT (D)

Indicates that the source program is in conventional ANSI format.
The compiler then expects 80-character card image records with
optional sequence numbers in character positions 1 through 6,
indicators in position 7, Area A beginning in position 8, Area B
beginning in position 12, and the identification area in
positions 73 through 80.

The default is /NOANSI_FORMAT; that is, the compiler assumes
that the source program is in DIGITAL's terminal format, where
Area A begins in position 1, Area B begins in position 5, and the
source program records do not have line numbers.
