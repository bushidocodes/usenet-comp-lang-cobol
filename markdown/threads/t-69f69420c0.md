```
>1) Do you call the subprogram once or for every file read?

## You call a subprogram every time you need it to do something.
It does its thing and returns control to the calling program. If its
thing is to read the entire file and return the number of records,
you call it once. If it is to read one record and return that record
to the calling program for further processing, you call the subprogram
for every record in the file.
The number of times a subprogram can get called varies from zero to
infinity.

>
>2) Can I move the new data from the subprogram into working-storage in
>the calling program?


## Yes. The calling program (generally) passes WORKING-STORAGE
data which, in turn, is defined in the LINKAGE SECTION of the called
program. From the LINKAGE SECTION in the called program, you may
move data to the local WORKING STORAGE section of the subprogram.


>I'm using an ibm-risc-6000 computer at school.


## For the questions you raised, the type of computer is (usually)
irrelevant.
```

```
I can't tell from this note, exactly who is saying what to whom, BUT,  it is
*always* legal to move data from the Linkage Section to Working-Storage
section in a sub-program - in an ANSI conforming program.

There *may* be issues issues for some programs that are not ANSI conforming
(i.e. that use extensions).  I can think so some RARE cases where
"addressability" to a linkage section item isn't available - such where a
SET ADDRESS statement hasn't been performed yet, or for CICS items that have
yet to be "set".  However, once addressability to such items has been
established, even in these "extension" applications, it is then legal
(valid) to move data from linkage section to working-storage.
```
