Though they say 'screen section' it is the whole MF ADIS module that
is not present.  You can use ANSI ACCEPT/DISPLAY but the whole ACCEPT
AT, FROM CRT, CRT-STATUS.

There are also no CBL_ or PC_ routines - though these are easily
replaced.

There are many minor differences, such as requiring an actual variable
to be defined for 'SELECT .. ASSIGN dynamic-name' which can be
implicit in MF.

There is no MF DE-EDIT"1", so I have had to change all these to
FUNCTION NUMVAL.

> (c) Creates both statically and dynamically linked executables
>               Yes.

Yes, that works fine, it creates .so.  It is case sensitive though, so
while I had all program names as lower case under MF I had to make
them upper case for Fujitsu.  Not really a problem though.
