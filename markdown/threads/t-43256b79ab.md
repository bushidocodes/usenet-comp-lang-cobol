In COBOL you have to identify the input and the output record descriptions in
the body of the program.  There isnt much automatic about anything in COBOL. 
The programmer is expected to know what the file looks like and decide what
the output looks like.  Usually the programmer uses a print chart and just
designs the output which is coded and tested. Often a file description is
made up and catalogged on the system so it can be copied into the program. 
There are probably some third party products that can read file descriptions
or program sourcecode and do what you want.

However, if all you want to do is move input to output that is pretty simple. 
Often for a programming assignment I use to make a simple program that moved
the input all as one text field to the printline and just write one line at a
time.  That way someone else can see what the input looked like for the
program.  This makes a good simple first program for a class.

You will have to figure it out on your own.  Even the crudest of COBOL books
have some examples.  Try a book like Learn COBOL in 24 Hours.
