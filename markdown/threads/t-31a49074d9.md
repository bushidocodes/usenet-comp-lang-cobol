Report on October 3-7 ANSI X3J4 COBOL Committee meeting
Report on October 24-28 ISO/IEC JTC1/SC22/WG4 COBOL Committee meeting

These two meetings were spent discussing proposals for the next revision
and the process for the next revision. Our present timetable requires
that all major proposals be complete by February 1995 (formerly August
1994). After that, we will start the ISO review cycles (first review in
March '95), culminating in a new standard in 1997 (or so we hope).
Anyone who wants to see any of these proposals or the draft standard can
contact me. None of them are on-line or in any generally accessible
electronic form, so a paper copy is all I can provide.

At the X3J4 meeting, the following proposals were passed by letter
ballot before or at this meeting: Free-form source, and standard
arithmetic. Both are major proposals.

The major topics/proposals we discussed were:

1. The new revision of the base document was distributed to the
Committee. This revision includes about 20 proposals that had passed
(all of them are fairly small and minor). We had a special editing
meeting the Sunday before the regular meeting. Several minor problems
in the document were found and some assignments for writing missing
parts and fixing parts that don't conform to the ISO rules were made.
There is a great deal of this kind of work that needs to be done.

2..The multi-octet character set proposal was passed and will be added
to the next version of the base document. This includes bit/boolean
items and boolean expressions as well as allowing multi-octet characters
for names and data.

3. We spent quite a bit of time on my exception processing proposal. A
paper came in from a WG4 member (in England) who wants to change it
radically. The committee rejected this attempt and I submitted a
rewrite to the committee for another letter ballot.

4. The Object-oriented COBOL proposal was discussed for two days (the
OO COBOL group met in parallel with the main group for the other three
days). Lots of votes were taken and so on. Most of this is minor
clean-up. The major parts of the proposal are more or less finalized.
Getting rid of unused objects was discussed at length and the SELECT
statement was removed. The Micro Focus book on OO has been revised and
is now available from Micro Focus. It follows the proposal closely (the
author is the editor of the proposal). The proposal was rewritten after
the meeting and submitted for another letter ballot.

5. Some minor topics: FIMS (Forms Interface Management System) has a
binding to COBOL undergoing ISO ballots - there appears to be no problem
with approval - their standard has been approved by ISO but is being
hung up in the US because one guy says they should use "choices" rather
than "menu"; a proposal on user-defined types (allows you to describe a
structure and use the name of it in other places to get the same
structure - also forces strong type checking) was hacked to death and
sent back for rewrite; and letter ballot comments on the proposal on
file sharing and record locking (it failed the letter ballot) were
discussed and it was sent back for rewrite. It should be available for
the January 1995 meeting.

The next X3J4 meeting is in La Jolla, California, on December 5-9. The
main topics will probably be object-oriented COBOL and common exception
processing (there will be a letter ballot on both closing before the
next meeting).

At the WG4 meeting, the following topics were discussed:

1. We passed a resolution that the Multi-octet Character Set amendment
will not be created. This is being done because MOCS will be in the
next standard and it will not be necessary to make an amendment.

2. One paper suggested drastic changes to the common exception
processing proposal, but the changes were rejected. The group voted in
favor of the approach that is now in the paper. There seemed to be some
sentiment for an in-line capability, so I will look into that.

3. There was a great deal of discussion on the base document and the
scheduling of ISO reviews.

4. The MOCS/bit/boolean proposal was presented and discussed. There is
still more to be done on this, but the format was there.

5. Free-format was discussed. It seemed to be well-accepted. The only
complaint was with in-line comments. Some felt they should be like Ada
and C++ with an opening and closing flag (like /* ... */) to allow them
to cross lines. Nothing concrete came out of these discussions.

6. Problems with zero-length data items were discussed. Very close
votes said to allow zero-length records in sequential files but not in
relative files. It is not clear what the final result will be.

7. Variable-length data items brought out lots of opinions. We
currently plan on adding only C style items (delimited by a perhaps a
zero byte). We also have PL/1 type strings on the list (a count is
used). The proposed method is an invisible length field with dynamic
data allocation for the strings. How they get to files is not clear.
Because of time constraints, we will concentrate on the C style.

The next meeting is in San Francisco, California, USA, on August 14-18,
1995. The meeting will be devoted to resolving comments on the first
ISO review of the base document.
