```
Thanks to feedback from the people who downloaded it, this tool has had some
significant fixes applied to it.

If you already downloaded it, but haven't used it for a while, try using it
again.

It SHOULD offer you an update, and, if you agree, it should automatically
update itself on your system.

If it doesn't, it might be because you have used it recently and the period
between update checks has not been completed.

Anyone who is using COBOL on a daily basis (whatever platform, including
Mainframe) and has to look at COBOL record layouts, and count bytes in
different usages, will probably find this tool helpful.

As long as you are able to cut/paste your record layout from a window, the
tool will analyze the fields you select on the system clipboard and give you
the length of them. (You can also configure it to look at the COPY book, if
the COPY book is available to your PC) The Tool is aware of REDEFINES and
OCCURS and provides correct offsets or positions for fields subject to
these.

The engine for this tool (COBSTRUCT) can be used as a component to provide
offset and length of any COBOL field in a COBOL buffer and this is very
useful when accessing COBOL records from other languages. The PRIMA DAL
generation for both C#/LINQ and COBOL/ESQL would not be possible without
this engine.

At the moment, a machine in Taiwan is using it the most... :-) (I have no
idea what they're doing with it or even who they are...) but there is a
sprinkling of machines around the world that have downloaded it.

You can get it from: http://primacomputing.co.nz/PRIMAMetro/newCOBDATA.aspx

Many thanks to all the people who wrote to me with suggestions and requests
(and bugs...); I have implemented everything requested.

Pete.
"I used to write COBOL...now I can do anything."
```
