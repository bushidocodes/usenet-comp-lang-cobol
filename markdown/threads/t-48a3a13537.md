COBOL does not lack a graphical interface any more than C++ does. I
was doing GUI forms and full graphical* windows in COBOL nearly a
quarter century ago (probably before you were born). It was done like
many other languages by calling procedures to draw forms or lines and
elements and processing events to determine what the user wants.


* This included graphical interactive views of container ships and the
layout of containers in the bays, calculation of hydrostatics, bending
and shearing and graphically showing these curves. There was also a
'helm view' with a perspective of the view ahead over the containers.
The 'helm point' could be set anywhere and then moved along a path
with each view recalculated. This could, on a fast enough machine (eg
a 486DX 100MHz), by starting a few hundred metres aft, a hundred or so
high and to one side, give a 'fly by'.

There are several libraries that cater for graphical UI for form
dialogs. For example SP2 from Flexus has been available for almost all
COBOL compilers since Windows 3.0.

http://www.flexus.com/main/index.php?option=com_content&view=article&id=48&Itemid=58

Several others were also available, mostly for specific vendor's
products. eg Fujitsu PowerCOBOL which I had copies of since Windows
3.1 (but didn't use), Microfocus Dialog , ScreenIO, MBP Visual COBOL,
Hi-Screen Pro, ...

With several COBOLs - MicroFocus, MicroSoft, Fujitsu, OpenCOBOL, etc -
CALLs can be made to C based libraries (just like other languages do)
or to Graphic APIs such as those for OS/2 or Windows (just like other
languages do).

In the example Container Ship system I used direct Windows API calls
for the graphics, SP2 for some dialogs and stuff like FarPoint
Toolbox.

Exactly like many other languages do graphics.
