You might find a disassembler, or a program that you can "see" the code
there in the object module.

Our cobol compiler has pmap option (I think) that shows the assembler
generated by the COBOL program.

Thru irritating trial and error you Might be able to figure out what
the original source code was. But, come to think of it, this task
is down right frightening.

For example:
if the cobol program has the following in it:

move 0 to ws-pork-a-rama-count.

The assembler (might be) something like this:

ZAP R10
MOV WS-PORK-A-RAMA-COUNT, R10

But it's worse than that...

ZAP R10
MOV SOME-OTHER-GARBLED-NAME, R10

Gosh, I don't envy you...

If you have the program, run some tests on it in a test environment and
treat it like a black box, and try finding documentation for it.

Without the source, you maybe better of remaking another program...
