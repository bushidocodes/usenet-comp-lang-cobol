```
I have a Report Writer problem I've been struggling with and would like
to ask for help. I have input data that includes employee numbers and
salaries. For IN-EMPLOYEE-NO = 999 I need to add asterisks in front of
the name and have an output that is a different PIC than the
others(employee numbers are not unique; there can be more than one
employee coded 999).  The PIC clause for all others employees is
$$$,$$9, whereas the PIC for the employees with number 999 is ***,**9. I
have the asterisks part done and thought I'd use that field as a control
break, but it causes the employee's salary to print three times at the
control break. Any ideas for something simpler and/or better? Thanks in
advance for any advice.
```

```
Pat:

There have been some improvements in this are of Report Writer (i.e. 
choices of contents) in the latest Standard and some implementations that 
anticipate it.  But I'll assume you're using a current standard, such as 
IBM OS/VS COBOL or Micro Focus or Fujitsu etc.  (This way would work 
on any platform.)  And I'll sketch the improved method later.

First of all, forget control breaks.  These cause extra lines (well, 
extra report groups) to be printed and you DON'T want extra lines here, 
right?

You have to define a Working-Storage area and use that as a Source. Then 
before you GENERATE the line(s) set it up in the format you want.  E.g.

 05  WS-REP-EMPLOYEE.
   07  WS-REP-EMP-999.
      09  WS-REP-EMP-SAL-AST      PIC ***,**9.
   07  WS-REP-EMP-OTH  REDEFINES WS-REP-RMP-999.
      09  WS-REP-EMP-SAL-OTH      PIC $$$,$$9.
...
(in Report Section)
 01  EMPLOYEE-LINES   TYPE DE.
  05  LINE ...
    07  COLUMN ...   PIC X(7)  SOURCE IS WS-REP-EMPLOYEE.
...
 (in Procedure Division)
   IF  IN-EMPLOYEE-NO = 999 
       MOVE IN-EMPLOYEE-SAL TO WS-REP-EMP-SAL-AST
   ELSE
       MOVE IN-EMPLOYEE-SAL TO WS-REP-EMP-SAL-OTH.
   GENERATE EMPLOYEE-LINES

You'll get the general idea even if the details vary a little.

With the new standard, you can put the condition in the Report Section, 
thus:
 01  EMPLOYEE-LINES   TYPE DE.
  05  LINE ...
    07  COLUMN ...   PIC ***,**9  SOURCE IS IN-EMPLOYEE-SAL
        PRESENT WHEN IN-EMPLOYEE-NO = 999.
    07  COLUMN ...   PIC $$$,$$9  SOURCE IS IN-EMPLOYEE-SAL
        PRESENT WHEN IN-EMPLOYEE-NO NOT = 999.

For further info and a whole gallery of examples, see
  www.spc-systems.com/refce/cobrwr.htm.
```

```
    IF IN-EMPLOYEE-NO NOT = 999

        999 only) DETAIL line.
```
