It sounds like its Level II COBOL. RUN.COM is the run time system. It will be
running either a .GNT file which is native i86 code or a .INT file which is
code compiled for our abstract machine. It's not interpreted per se.

› As a non cobol programmer I have no
› knowledge of cobol file formats/systems. Is there some little utility
› available to do this? Or where would I find information Micro Focus
› file formats. Any information would be greatly appreciated.

It depends on the file format and the type of corruption. We have a utility
called REBUILD funnily enough that can repair quite a few files but I'm not
sure if Level II had that. I presume you don't have a full copy of Level II
COBOL, just the run time needed for the application, so you probably wouldn't
have REBUILD anyway.

Without knowing the format of the files and their structure it can be
difficult to rebuild them. Indexed files are easy to tell as they have
.IDX files to match the data file.

Sorry I couldn't be more accurate with the Level II info. I don't have the
manuals anymore myself and the only disks I've got are 360K 5.25" disks. I
haven't had a 5.25" drive for about 5 years now so it's vague memories for the
most part.
