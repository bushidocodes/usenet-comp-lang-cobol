Speaking of MVS (or more accurately OS/390) which is the area that I know
best ...
I am NOT saying that you should try to convince your company to upgrade
compilers every 6 months or so - but COBOL/370 (the 1st compiler with
Intrinsic Function Support) came out in the early 90's and I *do* think that
any shop that is still creating (and probably any that are maintaining)
COBOL code should be on one of the LE-enabled compilers by now.

In fact, IBM's Operating System support "policy" is that they are now ONLY
supporting the OS (where LE now lives) for 2 years.  Therefore, I think that
any shop that is NOT looking at a policy that lets them upgrade "safely" in
the 2 to 3 year range is asking for trouble.

P.S.  My comment about talking to the vendor was NOT aimed at IBM - where
each new compiler IS more efficient than the previous one.  However, there
have been reports (in this NG) about some of the '85 Standard compilers
having SIGNIFICANT overhead over the '74 compilers - for the same programs.
This is something that I think users should complain about.
