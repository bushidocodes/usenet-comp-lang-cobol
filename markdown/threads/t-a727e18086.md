In answer to your newsgroup problem I can only speak from our own
experience.

NetExpress is a strange product when compared to the older MF
Workbench offerings, but it's one 
great strenth is in VB integration.

Linking from COBOL to VB is not the easiest thing to do, as there are
problems with transferring data
between the two - indeed we have written a program generator to write
most of the code required
(in both COBOL and VB) to pass complete COBOL FD definitions between
the two.

We would make our program generators available if there was sufficient
commercial interest,
but this seems lacking for now.

If I never use NetExpress for anything else (and that is a
possibility!) the fact that you can step
straight from the VB debugger into the COBOL animator makes the
product worth having - essential,
even, I would think.

I hope that this helps.
