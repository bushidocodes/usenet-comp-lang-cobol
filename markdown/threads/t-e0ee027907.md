Alain,

The reason that ODBC drivers for COBOL files seem relatively expensive is
that they are, generally, as complicated and difficult to build correctly as
a relational database management system and the market for them is somewhat
limited.

A COBOL VSAM ODBC "driver" is unlike the Intersolv (Merant) drivers in that
it must implement (in addition to the ODBC API) a SQL parser, execution plan
generator, query optimizer, and execution plan interpreter that is capable
of presenting a relational (i.e., normalized tables and row/column values)
view of unnormalized indexed files with, typically, highly overlaid record
structures.  Doing this correctly is very difficult.  Much more difficult
than interfacing to an existing relational database or "flat" text file.

The methods used to accomplish this by our products are, in fact, patented
in the U.S. and Britain.  In our case, (Relativity Data Server), we give
away the client ODBC driver, but the server engine which runs on the machine
with the COBOL data files costs something.

Regards,
John Bradley (Liant)
