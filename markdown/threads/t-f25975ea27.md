Chapter 21. Programming for a DB2 environment

In general, the coding for your COBOL program will be the same whether or
not
you want it to access a DB2 database. However, to retrieve, update, insert,
and
delete DB2 data and use other DB2 services, you must use SQL statements.

To communicate with DB2, you need to do the following:
  () Code any SQL statements you need, delimiting them with EXEC SQL and
END-EXEC statements.
  () Use the DB2 precompiler or compile with the SQL compiler option if
using DB2

James Martin's book "DB2 Concepts, Design. and Programming
is relatively short (325 pages), and covers DB2, SQL,
QMF (Query Management Facility) and DXT (Data Extract).

An oldie (1989) but goodie, it has a chapter on Embedded SQL statements,
with examples in COBOL, PL/I, C, FORTRAN and Assembler,
so I'd recommend that to get you up and running.

For a more comprehensive User Guide, I also have Don Chamberlin's
"Using the New DB2".  681 pages.  Practical tips.  100s of tested examples.
Dr Chamberlin is a co-inventor of the original SQL database language.

For a certification guide, I've got the 4th Edition of IBM's
"DB/2 Universal Database v7.1 for Unix, Linux, Windows and OS/2",
written by George Baklarz and Bill Wong, DB2 experts in IBM's Toronto Lab.
All 790 pages of it.  Comes with a CD-ROM containing a trial version
of DB/2 UDB v7.1 for Windows NT, Windows 2000 and Windows 95/98.

JCL is something you need to understand at better than beginner's level.
You wouldn't be the first programmer to leave of a few critical characters
and end up deleting every member of a production Generation Data Group,
or every member in a test copy library.  I've always used a book by
Gary De Ward Brown "System 370 Job Control Language".
The first edition came out in 1977 and sold over 250,000 copies
before the second edition came out in 1987.

* DB2 administration: populating tables with import and load utilities,

* Security: controlling data access through authentication, assigned
authorities,
   user/group privileges and more
* Database backup and recovery, step by step
* DB2 SQL, in depth: database objects, Data Definition Language (DDL),

into a host program.
