I have systems that use barcode readers on PCs, serial terminals and RF
guns.  These typically use a 'keyboard wedge' connection so that
scanning a barcode is exactly like keying in the number - the computer
or application can't tell the difference.

Usually the barcode scanner can be set to add a 'tab' or 'enter' at the
end of a scan so that some applications can be done entirely from the
scanner without having to do keyboard entry.  As an aid to this a page
of common and useful barcodes can be attached close by the machine so
that the operator can scan this rather than have to put down the reader
just to hit a couple of keys.
