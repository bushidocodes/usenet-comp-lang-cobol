There is a RECFM=FBA DCB parameter (fixed blocked ansi carriage control). 
I don't recall using it on input but only for output.  If your print file
is catalogued you should be able to read it in with just
DSN=filename,DISP=SHR.  Make sure your program record size in the FD
matches the LRECL=length that the file was created with.  This is probably
LRECL=133 (132 for the data and 1 for the carriage control character).

Hi George,

I'd define both i/p and o/p as recfm=fb and define the lrecl as the data len +
the print cntl char.

In the pgm I'd treat both files as "data" files not "report" files. By that I
mean don't use the "advancing" option, just move the cntl char you need into
the 1st position of the o/p record.

On the input side look for a "1" in the 1st pos of the record for TOF. Can't
remember the others, sorry (0,+,- seems to ring a bell).

Then, when you want to print it, define it as
recfm=fba.

IIRC there  are two ways to create a print file with ansi print control
characters.

The first way is to use 133 character records in your program and set the
print contol character yourself.  Int his case your JCL would have
RECFM=FB,LRECL=133......

The other way is to use 132 character records in your program and have
WRITE .... ADVANCING put in the print control.  You need the compiler
option ADV for this and your JCL would have RECFM=FBA,LRECL=133.........

First double check the print file to make certain that the print control
characters are indeed being created.  If they are, then reading into a 133
character record should give you the print control character in the first
position. Read each record and make any changes you want, then use method 1
above to write out the record.
