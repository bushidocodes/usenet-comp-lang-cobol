No, I include GUI as portable. All my stuff have a virtual screen which
includes
the standard GUI element (pull-downs, push buttons, pop-ups, etc). For each
platform, there is a peer component that handles the GUI from this virtual
screen. The peers may generate ANSI control sequences, HTML for a
Web-based interface, or a Windows interface, all depending on the platform.
This is somewhat similar to the Java AWT approach. Netscape's browser
used to run on many different platforms too (PC, Mac, Unix) using the
same approach.
The high-level control of the program is event-driven on all platforms,
and the actual API calls are part of code GENERATED for the platform
and thus invisible to the programmer.
So, I stand by the portability even in a GUI world.
