I often wonder at why people want to do this. Why convert from
a 35 year old language that is suitable for the type of programs
written in it to a 25 year old language that is not suitable for
those programs.

Presumably the idea is that 'C programs' are easier to maintain
based on the programs written as C course examples compared to
Cobol programs written on Business courses.

When code is converted from Cobol to C it is likely to still be a
Cobol program but written in the C syntax. It will not look like
a C program. If the Cobol has bad structure (for Cobol) the C will
have appalling structure (for C).

If you expect to get *identical* results then consider how you
will access the structured files, how you will get structs to
match records exactly, how C will cope with all the Cobol data
types and sizes, how it will do the dynamic loading.

Cobol syntax is a challange, especiall the dear olde little period. The
parsing rules differ by Division. Makes for great fun or strange
