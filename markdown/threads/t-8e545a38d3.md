PROCEDURE DIVISION.
DRIVER.
PERFORM START-UP
THRU START-UP-EXIT.
PERFORM WRAP-UP
THRU WRAP-UP-EXIT.

******************************************************************************************

START-UP.
DISPLAY ' HERE WE GO AGAIN!'.

DISPLAY ' JUST WORK...ONCE'.
DISPLAY ' PLEASSSSSE!'

*******************************************************************************************
* JUST A SMALL PRAYER TO THE COBOL GOD.

*******************************************************************************************
OPEN INPUT INVENT-FILE.
OPEN OUTPUT REPORT-FILE.
ACCEPT WS-DATE FROM DATE.
MOVE WS-D-YY TO TL-YY.
MOVE WS-D-MM TO TL-MM.
MOVE WS-D-DD TO TL-DD.
MOVE TITLE-LINE TO PRINT-LINE.
WRITE PRINT-LINE AFTER ADVANCING PAGE.

MOVE HEADING-1 TO PRINT-LINE.
WRITE PRINT-LINE
AFTER ADVANCING 2 LINES.
MOVE HEADING-2 TO PRINT-LINE.
WRITE PRINT-LINE
AFTER ADVANCING 1 LINES.
MOVE SPACE TO PRINT-LINE.
WRITE PRINT-LINE
AFTER ADVANCING 2 LINES.

*****************************************************
* This starts the looping of the individual records
* that were read by INVENT-FILE. There are only 15 files.
*****************************************************
PERFORM READ-INVENT
THRU READ-INVENT-EXIT
VARYING WS-COUNT FROM 1 BY 1
UNTIL WS-COUNT > 16.
* UNTIL WS-EOF = 'YES'.

START-UP-EXIT.
EXIT.

************************************************************************************************

READ-INVENT.
READ INVENT-FILE
AT END
MOVE 'YES' TO WS-EOF.
PERFORM MAIN-RTN
THRU MAIN-RTN-EXIT.

READ-INVENT-EXIT.
EXIT.

MAIN-RTN.
PERFORM 90-RES-CODE.
PERFORM 95-TIME-CODE.
PERFORM 100-TUITION-COST.
PERFORM 200-BOOK-COST.

MAIN-RTN-EXIT.
EXIT.


***************************************************************************************
* In 90-RES-CODE a number is read into IR-RES-CODE and then spits
out
* a response.

***************************************************************************************

90-RES-CODE.
IF IR-RES-CODE = 1 THEN
MOVE 'RES' TO WS-RES-CODE.
IF IR-RES-CODE = 2 THEN
MOVE 'NONRES' TO WS-RES-CODE.
IF IR-RES-CODE = 3 THEN
MOVE 'FOREIGN' TO WS-RES-CODE.


***************************************************************************************
* In 95-TIME-CODE a number is read into IR-TIME-CODE and then spits
out
* a response. Anything over 3 will default to FULL TIME.

***************************************************************************************
95-TIME-CODE.
IF IR-TIME-CODE = 0 THEN
MOVE '1/4 TIME' TO WS-TIME-CODE.
IF IR-TIME-CODE = 1 THEN
MOVE '1/2 TIME' TO WS-TIME-CODE.
IF IR-TIME-CODE = 2 THEN
MOVE '3/4 TIME' TO WS-TIME-CODE.
IF IR-TIME-CODE > 2 THEN
MOVE 'FULL TIME' TO WS-TIME-CODE.
IF IR-TIME-CODE > 3
MOVE 3 TO IR-TIME-CODE.

*****************************************************
* This takes the in coming record via IR-RES-CODE, IR-TIME-CODE
* which give the coordinates to the TIME-TABLE and saves the results
* to WS-TUITION-AMT. Then it adds it to the total cost.
*****************************************************

100-TUITION-COST.
MOVE TIME-TABLE (IR-RES-CODE, IR-TIME-CODE)
TO WS-TUITION-AMT.
ADD WS-TUITION-AMT TO WS-TUITION-COST-TOT.

*****************************************************
* This is where the problems start. In order to read and process
* the total book costs, each input record was done separately.
* Each was positioned to the table with the book costs and then
* put into working storage memory locations.
*****************************************************
200-BOOK-COST.
MOVE ZERO TO WS-BOOK-COST.
IF IR-BOOK-CODE-1 > 0
ADD BOOK-LOAD (IR-BOOK-CODE-1) TO WS-BOOK-COST.
* ADD WS-BOOK-COST TO WS-BOOK-COST-TOT.

IF IR-BOOK-CODE-2 > 0
ADD BOOK-LOAD (IR-BOOK-CODE-2) TO WS-BOOK-COST.
* ADD WS-BOOK-COST TO WS-BOOK-COST-TOT.

IF IR-BOOK-CODE-3 > 0
ADD BOOK-LOAD (IR-BOOK-CODE-3) TO WS-BOOK-COST.
* ADD WS-BOOK-COST TO WS-BOOK-AMT.

IF IR-BOOK-CODE-4 > 0
ADD BOOK-LOAD (IR-BOOK-CODE-4) TO WS-BOOK-COST.
* ADD WS-BOOK-COST TO WS-BOOK-AMT.


ADD 1 TO IR-TIME-CODE.

*****************************************************
* This is where the total bok costs are calculated.
* The WS-BOOK-COST is added to the running total.
*****************************************************
ADD WS-BOOK-COST TO WS-BOOK-COST-TOT.
MOVE WS-BOOK-COST-TOT TO PL-BOOK-COST-TOT.

*****************************************************
* This is populates BOOK-AMT-TBL. Each total of the
* book costs is loaded into the table BOOK-AMT-TBL.
*****************************************************
MOVE WS-BOOK-COST TO BOOK-AMT-TBL (WS-COUNT).

DISPLAY PL-BOOK-COST-TOT.
MOVE WS-BOOK-COST TO PL-BOOK-AMT.
MOVE WS-RES-CODE TO PL-RES-CODE.
MOVE WS-TIME-CODE TO PL-TIME-CODE.
MOVE IR-STU-NAME TO PL-STU-NAME.
MOVE WS-TUITION-AMT TO PL-TUITION-AMT.
WRITE PRINT-LINE
AFTER ADVANCING 1 LINE.
MOVE SPACE TO PRINT-LINE.
MOVE WS-LARGE TO PL-LARGE.
DISPLAY PL-LARGE.


WRAP-UP.
MOVE WS-BOOK-COST-TOT TO PL-BOOK-COST-TOT.
MOVE WS-TUITION-COST-TOT TO PL-TUITION-COST-TOT.
WRITE PRINT-LINE
AFTER ADVANCING 2 LINES.
MOVE TOTAL-LINE-1 TO PRINT-LINE.
WRITE PRINT-LINE
AFTER ADVANCING 2 LINES.

*****************************************************
* This does the actual looping that looks at the highest cost that
* one student paied for all of his books.
*****************************************************
MOVE BOOK-AMT-TBL (1) TO WS-LARGE.
PERFORM FIND-SMALLEST-LOOP
VARYING WS-SUB FROM 2 BY 1
UNTIL WS-SUB > 20.

****************************************************
* This is the subroutine that stores the first value and
* compares the next one in line. If greater it stores it
* in WS-LARGE. If less it is passed by.
***************************************************
FIND-SMALLEST-LOOP.
IF BOOK-AMT-TBL (WS-SUB) > WS-LARGE
MOVE BOOK-AMT-TBL (WS-SUB) TO WS-LARGE.

MOVE WS-LARGE TO PL-LARGE.

MOVE SEARCH-LINE-1 TO PRINT-LINE.
WRITE PRINT-LINE
AFTER ADVANCING 3 LINES.

CLOSE INVENT-FILE REPORT-FILE.

WRAP-UP-EXIT.
EXIT.
