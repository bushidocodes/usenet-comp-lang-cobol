It depends on what you mena by "icon." If you are referring to icons
which launch the application (such as from the Windows Program Manager,
you would either install these icons with an installation program, or
they would have to be bound to the executable with the Software
Development Kit provided by the GUI environment within which you are
working.

If you want icon fields within your application user interface, read on!

Yes, COBOL spII supports the display of bitmap-based icons in Windows,
Windows 95, Windows NT, OS/2 and UNIX Motif. In addition, function keys
may be assigned to the icon fields so that you can detect the mouse
click on the icon in the same fashion as you would process a traditional
COBOL-based user interface.

The programmer can also place "focus" on the icon with COBOL spII so
that as the operator of your application tabs through the fields on
the screen, the icon will become "highlighted" with a dark border
surrounding the icon field.

Icons of any size are supported.
