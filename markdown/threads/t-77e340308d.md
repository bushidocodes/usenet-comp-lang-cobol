> FILE-CONTROL.
>            SELECT O-CLM0110-XTRCT-FILE        ASSIGN TO CLM0100.

You probaly also need the word 'external' after 'to' to get it to use
the environment variables.  As it is it probably want to have the name
of the file in the implicitly defined working-storage variable
CLM0100.

ie it expects:

       MOVE "dirctory/nameoffile"    TO CLM0100
       OPEN OUTPUT O-CLM0110-Xtract-File

To take the name from an _external_ variable may need it to be
specified as such.
