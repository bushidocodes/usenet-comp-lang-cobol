```
Is there a way to read the attributes of a VSAM file in a COBOL
program and utilize those statistics??

I have a program that runs in 3 sepearte batch cycles. In 1 of them
this file carries about 1.5 million records, in the other 2 cycles it
has only one HIGH-VALUES record in it.

The program accesses the file Randomly and the other inputs are sorted
prior to execution.

I am thinking that if I can get a read on how many records are in the
file, and if it is 1 record, then I can verify that it is a
HIGH-VALUES record and not make the and IO call 950,000 times in the
program if it is.

I am also looking at reading the file sequentially and using TPS logic
on it to find EOF and then shut off the READ.

Any other ideas on how to verify this? I would love to get rid of
950,000 IO calls to a virtually empty file.
```

If the file is made OPTIONAL then, where it is to contain only
a HIGH-VALUES record, do not make the file available and the
OPEN will show file status '05'. If file status is '05', don't read
the file; just indicate the record was not found, or whatever.

This, I think, is a standard COBOL solution and I am not
certain as to what changes may be required in JCL.
