One of us is missing something horribly obvious, and I think it might
be you. :-) Are you *sure* that your "INVALID KEY" and "NOT INVALID KEY" 
clauses are correct? At first glance, they appear *reversed* to me:
+
+151-ADD-RECORD.
+           WRITE MASTER-RECORD FROM WS-MASTER-RECORD
+               INVALID KEY
This clause is executed if the write *fails* -- but I think you want
to do these moves if it *succeeds*.
+                   MOVE TR-INFO TO WS-STOCK-INFO
+                   MOVE TR-CURRENT-INFO TO WS-STOCK-CURRENT-INFO
+                   MOVE TR-PROJECTION-INFO TO WS-STOCK-PROJECTION-INFO
+               NOT INVALID KEY
And this clause is executed if the write *succeeds* -- and I'm sure that you 
don't want to be displaying a duplicate-record error message after a 
successful write.
+                   MOVE TR-NAME TO RPT-STOCK-NAME
+                   MOVE 'Error - A - Duplicate Record Found'
+                       TO RPT-ERROR-REASON
+                   WRITE REPORT-RECORD FROM REPORT-RECORD-DET
+                       AFTER ADVANCING 1 LINE
+           END-WRITE.

I think if you change "INVALID KEY" to "NOT INVALID KEY", and vice versa,
the program will behave as you expect it to.
