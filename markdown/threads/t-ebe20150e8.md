Simple question, simple answer.  Yes, you can do it, and this is how:


1) Write another COBOL program that will be *CALL*ed & name it
DDCHANGE.  In its LINKAGE SECTION, have an 8 character field and a 47
character field, one is NEW-DDNAME and the last field is
FD-TO-BE-CHANGED.  In the PROCEDURE DIVISION, all you do is MOVE
NEW-DDNAME TO the actual DD name in the FD area, which is located in
positions 40 - 47 and then GOBACK.

2) In your currently existing program, at the point you test to want
to dynamically alter the DD name, issue a CALL to *DDCHANGE* USING
NEW-DDNAME and the *FD* name of the DD that you want to change.

Hope that this helps you out.  Please let me know.  You might have to
play around with the length of the FD area to get this one right.  I
did it by writing it to a VB (Variable length record, blocked) file &
using File-Aid to view the record, which contained the FD "record"
area.

Jay

this sample jcl is ok. but the problem is that we cannot change the jcl. the
program that needs the dynamic assign is a subprogram. it could be used by a
main-program or by a another subprogram. the controlling of all programs is
possible with the textfile. this is a relict from older times. the textfile
look like this:

26.03.1999                                       AVK
U                                                USK
file_1 file_2                                 prog1K
1 4 675                                       prog2K
1241 sampletext                               enrg1K


each program can get its own parameters from this file at runtime. the program
with the name 'prog' would get the parameters from the lines that are marked
with prog1K and prog2K at end of line. similar, the program with the name
'enrg' get the parameters from the line marked with 'enrg1K'.

like i wrote, the jcl is unchangeable, but all files that will be possibly
used by the program are defined in the jcl.

at the moment we solved the problem with extra sections for all fileworking
actions. each sections contains a big evaluate that decides which file should
be used. Each file that could be used has its own select .. assign clause in
cobol-program.
Example (using the example textfile above. file_1 is input and file_2 is
output)

read-input-file section.
  evaluate true
     when prog1K-filename_1 = "file_1"
        read file_1
     when prog1K-filename_1 = "file_4"
        read file_4
     when prog1K-filename_1 = "file_7"
        read file_7
     when other
        perform abend
  end-evaluate.

write-output-file section.
  evaluate true
     when prog1K-filename_2 = "file_2"
        write file_1
     when prog1K-filename_2 = "file_10"
        write file_10
     when other
        perform abend
  end-evaluate.


but this is coding that i dislike.
