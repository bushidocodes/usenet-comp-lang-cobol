Unfortunately, there is not consistent, easy solution to your problem
with the signed fields. Virtually every vendor implements the sign
value differently. You will probably not be able to load this data
directly into your new files, and will have to convert it first.

The only hope you have is that the product you use for you RDMS has
translation support for the source data. Some PC Cobol compilers can
translate IBM mainframe sign conventions by simply specifying a compile
or run-time option.

Mike Dodas

Well, you understand that if you wrote PIC S999V9 on the DEC machine and
read PIC S999V9 on the Intel machine, you wouldn't have to know anything
about the format because the run-time library takes care of it. So, not
using CoBOL may not necessarily have been the easiest way to go. But,
OK, you want to know how overpunched signs work in ASCII.

A packed +1 is X"1C". When converted to zoned, this comes out as X"C1",
which is "A" in EBCDIC. In ASCII, however, "A" is X"41", so if you have
only the ASCII value, it's not easy to see where it came from. Here's a
list of all the overpunches. Notice how neat the EBCDIC columns are.
You can recover the digit value from any EBCDIC overpunch by ANDing it
with X"0F". Not so the ASCII translations.

Unsigned Positive Negative
Digit EBCDIC Char ASCII Digit EBCDIC Char ASCII Digit EBCDIC Char ASCII

0F F0 0 30 0C C0 { [*] 0D D0 } [**]
1F F1 1 31 1C C1 A 41 1D D1 J 4A
2F F2 2 32 2C C2 B 42 2D D2 K 4B
3F F3 3 33 3C C3 C 43 3D D3 L 4C
4F F4 4 34 4C C4 D 44 4D D4 M 4D
5F F5 5 35 5C C5 E 45 5D D5 N 4E
6F F6 6 36 6C C6 F 46 6D D6 O 4F
7F F7 7 37 7C C7 G 47 7D D7 P 50
8F F8 8 38 8C C8 H 48 8D D8 Q 51
9F F9 9 39 9C C9 I 49 9D D9 R 52

Note: There is more than one scheme of translation from EBCDIC to ASCII
[*] +0 might be translated to ( 28, [ 5B, or ? 3F
[**] -0 might be translated to ) 29, ] 5D, or ! 21


For a one-time project, if you can get hold of the CoBOL program on the
DEC machine and add SIGN IS { LEADING | TRAILING } SEPARATE, it might
make your program on the Intel machine easier, but it is too inefficient
for normal operation.
