My COBOL skills are quite a bit rusty and I have a production problem
that needs solving.  I have an old COBOL program that is our ODBC/SQL
log program.  It makes some references to some fields in library
QSYSINC/ file EZDAEP that must have changed between V4R4 (the last
time the program was compiled) and V5R4 (the release we are on now).

First - I reference these fields:

LIB-NAME OF QZDA-NDB-FORMAT2 (1)
LIB-NAME OF QZDA-NDB-FORMAT2 (2)
LIB-NAME OF QZDA-NDB-FORMAT2 (3)
LIB-NAME OF QZDA-NDB-FORMAT2 (4)
LIB-NAME OF QZDA-NDB-FORMAT2 (5)

This used to work but the copybook now is defined as:

       05  NBR-OF-LIBRARIES                    PIC S9(00009) BINARY.
  *                                   Number of libraries in list
  *    05  LIB-NAME                            PIC  X(00010)
  *                                            OCCURS xxxxx TIMES.
  *                                   Varying number
  *                                   of library names

with LIB-NAME commented out.  How can I readily access the first 'x'
number of occurrences of LIB-NAME (up to 5 of them) so I can move
these fields to the logfile record?

Second - I reference this field:
 SQL-STATEMENT-TEXT OF QZDA-SQL-FORMAT2

which now is defined as:

      05  SQL-STATEMENT-TEXT-LEN              PIC S9(00009) BINARY.
 *                                   Length of SQL Stmt text
 *    05  SQL-STATEMENT-TEXT                  PIC  X(xxxxx).
 *                                   Varying length
 *                                   SQL statement text

Again, SQL-STATEMENT-TEXT  is now commented out.  How I can reference
the SQL-STATEMENT-TEXT?

TIA,
Scott
```

I  rec'd some great help from the COBOL400-L on midrange.com.  For the
archives here are the relative snippets of code:

(working storage:)

05  MY-PTR                         USAGE POINTER.
05  LIBNAME-ARRAY-X.
    10  LIBNAME-ARRAY              PIC X(10)
                                   OCCURS 100 TIMES.
LINKAGE SECTION.

  COPY EZDAEP OF QSYSINC-QLBLSRC.

  01  LIBNAME              PIC X(10).
  01  SQL-STATEMENT-TEXT2  PIC X(4096).

SET MY-PTR TO ADDRESS OF SQL-STATEMENT-TEXT-LEN
   OF QZDA-SQL-FORMAT2
SET MY-PTR UP by length of SQL-STATEMENT-TEXT-LEN
SET ADDRESS OF SQL-STATEMENT-TEXT2 TO MY-PTR

 if SQL-STATEMENT-TEXT-LEN > 0

   if SQL-STATEMENT-TEXT-len > 4096
      move 4096 to SQL-STATEMENT-TEXT-len
   end-if

  STRING TODAYS-DATE SPACE TODAYS-TIME SPACE
        <snip>
        SQL-STATEMENT-TEXT2
         (1:SQL-STATEMENT-TEXT-LEN)
           DELIMITED BY SIZE INTO FD-ODBCLOG-RECORD
    else
     STRING TODAYS-DATE SPACE TODAYS-TIME SPACE
           <snip>
           PACKAGE-LIB-NAME
             OF QZDA-SQL-FORMAT2 '|'
           DELIMITED BY SIZE INTO FD-ODBCLOG-RECORD
    end-if
   end-if
=============================================================================
  MOVE SPACES TO LIBNAME-ARRAY-X

  SET MY-PTR TO ADDRESS OF NBR-OF-LIBRARIES
  SET MY-PTR UP BY LENGTH OF NBR-OF-LIBRARIES
  SET ADDRESS OF LIBNAME TO MY-PTR

  PERFORM VARYING SUB FROM 1 BY 1
      UNTIL SUB > NBR-OF-LIBRARIES
      MOVE LIBNAME
          TO LIBNAME-ARRAY (SUB)
      SET MY-PTR UP BY LENGTH OF LIBNAME
      SET ADDRESS OF LIBNAME TO MY-PTR
  END-PERFORM

STRING TODAYS-DATE SPACE TODAYS-TIME SPACE
       <snip>
       LIBNAME-ARRAY-X
        '|'
       DELIMITED BY SIZE INTO FD-ODBCLOG-RECORD
```
