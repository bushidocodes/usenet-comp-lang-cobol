Well, RGB isn't that easy. At least from the perspective of Windows
programming, it is a mix of three bytes;
    RedByte, GreenByte, BlueByte

Which each may have a value between 0 and 255. For the Windows API, this
implies shades of the three colors, that is, if all are set to 0, the result
is black, are all set to 255, the result is white.

For the Windows API, there is a RGB macro to call, for instance;
    myRGBColor = RGB(255, 128, 144);
which expands into:
    myRGBColor =
((((BYTE)(r)|((WORD)((BYTE)(g))<<8))|(((DWORD)(BYTE)(b))<<16)))

The result of the macro is a DWORD value which is a 32-bit unsigned integer.

Obviously, this won't fit in any storage intended to hold a value ranging
from 0 to 7.

> HOWEVER, you might want to send a note to J4
