```
I'm having a difficult time trying to communicate to a Fujitsu COBOL
program from a VB 6.0 program thru TCP/IP.  We are using the Dart
Communications Power TCP control on the VB side.  In that environment,
you send data to an IP address given a specific port.  On the Fujitsu
side, there seems to be some Simplified Queue system that holds TCP
messages and routes them.  The problem is, that if you send to the IP
address and the port of the Simplified Queue, nothing happens.  It
never hears the message.  Now, there are also Logical Destinations
that have to be dealt with on the Simplified system, that quite
frankly are a pain....

So, I am wondering if anyone out there has ever communicated using
TCP/IP to talk to a Fujitsu COBOL program without using the Simplified
Inter-application Communication Server.  It would seem that at some
point in history, a VB program needed to send data to a Fujitsu COBOL
program using TCP communication.
```
