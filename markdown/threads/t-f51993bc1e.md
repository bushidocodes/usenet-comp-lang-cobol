```
In your source code you have:

           SELECT CUSTOMER-MASTER-IO
               ASSIGN TO 'C:\CBL\LAB15\DATA\M1501.NDX'
               ORGANIZATION IS INDEXED
               ACCESS IS RANDOM
               RECORD KEY IS MAST-CUST-NO.
                           ....
         OPEN ...
                I-O   CUSTOMER-MASTER-IO.

This assumes that the file 'C:\CBL\LAB15\DATA\M1501.NDX' already exists and
is an "indexed file".

What you need (as your question implied) is a way to "make" your existing
master file (that ends in ".txt" - and which APPEARS to be a "sequential"
file into an indexed file.

There are a couple of ways to do this (assuming that you class didn't
actually "give" you an indexed file to copy to your PC - that you haven't
seen yet).

1) Write a "quick and dirty" program that simply reads the existing
sequential (.txt) file and writes out each record to a new indexed (.ndx)
file.

2) If you have (as I think you do) one of the MERANT products that includes
"WFL" - use it to create an "indexed file" from a "sequential file" (this is
a vendor supplied utility - that I think is still part of Mainframe
Express - but may or may not be a part of Personal COBOL)

P.S.  Good coding "technique" - especially for indexed files
   *ALWAYS* check your file status codes after every I/O statement
(including OPEN, WRITE, READ, etc).  If it is not zero (or at least the 1st
byte as '0') the chances are that "something" is wrong.  This may be a
problem with your program logic OR it may be a "real world problem" that you
need to provide code in your program to handle.
```
