Unless they have changed Personal COBOL, you can't. To build .EXE's you
need .OBJ's. You can only get the .OBJ's out of the real product. If you
really want to run from the command line you can always type
{whatever the workbench executable is} run {program name}. For example,
on my system you can just type "wb run myprog" and myprog will be run.
