You could try http://www.xml4cobol.com. When last I looked it's premise was to provide a callable routine based on parsing specific DTDs, which is probably OK for a lot of mainframe applications. Features may have changed.

I wrote my own SAX-ish parser in COBOL because I wanted to handle XML Schema and provide some dynamic support for unrecognized DTDs. If 'twere me, I'd probably set up a java application on the mainframe that returns the data from an XML document as a COBOL structure, or use MQSeries to accomplish the same thing rather than trying to use something proprietary with an evolving technology like XML. Otherwise you're obsolete before you start.

- Karl
