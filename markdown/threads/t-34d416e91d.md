### MF Cobol indexed files across NFS?

We have an MF Cobol (1.2) application running on two U6000 Unix boxes.
One machine holds the data files and shares this filesystem via NFS to
the other.

If two processes, one on each machine attempt to update the same file
simultaneously, the file is corrupted.

Does the MF file handler not work with NFS?

Tracing a file update process via truss seems to indicate that MF
synchronises multiple updates via fcntl locking only. Tests have shown
that this form of locking _is_ propagated across the NFS via lockd
successfully, so what the hell is going wrong? Are the NFS buffers
being flushed _after_ the lock is released or something?

Any information at all on this issue would be appreciated. Please
reply (or Cc:) via email, as our news feed is sloooowww.
