```
Not really a COBOL issue, but of possible interest to many of you running on 
OS/390 or z/OS. (If you aren't a "systems programmer type" you may want to run 
this by the appropriate people in your shop.)

From "Cheryl's List #93"

"Last Day to Order z/OS 1.4

Please note that 9Sept2004 is the last day to order z/OS 1.4 or z/OS 1.5.  Why 
does this matter?  If you are running OS/390 R10, that release goes out of 
service at the end of this month (30Sep2004), and you need to order another 
release to stay in service.  Your choices are one of the following:

      z/OS 1.4 - End of service 31Mar2007, the only bimodal release available
      z/OS 1.5 - End of service 31Mar2007, the last release to run on 
non-zSeries machines
      z/OS 1.6 - GA date is 24Sep2004, but it only runs on zSeries machines

So if you're on a G5/G6 (or similar) machine using OS/390 R10 (or even early 
z/OS releases), you must order either z/OS 1.4 or 1.5 as soon as possible.  z/OS 
1.6 won't work on your machines.  Although the last day to order is 9Sep2004, 
IBM recommends ordering by the end of August (yesterday!).  The difference 
between z/OS 1.4 or 1.5 is that 1.4 provides bimodal support, which provides a 
short-term fall-back to 31-bit mode when you finally move to a zSeries machine. 
You only have about a week left to make a decision, so please consider this 
carefully.

Even if you're running z/OS, you may still need to order z/OS 1.4 or 1.5.  z/OS 
1.1 went out of service 31Mar2004, z/OS 1.2 goes out of service 31Oct2004, and 
z/OS 1.3 goes out of service 31Mar2005.

If you're running OS/390 R10, even on a zSeries machine, then you will probably 
want to upgrade before that release goes out of service at the end of September.

End of service dates can be found at 
http://www.ibm.com/servers/eserver/zseries/zos/support/zos_eos_dates.html."

   ***

NOTE:
   As far as COBOL goes, you should also be aware that as of October 2004, you 
will need to be on at least z/OS 1.3 for the LE link-edit and/or run-time 
library for whatever COBOL object code you have - to be supported by IBM. This 
is true whether you are using any of the following compilers (only the last of 
which is still supported as a compiler on an MVS, OS/390, or z/OS system):

  OS/VS COBOL
  VS COBOL II
  SAA AD/Cycle COBOL/370
  IBM COBOL for OS/390 & VM
  IBM COBOL for z/OS & VM
  Enterprise COBOL
```
