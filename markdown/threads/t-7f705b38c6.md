The dreaded day has arrived that I might have to upgrade my machine
from Win 3.1 to Win 95 due to the fact I want to use Dragon
NaturallySpeaking and it only runs under Win 95. However, I have what
one might call a "mission critical" application written in Realia 4.2
and Screen IO 2.1 which I run under DOS 6.2. I don't want any problems
with this application under Win 95 (I understand it will still be
running under DOS) and I want to still continue to enhance it, change
it, and write new programs in Realia & Screen IO.

Does anyone run this sort of combination (I imagine a lot of people
do) and has anyone had any notable problems?

Thanks in advance for your advice.

Yes. Realdbug stops working under Windows 95. You can shutdown and
restart in Windows 95 and still run it. Screenio has some problems
with the window being less that a full screen in Windows 95 - but it's
workable.

If you want to update your code to Windows, you can get COBOL SP2 from
Flexus - I converted several screenio applications with it, and they
now have a screenio emulator. If I can be of any assistance at all,
let me know. Realia with Screenio is my forte.

The flexus products can be downloaded from the internet with 60 day
eval versions from http://www.flexus.com
