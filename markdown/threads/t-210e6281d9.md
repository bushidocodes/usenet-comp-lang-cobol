       PROCEDURE DIVISION.
       0000-START-HERE.
           EXIT PROGRAM.

       ENTRY "VERIFY-DATE" USING VD-DATE, VD-VALID.
           MOVE VD-YEAR to IVD-YEAR.
           MOVE VD-MONTH TO IVD-MONTH.
           MOVE VD-DAY-OF-MONTH to IVD-DAY-OF-MONTH.
           PERFORM 1004-ISVALIDDATE.
           MOVE IVD-VALID TO VD-VALID.
           EXIT PROGRAM.

       ENTRY "FIELDED-TO-LINEAR" USING FTL-DATE, FTL-LINEAR.
           MOVE FTL-YEAR TO GTL-YEAR.
           MOVE FTL-MONTH TO GTL-MONTH.
           MOVE FTL-DAY-OF-MONTH TO GTL-DAY-OF-MONTH.
           PERFORM 1006-GREGORIAN-TO-LINEAR1601.
           MOVE GTL-LINEAR TO FTL-LINEAR.
           EXIT PROGRAM.

       ENTRY "LINEAR-TO-FIELDED" USING LTF-LINEAR, LTF-DATE.
           MOVE LTF-LINEAR TO LTG-LINEAR.
           PERFORM 1008-LINEAR1601-TO-GREGORIAN.
           MOVE LTG-YEAR TO LTF-YEAR.
           MOVE LTG-MONTH TO LTF-MONTH.
           MOVE LTG-DAY-OF-MONTH TO LTF-DAY-OF-MONTH.
           MOVE LTG-DAY-OF-YEAR TO LTF-DAY-OF-YEAR.
           MOVE LTG-DAY-OF-WEEK TO LTF-DAY-OF-WEEK.
           EXIT PROGRAM.

      ******************************************************************
      * The following "Floor Division Modulus" routine calculates the  *
      * proper modulus of a negative number. For negative numbers, a   *
      * modulus is different than a remainder. The modulus is always   *
      * positive.                                                      *
      *                                                                *
      * For example, -1 floordivmod 2 gives a FDM-DIV of -1 with a     *
      * FDM-MOD of +1. In other words, -1/2 ==> -1 + 1/2               *
      *                                                                *
      * In contrast, a remainder approach gives a different answer:    *
      * -1/2 ==> 0 + -1/2                                              *
      ******************************************************************
       1001-FLOOR-DIVMOD.
           IF FDM-X >= 0
               DIVIDE FDM-Y INTO FDM-X GIVING FDM-DIV REMAINDER FDM-MOD
           ELSE
               ADD 1 TO FDM-X GIVING FDM-TMP
               SUBTRACT FDM-Y FROM FDM-TMP
               DIVIDE FDM-Y INTO FDM-TMP GIVING FDM-DIV
                   REMAINDER FDM-MOD
               ADD FDM-Y TO FDM-MOD
               SUBTRACT 1 FROM FDM-MOD
           END-IF.

      ******************************************************************
      * The following "Floor Division" performs the same division      *
      * operation as 1001-FLOOR-DIVMOD above. See above for a more     *
      * detailed explanation.                                          *
      ******************************************************************
       1002-FLOOR-DIV.
           IF FD-X >= 0
               DIVIDE FD-Y INTO FD-X GIVING FD-DIV
           ELSE
               ADD 1 TO FD-X GIVING FD-TMP
               SUBTRACT FD-Y FROM FD-TMP
               DIVIDE FD-Y INTO FD-TMP GIVING FD-DIV
           END-IF.

      ******************************************************************
      * The following code determines whether a year is a leap year or *
      * not. It works for negative years.                              *
      ******************************************************************
       1003-ISLEAPYEAR.
           MOVE ILY-YEAR TO FDM-X.
           MOVE 4 to FDM-Y.
           PERFORM 1001-FLOOR-DIVMOD.
           IF FDM-MOD <> 0
               MOVE "N" TO ILY-LEAP
           ELSE
               MOVE 400 TO FDM-Y
               PERFORM 1001-FLOOR-DIVMOD
               IF (FDM-MOD = 100) OR (FDM-MOD = 200) OR (FDM-MOD = 300)
                   MOVE "N" TO ILY-LEAP
               ELSE
                   MOVE "Y" TO ILY-LEAP
               END-IF
           END-IF.

      ******************************************************************
      * The following code determines whether a given date is a valid  *
      * one or not. This test guarantees that the given date will be   *
      * convertible to a 'linear' day number without the overhead of   *
      * over actually calculating it.                                  *
      ******************************************************************
       1004-ISVALIDDATE.
           MOVE "N" TO IVD-VALID.
           IF (-27256 <= IVD-YEAR) AND (IVD-YEAR <= 30826)
           AND (1 <= IVD-MONTH) AND (IVD-MONTH <= 12)
           AND (1 <= IVD-DAY-OF-MONTH)
           AND (IVD-DAY-OF-MONTH < IVD-DAYS-IN-MONTH(IVD-MONTH))
               IF (IVD-MONTH = 2) AND (IVD-DAY-OF-MONTH = 29)
                   MOVE IVD-YEAR TO ILY-YEAR
                   PERFORM 1003-ISLEAPYEAR
                   IF (ILY-IS-LEAP-YEAR)
                       MOVE "Y" TO IVD-VALID
                   END-IF
               ELSE
                   MOVE "Y" TO IVD-VALID
               END-IF
           END-IF.

      ******************************************************************
      * The following date conversion routine takes a Gregorian        *
      * Calendar date and calculates a linear numbering of days from   *
      * the Epoch of 1 R.D. January 1                                  *
      *                                                                *
      * The date is assumed valid (see 1004-ISVALIDDATE)               *
      ******************************************************************
       1005-GREGORIAN-TO-LINEAR.
           SUBTRACT 1 FROM GTL-YEAR GIVING GTL-YEAR-LESS-1.
           MULTIPLY GTL-YEAR-LESS-1 BY 365 GIVING GTL-LINEAR.

           MOVE GTL-YEAR-LESS-1 TO FD-X.
           MOVE 4 TO FD-Y.
           PERFORM 1002-FLOOR-DIV.
           ADD FD-DIV TO GTL-LINEAR.

           MOVE 100 TO FD-Y.
           PERFORM 1002-FLOOR-DIV.
           SUBTRACT FD-DIV FROM GTL-LINEAR.

           MOVE 400 to FD-Y.
           PERFORM 1002-FLOOR-DIV.
           ADD FD-DIV TO GTL-LINEAR.

           MULTIPLY 367 BY GTL-MONTH GIVING GTL-TEMP-DAYS.
           SUBTRACT 362 FROM GTL-TEMP-DAYS GIVING FD-X.
           MOVE 12 TO FD-Y.
           PERFORM 1002-FLOOR-DIV.
           ADD FD-DIV TO GTL-LINEAR.

           ADD GTL-DAY-OF-MONTH TO GTL-LINEAR.

           IF GTL-MONTH > 2
               MOVE GTL-YEAR TO ILY-YEAR
               PERFORM 1003-ISLEAPYEAR
               IF ILY-IS-LEAP-YEAR
                   SUBTRACT 1 FROM GTL-LINEAR
               ELSE
                   SUBTRACT 2 FROM GTL-LINEAR
               END-IF
           END-IF.

      ******************************************************************
      * The above code calculates GTL-LINEAR with an Epoch of 1 R.D    *
      * We make an adjustment now to change the Epoch to 1601 January  *
      * 1st.                                                           *
      ******************************************************************
       1006-GREGORIAN-TO-LINEAR1601.
           PERFORM 1005-GREGORIAN-TO-LINEAR.
           SUBTRACT 584389 FROM GTL-LINEAR.

      ******************************************************************
      * The following date conversion routine takes a linear numbering *
      * of days from the Epoch of 1601 January 1st and calculates the  *
      * corresponding date in the Gregorian Calendar.                  *
      ******************************************************************
       1007-LINEAR-TO-GREGORIAN.
           SUBTRACT 1 FROM LTG-LINEAR GIVING LTG-D0.

           MOVE LTG-D0 TO FDM-X.
           MOVE 146097 TO FDM-Y.
           PERFORM 1001-FLOOR-DIVMOD.
           MOVE FDM-DIV TO LTG-N400.
           MOVE FDM-MOD TO LTG-D1.

           MOVE LTG-D1 TO FDM-X.
           MOVE 36524 TO FDM-Y.
           PERFORM 1001-FLOOR-DIVMOD.
           MOVE FDM-DIV TO LTG-N100.
           MOVE FDM-MOD TO LTG-D2.

           MOVE LTG-D2 TO FDM-X.
           MOVE 1461 TO FDM-Y.
           PERFORM 1001-FLOOR-DIVMOD.
           MOVE FDM-DIV TO LTG-N4.
           MOVE FDM-MOD TO LTG-D3.

           MOVE LTG-D3 TO FDM-X.
           MOVE 365 TO FDM-Y.
           PERFORM 1001-FLOOR-DIVMOD.
           MOVE FDM-DIV TO LTG-N1.
           MOVE FDM-MOD TO LTG-D4.

           ADD 1 TO LTG-D4.

           COMPUTE LTG-YEAR = 400*LTG-N400 + 100*LTG-N100
                            + 4*LTG-N4 + LTG-N1.

           IF (LTG-N100 <> 4) AND (LTG-N1 <> 4)
               ADD 1 TO LTG-YEAR
           END-IF.

           MOVE LTG-YEAR TO GTL-YEAR.
           MOVE 1 TO GTL-MONTH.
           MOVE 1 TO GTL-DAY-OF-MONTH.
           PERFORM 1005-GREGORIAN-TO-LINEAR.
           MOVE GTL-LINEAR TO LTG-JAN01.

           MOVE 3 TO GTL-MONTH.
           PERFORM 1005-GREGORIAN-TO-LINEAR.
           MOVE GTL-LINEAR TO LTG-MAR01.

           IF (LTG-LINEAR  < LTG-MAR01)
               MOVE 0 TO LTG-CORRECTION
           ELSE
               MOVE LTG-YEAR TO ILY-YEAR
               PERFORM 1003-ISLEAPYEAR
               IF ILY-IS-LEAP-YEAR
                   MOVE 1 TO LTG-CORRECTION
               ELSE
                   MOVE 2 TO LTG-CORRECTION
               END-IF
           END-IF.

           SUBTRACT LTG-JAN01 FROM LTG-LINEAR GIVING LTG-PRIOR-DAYS.
           ADD 1 TO LTG-PRIOR-DAYS GIVING LTG-DAY-OF-YEAR.
           COMPUTE LTG-TEMP =
               (12 * (LTG-PRIOR-DAYS + LTG-CORRECTION) + 373)/367.
           MOVE LTG-TEMP TO LTG-MONTH.

           MOVE LTG-MONTH TO GTL-MONTH.
           PERFORM 1005-GREGORIAN-TO-LINEAR.
           MOVE GTL-LINEAR TO LTG-1ST.

           COMPUTE LTG-DAY-OF-MONTH = LTG-LINEAR - LTG-1ST + 1.

           MOVE LTG-LINEAR TO FDM-X.
           MOVE 7 TO FDM-Y.
           PERFORM 1001-FLOOR-DIVMOD.
           MOVE FDM-MOD TO LTG-DAY-OF-WEEK.

      ******************************************************************
      * The above code uses LTG-LINEAR with an Epoch of 1 R.D.         *
      * We make an adjustment now so that we may accept LTG-LINEAR     *
      * with an Epoch of 1601 January 1st.                             *
      ******************************************************************
       1008-LINEAR1601-TO-GREGORIAN.
           ADD 584389 TO LTG-LINEAR.
           PERFORM 1007-LINEAR-TO-GREGORIAN.
       END PROGRAM JERM.

       IDENTIFICATION DIVISION.
       PROGRAM-ID. DRIVER.
       ENVIRONMENT DIVISION.
       INPUT-OUTPUT SECTION.
       FILE-CONTROL.
           SELECT OUTPUTFILE
               ASSIGN TO "OUTPUT"
               ACCESS MODE IS SEQUENTIAL.

       DATA DIVISION.
       FILE SECTION.
       FD OUTPUTFILE.
       01  DATE-RECORD.
           05  DR-LINEAR-DATE     PIC S9(8)   COMP.

       WORKING-STORAGE SECTION.

       01  A               PIC S9(8) COMP VALUE -1.

      *
      * Size of this structure is 8 bytes.
      *
       01  FIELDED-DATE.
           05  YEAR        PIC S9(5)   COMP.
           05  MONTH       PIC 99      COMP.
           05  DOM         PIC 99      COMP.
           05  DOY         PIC 999     COMP.
           05  DOW         PIC 9       COMP.

       01  BOOL            PIC X.
           88  IS-VALID    VALUE "Y".
           88  NOT-VALID   VALUE "N".

      *
      * This size of this item is 4 bytes.
      *
       01  LINEAR-DATE     PIC S9(8)   COMP.

       PROCEDURE DIVISION.
       0000-START-HERE.
           CALL "JERM".
           MOVE 2000 TO YEAR.
           MOVE 2 TO MONTH.
           PERFORM VARYING A FROM 1 BY 1 UNTIL A > 20
               MOVE A TO DOM
               CALL "VERIFY-DATE" USING FIELDED-DATE, BOOL
               IF NOT-VALID
                   DISPLAY "DATE NOT VALID."
               ELSE
                   CALL "FIELDED-TO-LINEAR" USING FIELDED-DATE,
                       LINEAR-DATE
                   DISPLAY YEAR " " MONTH " " DOM " " LINEAR-DATE
                       WITH NO ADVANCING

                   CALL "LINEAR-TO-FIELDED" USING LINEAR-DATE,
                       FIELDED-DATE
                   DISPLAY " " YEAR " " MONTH " " DOM " " DOY " " DOW
               END-IF
           END-PERFORM.
           OPEN OUTPUT OUTPUTFILE.
           MOVE LINEAR-DATE TO DATE-RECORD.
           WRITE DATE-RECORD.
           CLOSE OUTPUTFILE.
           GO TO 9999-STOP-RUN.

       9999-STOP-RUN.
           STOP RUN.
       END PROGRAM DRIVER.

> What's a Lilian date?

A Lilian date is an integer date system in which day 1 is October 15,
1582, the first day of the Gregorian calendar. It is named for Luigi
Lilio, who provided most of the ideas for the Gregorian calendar reform,
including the rule for leap years.   

IBM wrote their language environment before COBOL had standardized on
true Julian and the LE provided languages with a 64 bit double precision
floating point number containing the number of seconds elapsed since
00:00:00 14 October 1582.

IBM also has a date which they incorrectly call Julian, which was
developed when they had an early case of a Y2K type problem.  It is a
seven digit integer in the form of YYYYDDD.  Because of this, they
called their version of Julian date "Integer date", which is the number
of days since March 15, 2000.

Astronomers use Julian date: 
A serial number used since 1582 in astronomical calculations, equal to
the number of days elapsed since January 1, 4713 BC.  This system was
invented in 1582 by Joseph Scaliger, who named it for his father, Julius
Caesar Scaliger.
