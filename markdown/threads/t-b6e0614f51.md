Peter,

EXTFH should always be your file handler entry point if you are using
the system file handlers.
There are some products (OCDS and pre-NetExpress 32-bit PC products) in
which EXTFH is renamed to mFFH. This is because some file types on those
systems are handled by a different file handler to EXTFH. The 'EXTFH'
entry point on those systems is a router which interrogates the file
type in the FCD and passes it to the correct file handler.
Some code in the router version of EXTFH also does other work (for
instance, on a successful OPEN, it makes sure that the file will be
closed when the program that opened it is CANCELled) so you're better
off not bypassing it if you can help it.

When you say 'some but not all of our files had corrupt indices', I'm
guessing that those are the ones that would have been routed *away* from
mFFH, had EXTFH been called instead.

Cheers, Kev.
