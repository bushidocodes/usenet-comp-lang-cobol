```
The problem is that COBOL.EXE is a dual mode executable. It'll
run in DOS or OS/2. What's happening is that it's trying to
execute as an OS/2 executable and it's trying to load the OS/2
version of the run time system (COBLIB.DLL). You have three
options:

1) Install the OS/2 support for COBOL and if it's installed add
the COBOL\EXEDLL dir to the OS2LIBPATH environment variable in
NT (Control Panel and System Icon in Main).

2) Use FORCEDOS COBOL.EXE on the command prompt line. This'll
force the executable to run in DOS mode.

3) Execute COBOL.EXE using XM (i.e. XM COBOL). This'll not
only force it to run in DOS mode, since XM is a DOS-only
executable, but it'll run it in extended memory.
```
