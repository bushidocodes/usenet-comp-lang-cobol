Hello COBOL lovers,

I am considering a rather grand plan for making information on the
COBOL programming language commonly available on the net, and since
I'm new to comp.lang.cobol, as well as to the language itself, I
thought it appropriate to raise this idea for discussion on the net
before I unvail anything ... :)

The WWW page I have in mind should contain information on COBOL
literature and software. So far, all this is captured in the COBOL
FAQ. I also want to make the COBOL FAQ available on-line, but this is
beside the point.

I would like the WWW page to contain a complete guide to the syntax of
the COBOL language. I know this is a grand project, and I was hoping
to find volunteers who would be willing to make additions of their own
and donate them to the page. The syntax of COBOL is rich enough and
complex enough to make a hypertext reference extremely attractive.

Here is basically what I have in mind for the syntax guide:

(1) Complete hypertext based syntax guide for the COBOL language.

(2) Should annotate the syntax with information as to which version of
COBOL they apply. This should include COBOL 74, COBOL 85, the coming
COBOL 97.

(3) Information on implementation-specific syntax and functions, for
as many implementations as possible. [IS THIS A VIOLATION OF THE
COPYRIGHTS?? HOW WOULD THIS INFO APPEAR IN TEXTBOOKS?? DOES THIS
NEED EXPLICIT PERMISSION FROM THE VARIOUS COMPANIES??]


I am open to suggestions, additions, and [hint, hint] offers to help.
I'm currently typing in some general COBOL syntax. Once most of the
syntax is in, I'll make it available. At that point, I'll need all the
help I can get in:

(1) Finding errors.
(2) Suggesting better formats.
(3) Making [possibly implementation specific] additions.
(4) Semantics: One or two lines explaining the behavior of each
statement/construct/etc. I want to UNINTRUSIVELY insert links for
those in various places within the syntax chats.
(5) Suggestions for improving the page.

I know this is a tough one to ask, but PLEASE no flames. Any
constructive criticism is of course welcome.
