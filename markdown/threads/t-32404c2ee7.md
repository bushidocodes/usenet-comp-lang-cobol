If you use the runtime for Sco under Linux you are bound to find
problems.
If you use the correct runtime, the only problems that I am aware will
be related with printers (and they can be solved by changing some 
configuration files), or with terminfos/termcaps (and once again you 
can change these).

As for recompiling the programs, this is NOT required, as you can move
the objects from one system to the other directly.

Error 204 means that the program was not found, either on the loaded
table of programs, or as a valid file+"valid cobol extension".

Is it possible that in your SCO machine you have a configuration
file using`the keyword
EXTENSION-NAMES OBJECT=obj
If this is the case you need to make sure you have the same on 
the linux runtime configuration.
other option is that you do indeed have this on the configuration,
but as you have recompiled the programs they are now being create
as ".cob".
