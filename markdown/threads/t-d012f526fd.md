I don't think there is any chance that the conversion from the 16-bit to
32-bit products can change the reading of files. My *guess* is that the
default (or explicit) directives are different. What I suggest you do is
to compile the same minimal program with both compilers and specify the
SETTING directive - then go thru the listing one by one to see which is
different. Obviously, some possibilities of causing "odd results" are the
SEQUENTIAL, COMP, IBMCOMP, COMP-5 directives - but it could be almost
anything.
