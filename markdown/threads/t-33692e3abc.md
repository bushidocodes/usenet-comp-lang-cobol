>names of the calling .COB files.

Cobol-85 is not a Cobol vendor, it is a standard, however. It might be that
you are dealing with Acucobol, which uses the term Cobol-85, in that case
the 98,01 error is due to the index being broken. See if you can find a
utility: vutil.exe (possibly vutil32.exe). and use to it to rebuild the
index of the particular file:

    vutil -rebuild <filename>

Cheesle
