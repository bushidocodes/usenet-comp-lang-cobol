means "continue execution at next possible statement"  -- may sound
similar but very different. 

My belief is that every paragraph should have just one period in it.
Why? because almost every serious programming error I've encountered
was due to a period in the wrong place -- and always unnecessary.  the
period no longer serves any useful purpose, now that ansi 85 features
are available.

If you are going to use full stops then please don't 'bury'
them on the end of a line where they are hard to see.  They
are far more obvious if on a line by themselves:

           IF ( a = b )
               IF ( b = c )
                   IF ( d = e )
                        PERFORM Something
                        .
            PERFORM Something-else
            .

Having some lines with '.' on the end and some without leads
to problems.  This can be solved by 'only having . at the
end of a paragraph' or by 'having . in its own line', or
both (preferably).

By never having a '.' on the end of a line you remove the
problem of needing to add or remove them when the code
requires to be changed to being under a conditional
or not.

I have also started to adopt parantheses around conditions
(as above).  I find this tends to make them stand out
better, especially the multiline IFs:

           IF ( File-Status = Record-Missing
            AND Inserting-Option = "Y"
              )
               PERFORM .....
