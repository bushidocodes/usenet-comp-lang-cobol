Hi all,

I want to create a program with MF NetExpress V 1.0 using the
"Callable Rebuild" facility (CALLRB) to test the status of
Indexed files. If it detects a corrupt file it should then
perform a rebuild of the file(s). So far my code is just to
check for file status.  The rebuild portion will be coded once
I overcome the current problem.

My problem is that the call to "CALLRB" can only be executed
once in a run unit. The second call results in the following
runtime message:

        object Code error : file 'idxcheck'
        error code: 153, pc=A99, call=1 seg=0


The definition of error code 153 is:

        153     Subscript out of range  (Fatal)

        A subscript which you have used in your program is out
        of the defined range, that is, it is either less than
        one or it is greater than the number of occurrences of
        the item.

        Resolution:
        You should recode your program.

The Object code 'idxcheck' is linked into the program by the
following:
        cbllink -t dbcheck.cbl callrb rbldmain rbldsub idxcheck
                   cdfh fhredir

Program code follows:

      -----------------------------------------------------------------
      $SET COMP NOALTER NOQUAL ANS85 SCHEDULER TARGET"486"
       identification division.
      *----------------------------------------------------------------
      *  Program must be Compiled and linked as follows:
      *    cbllink -t dbcheck.cbl callrb rbldmain rbldsub idxcheck cdfh
      *               fhredir
      *----------------------------------------------------------------
       program-id. dbcheck.
       data division.
       working-storage section.
       01 Parameters        pic x(600).
       01 Call-Status.
          03 CS-Stat        pic x.
          03 CS-Code        pic x comp-x.
       01 Delay             pic x.

       linkage section.
       procedure division.
      *
      * Set Input & Output Filenames with options to Rebuild
      *
           initialize Call-Status parameters
           cancel "callrb"
           move "idrsmas /f" to Parameters
           call "callrb" using
                Parameters
                Call-Status
           end-call

           if return-code = 0
              display ">>> IDRSMAS  Database is OK ... "
              display "             Call-Status .. " CS-Stat
              display "             Call-Code   .. " CS-Code
           else
              display ">>> IDRSMAS  is Bad!! ..."
              display "             Return-Code .. " return-code
              display "             Call-Status .. " CS-Stat
              display "             Call-Code   .. " CS-Code
           end-if

           display ">>> Hit Enter Key"
           accept delay

           initialize Call-Status parameters
           cancel "callrb"
           move  "logmas /f" to Parameters
           call "callrb" using
                Parameters
                Call-Status
           end-call

           if return-code = 0
              display ">>> LOGMAS   Database is OK ... "
              display "             Call-Status .. " CS-Stat
              display "             Call-Code   .. " CS-Code
           else
              display ">>> LOGMAS   is Bad!! ..."
              display "             Return-Code .. " return-code
              display "             Call-Status .. " CS-Stat
              display "             Call-Code   .. " CS-Code
           end-if

           display ">>> Hit Enter Key"
           accept delay

           initialize Call-Status parameters
           cancel "callrb"
           move "idrsparm /f" to Parameters
           call "callrb" using
                Parameters
                Call-Status
           end-call

           if return-code = 0
              display ">>> IDRSPARM Database is OK ... "
              display "             Call-Status .. " CS-Stat
              display "             Call-Code   .. " CS-Code
           else
              display ">>> IDRSPARM is Bad!! ..."
              display "             Return-Code .. " return-code
              display "             Call-Status .. " CS-Stat
              display "             Call-Code   .. " CS-Code
           end-if

           display ">>> Hit Enter Key"
           accept delay

           initialize Call-Status parameters
           cancel "callrb"
           move "iddrno /f" to Parameters
           call "callrb" using
                Parameters
                Call-Status
           end-call

           if return-code = 0
              display ">>> IDDRNO   Database is OK ... "
              display "             Call-Status .. " CS-Stat
              display "             Call-Code   .. " CS-Code
           else
              display ">>> IDDRNO   is Bad!! ..."
              display "             Return-Code .. " return-code
              display "             Call-Status .. " CS-Stat
              display "             Call-Code   .. " CS-Code
           end-if
           stop run
           .

-------------------------------------------------------------------------
The First call works normally but the next call causes the program
to terminate with the RTS message about 'idxcheck'.

Anybody run across this situation?  And if so how did you solve
it. Any suggestions or ideas would be greatly appreciated.

Dave
