```
Hi,

I'm about at my wits end trying to figure out the proper syntax for
Fujitsu Cobol's INVOKE statement.  I've been searching web,
newsgroups, Fujitsu's site and have yet to find anything that might be
useful or up to date.  What I'm doing is taking code that worked with
Microfocus and attempting to get it running in Fujitsu NetCobol 6.1.

Here is the problem child:
      INVOKE WS-XMLDOCUMENT "createElement"
             USING "OLifEs"
             RETURNING WS-NEW-NODE.
Here is the error:
PZOLIFE2.COB 679: JMN3410I-S  BY CONTENT PHRASE CANNOT BE SPECIFIED IN
THE INVOKE STATEMENT,WHEN UNTYPED OBJECT REFERENCE IDENTIFIER,SPECIAL
CLASS OBJECT REFERENCE IDENTIFIER OR SPECIAL CLASS CLASS-NAME
'WS-XMLDOCUMENT' IS SPECIFIED IN INVOKE STATEMENT.

Fujitsu seems to have trouble with having a string literal in the
using clause in this case.

I've tried "USING BY VALUE 'OLifEs'" but that tells me I can't use BY
VALUE.  I've tried moving the string literal to a working storage
field, "USING WS-PROPERTY" and that seems to compile fine, but that's
not the answer I want.

I've tried declaring WS-XMLDOCUMENT as OBJECT REFERENCE COM where COM
is defined the the usual way CLASS COM AS "*COM" and that doesn't
help.

Why oh why doesn't the supplied documentation have anything better
than the "Getting started using COM/ActiveX" pdf?  Even the supplied
language reference doesn't have a proper entry for INVOKE.
```
