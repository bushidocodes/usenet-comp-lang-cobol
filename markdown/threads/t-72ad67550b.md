I am trying to write a COBOL program that would call an existing C program.

This is on a mainframe (OS/390 with LE).

The C program has a main() function and expects one argument.  For some
reason, the name of the C program is 'CALL'.  It does not return a result.

***************************************
void main(int  argc,char *argv[])
***************************************

My COBOL program (in relevant part), is coded like this:

********************************************************************
IDENTIFICATION DIVISION.
PROGRAM-ID.    CSQUTILD.

...

WORKING-STORAGE SECTION.

...

 01  PGM-CALL             PIC X(08) VALUE 'CALL    '.
 01  PARM-EMM             PIC X(03) VALUE 'EMM'.

...

PROCEDURE DIVISION USING PARM-INPUT.

...

CALL PGM-CALL USING BY CONTENT PARM-EMM.

...


**********************************************************************


The following complile options were used:

'DYNAM,OPT,RENT,DATA(24),MAP,XREF',
'NOCMPR2,NUMPROC(NOPFD),NOLIST',

***********************************************************************

The following link-edit options were used:

'LIST,LET,XREF,REUS,AMODE(31),RMODE(ANY),RENT,RES',

The link-edit SYSIN card looked like this :

//SYSIN    DD *
  INCLUDE SYSLIB(EDCSTART)
  INCLUDE SYSLIB(CEEROOTB)
  INCLUDE SYSLIB(@@CBL2C)
  INCLUDE SYSLMOD(CALL)
  ENTRY CSQUTILD
  NAME CALL(R)
//*
***********************************************************************

The compile and link-edit went successfully, but when the program is run,
it abends
with U4038 with the following message:

Is this the LKED parms for the Cobol compile?  You are mixing the C
module name "CALL" with the entry name of your Cobol program,
"CSQUTILD".

I would suggest just using a straight dynamic call to the C module and
saving yourself the trouble.

Compile and Link the "Call" program such that it can be executed via
JCL.  Then make sure it is in a steplib or the system linklist and
execute CSQUTILD with the "CALL myCpgm USING myParm RETURNING
dummyArea END-CALL".

Even without a return parameter I suggest using the RETURNING clause,
otherwise you might someday find yourself in the position of having
strange return codes for your job step.  Then again, maybe not.

Let me know if it helps,

Joe Z.

Thanks, after changing my linkage statements and making my program
call the C program dynamically, it worked.  The one additional thing I
had to do was to pass the number of parameters also as a parameter. 
This is because I am calling the main() which expects (int argc, char*
argv[]).

So I had to do CALL PGM-CALL USING EMM-COUNT,PARM-EMM.  (EMM-COUNT=1).
If calling from JCL, this is not needed for some reason.  We are just
passing the argument.
