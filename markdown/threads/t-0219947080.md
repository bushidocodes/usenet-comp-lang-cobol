Charlie,

We used to have an application written in Microfocus COBOL 3.4.23 that
interfaced with Winfax Pro 10.  The PC (Windows 95) the application was
running on had the "default" printer set to "Winfax" and we used DDE
commands to send report files out automatically, via the Print command.  It
didn't have any specific PCL commands within the reports, but we let Winfax
Pro do all the work and keep track of the events (i.e. pass, fail, retries,
etc.) and Winfax handled the modem interface and cover pages (we didn't use
one, but we could have) as well.  I'm just providing this information as a
place to start looking, since Symantec probably has a W2K version of Winfax
Pro and I would hope that NetExpress still has DDE/OLE type interfaces with
software such as Winfax Pro.  We've since moved to Fujitsu 6.1 and no longer
use the Mircofocus program.  But no need to create a bunch of new code if
existing DDE (yeah, I know it's old fashioned, but it worked) conversations
might do the job for you.

Hope this helps,

Denny
