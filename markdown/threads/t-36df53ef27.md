You can definitely do this.  We have a working application that communicates
from a PC client to AS/400 server.  Both client and server are Cobol and do
direct socket communications.

For the PC side, check out Microsoft's sockets API documentation.  You should be
able to call all the relavent functions from MF Cobol.  The MF Cobol
documentation provides information on how to interface to the Windows API
functions.

The Windows Sockets API documentation is at
http://msdn.microsoft.com/library/sdkdoc/winsock/apistart_520i.htm.

Good luck!
