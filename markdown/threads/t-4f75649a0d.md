```
› COBOL keeps on translating the TAB's into multiple spaces.

Presumably you're reading the records in as a line sequential file? As far as I
know, TABs will always get expanded to spaces. There is a +T RTS switch which
allows spaces to be compressed to TABs on writing the file but that doesn't help
for reading the file.

I'd probably use a byte stream file using CBL_OPEN_FILE, CBL_READ_FILE etc and
probably a heap for speed. ie. read the file into a heap first. Then just read a
byte in at a time watching out for TAB characters and chucking records at your
program as you come across them.

Shaun.
```
