A CICS program can issue a dynamic CALL to a subroutine a does not have
to pass it the DFHEIBLK and DFHCOMMAREA if the subroutine uses no CICS
commands.

I don't have DB2 experience, and my question is what if the subroutine
has EXEC SQL code to access DB2 database ?

What I'm after is the ability to code a COBOL subroutine that accesses
DB2 tables, and compile it once (with no CICS translation) and then be
able to call it dynamically from both CICS and batch programs.

>From what I've read thus far, it looks like this should work, but I'd
like to hear from anyone who's actually tried it.

In doing this, remember that when you're running under CICS, you cannot
issue the DB2 COMMIT statement. That preogative lies with CICS (i.e. CICS
SYNCPOINT must be used).

Also, I guess you won't pass the subroutine source thru CICS's translator.
Else, it'll get the two extra parameters by default.

- Darius Cooper
