I have an application running on WIN/NT 3.51 using Microfocus COBOL V4 - 32bit SDK.
The application is being invoked from a multi-threaded C++ (VC 4.0) application. Things
work fine when invoked by a single thread at a time. When we open up more threads things get ugly.

The C program creates multiple threads and each can invoke program COBOL1 and COBOL1 can invoke a varitey
of other COBOL modules, say COBOL2A and COBOL2B, for example. COBOL2* can invoke a variety of COBOL3*
modules. All the COBOL modules are DLLs.

What we observe is a variety of run-time errors within many of the third level COBOL3 modules. The most prevalent
being Run time error 166 - recursive program call. This leads us to believe that the run-time system thinks that invocations
of the (say) COBOL3A module from different threads is actually an attempt at a recurisve call.

What really confuses me is why didn't it complain about this for program COBOL1 which would be called be all threads
and is the modules that indirectly invokes module COBOL3A. Anybody have any ideas?

Since this is meant to be a portable application (the cobol parts anyhow) I don't want to make use of microfocus
extensions unless I really have to.
