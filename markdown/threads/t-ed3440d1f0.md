The new ANS COBOL Standard will likely be COBOL 2000,
rather than COBOL 97, or COBOL 98 (as I've been referencing for
some years now.) The recent meeting of the ISO COBOL standards
committee pretty much has pushed the "official" release date
to 2000. However we will see compilers on the streets with
the new COBOL 2000 features well before then. (They are already
on the streets today.) To see some of these new features
see http://members.aol.com/JERRYCOBOL/blackjg.html
Click on the link to the new COBOL Standard.
J Garfunkel

Description of Features

31-digit fixed point numeric data type - The maximum number of digits that may
be specified in a numeric literal or in a PICTURE character-string that
describes a numeric or numeric-edited data item is increased from 18 to 31. The
maximum number of characters that may be specified in a PICTURE
character-string is increased from 30 to 50.
<>

ACCEPT options and new functions for 4-digit years - Enhance the ACCEPT
statement to return a 4-digit year through the use of new keywords. Provide
functions that take as arguments a date with a 2-digit year and an optional
100-year range and return a date with a 4-digit year.
<>

Boolean datatype for bit handling and Boolean operations - The capability of
defining bit strings and setting or testing Boolean values is added. Data of
class and category Boolean may be represented as bits, alphanumeric characters,
or national characters by specifying usage bit, display, or national,
respectively. Boolean data items are specified by picture character strings
containing the symbol '1'; Boolean literals are identified by the separator
'B"'. Boolean operations B-XOR, B-AND, B-OR, and B-NOT are provided for use in
Boolean expressions.
<>

CALL enhancements include:
- New data types for interlanguage calls - pointers, floating-point, native
binary
- User-defined functions
- Call prototypes, so that type checking of parameters is possible
- Passing parameters by value
- Case-sensitive external names
- Use of the reserved word FUNCTION optional
- Omitted parameters
- Passing literals as parameters
- Recursion
- GOBACK statement
- EXIT WITH STATUS
<>

Compiler directives - Compiler directives instruct the compiler to take
specific actions while compiling a source program. Compiler directives are
provided:
- to control the source program listing
- to specify the extended letters of the COBOL character set
- to request flagging of syntax that might be incompatible between the
previous COBOL standard and the current standard
- to specify page ejection
- to cause propagation of an exception condition to a calling program or
invoking method
- to specify whether the reference format of the source text that follows if
fixed form or free form
- to turn checking for certain exception conditions on or off
- for specifying options that are defined by the implementor.
<>

Conditional Compilation - One defines constants with a CONSTANT directive
(these can be used anywhere in a program, not just in the EVALUATE and IF
directives) and uses EVALUATE and IF directives to delimit code.
<>

Currency sign enhancements - Add the ability to specify multiple-character
currency signs and single-character ones that are any character.
<>

Exception handling, additional features for detection and reporting - A common
method for handling exceptions which provides exceptions for all rules in the
specifications that state not to do something; a method of turning checking on
and off; Declaratives for processing all exceptions; and functions for
determining where an exception occurred and what it was. The RECOVER statement
allows specification of where execution is to resume after an exception
declarative is executed.
<>

EXIT options - Add the ability to immediately exit an in-line PERFORM
statement, a paragraph, or a section.

File sharing and record locking - provided by merging the capabilities from the
JOD and the X/Open specifications.

Free-form source text - Free-form reference format provides varying-length
source lines and permits a program to be written with a minimum of restrictions
on the placement of source code on a line. A compiler directive permits
selection and intermixing of free-form or fixed-form reference format, with
fixed-form as the default. Includes in-line comments and concatenation of
literals.

HIGHEST-ALGEBRAIC and LOWEST-ALGEBRAIC functions - return the highest or lowest
possible value for a data item

INITIALIZE FILLER - The FILLER phrase was added to the INITIALIZE statement to
give the option of initializing filler data items.

National character datatype and internationalization - The capability is added
for using large character sets, such as ISO/IEC 10646-1:1993, in the source
program and in program data at execution time. Class national and categories
national and national-edited are specified by PICTURE character-strings
containing the symbol 'N'; national literals are identified by the separator
'N"'. Usage national specifies representation of data in a national character
set. User-defined words may contain extended letters from a national character
set. Processing of data of class national is comparable to processing data of
class alphanumeric, though there are some minor differences. Conversion between
data of classes alphanumeric and national are provided by intrinsic functions.
Support for cultural conventions (locales) for number and date formatting and
collating sequences.

Object-oriented programming enhancements - Support for object oriented
programming has been added to the language.

Portability of arithmetic increased - The ARITHMETIC IS STANDARD phrase
specifies that certain arithmetic will be performed in a well-defined manner
and may yield results that are portable. When standards arithmetic is in
effect, there is no restriction on the composite of operands.

Report Writer enhancements - The report writer facility is integrated into the
COBOL specification and it must be implemented by a conforming implementation.
Previously, the report writer was a separate module and its implementation was
optional. Additional features have been provided.

Screen handling facility - The screen section provides a non-procedural means
of declaring screen items that are to appear on a terminal, their position, and
various attributes. The screen item may be used for input or output and be
associated with data items described in other sections within the data
division.

Table sort - A SORT statement may be used to sort a table. This sort may be
done using the fields specified in the KEY phrase defining the table, by using
the entire table element as the key, or by using specified key data items.
<>

Tables that are automatically-expanded - Tables may be declared with no maximum
number of occurrences. The system automatically expands the size allocated for
a table when the table is referenced as a receiving item. The space allocated
for the table may be increased or decreased with a new format of the SET
statement.
<>

User-defined types - The TYPEDEF clause identifies a type declaration which
creates a user-defined type. The TYPE clause is used to apply this user-defined
type to the description of a data item. No storage is allocated for the type
declaration.
<>

Validation features for data - The VALIDATE statement gives a program the
ability to perform comprehensive data validation. The new data division clauses
ALLOW, CLASS, DEFAULT, DESTINATION, ERROR, INVALID, PRESENT WHEN, and VARYING
were also added. The PRESENT WHEN and VARYING clauses have a similar function
to those in the report section. The new clauses, together with the VALID,
INVALID, and WHEN phrases of the level-88 VALUE clause, are ignored when they
appear in a data description which is the operand of any statement other than
VALIDATE.
<>
