```
We just got vers. 2.0 (beta) up & running on a test bed, & it looks good.
But even the old 1.1 has remote stuff like I didn't think possible.

I can compile a pgm on my PC, set up JCL & proc, and have a test, reading
some files on the PC & some on the M/F. It will even read DLI & DB2 tables
on the M/F. So we have a full size test copy of a major database in our test
environment. I can access it from the PC, and don't have to cut it down to
XFER to the PC. I can run a sequential OSAM read and get the same
records the production application runs. So I can do a 10,000,000 record
BEFORE/AFTER, all on the PC. I haven't tried the CICS yet, but if it's
like the rest it should be cool. I even use MFE's drag'n'drop file transfer,
when I'm not going to use the files in MFE - it's better/faster than my
emulator
(Extra) file transfer facility. The MFE D'n'D uses a separate logon to the
mainframe, separate from your emulators, and it can then act a bit like
TSO/ISPF's 3.4 option. Set up the DSN range, set up a destination, you can
even set up a semi-permanent conversion (EBCDIC->ANSI), for everything
 going to that particular destination.

So, I give MFE lots of votes. But then, hey.......I didn't have to pay for
it.
```
