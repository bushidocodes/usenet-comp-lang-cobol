- **Message-ID:** `<6u339h$rqt$1@news02.btx.dtag.de>`

```
I have recently started to recompile some projects which I created with the
Microsoft COBOL 5.0 PDS under Windows 98 and found that the behaviour under
W98 is totally different.
Under W95 and the earlier W 3.11 I used a rather complex directives file
(COBOL.DIR) to control the compilers behaviour.
After updating to W98 everything changed in that many COBOL instructions no
longer behave as one would expect:
For example: The STRING END-STRING structure no longer strings text. The
SCREEN SECTION and general screen-I/O is different in that the PROMPT
instruction loses its functionality. There must be many other instructions
that lost their standard functionality.
Has anybody come across anything similar using MS COBOL 5.0 or MF COBOL/2
3.x? Is there a solution?
I do NOT intend to update to any release of MF COBOL - hate Dongleware!

Any comment is appreciated. Please answer by email.
```
