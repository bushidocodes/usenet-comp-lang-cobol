First of all I would recommend amending the User Id that the web
server is using to a Administrator Id as that would at least rule out
a permissions problem. IIS by default uses an account IUSR_MACHINENAME
that only has guest rights. 

Check out the NT Application event log. IIS can sometimes log a
message that gives a clue as to the cause of the failure.

Try updating the PATH (and restarting the mahine) with the location of
the DLL (or put it in the windows system folder).

If your using MERANT Net Express then you could try putting a call to
CBL_DEBUGBREAK as the first statement in the program to see if it gets
that far and also make sure that you have built you DLL using the
multi threaded runtime.

Try also a simple C routine via JNI if that does not work check with
the supplier of JRUN to see if it supports JNI.
