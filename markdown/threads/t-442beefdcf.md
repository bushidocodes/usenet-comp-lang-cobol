'Nightfly' also posted this question in a BASIC newsgroup.  So either
it is a troll, or 'Nightfly' is interested in the subject generally.

I recommend any or all of the following books:

    "Teach Yourself COBOL in 24 Hours", Thane Hubbell

    "COBOL Unleashed", Jon Wessler, et al.

    "Algorithms + Data Structures = Programs", Niklaus Wirth

    "Algorithms in (pick language)", Robert Sedgewick

    "The Art of Computer Programming", Donald Knuth

The above books all cover data structures and basic algorithms.  The
Knuth books (there are several in the series) are considered by many
to be 'The Programmer's Reference'.  However, only the first two are
specifically COBOL books.  "Teach Yourself COBOL in 24 Hours" is an
excellent introduction to COBOL.  "COBOL Unleashed" is a more advanced
book for those who already know at least some COBOL.  Both books come
with free COBOL compilers on CD-ROM, and the authors (several for the
'Unleashed' book) frequent this newsgroup.  Sedgewick has written
several books in the "Algorithms in ..." series (e.g. "Algorithms in C"
and "Algorithms" which uses Pascal), but I don't believe there is one
specifically for COBOL.  "Algorithms + Data Structures = Programs" is
a classic that I consider a 'must read' for any serious programmer.
The language used is Pascal naturally, for Wirth (pronounced 'Veert')
is the inventor of Pascal.

In COBOL the variables or fields are described in the top part of the
program.  To describe them they use a Picture Clause which has the
letters PIC a space then the data type followed by parentheses(spell)
with the size of the field in it.  The programs I have done have fields
with a set field length.

EXAMPLE:  01 ENE-OF-FILE PIC X(03) VALUE "YES".

X is for an Alpha-Numeric field.  In COBOL there is no string class
like JAVA has or some variations of C++.  However there is a STRING
command that allows concatanation, etc. "A" would be for strictly
Alphabetic fields and isnt used much, and "9" is for numeric fields.
There are variations for Packed numbers, hex characters, and binary
data, but I wont go in that direction.  In COBOL you can create tables
or arrays using the "OCCOURS" statement creating a series of fields
that can be referred to with a sub-script method.  For isntance if
TOTAL OCCURS 5 TIMES, then TOTAL (1) is the first one, etc.  In C++
that would be TOTAL[0].  In COBOL the subscripts must start at 1 and be
whole numbers.  SOME compilers let you choose part of a field in the
following manner:

FIELD-1 (5,3)

This translates to the variable "FIELD-1" STARTING AT THE FIFTH ELEMENT
USE 3 CHARACTERS.  This is similar to ASSEMBLY; in fact a Mainframe's
operating system uses ASSEMBLY.  This is a type of referential
addressing.

Pardon my spelling.  As the others stated I suggest you get a
referrence book for complete COBOL SYNTAX.  Another suggestion is a
book on Structured Programming Logic.  If they offer a class in this
subject at your school someone may have one they could let you use.

Charles
