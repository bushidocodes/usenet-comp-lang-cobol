MOVE FUNCTION UPPER-CASE(DISPLAY-FIELD) TO DISPLAY-FIELD.

MOVE SPACE TO BYTE-FIELD-256.
PERFORM VARYING DISPLAY-INDX FROM 1 BY 1

   IF DISPLAY-FIELD(DISPLAY-INDX:1) < "A" OR > "Z"

Sure. Why not?

Word processing programs correct spelling as you type.

Suppose a program had to validate a credit card number but the user
typed the number as displayed on the card (with imbedded spaces).
Wouldn't a user-friendly program squeeze out the spaces before the
validation test?

The NCIS computers (National Crime Information Center) routinely
converts user-supplied alphabetic "I" and "O" to one and zero before
using the query in a serial number look-up.

How about telephone numbers? If the program likes to store telephone
numbers as 1234567890 why not permit
(123)456-7890
123-456-7890
123.456.7890
as well as
1234567890

Same with dates:
01/01/01
01-01-2001
Jan 1, 2001
1 Jan 2001
should all be converted to (say) 20010101 before being tested for
validity.

I, for example, like ROUND numbers. All angular numbers (1, 7, 4,
sometimes 5) are converted to 2, 6, 0, and my favorite, 8, before I
let them in my systems.
