(There is no compiler error reported here on either ACCEPT statement.)

A WRITE statement tells the computer to transfer a whole record from
memory to a file.

You cannot WRITE a single field to a file.  What you can do is ACCCEPT
all the fields into the record.  If and when you are ready, you can WRITE
the whole record, that is

    WRITE ReleaseProfile.

At first blush, this is a simple application, but it gets deep fast.
Notice that there is only one Tracks in your ReleaseProfile record!

One way to have more than one Tracks in your file without modifying your
DATA DIVISION is to write a separate ReleaseProfile record for each
Tracks.  You must then change your PROCEDURE DIVISION to fill in all the
other fields before entering the Tracks loop.  This method has a lot of
duplicate data in the file but no limit on the number of Tracks.

Another way is to modify your DATA DIVISION to have more than one Tracks
per ReleaseProfile record.  In that case, you can leave the Tracks loop
where it is in the PROCEDURE DIVISION, but you will have to choose an
absolute maximum number of Tracks per ReleaseProfile record.  This method
has no duplicate data but a lot of wasted space in the average record.

Another way is to have a separate file containing only Tracks, and point
to them as needed in the ReleaseProfile record.
