After trying to figure out over the last few days how to accept a julian

Here is a solution for a program which is only expected to be in use for the
next four years(max).

The output format is YYDDD.

In working storage I declared a table thus

01 Day-Totals.

05 Month-01 Pic 9(3) value 0.
05 Month-02 Pic 9(3) value 31.
05 Month-03 Pic 9(3) value 59.
05 Month-04 Pic 9(3) value 90.
05 Month-05 Pic 9(3) value 120.
05 Month-06 Pic 9(3) value 151.
05 Month-07 Pic 9(3) value 181.

05 Month-09 Pic 9(3) value 243.
05 Month-10 Pic 9(3) value 273.
05 Month-11 Pic 9(3) value 304.
05 Month-12 Pic 9(3) value 334.

01 filler redefines Day-Totals.

05 Month-Total Pic 9(3) occurs 12.

01 Date-In.

05 In-yy Pic 9(2).
05 In-mm Pic 9(2).
05 In-dd Pic 9(2).

01 Julian-Date.

05 Julian-YY Pic 9(2).
05 Julian-Day Pic 9(3).

01 Date-store pic 9(2).
01 ws-rem pic 9(2).

In the procedure division the code was thus.

Julian-Date Section.

JD-010.

Accept Date-In from Date.

JD-020.

Compute Julian-Day = Month-Total(In-mm) + In-dd.

Divide In-yy by 4 giving Date-Store remainder ws-rem.

If Ws-rem = 0

Add 1 to Julian-Day

end-if.

JD-030.

Move In-yy to Julian-YY.

JD-999.

Exit.

This procedure does not allow for the leap year calculated on century as
the year 2000 is a leap year and the program will not be used much after
this date.
