Is it just one of your clients that has SQL Server 7.0 - or just one who has
COBOL?  If you are using COBOL (but not SQL Server), then almost all the
major players in the WinTel COBOL compiler have a way to reach SQL Server.
(This includes - but is not limited to Fujitsu, Micro Focus, AcuCOBOL, RM,
and IBM).  However, what you need for each of the compilers may vary.
Therefore, if you tell us which COBOL compiler you are using for other
things, we can tell you out to get SQL Server working.  If, on the other
hand, it is "COBOL" that only one of your clients uses, that significantly
impact what we might recommend.
