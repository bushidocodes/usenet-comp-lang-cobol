```
>MicroFocus Cobol without using linkage section?

Is there a reason WHY you want to do this?  If so, it might help us to
answer.

You can use the EXTERNAL attribute on any Working-Storage 01-level item.
Define it this way in both the calling and called program - and its data
will be available in both programs.  Similarly, you could write your data
out to a file in the calling program, then close the file and open it again
in the called subprogram.  However, (as stated earlier) depending on what
you really WANT, these may or may not solve your problems.
```
