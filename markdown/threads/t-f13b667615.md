Ken Foskey <waratah@zip.com.au> wrote:

: Moderately agree, carefully indented paragraphs is no substitute for

And is far from having the most important aspect: getting the
compiler to understand what you are doing.

Never trust someone else's indenting, nor your own, it may fool the
reader, but it will never 'fool' the compiler.

OTOH, if you don't know what you are doing, do it neatly   ;-)

: Code that is crafted (i.e. conforms to what is generally perceived as
: best practice) will invariably be good code.  I personally believe
: that using continue and removing full stops has made my code cleaner
: and easier to understand.  The compiler helps find those misplaced
: periods (full stops).

Absolutely, the more bugs that the compiler can find the less
the programmer has to search for.  I try to program to the
point that a clean compile means that the program will work,
however, I do know the limitations of this.  Putting in every
END- statement possible does allow the compiler to give
more warnings - thus achieving less bugs.

Using silly tricks, such as mixing NEXT SENTENCE and END-IF,
denys the compiler the ability to warn of introduced bugs.
