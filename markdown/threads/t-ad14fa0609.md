```
I've been caught out a couple of times recently when I used
PERFORM VARYING X to access an array.

The first time I used
MOVE ARRAY-SIZE + 1 TO X
to exit the loop pre-maturely.
Which would have been fine except ARRAY-SIZE was 99 and
X was PIC 99.

Another time I used
MOVE HIGH-VALUES TO X
and then expected X = HIGH-VALUES after the loop,
after spending some time finding these bugs I started to look for
thicky-proof ways to avoid them in the future.

I know that VB has the exit-for clause for exiting for next loops,
does COBOL have anything similar?
```

```
The cleanest, safest, and simplest method is to have an explicit
exit variable, e.g.:

MOVE "N" TO PREMATURE-EXIT
PERFORM XYZ
VARYING X FROM ... BY 1
UNTIL X > MAX-X
OR PREMATURE-EXIT = "Y"

...

XYZ.
....
IF it's time to exit
MOVE "Y" TO PREMATURE-EXIT
ELSE
....
.

Even the X > MAX-X test can be dangerous if both X and MAX-X are,
say PIC 99, and MAX-X is 99 (as you have found). Make sure that
the picture has enough digits. Monkeying with the index like
COMPUTE X = MAX-X + 1 is always a bad idea.
```
