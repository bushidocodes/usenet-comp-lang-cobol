I am a big defender of COBOL, but don't think it will make you happy.  It was
defined as stable (boring) business language.  The only reason I would use it
as a hobbyist is because I speak COBOL like a native.  If you want to have a
more complete knowledge of computer languages, I would learn COBOL right
after leaning assembler.  If you want to have fun as a hobbyist, I would lean
towards Java or LISP, depending on what types of application you're
interested in.
