```
I wonder if anyone else has run into this problem: When printing reports
on multi-user systems, I typically open printer files with OPEN EXCLUSIVE
EXTEND in order to prevent two users from accidentally merging their
reports. That works fine if I only want to print text information.
However more and more often, I am required to print graphics such as logos
and signatures on reports along with text. Because this is typically
binary information I don't want to add the form control characters that
records printed to files ASSIGNED to PRINTER always include. So I have
been using a second FD for the binary information which is ASSIGNed to
DISK with the same file name. Then depending on whether I am printing the
text or the graphics portion of the report I close one FD and open
(EXCLUSIVE EXTEND) the other. The problem is the chance of a second user
merging his report into the first if it happens that his program grabs the
file between the close and the open.

I've looked at two possible solutions, but I'm not crazy about either one
of them:

1. Use only the binary FD and add my own form controls when printing
text. This would be a pretty big hassle because the software I write runs
on a variety of OSes that sometimes use different combinations of form
controls.

2. Using the two FD scheme, write the whole report to a work file first,
and then copy it to the printer file opened EXCLUSIVE. This would be
somewhat easier to do, but my biggest client doesn't want anything to slow
down the speed of his printing. Also he worrys about the case when the
printer isn't available for a long time after the disk file is ready.
(The current software immediately warns when the printer is in use.)

Does anyone know of another way with standard COBOL to handle graphics &
text on a multi-user system? What I imagine I would like would be the
ability to open a file exclusively for a single run unit. i.e. Only the
session that opened the file could access the printer, but different parts
of that same session would be permitted to access the printer at the same
time.

I would appreciate any ideas short of changing languages .

Sal
```
