```
vanilla CSV works well for me:

1)  Use " as the text delimiter, and the comma as a field
delimiter.
2)  Double all occurrences of the text delimiter.
3)  Enclose text in text delimiter.
4)  Format numerics such that they do not use the field
delimiter.
5)  Seperate fields with the field delimiter.

So, sample text would look like:

"Name", "DOB", "HourlyRate"
"Jake ""The Snake"" Plisken", 01/01/1960, 45.00

If you're not interested in providing formatting information
above what CSV can offer, you don't even have to use OLE to
drive Excel--you can just open the csv file, and it will
interpret the results.  Excel is not particularily typed, so
that can cause problems, but ones that can be worked around.
```

```
Thought I'd pass on more info about the Import Spec.

 When importing the data (file, get external data, import) select your text
file and work through the wizard.  Always pressing next instead of finish.
The wizard lets you define 'field options'.  This is where you tell it that
it is 'text' and not 'long integer'.  Thus it will keep your leading zeros.

At the end of the wizard, select 'advanced.'  It is here that you will see
the results of the wizard where you can 'save' your 'import spec.'  Then,
when importing again, the first screen of the wizard also has the 'advanced'
button.  Click that to get the import spec you saved.  At this point you
could 'finish'.  It will overwrite your old table (but it also adds primary
key... if you don't want one, continue to select 'next' to be sure to select
'no primary key').  Or, continue to the end and enter a new table name if
you don't want to overwrite the one you created previously.

Note that if Access does not like the way you defined a field for the data
it wants to import, it will create a separate error table.

When your table is ready to be 'exported' (file, export), select excel as
your file type.  The excel file will contain the Access field names in the
first row.  That should be it.  Good luck!
```
