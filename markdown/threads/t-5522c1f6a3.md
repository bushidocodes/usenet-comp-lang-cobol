At the risk of opening a can of worms, I think I can shed some light
on what Ed is probably remembering. There is a COBOL rule that applies
when you have nested PERFORMs. If you branch out of an inner PERFORM,
you must not allow control to reach the exit of the outer PERFORM; the
inner PERFORM must be satisfied first. This is a very easy rule to
break, and various compilers handle the situation differently.

With some compilers, all the exit points of all the PERFORMs currently
being executed are active simultaneously. (I think this is typical
with IBM COBOL compilers.) With other compilers, only the exit point
of the innermost PERFORM is active; if you branch out of the inner
PERFORM and eventually reach the exit of the outer PERFORM without
satisfying the inner one, the return branch from the outer PERFORM will
not be taken. (I know that HP COBOL II works this way, and I think
it's the default for Micro Focus.)

It is very easy to have code that inadvertently relies on one behavior
or the other, and it can come as a rude shock when porting code to a
different compiler or platform. This is not a compiler bug. The
COBOL standard allows both implementations, and it's up to the compiler
writer to choose how to do it. A program that follows the rules of
the standard will never get into this gray area.

Here's some typical code that demonstrates the problem. Can you
accurately predict how this program will behave on YOUR system?

-----cut here-----
IDENTIFICATION DIVISION.
PROGRAM-ID. PERFORM-TEST.
ENVIRONMENT DIVISION.
DATA DIVISION.
WORKING-STORAGE SECTION.
01 TEST-GROUP VALUE "AB1*C23D".
02 TEST-DIGIT OCCURS 8 PIC X.
01 COUNTER PIC 9 VALUE 0.
01 SUB PIC 9.
PROCEDURE DIVISION.
MAIN SECTION.
BEGIN.
PERFORM PROCESS
DISPLAY "SCANNED " COUNTER " DIGITS"
DISPLAY "NORMAL TERMINATION"
STOP RUN.
PROCESS SECTION.
* SCAN RIGHT-TO-LEFT COUNTING DIGITS.
* TERMINATE SCAN IF ASTERISK ENCOUNTERED.
PROCESS-LOOP.
PERFORM SCAN VARYING SUB FROM 8 BY -1 UNTIL SUB < 1
GO TO PROCESS-EXIT.
SCAN.
IF TEST-DIGIT (SUB) = "*" GO TO PROCESS-EXIT.
IF TEST-DIGIT (SUB) NUMERIC ADD 1 TO COUNTER.
PROCESS-EXIT.
EXIT.
FALL-THROUGH SECTION.
STRANGE.
DISPLAY "*** DID NOT EXPECT TO GET HERE ***"
STOP RUN.
END PROGRAM PERFORM-TEST.
-----cut here-----
