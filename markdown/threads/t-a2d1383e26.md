```
IDENTIFICATION DIVISION.
PROGRAM-ID. SAMPLE.
SPECIAL-NAMES.
CALL-CONVENTION 8 IS LITLINK.
*
WORKING-STORAGE SECTION.
*
LINKAGE SECTION.
*
01 WSBUFFER PIC X(10).
01 LSRETURNCODE PIC S9(9) COMP-5.
*
****************************************************************
** ONE POSSIBLE CALLING CONVENTION. YOU NEED TO KNOW WHAT **
** THE CALLING CONVENTION OF THE LANGUAGE THAT IS CALLING **
** THE DLL IS. IF IT IS C/C++ IT IS PROBABLY SYSTEM, OR STD **
** LOOK IN THE ONLINE HELP FOR CALLING CONVENTIONS AFTER YOU **
** KNOW WHICH ONE YOU NEED. IF THE CALLING CONVENTION IS **
** SYSTEM YOU DO NOT NEED THE SPECIAL NAMES OR LITLINK NAME **
****************************************************************
*
PROCEDURE DIVISION LITLINK USING BY REFERENCE WSBUFFER
RETURNING LSRETURNCODE.
*
EXIT PROGRAM.
*
****************************************************************
** SAMPLE MAKE FILE **
****************************************************************
* PROGRAM = SAMPLE
* all: $(PROGRAM).DLL
* #
* $(PROGRAM).DLL: $(PROGRAM).OBJ
* cbllink -d $(PROGRAM)
* #
* .cbl.obj:
* COBOL $*.CBL;
****************************************************************
** OR TO MAKE FROM THE COMMAND LINE YOURSELF WOULD BE **
****************************************************************
* COBOL SAMPLE;
* CBLLINK -d SAMPLE.OBJ
****************************************************************
** Your must either have the appropriate path, cobdir, cobcpy **
** path setup or do this from a NetExpress command prompt **
****************************************************************
```
