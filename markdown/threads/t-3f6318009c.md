Go to cobolreport.com and check Bill Klein's COBOL FAQ (Frequently Asked
Questions), plus the latest copy of cobolreport.com has an article with a
list of sample references. Would be useful to get a copy of Thane Hubbell's
'Teach Yourself COBOL in 24 hours". There are many options to using SORT. To
get you started. here's an example based on Thane's text :-

*>-------------------- collect2.cbl----------------------------

     *> Output sorted into the sequence :-
     *> Public, MethodName then Class

Input-Output Section.
File-Control.
    Select In-File
        Assign          In-Filename
        Organization    Line Sequential
        Access          Sequential
        File Status     In-FileStatus.

    Select Sort-File
        Assign          Sort-Work.

    Select OUT-File
        Assign          OUT-Filename
        Organization    Line Sequential
        Access          Sequential.


Data Division.
File Section.

FD  IN-File.
01  IN-Record.
    05 PublicPrivate  pic x(08).      *> This contains "Public" or "Private"
    05 MethodName            pic x(32).
    05 ClassName             pic x(09).
    05 Params.
       10 XUsing             pic x(80).
       10 XReturning         pic x(80).

SD  SORT-File.
01  SORT-Record.
    05 PublicPrivate         pic x(08).
    05 MethodName            pic x(32).
    05 ClassName             pic x(09).
    05 Params.
       10 XUsing             pic x(80).
       10 XReturning         pic x(80).

FD OUT-File.
01 OUT-Record pic x(209).

Working-Storage Section.

01 IN-Filename pic x(100) value
   "c:\cobol\examples\Collection-Classes\collectA.txt".
01 Out-FileName  pic x(100) value
   "c:\cobol\examples\Collection-Classes\collectM.txt".

01  Done-Flag   Pic X Value Spaces.
    88  All-Done      Value "Y".
01  In-FileStatus Pic XX Value "00".
Procedure Division.
Declaratives.
In-File-Error Section.
    Use After Standard Error Procedure On In-File.
Dealer-Error.
    Display "Unhandled error on In File " In-FileStatus
    Set All-Done To True
    .
End Declaratives.
Mainline.
    Sort Sort-File Descending Key PublicPrivate of Sort-Record
                   Ascending  Key MethodName    Of Sort-Record
                                  ClassName     Of Sort-Record
         Input Procedure Sort-In
         Giving Out-File
    Display "Sort Complete"
    Stop Run
    .
Sort-In.
    Open Input In-File
    Perform Until All-Done
       Read In-File
            At End Set All-Done To True
            Not At End
               Move Corresponding In-record To Sort-Record
               display IN-record
               Release Sort-Record
       End-Read
    End-Perform
    Close In-File
    .
*>-------------------------------------------------------------------------------

Firstly, if you want to see some really good 'stylish' programming - get Pete Dashwood
to send you a sample of his - now Kiwi don't get a swollen head <G>.

As to my own style - not quite sure where this starts. Back with RM/COBOL('74)  it was
of course all Uppercase. Differentiated between  W/S and Files with prefixes like :-

    WS-ITEM-NUMBER
    CUS-PRIME-KEY

Then into MS/COBOL and MF/COBOL and switched to Lowercase - much 'softer' on the eyes
and easier to read. ( I shudder when I see mainframe Uppercase examples here - just
find it hard to read).

Then into OO and found that folks like Will Price were talking 'CamelBack' - so above
became :-

ws-ItemNumber
CUS-PrimeKey

Note I really like to emphasize 'file formats' with caps in the prefix. Another
feature Will Price uses is :-

        Evaluate
        End-Evaluate

In fact I think most of his 'reserved' words are UpperCase. I tend to go with the
'caps' as shown above, but sometimes forget and finish up with :-

    perform
    End-perform - ( There's inconsistency on the first (P)erform  !!!!<G>)

Then move on to OO :-

 invoke anObject "doThis"    - ('ORRIBLE !, 'ORRIBLE !)

I H-A-T-E that 'anObject' and prefer :-

invoke os-CustomerCollection "doThis"

You *know* exactly which object I'm referring to.

Coupled with OO I have my 'own standard' :-

    - Working-Storage (i.e. 'Global'), mostly prefixed with ws-
    - Object-Storage Section (again 'Global') and it IS W/S in COBOL 2002 -
      here I prefix with os-
    - Method - Local-Storage section - prefixed with ls-
    - Method - Linkage section prefixed with lnk- (both using and returning)

Interesting thing about the latter, both Will and I believe J4 examples use the prefix
'ls' for Linkage - just seems more concise as "lnk-".

Now as to the example you were looking at - that's a bit of a mish-mash. Why re-invent
the wheel; took the code from Thane's CD and much of it shows his original code like
the Declaratives and Flags; then I adapted bits to suit my preference.

One thing I'm 'thrilled' about is the Micro Focus feature which lets me leave out the
'red tape' when not required. If you look at my original source I don't have
Program-id - that appears as my first comment line. (I haven't checked but Im pretty
certain I could have left out Thane's lines 'Input-Output Section.' and 'Data
Division.'). The payoff here is your eye is focused on the immediate logic you are
perusing rather than laboriously following the full COBOL syntax.

I'm also a great believer in copyfiles - so that I can prefix with CUS-, ls- or lnk-
as appropriate, with a copy replace ==(tag)== by ==xxxx==.

Really just a question of what appears pleasing to the author, and retains clarity.

As Michael would say "It's the house decorator not the paint roller" <G>. (Want to see
some elegant code - get a copy of some of his BASIC).

> advantages and/or disadvantages?

As a contrast to the example I use 'Modula2' style with all reserved
words in upper case and user words in mixed case.  I find it easier to
visually separate the user words and the reserved, also my editor has
syntax colouring so misspelt reserved words stand out because an upper
case word is coloured wrongly.

However, another reason for this is that my editor generates reserved
words from keystroke macros in upper case - eg F9-P gives PERFORM, F9-V
gives VARYING, etc.

There are two kinds of Cobol sorts:
1. file sort
2. procedure sort
You may combine it: one file in input and one procedure in output,
or reverse.
Here, a sample of each kind in nearly standard 85 Cobol:
1. file sort
       identification division.
      * file sort
       program-id. trific.
       environment division.
       input-output section.
       file-control.
           select entree assign to "input.txt"
               organization is line sequential
               file status fs-entree.
           select sort-file assign to "sorttemp"
               sort status is sort-status.
           select sortie assign to "output.txt"
               organization is line sequential
               file status fs-sortie.
       data division.
       file section.
       fd entree.
       01 entree-enr           pic x(4).
       fd sortie.
       01 sortie-enr           pic x(4).
       sd  sort-file.
       01  sort-enr .
           02 terme            pic x(4).
       working-storage section.
       01 sort-status pic xx.
       01 fs-entree                      pic xx.
       88 eof-entree value "10".
       01 fs-sortie                      pic xx.
       88 eof-sortie value "10".
       01 fs-sort                        pic xx.
       88 eof-sort value "10".
       procedure division.
       tri section.
       x.
           display "Sorting..."
           sort sort-file ascending terme
               using entree
               giving sortie
           display "Sort is finished"
           stop run.

2. procedure sort
       identification division.
      * sort with procedures
       program-id. tri.
       environment division.
       input-output section.
       file-control.
       select entree assign to "input.seq"
               file status fs-entree.
       select sort-file assign to "sorttemp"
              sort status is sort-status.
       data division.
       file section.
       fd entree label record standard.
       01 entree-enr pic x(13).
       sd  sort-file.
       01  sort-enr .
           02 insee pic x(13).
       working-storage section.
       01 sort-status pic xx.
       01 fs-entree                      pic xx.
       88 eof-entree value "10".
       01 fs-sort                      pic xx.
       88 eof-sort value "10".
       procedure division.
       sort sort-file descending insee
           input procedure proc-entree
           output procedure proc-sortie.
       stop run.
       proc-entree section.
           open input entree.
           perform until eof-entree
               read entree end-read
               move entree-enr to sort-enr
               release sort-enr
           end-perform.
       proc-sortie section.
           perform until eof-sort
               return sort-file at end move "10" to fs-sort end-return
               display insee
           end-perform.
