There is a library of routines included in the Micro Focus COBOL product
providing printing support, including the function to print a bitmap. The
functions are prefixed PC_PRINTER_... for the generic 32-bit versions and
PC_WIN_... for the 16-bit Windows versions. Your code will look something
like

call "PC_PRINTER_OPEN" using printer-handle ...
call "PC_PRINTER_WRITE_BMP" using printer-handle ...
call "PC_PRINTER_CLOSE" using printer-handle

Pardon me, but if you are under the impression that you cannot print
graphics using Micro Focus COBOL (on a PC), then you are misinformed.
I wrote an entire low-level graphics library in COBOL for a mainframe
application (COBOL was the only language available), and I prototyped
it on the PC using Microfocus COBOL. Once you know the format of the
graphics image and the printer expects graphics commands, it is simply
a matter of reading the graphics image, crunching the data, and writing
the output image. COBOL may not be the ideal language for such an
application, but it can be done.

In my particular application, it was complicated by the fact that the
8-bit binary graphics data on the EBCDIC mainframe had to be transmitted
over a 7-bit ASCII communications line to a laser printer attached to a
PC. I had to take the 8-bit binary graphics data on the EBCDIC mainframe
and split it into 6-bit chunks, then translate those 6-bit chunks into
printable EBCDIC characters which when translated to ASCII would give me
another printable character which I could bit-shift on the PC to the
appropriate 6-bit chunk, which when reassembled, reconstituted the 8-bit
graphics data. Though I had to write an assembly language printer driver
for the PC to convert and reassemble the chunks while printing, COBOL
handled everything on the mainframe side. By comparison, simply printing
a GIF or JPG is a piece of cake. ;-)
