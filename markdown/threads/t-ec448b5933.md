Good day... I've been asked to offer a bit of assisstance and it is beyond 
my knowledge; if someone can offer a pointer or two it would be 
appreciated.

A buddy o' mine is impressed with the way I do backups (as mentioned here
earlier I use a Jaz drive and do image-copies with HD98copy) and wants to
do something similar on his laptop.  The problem is... he has an external
SCSI Jaz drive which can attach either through the parallel port or 
through a PCMCIA-to-SCSI card.  When he boots to DOS and attaches the 
drive to the parallel port (using the Jaz traveller) the system will 
recognise the drive (using the Iomega Guest utility0 but transfer rates 
make backing up 10G of disk a weary, weary time.

If he attaches the drive using the PCMCIA card the Guest utility will not 
recognise it as existing; this is, I believe, because DOS does not 
recognise PCMCIA cards unless one does... something.

Is there anyone who can point me towards what this 'something' might be?  
I have searched on 'card and socket services' but it seems that this only 
directs me towards people who want to sell me software without any 
guarantees.

Thanks much!

DD

It sounds like you need an MS-DOS mode driver.

DOS has parallel port support built in, but no PCMCIA support.  The 
manufacturer of the PCMCIA-to-SCSI card might have a DOS only driver.  
That would be a sure fix.

In the event that they do not have a DOS driver for the card, can you 
run the HD98COPY from a DOS window in Windows?  In that case it would be 
the Windows driver that would handle the interface with the card.

Finally, if a DOS driver is unavailable, and HD98COPY will not run under 
Windows, you might be able to set up a lightweight Linux disk with 
support for the cards that would do the image copy using different 
tools.  This could be as simple as a floppy with a kernel supporting the 
SCSI card and the DD utility (Device to Device copy).

Best bet is a DOS driver though...
