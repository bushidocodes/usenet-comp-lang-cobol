inadvertantly created a record larger than what your 01 Level says it
is.

important here.   In addition, since we don't know the length of the

You had said that you expected the data of the record to be 100
characters. Given that you haven't supplied the lengths relevant to
the string operation this may give the actual 100.

With a line sequential the trailing spaces will be removed and a CR/LF
added. You haven't supplied room for this. It is possible that there
is an issue with this particular compiler version when there is no
room for the terminator to be added. The program would then write out
the complete block buffer which contains nulls for the remainder.

The solution/avoidance is probably to make the record area larger:

        01   SPULE-PA  PIC X(256).

Some experimentation with various sizes may show that this particualar
Microfocus version has a bug when the last character of the record is
non-space.
