ActiveX Data Objects (ADO)

The Fujitsu ADODataSourceControl is a wrapper around two Microsoft Classes:

1. ADO connection object
2. ADO resultset object

The Fujitsu control has no methods of its own, but you can use the standard
MS methods documented on MSDN.

Looking at the code you posted...

INVOKE "Recordset" OF CmADODataSource1 "AddNew"


This is meaningless. You need to pass the optional parameters to it. If you
don't, it is trying to insert an empty row on an empty dataset. There is
little chance of that ever working...
You cannot pass these parameters by CONTENT because it is an ActiveX (COM)
control; you must set them up in ws and pass their addresses (BY REFERENCE).
If you use the same kind of values you would use on an SQL 'INSERT' (A field
list followed by a list of values for each field), it will work.


INVOKE "Recordset" of CmADODataSource1 "AddNew"
USING wsFieldList,
wsValueList
RETURNING wsErrorCode
END-INVOKE


Here's the full story:

https://msdn.microsoft.com/en-us/library/ms677536(v=vs.85).aspx

Also, make sure that CmADODataSource1 is not null... (Step through and check
it in the debugger...)

Espero que esta información te ayude,
