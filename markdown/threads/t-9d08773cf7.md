The negative of a binary is often stored in 2's complement notation.  Each bit is
inverted (1's complement at this point) and 1 is added to the result to get 2's
complement.  Also, the leading bit of the number indicates the sign.  This limits
the highest signed binary two byte field to the value of 32767.  It works like
the old adding machines where if you subtracted a large value from a small one,
you got a lot of leading 9's.

+4 = 0000 0000 0000 0100
-4 = 1111 1111 1111 1100

+7 0000 0000 0000 0111
-4 1111 1111 1111 1100
------------------------------
+3 0000 0000 0000 0011

Note when the 1's get added in the 3rd from the right position, the carry gets
propagated all the way to the left end.  HTH

I am taking a shot at this, but keep in mind that everything I say is only true in respect
to MVS etc:

Also, all the examples are for 2 byte hex data only, appropriate changes have to be made
for 4 or 8 byte fields...

An UNSIGNED BINARY number is stored as its direct hexadecimal representation, i.e. it can
have a range of 0 to 65535 (X'0000' TO X'FFFF')

77 VAR-1  PIC 9(4) BINARY VALUE '37212'.

would store as X'915C' (as long as the TRUNC compiler option allows it)

A SIGNED BINARY number has the sign in its first bit position.  If it is 0, the number is
considered positive (or zero), if is one the number is considered negative.  The range
therefore is -32768 to + 32767 (X'8000' to X'7FFF')

A positive signed number will just use the standard hex representation, just as the
unsigned number. 

A  NEGATIVE signed number uses the 2-complement format:
As an example, let us use the number -28324

Take the absolute value of the number in question (28324)
Convert to hexadecimal representation X'6EA4'
Flip all the bits ( i.e. 1 --> 0, and 0 --> 1)
This is an XOR operation with X'FFFF' and yields X'915B'
now add 1, giving X'915C'

This is the requested negative number -28324
Proof:

-28324 + 28324 = X'915C' + X'6EA4' = X'0000' + carry = 0

Depends on the architecture (hardware). The two systems that I am aware of are
twos-complement and complement. 

In the complement system one flips every bit. There is both a positive and
negative zero.

In the twos-complement one flips every bit and adds one. The highest negative
number is one more than the highest positive number.

In both systems a number is negative if the first bit is on.
