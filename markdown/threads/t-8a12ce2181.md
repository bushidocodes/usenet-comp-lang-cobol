typedef struct t_names

The C sprintf function will expect that nameslist[n].firstname will be null
terminated.  The Cobol that I use will not give the C subroutine a null
terminated string - they will be space padded.  You would first have to scan
backwards from the end of namelist[n].firstname for the last non-space
character and stick a '\0' after it if you want to use sprintf or any C
library function that deals with strings.

The sprintf will add a '\0' to the end of the string and this could be a
problem
depending on what the COBOL program is doing with the data. Also, if the C
program uses
a string variable in the sprintf statement and that variable contains a
string longer
than 19 characters (one needed for the null terminator) it will flow over
into the
next element. This is not what you want, this is not how COBOL acts.

I would use memcpy to copy up to a max of 20 characters and if there are
fewer than 20 characters use a second memcpy to pad with spaces.

OCCURS is not allowed on an 01 level.

"The OCCURS clause should not be specified ... at the 01 level or as a
77 level-number."

> CALL 'FunctioncalledbyCobolprogram' USING WS01-NAME-REC.

level and try again.
