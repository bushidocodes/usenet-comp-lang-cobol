### Optimizing mF COBOL on AIX

- **Date:** 1996-12-27T19:00:01+00:00
- **Newsgroups:** comp.lang.cobol
- **Message-ID:** `<5a450m$rdc@flood.xnet.com>`

```

This is a request for techniques used to optimizing mF
COBOL 4.0 on AIX 4.X.

FACTS AS I SEE THEM:
The programs that I am trying to optimize are already being compiled
to a statically linked executible.

The -O flag on the cob command seems to have no effect on run-time. I
confirmed this with mF tech support, but the support person's
non-committal style of answering the question did not inspire my
confidence in his answer. (I did request that mF change the
documentation).

My need is to reduce run-time from 14 minutes to 7 minutes (at least).

I am interested in techniques that do not require knowledge of the
code. Another person in my company is working on techniques that
involve code knowledge (such as changing the code or using the
-C OPTXXX compile options.) He is also planning on using the profiling
techniques.

MY QUESTIONS:
1) Is the -O option really useless on AIX?

2) Is there a third party object code optimizer available for AIX?
In particular the SMP PowerPC version. what I am looking for
would read the .o file produced by the compiler and output
another optimized .o file.

3) Do you know of other methods of optimization? If so, please send them
to me.

4) Is there a disassembler available for .o files on AIX?
I would like to use it to see how efficient the mF COBOL compiler is.

5) Is there a way to get the mF COBOL compiler to output assembly
language? I know there is a way to do that is 16 bit environments,
but I have not found a way to do it on AIX. Even undocumented
methods would be fine with me.

Michael Potter
pot··.@li··p.com
```
