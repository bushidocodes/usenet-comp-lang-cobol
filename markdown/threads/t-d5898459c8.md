The kind of output you see in Moicrosoft Outlook Express is a standard
Windows(r) Listview control, into which "sort on the clicked column" is
built as a standard feature. The listview control is part of the "Common
Controls" series of the Win/32 API.

I don't know how Net Express implements the Listview control (I do not write
in any any language using the "properties and methods" syntax; I use the
"raw" API calls),  but I would think that implementation would include the
sort interface.

Now, the Outlook Express screens which feature multiple panels which
automatically resize to fit inside the client area of the "main" window
without overlapping each other, that's horse of another color. (Or at least
it was about a year ago when I started playing with it).

I use GUIs and OO, not Dialog System. The only thing I can suggest is that you
look at the demo programs for Dialog System when you get your copy of N/E - as I
said, I don't use D/S but am aware it has more whistles and bells since your
last use.

As to whether you can do that 'header click' for ascending/descending, you'll
have to check out. Failing that, it might be a partial use of features in Dialog
System and using some OO methods to achieve what you want.

Best bet, I would suggest, is ask Merant directly.

PS: Before you ask - I don't 'header click' in OO. Where necessary I have three
buttons, associated with a Listbox,  allowing user to select ByName (default),
ByKey and ByOther (if appropriate); I then go back to my (Sorted)collection,
rearranging the listbox text so that I get a complete sort of data, (all
columns), ascending.
