```
In message <<1995Aug29.165321@wombat>> ha94mayp@wombat writes:
› convert a 5 digit number into words (ie: 20345 ---> Twenty thousand three
› hundred and forty five)

Something along the lines of:

MOVE SPACES TO Output-Buffer
MOVE 1 TO Output-Pointer

IF Number > 999
DIVIDE Number BY 1000 GIVING Word-Index
IF Word-Index < 20
STRING Teens-Number(Word-Index) DELIMITED BY "|"
INTO Output-Buffer
WITH POINTER Output-Pointer
ELSE
DIVIDE 10 INTO Word-Index
GIVING Word-Index
REMAINDER Unit-Index
STRING Tens-Number(Word-Index) DELIMITED BY "|"
INTO Output-Buffer
WITH POINTER Output-Pointer
IF Unit-Index > ZERO
STRING Unit-Number(Unit-Index) ....
END-IF
END-IF
DIVIDE 100 INTO Number
GIVING Word-Index
REMAINDER Number
STRING "Thousand " DELIMITED BY SIZE
INTO ....
END-IF

IF Number > 99
etc etc

The various words should be given a value such as "Two |" to cater
for the delimited by, or perhaps you could use the space as a
terminator and then add a " " Delimited by size after each word.

The WITH POINTER allows each successive string to add to the output,
rather than starting at the start again.
```

```
the math operation modulus.

then use string to string the literals together.

divide ws-accumulator by 10000 giving ws-result remainder ws-remainder.
ws-result will show your 10000's digit. repeat until out of digits.
but each time, divide by a lower power of 10, getting the digits
from left to right, moving ws-remainder to the ws-accumulator at
the appropriate time.
```
