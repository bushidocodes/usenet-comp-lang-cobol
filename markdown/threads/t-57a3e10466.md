The first line of the source file set the conditions.

     1$set ans85 flag"ans85" flagas"s"

Allow ANS85 syntax, FLAG non-ANS85 usage, FLAG violations AS
severe.

With FLAGAS"S" no code is produced.

In the absence of FLAGAS (or with other options for FLAGAS),
code is produced and appears to work correctly.

As I understood it, the idea was presented as a means to provide an
alternative to a long string of END-IF scope-delimiters. I also understood,
perhaps inferred, that the alternative would work with all translators.

In using directives to enforce the strict ("tight-assed J4") interpretation,
I confirmed that it may not work with all translators.
