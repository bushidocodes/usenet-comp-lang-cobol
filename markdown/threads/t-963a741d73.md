Hi,

After establishing the connection to a server process, cs-binding gives
you some information in lnk-client-id and lnk-tagname. afaik lnk-tagname
is built combining the servername from mfclisrv.cfg and a consecutive
number.
If you have any further questions, you may ask me.
CS Binding - to my mind - is a very easy and cheap way for small client
server solutions, once it is set up correctly (!).
The documentation, that comes with NetExpress is very poor, especially
if you want to do real two-way communication with your server. So you
depend on try and error :-(
