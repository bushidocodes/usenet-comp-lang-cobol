I might be able to explain the differences, in COBOL, by taking
some liberties with the COBOL language.

Each paragraph/section has, associated with it, an active next
address and a default next address. At program start-up these
addresses are the same.

IBM (see the comments, below) and RM appear to be similar
in using memory locations to hold the active next address.
Given the statement:

Perform para-A thru para-C.

Something like the following will take place at the Perform.

    move address of ra-000 to active-next-address of para-C.
    go to para-A.
ra-000.

At the ends of para-A thru para-C, the following will occur.

para-A.
    ...
    move active-next-address of para-A to next-address.
    move address of para-B to active-next-address of para-A
    go to next-address.
para-B.
    ...
    move active-next-address of para-B to next-address.
    move address of para-C to active-next-address of para-B
    go to next-address.
para-C.
    ...
    move active-next-address of para-C to next-address.
        *> at this point next-address has the return address, ra-000
    move address of para-D to active-next-address of para-C
    go to next-address. *> return from perform
para-D.

Since the active next addresses for paragraphs A and B were
unchanged from there initial values, they essentially "fall-thru"
to the next paragraph.

For a stack-based implementation, the following may be done.

At the Perform:

    move address of para-A to next-address.
    perform
        call next-address *> leaves return address on stack
        if next-address not equal default-next-address of para-C
            exit perform cycle
                *> repeatly call the next address until addresses match
        end-if
    end-perform.

At the ends of para-A thru para-C, the following will occur.

para-A.
    ...
    move default-next-address of para-A to next-address.
        *> default-next-address of para-A is address of para-B
    exit paragraph. *> essentially return
para-B.
    ...
    move default-next-address of para-B to next-address.
        *> default-next-address of para-B is address of para-C
    exit paragraph. *> essentially return
para-C.
    ...
    move default-next-address of para-C to next-address.
        *> default-next-address of para-C is address of para-D
    exit paragraph. *> return with matching address
para-D.

The example given as IBM is more like COBOL F, not VS
and later. As I understand it, VS and later has two active next
addresses available.

One difference between IBM and some others, is that IBM
treats the end of a section as the same location as the end
of the last paragraph of that section. RM and MF treat the
end of a section as occurring separately and after the end
of the last paragraph of that section.

As I mentioned elsewhere, both RM and MF have options,
for well-structured programs, to eliminate the normal
"fall-thru" processing replacing it with simple CALL/RET.
This improves performance.

For a given computer, a stack-based implementation will
give better performance than non-stack methods when
performing single paragraphs. This is because the
stack-based solution need not check for the end of a perform
range for that single paragraph. A non-stack method must
always work with its active and default next addresses even
if the perform was of a single paragraph. (Whether the
implementation gives better overall performance is debatable;
but the performs will be faster.)
