```
In preparation for giving a brief dump debugging session using the
CEEDUMP, I found an anomaly when the abend occurs in a SORT OUTPUT
PROCEDURE.  The COBOL compiler is COBOL FOR MVS and VM.  LE is the LE
for OS390 release 2.5.  I found that the record area associated with the
SD doesn't appear in the dump and if the SD is not the last descriptor
in the list the BLF numbers for the files after the SD are off by 1 (see
example below).  This is a minor annoyance for the CEEDUMP in R2.5 (LE
1.8 I think) is a vast improvement over that associated with OS390 1.3
(LE 1.5).  You can find the record last read (or to be written) for a
file by doing a find for BLF 1 20.  The indexes are before the first
BLF.  The WORKING STORAGE can be found by doing a find on BLW 1 20 while
the LINKAGE SECTION can be found by doing a find on BLL 1 20.  The
CEEDUMP is manageable even if you blow up in a SORT as was done
intentionally in this example.  

I suspect that the BLF associated with an SD is a pseudo BLF (Base
Locator File) and that LE is unaware of it because there probably is no
associated FIB (File Information Block).  The reasons that this anomaly
concerns me is that for programs with SD statements at other than the
end, the BLF numbers do not match the listing.  This will be even more
confusing if the records on one or more of the files or associated with
the SORT file are over 4K in length thus causing multiple BLF numbers to
be assigned (BLF-n for the first 4K, BLF-n+1 for the second 4K, etc.). 
Given that in general it is easy to correlate the information in the
dump to the COBOL listing, this is a shame.    

The program File Section is as follows where the DDNAME associated with
a file is the same as the name of the file in the FD statement: 

FD  FILEA BLOCK 0.
01  RECORD-A.                    BLF-00
    COPY AREC.
     snip
SD  SORTWK01.
01  SORTWK-RECORD.               BLF-01
    COPY SREC.
    snip
FD  FILE-B BLOCK 0. 
01  RECORD-B.                    BLF-02 
    COPY BREC. 
FD  FILE-C BLOCK 0.
01  RECORD-C.                    BLF-03  
    COPY CREC.                    	

The CEEDUMP has

BLF-00 FILEA

record for FILEA

BLF-01 FILEB

record for FILEB

BLF-02 FILEC

record for FILEC

rest of program.

Clark Morris, CFM Technical Programming Services morrisc@nbnet.nb.ca
```
