I'm in the middle of a slow migration of my COBOL
text based application to Windows.

My application writes it's reports to a
temporary, relative file.  In this file I have
"Control Words" that are interpreted by my print
program to change form 10cpi, to 16cpi and back,
bold face, and when to page eject.  When my print
program encounters one of these Control Words, it
searches a table, looking for the selected
printer, to emit the proper escape codes.

In early releases of Windows 95 this worked
fine.  When the OEM release of '95 came out I
began having problems where the escape code were
ignored.

As a solution, I wrote a new version of the print
program in Micro Focus Cobol, using the PC_Print
commands.  This has been a considerable
improvement when I use a Mono Spaced Font such a
Courier New.

The problem I have now is that when I go into
compressed mode (Pitch 6), my 6 lines per inch
spacing shrinks down to about 11 lines per inch,
making it very difficult to read, or produce
faxable pages.

I tried various fonts with no luck and the Micro
Focus functions seem to only want to use True
Type fonts.  I'm thinking that the Courier 10,
12, 15 bit map font may solve my problem.

Can anyone recommend how to do any of the
following:

1. Maintain the default font built into a printer
and maintain the 6 lines per inch print.
2. How do I set printer line spacing using Micro
Focus Cobol.  I'm willing to use API calls if you
can point me twards and example or even the API I
need.
3. How can I get Micro Focus Cobol to recognize
Bit Map Fonts?

Any suggestions or new directions would be of
great help.

    Most printers (Some of the bubble jet printers from Canon and others are
exeptions) use more or less standard HP escape codes.  PCL-4 is basic and
widely accepted.  Although some of the PCL codes are printer specific, stuff
like lines per inch are reasonably standard.

    Also note that many printers have a printer resident font called "line
printer" that works well at 17 pitch. (132 characters on a std 8.5 inch
peice of paper).  If you use line drawing characters you will need to adjust
the character set as well (PC-8 or some such).

    Problem:  HP does not give these books away.  But you might get summarys
from the HP web site.

If you use the windows printer driver, you possibly have to use Cobol's windows
printer functions for printing which possibly does not allow escape sequences
because it uses the windows fonts. 

Not the state of the art but the following method is simple, fast and
easy-to-use:

1. print into a sequential file
2. then copy the file to the good old lpt1

BY this you use the printer-built-in fonts that can be changed (aside with page
orientation etc) by escape sequences.

You may also do the two steps in one program, but performing two steps allows
you to write a copy program that translates your own control words into PCL5 or
other control codes. With PC_print, you are delimited to use one font for the
whole document.

If you use a network printer, simply assign it to a lpt connection with the net
use command.

Hoe this helps

    Note that lines per inch refers to verticle, ie top to bottom. Lines per
inch has NOTHING to do with pitch or font size.  Pitch and font size are
both different ways of saying the same thing, ie how WIDE (not tall) is my
stuff.


;*******************************
; Portrait, 6 LINES PER INCH
132-COL-XDATA=1B
132-COL-DATA=&l0o6D
; PC-8 SYMBOL SET
132-COL-XDATA=1B
132-COL-DATA=(10U
; Fixed, 16.67 pitch, height=8.5 points, upright,
; Medium (not light or bold), Line printer
132-COL-XDATA=1B
132-COL-DATA=(s0p16.67h8.5v0s0b0T

    Above - XDATA means hexadecimal (normally escape)

    Note that Courier also works well for print outs.
