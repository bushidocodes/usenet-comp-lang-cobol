Orbix is a software environment that allows you to build and integrate
distributed applications. Orbix is a full implementation of the Object
Management Group's (OMG) Common Object Request Broker Architecture (CORBA)
specification.  CORBA objects are standard software objects implemented in
any supported programming language. CORBA supports several languages,
including COBOL, PL/I, C++, and Java.

The first step in developing a CORBA application in COBOL is to define the
interfaces to objects in your system, using CORBA IDL. You then compile
these interfaces using an IDL -> COBOL compiler - which is available from
IONA as part of their COBOL solution.  This IDL compiler generates COBOL
code from IDL definitions, and produces the following:

1) Server skeleton code, which allows you to develop the mainframe COBOL
server programs which can accept CORBA requests from off-platform clients,
and

2) COBOL copybook files that define the program attributes and operations in
COBOL format based on the IDL interface for this service

CORBA Clients and servers communicate their request/ replies using the IIOP
protocol which runs over TCP/IP -  i.e. a C++ or Java client on Windows or
Unix can access a COBOL program running in Batch mode, IMS or CICS on OS/390
using IONA's Orbix, iPortal and iPortal OS/390 Server products.
