No IBM mainframe compiler (that I know of) supports -INC directly.  It is
supported by Librarian (unless I have it confused with Panvalet - one
uses -INC and one uses ++INCLUDE - but they both work basically the same.)

In most cases, these statements are handled by a "preprocessor" step.
However, CA (which owns them both) also supports a mode where members can
actually be handled as if they were COPYLIB members (via the COPY
statement).

Finally, with the newer compilers (I think anything after VS COBOL II,
Release 3.0) it is also possible to code (or have code) which is handled via
an EXIT compiler-option.  This to could "intercept" (and handle) -INC or
++INCLUDE statements.

A few other "tidbits" of related information:

A) Micro Focus provides compiler support for -INC and ++INCLUDE statements
directly (as an emulation of the mainframe products). Other PC vendors (CA?
IBM? Fujitsu?) may also support this syntax - I don't know.

B) All IBM mainframe compilers (at least OS/VS COBOL and above) support
"INSERT" "DELETE" and "REPLACE" statements. (This replace is NOT the new
ANSI'85 standard replace - which can cause confusion).  Although I know of
this feature, I personally have only run into one shop that ever used it.

As Bill Klein wondered, the "-INC" feature/function is supported by
Librarian and it is resolved in the Librarian step. This means you can
-INC procedure division code containing CICS commands and not have a
problem; until you decide to migrate to Endevor or some other software
which wants COPY statements (which get resolved in the COBOL step).
