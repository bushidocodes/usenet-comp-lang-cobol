The  non-subordinate fields within a group  that follow an  Occurs
Depending On are variably located. Their address is determined at the
time of reference. An 01 level following an ODO is not affected.  For
example:

01  ws-depends   PIC S9(4) COMP.
01 the-group.
    02  a-byte OCCURS 1 TO 10 DEPENDING ws-depends
                    PIC X.
    02  a-field-that-follows PIC X.

MOVE 10 TO  ws-depends
MOVE 'Z' TO a-field-that-follows .
DISPLAY a-field-that-follows               <<< displays a 'Z'
MOVE '1234567890' TO the-group.   <<< a-byte(ALL)  contains the entire
string and a-field-that-follows is located in position 11 of the group.
MOVE 3 to ws-depends.                     <<<<only 3 bytes of a-byte( )
are available, and the COBOL thinks a-field-that-follows is now in
position 4.
DISPLAY a-field-that-follows               <<< displays a '4'

Unless you intend to write an ODO table to an external medium, there is
no advantage to using an ODO in working storage.  As I stated earlier,
the table will still occupy the maximum size, only not all bytes will be
able to be referenced,  and the fields that follow will keep slip sliding
away.

Hope this helped.
