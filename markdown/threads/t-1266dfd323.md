For sorting a small table like this, an insertion sort works well.
An insertion sort works by assuming the beginning of the table is
in sequence (initially, only the first element in the table).  Then
each element beyond the last sorted element is inserted into the
sorted part of the table by saving it in a temporary hold area,
moving all of the elements in the sorted part of the table up by one
position, until an element is found that is not greater than the
element to be inserted, or you drop below the first element in the
table.  Then the saved element is inserted in the position above the
position being examined (which would be 0 + 1 = 1 if the new element
is lowest).  I would give you code or pseudo-code if this were not
homework. :-)
