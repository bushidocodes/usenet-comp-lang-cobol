```
What????? Not at all!

The progression is normally:

OS/VS COBOL -> COBOL II -> COBOL/370 -> COBOL for MVS -> COBOL for OS/390 ->
Enterprise COBOL

The biggest jump is OS/VS COBOL -> COBOL II
Some shops went OS/VS COBOL -> COBOL/370
Or even OS/VS COBOL -> COBOL for MVS
Or, in at least one case, OS/VS COBOL -> COBOL for OS/390

These four jumps are relatively the same magnitude.


Now consider COBOL/370 -> COBOL for MVS -> COBOL for OS/390

these steps are easy, almost trivial (one issue: new reserved words that you
have been using for data items or paragraph names in earlier code; but not real
likely).

>
>Shall I use the cookbook for converting OS/VS COBOL to COBOL/390 for
>my migration from COBOL/370? Is there anything specific I need to take
>care?

If you are at COBOL/370 and converting to COBOL for OS/390, just recompile and
re-bind. Odds are there will be few, if any, problems. [I would recompile and
bind from the lowest level subroutines up.]


Hope this helps.
```

```
As Steve says, upgrading from IBM SAA AD/Cycle COBOL/370 code to IBM COBOL
for OS/390 & VM (or the even newer IBM Enterprise COBOL for OS/390 & z/OS)
is normally QUITE simple.  It is discussed in the current IBM Migration
Guide at:

 http://publibz.boulder.ibm.com/cgi-bin/bookmgr_OS390/BOOKS/IGY3MG00/4.5

There is ONE thing that *might* cause you some work.  If your existing
programs are compiled with the CMPR2 compiler option, this option is still
available with COBOL for OS/390 & VM - but goes away with Enterprise COBOL.
Therefore, you should study the migration requirements for this change.
```
