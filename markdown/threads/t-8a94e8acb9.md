Question
========
In Communications Manager/2, what is the difference between CPICOBOL.DLL
and CPIC.DLL? Is there something different about COBOL calling convention
that requires a different DLL? Would this same problem exist on other
platforms like Windows/NT and SNA Server? (The DLL I am using there is
WINCPIC.DLL)

Background
==========
I have a CPI-C application working in production using cm/2 V2.1 and
higher. The application is written in Microfocus COBOL. It works great.

I want to port this application to Windows/NT using MS SNA Server V2.1
(Intel Pentium Processor). Again I am using Microfocus COBOL (32bit sdk for NT).
Some of our call parameters seem to be mis-interpreted.

I am aware of the big versus little-endian architecture problem. SNA
Server, like CM/2, is expecting integers in native Intel format.
