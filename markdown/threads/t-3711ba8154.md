I can't speak to the scrollbar or control implementations in
Totem, but if you must do this, the logic would be:

1)  When a control gets focus, determine if it is visible.

     You may need to inspect the scrollbar's value and use
that, with information about the client area of your window, to
calculate it's position relative to that client area.

2)  If it is not visible, determine the closest scrollbar
position that would make it visible -- and set that value.

It seems unlikely that anything native to COBOL will help here.
The Windows API could be used, I suppose, but any information
you need _should_ be exposed by the control's properties.
