Probably the best method is to declare a USAGE PROCEDURE-POINTER
variable, then set it to the names of each of the DLLs you need to
use.  I'm assuming they're DLLs; if they're static libraries, then I
think you may have to compile your COBOL modules to objects and link
them into an executable with the libraries.

So for the DLL route you'd have something like:

       77  dll-loader          usage procedure-pointer.

       ...

       set dll-loader to entry "calwin32"
       if dll-loader = null
          [handle error]
       end-if
       set dll-loader to entry "clxwin32"
       [and so on]

(That's untested, but looks right off the top of my head.  I think
it should work in Object COBOL.)

I've just learned that this check doesn't catch loading failures in
earlier versions of MF Net Express, so it probably won't work in
Object COBOL either.  In NX4 a failure to load a DLL does set a
procedure-pointer to null, but in earlier versions it would set it
to point to a routine in the RTS so that an attempt to call through
it would be trapped.  (Now the RTS just catches the attempt to call
through a null procedure-pointer.)

In any version of MF COBOL that supports procedure-pointers, you can
check for load failure like this:

       77  dll-not-found        usage procedure-pointer.


       set dll-not-found to entry "somenamethatwillnotbefound"
       set dll-loader to entry "calwin32"
       if dll-loader = dll-not-found
          [handle error]
       end-if

(I've actually not done very much with the COBOL side of things
until very recently, so my experience is almost exclusively with
NX4 / SX4.)
