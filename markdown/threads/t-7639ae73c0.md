The client is running under windows, so you don't actually have a user-id in
the unix sense. If you are looking for the username (i.e. what you logged
onto windows as), we accomplished that by creating a new c call and
recompiling and linking the runtime (also created one for workstation name).
This was the easiest, but not the best way of doing it. If I had the time to
do it over, I'd do an external cobol subroutine that loads and calls a c
dll - that way you don't have to worry about relinking the runtime every
time you get an upgrade.

On that subject, if anyone has actually written a cobol program to call an
external dll, any sample code would be appreciated - the docs that acucorp
provide are thin at best.
