Was this code sample cut and pasted into your message, or did you type it in
fresh? If you had typed it in fresh, go back to the original and make sure you
have a period outside of the quotes on the line,

DISPLAY "Move to JUNK1 Failed".

If there is no period after the closing quote in the original, high values
wouldn't have been moved to JUNK2. Misplaced periods in the IF construct can
make hard-to-find problems.

If you are using a compiler that conforms to the 85 standard, it would be
easier to see if there are misplaced periods if you use END-IFs, e.g.,

MOVE HIGH-VALUES to JUNK1
IF JUNK1 IS EQUAL TO HIGH-VALUES
DISPLAY "Move to JUNK1 Worked"
ELSE
DISPLAY "Move to JUNK1 Failed"
END-IF

MOVE HIGH-VALUES to JUNK2.
IF JUNK2 IS EQUAL TO HIGH-VALUES
DISPLAY "Move to JUNK2 Worked"
ELSE
DISPLAY "Move to JUNK2 Failed"
END-IF.
