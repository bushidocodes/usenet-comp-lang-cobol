Since you won't tell me where you're running, the answers are:

For OO COBOL you can probably use Microsoft's XML parser, assuming your
compiler can interact with C++ objects.
(http://msdn.microsoft.com/xml/default.asp). There are also Java parsers
available. from Apache, Sun and others if that helps.

There's a product for the mainframe called XML4COBOL
(http://www.xml4cobol.com) which creates assembler code to parse specific
documents based on a DTD. I haven't looked at it in a while so I don't know
if it has kept up with development.

I mucked with a SAX-compliant parser for COBOL for a while before I decided
it was easier to parse the XML in some other language, like Java, and
restructure it into a form that was more readily digestible by COBOL.

Karl
