TCP/IP is a transport. What you need is winsock.ddl which is
the api of access to the tcp/ip transport. Of course if your
applictions are capable of using DDE this could also be used.

Get the winsock 1.1 specifactions which should have all of the
callable routines and the parameters that the called routines
expect and what the calling program will return. These
specifcations were available from Microsoft at one time. They
still may be available.

What type of COBOL are you using? Are you experiencing
any performance problems. Do you think that COBOL runs as
fast as other programming languages that have been around the
PC world? Is it a resource hog?

Sorry for all of the questions, but I have developed a number
of application systems over the past 12 years using the COBOL
language and enjoy using it very much. Lately I've been
learning to use Object Oriented COBOL and again have enjoyed
the learning experience.

Thanks,

Dan Yates

U of I @ Springfield
