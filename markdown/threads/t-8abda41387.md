```
I would appreciate suggestions and comments for a
cobol compiler. My application is as follows...

I have a relatively small program which we use on our
SCO UNIX system. We currently use Ryan/McFarland COBOL.
This application works fine using either the old RM/COBOL-2
or RM/COBOL-85.

I want to sell the program for PC use. The easy part of RM
is that I can simply copy the compiled code over the the PC
and it runs perfectly.

The problem is that this is a low-priced product (under $200)
and RM would want about half of that price for runtime module
licenses. There would be no point in my selling the product.

This is a SIMPLE program. There isn't even any file access.

I need to find a PC compiler that produces output that does not
require a runtime module. I do not wish to make ANY modifications
to the source. We will continue to to do our development work
on the UNIX system using RM. I need to be able to simply copy
the source over to the PC, and recompile.

Everyone's literature seems to say they are "compatible" with
RM source code, using certain compiler options, but I don't want
to spend money *first*, and find the problems *later*. I simply
don't have the time.

Suggestions? Micro-Focus? Microsoft? Accu-COBOL? Others?

thanks,
don

I don't get time to read this conference regularly. I would
appreciate responses emailed to fer··.@ep··x.net. Thank you.
```
