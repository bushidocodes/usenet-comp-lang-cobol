There is no outstanding IF statement at this point - they were
both terminated by the period. If you're using ANSI 85 COBOL,

the code. Always remember that the compiler will take no notice
of your indentation to guess your intentions.

Unfortunately, the compiler looks only at the code you write, not how you
format that code on the page. Every programer can use practice in
looking at his code through the compiler's eyes. So follow along for a
moment, and try to "see" what the compiler "sees":

IF N-BANA = 1
THEN ADD 1 TO N-SCHEMA1-RAD-NR
MOVE A-KLOCKA TO A-SCHEMA1-TID
MOVE A-ART-ID TO A-SCHEMA1-ART-ID
MOVE A-LOGO TO A-SCHEMA1-LOGO
MOVE N-ORDER-ID TO N-SCHEMA1-ORDER-ID
IF N-ANTAL > N-PROD-PER-TIM
THEN MOVE N-PROD-PER-TIM TO N-SCHEMA1-ANTAL
ELSE MOVE N-ANTAL TO N-SCHEMA1-ANTAL
WRITE SCHEMA1-POST
IF A-KLOCKA = '15:00'
THEN ADD 1 TO N-BANA.

PERFORM RAKNA-UPP-KLOCKA
ELSE IF N-BANA = 2 THEN

Oops! Not what you thought you were writing, is it?

You know when you are finished with an IF-statement, but the compiler
can't read your mind. You need to tell the compiler where the end of the
statement is, and the way you do that is with the END-IF phrase:

IF N-BANA = 1
THEN
ADD 1 TO N-SCHEMA1-RAD-NR
MOVE A-KLOCKA TO A-SCHEMA1-TID
MOVE A-ART-ID TO A-SCHEMA1-ART-ID
MOVE A-LOGO TO A-SCHEMA1-LOGO
MOVE N-ORDER-ID TO N-SCHEMA1-ORDER-ID

IF N-ANTAL > N-PROD-PER-TIM
THEN
MOVE N-PROD-PER-TIM TO N-SCHEMA1-ANTAL
ELSE
MOVE N-ANTAL TO N-SCHEMA1-ANTAL
END-IF

WRITE SCHEMA1-POST

* kna upp bana
IF A-KLOCKA = '15:00'
THEN
ADD 1 TO N-BANA
END-IF

* kna upp klocka
PERFORM RAKNA-UPP-KLOCKA

ELSE IF N-BANA = 2
etc.

As an exercise, you might find it useful to write a CoBOL program that
reads another CoBOL program and reports if the indentation does not match
the structure. This is an easy way to find missing END-IFs before they
can do any damage.
