You should probably go to Microfocus.com and go to the student section
and order the Personal COBOL for DOS or Windows (if you have 3.1 or
version A of W95 you can order the Windows version; there are problems
with version B and the Windows version.)

Once you get the diskettes, you need to know that the text display may
work poorly; in certain books you can find descriptions of the COBOL
commands that allow precise placement of text on the screen (at line
x, col y, etc); use this commands.

There are some free compilers being offered just now; they are
actually commercial products being offered free in hopes that they
will catch on. I don't know how good they are. In the COBOL for
Dummies book there are several. For example, the Fujitsu compiler is
in the CD in that book, but it seems incomplete as compared to the
free CD compiler you can order from the Fujitsu website (They will
respond to your query as well). I have not used their product at all,
but I did install it, both the book compiler and the free CD from
Fujitsu.
Also, Casegen is offering a free compiler, I think; I ordered one and
it is on its way.
I would use the Microfocus compiler at first because it has been used
by many students before and you are more likely to find help on it
than the others, possibly.
Good luck.

An exchange took place between someone trying to use a COBOL compiler to
do the Hello program and the difficulties encountered.

The complaint boiled down to the fact that "vendors" often do not
include a step by step outline of how to start using the compiler,
editor, and sample code provided. Yes, help files and documentation is
provided, but it takes a month of sundays to ferret out the details
needed to do the first thing required to use the tools.

I've noted this problem and refrained from commenting on it until now.
I'd like to take this opportunity though to mention that most personal
computer users are familiar with the kind of "help" that comes with
shareware. In those cases, the author/vendor realizes that the new
potential customer is in a hurry to see how the tool works. The tool is
not a hammer, it's more like a swiss knife. These vendors/authors
usually include something that says, "if you are in a hurry to try this,
then do yada yada yada, and be sure to yada, then go." That's a loose
translation. It really reads more like a recipe to make a dish for
dinner.

As the current holder of several compilers for COBOL as a means for
getting up to date, I have not found one that does not assume the user
of the tool knows all about how to make the tool work on the system. May
I remind those of you who are in the position to do something about this
problem that most of us use Win95 or some similar "swiss knife" as an
operation system with out knowing very much about it either. Yet, any
application will tell you how to install it, and how to set up the
system to use it. Vendors/authors need to back up so they can see the
trees.

Pardon my use of all the clichÃ©s.
