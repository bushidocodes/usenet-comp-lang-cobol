It all started at the university of Berlin
([url]http://web.f4.fhtw-berlin.de/ginnold/stuff/sapat/at19/primsch.pdf[/url])
The original adabas is a non-relational database designed to run on
mainframes and VAX-VMS. In combination with the 4GL Natural a great
combination!
When Entire SQL-DB was renamed to adabas-D, the original was referred
to as adabas-C. People thought C for "Classic" but Software AG never
liked that :)

Adabas-D can still be downloaded (evaluation copy) and used *including*
a cobol pre-processor. The only "free" one around as far as I can
judge.

The big endian issue: you can test at execution time on what machine
you are running

Code:
--------------------
         01  Machinetest.
  03 binair-0000-0100-pic-X   pic X(2)  value X'0004'.
  03 which-endian             redefines
  binair-0000-0100-pic-X   pic S9(4) comp-5.
  
  03 B-I-G-endian value +0004 pic S9(4) comp-5.
  03 litle-endian value +1024 pic S9(4) comp-5.
  PROCEDURE DIVISION.
  ********************
  000-MAIN Section.
  000-010.
  evaluate which-endian
  when B-I-G-endian
  display "this is the AIX (powerPC)"
  when litle-endian
  display "this is a PC (intel inside!)"
  when other
  display "how the *beep* do you run this program?"
  end-evaluate.
  
  
--------------------
I've tested this code with Micro Focus NetExpress on NT and
ServerExpress on AiX. Results as expected ;)

Your statement was that 'Software AG' _is_ 'the SAP folks'.

Software AG and SAP are two different companies.  Software AG did
develop Adabas in the '70s which is now Adabas-C.

Adabas-D (aka SAP/DB Max/DB) was developed by University of Berlin in
the '80s and paid for and owned by Nixdorf as DDB/4.  Nixdorf licenced
it to Software-AG, who also added development, where it was named
Adabas-D.  SAP used it and in 1997 bought it from Nixdorf renaming it
SAP/DB.

You are confusing the Adabas of the '70s (which was owned by Software
AG not SAP) with SAP/DB.
