There is no way unless you want to write a routine to mimic it. This
would be fairly trivial. What value would you return for garbage?
My choice (I'm the one who implemented NUMVAL) for bad values for
function parameters was to produce a runtime message that says you
had a bad value and quit. That is because the users I have talked to
want to quit when they have bad data rather than continue. In the
1997 standard there will be a method of choosing whether to quit or
continue.

I would be interested in the opinions some of you might have. It
seems to me that it would be a disaster to continue executing with
garbage data. Nothing using it would make sense. What probably
should have been done was to have an error return as was defined by
the CODASYL COBOL Committee (in this case it was - 999...999 (18 9s)
- which, of course, is a legal value as well). However, the ANSI
committee did not like the idea of error returns.
