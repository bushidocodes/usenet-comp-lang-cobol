Joseph Ostreicher wrote:
> 
>  special-names.
>      crt status is keyboard-status

The CRT status is basically what caused the ACCEPT ... AT statement to
return to the program.  The ACCEPT waits for the user to key into a
field (or fields).  When the user wants the program to do something he
ends the input with the enter key, or esc key or a function key or mouse
click or ...

It may be necessary to set up what can be used to terminate the ACCEPT
using a configuration utility or by the program calling routines.

>      Cursor is cursor-position.

This has two functions, the first is to set the cursor to a particular
place within the fields to be accepted, the second is to know where the
cursor was when the accept terminated.

The cursor is where typing will put characters.

For example I may want to accept 3 fields with:

    01  Field-Group.
        03  FILLER     PIC X(20) VALUE "  Customer Number: [".
        03  In-Cust    PIC X(8).
        03  FILLER     PIC X(52) VALUE "]".
        03  FILLER     PIC X(20) VALUE "    Customer Name: [".
        03  In-Name    PIC X(30).
        03  FILLER     PIC X(30) VALUE "]".
        03  FILLER     PIC X(20) VALUE "          Address: [".
        03  In-Address PIC X(30).
        03  FILLER     PIC X(30) VALUE "]".
    01  Field-Block REDEFINES Field-Group   PIC X(240).

        DISPLAY Field-Block   AT 0101
   *    this will display all 240 characters onto 3 lines at top of
screen     

        MOVE 0221      TO Cursor-Position
        ACCEPT Field-Group    AT 0101
   *    this will only accept the 3 non-FILLER fields with the
   *    cursor starting in the field In-Name
   *    the user can move to any of the 3 fields but not to
   *    anywhere else on the screen

        IF ( cursor-Position > 0300 )
   *        the cursor was in the 3rd field when the used completed
input
            ....

        IF ( Function-Key = 3 )
   *        the user pressed the F3 key to end the input 


>  working-storage section.
>  01  keyboard-status.
…[8 more quoted lines elided]…
> the field, with starting position closest to, but not less then???

The cursor can only be within one of the fields - either a field defined
in SCREEN SECTION or a non-FILLER within a group accept (as above) or a
simple field accept.  If the position is set to be outside the fields
(eg MOVE 1645 TO Cursor-Position) then it will be positioned within the
closest field.
