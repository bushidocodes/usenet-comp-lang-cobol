Current Schedule for ACUCOBOL-85 Training Courses
1995

Course Name August 1995 October 1995 November 1995

Introductory Wed., Thurs. Wed., Thurs Wed, Thurs


Location:
Each of the courses above will be given in the Acucobol
training facility in San Diego.

Hours:
Each course is covered in two days. Class hours are 8:00am to
5:00pm, with a one hour lunch break. One class day consists of
four units. Each unit consists of approximately one hour of
lecture and one hour of lab exercises.

Tuition:
The tuition for each two-day training course is $525.00.
Courses should be booked and paid for in advance. Mastercard
and Visa will be accepted. Although we will accept payment at
the door, the student's spot in the class is not reserved
until payment is received. The maximum number of students per
class is 12. Lab exercises are done in the training facility,
and there will be one lab computer for every two students.

Course Descriptions:

ACUCOBOL-85 Introductory

This course is a BEGINNING course for COBOL programmers who need
to know specifics about ACUCOBOL-85. The student will learn the
basics of how to use the development system, including the
ACUCOBOL debugger. The course will also cover ACUCOBOL
extensions to the ANSI-85 standard, such as the screen section.
Prerequisites: Student should have 6 months or more experience
in COBOL programming.

AcuScreens Introductory

This course is a BEGINNING course for COBOL programmers who
wish to learn to use the AcuScreens product. The student will
learn how to use the AcuScreens Panel Editor/Code Generator to
create a data entry screen, and how to access data once it's
been entered by the user.
Prerequisites: Student should have 6 months or more experience
in COBOL programming.

----------------------------------------------------------------


ACUCOBOL-85 Introductory
Training Course

Each unit consists of approximately 2 hours. One hour of each
unit involves hands-on lab exercises. Class hours are from 8 AM
to 5 PM with an hour for lunch. For every hour of lecture there
will be a 10 minute break.

Prerequisites: Students should have some experience in COBOL
programming. Six months or more COBOL experience recommended).

Day One.

Unit 1. Module Overview
Brief description of ACUCOBOL-85 s components.

Unit 2. The Compiler and Runtime System
Compiler and runtime options. Configuration file
usage.

Unit 3. The Debugger
Debugger modes and commands.

Unit 4. DISPLAY and ACCEPT
Enhancing the look and feel of your applications.
Includes detailed information on Acucobol s Screen
Section and other enhancements to the 85 standard including
pop-up windows, boxes, etc.

Day Two.

Unit 5. Runtime Configuration Variables
What is the runtime configuration file, how to tell the
runtime where to find it, and how to use it to change
the behavior of your application.

Unit 6. Calling subprograms
Calling C routines and special library routines.
Includes information on basic COBOL calling
conventions, and an introduction to the available
routines.

Unit 7. Vision file system
How Vision works. Vision architecture. Avoiding file
corruption.

Unit 8. Portability issues
Differences between operating systems and hardware
platforms. Terminal differences.


-----------------------------------------------------------------


AcuScreens Introductory
Training Course


Prerequisites: Students should have some experience in COBOL
programming. (Six months or more COBOL experience recommended).

Day One.

Unit 1. The Big Picture
Become familiar with the components of ACUCOBOL-85 and
AcuScreens, and learn how the two products interact.

Unit 2. Introduction to the Panel Editor
Learn to navigate the Panel Editor by becoming familiar with
its four main components: Menubar, Toolbar, Field
Characteristics Ribbon, and the Format Window.

Unit 3. Mastering the Panel Editor
Learn to set field and panel characteristics using the Panel
Editor. Become familiar with the many dialog boxes used in
the Panel Editor.

Unit 4 Creating menus
Use the Panel Editor to create a menu bar for a sample
application. Use the Code Generator to generate a COBOL
program. Add necessary code to the generated program to
process the menu.

Day Two.

Unit 5. Creating panels
Use the Panel Editor to create a data entry screen (panel)
which makes use of many GUI controls, such as listboxes and
radio buttons.

Unit 6. Processing data from an AcuScreens panel
Add necessary code to a program to process data that is
retrieved from an AcuScreens panel.

Unit 7. Adding bells and whistles
Add secondary panels that "pop up" over existing panels.
Add code to change colors dynamically from COBOL..

Unit 8. Introduction to advanced AcuScreens programming
Learn basic concepts involving AcuScreens functions and data
structures.

For additional information contact Acucobol at 1-800-COBOL85 or
619-689-7220.
