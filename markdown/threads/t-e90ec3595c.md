Started to answer this - then thought I'd gone nuts ! Your text keeps saying
Micro Focus - whereas the title mentions Fujitsu V 4 <G>

Unfortunately there's no quick answer other than 'suck it and see'. Let's
assume you are asking about the difference between Fujitsu V4 and Net Express
which is the LATEST Micro Focus. (There are two here who have both F/J and N/E
- Judson and Thane - perhaps they can expand).

If you restrict yourself to the STANDARD '74 or '85 features - don't use
vendor extensions - it's a fair bet it will compile cleanly, either direction.
(Just recently I took a Fujitsu V ????? Structured program written by Thane
and compiled it cleanly in Net Express).

Now if you move up to what we refer to as the '97 Standard, (includes
Intrinsic functions etc.), that's a slightly grey area, but really boils down
to the EXTENSIONS the respective vendors have introduced. Both allow for
CBL_FILE routines - but I'm guessing they probably have different syntax and
possibly return different results (?). Similarly both as an extension have
Screen Section - I know Merant has CBL_screenAttributes and CBL_moveMouse
routines; I don't know if F/J has equivalents - If the answer is 'Yes' - it's
a fair bet that they are different

Both allow for using comp-5 - though I think they are different. Merant(Micro
Focus), as an extension allows you to code :-

78 ThisIsMyConstant                   value 22.
78 MySecondConstant                value 'Hello World'.

01 MyRecord                                pic x(ThisIsMyConstant).

Note : the examples above are replaced by the NEW syntax CONSTANT in the draft
of COBOL 200X.

The above is just a 'sampling' which immediately springs to mind.

Chances are, using an 'up to date' coded program in one or the other - you are
going to get an hiccup when you try to convert.
