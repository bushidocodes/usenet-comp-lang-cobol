Hello All,
I'm a mainframe Cobol programmer (Unisys A-Series) with little to no
knowledge/experience of Windows programming or other PC-based
languages such as VB, C++, etc. I have an application that my company
wants to write in MF Cobol 4.0 that will gather information from the
user into a file on the PC then transmit that information, record by
record, to our mainframe via a ASCII TTY connection through our
front-end processor. We currently use Procomm, Dynacomm, and other
communication packages to do this (kind of like a BBS system that
prompts you for input) but we want a PC-based application to batch up
the information then transmit the accumulated inquiries. After each
record is transmitted, the host will send back an ID# that was
assigned to the inquiry. A file/database on the PC will use this for
tracking/reporting purposes.

My big problem is how to communicate with the modem. How do you go
about accessing the modem from within Cobol? Should I write API calls
and if so where do you learn to do this? Any reference books on
accessing the Windows API from within COBOL? From what I can find out
from others who know VB or C++, this is a no-brainer in those
languages. Are there 3rd party tools/add-in modules for Cobol?

Any and ALL information would be GREATLY appreciated!

You wrote (among many other things I snipped):

› My big problem is how to communicate with the modem. How do you go
› about accessing the modem from within Cobol?

Two possibilities:1) CALL functionality from dedicated software. See for example
contribution of Fujitsu Software Corporation Developer Tools Group in this
thread.
2) CALL windows functions yourself.

› Should I write API calls and if so where do you learn to do this?

Under option 2: yes. You have to learn:- the general structure of windows
internally. Get some books or a course in a Microsoft training shop.
- the general structure of the windows API. Same advice.
- the specific API-calls needed and theid interface. A lot of books available.

› Any reference books on
› accessing the Windows API from within COBOL?

Each copmpany that is a vendor of a COBOL flavor in which you can CALL windows
API, states how to do so in the docs/manuals.

For example: options in environment division like
CALL CONVENTION number name
where number is your choice and name the name you choose (WINAPI widely used).

The number says the compiler/runtime how to pass parameters (order, stacking,
who allocs/who cleans etc).

The calls themselves are like "CALL WINAPI "APIFunctionThisOrThat" USING etc.
and are documented in the books mentioned earlier. Microsofts sells a so-called
Software development Kit (SDK). See Thane Hubbell's contribution. The SDK is
sometimes included with other software, for example Micro Focus includes the
32-bit version with NetExpress. BTW, the SDK contains a complete reference of
the API
