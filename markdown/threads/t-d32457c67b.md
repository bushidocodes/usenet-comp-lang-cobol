Mr. Moore:

We have a close working relationship with England Technical Services.
I do know that Bob England just completed a prelimiary VAX VMS port of
the COBOL RPC product embedded in our COBOL sp2 Thin Client.

You should probably contact Bob England directly to determine when he
plans to release a production version of COBOL RPC for the VAX. The PC
support has been available for both 16 bit and 32 bit Windows
platforms for a long time, so that should be no problem at all.  I
don't believe that he supports OS/400 yet, but perhaps COBOL RPC can
also be ported to that operating system as well.

COBOL RPC essentially allows a COBOL program on machine A call a COBOL
program on machine B and pass the associated paramter block.  This is
accomplished through the TCP/IP communications protocol, so it can
easily work across the Internet.

Bob's product is actually quite easy to use because the programmer
doesn't really have to know a lot about TCP/IP and all programming is
done in good ol' COBOL.  Both machines do have to support a TCP/IP
stack.
