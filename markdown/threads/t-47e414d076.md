Any compiler that does NOT give an error (or at least an ANS85 extension
flag) for this syntax is in ERROR. (In Micro Focus, for example, if you use
the FLAG(ANS85) directive, this is accepted - but provides the correct
message indicating it is an extension).

According to the ANS85 Standard a numeric (or integer) function may ONLY be
used where an arithmetic expression may be used. (A fairly small list,
IF-statement, SEARCH-statement, reference modification, compute - and maybe
one or more places). Several implementations do allow such functions (AS AN
EXTENSION) to be used wherever a numeric (integer) item may be used as a
sending item (for example a MOVE statement) and, in fact, the next Standard
will also expand the language to allow this.
