Well, I believe it was assembler. We just called a small assembler
routine to invoke the APIs as they were all interrupt driven.

We used Realia, and could have probably done it all with MLIs.
All we really use it for it to find out if we are on Novell, and
if so, what station number we are. We use this to create unique
file names etc. To do any serious netware interaction, I would recommend
you execute the Novell utility (NPRINT, etc) because the low-level
APIs have changed at least once on us forcing a re-write of the
assembler.

The developers kit was expensive then, probably moreso now. But if
you find anyone with one, they can write you a little routine to
do this without violating anyones copyright because there is no
vender code involved.

All you have to do is call the assembler from your cobol.
