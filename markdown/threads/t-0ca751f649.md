Hans,

Are you sure that you are going to score a higher grade if you add
so-called, 'cool things', to your program ?

My understanding of university assignments is that certain tasks are
specified, and specific marking criteria are applied to each of those tasks,
as well as to the project overall. I'm not sure though, that bonus marks are
normally awarded for simply adding extra facilities that were not specified
in the first place, and, in fact, you might even be penalised by having
added something that was not originally asked for, since this means you did
not adhere to the assigned specification !

Might it not be possible to gain a higher grade if you took greater care to:

* Ensure your program(s) is clearly, and comprehensively documented,
   both by comments within the code, and supporting external
   documentation

* Ensure that the 'quality' aspect of your code is high. For example:

  - Adhere strictly to any coding standards that have been set
  - Use consistent naming conventions for variables, and
    procedures
  - Use extensive error checking / reporting. For example:
    * ON SIZE ERROR checks for arithmetic operations
    * ON EXCEPTION checks when calling subroutines
    * FILE STATUS checks in file I/O
    * Checking for missing / incomplete / corrupted data

* Present a comprehensive 'test plan' outlining things such as:

  - Test data used
  - Expected and actual output results obtained
  - White and black box tests performed

In short, if you can present a project that does exactly what was asked,
does it very, very, well, and that it possesses qualities that make it easy
to understand and modify, you are, surely, bound to score a perfect grade !

I know that I did not mention any 'cool things', but I hope that at least a
couple of these suggestions help.
