You can initialize any level having a name.

initialize CIF-Name in CIF-Temp2 (m, n)

Remember that initialize does not clear FILLERs (unnamed areas), indexes nor pointers. Nor
does it use your VALUEs. Both were fixed in the 2002 standard, but it is generally not
available.   I prefer move low-values, because it works for binary numbers and pointers as
well as pic x. The most common technique is 

move spaces to CIF-Data in CIF-Temp2 (m)
initialize CIF-Data in CIF-Temp2 (m) replacing numeric by zero

For highest speed, create an extra structure, initialize it once, move the whole thing or
the part you want to initialize. That forces you to use qualification on references to 
the base structure, which might otherwise not require qualification. Maximum speed is not
justified on every initialization, or even most. I don't recommend this technique unless
you are processing very high transaction volume. 

As a matter of style, initialization should be done INITIALLY i.e. at the beginning of a
process. You shouldn't be initializing parts of the structure in the middle of a process,
and definitely not at the end. Since you are initializing with procedural code, there
should not be a need for VALUEs in the data division except for constants that never
change. Constants can usually be handled better with 78 level. 

Indicators, one byte true/false fields with 88s, should be initialized by setting the
condition to false (Micro Focus) or by moving low-values to a group name. They should not
have data names. Setting and testing should both be done with 88s. It is bad style to
sometimes use a name and sometimes an 88.
