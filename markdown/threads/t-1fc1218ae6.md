Good for you, Stanley. If I were you I would get a good book on COBOL,
perhaps an introductory one if you think you need that much coaching. And
also get a learning version of a COBOL compiler (or a professional version,
if you want to spend that much). There are several good possibilities
around. Then I would get in there and reacquaint myself with the language.
Once that was underway, I would start looking for a job. Pretty soon they're
going to be hiring anybody that can spell COBOL for Y2k.

It will take you a little while to get up to speed after more than ten years.
But COBOL programming hasn't changed a lot in that time. The main thing you
will likely notice is the programming IDEs and debugging tools that are
available now. They're nice.
