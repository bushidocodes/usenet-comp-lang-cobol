I'm doing some development with Fujitsu V4 (got it from one
of the books on a CD).

What I have done is written at least one subroutine and one
main program. They compile correctly when done individually.
However, when the BUILD is done (which ultimately brings in
the LINKER -- or is that clinker?), the link fails
complaining about there not being a valid entry.

Now this is rather interesting, and the manual says that you
must change the parms so that the "main" module is marked
MAIN. The problem is, this causes the subroutine to now get
compiled with MAIN! (what a way to run an army!) The build
function seems to want to use the same options/parms for ALL
the "cobol" parts!?!

The whole reason that I'm doing all of this is so that I can
develope and test on an NT system and then move the code to
a different platform (and all we have to do is change the
common subroutine(s) to use the I/O of the intended platform
and then recompile and link). So yes, I want static linkage,
not dynamic.

Anyone have any ideas as to what the problem is?
