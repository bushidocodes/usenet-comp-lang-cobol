I've been playing with the word processing challenge thing - I've given 
it to a guy I'm supervising as well, to help him learn string 
manipulation and the like.  Anyway, I've been consistently getting the 
wrong counts on both occasions (fewer total words, and more dupes) - I 
swiped WMK's conversion routine, and the counts were perfect the first time.

Here's what I swiped, that worked...

=-=-=-= This works!

     move function lower-case (kjvRec) to wsRec
     inspect wsRec replacing all "'s" by "s "
     inspect wsRec converting "."",;?:)(!-'0123456789"
                            to "                     "
     move wsRec to wsCompressedRec

=-=-=-=

Now, what I tried to do.  I have two tables, both 80 characters.  I'm 
trying to move the data, a character at a time, to a line that will have 
the words lower-case, the first word starting in column 1, and each word 
separated by a single space.  Here's the logic...

=-=-=-=-= This doesn't work!

     *>  Input record breakout area
     01  wsRec  value spaces.
         09  wsChar      occurs 80 times
                           indexed by wsIdx
                              pic X(01).

     *>  Input record minus all the extra spaces & special chars
     01  wsCompressedRec value spaces.
         09  wsCompChar  occurs 80 times
                           indexed by wsCompIdx
                              pic X(01).

...

     move kjvRec    to wsRec
     set  wsCompIdx to 1
     move spaces    to wsCompressedRec

     perform varying wsIdx from 1 by 1 until wsIdx > 80
         evaluate wsChar (wsIdx)
           when "A" thru "Z"
*> This makes them lower-case
             move function lower-case (wsChar (wsIdx)) to wsCompChar 
(wsCompIdx)
             set  wsCompIdx                            up by 1
             set  bMovedAChar                          to true
           when "a" thru "z"
*> This character's fine
             move wsChar (wsIdx) to wsCompChar (wsCompIdx)
             set  wsCompIdx      up by 1
             set  bMovedAChar    to true
           when "'"
*> Skip it if it's in a word, otherwise move a space
             if wsChar (wsIdx + 1) > space
                 continue
             else
                 move space        to wsCompChar (wsCompIdx)
                 set  bMovedASpace to true
             end-if
           when other
*> Masks off everything else, but doesn't move spaces
*> until the first word has been found
             if   wsCompIdx = 1
               or bMovedASpace  then
                 continue
             else
                 move space         to wsCompChar (wsCompIdx)
                 set  wsCompIdx    up by 1
                 set  bMovedASpace to true
             end-if
         end-evaluate
     end-perform

=-=-=-=-=

Various random displays I've put in the program appear to be working 
correctly, yet my totals are consistently off (but the same) on two very 
different architectures.  What am I missing?  I thought that my way 
would be good, because it's not limited to a finitie list of symbols to 
disallow - instead, it takes the letters it wants, and leaves the rest 
alone.

As an aside - I mentioned that the SEARCH on a single table was NOT the 
way to do things...  My version now uses a 2-D table (26 x 5000), and 
although it's not sub-second times, it's down from over a minute to 
about 9 seconds (on a 750 MHz laptop).  Once I resolve this, I'll post 
my solution.

Thanks...

   Well, probably I have only misunderstood, and these are two level- 
88 condition-names for the same variable, as in my version, which  
finally got the parsing right:


Verarbeiten-Eingabesatz SECTION.
 Anfang.
D    DISPLAY laenge-Eingabezeile ">" zeichen-Eingabe(laenge-Eingabezeile) "> " text-line UPON Konsole
     MOVE ZERO TO Laenge OF einzelnes-Wort
     MOVE SPACE TO FIX OF einzelnes-Wort
     SET ist-nicht-im-Wort TO TRUE
     MOVE 1 TO Pointer-Char-in-Line
     PERFORM UNTIL Pointer-Char-in-Line > laenge-Eingabezeile
       EVALUATE zeichen-Eingabe (Pointer-Char-in-Line)
         WHEN "A" THRU "Z"           *>  ALPHABETIC
         WHEN "a" THRU "z"
           ADD 1 TO Laenge OF einzelnes-Wort
           MOVE FUNCTION LOWER-CASE(zeichen-Eingabe(Pointer-Char-in-Line))
                TO FIX OF Einzelnes-Wort(Laenge OF einzelnes-Wort:1)
           IF ist-nicht-im-Wort
           THEN
             SET ist-im-Wort TO TRUE
           END-IF
         WHEN "'"
           CONTINUE
         WHEN OTHER
           IF ist-im-Wort
           THEN
             PERFORM verarbeiten-gefundenes-Wort
             MOVE ZERO TO Laenge OF einzelnes-Wort
             MOVE SPACE TO FIX OF einzelnes-Wort
             SET ist-nicht-im-Wort TO TRUE
           END-IF
       END-EVALUATE
       ADD 1 TO Pointer-Char-in-Line
     END-PERFORM  *> finding individual words in one line
     IF ist-im-Wort THEN
       PERFORM verarbeiten-gefundenes-Wort
     END-IF
     .


   Some translation hints:

   German                      English
   ------------------          ------------------
   Eingabe                     input
   Eingabesatz                 input record
   Eingabezeile                input line
   einzeln                     single
   einzelnes-Wort              single word
   gefunden                    found
   gefundenes-wort             word found
   ist-im-wort                 is in word
   ist-nicht-im-wort           is not in word
   Laenge                      length
   Verarbeiten                 to process
   wort                        word
   zeichen                     character (originally: sign)


  So, I dropped the UNSTRING and went thru the input line character by  
character, but not to reproduce the line again with just spaces as  
delimiters, but to extract single words right away and process them  
word by word during parsing of the input line.
