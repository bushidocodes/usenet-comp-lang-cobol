You can download a free tutorial on COBOL data types from
www.flexus.com/softwaredownload.html

Get file COBDATA.ZIP, explode (er, un-zip) and read the text and/or play
with the graphics.

Then, if you still need to convert the numeric data from 'COBOL  datatypes'
to 'IEEE datatypes', I have a tutorial on that, as well as a Windows 32-bit
DLL to handle it on the PC side. On a mainframe, you can actually unpack and
format the data using something like SyncSort, provided you get all the
field sizes and types correct.

Cobol data file is a sequence of records, possibly compressed,
possibly indexed. They are not supposed to be readable by
a human. Only Cobol programs are suposed to read them.

We developed a series of converters that can read Cobol data files
and convert them to text or DBF format, or even make them readable
to Crystal Reports.

All you need to have is your Cobol data file and at least one 
Cobol program in source that can read or write this file.
Then our converters would be able to read this file.

More on this at http://www.siber.com/sct/datafile/
