MF Cobol has the +B switch which changes the behaviour to how you want
to work.  With the +B run-time switch OFF (as it is by default) then
after a READ NEXT has found a locked record the following READ NEXT on a
file opened in I-O mode will re-read the same record again (as you have
found).

If the +B switch is ON then the READ NEXT will get the actual next
record and not re-read the locked record.

Both behaviours are useful in different circumstances.

This +B switch can be set on the COBSW environment variable, on the
program run command or programatically.

Alternately use the NORETRYLOCK compiler directive.

If the file is open in INPUT mode the lock won't be detected nor will it
re-read the locked record.  If user 2 is not updating then you could
close and re-open as appropriate so that it is in INPUT mode when
reading through alternate key.
