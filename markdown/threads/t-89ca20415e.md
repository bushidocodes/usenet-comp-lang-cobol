support for the Fujitsu Compiler version 3.0.  The same code works fine in a
normal strucuted COBOL program (Thanks Thane) but doesn't work in an Object
COBOL class library.  The error that is returned is VERY strange indeed.

      *

       Procedure Division.

       Data Division.

       01 sampleText.
          05 filler   pic x(06) value 'Hello'.

           display sampleText at line 01 column 01.
      *

What the message is saying (as I read it) is that you may not use an
"extension" display statement in a method (but can use it in a program).
If you remove

   "at line 01 column 01"

(so you have an ANSI conforming DISPLAY statement, I'll bet your program
will compile and run OK).

FYI, there was quite a bit of discussion in the ANSI and ISO committees
about whether methods should be allowed to have

    - screen section
    - report writer
    - communication section

I don't remember exactly where it ended up in the draft Standard, but it
looks like Fujitsu is (currently) not allowing the enhanced Accept/Display
in methods.

I don't think you can do a display statement in the object cobol support
other than one without a line or column specification.  You can not even
define a screen section without getting the following error.  (I tried it
with and without a line and column number).

I have not received a respone from Fujitsu.  I am becoming very discouraged
with the object cobol support included with Fujitsu.  I can not speak to the
standard cobol support or even the GUI tools. Here are some VERY big holes I
see in the Fujitsu support:

Limited classes supplied with the compiler
No base support for collection classes
No support for call backs
You can not even define procedure pointers so you could write your own call
backs

If you are going to be doing OO COBOL I would strongly suggest the
NetExpress product or looking at the IBM product.
