"Russell Styles" <RWSTYLES@worldnet.att.net> wrote:
> The Net Express editor does have column block abilities.  If this is
> a bottom of the rung editor, than a good one (Multi-edit?) must be

You're right, Net Express does have has an option to switch between line
and column blocks.  But you can't select streams of text at all, which is
extremely useful (I'll give examples if anyone wants).  And you must use
the mouse to use both on the fly, and using the mouse for editing functions
in a programmer's editor makes little sense.  Unless you're a hunt-and-peck
typist, taking your hands off the keyboard to do any editing function is far
slower than using the keyboard.  You can type dozens of keystrokes in the
time it takes to move your hand to a mouse, move the mouse, click, drag or
select, then move your hand back to the keyboard.  It is intuitively obvious
and easily verified, and is one reason why GUI interfaces don't make sense
for some applications.

Believe me, compared to a top flight programmer's editor (Multi-Edit is
only one of many), any IDE editor I have ever seen is like using crayons,
by comparison; no hooey.  Any programmer's editor worth the name will
have regular expression search and replace, block, column and line mark,
copy and move, convert case to upper/lower, file compare, keyboard
macros, a macro language, full keyboard remap, and a vast array of other
functions.  The last hardcopy manual printed for Multi-Edit is two major
releases ago, and it is over 1" thick.  The macro language in Brief was so
powerful there were games written using it.  Brief set the standard for
DOS based programmer's editors.  Unfortunately, Borland bought Brief to
get the technology, then let it die on the vine more than 10 years ago.  Even
now, Brief is better than many if not most of the current editors, and almost
all good programmer's editors still offer a Brief keyboard emulation.

By 'bottom rung' I mean that the NE IDE is poor in text editing functions
when compared with other IDEs, such as Visual Studio (which has such
features as regular expressions and keyboard emulation for a few big
name editors, such as Brief) or even old DOS IDEs such as QuickBasic
and the Borland products.  The Net Express IDE's debugging features and
integration with the compiler are okay.
