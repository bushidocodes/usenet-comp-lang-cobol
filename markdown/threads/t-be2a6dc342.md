VALUE OF file-attribute was part of COBOL-74. IBM always accepted it as
comments, but Unisys and others used it for all sorts of purposes. In
IBM, the only file attribute inside the program is the DDNAME (which is
specified on the SELECT); everything else is specified in JCL. On
Unisys, you can specify everything inside the program, eg, "VALUE OF
AREASIZE IS 5000". You don't any DD statements (we call them file
equates), but they're available when needed.

Because IBM never used it, VALUE OF was a deprecated feature in
Cobol-81. I believe it was called "transitional" which was later called
"obsolescent", but I'm not going to say what it's status was in
X3.23-1985 until I actually have a copy.

Cobol-81 had an interesting feature: It was observed that the division
of clauses between the SELECT (ENVIRONMENT DIVISION) and FD (DATA
DIVISION) was not following the theoretical model that the SELECT looks
outward and the FD looks inward. Therefore VALUE OF and BLOCKSIZE were
moved to the SELECT, and some other clauses were moved to the FD.
Though I praised the reallignment in my comments, it was widely derided
and was removed from the pink draft.
